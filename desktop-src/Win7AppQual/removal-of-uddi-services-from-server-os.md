# <a name="removal-of-uddi-services-from-server-operating-system"></a>Удаление служб UDDI с серверной операционной системы

## <a name="affected-platform"></a>Затронутая платформа

**серверы** — Windows Server 2008 R2  



## <a name="feature-impact"></a>Воздействие на функции

**Серьезность** — средний  
**Частота** -низкая  

## <a name="description"></a>Описание

корпорация майкрософт удалила роль сервера служб UDDI (универсальное описание, обнаружение и интеграция) из Windows server 2008 R2. Будущие выпуски служб UDDI будут входить в состав продукта Microsoft BizTalk. При повторном выравнивании и консолидации продукта Корпорация Майкрософт будет лучше обслуживать на рынке архитектуры, ориентированной на службы (SOA).

первый выпуск служб uddi, выпущенный Windows Server 2008, будет соответствовать стандартам uddi v 3.0. он будет поставляться как часть выпуска BizTalk Server 2009. чтобы обеспечить удовлетворительное взаимодействие с пользователем, мы предлагаем пакет установки служб UDDI v3 для Windows Server 2008 R2.

## <a name="manifestation"></a>Проявление

наличие предыдущих версий компонентов служб UDDI в системе блокирует обновление до Windows Server 2008 R2.

## <a name="mitigation-of-impact"></a>Снижение влияния

пользователи, которые развернули сайт служб uddi с Windows server 2003 или Windows server 2008, могут отказаться от обновления серверов, на которых работают службы uddi, до Windows Server 2008 R2. они также могут перемещать службы uddi на выделенные поля Windows server 2003 или Windows server 2008, если необходимо обновить текущие поля служб UDDI.

## <a name="solution"></a>Решение

рекомендуемое решение — развернуть службы UDDI версии 3.0 с BizTalk Server 2009 на компьютере Windows Server 2008 R2, а затем перенести старый сайт на новый сайт. Службы UDDI версии 3.0 обеспечивают обратную совместимость со службами UDDI 2,0, поэтому все приложения, использующие службы UDDI, не затрагиваются.

Для этого выполните следующие действия:

1.  настройте новый сайт служб UDDI версии 3.0 на компьютере, который уже загружен с Windows Server 2008 R2. (примечание. в распределенной настройке сайт UDDI версии 3.0 может состоять из нескольких компьютеров Windows Server 2008 R2.)
2.  Используйте средство переноса данных UDDI для переноса данных из старой базы данных служб UDDI в новую базу данных.
3.  Создайте резервную копию старой базы данных служб UDDI, чтобы обеспечить возможность отката.
4.  удалите старое программное обеспечение служб UDDI, а затем обновите эти поля до Windows Server 2008 R2.
