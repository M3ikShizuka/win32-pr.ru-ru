# <a name="vss-backup-and-restore-of-the-active-directory"></a>Резервное копирование и восстановление VSS для Active Directory

Модуль записи Active Directory не требует специальных действий во время операций резервного копирования. Модуль записи предоставит инициатору запроса сведения о компоненте и наборе файлов, а инициатор запроса использует эти сведения, чтобы решить, какие файлы следует копировать на носитель резервной копии. Для резервного копирования Active Directory нет необходимости использовать специальные API-интерфейсы.

Способ выполнения восстановления зависит от того, восстанавливается ли Active Directory в ходе аварийного восстановления или если восстановление выполняется в систему, в которой выполняется Active Directory. Кроме того, возраст резервной копии состояния Active Directory может быть проблемой из-за Active Directory захоронений.

## <a name="active-directory-restoration-following-disaster-recovery"></a>Восстановление Active Directory после аварийного восстановления

После сбоя, нуждающегося в аварийном восстановлении, Active Directory можно восстановить в ходе восстановления состояния операционной системы.

Эта операция восстановления, по сути, является восстановлением, не предназначенным для записи.

## <a name="active-directory-restoration-on-the-system-where-it-is-running"></a>Active Directory восстановление в системе, где она работает

Если в данный момент на сервере выполняется Active Directory, система должна быть перезагружена в режиме восстановления служб каталогов.

После этого операционная система будет работать без Active Directory, и проверка всех пользователей выполняется через диспетчер учетных записей безопасности (SAM) в реестре. Только администратор имеет разрешение на восстановление Active Directory.

В режиме восстановления службы каталогов восстановление VSS может выполняться обычным образом. Нет причин использовать интерфейсы API Win32 Active Directory, не связанные с VSS, для восстановления состояния Active Directory.

## <a name="active-directory-restores-and-active-directory-tombstones"></a>Active Directory восстановления и Active Directory захоронений

Любой план восстановления должен гарантировать, что возраст резервной копии не должен превышать время жизни захоронения Active Directory (по умолчанию — 60 дней).

Восстановление резервной копии старше, чем захоронение, приведет к тому, что контроллер домена будет иметь объекты, которые не реплицируются на другие серверы.

Эти объекты, которые не реплицируются, не будут автоматически удалены на этом (восстановленном) контроллере домена, так как объекты-захоронения этих объектов на других репликах уже были удалены.

Администратору придется вручную удалить каждый из объектов на восстановленном контроллере домена, который не реплицируется. Добавочное резервное копирование Active Directory не поддерживается; требуется полная резервная копия.
