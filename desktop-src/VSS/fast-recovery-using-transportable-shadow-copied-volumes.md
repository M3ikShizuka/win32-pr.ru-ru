# <a name="fast-recovery-using-transportable-shadow-copied-volumes"></a>Быстрое восстановление с помощью переносимых теневых скопированных томов

Для *быстрого восстановления* можно использовать [*переносимые теневые копии*](vssgloss-t.md) .

Быстрое восстановление можно использовать для быстрого возврата к более ранней теневой копии. Ниже приведены необходимые действия.

1.  Создайте транспортную теневую копию соответствующих LUN. Теневая копия может быть либо [*постоянной*](vssgloss-p.md) , либо неустойчивой.
2.  Удаление исходных LUN
    1.  Если компьютер находится в кластере, пометьте затронутые дисковые ресурсы как автономные или включите [режим обслуживания](/previous-versions/windows/desktop/mscs/maintaining-physical-disk-resources) для этих дисковых ресурсов.
    2.  Маскирование затронутых LUN с этого компьютера (и всех остальных узлов, если компьютер находится в кластере).
3.  Импортируйте теневую копию с помощью метода [**ивссбаккупкомпонентс:: импортснапшотс**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-importsnapshots) .
4.  Разбейте теневую копию с помощью метода [**ивссбаккупкомпонентс:: бреакснапшотсет**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-breaksnapshotset) .
5.  Пометьте новые тома теневых копий как доступные для чтения и записи.
6.  Если компьютер находится в кластере, предоставьте новые номера LUN теневого копирования в качестве исходных.
    1.  Снимите маску LUN теневой копии на другие узлы в кластере.
    2.  Пометьте ресурсы, которые ранее были помечены как подключенные к сети, или отключите режим обслуживания для этих дисковых ресурсов.

> [!Note]  
> Перепереносимые теневые копии в кластере не поддерживаются до Windows Server 2003 с пакетом обновления 1 (SP1). Эта поддержка поддерживается только с соответствующими номерами LUN, у которых есть по крайней мере одна страница данных продукта (VPD) 0x83 с \_ идентификаторами хранилища 1, 2 или 8 и сопоставление 0, а LUN должны поддерживать базовый диск с разделением MBR.
