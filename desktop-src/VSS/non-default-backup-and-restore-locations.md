# <a name="non-default-backup-and-restore-locations"></a>Расположения резервного копирования и восстановления не по умолчанию

Операции резервного копирования и восстановления обычно копируют файлы из заданного расположения по умолчанию на диск для резервного носителя, а затем восстанавливаются с этого носителя в то же расположение по умолчанию на диске.

Существует множество причин, особенно при работе с запущенными процессами, а не для выполнения этой простой модели. VSS поддерживает использование нестандартных источников для резервного копирования и альтернативных целевых объектов для восстановления и включает механизмы для работы с подмножествами файлов и сопоставления файлов.

-   [Работа с альтернативными расположениями во время восстановления](working-with-alternate-locations-during-restore.md)
-   [Работа с новыми целевыми объектами во время восстановления](working-with-new-targets-during-restore.md)
-   [Работа с частичными файлами](working-with-partial-files.md)
-   [Работа с направленными целями](working-with-directed-targets.md)
