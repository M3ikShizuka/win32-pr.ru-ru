# <a name="working-with-the-backup-components-document"></a>Работа с документом компонентов резервного копирования

Инициатор запроса создает документ компонентов резервного копирования в начале выполнения резервного копирования. Изначально документ содержит только описание состояния операции резервного копирования. Во время восстановления документ должен предоставить инструкции о том, как инициатор запроса должен продолжать копирование файлов обратно на диск. В ходе операции восстановления документ компонентов резервного копирования изменяется и содержит состояние этой операции.

В отличие от документа метаданных модуля записи, который доступен только для чтения, существует окно, в котором документ компонентов резервного копирования может быть изменен как инициаторами запроса, так и модулями записи. Документ можно обновить до создания события [*баккупкомплете*](vssgloss-b.md) или [*баккупшутдовн*](vssgloss-b.md) во время операций резервного копирования и до события, выполняемого при [*восстановлении*](vssgloss-p.md) .

Для использования документации по компонентам резервного копирования инициатора запроса необходимо понимание следующих разделов:

-   [Жизненный цикл документа компонентов резервного копирования](backup-components-document-life-cycle.md)
-   [Содержимое документа компонентов резервного копирования](backup-components-document-contents.md)
