# <a name="vss-restore-configurations"></a>Конфигурации восстановления VSS

Восстановление файлов в работающей системе может быть проблематичным. Важно, чтобы работающие приложения (модули записи) указывали, что делать при возникновении трудностей во время восстановления, например, если восстанавливаемый файл используется в данный момент.

В VSS модули записи имеют два дополнительных способа управления восстановлением —[*методы восстановления*](vssgloss-r.md) и [*целевые объекты восстановления*](vssgloss-r.md).

Кроме того, инициаторы запроса могут восстанавливать файлы в ранее неуказанных расположениях и уведомлять модули записи (см. [**ивссбаккупкомпонентс:: аддневтаржет**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-addnewtarget)).

Метод Restore (также называемый исходным целевым объектом восстановления) задается модулем записи во время резервного копирования и задает определение метода в масштабе записи, которое будет использоваться для восстановления всех его компонентов в будущем. Все файлы и компоненты, управляемые модулем записи, используют один и тот же метод Restore.

Целевые объекты восстановления позволяют модулям записи изменять способ восстановления конкретных компонентов во время восстановления. В отличие от методов восстановления, целевые объекты восстановления определены для набора компонентов.

Подробное обсуждение использования методов восстановления и целевых объектов восстановления см. в разделах, перечисленных ниже.

-   [Состояние восстановления VSS](vss-restore-state.md)
-   [Настройка методов восстановления VSS](setting-vss-restore-methods.md)
-   [Настройка целевых объектов восстановления VSS](setting-vss-restore-targets.md)
-   [Настройка параметров восстановления VSS](setting-vss-restore-options.md)

(Сведения о восстановлении, которые не используют эти механизмы, см. в разделе [восстановление без участия в записи](restores-without-writer-participation.md).)
