# <a name="s-volume-shadow-copy-service"></a>S (служба теневого копирования томов)

<dl> <dt>

<span id="base.vssgloss_selectability_for_backup"></span><span id="BASE.VSSGLOSS_SELECTABILITY_FOR_BACKUP"></span>**выбор для резервного копирования**
</dt> <dd>

Компонент считается выбираемым для резервного копирования, если его явное включение в операцию резервного копирования является необязательным. Возможность выбора для резервного копирования включена, если значение элемента **бселектабле** структуры [**\_ компонентинфо VSS**](/windows/desktop/api/VsBackup/ns-vsbackup-vss_componentinfo) равно **true**. Значение по умолчанию для выбора компонента для резервного копирования отсутствует. средство записи должно всегда задавать это значение явным образом.

Модули записи также используют возможность выбора для восстановления, чтобы упорядочить участие компонента в операциях восстановления.

См. также *выбираемый компонент*, возможность *выбора для восстановления*, [*явное включение компонентов*](vssgloss-e.md), [*Включение неявных компонентов*](vssgloss-i.md).

</dd> <dt>

<span id="base.vssgloss_selectability_for_restore"></span><span id="BASE.VSSGLOSS_SELECTABILITY_FOR_RESTORE"></span>**выбор для восстановления**
</dt> <dd>

Считается, что компонент является выбираемым для восстановления, если он может быть неявно резервным копированием в составе набора компонентов, а затем по отдельности восстановлен без остальной части набора компонентов. Возможность выбора для восстановления включена, если значение элемента **бселектаблефорресторе** структуры [**\_ компонентинфо VSS**](/windows/desktop/api/VsBackup/ns-vsbackup-vss_componentinfo) равно **true**. По умолчанию выбор компонента для восстановления имеет **значение false**. Выбор способа восстановления имеет значение только в том случае, если компонент не был добавлен в документ резервного копирования.

См. также *выбираемый компонент*, возможность *выбора для резервного копирования*, [*явное включение компонентов*](vssgloss-e.md), [*Включение неявных компонентов*](vssgloss-i.md).

</dd> <dt>

<span id="base.vssgloss_selectable_component"></span><span id="BASE.VSSGLOSS_SELECTABLE_COMPONENT"></span>**Выбираемый компонент**
</dt> <dd>

Считается, что компонент является выбираемым, если его явное включение в операцию резервного копирования или восстановления является необязательным.

Выбор варианта резервного копирования и возможность выбора для восстановления не зависят друг от друга — компонент может быть выбран для обеих операций, для восстановления, а не для резервного копирования, для резервного копирования и не восстановления.

Модули записи также используют возможность выбора для организации компонентов в группы:

-   Невыбираемые компоненты без выбираемых для них предков в их логических путях всегда должны включаться явным образом, если необходимо выполнить резервное копирование или восстановление модуля записи.
-   Невыбираемые компоненты с выбираемыми для выбора предками будут включаться в резервную копию или восстановление только в том случае, если включен хотя бы один элемент, который можно выбрать.
-   Выбираемые компоненты без выбора предков могут быть явно добавлены в операцию резервного копирования или восстановления.
-   Выбираемые компоненты можно явно включать в операцию резервного копирования или восстановления, а также можно неявно включать, если включен один из их выбираемых предков.

См. также [*компоненты*](vssgloss-c.md), возможность *выбора для резервного копирования*, *Выбор для восстановления*, [*явное включение компонентов*](vssgloss-e.md), [*Включение неявных компонентов*](vssgloss-i.md).

</dd> <dt>

<span id="base.vssgloss_shadow_copy"></span><span id="BASE.VSSGLOSS_SHADOW_COPY"></span>**Теневое копирование**
</dt> <dd>

Реплика на момент времени, доступная только для чтения для содержимого исходного тома. Каждая теневая копия шифруется с помощью постоянного идентификатора GUID. Также называется теневой копией тома. См. также *Набор теневых копий*.

</dd> <dt>

<span id="base.vssgloss_shadow_copies_for_shared_folders"></span><span id="BASE.VSSGLOSS_SHADOW_COPIES_FOR_SHARED_FOLDERS"></span>**теневые копии общих папок**
</dt> <dd>

функция Windows, которая создает упрощенные оперативные резервные копии томов с помощью VSS. клиенты могут получить доступ к этим резервным копиям через оболочку Windows для просмотра старых версий файлов и отмены ошибок без полного восстановления. См. также [*теневую копию, доступную для клиента*](vssgloss-c.md).

</dd> <dt>

<span id="base.vssgloss_shadow_copy_set"></span><span id="BASE.VSSGLOSS_SHADOW_COPY_SET"></span>**Набор теневых копий**
</dt> <dd>

Коллекция теневых копий томов, созданная одновременно и определяемая общим ИДЕНТИФИКАТОРом набора теневых копий (постоянный GUID). Это механизм, используемый для управления операциями теневого копирования на томах. См. также *теневое копирование*.

</dd> <dt>

<span id="base.vssgloss_software_based_provider"></span><span id="BASE.VSSGLOSS_SOFTWARE_BASED_PROVIDER"></span>**поставщик программного обеспечения**
</dt> <dd>

Поставщик, который перехватывает запросы ввода-вывода на уровне программного обеспечения между файловой системой и диспетчером томов. См. также поставщик [*оборудования*](vssgloss-h.md), [*поставщик*](vssgloss-p.md).

</dd> <dt>

<span id="base.vssgloss_subcomponent"></span><span id="BASE.VSSGLOSS_SUBCOMPONENT"></span>**вспомогательный компонент**
</dt> <dd>

Любой компонент, имеющий логический путь, содержащий выбираемый родительский компонент (для резервного копирования или восстановления). Подкомпоненты должны иметь логический путь к форме { \_ имя компонента} \\ {имя подкомпонента \_ }. Если можно выбрать подкомпонент (для резервного копирования или восстановления) предка явным образом в резервную копию или восстановление, то этот компонент неявно включается в операцию. Неявно включенные подкомпоненты не содержат данные, включенные в документ компонентов резервного копирования, независимо от того, являются ли они выбираемыми (для восстановления или резервного копирования). См. также: [*компонент*](vssgloss-c.md), [*логический путь*](vssgloss-l.md).

</dd> <dt>

<span id="base.vssgloss_surfaced_shadow_copy"></span><span id="BASE.VSSGLOSS_SURFACED_SHADOW_COPY"></span>**Теневая копия с поверхностью**
</dt> <dd>

Теневое копирование тома, видимое для пространства имен диспетчера подключения системы — это означает, что **финдфирстволуме** и **финднекстволуме** могут найти его, а уведомления о получении и отправлении тома будут созданы. Все доступные теневые копии также отображаются на теневых копиях. Однако не требуется предоставлять доступ к поверхностной теневой копии. Если теневая копия является транспортной, она не может быть выповерхности. См. также: [*доступна теневая копия*](vssgloss-e.md), [*Переносимая теневая копия*](vssgloss-t.md).

</dd> <dt>

<span id="base.vssgloss_system_file_protection"></span><span id="BASE.VSSGLOSS_SYSTEM_FILE_PROTECTION"></span>**Защита системных файлов**
</dt> <dd>

см. раздел [*защита файлов Windows*](vssgloss-w.md).

</dd> <dt>

<span id="base.vssgloss_system_level_applications"></span><span id="BASE.VSSGLOSS_SYSTEM_LEVEL_APPLICATIONS"></span>**приложения уровня системы**
</dt> <dd>

Указывает точку, в которой VSS уведомляет модуль записи о замораживании. Модули записи, инициализированные как приложения уровня системы, получают уведомления после того, как модули записи инициализированы как приложения внешнего интерфейса или как приложения серверного уровня. См. также приложения уровня [*приложения*](vssgloss-a.md), [*серверного*](vssgloss-b.md)уровня, [*приложения внешнего*](vssgloss-f.md)интерфейса.

</dd> <dt>

<span id="base.vssgloss_system_provider"></span><span id="BASE.VSSGLOSS_SYSTEM_PROVIDER"></span>**поставщик системы**
</dt> <dd>

Предустановленный поставщик по умолчанию, предоставленный в составе Windows.

</dd> <dt>

<span id="base.vssgloss_system_volume_folder"></span><span id="BASE.VSSGLOSS_SYSTEM_VOLUME_FOLDER"></span>**Папка системного тома**
</dt> <dd>

Общий каталог, в котором хранятся общие копии общих файлов домена, то есть файлов, которые реплицируются между всеми контроллерами домена в домене.
