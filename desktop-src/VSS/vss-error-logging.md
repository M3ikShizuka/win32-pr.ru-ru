# <a name="vss-error-logging"></a>Ведение журнала ошибок VSS

Следующие сведения об ошибке и состоянии VSS записываются в журнал событий приложений:

-   Ошибки инициатора, созданные с помощью интерфейса [**ивссбаккупкомпонентс**](/windows/desktop/api/VsBackup/nl-vsbackup-ivssbackupcomponents)
-   Ошибки модуля записи, возникшие при использовании класса [**квссвритер**](/windows/desktop/api/VsWriter/nl-vswriter-cvsswriter) , включая переопределяющие методы
-   Ошибки, сформированные поставщиком по умолчанию
-   Ошибки службы VSS, созданные при координации действий поставщика, модуля записи и инициатора запроса (например, создание событий).

Эти ошибки могут быть вызваны несколькими причинами, включая ошибку программирования в коде сторонних разработчиков или ошибки конфигурации, связанные с VSS.

Драйверы VSS и функциональные возможности более низкого уровня записывают ошибки в системный журнал. Сторонние программы (инициатор запроса, поставщик, модуль записи) могут выбрать журнал приложения, системный журнал или и то, и другое, чтобы записать записи журнала ошибок.

Рекомендуется, чтобы приложения высокого уровня (например, код пользовательского режима) использовали журнал приложений. Для приложений более низкого уровня, таких как аппаратные интерфейсы и драйверы, следует использовать системный журнал.
