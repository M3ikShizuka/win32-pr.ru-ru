# <a name="backing-up-and-restoring-the-com-class-registration-database-under-vss"></a>Резервное копирование и восстановление базы данных регистрации классов COM+ в VSS

Модель COM является двоичным стандартом для написания программного обеспечения компонентов в среде распределенных вычислений.

Исходная реализация Win32 идентификаторы компонентов (GUID) и другое состояние COM в реестре в **\_ \_ корневом \\ CLSID для классов hKey**.

Текущее поколение объектной модели компонента COM+ предоставляет базу данных, не зависящую от реестра, для хранения сведений о регистрации компонента: база данных регистрации классов COM+.

База данных регистрации классов COM+ поддерживает модуль записи VSS, который позволяет запрашивающим стороне создавать резервную копию базы данных регистрации с помощью данных, хранящихся на теневом томе.

Чтобы восстановить базу данных регистрации COM+, инициатор запроса должен вызвать метод [икомадминкаталог:: ресторерегдб](/windows/win32/api/comadmin/nf-comadmin-icomadmincatalog-restoreregdb) .

Дополнительные сведения о модуле записи базы данных регистрации COM+ см. [в разделе модули записи VSS в Box](in-box-vss-writers.md).
