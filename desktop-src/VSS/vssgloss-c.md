# <a name="c-volume-shadow-copy-service"></a>C (служба теневого копирования томов)

<dl> <dt>

<span id="base.vssgloss_certification_authority"></span><span id="BASE.VSSGLOSS_CERTIFICATION_AUTHORITY"></span>**центр сертификации**
</dt> <dd>

Сущность сохранность для выдачи сертификатов, подтверждающих, что получатель, компьютер или организация, запрашивающие сертификат, удовлетворяет условиям установленной политики.

</dd> <dt>

<span id="base.vssgloss_client_accessible_shadow_copy"></span><span id="BASE.VSSGLOSS_CLIENT_ACCESSIBLE_SHADOW_COPY"></span>**Теневая копия, доступная для клиента**
</dt> <dd>

Теневая копия, созданная поставщиком системы для поддержки теневые копии общих папок и других механизмов отката, которые позволяют клиентам просматривать старые версии файлов и выполнять отмену операций без полного восстановления. Теневая копия, доступная для клиента, создается с использованием значения, **\_ \_ \_ доступного** для перечисления [**\_ \_ \_ контекста моментальных снимков VSS**](/windows/desktop/api/Vss/ne-vss-vss_snapshot_context) в VSS CTX. Кроме того, \_ \_ \_ \_ для теневых копий, доступных для клиента, перечисление атрибутов VSS VOLSNAP attr, доступное для клиентского [**\_ \_ \_ моментального снимка \_ тома VSS**](/windows/desktop/api/Vss/ne-vss-vss_volume_snapshot_attributes) , устанавливается автоматически. См. также [*теневые копии общих папок*](vssgloss-s.md).

</dd> <dt>

<span id="base.vssgloss_component"></span><span id="BASE.VSSGLOSS_COMPONENT"></span>**см**
</dt> <dd>

Группа файлов, определяемая модулем записи, которая должна обрабатываться как единица во время операций резервного копирования и восстановления. См. также [*компонент базы данных*](vssgloss-d.md), [*компонент файловой группы*](vssgloss-f.md).

</dd> <dt>

<span id="base.vssgloss_component_dependency"></span><span id="BASE.VSSGLOSS_COMPONENT_DEPENDENCY"></span>**зависимость компонентов**
</dt> <dd>

В ситуации, когда компонент (и определяемый им набор компонентов), управляемый одним модулем записи, не может быть скопирован или восстановлен независимо от компонентов, управляемых модулями записи других пользователей. Зависимость не указывает порядок предпочтения между компонентом с документированными зависимостями и компонентами, от которых он зависит: зависимость просто указывает, что компонент и компоненты, от которых он зависит, всегда должны быть резервными копиями или восстановлены вместе.

</dd> <dt>

<span id="base.vssgloss_component_mode"></span><span id="BASE.VSSGLOSS_COMPONENT_MODE"></span>**режим компонента**
</dt> <dd>

Режим, в котором операция резервного копирования или восстановления использует сведения о компоненте модуля записи. См. также компонент, который можно [*выбрать*](vssgloss-s.md).

</dd> <dt>

<span id="base.vssgloss_component_set"></span><span id="BASE.VSSGLOSS_COMPONENT_SET"></span>**набор компонентов**
</dt> <dd>

Группа компонентов, имеющих по крайней мере одну выбираемую (для резервного копирования или восстановления) компонент и ряд невыбираемых компонентов, организованных по их логическим путям. Неявное участие в операции резервного копирования или восстановления зависит от явного включения выбираемого компонента верхнего уровня. В документ компоненты резервного копирования включаются только сведения о компонентах этого выбираемого компонента. Набор компонентов может включать в себя выбираемые и невыбираемые подкомпоненты. См. также [*логический путь*](vssgloss-l.md), [*выбираемый компонент*](vssgloss-s.md).

</dd> <dt>

<span id="base.vssgloss_copy_on_write_shadow_copy"></span><span id="BASE.VSSGLOSS_COPY_ON_WRITE_SHADOW_COPY"></span>**Теневое копирование при записи**
</dt> <dd>

Теневая копия, созданная путем сохранения только отличий от исходного тома.

</dd> <dt>

<span id="base.vssgloss_crash_consistent_state"></span><span id="BASE.VSSGLOSS_CRASH_CONSISTENT_STATE"></span>**состояние отказоустойчивости**
</dt> <dd>

Состояние дисков, эквивалентное тому, что будет найдено после разрушительного сбоя, который внезапно завершает работу системы. Восстановление из такого набора теневых копий будет эквивалентно перезагрузке после внезапного завершения работы. Это состояние по умолчанию для данных, скопированных с помощью теневого копирования без поддержки модулей записи.

</dd> </dl>
