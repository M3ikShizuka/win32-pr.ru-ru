# <a name="com-crm-registry-settings"></a>Параметры реестра COM+ CRM

Существует несколько параметров реестра, позволяющих изменить поведение CRM для помощи при устранении неполадок и разработке. Все эти параметры реестра, перечисленные и описанные в следующей таблице, являются необязательными. для нормальной работы CRM нет необходимости.

Все параметры реестра CRM находятся в разделе **hKey \_ Local \_ Machine \\ Software \\ Microsoft \\ COM3 \\ CRM**. Создайте ключ **CRM** с ключом **COM3** , если он еще не создан.



| Параметры реестра CRM                  | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|----------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **VTRACE1**<br/>                 | REG \_ значение DWORD. Если задать для этого параметра значение, отличное от нуля, можно включить отладку сообщений трассировки по предупреждениям или ошибкам. Эти сообщения можно просмотреть в окне вывод отладчика. Это значение должно быть задано только для разработки, а не во время нормального развертывания. Это значение считывается при запуске любого приложения сервера CRM.<br/>                                                                                                                                                                                                                                                   |
| **игнорекомпенсатореррорс**<br/> | REG \_ значение DWORD. Если задать для этого параметра значение, отличное от нуля, инфраструктура CRM будет игнорировать все ошибки, возвращенные компенсаторами CRM. Если восстановление завершилось сбоем из-за ошибки в компенсаторе CRM, задание этого значения позволяет завершить восстановление. Это значение считывается при запуске любого приложения сервера CRM.<br/>                                                                                                                                                                                                                                           |
| **чеккпоинтинтервалинсек**<br/> | REG \_ значение DWORD. Интервал контрольной точки в секундах. Интервал контрольных точек по умолчанию составляет 30 секунд. Контрольные точки используются для освобождения пространства в файле журнала CRM. Увеличение интервала контрольной точки может повысить производительность, но за счет увеличения времени восстановления и большого файла журнала CRM. Это значение считывается при запуске любого приложения сервера CRM.<br/>                                                                                                                                                                                                  |
| **инитиаллогфилесизеинкб**<br/>  | REG \_ значение DWORD. Это исходный размер файла журнала CRM в килобайтах. Размер файла журнала CRM по умолчанию составляет 1024 КБ (1 МБ). Файл журнала CRM автоматически разворачивается в соответствии с нагрузкой транзакций, но если ожидается большая нагрузка, может потребоваться увеличить это значение. Это значение считывается при запуске любого серверного приложения COM+ с поддержкой CRM, но если файл журнала CRM уже существует для серверного приложения CRM, это значение игнорируется для этого серверного приложения.<br/>                                                                               |
| **рековеритрацеенаблед**<br/>    | REG \_ значение DWORD. Если задать для этого параметра значение, отличное от нуля, включается трассировка восстановления. Трассировка восстановления — это текстовый файл, имя которого совпадает с именем файла журнала CRM и следующим дополнительным расширением: .recoverytrace.txt. <br/> Этот файл находится в том же каталоге, что и файл журнала CRM. Трассировка восстановления предоставляет трассировку активности CRM во время восстановления, которую можно использовать для диагностики проблем. Это значение считывается при запуске любого приложения сервера CRM. Однако для каждого приложения сервера CRM создается уникальный файл трассировки восстановления.<br/> |
