# <a name="designing-for-availability"></a>Проектирование для обеспечения доступности

Доступность — это способность приложения допускать сбои в ресурсах сервера. Это означает, что клиент по-своему будет обрабатываться с ошибкой, и в идеале этот сбой прозрачен для клиента. Очевидно, что сбой может происходить из аппаратных или программных источников, поэтому необходимо разработать оба варианта.

На доступность могут влиять следующие факторы.

-   Модель приложения. Для обеспечения наивысшей доступности убедитесь, что критическая логика приложения выполняется с помощью службы [транзакций COM+](com--transactions.md) . Кроме того, использование механизма компенсации может быть эффективным, чтобы ресурсы оставались в работоспособном состоянии после сбоев.
-   Клиентская модель. Интегрируйте логику повторных попыток в клиентское приложение и стремитесь к постепенному снижению производительности приложения, если ресурсы или службы недоступны. Узнайте, что клиент ожидает от приложения, и создайте конструкцию, которая позволяет альтернативам при возникновении сбоя.
-   Доступность данных и состояния. Для обеспечения единообразного доступа к постоянным данным используйте кластеризацию Windows, чтобы обеспечить поддержку отработки отказа.
-   Доступность службы. Балансировку сетевой нагрузки можно использовать для балансировки нагрузки входящих IP-запросов в кластере серверов.
