# <a name="default-partitions"></a>Секции по умолчанию

Когда пользователю назначается секция по умолчанию, COM+ сначала пытается активировать компоненты в этом разделе. В случае сбоя активации COM+ выполняет поиск компонента для активации в глобальном разделе. Исключением из этого процесса является ситуация, когда компонент COM+ включает моникер секции, который явно указывает секцию, в которой находится компонент. В этом случае моникер раздела имеет приоритет над любой конфигурацией секции по умолчанию.

При использовании локальных секций раздел по умолчанию назначается пользователям с помощью папки " **Пользователи раздела COM+** " в средстве администрирования "службы компонентов" на сервере приложений. При использовании наборов разделов в Active Directory Секция по умолчанию пользователя определяется набором разделов пользователя.
