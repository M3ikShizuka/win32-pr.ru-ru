# <a name="receiving-a-response"></a>Получение ответа

Поскольку поставленные в очередь компоненты предназначены для работы в асинхронном режиме, клиентские приложения не должны блокироваться при ожидании ответа от запроса, поставленного в очередь. Тем не менее часто бывает полезно, чтобы клиентское приложение или связанное приложение на клиентском компьютере получало ответ в конечном итоге. Например, клиент может получать уведомления о том, что запрошенная транзакция успешно завершена.

Существует множество способов, с помощью которых компонент в очереди отправляет ответ обратно вызывающему объекту в асинхронном режиме. Например, он может отправить сообщение электронной почты. Кроме того, сервер может публиковать слабо связанные события, на которые клиент может подписываться.

Другой способ получения ответа от компонента очереди, запущенного на сервере, заключается в том, что клиент передает вызванный метод объекту уведомления. Объект уведомления — это экземпляр компонента в очереди, который выполняется на клиенте. Такой объект уведомления может быть довольно простым, содержащий только целое число, используемое для представления значения ошибки, или может быть довольно сложным, содержащим всю информацию, необходимую для отката транзакции на клиенте. В любом случае вызывающий клиент передает объект уведомления в качестве входного параметра каждый раз, когда он помещает ответ от компонента очереди, который выполняется на сервере. Так как объект уведомления помещается в очередь, сервер может вызвать метод для изменения его состояния, которое впоследствии может быть считано клиентом. В этом сценарии служба очередей компонентов COM+ используется как на клиенте, так и на сервере для обеспечения асинхронного взаимодействия в обоих направлениях.
