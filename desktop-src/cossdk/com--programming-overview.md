# <a name="com-programming-overview"></a>Общие сведения о программировании COM+

COM+ предоставляет среду разработки предприятия на основе модели COM для создания распределенных приложений на основе компонентов. Он также предоставляет средства для создания транзакционных многоуровневых приложений. COM+ сочетает усовершенствования традиционной разработки на основе COM с множеством полезных служб программирования и администрирования. Полный список этих служб см. в разделе [службы COM+](com--services.md) .

Улучшения модели COM включают улучшения как в многопоточной среде, так и в безопасности, а также общие сведения о службах синхронизации. Службы включают в себя средство администрирования "службы компонентов".

Для тех, кто знаком с программированием COM, улучшения COM+ являются значительными, включая следующие:

-   COM+ реализует потоковую модель, называемую нейтральным апартаментным потоком, которая позволяет компоненту осуществлять сериализацию доступа вместе с возможностью выполнения в любом потоке.
-   COM+ поддерживает компоненты с особой средой, называемой [контекстом](com--contexts.md), который предоставляет расширяемый набор свойств, определяющих среду выполнения для компонента.
-   COM+ обеспечивает безопасность на основе ролей, асинхронное выполнение объектов и встроенное специальное имя, которое представляет ссылку на экземпляр объекта, выполняющийся на сервере вне процесса.

## <a name="application-and-component-administration"></a>Администрирование приложений и компонентов

В COM+ база данных регистрации с именем Регдб хранит метаданные, описывающие компоненты. Эта база данных высоко оптимизирована для типа информации, которую COM+ требуется для активации компонентов и используется вместо системного реестра. Кроме того, COM+ предоставляет доступ к *каталогу com+*, который обращается к информации в регдб. Каталог COM+ — это системное хранилище данных, которое содержит сведения о конфигурации для приложений COM+ на данном компьютере сервера.

Наконец, средство администрирования служб компонентов предоставляет полнофункциональный пользовательский интерфейс для разработчиков и администраторов для администрирования компонентов, а также развертывания многоуровневых приложений на стороне клиента и сервера. Дополнительные сведения см. в разделе [развертывание приложений COM+](deploying-com--applications.md).

## <a name="automatic-transactions"></a>Автоматические транзакции

COM+ поддерживает всю семантику сервера Microsoft Transaction Server (MTS) 2,0 и добавляет функцию [автоматического завершения](enabling-auto-done-for-a-method.md) , которую можно установить с помощью средства администрирования "службы компонентов". Эта функция позволяет системе прерывать транзакцию автоматически при срабатывании исключения или при фиксации. Дополнительные сведения см. в разделе [транзакции COM+](com--transactions.md)и [JIT-активация com+](com--just-in-time-activation.md).
