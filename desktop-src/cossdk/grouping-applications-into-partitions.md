# <a name="grouping-applications-into-partitions"></a>Группирование приложений в секции

При принятии решения о группировании приложений на разделы администраторам необходимо учитывать определенные правила и ограничения, включая следующие:

-   Приложение можно установить в одну или несколько секций.
-   В приложении может существовать только один экземпляр данного компонента.

## <a name="public-and-private-components"></a>Общедоступные и частные компоненты

При принятии решения о группировании приложений COM+ существуют другие ограничения. Эти ограничения относятся к общедоступным и частным компонентам в приложении. Как правило, компоненты приложения могут быть общедоступными или частными. Однако при группировании приложений на разделы администраторам следует помнить о некоторых ограничениях, связанных с общими и частными компонентами. Эти ограничения перечислены в следующей таблице.



| Если приложение:              | Затем компоненты в разделе могут быть:                                                                                   |
|------------------------------------|--------------------------------------------------------------------------------------------------------------------------|
| Серверное приложение<br/>    | Общедоступные и частные.<br/>                                                                                           |
| Приложение библиотеки<br/>   | Только общедоступные. В противном случае приложение вызывающих объектов может иметь те же компоненты, что привело бы к неоднозначности.<br/> |
| В глобальном разделе<br/> | Только общедоступные. Это обеспечивает обратную совместимость.<br/>                                                             |



 

## <a name="application-ids"></a>Идентификаторы приложений

Каждое приложение COM+, установленное на компьютере, имеет уникальный идентификатор приложения. Однако имена приложений должны быть уникальными только в пределах одной секции, а не на всем компьютере.

В следующей таблице показано, что происходит с ИДЕНТИФИКАТОРом приложения, когда приложение импортируется в секцию или экспортируется из секции.



| Если приложение:                                              | Идентификатор приложения:                    |
|--------------------------------------------------------------------|---------------------------------------------|
| Импортировано в глобальный раздел<br/>                        | Остается тем же<br/>                 |
| Импортировано в секцию, отличную от глобальной секции<br/> | Изменяется<br/>                       |
| Экспортированный сертификат<br/>                                                | Входит в экспортируемый файл<br/> |
