# <a name="creating-queuable-components"></a>Создание компонентов Куеуабле

Компонент, имеющий по крайней мере один интерфейс куеуабле, является *компонентом куеуабле*. Чтобы компонент вызывался очередью, интерфейсы должны быть помечены как куеуабле, а компонент должен быть установлен в очереди приложения. Однако компонент куеуабле может быть компонентом приложения без очереди.

Интерфейс куеуабле должен содержать только параметры in — без параметров out и без возвращаемых значений. Эти характеристики проверяются путем анализа сведений о типе во время установки компонента. Если интерфейс не куеуабле, то очередь приложения, содержащей компонент, не может быть активирована.

Чтобы указать интерфейс COM+ в качестве куеуабле, выполните следующие действия.

1.  В дереве консоли средства администрирования "службы компонентов" в разделе " **службы компонентов**" Откройте папку **приложения COM+** , связанную с компьютером, которым вы хотите управлять.

2.  Откройте папку **интерфейсы** компонента приложения COM+, которое требуется сделать куеуабле.

3.  Щелкните правой кнопкой мыши интерфейс, который нужно пометить как куеуабле, и выберите пункт **Свойства**.

4.  Выберите вкладку **очередь** в диалоговом окне Свойства.

5.  Активируйте флажок с меткой **поставлено в очередь**.

    > [!Note]  
    > Если флажок в **очереди** неактивен, интерфейс не удовлетворяет описанным выше ограничениям куеуабле.

     

6.  Нажмите **OK**.

    Компонент куеуабле может быть идентифицирован таким образом, добавляя макрос атрибута QUEUEd в раздел Interface исходного файла IDL для всех интерфейсов, куеуабле.

    ``` syntax
#include "mtxattr.h"
    [ object, dual, uuid(), helpstring(IShiphip"), QUEUEABLE ]
    interface IShip:IDispatch{
       [propput, id(1)] HRESULT CustomerId ([in] long CustId);
       [propput, id(2)] HRESULT OrderId ([in] long OrderID);
       [id(3)] HRESULT LineItem ([in] long Qty);
       [id(4)] HRESULT Process ();
    }
    ```
