# <a name="configuring-transaction-isolation-levels"></a>Настройка уровней изоляции транзакций

COM+ предоставляет разработчикам больший контроль над своими приложениями, позволяя настраивать уровни изоляции транзакций. Версии COM+ до COM+ 1,5 всегда использовали самый высокий уровень изоляции транзакций. Хотя этот уровень гарантирует, что целостность данных всегда сохраняется, это может привести к проблемам с производительностью, таким как время ожидания, когда много транзакций необходимо выполнить в большой базе данных. С помощью настраиваемых уровней изоляции опытные разработчики могут увеличить параллелизм, чтобы повысить производительность и масштабируемость.

COM+ предоставляет следующие уровни изоляции транзакций.



| Уровень            | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Сериализованные       | Данные, считанные текущей транзакцией, не могут быть изменены другой транзакцией до завершения текущей транзакции. Новые данные не могут быть вставлены, что повлияет на текущую транзакцию. Это самый надежный уровень изоляции, который используется по умолчанию.                                                                                                                                                                                                                                                                                                                                                    |
| Уровень изоляции repeatable read  | Данные, считанные текущей транзакцией, не могут быть изменены другой транзакцией до завершения текущей транзакции. Во время транзакции можно вставлять любые типы новых данных.                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Уровень изоляции read committed   | Транзакция не может считывать данные, которые изменяются другой транзакцией, которая не зафиксирована. Это уровень изоляции по умолчанию в Microsoft SQL Server.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Уровень изоляции read uncommitted | Транзакция может считывать любые данные, даже если она изменяется другой транзакцией. Это наименее надежный уровень изоляции, но обеспечивает наибольшую степень параллелизма.                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Любой              | Поддерживается любой уровень изоляции. Этот параметр чаще всего используется нисходящими компонентами во избежание конфликтов. Этот параметр полезен, так как любой нисходящий компонент должен быть настроен на уровне изоляции, который равен или меньше уровня изоляции немедленного вышестоящего компонента. Таким образом, нисходящий компонент с уровнем изоляции, настроенным как Any, всегда использует тот же уровень изоляции, который использует его непосредственный восходящий компонент. Если корневой объект в транзакции имеет уровень изоляции, настроенный на Any, его уровень изоляции будет сериализован. |



 

> [!Note]  
> Если нисходящий компонент настроен на более высокий уровень изоляции, чем вышестоящий компонент, и пытается присоединиться к транзакции, возникает ошибка и транзакция прерывается.
