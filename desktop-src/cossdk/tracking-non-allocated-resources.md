# <a name="tracking-non-allocated-resources"></a>Отслеживание нераспределенных ресурсов

Диспетчер распределителя может отслеживанию ресурса, который не поддается инвентаризации, на основе знаний о времени существования объекта. При освобождении отслеживания невыделенный ресурс уничтожается и, следовательно, не возвращается в опись. Этот режим только для наблюдения за ресурсами, которые недорого создавать и уничтожать, более удобен, чем хранение их в инвентаризации. Этот режим также полезен для управления распределителем памяти или для ресурса, который трудно выполнять инвентаризацию из-за слишком большого числа различных типов.

В режиме "только трассировка" распределитель ресурсов непосредственно создает запрошенный ресурс, вместо того чтобы запрашивать от менеджера распределителя распределение одного из запасов. Перед возвращением этого ресурса в компонент запрашивающего приложения распределитель ресурсов сообщает диспетчеру распределителя, что он отслеживает ресурс, что гарантирует, что даже если компонент не сможет освободить ресурс, диспетчер распределителя будет делать это, когда время существования компонента превышено.
