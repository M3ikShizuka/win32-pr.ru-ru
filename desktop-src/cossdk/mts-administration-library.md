# <a name="mts-administration-library"></a>Библиотека администрирования MTS

Библиотека администрирования MTS, поставляемая с интерфейсами администрирования COM+, обеспечивает обратную совместимость с библиотекой администрирования MTS 2,0. Большая часть существующего кода администрирования MTS 2,0 по-прежнему будет работать с предоставленной библиотекой Мтсадмин. Однако некоторые функциональные возможности изменились.

Чтобы воспользоваться преимуществами новых функций или при первом написании кода администрирования, следует использовать библиотеку Комадмин.

Следующие элементы в текущей библиотеке администрирования MTS изменяются из библиотеки администрирования MTS 2,0:

-   Интерфейс **иремотекомпонентутил** больше не доступен.
-   Коллекция **ремотекомпонентс** больше недоступна.
-   Свойство RoleID больше не доступно, имя роли теперь используется в качестве ключа для этого.
-   Метод **експортпаккаже** больше не экспортирует client.exe.
-   Свойство Ремотекомпонентинсталлпас больше не предоставляется в коллекции [**локалкомпутер**](localcomputer.md) .
-   Свойство Аппликатионинсталлпас больше не будет предоставляться в коллекции [**локалкомпутер**](localcomputer.md) .
-   Системный пакет теперь называется системным приложением.
-   Пакет служебных программ теперь называется служебными программами COM+.
-   Свойство IsCollection существует в коллекции [**Components**](components.md) в мтсадмин, но будет возвращать "нотимпл", если установлен флажок и не будет сохранен, если задано значение.
-   Свойство PackageName больше не поддерживается коллекцией [**Components**](components.md) . Чтобы выяснить имя пакета, необходимо получить PackageID, а затем найти соответствующий пакет в коллекции пакетов.
-   Следующие свойства больше не существуют в коллекции [**интерфацесфоркомпонент**](interfacesforcomponent.md) :
    -   **проксиклсид**
    -   **проксидлл**
    -   **проксисреадингмодел**
    -   **типелибфиле**
    -   **типелибид**
    -   **типелиблангид**
    -   **типелибплатформ**
    -   **TypeLibVersion**
