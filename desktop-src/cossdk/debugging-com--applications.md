# <a name="debugging-com-applications"></a>Отладка приложений COM+

Методы отладки для приложений COM+ зависят от языка, на котором вы решили писать компонент.

при выполнении кода в Microsoft Visual C++ можно запустить отладчик на C++ или, используя удаленный клиент, можно выполнить отладку с помощью удаленного управления процедурой (RPC) OLE и jit-отладкой. Вы всегда можете использовать средство администрирования "службы компонентов" для отладки компонента с помощью параметра " **запустить com+ в отладчике** " на вкладке " **Дополнительно** " диалогового окна " **Свойства** " приложения COM+. Дополнительные сведения об отладке компонентов, закодированных в C++, см. [в разделе Отладка компонентов, написанных на Visual C++](debugging-components-written-in-visual-c--.md).

если вы сейчас не выполняете отладку многопоточности, отслеживания компонентов, удаленных вызовов или изоляции процессов, вы можете использовать среду Microsoft Visual Basic в целях отладки. [отладка компонентов, написанных на Visual Basic](debugging-components-written-in-visual-basic.md) описывает процесс отладки в среде Visual Basic.

в разделе [отладка в Visual Basic интегрированной среде](debugging-in-the-visual-basic-ide.md) разработки содержатся общие сведения о рекомендациях, советах и процедурах отладки в интегрированной среде разработки (ide).

дополнительные сведения об отладке дополнительных процессов см. в разделе [отладка, скомпилированная Visual Basic компонентов](debugging-compiled-visual-basic-components.md).

> [!Note]  
> некоторые ограничения, связанные с использованием среды Visual Basic для отладки компонентов с помощью MTS, были разрешены для COM+. дополнительные сведения см. в разделе [поддержка отладки COM+ Visual Basic, которая отличается от MTS](com--visual-basic-debugging-support-contrasted-with-mts.md).
