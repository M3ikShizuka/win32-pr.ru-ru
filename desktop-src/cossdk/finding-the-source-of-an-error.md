# <a name="finding-the-source-of-an-error"></a>Поиск источника ошибки

Вы можете диагностировать источник и получить описание ошибок приложения с помощью COM+ и других средств. Если обнаруживается, что ошибка приложения вызвана COM+, сообщение об ошибке можно интерпретировать при просмотре файла Winerror. h или с помощью служебной программы Microsoft Visual C++ Error.

Если серверное приложение завершается сбоем или вызывает непредвиденное поведение, необходимо сначала определить, где произошла ошибка. Система Просмотр событий отслеживает события приложений, безопасности и системы. Здесь записываются многие ошибки "без предупреждения". Однако не все ошибки отображаются в Просмотр событий.

Сначала следует обратиться к журналу приложений в Просмотр событий, чтобы проверить приложение, связанное с сообщением о событии. (Так как можно также архивировать журналы событий, можно отвести журнал событий ошибки.) Двойной щелчок записи в журнале активирует страницу **свойств события** , которая предоставляет дополнительные сведения о системном событии.

Дополнительные сведения об отладке приложения COM+ см. в разделе [Отладка приложений COM+](debugging-com--applications.md).
