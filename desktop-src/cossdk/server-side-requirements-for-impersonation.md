# <a name="server-side-requirements-for-impersonation"></a>Server-Side требования для олицетворения

Сервер выполняет олицетворение программным способом. Он явно предполагает учетные данные безопасности клиента с помощью [**коимперсонатеклиент**](/windows/desktop/api/combaseapi/nf-combaseapi-coimpersonateclient). Когда клиент предоставил серверу достаточно полномочий, это приводит к замене учетных данных безопасности клиента токеном потока сервера вместо токена процесса.

Когда это будет сделано, сервер может, например, использовать маркер клиента для доступа к ресурсам, защищенным с помощью дескриптора безопасности. Кроме того, если включена маскировка, она может выполнять вызовы под удостоверением клиента.

Сервер может явно задать маскировку программно или может полагаться на административный параметр. По умолчанию приложения COM+ настроены для использования динамической маскировки. Дополнительные сведения см. в разделе [маскировка](cloaking.md).

Если сервер реализует делегирование от имени клиента, используя удостоверение клиента по сети, удостоверение процесса сервера должно быть помечено как "доверенное для делегирования" в службе Active Directory; в противном случае произойдет сбой делегирования.

После завершения использования удостоверения клиента сервер может вернуться к собственному маркеру процесса с помощью [**кореверттоселф**](/windows/desktop/api/combaseapi/nf-combaseapi-coreverttoself).

Дополнительные сведения о реализации олицетворения и делегирования см. в разделе [Делегирование и олицетворение](/windows/desktop/com/delegation-and-impersonation).
