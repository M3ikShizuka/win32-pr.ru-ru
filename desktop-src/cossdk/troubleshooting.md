# <a name="troubleshooting"></a>Устранение неполадок

Если возникают проблемы при диагностике ошибок приложения, см. следующие советы по устранению неполадок.

-   Убедитесь, что координатор распределенных транзакций (DTC) работает на всех серверах.
-   Проверьте сетевое взаимодействие путем первого тестирования на локальном компьютере, чтобы убедиться, что приложение работает. При использовании протокола TCP/IP в сети можно использовать служебную программу ping.exe, чтобы убедиться, что компьютеры находятся в сети.
-   Убедитесь, что SQL и DTC расположены на одном компьютере или программа настройки клиента DTC указывает, что DTC находится на другом компьютере. В противном случае SQLConnect будет возвращать ошибку внутренне при вызове из транзакционного компонента.
-   Задайте для времени ожидания транзакции большее значение, чем значение по умолчанию 60 секунд. По истечении времени ожидания транзакции COM+ прерывает транзакцию. Все последующие вызовы компонента возвращают немедленный результат с КОНТЕКСТом \_ E \_ .
-   Убедитесь, что драйверы ODBC являются потокобезопасными и не имеют сходства потоков.
-   Если у вас возникли трудности с тем, чтобы приложение работало на нескольких серверах, перезагрузите клиент и убедитесь, что контроллер домена настроен правильно.
