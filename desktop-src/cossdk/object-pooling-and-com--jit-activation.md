# <a name="object-pooling-and-com-jit-activation"></a>Объединение объектов в пул и JIT-активация COM+

JIT-активация COM+, по сути, дает компромисс между жадными клиентами и жадными серверами для обеспечения оптимальной производительности. Клиенты получают ссылку на сохранение ссылок на объекты, а серверы могут более точно управлять использованием памяти.

Это можно уточнить, используя [пул объектов COM+](com--object-pooling.md). При помощи пулов объектов, активируемых JIT-активацией, можно выделить определенный объем памяти для хранения определенного числа объектов, активных в памяти, готовых к немедленному повторному использованию. Это лучше всего подходит для создания объектов, как в случае, когда они хранят несколько ресурсов.

Таким образом, для создания пулов JIT-активируемых объектов можно получить следующие преимущества.

-   Значительно ускорить повторную активацию объектов.
-   Повторное использование любых ресурсоемких ресурсов, хранящихся в объектах.
-   Более точное управление использованием памяти и ресурсов для объектов в составе пула.
-   Хранение административной гибкости, благодаря которой приложение может масштабироваться для использования доступных ресурсов и адаптироваться к изменяющимся требованиям к производительности.
