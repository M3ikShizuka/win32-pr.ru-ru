# <a name="com-application-pooling-concepts"></a>Основные понятия пула приложений COM+

Пул приложений COM+ позволяет масштабировать однопотоковые процессы, аналогично тому, как пул потоков позволяет масштабировать однопотоковые объекты. Пулы приложений также могут помочь в восстановлении после сбоев в отдельных процессах, предоставляя другим работающим процессам возможность обработки активаций.

> [!Note]  
> Библиотечные приложения имеют свойства перезапуска и группирования для своего хостового процесса.

 

Все методы интерфейса [**ICOMAdminCatalog2**](/windows/desktop/api/ComAdmin/nn-comadmin-icomadmincatalog2) поддерживают пулы приложений.

Пул приложений COM+ можно настроить для каждого приложения с помощью свойства Конкуррентаппс объекта [**комадминкаталогобжект**](comadmincatalogobject.md) в коллекции [**приложений**](applications.md) . Конкуррентаппс — это целочисленное значение, указывающее максимальное количество одновременных процессов Dllhost, которые должны быть запущены для активации службы для приложения. Это свойство можно задать с помощью средства администрирования служб компонентов или программно.

Если свойство Конкуррентаппс имеет значение 1, которое является значением по умолчанию, служба пулов приложений COM+ отключена.
