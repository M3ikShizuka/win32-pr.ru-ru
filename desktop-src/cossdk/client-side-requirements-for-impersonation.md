# <a name="client-side-requirements-for-impersonation"></a>Client-Side требования для олицетворения

Следующие два параметра могут быть указаны относительно олицетворения на стороне клиента:

-   Уровень олицетворения, который указывает на готовность клиента к тому, что сервер использует его удостоверение.
-   Параметр в службе Active Directory, с помощью которого учетная запись клиента может быть помечена как "учетная запись является конфиденциальной и не может быть делегирована", что отключит делегирование.

Уровень олицетворения может быть задан различными способами. Если клиент не указывает уровень олицетворения, COM-сервер использует значение по умолчанию на уровне компьютера. Значение по умолчанию на уровне компьютера можно задать с помощью средства администрирования "службы компонентов" или Dcomcnfg.exe. Клиент также может указать предпочтительный уровень олицетворения в административном виде с Dcomcnfg.exe.

Клиент может указать уровень олицетворения программным путем с помощью метода [**CoSetProxyBlanket**](/windows/desktop/api/combaseapi/nf-combaseapi-cosetproxyblanket)— эквивалента [**Иклиентсекурити:: сетбланкет**](/windows/desktop/api/objidl/nf-objidl-iclientsecurity-setblanket), который может вызываться как можно чаще, или [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity), который может вызываться один раз для каждого процесса.

Если клиент указывает на олицетворение на уровне делегата — самый широкий из них, который он может предоставить серверу, то клиент должен работать под удостоверением, правильно настроенным в службе Active Directory, чтобы предоставить удостоверение для делегирования.

Дополнительные сведения о уровнях олицетворения и требованиях делегирования к работе см. в разделе [Делегирование и олицетворение](/windows/desktop/com/delegation-and-impersonation).

Само собой, приложения COM+ всегда могут действовать как клиенты. Когда приложение COM+ выполняет вызов к другому приложению или ресурсу, оно выражает уровень олицетворения. Для серверных приложений COM+ можно задать уровень олицетворения административно. Приложения библиотеки COM+ не могут задавать собственный уровень олицетворения; Вместо этого они используют процесс хоста. Сведения о настройке олицетворения для приложения COM+ см. в разделе [Установка уровня олицетворения](setting-an-impersonation-level.md).
