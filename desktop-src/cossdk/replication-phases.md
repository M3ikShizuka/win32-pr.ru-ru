# <a name="replication-phases"></a>Этапы репликации

COMREPL выполняет свою работу в три этапа. Процесс репликации делится на отдельные этапы по следующим двум причинам.

-   Разделение работы для подготовки источника от работы, выполненной на каждом целевом объекте
-   Задержка изменения целевого объекта до получения всех данных из источника

Этапы репликации приведены ниже.



| Этап         | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|---------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Фаза подготовки | Экспортирует все установленные приложения локально на исходном компьютере. Этот этап не затрагивает настройку целевых объектов.                                                                                                                                                                                                                                                                                                                                                                                                           |
| Стадия копирования    | Копирует данные на конечный компьютер. Все приложения, экспортированные в источнике, копируются в целевой объект. Это операция копирования файлов. (См. раздел [Управление файлами](file-management.md).) На этом шаге также запрашиваются некоторые данные с исходного компьютера, которые не инкапсулированы в экспортированных файлах приложения (например, удостоверения приложений). Эти данные хранятся в памяти в COMREPL. Этот этап не затрагивает настройку конечных компьютеров.                                                                               |
| Этап установки | Устанавливает исходный каталог на целевом компьютере. При запуске этого шага все данные, необходимые для перенастройки целевого объекта, находятся в файлах приложения в целевой файловой системе или хранятся как данные экземпляра в COMREPL. На этом этапе будут удалены все установленные приложения на целевом компьютере (за исключением приложений, описанных в разделе [что реплицируется](what-gets-replicated.md)) перед установкой приложений, скопированных из источника. COMREPL устанавливается параллельно с несколькими целевыми объектами с помощью потока установки для каждого целевого объекта. |
