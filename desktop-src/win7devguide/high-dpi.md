# <a name="high-dpi"></a>Высокое разрешение

По мере продвижения технологий отображения, производители увеличили число пикселей, поддерживаемое их отображением. В то время как текст, изображения и элементы пользовательского интерфейса выглядят гораздо более четкими и удобочитаемыми при отображении высокого разрешения, операционная система должна масштабироваться для поддержки визуального интерфейса; в противном случае все просто выглядит меньше.

Windows 7 поддерживает дисплеи с высоким *разрешением* . на рынке данных предполагается, что развертывание экранов с высоким *разрешением* (120-144 точек на дюйм) увеличится в течение Windows 7. При выполнении собственных разрешений на этих экранах многие приложения отображаются очень мелкими, если они не используют *высокое разрешение dpi*. некоторые приложения (например, Windows Internet Explorer) имеют функции масштабирования шрифтов, позволяющие пользователям изменять масштаб, но многие приложения этого не делают. функция *высокого DPI* в Windows 7:

-   обеспечивает оптимальное взаимодействие Windows и приложений на стандартном оборудовании (параметры *DPI* оптимизированы в соответствии с возможностями оборудования).
-   позволяет оболочке Windows и другим приложениям, основанным на Windows, иметь хорошее представление о различных параметрах *DPI* .
-   Учитывает параметры *dpi* по умолчанию, основанные на спецификациях оборудования и возможностях.
-   Позволяет пользователям персонализировать параметры *dpi* без перезагрузки.
-   Гарантирует, что экран всегда будет иметь собственное разрешение.

функция масштабирования Windows *DPI* масштабирует шрифты и элементы пользовательского интерфейса (например, кнопки, значки и поля ввода) на вычисленный процент, как указано в параметре *DPI* . Это отличается от масштабирования, которое происходит при понижении разрешения экрана. в случае масштабирования *DPI* Windows предоставляет шрифты и элементы пользовательского интерфейса, которые рисуются более чем в пикселях, что повышает точность и четкость Windows. сторонние Windows приложения могут использовать параметры *высокого dpi* и соответствующим образом настроить пользовательский интерфейс путем объявления самих *точек с высоким dpi* . Разработчикам приложений больше не следует рассчитывать на то, что 96 dpi является идеальным решением для всех приложений.

Дополнительные сведения о *высоком уровне dpi* и о создании приложений с поддержкой *высокого DPI* см. в разделе [высокое разрешение dpi](../hidpi/high-dpi-desktop-application-development-on-windows.md).
