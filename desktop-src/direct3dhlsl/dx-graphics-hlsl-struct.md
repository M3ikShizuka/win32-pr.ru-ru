# <a name="struct-type"></a>Тип структуры

Используйте следующий синтаксис, чтобы объявить структуру с помощью HLSL.



|                                                                                           |
|-------------------------------------------------------------------------------------------|
| *имя* структуры { \[ *интерполатионмодифиер* \] *Type* \[ *R* x *C* \] *MemberName*;     ... }; |



 

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="Name"></span><span id="name"></span><span id="NAME"></span>*Безымян*
</dt> <dd>

Строка ASCII, однозначно определяющая имя структуры.

</dd> <dt>

<span id="_InterpolationModifier_"></span><span id="_interpolationmodifier_"></span><span id="_INTERPOLATIONMODIFIER_"></span>\[*интерполатионмодифиер*\]
</dt> <dd>

Необязательный модификатор, задающий тип интерполяции. Подробные сведения см. в разделе [Примечания](#remarks).

</dd> <dt>

<span id="Type_RxC_"></span><span id="type_rxc_"></span><span id="TYPE_RXC_"></span>*Тип* \[ *R* x *C*\]
</dt> <dd>

Тип элемента с необязательным размером массива столбца Row (*R*) x (*C*). Структура содержит по крайней мере один элемент. Если он содержит несколько элементов, все они имеют один и тот же тип. Число строк и столбцов — это целые числа без знака в диапазоне от 1 до 4 включительно.

</dd> <dt>

<span id="MemberName"></span><span id="membername"></span><span id="MEMBERNAME"></span>*MemberName*
</dt> <dd>

Строка ASCII, однозначно определяющая имя элемента.

</dd> </dl>

## <a name="remarks"></a>Примечания

Модификатор интерполяции можно указать для любого члена структуры или аргумента в функции шейдера пикселей. Если модификатор присутствует в обоих местах, модификатор снаружи (модификатор аргумента пиксельного шейдера) переносит модификаторы внутрь (модификатор структуры).

При компиляции шейдера или воздействия на пакеты компилятор шейдера члены структуры зависят от [правил упаковки HLSL](dx-graphics-hlsl-packing-rules.md).

### <a name="interpolation-modifiers-introduced-in-shader-model-4"></a>Модификаторы интерполяции, появившиеся в модели шейдеров 4

Выходные данные шейдера вершин, используемые для входных шейдеров пикселей, линейно интерполируются для получения значений в пикселях во время растрирования. Чтобы задать метод интерполяции, используйте любое из следующих значений, которые поддерживаются в [модели шейдеров 4](dx-graphics-hlsl-sm4.md) или более поздней версии. Модификатор игнорируется на выходных данных шейдера вершин, которые не используются в качестве входных шейдеров пикселей.



| Модификатор интерполяции | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **линейная**             | Интерполяция между входными данными шейдера; значение по умолчанию — **линейное** , если модификатор интерполяции не задан.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **центроид**           | Интерполяция между образцами в охваченной области пикселя (для этого может потребоваться экстраполяция конечных точек из пиксельного центра). Выборка центроид может улучшить сглаживание при частичном покрытии пикселя (даже если он не охватывает точек). Модификатор **центроид** должен сочетаться с модификатором **линейной** или **проекции** .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Интерполяция**    | Не выполнять интерполяцию.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Перспектива**      | Не выполнять исправление перспективы во время интерполяции. Модификатор **"Перспектива"** можно сочетать с модификатором **центроид** .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Следующий**             | **Доступно в модели шейдеров 4,1 и более поздних версий** Интерполяция в месте выборки, а не в центре пикселей. Это приводит к тому, что шейдер пикселей выполняется для каждого образца, а не для пикселя. Другой способ выполнения отдельных примеров — наличие входных данных с [семантическим SV \_ SampleIndex](dx-graphics-hlsl-semantics.md), который указывает текущий пример. Только входные данные с указанным **образцом** (или \_ SampleIndex SV) отличаются между вызовами шейдера в пикселе, тогда как другие входы, которые не задают модификаторы (например, при смешении модификаторов для различных входов), по-прежнему интерполируются в пикселном центре. При каждом охваченном образце в пикселе выводятся как вызов шейдера пикселей, так и проверка глубины или трафарета. Иногда это называется « *ресамплинг*». В отличие от того, что в случае отсутствия выборки частоты, известной как *многовыборочная*, построитель текстуры вызывается один раз на пиксель, независимо от количества охваченных выборок, в то время как при тестировании глубины или шаблона возникает частота выборки. Оба режима обеспечивают эквивалентное сглаживание краев. Тем не менее, ресамплинг обеспечивает лучшее качество заливки, чаще вызывая построитель текстуры.<br/> |



 

<dl> 1. При использовании типа int/uint единственным допустимым параметром является " **интерполяция**".  
</dl>

Модификаторы интерполяции могут применяться к членам структуры или [аргументам функций](dx-graphics-hlsl-function-parameters.md)или к обоим.

## <a name="examples"></a>Примеры

Ниже приведено несколько примеров объявлений структуры.


```
struct struct1
{
  int    a;
}
```



Это объявление включает массив.


```
struct struct2
{
  int    a;
  float  b;
  int4x4 iMatrix;
}
```



Это объявление включает модификатор интерполяции.


```
struct In
{
  centroid float2 Texcoord;
};
```
