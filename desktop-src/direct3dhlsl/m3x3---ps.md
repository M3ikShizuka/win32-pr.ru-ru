# <a name="m3x3---ps"></a>m3x3-PS

Умножает вектор трех компонентов на матрицу 3X3.

## <a name="syntax"></a>Синтаксис



| m3x3 DST, src0, src1 |
|----------------------|



 

where

-   DST — это регистр назначения. Result является вектором из трех компонентов.
-   src0 — это исходный регистр, представляющий 3-компонентный вектор.
-   src1 — это исходный регистр, представляющий матрицу 3x3, которая соответствует первому из трех последовательных регистров.

## <a name="remarks"></a>Remarks



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| m3x3                  |      |      |      |      | x    | x    | x     | x    | x     |



 

Для регистра назначения требуется маска XYZ. Модификаторы отрицания и свиззле разрешены для src0, но не для src1.

В следующем фрагменте кода показаны выполняемые операции.


```
dest.x = (src0.x * src1.x) + (src0.y * src1.y) + (src0.z * src1.z);
dest.y = (src0.x * src2.x) + (src0.y * src2.y) + (src0.z * src2.z);
dest.z = (src0.x * src3.x) + (src0.y * src3.y) + (src0.z * src3.z);
```



Входной вектор находится в регистре src0. Входная матрица 3x3 находится в регистре src1 и следующих двух старших регистров, как показано в расширении ниже. Создается объемный результат, при этом другой элемент регистр назначения (dest. w) не затронут.

Эта операция обычно используется для преобразования обычных векторов во время вычислений освещения. Эта инструкция реализована в виде набора изделий с точками, как показано ниже.


```
m3x3 r0.xyz, r1, c0  which will be expanded to:

dp3   r0.x, r1, c0
dp3   r0.y, r1, c1
dp3   r0.z, r1, c2
```
