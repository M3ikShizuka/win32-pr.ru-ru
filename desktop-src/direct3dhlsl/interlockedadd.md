# <a name="interlockedadd-function-hlsl-reference"></a>Функция Интерлоккедадд (Справочник по HLSL)

Выполняет гарантированное атомарное Добавление значения в переменную ресурса-приемника.

## <a name="syntax"></a>Синтаксис

``` syntax
void InterlockedAdd(
  in  R dest,
  in  T value,
  out T original_value
);
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*конечный адрес* \[ окне\]
</dt> <dd>

Тип: **R**

Адрес назначения.

</dd> <dt>

*значение* \[ окне\]
</dt> <dd>

Тип: **T**

Входное значение.

</dd> <dt>

*исходное \_ значение* \[ out\]
</dt> <dd>

Тип: **T**

Необязательный параметр. Исходное входное значение.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Примечания

Эта операция может выполняться только с типизированными ресурсами типа int или uint и переменными общей памяти. Существует два возможных использования этой функции. Первый — когда R является переменной общей памяти. В этом случае функция выполняет атомарное Добавление значения в регистр общей памяти, на который ссылается dest. Второй сценарий заключается в том, что R является типом переменной ресурса. В этом сценарии функция выполняет атомарное Добавление значения в расположение ресурса, на которое ссылается dest. Перегруженная функция имеет дополнительную выходную переменную, которой будет присвоено исходное значение dest. Эта перегруженная операция доступна только в том случае, если R доступен для чтения и записи.
