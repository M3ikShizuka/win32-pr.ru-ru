# <a name="vs"></a>vs

Эта инструкция указывает номер версии шейдера. Эта инструкция работает со всеми версиями шейдера.

## <a name="syntax"></a>Синтаксис


```
vs_mainVer_subVer
```



## <a name="input-arguments"></a>Входные аргументы

Входные аргументы содержат один основной номер версии с одним вложенным номером версии. Допустимые сочетания перечислены в таблице ниже.



| Основные версии | Подверсии                   |
|---------------|--------------------------------|
| 1             | 1                              |
| 2             | 0, SW (программное обеспечение), x (расширенная) |
| 3             | 0, SW (программное обеспечение)               |



 

## <a name="remarks"></a>Примечания



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|------|-------|------|-------|
| vs                     | x    | x    | x    | x     | x    | x     |



 

Эта инструкция должна быть первой инструкцией, отличной от комментария, в шейдере вершин.

Эта инструкция поддерживается во всех версиях шейдеров вершин.

Аппаратно-ускоренные версии программного обеспечения (версии без \_ SW в номере версии) могут обрабатывать вершины с акцелеаратион оборудования или использовать программную обработку вершин. Версии программного обеспечения (версии с \_ SW по номеру версии) обрабатывают вершины только с программным обеспечением.

## <a name="examples"></a>Примеры

В этом частичном примере объявляется \_ шейдер вершин версии 1 1.


```
vs_1_1
```



В этом частичном примере объявляется программный шейдер вершин версии 2.


```
vs_2_sw
```
