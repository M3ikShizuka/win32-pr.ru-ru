# <a name="consumestructuredbuffer"></a>консуместруктуредбуффер

Входной буфер, который отображается в виде потока, который шейдер может извлекать из значений. Только структурированные буферы могут принимать типы T, являющиеся структурами.



| Метод                                                                    | Описание                                 |
|---------------------------------------------------------------------------|---------------------------------------------|
| [**Использование**](sm5-object-consumestructuredbuffer-consume.md)             | Удаляет значение из конца буфера. |
| [**GetDimensions**](sm5-object-consumestructuredbuffer-getdimensions.md) | Возвращает размеры ресурсов.               |



 

Формат UAV, привязанный к этому ресурсу, должен быть создан с \_ \_ неизвестным форматом DXGI.

UAV, привязанный к этому ресурсу, должен быть создан с [**\_ \_ \_ \_ добавлением флага D3D11 buffer UAV**](/windows/desktop/api/d3d11/ne-d3d11-d3d11_buffer_uav_flag).

Дополнительные сведения об использовании структурированных буферов см. в обоих разделах: [Добавление и использование буфера](/windows/desktop/direct3d11/direct3d-11-advanced-stages-cs-resources) и [структурированного буфера](/windows/desktop/direct3d11/direct3d-11-advanced-stages-cs-resources).

## <a name="minimum-shader-model"></a>Минимальная модель шейдера

Этот объект поддерживается в следующих моделях шейдеров.



| Модель шейдера                                                                | Поддерживается |
|-----------------------------------------------------------------------------|-----------|
| [Модели шейдера 5](d3d11-graphics-reference-sm5.md) и более поздних моделей шейдеров | Да       |



 

Этот объект поддерживается для следующих типов шейдеров:



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
|        |      |        |          | x     | x       |
