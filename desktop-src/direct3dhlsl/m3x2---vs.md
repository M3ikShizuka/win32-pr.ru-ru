# <a name="m3x2---vs"></a>m3x2 — VS

Умножает вектор трех компонентов на матрицу 3x2.

## <a name="syntax"></a>Синтаксис



| m3x2 DST, src0, src1 |
|----------------------|



 

where

-   DST — это регистр назначения. Result является вектором из двух компонентов.
-   src0 — это исходный регистр, представляющий 3-компонентный вектор.
-   src1 — это исходный регистр, представляющий матрицу 3x2, которая соответствует первому из двух последовательных регистров.

## <a name="remarks"></a>Remarks



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|------|-------|------|-------|
| m3x2                   | x    | x    | x    | x     | x    | x     |



 

Для регистрации назначения требуется маска XY. Модификаторы отрицания и свиззле разрешены для src0, но не для src1.

В следующих уравнениях показано, как работает инструкция.


```
dest.x = (src0.x * src1.x) + (src0.x * src1.y) + (src0.x * src1.z);
dest.y = (src0.x * src2.x) + (src0.y * src2.y) + (src0.z * src2.z);
```



Входной вектор находится в регистре src0. Входная матрица 3x2 находится в регистре src1 и следующем более высоком регистре в том же файле регистрации, как показано в следующем развертывании. Создается 2D-результат, в результате чего остальные элементы регистра назначения (dest. z и dest. w) не затрагиваются.

Эта операция обычно используется для двумерных преобразований. Эта инструкция реализуется как пара изделий с точками, как показано здесь.


```
m3x2   r0.xy, r1, c0   which will be expanded to:

dp3   r0.x, r1, c0
dp3   r0.y, r1, c1
```



Модификаторы свиззле и отрицания недопустимы для регистра src1. Регистр dest и src0 или любой из src1 + i не могут быть одинаковыми.
