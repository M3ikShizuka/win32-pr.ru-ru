# <a name="bem---ps"></a>Бем-PS

Примените фиктивное преобразование «среда выпуклости».

## <a name="syntax"></a>Синтаксис



| Бем DST. RG, src0, src1 |
|------------------------|



 

где

-   DST. RG DST — это регистр назначения. Необходимо использовать маску записи красного и зеленого компонентов.
-   src0 является исходным регистром.
-   src1 является исходным регистром.

## <a name="remarks"></a>Remarks



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| бем                   |      |      |      | x    |      |      |       |      |       |



 

Эта инструкция выполняет следующие вычисления.


```
(Given n == dest register #)
dest.r = src0.r + D3DTSS_BUMPENVMAT00(stage n) * src1.r 
                + D3DTSS_BUMPENVMAT10(stage n) * src1.g

dest.g = src0.g + D3DTSS_BUMPENVMAT01(stage n) * src1.r
                + D3DTSS_BUMPENVMAT11(stage n) * src1.g
```



Правила использования Бем:

1.  Бем должен располагаться на первом этапе шейдера (то есть перед маркером этапа).
2.  Бем использует два слота арифметических инструкций.
3.  Для шейдера допускается только одно использование этой инструкции.
4.  Целевой вритемаск должен быть. RG/.КСИ.
5.  Эта инструкция не может быть выполнена совместно.
6.  Кроме ограничения, маска записи назначения имеет значение. RG, модификаторы для исходных src0, src1 и модификаторы инструкций не ограничены.
