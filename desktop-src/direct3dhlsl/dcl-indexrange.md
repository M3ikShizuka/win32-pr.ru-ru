# <a name="dcl_indexrange-sm4---asm"></a>дкл \_ индексранже (SM4-ASM)

Объявляет диапазон регистров, доступ к которым будет осуществляться по индексу (целочисленное значение, вычисленное в шейдере).



| дкл \_ Индексранже *минрегистерм*, *максрегистерн* |
|------------------------------------------------|

На все регистры распространяются следующие ограничения.

-   Регистры min и Max должны быть одного типа и иметь одинаковые маски компонентов (при объявлении масок).
-   Регистр может иметь несколько диапазонов индекса, если они не перекрываются.
-   Минимальное число регистров должно быть меньше максимального.
-   Регистр индекса не может содержать [системное значение](dx-graphics-hlsl-semantics.md).
-   Индексирование регистра за пределами определения максимального индекса приводит к неопределенным результатам.

Входные регистры шейдера пикселей должны использовать один и тот же режим интерполяции. выходные регистры шейдера пикселей не индексируются.

Входной регистр шейдера geometry имеет два измерения (ось вершин, ось атрибутов); диапазон индексов применяется только к оси атрибутов, так как ось вершин всегда полностью индексируется.

Эта инструкция применяется к следующим этапам шейдера:



| Вершинный построитель текстуры | Шейдер геометрии | Построитель текстуры |
|---------------|-----------------|--------------|
| x             | x               | x            |



 

Эта инструкция включена для облегчения отладки шейдера в сборке; нельзя создать шейдер на языке ассемблера с использованием модели шейдеров 4.

## <a name="example"></a>Например, .

Ниже приведен пример.


```
dcl_indexRange v1, v3
dcl_indexRange v4, v9
```
