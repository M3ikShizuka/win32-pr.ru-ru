# <a name="ps"></a>ps

Эта инструкция указывает номер версии шейдера и работает во всех версиях шейдера.

## <a name="syntax"></a>Синтаксис


```
ps_mainVer_subVer
```



## <a name="input-arguments"></a>Входные аргументы

Входные аргументы содержат один основной номер версии с одним вложенным номером версии. Допустимые сочетания перечислены в таблице ниже.



| Основные версии | Подверсии                   |
|---------------|--------------------------------|
| 1             | 1, 2, 3, 4                     |
| 2             | 0, x (расширенная), SW (программное обеспечение) |
| 3             | 0, SW (программное обеспечение)               |



 

## <a name="remarks"></a>Примечания



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| ps                    | x    | x    | x    | x    | x    | x    | x     | x    | x     |



 

Эта инструкция должна быть первой инструкцией, не являющейся комментарием, в шейдере пикселей.

Аппаратно-ускоренные версии программного обеспечения (версии без \_ SW в номере версии) могут обрабатывать вершины с акцелеаратион оборудования или использовать программную обработку вершин. Версии программного обеспечения (версии с \_ SW по номеру версии) обрабатывают вершины только с программным обеспечением.

## <a name="examples"></a>Примеры

В этом частичном примере объявляется шейдер версии 1 \_ 1 пиксель.


```
ps_1_1
```



В этом частичном примере объявляется шейдер версии 1 \_ 4 пикселей.


```
ps_1_4
```
