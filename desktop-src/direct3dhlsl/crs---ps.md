# <a name="crs---ps"></a>CR-PS

Вычисление перекрестного произведения с помощью правого правила.

## <a name="syntax"></a>Синтаксис



| CR, летнее время, src0, src1 |
|---------------------|



 

где

-   DST — это регистр назначения.
-   src0 является исходным регистром.
-   src1 является исходным регистром.

## <a name="remarks"></a>Примечания



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| запросы                   |      |      |      |      | x    | x    | x     | x    | x     |



 

Эта инструкция работает, как показано здесь.


```
dest.x = src0.y * src1.z - src0.z * src1.y;
dest.y = src0.z * src1.x - src0.x * src1.z;
dest.z = src0.x * src1.y - src0.y * src1.x;
```



Некоторые ограничения на использование:

-   src0 не может быть тем же регистром, что и dest.
-   src1 не может быть тем же регистром, что и dest.
-   src0 не может иметь свиззле, кроме свиззле по умолчанию (ксизв).
-   src1 не может иметь свиззле, кроме свиззле по умолчанию (ксизв).
-   dest должен иметь ровно одну из следующих семи масок:. x \| . y \| . z \| . XY \| . КСЗ \| . из \| . xyz.
-   конечный объект должен быть временным регистром.
-   конечный объект не должен совпадать с регистром src0 или src1
