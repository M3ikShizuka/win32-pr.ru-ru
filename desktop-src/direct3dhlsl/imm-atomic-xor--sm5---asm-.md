# <a name="imm_atomic_xor-sm5---asm"></a>IMM \_ Atomic \_ XOR (SM5-ASM)

Немедленная атомарная операция XOR в память. Возвращает значение в памяти перед исключающим XOR.



| IMM \_ Atomic \_ XOR dst0 \[ . один \_ компонент \_ Маска \] , dst1, дстаддресс \[ . свиззле \] , src0 \[ . Select \_ Component\] |
|-------------------------------------------------------------------------------------------------------------|



 



| Элемент                                                                                                           | Описание                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|
| <span id="dst0"></span><span id="DST0"></span>*dst0*<br/>                                                | \[в \] содержит значение из *DST1* перед XOR.<br/>                                                                  |
| <span id="dst1"></span><span id="DST1"></span>*dst1*<br/>                                                | \[в \] представлении неупорядоченного доступа (UAV) (u \# ). В шейдере вычислений это также может быть общей памятью группы потоков (g \# ). <br/> |
| <span id="dstAddress"></span><span id="dstaddress"></span><span id="DSTADDRESS"></span>*дстаддресс*<br/> | \[в \] адресе памяти.<br/>                                                                                             |
| <span id="src0"></span><span id="SRC0"></span>*src0*<br/>                                                | Значение XOR с *dst1*.<br/>                                                                                          |



 

## <a name="remarks"></a>Примечания

Эта инструкция выполняет один компонент 32-bit битовый ИСКЛЮЧАЮЩее *src0* с *dst1* в 32-бит на компонент, адрес *дстаддресс*.

Если *dst1* имеет тип u \# , то он мог быть объявлен как необработанный, типизированный или структурированный. При вводе он должен быть объявлен как UINT/Синт с форматом привязанного ресурса, R32 \_ uint/ \_ Синт.

Если *dst1* имеет значение g \# , то его необходимо объявить как необработанный или структурированный.

Значение в памяти *dst1* перед возвратом XOR в *dst0*.

Вся операция выполняется атомарно.

Количество компонентов, взятых из адреса, определяется размерностью ресурса, объявленного на *dst1*.

Если вызов шейдера неактивен, для ексаммпле, если точка была удалена раньше при выполнении, или если существует только один из них, который будет использоваться в качестве вспомогательного метода для фактического пикселя или примера для производных, эта инструкция не изменяет *dst1* память вообще, а возвращаемое значение не определено.

Исходящие адреса в u \# не записываются в память, за исключением случаев, когда u \# является структурированным, а смещение в байтах в структуре (второй компонент адреса) вызывает неограниченный доступ, а все содержимое UAV становится неопределенным.

Исходящие адреса в u \# или g \# вызывают неопределенный результат, возвращаемый шейдеру в *dst0*.

Эта инструкция применяется к следующим этапам шейдера:



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
|        |      |        |          | X     | X       |



 

Так как Уавс доступны на всех стадиях шейдера для Direct3D 11,1, эта инструкция применяется ко всем этапам шейдера для среды выполнения Direct3D 11,1, которая доступна начиная с Windows 8.



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
| X      | X    | X      | X        | X     | X       |
