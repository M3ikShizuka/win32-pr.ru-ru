# <a name="modifiers-for-ps_1_x"></a>Модификаторы для PS \_ 1 \_ X

Модификаторы инструкций влияют на результат инструкции перед его записью в регистр назначения. Например, используйте их для умножения или деления результата на множитель двух или для фиксации результата между нулем и единицей. Модификаторы инструкций применяются после выполнения инструкции, но перед записью результата в регистр назначения.

Ниже показан список модификаторов.



| Модификатор | Описание                   | Синтаксис           | Version |      |      |      |
|----------|-------------------------------|------------------|---------|------|------|------|
|          |                               |                  | 1\_1    | 1\_2 | 1 \_ 3 | 1\_4 |
| \_штук     | Умножить на 2                 | Инструкция \_ x2  | X       | X    | X    | X    |
| \_X4     | Умножить на 4                 | Инструкция \_ X4  | X       | X    | X    | X    |
| \_скорости     | Умножить на 8                 | Инструкция \_ x8  |         |      |      | X    |
| \_D2     | Разделить на 2                   | Инструкция \_ D2  | X       | X    | X    | X    |
| \_D4     | Разделить на 4                   | Инструкция \_ D4  |         |      |      | X    |
| \_D8     | Разделить на 8                   | Инструкция \_ D8  |         |      |      | X    |
| \_Кот    | Насыщенность (срез от 0 до 1) | Инструкция \_ Кот | X       | X    | X    | X    |



 

-   Модификатор перемножения умножает данные регистров на степень двух после считывания. Это то же самое, что и сдвиг влево.
-   Модификатор деления делит данные регистров на степень двух после считывания. Это то же самое, что и право сдвига вправо.
-   Модификатор «насыщенность» служит для фиксации диапазона значений регистров от нуля до одного.

В арифметических инструкциях можно использовать модификаторы инструкций. Они не могут использоваться в инструкциях по адресам текстур.

Модификатор умножения

В этом примере загружается регистр назначения (dest) с суммой двух цветов в исходных операндах (src0 и src1) и результат умножается на два.


```
add_x2 dest, src0, src1
```



В этом примере объединяются два модификатора инструкции. Во-первых, добавляются два цвета в исходных операндах (src0 и src1). Затем результат умножается на два, а между 0,0 — 1,0 для каждого компонента. Результат сохраняется в реестре назначения.


```
add_x2_sat dest, src0, src1
```



Модификатор деления

В этом примере загружается регистр назначения (dest) с суммой двух цветов в исходных операндах (src0 и src1) и делит результат на два.


```
add_d2 dest, src0, src1
```



Модификатор насыщенности

Для арифметических инструкций модификатор насыщенности преобразует результат этой инструкции в диапазон от 0,0 до 1,0 для каждого компонента. В следующем примере показано, как использовать этот модификатор инструкции.


```
dp3_sat r0, t0_bx2, v0_bx2    ; t0 is bump, v0 is light direction
```



Эта операция выполняется после любого модификатора инструкции умножения или деления. \_почти чаще всего используется для фиксации результатов продукта в виде точки. Однако он также обеспечивает последовательную эмуляцию многопроходных методов, где буфер фрейма всегда находится в диапазоне от 0 до 1, а также синтаксиса многотекстуры DirectX 6 и 7,0, в котором насыщенность определяется на каждом этапе.

В этом примере загружается регистр назначения (dest) с суммой двух цветов в исходных операндах (src0 и src1) и выполняется фиксация результата в диапазоне от 0,0 до 1,0 для каждого компонента.


```
add_sat dest, src0, src1
```



В этом примере объединяются два модификатора инструкции. Во-первых, добавляются два цвета в исходных операндах (src0 и src1). Результат умножается на два и обменяются в диапазоне от 0,0 до 1,0 для каждого компонента. Результат сохраняется в реестре назначения.


```
add_x2_sat dest, src0, src1
```
