# <a name="dp4---ps"></a>DP4-PS

Выполняет вычисление 4-компонентного произведения исходных регистров.

## <a name="syntax"></a>Синтаксис



| DP4 DST, src0, src1 |
|---------------------|



 

где

-   DST — это регистр назначения.
-   src0 является исходным регистром.
-   src1 является исходным регистром.

## <a name="remarks"></a>Примечания



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| dp4                   |      | x    | x    | x    | x    | x    | x     | x    | x     |



 

В следующем фрагменте кода показаны выполняемые операции.


```
dest.x = dest.y = dest.z = dest.w = 
    (src0.x * src1.x) + (src0.y * src1.y) + 
    (src0.z * src1.z) + (src0.w * src1.w);
```



Ограничения для PS \_ 1 \_ 2 и PS \_ 1 \_ 3:

-   Каждый шейдер может использовать не более четырех инструкций DP4.
-   Каждая инструкция DP4 считается двумя арифметическими инструкциями.

Ограничения для \_ версий 1 X:

-   Эта инструкция не может быть выдана, так как DP4 выполняется как в векторном, так и в альфа-канале.
