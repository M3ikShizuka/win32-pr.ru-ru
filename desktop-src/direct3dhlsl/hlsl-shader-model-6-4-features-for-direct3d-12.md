# <a name="hlsl-shader-model-64"></a>Модель шейдера HLSL 6,4

Описывает встроенные функции машинного обучения, добавленные в модель шейдера HLSL 6,4.

## <a name="shader-model-64"></a>Модель шейдера 6,4
Эти встроенные функции являются обязательной и поддерживаемой функцией шейдера 6,4. Следовательно, отдельная проверка поразрядности функций не требуется, помимо того, чтобы гарантировать использование модели шейдера 6,4. Минимальный поддерживаемый клиент для этих подпрограмм — Windows 10, версия 1903.

## <a name="shading-language-intrinsics"></a>Встроенные функции языка заливки

### <a name="unsigned-integer-dot-product-of-4-elements-and-accumulate"></a>Целое число без знака Dot-Product 4 элемента и накопление
```syntax
uint32 dot4add_u8packed(uint32 a, uint32 b, uint32 acc); // ubyte4 a, b;
```
 
4-мерный целое число без знака, имеющее значение "точка — произведение" с добавлением. Умножает каждую соответствующую пару 8-разрядных байтов int в два входных DWORD и суммирует результаты в 32-разрядное целое число без знака. Эта инструкция работает в пределах одной 32-разрядной глобальной полосы. В качестве входных данных также предполагается 32-разрядное количество.
 
### <a name="signed-integer-dot-product-of-4-elements-and-accumulate"></a>Целое число со знаком Dot-Product 4 элемента и накопление
```syntax
int32 dot4add_i8packed(uint32 a, uint32 b, int32 acc); // signed byte4 a, b;
```

4-мерный целое число со знаком — поточечное произведение с помощью Add. Умножает каждую соответствующую пару из 8-разрядных байтов int в два входных DWORD и суммирует результаты по 32-разрядному целому числу со знаком. Эта инструкция работает в пределах одной 32-разрядной глобальной полосы. В качестве входных данных также предполагается 32-разрядное количество.
 
### <a name="single-precision-floating-point-2-element-dot-product-and-accumulate"></a>Одинарная точность с плавающей запятой 2-element Dot-Product и accumulate
```syntax
float dot2add( half2 a, half2 b, float acc );
```

Двухмерный half2 векторов с плавающей запятой с помощью Add. Умножает элементы двух входных векторов с половинной точностью и суммирует результаты в 32-разрядном агрегате с плавающей запятой. Эти инструкции работают в пределах одной 32-битной глобальной полосы. Входные данные представляют собой 16-битные количества, Упакованные в одну и ту же полосу.

Это рассматривается с помощью битовой функции с наименьшей точностью (это означает, что имеется собственная половина и небольшая поддержка).

### <a name="sv_shadingrate"></a>SV_ShadingRate
```syntax
uint shadingRate : SV_ShadingRate
```

Целое число без знака, представляющее количество целевых пикселей, записываемых при каждом вызове шейдера пикселей. Допустимые значения принадлежат к набору значений перечисления [D3D12_SHADING_RATE](/windows/win32/api/d3d12/ne-d3d12-d3d12_shading_rate).

Это системное значение доступно на платформах, которые имеют [D3D12_VARIABLE_SHADING_RATE_TIER_2](/windows/win32/api/d3d12/ne-d3d12-d3d12_variable_shading_rate_tier) или более поздней версии. Его можно записать не более чем на одном из стадий шейдера вершин или геометрии. Его можно прочитать на этапе шейдера пикселей. Дополнительные сведения см. в разделе [Заливка переменной скорости](../direct3d12/vrs.md).
