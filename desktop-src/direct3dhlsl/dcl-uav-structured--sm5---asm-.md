# <a name="dcl_uav_structured-sm5---asm"></a>дкл \_ UAV \_ Structured (SM5-ASM)

Объявите неупорядоченное представление доступа (UAV) для использования шейдером.



| дкл \_ UAV \_ Structured \[ \_ ГЛК \] дстуав, структбитестриде |
|--------------------------------------------------------|



 



| Элемент                                                                                                                                   | Описание                                           |
|----------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------|
| <span id="dstUAV"></span><span id="dstuav"></span><span id="DSTUAV"></span>*дстуав*<br/>                                         | \[в \] UAV.<br/>                            |
| <span id="structByteStride"></span><span id="structbytestride"></span><span id="STRUCTBYTESTRIDE"></span>*структбитестриде*<br/> | \[\]Размер структуры в байтах.<br/> |



 

## <a name="remarks"></a>Remarks

*дстуав* — это регистрация в формате u, \# объявленная как ссылка на унордередакцессвиев структурированного буфера с заданным шагом, который должен быть привязан к слоту UAV \# в API.

Содержимое структуры не имеет типа; операции, выполняемые в памяти, могут неявно интерпретировать данные как имеющие тип.

*структбитестриде* — это размер структуры в байтах в объявляемом буфере. Это значение должно быть больше нуля. *структбитестриде* имеет тип uint и должен быть кратен 4.

Инструкции, которые ссылаются на структурированный u \# , принимают 2D-адрес, где первый компонент выбирает \[ структуру \] , а второй компонент выбирает \[ смещение в пределах структуры в соответствии с выделенными байтами \] .

\_Флаг ГЛК означает «глобально согласованность». Отсутствие \_ ГЛК означает, что UAV объявляется только как "согласование с группами" в шейдере вычислений или "локальное согласование" в одном вызове шейдера пикселей.

\_Флаг OPC — это счетчик с сохранением заказов. Это означает, что если UAV привязан к слоту \# (u \# ), он должен быть создан с ФЛАГом счетчика. Это означает, что операции [ \_ \_ распределения IMM Atomic](imm-atomic-alloc--sm5---asm-.md) или [IMM \_ атомарных операций \_ использования](imm-atomic-consume--sm5---asm-.md) в шейдере управляют счетчиком, значения которого можно использовать в шейдере в качестве постоянной ссылки на расположение в UAV. Невозможно изменить порядок данных после поверх шейдера.

Отсутствие \_ флага OPC означает, что если шейдер использует инструкции[IMM \_ Atomic \_ Alloc](imm-atomic-alloc--sm5---asm-.md) или [IMM \_ Atomic \_ ](imm-atomic-consume--sm5---asm-.md) , а UAV привязан к слоту \# (u), он должен быть создан с флагом Append, который предоставляет счетчик, не гарантирующий, что порядок сохраняется после вызова шейдера.

Если \_ флаг OPC отсутствует, а шейдер не содержит инструкций [IMM \_ Atomic \_ Alloc](imm-atomic-alloc--sm5---asm-.md) или [IMM \_ Atomic \_ ](imm-atomic-consume--sm5---asm-.md) , то можно создать UAV, привязанный к СЛОТу \# (u), с флагом счетчика (счетчик будет использоваться этим шейдером), флаг (нет счетчика), но не с флагом Append.

> [!Note]  
> CS \_ 4 \_ 0 и CS \_ 4 \_ 1 поддерживают **дкл \_ тгсм \_ Structured**, но не [дкл \_ тгсм \_ RAW](dcl-tgsm-raw--sm5---asm-.md).

 

Эта инструкция применяется к следующим этапам шейдера:



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Службы вычислений |
|--------|------|--------|----------|-------|---------|
|        |      |        |          | X     | X       |



 

Так как Уавс доступны на всех стадиях шейдера для Direct3D 11,1, эта инструкция применяется ко всем этапам шейдера для среды выполнения Direct3D 11,1, которая доступна начиная с Windows 8.



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Службы вычислений |
|--------|------|--------|----------|-------|---------|
| X      | X    | X      | X        | X     | X       |
