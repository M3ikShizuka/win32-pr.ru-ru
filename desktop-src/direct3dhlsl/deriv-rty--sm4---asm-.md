## <a name="remarks"></a>Примечания

Для каждой отметки 2x2 пикселей вычислена только одна производная пара x, y.

Эта операция зависит от оборудования.

Ссылка на реализацию средства программной прорисовки для треугольников:

-   Шейдер пикселей всегда запускает шейдер на четырех пикселях в локкстеп (даже через Управление потоком, маскирование отключенных пикселов).
-   Четыре всегда имеют даже пронумерованные координаты пикселей (x и y) для верхнего левого пикселя.
-   Фиктивные Пиксели выключаются из примитивов, если примитив слишком мал для заполнения 2x2 четырех.
-   [наследование \_ RTX](deriv-rtx--sm4---asm-.md) вычислено при первом выборе 2 пикселя: текущего пикселя и другого пикселя с той же координатой y из четырех. Затем результат вычисляется следующим образом: *src0*(нечетный x пиксель) — *src0*(даже x пиксель) \[ для каждого компонента\]
-   **наследование \_ РТИ** вычислено при первом выборе 2 пикселя: текущего пикселя и другого пикселя с той же координатой x из четырех. Затем результат вычисляется следующим образом: *src0*(нечетный y пиксель) — *src0*(даже y пиксель) \[ для каждого компонента.\]

Эта инструкция применяется к следующим этапам шейдера:



| Вершинный построитель текстуры | Шейдер геометрии | Построитель текстуры |
|---------------|-----------------|--------------|
|               |                 | x            |
