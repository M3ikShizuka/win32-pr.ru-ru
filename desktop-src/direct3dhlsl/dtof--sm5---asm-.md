# <a name="dtof-sm5---asm"></a>дтоф (SM5-ASM)

Компонентное преобразование из данных с плавающей запятой двойной точности в данные с плавающей запятой одиночной точности.



| дтоф dest \[ . Mask \] , \[ - \] src \[ . свиззле \] , |
|-------------------------------------------|



 



| Элемент                                                            | Описание                                          |
|-----------------------------------------------------------------|------------------------------------------------------|
| <span id="dest"></span><span id="DEST"></span>*dest*<br/> | \[в \] адрес преобразованных данных.<br/> |
| <span id="src0"></span><span id="SRC0"></span>*src0*<br/> | \[\]преобразуемые данные.<br/>          |



 

## <a name="remarks"></a>Примечания

Каждый компонент источника преобразуется из представления двойной точности в представление с одиночной точностью с использованием округления до ближайшего числа.

Допустимый свиззлес для параметра source:. ксизв,. ксикси,. звкси,. звзв.

Допустимые маски *приемников* — это один или два компонента. Это:. x,. y,. z,. w,. XY,. КСЗ,. КСВ,. из,. Ив,. ZW результат первого преобразования переходит к первому компоненту маски, а результат второго компонента переходит во второй компонент маски, если он есть.

компоненты *dest* — float32.

*src* — это двойное vec2 в (x 32LSB, y 32MSB) и (z 32LSB, w 32MSB) POST свиззле.

Для преобразований типа float32<->, реализации могут либо учитывать значение float32, либо могут сбрасывать их.

Эта инструкция применяется к следующим этапам шейдера:



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
| X      | X    | X      | X        | X     | X       |
