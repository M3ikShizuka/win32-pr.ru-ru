## <a name="remarks"></a>Remarks

Эти числовые типы имеют скалярные, векторные и матричные расширения ключевых слов.

-   float, int, uint, bool
-   min10float, min16float
-   min12int, min16int
-   min16uint

Расширения этих числовых типов следуют этому шаблону, в котором в качестве примера используется float.

-   скалярная

    <dl> FLOAT  
    </dl>

-   Вектор

    <dl> float1, float2, float3, float4  
    </dl>

-   Матрица

    <dl> float1x1, float1x2, float1x3, float1x4  
    float2x1, float2x2, float2x3, float2x4  
    float3x1, float3x2, float3x3, float3x4  
    float4x1, float4x2, float4x3, float4x4  
    </dl>

HLSL поддерживает более низкие [текстуры](dx-graphics-hlsl-texture.md) и [образцы](dx-graphics-hlsl-sampler.md) для устаревших целей. Вместо этого для новых приложений рекомендуется использовать новые объекты текстуры HLSL ([Texture2D](sm5-object-texture2d.md), [Texture3D](sm5-object-texture3d.md)и т. д.) и объекты образцов ([самплерстате и самплеркомпарисонстате](dx-graphics-hlsl-sampler.md)).

### <a name="export"></a>экспорт

Используйте **Export** , чтобы отмечать функции, Упакованные в библиотеку.

Пример:

``` syntax
export float identity(float x)
{
    return x;
}
```

Пометив функцию **Identity** с помощью ключевого слова **Export** , вы сделаете функцию **Identity** доступной из библиотеки для последующей компоновки. Без маркировки **экспорта** функция **Identity** недоступна для последующей компоновки.

Компилятор игнорирует ключевое слово **экспорта** для компиляции, не являющейся библиотекой.

> [!Note]  
> Ключевое слово **Export** требует D3dcompiler \_47.dll или более поздней версии библиотеки DLL.
