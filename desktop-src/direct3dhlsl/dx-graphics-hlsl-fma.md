# <a name="fma"></a>fma

Возвращает предохранитель с двойной точностью, умноженный на " \* b + c".



| *RET* FMA (двойное *a*, *b*, *c*); |
|----------------------------------|



 

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="a"></span><span id="A"></span>*конкретного*
</dt> <dd>

\[в \] первом значении с плавким предохранителем.

</dd> <dt>

<span id="b"></span><span id="B"></span>*&*
</dt> <dd>

\[во \] втором значении с плавким предохранителем.

</dd> <dt>

<span id="c"></span><span id="C"></span>*ц*
</dt> <dd>

\[в \] третьем значении с плавким предохранителем.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Предохранитель с двойной точностью — сложение параметров *a* \* *b*  +  *c*. Возвращаемое значение должно быть точным до 0,5 единиц с наименьшей точностью (ULP).

## <a name="remarks"></a>Примечания

Встроенная функция **FMA** должна поддерживать значений NaN, INF и денормы.

Чтобы использовать встроенную функцию **FMA** в коде шейдера, вызовите метод [**ID3D11Device:: чеккфеатуресуппорт**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkfeaturesupport) с [**\_ \_ \_ параметром D3D11 Feature D3D11**](/windows/desktop/api/d3d11/ne-d3d11-d3d11_feature) , чтобы убедиться, что устройство Direct3D поддерживает параметр [**екстендеддаублесшадеринструктионс**](/windows/desktop/api/d3d11/ns-d3d11-d3d11_feature_data_d3d11_options) . Для встроенной функции **FMA** требуется драйвер дисплея WDDM 1,2, а все драйверы дисплея WDDM 1,2 должны поддерживать **FMA**. Если приложение создает устройство отрисовки с [уровнем функций](/windows/desktop/direct3d11/overviews-direct3d-11-devices-downlevel-intro) 11,0 или 11,1, а целью компиляции является модель шейдера 5 или более поздней версии, исходный код HLSL может использовать встроенную функцию **FMA** .
