# <a name="output-registers"></a>Выходные регистры

-   Регистр цвета вершин
-   Регистр тумана
-   \_Регистр позиций
-   \_Регистр размера \_ точки
-   \_Регистр координаты \_ текстуры

Именам регистров предшествуют строчная буква o, указывающая, что выходные регистры доступны только на запись.

## <a name="vertex-color-register---od0-od1"></a>Регистр цветов вершин — oD0, oD1

oD0 — это регистр рассеянного цвета. oD1 — регистр отраженного цвета. Значение oD0 интерполируются и записывается во входной цвет регистр 0 (v0) шейдера пикселей. Значение oD1 интерполируются и записывается во входной цвет регистр 1 (v1) шейдера пикселей. Дополнительные сведения о цветовых регистрах шейдера пикселей см. в разделе registers.



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ SW | 2 \_ x | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|-------|------|------|-------|
| Регистр цвета вершин  | x    | x    | x     | x    |      |       |



 

## <a name="fog-register---ofog"></a>Регистр тумана — Офог

Регистры значений выходных тумана. Значение — это коэффициент тумана для интерполяции, который затем направляется в таблицу тумана. Используется только скалярный компонент x-компонента тумана. Перед передачей в средство программной прорисовки значения отправляются между нулем и единицей.



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ SW | 2 \_ x | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|-------|------|------|-------|
| Регистр тумана           | x    | x    | x     | x    |      |       |



 

## <a name="position-register---opos"></a>Регистр позиций — oPos

Регистрация расположения выходных данных. Значение является позицией в однородной вырезанной области. Это значение должно быть записано шейдером вершин.



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ SW | 2 \_ x | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|-------|------|------|-------|
| Регистр позиций      | x    | x    | x     | x    |      |       |



 

## <a name="point-size-register---opts"></a>Регистр размера точки — наведении

Регистры выходных данных размера точки вывода. Используется только скалярный x-компонент размера точки.



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ SW | 2 \_ x | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|-------|------|------|-------|
| Регистр размера точки    | x    | x    | x     | x    |      |       |



 

## <a name="texture-coordinate-register---ot0-to-ot7"></a>Регистрация координаты текстуры — oT0 в oT7

Выходные координаты выходной текстуры регистрируются. В частности, это массив регистров выходных данных, которые просматриваются и используются в качестве координат текстуры на стадиях выборки текстуры. Эти данные передаются в шейдер пикселей.



| Версии шейдеров вершин      | 1\_1 | 2 \_ 0 | 2 \_ SW | 2 \_ x | 3 \_ 0 | 3 \_ SW |
|-----------------------------|------|------|-------|------|------|-------|
| Регистр координаты текстуры | x    | x    | x     | x    |      |       |



 

При записи в регистр текстурных координат рекомендуется передавать в качестве измерения соответствующей карты текстуры только столько чисел с плавающей запятой. Управление значениями, передаваемыми с помощью модификатора. Например, используйте. XY для двухмерной текстурной гиперкарты.

Если для этапа текстуры включена проекция текстуры, все четыре значения с плавающей запятой должны быть записаны в соответствующий регистр текстуры.

\*При использовании программируемого конвейера любой из флагов преобразования текстуры D3DTTFF должен быть равен нулю.

### <a name="texture-coordinate-range"></a>Диапазон координат текстуры

Данные вершин объекта предоставляют координаты текстуры ввода. Объекты, которые не используют мозаичные текстуры, обычно имеют координаты текстуры в диапазоне \[ 0, 1 \] . Объекты, использующие мозаичные текстуры, такие как ландшафта, обычно имеют координаты текстуры в диапазоне от \[ -?, +? \] где? может быть большим числом с плавающей запятой.

Интерполяция текстурных координат выполняется для данных вершин для растрирования. Во время растрирования координаты текстуры интерполируются между вершинами объекта, изменены с помощью переноса текстур и масштабируются по размеру текстуры (также принимая во внимание режим адресации текстуры) для создания целочисленного индекса. Затем индекс используется для поиска текстур. Макстекстуререпеат можно использовать для определения того, сколько раз можно выполнить мозаичное заполнение текстуры.

Если координаты текстуры считываются непосредственно в пиксельный шейдер (с помощью текскурд или текскрд), диапазон координат текстуры зависит от инструкции и версии построителя текстуры.
