# <a name="ifdef-and-ifndef-directives"></a>\#\#директивы ifdef и ifndef

Директивы препроцессора, которые определяют, определена ли конкретная константа препроцессора или макрос.



| \#*идентификатор* ifdef...  |
|---------------------------|
| \#endif                   |
| \#*идентификатор* ifndef... |
| \#endif                   |



 

## <a name="parameters"></a>Параметры



| Элемент                                                                              | Описание                                               |
|-----------------------------------------------------------------------------------|-----------------------------------------------------------|
| <span id="identifier"></span><span id="IDENTIFIER"></span>*Идентификатор*<br/> | Идентификатор проверяемой константы или макроса. <br/> |



 

## <a name="remarks"></a>Примечания

\#Директивы ifdef и ifndef можно использовать \# в любом месте, где можно использовать, [ \# Если](dx-graphics-hlsl-appendix-pre-if.md) может использоваться. \#Инструкция ifdef эквивалентна директиве. Эти директивы проверяют только на наличие или отсутствие идентификаторов, определенных с помощью директивы [ \# define](dx-graphics-hlsl-appendix-pre-define.md) , а не для идентификаторов, объявленных в исходном коде C или C++.

Эти директивы предназначены только для совместимости с предыдущими версиями языка. Рекомендуется использовать [определенный](dx-graphics-hlsl-appendix-pre-if.md) оператор с \# директивой if.

\#Директива ifndef проверяет наличие противоположного условия, установленного параметром \# ifdef. Если идентификатор не определен, условие имеет значение true (не равно нулю); в противном случае условие имеет значение false (ноль).

## <a name="examples"></a>Примеры

Значение идентификатора можно передать из командной строки при помощи параметра /D. Он позволяет определить до 30 макросов. Это позволяет проверить, существует ли определение, поскольку определения можно передавать из командной строки. В следующем примере это поведение используется для определения того, следует ли запускать приложение в тестовом режиме.


```
// PROG.CPP
#ifndef test
  #define final
#endif
int main()
{
}
```



При компиляции с помощью следующей команды Prog. cpp будет компилироваться в тестовом режиме. в противном случае он будет компилироваться в окончательном режиме.


```
CL.EXE /Dtest prog.cpp
```
