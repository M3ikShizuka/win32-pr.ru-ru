# <a name="ftod-sm5---asm"></a>фтод (SM5-ASM)

Компонентное преобразование из данных с плавающей запятой одиночной точности в данные двойной точности с плавающей запятой.



| фтод dest \[ . Mask \] , \[ - \] src \[ . свиззле \] , |
|-------------------------------------------|



 



| Элемент                                                            | Описание                                          |
|-----------------------------------------------------------------|------------------------------------------------------|
| <span id="dest"></span><span id="DEST"></span>*dest*<br/> | \[в \] адрес преобразованных данных.<br/> |
| <span id="src0"></span><span id="SRC0"></span>*src0*<br/> | \[\]преобразуемые данные.<br/>          |



 

## <a name="remarks"></a>Remarks

Каждый компонент источника преобразуется из представления с одиночной точностью в представление двойной точности.

Допустимые маски *приемников* : XY, ZW и. ксизв. . XY получает результат первого преобразования, а. ZW получает результат второго преобразования.

*dest* — это двойное vec2 в (x 32LSB, y 32MSB) и (z 32LSB, w 32MSB).

*src* — это vec2 с плавающей запятой в пределах x и y (ZW игнорируется) (POST свиззле).

Для преобразований типа float32<->, реализации могут либо учитывать значение float32, либо могут сбрасывать их.

Эта инструкция применяется к следующим этапам шейдера:



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
| X      | X    | X      | X        | X     | X       |
