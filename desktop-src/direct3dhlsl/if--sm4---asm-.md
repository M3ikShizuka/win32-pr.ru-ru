# <a name="if-sm4---asm"></a>If (SM4-ASM)

Ветвь на основе логического или результата.



| If { \_ z \|\_NZ} src0. Выбор \_ компонента |
|--------------------------------------|



 



| Элемент                                                            | Описание                                                              |
|-----------------------------------------------------------------|--------------------------------------------------------------------------|
| <span id="src0"></span><span id="SRC0"></span>*src0*<br/> | \[в \] содержит компонент, для которого необходимо проверить условие.<br/> |



 

## <a name="remarks"></a>Remarks

Формат токена содержит смещение соответствующей инструкции endif в шейдере для удобства.

В следующем примере показано, как использовать эту инструкцию.

``` syntax
                if_z r0.x // if all bits in r0.x are zero
                   ...
                else // (optional)
                   ...
                endif
                if_nz r1.x // if any bit in r0.x is nonzero
                   ...
                else // (optional)
                   ...
                endif
```

### <a name="restrictions"></a>Ограничения

-   Исходные операнды (если 4 вектора компонентов) должны использовать один селектор компонента.
-   32-разрядный регистр, предоставляемый *src0* , тестируется на битовом уровне. Если любой бит не равен нулю, **Если \_ z** будет иметь значение true. Значение, если все биты равны нулю, **Если \_ NZ** будет иметь значение true.
-   блоки управления Flow могут вкладывать до 64 глубины в одну подпрограммы (и главную). Компилятор HLSL не будет создавать подпрограммы, превышающие это ограничение. Поведение инструкций потока управления свыше 64 уровней в глубину (по подподпрограмме) не определено.

Эта инструкция применяется к следующим этапам шейдера:



| Вершинный построитель текстуры | Шейдер геометрии | Построитель текстуры |
|---------------|-----------------|--------------|
| x             | x               | x            |
