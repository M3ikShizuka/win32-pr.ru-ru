# <a name="texture-coordinate-register-hlsl-ps-reference"></a>Регистр текстурной координаты (Справочник по HLSL PS)

Входной регистр шейдера пикселей, содержащий координаты текстуры.



| Версии шейдеров пикселей       | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ SW | 2 \_ x | 3 \_ 0 | 3 \_ SW |
|-----------------------------|------|------|------|------|------|-------|------|------|-------|
| Регистр координаты текстуры |      |      |      |      | x    | x     | x    | x    | x     |



 

Регистр координат текстуры содержит данные о координатах текстуры. Перед использованием регистра координат текстуры его необходимо объявить объявлением шейдера пикселей. Дополнительные сведения об объявлении регистра текстуры см. в разделе [дкл-(SM2, SM3-PS ASM)](dcl---ps.md).

Кроме того, ниже приведены некоторые другие свойства регистров координат текстуры.

-   Существует восемь регистров координат текстуры шейдера пикселей, t0 в T7.
-   Это регистры только для чтения.
-   Они содержат значения RGBA из четырех компонентов, перебираемые из входных вершин.
-   Они содержат высокую точность, высокие значения динамических диапазонов данных, которые интерполируются из данных вершин. Значения создаются с интерполяцией с правильной перспективой. Данные являются точностью с плавающей запятой и подписываются.
-   В одной инструкции существует максимум один.
-   При нескольких операциях чтения регистра координат текстуры в шейдере должна использоваться идентичная [маска записи регистра назначения](dx9-graphics-reference-asm-ps-registers-modifiers-write-mask.md).
-   Необязательный модификатор частичной точности \[ \_ PP \] применяется к зависимым операциям чтения. Это связано с тем, что частичная точность влияет на арифметические операции, в которых используется регистр текстурной координаты. Он не влияет на точность инструкций по адресу текстуры, так как не влияет на итераторы координат текстуры.
