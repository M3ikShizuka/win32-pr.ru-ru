# <a name="dcl_usage-output-sm1-sm2-sm3---vs-asm"></a>\_выходные данные использования дкл (SM1, SM2, SM3-VS ASM)

Различные типы выходных регистров были свернуты в двенадцать выходных регистров (два для цвета, восемь для текстуры, одна для позиции, а другая для тумана и размера точки). Они могут использоваться для того, что нужно выполнить для интерполяции для шейдера пикселей: текстурные координаты, цвета, туман и т. д.

Для выходных регистров требуются объявления, которые включают семантику. Например, старые регистры позиции и размера точек заменяются путем объявления выходного регистра с указанием позиции или семантики размера точки.

Из двенадцати выходных регистров любое десять (не обязательно O0 на O9) состоит из четырех компонентов (ксизв), а другое должно быть объявлено как положенное (а также включать все четыре компонента), и, по желанию, один из них может быть скалярным размером точки.

## <a name="syntax"></a>Синтаксис

Синтаксис для объявления выходных регистров аналогичен объявлениям для входного регистра:



|                                  |
|----------------------------------|
| \_семантика дкл o \[ . запись \_ маски\] |



 

Где:

-   \_семантика дкл может использовать тот же набор семантики, что и для объявления ввода. Семантические имена берутся из [**D3DDECLUSAGE**](/windows/desktop/direct3d9/d3ddeclusage) (и сопряжены с индексом, например position3). Всегда должен быть один выходной регистр с семантикой positiont0, если не используется для обработки вершин. Семантика positiont0 и семантика pointsize0 являются единственными, которые имеют смысл за пределами простого разрешения компоновки из вершины в шейдеры пикселей. Для шейдеров с управлением потоком предполагается, что выводится наихудший результат. Значения по умолчанию отсутствуют, если шейдер фактически не выводит то, что его объявляет (из-за управления потоком).
-   o — это выходной регистр. См. раздел [выходные \_ регистры](dx9-graphics-reference-asm-vs-registers-vs-3-0.md).
-   \_маска записи указывает на один и тот же выходной регистр, который можно объявить несколько раз (так что разные семантики могут применяться к отдельным компонентам), каждый раз с уникальной маской записи. Однако одна и та же семантика не может использоваться несколько раз в объявлении. Это означает, что векторы должны состоять из четырех компонентов или меньше и не могут быть передаваться по границам регистров для четырех компонентов (отдельные регистры). При использовании семантики размера точки она должна иметь полную маску записи, так как она считается скалярной. При использовании семантики позиций она должна иметь полную маску записи, так как необходимо записать все четыре компонента.

## <a name="declaration-examples"></a>Примеры объявлений


```
vs_3_0
dcl_color4     o3.x    // color4 is a semantic name.
dcl_texcoord3  o3.yz   // Different semantics can be packed into one register.
dcl_fog        o3.w 
dcl_tangent    o4.xyz
dcl_position   o7.xyzw // position must be declared to some unique register 
                       //   in a vertex shader, with all 4 components.

dcl_psize      o6      // Pointsize cannot have a mask 
                       //   (that is, mask is full .xyzw)
                       // This is an implied scalar register. 
                       // No other semantics can be assigned to any components
                       //   of this register.
                       // If pointsize declaration is not used (typical),
                       //   only 11 "out" registers are available, not 12.
                       // Pixel shaders cannot see this value.

```
