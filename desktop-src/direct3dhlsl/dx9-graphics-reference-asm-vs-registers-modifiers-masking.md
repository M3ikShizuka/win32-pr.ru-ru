# <a name="destination-register-masking"></a>Маскирование регистров назначения

Маскирование указывает, какие компоненты регистра назначения будут обновлены в результате выполнения инструкции. Регистры текстур имеют один набор правил и нетекстурные регистры имеют еще один набор правил.

-   DX9 \_ графика \_ Справочник \_ по \_ \_ шаблонам ASM VS registers \_ \_ . Этот раздел содержит правила применения масок к целевым регистрам.
-   \_Маски регистров текстур \_ — регистры текстур имеют уникальные правила.

## <a name="destination-register-masking"></a>Маскирование регистров назначения

Как показано в следующей таблице, Маскирование (где — это один из допустимых [регистров шейдера](dx9-graphics-reference-asm-vs-registers.md)вершин шейдера вершин) может применяться к отдельным компонентам векторных данных.



| Модификатор компонента | Описание      |
|--------------------|------------------|
| r. {x} {y} {z} {w}     | Маска назначения |



 

-   В целом, указание масок записи для регистра назначения является хорошим стилем написания кода. Это упрощает чтение и обслуживание кода.
-   Можно указать любое сочетание компонентов (в том числе None), если x предшествует y, y – z, и z предшествует w.
-   Регистры выходных и Офог должны использовать только одну маску.
-   Для некоторых инструкций требуется, чтобы регистр назначения использовал одну маску записи: EXP, експп, log, логп, pow, rcp и РСК.
-   В версии 1,0 инструкции ФРК требовалось одно из следующих сочетаний маски:. x или. y или. XY. Версия 2,0 не имеет ограничений маски.
-   для синкос требуется одно из следующих сочетаний маски:. x или. y или. xyz.
-   для m3x2 требуется маска записи. XY.
-   для m3x3 и m4x3 требуется маска записи. xyz.
-   m3x4 и m4x4 должны иметь маску записи. XYZ или маску записи по умолчанию (ксизв).

## <a name="texture-register-masks"></a>Маски регистров текстур

Правила проверки использования модификаторов в регистрах координат текстуры более тесны, чем правила проверки для других регистров.

-   При написании отн все предыдущие регистры (отн-1 ~ oT0) также должны быть записаны.
-   "Объединенная" маска записи для любого \# регистра объекта должна быть в точности одним из следующих:
    -   . x
    -   . XY
    -   . XYZ
    -   . ксизв (эквивалентно не использованию модификаторов компонентов)

Например, шейдер вершин может выводить данные в регистры текстур в отдельных инструкциях.


```
    oT1.y  
    oT0.y  
    oT2  
    oT0.xz  
    oT1.x
```



Инструкции можно также объединить.


```
    oT0.xyz  
    oT1.xy  
    oT2.xyzw    
```



Эти ограничения применяются только к регистрам координат текстуры.
