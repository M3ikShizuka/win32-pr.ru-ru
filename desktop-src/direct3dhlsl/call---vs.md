# <a name="call---vs"></a>вызов — VS

Выполняет вызов функции в инструкцию, помеченную указанной меткой.

## <a name="syntax"></a>Синтаксис



| вызов l\# |
|----------|



 

где l \# — [Метка — VS](label---vs.md) отмечает начало подпрограммы, которую необходимо вызвать.

## <a name="remarks"></a>Примечания



| Версии шейдеров вершин | 1\_1 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|------------------------|------|------|------|-------|------|-------|
| вызывает                   |      | x    | x    | x     | x    | x     |



 

Эта инструкция выполняет следующие действия:

1.  Адрес принудительной отправки следующей инструкции в стек адресов возврата.
2.  Продолжить выполнение из инструкции, помеченной меткой.

В шейдере вершин 2 \_ 0 вызовы вложений запрещены.

В шейдере вершин 2 \_ x глубина вложения ограничена элементом статикфловконтролдепс структуры [**D3DVSHADERCAPS2 \_ 0**](/windows/desktop/api/d3d9caps/ns-d3d9caps-d3dvshadercaps2_0) . Дополнительные сведения см. в разделе [**жетдевицекапс**](/windows/desktop/api/d3d9/nf-d3d9-idirect3d9-getdevicecaps).

В шейдере вершин 3 \_ 0 разрешено использовать четыре уровня вложенности вызовов.

Допускаются только прямые вызовы. Это означает, что расположение метки внутри шейдера вершин должно быть после инструкции вызова, ссылающейся на него.

Если инструкция вызова вызывается внутри [цикла](loop---vs.md)... блок [ендлуп](endloop---vs.md) . значение [регистра счетчика циклов](dx9-graphics-reference-asm-vs-registers-loop-counter.md) (Al) доступно внутри подпрограммы.

Если подпрограмма ссылается на [регистр счетчиков циклов](dx9-graphics-reference-asm-vs-registers-loop-counter.md) (Al), расположенный за пределами подпрограммы, каждый экземпляр вызова этой подпрограммы должен быть заключен в [цикл](loop---vs.md)... блок [ендлуп](endloop---vs.md) .
