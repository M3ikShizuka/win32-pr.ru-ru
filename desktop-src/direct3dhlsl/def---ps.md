# <a name="def---ps"></a>DEF-PS

Определяет константы с плавающей точкой шейдера пикселей.

## <a name="syntax"></a>Синтаксис



| DEF, fVvalue1, fValue2, fValue3, fValue4 |
|----------------------------------------------|



 

Где:

-   DST — это регистр назначения.
-   fValue1 fValue4 — это значения с плавающей запятой.

## <a name="remarks"></a>Remarks



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| def                   | x    | x    | x    | x    | x    | x    | x     | x    | x     |



 

Существует два способа задания константы с плавающей запятой в построителе пикселей.

1.  Используйте DEF для определения константы непосредственно внутри шейдера.
2.  Используйте API, чтобы задать константу с помощью [**сетпикселшадерконстантф**](/windows/desktop/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-setpixelshaderconstantf).

DEF определяет константу шейдера, значение которой загружается каждый раз, когда шейдер задается устройством. Они называются прямыми константами. Немедленные константы имеют приоритет над константами, заданными методом API.

-   Должен располагаться перед первой арифметической или инструкцией адресации в шейдере.
-   Можно смешивать с инструкциями [дкл-(SM2, SM3-PS ASM)](dcl---ps.md) (другими типами инструкций, которые находятся в начале шейдера).
-   Регистр летнего времени должен быть [регистром констант](dx9-graphics-reference-asm-ps-registers-ps-1-x.md).
-   Маска записи должна быть заполнена (по умолчанию).
-   Если [регистр константы](dx9-graphics-reference-asm-ps-registers-ps-1-x.md) определен несколько раз в шейдере, последний сохраняется.
