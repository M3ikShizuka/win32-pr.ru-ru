# <a name="ld_raw-sm5---asm"></a>LD \_ необработанных (SM5-ASM)

Чтение произвольного доступа с 1-4 32-разрядных компонентов из необработанного буфера.



| LD \_ необработанных dst0 \[ . Mask \] , сркбитеоффсет \[ . Select \_ компонент \] , src0 \[ . свиззле\] |
|------------------------------------------------------------------------------|



 



| Элемент                                                                                                                       | Описание                                                   |
|----------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------|
| <span id="dst0"></span><span id="DST0"></span>*dst0*<br/>                                                            | \[в \] адресе результата операции.<br/> |
| <span id="srcByteOffset"></span><span id="srcbyteoffset"></span><span id="SRCBYTEOFFSET"></span>*сркбитеоффсет*<br/> | \[в \] задает смещение для чтения.<br/>          |
| <span id="src0"></span><span id="SRC0"></span>*src0*<br/>                                                            | \[в \] компоненте для чтения. <br/>                     |



 

## <a name="remarks"></a>Примечания

*src0* должен быть:

-   Любой этап шейдера: SRV (t \# ) LD St
-   Вычисление шейдера или шейдера пикселей: UAV (u \# )
-   Шейдер вычислений: Общая память группы потоков (g \# )

*сркбитеоффсет* задает базовое 32-разрядное значение в памяти для окна из 4 последовательных 32-разрядных значений, в которых данные могут быть считаны, в зависимости от свиззле и маски на других параметрах.

Данные, считанные из необработанного буфера, эквивалентны следующему псевдокоду: где у нас есть смещение, адрес, указатель на содержимое буфера, шаг источника и данные, хранимые линейно.

``` syntax
                    BYTE *BufferContents;         // from src0
                    UINT srcByteOffset;           // from srcRegister
                    BYTE *ReadLocation;           // value to calculate
                    ReadLocation = BufferContents 
                                + srcByteOffset;

                    UINT32 Temp[4];  // used to make code shorter

                    // apply the source resource swizzle on source data
                    Temp = read_and_swizzle(ReadLocation, srcSwizzle);

                    // write the components to the output based on mask
                    ApplyWriteMask(dstRegister, dstWriteMask, Temp);
```

Недопустимые адреса в u \# /t для \# любого заданного 32-разрядного компонента возвращают 0 для этого компонента.

Исходящие адреса в g \# (границы этого конкретного g \# , а не всю общую память) для любого заданного 32-разрядного компонента возвращают неопределенный результат.

CS \_ 4 \_ 0 и CS \_ 4 \_ 1 поддерживают эту инструкцию для UAV и SRV.

Эта инструкция применяется к следующим этапам шейдера:



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
| X      | X    | X      | X        | X     | X       |



 

Так как Уавс доступны на всех стадиях шейдера для Direct3D 11,1, эта инструкция применяется ко всем этапам шейдера для Уавс в среде выполнения Direct3D 11,1, которая доступна начиная с Windows 8.



| Вершина | Поверхности | Домен | Геометрия | Пиксель | Вычисления |
|--------|------|--------|----------|-------|---------|
| X      | X    | X      | X        | X     | X       |
