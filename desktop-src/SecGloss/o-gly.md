<dl> <dt>

<span id="_security_object_identifier_gly"></span><span id="_SECURITY_OBJECT_IDENTIFIER_GLY"></span>**Идентификатор объекта**
</dt> <dd>

КОДА Число, уникально идентифицирующее класс объекта или атрибут. Идентификатор объекта представляется в виде точечно-десятичной строки, например 1.2.3.4. Идентификаторы объектов организованы в глобальную иерархию. Национальные органы регистрации выдают корневые идентификаторы объектов отдельным лицам или организациям, которые управляют иерархией, лежащей ниже своего корневого идентификатора объекта.

См. также [*абстрактный синтаксис нотации 1*](a-gly.md), [*Attribute*](a-gly.md).

</dd> <dt>

<span id="_security_ocsp_gly"></span><span id="_SECURITY_OCSP_GLY"></span>**ВЫПОЛНЕН**
</dt> <dd>

См. раздел *протокол состояния сертификатов в сети*.

</dd> <dt>

<span id="_security_ofb_gly"></span><span id="_SECURITY_OFB_GLY"></span>**офб**
</dt> <dd>

См. *отзыв о выходе*.

</dd> <dt>

<span id="_security_oid_gly"></span><span id="_SECURITY_OID_GLY"></span>**КОДА**
</dt> <dd>

См. *идентификатор объекта*.

</dd> <dt>

<span id="_security_online_certificate_status_protocol_gly"></span><span id="_SECURITY_ONLINE_CERTIFICATE_STATUS_PROTOCOL_GLY"></span>**протокол состояния сертификатов в сети**
</dt> <dd>

ВЫПОЛНЕН Протокол Интернета, используемый для получения состояния отзыва цифрового сертификата X. 509.

</dd> <dt>

<span id="_security_opaque_blob_gly"></span><span id="_SECURITY_OPAQUE_BLOB_GLY"></span>**непрозрачный BLOB-объект**
</dt> <dd>

Последовательность байтов, используемая для хранения ключей сеанса. Непрозрачные BLOB-объекты содержат материал базового ключа и все сведения о текущем состоянии. Сюда входят такие сведения, как расширяющее значение, вектор инициализации и таблица ключей. Формат непрозрачных больших двоичных объектов не опубликован. Каждый поставщик CSP определяет собственный формат больших двоичных объектов, который должен включать шифрование непрозрачных больших двоичных объектов с помощью какого-либо симметричного ключа.

</dd> <dt>

<span id="_security_outer_content_gly"></span><span id="_SECURITY_OUTER_CONTENT_GLY"></span>**внешнее содержимое**
</dt> <dd>

Улучшения для некоторых инкапсулированных данных. Этот термин используется в основном при обсуждении расширенных данных (внутреннего содержимого) в \# сообщениях PKCS 7.

</dd> <dt>

<span id="_security_output_feedback_gly"></span><span id="_SECURITY_OUTPUT_FEEDBACK_GLY"></span>**Отзыв о выходе**
</dt> <dd>

(ОФБ) Режим блочного шифрования, в котором используется обратная связь, аналогичная режиму обратной связи шифра (CFB). Единственное различие между двумя режимами заключается в том, как заполняется регистр сдвига.

См. также [*отзыв шифра*](c-gly.md).

</dd> </dl>
