# <a name="b-security-glossary"></a>B (глоссарий по безопасности)

[A](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F [G](g-gly.md) [р](h-gly.md) [.](i-gly.md) J [K](k-gly.md) [L](l-gly.md) [M](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [U](u-gly.md) [V](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_backup_authority_gly"></span><span id="_SECURITY_BACKUP_AUTHORITY_GLY"></span>**Центр архивации**
</dt> <dd>

Доверенное приложение, работающее на защищенном компьютере, предоставляющем вторичное хранилище для ключей сеансов своих клиентов. Центр архивации сохраняет ключи сеанса в виде ключевых BLOB-объектов, которые шифруются с помощью открытого ключа резервного центра.

</dd> <dt>

<span id="_security_base_content_type_gly"></span><span id="_SECURITY_BASE_CONTENT_TYPE_GLY"></span>**базовый тип содержимого**
</dt> <dd>

Тип данных, содержащихся в сообщении PKCS \# 7. Базовые типы содержимого содержат только данные без криптографических расширений, таких как хэши или подписи. В настоящее время единственным базовым типом содержимого является тип содержимого данных.

</dd> <dt>

<span id="_security_base_cryptographic_functions_gly"></span><span id="_SECURITY_BASE_CRYPTOGRAPHIC_FUNCTIONS_GLY"></span>**базовые криптографические функции**
</dt> <dd>

Самый низкий уровень функций в архитектуре CryptoAPI. Они используются приложениями и другими высокоуровневые функции CryptoAPI для предоставления доступа к алгоритмам шифрования, предоставляемым CSP, созданию безопасного ключа и защищенному хранению секретов.

См. также [*поставщики служб шифрования*](c-gly.md).

</dd> <dt>

<span id="_security_basic_encoding_rules_gly"></span><span id="_SECURITY_BASIC_ENCODING_RULES_GLY"></span>**Базовые правила кодирования**
</dt> <dd>

ЛИЧЕСТВО Набор правил, используемый для кодирования ASN. 1 определенных данных в поток битов (нулей или единиц) для внешнего хранилища или передачи. Один объект ASN. 1 может иметь несколько эквивалентных закодированных ЛИЧЕСТВО. ЛИЧЕСТВО определяется в рекомендации CCITT X. 209. Это один из двух методов кодирования, используемых в настоящее время интерфейсом CryptoAPI.

</dd> <dt>

<span id="_security_ber_gly"></span><span id="_SECURITY_BER_GLY"></span>**ЛИЧЕСТВО**
</dt> <dd>

См. раздел *основные правила кодирования*.

</dd> <dt>

<span id="_security_big_endian_gly"></span><span id="_SECURITY_BIG_ENDIAN_GLY"></span>**с обратным порядком байтов**
</dt> <dd>

Формат памяти или данных, в котором наиболее значимый байт хранится по нижнему адресу или сначала прибывается.

См. также [*прямой порядок байтов*](l-gly.md).

</dd> <dt>

<span id="_security_blob_gly"></span><span id="_SECURITY_BLOB_GLY"></span>**ОБЪЕКТЕ**
</dt> <dd>

Универсальная последовательность битов, содержащая одну или несколько структур заголовков фиксированной длины и конкретных данных контекста.

См. также [*Ключевые BLOB*](k-gly.md)-объекты, [*Сертификаты BLOB*](c-gly.md)-объектов, [*имена сертификатов*](c-gly.md)и [*BLOB-объекты атрибутов*](a-gly.md).

</dd> <dt>

<span id="_security_block_cipher_gly"></span><span id="_SECURITY_BLOCK_CIPHER_GLY"></span>**блочный шифр**
</dt> <dd>

Алгоритм шифрования, который шифрует данные в дискретных единицах (называемых блоками), а не в виде непрерывного потока битов. Наиболее распространенный размер блока — 64 бит. Например, DES является блочным шифром.

См. также [*Stream Cipher*](s-gly.md).

</dd> <dt>

<span id="_security_bulk_encryption_key_gly"></span><span id="_SECURITY_BULK_ENCRYPTION_KEY_GLY"></span>**ключ для групповой шифрования**
</dt> <dd>

Ключ сеанса, полученный из главного ключа. В шифровании [*SChannel*](s-gly.md) используются ключи с массовым шифрованием.

</dd> </dl>
