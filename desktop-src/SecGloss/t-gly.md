<dl> <dt>

<span id="_security_t_0_protocol_gly"></span><span id="_SECURITY_T_0_PROTOCOL_GLY"></span>**T = 0 протокол**
</dt> <dd>

Асинхронный, ориентированный на символьный протокол передачи данных с половинным дуплексом.

</dd> <dt>

<span id="_security_t_1_protocol_gly"></span><span id="_SECURITY_T_1_PROTOCOL_GLY"></span>**T = 1 протокол**
</dt> <dd>

Асинхронный, ориентированный на блокировку протокол передачи с половинным дуплексом.

</dd> <dt>

<span id="_security_terminal_gly"></span><span id="_SECURITY_TERMINAL_GLY"></span>**терминалов**
</dt> <dd>

Сочетание мониторов, клавиатуры, мыши и совместно расположенных периферийных устройств, таких как устройства чтения смарт-карт. С одним терминалом может быть связано несколько процессов, но только один процесс управляет терминалом в любой конкретный момент времени.

</dd> <dt>

<span id="_security_three_phase_key_exchange_protocol_gly"></span><span id="_SECURITY_THREE_PHASE_KEY_EXCHANGE_PROTOCOL_GLY"></span>**Протокол обмена ключами с тремя этапами**
</dt> <dd>

Протокол, используемый для создания проверенного и зашифрованного соединения между двумя пользователями в незащищенной сети. Пользователи обмениваются набором сообщений для согласования пары ключей шифрования. Один ключ используется отправителем для шифрования сообщений, отправленных получателю, а другой используется получателем для шифрования сообщений, отправляемых отправителю. Этот протокол гарантирует, что оба пользователя активны и отправляют сообщения непосредственно друг другу.

</dd> <dt>

<span id="_security_thumbprint_gly"></span><span id="_SECURITY_THUMBPRINT_GLY"></span>**отпечатк**
</dt> <dd>

Хэш [*SHA-1*](s-gly.md) сертификата.

</dd> <dt>

<span id="_security_tls_gly"></span><span id="_SECURITY_TLS_GLY"></span>**АУТЕНТИФИЦИРОВАН**
</dt> <dd>

См. раздел *протокол безопасности транспортного уровня*.

</dd> <dt>

<span id="_security_token_gly"></span><span id="_SECURITY_TOKEN_GLY"></span>**Лекс**
</dt> <dd>

См. раздел [*маркер доступа*](a-gly.md).

</dd> <dt>

<span id="_security_transaction_gly"></span><span id="_SECURITY_TRANSACTION_GLY"></span>**операции**
</dt> <dd>

Операция, позволяющая выполнять несколько действий в рамках одной операции. Промежуточные шаги фактически не выполняются до тех пор, пока все взаимодействия не будут выполнены успешно. В случае сбоя любого взаимодействия все шаги возвращаются к исходному параметру.

</dd> <dt>

<span id="_security_transport_layer_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_GLY"></span>**транспортный уровень**
</dt> <dd>

Сетевой уровень, отвечающий за качество обслуживания и доставку данных без ошибок. К задачам, выполняемым на этом уровне, относятся обнаружение и коррекция ошибок.

</dd> <dt>

<span id="_security_transport_layer_security_protocol_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_SECURITY_PROTOCOL_GLY"></span>**Протокол безопасности транспортного уровня**
</dt> <dd>

АУТЕНТИФИЦИРОВАН Протокол, обеспечивающий конфиденциальность и безопасность обмена данными между двумя приложениями, взаимодействующими по сети.

</dd> <dt>

<span id="_security_triple_des_gly"></span><span id="_SECURITY_TRIPLE_DES_GLY"></span>**тройной алгоритм DES**
</dt> <dd>

Разновидность алгоритма блочного шифра DES, которая шифрует обычный текст с помощью одного ключа, шифрует полученный текст с помощью второго ключа и, наконец, шифрует результат второго шифрования с помощью третьего ключа. Triple DES — это симметричный алгоритм, использующий тот же алгоритм и ключи для шифрования и расшифровки.

</dd> <dt>

<span id="_security_trustee_gly"></span><span id="_SECURITY_TRUSTEE_GLY"></span>**доверенное лицо**
</dt> <dd>

Учетная запись пользователя, учетная запись группы или сеанс входа в систему, к которым применяется запись управления доступом (ACE). Каждый элемент ACE в списке управления доступом (ACL) применяется к одному доверенному лицу.

См. также [*запись управления доступом*](a-gly.md) и [*список управления доступом*](a-gly.md).

</dd> <dt>

<span id="_security_trust_list_gly"></span><span id="_SECURITY_TRUST_LIST_GLY"></span>**список доверия**
</dt> <dd>

См. [*список доверия сертификатов*](c-gly.md).

</dd> <dt>

<span id="_security_trust_provider_gly"></span><span id="_SECURITY_TRUST_PROVIDER_GLY"></span>**поставщик доверия**
</dt> <dd>

Программное обеспечение, определяющее, можно ли доверять заданному файлу. Решение принимается на основе сертификата, связанного с файлом.

</dd> </dl>
