<span id="_security_data_encryption_standard_gly"></span><span id="_SECURITY_DATA_ENCRYPTION_STANDARD_GLY"></span>**Стандарт шифрования данных**
</dt> <dd>

3DES Блочный шифр, который шифрует данные в 64-битных блоках. DES — это симметричный алгоритм, использующий тот же алгоритм и ключ для шифрования и расшифровки.

Разработанный на ранних 1970-х, DES также известен как Деа (алгоритм шифрования данных) ANSI и ДЕА-1 по ISO.

</dd> <dt>

<span id="_security_datagram_gly"></span><span id="_SECURITY_DATAGRAM_GLY"></span>**датаграммы**
</dt> <dd>

Коммуникационный канал, использующий данные, направляемые через сеть коммутации пакетов. Эти сведения включают отдельные пакеты информации и сведения о доставке, связанные с этими пакетами, например адрес назначения. В сети с коммутацией пакетов пакеты данных направляются независимо друг от друга и могут следовать разным маршрутам. Они также могут поступать в порядке, отличном от того, в котором они были отправлены.

</dd> <dt>

<span id="_security_decoding_gly"></span><span id="_SECURITY_DECODING_GLY"></span>**декодирования**
</dt> <dd>

Процесс перевода закодированного объекта (например, сертификата) или данных обратно в исходный формат.

В общих чертах данные декодированы на уровне кодирования и декодирования протокола связи. Сертификаты декодированы путем вызова функции **криптдекодеобжект** .

</dd> <dt>

<span id="_security_decryption_gly"></span><span id="_SECURITY_DECRYPTION_GLY"></span>**расшифровки**
</dt> <dd>

Процесс преобразования зашифрованного текста в обычный текст. Расшифровка — это противоположность шифрованию.

</dd> <dt>

<span id="_security_default_mode_gly"></span><span id="_SECURITY_DEFAULT_MODE_GLY"></span>**режим по умолчанию**
</dt> <dd>

Параметры по умолчанию, такие как режим шифрования блоков или метод заполнения блочного шифрования.

</dd> <dt>

<span id="_security_der_gly"></span><span id="_SECURITY_DER_GLY"></span>**ФОРМАТЕ**

см. *Distinguished Encoding Rules*.

</dd> <dt>

<span id="_security_derived_key_gly"></span><span id="_SECURITY_DERIVED_KEY_GLY"></span>**производный ключ**
</dt> <dd>

Криптографический ключ, созданный при вызове функции **криптдеривекэй** . Производный ключ можно создать на основе пароля или любых других данных пользователя. Производные ключи позволяют приложениям создавать ключи сеансов по мере необходимости, устраняя необходимость в хранении определенного ключа.

</dd> <dt>

<span id="_security_des_gly"></span><span id="_SECURITY_DES_GLY"></span>**3DES**

<span id="_security_dh_gly"></span><span id="_SECURITY_DH_GLY"></span>**ХЕЛМАНА**

См. раздел *алгоритм Диффи-Хелмана*.

</dd> <dt>

<span id="_security_dh_keyx_gly"></span><span id="_SECURITY_DH_KEYX_GLY"></span>**DH \_ KEYX**
</dt> <dd>

Имя алгоритма CryptoAPI для Diffie-Hellman алгоритма обмена ключами.

См. также *алгоритм Диффи-Хелмана*.

</dd> <dt>

<span id="_security_diffie_hellman_algorithm_gly"></span><span id="_SECURITY_DIFFIE_HELLMAN_ALGORITHM_GLY"></span>**Алгоритм Диффи-Хелмана**
</dt> <dd>

ХЕЛМАНА Алгоритм открытого ключа, используемый для безопасного обмена ключами. Diffie-Hellman нельзя использовать для шифрования данных. Этот алгоритм указывается в качестве алгоритма обмена ключами для \_ \_ типов поставщиков Диффи-Хелмана Prov DSS.

См. также алгоритм обмена ключами *Диффи-Хелмана (хранение и переадресация)* и *алгоритм обмена ключами Диффи-Хелмана (эфемерный)*.

</dd> <dt>

<span id="_security_diffie_hellman_store_and_forward_key_exchange_algorithm_gly"></span><span id="_SECURITY_DIFFIE_HELLMAN_STORE_AND_FORWARD_KEY_EXCHANGE_ALGORITHM_GLY"></span>**Алгоритм обмена ключами Диффи-Хелмана (хранение и переадресация)**
</dt> <dd>

Алгоритм Diffie-Hellman, в котором сохраняются значения ключа обмена (в CSP) после уничтожения маркера ключа.

См. также *алгоритм обмена ключами Диффи-Хелмана (эфемерный)*.

</dd> <dt>

<span id="_security_diffie_hellman_ephemeral_key_exchange_algorithm_gly"></span><span id="_SECURITY_DIFFIE_HELLMAN_EPHEMERAL_KEY_EXCHANGE_ALGORITHM_GLY"></span>**Алгоритм обмена ключами Диффи-Хелмана (эфемерные)**
</dt> <dd>

Алгоритм Diffie-Hellman, в котором значение ключа обмена удаляется из CSP при уничтожении маркера ключа.

См. также *алгоритм обмена ключами Диффи-Хелмана (хранение и переадресация)*.

</dd> <dt>

<span id="_security_digested_data_gly"></span><span id="_SECURITY_DIGESTED_DATA_GLY"></span>**Дайджест-данные**
</dt> <dd>

Тип содержимого данных, определенный в PKCS \# 7 и состоящий из любого типа данных и хэш-кода сообщения (дайджест) содержимого.

</dd> <dt>

<span id="_security_digital_certificate_gly"></span><span id="_SECURITY_DIGITAL_CERTIFICATE_GLY"></span>**цифровой сертификат**
</dt> <dd>

См. [*сертификат*](c-gly.md).

</dd> <dt>

<span id="_security_digital_envelope_gly"></span><span id="_SECURITY_DIGITAL_ENVELOPE_GLY"></span>**цифровой конверт**
</dt> <dd>

Частные сообщения, зашифрованные с помощью открытого ключа получателя. Запечатанные сообщения могут быть расшифрованы только с помощью закрытого ключа получателя, что позволяет только пользователю понять сообщение.

</dd> <dt>

<span id="_security_digital_signature_gly"></span><span id="_SECURITY_DIGITAL_SIGNATURE_GLY"></span>**Цифровая подпись**
</dt> <dd>

Данные, связывающие личность отправителя с передаваемой информацией. Цифровая сигнатура может включаться в сообщение, файл или другой блок информации, представленной в цифровом виде, либо передаваться отдельно. Цифровые сигнатуры используются в средах открытых ключей и предоставляют возможность проверки подлинности и целостности данных.

</dd> <dt>

<span id="_security_digital_signature_algorithm_gly"></span><span id="_SECURITY_DIGITAL_SIGNATURE_ALGORITHM_GLY"></span>**Алгоритм цифровых подписей**
</dt> <dd>

NИСХОДНЫЙ Алгоритм открытого ключа, заданный стандартом цифровой подписи (DSS). DSA используется только для создания цифровых подписей. Его нельзя использовать для шифрования данных.

</dd> <dt>

<span id="_security_digital_signature_key_pair_gly"></span><span id="_SECURITY_DIGITAL_SIGNATURE_KEY_PAIR_GLY"></span>**пара ключей цифровых подписей**
</dt> <dd>

См. [*раздел пара ключей подписи*](s-gly.md).

</dd> <dt>

<span id="_security_digital_signature_standard_gly"></span><span id="_SECURITY_DIGITAL_SIGNATURE_STANDARD_GLY"></span>**Стандарт цифровой подписи**
</dt> <dd>

DSS Стандарт, указывающий алгоритм цифровой подписи (DSA) для своего алгоритма подписи и SHA-1 в качестве хэш-алгоритма сообщения. DSA — это шифр открытого ключа, который используется только для создания цифровых подписей и не может использоваться для шифрования данных. DSS определяется \_ типами поставщиков Prov DSS, Prov \_ DSS \_ DH и Prov \_ Fortezza.

</dd> <dt>

<span id="_security_discretionary_access_control_list_gly"></span><span id="_SECURITY_DISCRETIONARY_ACCESS_CONTROL_LIST_GLY"></span>**список управления доступом на уровне пользователей**
</dt> <dd>

КОНТРОЛЯ Список управления доступом, который управляется владельцем объекта и указывает, какие пользователи или группы могут иметь доступ к объекту.

См. также список [*управления доступом*](a-gly.md) и [*системный список управления доступом*](s-gly.md).

</dd> <dt>

<span id="_security_distinguished_encoding_rules_gly"></span><span id="_SECURITY_DISTINGUISHED_ENCODING_RULES_GLY"></span>**Distinguished Encoding Rules**
</dt> <dd>

ФОРМАТЕ Набор правил для кодирования ASN. 1 определенных данных в виде потока битов для внешнего хранилища или передачи. Каждый объект ASN. 1 имеет только одну соответствующую кодировку DER. DER определяется в рекомендации CCITT X. 509, раздел 8,7. Это один из двух методов кодирования, используемых в настоящее время интерфейсом CryptoAPI.

</dd> <dt>

<span id="_security_dll_gly"></span><span id="_SECURITY_DLL_GLY"></span>**КОМПОНОВКИ**

<span id="_security_dss_gly"></span><span id="_SECURITY_DSS_GLY"></span>**DSS**

<span id="_security_dynamic_access_control_gly"></span><span id="_SECURITY_DYNAMIC_ACCESS_CONTROL_GLY"></span>**Динамический контроль доступа**

DAC Возможность указывать политики управления доступом на основе утверждений пользователей, устройств и ресурсов. Это обеспечивает более гибкую проверку подлинности для приложений, сохраняя требования к безопасности и соответствию.
