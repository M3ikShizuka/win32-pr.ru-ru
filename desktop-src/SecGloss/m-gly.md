<dl> <dt>

<span id="_security_mac_gly"></span><span id="_SECURITY_MAC_GLY"></span>**АДРЕС**
</dt> <dd>

См. *код проверки подлинности сообщения*.

</dd> <dt>

<span id="_security_mac_key_gly"></span><span id="_SECURITY_MAC_KEY_GLY"></span>**Ключ MAC**
</dt> <dd>

Ключ кода проверки подлинности сообщения, используемый с протоколами [*SChannel*](s-gly.md) .

</dd> <dt>

<span id="_security_master_key_gly"></span><span id="_SECURITY_MASTER_KEY_GLY"></span>**Главный ключ**
</dt> <dd>

Ключ, используемый клиентом и сервером для всех поколенных ключей сеансов. Главный ключ используется для создания ключа считывания клиента, ключа записи клиента, ключа для чтения с сервера и ключа записи на сервер. Главные ключи можно экспортировать как простые ключевые BLOB-объекты.

</dd> <dt>

<span id="_security_md2_gly"></span><span id="_SECURITY_MD2_GLY"></span>**MD2**
</dt> <dd>

Имя алгоритма CryptoAPI для хэш-алгоритма MD2. Другие алгоритмы хеширования включают *MD4*, *MD5* и [*SHA*](s-gly.md).

См. также *алгоритм MD2*.

</dd> <dt>

<span id="_security_md2_algorithm_gly"></span><span id="_SECURITY_MD2_ALGORITHM_GLY"></span>**Алгоритм MD2**
</dt> <dd>

MD2 Алгоритм хэширования, создающий 128-разрядное хэш-значение. Технологии MD2 оптимизированы для использования с 8-разрядными компьютерами. CryptoAPI ссылается на этот алгоритм по его типу (КАЛГ \_ MD2), имени (Mac) и классу ( \_ хэш класса алгоритма \_ ). Технологии MD2 были разработаны RSA Data Security, Inc.

См. также алгоритм *MD4*, *алгоритм MD5*.

</dd> <dt>

<span id="_security_md4_gly"></span><span id="_SECURITY_MD4_GLY"></span>**MD4**
</dt> <dd>

Имя алгоритма CryptoAPI для хэш-алгоритма MD4. Другие алгоритмы хеширования включают *технологии MD2*, *MD5* и [*SHA*](s-gly.md).

См. раздел *алгоритм MD4*.

</dd> <dt>

<span id="_security_md4_algorithm_gly"></span><span id="_SECURITY_MD4_ALGORITHM_GLY"></span>**Алгоритм MD4**
</dt> <dd>

MD4 Алгоритм хэширования, создающий 128-разрядное хэш-значение. MD4 был оптимизирован для 32-разрядных компьютеров. Теперь она считается неработающей, поскольку конфликты можно найти слишком быстро и легко. MD4 был разработан RSA Data Security, Inc.

См. также *алгоритм MD2*, *алгоритм MD5*.

</dd> <dt>

<span id="_security_md5_gly"></span><span id="_SECURITY_MD5_GLY"></span>**MD5**
</dt> <dd>

Имя алгоритма CryptoAPI для хэш-алгоритма MD5. Другие алгоритмы хеширования включают *технологии MD2*, *MD4* и [*SHA*](s-gly.md).

См. также *алгоритм MD5*.

</dd> <dt>

<span id="_security_md5_algorithm_gly"></span><span id="_SECURITY_MD5_ALGORITHM_GLY"></span>**Алгоритм MD5**
</dt> <dd>

MD5 Алгоритм хэширования, создающий 128-разрядное хэш-значение. MD5 оптимизирован для 32-разрядных компьютеров. CryptoAPI ссылается на этот алгоритм по идентификатору алгоритма (КАЛГ \_ MD5), имени (MD5) и классу ( \_ хэш класса алгоритма \_ ). MD5 был разработан RSA Data Security, Inc. и указывается с помощью PROV \_ RSA \_ Full, Prov \_ RSA \_ SIG, Prov \_ DSS, Prov \_ DSS \_ DH и Prov \_ MS \_ Exchange Types.

См. также *алгоритм MD2*, *алгоритм MD4*.

</dd> <dt>

<span id="_security_message_gly"></span><span id="_SECURITY_MESSAGE_GLY"></span>**Сообщение**
</dt> <dd>

Любые данные, закодированные для передачи или получения от лица или сущности. Сообщения могут быть зашифрованы для обеспечения конфиденциальности, цифровых подписей в целях проверки подлинности или и того и другого.

</dd> <dt>

<span id="_security_message_digest_gly"></span><span id="_SECURITY_MESSAGE_DIGEST_GLY"></span>**дайджест сообщения**
</dt> <dd>

См. раздел [*hash*](h-gly.md).

</dd> <dt>

<span id="_security_message_authentication_code_gly"></span><span id="_SECURITY_MESSAGE_AUTHENTICATION_CODE_GLY"></span>**код проверки подлинности сообщения**
</dt> <dd>

АДРЕС Алгоритм хэширования с ключом, использующий симметричный ключ сеанса, чтобы обеспечить сохранность блока данных с момента отправки до момента получения. При использовании этого типа алгоритма принимающее приложение также должно обладать ключом сеанса для повторного вычисления хэш-значения, чтобы можно было проверить, что базовые данные не изменились. CryptoAPI ссылается на этот алгоритм по его типу (КАЛГ \_ Mac), имени (Mac) и классу ( \_ хэш класса алгоритма \_ ).

</dd> <dt>

<span id="_security_message_encoding_type_gly"></span><span id="_SECURITY_MESSAGE_ENCODING_TYPE_GLY"></span>**тип кодирования сообщений**
</dt> <dd>

Определяет, как кодируется сообщение. Тип кодирования сообщений хранится в строгом слове структуры типа кодировки. Текущие определенные типы кодировки: шифрование \_ ASN \_ , кодировка X509 \_ ASN \_ и кодирование PKCS \_ 7 \_ ASN \_ .

</dd> <dt>

<span id="_security_message_management_functions_gly"></span><span id="_SECURITY_MESSAGE_MANAGEMENT_FUNCTIONS_GLY"></span>**функции управления сообщениями**
</dt> <dd>

Функции, обеспечивающие два уровня управления сообщениями: низкоуровневые функции сообщений и упрощенные функции сообщений. Низкоуровневые функции сообщений обеспечивают большую гибкость, чем упрощенные функции сообщений. Однако для них требуется больше вызовов функций.

</dd> <dt>

<span id="_security_message_signing_functions_gly"></span><span id="_SECURITY_MESSAGE_SIGNING_FUNCTIONS_GLY"></span>**функции подписания сообщений**
</dt> <dd>

Функции, используемые для подписи сообщений и данных.

См. также [*упрощенные функции сообщений*](s-gly.md).

</dd> <dt>

<span id="_security_mpr_gly"></span><span id="_SECURITY_MPR_GLY"></span>**MPR**
</dt> <dd>

См. раздел *маршрутизатор с несколькими поставщиками*.

</dd> <dt>

<span id="_security_multiple_provider_router_gly"></span><span id="_SECURITY_MULTIPLE_PROVIDER_ROUTER_GLY"></span>**Маршрутизатор с несколькими поставщиками**
</dt> <dd>

MPR Системный компонент, который обрабатывает обмен данными между системой и всеми поставщиками сети. Многопротокольный маршрутизатор вызывает функции поставщика сети, предоставляемые каждым поставщиком сети.

</dd> </dl>
