# <a name="about-imftransform"></a>О Имфтрансформ

Интерфейс **имфтрансформ** — это один из интерфейсов, который должен быть реализован подключаемым модулем DSP с двойным режимом. проигрыватель Windows Media вызывает методы интерфейса **имфтрансформ** , чтобы предоставить данные подключаемому модулю и получить обработанные данные обратно из подключаемого модуля. полную документацию по интерфейсу **имфтрансформ** см. в разделе Media Foundation Windows SDK.

Методы **имфтрансформ** можно классифицировать следующим образом.

## <a name="methods-that-handle-format-negotiation"></a>Методы, обрабатывающие согласование формата

проигрыватель Windows Media вызывает следующие методы для получения сведений о форматах данных, поддерживаемых подключаемым модулем.

-   **жетстреамкаунт**
-   **жетстреамлимитс**
-   **жетинпутстреаминфо**
-   **жетаутпутстреаминфо**
-   **жетинпутаваилаблетипе**
-   **жетаутпутаваилаблетипе**
-   **сетинпуттипе**
-   **сетаутпуттипе**
-   **GetAttributes**
-   **жетинпутстреаматтрибутес**
-   **жетаутпутстреаматтрибутес**
-   **жетстреамидс**

## <a name="methods-that-specify-or-retrieve-state-information"></a>Методы, указывающие или получающие сведения о состоянии

проигрыватель Windows Media вызывает следующие методы для получения или задания значений, связанных с текущим состоянием подключаемого модуля.

-   **сетинпуттипе**
-   **сетаутпуттипе**
-   **жетинпуткурренттипе**
-   **жетаутпуткурренттипе**
-   **жетинпутстатус**
-   **аддинпутстреамс**
-   **делетеинпутстреамс**
-   **жетаутпутстатус**
-   **сетаутпутбаундс**

> [!Note]  
> **Сетинпуттипе** и **сетаутпуттипе** используются как для согласования формата, так и для указания и получения сведений о состоянии.

 

## <a name="methods-that-handle-buffering-and-processing-data"></a>Методы, обрабатывающие буферизацию и обработку данных

проигрыватель Windows Media вызывает следующие методы для инициации различных процессов, которые выполняет подключаемый модуль для обработки цифровых сигналов.

-   **ProcessInput**
-   **ProcessOutput**
-   **ProcessMessage**
-   **процессевент**
