# <a name="rich-media-streaming"></a>Мультимедийная потоковая передача мультимедиа

Сложные веб-страницы содержат множество различных компонентов, которые обычно передаются отдельно. При низком подключении такие страницы отображаются по одному элементу за раз и может занять несколько минут для полного вывода. Это может помешать эффективно синхронизировать веб-страницы с цифровым носителем. Решением этой проблемы является мультимедийная потоковая передача мультимедиа, что означает добавление веб-страниц в поток цифрового мультимедиа, чтобы они доставлялись одновременно с аудио-или видеоданными.

Насыщенная потоковая передача мультимедиа использует простой макет набора фреймов и работает точно так же, как обычное переключение URL-адресов. Как и при отражении URL-адресов, команды сценария URL-адреса, внедренные в цифровые мультимедийные потоки, используются для отображения содержимого в целевом кадре. однако вместо того, чтобы указывать на страницы в интернете, команды сценария URL-адреса используются в проигрыватель Windows Media 9 Series или более поздней версии для доступа к файлам в кэше Internet Explorer, где помещается потоковая передача содержимого HTML по мере их получения. как и при обычной перезаписи URL-адресов, можно написать обработчики событий, которые отвечают на эти команды сценария URL-адреса, или позволить проигрыватель Windows Media элемент управления обрабатывать все автоматически.

> [!Note]  
> Любое содержимое HTML, доставленное через мультимедийную потоковую передачу мультимедиа, воспроизводится в зоне Интернет-безопасности и подлежит настройке безопасности браузера пользователя.

 

дополнительные сведения о создании мультимедийных презентаций см. в статье Windows media Format 9 серии или более поздней версии пакета sdk для Windows media Encoder 9.
