# <a name="banner-element"></a>Элемент БАННЕРа

Элемент **баннера** определяет URL-адрес графического файла, который будет отображаться на панели отображения.

``` syntax
<BANNER
   HREF = "URL"
>
</BANNER>
```

## <a name="attributes"></a>Атрибуты

**Href** (обязательно)

URL-адрес графического файла, отображаемого на панели отображения.

## <a name="parentchild-elements"></a>Родительские и дочерние элементы



| Иерархия       | Элементы                   |
|-----------------|----------------------------|
| Родительские элементы | **ASX**, **запись**         |
| Дочерние элементы  | **abstract**, **мореинфо** |



 

## <a name="remarks"></a>Комментарии

этот элемент определяет URL-адрес графического файла, который отображается на панели отображения проигрыватель Windows Media под содержимым видео. Если носитель имеет только аудио, отображается баннер. проигрыватель Windows Media резервирует пространство 32 пикселей высотой в ширину 194 пикселей (заголовок) для графика. Если рисунок, определенный в URL-адресе, меньше указанного, он отображается в исходном размере. если изображение больше, чем зарезервированное, проигрыватель Windows Media будет обрезать изображение в соответствии с пространством.

Можно использовать **абстрактный** элемент в области элемента **баннера** для отображения текста в виде подсказки, когда пользователь приостанавливает указатель мыши над изображением баннера. Элемент **мореинфо** внутри элемента **баннера** определяет URL-адрес, на который пользователь выполняет, когда пользователь щелкает рисунок баннера. (URL-адрес может быть любым путем или протоколом, например ссылкой на электронную почту, URL-адресом http для веб-сайта или даже командой Microsoft JScript). Когда указатель перемещается над графиком, он становится рельефным и выглядит как кнопка.

Элемент **баннера** , определенный для элемента **ASX** , отображается во время воспроизведения всех клипов в списке воспроизведения. Элемент **баннера** , определенный в элементе **entry** , отображается только во время воспроизведения клипа, и в течение этого времени все баннеры, определенные в родительском элементе **ASX** , переопределяются. можно указать, как проигрыватель Windows Media резервирует пространство для баннера, установив атрибут **баннербар** элемента **ASX** .

изображения с баннерами не поддерживаются в файлах DRM или при внедрении проигрыватель Windows Media на веб-страницу.

## <a name="examples"></a>Примеры

Ниже приведен пример элемента **баннера** без дочерних элементов.


```XML
<BANNER HREF="https://sample.microsoft.com/art/banner1.bmp" />
```



Ниже приведен пример элемента **баннера** , содержащего **абстрактные** и **мореинфо** элементы.


```XML
<BANNER HREF="https://www.proseware.com/logos/banner1.bmp">
    <ABSTRACT>Click here to go to our website.</ABSTRACT>
    <MOREINFO HREF="https://sample.microsoft.com" />
    <!-- The text in the Abstract element displays as a 
         ToolTip when the mouse hovers over the banner 
         graphic. When a user clicks the banner, the URL 
         given in the MoreInfo element opens in the 
         browser. -->
</BANNER>
```
