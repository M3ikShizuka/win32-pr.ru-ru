# <a name="playermousemove-event"></a>Событие Player. MouseMove

Событие **MouseMove** возникает при перемещении указателя мыши.

## <a name="syntax"></a>Синтаксис


```JScript
Player.MouseMove(
  nButton,
  nShiftState,
  fX,
  fY
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*нбуттон* 
</dt> <dd>

**Число** (**int**), задающее битовую глубину, соответствующую левой кнопке (бит 0), правой кнопке (бит 1) и средней кнопке (бит 2). Эти биты соответствуют значениям 1, 2 и 4 соответственно. Могут быть заданы некоторые, все или никакие биты, указывающие на то, что нажаты некоторые, все или ни одна из кнопок.

</dd> <dt>

*ншифтстате* 
</dt> <dd>

**Число** (**int**), задающее битовую глубину, которая соответствует клавише Shift (бит 0), клавиша Ctrl (бит 1) и клавиша Alt (бит 2). Эти биты соответствуют значениям 1, 2 и 4 соответственно. Аргумент shift указывает состояние этих ключей. Можно установить некоторые, все или ни одного бита, указывая, что не нажаты некоторые, все или ни один из этих ключей.

</dd> <dt>

*fX* 
</dt> <dd>

**Число** (**длинное**), определяющее координату x указателя мыши относительно левого верхнего угла элемента управления.

</dd> <dt>

*Обозначения* 
</dt> <dd>

**Число** (**длинное**), указывающее координату y указателя мыши относительно левого верхнего угла элемента управления.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Это событие не возвращает значение.

## <a name="remarks"></a>Remarks

значение параметров события задается проигрыватель Windows Media, и к нему можно получить доступ или передать в метод в импортированном JScriptном файле, используя имя параметра. Имя этого параметра должно быть введено в точности так, как показано, включая прописные буквы.

**проигрыватель Windows Media 10 Mobile:** Это событие не поддерживается.
