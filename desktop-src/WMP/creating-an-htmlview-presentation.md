# <a name="creating-an-htmlview-presentation"></a>Создание презентации Хтмлвиев

Чтобы создать базовую Хтмлвиев презентацию, требуется по крайней мере три элемента:

-   **Мультимедийное содержимое.** Это один или несколько звуковых или видеофайлов, которые воспроизводит проигрыватель Windows Media.
-   **Веб-страница.** Это веб-содержимое, которое отображается в **воспроизводимой** в данный момент функции пользовательского интерфейса проигрывателя.
-   **Метафайл Windows Media.** Это список воспроизведения метафайлов, который указывает проигрывателю Windows Media объединить мультимедийное содержимое с веб-страницей.

ASX-файл — это текстовый файл, содержащий сведения о файловом потоке и его представлении. В зависимости от синтаксиса язык XML (XML) ASX-файлы могут содержать множество элементов, каждый из которых определяется тегом с соответствующими атрибутами. Элемент **param** предоставляет способ связать пользовательский параметр с определенной записью в списке воспроизведения метафайла или в целом метафайле. Одно из предопределенных имен параметров, доступных для использования, — "Хтмлвиев". Это параметр, который вызывает отображение веб-страницы, указанной значением URL-адреса, в проигрывателе Windows Media.

В следующем примере кода показан файл ASX, объединяющий один файл мультимедиа с одной веб-страницей:


```XML
<ASX version="3.0">
<PARAM name="HTMLView" value="https://www.proseware.com/htmlview.htm"/>

<ENTRY>
   <REF href="rtsp://www.proseware.com/content1.wma"/>
</ENTRY>

</ASX>

```



Когда проигрыватель Windows Media открывает ASX-файл в предыдущем примере, он воспроизводит звук из файла с именем content1. WMA и открывает веб-страницу с именем htmlview.htm в **воспроизводимой** функции проигрывателя в полноэкранном режиме. Пользователь может приостанавливать, искать и останавливать аудио содержимое с помощью элементов управления проигрывателем Windows Media.

Вы можете легко изменить веб-страницу, отображаемую для каждого фрагмента содержимого, связав элемент **param** с каждой записью, как показано в следующем примере кода:


```XML
<ASX version="3.0">

<ENTRY>
   <PARAM name="HTMLView" value="https://www.proseware.com/htmlview1.htm"/>
   <REF href="rtsp://www.proseware.com/content1.wma"/>
</ENTRY>

<ENTRY>
   <PARAM name="HTMLView" value="https://www.proseware.com/htmlview2.htm"/>
   <REF href="rtsp://www.proseware.com/content2.wma"/>
</ENTRY>

</ASX>

```



В предыдущем примере проигрыватель Windows Media сначала отображает веб-страницу htmlview1.htm при воспроизведении цифрового звукового файла content1. WMA. Когда проигрыватель открывает следующую запись в списке воспроизведения content2. WMA, веб-страница, отображаемая в окне **воспроизведения** , изменяется на htmlview2.htm. Таким образом можно указать, какую веб-страницу будет видеть пользователь для каждого фрагмента мультимедийного содержимого.
