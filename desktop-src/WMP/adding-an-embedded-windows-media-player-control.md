# <a name="adding-an-embedded-windows-media-player-control"></a>добавление встроенного элемента управления проигрыватель Windows Media

существует две причины, по которым можно добавить встроенный экземпляр проигрыватель Windows Media в презентацию хтмлвиев. прежде всего, если требуется отобразить видеосодержимое, необходимо использовать элемент управления проигрыватель Windows Media ActiveX. во-вторых, если вы хотите воспользоваться преимуществами функций объектной модели проигрыватель Windows Media, изнутри страницы хтмлвиев, необходимо использовать для этого экземпляр элемента управления Player.

## <a name="using-the-player-control-to-display-video-in-htmlview-content"></a>Использование элемента управления "проигрыватель" для показа видео в Хтмлвиев содержимом

как правило, проигрыватель Windows Media отображает видео с помощью панели видео и визуализации в **воспроизводимой** функции. Так как Хтмлвиев использует эту область для вывода веб-страницы, необходимо предоставить дополнительную область экрана, если вы хотите, чтобы проигрыватель воспроизводил видео. это легко сделать с помощью элемента управления проигрыватель Windows Media ActiveX.

Чтобы использовать элемент управления проигрывателя для показа видео, внедрите элемент управления на веб-страницу Хтмлвиев с помощью тега **Object** . Это тот же метод, который используется для внедрения элемента управления Player в любую веб-страницу, в которой нужно отобразить видео. В следующем примере кода показан базовый синтаксис для внедрения элемента управления Player в Internet Explorer:


```XML
<OBJECT id = "Player" 
    CLASSID = "CLSID:6BF52A52-394A-11d3-B153-00C04F79FAA6"> 
        <PARAM Name = "autoStart"  Value = "true">
        <PARAM Name = "uiMode" Value = "none">
</OBJECT>

```



Параметр *автозапуска* обеспечивает автоматическое воспроизведение содержимого при указании нового URL-адреса. Значение, заданное для *uiMode* , не зависит от вас, но при создании содержимого для презентаций хтмлвиев обычно требуется указать "нет". при внедрении элемента управления проигрыватель Windows Media для показа видео таким образом пользователь может управлять воспроизведением с помощью элементов управления проигрывателя в полноэкранном режиме, поэтому на веб-странице не нужно предоставлять дополнительные элементы управления для передачи данных. Можно использовать пространство, которое обычно выделяется для элементов управления транспорта, для вывода большего текста, графики или ссылок на другое содержимое.

не указывайте параметр *URL-адреса* при внедрении элемента управления проигрыватель Windows Media в веб-страницу, предназначенную для отображения в хтмлвиев презентации. Вместо этого укажите цифровые файлы мультимедиа в файле ASX, который открывает содержимое.

Так как вы предоставляете область отображения видео на веб-странице Хтмлвиев, вы можете выбрать расположение видео и размер области отображения. Например, можно разместить объект **Player** в элементе HTML **div** , а затем указать расположение элемента **div** , ситуате отображение видео на веб-странице. Можно изменить размеры экрана видео, указав значения атрибутов Height и Width элемента **Object** . Эти значения также можно указать с помощью кода скрипта.

## <a name="using-the-player-object-model"></a>Использование объектной модели проигрывателя

объектная модель проигрыватель Windows Media предоставляет свойства, методы и события, которые можно использовать на веб-страницах хтмлвиев. при внедрении элемента управления проигрыватель Windows Media ActiveX на веб-странице хтмлвиев вы автоматически получите доступ к объектной модели проигрывателя.

если вы внедряете элемент управления проигрыватель Windows Media на веб-странице хтмлвиев, не используйте объектную модель проигрывателя, чтобы указать воспроизводимый мультимедийный файл. Например, если вы используете код скрипта для указания значения свойства **URL-адрес** внедренного элемента управления, веб-страница хтмлвиев будет выгружена из функции **Now** при воспроизведении файла мультимедиа. Чтобы избежать этого, всегда открывать файлы. ASX, которые содержат параметры Хтмлвиев, когда необходимо использовать сценарий для открытия мультимедийного содержимого на веб-странице Хтмлвиев.
