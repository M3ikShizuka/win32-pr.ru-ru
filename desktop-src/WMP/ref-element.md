# <a name="ref-element"></a>REF, элемент

Элемент **ref** указывает URL-адрес для мультимедийного содержимого.

``` syntax
<REF
   HREF = "URL"
>
</REF>
```

## <a name="attributes"></a>Атрибуты

**Href** (обязательно)

URL-адрес любого фрагмента мультимедийного содержимого, поддерживаемого проигрывателем Windows Media.

## <a name="parentchild-elements"></a>Родительские и дочерние элементы



| Иерархия       | Элементы                                                                         |
|-----------------|----------------------------------------------------------------------------------|
| Родительские элементы | **ОПЕРАЦИИ**                                                                        |
| Дочерние элементы  | **DURATION**, **ендмаркер**, **превиевдуратион**, **стартмаркер**, **StartTime** |



 

## <a name="remarks"></a>Комментарии

Этот элемент задает URL-адрес для фрагмента мультимедийного содержимого. URL-адрес может указывать на любой поддерживаемый тип носителя, используя любой протокол, поддерживаемый проигрывателем Windows Media.

Поддерживаемые типы мультимедиа включают изображения в формате GIF и JPG, а также файлы Flash с расширением. SWF. Эти типы носителей полезны для включения рекламного содержимого в список воспроизведения. При использовании файлов изображений и Flash-файлов, которые воспроизводятся в цикле, необходимо также указать количество времени для вывода элемента мультимедиа, включив элемент **DURATION** в элемент **ref** . Если вы хотите, чтобы изображение продолжалось отображать во время буферизации следующей записи в списке воспроизведения, включите элемент **param** в элемент **entry** , задайте для его атрибута **Name** значение шоввхилебуфферинг и установите для атрибута **value значения** true.

Для ссылки на содержимое компакт-диска или DVD-диска, который допускает его, предоставляются протоколы вмпкд и вмпдвд. Например, если задать для атрибута **href** значение «wmpdvd://f/5/3», будет воспроизводится глава 3 из заголовка 5 на DVD, но только в том случае, если DVD-диск был создан для разрешения.

Приложения, которые открывают цифровые мультимедиа из защищенного брандмауэра, будут иметь лучшую производительность при открытии элементов мультимедиа, если адрес указан с использованием имени сервера доменных имен (DNS) вместо IP-адреса.

Наиболее часто этот элемент используется для смены URL-адреса. Если проигрывателю Windows Media не удается открыть часть носителя, определенную в элементе **ref** , он пытается получить URL-адрес в следующем элементе **ref** . Когда проигрыватель Windows Media открывает мультимедийное содержимое из URL-адреса, определенного в области одного элемента **entry** , он игнорирует последующие теги **ref** внутри этого элемента **entry** . После завершения воспроизведения части содержимого проигрыватель Windows Media переходит к следующему элементу **entry** , если он есть.

-   **Важно!** Когда проигрыватель Windows Media устанавливает соединение с фрагментом содержимого, на который ссылается ссылка, он игнорирует все остальные элементы **ref** в этой **записи**, независимо от того, завершается ли соединение нормально или аварийно.

Если элемент мультимедиа, на который указывает ссылка, является файлом изображения, то для указания времени показа изображения должен использоваться элемент **DURATION** .

**Примечание**

Попытка воспроизведения Flash-носителя, который содержит звук с первым кадром, может привести к непредвиденным результатам. Содержимое Flash следует создавать, чтобы воспроизводить звук, начинающийся не раньше, чем второй кадр.

## <a name="examples"></a>Примеры


```XML
<ASX VERSION="3.0">
   <ENTRY>
   <TITLE>Example Clip</TITLE>
      <REF HREF="mms://example.microsoft.com/selection1.asf" />
      <REF HREF="mms://sample.microsoft.com/mirror/selection1.asf" />
   </ENTRY>
</ASX>
```
