# <a name="differences-between-the-object-models"></a>Различия между объектными моделями

Существует два основных различия между объектной моделью проигрывателя Windows Media 6,4 и объектной моделью проигрывателя Windows Media 7 или более поздней версии.

-   **Идентификатор CLSID** Объектная модель проигрывателя Windows Media 7 или более поздней версии — это законченная версия объектной модели версии 6,4. Спецификация модели COM требует, чтобы все существующие интерфейсы продолжали функционировать в новых версиях COM-компонента. Это означает, что новые интерфейсы можно добавить в COM-компонент, но существующие интерфейсы никогда не должны изменяться, гарантируя, что более старый код клиента будет всегда работать с конкретным компонентом, который он должен использовать. Поэтому в проигрывателе Windows Media 7 или более поздней версии элемент управления ActiveX имеет новый идентификатор класса: 6BF52A52-394A-11D3-B153-00C04F79FAA6. Если вы хотите воспользоваться преимуществами новых функций элемента управления для этой версии, необходимо изменить идентификатор CLSID.
-   **Иерархическая объектная модель** Если вы использовали элемент управления ActiveX проигрывателя Windows Media 6,4, вы могли заметить, что доступ ко всем свойствам, методам и событиям осуществляется через один и тот же объект: объект **Player** . В отличие от этого, объектная модель проигрывателя Windows Media 7 или более поздней версии организована как иерархия объектов. Объект **Player** по-прежнему является корневым объектом, но доступ к функциям теперь осуществляется через разнообразные дочерние объекты.
