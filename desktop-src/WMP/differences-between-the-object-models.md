# <a name="differences-between-the-object-models"></a>Различия между объектными моделями

существует два основных различия между объектной моделью проигрыватель Windows Media 6,4 и объектной моделью проигрыватель Windows Media 7 или более поздней.

-   **Идентификатор CLSID** объектная модель проигрыватель Windows Media 7 или более поздней версии — это законченный уход от объектной модели версии 6,4. Спецификация модели COM требует, чтобы все существующие интерфейсы продолжали функционировать в новых версиях COM-компонента. Это означает, что новые интерфейсы можно добавить в COM-компонент, но существующие интерфейсы никогда не должны изменяться, гарантируя, что более старый код клиента будет всегда работать с конкретным компонентом, который он должен использовать. таким образом, элемент управления ActiveX проигрыватель Windows Media 7 или более поздней версии имеет новый идентификатор класса: 6BF52A52-394A-11D3-B153-00C04F79FAA6. Если вы хотите воспользоваться преимуществами новых функций элемента управления для этой версии, необходимо изменить идентификатор CLSID.
-   **Иерархическая объектная модель** если вы используете элемент управления ActiveX проигрыватель Windows Media 6,4, вы могли заметить, что доступ ко всем свойствам, методам и событиям осуществляется через один и тот же объект: объект **Player** . в отличие от этого, объектная модель проигрыватель Windows Media 7 или более поздней версии организована как иерархия объектов. Объект **Player** по-прежнему является корневым объектом, но доступ к функциям теперь осуществляется через разнообразные дочерние объекты.
