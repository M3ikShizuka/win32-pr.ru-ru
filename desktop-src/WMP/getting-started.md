# <a name="getting-started-with-the-windows-media-player-sdk"></a>начало работы с помощью пакета SDK для проигрыватель Windows Media

чтобы создать подключаемый модуль, сначала необходимо установить Microsoft eMbedded Visual C++ 4,0 и Visual Studio 2003. необходимо также установить пакет sdk карманный ПК 2003 или пакет sdk для Smartphone 2003, который является отдельными скачиваниями. чтобы скомпилировать и запустить проект, карманный ПК или смартфон, на котором запущена операционная система Windows Mobile 2003 второго выпуска, необходимо синхронизировать с компьютером разработчика с помощью Microsoft ActiveSync® 3.7.1 или более поздней версии.

поскольку подключаемые модули пользовательского интерфейса на Windows мобильных устройствах используют ту же модель подключаемых модулей, что и настольные версии, вы можете использовать мастер подключаемых модулей проигрыватель Windows Media в качестве отправной точки при создании фонового подключаемого модуля пользовательского интерфейса, который будет работать с проигрыватель Windows Media 10 Mobile.

Чтобы создать код подключаемого модуля пользовательского интерфейса, выполните следующие действия.

1.  откройте Visual Studio 2003 и запустите новый проект в Visual C++.
2.  в области **шаблоны** выберите **проигрыватель Windows Media мастер подключаемых модулей** .
3.  Присвойте проекту имя Нетворкплугин и нажмите кнопку **ОК**.
4.  Выберите **подключаемый модуль пользовательского интерфейса** и нажмите кнопку **Далее**.
5.  Выберите **фон** и нажмите кнопку **Далее**.
6.  Нажмите кнопку **Далее**, чтобы завершить работу мастера. если вы собираетесь отслеживать события с помощью подключаемого модуля, перед завершением работы мастера необходимо проверить **события прослушивания** , а также ознакомиться с документацией по интерфейсу **ивмпевентс** , чтобы узнать, какие события поддерживаются в проигрыватель Windows Media 10 Mobile.

теперь, когда вы создали код подключаемого модуля пользовательского интерфейса, необходимо создать пустой проект Windows CE DLL в eMbedded Visual C++ 4,0. Затем скопируйте созданные мастером исходные файлы в этот новый проект, чтобы они были скомпилированы с помощью компилятора ARM4.

Создание пустого проекта

1.  Запустите новый проект во встроенной Visual C++, а затем выберите **вце Dynamic-Link Library (библиотека** ) в области **проекты** .
2.  Присвойте проекту имя и нажмите кнопку **ОК**.
3.  убедитесь, что выбран **пустой проект Windows CE DLL** , и нажмите кнопку **готово**.
4.  щелкните элемент меню **Project** .
5.  выберите **добавить в Project**, а затем щелкните **файлы**.
6.  Выберите файлы C++, созданные с помощью мастера подключаемых модулей, и нажмите кнопку **ОК**.
