# <a name="simple-art-example"></a>Пример простой иллюстрации

Для создания простой обложки с двумя кнопками необходимы три рисунка. Требуется основной образ и изображение сопоставления, а альтернативное изображение предоставляет пользователю визуальную подсказку о том, что кнопка может быть нажата.

Эти графические файлы были созданы в художественной программе, использующей слои. Использование слоев упрощает обеспечение того, что основной, сопоставленный и альтернативный изображения имеют одинаковый размер и построчно.

Подробные инструкции по созданию картинок см. в статье [Создание обложки](skin-creation-guide.md).

## <a name="primary-image"></a>Основной образ

основной образ — это простой желтый овал с двумя кнопками, розовый один для начала проигрыватель Windows Media и фиолетовые кнопки, чтобы ее отключить. Фон немного темнее, чем овал. Это изображение показано на следующем рисунке.

![основной образ](images/absam01b.png)

Основной образ находился на следующих изображениях, каждый в отдельном слое. Первый овал был создан с эффектом выпуклости и рельефности слоя. Это изображение показано на следующем рисунке.

![изображение овала](images/absam01s.png)

Затем создаются две кнопки, а также эффекты рельефа и рельефа слоя. Это показано на следующем рисунке.

![две кнопки](images/absam01p.png)

После создания фона изображения. Был выбран немного более темный желтый цвет, чтобы не было замечено сглаживание между овалом и фоном. Значение цвета — \# CCCC00. Это изображение показано на следующем рисунке.

![фоновое изображение](images/absam01y.png)

Слои, содержащие эти изображения, были сделаны видимыми и сохранены в виде копии в растровом формате, создавая первичный образ. Основной составной образ будет использоваться атрибутом **backgroundImage** элемента **View** .

## <a name="mapping-image"></a>Изображение сопоставления

Изображение сопоставления необходимо для указания времени и места щелчка обложки. Изображение сопоставления было создано с красной областью и зеленой областью. Это изображение показано на следующем рисунке.

![изображение сопоставления](images/absam01m.png)

зеленая область будет использоваться для того, чтобы обозначить область обложки, которая будет начинаться проигрыватель Windows Media а красная область будет использоваться для ее отмены. Изображение сопоставления имеет тот же размер, что и основной образ.

Изображение сопоставления было создано путем копирования слоя кнопок в новый слой и отключения эффекта рельефа и рельефа. для сопоставления требуются плоские изображения, поскольку проигрыватель Windows Media будет искать отдельные значения цвета в каждой области. Он может искать только определенный цвет, например красный ( \# FF0000). Если изображение имеет багетную рамку или другой результат, не все из них будут полностью красными.

Чтобы запомнить кнопки сопоставления, изображения были заполнены красным и зеленым цветом, но можно использовать любой цвет. Необходимо запомнить цветовые номера на карте, чтобы их можно было указать в файле определения обложки XML. В этом случае красный цвет — \# FF0000, а зеленый — \# 00FF00.

Затем, если виден только новый слой, образ был сохранен как копия в файл BMP. Он будет вызываться атрибутом **маппингимаже** элемента **BUTTONGROUP** .

## <a name="alternate-image"></a>Альтернативное изображение

Альтернативные образы не требуются, но очень полезны для предоставления пользователю визуальных подсказок. В этом случае рекомендуется использовать изображение для наведения, чтобы пользователь знал, на какие области можно щелкнуть.

Был создан альтернативный образ с двумя желтыми кнопками. Это показано на следующем рисунке.

![изображение при наведении](images/absam01h.png)

Альтернативный образ был создан путем копирования исходного слоя кнопки на новый слой и последующего изменения цвета заливки на желтый. Эффект рельефа и выпуклости сохранен. После этого создается новый слой и добавляются изображения: стрелка указывает "Воспроизвести", а квадрат — "останавливаться". Затем, если видны только новые желтые кнопки и слои типа, изображение сохраняется как копия в файл точечного рисунка.

результат заключается в том, что при наведении указателя мыши на область, определенную изображением сопоставления, отображается изображение для наведения, которое предупреждает читателя о том, что, если щелкнуть это место, они могут играть или прекращать проигрыватель Windows Media.

## <a name="xml-code-for-the-art-example"></a>XML-код для примера рисунка

Сведения о том, как писать код XML, приведены в разделе [руководство по созданию обложки](skin-creation-guide.md), но чтобы продемонстрировать, как небольшой код необходим для создания рабочей обложки, здесь приведен код для иллюстрации в этом примере.

Для функций Play и останавливают используются стандартные кнопки. необходимо загрузить файл или список воспроизведения из Windows привязки носителя. когда проигрыватель Windows Media изменяется в режиме обложки, в правом нижнем углу экрана появляется небольшое поле. Это поле называется "привязка". если щелкнуть привязку, вы получите требуемую минимальную функциональность, если обложка не предоставляет способа вернуться в полный режим проигрыватель Windows Media. Пользователь может переключаться между режимами с помощью меню **вид** в полном режиме или путем щелчка привязки в режиме обложки.


```C++
<THEME>
    <VIEW
        clippingColor = "#CCCC00"
        backgroundImage = "background.bmp"
        titleBar = "false">
         
        <BUTTONGROUP
            mappingImage = "map.bmp"
            hoverImage = "hover.bmp">
                
        <PLAYELEMENT
            mappingColor = "#00FF00"/>

        <STOPELEMENT
            mappingColor = "#FF0000"/>
                
        </BUTTONGROUP>
    </VIEW>
</THEME>

```
