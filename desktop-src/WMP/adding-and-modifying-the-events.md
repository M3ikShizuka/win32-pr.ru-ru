# <a name="adding-and-modifying-the-events"></a>Добавление и изменение событий

Необходимо предоставить обработчики событий для \_ событий изменения EN, происходящих при изменении пользователем текста в полях редактирования страницы свойств. Эти обработчики событий имеют простую реализацию, которая просто включает параметр **Применить** в диалоговом окне страницы свойств.

## <a name="modifying-the-scale-factor-event-handler"></a>Изменение обработчика событий коэффициента масштабирования

Необходимо изменить имя существующего обработчика событий, предоставленного мастером подключаемых модулей для поля ввода коэффициента масштабирования. Необходимо изменить имя с Ончанжескале на Ончанжеделай в трех местах:

1.  В Ечопроппаже. h измените имя в разделе макроса схемы сообщения. Замените строку, которая сопоставляет событие изменения коэффициента масштабирования с методом Ончанжескале, следующим кодом:
    ```C++
    COMMAND_HANDLER(IDC_DELAYTIME, EN_CHANGE, OnChangeDelay)
    
    ```

    

2.  В Ечопроппаже. h измените имя в строке, которая прототипет функцию Ончанжескале:
    ```C++
    LRESULT (OnChangeDelay)(WORD wNotifyCode, WORD wID, HWND hWndCtl, BOOL& bHandled);
    
    ```

    

3.  В Ечопроппаже. cpp измените имя в заголовке функции:
    ```C++
    LRESULT CEchoPropPage::OnChangeDelay(WORD wNotifyCode, WORD wID, HWND hWndCtl, BOOL& bHandled)
    
    ```

    

## <a name="adding-the-wet-mix-event-handler"></a>Добавление обработчика событий со смешением

Вы можете легко добавить обработчик событий для \_ события смены языка, присоединенного к \_ элементу управления "поле ввода" IDC ветмикс. В редакторе ресурсов диалогового окна:

1.  Щелкните правой кнопкой мыши \_ поле ввода IDC ветмикс и выберите пункт **события**. Откроется диалоговое окно Новое сообщение Windows и обработчики событий.
2.  В поле **класс или объект для обработчика** щелкните имя ресурса поля редактирования, IDC \_ ветмикс.
3.  В окне **новые сообщения или события Windows** щелкните EN Change (смена языка), \_ чтобы выбрать его.
4.  Нажмите кнопку **Добавить обработчик**. Откроется диалоговое окно Добавление функции члена.
5.  В поле **имя функции члена** введите имя ончанжеветмикс.
6.  Нажмите кнопку **ОК** , чтобы закрыть диалоговое окно Добавление функции члена.
7.  Нажмите кнопку **ОК** , чтобы вернуться в редактор ресурсов диалогового окна.

Visual C++ автоматически добавляет код для схемы сообщений и для функции обработчика событий в Ечопроппаже. h. Код, который он вставляет, содержит комментарий TODO, в котором можно добавить реализацию в заголовок функции. Этот стиль немного отличается от образца кода мастера подключаемого модуля проигрывателя Windows Media, но является приемлемым.

Требуется ли писать реализацию в файле заголовка или переместить ее в Ечопроппаже. cpp. В любом случае для реализации **необходимо включить в** диалоговом окне страницы свойств только одну дополнительную строку кода. Вставьте эту строку кода перед строкой, которая возвращает из функции:


```C++
SetDirty(TRUE);  // Enable Apply.

```
