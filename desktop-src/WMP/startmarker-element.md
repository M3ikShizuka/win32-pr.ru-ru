# <a name="startmarker-element"></a>СТАРТМАРКЕР, элемент

Элемент **стартмаркер** указывает маркер, с помощью которого проигрыватель Windows Media начнет отрисовку потока.

``` syntax
<STARTMARKER
   NUMBER = "marker number"
   NAME = "marker name"
/>
```

## <a name="attributes"></a>Атрибуты

**Нумерация**

Номер числового маркера в индексе. Значение по умолчанию — это начало содержимого по запросу или текущего расположения в потоке в режиме реального времени.

**ИМЯ**

Имя именованного маркера в индексе. Значение по умолчанию — это начало содержимого по запросу или текущего расположения в потоке в режиме реального времени.

## <a name="parentchild-elements"></a>Родительские и дочерние элементы



| Иерархия       | Элементы           |
|-----------------|--------------------|
| Родительские элементы | **entry**, **ref** |
| Дочерние элементы  | Нет               |



 

## <a name="remarks"></a>Remarks

Этот элемент задает маркер, с помощью которого проигрыватель Windows Media начинает отрисовку потока, определенного в родительской **записи** или элементе **ref** .

**Примечание**

Используйте этот элемент с атрибутом **Number** или **Name** , но не обоими.

Элемент **стартмаркер** , определенный внутри **ссылочного** элемента, имеет приоритет над элементом **стартмаркер** , определенным в родительском элементе **entry** элемента **ref** . Элемент **стартмаркер** также имеет приоритет над элементом **StartTime** .

Если маркер, заданный элементом **стартмаркер** , происходит позже в потоке, отличном от маркера, определенного элементом **ендмаркер** , содержимое не будет воспроизводиться, но ошибки не создаются.

## <a name="examples"></a>Примеры


```XML
<STARTMARKER NUMBER="14" />
<STARTMARKER NAME="Marker_StartHere" />
```
