# <a name="entry-element"></a>ENTRY, элемент

элемент **ENTRY** указывает Windowsный файл мультимедиа для отображения в виде клипа.

``` syntax
<ENTRY
   CLIENTSKIP = "YES" | "NO"
   SKIPIFREF = "YES" | "NO"
>
</ENTRY>
```

## <a name="attributes"></a>Атрибуты

**клиентскип**

Значение, указывающее, может ли пользователь пропускать вперед после клипа.

Ниже приведены возможные значения.



| Значение | Описание                                   |
|-------|-----------------------------------------------|
| YES   | По умолчанию. Пользователь может пропускать пересылку после клипа. |
| NO    | Пользователь не может пропускать вперед после клипа.       |



 

**скипифреф**

значение, указывающее, должен ли проигрыватель Windows Media пропустить этот клип, если элемент **ENTRY** включен во второй метафайл с помощью элемента **ентриреф** .

Ниже приведены возможные значения.



| Значение | Описание                                                                                 |
|-------|---------------------------------------------------------------------------------------------|
| YES   | проигрыватель Windows Media будет игнорировать эту запись при ссылке через элемент **ентриреф** . |
| NO    | По умолчанию. проигрыватель Windows Media не будет пропускать эту запись.                                   |

## <a name="remarks"></a>Remarks

этот элемент является фундаментальным конструкцией в метафайле Windows Media. Элемент **entry** и связанные с ним атрибуты определяют метаданные для отдельного логического фрагмента содержимого, называемого клипом. дочерние элементы в области элемента **ENTRY** определяют мультимедийное содержимое для проигрыватель Windows Media открытия (**REF**), сведения о клипе, который проигрыватель Windows Media будет отображаться как текст (**автор**, **авторские права**, **заголовок**) и другие параметры, связанные с этим клипом. Дочерний элемент **ref** связывает содержимое, которое должно быть передаваться в поток для родительского элемента **entry** . Несмотря на то, что скрипт не будет прерываться, **запись** не имеет смысла без дочернего элемента **ref** .

Если значение атрибута **клиентскип** равно «нет», то пользователь не сможет пропускать вперед после фрагмента содержимого, определенного элементом **entry** . Это не работает, если элемент дочерней **ссылки** ссылается на другой метафайл. На вложенные метафайлы следует ссылаться с помощью элемента **ентриреф** .

Атрибут **скипифреф** относится только к элементам **записи** , которые включены во второй метафайл посредством использования элемента **ентриреф** . Элемент **ентриреф** ссылается на другой метафайл для логического включения в текущий файл. если значение атрибута **скипифреф** для элемента **ENTRY** из файла метафайла, на который указывает ссылка, равно YES, проигрыватель Windows Media пропускает эту запись и переходит к следующему элементу **entry** , если он есть. Следующий элемент **entry** может быть следующей записью в исходном файле или следующей записью в метафайле, на который ссылается элемент **ентриреф** .

## <a name="examples"></a>Примеры


```XML
<ASX VERSION="3.0">
   <TITLE>Example Windows Media Player Show</TITLE>
   
   <ENTRY>
      <TITLE>Example Clip</TITLE>
      <REF HREF="https://example.microsoft.com/media.asf" />
   </ENTRY>
   
   <ENTRY>
      <TITLE>Another Clip</TITLE>
      <REF HREF="https://example.microsoft.com/more_media.asf" />
   </ENTRY>
</ASX>

```
