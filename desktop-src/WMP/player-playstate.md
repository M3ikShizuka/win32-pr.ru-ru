# <a name="playerplaystate"></a>Player. Плайстате

Свойство **плайстате** извлекает значение, указывающее состояние операции проигрывателя Windows Media.

## <a name="syntax"></a>Синтаксис

*проигрыватель* . **плайстате**

## <a name="possible-values"></a>Возможные значения

Это свойство является **числом** только для чтения (**длинное целое**). Константу перечисления в стиле C можно получить с помощью префикса "вмппс" в качестве значения состояния. Например, константа для воспроизводимого состояния — **вмппсплайинг**.



| Значение | Состояние         | Описание                                                                                                                 |
|-------|---------------|-----------------------------------------------------------------------------------------------------------------------------|
| 0     | Не определено.     | Проигрыватель Windows Media находится в неопределенном состоянии.                                                                              |
| 1     | Остановлена       | Воспроизведение текущего элемента мультимедиа остановлено.                                                                              |
| 2     | Пауза        | Воспроизведение текущего элемента мультимедиа приостановлено. Когда элемент мультимедиа приостанавливается, возобновление воспроизведения начинается с того же расположения. |
| 3     | Воспроизведение       | Текущий элемент мультимедиа воспроизводится.                                                                                          |
| 4     | сканфорвард   | Текущий элемент мультимедиа обеспечивает быструю пересылку.                                                                                  |
| 5     | сканреверсе   | Текущий элемент мультимедиа — быстрое перемотка назад.                                                                                   |
| 6     | ответов     | Текущий элемент мультимедиа получает дополнительные данные с сервера.                                                          |
| 7     | Ожидание       | Соединение установлено, но сервер не отправляет данные. Ожидание начала сеанса.                                |
| 8     | MediaEnded    | Воспроизведение элемента мультимедиа завершено.                                                                                          |
| 9     | Переход | Подготовка нового элемента мультимедиа.                                                                                                   |
| 10    | Ready         | Все готово для начала воспроизведения.                                                                                                     |
| 11    | Повторное подключение  | Повторное подключение к потоку.                                                                                                     |



 

## <a name="remarks"></a>Комментарии

Состояния проигрывателя Windows Media не гарантированно выполняются в определенном порядке. Кроме того, не все состояния должны происходить во время последовательности событий. Не следует писать код, зависящий от порядка состояний.

## <a name="examples"></a>Примеры

В следующем коде JScript показано использование *проигрывателя*. Свойство **плайстате** . В текстовом элементе HTML с именем "myText" отображается текущее состояние. Объект **Player** создан с идентификатором "Player".


```JScript
// Test whether Windows Media Player is in the playing state.
if (3 == Player.playState)
    myText.value = "Windows Media Player is playing!";
else
    myText.value = "Windows Media Player is NOT playing!";
```
