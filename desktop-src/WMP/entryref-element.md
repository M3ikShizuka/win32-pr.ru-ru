# <a name="entryref-element"></a>ЕНТРИРЕФ, элемент

Элемент **ентриреф** указывает на элементы **записи** во внешнем метафайле.

``` syntax
<ENTRYREF
   HREF = "URL"
/>
```

## <a name="attributes"></a>Атрибуты

**Href** (обязательно)

URL-адрес метафайла, на который указывает ссылка.

**клиентбинд**

Этот атрибут больше не поддерживается.

## <a name="parentchild-elements"></a>Родительские и дочерние элементы



| Иерархия       | Элементы                       |
|-----------------|--------------------------------|
| Родительские элементы | **ASX**, **событие**, **Повтор** |
| Дочерние элементы  | None                           |



 

## <a name="remarks"></a>Remarks

Этот элемент указывает на содержимое внешнего метафайла. проигрыватель Windows Media обрабатывает элементы **записи** указанного метафайла, как если бы они были добавлены в основной метафайл в позиции элемента **ентриреф** . Однако он пропускает все элементы **записи** в метафайле, на который указывает ссылка, для которого атрибут **СКИПИФРЕФ** имеет значение Да.

проигрыватель Windows Media 7,0, 7,1 и проигрыватель Windows Media для Windows XP игнорируют все элементы **ентриреф** в метафайле, на который указывает ссылка. Поэтому при использовании этих версий проигрывателя метафайлы могут быть вложены только на один уровень глубже. проигрыватель Windows Media также игнорирует элемент **ASX** в упоминаемом файле вместе с его атрибутами. проигрыватель Windows Media 9 Series или более поздней версии поддерживает вложение метафайлов размером до пяти уровней.

URL-адрес, указанный в атрибуте **href** , станет базовым URL-адресом для всех относительных URL-адресов во внешнем метафайле. Этот URL-адрес используется, когда воспроизводится запись из внешнего метафайла и пользователь добавляет его в библиотеку.

## <a name="examples"></a>Примеры


```XML
<ASX VERSION="3.0">
   <TITLE>Example of Using EntryRef</TITLE>
   <ENTRYREF HREF="https://sample.microsoft.com/metafile.asx" />
</ASX>

```
