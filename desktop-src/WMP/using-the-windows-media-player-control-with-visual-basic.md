# <a name="using-the-windows-media-player-control-with-visual-basic"></a>Использование элемента управления проигрывателя Windows Media с Visual Basic

В этом разделе описывается, как использовать Windows Media Player 9 Series или более поздней версии элемента управления ActiveX в приложениях, созданных с помощью Microsoft Visual Basic 6,0.

## <a name="getting-started"></a>Приступая к работе

Чтобы добавить элемент управления проигрывателя Windows Media на панель элементов, сначала выберите **компоненты** в меню **проект** . В диалоговом окне Компоненты установите флажок рядом с полем "проигрыватель Windows Media". В нижней части диалогового окна убедитесь, что выбранный файл имеет wmp.dll. После закрытия диалогового окна экземпляр элемента управления проигрывателя Windows Media можно поместить в форму обычным образом.

Вы можете задать множество свойств управления с помощью окно свойств. Чтобы задать некоторые свойства, необходимо использовать диалоговое окно Свойства проигрывателя Windows Media, которое открывается с помощью элемента "(пользовательский)" в окно свойств.

## <a name="object-references"></a>Ссылки на объекты

Для получения ссылок на определенные объекты используются определенные свойства управления проигрывателем. Например, свойство **кдромколлектион** возвращает ссылку на объект **кдромколлектион** . Такую ссылку необходимо назначить переменной, объявленной в качестве соответствующего интерфейса. Например, при использовании свойства **кдромколлектион** его возвращаемое значение присваивается переменной типа **ивмпкдромколлектион**.

Ознакомьтесь с разделом [интерфейсы](interfaces.md) в [справочнике по объектной модели для C++](object-model-reference-for-c.md) , чтобы узнать, какие объекты реализуют несколько интерфейсов. В таких случаях необходимо объявить объектную переменную в качестве самого высокого номера интерфейса, описанного в этом пакете SDK, чтобы получить доступ ко всем свойствам и методам этого объекта. Например, следует присвоить значение свойства **куррентмедиа** элемента управления проигрывателя Windows Media переменной, объявленной как **IWMPMedia3** , чтобы гарантировать доступ к методам **жетаттрибутекаунтбитипе** и **жетитеминфобитипе** .

> [!Note]  
> Объект **виндовсмедиаплайер** реализует все свойства и методы интерфейсов **ивмпкоре**, **IWMPCore2**, **IWMPCore3**, **ивмпплайер**, **IWMPPlayer2**, **IWMPPlayer3** и **IWMPPlayer4** . Для этих интерфейсов не нужно объявлять отдельные переменные. Доступ ко всем членам можно получить с помощью имени, назначенного экземпляру **виндовсмедиаплайер** .

 

В обозревателе объектов Visual Basic вы увидите множество интерфейсов, предназначенных для частного использования элементом управления проигрывателя Windows Media, включая некоторые, которые поддерживаются разработчиками обложки. Следует использовать только объекты, свойства, методы и события, описанные в этом пакете SDK.

## <a name="additional-tips"></a>Дополнительные советы

-   В справочной документации показан синтаксис JScript. В JScript аргументы, передаваемые методам, всегда заключаются в круглые скобки. В Visual Basic 6,0 аргументы, передаваемые методам, не возвращающим значение, не должны заключаться в круглые скобки.
-   Некоторые свойства или методы могут не отображаться в функции автосписковое автозавершение кода в Visual Basic редакторе кода. Вы по-прежнему можете использовать эти члены, вводя их имена точно так же, как они отображаются в этой документации.
-   Управление внешним видом элемента управления с помощью свойства **UIMODE** . Это можно сделать двумя способами. Можно использовать раскрывающийся список **выбрать режим** в диалоговом окне Свойства проигрывателя Windows Media или ввести правильное значение в окно свойств.

    В частности, не используйте свойство **Visible** для скрытия элемента управления. Вместо этого присвойте свойству **UIMODE** значение «невидимый».
