# <a name="using-the-windows-media-player-control-with-visual-basic"></a>использование элемента управления проигрыватель Windows Media с Visual Basic

в этом разделе описано, как использовать элемент управления проигрыватель Windows Media 9 Series или более поздней версии ActiveX в приложениях, созданных с помощью Microsoft Visual Basic 6,0.

## <a name="getting-started"></a>Приступая к работе

чтобы добавить элемент управления проигрыватель Windows Media на панель элементов, сначала выберите **компоненты** в меню **Project** . в диалоговом окне компоненты установите флажок рядом с проигрыватель Windows Media. В нижней части диалогового окна убедитесь, что выбранный файл имеет wmp.dll. после закрытия диалогового окна экземпляр элемента управления проигрыватель Windows Media можно поместить в форму обычным образом.

Вы можете задать множество свойств управления с помощью окно свойств. чтобы задать некоторые свойства, необходимо использовать диалоговое окно свойства проигрыватель Windows Media, которое открывается с помощью элемента "(пользовательский)" в окно свойств.

## <a name="object-references"></a>Ссылки на объекты

Для получения ссылок на определенные объекты используются определенные свойства управления проигрывателем. Например, свойство **кдромколлектион** возвращает ссылку на объект **кдромколлектион** . Такую ссылку необходимо назначить переменной, объявленной в качестве соответствующего интерфейса. Например, при использовании свойства **кдромколлектион** его возвращаемое значение присваивается переменной типа **ивмпкдромколлектион**.

Ознакомьтесь с разделом [интерфейсы](interfaces.md) в [справочнике по объектной модели для C++](object-model-reference-for-c.md) , чтобы узнать, какие объекты реализуют несколько интерфейсов. В таких случаях необходимо объявить объектную переменную в качестве самого высокого номера интерфейса, описанного в этом пакете SDK, чтобы получить доступ ко всем свойствам и методам этого объекта. например, следует присвоить значение свойства **куррентмедиа** проигрыватель Windows Media элемента управления переменной, объявленной как **IWMPMedia3** , чтобы гарантировать доступ к методам **жетаттрибутекаунтбитипе** и **жетитеминфобитипе** .

> [!Note]  
> Объект **виндовсмедиаплайер** реализует все свойства и методы интерфейсов **ивмпкоре**, **IWMPCore2**, **IWMPCore3**, **ивмпплайер**, **IWMPPlayer2**, **IWMPPlayer3** и **IWMPPlayer4** . Для этих интерфейсов не нужно объявлять отдельные переменные. Доступ ко всем членам можно получить с помощью имени, назначенного экземпляру **виндовсмедиаплайер** .

 

в обозревателе объектов Visual Basic вы увидите множество интерфейсов, предназначенных для частного использования элементом управления проигрыватель Windows Media, включая некоторые, которые поддерживаются разработчиками обложки. Следует использовать только объекты, свойства, методы и события, описанные в этом пакете SDK.

## <a name="additional-tips"></a>Дополнительные советы

-   в справочной документации показан синтаксис JScript. в JScript аргументы, передаваемые методам, всегда заключаются в круглые скобки. в Visual Basic 6,0 аргументы, передаваемые методам, не возвращающим значение, не должны заключаться в круглые скобки.
-   некоторые свойства или методы могут не отображаться в функции автосписковое автозавершение кода в Visual Basic редакторе кода. Вы по-прежнему можете использовать эти члены, вводя их имена точно так же, как они отображаются в этой документации.
-   Управление внешним видом элемента управления с помощью свойства **UIMODE** . Это можно сделать двумя способами. можно использовать раскрывающийся список **выбрать режим** в диалоговом окне проигрыватель Windows Media свойства или ввести правильное значение в окно свойств.

    В частности, не используйте свойство **Visible** для скрытия элемента управления. Вместо этого присвойте свойству **UIMODE** значение «невидимый».
