# <a name="art-files"></a>Файлы изображений

Для обложки необходимо создать один или несколько графических файлов. Без рисунка у пользователя не будет ничего взглянуть на. Можно создать невидимую обложку, но никто не увидит ее. И даже затем придется создавать файлы изображений для размещения невидимых изображений, так как для файла определения обложки требуются графические файлы для конкретных элементов.

Существует три способа использования иллюстрации в обложках: основные образы, изображения сопоставления и альтернативные изображения.

## <a name="primary-images"></a>Основные образы

Необходимо создать основной образ для обложки. Это то, что пользователи увидят при установке обложки. Основной образ состоит из одного или нескольких образов, созданных конкретными элементами управления обложки.

При наличии нескольких элементов управления необходимо указать z-порядок, который определяет, какие элементы управления отображаются перед другими элементами управления. Каждый элемент **представления** будет иметь фоновое изображение, в которое можно добавить другие изображения элементов в, что позволит создать первичный составной образ.

Кроме того, могут иметься дополнительные образы, например скользящий лоток, который не отображается при первом отображении обложки, но отображается, когда пользователь выполняет какое-либо действие. Они следуют тем же правилам, что и основные образы, в том, что они создаются с помощью набора элементов управления.

## <a name="mapping-images"></a>Сопоставление изображений

Одной из самых эффективных функций обложек проигрывателя Windows Media является возможность использовать сопоставление изображений для активации событий для обложки. Карты изображений — это файлы, содержащие специальные образы. Однако изображения в файле отображения изображений не предназначены для просмотра пользователем, но используются проигрывателем Windows Media для выполнения действий, когда пользователь щелкает обложку.

Различным элементам управления требуются различные типы карт изображений. Например, если цветная часть изображения соответствует определенному красному значению, а пользователь щелкает соответствующую область основного изображения, то кнопка будет срабатывать на событие. Цвет используется для определения событий, которые вызываются щелчком в определенной области обложки.

## <a name="alternate-images"></a>Альтернативные образы

Кроме того, можно настроить отображение альтернативных изображений, когда пользователь выполняет какие-либо действия. Например, можно создать альтернативное изображение кнопки, которое будет отображаться только при наведении указателя мыши на кнопку. Это хороший способ предоставить пользователям сведения о том, что они могут делать, а также предоставляет возможность легко обнаруживаемого пользовательского интерфейса. При тщательном использовании всплывающих подсказок и изображений для наведения можно создавать необычные пользовательские интерфейсы, которые по-прежнему предоставляют пользователю Отзывы о доступных параметрах.
