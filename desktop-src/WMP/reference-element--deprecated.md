# <a name="reference-element-deprecated"></a>Элемент Reference (не рекомендуется)

В этом разделе документируется функция, которая больше не используется в текущей версии метафайлов Windows Media.

Элемент **Reference** содержит список ссылок на URL-адреса. Каждый элемент в списке имеет   =  *URL-адрес* ref XX, где *XX* — целое число, а *URL* — URL-адрес файла ASF.

**Синтаксис**


```XML
[reference]
RefXX = URL
RefXX = URL
   
<entity type="hellip"/>
```



Целые числа, представленные значением *XX* , должны быть в возрастающем порядке. То есть целое число в каждой строке должно быть больше, чем целое число в предыдущей строке.

Когда клиентская программа считывает элемент Reference, он пытается подключиться к файлу мультимедиа, представленному первым URL-адресом в списке. Если это не удается, клиентская программа пытается подключиться к файлу, представленному следующим URL-адресом в списке. Клиентская программа переходит в список до тех пор, пока не будет выполнено успешное подключение или не достигнет конца списка. Обратите внимание, что если клиентская программа устанавливает успешное подключение, она не считывает ни одного из последующих URL-адресов в списке.

**Пример кода**

В следующем примере клиентская программа сначала пытается подключиться к файлу, представленному URL-адресом MMS. Если это не удалось, клиентская программа попытается подключиться к файлу, представленному URL-адресом HTTP.


```XML
[reference]
Ref01 = mms://example.com/MusicFile.wma
Ref02 = https://example.com/MusicFile.wma

```



## <a name="remarks"></a>Комментарии

Формат файлов ASF включает в себя разнообразные типы мультимедиа, включая аудио и видео. Файлы ASF также используют несколько расширений имен файлов, включая файлы WMA и WMV.
