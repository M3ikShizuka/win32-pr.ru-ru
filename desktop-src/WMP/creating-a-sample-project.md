# <a name="creating-a-sample-project"></a>Создание примера проекта

Чтобы создать новый проект с помощью мастера проектов, выполните следующие действия.

1.  Откройте среду Microsoft Visual Studio.
2.  В меню **Файл** выберите пункт **Создать**, а затем команду **Проект**.
3.  В списке **типы проектов** выберите **Visual C++ проекты**.
4.  В списке **шаблоны** выберите **Мастер Интернет-магазинов проигрывателя Windows Media**.
5.  Введите имя и расположение проекта.
6.  Нажмите кнопку **ОК** , чтобы запустить мастер проектов.
7.  Выберите **тип 2 (интеграция с базовой интеграцией)** и нажмите кнопку **Далее**.
8.  Введите понятное имя и идентификатор распространителя содержимого для службы. Введите URL-адрес веб-страницы, которая удаляет службу с компьютера пользователя.
    > [!Note]  
    > Значение, которое вы задают для идентификатора распространителя содержимого, не должно содержать пробелы. Мастер удаляет все пробелы из предоставленной строки.

     

9.  Нажмите кнопку **Готово**, чтобы создать проект.

Мастер автоматически создает новый проект C++, который создает подключаемый модуль магазина типа 2, реализующий интерфейсы **ивмпсубскриптионсервице** и **IWMPSubscriptionService2** . Каждый раз при запуске мастера создается тот же проект, но компонент, создаваемый при компиляции проекта, имеет уникальный идентификатор класса. Имя проекта, понятное имя и идентификатор распространителя содержимого могут также отличаться в зависимости от значений, указанных при запуске мастера. Пример проекта регистрирует компонент, используя имя ключа, которое соответствует значению, указанному для понятного имени.

В примере используется код библиотеки активных шаблонов (ATL) для реализации COM-реализаций.

Мастер создает следующие файлы:

-   *Имя_проекта* DLL. cpp. Реализует экспорт библиотеки динамической компоновки (DLL), например основную функцию точки входа библиотеки DLL. Также содержит декларацию объектов ATL и объявление модуля.
-   *Имя_проекта*. cpp. Содержит реализации по умолчанию для методов интерфейсов Ивмпсубскриптионсервице и IWMPSubscriptionService2. Также содержит код для определения диалоговых окон, которые открывается при вызове метода проигрывателем Windows Media.
-   *Имя_проекта*. def. Объявляет экспорты DLL.
-   StdAfx. cpp. Включает стандартные заголовки.
-   WMP. h. Файл заголовка проигрывателя Windows Media.
-   вмпплуг. h. Файл заголовка подключаемого модуля проигрывателя Windows Media.
-   субскриптионсервицес. h. Файл заголовка Windows Media Player в Интернете.
-   Resource. h. Содержит определения ресурсов.
-   **Имя_проекта**. h. Содержит определения классов для объекта Интернет-магазина и диалоговых окон. Определяет идентификатор класса объекта и константу идентификатора распространителя содержимого.
-   Файл StdAfx. h. Содержит стандартные системные включения.
-   *Имя_проекта*. rc. Файл описания ресурсов. Это содержит определения для ресурсов диалоговых окон и элементов управления. Здесь также хранится таблица строк. Таблица строк содержит имя проекта и строковые константы с понятными именами. Обычно этот файл работает в редакторе ресурсов Visual Studio, а не в виде обычного текста.
-   *Имя_проекта*. RGS. Файл скрипта реестра. Этот файл содержит сведения, используемые для добавления класса компонента в реестр, чтобы проигрыватель Windows Media мог его разместить. Имя ключа для службы можно изменить в этом файле.

> [!Note]  
> Необходимо задать базовый адрес для библиотеки DLL 0x0F100000 во избежание конфликтов с проигрывателем Windows Media.
