# <a name="logurl-element"></a>ЛОГУРЛ, элемент

Элемент **логурл** указывает проигрывателю Windows Media отправлять любые данные журнала по указанному URL-адресу.

``` syntax
<LOGURL
   HREF = "URL"
/>
```

## <a name="attributes"></a>Атрибуты

**Href** (обязательно)

URL-адрес узла, который может обрабатывать запросы на ведение журнала.

## <a name="parentchild-elements"></a>Родительские и дочерние элементы



| Иерархия       | Элементы           |
|-----------------|--------------------|
| Родительские элементы | **ASX**, **запись** |
| Дочерние элементы  | Нет               |



 

## <a name="remarks"></a>Remarks

Элемент **логурл** позволяет списку воспроизведения клиентского метафайла отправить дополнительные сведения о ведении журнала на указанные серверы. Данные журнала автоматически отправляются на сервер-источник списка воспроизведения при его открытии и для каждого **логурл** , указанного для элемента **ASX** , если таковые имеются. Сведения о ведении журнала также отправляются в каждый **логурл** , указанный для элемента **entry** при достижении этой записи. URL-адрес, указанный в атрибуте **href** элемента **логурл** , должен быть адресом узла, который может обрабатывать запросы на ведение журнала. URL-адрес может быть любым допустимым URL-адресом HTTP. URL-адрес также может указывать расположение сценария CGI.

Единственными допустимыми протоколами для элемента **логурл** являются HTTP и HTTPS.

Элемент **логурл** внутри области элемента **ASX** применим только к метафайлу, в котором он находится, независимо от того, есть ли ссылка на этот метафайл из другого метафайла. Элемент **логурл** вызывает принудительную отправку данных журнала для всего потока содержимого из заданной области и только для потока содержимого из его заданной области. Данные журнала будут отправлены на сервер-источник и все URL-адреса, перечисленные в каждом элементе **логурл** в области. Данные журнала будут отправляться только один раз на каждый указанный URL-адрес, даже если один и тот же URL-адрес указан более одного раза в заданной области. Повтор **записи** приведет к другой отправке в указанные URL-адреса.

Число элементов **логурл** в списке воспроизведения метафайлов не ограничено.

## <a name="examples"></a>Примеры


```XML
<ASX VERSION="3.0">
  <TITLE>Example Media Player Show</TITLE>
  <LOGURL HREF="https://example.microsoft.com/info/showlog.asp?whatsup" />
  <ENTRY>
    <REF href="mms://ucast.proseware.com/Media1.asf" />
    <LOGURL HREF="https://www.proseware.com/cgi-bin/logging.pl?SomeArg=SomeVal"/>
  </ENTRY>
</ASX>
```



Ниже приведены примеры допустимых URL-адресов.

URL-адрес приложения ISAPI:


```XML
https://www.proseware.com/logs/WMSLogging.dll
```



URL-адрес скрипта CGI:


```XML
https://www.proseware.com/cgi-bin/My_WMS_Logging_Script.pl
```



Допустимый URL-адрес HTTP:


```XML
https://www.proseware.com/some/arbitrary/path/My_WMS_Logging_Page.asp?PubPoint=FooPubPoint&AnotherName=AnotherValue
```
