# <a name="personalizing-media-delivery"></a>Настройка доставки мультимедиа

в отличие от односторонней связи, которая передает идентичное содержимое массовой аудитории, Windows мультимедийные технологии предоставляют средства для использования демографических данных для индивидуализируйте вещания. Благодаря Интернету двусторонняя связь в крупном масштабе доступна. Этот динамический обмен информацией позволяет поставщикам содержимого узнавать свою аудиторию и реагировать на настроенные презентации.

В следующем примере с помощью вымышленной компании показано, как можно персонализировать потоковую передачу содержимого. В этом обсуждении предполагается, что вы знакомы с Active Server страницами (ASP или файлы ASP) и определением переменных.

«Новости» — это вымышленная организация с новостями, которая развернула свои операции для включения веб-сайта. Основной компонент сайта — это раздел, в котором пользователи могут создавать собственные персонализированные невскастс. Вместо того чтобы просматривать традиционные невскаст, предназначенные для массовой аудитории, пользователь видит полную программу новостей, которая содержит только темы личного интереса. Следующая последовательность описывает типичное взаимодействие с пользователем.

1.  Новый пользователь переходит на сайт и нажимает кнопку **создать личный невскаст**.
2.  Откроется форма предпочтений. В этой форме пользователь отвечает на вопросы о личных предпочтениях, например о избранных новостях, наименее избранных новостях, хобби и обычном способе получения ежедневных новостей.
3.  Пользователь отправляет эти сведения, и через несколько секунд вы просматриваете полный, 15-минутный, персональный невскаст, содержащий содержимое программы, переходы и коммерческие приложения. выбор каждого элемента мультимедиа, включая коммерческие, зависит от профиля пользователя и выполняется автоматически с Windows компонентами технологий мультимедиа и готовыми интернет-инструментами.

В следующем списке описано, как различные средства взаимодействуют для создания персонализированной невскаст.

1.  Форма предпочтения, заполняемая пользователем, — это страница Active Server (ASP) (Choice. ASP). Данные, полученные из формы предпочтений, анализируются двумя серверными компонентами. один компонент использует эти сведения для запроса Microsoft SQL Server базы данных новостей. Другой компонент — это сервер AD, использующий сложные наборы правил, основанные на контрактных требованиях и демографических данных, чтобы запланировать в это время рекламу, соответствующую пользователю.
2.  Две базы данных возвращают различные части списка воспроизведения. База данных новостей о новостях возвращает набор соответствующих записей истории, а сервер AD возвращает набор соответствующих коммерческих записей.
3.  Вторая ASP-страница (Плайшов. ASP) получает записи из базы данных новостей и сервера AD и объединяет их со стандартными записями Open, Close и Transition. Все записи затем размещаются в соответствии с шаблоном, предоставленным Плайшов. ASP, и страница ASP возвращает список воспроизведения пользователю.
4.  встроенный элемент управления проигрыватель Windows Media на компьютере пользователя воспроизводит список воспроизведения от начала до конца, и пользователь просматривает персонализированный невскаст.

В следующем примере показана часть файла списка воспроизведения, который может получить пользователь. В нее добавлены баннеры, ссылки МОРЕИНФО и АБСТРАКЦИи.


```XML
<ASX Version="3">
<TITLE>MyPersonalized NewsCast</TITLE>
<ENTRY ClientSkip="no">
    <!<entity type="mdash"/>- Commercial Element 1 -->
    <REF HREF="mms://proseware.com/Commercial.wma" />
    <BANNER HREF="https://www.proseware.com/images/MoreInfo.gif" >
        <MOREINFO HREF="https://www.proseware.com" target="_blank" />
    <ABSTRACT>Courtesy of Windows Media Technologies
    </ABSTRACT>
    </BANNER>
</ENTRY>
<ENTRY>
    <!<entity type="mdash"/>- Program Element 1 -->
    <TITLE>A Celebrity's Life</TITLE>
    <COPYRIGHT>Copyright 2004</COPYRIGHT>
    <REF HREF="mms://proseware.com/SomePath/TheFile.wma" />
    <ABSTRACT>
     :: A celebrity looks back on her career after 40 years in public life.
    </ABSTRACT>
    <COPYRIGHT>Copyright 2004-- All Rights
         Reserved
    </COPYRIGHT>
</ENTRY>

<ENTRY>
    <!<entity type="mdash"/>Program Element 2 -->
    <TITLE>City council votes to build new bicycle path</TITLE>
    <COPYRIGHT>Copyright 2004</COPYRIGHT>
    <REF HREF="mms://proseware.com/SomePath/MyFile.wma" />
    <ABSTRACT>
        :: Some residents opposed changing the landscape in the public parks to accommodate bicycles.
    </ABSTRACT>
    <COPYRIGHT>Copyright 2004 -- All Rights Reserved
    </COPYRIGHT>
</ENTRY>
</ASX>

```



-   Названия организаций, предприятий и изделий, а также имена и события, используемые в качестве примеров, являются вымышленными. Возможное сходство с реально существующими организациями, предприятиями, изделиями, лицами и событиями следует рассматривать как случайное.
