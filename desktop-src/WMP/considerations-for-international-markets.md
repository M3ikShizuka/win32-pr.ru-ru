# <a name="considerations-for-international-markets"></a>Рекомендации для международных рынков

Если ваша компания создает Интернет-магазины для нескольких рынков, для каждого рынка необходимо предоставить корпорации Майкрософт адрес URL документа ServiceInfo. Это можно сделать одним из следующих двух способов:

-   Создайте отдельный документ ServiceInfo для каждого рынка. В этом случае вы предоставляете корпорации Майкрософт URL-адреса, указывающие на отдельные документы ServiceInfo для каждого из Интернет-магазинов на каждом рынке.
-   Создайте отдельный документ ServiceInfo для всех рынков. При этом вы предоставляете корпорации Майкрософт один и тот же URL-адрес для каждого рынка. Документ ServiceInfo, созданный в виде ASP-страницы, может динамически определять расположение пользователя на основе параметров строки запроса.

проигрыватель Windows Media добавляет строку запроса в запрос URL-адреса ServiceInfo, который предоставляет сведения о локали и параметрах расположения пользователя. Если Интернет-магазин использует эти сведения для определения отображаемого содержимого, следует динамически добавлять эти значения к своим URL-адресам в документе ServiceInfo. Это лучший способ гарантировать, что URL-адреса веб-страниц всегда будут содержать нужные параметры.

Определив расположение пользователя и предпочитаемый язык, вы можете сохранить эти сведения для будущих сеансов. Это можно сделать с помощью любого из методов, обычно используемых на веб-странице, таких как файлы cookie, или с помощью COM-объекта.
