# <a name="sync-attributes"></a>Атрибуты синхронизации

атрибуты **Sync01** по **Sync16** представляют собой строковые представления 32-разрядных значений, которые проигрыватель Windows Media использует при синхронизации списков воспроизведения с одним из 16 портативных устройств.

## <a name="applies-to"></a>Применяется к

-   [Списки воспроизведения](playlist-attributes-ref.md)

## <a name="remarks"></a>Remarks

Они являются атрибутами чтения и записи. нулевое значение означает, что проигрыватель Windows Media не должны синхронизировать список воспроизведения с соответствующим устройством. Значение больше нуля указывает приоритет синхронизации для данного списка воспроизведения.

на основе введенных пользователем данных проигрыватель Windows Media задает этот атрибут для каждого списка воспроизведения в библиотеке. когда проигрыватель Windows Media пытается синхронизировать список воспроизведения с портативным устройством, он проверяет каждый список воспроизведения для сравнения значений атрибутов **Sync**_XX_ , представляющих связь устройства. Списки воспроизведения, имеющие более низкие значения параметров **синхронизации**_XX_ , получают более высокий приоритет синхронизации.

Например, Библиотека может содержать следующие четыре списка воспроизведения и соответствующие значения **синхронизации**_XX_ :



| Имя списка воспроизведения | Значение Sync01 |
|---------------|--------------|
| Гиммусик. WPL  | 0x0000000D   |
| Ослабить. WPL     | 0x00000020   |
| Дривехоме. WPL | 0x00000001   |
| Ного. WPL      | 0x00000000   |



 

когда проигрыватель Windows Media синхронизирует устройство, связанное с атрибутом, он синхронизирует списки воспроизведения в следующем порядке:

1.  Дривехоме. WPL
2.  Гиммусик. WPL
3.  Ослабить. WPL

Чтобы определить, можно ли изменить значение этого атрибута, используйте метод [Media. исреадонлитем](media-isreadonlyitem.md) .
