# <a name="sync-attributes"></a>Атрибуты синхронизации

Атрибуты **Sync01** через **Sync16** представляют собой строковые представления 32-разрядных значений, которые проигрыватель Windows Media использует при синхронизации списков воспроизведения с одним из 16 портативных устройств.

## <a name="applies-to"></a>Применение

-   [Списки воспроизведения](playlist-attributes-ref.md)

## <a name="remarks"></a>Комментарии

Они являются атрибутами чтения и записи. Нулевое значение указывает, что проигрыватель Windows Media не должен синхронизировать список воспроизведения с соответствующим устройством. Значение больше нуля указывает приоритет синхронизации для данного списка воспроизведения.

В зависимости от вводимых пользователем данных проигрыватель Windows Media устанавливает этот атрибут для каждого списка воспроизведения в библиотеке. Когда проигрыватель Windows Media пытается синхронизировать список воспроизведения с портативным устройством, он проверяет каждый список воспроизведения для сравнения значений атрибутов **синхронизации**_XX_ , которые представляют связь устройства. Списки воспроизведения, имеющие более низкие значения параметров **синхронизации**_XX_ , получают более высокий приоритет синхронизации.

Например, Библиотека может содержать следующие четыре списка воспроизведения и соответствующие значения **синхронизации**_XX_ :



| Имя списка воспроизведения | Значение Sync01 |
|---------------|--------------|
| Гиммусик. WPL  | 0x0000000D   |
| Ослабить. WPL     | 0x00000020   |
| Дривехоме. WPL | 0x00000001   |
| Ного. WPL      | 0x00000000   |



 

Когда проигрыватель Windows Media синхронизирует устройство, связанное с атрибутом, списки воспроизведения будут синхронизироваться в следующем порядке:

1.  Дривехоме. WPL
2.  Гиммусик. WPL
3.  Ослабить. WPL

Чтобы определить, можно ли изменить значение этого атрибута, используйте метод [Media. исреадонлитем](media-isreadonlyitem.md) .
