# <a name="adding-the-play-buttonelement"></a>Добавление БУТТОНЕЛЕМЕНТ Play

Наконец, можно добавить элементы **буттонелемент** , которые соединяют визуальные кнопки на экране, с действиями проигрывателя Windows Media. Это ядро обложки, и его можно рассматривать как привязку поверхности обложки к внутреннему механизму проигрывателя Windows Media.

**Буттонелемент** содержатся в **BUTTONGROUP**. В каждом **BUTTONGROUP** всегда должен быть хотя бы один **буттонелемент** .

Вставьте код Play **буттонелемент** после закрывающей угловой скобки **BUTTONGROUP**.


```C++
        <BUTTONELEMENT
            mappingColor = "#00FF00"
            upToolTip = "Play"
            onClick = "JScript: player.URL='https://proseware.com/laure.wma';" />

```



Для определения **буттонелемент** кнопки воспроизведения используются следующие атрибуты:

**маппингколор**

Это значение цвета для региона в файле рисунка сопоставления, созданном ранее. В данном случае это сплошной зеленый цвет. Этот атрибут является обязательным для любого **буттонелемент**. Определив этот цвет, вы указываете проигрывателю Windows Media связать эту цветовую область с XML-кодом этой кнопки.

**Подсказка**

Определяет текст, который будет отображаться, если пользователь наведет указатель мыши на кнопку. Не путайте это с назначением, которое будет отображаться. Всплывающая подсказка — это небольшой заголовок, который занимает некоторое время. Однако изображение с наведением будет мгновенно отображаться в любом выбранном цвете и форме.

**onClick**

Определяет событие, возникающее при нажатии кнопки мыши. Значение этого атрибута события называется обработчиком событий и будет либо строкой кода Microsoft JScript, либо функцией JScript во внешнем текстовом файле, который загружается атрибутом **Лоадскрипт** **представления**. В этом случае код JScript вызывает метод **URL-адреса** проигрывателя Windows Media, который загружает и начинает воспроизводить файл с именем «Лауре. WMA». Обратите внимание, что строка заканчивается точкой с запятой внутри кавычек, что является хорошей практикой написания кода в JScript. Обратите внимание также на то, что в двойных кавычках используется одинарные кавычки для задания имени файла. Дополнительные сведения о JScript см. в статье [Использование JScript](using-jscript.md) в разделе "сведения о обложках" этого пакета SDK.

Обратите внимание, что отсутствует конечный тег **буттонелемент** . Если элемент не заключает другой элемент, его можно закрыть с помощью косой черты прямо перед закрывающей угловой скобкой. Это говорит о том, что XML завершен с этим элементом. Например,


```C++
<BUTTONELEMENT></BUTTONELEMENT>
```



и


```C++
<BUTTONELEMENT/>
```



передает те же сведения в формате XML.

Возможности обложек обусловлены использованием обработчиков событий. Если пользователь делает что-то с помощью мыши, это событие можно обменять с помощью JScript. Код может быть единственной строкой, которая делает проигрыватель Windows Media простым, например Play, или полноценным приложением, написанным на языке JScript.
