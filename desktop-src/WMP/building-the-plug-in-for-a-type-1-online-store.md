# <a name="building-the-plug-in-for-a-type-1-online-store"></a>Создание подключаемого модуля для Интернет-магазина типа 1

Интернет-магазин типа 1 должен предоставлять подключаемый модуль, реализующий интерфейс [**ивмпконтентпартнер**](/previous-versions/windows/desktop/api/contentpartner/nn-contentpartner-iwmpcontentpartner) . подключаемый модуль выполняется в отдельном процессе от проигрыватель Windows Media.

Ниже приведены шаги по созданию подключаемого модуля, запускаемого вне процесса.

1.  Создайте подключаемый модуль, как если бы он был внутрипроцессный COM-сервером.
2.  Создайте запись реестра **дллсуррогате** на компьютере пользователя. Запись **дллсуррогате** информирует среду выполнения com о том, что подключаемый модуль должен быть создан в экземпляре СУРРОГАТНОЙ библиотеки DLL по умолчанию, dllhost.exe.

Подробные сведения о регистрации подключаемого модуля см. в разделе [разделы реестра и записи для Интернет-магазина типа 1](registry-keys-and-entries-for-a-type-1-online-store.md).

Для подключаемого модуля не нужно создавать прокси-сервер или код заглушки. вся поддержка упаковки обеспечивается проигрыватель Windows Media.

Для быстрого создания подключаемого модуля, который служит отправной точкой, можно использовать мастер подключаемых модулей интернет-магазина. Дополнительные сведения см. в статье [Установка подключаемого модуля интернет-магазина](installing-the-online-store-plug-in-wizard.md).
