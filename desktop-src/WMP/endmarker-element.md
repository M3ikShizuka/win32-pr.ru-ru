# <a name="endmarker-element"></a>ЕНДМАРКЕР, элемент

элемент **ендмаркер** указывает маркер, по которому проигрыватель Windows Media будет прерывать отрисовку потока.

``` syntax
<ENDMARKER
   NUMBER = "marker number" |
   NAME = "marker name"
/>
```

## <a name="attributes"></a>Атрибуты

**Нумерация**

Номер числового маркера в индексе. Значение по умолчанию — конец содержимого.

**ИМЯ**

Имя именованного маркера в индексе. Значение по умолчанию — конец содержимого.

## <a name="parentchild-elements"></a>Родительские и дочерние элементы



| Иерархия       | Элементы           |
|-----------------|--------------------|
| Родительские элементы | **entry**, **ref** |
| Дочерние элементы  | Нет               |



 

## <a name="remarks"></a>Remarks

этот элемент задает маркер, где проигрыватель Windows Media — останавливает отрисовку потока, определенного в родительской **записи** или элементе **REF** .

Примечание

Используйте элемент **ендмаркер** с атрибутом **Number** или **Name** , но не обоими.

В режиме предварительного просмотра достижение завершающего маркера останавливает предварительную версию, даже если не прошло время, заданное элементом **превиевдуратион** . Элемент **ендмаркер** также имеет приоритет над элементом **DURATION** .

Элемент **ендмаркер** , определенный внутри **ссылочного** элемента, имеет приоритет над **ендмаркер** , определенным в родительском элементе **entry** элемента **ref** .

Если маркер, заданный элементом **ендмаркер** , находится в потоке раньше, чем маркер, заданный элементом **стартмаркер** , содержимое не будет воспроизводиться, но ошибки не создаются.

## <a name="examples"></a>Примеры


```XML
<ENDMARKER NUMBER="17" />
<ENDMARKER NAME="Marker_StopHere" />

```
