# <a name="wm_keydown-message"></a>\_Сообщение WM KeyDown

Отправляется в окно с фокусом клавиатуры при нажатии несистемного ключа. Несистемный ключ — это ключ, который нажимается, когда клавиша ALT не нажата.


```C++
#define WM_KEYDOWN                      0x0100
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Код виртуального ключа несистемного ключа. См. раздел [коды виртуальных клавиш](virtual-key-codes.md).

</dd> <dt>

*lParam* 
</dt> <dd>

Число повторов, код сканирования, флаг расширенного ключа, код контекста, предыдущий флаг состояния ключа и флаг состояния перехода, как показано ниже.



| Bits  | Значение                                                                                                                                                                                                                                                               |
|-------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 0—15  | Число повторов для текущего сообщения. Значение — это число повторных попыток нажатия клавиши в результате, когда пользователь удерживает клавишу. Если нажатие клавиши удерживается достаточно долго, отправляется несколько сообщений. Однако число повторов не является накопительным. |
| 16—23 | Код сканирования. Значение зависит от поставщика вычислительной техники.                                                                                                                                                                                                                          |
| 24    | Указывает, является ли ключ расширенным ключом, таким как правая клавиша ALT и CTRL, которые появляются на клавиатуре с 101 или 102. Значение равно 1, если это расширенный ключ; в противном случае — 0.                                                              |
| 25-28 | Процессу не используйте.                                                                                                                                                                                                                                                 |
| 29    | Код контекста. Значение всегда равно 0 для сообщения **WM \_ KeyDown** .                                                                                                                                                                                                |
| 30    | Предыдущее состояние ключа. Значение равно 1, если ключ не работает до отправки сообщения, или равен нулю, если ключ работает.                                                                                                                                                 |
| 31    | Состояние перехода. Значение всегда равно 0 для сообщения **WM \_ KeyDown** .                                                                                                                                                                                            |

Дополнительные сведения см. в разделе [Флаги сообщения о нажатии клавиш](about-keyboard-input.md#keystroke-message-flags).
 

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Приложение должно вернуть нуль, если оно обрабатывает это сообщение.

## <a name="example"></a>Пример

```cpp
LRESULT CALLBACK HostWndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam)
{
    switch (message) 
    {
    case WM_KEYDOWN:
        if (wParam == VK_ESCAPE)
        {
            if (isFullScreen) 
            {
                GoPartialScreen();
            }
        }
        break;

    // ...
    default:
        return DefWindowProc(hWnd, message, wParam, lParam);
    }
    return 0;  
}

```

пример из [Windows классических примеров](https://github.com/microsoft/Windows-classic-samples/blob/1d363ff4bd17d8e20415b92e2ee989d615cc0d91/Samples/Magnification/cpp/Windowed/MagnifierSample.cpp) на GitHub.


## <a name="remarks"></a>Remarks

Если нажата клавиша F10, функция [**дефвиндовпрок**](/windows/desktop/api/winuser/nf-winuser-defwindowproca) устанавливает внутренний флаг. Когда **дефвиндовпрок** получает сообщение [**WM \_ KEYUP**](wm-keyup.md) , функция проверяет, установлен ли внутренний флаг, и, если это так, отправляет сообщение [**WM \_ сискомманд**](/windows/desktop/menurc/wm-syscommand) в окно верхнего уровня. Параметр **WM \_ сискомманд** сообщения имеет значение SC \_ кэймену.

Из-за функции автоповтора может быть отправлено более одного сообщения **WM \_ KeyDown** , прежде чем будет отправлено сообщение [**WM \_ KEYUP**](wm-keyup.md) . Предыдущее состояние ключа (бит 30) можно использовать, чтобы определить, указывает ли сообщение **WM \_ KeyDown** на первый переход или повторный переход.

Для расширенных клавиш с 101 и 102, расширенными ключами являются сочетания клавиш ALT и CTRL в основном разделе клавиатуры. клавиши INS, DEL, ДОМАШНяя, КОНЕЧная страница вверх, PAGE DOWN и стрелка в кластерах слева от цифровой клавиатуры; и клавиши деления (/) и ENTER на цифровой клавиатуре. Другие клавиатуры могут поддерживать бит расширенного ключа в параметре *lParam* .

Приложения должны передавать *wParam* в [**TranslateMessage**](/windows/desktop/api/winuser/nf-winuser-translatemessage) , не изменяя его вообще.
