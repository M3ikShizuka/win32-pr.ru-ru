# <a name="windows-event-collector"></a>Сборщик событий Windows

Вы можете подписываться на получение и хранение событий на локальном компьютере (сборщике событий), пересылаемом с удаленного компьютера (источник события). [функции сборщика событий Windows](windows-event-collector-functions.md) поддерживают подписку на события с помощью протокола WS-Management. дополнительные сведения о службе WS-Management см. в разделе [about служба удаленного управления Windows](/windows/desktop/WinRM/about-windows-remote-management).

## <a name="event-forwarding-and-event-collection-architecture"></a>Архитектура пересылки событий и сбора событий

Сбор событий позволяет администраторам получать события с удаленных компьютеров и сохранять их в локальном журнале событий на компьютере сборщика. Путь к журналу назначения для событий является свойством подписки. Все данные перенаправленного события сохраняются в журнале событий компьютера сборщика (ни одна из этих данных не теряется). Дополнительные сведения, связанные с пересылкой событий, также добавляются в событие. Дополнительные сведения о том, как разрешить компьютеру получать собранные события или пересылать события, см. в разделе [Настройка компьютеров для пересылки и сбора событий](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc748890(v=ws.11)).

## <a name="subscriptions"></a>Подписки

В следующем списке описываются типы подписок на события.

-   Подписки, инициированные источником. позволяет определить подписку на события на компьютере сборщика событий без определения исходных компьютеров событий. Затем можно настроить несколько удаленных компьютеров источника событий (с помощью параметра групповой политики), чтобы перенаправить события на компьютер сборщика событий. Дополнительные сведения см. [в разделе Настройка инициированной источником подписки](setting-up-a-source-initiated-subscription.md). Этот тип подписки полезен, если нет необходимости указывать все источники событий компьютеры, которые будут пересылать события.
-   Подписки, инициированные сборщиком. позволяет создать подписку на событие, если известно, что все исходные компьютеры событий будут пересылать события. Вы указываете все источники событий во время создания подписки. Дополнительные сведения см. [в разделе Создание подписки, инициированной сборщиком](creating-an-event-collector-subscription.md).
