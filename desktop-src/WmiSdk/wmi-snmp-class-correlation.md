# <a name="wmi-snmp-class-correlation"></a>Корреляция классов SNMP WMI

Корреляция влияет на набор классов, возвращаемых из Смир. Коррелированные классы — это те, которые поддерживаются данным устройством SNMP при перечислении времени. Некоррелированное перечисление возвращает все классы, находящиеся в Смир, независимо от того, поддерживает ли устройство их. Корреляция является функцией поставщика WMI SNMP и может быть выключена или включена (по умолчанию).

> [!Note]  
> Дополнительные сведения об установке поставщика см. в разделе [Настройка среды SNMP WMI](setting-up-the-wmi-snmp-environment.md).

 

Корреляция может препятствовать просмотру всех классов, которые на самом деле поддерживаются устройством. Если известно, что конкретный класс поддерживается, но не отображается в Смир, это может быть вызвано несколькими причинами, одним из которых является корреляция. Рассмотрите возможность отключения корреляции перед записью на рассматриваемое устройство. Однако отключение корреляции приводит к невозможности, что многие классы, не поддерживаемые устройством, будут отображаться в Смир. Кроме того, при использовании корреляции возникает снижение производительности, так как на устройстве запрашиваются сведения о поддерживаемых классах. Если функция корреляции включена, поставщик SNMP вызывает перечисление классов, поддерживаемых устройствами, подобно результату выполнения средства *анализа MIB* .

Например, диспетчер сети хочет узнать несколько ключевых фрагментов данных обо всех управляемых концентраторах в определенной сети. База данных текущих устройств и их поддерживаемые MIB уже существуют, поэтому нет необходимости полагаться на функцию корреляции устройства для предоставления поддерживаемых элементов данных. В этом случае может быть отключена корреляция.

В следующем примере показано, как отключить корреляцию.


```VB
set objNamedValueSet = CreateObject( _
    "WbemScripting.SWbemNamedValueSet") 
objNamedValueSet.Add "Correlate", FALSE, 0
```



С другой стороны, другим диспетчером сети может потребоваться мониторинг всех дисков компьютера UNIX в сети, но не знаком с данными MIB на этих компьютерах. В этом случае будет включена корреляция.
