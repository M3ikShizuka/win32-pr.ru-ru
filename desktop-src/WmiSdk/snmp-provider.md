# <a name="wmi-and-snmp"></a>WMI и SNMP

Поставщик SNMP позволяет клиентским приложениям получать доступ к информации SNMP с помощью инструментарий управления Windows (WMI) (WMI). Поставщик SNMP не устанавливается по умолчанию. Дополнительные сведения об установке поставщика см. в разделе [Настройка среды SNMP WMI](setting-up-the-wmi-snmp-environment.md).

Протокол SNMP использует схему для определения объектов. Схема отличается от схемы, используемой в WMI. Схема CLR и SNMPv2C называется структурой управляющих данных (SMI) и упаковывается в виде файлов MIB. Файлы MIB определяют объекты на стандартном языке — абстрактный синтаксис Notation 1 (ASN. 1), а также ряд определений макросов, которые служат шаблонами для описания объектов. Макросы предоставляют такие сведения, как имя объекта, идентификатор, синтаксис, описание, права доступа, операции протокола и кодировка протокола. В следующей таблице указано, как поставщик SNMP обрабатывает различные аспекты объекта MIB.

| [Макрос типа уведомления](notification-type-macro.md)   | Сопоставление макросов **типа уведомления** из SNMP с WMI.  |
| [Макрос типа OBJECT-TYPE](object-type-macro.md)               | Способ сопоставления макросов **объектов** из SNMP с WMI.        |
| [Макрос ТЕКСТОВОГО соглашения](textual-convention-macro.md) | Сопоставление макросов **ТЕКСТОВОГО соглашения** из SNMP с WMI. |
| [Макрос типа ЛОВУШКи](trap-type-macro.md)                   | Сопоставление макросов **типа ловушек** с SNMP и WMI.          |



 

Поставщик SNMP поставляется с компилятором, который преобразует объекты MIB в объекты WMI. Компилятор SNMP также может выводить сообщение об ошибке или другие сообщения. Дополнительные сведения см. в разделе [сообщения об ошибках компилятора SNMP](snmp-compiler-error-messages.md) и [общие информационные сообщения](general-information-messages.md).
