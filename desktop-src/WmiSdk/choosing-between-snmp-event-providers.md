# <a name="choosing-between-snmp-event-providers"></a>Выбор поставщиков событий SNMP

Поставщики событий SNMP получают пакеты событий SNMP из стека WINSNMP и преобразуют сведения, содержащиеся в пакетах, в события WMI.

WMI включает следующие два поставщика событий SNMP:

-   Поставщик инкапсулированных событий SNMP (СиП)

    Инкапсулированная инициализация означает, что экземпляр события имеет простые свойства, описывающие информацию, сопоставленную непосредственно с SNMP-ловушкой.

-   Поставщик событий SNMP референт (СРЕП)

    Референтная инициализация абстрагирует сведения, присутствующие в ловушке SNMP, чтобы свойства, использующие один и тот же класс и экземпляр, представлялись как внедренные объекты. Это позволяет получить уникальный экземпляр, с которым связана ловушка, после получения события с помощью SNMP \_ \_ релпас.

> [!Note]  
> Дополнительные сведения об установке поставщика см. в разделе [Настройка среды SNMP WMI](setting-up-the-wmi-snmp-environment.md).

 

Независимо от того, является ли событие SNMP незашифрованным или SNMPv2C уведомлением, стек сообщает об этом как уведомление SNMPv2. Поэтому поставщики событий обрабатывают все события SNMP как уведомления SNMPv2.

Для описания событий SNMP для потребителей WMI каждый поставщик событий поддерживает иерархию классов, производных от класса System [**\_ \_ екстринсицевент**](--extrinsicevent.md) . Класс [снмпнотификатион](snmpnotification.md) является родительским классом для иерархии СиП; класс [снмпекстендеднотификатион](snmpextendednotification.md) является родительским классом для иерархии среп.
