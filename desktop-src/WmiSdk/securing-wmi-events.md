# <a name="securing-wmi-events"></a>Защита событий WMI

События WMI доставляются [*поставщиком событий*](gloss-e.md) [*временному*](gloss-t.md) или [*постоянному*](gloss-p.md) потребителю. Система событий WMI использует сравнение [*дескрипторов безопасности*](gloss-s.md) для событий и [*идентификаторов*](gloss-s.md) безопасности учетной записи пользователя для управления доступом к событиям.

События доставляются в виде экземпляра класса событий (как правило, но не обязательно), производного от [**\_ \_ события**](--event.md). Инструментарий WMI предоставляет несколько общих классов событий в [системных классах WMI](wmi-system-classes.md), таких как [**\_ \_ инстанцемодификатионевент**](--instancemodificationevent.md). Поставщики также могут предоставлять собственные классы событий. Например, [поставщик системного реестра](/previous-versions/windows/desktop/regprov/system-registry-provider) имеет классы событий, которые сообщают об изменении раздела или значения реестра, например [**регистрикэйчанжеевент**](/previous-versions/windows/desktop/regprov/registrykeychangeevent).

Следующие разделы содержат сведения о защите доставки событий поставщикам и о получении событий для клиентских сценариев и приложений.

-   [Безопасное предоставление событий](providing-events-securely.md)
-   [Безопасное получение событий](receiving-events-securely.md)
