# <a name="cimtype-qualifier"></a>Квалификатор Цимтипе

Квалификатор **Цимтипе** содержит текст, описывающий тип свойства.

Этот текст может представлять собой MOF-тип, например "String" и "Sint32", или тип CIM, например "CIM \_ String" и "CIM \_ Sint32". Между квалификатором **Цимтипе** и типом свойства, используемым в параметре *пвттипе* метода [**ивбемклассобжект:: Get**](/windows/desktop/api/WbemCli/nf-wbemcli-iwbemclassobject-get) , существует соответствие "один к одному".

Это два исключения:

-   [*Свойства ссылки*](gloss-r.md), имеющие **\_ ссылку на тип CIM**, которая содержит значение "ref: ClassName". Значение ClassName описывает тип класса ссылочного свойства.
-   Свойства внедренного объекта, имеющие тип **\_ объекта CIM** .

Однако обратите внимание, что квалификатор **Цимтипе** может и должен использоваться для более точного ввода ссылочного свойства. Например, если свойство всегда ссылается на экземпляры класса [**Win32 \_ LogicalDisk**](/windows/desktop/CIMWin32Prov/win32-logicaldisk) , его квалификатор **Цимтипе** должен иметь значение:


```mof
"ref:Win32_LogicalDisk"
```



По умолчанию квалификатор **Цимтипе** ссылочного свойства имеет тип **ref**.

Как и в случае со свойствами ссылки, квалификатор **Цимтипе** должен использоваться для ввода свойства внедренного объекта более точно с помощью следующего синтаксиса:


```mof
"object:EmbedClass"
```



Поскольку Инструментарий WMI поддерживает больше типов, чем могут быть выражены стандартными константами с помощью \_ префикса VT, квалификатор **Цимтипе** может помочь интерпретировать значения типа. Квалификатор **Цимтипе** добавляется автоматически. Дополнительные сведения см. в разделе [типы данных MOF](mof-data-types.md).
