# <a name="sending-email-based-on-an-event"></a>Отправка сообщения электронной почты на основе события

С помощью класса [смтпевентконсумер](smtpeventconsumer.md) можно отправить сообщение электронной почты назначенному пользователю при наступлении указанного события. Этот класс является [стандартным потребителем событий](standard-consumer-classes.md) , ПРЕДОСТАВЛЯЕМым инструментарием WMI.

Класс [смтпевентконсумер](smtpeventconsumer.md) требует следующих условий для отправки сообщения электронной почты в ответ на событие:

-   Класс [смтпевентконсумер](smtpeventconsumer.md) должен быть скомпилирован в правильное пространство имен. Дополнительные сведения см. [в разделе Мониторинг и реагирование на события с помощью стандартных потребителей](monitoring-and-responding-to-events-with-standard-consumers.md).
-   SMTP-сервер должен существовать в сети.
-   Сообщение электронной почты не может иметь вложение.
-   Сообщение электронной почты должно быть закодировано в US-ASCII.

Базовая процедура использования стандартных потребителей всегда одинакова и находится в [мониторинге и реагировании на события со стандартными потребителями](monitoring-and-responding-to-events-with-standard-consumers.md). Следующая процедура добавляет к основной процедуре. относится только к классу [смтпевентконсумер](smtpeventconsumer.md) ; и описывает создание объекта-получателя события, отправляющего электронную почту.

В следующей процедуре описывается создание объекта-получателя события, отправляющего электронную почту.

**Создание потребителя событий, отправляющего электронную почту**

1.  При необходимости установите и зарегистрируйте класс [смтпевентконсумер](smtpeventconsumer.md) .

    Класс [смтпевентконсумер](smtpeventconsumer.md) компилируется в \\ пространство имен корневой подписки программой установки WMI.

2.  Найдите событие, которое необходимо отслеживать, и создайте запрос события.

    Возможно, имеется встроенное событие, которое используется для отслеживания события. Большинство внутренних событий связаны с изменениями экземпляров класса в \\ пространстве имен root cimv2. Анализируя классы в справочнике по [классам WMI](wmi-classes.md) , можно, вероятно, найти класс, определяющий событие, которое необходимо отслеживать. Например, используйте класс [**Win32 \_ LogicalDisk**](/windows/desktop/CIMWin32Prov/win32-logicaldisk) для отслеживания изменений на жестком диске.

    Если нет существующих внутренних событий, которые используют, может существовать внешний поставщик событий, который может работать. Например, используйте класс [**регистритричанжеевент**](/previous-versions/windows/desktop/regprov/registrytreechangeevent) поставщика реестра для отслеживания изменений в системном реестре.

    Если класс, определяющий событие, которое необходимо отслеживать, не существует, необходимо создать собственный поставщик событий и определить новые классы внешних событий. Дополнительные сведения см. в разделе [запись поставщика событий](writing-an-event-provider.md).

3.  В файле MOF-файл (MOF) создайте экземпляр [смтпевентконсумер](smtpeventconsumer.md) для получения событий.

    Используйте свойства экземпляра, чтобы определить сообщение электронной почты, которое будет отправлено при возникновении события. Например, свойство **поле ToLine** определяет адрес электронной почты, а свойство **Message** определяет текст сообщения электронной почты. Необходимо определить адрес электронной почты, тему и текст сообщения, но сообщение электронной почты не может содержать вложение. Дополнительные сведения см. в разделе [Конструирование классов MOF-файл (MOF)](designing-managed-object-format--mof--classes.md).

4.  Создайте запрос события, указывающий события, которые необходимо отслеживать.

    Дополнительные сведения см. [в разделе запросы с помощью WQL](querying-with-wql.md).

5.  Создайте экземпляр [**\_ \_ EventFilter**](--eventfilter.md) и сохраните запрос в свойстве **Query** .

    Дополнительные сведения см. в разделе [запросы с помощью WQL](querying-with-wql.md).

6.  Создайте экземпляр [**\_ \_ филтертоконсумербиндинг**](--filtertoconsumerbinding.md) , чтобы связать фильтр и потребитель.
7.  Скомпилируйте MOF-файл с помощью [**Mofcomp.exe**](mofcomp.md).


## <a name="example"></a>Пример

Пример в этом разделе находится в MOF-коде, но экземпляры можно создавать программно с помощью [API скриптов для WMI](scripting-api-for-wmi.md) или [COM API для WMI](com-api-for-wmi.md).

В следующей процедуре описывается использование примера.

**Использование примера**

1.  Скопируйте следующий MOF в текстовый файл и сохраните его с расширением. mof.
2.  В окне командной строки Скомпилируйте MOF-файл с помощью следующей команды:

    **Mofcomp** *filename * * *. mof**

> [!Note]  
> Когда MOF-код компилируется в \\ пространство имен корневой подписки, [смтпевентконсумер](smtpeventconsumer.md) компилируется в то же пространство имен.

 

``` syntax
#pragma namespace ("\\\\.\\root\\subscription")

instance of __EventFilter as $FILTER
{
    Name = "LowDiskspaceFilter";
    
    EventNamespace = "\\\\.\\root\\cimv2";  

    Query = "SELECT * FROM __InstanceModificationEvent WITHIN 10 "
            "WHERE TargetInstance ISA \"Win32_LogicalDisk\" "
            "AND TargetInstance.FreeSpace < 846000000 "
            "AND PreviousInstance.FreeSpace >= 846000000 "
            "AND (TargetInstance.DeviceID = \"C:\" "
            "OR TargetInstance.DeviceID = \"D:\")";
    QueryLanguage = "WQL";
};


instance of SMTPEventConsumer as $CONSUMER
{
    Name = "LowDisk";
    ToLine = "SysAd@MyCompany.com, MyAlias@MyCompany.com";
    CcLine = "MyHome@MyISP.com";
    ReplyToLine = "MyAlias@MyCompany.com";
    SMTPServer = "SmartHost";
    Subject = "WARNING: Low disk space";
    Message = "WARNING: Your %TargetInstance.DeviceID% is"
        " getting dangerously low.";
};

instance of __FilterToConsumerBinding
{
    Consumer = $CONSUMER ;
    Filter = $FILTER ;
};
```
