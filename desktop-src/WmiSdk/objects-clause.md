# <a name="objects-clause"></a>Предложение OBJECTs

Предложение OBJECTs макроса [типа уведомления](notification-type-macro.md) перечисляет набор объектов, связанных с объектом уведомления.

> [!Note]  
> Дополнительные сведения об установке поставщика см. в разделе [Настройка среды SNMP WMI](setting-up-the-wmi-snmp-environment.md).

 

При сопоставлении с классами CIM применяются следующие правила.

-   Каждый член предложения OBJECTs сопоставляется со свойством определения класса CIM. Дескриптор объекта сопоставляется буквально с именем свойства CIM. Например, **ifIndex** преобразуется в **ifIndex**.
-   Каждое свойство CIM, создаваемое процессом сопоставления, содержит квалификатор **варбиндиндекс** .

    **Варбиндиндекс** является обязательным квалификатором **UInt32** , описывающим расположение объекта в том виде, в котором оно отображается в предложении Objects [типа Trap-Type](trap-type-macro.md) или [Notification-Type](notification-type-macro.md) . Целое число также задает расположение свойства в том виде, в котором оно отображается в списке привязок переменных для PDU ловушек SNMPv2C (единица данных протокола) соответственно. (Поскольку первые две привязки переменных всегда имеют метку времени и код, все дополнительные переменные нумеруются после значения 2.)

    Когда класс событий CIM создается из макроса с [типом данных Trap](trap-type-macro.md) , квалификатор **варбиндиндекс** имеет начальное значение 1 для первой переменной в списке привязок переменных.

    Когда класс событий CIM создается из макроса [типа уведомления](notification-type-macro.md) SNMPv2c, квалификатор **варбиндиндекс** имеет начальное значение 3 для первой переменной в списке привязок переменных.

При сопоставлении инкапсулированного класса CIM каждый элемент предложения OBJECTs сопоставляется со свойством CIM, которое отражает имя, тип и значение соответствующего объекта MIB. Используемые процедуры сопоставления указываются в следующих разделах:

-   [Предложение СИНТАКСИСа](syntax-clause.md)
-   [Макрос ТЕКСТОВОГО соглашения](textual-convention-macro.md)
-   [Предложения доступа и MAX-ACCESS](access-and-max-access-clauses.md)

При использовании класса CIM референт каждый член предложения OBJECTs сопоставляется следующим образом:

-   Каждый член предложения OBJECTs сопоставляется с одним свойством класса CIM. Свойство строго типизировано как внедренный объект.
-   Класс внедренного объекта создается с помощью стандартной процедуры сопоставления объектов. Дополнительные сведения см. в разделе [объект-тип макроса](object-type-macro.md). Например, **ифтабле** сопоставляется с внедренным классом с именем **SNMP \_ RFC1213 \_ MIB \_ ифтабле**.
-   Значения, связанные с привязками переменных PDU ЛОВУШКи, создаются как внедренные объекты объекта класса референт. Каждый внедренный объект содержит значения для каждого из ключевых свойств объекта и значение свойства в сопоставляемой привязке переменной.
