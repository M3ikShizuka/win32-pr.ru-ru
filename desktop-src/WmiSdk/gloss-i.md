<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**олицетворения**
</dt> <dd>

Способность потока в одном процессе предположить удостоверение безопасности и привилегии другого процесса.

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**поставщик экземпляров**
</dt> <dd>

COM-сервер, предоставляющий экземпляры классов и реализующий методы интерфейса [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) для поддержки извлечения, изменения, удаления, перечисления или обработки запросов данных.

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**Внутреннее событие**
</dt> <dd>

Событие, предоставляемое инструментарием WMI, которое является уведомлением об изменении стандартной модели данных WMI. Например, создание нового экземпляра [**\_ процесса Win32**](/windows/desktop/CIMWin32Prov/win32-process) — это [**\_ \_ инстанцекреатионевент**](--instancecreationevent.md) тип внутреннего события. См. также [*внешние события*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**инструментирования**
</dt> <dd>

Механизм, используемый для сообщения о состоянии оборудования и программного обеспечения системы. Инструментарий WMI реализует инструментирование, моделирующее систему с помощью классов на основе [*модель CIM (CIM)*](gloss-c.md). Инструментарий WMI в платформа .NET Framework создает инструментированные классы, добавляя атрибуты.

</dd> </dl>
