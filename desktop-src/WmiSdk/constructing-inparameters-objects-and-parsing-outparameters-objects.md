# <a name="constructing-inparameters-objects-and-parsing-outparameters-objects"></a>Построение объектов параметров и анализ объектов параметров

Как правило, прямой доступ подходит для вызова [*метода поставщика*](gloss-p.md)WMI. Прямой доступ означает исполнение метода с помощью синтаксиса *Object. Method* . Однако в некоторых случаях нельзя использовать прямой доступ. Кроме того, для асинхронного вызова метода поставщика из скрипта требуется тип вызова [**ексекмесодасинк**](swbemobject-execmethodasync-.md) .

> [!Note]  
> Поскольку обратный вызов приемника может не возвращаться на том же уровне проверки подлинности, что и клиент, рекомендуется использовать семисинчронаус вместо асинхронного взаимодействия. Дополнительные сведения см. [в разделе вызов метода](calling-a-method.md).

 

Порядок входных и выходных параметров метода определяется в схеме MOF-файл (MOF) для метода. Инструментарий WMI не предотвращает изменение порядка параметров при повторной компиляции класса с помощью [mofcomp](mofcomp.md). Используя объект [**параметров**](swbemmethod-inparameters.md) , можно избежать проблем, вызванных измененной схемой, поскольку входные параметры идентифицируются по имени. Правильный параметр можно увидеть, изучив квалификатор **идентификатора** каждого входного параметра. Первый параметр имеет значение **ID** , равное 0 (нулю).

Методы [**SWbemObject.Exeкмесод \_**](swbemobject-execmethod-.md), [**SWbemObject.Exeкмесодасинк \_**](swbemobject-execmethodasync-.md), [**SWbemServices.Exeкмесод**](swbemservices-execmethod.md)и [**SWbemServices.Exeкмесодасинк**](swbemservices-execmethodasync.md) предоставляют альтернативный способ выполнения метода поставщика в случаях, когда невозможно выполнить метод напрямую. Дополнительные сведения см. в разделе [Управление сведениями о классе и экземпляре](manipulating-class-and-instance-information.md).

Дополнительные сведения о параметрах см. в разделе [Построение объектов](constructing-inparameters-objects.md) параметров и [анализ объектов параметров](parsing-outparameters-objects.md).
