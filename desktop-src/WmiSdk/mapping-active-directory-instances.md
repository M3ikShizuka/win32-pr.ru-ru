# <a name="mapping-active-directory-instances"></a>Сопоставление экземпляров Active Directory

В общем случае каждый объект Active Directory сопоставляется только с одним экземпляром WMI. Класс WMI, соответствующий экземпляру WMI, аналогичен классу, предоставленному поставщиком класса, из соответствующего класса Active Directory. Ключевое свойство **адсипас** каждого экземпляра заполняется с помощью пути ADSI объекта.

В этом разделе обсуждаются следующие разделы:

-   [Сопоставление пространств имен](#mapping-namespaces)
-   [Сопоставление значений атрибутов](#mapping-attribute-values)
-   [Сопоставление ассоциаций экземпляров](#mapping-instance-associations)

> [!Note]  
> Дополнительные сведения о поддержке и установке этого компонента в определенной операционной системе см. в статье [доступность компонентов WMI в операционной системе](operating-system-availability-of-wmi-components.md).

 

## <a name="mapping-namespaces"></a>Сопоставление пространств имен

Каждое пространство имен в ADSI сопоставляет один к одному пространствам имен в \\ \\ пространстве имен корневого каталога WMI. Имя пространства имен WMI совпадает со значением **ProgID** поставщика служб каталогов, предоставляющего пространство имен. В частности, Active Directory сопоставляется с \\ пространством имен LDAP в \\ \\ пространстве имен корневого каталога. Инструментарий WMI создает \\ пространство имен LDAP как часть процесса регистрации поставщика класса.
