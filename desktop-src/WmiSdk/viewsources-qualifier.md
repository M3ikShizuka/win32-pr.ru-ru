# <a name="viewsources-qualifier"></a>Квалификатор Виевсаурцес

Все классы представлений должны иметь квалификатор массива строк с именем **виевсаурцес**. Квалификатор **виевсаурцес** содержит исходные запросы, определяющие исходные экземпляры, используемые в классе представления. Значение квалификатора **виевсаурцес** — это строковый массив, содержащий запросы [*язык запросов WMI (WQL)*](gloss-w.md) . Можно определить исходные классы и ограничить экземпляры источника, используемые классом представления, с [запросом с помощью](querying-with-wql.md)[предложения WHERE](where-clause.md) WQL для создания отфильтрованного представления.

[Поставщик представления](view-provider.md) сопоставляет исходные запросы в квалификаторе **виевсаурцес** с пространствами имен, перечисленными в [квалификаторе виевспацес](viewspaces-qualifier.md) в том порядке, в котором перечислены запросы и пространства имен. Число исходных запросов должно совпадать с количеством пространств имен, перечисленных в квалификаторе Виевспацес. Порядок, в котором перечисляются исходные запросы, определяет пространства имен, из которых рисуются исходные экземпляры.

В следующем примере выбираются только экземпляры класса **локалдиск** , где значение свойства **FILESYSTEM** равно "NTFS", и экземпляры класса **Ремотедиск** , где значение свойства **Freespace** превышает 45 МБ.


```sql
ViewSources{
"SELECT __Namespace, 
   Description, 
   DeviceID, 
   FileSystem, 
   FreeSpace, 
   VolumeName FROM LocalDisk 
 WHERE FileSystem = \"NTFS\"", 
   "SELECT __Namespace, 
   Description,
   DeviceID, 
   FileSystem, 
   FreeSpace, 
   VolumeName FROM RemoteDisk 
 WHERE FreeSpace > 45000000"}
```



> [!Note]  
> Количество исходных запросов, которые можно определить для классов представления соединения, зависит от числа экземпляров, возвращаемых этими запросами, и от того, сколько способов можно объединить с этими экземплярами. Количество возможных сочетаний экземпляров исходного кода для классов представления растет экспоненциально, поэтому необходимо как можно более быстро принимать исходные запросы на соединение с классами представления.
