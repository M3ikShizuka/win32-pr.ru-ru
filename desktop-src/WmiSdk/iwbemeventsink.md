# <a name="iwbemeventsink-interface"></a>Интерфейс Ивбемевентсинк

Интерфейс **ивбемевентсинк** инициирует связь с поставщиком событий с помощью ограниченного набора запросов. Этот интерфейс расширяет [**ивбемобжектсинк**](iwbemobjectsink.md), предоставляя новые методы, работающие с безопасностью и производительностью. Дополнительные сведения об использовании этого интерфейса см. в разделе [написание поставщика событий](writing-an-event-provider.md) и [обеспечение безопасности событий WMI](securing-wmi-events.md).

## <a name="members"></a>Элементы

Интерфейс **ивбемевентсинк** имеет следующие типы членов:

-   [Методы](#methods)

### <a name="methods"></a>Методы

Интерфейс **ивбемевентсинк** содержит следующие методы.



| Метод                                                                | Описание                                                           |
|:----------------------------------------------------------------------|:----------------------------------------------------------------------|
| [**жетрестриктедсинк**](/windows/desktop/api/Wbemprov/nf-wbemprov-iwbemeventsink-getrestrictedsink)         | Вызывается потребителем для настройки запросов с ограниченными событиями.<br/> |
| [**IsActive**](/windows/desktop/api/Wbemprov/nf-wbemprov-iwbemeventsink-isactive)                           | Проверяет состояние приемника событий.<br/>                               |
| [**сетбатчингпараметерс**](/windows/desktop/api/Wbemprov/nf-wbemprov-iwbemeventsink-setbatchingparameters) | Вызывается потребителем для задания пакетных параметров.<br/>         |
| [**сетсинксекурити**](/windows/desktop/api/Wbemprov/nf-wbemprov-iwbemeventsink-setsinksecurity)             | Используется для обновления дескриптора безопасности в приемнике событий.<br/>   |



 

## <a name="remarks"></a>Комментарии

При реализации приемника подписки на события ([**ивбемобжектсинк**](iwbemobjectsink.md) или **ивбемевентсинк**) не следует вызывать WMI из методов объекта приемника. Например, вызов [**IWbemServices:: канцеласинккалл**](/windows/desktop/api/WbemCli/nf-wbemcli-iwbemservices-cancelasynccall) для отмены приемника из реализации [**Ивбемевентсинк:: сетсинксекурити**](/windows/desktop/api/Wbemprov/nf-wbemprov-iwbemeventsink-setsinksecurity) может повлиять на состояние WMI. Чтобы отменить подписку на события, установите флаг и вызовите **IWbemServices:: канцеласинккалл** из другого потока или объекта. Для реализаций, которые не связаны с приемником событий, например объектом, перечислением и получением запросов, можно выполнить обратный вызов к инструментарию WMI.

Реализации приемника должны обрабатывать уведомление о событии в течение 100 мс, поскольку поток WMI, доставляющий уведомление о событии, не может выполнить другие действия, пока объект приемника не завершит обработку. Если для уведомления требуется большой объем обработки, приемник может использовать внутреннюю очередь для другого потока, чтобы обработать обработку.
