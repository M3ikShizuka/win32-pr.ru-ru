# <a name="creating-a-timer-event-with-__timerinstruction"></a>Создание события таймера с помощью \_ \_ тимеринструктион

Событие таймера создается путем создания экземпляра классов, производных от класса [**\_ \_ тимеринструктион**](--timerinstruction.md) в любом пространстве имен WMI. Затем Инструментарий WMI создает событие таймера в соответствующее время. Если вы пропустили событие таймера из-за простоя компьютера, Инструментарий WMI сообщит о пропущенном событии. Инструментарий WMI поддерживает события таймера для обеспечения обратной совместимости и в тех случаях, когда необходимо выяснить, сколько событий было пропущено с момента последнего доставленного события. Однако для большинства событий таймера следует создать фильтр событий для [**Win32 \_ localtime**](/previous-versions/windows/desktop/wmitimepprov/win32-localtime) или [**Win32 \_ утктиме**](/previous-versions/windows/desktop/wmitimepprov/win32-utctime). Дополнительные сведения см. в разделе [Создание события таймера с помощью Win32 \_ LocalTime или Win32 \_ утктиме](creating-a-timer-event-with-win32-localtime-or-win32-utctime.md).

В следующей процедуре описывается создание и получение события таймера с помощью \_ \_ тимеринструктион.

**Создание и получение события таймера с помощью \_ \_ тимеринструктион**

1.  Создайте экземпляр классов [**\_ \_ абсолутетимеринструктион**](--absolutetimerinstruction.md) или [**\_ \_ интервалтимеринструктион**](--intervaltimerinstruction.md) .

    Классы [**\_ \_ абсолутетимеринструктион**](--absolutetimerinstruction.md) и [**\_ \_ интервалтимеринструктион**](--intervaltimerinstruction.md) являются производными от класса [**\_ \_ тимеринструктион**](--timerinstruction.md) , который содержит уникальную строку, назначенную разработчиком, которая определяет тип события таймера. Класс **\_ \_ тимеринструктион** также содержит значение, указывающее, должен ли Инструментарий WMI отправить уведомление белатед, если событие Timer возникает при недоступности инструментария WMI.

    Используйте [**\_ \_ абсолутетимеринструктион**](--absolutetimerinstruction.md) для отправки абсолютных событий таймера, которые происходят в указанную дату в определенное время. Используйте [**\_ \_ интервалтимеринструктион**](--intervaltimerinstruction.md) для отправки событий интервала таймера, которые происходят на регулярной основе.

2.  Настройте приложение для получения экземпляра [**\_ \_ тимеревент**](--timerevent.md) .

    Чтобы создать событие, WMI создает экземпляр класса [**\_ \_ тимеревент**](--timerevent.md) и пересылает экземпляр потребителю. Экземпляр **\_ \_ тимеревент** содержит идентификатор инструкции таймера от потребителя. Экземпляр также содержит значение, указывающее, сколько раз WMI должен отправить уведомление о событии таймера в течение любого интервала, когда WMI не может связаться с потребителем.
