# <a name="describing-a-qualifier-with-a-qualifier-flavor"></a>Описание квалификатора с флагом квалификатора

[*Разновидность квалификатора*](gloss-q.md) — это флаг, описывающий дополнительные сведения об квалификаторе. Например, ограничение ограниченного квалификатора указывает, что инструментарий WMI не должен распространять связанный квалификатор на все производные классы или экземпляры. Установить флаги можно с помощью кода MOF или программно. Несмотря на то, что можно описать разнообразные эффекты с помощью разновидностей, основное назначение флагов разновидностей заключается в том, чтобы определить, как WMI распространяет квалификаторы с помощью наследования.

Инструментарий WMI определяет несколько разновидностей квалификаторов, которые можно присоединить к любому квалификатору независимо от источника квалификатора. Однако некоторые разновидности не подходят для всех типов квалификаторов. Например, флаг **ToSubClass** подходит только для квалификаторов, определенных для класса. Нельзя присоединить **ToSubClass** к квалификатору, используемому для описания экземпляра.

С помощью разновидностей можно описать различные эффекты для квалификаторов. Например, версия может указывать, можно ли локализовать квалификатор. Однако одна из основных целей разновидности квалификатора заключается в том, чтобы описать, может ли родительский класс передавать квалификаторы в класс или экземпляр класса. Кроме того, с помощью флагов можно определить, передает ли свойство класса квалификатор в свойство экземпляра. Наконец, используйте флаги, чтобы указать, может ли подкласс переопределять исходное значение унаследованного квалификатора. Однако квалификаторы, объявляемые для класса или экземпляра, не распространяются на свойства этого класса или экземпляра. Кроме того, разновидности, устанавливающие разрешения на переопределение, допустимы только в том случае, если заданы флаги **тоинстанце** или **ToSubClass** .

Разновидность можно глобально назначить квалификатору для всего MOF-файла с помощью следующего синтаксиса, в котором пробел действует как разделитель, если задано несколько разновидностей.

``` syntax
Qualifier QualifierName : flavor1 <flavor2...>;
```

Глобальные разновидности применяются ко всем последующим применениям квалификатора в MOF-файле. Глобальные инструкции типа могут находиться в файле за пределами блока объявления объекта. Флаги, переопределенные на уровне класса, экземпляра или свойства, переопределяют глобальные объявления разновидностей для области этого объекта.

Невозможно определить новую разновидность. Хотя можно создать новый квалификатор, используйте только существующие [разновидности квалификаторов](qualifier-flavors.md) для описания нового квалификатора.

**Определение разновидностей квалификаторов в MOF**

-   Объявите флаги, описывающие заданный квалификатор после имени квалификатора, между квадратными скобками. Используйте пробелы в качестве разделителя между несколькими разновидностями.

    В следующем примере показан шаблон для присоединения предопределенных описателей.

    ``` syntax
    [qualifier1 : flavor1 flavor2 flavor3, qualifier2 : flavor1]
    ```

Атрибуты квалификатора можно добавлять программным способом только в C++. Эта операция недоступна в [API скриптов для WMI](scripting-api-for-wmi.md), хотя можно добавить новый квалификатор, вызвав [**свбемкуалифиерсет. Add**](swbemqualifierset-add.md).

**Назначение разновидности с помощью C++**

-   Вызовите метод [**ивбемкуалифиерсет::P UT**](/windows/desktop/api/Wbemcli/nf-wbemcli-iwbemqualifierset-put) с параметром *лфлавор* , для которого задана одна из констант, определенных для метода.
