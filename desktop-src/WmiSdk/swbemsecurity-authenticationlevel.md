# <a name="swbemsecurityauthenticationlevel-property"></a>Свбемсекурити. Аусентикатионлевел, свойство

Свойство **аусентикатионлевел** — это целое число, определяющее уровень проверки подлинности COM, назначенный этому объекту. Этот параметр определяет, как вы защищаете данные, отправляемые из WMI. Дополнительные сведения об уровнях проверки подлинности см. в разделе [Установка параметров \_ \_ безопасности процессов для клиентских приложений](setting-client-application-process-security.md). Как правило, при выполнении вызовов API WMI не требуется устанавливать уровень проверки подлинности. Если это свойство не задано, используется уровень проверки подлинности COM по умолчанию для системы.

Описание этого синтаксиса см. в разделе [соглашения о документе для API скриптов](document-conventions-for-the-scripting-api.md).

Это свойство доступно для чтения и записи.

## <a name="syntax"></a>Синтаксис


```VB
SWbemSecurity.AuthenticationLevel As Integer
```



## <a name="property-value"></a>Значение свойства

## <a name="remarks"></a>Комментарии

Параметр Аусентикатионлевел позволяет запрашивать уровень проверки подлинности и конфиденциальности DCOM для использования в рамках всего соединения. Параметры находятся в диапазоне от без проверки подлинности до зашифрованной проверки подлинности для каждого пакета.



| Значение        | Описание                                                                                                                                                                                                                                                                                                            |
|--------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Нет         | Не использует проверку подлинности. Все параметры безопасности игнорируются.<br/>                                                                                                                                                                                                                                         |
| Значение по умолчанию      | Использует стандартное согласование безопасности для выбора уровня проверки подлинности. Это рекомендуемый параметр, так как клиент, вовлеченный в транзакцию, будет согласовываться с уровнем проверки подлинности, указанным на сервере.<br/> DCOM не будет выбирать значение None во время сеанса согласования.<br/> |
| Подключение      | Проверяет подлинность учетных данных клиента только в том случае, если клиент пытается подключиться к серверу. После установки соединения дополнительные проверки подлинности не выполняются.<br/>                                                                                                                          |
| Вызов         | Проверяет подлинность учетных данных клиента только в начале каждого вызова, когда сервер получает запрос. Заголовки пакетов подписываются, но пакеты данных, которыми обмениваются клиент и сервер, не подписываются и не шифруются.<br/>                                                     |
| PKT          | Проверяет, что все пакеты данных получены от ожидаемого клиента. Аналогично вызову; заголовки пакетов подписываются, но не шифруются. Сами пакеты не подписаны и не зашифрованы.<br/>                                                                                                               |
| пктинтегрити | Проверка подлинности и проверка того, что ни один из пакетов данных, передаваемых между клиентом и сервером, не был изменен. Каждый пакет данных подписывается, гарантируя, что пакеты не были изменены во время передачи. Ни один из пакетов данных не зашифрован.<br/>                                            |
| пктприваци   | Проверяет подлинность всех предыдущих уровней олицетворения и подписывает и шифрует каждый пакет данных. Это гарантирует, что весь обмен данными между клиентом и сервером будет конфиденциальным.<br/>                                                                                                                             |



 

Можно задать уровень проверки подлинности объектов [**SwbemServices**](swbemservices.md), [**SWbemObject**](swbemobject.md), [**SWbemObjectSet**](swbemobjectset.md), [**Свбемобжектпас**](swbemobjectpath.md)и [**SwbemLocator**](swbemlocator.md) , задав для свойства **аусентикатионлевел** нужное значение.

В следующем примере показано, как задать уровень проверки подлинности для объекта **SWbemObject** .


```VB
objinstance.Security_.AuthenticationLevel = wbemAuthenticationLevelPkt
```



Можно также указать уровни проверки подлинности как часть моникера. В следующем примере задается уровень проверки подлинности и уровень олицетворения, а также извлекается экземпляр [**Win32 \_ LogicalDisk**](/windows/desktop/CIMWin32Prov/win32-logicaldisk).


```VB
Set objinst = GetObject("WinMgmts:{impersonationLevel=impersonate,authenticationLevel=pktPrivacy}!root/cimv2:Win32_LogicalDisk='c:'")
```
