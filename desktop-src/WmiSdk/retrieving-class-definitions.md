# <a name="retrieving-class-definitions"></a>Получение определений классов

Запросы схемы — это инструкции WQL, которые запрашивают определения классов и сведения о [связях схем](schema-associations.md). Поставщики классов используют запросы схемы в своих экземплярах класса [**\_ \_ класспровидеррегистратион**](--classproviderregistration.md) для указания классов, которые они поддерживают при регистрации в WMI. Запросы схемы размещаются в свойствах **ресултсеткуериес**, **референцедсеткуериес** и **унсуппортедкуериес** экземпляра **\_ \_ класспровидеррегистратион** .

Запросы схемы похожи на запросы данных в том, что они поддерживают следующие инструкции WQL:

-   [SELECT](select-statement-for-schema-queries.md)
-   [СОЕДИНИТЕЛи](schema-associations.md)
-   [ССЫЛКИ НА](schema-associations.md)
-   [ISA](isa-operator-for-schema-queries.md)

Запрос схемы похож на [ссылку](references-of-statement.md) на запрос данных, указывающий ключевое слово **классдефсонли** . Иными словами, он возвращает результирующий набор объектов определения класса, а не фактические экземпляры классов взаимосвязей. Однако ссылки на определения класса возвращаются только при наличии экземпляров. Запрос схемы возвращает определения классов независимо от наличия экземпляров.
