# <a name="__timerevent-class"></a>\_\_Класс Тимеревент

Системный класс **\_ \_ тимеревент** сообщает о событии, формируемом WMI, в ответ на запрос получателя о событии интервала таймера или на абсолютное событие таймера. Событие интервала времени — это событие, которое происходит через регулярные интервалы времени. Абсолютное событие таймера — это событие, возникающее в определенное время. События таймера могут возникать в любом пространстве имен.

Приведенный ниже синтаксис является упрощенной версией кода MOF и включает все унаследованные свойства. Свойства перечислены в алфавитном порядке, а не в MOF.

## <a name="syntax"></a>Синтаксис

``` syntax
class __TimerEvent : __Event
{
  uint32 NumFirings;
  uint8  SECURITY_DESCRIPTOR[];
  string TimerId;
  uint64 TIME_CREATED;
};
```

## <a name="remarks"></a>Комментарии

Класс **\_ \_ тимеревент** является производным от [**\_ \_ события**](--event.md).

Потребители событий регистрируются для выполнения абсолютного события таймера путем создания экземпляра класса System [**\_ \_ абсолутетимеринструктион**](--absolutetimerinstruction.md) . Они регистрируются для события интервала времени путем создания экземпляра класса System [**\_ \_ интервалтимеринструктион**](--intervaltimerinstruction.md) .

Во время нормальной работы свойство **нумфирингс** имеет значение 1. Если доступ к потребителю невозможен, а интервал срабатывания намного быстрее, чем возможность доставки события, для **нумфирингс** устанавливается значение больше 1. Если **нумфирингс** больше 1, WMI автоматически объединяет многие события таймера в одно и то же событие. Такое объединение аналогично тому, что происходит с сообщениями [**\_ таймера WM**](/windows/desktop/winmsg/wm-timer) в программировании Windows.
