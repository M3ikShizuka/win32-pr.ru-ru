# <a name="iswbemsinkeventsonobjectready-event"></a>Событие Исвбемсинкевентс:: Онобжектреади

Событие **онобжектреади** объекта [**свбемсинк**](swbemsink.md) активируется, когда асинхронная операция возвращает объект. Это событие используется для обработки объектов из асинхронных вызовов, таких как [**SWbemObject \_ . инстанцесасинк**](swbemobject-instancesasync-.md) или [**SWbemServices.Exeккуерясинк**](swbemservices-execqueryasync.md). **Онобжектреади** возвращает один [**SWbemObject**](swbemobject.md) каждый раз, пока перечисление не будет завершено.

Описание этого синтаксиса см. в разделе [соглашения о документе для API скриптов](document-conventions-for-the-scripting-api.md).

## <a name="syntax"></a>Синтаксис


```VB
SWbemSink.OnObjectReady( _
  ByVal objWbemObject, _
  ByVal objWbemAsyncContext _
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*обжвбемобжект* 
</dt> <dd>

Объект [**SWbemObject**](swbemobject.md) . Это похоже на то, что возвращается синхронным эквивалентом асинхронного вызова, который запускает это событие. Например, вызов метода [**SwbemServices. Async**](swbemservices-getasync.md) возвращает **SWbemObject** в параметре *обжвбемобжект* события **онобжектреади** объекта [**свбемсинк**](swbemsink.md) , который передается как параметр *обжвбемобжект* исходного вызова. Один и тот же объект **SWbemObject** можно получить с помощью эквивалентного синхронного вызова [**SwbemServices. Get**](swbemservices-get.md).

</dd> <dt>

*обжвбемасинкконтекст* 
</dt> <dd>

Объект [**свбемнамедвалуесет**](swbemnamedvalueset.md) , который передается в исходный асинхронный вызов. Используйте этот параметр, чтобы указать происхождение асинхронного вызова, который активирует это событие, когда с помощью этого приемника объектов выполняется несколько асинхронных вызовов.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Это событие не возвращает значение.

## <a name="remarks"></a>Комментарии

Асинхронный обратный вызов позволяет пользователю без проверки подлинности предоставлять данные в приемник. Это создает угрозы безопасности для сценариев и приложений. Чтобы устранить риски, используйте либо частично синхронное соединение, либо синхронное взаимодействие. Дополнительные сведения см. [в разделе вызов метода](calling-a-method.md).
