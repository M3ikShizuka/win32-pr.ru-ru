# <a name="accessing-an-unnamed-registry-value"></a>Доступ к неименованному значению реестра

По умолчанию или безымянное значение раздела реестра отображается как (по умолчанию) или <No Name> в редакторе реестра Regedit. Для доступа к неименованному разделу реестра можно использовать поставщик системного реестра. Аналогично, можно также использовать поставщик системного реестра для доступа к описаниям точечных рисунков, которые определены как безымянные значения.

В следующей процедуре описывается, как получить неименованное значение реестра.

**Получение неименованного значения реестра**

1.  Определите свойство и задайте в качестве квалификатора **пропертиконтекст** этого свойства пустую строку.

    В следующем примере кода показано, как класс определяет свойства для хранения значений ключа, заданного квалификатором **классконтекст** . Значение по умолчанию хранится в свойстве **по умолчанию** .

    ``` syntax
    [dynamic, 
     provider("RegProv"), 
     ClassContext("local|hkey_local_machine\\software\\"
     "microsoft\\Active Setup\\Installed Components")]

    class RegTrans{
      [key] String Transports="";
      [PropertyContext("")] String Default;
      [PropertyContext("ComponentId")] String ComponentID;
      [PropertyContext("Locale")] String Locale;
    };
    ```

    Ключ транспортов не использует безымянное значение, поэтому компиляция этого MOF-файла не создает никакого значения для свойства **по умолчанию** , если только средство редактирования реестра не используется для изменения безымянного значения.

2.  Для файла точечного рисунка определите свойство и задайте **пропертиконтекст** этого свойства.

    В следующем примере кода показано, как определить свойство.

    ```mof
    Local|hkey_classes_root\\.bmp
    ```
