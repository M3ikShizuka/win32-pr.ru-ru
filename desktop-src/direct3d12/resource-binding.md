# <a name="resource-binding"></a>Привязка ресурсов

Привязка — это процесс связывания объектов ресурсов с шейдерами графического конвейера.

| Раздел | Описание |
|-|-|
| [Общие сведения о привязке ресурсов](resource-binding-flow-of-control.md) | Ключом к привязке ресурсов в DirectX 12 являются понятия *дескриптора*, *таблиц дескрипторов*, *куч дескрипторов* и *корневой подписи*. |
| [Различия в модели привязки от Direct3D 11](binding-model.md) | Одним из основных решений по проектированию привязок DirectX12 является отделение ее от других задач управления. Это помещает некоторые требования к приложению для управления определенными потенциальными причинами опасности. |
| [Дескрипторы](descriptors.md) | Дескрипторы являются основной единицей привязки для одного ресурса в D3D12. |
| [Кучи дескрипторов](descriptor-heaps.md) | Куча дескрипторов — это набор смежных выделений дескрипторов, по одному выделению для каждого из них. |
| [Таблицы дескрипторов](descriptor-tables.md) | Таблица дескрипторов логически представляет собой набор дескрипторов. |

| [Запрашивание возможностей](capability-querying.md) | Приложение может обнаружить уровень поддержки привязки ресурсов и многие другие функции с помощью вызова [**ID3D12Device:: чеккфеатуресуппорт**](/windows/desktop/api/d3d12/nf-d3d12-id3d12device-checkfeaturesupport). |
| [Привязка ресурсов в HLSL](resource-binding-in-hlsl.md) | В этом разделе описываются некоторые особенности использования [модели шейдера](/windows/desktop/direct3dhlsl/shader-model-5-1) высокого уровня Shader (HLSL) 5,1 с Direct3D 12. Все оборудование Direct3D 12 поддерживает модель шейдера 5,1, поэтому поддержка этой модели не зависит от уровня компонентов оборудования. |
| [Оптимизация UMA: доступные ЦП текстуры и стандартное преобразование ссылок при перемещении данных с внешнего устройства](default-texture-mapping.md) | Графические процессоры с архитектурой "Универсальная память" (для архитектуры) предлагают некоторые эффективные преимущества по сравнению с отдельными GPU, особенно при оптимизации для мобильных устройств. Предоставление ресурсов доступа к ЦП при наличии GPU может уменьшить объем копируемых данных между ЦП и GPU. Хотя мы не рекомендуем, чтобы приложения применялись к ресурсам ЦП для всех ресурсов в области разработки, существуют возможности повысить эффективность, обеспечивая доступ к нужным ресурсам ЦП. В отличие от дискретных графических процессоров, в ЦП технически может быть указатель на все ресурсы, к которым может получить доступ GPU. |

| [Объемные плиточные ресурсы](volume-tiled-resources.md) | Объемные (трехмерные) текстуры можно использовать в качестве мозаичных ресурсов, отметив, что разрешение плитки состоит из трех измерений. |
| [Подресурсы](subresources.md) | Описывает разделение ресурса на подресурсы и ссылку на один, несколько или срез подресурсов. |
