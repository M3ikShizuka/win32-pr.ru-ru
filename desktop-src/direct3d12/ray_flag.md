# <a name="ray_flag-enumeration"></a>\_Перечисление флагов лучей

Флаги, передаваемые функции [**трацерай**](traceray-function.md) для переопределения прозрачности, отбора и поведения при начальном выходе.

## <a name="syntax"></a>Синтаксис


```
enum RAY_FLAG : uint
{
    RAY_FLAG_NONE                            = 0x00,
    RAY_FLAG_FORCE_OPAQUE                    = 0x01,
    RAY_FLAG_FORCE_NON_OPAQUE                = 0x02,
    RAY_FLAG_ACCEPT_FIRST_HIT_AND_END_SEARCH = 0x04,
    RAY_FLAG_SKIP_CLOSEST_HIT_SHADER         = 0x08,
    RAY_FLAG_CULL_BACK_FACING_TRIANGLES      = 0x10,
    RAY_FLAG_CULL_FRONT_FACING_TRIANGLES     = 0x20,
    RAY_FLAG_CULL_OPAQUE                     = 0x40,
    RAY_FLAG_CULL_NON_OPAQUE                 = 0x80,
}; 
```

<dl> <dt>

<span id="RAY_FLAG_FORCE_NON_OPAQUE"></span><span id="ray_flag_force_non_opaque"></span>**\_флаг луча \_ Force \_ не \_ прозрачный**

Все пересечения лучей-примитивов, обнаруженные в райтраце, считаются не непрозрачными. </dd> <dt>

<span id="RAY_FLAG_ACCEPT_FIRST_HIT_AND_END_SEARCH"></span><span id="ray_flag_accept_first_hit_and_end_search"></span>**\_флаг луча \_ допускает \_ первое \_ Нажатие \_ и \_ Завершение \_ поиска**
</dt> <dd>

Первое пересечение-примитив лучей, обнаруженное в райтраце, автоматически вызывает **акцепситандендсеарч** сразу после любого шейдера попадания, в том числе при отсутствии шейдера попаданий. 
 
Единственным исключением является ситуация, когда предшествующий любой шейдер вызывает **игнорехит**, в этом случае, когда луч остается нетронутым, так что следующее попадание становится еще одним кандидатом.  Чтобы это исключение было применено, любой шейдер нажатий должен быть выполнен.  Таким образом, если любой шейдер нажатий пропускается, так как попадание считается непрозрачным (например, из-за флага луч Force или непрозрачного флага \_ \_ \_ D3D12 райтраЦинг или непрозрачного \_ \_ \_ \_ D3D12 \_ \_ \_ флага экземпляра райтраЦинг \_ ), вызывается **акцепситандендсеарч** .

Если при первом попадании появится ближайшее построитель текстуры, он вызывается, если \_ также имеется флаг луча, \_ пропуская \_ ближайший \_ \_ построитель текстуры.  Найденное совпадение считается ближайшим, даже несмотря на то, что другие потенциальные попадания, которые могут быть ближе к луча, могут быть не посещены.

Обычно этот флаг используется для теней, где необходимо найти только одно нажатие.


</dd> <dt>

<span id="RAY_FLAG_SKIP_CLOSEST_HIT_SHADER"></span><span id="ray_flag_skip_closest_hit_shader"></span>**\_флаг луча \_ пропуск \_ ближайшего \_ \_ шейдера попаданий**
</dt> <dd>
