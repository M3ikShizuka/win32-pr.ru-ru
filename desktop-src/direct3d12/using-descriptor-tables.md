# <a name="using-descriptor-tables"></a>Использование таблиц дескрипторов

Таблицы дескрипторов, каждый из которых идентифицирует диапазон в куче дескрипторов, привязываются к слотам, определенным текущей корневой подписью в списке команд.

-   [Таблицы дескрипторов индексации](#indexing-descriptor-tables)
-   [Связанные темы](#related-topics)

Шейдеры могут искать ресурсы, на которые ссылаются дескрипторы, составляющие таблицу индексов. Другие привязки ресурсов — буферы индексов, буфер вершин, буферы вывода потока, целевые объекты отрисовки и трафарет глубины выполняются непосредственно в списке команд, а не с помощью дескрипторов. Подведение итогов.

Следующие ссылки на ресурсы могут совместно использовать одну и ту же таблицу дескрипторов и кучу:

-   Представления ресурсов шейдера
-   Представления неупорядоченного доступа
-   Представления буфера констант

Следующие ссылки на ресурсы должны находиться в собственной куче дескрипторов:

-   Образцы.

Следующие ресурсы не помещаются в таблицы дескрипторов или кучи, но привязываются напрямую с помощью списков команд:

-   Буферы индексов
-   Буферы вершин
-   Потоковые буферы вывода
-   Целевые объекты рендеринга
-   Представления трафарета глубины

## <a name="indexing-descriptor-tables"></a>Таблицы дескрипторов индексации

Шейдеры не могут динамически индексироваться между границами таблиц дескрипторов с данного узла вызова в шейдере. Однако выбор дескриптора в таблице дескрипторов может динамически индексироваться в коде шейдера в пределах диапазонов одного и того же типа дескриптора (например, индексирование по смежной области СРВС).
