# <a name="dns-wmi-provider-overview"></a>Общие сведения о поставщике WMI DNS

поставщик является элементом архитектуры *инструментарий управления Windows (WMI) (WMI)*. WMI определяет единую архитектуру для описания, доступа и инструментирования объектов. Частью этой архитектуры является большая база данных классов WMI, используемых для выполнения задач удаленного управления на конкретных объектах.

Поставщики WMI действуют как посредники между WMI и одним или несколькими управляемыми объектами. Когда WMI получает запрос от приложения управления для данных, недоступных из репозитория CIM, или для уведомлений о событиях, которые не поддерживаются инструментарием WMI, он перенаправляет запрос поставщику. Поставщики предоставляют данные и уведомления о событиях для управляемых объектов, которые находятся в их домене. Поставщик расширяет схему WMI классов, позволяя инструментарию WMI работать с новыми типами объектов. Поставщик WMI для DNS определяет классы для запроса и настройки DNS-сервера, а также связанные с ним зоны DNS и записи DNS.

Поставщик WMI для DNS предоставляет клиентам несколько объектов DNS, включая DNS-сервер, домен DNS и объекты RR DNS. С помощью этих объектов клиенты могут выполнять действия по управлению DNS.

С помощью поставщика WMI для DNS можно создавать собственные средства для выполнения большинства задач администрирования DNS-сервера. Например, можно создавать, удалять и просматривать зоны и записи. Сброс свойств сервера и зоны; и выполняют стандартные операции администрирования, такие как обновление зоны, повторная загрузка зоны, обновление зоны, запись зоны обратно в файл или Active Directory, приостановка и возобновление работы зоны, очистка кэша, остановка и запуск службы DNS, а также просмотр статистики.

Поставщик WMI DNS имеет набор уникальных поведений, не найденных в других поставщиках. Сведения о классах см. в [справочнике по поставщику WMI DNS](dns-wmi-provider-reference.md) .
