# <a name="monitor-architecture"></a>Мониторинг архитектуры

На следующем рисунке показана связь между [*мониторами*](m.md) и другими компонентами архитектуры сетевой монитор.

![связь между мониторами и другими компонентами сетевого монитора](images/nmarch2.png)

Сетевой трафик собираются (отдельные кадры) из драйвера NDIS. Затем драйвер сетевой монитор (Nmnt.sys) направляет кадры в поставщик сетевых пакетов (НПП), который, в свою очередь, захватывает данные в режиме реального времени. НПП — это коллекция COM-интерфейсов, используемых для сбора данных. В этом случае для выполнения записи в режиме реального времени используется интерфейс [**ИРТК**](irtc.md) .

> [!Note]  
> НПП используется для отложенных и в реальном времени записей. Для отложенных захватов, используемых экспертами и анализаторами, используется интерфейс [**иделайдк**](idelaydc.md) .

 

Затем монитор проверяет захваченные данные в режиме реального времени, выявляя определенные условия сети и создавая события по мере необходимости.

Для мониторинга приложений используются три других компонента: средство управления монитором, служба контроля мониторинга (МКСВК) и Просмотр событий:

-   Средство управления монитором используется для управления приложениями мониторинга. Сюда входит настройка и запуск всех приложений мониторинга.
-   Служба управления монитором (МКСВК) запускает события, предоставляет ссылку для обмена данными на WMI для просмотра событий и координирует обработку операций мониторинга.
-   В Просмотр событий отображаются события, инициированные службой управления монитором.
