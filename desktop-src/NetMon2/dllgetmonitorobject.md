# <a name="dllgetmonitorobject-callback-function"></a>Функция обратного вызова Дллжетмониторобжект

Функция **дллжетмониторобжект** должна быть реализована монитором. МКСВК вызывает эту функцию для создания экземпляра монитора.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT DllGetMonitorObject(
  _In_  REFIID riid,
  _Out_ LPVOID *ppObj
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*riid* \[ окне\]
</dt> <dd>

UUID мониторов, показанный ниже, как определено в файле заголовка Имонитор. h. Если указан недопустимый UUID, функция завершается ошибкой, а монитор должен возвращать E- \_ интерфейс.

``` syntax
IID_IMonitor
```

</dd> <dt>

*ппобж* \[ заполняет\]
</dt> <dd>

Указатель на указатель, который получает интерфейс, запрошенный в *riid*.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно, возвращается значение S \_ OK (то же самое, что и ошибка).

Если функция завершается неудачно, возвращаемое значение является кодом ошибки. При возвращении кода ошибки МКСВК не создает объект Monitor, а метод [IUnknown:: Release](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) не вызывается для указателя интерфейса.

## <a name="remarks"></a>Комментарии

Функция **дллжетмониторобжект** вызывается каждый раз, когда мксвк пытается создать экземпляр монитора. Эта функция намеренно несет серьезную сходство с наиболее распространенной функцией [**DllGetClassObject**](/windows/win32/api/combaseapi/nf-combaseapi-dllgetclassobject) . Основное отличие заключается в том, что CLSID не передается в **дллжетмониторобжект**.
