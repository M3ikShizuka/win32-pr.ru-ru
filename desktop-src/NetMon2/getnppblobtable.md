# <a name="getnppblobtable-function"></a>Функция Жетнппблобтабле

Функция **жетнппблобтабле** извлекает таблицу больших двоичных объектов НПП, представляющую сетевые карты ККМ на локальном компьютере.

## <a name="syntax"></a>Синтаксис


```C++
DWORD GetNPPBlobTable(
  _In_  HBLOB       hFilterBlob,
  _Out_ PBLOB_TABLE *ppBlobTable
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хфилтерблоб* \[ окне\]
</dt> <dd>

Маркер для фильтра BLOB, ограничивающий большие двоичные объекты НПП, возвращаемые в таблице.

</dd> <dt>

*ппблобтабле* \[ заполняет\]
</dt> <dd>

Указатель на структуру [ \_ таблицы больших двоичных объектов](blob-table.md) , содержащую по крайней мере один указатель BLOB-объекта.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно, возвращается значение НМЕРР \_ Success.

Если функция не выполнена, возвращаемое значение является одним из следующих кодов ошибок:



| Код возврата                                                                                                | Описание                                                            |
|------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------|
| <dl> <dt>**НМЕРР \_ нет \_ \_ библиотеки DLL НПП**</dt> </dl>         | В каталоге НПП не найдены библиотеки DLL.<br/>                    |
| <dl> <dt>**НМЕРР \_ нет \_ допустимых \_ \_ библиотек DLL НПП**</dt> </dl> | Ни одна из библиотек DLL в каталоге НПП не была допустимой библиотекой DLL НПП.<br/>  |
| <dl> <dt>**НМЕРР \_ без \_ соответствующего \_ НППС**</dt> </dl>   | Обнаружены большие двоичные объекты НПП, но проверка фильтра не пройдена.<br/> |
| <dl> <dt>**НМЕРР \_ из \_ поля \_ MEMO**</dt> </dl>       | Сетевой монитор не удалось выделить память.<br/>            |



 

## <a name="remarks"></a>Remarks

Большой двоичный объект с именем *хфилтерблоб* также может быть специальным BLOB-объектом.

Если установить флаг в поле Фильтровать большой двоичный объект в **значение true**, то возвращаемая таблица больших двоичных объектов также может содержать специальные большие двоичные объекты.

Если большой двоичный объект с именем *хфилтерблоб* является специальным BLOB-объектом, сетевой монитор пользовательский интерфейс попытается обработать его. Например, предположим, что предыдущий вызов возвращает специальный большой двоичный объект из удаленного НПП. Приложение вставит требуемый тег, имя компьютера \_ . Затем метод поиска передает этот большой двоичный объект удаленному НПП, который затем возвращает таблицу больших двоичных объектов НПП, связанных с именем компьютера.

Для уничтожения всех возвращенных больших двоичных объектов и таблицы больших двоичных объектов вызывающий объект отвечает за вызов функции **дестройблоб** .
