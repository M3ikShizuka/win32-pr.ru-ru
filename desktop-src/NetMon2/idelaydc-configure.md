# <a name="idelaydcconfigure-method"></a>Метод Иделайдк:: Configure

Метод **Configure** отправляет сведения о конфигурации, используемые для записи.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT STDMETHODCALLTYPE Configure(
  [in]  HBLOB hConfigurationBlob,
  [out] HBLOB hErrorBlob
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хконфигуратионблоб* \[ окне\]
</dt> <dd>

Обработайте с BLOB-объектом, который содержит сведения о конфигурации.

</dd> <dt>

*херрорблоб* \[ заполняет\]
</dt> <dd>

Обработайте с BLOB-объектом ошибок, который содержит дополнительные сведения об ошибке. Сведения о содержимом большого двоичного объекта ошибки см. в подразделе "Примечания" этого раздела.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если этот метод выполнен успешно, возвращается значение НМЕРР \_ Success.

Если метод завершается неудачно, возвращаемое значение является одним из следующих кодов ошибок:



| Код возврата                                                                                                      | Описание                                                                               |
|------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------|
| <dl> <dt>**НМЕРР \_ запись**</dt> </dl>                  | НПП сообщает, что сеанс отслеживания запущен.<br/>                          |
| <dl> <dt>**\_ \_ фиксированный диск нмерр не является \_ локальным \_**</dt> </dl>    |                                                                                           |
| <dl> <dt>**НМЕРР \_ \_ не удалось \_ создать \_ память**</dt> </dl> |                                                                                           |
| <dl> <dt>**НМЕРР \_ недостаточно \_ \_ памяти**</dt> </dl>            | Недостаточно памяти. Завершите работу Windows, чтобы освободить ресурсы.<br/>               |
| <dl> <dt>**НМЕРР \_ Недопустимый \_ параметр**</dt> </dl>         | Большой двоичный объект конфигурации, заданный параметром Хконфигуратион, является недопустимым.<br/> |



 

## <a name="remarks"></a>Комментарии

Этот метод необходимо применить для перезапуска НПП, который был запущен, остановлен, но не отключен.

BLOB-объект ошибки, возвращенный *херрорблоб* , содержит записи, которые сетевой монитор не удалось распознать или найти в большом двоичном объекте конфигурации, указанном в *хконфигуратионблоб*. Возвращенный BLOB-объект ошибки содержит сведения об ошибке, которые приложение может использовать для устранения неполадок. Например, если \_ запись BLOB-объекта нмерр \_ \_ \_ не \_ существует, запись, сетевой монитор не удалось найти, включается в возвращенный большой двоичный объект ошибки.
