# <a name="monitor-events"></a>Мониторинг событий

Мониторы предназначены для запуска событий на локальных или удаленных компьютерах с помощью [инструментарий управления Windows (WMI)](/windows/desktop/WmiSdk/wmi-start-page) (WMI).

> [!Note]  
> Так как мониторы используют запись в режиме реального времени, они могут записывать только данные на локальном компьютере. Это ограничение интерфейса **ИРТК** поставщика сетевых пакетов. Однако с помощью событий WMI захваченные данные могут быть проверены локально, а события могут отправляться на локальные или удаленные компьютеры.

 

Мониторы не взаимодействуют напрямую с WMI. [*Служба контроля мониторинга*](m.md) (мксвк) предоставляет интерфейс (именуемый [имониторевентер](imonitoreventer.md)), который монитор может использовать для получения структуры данных событий и заполнения ее соответствующими данными. Затем МКСВК может отправить структуру данных события в WMI, что, в свою очередь, вызывает событие.
