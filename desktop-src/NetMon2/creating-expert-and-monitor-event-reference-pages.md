# <a name="creating-expert-and-monitor-event-reference-pages"></a>Создание страниц со ссылками на события экспертов и мониторинга

В этом разделе описывается планирование, разработка и тестирование страниц справочника по событиям (ERP) и их развертывание в эксперте или мониторе. Сведения о ERP и об их использовании с сетевой монитор см. [на странице справочника по событиям](event-reference-page.md).

Для разработки нового ERP первым шагом является определение событий, требующих отдельного ERP для Вашего специализированного [эксперта](experts.md) или [монитора](monitors.md). Для каждого события, которое необходимо отобразить в сетевой монитор Просмотр событий, необходимо создать отдельный ERP. Например, предположим, что:

-   Вы разрабатываете монитор с именем "игровой монитор" и "наблюдатель запасов".
-   Монитор находится в файле с именем Gamemon.dll.
-   Монитор создает два события: игра запущена и моя Интернет Nowalking.
-   Планируется разработка двух ERP, Gamemon1.htm и Gamemon2.htm.

> [!Note]  
> Нет необходимости иметь соответствие "один к одному" ERP для отслеживания или экспертных мероприятий.

 

После создания списка уникальных ERP выполните следующие действия, чтобы создать каждый ERP-процесс.

-   [Напишите исходный HTML-документ](writing-an-event-reference-page.md).
-   [Сохраните и присвойте](naming-an-event-reference-page.md) исходному файлу HTML имя в виде HTM-файла.
-   [Протестируйте ERP](testing-an-event-reference-page.md) с завершенным экспертом или монитором.
