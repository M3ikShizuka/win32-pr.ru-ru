# <a name="about-capture-filters"></a>О фильтрах записи

Фильтр записи — это элемент приложения НПП, который указывает драйверу сетевой монитор (Nmnt.sys), какие кадры сетевых данных должны храниться и какие кадры игнорировать. Преимущество настройки фильтра записи более эффективно. Не нужно изучать захваченные кадры; драйвер сетевой монитор изучает их. Этот подход исключает копирование кадров, что обеспечивает преимущество использования памяти.

## <a name="capture-filter-structure"></a>Структура фильтра записи

Фильтры записи состоят из трех элементов:

-   Параметры ETYPE/SAP
-   Адрес
-   Соответствие шаблону

Вместе эти элементы логически оценивают, какие кадры включить или исключить, во время работы приложения НПП. Фильтр записи предоставляет комплексные совпадения и исключения элементов Frame в приложение НПП.

Сетевой монитор реализует фильтр записи через структуру данных, [**каптурефилтер**](the-capturefilter-structure.md), передается в НПП, а затем передается драйверу сетевой монитор при запуске записи.

Дополнительные сведения об операциях НПП и больших двоичных объектов см. в разделе [поставщики сетевых пакетов](network-packet-providers.md) и [сетевой монитор большие двоичные объекты](network-monitor-blobs.md).
