# <a name="blob-types"></a>Типы больших двоичных объектов

Сетевой монитор использует три типа больших двоичных объектов (BLOB) для выбора и подключения сетевых карт, сбора данных и управления данными НПП.

## <a name="npp-blob"></a>БОЛЬШОЙ ДВОИЧНЫЙ ОБЪЕКТ НПП

Приложения используют большие двоичные объекты НПП для подключения к определенной сетевой карте через НПП. (Приложение устанавливает соединение с вызовом **креатенппинтерфаце** и данными о расположении в большом ДВОИЧНОМ объекте НПП.) Приложение также может хранить свои данные конфигурации в большом двоичном объекте НПП, чтобы настроить НПП. Кроме того, приложение, сохраняющее большой двоичный объект НПП, не обязательно должно проходить через поиск для повторного использования этого большого двоичного объекта.

## <a name="filter-blob"></a>Фильтровать большой двоичный объект

Большой двоичный объект фильтра содержит определяемые приложением условия, которые можно использовать для выбора НПП и сетевого адаптера. Например, приложение может запросить определенную сетевую карту, только карты Ethernet или карты, поддерживающие конкретный темп записи кадров.

## <a name="special-blob"></a>Специальный большой двоичный объект

Если НПП требует дополнительных данных, прежде чем он сможет вернуть НПП большой двоичный объект в приложение, НПП использует специальный большой двоичный объект. Чаще всего приложению не требуются или не используются специальные большие двоичные объекты, поэтому они не возвращаются в приложение, если в одном вызове функции не задан определенный флаг. Например, удаленный НПП использует специальный большой двоичный объект, так как для создания соединения требуется имя пути. Приложение, которое получает удаленный Специальный большой двоичный объект НПП, может добавить строку и выполнить обратный вызов в Finder для получения таблицы BLOB-объектов НПП. Дополнительные сведения см. в разделе [специальные записи большого двоичного объекта](special-blob-entries.md).
