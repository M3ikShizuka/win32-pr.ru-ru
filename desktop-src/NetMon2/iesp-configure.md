# <a name="iespconfigure-method"></a>Метод ИЕСП:: Configure

Метод **Configure** отправляет сведения о конфигурации для записи.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT STDMETHODCALLTYPE Configure(
  [in]  HBLOB hConfigurationBlob,
  [out] HBLOB hErrorBlob
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хконфигуратионблоб* \[ окне\]
</dt> <dd>

Обработайте с BLOB-объектом, который настраивает вызывающий объект.

</dd> <dt>

*херрорблоб* \[ заполняет\]
</dt> <dd>

Обработайте с BLOB-объектом ошибок, который содержит дополнительные сведения об ошибке. Сведения о содержимом большого двоичного объекта ошибки см. в подразделе "Примечания" этого раздела.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если метод выполнен успешно, возвращается значение НМЕРР \_ Success.

## <a name="remarks"></a>Remarks

Этот метод необходимо применить для перезапуска НПП, который был запущен и остановлен, но не был отключен.

BLOB-объект ошибки, возвращенный параметром *херрорблоб* , содержит записи, которые сетевой монитор не удалось распознать или найти в большом двоичном объекте конфигурации, указанном в *хконфигуратионблоб*. Возвращенный BLOB-объект ошибки содержит сведения об ошибке, которые приложение может использовать для устранения неполадок. Например, если \_ запись BLOB-объекта нмерр \_ \_ \_ не \_ существует, запись, сетевой монитор не удалось найти, включается в возвращенный большой двоичный объект ошибки.
