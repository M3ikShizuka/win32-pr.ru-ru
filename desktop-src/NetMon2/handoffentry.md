# <a name="handoffentry-structure"></a>Структура ХАНДОФФЕНТРИ

Структура **хандоффентри** определяет определенную запись протокола в структуре **хандоффтабле** .

Эта структура заполняется сетевой монитор на основе информации в предоставленном пользователем .ini файле, предоставленном при вызове функции [**креатехандоффтабле**](createhandofftable.md) . Эта структура никогда не должна быть явно изменена приложением.

## <a name="syntax"></a>Синтаксис


```C++
typedef struct _SESSIONSTATS {
  DWORD      hoe_sig;
  DWORD      hoe_ProtIdentNumber;
  HPPROTOCOL hoe_ProtocolHandle;
  DWORD      hoe_ProtocolData;
} HANDOFFENTRY, *LPHANDOFFENTRY;
```



## <a name="members"></a>Члены

<dl> <dt>

**описывается \_ SIG**
</dt> <dd>

Сигнатура, которая идентифицирует эту запись как запись в таблице передачи.

</dd> <dt>

**описывается \_ протидентнумбер**
</dt> <dd>

Номер протокола, предоставленный пользователем .ini файле.

</dd> <dt>

**описывается \_ протоколхандле**
</dt> <dd>

Маркер протокола, созданного с использованием имени протокола, предоставленного пользователем .ini файле.

</dd> <dt>

**описывается \_ протоколдата**
</dt> <dd>

Данные экземпляра протокола, предоставленные пользователем .ini файле.

</dd> </dl>

## <a name="remarks"></a>Remarks

Эта структура заполняется сетевой монитор когда сетевой монитор создает таблицу передачи.
