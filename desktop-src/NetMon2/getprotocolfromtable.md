# <a name="getprotocolfromtable-function"></a>Функция Жетпротоколфромтабле

Функция **жетпротоколфромтабле** возвращает маркер для протокола на основе заданной переданной таблицы и значения.

## <a name="syntax"></a>Синтаксис


```C++
HPROTOCOL WINAPI GetProtocolFromTable(
  _In_  LPHANDOFFTABLE hTable,
  _In_  DWORD          ItemToFind,
  _Out_ PDWORD_PTR     lpInstData
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хтабле* \[ окне\]
</dt> <dd>

Обработчик для переданной таблицы.

</dd> <dt>

*Итемтофинд* \[ окне\]
</dt> <dd>

Значение, используемое для нахождение протокола в таблице передачи. Значение должно быть доступно в данных протокола.

</dd> <dt>

*лпинстдата* \[ заполняет\]
</dt> <dd>

При наличии в таблице передачи данные экземпляра для следующего протокола. Длина данных экземпляра не может превышать значение типа DWORD \_ .

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно, возвращаемое значение является маркером протокола.

Если функция завершилась неудачно, возвращается значение **null**.

## <a name="remarks"></a>Комментарии

При реализации функции экспорта [рекогнизефраме](recognizeframe.md) функция **жетпротоколфромтабле** используется для получения маркера для следующего протокола. Функция **жетпротоколфромтабле** вызывается для получения маркера из следующего протокола, если протокол определяет, какой именно протокол следует.

**Данные экземпляров**

Данные экземпляра могут быть любыми данными, размер которых меньше или равен DWORD или \_ указателем на данные, например необработанные данные кадра, которые не должны выделяться или освобождаться анализатором.
