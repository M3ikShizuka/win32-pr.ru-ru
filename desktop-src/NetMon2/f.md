# <a name="f-network-monitor"></a>F (сетевой монитор)

<dl> <dt>

<span id="_netmon_filter_blob_gly"></span><span id="_NETMON_FILTER_BLOB_GLY"></span>**фильтровать большой двоичный объект**
</dt> <dd>

Большой двоичный объект (BLOB), используемый для ограничения поиска сетевого адаптера. Каждая запись в объекте фильтра BLOB должна соответствовать эквивалентной записи в большом двоичном объекте поставщика сетевых пакетов (НПП), представляющем сетевую карту. Например, приложение может запрашивать только карты Ethernet или карты, которые поддерживают определенную частоту кадров. Для вызова функций **жетнппблобфромуи** и **жетнппблобтабле** можно использовать большой двоичный объект фильтра.

</dd> <dt>

<span id="_netmon_follow_set_gly"></span><span id="_NETMON_FOLLOW_SET_GLY"></span>**следовать за набором**
</dt> <dd>

Набор протоколов, который следует за протоколом. Сетевой монитор использует следующий набор протоколов, если средство синтаксического анализа не может обнаружить следующие протоколы из данных в записи. См. также « [ *переданный набор* »](h.md)

</dd> <dt>

<span id="_netmon_frame_gly"></span><span id="_NETMON_FRAME_GLY"></span>**NOFRAMES**
</dt> <dd>

Пакет данных, передаваемый в виде одной единицы по сети. Каждый кадр соответствует одной базовой организации и содержит управляющую информацию, например синхронизацию символов, адреса станции, значение проверки ошибок и переменный объем данных. Также называется пакетом.

</dd> <dt>

<span id="_netmon_frame_viewer_window_gly"></span><span id="_NETMON_FRAME_VIEWER_WINDOW_GLY"></span>**окно средства просмотра кадров**
</dt> <dd>

Сетевой монитор окно, в котором отображаются подробные сведения об отдельных кадрах. Можно открыть несколько окон средства просмотра кадров за один раз.

</dd> </dl>
