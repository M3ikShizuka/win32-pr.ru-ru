# <a name="getcapturecomment-function"></a>Функция Жеткаптурекоммент

Функция **жеткаптурекоммент** возвращает указатель на комментарий записи.

## <a name="syntax"></a>Синтаксис


```C++
LPSTR WINAPI GetCaptureComment(
  _In_ HCAPTURE hCapture
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хкаптуре* \[ окне\]
</dt> <dd>

Маркер записи. Дополнительные сведения о получении маркера записи см. в разделе "Примечания".

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно (то есть, если запись содержит комментарий), возвращаемое значение является указателем на строку комментария.

Если функция завершилась неудачно, возвращается значение **null**.

## <a name="remarks"></a>Комментарии

Не изменяйте возвращенную строку комментария, которая является действительной строкой комментариев, которая находится в загруженной записи. Любые изменения могут повредить запись. Попытка изменить строку приводит к нарушению прав доступа.

Маркер записи можно получить несколькими способами, в зависимости от того, сделан ли вызов экспертом или средством синтаксического анализа. Для экспертов этот маркер указывается в элементе **хкаптуре** структуры [експертстартупинфо](expertstartupinfo.md) . Для средств синтаксического анализа можно получить маркер записи, вызвав функцию [жетфрамекаптурехандле](getframecapturehandle.md) .

Чтобы получить комментарий записи из файла записи, вызовите функцию [жеткаптурекомментфромфиленаме](getcapturecommentfromfilename.md) .

[*Эксперты*](e.md) и [*средства синтаксического анализа*](p.md) могут вызывать функцию **жеткаптурекоммент** .
