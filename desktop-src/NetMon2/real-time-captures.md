# <a name="real-time-captures"></a>Записи Real-Time

Мониторы получают захваченные кадры сети в режиме реального времени. Кадры фиксируются [*поставщиком сетевых пакетов*](n.md) (НПП) с помощью COM-интерфейса [ИРТК](irtc.md) поставщика и передаются монитору в одном из двух потоков выполнения монитора.

Мониторы могут ограничивать количество кадров, которые они должны обработать, передавая [*фильтр записи*](c.md) НПП, который предоставляет кадры. Как правило, конкретный монитор предназначен для наблюдения за конкретными типами трафика, такими как HTTP, RIPX или SMB. В отличие от экспертов, которые используют сложные средства синтаксического анализа для выбора анализируемых данных, монитор должен изучить каждый кадр на предмет условий, которые он был предназначен для обнаружения. Причина этого подхода в кадрах заключается в производительности. Монитор должен быстро обрабатывать свои кадры, чтобы он не выпадал за пределы драйвера, предоставляя кадры НПП. В противном случае данные будут потеряны.
