# <a name="implementing-the-capture-filter-code"></a>Реализация кода фильтра записи

Определив способ организации фильтра с помощью структуры [**каптурефилтер**](capturefilter.md) , необходимо выделить и заполнить структуру, которая затем передается драйверу сетевой монитор с помощью большого двоичного объекта конфигурации.

Прямые НПП пользователи добавляют фильтр записи в большой двоичный объект конфигурации, который передается для **подключения**, [**настройки**](configure.md)или обоих. Список COM-интерфейсов, которые можно использовать для взаимодействия с НПП, см. в разделе [интерфейсы НПП](npp-interfaces.md).
