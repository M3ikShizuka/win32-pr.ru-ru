# <a name="set-structure"></a>ЗАДАТЬ структуру

Структура **набора** определяет набор значений.

## <a name="syntax"></a>Синтаксис


```C++
typedef struct _SET {
  DWORD nEntries;
  union {
    LPBYTE               lpByteTable;
    LPWORD               lpWordTable;
    LPDWORD              lpDwordTable;
    LPLARGEINT           lpLargeIntTable;
    LPSYSTEMTIME         lpSystemTimeTable;
    LPLABELED_BYTE       lpLabeledByteTable;
    LPLABELED_WORD       lpLabeledWordTable;
    LPLABELED_DWORD      lpLabeledDwordTable;
    LPLABELED_LARGEINT   lpLabeledLargeIntTable;
    LPLABELED_SYSTEMTIME lpLabeledSystemTimeTable;
    LPLABELED_BIT        lpLabeledBit;
    LPVOID               lpVoidTable;
  };
} SET, *LPSET;
```

## <a name="remarks"></a>Remarks

Структура **набора** используется для определения набора данных сравнения, которые сетевой монитор могут использовать для интерпретации значения свойства в пакете протокола. Если требуется набор данных сравнения, указатель на структуру **Set** указывается в элементе **лпсет** структуры [PROPERTYINFO](propertyinfo.md) .

Библиотека DLL средства синтаксического анализа может предоставлять набор значений и метку. Член **объединения** , выбранный в **наборе** , указывает на массив структур, определяющих каждый элемент набора.

-   Набор значений

    Набор значений используется, если требуется, чтобы сетевой монитор включить индикатор in-Set или unin-Set со значением, найденным в пакете протокола. Например, если задан параметр DWORD, сетевой монитор отображает метку для каждого значения DWORD, найденного в пакете протокола, указывая, что параметр DWORD имеет значение или не указан в наборе.

    Набор значений может основываться на типах данных BYTE, WORD, DWORD, ЛАРЖЕИНТ и SYSTEMTIME.

-   Набор меток

    Набор меток используется, если требуется, чтобы сетевой монитор отображала определенная пользователем метка вместо значений свойств, указанных в наборе.

    Набор меток может быть основан на парах BYTE, WORD, DWORD, ЛАРЖЕИНТ, SYSTEMTIME и BIT меток.
