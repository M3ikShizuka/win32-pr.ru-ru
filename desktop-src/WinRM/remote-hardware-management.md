# <a name="remote-hardware-management"></a>Удаленное управление оборудованием

Служба удаленного управления Windows управление оборудованием предназначено для снижения общей стоимости ИТ-администрирования за счет мониторинга и контроля удаленных аппаратных компонентов, особенно перед запуском системы и после сбоя операционной системы.

Изготовители оборудования (OEM) разработали общую архитектуру для решения этой потребности в управлении оборудованием. Важной частью этой архитектуры является [*контроллер управления основной платой*](windows-remote-management-glossary.md) (BMC). Контроллер BMC — это специализированное устройство, отслеживающее состояние компьютера сервера. Контроллер BMC обеспечивает удаленное управление аппаратным обеспечением сервера, получением данных о состоянии и получением уведомлений о критических ошибках и других изменениях состояния оборудования. Сценарий или приложение, которое наблюдает за удаленным сервером, может получать данные с сервера с помощью [*аппаратного контроллера управления*](windows-remote-management-glossary.md), через удаленную операционную систему или по внешнему [*каналу*](windows-remote-management-glossary.md)непосредственно из BMC.

В контроллере BMC есть датчики, которые могут обнаружить, например, когда компьютер сервера перегревается или когда напряжение выходит за пределы допустимого диапазона. Для определения архитектуры BMC существует несколько стандартов. Стандартный [*интерфейс управления платформой (IPMI)*](windows-remote-management-glossary.md) — это один из таких стандартов, который часто используется. Однако, несмотря на стандарт IPMI, доступ управления к оборудованию сервера является собственным и требует использования средств управления, предоставляемых поставщиками оборудования. Кроме того, удаленный доступ к контроллеру BMC предоставляется с помощью специального сетевого протокола (Remote Management Protocol), который имеет нестандартные механизмы безопасности для проверки подлинности доступа.

Поставщик Microsoft [IPMI](/previous-versions/windows/desktop/ipmiprv/ipmi-provider) и драйвер IPMI позволяют получать данные BMC с удаленных серверных компьютеров с помощью стандартного поставщика WMI с [классами](/previous-versions/windows/desktop/ipmiprv/ipmi-provider)WMI. Хотя вы можете написать стандартный сценарий WMI, который получает удаленные данные через DCOM, во многих случаях предпочтительным способом получения данных IPMI является использование служебной программы командной строки **WinRM** или [API сценариев WinRM](winrm-scripting-api.md) или [интерфейса API WinRM C++](winrm-c---api.md). Служебная программа WinRM и API службы WinRM используют протокол WS-Management и могут получать данные IPMI с локального или удаленного компьютера без использования DCOM.

В BMC также есть база данных событий, называемая журналом системных событий (SEL), которая записывает события на наблюдаемом компьютере. Нельзя подписываться на то, что эти события доставляются в сценарий, как это можно делать с классами событий WMI. Однако вы можете подписываться на них с помощью программы командной строки Wecutil.exe. Для получения дополнительных сведений об использовании этого средства в командной строке введите **wecutil/?**.
