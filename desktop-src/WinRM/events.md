# <a name="events-windows-remote-management"></a>события (служба удаленного управления Windows)

Служба сборщика событий использует протокол WS-Management для сбора событий с удаленных компьютеров. служба "сборщик событий" позволяет создавать подписки на Windows событий на удаленных компьютерах и событиях оборудования, формируемых контроллерами управления основной платой (bmc). Контроллеры BMC должны поддерживать протокол WS-Management.

При создании подписки события перенаправляются на компьютер, на котором была создана подписка. Они будут отображаться в Просмотр событий.

события оборудования, создаваемые интеллектуальным интерфейсом управления платформой (IPMI) на компьютерах на базе Windows, собираются локально с помощью драйвера ipmi и хранятся в журнале событий оборудования, после чего они могут пересылаться, как и другие Windows события операционной системы.

Подписки можно создавать с помощью средства командной строки Wecutil.exe. Для получения дополнительных сведений об использовании этого средства в командной строке введите **wecutil/?**. дополнительные сведения о службе сборщика событий см. в разделе [Windows сборщика событий](/windows/desktop/WEC/windows-event-collector).

> [!Note]  
> Windows Для интерфейсов API удаленного управления требуется, чтобы все IPv6-адреса заключались в квадратные скобки.

 

> [!Caution]
>
> при использовании подписок, инициированных сборщиком, ограничьте число удаленных компьютеров до 500 и изолируйте [Windows службу сборщика событий](/windows/desktop/WEC/windows-event-collector) (вексвк) в отдельном хост-процессе.
>
> Ошибка подключения будет содержать поток до истечения времени ожидания. Большое количество одновременных ошибок подключения может привести к нехватке пула потоков и отрисовывать сервер без ответа.
