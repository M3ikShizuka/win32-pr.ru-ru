# <a name="events-windows-remote-management"></a>События (служба удаленного управления Windows)

Служба сборщика событий использует протокол WS-Management для сбора событий с удаленных компьютеров. Служба "сборщик событий" позволяет создавать подписки на события Windows на удаленных компьютерах и событиях оборудования, создаваемых контроллерами управления основной платой (BMC). Контроллеры BMC должны поддерживать протокол WS-Management.

При создании подписки события перенаправляются на компьютер, на котором была создана подписка. Они будут отображаться в Просмотр событий.

События оборудования, создаваемые интеллектуальным интерфейсом управления платформой (IPMI) на компьютерах под управлением Windows, собираются локально с помощью драйвера IPMI и хранятся в журнале событий оборудования, после чего они могут пересылаться, как и другие события операционной системы Windows.

Подписки можно создавать с помощью средства командной строки Wecutil.exe. Для получения дополнительных сведений об использовании этого средства в командной строке введите **wecutil/?**. Дополнительные сведения о службе "сборщик событий" см. в разделе [сборщик событий Windows](/windows/desktop/WEC/windows-event-collector).

> [!Note]  
> Служба удаленного управления Windows API требует, чтобы все IPv6-адреса заключались в квадратные скобки.

 

> [!Caution]
>
> При использовании подписок, инициированных сборщиком, ограничьте число удаленных компьютеров до 500 и изолируйте службу [сборщика событий Windows](/windows/desktop/WEC/windows-event-collector) (вексвк) в отдельном хост-процессе.
>
> Ошибка подключения будет содержать поток до истечения времени ожидания. Большое количество одновременных ошибок подключения может привести к нехватке пула потоков и отрисовывать сервер без ответа.
