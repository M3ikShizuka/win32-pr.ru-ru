# <a name="upload-best-practices"></a>Upload Рекомендации

Хигхлоадс может вызвать различные условия времени ожидания сервера, которые могут, в свою очередь, увеличить нагрузку при попытке клиента. Кроме того, большое количество необработанных подключений будет потреблять больше ресурсов сервера и усложнить ситуацию. Поверх этого, если серверное приложение не записывается для работы с высокими условиями загрузки, оно может привести к сбою или некорректной работе. Чтобы ограничить нагрузку на серверную часть, приложение должно выполнить следующие действия.

Если серверное приложение не записывается для работы с большими объемами томов, могут возникать условия времени ожидания, которые, в свою очередь, увеличивают нагрузку при попытке клиента. Кроме того, большое количество необработанных подключений будет потреблять больше ресурсов сервера.

При тестировании серверного приложения протестируйте максимально возможную нагрузку. Следует использовать несколько клиентских компьютеров, каждый из которых имеет несколько параллельных заданий битов и измеряет максимальную пропускную способность в серверной части. Если вы не можете измерить пропускную способность, вам придется оценить пропускную способность.

Серверное приложение должно располагаться на другом URL-адресе для отправки URL (см. свойство BITS IIS, **битссервернотификатионурл**).

Рекомендуется ограничить нагрузку на сервер приложений на основе проверенных значений пропускной способности. Для ограничения нагрузки на сервер приложений следует использовать свойства IIS **MaxBandwidth** и **MaxConnections**.
