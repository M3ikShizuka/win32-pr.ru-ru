# <a name="_ianalysisproxyeventspopulatecontextnode-event"></a>\_Ианалисиспроксевентс: событие Опулатеконтекстноде:P

Происходит перед тем, как [**иинканализер**](iinkanalyzer.md) выполняет анализ внутри региона частично заполненного объекта [**иконтекстноде**](icontextnode.md) .

## <a name="syntax"></a>Синтаксис


```C++
HRESULT PopulateContextNode(
  [in] IInkAnalyzer *pInkAnalyzer,
  [in] IContextNode *pContextNodeToPopulate
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пинканализер* \[ окне\]
</dt> <dd>

Объект [**иинканализер**](iinkanalyzer.md) , о котором необходимо выполнить анализ.

</dd> <dt>

*пконтекстнодетопопулате* \[ окне\]
</dt> <dd>

Частично заполненный объект [**иконтекстноде**](icontextnode.md) .

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Описание возвращаемых значений см. в разделе [классы и интерфейсы — анализ рукописного ввода](classes-and-interfaces---ink-analysis.md).

## <a name="remarks"></a>Remarks

Это событие используется, когда приложение поддерживает собственную структуру данных, которая синхронизируется с [**иинканализер**](iinkanalyzer.md). Когда **иинканализер** вызывает это событие, приложение должно заполнить *пконтекстнодетопопулате*. На этапе анализа **иинканализер** создает это событие для получения сведений о областях, в которых он анализирует рукописный ввод.

Если документ содержит ссылки для *пконтекстнодетопопулате*, приложение должно создать и добавить эти ссылки. Для этого процесса требуется, чтобы исходный и конечный узлы, включая их предки, были полностью заполнены до того, как обработчик событий для этого события выходит из системы (см. раздел [**иконтекстлинк:: жетсаурценоде**](icontextlink-getsourcenode.md) and [**Иконтекстлинк:: жетдестинатионноде**](icontextlink-getdestinationnode.md)).

Дополнительные сведения о синхронизации данных приложения с помощью [**иинканализер**](iinkanalyzer.md)см. в разделе [учетная запись-посредник данных с помощью анализа рукописного ввода](data-proxy-with-ink-analysis.md).

Во время фонового анализа [**иинканализер**](iinkanalyzer.md) создает это событие после того, как оно вызывает событие [**\_ ианалисисевентс:: реадитореконЦиле**](-ianalysisevents-readytoreconcile.md) .
