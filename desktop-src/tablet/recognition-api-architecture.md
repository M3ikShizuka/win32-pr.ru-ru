# <a name="recognition-api-architecture"></a>Архитектура API распознавания

Приложение с поддержкой рукописного ввода взаимодействует с системой распознавания через интерфейсы API платформы Tablet PC. Для этого в приложениях используется объект [**иинкрекогнизер**](/windows/desktop/api/msinkaut/nn-msinkaut-iinkrecognizer) . Платформа Tablet PC взаимодействует с распознавателем с помощью интерфейсов в стиле C, описанных в этом разделе. На следующем рисунке область внутри пунктирной линии показывает, что описано в этом разделе.

![Иллюстрация архитектуры распознавания с выделенным распознавателем](images/96ee7367-7b8a-4794-99c1-bcac9daed645.gif)

Пользовательский распознаватель должен включать в себя краткое руководство. h (устанавливается по умолчанию в C: \\ Program Files \\ Microsoft Tablet PC Platform SDK \\ include). Если не указано иное, Библиотека динамической компоновки (DLL) должна экспортировать все функции API, даже если вы решили, чтобы некоторые из них возвращали E \_ нотимпл.

При отсутствии каких-либо обстоятельств ваш распознаватель будет вызываться непосредственно приложением с поддержкой рукописного ввода. Вместо этого приложения будут вызывать API автоматизации или управляемые API для получения результатов от распознавателя.
