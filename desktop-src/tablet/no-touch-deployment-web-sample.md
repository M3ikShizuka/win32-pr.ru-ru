# <a name="no-touch-deployment-web-sample"></a>Пример веб-примера развертывания No-Touch

В этом примере показано, как развернуть управляемое приложение для планшетных ПК через Интернет с помощью развертывания без вмешательства пользователя. вы должны быть знакомы с концепциями, обсуждаемыми в разделе [несенсорное развертывание в платформа .NET Framework](/documentation/?url=%2flibrary%2fdv_vstechart%2fhtml%2fvbtchno-touchdeploymentinnetframework.asp). для запуска этого примера на компьютере должен быть установлен Microsoft IIS (IIS).

## <a name="overview"></a>Обзор

Благодаря несенсорному развертыванию приложения для планшетных Пквиндовс Forms — приложения для настольных систем, созданные с помощью классов в системе. Windows. пространство имен forms пакета средств разработки microsoft платформа .NET Framework и microsoft Windows XP Tablet PC Edition 1,7 — можно скачать, установить и запустить непосредственно на компьютерах пользователей без изменения реестра или общих компонентов системы.

Этот пример принимает исходный проект для [образца формы Auto утверждений, автоматическую](auto-claims-form-sample.md)заявку и предоставляет проект установщика с автоутверждением \_ нотаучвеб. После компиляции и запуска проект установщика создает новый виртуальный корень, который также называется автозаявками \_ нотаучвеб. Программа установки копирует файл default.htm, содержащий ссылку на сборку AutoClaims.exe. Чтобы запустить клиентское приложение с богатыми возможностями, перейдите к виртуальному корню с помощью Microsoft Internet Explorer и щелкните ссылку на странице default.htm.

> [!Note]  
> Для работы приложения в домене приложения Internet Explorer необходимо перейти к виртуальному корню с помощью IIS (например, https://localhost/AutoClaims\_NoTouchWeb/default.htm) а не напрямую через файловую систему).

 


```C++
<html>
    <head>
        <title>AutoClaims (No Touch Web)</title>
      </head>
      <body>
          <a href="AutoClaims.exe">Launch AutoClaims Sample</a>
      </body>
</html>
```



## <a name="no-touch-deployment-requirements"></a>Требования к развертыванию No-Touch

Все зависимые сборки должны находиться либо в пути поиска сборок, либо в корне виртуального каталога веб-сайта. Проект развертывания нотаучвеб автоматических утверждений \_ устанавливает сборку и ссылающуюся страницу default.htm в один и тот же виртуальный корень (автоутверждения \_ нотаучвеб).

> [!Note]  
> Скомпилированные веб-образцы не устанавливаются с помощью параметра установки по умолчанию для пакета SDK. Необходимо выполнить выборочную установку и выбрать подпараметр "предварительно скомпилированные веб-примеры", чтобы установить их.

 

Дополнительные сведения об использовании рукописного ввода в Интернете см. в разделе [Рукописный ввод в Интернете](ink-on-the-web.md).
