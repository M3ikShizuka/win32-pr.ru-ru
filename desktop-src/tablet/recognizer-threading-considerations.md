# <a name="recognizer-threading-considerations"></a>Рекомендации по потокам распознавателя

Несмотря на то, что доступ к [контексту распознавателя](hrecocontext-handle.md) невозможен в двух потоках одновременно платформой Tablet PC, функция [**адвисеинкчанже**](/windows/desktop/api/recapis/nf-recapis-adviseinkchange) может быть вызвана в любое время в любом потоке. Поскольку платформа Tablet PC не гарантирует, что в каждый момент времени существует только один контекст распознавателя, два отдельных контекста распознавателя в отдельных потоках могут одновременно попытаться получить доступ к [распознавателю](hrecognizer-handle.md). Таким образом, глобальные переменные в подсистеме распознавания должны быть потокобезопасными.

Списки слов являются необязательными реализациями, используемыми для повышения точности. Если распознаватель реализует списки слов, он должен быть потокобезопасным. Дополнительные сведения об использовании списков Word см. [в разделе Использование словарей приложений](using-application-dictionaries.md).

Дополнительные сведения о других вопросах, связанных с потоками, см. в статье [вопросы многопоточной обработки в планшетных ПК](tablet-pc-threading-considerations.md).
