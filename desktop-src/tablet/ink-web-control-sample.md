# <a name="ink-web-control-sample"></a>Пример веб-элемента управления рукописного ввода

В этом примере показано, как создать элемент управления с поддержкой рукописного ввода для использования в веб-браузере. Образец принимает исходный [образец автоматической заявки](auto-claims-form-sample.md) и превращает его в элемент управления, помещаемый на веб-страницу.

Дополнительные сведения об использовании рукописного ввода в Интернете см. в разделе [Рукописный ввод в Интернете](ink-on-the-web.md).

## <a name="modifications-to-the-original-sample-project"></a>Изменения в исходном образце Project

Этот пример состоит из решения, включающего два проекта и HTML-файл. Первый проект, Автоутверждение, представляет собой \# проект библиотеки элементов управления Microsoft Visual C (пользовательский элемент управления). Исходный код для этого элемента управления практически идентичен образцу автозаявок с двумя отличиями:

-   `AutoClaims`Класс в этом примере наследует от класса [UserControl](/dotnet/api/system.windows.forms.usercontrol?view=netcore-3.1) , а не от класса [Form](/dotnet/api/system.windows.forms.form?view=netcore-3.1) .

    ```C++
    public class AutoClaims : System.Windows.Forms.UserControl 
    ```

    

-   Класс автозаявок в этом образце содержит добавленный открытый метод, `DisposeResources` удаляющий внутренние дочерние элементы управления, используемые для сбора рукописных данных. Этот метод должен вызываться севебпажеон, который используется при завершении страницы с помощью элемента управления.

## <a name="referencing-the-control-in-html"></a>Ссылка на элемент управления в HTML

Решение включает HTML-файл default.htm. Этот файл представляет собой страницу, к которой переходит браузер, чтобы загрузить элемент управления. Файл содержит &lt; &gt; тег объекта, который ссылается на элемент управления. Он также включает скрипт, который вызывается при выгрузке страницы, как указано в атрибуте OnLoad = " `OnUnload()` " в &lt; &gt; теге body. Эта функция вызывает `DisposeResources` метод для элемента управления, чтобы обеспечить правильную отпускания всех ресурсов при завершении работы.


```C++
<html>
    <script language="jscript">
        // Release any resources held by the AutoClaims control
        function OnUnload()
        {
            autoClaimsControl.DisposeResources();
        }
    </script>
    <head>
        <title>AutoClaims (Web Control)</title>
    </head>
    <body onunload="OnUnload()">
        <object 
          id="autoClaimsControl" 
          classid="AutoClaims.dll#AutoClaims.AutoClaims">
        </object>
    </body>
</html> 
```



Обратите внимание на формат значения атрибута ClassID для &lt; тега объекта &gt; . Он именует сборку, за которой следует \# Разделитель знаков, а затем пространство имен, содержащее элемент управления, а затем имя класса элемента управления.

Реальный пользовательский элемент управления, скорее всего, будет включать дополнительные методы, используемые для сохранения или отправки данных, собираемых в приложении.

## <a name="the-autoclaims_webcontrol-project"></a>Project WebControl для автозаявок \_

проект WebControl для автозаявок \_ — это Project развертывания, который создает программу установки, которая добавляет виртуальный корневой каталог, автозапрос \_ WebControl на веб-сервере при установке. Элемент управления и HTML-файл помещаются в этот виртуальный корневой каталог.
