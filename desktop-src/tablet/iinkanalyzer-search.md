# <a name="iinkanalyzersearch-method"></a>Метод Иинканализер:: Search

Предоставляет поиск с учетом регистра нечетких символов, не учитывающих регистр, для проанализированных рукописных штрихов и проанализированных графических штрихов, имеющих распознаваемые типы.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Search(
  [in]      BSTR  bstrPhraseToMatch,
  [in, out] ULONG *pulSearchResultCount,
  [out]     ULONG **ppulStrokeCountPerResult,
  [in, out] ULONG *pulStrokeIdsCount,
  [out]     ULONG **ppulStrokeIds
);
```

## <a name="remarks"></a>Remarks

В результате поиска будут найдены подстроки из нескольких слов и отдельных слов. Поиск выполняется в альтернативных результатах распознавания и в альтернативных сегментах.

Все входящие строки будут преобразованы в единый регистр для сравнения с использованием LCID текущего потока для выполнения этого преобразования в соответствии с региональными соглашениями о регистрах.

Переданная строка обрабатывается как фраза. Слова и символы должны присутствовать в алтерантес для штрихов в указанном порядке. Первое и Последнее слова фразы могут сопоставляться как подстроки (первое слово, отображаемое в конце альтернативного, и Последнее слово, отображаемое в беггингинге), но все остальные слова (внутри фразы) должны быть целыми словами.

Если переданная строка не содержит пробелов между символами, то подстрока может находиться в любом месте внутри одного слова в альтернативном.

Результаты поиска изменяются только присутствием или отсутствием пробелов между символами. Пробелы, не заключенные в символы, игнорируются. Тип пробела игнорируется (знак табуляции или пробел между символами дает одинаковый результат). Количество пробелов не имеет значения — один или два пробела между символами дают одинаковый результат.

Поиск не создает событий Популатеконтекстноде. Будет выполнен поиск только уже заполненных штрихов.
