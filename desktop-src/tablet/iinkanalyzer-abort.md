# <a name="iinkanalyzerabort-method"></a>Метод Иинканализер:: Abort

Отменяет текущую операцию анализа.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Abort(
  [out] IAnalysisRegion **ppAbortedRegion
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*ппабортедрегион* \[ заполняет\]
</dt> <dd>

Указатель на объект [**ианалисисрегион**](ianalysisregion.md) , представляющий "грязную" область (см. [**метод Иинканализер:: жетдиртирегион**](iinkanalyzer-getdirtyregion.md)) текущей операции анализа.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Описание возвращаемых значений см. в разделе [классы и интерфейсы — анализ рукописного ввода](classes-and-interfaces---ink-analysis.md).

## <a name="remarks"></a>Комментарии

Вызовите [**IUnknown:: Release**](/windows/desktop/api/unknwn/nf-unknwn-iunknown-release) в *ппабортедрегион* , когда больше не нужно использовать объект.

Этот метод отменяет текущую операцию анализа.

Если *ппабортедрегион* имеет **значение NULL**, метод Abort выполняется как обычная, так как это указывает на то, что вызывающий объект не имеет интереса в возвращаемом значении.

**Метод иинканализер:: Abort** останавливает события [**\_ ианалисисевентс:: Results**](-ianalysisevents-results.md) и [**\_ ианалисисевентс:: Activity**](-ianalysisevents-activity.md) для текущей операции анализа.

**Метод иинканализер:: Abort** выполняется асинхронно до отмены текущей фоновой операции анализа. Поскольку процесс отмены является асинхронным, приложение может выполнять другие задачи во время отмены текущего анализа опертионс.

Если операций анализа не выполняется, этот метод возвращает пустой регион анализа.

Если выполняется одна операция анализа, этот метод отменяет операцию.

Если выполняется как синхронные, так и асинхронные операции анализа, этот метод отменяет синхронную операцию.

Если этот метод вызывается более одного раза для одной и той же операции анализа, первый вызов возвращает "грязную" область для операции, а последующие вызовы возвращают пустой регион.

Если приложение поддерживает собственную структуру данных, которая синхронизирована с [**иинканализер**](iinkanalyzer.md), вызов **метода Иинканализер:: Abort** может привести к невозможности выполнения документа с частичными результатами. Чтобы избежать этого, не вызывайте **метод иинканализер:: Abort** между моментом, когда **иинканализер** получает событие [**\_ ианалисиспроксевентс:: Инканализерстатечангинг**](-ianalysisproxyevents-inkanalyzerstatechanging.md) , и время, которое **иинканализер** получает событие [**\_ IAnalysisEvents:: IntermediateResults**](-ianalysisevents-intermediateresults.md) или [**\_ IAnalysisEvents:: Results**](-ianalysisevents-results.md) .

Дополнительные сведения о синхронизации данных приложения с помощью анализатора рукописного ввода см. в разделе [учетная запись-посредник данных с анализом рукописного ввода](data-proxy-with-ink-analysis.md).
