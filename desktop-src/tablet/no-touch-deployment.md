# <a name="no-touch-deployment"></a>Без сенсорного развертывания

Microsoft платформа .NET Framework предоставляет возможность развертывать приложения с веб-сервера или с файловым сервером. Эта методика, называемая «несенсорным развертыванием», сочетает в себе производительность и интерактивность приложения смарт-клиента со всеми преимуществами развертывания веб-приложения. Чтобы развернуть приложение через Интернет, щелкните правой кнопкой мыши папку, содержащую исполняемый файл, и выберите пункт **Свойства**. На вкладке « **общий доступ в Интернете** » выберите **Открыть общий доступ к этой папке**. Введите псевдоним для папки. Теперь исполняемый файл можно запустить через Интернет, используя этот псевдоним в качестве каталога на сервере. дополнительные сведения о развертывании без вмешательства пользователя см. в разделе [интеллектуальные клиенты .net](/archive/msdn-magazine/2002/july/net-zero-deployment-security-and-versioning-models-in-the-windows-forms-engine-help-you-create-and-deploy-smart-clients) и [развертывание без вмешательства в платформа .NET Framework](/documentation/?url=%2flibrary%2fdv_vstechart%2fhtml%2fvbtchno-touchdeploymentinnetframework.asp).

> [!Note]  
> для включения вкладки " **доступ в интернете** " в диалоговом окне " **свойства** " необходимо установить на компьютере Microsoft IIS (IIS).

 

несенсорное развертывание применяется к приложениям с поддержкой рукописного ввода так же, как и любому другому приложению Windows Forms. примеры, предоставляемые пакетом средств разработки программного обеспечения (SDK) Microsoft Windows XP Tablet PC Edition, включают версию развертывания без участия пользователя в примере "автоматические утверждения" в разделе образцы веб-страниц образцов. В этом примере используется [образец автоматической формы утверждений](auto-claims-form-sample.md) и предоставляется установщик, который помещает в веб-общую папку. Дополнительные сведения о примере см. в разделе [веб-пример Несенсорного развертывания](no-touch-deployment-web-sample.md) .

> При развертывании приложения .NET через Интернет это может повлиять на приложение с помощью модели безопасности .NET. В разделе [безопасность и доверие](security-and-trust.md) описано, как безопасность работает с веб-приложениями с поддержкой рукописного ввода.
