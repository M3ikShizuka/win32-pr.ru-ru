# <a name="inkrecognizerguide-class"></a>Класс Инкрекогнизергуиде

Представляет область, используемую распознавателем для рисования рукописного ввода. Область называется руководством по распознаванию.

**Инкрекогнизергуиде** имеет следующие типы членов:

-   [Интерфейсы](#interfaces)
-   [Свойства](#properties)

### <a name="interfaces"></a>Интерфейсы

Класс **инкрекогнизергуиде** определяет эти интерфейсы.



| Интерфейс               | Описание                                                                  |
|:------------------------|:-----------------------------------------------------------------------------|
| **иинкрекогнизергуиде** | Этот объект реализует COM-интерфейс **иинкрекогнизергуиде** .<br/> |

Класс **инкрекогнизергуиде** имеет следующие свойства.



| Свойство                                                       | Тип доступа           | Описание                                                                                                                    |
|:---------------------------------------------------------------|:----------------------|:-------------------------------------------------------------------------------------------------------------------------------|
| [**Столбцы**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizerguide-get_columns)<br/>       | Чтение/запись<br/> | Возвращает или задает количество столбцов в поле с руководством.<br/>                                                                |
| [**дравнбокс**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizerguide-get_drawnbox)<br/>     | Чтение/запись<br/> | Возвращает или задает поле, которое физически рисуется на экране планшета и в котором происходит запись.<br/>              |
| [**гуидедата**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizerguide-get_guidedata)<br/>   | Чтение/запись<br/> | Возвращает или задает данные руководств для разработчиков на C++.<br/>                                                                         |

## <a name="remarks"></a>Комментарии

Для создания экземпляра этого объекта можно вызвать метод [**CoCreateInstance**](/windows/desktop/api/combaseapi/nf-combaseapi-cocreateinstance) .

По умолчанию нет руководств по распознавателю. По умолчанию для всех свойств задано значение 0. Для настройки программы необходимо использовать свойства этого объекта.

Если приложение выводит рекомендации на экране, на котором пользователь ожидает записи, приложение должно задать значения свойств руководства распознавателя, чтобы информировать распознаватель. Эти свойства предназначены только для использования распознавателя. В противном случае Нарисуйте визуальные подсказки на экране. Приложение или элемент управления рисует визуальные подсказки.

Средство распознавания может состоять из строк и столбцов, и они предоставляют распознавателю Улучшенный контекст, в котором выполняется распознавание. Такие буквы, как «t» и «I», легче распознаются при использовании руководств для предоставления контекста рукописному вводу. Например, можно нарисовать горизонтальные линии на экране, которые показывают, где должно происходить запись (этот тип руководства будет состоять только из строк и не содержит столбцов). При написании строк вместо какого-либо произвольного пространства распознавание распознавания улучшается.

В данном руководством определяются границы рукописного ввода в координатах рукописного ввода.

Свойство [**дравнбокс**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizerguide-get_drawnbox) может определять поле, размер которого совпадает с размером поля, определенного свойством [**вритингбокс**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizerguide-get_writingbox) , или меньше него.

На следующем рисунке показаны элементы в руководству распознавателя с двумя строками и без столбцов.

![Иллюстрация, демонстрирующая элементы с руководством распознавателя](images/927cc2f3-549f-4279-aab9-bd5ade070eaf.jpg)

В дополнение к рисованию линий на экране, которые показывают пользователей, на которых нужно писать, можно нарисовать ячейки на экране, где записываются символы или слова. Это называется Boxed Input и полезно для некоторых азиатских языков. Чтобы определить, поддерживает ли распознаватель Упакованные входные данные, вызовите свойство [**capabilities**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizer-get_capabilities) объекта [**иинкрекогнизер**](/windows/desktop/api/msinkaut/nn-msinkaut-iinkrecognizer) .

На следующем рисунке показано средство распознавания с четырьмя столбцами.
