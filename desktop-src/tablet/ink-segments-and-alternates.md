# <a name="ink-segments-and-alternates"></a>Сегменты и варианты рукописного ввода

Распознаватель может найти несколько способов разбить образец рукописного ввода на сегменты распознавания. По этой причине количество вариантов распознавания может располагаться в шахматном порядке, даже для небольшого примера рукописного ввода.

Например, «t o g e t h e r» может привести к следующим результатам:

-   "чтобы получить его" (плюс альтернативные варианты для каждого слова)
-   "для сбора" (плюс альтернативные варианты для каждого слова)
-   "тог Ether" (плюс альтернативные варианты для каждого слова)
-   «Тог» (плюс альтернативные варианты для каждого слова)
-   "вместе" (плюс альтернативы для слова)

Объект [**рекогнитионресулт**](/windows/desktop/api/msinkaut/nn-msinkaut-iinkrecognitionresult) поддерживает эффективное хранение полных результатов распознавания в объекте [**рукописного ввода**](inkdisp-class.md) . Объект **рукописного ввода** сопоставляет варианты распознавания с штрихами в объекте **рукописного ввода** и может также содержать другие сведения, такие как базовая линия, индексы строк и диапазоны языков.
