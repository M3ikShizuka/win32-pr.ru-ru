# <a name="peninputpanelinputfailed-event"></a>Пенинпутпанел. Инпутфаилед, событие

Не рекомендуется. [**Пенинпутпанел**](peninputpanel-class.md) был заменен [панелью ввода текста (TIP)](text-input-panel-reference.md).

Происходит при изменении фокуса ввода до того, как объект [**пенинпутпанел**](peninputpanel-class.md) смог вставить входные данные пользователя в присоединенный элемент управления.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT InputFailed(
  [in] long  hWnd,
  [in] long  Key,
  [in] BSTR  Text,
  [in] short ShiftKey
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*HWND* \[ окне\]
</dt> <dd>

Маркер окна элемента управления, вызвавшего объект [**пенинпутпанел**](peninputpanel-class.md) .

</dd> <dt>

*Ключ* \[ окне\]
</dt> <dd>

Виртуальный ключ, соответствующий нажатой клавише.

</dd> <dt>

*Текстовая надпись* \[ окне\]
</dt> <dd>

Строка, которая должна быть вставлена в элемент управления, представленный параметром *HWND* при возникновении события **инпутфаилед** .

Дополнительные сведения о типе данных BSTR см. в разделе [Использование библиотеки COM](using-the-com-library.md).

</dd> <dt>

*Шифткэй* \[ окне\]
</dt> <dd>

Состояние модификаторов клавиатуры, включая SHIFT, CAPS, CTRL и ALT.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если это событие завершается успешно, возвращается значение **S \_ ОК**. В противном случае возвращается код ошибки **HRESULT** .

## <a name="remarks"></a>Remarks

Событие **инпутфаилед** возникает при изменении фокуса ввода до вставки пользовательского ввода в присоединенный элемент управления. Например, если пользователь вводит рукописный ввод в панель рукописного ввода, а затем переключается на другой элемент управления "поле ввода" до того, как распознаватель уже готов, это событие срабатывает.

С помощью маркера окна, переданного в это событие, можно выбрать вставку текста самостоятельно при возникновении этого события.

> [!Note]  
> начиная с Microsoft Windows XP Tablet PC Edition 2005, событие **инпутфаилед** больше не применяется. Текст всегда вставляется перед изменением фокуса.
