# <a name="input-scopes-and-factoids"></a>Области ввода и Фактоидс

Область ввода — это определенный набор слов, чисел, знаков препинания и синтаксических порядков, допустимых в пределах данной языковой модели. Области ввода могут использоваться приложениями для ограничения языковой модели, используемой распознавателем, определенным контекстом. Например, область ввода \_ смтпемаиладдресс электронной почты \_ влияет на результаты распознавания, указывая, что определенное поле является адресом электронной почты. Таким образом, поле, скорее всего, будет содержать такие символы, как "" @" and " \_ , и не может содержать такие символы, как " \* " и пробелы.

> [!Note]  
> Другие технологии Майкрософт также используют области ввода. В этой статье рассматривается использование контекста для повышения точности модулей распознавания рукописного текста для приложений для планшетных ПК.

 

Предыдущие версии API технологии Tablet PC использовали фактоидс для определения контекста. В практических целях фактоид — это то же самое, что и входная область. Версия одна из платформ пакета SDK для планшетных ПК определила набор значений фактоид в объекте [**фактоид**](factoid-constants.md) . Эти значения использовались для задания контекста и влияния на результаты распознавания при использовании объекта [**рекогнизерконтекст**](inkrecognizercontext-class.md) для распознавания. для распознавателей латиницы, начинающихся с Windows XP Tablet PC Edition 2005, по-прежнему используется свойство [**фактоид**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkrecognizercontext-get_factoid) объекта **рекогнизерконтекст** для задания контекста, но необходимо передать в область ввода, список фраз или значение регулярного выражения рукописного ввода вместо одной версии фактоид значений. Средства распознавания символов восточноазиатских языков не поддерживают использование перечислимых значений области ввода. Следует продолжать использовать значения фактоид для распознавания символов восточноазиатских языков.

Области ввода и фактоидс — это ограничения для альтернативного уровня слов; альтернативы символов могут находиться за пределами входной области, даже если установлен флаг **приведение** .

Если флаг **приведения** установлен с ограниченным фактоид, то может произойти, что распознаватель не может создать ответ, совпадающий с рукописным вводом и удовлетворяющий фактоид. Другой сценарий, в котором распознаватель может не дать никакого удовлетворительного ответа, заключается в том, что язык распознавателя не соответствует языку рукописного ввода (например, распознаватель английского языка и знаки китайского языка (США)).

Если распознаватель рукописного ввода не может распознать рукописный ввод для данного слова или символа, он может либо вернуть пустой список альтернатив, либо альтернативный список, где первый вариант — кодовая точка 0xFFFF (неопределенный символ). Это особенно полезно в режимах ввода в интерактивном режиме, где все поля с рукописным вводом должны иметь непустой список вариантов. Приложение может затем отобразить этот неопределенный символ любым способом (например, "???").

> [!Note]  
> Значения фактоид продолжают работать с распознавателями латинского алфавита для обеспечения обратной совместимости.

 

Дополнительные сведения о фактоидс см. в разделе [Настройка контекста для элементов управления Ink-Enabled](setting-context-for-ink-enabled-controls.md).

Сведения о восточно-азиатских фактоидс см. [в разделе Supported фактоидс from версии 1](supported-factoids-from-version-1.md).
