# <a name="using-active-accessibility-to-expose-custom-controls"></a>Использование Active Accessibility для предоставления настраиваемых элементов управления

Active Accessibility Майкрософт можно использовать как эффективный способ обеспечения совместимости пользовательских элементов управления с вспомогательными средствами. Для Active Accessibility требуется, чтобы приложение:

-   Создание объектов модели COM, представляющих отдельные пользовательские элементы управления или группы элементов управления, которые должны поддерживать интерфейс **IAccessible** . (Объект может быть создан по запросу, когда запрос запрашивается клиентом Active Accessibility.)
-   Вызывайте [**нотифивиневент**](/windows/win32/api/winuser/nf-winuser-notifywinevent) , когда элементы управления создаются или уничтожаются, получают или теряют фокус или иным образом меняют состояние.
-   Обрабатывайте \_ сообщение WM GetObject при использовании для запроса свойств объекта или объектов.

В этом обсуждении окно, содержащее другие пользовательские объекты, также должно быть предоставлено Active Accessibility, что позволяет клиенту обнаруживать дочерние объекты и переходить к ним. Дополнительные сведения о том, как сделать пользовательские элементы управления совместимыми с вспомогательными средствами, см. в разделе [Специальные возможности](../accessibility/accessibility.md).
