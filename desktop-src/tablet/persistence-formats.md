# <a name="persistence-formats"></a>Форматы сохраняемости

Приложение должно иметь возможность создавать и использовать данные из нескольких форматов. Они часто включают в себя собственные двоичные форматы, а также некоторые стандартные форматы, такие как RTF или HTML.

В следующей таблице перечислены некоторые форматы, которые могут содержать рукописные данные.



| Формат                                      | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|---------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Двоичные данные<br/>                           | Приложения должны использовать сериализованный формат рукописного ввода (ISF) для кодирования рукописного ввода в двоичные форматы.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| HTML<br/>                             | Для представления разнородного содержимого настоятельно рекомендуется использовать формат HTML. Приложения должны использовать формат GIF для кодирования рукописного ввода в HTML-документы. Дополнительные сведения о специальная GIF см. в разделе [стандартные блоки](building-blocks.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Образ<br/>                            | Для приложений, для которых отсутствует другой уровень совместимости, приложение с поддержкой рукописного ввода должно перемещать изображения битовой карты и отформатированные метафайлы в буфер обмена.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Формат ISF<br/>      | Формат ISF — наиболее компактное постоянное представление рукописного ввода. Хотя часто он содержит только рукописные данные, ISF является расширяемым. Приложения могут устанавливать настраиваемые атрибуты (определяемые глобальным уникальным идентификатором (GUID)) на объекте [**рукописного ввода**](inkdisp-class.md) , рукописном штрихе или точке рукописного ввода. Это позволяет хранить данные или метаданные в виде атрибута в потоке ISF. Для взаимодействия с буфером обмена рукописный ввод можно поместить в стандартный слот буфера обмена для ISF, который определен в файлах заголовков пакета средств разработки программного обеспечения (SDK).<br/> ISF — это формат, характерный для технологии Microsoft Tablet PC, и поддерживается только в методах [**Load**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkdisp-load) и [**Save**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkdisp-save) объекта [**Ink**](inkdisp-class.md) .<br/> |
| RTF<br/>                              | Можно создать формат буфера обмена RTF и кодировать рукописные данные в RTF как объекты OLE. это позволяет вставлять рукописный ввод в контейнер OLE, например Microsoft Word или приложение на основе RichEdit.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| расширяемый язык разметки (XML)<br/> | Приложения могут использовать один из форматов рукописного ввода Base-64 в кодировке для хранения рукописного ввода в формате XML-файла. Формат XML удобен для ввода рукописного содержимого в базу данных, как в случае с полем подписи, или даже в качестве основного формата файла приложения. Это устраняет необходимость в написании средства синтаксического анализа.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                        |
