# <a name="snipping-tool-support-in-windows-vista"></a>поддержка ножниц в Windows Vista

В этом разделе описывается, как в приложении можно указать URL-адрес, который должен получить средство выделения для планшетного ПК при записи приложения.

## <a name="specifying-the-url-via-registry-key"></a>Указание URL-адреса через раздел реестра

Ножницы позволяют пользователям захватить фрагмент (снимок экрана) любого объекта на экране, а затем добавить к нему заметки, сохранить или поделиться этим изображением. Если данные сохраняются в формате HTML или отправляются в почтовый клиент, поддерживающий встроенный HTML, средство выделения фрагментов кода может добавить URL-адрес к фрагменту, если приложение предоставляет сведения о том, как получить URL-адрес.

Средство выделения фрагментов получает URL-адрес через объекты специальных возможностей. Приложения должны указать необходимые сведения в следующих разделах реестра:

HKLM \\ Software \\ Microsoft \\ Windows \\ \\ средство выделения \\ линкфинжерпринтс TabletPC,

И должны создать подраздел, имя которого совпадает с именем класса окна, из которого должна быть получена ссылка. Имя класса Window должно быть самым верхним окном приложения.

HKLM \\ Software \\ Microsoft \\ Windows \\ \\ средство выделения \\ линкфинжерпринтс TabletPC\\<Window Class Name>

### <a name="window-class-key-details"></a>Сведения о ключе класса Window

В разделе "ключ класса окон" должны быть заданы соответствующие значения, чтобы указать, что средство выделения должно обнаруживать правильный объект специальных возможностей.



| ЗНАЧЕНИЕ                        | TYPE                  | ВИДЕ            | ХРАНЯЩИЕСЯ СВЕДЕНИЯ                                          |
|------------------------------|-----------------------|-----------------|-------------------------------------------------------------|
| Mask<br/>              | REG \_ DWORD<br/> |                 | Указывает, какие из следующих полей следует проверить<br/> |
| Имя<br/>              | REG \_ SZ<br/>    | 0x02<br/> | Имя специальных возможностей<br/>                               |
| Описание<br/>       | REG \_ SZ<br/>    | 0x04<br/> | Описание специальных возможностей<br/>                        |
| Роль<br/>              | REG \_ DWORD<br/> | 0x08<br/> | Роль специальных возможностей<br/>                               |
| ParentName<br/>        | REG \_ SZ<br/>    | 0x10<br/> | Имя родительского элемента<br/>                     |
| парентвалуе<br/>       | REG \_ SZ<br/>    | 0x20<br/> | Значение доступности родителя<br/>                    |
| парентроле<br/>        | REG \_ DWORD<br/> | 0x40<br/> | Роль доступности родителя<br/>                     |
| парентдескриптион<br/> | REG \_ SZ<br/>    | 0x80<br/> | Описание специальных возможностей родителя<br/>              |



 

Кроме того, если задано значение битовой маски 0x1, то URL-адрес должен быть взят из имени специальных возможностей. в противном случае URL-адрес должен быть получен из значения специальных возможностей.

Если приложение использует локализованные строки для значений REG \_ SZ, приведенных выше, строка должна быть указана в качестве косвенной строки в следующем формате:

@filename, ресурс

Строка извлекается из файла с именем и используется значение ресурса в качестве указателя. Если значение ресурса равно нулю или больше, то число становится индексом строки в двоичном файле. Если число отрицательное, оно становится идентификатором ресурса (ИДЕНТИФИКАТОРом).

> [!Note]  
> константы ролей можно найти в олеакк. h в Windows SDK. описанные здесь параметры реестра относятся только к Windows Vista.
