# <a name="ianalysisalternate-interface"></a>Интерфейс Ианалисисалтернате

Представляет возможные совпадения слов распознавания рукописного текста для объектов [**иконтекстноде**](icontextnode.md) .

## <a name="members"></a>Элементы

Интерфейс **ианалисисалтернате** наследует от интерфейса [**IUnknown**](/windows/desktop/api/unknwn/nn-unknwn-iunknown) . **Ианалисисалтернате** также имеет следующие типы членов:

-   [Методы](#methods)

### <a name="methods"></a>Методы

Интерфейс **ианалисисалтернате** содержит следующие методы.



| Метод                                                                          | Описание                                                                                                                                                          |
|:--------------------------------------------------------------------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**жеталтернатенодес**](ianalysisalternate-getalternatenodes.md)               | Извлекает объекты [**иконтекстноде**](icontextnode.md) , связанные с этим альтернативным.<br/>                                                       |
| [**жетрекогнитионконфиденце**](ianalysisalternate-getrecognitionconfidence.md) | Извлекает значение, указывающее уровень достоверности, который [**иинканализер**](iinkanalyzer.md) имеет точность **ианалисисалтернате**.<br/> |
| [**жетрекогнизедстринг**](ianalysisalternate-getrecognizedstring.md)           | Извлекает распознанное строковое значение объекта **ианалисисалтернате** .<br/>                                                                               |
| [**жетстрокеидс**](ianalysisalternate-getstrokeids.md)                         | Получает идентификаторы штриха, связанные с этим **ианалисисалтернате**.<br/>                                                                    |



 

## <a name="remarks"></a>Remarks

Существует множество вариантов рукописного ввода пользователей. По этой причине Распознаватели рукописного ввода иногда могут преобразовывать рукописный ввод в текст, отличный от предполагаемого пользователя. Когда [**иинканализер**](iinkanalyzer.md) выполняет анализ коллекции штрихов, **иинканализер** находит наиболее вероятный набор слов, которые представляет рукописный ввод. Кроме того, **иинканализер** находит наборы альтернативных совпадений распознавания, которые хранятся в коллекции [**ианалисисалтернатес**](ianalysisalternates.md) . Чтобы пользователь мог воспользоваться преимуществами распознавания, необходимо создать пользовательский интерфейс, позволяющий пользователю выбрать правильный **ианалисисалтернате**.

Объекты **ианалисисалтернате** обычно получаются с помощью [**метода иинканализер::-alternate**](iinkanalyzer-getalternates.md). Первый объект **ианалисисалтернате** в коллекции — это то, что [**иинканализер**](iinkanalyzer.md) считает наиболее вероятной альтернативой.

Этот интерфейс эквивалентен [System. Windows. Ink. аналисискоре. аналисисалтернатебасе](/previous-versions/ms610092(v=vs.100)) в платформа .NET Framework.
