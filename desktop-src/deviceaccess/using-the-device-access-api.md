# <a name="how-to-use-the-device-access-api"></a>Как использовать API доступа к устройствам

В этом разделе содержатся задачи и рекомендации по проектированию для использования API доступа к устройствам.

## <a name="steps"></a>Этапы

| Раздел | Описание |
|---|---|
| [Рекомендации по проектированию для пользовательских устройств](design.md)<br/> | В этом разделе описаны рекомендации по проектированию, которые помогут определить, требуется ли устройству пользовательский драйвер.<br/> |
| [Реализация объекта доступа к устройству](create-the-device-access-object.md)<br/> | В этом разделе объясняется, как создать объект доступа к устройству и использовать его для доступа к устройству. <br/>  |
| [Объявление возможностей устройства](declaring-the-device-capability.md)<br/> | В этом разделе объясняется, как объявить GUID возможностей устройства.<br/> |
| [Зарегистрируйте интерфейс устройства как ограниченные для привилегированных приложений](register-the-device-interface-class-as-privileged.md)<br/> | В этом разделе показано, как добавить свойство **Restricted** , указывающее, что только привилегированные приложения могут обращаться к классу интерфейса устройства.<br/> |
| [создание расширения среда выполнения Windows](create-a-windows-runtime-extension.md)<br/> | можно создать компонент среда выполнения Windows, чтобы заключить в оболочку компонент, обращающийся к вашему драйверу. затем можно написать приложение для устройства магазина Windows для устройства на C# или JavaScript.<br/> |
