# <a name="device-access-glossary"></a>Глоссарий доступа к устройствам

Ниже приведены термины, используемые во всей документации по API доступа к устройствам.

<dl> <dt>

**AppContainer**
</dt> <dd>

Среда выполнения с высокой степенью ограничений, в которой выполняется процесс с ограниченным подмножеством привилегий пользователя. Процесс, выполняющийся в AppContainer, называется процессом AppContainer. Процесс AppContainer изолирован от других процессов AppContainer и из профиля пользователя. Он имеет ограниченный доступ к очень небольшому подмножеству системных ресурсов, таких как файлы, устройства, разделы реестра, конечные точки вызова процедур (IPC)/ремоте (RPC) и сетевые ресурсы.

</dd> <dt>

**Привязка**
</dt> <dd>

Связывание объекта доступа к устройству с определенным интерфейсом устройства. если привязка выполнена успешно, Windows приложения магазина могут использовать объект доступа к устройству в качестве брокера для взаимодействия с драйвером устройства.

</dd> <dt>

**Broker**
</dt> <dd>

Компонент, предоставляющий доступ к ресурсу, который не предоставляется по умолчанию.

</dd> <dt>

**Привилегированное приложение**
</dt> <dd>

Приложение, определенное в метаданных конкретного устройства, связанное с этим устройством, чтобы оно может взаимодействовать с ограниченным интерфейсом драйвера устройства. Примером такого приложения может быть собственное приложение для воспроизведения музыки, которое имеет исключительные разрешения на синхронизацию с портативным музыкальным проигрывателем, когда приложения от конкурентов не могут. Дополнительные сведения о настройке метаданных устройства или о том, как ограничить драйвер для привилегированных приложений, см. в статье [приложения устройства UWP для внутренних устройств](/windows-hardware/drivers/devapps/uwp-device-apps-for-specialized-devices).

</dd> </dl>
