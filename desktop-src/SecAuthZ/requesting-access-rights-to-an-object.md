# <a name="requesting-access-rights-to-an-object"></a>Запрос прав доступа к объекту

При открытии обработчика для объекта возвращенный обработчик имеет некоторую комбинацию прав доступа к объекту. Для некоторых функций, таких как [**createsemaphore-**](/windows/desktop/api/winbase/nf-winbase-createsemaphorea), не требуется определенный набор запрошенных прав доступа. Эти функции всегда пытаются открыть маркер для полного доступа. Другие функции, такие как [**CreateFile**](/windows/desktop/api/fileapi/nf-fileapi-createfilea) и [**OpenProcess**](/windows/desktop/api/processthreadsapi/nf-processthreadsapi-openprocess), позволяют указать необходимый набор прав доступа. Следует запрашивать только необходимые права доступа, а не открывать полный доступ. Это предотвращает непреднамеренное использование этого маркера и увеличивает вероятность того, что запрос доступа будет выполнен, если DACL объекта допускает только ограниченный доступ.

Используйте [универсальные права доступа](generic-access-rights.md) , чтобы указать тип доступа, необходимый при открытии маркера для объекта. Обычно это проще, чем указывать все соответствующие стандартные и специальные права. Кроме того, используйте максимально \_ допустимую константу, чтобы запросить открытие объекта со всеми правами доступа, которые являются допустимыми для вызывающего.

> [!Note]  
> МАКСИМАЛЬНО \_ допустимую константу нельзя использовать в элементе управления доступом.

 

Чтобы получить или задать список SACL в [*дескрипторе безопасности*](/windows/desktop/SecGloss/s-gly)объекта, запросите [ \_ право доступа системы доступа \_ безопасности](sacl-access-right.md) при открытии дескриптора объекта.
