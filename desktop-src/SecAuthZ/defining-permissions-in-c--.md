# <a name="defining-permissions-in-c"></a>Определение разрешений в C++

В диспетчере авторизации вы определяете, какие пользователи имеют доступ к ресурсам приложения, создавая хранилище политик авторизации.

Дополнительные сведения об определении разрешений с помощью списков управления доступом см. [в разделе Определение разрешений с помощью ACL в C++](defining-permissions-with-acls-in-c--.md).

**Определение разрешений на доступ**

1.  Создайте хранилище, в котором определена политика авторизации:<dl>

[Создание объекта хранилища политики авторизации в C++](creating-an-authorization-policy-store-object-in-c--.md)  
    </dl>
2.  Создайте раздел в хранилище политик авторизации для определенного приложения:<dl>

[Создание объекта приложения на C++](creating-an-application-object-in-c--.md)  
    </dl>
3.  Определение операций, реализуемых приложением для доступа к ресурсам и их изменения:<dl>

[Определение операций в C++](defining-operations-in-c--.md)  
    </dl>
4.  Сгруппируйте операции по задачам высокого уровня, которые пользователи хотят выполнить:<dl>

[Группирование операций в задачи на C++](grouping-operations-into-tasks-in-c--.md)  
    </dl>
5.  Определите роли, состоящие из групп задач:<dl>

[Группирование задач в роли в C++](grouping-tasks-into-roles-in-c--.md)  
    </dl>Пользователь, назначенный роли, имеет разрешение на выполнение любой задачи, назначенной этой роли.
6.  Создание скриптов для определения доступа к задачам во время выполнения:<dl>

[Квалификация доступа с помощью бизнес-логики в C++](qualifying-access-with-business-logic-in-c--.md)  
    </dl>Это необязательный шаг.
7.  Определите группы пользователей:<dl>

[Определение групп пользователей в C++](defining-groups-of-users-in-c--.md)  
    </dl>Затем эти группы можно назначить ролям, чтобы определить, какие задачи они могут выполнять.
8.  Добавление пользователей в группы пользователей:<dl>

[Добавление пользователей в группу приложений в C++](adding-users-to-an-application-group-in-c--.md)  
    </dl>
