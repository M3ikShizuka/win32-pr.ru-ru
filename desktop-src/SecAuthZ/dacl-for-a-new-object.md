# <a name="dacl-for-a-new-object"></a>DACL для нового объекта

Система использует следующий алгоритм для создания списка DACL для большинства типов новых защищаемых объектов:

1.  DACL объекта — это список DACL из [*дескриптора безопасности*](/windows/desktop/SecGloss/s-gly) , указанного создателем объекта. Система объединяет все наследуемые ACE в указанный список DACL, если в \_ \_ контрольных битах дескриптора безопасности не установлен бит защищенного списка DACL SE.
2.  Если создатель не указал дескриптор безопасности, система создает DACL объекта из наследуемых ACE.
3.  Если дескриптор безопасности не указан и не существует наследуемых ACE, DACL объекта является DACL по умолчанию от [*основного*](/windows/desktop/SecGloss/p-gly) [*маркера или токена олицетворения*](/windows/desktop/SecGloss/i-gly) создателя.
4.  Если указанный, унаследованный или DACL по умолчанию нет, система создает объект без DACL, что позволяет всем пользователям иметь полный доступ к объекту.

Система использует другой алгоритм для создания списка DACL для нового объекта Active Directory. Дополнительные сведения см. [в разделе как устанавливаются дескрипторы безопасности для новых объектов каталога](/windows/desktop/AD/how-security-descriptors-are-set-on-new-directory-objects).
