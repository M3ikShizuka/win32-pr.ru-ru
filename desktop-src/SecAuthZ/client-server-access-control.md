# <a name="clientserver-access-control"></a>Управление доступом клиента/сервера

Серверное приложение предоставляет клиентам службы. Например, сервер может выполнять следующие службы от имени клиента:

-   Сохранение и получение сведений из частной базы данных
-   Доступ к сетевым ресурсам
-   Запуск [*процессов*](/windows/desktop/SecGloss/p-gly) в [*контексте безопасности*](/windows/desktop/SecGloss/s-gly) клиента на компьютере сервера

Защищенный сервер управляет доступом к своим службам. Windows обеспечивает поддержку безопасности, которая позволяет серверу выполнять следующие действия:

-   Олицетворяет клиентский [*контекст безопасности*](/windows/desktop/SecGloss/s-gly), что заставляет систему выполнять большинство проверок доступа и [*привилегий*](/windows/desktop/SecGloss/p-gly) для [*маркера доступа*](/windows/desktop/SecGloss/a-gly) клиента, а не сервера
-   Регистрация клиента на компьютере сервера
-   Подключение сетевых ресурсов с помощью контекста безопасности клиента
-   Создание [*дескрипторов безопасности*](/windows/desktop/SecGloss/s-gly) для защиты закрытых объектов
-   Определение возможности доступа к клиенту с помощью дескриптора безопасности
-   Определение того, включен ли набор привилегий в маркере клиента
-   Создание сообщений аудита в журнале событий безопасности для записи попыток клиента получить доступ к объектам или использовать привилегии
