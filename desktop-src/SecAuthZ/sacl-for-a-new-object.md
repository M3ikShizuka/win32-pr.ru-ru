# <a name="sacl-for-a-new-object"></a>Список SACL для нового объекта

Для создания списка SACL для большинства типов новых защищаемых объектов система использует следующий алгоритм:

1.  Список SACL объекта является списком SACL из [*дескриптора безопасности*](/windows/desktop/SecGloss/s-gly) , указанного создателем объекта. система объединяет все наследуемые ace в указанный список SACL, если в \_ \_ контрольных битах дескриптора безопасности не установлен бит защищенного списка sacl SE. атрибуты доступа к системным \_ ресурсам \_ \_ ace и \_ \_ идентификатор политики системы \_ в пределах \_ родительского объекта будут объединены в новый объект, даже если \_ \_ установлен бит защищенного списка SACL SE.
2.  Если создатель не указал дескриптор безопасности, система создает список SACL объекта из наследуемых ACE.
3.  Если указанный или унаследованный список SACL отсутствует, объект не имеет списка SACL.

чтобы указать список SACL для нового объекта, создателю объекта необходимо \_ \_ включить [привилегию](privileges.md) SE "имя безопасности". если указанный список SACL для нового объекта содержит только \_ \_ ace атрибута системных ресурсов \_ , то \_ привилегия SE имя безопасности \_ не требуется. Автор не нуждается в этой привилегии, если SACL объекта построен из унаследованных ACE.

Система использует другой алгоритм для создания списка SACL для нового объекта Active Directory. Дополнительные сведения см. [в разделе как устанавливаются дескрипторы безопасности для новых объектов каталога](/windows/desktop/AD/how-security-descriptors-are-set-on-new-directory-objects).
