# <a name="general-guidelines-for-implementing-raw-codecs"></a>Общие рекомендации по реализации необработанных кодеков

По сравнению с необработанными типами изображений, такими как JPEG или TIFF, существуют два важных отличия в том, как предполагается, что форматы RAW-изображений работают в Windows:

-   Большинство форматов необработанных изображений предполагается «только для чтения» и, возможно, не будет поддерживать кодирование пикселей в Необработанный формат. однако, поскольку компоненту Windows imaging (WIC) необходим кодировщик для поддержки обратной записи метаданных, авторы необработанных кодеков должны предусмотреть реализацию по крайней мере каркаса класса кодировщика.
-   Декодирование необработанного образа в полном размере может занять много времени по сравнению с другими форматами. По этой причине корпорация Майкрософт рекомендует принять определенные подходы к уменьшению задержки декодирования и обеспечить поддержку таких сценариев, как быстрое отображение эскизов и предварительных версий.

    Например, все авторы необработанного кодека должны реализовать интерфейс [**ивикбитмапсаурцетрансформ**](/windows/desktop/api/Wincodec/nn-wincodec-iwicbitmapsourcetransform) , который предоставляет механизм для уведомления декодера заранее до размера целевого растрового изображения, что позволяет оптимизировать декодирование до меньшего размера выходного изображения.
