# <a name="feature-completeness-recommended-interfaces"></a>Полнота компонентов: Рекомендуемые интерфейсы

в следующей таблице перечислены интерфейсы компонента обработки изображений Windows, которые должны быть реализованы в коде необработанных кодеков (WIC).




| Интерфейс | Обязательно для: | Описание | 
|-----------|--------------|-------------|
| <a href="/windows/desktop/api/Wincodec/nn-wincodec-iwicbitmapdecoder"><strong>ивикбитмапдекодер</strong></a> | Декодеров | Представляет начальную точку для декодирования файла изображения. Предоставляет доступ к свойствам уровня контейнера, таким как эскизы, фреймы и палитра.<br /> | 
| <a href="/windows/desktop/api/Wincodec/nn-wincodec-iwicbitmapframedecode"><strong>IWICBitmapFrameDecode</strong></a> | Декодеров | Представляет конкретный кадр изображения в контейнере, который предоставляет доступ к свойствам на уровне фрейма. Это интерфейс, который декодирует фактические биты изображения.<br /> | 
| <a href="/windows/desktop/api/Wincodecsdk/nn-wincodecsdk-iwicmetadatablockreader"><strong>ивикметадатаблоккреадер</strong></a> | Декодеров | Требуется для перечисления и итерации блоков метаданных и вызова соответствующих средств чтения метаданных при чтении из файла изображения. <br /><blockquote>[!Note]<br />Если Необработанный формат контейнера совместим с TIFF или использует стандартные IFD или Ирбс для хранения метаданных EXIF или XMP, авторы кодеков могут выбрать вызов встроенных средств чтения метаданных, а не писать их самостоятельно.</blockquote><br /><br /> | 
| <a href="/windows/desktop/api/Wincodec/nn-wincodec-iwicbitmapsourcetransform"><strong>ивикбитмапсаурцетрансформ</strong></a> | Декодеров | Позволяет вызывающему объекту указать требуемое масштабирование, обрезку, поворот или формат пикселей для декодированного изображения, что может значительно повысить производительность декодера. Например, декодеры в формате JPEG и беспроводной связи (формате WDP) корпорации Майкрософт используют схему оптимизации пирамиды для ускорения декодирования, если целевой точечный рисунок меньше исходного растрового изображения. Windows В Vista (и более поздних версиях) будет предпринята попытка использовать этот интерфейс для извлечения "быстрого" предварительного просмотра из необработанного образа при отсутствии или менее чем 1 024 пикселей в самом большом измерении.<br /> | 
| <a href="/windows/desktop/api/Wincodec/nn-wincodec-iwicdevelopraw"><strong>ивикдевелоправ</strong></a> | Декодеров | Требуется для необработанных форматов. Предоставляет параметры, характерные для обработки необработанных изображений. Необработанные кодеки должны поддерживать столько же параметров, сколько применяется к кодеку.<br /> | 
| <a href="/windows/desktop/api/wincodec/nn-wincodec-iwicbitmapencoder"><strong>ивикбитмапенкодер</strong></a> | Кодировщики | Представляет отправную точку для кодирования файла изображения. Этот интерфейс используется для задания свойств уровня контейнера, таких как эскизы, кадры и палитра. Также необходимо вызвать модуль записи метаданных, чтобы включить сохраняемость метаданных в файле изображения. По этим причинам этот интерфейс необходим даже в том случае, если кодирование первичного точечного рисунка в Необработанный формат не поддерживается.<br /> | 
| <a href="/windows/desktop/api/Wincodec/nn-wincodec-iwicbitmapframeencode"><strong>ивикбитмапфраминкоде</strong></a> | Кодировщики | Представляет конкретный кадр изображения в контейнере. Этот интерфейс используется для кодирования фактических битов изображения и для задания метаданных и свойств каждого кадра.<br /> | 
| <a href="/windows/desktop/api/Wincodecsdk/nn-wincodecsdk-iwicmetadatablockwriter"><strong>ивикметадатаблокквритер</strong></a> | Кодировщики | Требуется для прохода по блокам метаданных и вызова соответствующих модулей записи метаданных при сериализации файла изображения.<br /><blockquote>[!Note]<br />Если Необработанный формат контейнера совместим с TIFF, авторы кодеков могут выбрать вызов встроенных модулей записи метаданных, а не писать их самостоятельно.</blockquote><br /><br /> |
