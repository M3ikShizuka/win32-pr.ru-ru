# <a name="hierarchies"></a>Иерархии

Службы сертификатов поддерживают иерархии [*центров сертификации*](../secgloss/c-gly.md) (ЦС). [*Иерархия ЦС*](../secgloss/c-gly.md) состоит из ЦС верхнего уровня (называемого корневым ЦС) с одним или несколькими подчиненными ЦС, которые выдавали сертификаты корневым ЦС. Возможный сценарий иерархии ЦС будет в Организации с одним корневым центром сертификации, который использовался для выдаче сертификатов нескольким подчиненным ЦС. Затем подчиненные ЦС могут выдавать сертификаты конечного субъекта, например сертификаты, выданные пользователю. Сертификат пользователя будет содержать путь доверия для иерархии ЦС, в данном случае, включая сертификат для выдающего подчиненного ЦС, а также корневого ЦС.
