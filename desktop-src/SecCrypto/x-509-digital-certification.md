# <a name="x509-digital-certification"></a>Цифровая сертификация X.509

Основной задачей [*цифрового сертификата*](../secgloss/d-gly.md) является предоставление доступа к [*открытому ключу*](../secgloss/p-gly.md)субъекта. Сертификат также подтверждает, что открытый ключ сертификата принадлежит субъекту сертификата. Например, [*центр сертификации*](../secgloss/c-gly.md) может подписать специальное сообщение (сведения о сертификате), которое содержит имя какого-либо пользователя, например "Алиса", и его открытый ключ. Это необходимо сделать таким образом, чтобы любой пользователь мог проверить, был ли сертификат выдан и подписан не центром сертификации. Если центр сертификации является доверенным и можно проверить, что сертификат Алисы выдан этим ЦС, любой получатель сертификата Алисы может доверять открытому ключу Алисы от этого сертификата.

Типичная реализация цифровой сертификации включает в себя процесс подписи сертификата.

Процесс выглядит следующим образом:

1.  Алиса отправляет подписанный [*сертификат*](../secgloss/c-gly.md) , содержащий его имя, Открытый ключ и, возможно, некоторую дополнительную информацию для центра сертификации.
2.  ЦС создает сообщение *m* из запроса Алисы. ЦС подписывает сообщение с помощью закрытого ключа, создавая отдельное сообщение с подписью *SIG*. ЦС возвращает Алисе сообщение, *m* и подпись, *SIG*. Вместе, *m* и *SIG* формируют сертификат Алисы.
3.  Алиса отправляет Бобу обе части сертификата, чтобы предоставить ему доступ к своему открытому ключу.
4.  Боб проверяет подпись ( *SIG* ), используя открытый ключ ЦС. Если подпись подтверждается действительным, он принимает открытый ключ в сертификате как открытый ключ Алисы.

Как и в случае с любой [*цифровой подписью*](../secgloss/d-gly.md), любой получатель с доступом к открытому ключу ЦС может определить, подписан ли сертификат конкретным центром сертификации. Этот процесс не требует доступа к секретным сведениям. Только что представленный сценарий предполагает, что Боб имеет доступ к открытому ключу ЦС. Боб будет иметь доступ к этому ключу, если у него есть копия сертификата ЦС, который содержит этот открытый ключ.

Цифровые сертификаты [*X. 509*](../secgloss/x-gly.md) включают не только имя пользователя и открытый ключ, но и другие сведения о пользователе. Эти сертификаты больше, чем пошаговые камни в цифровой иерархии доверия. Они позволяют ЦС предоставить получателю сертификата возможность доверять не только открытому ключу субъекта сертификата, но и другим сведениям о субъекте сертификата. Другие сведения могут включать, помимо прочего, адрес электронной почты, авторизацию для подписывания документов заданного значения или авторизацию для того, чтобы стать центром сертификации и подписанием других сертификатов.

Сертификаты X. 509 и многие другие сертификаты имеют допустимый срок действия. Срок действия сертификата может истечь, и он перестанет быть действительным. Центр сертификации может отозвать сертификат по ряду причин. Для решения отзыва ЦС обслуживает и распространяет список отозванных сертификатов, называемый [*списком отзыва сертификатов*](../secgloss/c-gly.md) (CRL). Пользователи сети получают доступ к списку отзыва сертификатов для определения допустимости сертификата.
