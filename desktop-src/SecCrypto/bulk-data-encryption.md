# <a name="bulk-data-encryption"></a>Групповое шифрование данных

[*Ключ шифрования*](../secgloss/b-gly.md) создается путем [*хэширования*](../secgloss/h-gly.md) одного из ключей Mac с помощью [**крипсашсессионкэй**](/windows/desktop/api/Wincrypt/nf-wincrypt-crypthashsessionkey) вместе с содержимым сообщения и другими данными. Это сообщение шифруется или расшифровывается с помощью одного из ключей шифрования с обычным способом.

При использовании [*блочного шифра*](../secgloss/b-gly.md)ядро протокола [*SChannel*](../secgloss/s-gly.md) выполняет все необходимое [*Заполнение*](../secgloss/p-gly.md) *блочного шифра* . При вызове [**криптенкрипт**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptencrypt) и [**криптдекрипт**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptdecrypt) последний флаг всегда имеет **значение false** , а длина данных является кратной длине целых блоков.

> [!Note]  
> CSP не должен выполнять внутреннюю буферизацию данных. После того как данные зашифрованы (или расшифрованы), размер [*открытого текста*](../secgloss/p-gly.md) всегда должен точно совпадать с размером [*зашифрованного текста*](../secgloss/c-gly.md).
