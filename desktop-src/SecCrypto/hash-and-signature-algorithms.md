# <a name="hash-and-signature-algorithms"></a>Алгоритмы хэширования и подписи

Следующие алгоритмы вычисляют [*хэши*](../secgloss/h-gly.md) и [*цифровые подписи*](../secgloss/d-gly.md). Каждый из этих алгоритмов поддерживается в службах Microsoft Base, Strong и усовершенствованных поставщиков служб шифрования. Внутренние сведения об этих алгоритмах выходят за рамки данной документации. Список дополнительных источников см. в [дополнительной документации по шифрованию](additional-documentation-on-cryptography.md).



| Алгоритмы                                     | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Цепочка блочного шифрования (CBC) MAC<br/>     | Одним из алгоритмов (КАЛГ \_ Mac), реализованных поставщиками Майкрософт, является [*блочный шифр*](../secgloss/b-gly.md) [*код проверки подлинности сообщения*](../secgloss/m-gly.md) (Mac). Этот метод шифрует базовые данные с помощью блочного шифра, а затем использует последний зашифрованный блок в качестве [*хэш-*](../secgloss/h-gly.md) значения. Алгоритм шифрования, используемый для создания MAC, — это тот, который был указан при создании ключа сеанса. <br/>                                                                            |
| Код проверки подлинности сообщений с помощью хэш-функций<br/>                                | Алгоритм (КАЛГ \_ HMAC), реализованный поставщиками Майкрософт. Этот алгоритм также использует [*симметричный ключ*](../secgloss/s-gly.md) для создания [*хэша*](../secgloss/h-gly.md), но он более сложен по сравнению с алгоритмом [*шифрования*](../secgloss/c-gly.md) Mac. Его можно использовать с любым циклическим хэш-алгоритмом шифрования, например MD5 или SHA-1. Дополнительные сведения см. в разделе [Создание HMAC](creating-an-hmac.md).<br/>                                                                                       |
| Технологии MD2, MD4 и MD5<br/>                   | Эти алгоритмы хэширования были разработаны с помощью RSA Data Security, Inc. Эти алгоритмы были разработаны в последовательном порядке. Все три создания 128-разрядных хэш-значений. Все три имеют слабые места и должны использоваться только там, где это необходимо для обеспечения совместимости. Для нового кода рекомендуется использовать семейство хэшей SHA-2.<br/> Эти алгоритмы хорошо известны и могут быть подробно рассмотрены в любой ссылке на [*криптографию*](../secgloss/c-gly.md).<br/>                                                                                                                                                           |
| Код проверки подлинности сообщения (MAC)<br/>   | Алгоритмы MAC похожи на алгоритмы [*хэширования*](../secgloss/h-gly.md) , но вычисляются с помощью [*симметричного*](../secgloss/s-gly.md) ключа (сеанса). Исходный [*ключ сеанса*](../secgloss/s-gly.md) требуется для повторного вычисления хэш-значения. Повторно вычисленное хэш-значение используется для проверки того, что базовые данные не изменялись. Эти алгоритмы иногда называются алгоритмами хэширования с ключом. Сведения о том, какие поставщики Майкрософт поддерживают MAC, см. в [статье поставщики служб шифрования Майкрософт](microsoft-cryptographic-service-providers.md).<br/>    |
| Безопасный алгоритм хэширования (SHA-1)<br/>       | Этот алгоритм хеширования был разработан [*национальным институтом стандартов и технологий*](../secgloss/n-gly.md) (NIST) и национальным агентством по безопасности (NSA). Этот алгоритм был разработан для использования с DSA (алгоритмом цифровых подписей) или DSS (стандарт цифровой подписи). Этот алгоритм создает 160-разрядное [*хэш-*](../secgloss/h-gly.md) значение. Алгоритм SHA-1 известен под слабости, и его следует использовать только в том случае, если это необходимо для обеспечения совместимости. Для нового кода рекомендуется использовать семейство хэшей SHA-2.<br/> |
| Безопасный алгоритм хэширования — 2 (SHA-2)<br/>   | Этот алгоритм хэширования был разработан как преемник SHA-1 [*национальным институтом стандартов и технологий*](../secgloss/n-gly.md) (NIST) и Агентством национальной безопасности (NSA). Он имеет четыре варианта: SHA-224, SHA-256, SHA-384 и SHA-512, которые называются в соответствии с количеством битов в их выходных данных. Эти алгоритмы SHA-256, SHA-384 и SHA-512 реализованы в поставщике служб шифрования Microsoft AES.<br/>                                                                                                                                |
| Алгоритм авторизации клиента SSL3<br/> | Этот алгоритм используется для проверки подлинности клиента SSL3. В протоколе SSL3 объединение хэша MD5 и хэша SHA подписывается с помощью [*закрытого ключа*](../secgloss/p-gly.md)RSA. CryptoAPI 2.0 и Microsoft Base и усовершенствованные поставщики служб шифрования поддерживают это с типом [*ХЕША*](../secgloss/h-gly.md) калг \_ SSL3 \_ SHAMD5. Дополнительные сведения см. [в разделе Создание калг \_ SSL3 \_ SHAMD5 hash](creating-a-calg-ssl3-shamd5-hash.md).<br/>                                                                                                                                               |
