# <a name="writing-custom-modules"></a>Написание пользовательских модулей

Модули политики можно писать на языке программирования C, C++ или Visual Basic. Необходимый компилятор Microsoft доступен в Visual C++ версии 5,0 или более поздней или в Visual Basic версии 5,0 или более поздней. В [*центре сертификации*](../secgloss/c-gly.md) предприятия (ЦС) должен использоваться только модуль политики предприятия, предоставленный корпорацией Майкрософт.

При написании пользовательского модуля политики необходимо реализовать [**ицертполици**](/windows/desktop/api/Certpol/nn-certpol-icertpolicy) . Кроме того, при написании пользовательского модуля политики необходимо реализовать [**ицертманажемодуле**](/windows/desktop/api/Certmod/nn-certmod-icertmanagemodule) .

Модули политики могут вызывать методы из [**ицертсерверполици**](/windows/desktop/api/Certif/nn-certif-icertserverpolicy) для помощи в обработке [*запроса на сертификат*](../secgloss/c-gly.md); **Ицертсерверполици** позволяет задавать и получать свойства и расширения сертификатов.
