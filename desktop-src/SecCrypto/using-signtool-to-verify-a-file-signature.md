# <a name="using-signtool-to-verify-a-file-signature"></a>Использование средства SignTool для проверки подписи файла

Следующая команда проверяет подпись файла с именем *MyControl.exe*:

**Проверка средства SignTool** *MyControl.exe*

Если предыдущий пример завершается сбоем, то в сигнатуре может использоваться сертификат подписи кода. По умолчанию средство [SignTool](signtool.md) имеет политику драйвера Windows для проверки.

Следующая команда проверяет подпись с помощью политики проверки подлинности по умолчанию:

Средство **SignTool Verify/па** *MyControl.exe*

Следующая команда проверяет системный файл, который может быть подписан в каталоге:

Средство **SignTool Verify/a** *SysFile.dll*

Следующая команда проверяет системный файл, подписанный в каталоге с именем *MyCat.Cat*:

Средство **SignTool Verify/c** *MyCat.catMyFile.ini*

Для любой проверки [SignTool](signtool.md) можно получить сведения о подписавшем сертификата. Следующая команда проверяет системный файл и отображает сертификат подписавших:

Средство **SignTool проверка/v** *MyControl.exe*

Средство [SignTool](signtool.md) возвращает текст командной строки, который указывает результат проверки подписи. Кроме того, средство SignTool возвращает нулевой код выхода для успешного выполнения, один для неудачного выполнения и два для выполнения, которые завершились с предупреждениями.

Дополнительные сведения о [SignTool](signtool.md)см. в разделе [SignTool](signtool.md).
