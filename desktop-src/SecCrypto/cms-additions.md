# <a name="cms-additions"></a>Дополнения CMS

Синтаксис криптографических сообщений (CMS), производный от PKCS \# 7 версии 1,5, является синтаксисом, используемым для [*хэширования*](../secgloss/h-gly.md), [*цифровой подписи*](../secgloss/d-gly.md), проверки подлинности и шифрования произвольных сообщений. Там, где это возможно, обратная совместимость сохраняется; Однако были внесены изменения в соответствии с переносом сертификата атрибута и методами ключевых соглашений для управления ключами. CMS позволяет использовать несколько инкапсуляций, чтобы один конверт инкапсуляции можно было вложить в другой. Кроме того, одна сторона может подписать ранее инкапсулированные данные. произвольные атрибуты, например время подписи, могут быть подписаны вместе с содержимым сообщения. атрибуты и, такие как [*подписи от других сторон*](../secgloss/c-gly.md), могут быть связаны с сигнатурой. CMS может поддерживать различные архитектуры управления ключами на основе сертификатов.

Для поддержки дополнительных возможностей CMS базовые структуры данных получают новые поля. Также были добавлены новые флаги и значения параметров. Все структуры данных и все интерфейсы API, использующие CMS, имеют 100 процентов обратной совместимости с PKCS \# 7 версии 1,5.

К дополнениям относятся добавленные в [оболочку](enveloped-data-additions.md) дополнения и [добавления подписанных данных](signed-data-additions.md).
