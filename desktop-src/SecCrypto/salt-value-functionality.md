# <a name="salt-value-functionality"></a>Функция значения соли

Базовый поставщик создает 40-битные [*симметричные ключи*](../secgloss/s-gly.md) с одиннадцатью байтами [*расширяющего*](../secgloss/s-gly.md)значения, одиннадцати байтов ненулевой соли, если \_ \_ задано значение шифрования Create соли, или не имеет расширяющего значения. Однако 40-разрядный симметричный ключ с Salt-значением не эквивалентен 40-разрядному симметричному ключу без Salt. Для взаимодействия ключи должны создаваться без Salt. Эта проблема возникает из-за условия по умолчанию, которое выполняется только с ключами ровно 40 бит. Все остальные [*длины ключей*](../secgloss/k-gly.md) не имеют расширяющего выделения по умолчанию.

Как базовые поставщики, так и расширенный поставщик могут использовать флаг шифрования \_ без \_ помех, чтобы указать, что для 40-разрядного симметричного ключа не выделяется расширяющее значение. К функциям, принимающим этот флаг, относятся [**криптженкэй**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptgenkey), [**криптдеривекэй**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptderivekey)и [**криптимпорткэй**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptimportkey). По умолчанию эти функции обеспечивают обратную совместимость для случая с 40-битным симметричным ключом, продолжая использовать расширяющее значение длиной в 11 байтов.
