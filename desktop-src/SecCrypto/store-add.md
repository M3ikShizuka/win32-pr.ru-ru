# <a name="storeadd-method"></a>Метод Store. Add

\[Метод **Add** доступен для использования в операционных системах, указанных в разделе требования. Вместо этого используйте [**класс X509Store**](/previous-versions/windows/embedded/hh424027(v=msdn.10)) в пространстве имен [**System. Security. Cryptography. X509Certificates**](/dotnet/api/system.security.cryptography.x509certificates.publickey.-ctor?view=netcore-3.1) .\]

Метод **Add** добавляет объект [**сертификата**](certificate.md) в открытое [*хранилище сертификатов*](../secgloss/c-gly.md). Этот метод можно использовать только с хранилищем, которое было открыто с разрешением на чтение и запись.

## <a name="syntax"></a>Синтаксис


```VB
Store.Add( _
  ByVal Certificate _
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Сертификат* \[ окне\]
</dt> <dd>

Выражение, которое разрешается в объект [**сертификата**](certificate.md) для добавления в хранилище.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Remarks

> [!IMPORTANT]
> При вызове этого метода в системном хранилище из веб-скрипта сценарий должен получить доступ к цифровым сертификатам на локальном компьютере. Если разрешить сценарию доступ к цифровым сертификатам, веб-сайт, с которого выполняется сценарий, также будет получать доступ к личным сведениям, хранящимся в сертификатах. При первом вызове этого метода из определенного домена создается диалоговое окно, в котором пользователь должен указать, следует ли разрешить доступ к сертификатам.

 

Если хранилище не открыто в режиме чтения и записи, этот метод завершается ошибкой. Несмотря на то, что этот метод можно использовать с хранилищами памяти, любые изменения, внесенные в хранилище памяти, не сохраняются при закрытии хранилища.

Если сертификат, добавляемый в хранилище, совпадает с уже существующим, метод **Add** удаляет существующий сертификат из хранилища, а затем добавляет новый сертификат. Новый сертификат наследует свойства от существующего сертификата.
