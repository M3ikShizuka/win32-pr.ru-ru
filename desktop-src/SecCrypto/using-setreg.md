# <a name="using-setreg"></a>Использование SetReg

Инструмент SetReg задает значение разделов реестра, определяющих поведение процесса проверки сертификата Authenticode. Эти ключи, называемые ключами состояния публикации программного обеспечения, определяют, следует ли доверять тестовому корню, разрешать автономное утверждение сертификатов, проверять сроки действия сертификатов и выполнять проверки отзыва.

Следующая команда перечисляет синтаксис и параметры для использования SetReg.

**Setreg-?**

Следующая команда делает тест корневым доверенным. По умолчанию тестовый корень не является доверенным. После внесения любых изменений в значения ключей отображается список текущего значения всех ключевых значений. Если используется параметр **-q** , значения ключа не отображаются.

**Setreg 1 TRUE**

Следующая команда делает тестовый корень ненадежным и вызывает проверку на наличие отзыва. Параметр **-q** используется, чтобы список значений ключей не отображался.

**Setreg-q 1 FALSE 3 TRUE**

> [!Note]  
> Команды, параметры и аргументы не учитывают регистр.
