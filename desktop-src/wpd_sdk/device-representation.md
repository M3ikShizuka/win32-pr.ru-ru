# <a name="device-representation"></a>Представление устройства

устройства имеют два основных поведения, которые решаются с помощью архитектуры Windowsных портативных устройств (WPD):

-   Доступ к содержимому и его хранение. Например, приложения должны иметь возможность добавлять музыкальные файлы в портативный музыкальный проигрыватель.
-   Программирование устройства. Сюда входят такие простые операции, как изменение параметров и подготовка устройства к сбору данных, или более сложные операции, такие как Загрузка встроенного по. В качестве примеров можно привести команду "взять фотографию" для цифровой камеры.

В WPD эти поведения описаны путем представления устройства в виде иерархии объектов. На следующем рисунке показано представление объекта WPD для устройства с несколькими функциями, которое в данном случае является мобильным телефоном.

![Иллюстрация, показывающая иерархию объектов для мобильного телефона](images/wpd-overview-figure3.gif)

Эта иерархия иллюстрирует следующие функциональные возможности и содержимое.

Возможности

-   объект служба хранилища. Это устройство имеет хранилище данных.
-   Служба контактов. Эта служба является функциональным объектом, который можно использовать для синхронизации и хранения контактов на телефоне.
-   Служба SMS. Эта служба является функциональным объектом, который можно использовать для отправки, получения и хранения SMS-сообщений.

Содержимое:

-   Объекты мультимедиа. на этом устройстве хранятся изображения, музыкальные файлы и видеофайлы в папках объекта служба хранилища. Хотя файлы, показанные на рисунке, хранятся в одной папке, устройство может разделить содержимое на папки, упорядоченные по типу хранимых носителей. Например, папки изображений, музыкальные папки и видеопапки.
-   Объекты Contact. Это устройство хранит контактные данные (например, имя, номер телефона и адрес) в качестве дочерних элементов службы "Контакты".
-   Объекты сообщений. Это устройство хранит SMS сообщения в виде дочерних элементов службы SMS.
