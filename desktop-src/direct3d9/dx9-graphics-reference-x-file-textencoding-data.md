# <a name="data-x-file-format-text-encoding"></a>Данные (формат файла X, кодировка текста)

Объекты данных содержат фактические данные или ссылку на эти данные. Каждый объект данных имеет соответствующий шаблон, указывающий тип данных. В следующих разделах описывается форма и части объектов данных.

## <a name="form-identifier-and-name"></a>Форма, идентификатор и имя

Объекты данных имеют следующую форму.


```
        <Identifier> [name] { [<UUID>]
    <member 1>;
...
    <member n>;
}
```



Идентификатор является принудительным и должен соответствовать ранее определенному типу данных или примитиву. Однако имя является необязательным.

## <a name="data-members"></a>Элементы данных

Элементы данных могут иметь одно из следующих значений: объект данных, ссылка на данные, список целых чисел, список с плавающей запятой или список строк.

Объект данных — это вложенный объект данных. Это позволяет выражать иерархическую природу формата файла. Типы вложенных объектов данных, разрешенных в иерархии, могут быть ограничены.

Ссылка на данные — это ссылка на ранее обнаруженный объект данных, как показано в следующем примере.


```
{
  name |
  UUID |
  name UUID
}
```



Целочисленный список представляет собой разделенный точками с запятой список целых чисел, как показано в следующем примере.


```
1; 2; 3;
```



Список float — это разделенный точками с запятой список элементов float, как показано в следующем примере.


```
1.0; 2.0; 3.0;
```



Список строк представляет собой разделенный точками с запятой список строк, как показано в следующем примере.


```
"Moose"; "Goats"; "Sheep";
```
