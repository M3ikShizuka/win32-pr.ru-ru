# <a name="idirectxfilesaveobject-interface"></a>Интерфейс Идиректксфилесавеобжект

Приложения используют методы интерфейса Идиректксфилесавеобжект для создания объектов данных, а также для сохранения шаблонов и объектов данных. Обратите внимание, что шаблоны не являются обязательными для каждого файла. Например, можно разместить все шаблоны в одном файле Microsoft DirectX, а не дублировать их в каждый файл DirectX. Не рекомендуется.

## <a name="members"></a>Элементы

Интерфейс **идиректксфилесавеобжект** наследует от интерфейса [**IUnknown**](/windows/win32/api/unknwn/nn-unknwn-iunknown) . **Идиректксфилесавеобжект** также имеет следующие типы членов:

-   [Методы](#methods)

### <a name="methods"></a>Методы

Интерфейс **идиректксфилесавеобжект** содержит следующие методы.



| Метод                                                               | Описание                                                                    |
|:---------------------------------------------------------------------|:-------------------------------------------------------------------------------|
| [**креатедатаобжект**](idirectxfilesaveobject--createdataobject.md) | Создает объект данных. Не рекомендуется.<br/>                                  |
| [**SaveData**](idirectxfilesaveobject--savedata.md)                 | Сохраняет объект данных и его дочерние элементы в файле DirectX. Не рекомендуется.<br/> |
| [**саветемплатес**](idirectxfilesaveobject--savetemplates.md)       | Сохраняет шаблоны в файл DirectX. Не рекомендуется.<br/>                      |



 

## <a name="remarks"></a>Комментарии

Глобальный уникальный идентификатор (GUID) для интерфейса Идиректксфилесавеобжект — **IID \_ идиректксфилесавеобжект**.

Интерфейс **идиректксфилесавеобжект** получается путем вызова метода [**Идиректксфиле:: креатесавеобжект**](idirectxfile--createsaveobject.md) .

Тип **лпдиректксфилесавеобжект** определяется как указатель на этот интерфейс.


```
typedef interface IDirectXFileSaveObject *LPDIRECTXFILESAVEOBJECT;
```
