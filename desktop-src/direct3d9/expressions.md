# <a name="expressions-direct3d-9"></a>Выражения (Direct3D 9)

Выражения — это математические или логические операторы, которые используются в правой части знака равенства. Существует много типов выражений.

## <a name="expressions"></a>Выражения

1.  Ссылка на переменную
    ```
    ( variable ) or<variable >
    ```

    

2.  Числовые скалярные
    ```
    scalar 
    ```

    

3.  Числовое выражение

    ```
    ( numeric expression )
    ```

    

    Здесь поддерживаются все стандартные числовые выражения ХЛЛ.

4.  Конструктор
    ```
    type ( constructor arguments )
    ```

    

5.  Список инициализаторов

    ```
    { scalar value [, scalar value ...  ] }
    
    ```

    

    Скаляры должны быть литеральными скалярными значениями.

    Число инициализаторов должно быть совместимо с переменной (State) в левой части знака равенства.

6.  Выражение OR

    ```
    token [ | token ... ]
    ```

    

    Токены должны быть совместимы с переменной (штат) в левой части знака равенства.

    В токенах регистр не учитывается.

7.  NULL

    ```
    NULL
    ```

    

    Значение NULL может быть назначено шейдеру, образцу или объекту текстуры.

8.  Блок сборки

    ```
    asm { code }
    ```

    

    Блоки сборки PS должны быть назначены состоянию PIXELSHADER.

    Блоки сборок VS должны быть назначены состоянию ВЕРТЕКСШАДЕР.

9.  Блок состояния образца

    ```
    sampler_state { [ state = expression ; [ state = ... ] ] }
    ```

    

    Блоки состояний образцов — это последовательности неиндексированного состояния этапа образца или назначения текстур.

    Блоки состояния образцов должны быть назначены состоянию эффектов образца.

10. Блок состояния состояния эффектов

    ```
    stateblock_state { [ state [ [index] ] = expression; 
        [ state [ [index] ] = ... ] ] }
    ```

    

    Блоки состояния — это последовательности общего состояния. Блоки состояния могут быть вложенными, но не могут содержать циклических ссылок.

    Блоки состояния должны быть назначены состоянию влияния СТАТЕБЛОКК.

11. HLSL компилировать

    ```
    compile target entrypoint ( [ arguments ] )
    ```

    

    Целевой шейдер вершин и \_ m \_ n указывает версию D3DVSа \_ вершинного шейдера версии (m, n). Цель PS m n пиксельного шейдера \_ \_ указывает \_ версию шейдера D3DPS (m, n) пикселей.

    Выражения верхнего уровня для языковой компиляции вершинных шейдеров могут быть назначены только состоянию влияния ВЕРТЕКСШАДЕР. Выражения высокого уровня языковой компиляции для шейдера пикселей могут быть назначены только состоянию PIXELSHADER Effect.
