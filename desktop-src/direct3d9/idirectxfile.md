# <a name="idirectxfile-interface"></a>Интерфейс Идиректксфиле

Приложения используют методы интерфейса Идиректксфиле для создания экземпляров интерфейсов [**идиректксфилинумобжект**](idirectxfileenumobject.md) и [**идиректксфилесавеобжект**](idirectxfilesaveobject.md) , а также для регистрации шаблонов. Не рекомендуется.

## <a name="members"></a>Элементы

Интерфейс **идиректксфиле** наследует от интерфейса [**IUnknown**](/windows/win32/api/unknwn/nn-unknwn-iunknown) . **Идиректксфиле** также имеет следующие типы членов:

-   [Методы](#methods)

### <a name="methods"></a>Методы

Интерфейс **идиректксфиле** содержит следующие методы.



| Метод                                                       | Описание                                          |
|:-------------------------------------------------------------|:-----------------------------------------------------|
| [**креатинумобжект**](idirectxfile--createenumobject.md)   | Создает объект перечислителя. Не рекомендуется.<br/> |
| [**креатесавеобжект**](idirectxfile--createsaveobject.md)   | Создает объект Save. Не рекомендуется.<br/>        |
| [**регистертемплатес**](idirectxfile--registertemplates.md) | Регистрирует пользовательские шаблоны. Не рекомендуется.<br/>   |



 

## <a name="remarks"></a>Remarks

Глобальный уникальный идентификатор (GUID) для интерфейса Идиректксфиле — IID \_ идиректксфиле.

Интерфейс Идиректксфиле получается путем вызова функции [**директксфилекреате**](directxfilecreate.md) .

Тип ЛПДИРЕКТКСФИЛЕ определяется как указатель на этот интерфейс.


```
typedef interface IDirectXFile *LPDIRECTXFILE;
```
