# <a name="d3dlock"></a>D3DLOCK

Сочетание одного или нескольких параметров блокировки, описывающих тип выполняемой блокировки.



|                            |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|----------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| \#определенно                   | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| D3DLOCK \_ отбросить           | Приложение отклоняет всю память в заблокированном регионе. Для буферов вершин и индексов весь буфер будет удален. Этот параметр допустим только при создании ресурса с динамическим использованием (см. [D3DUSAGE](d3dusage.md)).                                                                                                                                                                                                                                                                                                                                                           |
| D3DLOCK \_ донотваит         | Позволяет приложению получать циклы ЦП, если драйвер не может немедленно заблокировать рабочую область. Если этот флаг установлен и драйвер не может немедленно заблокировать поверхность, вызов блокировки возвратит D3DERR \_ васстиллдравинг. Этот флаг можно использовать только при блокировке поверхности, созданной с помощью [**креатеоффскринплаинсурфаце**](/windows/desktop/api), [**креатерендертаржет**](/windows/desktop/api)или [**креатедепсстенЦилсурфаце**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-createdepthstencilsurface). Этот флаг также можно использовать с задним буфером.            |
| D3DLOCK \_ без \_ "грязного" \_ обновления | По умолчанию блокировка ресурса добавляет к этому ресурсу «грязную» область. Этот параметр предотвращает любые изменения в «грязном» состоянии ресурса. Приложения должны использовать этот параметр, если у них есть дополнительные сведения о наборе регионов, измененных во время операции блокировки.                                                                                                                                                                                                                                                                                                                    |
| D3DLOCK \_ нуверврите       | Указывает, что память, на которую ссылается вызов рисования с момента последней блокировки без этого флага, не будет изменена во время блокировки. Это может включить оптимизацию, когда приложение добавляет данные к ресурсу. Если указать этот флаг, драйвер будет немедленно возвращаться, если ресурс используется. в противном случае драйвер должен завершить использовать ресурс перед возвратом из блокировки.                                                                                                                                                                                            |
| D3DLOCK \_ носислокк         | Поведением по умолчанию блокировки видеопамяти является резервирование критической секции на уровне системы, гарантирующая, что в течение блокировки не произойдет никаких изменений режима просмотра. Этот параметр приводит к тому, что критически важный для всей системы раздел не будет удерживаться на время блокировки.<br/> Операция блокировки занимает много времени, но может позволить системе выполнять другие обязанности, например перемещать курсор мыши. Этот параметр полезен для долгосрочных блокировок, таких как блокировка заднего буфера для отрисовки программного обеспечения, которая в противном случае негативно повлияет на скорость реагирования системы.<br/> |
| D3DLOCK \_ ReadOnly          | Приложение не будет записывать в буфер. Это позволяет ресурсам, хранящимся в несобственных форматах, сохранять шаг повторного сжатия при разблокировании.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
