# <a name="getthreadselectedcpusets-function"></a>Функция Жетсреадселектедкпусетс

Возвращает явное назначение набора ЦП указанного потока, если какое-либо назначение было задано с помощью API [**сетсреадселектедкпусетс**](setthreadselectedcpusets.md) . Если явное присваивание не задано, **рекуиредидкаунт** имеет значение 0, а функция возвращает значение true.

## <a name="syntax"></a>Синтаксис


```C++
BOOL WINAPI GetThreadSelectedCpuSets(
  _In_      HANDLE Thread,
  _Out_opt_ PULONG CpuSetIds,
  _In_      ULONG  CpuSetIdCount,
  _Out_     PULONG RequiredIdCount
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Поток команд* \[ окне\]
</dt> <dd>

Указывает поток, для которого запрашиваются выбранные наборы ЦП. Этот обработчик должен иметь \_ \_ право доступа к ограниченным данным в запросе потока \_ . Значение, возвращаемое функцией [**жеткуррентсреад**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-getcurrentthread) , также можно указать здесь.

</dd> <dt>

*Кпусетидс* \[ out, необязательно\]
</dt> <dd>

Задает необязательный буфер для получения списка идентификаторов набора ЦП.

</dd> <dt>

*Кпусетидкаунт* \[ окне\]
</dt> <dd>

Указывает Емкость буфера, указанного в **кпусетидс**. Если буфер равен NULL, это значение должно быть равно 0.

</dd> <dt>

*Рекуиредидкаунт* \[ заполняет\]
</dt> <dd>

Указывает требуемую Емкость буфера для хранения всего списка наборов ЦП, выбранных в потоке. При успешном возвращении этот параметр указывает количество идентификаторов, заполняемых в буфер.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот API возвращает значение TRUE в случае успешного выполнения. Если буфер недостаточно велик, значение **GetLastError** не является ошибкой \_ \_ buffer. Этот API не может завершиться ошибкой, если переданы допустимые параметры, и буфер возврата достаточно большой.
