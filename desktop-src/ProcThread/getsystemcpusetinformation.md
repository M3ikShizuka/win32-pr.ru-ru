# <a name="getsystemcpusetinformation-function"></a>Функция Жетсистемкпусетинформатион

Позволяет приложению запрашивать доступные наборы ЦП в системе и их текущее состояние.

## <a name="syntax"></a>Синтаксис


```C++
BOOL WINAPI GetSystemCpuSetInformation(
  _Out_opt_  PSYSTEM_CPU_SET_INFORMATION  Information,
  _In_       ULONG                        BufferLength,
  _Out_      PULONG                       ReturnedLength,
  _In_opt_   HANDLE                       Process,
  _Reserved_ ULONG                        Flags
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Сведения о* \[ out, необязательно\]
</dt> <dd>

Указатель на [**\_ \_ \_ информационную структуру набора ЦП системы**](/windows/desktop/api/winnt/ns-winnt-system_cpu_set_information) , которая получает данные о наборе ЦП. Передайте значение NULL с длиной буфера 0, чтобы определить требуемый размер буфера.

</dd> <dt>

*BufferLength* \[ окне\]
</dt> <dd>

Длина выходного буфера (в байтах), переданного в качестве информационного аргумента.

</dd> <dt>

*Ретурнедленгс* \[ заполняет\]
</dt> <dd>

Длина в байтах допустимых данных в выходном буфере, если буфер достаточно большой, или требуемый размер выходного буфера. Если наборы ЦП не существуют, это значение будет равно 0.

</dd> <dt>

*Обработка* \[ в необязательное\]
</dt> <dd>

Необязательный обработчик для процесса. Этот процесс используется для определения значения флага **аллокатедтотаржетпроцесс** в \_ \_ структуре сведений о наборе ЦП системы \_ . Если набор ЦП выделен для указанного процесса, устанавливается флаг. В противном случае он очищается. Этот обработчик должен иметь право на \_ \_ доступ к ограниченным данным Process \_ . Значение, возвращаемое функцией [**жеткуррентпроцесс**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-getcurrentprocess) , также может быть указано здесь.

</dd> <dt>

*Флаги* 
</dt> <dd>

Зарезервировано, должно быть равно 0.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если API завершается, возвращается значение TRUE. В случае неудачи причина ошибки доступна через **GetLastError**. Если информационный буфер имел неопределенное значение или недостаточно большой, возвращается ошибка с кодом \_ недостаточный \_ Размер буфера. Этот API не может быть выполнен, если переданы допустимые параметры и буфер, достаточно большой для хранения всех возвращаемых данных.
