# <a name="psm_recalcpagesizes-message"></a>\_Сообщение ПСМ рекалкпажесизес

Повторно вычисляет размер страницы на странице свойств стандартного или мастера после добавления или удаления страниц. Это сообщение можно отправить явным образом или использовать макрос [**пропшит \_ рекалкпажесизес**](/windows/desktop/api/Prsht/nf-prsht-propsheet_recalcpagesizes) .

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Должен равняться нулю.

</dd> <dt>

*lParam* 
</dt> <dd>

Должен равняться нулю.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает **значение true** , если успешно, или **false** в противном случае.

## <a name="remarks"></a>Комментарии

При создании страницы свойств ее размер изменяется по размеру исходной коллекции страниц. Чтобы обеспечить совместимость с предыдущими версиями стандартных элементов управления, страницы свойств и мастера не изменяют размер автоматически при последующем добавлении или удалении страниц. При использовании стандартных элементов управления [версии 5,80](common-control-versions.md)приложения должны отправить **сообщение \_ ПСМ рекалкпажесизес** после добавления или удаления страниц с помощью [**ПСМ \_ ADDPAGE**](psm-addpage.md), [**ПСМ \_ инсертпаже**](psm-insertpage.md), [**ПСМ \_ REMOVEPAGE**](psm-removepage.md)или их эквивалентных макросов. Это гарантирует, что лист свойств будет иметь правильный размер для текущей коллекции страниц. Если это сообщение не отправлено, некоторые страницы листа свойств могут быть усечены или слишком велики.

> [!Note]  
> Это сообщение не поддерживается при использовании стиля мастера Aero ([**командном PSH \_ аеровизард**](/windows/desktop/api/Prsht/ns-prsht-propsheetheadera_v2)).
