# <a name="cb_selectstring-message"></a>\_Сообщение СЕЛЕКТСТРИНГ CB

Выполняет поиск элемента, который начинается с символов в указанной строке, в списке поля со списком. Если соответствующий элемент найден, он выбирается и копируется в элемент управления "поле ввода".

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Отсчитываемый от нуля индекс элемента, предшествующего первому элементу для поиска. Когда поиск достигает нижнего края списка, он переходит от верхней части списка к элементу, указанному параметром *wParam* . Если параметр *wParam* имеет значение-1, поиск выполняется по всему списку с самого начала.

</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая содержит символы для поиска. При поиске регистр не учитывается, поэтому эта строка может содержать любое сочетание прописных и строчных букв.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если строка найдена, возвращаемое значение является индексом выбранного элемента. Если поиск завершается неудачно, возвращается значение CB \_ Err, а текущее выделение не изменяется.

## <a name="remarks"></a>Комментарии

Строка выбирается только в том случае, если символы из начальной точки совпадают с символами в строке префикса.

Если создать поле со списком, используя стиль, рисуемый владельцем, но без [**стиля \_ хасстрингс CBS**](combo-box-styles.md) , то то, что делает сообщение **\_ селектстринг CB** , зависит от того, используется ли стиль [**\_ сортировки CBS**](combo-box-styles.md) . Если используется **стиль \_ сортировки CBS** , система отправляет сообщения [**WM \_ компареитем**](wm-compareitem.md) владельцу поля со списком, чтобы определить, какой элемент соответствует заданной строке. Если вы не используете стиль **\_ сортировки CBS** , **CB \_ Селектстринг** пытается сопоставить значение **DWORD** со значением параметра *lParam* .
