# <a name="lvm_getisearchstring-message"></a>\_Сообщение LVM жетисеарчстринг

Извлекает строку добавочного поиска элемента управления "представление списка". Это сообщение можно отправить явно или с помощью макроса [**\_ жетисеарчстринг ListView**](/windows/desktop/api/Commctrl/nf-commctrl-listview_getisearchstring) .

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>Должен равняться нулю.</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на буфер, который получает строку добавочного поиска. Чтобы просто получить длину строки, присвойте параметру *lParam* **значение NULL**.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает количество символов в строке добавочного поиска, не включая завершающий нуль-символ, или нуль, если элемент управления "представление списка" не находится в режиме добавочного поиска.

## <a name="remarks"></a>Remarks

**Предупреждение системы безопасности:** Неправильное использование этого сообщения может нарушить безопасность программы. Это сообщение не позволяет получить сведения о размере буфера. Если вы используете это сообщение, сначала вызовите сообщение, передав ему **значение NULL** в параметре *lParam*, после чего будет возвращено необходимое число символов, исключая **null** . Затем вызовите сообщение еще раз, чтобы получить строку. прежде чем продолжить, ознакомьтесь с [вопросами безопасности: Microsoft Windows](sec-comctls.md) .

*Строка добавочного поиска* — это последовательность символов, которую пользователь вводит, когда представление списка имеет фокус ввода. Каждый раз, когда пользователь вводит символ, система добавляет символ в строку поиска, а затем выполняет поиск соответствующего элемента. Если система находит совпадение, она выбирает элемент и, при необходимости, прокручивает его в представлении.

С каждым символом, определяемым пользователем, связан период времени ожидания. Если время ожидания истекает до того, как пользователь вводит другой символ, строка добавочного поиска сбрасывается.

Убедитесь, что буфер достаточно велик, чтобы вместить строку и завершающий нуль-символ. Если он слишком мал, возникнет немедленная ошибка страницы.
