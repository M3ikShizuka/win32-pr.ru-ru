# <a name="lvm_mapidtoindex-message"></a>\_Сообщение LVM мапидтоиндекс

Карты идентификатор элемента в индекс.

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Уникальный идентификатор элемента.

</dd> <dt>

*lParam* 
</dt> <dd>

Должен равняться нулю.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает самый текущий индекс.

## <a name="remarks"></a>Remarks

Элементы управления "представление списка" внутренне отслеживание элементов по индексу. Это может представлять проблемы, так как индексы могут изменяться во время существования элемента управления.

Элемент управления "представление списка" может помечать элемент с ИДЕНТИФИКАТОРом при создании элемента. Этот идентификатор можно использовать для обеспечения уникальности во время существования элемента управления "представление списка".

Чтобы однозначно идентифицировать элемент, возьмите индекс, возвращенный из вызова, например [**IComponent:: жетдисплайинфо**](/windows/desktop/api/mmc/nf-mmc-icomponent-getdisplayinfo) , и вызовите [**LVM \_ мапиндекстоид**](lvm-mapindextoid.md). Возвращаемое значение является уникальным ИДЕНТИФИКАТОРом.

Если требуется индекс элемента после создания идентификатора, можно вызвать **LVM \_ мапидтоиндекс** с уникальным идентификатором и возвратить самый актуальный индекс.

**LVM \_ МАПИДТОИНДЕКС** не поддерживается в стиле [**LVS \_ овнердата**](list-view-window-styles.md) .

> [!Note]  
> В многопоточной среде индекс гарантированно гарантируется только в потоке, где размещается элемент управления "представление списка", а не в фоновых потоках.

 

> [!Note]  
> Чтобы использовать это сообщение, необходимо указать манифест, указывающий Comclt32.dll версии 6,0. Дополнительные сведения о манифестах см. в разделе [Включение визуальных стилей](cookbook-overview.md).
