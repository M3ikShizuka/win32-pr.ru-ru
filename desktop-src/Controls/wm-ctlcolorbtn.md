# <a name="wm_ctlcolorbtn-message"></a>\_Сообщение КТЛКОЛОРБТН WM

Сообщение **WM \_ ктлколорбтн** отправляется в родительское окно кнопки перед рисованием кнопки. Родительское окно может изменять цвета текста и фона кнопки. Однако только рисуемые владельцем кнопки реагируют на родительское окно, обрабатывающее это сообщение.


```C++
WM_CTLCOLORBTN

    WPARAM wParam;
    LPARAM lParam; 
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Контекст **HDC** , задающий маркер для контекста вывода кнопки.

</dd> <dt>

*lParam* 
</dt> <dd>

**HWND** , указывающий дескриптор кнопки.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если приложение обрабатывает это сообщение, оно должно возвращать маркер кисти. Система использует кисть для рисования фона кнопки.

## <a name="remarks"></a>Remarks

Если приложение возвращает созданную кисть (например, с помощью функции [**креатесолидбруш**](/windows/desktop/api/wingdi/nf-wingdi-createsolidbrush) или [**креатебрушиндирект**](/windows/desktop/api/wingdi/nf-wingdi-createbrushindirect) ), приложение должно освободить кисть. Если приложение возвращает системную кисть (например, которая была получена функцией [**жетстоккобжект**](/windows/desktop/api/wingdi/nf-wingdi-getstockobject) или [**жетсисколорбруш**](/windows/desktop/api/winuser/nf-winuser-getsyscolorbrush) ), приложению не нужно освобождать эту кисть.

По умолчанию функция [**дефвиндовпрок**](/windows/desktop/api/winuser/nf-winuser-defwindowproca) выбирает системные цвета по умолчанию для кнопки. Кнопки с [**\_ кнопками BS**](button-styles.md), [**\_ дефпушбуттон**](button-styles.md)или [**BS \_ пушлике**](button-styles.md) не используют возвращенную кисть. Кнопки с этими стилями всегда отображаются с системными цветами по умолчанию. Для нажатия кнопок рисования требуется несколько различных кистей, выделения и тени, но сообщение **WM \_ ктлколорбтн** позволяет вернуть только одну кисть. Чтобы предоставить пользовательский внешний вид для кнопок, используйте рисуемые владельцем кнопки. Дополнительные сведения см. в разделе [Создание элементов управления Owner-Drawn](user-controls-intro.md).

Сообщение **WM \_ ктлколорбтн** никогда не передается между потоками. Он отправляется только в пределах одного потока.

Цвет текста флажка или переключателя применяется к полю или кнопке, его галочке и тексту. Прямоугольник фокуса для этих кнопок остается цветом системы по умолчанию (обычно черным). Цвет текста поля группы применяется к тексту, но не к линии, определяющей поле. Цвет текста кнопки «кнопка» применяется только к прямоугольнику фокуса; Он не влияет на цвет текста.

Если процедура диалогового окна обрабатывает это сообщение, необходимо привести требуемое возвращаемое значение к типу **int \_ ptr** и вернуть значение напрямую. Если процедура диалогового окна возвращает **значение false**, выполняется обработка сообщений по умолчанию. \_Значение МСГРЕСУЛТ DWL, заданное функцией [**SetWindowLong**](/windows/desktop/api/winuser/nf-winuser-setwindowlonga) , игнорируется.
