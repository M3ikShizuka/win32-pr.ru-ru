# <a name="lb_settabstops-message"></a>Сообщение сеттабстопс балансировки нагрузки \_

Задает позиции табуляции в поле со списком.

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Указывает количество единиц табуляции.

</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на первый элемент массива целых чисел, содержащий позиции табуляции. Целые числа представляют число кварталов средней ширины символов для шрифта, выбранного в списке. Например, позиция табуляции 4 помещается в 1,0 единиц знака, а позиция табуляции, равной 6, помещается по 1,5 среднему числу единиц символов. Однако если список является частью диалогового окна, то целые числа находятся в единицах шаблона диалогового окна. Позиции табуляции должны быть отсортированы в возрастающем порядке. обратные вкладки не допускаются.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если заданы все указанные вкладки, возвращаемое значение равно **true**; в противном случае — **false**.

## <a name="remarks"></a>Remarks

Чтобы ответить на сообщение **\_ Сеттабстопса балансировки нагрузки** , список должен быть создан с помощью стиля [**фунта \_ усетабстопс**](list-box-styles.md) .

Если параметр *wParam* равен 0, а *lParam* имеет **значение NULL**, то позиция табуляции по умолчанию равна двум единицам диалогового шаблона. Если параметр *wParam* равен 1, то в списке будут находиться позиции табуляции, разделенные расстоянием, заданным параметром *lParam*.

Если *lParam* указывает на больше одного значения, для каждого значения в *lParam* будет задана позиция табуляции до числа, указанного параметром *wParam*.

Значения, заданные параметром *lParam* , находятся в шаблонах диалоговых окон, которые являются аппаратно-независимыми устройствами, используемыми в шаблонах диалоговых окон. Чтобы преобразовать измерения из шаблона диалогового окна в единицы экрана (в пикселях), используйте функцию [**мапдиалогрект**](/windows/desktop/api/winuser/nf-winuser-mapdialogrect) .

Windows 95/Windows 98/Windows Millennium Edition (Windows Me). буфер, на который указывает *lParam* , должен находиться в памяти, доступной для записи, даже если сообщение не изменяет массив.
