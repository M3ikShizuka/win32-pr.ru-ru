# <a name="stm_setimage-message"></a>\_Сообщение СЕТИМАЖЕ STM

Приложение отправляет сообщение **STM \_ сетимаже** , чтобы связать новый образ со статическим элементом управления.

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Указывает тип изображения, связываемого с статическим элементом управления. Этот параметр может принимать одно из следующих значений:



| Значение                                                                                                                                                                     | Значение                       |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------|
| <span id="IMAGE_BITMAP"></span><span id="image_bitmap"></span><dl> <dt>**\_точечный рисунок изображения**</dt> </dl>                | Массив.<br/>            |
| <span id="IMAGE_CURSOR"></span><span id="image_cursor"></span><dl> <dt>**\_курсор изображения**</dt> </dl>                | Набора.<br/>            |
| <span id="IMAGE_ENHMETAFILE"></span><span id="image_enhmetafile"></span><dl> <dt>**\_ЕНХМЕТАФИЛЕ изображения**</dt> </dl> | Расширенный метафайл.<br/> |
| <span id="IMAGE_ICON"></span><span id="image_icon"></span><dl> <dt>**\_значок изображения**</dt> </dl>                      | Значок.<br/>              |



 

</dd> <dt>

*lParam* 
</dt> <dd>

Обрабатывайте изображение, чтобы связать его со статическим элементом управления.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращаемое значение является обработкой изображения, ранее связанного со статическим элементом управления, если таковой имеется; в противном случае он имеет **значение NULL**.

## <a name="remarks"></a>Remarks

Чтобы связать изображение со статическим элементом управления, элемент управления должен иметь соответствующий стиль. В следующей таблице показан стиль, необходимый для каждого типа изображений.



| Тип образа         | Статический стиль элемента управления |
|--------------------|----------------------|
| \_точечный рисунок изображения      | \_Битовая карта SS           |
| \_курсор изображения      | \_значок SS             |
| \_ЕНХМЕТАФИЛЕ изображения | СС \_ енхметафиле      |
| \_значок изображения        | \_значок SS             |



 

> [!IMPORTANT]
>
> В версии 6 элементов управления Microsoft Win32 точечный рисунок, переданный в статический элемент управления с **помощью \_ сетимаже** -сообщения STM, был таким же, как и последующее сообщение **STM \_ сетимаже** . Клиент несет ответственность за удаление любого растрового изображения, отправленного в статический элемент управления.
>
> в Windows XP, если точечный рисунок, передаваемый в сообщении **STM \_ сетимаже** , содержит пиксели с ненулевой альфа-составляющей, статический элемент управления принимает копию точечного рисунка. Этот скопированный точечный рисунок возвращается следующим сообщением **STM \_ сетимаже** . Клиентский код может независимо отслеживаниь точечных рисунков, переданных в статический элемент управления, но если он не проверяет и не освобождает растровые изображения, возвращенные из **\_ Сетимаже сообщений STM** , точечные рисунки будут утеряны.
