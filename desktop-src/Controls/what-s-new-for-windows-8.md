# <a name="whats-new-windows-controls"></a>новые возможности (элементы управления Windows)

в этом разделе описываются различия в поддержке этих стилей и визуального стиля между Windows 8 и предыдущими версиями Windows.

## <a name="through-windows-7"></a>до Windows 7

в Windows 7 визуальные стили включены по умолчанию, но пользователь может отключить их, выбрав пункт Windows классическая тема или отключив службу "темы". Когда визуальные стили отключены, весь пользовательский интерфейс получает классический вид, и большинство интерфейсов API визуальных стилей недоступны. визуальные стили отключены в режиме Windows 7 для поддержки различных тем высокой контрастности, а также Windows классической темы. Если вы хотите поддерживать как визуальные стили, так и темы с высокой контрастностью в одном приложении, обычно необходимо поддерживать два отдельных пути кода для отрисовки элементов управления.

## <a name="windows-8-and-later"></a>Windows 8 и более поздних версий

в Windows 8 стили оформления не могут быть отключены на странице **персонализация** **Параметры пк** или при отключении службы "темы". Windows Классический режим больше не существует, и режим высокой контрастности был изменен для работы с визуальными стилями. из-за этих изменений приложения, предназначенные только для Windows 8, больше не нуждаются в двух отдельных путях кода для поддержки визуальных стилей и тем с высокой контрастностью.

визуальные стили в Windows 8 включают поддержку обратной совместимости для Windows классическом режиме. любой код отрисовки пользовательского интерфейса, работающий в предыдущих версиях, продолжит работать с Windows 8 без изменений.

если вы хотите, чтобы приложение поддерживало темы с высокой контрастностью, основанные на стилях оформления, в Windows 8, необходимо включить Windows 8 GUID в раздел compatibility манифеста приложения. в противном случае система предполагает, что приложение разработано для предыдущей версии и визуализирует клиентскую область, имитируя Windows классические темы с высокой контрастностью. Дополнительные сведения см. в разделе [Поддержка тем высокая контрастность](supporting-high-contrast-themes.md).

как и в предыдущих версиях, Windows 8 поддерживает стандартные элементы управления версии 5 и 6, а версия 5 — значение по умолчанию. Поскольку стили оформления поддерживаются только в версии 6, необходимо указать версию 6 в манифесте приложения, если необходимо применить стили оформления к общим элементам управления в клиентской области приложения. Дополнительные сведения см. в разделе [Включение визуальных стилей](cookbook-overview.md).
