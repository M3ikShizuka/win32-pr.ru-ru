# <a name="cb_getlbtext-message"></a>\_Сообщение ЖЕТЛБТЕКСТ CB

Возвращает строку из списка поля со списком.

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Отсчитываемый от нуля индекс извлекаемой строки.

</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на буфер, который получает строку. Буфер должен содержать достаточно места для строки и завершающего нуль-символа. Вы можете отправить сообщение [**\_ жетлбтекстлен CB**](cb-getlbtextlen.md) перед сообщением **\_ жетлбтекст CB** , чтобы получить длину строки в **файле TCHAR**. Если это строка ANSI, то это число байтов, но если это строка в Юникоде, то это число символов.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращаемое значение — это длина строки в **файле TCHAR** s, исключая завершающего нуль-символа. Если параметр *wParam* не указывает допустимый индекс, возвращается значение CB \_ Err.

## <a name="remarks"></a>Комментарии

**Предупреждение системы безопасности:** Неправильное использование этого сообщения может нарушить безопасность программы. Это сообщение не позволяет получить сведения о размере буфера. Если вы используете это сообщение, сначала вызовите [**CB \_ жетлбтекстлен**](cb-getlbtextlen.md) , чтобы получить необходимое количество символов, а затем вызовите сообщение, чтобы получить строку. Прежде чем продолжить, ознакомьтесь с [вопросами безопасности: элементы управления Microsoft Windows](sec-comctls.md) .

Если создать поле со списком, используя стиль, рисуемый владельцем, но без [**стиля \_ хасстрингс CBS**](combo-box-styles.md) , то буфер, на который указывает *lParam* , получает данные, связанные с этим элементом.
