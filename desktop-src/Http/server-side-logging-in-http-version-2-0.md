# <a name="server-side-logging"></a>Ведение журнала Server-Side

Ведение журнала на стороне сервера доступно в группе URL-адресов или сеансе сервера. Если ведение журнала включено в сеансе сервера, оно работает как централизованное ведение журнала для всех групп URL-адресов в сеансе сервера. Если ведение журнала включено в группе URL-адресов, ведение журнала выполняется только для запросов, направляемых в группу URL-адресов. API HTTP-сервера поддерживает следующие типы ведения журнала:

-   [Ведение журнала W3C](w3c-logging.md): доступно в сеансе сервера и группе URL-адресов.
-   [Ведение журнала NCSA](ncsa-logging.md): доступно в группе URL-адресов.
-   [Ведение журнала IIS](iis-logging.md): доступно в группе URL-адресов.
-   [Централизованное ведение журнала в двоичном формате](centralized-binary-logging.md): доступно в сеансе сервера.

Чтобы воспользоваться функцией ведения журнала HTTP, приложение включает и настраивает один тип ведения журнала в сеансе сервера или группе URL-адресов. Дополнительные сведения см. в разделе [Настройка и включение ведения журнала на стороне сервера](configuring-and-enabling-server-side-logging.md) .
