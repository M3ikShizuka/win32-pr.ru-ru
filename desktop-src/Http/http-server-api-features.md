# <a name="http-server-api-features"></a>Функции API сервера HTTP

В следующих списках описаны поддерживаемые и неподдерживаемые функции API-интерфейса сервера HTTP.

## <a name="features-supported"></a>Поддерживаемые возможности

Протокол HTTP поддерживает следующие функции:

-   Функции сервера HTTP v 1.0 и v 1.1.
-   SSL 3,0 (SSL) с поддержкой сертификатов клиента и сервера.
-   Кэширование фрагментов данных для использования в последующих ответах.
-   Поддержка адресации IPv6 и IPv4.
-   Резервирование пространства имен URL-адресов для безопасности приложения.
-   Истинные дескрипторы для всех объектов.
-   Синхронные и асинхронные модели ввода-вывода.
-   поддержка WOW64 на компьютерах, работающих под управлением 64-разрядных Windows начиная с Windows Server 2003 с пакетом обновления 1 (SP1).

## <a name="features-not-supported"></a>Функции не поддерживаются

API сервера HTTP не поддерживает следующие функциональные возможности:

-   API сервера HTTP не выполняет проверку подлинности клиента или сервера на основе содержимого заголовков HTTP-запроса. Все необходимые проверки подлинности должны быть реализованы приложением.
-   подсистема WOW64 на компьютерах, работающих на 64-разрядной Windows, не поддерживается на Windows Server 2003 и Windows XP с пакетом обновления 2 (SP2) и более ранних версий.
-   API сервера HTTP не поддерживает ведение журнала HTTP-запросов и ответов.
-   API-сервер HTTP не выполняет фрагментацию исходящих HTTP-ответов. При необходимости приложение должно реализовать фрагмент ответа.
