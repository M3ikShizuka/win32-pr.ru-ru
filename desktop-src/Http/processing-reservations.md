# <a name="processing-reservations"></a>Обработка резервирований

Резервирование для частей пространства имен URL-адресов осуществляется системным администратором и размещается в хранилище постоянного резервирования. Корневой каталог пространства имен URL-адресов для HTTP принадлежит системному администратору. Для всех значений узлов и портов в корне пространства имен **relativeUri** всегда предполагается наличие следующих двух резервирований.



| Пространство имен зарезервировано                 | Зарезервировано для              |
|------------------------------------|---------------------------|
| https:// <host> :<port>/  | Администратор LocalSystem |
| https:// <host> :<port>/ | Администратор LocalSystem |



 

Эти неявные резервирования не могут быть удалены. Однако явное резервирование root можно делегировать другим пользователям. При резервировании, как показано ниже, будут созданы такие делегирования:



| Пространство имен зарезервировано        | Зарезервировано для |
|---------------------------|--------------|
| `https://+:80/`           | Пользователь a, UserC |
| `https://adatum.com:443/` | Пользователя b        |



 

Если эти делегированные резервирования удаляются системным администратором, владение пространством имен возвращается к неявному резервированию для соответствующих значений узла и порта.
