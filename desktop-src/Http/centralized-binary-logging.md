# <a name="centralized-binary-logging"></a>Централизованное ведение журнала в двоичном формате

Централизованное ведение журнала в двоичном формате — это тип ведения журнала на стороне сервера, который может быть включен в сеансе сервера. Централизованная функция ведения журнала в двоичном формате используется в качестве централизованной формы для всех групп URL-адресов, созданных в сеансе сервера, и позволяет всем группам URL-адресов в сеансе сервера передавать двоичные, неформатированные данные журнала в один файл журнала. Этот тип ведения журнала можно включить только в сеансе сервера. ее нельзя включить в группе URL-адресов.

## <a name="when-to-use-centralized-binary-logging"></a>Когда следует использовать централизованное ведение журнала в двоичном формате

Когда сеанс сервера содержит несколько групп URL-адресов, процесс создания сотен форматированных файлов журналов для отдельных групп URL-адресов и записи данных журнала на диск может быстро потреблять ценные ресурсы ЦП и памяти, тем самым создавая проблемы с производительностью и масштабируемостью. Централизованное ведение журнала в двоичном формате уменьшает объем системных ресурсов, используемых для ведения журнала, в то же время предоставляя подробные данные журналов для организаций, которым он необходим.

Централизованное ведение журнала в двоичном формате — это свойство сеанса сервера, которое при включении настраивает ведение журнала для всех групп URL-адресов в сеансе сервера. При включении централизованного ведения журнала в двоичном формате данные не могут быть записаны или отформатированы для отдельных групп URL-адресов. Централизованный файл журнала двоичного журнала имеет расширение двоичного файла журнала Интернета (. Ибл). Это расширение имени файла гарантирует, что текстовые программы не пытаются открыть и прочитать файл журнала центрального двоичного журнала.

## <a name="extracting-data-from-the-centralized-binary-log-file"></a>Извлечение данных из централизованного двоичного файла журнала

Для извлечения данных из необработанного файла журнала выполняются следующие действия.

-   Создайте инструмент, который находит и извлекает нужные данные из необработанного файла и преобразует их в форматированный текст. Описание формата файла заголовка и файла журнала можно просмотреть в пакете средств разработки программного обеспечения IIS 6,0 на сайте MSDN.
-   Используйте средство синтаксического анализа журналов для извлечения данных из необработанного файла. Средство синтаксического анализа журналов и сопутствующая документация для пользователей включены в средства комплекта ресурсов IIS 6,0.

## <a name="centralized-binary-logging-file-format"></a>Формат централизованного двоичного файла ведения журнала

Необработанный централизованный файл журнала двоичного журнала состоит из записей фиксированной длины или записей индекса, содержащих идентификаторы строк. Записи индекса появятся, так как для записи максимально возможной информации строковые поля переменной длины заменяются числовыми идентификаторами — индексами, которые сопоставляют строку переменной длины с зарегистрированным идентификатором.

Необработанный файл журнала не читается человеком и не может быть прочитан с помощью большинства доступных анализаторов журналов. Чтобы извлечь данные из необработанного файла журнала, можно создать средство, которое находит и извлекает данные, а затем преобразует их в форматированный текст.
