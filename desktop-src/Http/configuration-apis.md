# <a name="configuration-apis"></a>API-интерфейсы конфигурации

Интерфейсы API HTTP-сервера предоставляют системным администраторам функции для установки, запроса и удаления параметров конфигурации. API конфигурации управляют параметрами для всех приложений, использующих API-сервер HTTP в системе.

В перечисленных ниже разделах содержатся сведения о параметрах конфигурации.

> [!Note]  
> Все это необязательные этапы настройки, применяемые в соответствии с требованиями к системе и (или) приложению.

 

[Список прослушивания IP-адресов](ip-listen-list.md) — описывает настройку протокола HTTP для работы через определенные интерфейсы IP.

[SSL-сертификаты](ssl-certificates.md) — сведения о процедурах настройки сертификатов с помощью API сервера HTTP.

[Резервирование пространства имен, регистрации и маршрутизация](namespace-reservations-registrations-and-routing.md) — описывает процесс резервирования частей пространства имен для всех приложений на компьютере с помощью API сервера HTTP. Также описывается процесс регистрации и маршрутизации запросов к конкретным приложениям.

[HttpCfg.exe](httpcfg-exe.md) — пример программы, которая использует API настройки. Скомпилированная версия этого средства также поставляется вместе с операционной системой в составе средств поддержки вместе с файлом справки, который документируется, как использовать его.
