# <a name="process-memory-usage-information"></a>Сведения об использовании памяти процесса

Функция [**жетпроцессмеморинфо**](/windows/desktop/api/Psapi/nf-psapi-getprocessmemoryinfo) принимает в качестве входных данных обработку и заполняет структуру [**\_ \_ счетчиков памяти процесса**](/windows/desktop/api/Psapi/ns-psapi-process_memory_counters) данными о статистике памяти для процесса. Член **CB** получает размер структуры. Элемент **пажефаулткаунт** получает количество ошибок страниц. Остальные члены получают текущее и пиковое использование памяти в следующих категориях:

-   Рабочий набор
-   выгружаемого пула
-   невыгружаемого пула
-   Размер

*Рабочий набор* — это объем памяти, физически сопоставленный с контекстом процесса в определенный момент времени. Память в *выгружаемого пула* — это системная память, которую можно передать в файл подкачки на диске (с разбивкой на страницы), если он не используется. В *невыгружаемом пуле* памяти находится системная память, которая не может быть помещена на диск, пока выделены соответствующие объекты. Использование файла *подкачки* представляет объем памяти, заданный для процесса в файле подкачки системы. При слишком высоком потреблении памяти Диспетчер виртуальной памяти выбирает память на диске. Когда потоку требуется страница, которая не находится в памяти, диспетчер памяти перегружает его из файла подкачки.
