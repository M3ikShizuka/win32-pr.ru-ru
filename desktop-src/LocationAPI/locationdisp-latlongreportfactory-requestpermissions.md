# <a name="locationdisplatlongreportfactoryrequestpermissions-method"></a>Локатиондисп. Латлонгрепортфактори. Рекуестпермиссионс, метод

\[Объектная модель API расположения доступна для использования в операционных системах, указанных в разделе требования. В последующих версиях он может быть изменен или недоступен. Вместо этого для доступа к расположению с веб-сайта используйте [API географического расположения W3C](/previous-versions/windows/internet-explorer/ie-developer/samples/gg589513(v=vs.85)). Чтобы получить доступ к расположению из классического приложения, используйте API [**Windows. Devices. Географическое расположение**](/uwp/api/Windows.Devices.Geolocation) .\]

Открывает диалоговое окно системы, в котором запрашиваются разрешения пользователя для устройств с поддержкой расположения.

## <a name="syntax"></a>Синтаксис


```JScript
LocationDisp.LatLongReportFactory.RequestPermissions(
  hWnd
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*hWnd* 
</dt> <dd>

Этот параметр не используется и должен иметь значение 0.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Remarks

Вызов является синхронным, и вызывающий объект ожидает закрытия диалогового окна.

> [!Note]  
> Если приложение, работающее в защищенном режиме, например объект модуля поддержки браузера (BHO) для Internet Explorer, вызывает **рекуестпермиссионс**, и пользователь выбирает параметр **не включать этот датчик расположения** в диалоговом окне, поставщик расположения не будет включен, но Windows снова отобразит это диалоговое окно, если **рекуестпермиссионс** повторно вызывается этим пользователем. Приложения, работающие в защищенном режиме, могут не вызывать **рекуестпермиссионс** при запуске, чтобы пользователь не мог видеть потенциально нежелательное диалоговое окно при каждом запуске приложения.
