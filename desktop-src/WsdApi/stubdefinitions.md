# <a name="stubdefinitions-element"></a>Стубдефинитионс, элемент

Создает реализации для функций-заглушек для операций с типом порта.

## <a name="usage"></a>Использование

``` syntax
<stubDefinitions>
  child elements
</stubDefinitions>
```

## <a name="attributes"></a>Атрибуты

Атрибуты отсутствуют.

## <a name="child-elements"></a>Дочерние элементы



| Элемент                                                         | Описание                                                                                                             |
|-----------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------|
| [**не предусматривающую**](deallocator.md)<br/>                   | Указывает тип дераспределителя, используемый функцией-заглушкой.<br/> <br/>                                 |
| [**eventArg**](eventarg.md)<br/>                         | Указывает, включаются ли в создаваемые функции связанные аргументы события.<br/> <br/>               |
| [**событиях**](events.md)<br/>                             | Указывает, включаются ли в создаваемые функции связанные события.<br/> <br/>                        |
| [**фаултинфо**](faultinfo.md)<br/>                       | Указывает, включаются ли в создаваемые функции параметры, используемые для передачи сведений об ошибках.<br/> <br/> |
| [**операцию**](operation.md)<br/>                       | Указывает операцию, для которой создается код.<br/> <br/>                                        |
| [**оператиондеаллокатор**](operationdeallocator.md)<br/> | Указывает тип дераспределителя, используемый функцией-заглушкой операции.<br/> <br/>                    |
| [**Порта**](porttype.md)<br/>                         | Указывает тип порта, для которого создается код.<br/> <br/>                                       |
| [**серверкласс**](serverclass.md)<br/>                   | Указывает имя класса сервера на стороне узла.<br/> <br/>                                                |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
(
  portType?, 
  operation*, 
  serverClass, 
  eventArg?, 
  events?, 
  operationDeallocator*, 
  deallocator?, 
  faultInfo?
)
```

## <a name="parent-elements"></a>Родительские элементы



| Элемент                         | Описание                                                    |
|---------------------------------|----------------------------------------------------------------|
| [**File**](file.md)<br/> | Выводит файл из генератора кода.<br/> <br/> |
