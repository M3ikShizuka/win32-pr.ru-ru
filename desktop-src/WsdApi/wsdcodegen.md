# <a name="wsdcodegen-element"></a>Всдкодежен, элемент

Является корневым элементом XML-файла скрипта генератора кода WSDAPI.

## <a name="usage"></a>Использование

``` syntax
<wsdCodeGen
  ConfigFileVersion = "Any character string.">
  child elements
</wsdCodeGen>
```

## <a name="attributes"></a>Атрибуты



| attribute                        | Тип                             | Обязательно       | Описание                                                                                  |
|----------------------------------|----------------------------------|----------------|----------------------------------------------------------------------------------------------|
| **конфигфилеверсион**<br/> | Любая символьная строка.<br/> | Да<br/> | Версия файла конфигурации. Единственное допустимое значение — "1,0".<br/> <br/> |



## <a name="child-elements"></a>Дочерние элементы



| Элемент                                                         | Описание                                                                                                                                                                                                                 |
|-----------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**автостатический**](autostatic.md)<br/>                     | Указывает, должна ли Всдкодежен попытаться автоматически помечать определенные созданные поля как статические. Эта функция включена по умолчанию.<br/> <br/>                                                                 |
| [**File**](file.md)<br/>                                 | Направляет генератору кода создать файл.<br/> <br/>                                                                                                                                                       |
| [**хостметадата**](hostmetadata.md)<br/>                 | Метаданные размещения для реализуемого устройства. Этот элемент используется только для реализаций устройств (узлов).<br/> <br/>                                                                                 |
| [**лайернумбер**](layernumber.md)<br/>                   | Номер создаваемого уровня кода. Номера слоев используются в таблицах среды выполнения для различения одного уровня кода для другого. WSDAPI сам использует сформированный код с номером уровня 0.<br/> <br/> |
| [**лайерпрефикс**](layerprefix.md)<br/>                   | Префикс, используемый в созданном коде для обеспечения уникальности созданных символов. В WSDAPI используется префикс "WSD".<br/> <br/>                                                                                     |
| [**макровирусах**](macro.md)<br/>                               | Определяет текст или CDATA для повторного использования элементом [**include**](include.md) .<br/> <br/>                                                                                                                        |
| [**Имен**](namespace.md)<br/>                       | Описывает пространство имен, используемое для создания кода.<br/> <br/>                                                                                                                                                |
| [**релатионшипметадата**](relationshipmetadata.md)<br/> | Описывает размещение и размещение метаданных для устройства.<br/> <br/>                                                                                                                                               |
| [**сисмоделметадата**](thismodelmetadata.md)<br/>       | Изготовитель и метаданные модели для реализуемого устройства. Этот элемент используется только для реализаций устройств (узлов).<br/> <br/>                                                                  |
| [**файле**](wsdl.md)<br/>                                 | Указывает WSDL-файл для обработки сведений о контракте.<br/> <br/>                                                                                                                                           |
| [**схеме**](xsd.md)<br/>                                   | Указывает XSD-файл для обработки сведений о контракте.<br/> <br/>                                                                                                                                           |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
(
  layerNumber?, 
  layerPrefix?, 
  autoStatic?, 
  hostMetadata?, 
  thisModelMetadata?, 
  nameSpace*, 
  wsdl*, 
  xsd*, 
  file*, 
  macro*, 
  relationshipMetadata*
)
```

## <a name="parent-elements"></a>Родительские элементы

Нет родительских элементов.

## <a name="remarks"></a>Комментарии

Как правило, элементы [**File**](file.md) должны выполняться последними, так как они создают код, использующий данные, указанные другими элементами.
