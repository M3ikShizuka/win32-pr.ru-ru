# <a name="subscriptionproxyfunctionimplementations-element"></a>Субскриптионпроксифунктионимплементатионс, элемент

Создает реализации для функций-посредников подписки и отмены подписки для операций уведомления типа порта.

## <a name="usage"></a>Использование

``` syntax
<subscriptionProxyFunctionImplementations
  extensible = "boolean">
  child elements
</subscriptionProxyFunctionImplementations>
```

## <a name="attributes"></a>Атрибуты



| attribute                 | Тип               | Обязательно      | Описание                                                                                                                   |
|---------------------------|--------------------|---------------|-------------------------------------------------------------------------------------------------------------------------------|
| **легко**<br/> | Логическое<br/> | Нет<br/> | Возможность добавлять точки расширения в функции и интерфейсы. Это значение всегда равно true.<br/> <br/> |



## <a name="child-elements"></a>Дочерние элементы



| Элемент                                                           | Описание                                                                                            |
|-------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------|
| [**нотификатионинтерфаце**](notificationinterface.md)<br/> | Указывает имя интерфейса уведомления, используемого с подписками на события.<br/> <br/> |
| [**операцию**](operation.md)<br/>                         | Указывает операцию, для которой создается код.<br/> <br/>                       |
| [**Порта**](porttype.md)<br/>                           | Указывает тип порта, для которого создается код.<br/> <br/>                      |
| [**proxyClass**](proxyclass.md)<br/>                       | Указывает имя класса прокси на стороне клиента.<br/> <br/>                              |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
(
  portType?, 
  operation*, 
  proxyClass?, 
  notificationInterface?
)
```

## <a name="parent-elements"></a>Родительские элементы



| Элемент                         | Описание                                                    |
|---------------------------------|----------------------------------------------------------------|
| [**File**](file.md)<br/> | Выводит файл из генератора кода.<br/> <br/> |
