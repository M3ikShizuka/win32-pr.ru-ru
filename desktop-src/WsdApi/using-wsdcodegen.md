# <a name="using-wsdcodegen"></a>Использование Всдкодежен

**Создание приложения WSDAPI с помощью Всдкодежен**

1.  Определите функциональный контракт для узла устройства в WSDL-файле.
2.  Создайте файл конфигурации с помощью Всдкодежен. Дополнительные сведения см. в разделе [файл конфигурации всдкодежен](wsdcodegen-configuration-file.md) и [синтаксис командной строки всдкодежен](wsdcodegen-command-line-syntax.md).
3.  Откройте созданный файл конфигурации и при необходимости измените файл. Если вы создаете клиентское приложение, изменение не требуется. При создании ведущего приложения измените пользовательские элементы конфигурации (например, [**Manufacturer**](manufacturer.md) и [**modelName**](modelname.md)).
4.  Создайте код с помощью Всдкодежен, указав файл конфигурации в качестве входных данных. Дополнительные сведения см. в описании [синтаксиса командной строки всдкодежен](wsdcodegen-command-line-syntax.md).
5.  Используйте созданный код для создания клиента, узла или и того и другого.

в Windows SDK содержатся примеры WSDL-файлов, файлы конфигурации всдкодежен и созданный код. Дополнительные сведения см. в разделе [примеры WSDAPI](wsdapi-samples.md).
