# <a name="wsdcodegen-configuration-file"></a>Файл конфигурации Всдкодежен

Файл конфигурации Всдкодежен обычно создается средством Всдкодежен. Файлы конфигурации можно создавать вручную, но сложность и длина файла обычно не позволяют вручную программировать. Мы настоятельно рекомендуем использовать Всдкодежен для создания файла. Дополнительные сведения о создании файлов конфигурации см. [в разделе Использование](using-wsdcodegen.md) [синтаксиса командной строки](wsdcodegen-command-line-syntax.md)всдкодежен и всдкодежен.

Необходимо проверить созданный файл конфигурации и при необходимости изменить его, прежде чем использовать его для создания исходного кода. Файл конфигурации, созданный Всдкодежен, обычно достаточно для большинства клиентских приложений.

Чтобы использовать файл конфигурации для разработки сервера, требуются некоторые изменения. Если размещение включено (т. е. Если выбран режим "все" или "узел"), при необходимости измените содержимое элемента [**сисмоделметадата**](thismodelmetadata.md) и его дочерних элементов. Кроме того, при необходимости измените или удалите элементы [**пнпксдевицекатегори**](/windows/desktop/WsdApi/pnpxdevicecategory), [**пнпксхардвареид**](pnpxhardwareid.md)и [**Пнпкскомпатиблеид**](/windows/desktop/WsdApi/pnpxcompatibleid) внутри элемента **сисмоделметадата** или [**размещенных**](hosted.md) элементов.

Файл конфигурации состоит из последовательности элементов, предоставляющих входные данные для создания кода, за которыми следует любое количество элементов [**File**](file.md) , описывающих создаваемые файлы. Входные данные включают несколько глобальных свойств и ссылок на типы, выраженные в WSDL, XSD и управляемых сборках. Текст и CDATA в элементах **File** записываются в созданные файлы без изменения. Другие элементы в **файловых** элементах заменяются в созданных файлах на созданный код.

XML-файлы конфигурации должны соответствовать некоторым общим правилам для правильного форматирования для использования с служебной программой генератора кода. А именно:

-   Корневым элементом любого файла конфигурации является [**всдкодежен**](wsdcodegen.md).

-   Элементы, содержащие простые типы данных, взаимозаменяемы с атрибутами. Например:

    ``` syntax
    <wsdCodeGen>
        <layerNumber>1</layerNumber>
    </wsdCodeGen>
    ```

    эквивалентно выражению

    ``` syntax
    <wsdCodeGen layerNumber="1"/>
    ```

-   Как правило, ограничение на порядок элементов отсутствует. Например:

    ``` syntax
    <wsdCodeGen>
        <layerNumber>1</layerNumber>
        <layerPrefix>MEDIA_</layerPrefix>
    </wsdCodeGen>
    ```

    эквивалентно выражению

    ``` syntax
    <wsdCodeGen>
        <layerPrefix>MEDIA_</layerPrefix>
        <layerNumber>1</layerNumber>
    </wsdCodeGen>
    ```

    Однако генератор кода обрабатывает файл конфигурации за один проход, и порядок имеет некоторую релевантность. Например, элементы [**File**](file.md) , которые создают код, связанный с определенным типом порта, должны находиться после элемента, который указывает генератору кода на необходимость чтения контракта типа порта.

Полный список элементов, используемых в файлах конфигурации Всдкодежен, см. в разделе [Справочник по XML-файлу конфигурации всдкодежен](wsdcodegen-configuration-file-xml-reference.md).

примеры файлов конфигурации включены в Windows SDK. Дополнительные сведения см. в разделе [примеры WSDAPI](wsdapi-samples.md).
