# <a name="hosted-element"></a>размещенный элемент

Определяет элементы [**ServiceID**](serviceid.md), [**types**](types.md),[**пнпксхардвареид**](pnpxhardwareid.md)и [**пнпкскомпатиблеид**](pnpxcompatibleid.md) для служб, определенных узлом службы.

## <a name="usage"></a>Использование

``` syntax
<hosted>
  child elements
</hosted>
```

## <a name="attributes"></a>Атрибуты

Атрибуты отсутствуют.

## <a name="child-elements"></a>Дочерние элементы



| Элемент                                                 | Описание                                                                      |
|---------------------------------------------------------|----------------------------------------------------------------------------------|
| [**пнпкскомпатиблеид**](pnpxcompatibleid.md)<br/> | Указывает совместимый с PnP-X идентификатор службы.<br/> <br/> |
| [**пнпксхардвареид**](pnpxhardwareid.md)<br/>     | Указывает идентификатор оборудования PnP-X для службы.<br/> <br/>   |
| [**ServiceID**](serviceid.md)<br/>               | Определяет идентификатор службы для узла службы.<br/> <br/>        |
| [**Типы**](types.md)<br/>                       | Определяет список полных имен XSD.<br/> <br/>                    |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
(
  ServiceID, 
  Types, 
  PnPXHardwareId?, 
  PnPXCompatibleId?
)
```

## <a name="parent-elements"></a>Родительские элементы



| Элемент                                         | Описание                                                                   |
|-------------------------------------------------|-------------------------------------------------------------------------------|
| [**хостметадата**](hostmetadata.md)<br/> | Метаданные размещения для реализуемого устройства.<br/> <br/> |



## <a name="remarks"></a>Примечания

Каждая служба, предоставляемая узлом службы, должна иметь собственные сведения о **размещенном** элементе, чтобы обеспечить правильную публикацию службы в ответ на запросы метаданных.

Элементы [**пнпксхардвареид**](pnpxhardwareid.md) и [**пнпкскомпатиблеид**](pnpxcompatibleid.md) необязательны, но при их использовании их необходимо использовать вместе. Если таковой имеется, также должен присутствовать и другой.

Если имеется элемент [**пнпксдевицекатегори**](pnpxdevicecategory.md) , то по крайней мере один **размещенный** элемент должен содержать как элементы [**пнпксхардвареид**](pnpxhardwareid.md) , так и [**пнпкскомпатиблеид**](pnpxcompatibleid.md) . Аналогично, если элементы **пнпксхардвареид** и **пнпкскомпатиблеид** находятся в **размещенном** элементе, то внутри элемента [**сисмоделметадата**](thismodelmetadata.md) должен присутствовать хотя бы один элемент **пнпксдевицекатегори** .
