# <a name="wsdcodegen-command-line-syntax"></a>Синтаксис командной строки Всдкодежен

Всдкодежен имеет две функции: создание файлов конфигурации и создание исходного кода для клиентских и хостовых приложений WSDAPI. В этом разделе приводится синтаксис командной строки, используемый для выполнения каждой задачи.

## <a name="generating-a-configuration-file"></a>Создание файла конфигурации

### <a name="syntax"></a>Синтаксис

**WSDCODEGEN.EXE** **/женератеконфиг:**{ \| **узел** клиента \| **ALL**} **\[ /аутпутфиле:**_конфигфиленаме_ *_\]_* *всдлфиленамелист*

### <a name="parameters"></a>Параметры

<dl> <dt>

<span id="_generateconfig__client___host___all_"></span><span id="_GENERATECONFIG__CLIENT___HOST___ALL_"></span>**/женератеконфиг: { \| узел клиента \| все}**
</dt> <dd>

Тип кода, который будет создан выходным файлом конфигурации. **/женератеконфиг: Client** используется для создания файла конфигурации для создания кода клиента, **/женератеконфиг: Host** используется для создания файла конфигурации для создания кода узла, а **/женератеконфиг: ALL** используется для создания файла конфигурации для создания клиента и кода узла.

</dd> <dt>

<span id="_outputfile_ConfigFileName"></span><span id="_outputfile_configfilename"></span><span id="_OUTPUTFILE_CONFIGFILENAME"></span>**/аутпутфиле:**_конфигфиленаме_
</dt> <dd>

Этот необязательный параметр используется для указания имени выходного файла конфигурации. Если этот параметр исключен, то имя выходного файла конфигурации будет codegen.config.

</dd> <dt>

<span id="_pnpx"></span><span id="_PNPX"></span>*/пнпкс*
</dt> <dd>

Включите шаблон PnP-X в файл конфигурации.

</dd> <dt>

<span id="WSDLFileNameList"></span><span id="wsdlfilenamelist"></span><span id="WSDLFILENAMELIST"></span>*всдлфиленамелист*
</dt> <dd>

Разделенный пробелами список WSDL-файлов, которые должны быть обработаны Всдкодежен.

</dd> </dl>

## <a name="generating-source-code"></a>Создание исходного кода

### <a name="syntax"></a>Синтаксис

**WSDCODEGEN.EXE** **/женератекоде** **\[ /download \]** **\[ /ГБК \]** **\[ аутпутрут:**_путь_ *_\]_* **\[ /вритеакцесс:**_Command_ *_\]_* *конфигфиленаме*

### <a name="parameters"></a>Параметры

<dl> <dt>

<span id="_generatecode"></span><span id="_GENERATECODE"></span>**/женератекоде**
</dt> <dd>

Направляет Всдкодежен для создания исходного кода. Этот режим используется по умолчанию, если режим не указан.

</dd> <dt>

<span id="_download"></span><span id="_DOWNLOAD"></span>**/Download**
</dt> <dd>

Скачивает импортированные документы, на которые ссылается файл конфигурации. Это необязательный параметр.

</dd> <dt>

<span id="_gbc"></span><span id="_GBC"></span>**/гбк**
</dt> <dd>

Добавляет комментарии в исходный код, который указывает на созданный код. Эти комментарии начинаются с фразы "создано". Это необязательный параметр.

</dd> <dt>

<span id="_outputroot_path"></span><span id="_OUTPUTROOT_PATH"></span>**/аутпутрут:**_путь_
</dt> <dd>

Расположение выходных данных для созданных файлов. *путь* может быть абсолютным или относительным путем. Это необязательный параметр.

</dd> <dt>

<span id="_writeaccess_command"></span><span id="_WRITEACCESS_COMMAND"></span>**/вритеакцесс:**_Command_
</dt> <dd>

Направляет Всдкодежен для выполнения указанной команды перед изменением существующих файлов на диске. Выходные файлы, идентичные параметрам на диске, не будут получать эту команду и не будут записываться в. Если команда содержит последовательность " {0} ", эта последовательность будет заменена именем изменяемого файла. В противном случае имя файла будет добавлено к команде.

Примеры:

**/вритеакцесс: "attrib-r"**

**/вритеакцесс: "attrib-r {0} "**

**/вритеакцесс: "Copy {0} .. \\ Резервное копирование \\ »**

</dd> <dt>

<span id="ConfigFileName"></span><span id="configfilename"></span><span id="CONFIGFILENAME"></span>*конфигфиленаме*
</dt> <dd>

Имя файла конфигурации для обработки перед созданием кода.

</dd> </dl>

## <a name="formatting-legend"></a>Условные обозначения форматирования



| Формат                                                                    | Значение                                                 |
|---------------------------------------------------------------------------|---------------------------------------------------------|
| *Наклонный*                                                                  | Сведения, которые должен предоставить пользователь                  |
| **Полужирный**                                                                  | Элементы, которые пользователь должен ввести в точности так, как показано       |
| Между скобками ( \[ \] )                                                   | Необязательные элементы                                          |
| Между фигурными скобками ( {} ); варианты, разделенные вертикальной чертой ( \| ). Пример: {четных и \| нечетных} | Набор вариантов, из которых пользователь должен выбрать только один |
