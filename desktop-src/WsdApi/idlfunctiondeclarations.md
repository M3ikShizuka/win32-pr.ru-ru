# <a name="idlfunctiondeclarations-element"></a>Идлфунктиондекларатионс, элемент

Создает объявления IDL для функций-посредников для операций с типом порта.

## <a name="usage"></a>Использование

``` syntax
<idlFunctionDeclarations>
  child elements
</idlFunctionDeclarations>
```

## <a name="attributes"></a>Атрибуты

Атрибуты отсутствуют.

## <a name="child-elements"></a>Дочерние элементы



| Элемент                                   | Описание                                                                                                             |
|-------------------------------------------|-------------------------------------------------------------------------------------------------------------------------|
| [**async**](async.md)<br/>         | Указывает, включаются ли асинхронные операции в создаваемые прокси-функции.<br/> <br/>         |
| [**eventArg**](eventarg.md)<br/>   | Указывает, включаются ли в создаваемые функции связанные аргументы события.<br/> <br/>               |
| [**событиях**](events.md)<br/>       | Указывает, включаются ли в создаваемые функции связанные события.<br/> <br/>                        |
| [**фаултинфо**](faultinfo.md)<br/> | Указывает, включаются ли в создаваемые функции параметры, используемые для передачи сведений об ошибках.<br/> <br/> |
| [**операцию**](operation.md)<br/> | Указывает операцию, для которой создается код.<br/> <br/>                                        |
| [**Порта**](porttype.md)<br/>   | Указывает тип порта, для которого создается код.<br/> <br/>                                       |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
(
  portType?, 
  operation*, 
  eventArg?, 
  events?, 
  async?, 
  faultInfo?
)
```

## <a name="parent-elements"></a>Родительские элементы



| Элемент                         | Описание                                                    |
|---------------------------------|----------------------------------------------------------------|
| [**File**](file.md)<br/> | Выводит файл из генератора кода.<br/> <br/> |



## <a name="remarks"></a>Комментарии

Этот элемент создает объявления функций элементов, соответствующих операциям, вызываемым контрактом. Эти объявления находятся в форме, подходящей для использования компилятором MIDL и обычно используются в IDL-файлах.

Пример.

``` syntax
<idlFunctionDeclarations events = "true"/>
```
