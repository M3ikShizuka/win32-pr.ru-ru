# <a name="xsd-element"></a>элемент XSD

Указывает XSD-файл для обработки сведений о контракте.

## <a name="usage"></a>Использование

``` syntax
<xsd
  path = "pathname string">
  child elements
</xsd>
```

## <a name="attributes"></a>Атрибуты



| attribute           | Тип                       | Обязательно       | Описание                                                 |
|---------------------|----------------------------|----------------|-------------------------------------------------------------|
| **путь**<br/> | Строка пути<br/> | Да<br/> | Файл и путь к входному файлу XSD.<br/> <br/> |



## <a name="child-elements"></a>Дочерние элементы



| Элемент                               | Описание                                                          |
|---------------------------------------|----------------------------------------------------------------------|
| [**типеури**](typeuri.md)<br/> | Указывает тип, включаемый из XSD-файла.<br/> <br/> |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
typeUri*
```

## <a name="parent-elements"></a>Родительские элементы



| Элемент                                     | Описание                                                                          |
|---------------------------------------------|--------------------------------------------------------------------------------------|
| [**всдкодежен**](wsdcodegen.md)<br/> | Корневой элемент XML-файла скрипта генератора кода WSDAPI.<br/> <br/> |



## <a name="remarks"></a>Комментарии

Строка filename также должна содержать сведения о полном пути.
