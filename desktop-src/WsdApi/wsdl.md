# <a name="wsdl-element"></a>элемент WSDL

Указывает WSDL-файл для обработки сведений о контракте.

## <a name="usage"></a>Использование

``` syntax
<wsdl>
  child elements
</wsdl>
```

## <a name="attributes"></a>Атрибуты

Атрибуты отсутствуют.

## <a name="child-elements"></a>Дочерние элементы



| Элемент                                           | Описание                                                                                                                                       |
|---------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------|
| [**ексклудеимпорт**](excludeimport.md)<br/> | Предотвращает создание операторов импорта для указанных целевых объектов с именем в элементе <WSDL: Import> в WSDL-файле. <br/> <br/> |
| [**импорсинт**](importhint.md)<br/>       | Указывает расположение загрузки для директивы> <WSDL: Import, которая явно не задает расположение.<br/> <br/>           |
| [**путь**](path.md)<br/>                   | Файл и путь к входному файлу WSDL.<br/> <br/>                                                                                      |



### <a name="child-element-sequence"></a>Последовательность дочерних элементов

``` syntax
(
  importHint*, 
  excludeImport*, 
  path
)
```

## <a name="parent-elements"></a>Родительские элементы



| Элемент                                     | Описание                                                                          |
|---------------------------------------------|--------------------------------------------------------------------------------------|
| [**всдкодежен**](wsdcodegen.md)<br/> | Корневой элемент XML-файла скрипта генератора кода WSDAPI.<br/> <br/> |



## <a name="remarks"></a>Комментарии

Все элементы [**импорсинт**](importhint.md) и [**ексклудеимпорт**](excludeimport.md) , появляющиеся после элемента [**path**](path.md) в XML-файле конфигурации, будут игнорироваться.
