# <a name="providing-user-permissions-for-media-burning-devices"></a>Предоставление пользователям разрешений для устройств записи мультимедиа

по умолчанию как Windows Vista, так и Windows Server 2008 предоставляют администраторам и пользователям доступ на чтение и запись для пользователей, которые регистрируются непосредственно на компьютере (промежуточные пользователи). однако в Windows XP и Windows Server 2003 администратор должен предоставить этим устройствам права на чтение и запись для других групп пользователей.

Администратор может настроить определенные разрешения устройства для опытных пользователей и интерактивных пользователей.

чтобы открыть соответствующую панель разрешений группы в Windows XP, нажмите кнопку **пуск**, выберите команду **выполнить**, введите **gpedit. msc** и нажмите кнопку **ок**. в интерфейсе групповая политика разверните узел **конфигурация компьютера**, разверните узел **Windows Параметры**, разверните узел **безопасность Параметры**, разверните узел **локальные политики** и дважды щелкните **параметры безопасности**.

![Снимок экрана, на котором показано окно "групповая политика" с политикой, выбранной на панели "политика".](images/gpolpanel.jpg)

На этой панели администратор должен указать параметры двух параметров устройства, чтобы предоставить необходимые разрешения группы.

-   Установите флажок "устройства: разрешить доступ к компакт-ДИСКам только для локального пользователя", чтобы **включить**
-   Настройка "устройства: разрешено форматировать и извлекать съемные носители" для **администраторов и опытных пользователей**. можно также эмулировать разрешения Windows Vista, установив для этого параметра значение **администраторы и интерактивные пользователи**.

хотя конкретный пользовательский интерфейс не существует в Windows XP или Windows Server 2003 для использования [**сетсекуритинфо**](/windows/desktop/api/aclapi/nf-aclapi-setsecurityinfo) или [сетупдисетдевицерегистрипроперти](/windows/win32/api/setupapi/nf-setupapi-setupdisetdeviceregistrypropertya), эти api-интерфейсы можно использовать для предоставления пользовательских групп разрешений на устройства. Например, вызов **сетсекуритинфо** предоставит разрешения группам пользователей. Изменения разрешений с помощью этого API являются временными и не будут сохранены при перезагрузке. Однако при вызове [сетупдисетдевицерегистрипроперти](/windows/win32/api/setupapi/nf-setupapi-setupdisetdeviceregistrypropertya) будут реализованы изменения разрешений в реестре, которые будут сохраняться во время перезагрузки.
