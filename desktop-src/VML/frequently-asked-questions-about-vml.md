# <a name="frequently-asked-questions-about-vml"></a>Часто задаваемые вопросы о VML

В этом разделе описывается функция VML, которая является устаревшей в Windows Internet Explorer 9. Веб-страницы и приложения, использующие VML, должны быть перенесены в формат SVG или другие широко поддерживаемые стандарты.

> [!Note]  
> По состоянию на Декабрь 2011 этот раздел был архивирован. В результате он больше не поддерживается. Дополнительные сведения см. в разделе [архивированное содержимое](/previous-versions/windows/internet-explorer/ie-developer/). Сведения, рекомендации и рекомендации по текущей версии Windows Internet Explorer см. в [центре разработчиков Internet Explorer](https://msdn.microsoft.com/ie/).

 

## <a name="does-vml-compete-with-gif-and-png"></a>Конкурирует ли VML с GIF и PNG?

Нет. GIF и PNG являются растровыми (или битовыми) форматами, которые можно использовать для хранения векторной графики. Растровые форматы хранят каждый пиксель, тогда как векторные форматы, такие как VML, используют математические описания или контуры для описания графики. Векторная графика, хранящаяся в векторном формате, более компактнее, чем в растровых форматах, что приводит к ускоренному времени загрузки для веб-пользователей.

Кроме того, графика VML доставляется в HTML-страницу, а не на внешние файлы, как в случае с GIF и PNG уже сегодня. Это позволяет графикам VML масштабироваться и взаимодействовать с другими элементами веб-страницы при изменении размера страницы, тем самым улучшая взаимодействие с конечными пользователями.

## <a name="why-is-microsoft-supporting-another-xml-based-standard-when-xml-is-hardly-in-use-and-is-young-enough-as-it-is"></a>Почему корпорация Майкрософт поддерживает другой стандарт на основе XML, когда XML вряд ли используется и достаточно велик, чем это?

XML — это мощный, но простой способ представления структурированных данных в Интернете и полностью дополняющий HTML для отображения. VML — это один из примеров многих зависящих от приложения форматов на основе XML, которые будут разработаны и реализованы в ближайшие месяцы. Например, в следующей версии Office язык VML будет использоваться для добавления заметка к документам HTML, чтобы сохранять сведения о форматировании графических элементов Office в формате HTML, а значит, пользователи Office могут легко переключаться между двумя форматами.

## <a name="who-will-support-vml"></a>Кто будет поддерживать VML?

Мы планируем, чтобы многие поставщики поддерживали VML. Например, для Autodesk, Hewlett-Packard, Macromedia и VISIO предусмотрена поддержка VML в будущих версиях своих продуктов. Корпорация Майкрософт приработала поддержку VML в будущих версиях своих платформ, таких как Internet Explorer. Кроме того, VML будет использоваться в следующем поколении Office, чтобы включить циклическое преобразование между форматом Office и HTML.

## <a name="does-vml-support-animation"></a>Поддерживает ли VML анимацию?

Нет, сейчас нет. Однако мы работаем с нашими партнерами VML, чтобы добавить возможности анимации в формат VML. Поскольку VML основан на XML, набор тегов можно легко расширить для дополнительных функциональных возможностей.

## <a name="does-vml-replace-macromedia-flash-didnt-macromedia-just-submit-their-flash-format-for-vector-graphics-to-the-w3c"></a>Заменяет ли VML Macromedia Flash? Не только компания Macromedia отправила свой формат Flash для векторной графики в консорциум W3C?

Нет. VML — это текстовый формат обмена данными и доставки для векторной графики, тогда как Flash — это двоичный формат для доставки векторной графики и анимации.

Компания Macromedia не отправила свой формат файла консорциуму W3C. Однако они открывали свои форматы файлов для разработчиков приложений, веб-разработчиков и конечных пользователей. Подробнее см. в разделе <https://www.macromedia.com>.
