# <a name="registering-a-time-provider"></a>Регистрация поставщика времени

Система загружает поставщик времени на основе сведений о конфигурации, хранящихся в реестре. Каждый поставщик времени должен создать следующий раздел реестра:

**HKey \_ Локальный \_ компьютер \\ System** \\ **CurrentControlSet** \\ **Services** \\ **W32Time** \\ **тимепровидерс** \\ *providerName*

В следующей таблице описаны значения, которые должны существовать в ключе каждого поставщика.



| Значение             | Описание                                                                                                                                                                                                              |
|-------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **DllName**       | Имя библиотеки DLL, содержащей поставщик. Это значение имеет тип **reg \_ SZ**.                                                                                                                                     |
| **Enabled**       | Указывает, следует ли запускать поставщик. Если это значение равно 1, то поставщик запускается. В противном случае поставщик не запускается. Это значение имеет тип **reg \_ DWORD**.                                           |
| **InputProvider** | Указывает, является ли поставщик входным поставщиком или поставщиком выходных данных. Если это значение равно 1, поставщик является поставщиком входных данных. В противном случае поставщик является выходным поставщиком. Это значение имеет тип **reg \_ DWORD**. |



 

Диспетчер поставщиков времени перечисляет ключи в ключе **тимепровидерс** и запускает каждый включенный поставщик. Поставщики запускаются при запуске системы и при каждом изменении параметров.

Каждый поставщик времени может также хранить в реестре сведения о конфигурации приложения.
