# <a name="registry-element-size-limits"></a>Ограничения размера элементов реестра

В следующей таблице приведены ограничения по размеру для различных элементов реестра.



| Элемент реестра | Максимальный размер                                                                                                                                            |
|------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| Имя ключа         | 255 символов. Имя ключа включает абсолютный путь к разделу в реестре, который всегда начинается с базового ключа, например HKEY \_ локальный \_ компьютер. |
| Имя значения       | 16 383 символов **Windows 2000:** 260 ANSI символов или 16 383 символов юникода.<br/>                                                       |
| Значение            | Доступная память (последний формат) 1 МБ (стандартный формат)<br/>                                                                                     |
| Дерево             | Дерево реестра может иметь глубину 512 уровней. С помощью одного вызова API реестра можно создать до 32 уровней за раз.                                  |



 

Длинные значения (более 2 048 байт) должны храниться в файле, а расположение файла должно храниться в реестре. Это помогает реестру работать эффективно.

Расположением файла может быть имя значения или данные значения. Каждой обратной косой черте в строке расположения должна предшествовать другая обратная косая черта в виде escape-символа. Например, укажите "C: \\ \\ MyDir \\ \\ MyFile", чтобы сохранить строку "c: \\ MyDir \\ MyFile". Расположение файла также может быть именем ключа, если оно находится в пределах 255 символов в именах ключей и не содержит обратных косых черт, которые не допускаются в именах ключей.
