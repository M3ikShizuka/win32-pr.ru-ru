# <a name="object-manager"></a>Диспетчер объектов

Объект состоит из стандартного заголовка и атрибутов, зависящих от объекта. поскольку все объекты имеют одинаковую структуру, в Windows используется один диспетчер объектов, который обслуживает все объекты.

Заголовок объекта включает такие элементы, как имя объекта, чтобы другие процессы могли ссылаться на объект по имени и дескриптор безопасности, чтобы диспетчер объектов мог контролировать, какие процессы обращаются к системному ресурсу.

В число задач, выполняемых диспетчером объектов, входят следующие.

-   Создание объектов
-   Проверка того, что процесс имеет право использовать объект
-   Создание дескрипторов объектов и их возврат вызывающему объекту
-   Обслуживание квот ресурсов
-   Создание повторяющихся дескрипторов
-   Закрытие дескрипторов объектов
