# <a name="operating-system-version"></a>Версия операционной системы

[Вспомогательные функции API версии](version-helper-apis.md) используются для определения версии операционной системы, которая выполняется в данный момент. Дополнительные сведения см. [в разделе получение системной версии](getting-the-system-version.md).

В следующей таблице перечислены последние номера версий операционной системы.

| Операционная система | номер версии; |
|------------------|----------------|
| Windows 10       | 10,0\*         |
| Windows Server 2019 | 10,0\*      |
| Windows Server 2016 | 10,0\*      |
| Windows 8.1      | 6.3\*          |
| Windows Server 2012 R2 | 6.3\*    |
| Windows 8        | 6.2            |
| Windows Server 2012 | 6.2         |
| Windows 7        | 6.1            |
| Windows Server 2008 R2 | 6.1      |
| Windows Server 2008 | 6.0         |
| Windows Vista    | 6.0            |
| Windows Server 2003 R2 | 5,2      |
| Windows Server 2003 | 5,2         |
| Windows XP 64-bit Edition | 5,2   |
| Windows XP | 5.1                  |
| Windows 2000     | 5.0            |

**\*** для приложений, которые были объявлены для Windows 8.1 или Windows 10. приложения, не проявляющиеся манифестом для Windows 8.1 или Windows 10, возвращают значение Windows 8 версии ос (6,2). чтобы выполнить манифест приложений для Windows 8.1 или Windows 10, см. [сведения в разделе настройка приложения для Windows](targeting-your-application-at-windows-8-1.md).<br/>

Определение текущей операционной системы обычно не является лучшим способом определить, имеется ли определенная функция операционной системы. Это связано с тем, что операционная система могла добавить новые функции в распространяемую библиотеку DLL. Вместо использования [вспомогательных функций API версии](version-helper-apis.md) для определения платформы или номера версии операционной системы проверьте наличие самой функции.

Чтобы определить лучший способ проверки функции, ознакомьтесь с документацией по интересующей функции. В следующем списке обсуждаются некоторые распространенные методы обнаружения компонентов.

- Можно проверить наличие функций, связанных с компонентом. Чтобы проверить наличие функции в системной библиотеке DLL, вызовите функцию [**LoadLibrary**](/windows/desktop/api/libloaderapi/nf-libloaderapi-loadlibrarya) для загрузки библиотеки DLL. Затем вызовите функцию [**GetProcAddress**](/windows/desktop/api/libloaderapi/nf-libloaderapi-getprocaddress) , чтобы определить, имеется ли в библиотеке DLL интересующая функция. Используйте указатель, возвращенный **GetProcAddress** , для вызова функции. Обратите внимание, что даже если функция имеется, она может быть заглушкой, которая просто возвращает код ошибки, такой как « \_ вызов ошибки \_ не \_ реализован».
- Наличие некоторых функций можно определить с помощью функции [**жетсистемметрикс**](/windows/desktop/api/winuser/nf-winuser-getsystemmetrics) . Например, можно обнаружить несколько мониторов монитора, вызвав **жетсистемметрикс**(SM \_ кмониторс).
- Существует несколько версий распространяемых библиотек DLL, которые реализуют оболочку и общие функции управления. Сведения о том, как определить, какие версии находятся в системе, в которой выполняется приложение, см. в разделе [оболочка разделов и стандартных элементов управления](/previous-versions/windows/desktop/legacy/bb776779(v=vs.85)).

Если требуется конкретная операционная система, обязательно используйте ее как минимальную поддерживаемую версию, а не разрабатывать тест для одной операционной системы. Таким образом, код обнаружения будет продолжать работать в будущих версиях Windows.

Обратите внимание, что 32-разрядное приложение может определить, выполняется ли оно в эмуляторе WOW64, вызвав функцию [**IsWow64Process**](/windows/desktop/api/wow64apiset/nf-wow64apiset-iswow64process) . Он может получить дополнительные сведения о процессоре, вызвав функцию [**жетнативесистеминфо**](/windows/win32/api/sysinfoapi/nf-sysinfoapi-getnativesysteminfo) .

дополнительные сведения см. в разделе [Windows 10 сведения о выпуске](/windows/release-information/) и [Windows таблице фактов жизненного цикла](https://support.microsoft.com/help/13853/windows-lifecycle-fact-sheet).
