# <a name="the-role-of-network-list-manager"></a>Роль диспетчера списка сетей

до Windows XP и Windows Server 2003 приложениям требовалось вызывать ряд несвязанных интерфейсов api для получения данных о сетевых атрибутах, таких как IP-адрес, контроллер домена или система доменных имен (DNS). начиная с Windows XP и Windows Server 2003, в интерфейсе API Winsock, осведомленном о сетевом расположении, указан ограниченный набор функций сетевого расположения. в Windows Server 2008 и Windows Vista служба осведомленности сети захватывает сетевые атрибуты в одном расположении и позволяет приложениям запрашивать и получать определенные сети и атрибуты. Диспетчер списка сетей заменяет функциональные возможности предыдущего API Winsock (доступного в качестве поставщика пространства имен Winsock), сохраняя совместимость с более старыми приложениями с помощью API Winsock.
