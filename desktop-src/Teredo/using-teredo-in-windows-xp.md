# <a name="using-teredo-in-windows-xp"></a>использование Teredo в Windows XP

чтобы использовать клиент Teredo или ретранслятор для конкретного узла на компьютерах под управлением Windows XP с пакетом обновления 1 (sp1) и расширенным сетевым пакетом, Windows XP с пакетом обновления 2 (sp2), Windows Server 2003 с пакетом обновления 1 (sp1) или Windows Server 2003 с пакетом обновления 2 (SP2), разработчик приложения должен выполнить следующие действия.

-   убедитесь, что приложение совместимо с IPv6, используя новые Windows соединители 2 (функции и структуры), поддерживающие как IPv4, так и IPv6. дополнительные сведения см. в разделе [рекомендации по IPv6 для приложений Windows sockets](../winsock/ipv6-guide-for-windows-sockets-applications-2.md).
-   включите использование Teredo в приложении, установив \_ \_ для параметра уровень защиты IPV6 Windows сокеты sockets уровень защиты \_ \_ неограниченный уровень. Дополнительные сведения см. [в разделе Использование \_ \_ уровня защиты IPv6](/previous-versions/aa916826(v=msdn.10)). этот параметр также можно задать с помощью класса [System .net. sockets](/dotnet/api/system.net.sockets?view=netcore-3.1) платформа .NET Framework.
-   создайте исключение для брандмауэра Windows, чтобы разрешить незапрошенный входящий трафик Teredo. используйте [Windows api брандмауэра](/previous-versions/windows/desktop/ics/windows-firewall-start-page) , чтобы создать исключение порта для UDP-порта, назначенного для трафика Teredo. Дополнительные сведения и примеры, описывающие требования к безопасности и трафику для Teredo, см. [в разделе использование Teredo](using-teredo.md).

Чтобы обеспечить доступность Teredo для использования при запуске приложения, разработчики приложений должны выполнить следующие действия в процессе установки приложения:

-   Установите IPv6 с помощью команды **netsh interface IPv6 Install** . Windows Брандмауэр защищает компьютер пользователя от незапрошенного входящего трафика IPv6 таким же образом, как и трафик IPv4.
-   Включите Teredo с помощью команды **клиента netsh interface IPv6 Set Teredo** .

При необходимости можно проверить, установлен ли протокол IPv6 при каждом запуске приложения и установке протокола IPv6, и при необходимости включить Teredo. Следует также сообщить пользователю, что устанавливается IPv6 и включен ли Teredo.
