# <a name="using-teredo-in-windows-xp"></a>Использование Teredo в Windows XP

Чтобы использовать клиент Teredo или ретранслятор для конкретного узла на компьютерах под управлением Windows XP с пакетом обновления 1 (SP1) и пакетом расширенного сетевого пакета, Windows XP с пакетом обновления 2 (SP2), Windows Server 2003 с пакетом обновления 1 (SP1) или Windows Server 2003 с пакетом обновления 2 (SP2), разработчик приложения должен выполнить следующие действия.

-   Убедитесь, что приложение совместимо с IPv6, используя новые элементы программирования Windows Sockets 2 (функции и структуры), поддерживающие как IPv4, так и IPv6. Дополнительные сведения см. в разделе [рекомендации по IPv6 для приложений Windows Sockets](../winsock/ipv6-guide-for-windows-sockets-applications-2.md).
-   Включите использование Teredo в приложении, установив \_ \_ для параметра сокет защиты IPv6 Windows сокеты значение \_ \_ Неограниченный уровень защиты. Дополнительные сведения см. [в разделе Использование \_ \_ уровня защиты IPv6](/previous-versions/aa916826(v=msdn.10)). Этот параметр также можно задать с помощью класса [System .NET. sockets](/dotnet/api/system.net.sockets?view=netcore-3.1) платформа .NET Framework.
-   Создайте исключение для брандмауэра Windows, чтобы разрешить незапрошенный входящий трафик Teredo. Используйте API [брандмауэра Windows](/previous-versions/windows/desktop/ics/windows-firewall-start-page) для создания исключения порта для UDP-порта, назначенного для трафика Teredo. Дополнительные сведения и примеры, описывающие требования к безопасности и трафику для Teredo, см. [в разделе использование Teredo](using-teredo.md).

Чтобы обеспечить доступность Teredo для использования при запуске приложения, разработчики приложений должны выполнить следующие действия в процессе установки приложения:

-   Установите IPv6 с помощью команды **netsh interface IPv6 Install** . Брандмауэр Windows защищает компьютер пользователя от незапрошенного входящего трафика IPv6 таким же образом, как и трафик IPv4.
-   Включите Teredo с помощью команды **клиента netsh interface IPv6 Set Teredo** .

При необходимости можно проверить, установлен ли протокол IPv6 при каждом запуске приложения и установке протокола IPv6, и при необходимости включить Teredo. Следует также сообщить пользователю, что устанавливается IPv6 и включен ли Teredo.
