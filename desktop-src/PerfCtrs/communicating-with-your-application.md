# <a name="communicating-with-your-application"></a>Взаимодействие с приложением

Как правило, поставщик предоставляет данные от имени приложения. Например, сервер может создать библиотеку DLL производительности для предоставления данных счетчиков. Взаимодействие между приложением и его поставщиком отличается для приложений пользовательского режима и режима ядра. Поставщики выполняются в пользовательском режиме. В связи с этим приложения пользовательского режима, такие как приложения для печати и отображения, могут использовать любой способ межпроцессного взаимодействия, например именованные каналы, сопоставление файлов или RPC. Однако приложения в режиме ядра должны предоставлять интерфейс IOCTL, который возвращает поставщику данные о производительности.

> [!WARNING]
> Не используйте COM в качестве механизма IPC. Система не может гарантировать состояние инициализации COM потока, вызывающего интерфейс. Поэтому библиотека DLL не сможет успешно инициализировать COM и получить данные.
