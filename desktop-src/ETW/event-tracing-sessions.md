# <a name="event-tracing-sessions"></a>Сеансы трассировки событий

Сеансы трассировки событий записывают события от одного или нескольких [поставщиков](providing-events.md) , которые поддерживает [контроллер](controlling-event-tracing-sessions.md) . Сеанс также отвечает за управление и очистку буферов. Контроллер определяет сеанс, который обычно включает в себя указание имени сеанса и файла журнала, тип используемого файла журнала и разрешение метки времени, используемой для записи событий.

Трассировка событий поддерживает не более 64 сеансов трассировки событий, выполняющихся одновременно. В этих сеансах есть два особых сеанса назначения. Остальные сеансы доступны для общего использования. Два особых сеанса назначения:

-   Сеанс глобального журнала
-   Сеанс ведения журнала ядра NT

Сеанс трассировки глобального журнала событий регистрирует события, происходящие на ранних этапах процесса загрузки операционной системы, например созданные драйверами устройств. Сведения о настройке сеанса отслеживания событий в глобальном средстве ведения журнала см. в разделе [Настройка и запуск глобального сеанса ведения журнала](configuring-and-starting-the-global-logger-session.md).

Сеанс трассировки событий журнала ядра NT записывает стандартные системные события, создаваемые операционной системой, например, события ввода-вывода диска или ошибки страницы. Сведения о настройке сеанса трассировки событий системного журнала NT, [настройке и запуске сеанса ведения журнала ядра NT](configuring-and-starting-the-nt-kernel-logger-session.md).

Сведения об определении сеанса трассировки регулярных событий см. в разделе [Настройка и запуск сеанса трассировки событий](configuring-and-starting-an-event-tracing-session.md).

**Windows 2000:** Поддерживает только сеансы трассировки событий 32.
