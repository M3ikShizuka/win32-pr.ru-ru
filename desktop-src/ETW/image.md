# <a name="image-class"></a>Класс Image

Этот класс является родительским для событий изображений.

Следующий синтаксис упрощен из MOF-кода.

## <a name="syntax"></a>Синтаксис

``` syntax
[Guid("{2cb15d1d-5fc1-11d2-abe1-00a0c911f518}"), EventVersion(2)]
class Image : MSNT_SystemTrace
{
};
```

## <a name="members"></a>Члены

Класс **Image** не определяет никаких членов.

## <a name="remarks"></a>Remarks

Чтобы включить события изображений в сеансе ведения журнала ядра NT, укажите **флаг \_ \_ \_ \_ загрузки изображения флага трассировки событий** в элементе **енаблефлагс** структуры [**\_ \_ свойств трассировки событий**](/windows/win32/api/evntrace/ns-evntrace-event_trace_properties) при вызове функции [**старттраце**](/windows/win32/api/evntrace/nf-evntrace-starttracea) .

Потребители трассировки событий могут реализовать специальную обработку событий загрузки изображения, вызвав функцию [**сеттрацекаллбакк**](/windows/win32/api/evntrace/nf-evntrace-settracecallback) и указав [**имажелоадгуид**](nt-kernel-logger-constants.md) в качестве параметра *пгуид* . Используйте следующие типы событий для обнаружения событий загрузки изображений при использовании событий.



| Тип события                                                          | Описание                                                                                                                                                                                                                                      |
|---------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Событие \_ \_ \_ Загрузка типа трассировки**(значение типа события — 10)<br/>     | Событие загрузки изображения. Создается при загрузке библиотеки DLL или исполняемого файла. Поставщик создает только одно событие при первой загрузке заданной библиотеки DLL. Класс [**MOF \_ загрузки образа**](image-load.md) определяет данные события для этого события.      |
| **Событие \_ \_ \_ Конец типа трассировки**(значение типа события — 2)<br/>       | Событие выгрузки изображения. Создается при выгрузке библиотеки DLL или исполняемого файла. Поставщик создает только одно событие для последней выгрузки данной библиотеки DLL. Класс [**MOF \_ загрузки образа**](image-load.md) определяет данные события для этого события. |
| **Событие \_ \_Тип трассировки \_ \_ Start контроллера домена**(значение типа события — 3)<br/> | Событие начала сбора данных. Перечисляет все загруженные изображения в начале трассировки. Класс [**MOF \_ загрузки образа**](image-load.md) определяет данные события для этого события.                                                                  |
| **Событие \_ \_Тип трассировки \_ \_ конец контроллера домена**(значение типа события — 4)<br/>   | Событие окончания сбора данных. Перечисляет все загруженные изображения в конце трассировки. Класс [**MOF \_ загрузки образа**](image-load.md) определяет данные события для этого события.                                                                          |
