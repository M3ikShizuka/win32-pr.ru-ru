# <a name="d1181-enddraw-incorrect-state"></a>D1181: неправильное состояние EndDraw

EndDraw был вызван, но рендертаржет находится в недопустимом состоянии. Это может произойти из-за вызова EndDraw без соответствующего Бегиндрав.






 

## <a name="possible-causes"></a>Возможные причины

Приложение выполнило несколько вызовов EndDraw без вызова Бегиндрав в одном и том же целевом объекте. При использовании Direct2D приложения должны соответствовать каждому вызову EndDraw при предыдущем вызове Бегиндрав. Кроме того, это сообщение может появиться, если целевой объект находится в ошибочном состоянии и не может быть отображен.

## <a name="possible-fixes"></a>Возможные исправления

Убедитесь, что каждый вызов Бегиндрав соответствует последующей EndDraw. При отправке команд в целевой объект с помощью Flush или EndDraw \_ перед повторной попыткой использования целевого объекта убедитесь, что возвращенный результат имеет значение S OK.
