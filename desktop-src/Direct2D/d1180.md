# <a name="d1180-begindraw-incorrect-state"></a>D1180: неправильное состояние Бегиндрав

Бегиндрав вызывался несколько раз без соответствующего вызова EndDraw.






 

## <a name="possible-causes"></a>Возможные причины

Приложение выполнило несколько вызовов Бегиндрав без вызова EndDraw в одном и том же целевом объекте. При использовании Direct2D приложения должны соответствовать каждому вызову Бегиндрав с последующим EndDraw. Кроме того, это сообщение может появиться, если целевой объект находится в ошибочном состоянии, а вызов Бегиндрав не допускает отрисовку целевого объекта.

## <a name="possible-fixes"></a>Возможные исправления

Убедитесь, что каждый вызов Бегиндрав соответствует последующей EndDraw. При отправке команд в целевой объект с помощью Flush или EndDraw \_ перед повторной попыткой использования целевого объекта убедитесь, что возвращенный результат имеет значение S OK.
