# <a name="d1159-invalid-commandlist-topology"></a>D1159: недопустимая топология Коммандлист

Список команд не может ссылаться на эффекты, которые являются частью графов эффектов, использующих список команд.





 

## <a name="possible-causes"></a>Возможные причины

При заполнении списка команд приложение может включать вызов ID2D1DeviceContext::D Равимаже. ID2D1DeviceContext::D Равимаже можно использовать для рисования точечных рисунков, эффектов или списков команд. Как следствие, синтаксис ID2D1DeviceContext::D Равимаже может попытаться нарисовать список команд, если этот же список команд задан как целевой объект. Однако списки команд, ссылающиеся на себя, не являются допустимыми. Direct2D выявляет случаи, в которых это происходит, и сообщает об ошибке, и отладочный слой выводит это сообщение.

## <a name="possible-fixes"></a>Возможные исправления

При заполнении списка команд убедитесь, что контекст устройства не выполняет никаких операций, которые используют тот же список команд. Наиболее очевидным способом, в котором это может произойти, является вызов ID2D1DeviceContext::D Равимаже с тем же списком команд, который задан в качестве целевого. Однако есть и менее очевидные способы, в которых это может произойти: ID2D1DeviceContext::D Равимаже может ссылаться на граф эффектов, а один узел на графе эффектов может попытаться использовать список целевых команд. Перед использованием убедитесь, что списки команд заполнены правильно, и что вызовы ID2D1DeviceContext:: Сеттаржет относятся к предполагаемому целевому объекту.
