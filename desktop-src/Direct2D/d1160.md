# <a name="d1160-cycle-in-graph"></a>D1160: цикл в Graph

Предпринята попытка нарисовать граф изображения с циклом.






 

## <a name="possible-causes"></a>Возможные причины

Direct2D позволяет приложениям создавать графы эффектов для обеспечения гибкости в силе поведения. Однако приложения не должны создавать диаграмму эффектов, которая содержит цикл. Например, рассмотрим диаграмму эффектов, которая содержит узлы A, B и C: недопустимо связывание A с B, затем B к C, а также C к A. Direct2D определит циклические диаграммы эффектов и сообщит об ошибке; уровень отладки выводит это сообщение.

## <a name="possible-fixes"></a>Возможные исправления

Чтобы избежать этого сообщения, не следует создавать графы эффектов, которые имеют цикл. Эффективно создайте графы эффектов, включая по крайней мере один узел, который выступает только в качестве выходных данных.
