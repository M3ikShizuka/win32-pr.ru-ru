# <a name="d1119-bitmap-bound-as-target"></a>D1119: Растровое изображение привязано как целевое

Не удалось выполнить операцию рисования, так как целевой точечный рисунок и исходный точечный рисунок совпадают.






 

## <a name="possible-causes"></a>Возможные причины

Вероятная причина этого сообщения заключается в том, что приложение попыталось выполнить операцию рисования, в которой один и тот же точечный рисунок использовался как для источника, так и для целевого объекта. В семантических целях это возможно для взаимодействующих текстур Direct3D с \_ ресурсами шейдера привязки \_ и \_ \_ целевым объектом рендеринга.

## <a name="possible-fixes"></a>Возможные исправления

Убедитесь, что приложение не пытается использовать его целевой точечный рисунок в качестве источника операции рисования, включая эффекты. В некоторых случаях это сообщение может означать, что приложение установило в качестве целевого объекта контекста устройства другое точечное изображение, чем планировалось.
