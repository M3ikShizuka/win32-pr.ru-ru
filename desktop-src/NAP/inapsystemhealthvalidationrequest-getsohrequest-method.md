# <a name="inapsystemhealthvalidationrequestgetsohrequest-method"></a>Метод Инапсистемхеалсвалидатионрекуест:: Жетсохрекуест

> [!Note]  
> Платформа защиты доступа к сети недоступна начиная с Windows 10

 

Метод **инапсистемхеалсвалидатионрекуест:: жетсохрекуест** позволяет средствам проверки работоспособности системы получать и проверять сведения [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) , отправленные их аналогами агента работоспособности системы (SHA) на клиенте.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetSoHRequest(
  [out] SoHRequest **sohRequest,
  [out] BOOL       *napSystemGenerated
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*сохрекуест* \[ заполняет\]
</dt> <dd>

Указатель на указатель на структуру [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) .

</dd> <dt>

*напсистемженератед* \[ заполняет\]
</dt> <dd>

**Логическое** значение, равное **true** , если состояние работоспособности было создано напажент от имени SHA и **false** в противном случае. Он используется в основном для обозначения сбоя SHA в SHV.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Также могут возвращаться другие коды ошибок, относящиеся к COM.



| Код возврата                                                                                     | Описание                                                        |
|-------------------------------------------------------------------------------------------------|--------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> </dl>           | Операция успешно завершена.<br/>                                    |
| <dl> <dt>**Д \_ ACCESSDENIED**</dt> </dl> | Ошибка разрешений, доступ запрещен.<br/>                       |
| <dl> <dt>**Д \_ OUTOFMEMORY**</dt> </dl>  | Не удалось выполнить операцию из – за пределов системных ресурсов.<br/> |



 

## <a name="remarks"></a>Комментарии

Параметр *сохрекуест* может возвращать **значение NULL** , если клиент не отправлял [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) в SHV. В этом сценарии SHV может заполнить **сохреспонсе** кодом ошибки [**NAP \_ E \_ Missing No \_ SoH**](nap-error-constants.md).

Если параметр *напсистемженератед* имеет **значение true**, формат *сохрекуест* выглядит следующим образом:

-   [**сохаттрибутетипесистемхеалсид**](sohattributetype-enum.md)= <id>
-   [**сохаттрибутетипефаилурекатегори**](sohattributetype-enum.md) =  [ **фаилурекатегориклиенткомпонент**](/windows/win32/api/naptypes/ne-naptypes-failurecategory)
-   [**сохаттрибутетипирроркодес**](sohattributetype-enum.md)  =  [ **<SHA-Failure-Error-code>**](nap-error-constants.md)
