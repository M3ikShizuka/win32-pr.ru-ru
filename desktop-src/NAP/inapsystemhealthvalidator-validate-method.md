# <a name="inapsystemhealthvalidatorvalidate-method"></a>Метод Инапсистемхеалсвалидатор:: Validate

> [!Note]  
> Платформа защиты доступа к сети недоступна начиная с Windows 10

 

Метод **инапсистемхеалсвалидатор:: Validate** определяется разработчиком SHV и вызывается системой защиты доступа к сети для проверки [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) , полученного от клиента.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Validate(
  [in] INapSystemHealthValidationRequest *request,
  [in] UINT32                            hintTimeOutInMsec,
  [in] INapServerCallback                *callback
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*запрос* \[ окне\]
</dt> <dd>

COM-указатель на объект [**инапсистемхеалсвалидатионрекуест**](inapsystemhealthvalidationrequest.md) , определяющий объект запроса проверки.

</dd> <dt>

*хинттимеаутинмсек* \[ окне\]
</dt> <dd>

Длительность периода ожидания связи в миллисекундах. Средство проверки работоспособности системы (SHV) должно отвечать в течение этого периода времени; в противном случае ответ отбрасывается.

> [!Note]  
> Время ожидания по умолчанию для всех SHV составляет 2000 миллисекунд. Если используется значение, отличное от значения по умолчанию, время ожидания для всех зарегистрированных SHV изменится.

 

</dd> <dt>

*обратный вызов* \[ окне\]
</dt> <dd>

Указатель на объект обратного вызова [**инапсерверкаллбакк**](inapservercallback.md). Этот указатель обратного вызова используется SHV при возврате **E \_ в ожидании** от вызова **инапсистемхеалсвалидатор:: Validate**. Используется для асинхронной проверки. Средства SHV должны реагировать в течение *хинттимеаутинмсек* времени, иначе ответ будет удален.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если возвращается любой другой код ошибки, система предполагает, что произошел сбой Серверкомпонент и соответствующее сопоставление выполняется для передачи или сбоя.



| Код возврата                                                                                                | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
|------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>                       | Указывает, что средство проверки установило Сохреспонсе для объекта Request.<br/>                                                                                                                                                                                                                                                                                                                                                                                                        |
| <dl> <dt>**\_в ожидании**</dt> </dl>                  | Указывает, что OnComplete () будет вызываться в отдельном потоке.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                    |
| <dl> <dt>**\_сервер RPC \_ S \_ недоступен**</dt> </dl> | Указывает, что процесс средства проверки работоспособности системы (SHV) завершается без Напсервер фактической освобождения ссылки на него. Напсервер попытается повторно создать новую ссылку на SHV и снова выполнит вызов Validate. Если при создании объекта или повторного выполнения проверки произошел сбой, SHV удаляется из списка загруженных SHV. Единственным способом перезагрузки этого средства SHV является отмена регистрации и повторной регистрации SHV, а также перезагрузка Напсервер<br/> |



 

## <a name="remarks"></a>Remarks

Чтобы обеспечить обнаружение вторжений, SHV будет запрашивать проверку клиентского компьютера независимо от того, отправляет ли клиент [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) , предназначенный для SHV.

SHV должен выполнить следующие действия:

-   Получение [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) из *запроса* путем вызова [**запроса. Жетсохрекуест ()**](inapsystemhealthvalidationrequest-getsohrequest-method.md).
-   Если пакет [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) имеет значение NULL:
    -   Если SHV является системой обнаружения вторжений, заполните пакет [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) соответствующим [**кодом ошибки NAP**](nap-error-constants.md) , чтобы убедиться, что клиентский компьютер является вредоносным.
    -   Все остальные SHV должны заполнить пакет [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) кодом ошибки [**NAP \_ E \_ Missing No \_ SoH**](nap-error-constants.md).
-   Если *напсистемженератед* имеет **значение true** из вызова [**запроса. Жетсохрекуест ()**](inapsystemhealthvalidationrequest-getsohrequest-method.md), SHV должен ждать пакет [**SoH**](/windows/win32/api/naptypes/ns-naptypes-soh) со следующим 3 ТЛВС: [**сохаттрибутетипесистемхеалсид**](sohattributetype-enum.md), **сохаттрибутетипефаилурекатегори**, **sohAttributeTypeErrorCodes**. Этот **сохрекуест** создается напажент от имени агента работоспособности системы (SHA), так как произошла ошибка при получении пакета запроса от SHA.
-   Проверьте пакет [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) .
    -   Если [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) имеет неправильный формат, создайте пакет **сохреспонсе** с кодом ошибки [**\_ \_ Недопустимый \_ пакет NAP E**](nap-error-constants.md).
    -   Если SHV использует только кэшированные данные для проверки пакета [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh) (т. е. операций ввода-вывода не выполняется), то он может создать **сохреспонсе**, установить его для объекта в *запросе* и вернуть значение **S \_ ОК**.
    -   Если SHV выполняет операции ввода-вывода для взаимодействия с внутренними серверами для проверки работоспособности клиента, он должен поставить в очередь операции ввода-вывода и вернуть эту функцию в **\_ ожидании**. В этом случае SHV должен вызвать [**обратный вызов. OnComplete ()**](inapservercallback-oncomplete-method.md) в отдельном потоке в течение периода ожидания, *хинттимеаутинмсек*. В противном случае ответ SHV будет удален.
-   Не возвращайте никаких других ошибок, кроме перечисленных выше. Если SHV возвращает любой другой код ошибки (например, часть системной ошибки), пакет будет удален.

SHV должен возвращать TLV **сохаттрибутетипекомплианцересулткодес** или **Сохаттрибутетипефаилурекатегори** в своем [**сохрекуест**](/windows/win32/api/naptypes/ns-naptypes-soh).

-   [**TLV сохаттрибутетипекомплианцересулткодес**](sohattributetype-enum.md). Если SHV может проверить работоспособность клиента (т. е. работоспособность или неработоспособность), возвращается этот TLV.
-   [**TLV сохаттрибутетипефаилурекатегори**](sohattributetype-enum.md). Если на клиенте или сервере существовали какие либо компоненты или сбои связи, они должны быть обозначены этим TLV. Этот TLV будет сопоставлено с работоспособным или неработоспособным в зависимости от конфигурации SHV. Дополнительные сведения см. в разделе интерфейс [**инапсерверманажемент**](inapservermanagement.md) и структура [**фаилурекатегоримаппинг**](/windows/win32/api/naptypes/ns-naptypes-failurecategorymapping) .

SHV не должен хранить ссылки на *запрос* или *обратный вызов* после завершения вызова асинкронаус.
