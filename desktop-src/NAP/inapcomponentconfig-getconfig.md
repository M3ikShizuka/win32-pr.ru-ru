# <a name="inapcomponentconfiggetconfig-method"></a>Метод Инапкомпонентконфиг:: config

> [!Note]  
> Платформа защиты доступа к сети недоступна начиная с Windows 10

 

Метод **config** извлекает конфигурацию компонента средства проверки работоспособности системы (SHV).

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetConfig(
  [out] UINT16 *bCount,
  [out] BYTE   **data
) const;
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*бкаунт* \[ заполняет\]
</dt> <dd>

Размер большого двоичного объекта конфигурации *данных* (в байтах).

</dd> <dt>

*данные* \[ заполняет\]
</dt> <dd>

Указатель на адрес данных конфигурации компонента SHV.

> [!Note]  
> Данные конфигурации, экспортированные с компьютера x86 с помощью метода **config** , можно импортировать на компьютер x64 с помощью метода [**сетконфиг**](inapcomponentconfig-setconfig.md) и наоборот. Поэтому данные конфигурации должны быть в независимом от архитектуры формате, таком как XML. Использование XML вместо байтового потока упрощает использование данных конфигурации в различных архитектурах. Элементы XML, используемые в данных конфигурации, определяются разработчиком.

 

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает один из следующих кодов ошибок в зависимости от результата этой операции.



| Код возврата                                                                                     | Описание                                                        |
|-------------------------------------------------------------------------------------------------|--------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> </dl>           | Операция прошла успешно.<br/>                            |
| <dl> <dt>**Д \_ ACCESSDENIED**</dt> </dl> | Ошибка разрешений, доступ запрещен.<br/>                       |
| <dl> <dt>**Д \_ OUTOFMEMORY**</dt> </dl>  | Не удалось выполнить операцию из – за пределов системных ресурсов.<br/> |



 

## <a name="remarks"></a>Remarks

Параметр данных должен выделяться вызываемым объектом (реализующим компонентом) с помощью функции **CoTaskMemAlloc** и освобождены вызывающей стороной с помощью **CoTaskMemFree**.
