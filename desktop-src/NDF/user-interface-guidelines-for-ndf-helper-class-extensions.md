# <a name="ui-guidelines-for-ndf-helper-class-extensions"></a>Рекомендации по пользовательскому интерфейсу для расширений классов модуля поддержки NDF

При создании вспомогательного класса необходимо создать текст, чтобы помочь пользователю понять причину инцидента и возможные действия по восстановлению, которые можно предпринять.

## <a name="root-cause-and-repair"></a>Основная причина и восстановление

При возникновении инцидента появляется диалоговое окно, сообщающее пользователю о том, что произошло. Созданный текст будет отображаться в двух отдельных областях.

-   **Основная причина**: краткое описание причины проблемы. Содержит сведения, помогающие администратору или расширенному пользователю устранить проблему.
-   **Исправление**: раскрывает основную причину, чтобы предоставить дополнительные сведения о действиях, которые может выполнить пользователь, не прибегая к слишком длине.

Каждая основная причина или восстановление должны иметь заголовок и описание. Весь текст перед первым \\ символом "n" будет использован для заголовка. В описании будет использоваться весь текст, следующий за первым \\ символом "n" (включая все последующие разрывы строк).

## <a name="title-guidelines"></a>Рекомендации по названиям

Название основной причины или восстановления должно соответствовать следующим рекомендациям:

-   Длина должна составлять 128 символов или меньше. (рекомендуется не более 40 символов.)
-   Не должно заканчиваться точкой.

## <a name="description-guidelines"></a>Рекомендации по описанию

Описание основной причины или восстановления должно соответствовать следующим рекомендациям:

-   Длина должна составлять 512 символов или меньше.
-   Каждое предложение должно заканчиваться точкой.
-   Символ " \\ n" может использоваться для создания разрыва строки в описании.
