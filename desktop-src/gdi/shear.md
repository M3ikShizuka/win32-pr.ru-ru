# <a name="shear"></a>Сдвиг

Некоторые приложения предоставляют функции, которые заменяют объекты, отображаемые в клиентской области. Приложения, использующие возможности сдвига, используют функцию [**сетворлдтрансформ**](/windows/desktop/api/Wingdi/nf-wingdi-setworldtransform) для установки соответствующих значений в мировом пространстве для преобразования страничного пространства. Эта функция получает указатель на структуру [**XForm**](/windows/win32/api/wingdi/ns-wingdi-xform) , содержащую соответствующие значения. Члены eM12 и eM21 XFORM указывают горизонтальные и вертикальные константы пропортионалити соответственно.

Существует два компонента *преобразования «сдвиг*». Первый изменяет вертикальные линии в объекте; Вторая изменяет горизонтальные линии. На следующем рисунке показан прямоугольник с 20 по 20, который обрезается по горизонтали при копировании из мира в пространство страницы.

![Иллюстрация, показывающая прямоугольник в мировом пространстве и трапезиод в пространстве страницы](images/cstrn-13.png)

Горизонтальное наклонное значение может быть представлено следующим алгоритмом:

``` syntax
x' = x + (Sx * y) 
```

где x — исходная координата x, SX — константа пропортионалити, а x — результат преобразования «сдвиг».

Вертикальный сдвиг может быть представлен следующим алгоритмом:

``` syntax
y' = y + (Sy * x) 
```

где y — исходная координата y, SY — константа пропортионалити, а y — результат преобразования «сдвиг».

Преобразования «горизонтальное и вертикальное наклонное» можно объединить в одну операцию, используя матрицу размером 2 на 2.

``` syntax
|x' y'| == |x y| * |  1   Sx| 
                   | Sy    1| 
```

Матрица размером 2 на 2, которая вызвала сдвиг, содержит следующие значения:

``` syntax
|1    1| 
|0    1| 
```
