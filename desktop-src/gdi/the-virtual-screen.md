# <a name="the-virtual-screen"></a>Виртуальный экран

Ограничивающим прямоугольником всех мониторов является *виртуальный экран*. Рабочий стол охватывает виртуальный экран, а не один монитор. На следующем рисунке показано возможное расположение трех мониторов.

![Иллюстрация, демонстрирующая три поля, представляющие мониторы, расположенные внутри поля, представляющего виртуальный экран](images/multimon-1.png)

*Основной монитор* содержит источник (0, 0). Это относится к совместимости с существующими приложениями, которые предполагают наличие монитора с источником. Однако основной монитор не обязательно должен находиться в левом верхнем углу виртуального экрана. На рис. 1 он находится ближе к центру. Если основной монитор находится не в левом верхнем углу виртуального экрана, части виртуального экрана имеют отрицательные координаты. Так как расположение мониторов устанавливается пользователем, все приложения должны быть спроектированы для работы с отрицательными координатами. Дополнительные сведения см. в разделе [рекомендации по нескольким мониторам для старых программ](multiple-monitor-considerations-for-older-programs.md).

Координаты виртуального экрана представлены 16-битным значением со знаком, так как из 16-разрядных значений содержатся в нескольких существующих сообщениях. Таким образом, границы виртуального экрана:

``` syntax
SHORT_MIN    <= rcVirtualScreen.left   <= SHORT_MAX - 1
SHORT_MIN +1 <= rcVirtualScreen.right  <= SHORT_MAX
SHORT_MIN    <= rcVirtualScreen.top    <= SHORT_MAX - 1
SHORT_MIN +1 <= rcVirtualScreen.bottom <= SHORT_MAX
```
