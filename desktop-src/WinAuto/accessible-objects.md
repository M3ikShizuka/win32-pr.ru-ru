# <a name="accessible-objects"></a>Доступные объекты

С помощью Microsoft Active Accessibility элементы пользовательского интерфейса предоставляются клиентам как объекты модели COM. Эти *объекты со специальными возможностями* сохраняют части информации, называемые *свойствами*, которые описывают имя объекта, расположение экрана и другие сведения, необходимые для вспомогательных функций. Объекты со специальными возможностями также предоставляют *методы* , которые вызываются клиентами для выполнения некоторых действий объектом. Доступные объекты, имеющие простые элементы, связанные с ними, также называются *родителями* или *контейнерами*.

Доступные объекты реализуются с помощью интерфейса [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) на основе COM Microsoft Active Accessibility. Методы и свойства **IAccessible** позволяют клиентским приложениям получать сведения о элементах пользовательского интерфейса, необходимых пользователям. Например, метод [**IAccessible:: Get \_ аккпарент**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accparent) возвращает указатель интерфейса на родительский объект доступного объекта, а метод [**IAccessible:: аккнавигате**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accnavigate) предоставляет клиентам возможность получать сведения о других объектах в контейнере.

Дополнительные сведения о взаимосвязи между доступными объектами и простыми элементами см. в разделе [простые элементы](simple-elements.md).
