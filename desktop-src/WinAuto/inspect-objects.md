# <a name="accessibility-tools---inspect"></a>Средства специальных возможностей — проверка

> [!Important]
> **Проверка** является устаревшим инструментом. вместо этого рекомендуется использовать [Аналитика специальных возможностей](https://accessibilityinsights.io/) .

**проверка** (Inspect.exe) — это средство на основе Windows, которое позволяет выбрать любой элемент пользовательского интерфейса и просмотреть данные о специальных возможностях элемента. Вы можете просматривать свойства и шаблоны элементов управления модели автоматизации пользовательского интерфейса Майкрософт, а также свойства Microsoft Active Accessibility (MSAA). **Проверка** также позволяет тестировать навигационную структуру элементов автоматизации в дереве модели автоматизации пользовательского интерфейса, а также объекты, доступные в иерархии Microsoft Active Accessibility.

## <a name="requirements"></a>Требования

Для изучения автоматизации пользовательского интерфейса в системе должна присутствовать модель автоматизации пользовательского интерфейса. Дополнительные сведения см. в разделе "требования" [модели автоматизации пользовательского интерфейса](entry-uiauto-win32.md).

**проверка** устанавливается как часть общего набора средств в пакете sdk для Windows Software Development Kit, он не распространяется отдельно для загрузки. Windows SDK включает все средства, связанные со специальными возможностями, описанные в этом разделе.

[скачайте Windows SDK](https://developer.microsoft.com/en-us/windows/downloads/windows-10-sdk/).

> [!NOTE]
> более старые версии Windows SDK см. в [Windows SDK и в архиве эмулятора](https://developer.microsoft.com/en-us/windows/downloads/sdk-archive/).

**Inspect.exe** находится в \\ папке bin \\ < *версии* > \\ < > пути установки пакета SDK (обычно запускать от имени администратора не требуется).

## <a name="the-inspect-window"></a>Окно проверки

Окно **Проверка** состоит из нескольких основных частей:

- Заголовок окна. Отображает дескриптор окна **проверки** (HWND).
- Строка меню. Предоставляет доступ к функциям **проверки** .
- Панель инструментов. Предоставляет доступ к функциям **проверки** .
- Представление в виде дерева. Представляет иерархическую структуру элементов пользовательского интерфейса в виде древовидного представления, которое можно использовать для перемещения между элементами.
- Представление данных. Отображает все предоставленные свойства специальных возможностей для выбранного элемента пользовательского интерфейса.

Команды, доступные в строке меню, также доступны на панели инструментов. На следующем изображении показано средство **Inspect**, запрашивающее свойства модели автоматизации пользовательского интерфейса для элемента меню **Правка** в Блокноте.

## <a name="using-inspect"></a>Использование проверки

При запуске **проверки** в представлении в **виде дерева** отображается расположение выбранного в данный момент элемента пользовательского интерфейса в иерархии элементов, а в представлении **данных** отображаются сведения о свойствах для выбранного элемента пользовательского интерфейса. Вы можете перемещаться по пользовательскому интерфейсу, чтобы просматривать сведения о специальных возможностях каждого элемента в пользовательском интерфейсе. По умолчанию **Проверка** отслеживает клавиатуру или фокус мыши. При изменении фокуса представление **данных** обновляется со сведениями о свойствах элемента с фокусом.

Для перехода между элементами пользовательского интерфейса можно использовать любой из следующих элементов:

- Мышь.
- клавиатура;
- Элемент управления "дерево-представление" в представлении в **виде дерева** .
- Параметры навигации в меню **навигации** .
- Параметры навигации на панели инструментов.

Последние три параметра позволяют перемещаться по древовидной иерархии пользовательского интерфейса. Структура этого дерева может немного отличаться от модели автоматизации пользовательского интерфейса и режима Microsoft Active Accessibility.

## <a name="verifying-accessibility-property-information"></a>Проверка сведений о свойствах специальных возможностей

В представлении **данных** отображаются сведения о свойствах выбранного в данный момент элемента пользовательского интерфейса. Вы можете настроить **проверку** , чтобы отобразить сведения обо всех свойствах специальных возможностей или подмножестве этих свойств. Кроме того, можно указать другие параметры просмотра, например, должна ли **Проверка** отображаться поверх других пользовательских интерфейсов или следует ли **проверять** выделение ограничивающего прямоугольника вокруг выбранного элемента. После настройки **проверки** для работы в нужном виде можно начать переходы между элементами пользовательского интерфейса и просмотром сведений о свойствах. **Проверка** сохраняет параметры конфигурации при закрытии и использует их для инициализации следующего сеанса **проверки** .

### <a name="configure-property-settings"></a>настройка Параметры свойств

1. в меню **параметры** выберите **Параметры...** или на панели инструментов выберите пункт **показывать Параметрыное диалоговое окно** .
2. В списке **Отображать в главном окне** выберите свойства, которые должны отображаться в представлении **данных** **проверки**.
3. В списке **всплывающая подсказка отображать сведения** выберите свойства, которые должны отображаться во всплывающей подсказке.
4. Чтобы просмотреть свойства, которые могут не поддерживаться элементом пользовательского интерфейса, установите флажок **отобразить неподдерживаемые свойства** .
5. Нажмите кнопку **ОК**.

## <a name="verifying-accessibility-navigation"></a>Проверка навигации по специальным возможностям

после выбора элемента пользовательского интерфейса с помощью **проверки** можно проверить, что элемент предоставляет верную Windowsную навигацию по автоматизации для продуктов вспомогательной технологии.

### <a name="verify-accessibility-navigation"></a>Проверка навигации по специальным возможностям

1. Откройте окно **Проверка** и приложение, которое требуется протестировать.
2. Выберите элемент пользовательского интерфейса, из которого необходимо начать навигацию.
3. Убедитесь, что в представлении **данных** элемент предоставляет правильные свойства, связанные с навигацией.
4. Используйте представление в **виде дерева** , меню **навигации** или кнопок навигации на панели инструментов для навигации по пользовательскому интерфейсу и проверки того, что каждый элемент предоставляет правильные свойства, связанные с навигацией.
    > [!Note]  
    > Параметры меню **навигации** и кнопки панели инструментов навигации изменяются в зависимости от расположения выбранного элемента в дереве.

## <a name="interacting-with-ui-elements"></a>Взаимодействие с элементами пользовательского интерфейса

Windows Автоматизация предоставляет методы, которые позволяют продуктам с поддержкой специальных возможностей взаимодействовать с элементом пользовательского интерфейса, как если бы использовались мышь или клавиатура (например, для нажатия кнопки). меню **проверка** действия позволяет тестерам вызывать методы автоматизации Windows для элемента (например, **invoke. invoke** вызывает метод [**иуиаутоматионинвокепаттерн:: invoke**](/windows/desktop/api/UIAutomationClient/nf-uiautomationclient-iuiautomationinvokepattern-invoke) ).

### <a name="interact-with-ui-elements"></a>Взаимодействие с элементами пользовательского интерфейса

1. Откройте окно **Проверка** и приложение, которое требуется протестировать.
2. Выберите элемент пользовательского интерфейса, с которым необходимо взаимодействовать.
3. в меню **действие** или на панели инструментов выберите действие, соответствующее Windows методу автоматизации, который требуется вызвать.

Меню « **действие** » содержит элементы « **Обновить** » и « **фокус** », а также другие элементы, которые изменяются в зависимости от того, выбран ли режим модели автоматизации пользовательского интерфейса или режим MSAA. В режиме автоматизации пользовательского интерфейса другие элементы отражать шаблоны элементов управления, поддерживаемые текущим выбранным элементом пользовательского интерфейса. В режиме MSAA остальные элементы всегда состоят из следующих элементов:

| Действие                | Описание                                                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------------------------------------|
| Обновить               | Обновляет пользовательский интерфейс. Доступно в режимах MSAA и UI Automation.                                               |
| Действие по умолчанию        | Выполняет действие по умолчанию для элемента.                                                                          |
| Фокус                 | Устанавливает фокус на элемент. Доступно в режимах MSAA и UI Automation.                                                  |
| Выберите пункт                | Выбирает элемент.                                                                                                  |
| Расширить область выбора      | Расширяет выбор элементов для включения всех элементов между первым выбранным элементом и текущим элементом. |
| Добавить к выделенному      | Выбирает текущий элемент (например, элемент списка).                                                                    |
| Удалить из выделенного фрагмента | Удаляет текущий элемент из выделенного фрагмента.                                                                       |
| сетакквалуе           | Устанавливает значение Microsoft Active Accessibility элемента в указанную строку.                                 |
| Дочерний элемент         | Выполняет переход к дочернему элементу элемента, на который в данный момент находится фокус.                                                       |
| HitTest в курсоре     | Выполняет переход к дочернему элементу элемента, заданного курсором мыши.                                                      |
| HitTest...            | Открывает диалоговое окно **HitTest** .                                                                                     |
