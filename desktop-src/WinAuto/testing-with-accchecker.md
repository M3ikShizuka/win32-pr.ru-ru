# <a name="testing-with-accchecker"></a>Тестирование с помощью Аккчеккер

В следующем сценарии описаны действия, которые обычно выполняются при тестировании с помощью Аккчеккер.

> [!Note]  
> Когда выполняются подпрограммы проверки Аккчеккер, взаимодействие пользователей с ведущим компьютером может мешать некоторым подпрограммам. Рекомендуется не выполнять никаких действий пользователя, пока Аккчеккер не сообщит пользователю о том, что все задачи завершены.

 

1.  Выполните проверки Аккчеккер для определенного целевого приложения или элемента управления. Дополнительные сведения см. на [вкладке проверки](the-accchecker-graphical-user-interface.md).
    > [!Note]  
    > Аккчеккер не исследует все перестановки пользовательского интерфейса в приложении. Элементы, скрытые в таких элементах управления, как окна, панели, вкладки и меню, должны предоставляться вручную.

     

2.  Изучите журнал ошибок Аккчеккер и оцените или рассмотрите результаты. Устраните тривиальные проблемы и заносить серьезные ошибки в журнал соответствующим образом.
3.  Создание файла подавления для ошибок и ошибок, которые можно пропустить до регрессионного тестирования.
4.  Повторно запустите проверки Аккчеккер с исправлением файла подавления и начальными ошибками. Это позволяет создать моментальный снимок проблем в текущей реализации Microsoft Active Accessibility и установить базовый план для регрессионного тестирования.
5.  Интегрируйте интерфейсы API Аккчеккер и файл подавления в среду автоматизированного тестирования для регрессионного тестирования на предмет ошибок, регистрируемых при первоначальной проверке Аккчеккер.
    > [!Note]  
    > При исправлении ошибок файл подавления следует изменять по мере необходимости.

     

6.  Убедитесь, что в приложение или элемент управления не появились регрессии или новые ошибки специальных возможностей.
