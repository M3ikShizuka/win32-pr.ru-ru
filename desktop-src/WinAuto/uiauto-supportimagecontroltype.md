# <a name="image-control-type"></a>Тип элемента управления Image

В этом разделе содержатся сведения о поддержке автоматизации пользовательского интерфейса Майкрософт для типа элемента управления **Image** .

Элементы управления "изображение", используемые в качестве значков, информационных изображений и диаграмм, будут поддерживать тип элемента управления **Image** . Элементы управления, используемые как изображения фона или водяного знака, не поддерживают тип элемента управления **Image** .

В следующих разделах определяется необходимая древовидная структура модели автоматизации пользовательского интерфейса, свойства, шаблоны элементов управления и события для типа элемента управления **Image** . Требования к автоматизации пользовательского интерфейса применяются ко всем элементам управления "изображение", в которых платформа или платформа пользовательского интерфейса интегрирует поддержку автоматизации пользовательского интерфейса для типов элементов управления и шаблонов элементов управления.

В этом разделе содержатся следующие подразделы.

- [Типичная древовидная структура](#typical-tree-structure)
- [Соответствующие свойства](#relevant-properties)
- [Обязательные шаблоны элементов управления](#required-control-patterns)
- [Обязательные события](#required-events)
- [Связанные темы](#related-topics)

## <a name="remarks"></a>Комментарии

Консорциум W3C (W3C) определяет декоративный рисунок, который не добавляет сведения в содержимое страницы. Дополнительные сведения см. в разделе консорциума W3C по [декоративным изображениям](https://www.w3.org/WAI/tutorials/images/decorative/).

В отношении модели автоматизации пользовательского интерфейса:

- Если изображение является чисто декоративным, не является интерактивным и не передает никакой информации, изображение:
  - Может или не находиться в дереве UIA
  - Может или не находиться в представлении UIA RAW
  - Не должно находиться в представлении элемента управления UIA
  - Не должно находиться в представлении содержимого
  - Может иметь или не иметь имени
- Если изображение передает информацию, но есть четкий связанный текст, который предоставляет те же сведения (например, кнопку воспроизведения, содержащую направленный влево треугольник вместе с текстом «Play»), изображение считается декоративным и изображением:
  - Должен находиться в представлении RAW
  - Должен быть в представлении элемента управления
  - Не должно находиться в представлении содержимого
  - Свойство Name может иметь значение или не иметь значения
  - Текст, который также передает значение изображения, должен находиться в представлении содержимого
- Если изображение является информативным и передает сведения, не предоставленные каким-либо связанным текстом, изображение:
  - Должен находиться в представлении RAW
  - Должен быть в представлении элемента управления
  - Должен находиться в представлении содержимого
  - Должно иметь значение имени, описывающее изображение и его значение
