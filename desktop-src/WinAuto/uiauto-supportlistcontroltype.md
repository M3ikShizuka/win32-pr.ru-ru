# <a name="list-control-type"></a>Тип элемента управления List

В этом разделе содержатся сведения о поддержке автоматизации пользовательского интерфейса Майкрософт для типа элемента управления **List** .

Тип элемента управления " **список** " предоставляет способ организации плоской группы или групп элементов и позволяет пользователю выбрать один или несколько этих элементов. Тип элемента управления " **список** " имеет слабое ограничение на типы дочерних элементов, которые он может содержать. Это позволяет поставщикам автоматизации пользовательского интерфейса поддерживать знакомые элементы для контейнеров выбора.

В следующих разделах определяется необходимая древовидная структура модели автоматизации пользовательского интерфейса, свойства, шаблоны элементов управления и события для типа элемента управления **List** . Требования к автоматизации пользовательского интерфейса применяются ко всем элементам управления "список", в которых платформа или платформа пользовательского интерфейса интегрирует поддержку автоматизации пользовательского интерфейса для типов элементов управления и шаблонов элементов управления.

В этом разделе содержатся следующие подразделы.

Представление для элемента управления, который реализует тип List (например, элемент управления "Список"), включает следующие элементы:

-   Ноль или более элементов в элементе управления "список" (элементы могут основываться на типах элементов управления [ListItem](uiauto-supportlistitemcontroltype.md) или [DataItem](uiauto-supportdataitemcontroltype.md) )
-   ноль или более элементов управления "Группа" в элементе управления "Список";
-   ноль, один или два элемента управления "Полоса прокрутки".

Представление содержимого элемента управления, который реализует тип List (например, элемент управления "Список"), включает следующие элементы:

-   Ноль или более элементов в элементе управления "список" (элементы могут основываться на типах элементов управления [ListItem](uiauto-supportlistitemcontroltype.md) или [DataItem](uiauto-supportdataitemcontroltype.md) )
-   ноль или более групп в элементе управления "Список".

Элемент управления "Список" не должен содержать элементы, которые имеют какую-либо иерархическую связь кроме общей группы. Если у элементов есть дочерние элементы в дереве модели автоматизации пользовательского интерфейса, то контейнер списка должен основываться на типе элемента управления " [дерево](uiauto-supporttreecontroltype.md) ".

Доступные для выбора элементы в элементе управления "список" будут доступны из потомков в дереве модели автоматизации пользовательского интерфейса элемента управления "список". Все элементы в элементе управления "Список" должны принадлежать одной группе выбора. Выбираемые элементы списка должны быть представлены как типы элементов управления [ListItem](uiauto-supportlistitemcontroltype.md) (а не [DataItem](uiauto-supportdataitemcontroltype.md)).
