# <a name="object-state-constants"></a>Константы состояния объектов

В этом разделе описаны константные значения, используемые для описания состояния объектов в пользовательском интерфейсе приложения. Константы состояния определяются в олеакк. h.

Объект связан с одним или несколькими значениями состояния в любое время. Следующие константы состояния объекта не используются: \_ высокий системный оповещение состояния, средний уровень оповещений системы состояний, низкая система состояний \_ \_ \_ \_ \_ \_ \_ \_ и \_ Система состояний \_ с плавающей запятой.

Клиенты получают состояние объекта путем вызова метода [**IAccessible:: Get \_ аккстате**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accstate), который возвращает целое число, являющееся сочетанием следующих битовых флагов. Клиенты вызывают [**жетстатетекст**](/windows/desktop/api/Oleacc/nf-oleacc-getstatetexta) со значением State, чтобы получить локализованную строку, описывающую состояние объекта.

При изменении состояния объекта серверы должны вызвать [**нотифивиневент**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) с помощью константы события [**\_ \_ STATECHANGE объекта события**](event-constants.md) . Однако объекты с невидимой системой состояния, состоянием "выбранные" системные \_ \_ \_ \_ \_ константы и состояния \_ объектов имеют собственные константы событий. Для этих объектов не используйте **\_ объект события \_ STATECHANGE**. Вместо этого используйте отдельную константу события.
