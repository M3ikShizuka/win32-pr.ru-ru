# <a name="side-by-side-assembly-sharing"></a>Совместное использование сборок параллельно

На следующем рисунке показано, как два приложения могут совместно использовать сборку с помощью традиционного метода совместного использования сборок. Проблема с традиционным совместно используемыми сборками возникает, когда приложение устанавливает версию сборки (обычно это DLL), которая не совместима с обратной совместимостью. Только что установленное приложение работает, но приложения, зависящие от предыдущей версии DLL, могут перестать работать.

![представление двух приложений, совместно использующих сборку](images/sxs1.png)

Изолированное приложение и параллельное решение сборки позволяют разным версиям одной и той же сборки Win32 выполняться одновременно в одной системе без конфликтов. Указав версию параллельной сборки, которую должно использовать приложение, разработчики могут гарантировать, что тестируемая конфигурация будет всегда дублироваться на компьютере пользователя. Параллельное совместное использование показано на следующем рисунке.

![реализация совместного использования сборок параллельными сборками](images/sxs2.png)
