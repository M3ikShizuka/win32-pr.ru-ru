# <a name="creating-side-by-side-windows-classes"></a>создание параллельных Windows классов

контексты приложений можно использовать для создания параллельных Windows классов. используя параллельные Windows классы, приложение может одновременно использовать разные версии класса в отдельном родительском окне. чтобы создать параллельно Windows класс, приложение должно активировать контекст активации перед вызовом [**CreateWindow**](/windows/win32/api/winuser/nf-winuser-createwindowa) для получения маркера для нового окна.

например, Windows общие элементы управления версии 5,82 и 6,0 имеют элемент управления Edit. Windows по умолчанию использует элемент управления Edit версии 5,82. Чтобы получить параллельное окно с элементом управления редактирования версии 6,0, перед вызовом [**CreateWindow**](/windows/win32/api/winuser/nf-winuser-createwindowa) в обычном режиме приложение может ссылаться на сборку, которая предоставляет именованные классы окон, а затем активировать контекст активации, который ссылается на элементы управления версии 6,0.
