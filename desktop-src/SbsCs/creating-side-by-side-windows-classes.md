# <a name="creating-side-by-side-windows-classes"></a>Создание параллельных классов Windows

Контексты приложений можно использовать для создания параллельных классов Windows. При использовании параллельных классов Windows приложение может одновременно иметь разные версии класса в отдельном родительском окне. Чтобы создать параллельный класс Windows, приложение должно активировать контекст активации перед вызовом [**CreateWindow**](/windows/win32/api/winuser/nf-winuser-createwindowa) для получения маркера для нового окна.

Например, стандартные элементы управления Windows версии 5,82 и 6,0 имеют элемент управления Edit. Windows по умолчанию использует элемент управления Edit версии 5,82. Чтобы получить параллельное окно с элементом управления редактирования версии 6,0, перед вызовом [**CreateWindow**](/windows/win32/api/winuser/nf-winuser-createwindowa) в обычном режиме приложение может ссылаться на сборку, которая предоставляет именованные классы окон, а затем активировать контекст активации, который ссылается на элементы управления версии 6,0.
