# <a name="creating-signed-files-and-catalogs"></a>Создание подписанных файлов и каталогов

Чтобы подписать файл и создать каталог для него, сначала необходимо выполнить процесс подписывания файлов, сертификата и открытого ключа.

**Подписание файла и создание каталога**

1.  Чтобы извлечь [*маркер открытого ключа*](p-sbscs-gly.md) из файла сертификата, используйте [Pktextract.exe](pktextract-exe.md) . Файл сертификата должен присутствовать в том же каталоге, что и программа.
2.  Используйте значение токена открытого ключа, чтобы обновить атрибут **PublicKeyToken** элемента **assemblyIdentity** в файле манифеста.
3.  Используйте [MT.exe](mt-exe.md) для создания хэшей файлов, содержащихся в манифесте сборки, и для создания файла описания каталога (CDF-файлов).
4.  Чтобы создать каталог безопасности для сборки, используйте Makecat.exe с созданным CDF. Это средство входит в интерфейс CryptoAPI.
5.  Используйте программу [SignTool](/windows/desktop/SecCrypto/signtool) для подписывания каталога, созданного с помощью сертификата, используемого на шаге 1. . CDF из шагов 3 и 4 можно удалить после создания каталога.

См. также [пример подписывания сборки](assembly-signing-example.md).
