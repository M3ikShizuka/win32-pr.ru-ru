# <a name="jetseek-function"></a>Функция Жетсик


_**Применимо к:** Windows | Windows Server_

## <a name="jetseek-function"></a>Функция Жетсик

Функция **жетсик** эффективно позиционирует курсор на запись индекса, которая соответствует условиям поиска, заданным в ключе поиска в этом курсоре, и заданному неравенству. Ключ поиска должен быть ранее создан с помощью [жетмакекэй](./jetmakekey-function.md).

```cpp
    JET_ERR JET_API JetSeek(
      __in          JET_SESID sesid,
      __in          JET_TABLEID tableid,
      __in          JET_GRBIT grbit
    );
```

### <a name="parameters"></a>Параметры

*сесид*

Сеанс, используемый для этого вызова.

*TableID*

Курсор, используемый для этого вызова.

*грбит*

Группа битов, содержащая параметры, используемые для этого вызова. *Грбит* должен быть ненулевым и должен включать одно или несколько значений, перечисленных в следующей таблице.

При успешном выполнении курсор будет размещен в записи индекса, соответствующей условиям поиска. Если запись подготовлена к обновлению, это обновление будет отменено. Если действует диапазон индексов, то диапазон индексов будет отменен. Если для курсора был создан ключ поиска, то этот ключ поиска будет удален. Изменение состояния базы данных не выполняется. Если несколько записей индекса имеют одинаковое значение, всегда выбирается запись, ближайшая к началу индекса.

В случае сбоя позицию курсора останется неизменной, если не было возвращено JET_errRecordNotFound. В этом случае курсор будет размещается в том месте, где элемент индекса, удовлетворяющий условиям поиска, заданным в поисковом ключе в этом курсоре, и заданное неравенство было бы. Курсор может быть перемещен относительно этой позиции, но по-прежнему не находится в допустимой записи индекса. Если запись подготовлена к обновлению, это обновление будет отменено. Если действует диапазон индексов, то диапазон индексов будет отменен. Если для курсора был создан ключ поиска, то этот ключ поиска будет удален. Изменение состояния базы данных не выполняется.
