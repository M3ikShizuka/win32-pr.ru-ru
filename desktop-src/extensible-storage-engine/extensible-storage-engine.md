# <a name="extensible-storage-engine"></a>Расширяемая подсистема хранилища


_**Применимо к:** Windows | Windows Server_

## <a name="extensible-storage-engine"></a>Расширяемая подсистема хранилища

Расширяемая подсистема хранения (ESE) — это расширенная технология для хранения с индексированными и последовательными методами доступа (ISAM). ESE позволяет приложениям сохранять и извлекать данные из таблиц с помощью индексированной или последовательной навигации по курсору. Он поддерживает денормализованные схемы, включая широкие таблицы с множеством разреженных столбцов, Многозначные столбцы и разреженные и насыщенные индексы. Это позволяет приложениям работать с целостным состоянием данных, используя обновление и получение данных транзакций. Механизм восстановления после сбоя обеспечивается таким образом, чтобы согласованность данных поддерживалась даже в случае сбоя системы. Он предоставляет транзакции ACID (атомарные изолированные устойчивые) транзакций по данным и схемам с помощью журнала упреждающего чтения и модели изоляции моментальных снимков. Транзакции в ESE имеют высокую степень параллельности, что позволяет использовать ESE для серверных приложений. Он кэширует данные, чтобы обеспечить максимальную производительность доступа к данным. Кроме того, это упрощенный, что делает его полезным для приложений, которые служат дополнительными ролями.

ESE предназначен для использования в приложениях, требующих быстрого и/или светло структурированного хранилища данных, когда доступ необработанных файлов или реестр не поддерживает требования к индексированию или размеру данных приложения.

Он используется приложениями, которые никогда не хранят более 1 мегабайта данных и использовались в приложениях с базами данных в экстремальных ситуациях, превышающих 1 терабайт, и обычно более 50 гигабайт.

Эта документация предназначена для разработчиков, знакомых с C и C++, а также основными концепциями баз данных, такими как таблицы, столбцы, индексы, восстановление и транзакции. Единственный метод доступа для ESE — это API C, описанный в этой документации.

Расширяемая подсистема хранилища — это компонент Windows, который появился в Windows 2000. Не все функции и API доступны во всех версиях операционных систем Windows.

ESE предоставляет механизм хранилища в пользовательском режиме, который управляет данными внутри неструктурированных двоичных файлов, доступных через API-интерфейсы Windows. Доступ к ESE осуществляется через библиотеку DLL, которая загружается непосредственно в процесс приложения; ни один из методов удаленного доступа не требуется или предоставляется ядром СУБД. Хотя ESE не имеет удаленного или межпроцессного доступа, используемые файлы данных можно предоставлять удаленно с помощью протокола SMB через API-интерфейсы Windows, но это не рекомендуется.

**Примечание**  .  Windows XP 64-bit Edition аналогичен Windows Server 2003 в целях определения поддерживаемого набора функций ESE.

### <a name="notes"></a>Примечания

Подсистема ESE ранее называлась синим, и поэтому часто термин "JET Blue" или "JET" используется с термином ESE за пределами этой документации. Однако на самом деле есть две полностью отдельные реализации API JET, именуемые JET Blue и JET Red. Термин "JET" часто используется для ссылки на JET Red, который является ядром СУБД, используемым для Microsoft Officeного доступа. Две реализации JET совершенно отличаются друг от друга, отдельно поддерживаются, имеют значительно отличающийся набор функций и не взаимозаменяемы. В документации ESE термин "JET" относится к ESE или API JET, который реализуется подсистемой ESE. Все ссылки на красный модуль JET всегда будут явно помечены как "JET Red".
