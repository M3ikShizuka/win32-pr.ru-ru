# <a name="creating-databases"></a>Создание баз данных


_**Применимо к:** Windows | Windows Server_

## <a name="creating-databases"></a>Создание баз данных

База данных ESE состоит из одной или нескольких таблиц, которые упорядочивают данные по столбцам и строкам. База данных ESE идентифицируется по имени и ИДЕНТИФИКАТОРу базы данных. База данных ESE выглядит как один файл в операционной системе Microsoft Windows; Однако внутренняя база данных хранится в виде коллекции страниц. Эти страницы содержат метаданные, описывающие данные в базе данных, сами данные и один или несколько индексов, которые хранят различные заказы данных. База данных может содержать до 2 ^ 31 страниц или 16 терабайт данных для базы данных с 8 КБ страниц.

Приложение создает экземпляр ядра СУБД, затем создает базу данных и прикрепляет ее к экземпляру. К экземпляру с [жеткреатедатабасе](./jetcreatedatabase-function.md) или [жетаттачдатабасе](./jetattachdatabase-function.md)можно одновременно подключить до 6 баз данных. В базе данных должен быть запущен один или несколько сеансов, так как все последующие операции ESE выполняются в контексте сеанса. Для каждого потока, использующего ESE, должен быть открыт отдельный сеанс.

Эта процедура позволяет инициализировать ESE и создать базу данных.

### <a name="to-initialize-ese-and-create-a-database"></a>Инициализация ESE и создание базы данных

1.  [Жеткреатеинстанце](./jetcreateinstance-function.md): создает экземпляр ядра СУБД.
    
    **Windows XP и более поздние версии:** Эта функция доступна в Windows XP и более поздних версиях. В Windows 2000 поддерживается только один экземпляр, и этот экземпляр создается неявно

2.  [Жетсетсистемпараметер](./jetsetsystemparameter-function.md): системные параметры, влияющие на физическую структуру, такие как JET_paramLogFilePath и JET_paramSystemPath, должны быть установлены перед инициализацией экземпляра с помощью [жетинит](./jetinit-function.md). Параметры, заданные на этом этапе, задаются для всех баз данных, создаваемых в экземпляре. [Жетсетсистемпараметер](./jetsetsystemparameter-function.md) — это единственная функция, которая может использовать экземпляр перед его инициализацией с помощью [жетинит](./jetinit-function.md).

3.  [Жетинит](./jetinit-function.md): Инициализирует экземпляр. Экземпляр должен быть инициализирован с помощью [жетинит](./jetinit-function.md) , прежде чем его можно будет использовать с другими функциями.

4.  [Жетбегинсессион](./jetbeginsession-function.md): создает сеанс для всех последующих транзакций. Все транзакции базы данных выполняются в контексте сеанса ([JET_SESID](./jet-sesid.md)).

5.  [Жеткреатедатабасе](./jetcreatedatabase-function.md): создает базу данных и возвращает маркер в идентификатор базы данных ([JET_DBID](./jet-dbid.md)).
    
    Если база данных уже существует, шаг 5 выше заменяется следующими двумя шагами.
    
    1.  [Жетаттачдатабасе](./jetattachdatabase-function.md): присоединение базы данных по имени к сеансу
    
    2.  [Жетопендатабасе](./jetopendatabase-function.md): Возвращает идентификатор базы данных, используемый в последующих операциях с базой данных.

Базу данных можно отсоединить от одного экземпляра ESE с помощью [жетдетачдатабасе](./jetdetachdatabase-function.md) и более поздних присоединенных к другому экземпляру с помощью [жетаттачдатабасе](./jetattachdatabase-function.md). При отсоединении базы данных ее можно скопировать как один файл с помощью стандартных служебных программ Windows. Однако если база данных подключена к экземпляру ESE, она не может быть скопирована, так как ESE открывает только файлы базы данных. Кроме того, в случае сбоя экземпляра файл базы данных не может быть скопирован отдельно, так как для восстановления после сбоя требуется, чтобы файлы журнала транзакций были связаны с ним.
