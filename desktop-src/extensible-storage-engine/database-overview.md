# <a name="database-overview"></a>Обзор базы данных


_**Применимо к:** Windows | Windows Server_

## <a name="database-overview"></a>Обзор базы данных

База данных ESE — это индексированный последовательный метод доступа (ISAM) для хранения и извлечения данных. База данных ESE хранится в одном файле и состоит из одной или нескольких определяемых пользователем таблиц. Данные упорядочиваются по записям таблицы с одним или несколькими определяемыми пользователем столбцами. Создаваемые индексы предоставляют другую организацию для всего набора или подмножества записей в таблице. С помощью API ESE приложения могут создавать курсоры, которые переходить по записям в базе данных в различных последовательных заказах. Элементы таблицы определены ниже:

  - **Столбец**: столбец является полем в таблице, в котором хранится конкретный тип информации. Столбцы могут быть фиксированными или переменными длиной в зависимости от типа данных, хранящихся в них. Некоторые столбцы, например помеченные столбцы, не получают пробелов, если значение **null** или задано как значение по умолчанию, и могут содержать несколько значений.

  - **Записи**. запись — это коллекция значений столбцов, имеющих уникальное удостоверение, определенное первичным ключом.

  - **Index**: индекс представляет собой коллекцию ключевых столбцов, определяющих сохраненный порядок записей в таблице. Кластеризованный или первичный индекс определяет порядок, в котором записи хранятся в таблице. Для указания различных порядков прохождения по записям в таблице можно определить несколько индексов. Индекс также может ограничить набор записей, отображаемых на основе простых критериев, таких как присутствие или отсутствие определенного значения ключевого столбца в записи.

  - **Cursor**: курсор указывает текущую запись в таблице и переходит к записям в таблице, используя текущий индекс. Курсор также содержит сведения о состоянии подготовленного обновления.

Столбцы и индексы могут быть добавлены или удалены из таблицы в любое время. Хотя можно определить несколько индексов, данные в таблице физически хранятся и логически кластеризованы в соответствии с определением первичного индекса в дереве B +. Каждый вторичный индекс хранится в отдельном дереве B +, содержащем только логические указатели на фактические данные, хранящиеся в первичной таблице. Если индекс не определен, записи в таблице хранятся в дереве B + в порядке вставки и называются последовательным индексом.

На схеме ниже приведен пример того, как данные для таблицы хранятся в дереве B + в соответствии с первичным индексом. Первичный индекс предназначен для имени и идентификатора, а вторичный индекс создается для номера офиса сотрудника. Записи для вторичного индекса хранятся в отдельном дереве B +, содержащем только указатели на записи, хранящиеся в основной таблице. Например, номер Office 12348 во вторичной таблице связан с записью 3 в основной таблице. Запись 3 содержит значения столбцов для сотрудника в Office 12348. Дополнительные сведения см. в описании [индекса в разделе таблицы](./indexing-in-the-table.md) .
