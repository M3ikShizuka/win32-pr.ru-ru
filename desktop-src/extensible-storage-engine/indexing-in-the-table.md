# <a name="indexing-in-the-table"></a>Индексирование в таблице


_**Применимо к:** Windows | Windows Server_

## <a name="indexing-in-the-table"></a>Индексирование в таблице

Индекс — это набор ключевых столбцов, которые определяют постоянный порядок записей в таблице. Записи — это индексные записи в первичном индексе. Один первичный индекс и несколько вторичных индексов можно определить для создания различных заказов данных в таблице.

Хотя можно определить несколько индексов, записи физически хранятся в деревьях B + в порядке, указанном первичным индексом. Первичный индекс всегда является кластеризованным индексом и также должен быть уникальным. Первичный индекс должен быть объявлен перед первой обновлением таблицы для сохранения порядка индексов. Если в приложении не определен первичный индекс, данные сохраняются в том порядке, в котором записи добавляются в таблицу. Этот особый индекс называется последовательным индексом.

Отдельные деревья B + используются для упорядочивания записей в соответствии с вторичным индексом. Элементы индекса во вторичном индексе содержат указатели на данные, хранимые в соответствии с первичным индексом. Записи индекса для записей в первичном индексе должны быть уникальными, так как вторичный индекс указывает на запись, используя первичный ключ записи. Вторичные индексы могут иметь или не иметь ограничений уникальности. Дополнительные сведения см. в разделе " [Обзор базы данных](./database-overview.md) ".
