# <a name="windows-media-mpeg4-v1v2-decoder"></a>Windows Декодер Media MPEG4 v1/v2

декодер Windows Media mpeg4 версии 1/v2 декодирует видеопотокы версии mpeg4 v1/v2.

## <a name="class-identifier"></a>Идентификатор класса

идентификатор класса (CLSID) для декодера Windows Media MPEG4 V1/V2 представлен константой **CLSID \_ CMpeg4DecMediaObject**. Можно создать экземпляр декодера MPEG4 версии 1/v2, вызвав **CoCreateInstance**.

## <a name="formats"></a>Форматы

декодер Windows Media MPEG4 V1/V2 поддерживает следующие входные типы носителей.

-   МЕДИАСУБТИПЕ \_ MPG4
-   МЕДИАСУБТИПЕ \_ MPG4
-   МЕДИАСУБТИПЕ \_ MP42
-   МЕДИАСУБТИПЕ \_ MP42

декодер Windows Media MPEG4 V1/V2 поддерживает следующие подтипы выходных данных, когда он выступает в качестве объекта мультимедиа DirectX (DMO).

-   МЕДИАСУБТИПЕ \_ YUY2
-   МЕДИАСУБТИПЕ \_ UYVY
-   МЕДИАСУБТИПЕ \_ RGB32
-   МЕДИАСУБТИПЕ \_ Rgb24
-   МЕДИАСУБТИПЕ \_ RGB565
-   МЕДИАСУБТИПЕ \_ RGB8
-   МЕДИАСУБТИПЕ \_ RGB555

декодер Windows Media MPEG4 V1/V2 поддерживает следующие подтипы выходных данных, когда он выступает в качестве Media Foundation преобразования (MFT).

-   Мфвидеоформат \_ YUY2
-   Мфвидеоформат \_ UYVY
-   Мфвидеоформат \_ RGB32
-   Мфвидеоформат \_ Rgb24
-   Мфвидеоформат \_ RGB565
-   Мфвидеоформат \_ RGB8
-   Мфвидеоформат \_ RGB555

## <a name="remarks"></a>Remarks

объект декодера Windows Media MPEG4 V1/V2 предоставляет интерфейс [**имедиаобжект**](/previous-versions/windows/desktop/api/mediaobj/nn-mediaobj-imediaobject) , чтобы объект можно было использовать в качестве объекта мультимедиа DirectX (DMO), и он предоставляет интерфейс [**имфтрансформ**](/windows/desktop/api/mftransform/nn-mftransform-imftransform) , чтобы объект можно было использовать в качестве Media Foundation преобразования (MFT). объект имеет тот же идентификатор класса (CLSID), независимо от того, выступает ли он в качестве DMO или MFT.

декодеры MPEG4 версии 1 или V2 работают в виде DMO или MFT в зависимости от того, какие интерфейсы получены и какая версия Windows выполняется. в следующей таблице показаны условия, при которых декодер MPEG4 версии 1 или V2 ведет себя в виде DMO или MFT.



| Операционная система            | Поведение декодера                                                                                                                                                                |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Windows XP                  | Декодер MPEG4 версии 1 или v2 всегда ведет себя как DMO.                                                                                                                                 |
| Windows Vista и Windows 7 | По умолчанию декодеры MPEG4 версии 1 и v2 ведет себя как DMO. Если вы получаете интерфейс [GUID подтипа видео](video-subtype-guids.md) для декодера MPEG4 версии 1/v2, он ведет себя как MFT. |



 

глобальные уникальные идентификаторы (guid) для подтипов мультимедиа RGB различаются в зависимости от того, выступает ли декодер в качестве DMO или MFT. идентификаторы guid для подтипов мультимедиа, отличных от RGB, одинаковы, независимо от того, выступает ли декодер в качестве DMO или MFT. Сведения об идентификаторах GUID, представляющих подтипы видео, см. в разделе [GUID для подтипов видео](video-subtype-guids.md).
