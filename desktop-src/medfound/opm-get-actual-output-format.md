# <a name="opm_get_actual_output_format"></a>ОПМ \_ получить \_ фактический \_ выходной \_ Формат

Возвращает описание видеосигнала, передаваемого через соединитель.



| Требование | Значение |
|--------------|------------------------------------------------------------------------------|
| GUID запроса | ОПМ \_ получить \_ фактический \_ выходной \_ Формат                                             |
| Входные данные   | Нет                                                                         |
| Возвращать данные  | Структура [**\_ реального \_ \_ формата выходных данных ОПМ**](/windows/desktop/api/opmapi/ns-opmapi-opm_actual_output_format) |



 

## <a name="remarks"></a>Комментарии

Видеосигнал, передаваемый через соединитель, не обязательно имеет тот же формат, что и режим экрана рабочего стола. Например, режим экрана рабочего стола может быть 1024 x 768 пикселей по 85 Гц, а соединитель может быть соединителем S-Video, передающий видеосигналы в 720 x 480 пикселей, 60/1.01 Гц с чередованием. В этом случае драйвер вернет разрешение видеосигнала S-Video, а не разрешение рабочего стола.

Этот запрос эквивалентен \_ запросу дксва коппкуеридисплайдата, используемому в сертифицированном протоколе защиты выходных данных (Копп).
