# <a name="mf_mt_depth_measurement-attribute"></a>\_ \_ \_ Атрибут измерения глубины MT

\[Некоторые сведения относятся к предварительно выпущенному продукту, который может быть значительно изменен перед коммерческой выпуском. Корпорация Майкрософт не предоставляет никаких гарантий, явных или подразумеваемых, относительно предоставленной здесь информации.\]

Значение, определяющее систему измерения для значения глубины в видеокадре.

## <a name="data-type"></a>Тип данных

**UINT32**

## <a name="remarks"></a>Remarks

Это значение является членом перечисления [**\_ мфдепсмеасуремент**](/windows/win32/api/mfapi/ne-mfapi-mfdepthmeasurement)

Если этот атрибут отсутствует, предполагается, что он является **дистанцетофокалплане**. Расстояние до фокальной плоскости обычно проще использовать в трехмерной евклидовой системе координат.

![Иллюстрация дистанцетофокалплане](images/distance-to-focal-plane.png)

Расстояние до центра фокуса — это обычно необработанные данные от датчика, например время видеокамер.

![Иллюстрация дистанцетуптикалцентер](images/distance-to-optical-center.png)

Камеры глубины не могут понять глубину всех пикселов. Если достоверность пикселя мала, из-за материала, перекрытия или вне диапазона и т. д. значение глубины на этом пикселе может быть недопустимым.

Если значение в пикселях глубины равно 0, пиксель является недопустимым.

Некоторые камеры с глубиной присоединяют метаданные битовой маски для каждого пикселя в дополнение к значению глубины, представляющему причину недопустимой глубины пикселя из-за материала, перекрытия или вне диапазона и т. д. Мы рекомендуем не присоединять такие метаданные как биты в значении глубины, так как они обычно приводят к сложности при использовании таких значений в шейдере пикселей. Используйте. рекомендуется использовать отдельный буфер образа 8 бит с тем же разрешением и присоединить его как атрибут [**имфсампле**](/windows/desktop/api/mfobjects/nn-mfobjects-imfsample). Такие метаданные зависят от поставщика камеры и не стандартизированы платформой. Мы рекомендуем использовать полные 16 бит для значения глубины, чтобы упростить обработку и использовать фиксированное значение, например 0 для недействительности.
