# <a name="mfpkey_wmaaecma_devicepair_guid-property"></a>МФПКЭЙ \_ вмааекма \_ Девицепаир, \_ свойство GUID

Определяет комбинацию звуковых устройств, которые в настоящее время использует приложение, с помощью схемы DSP для записи голоса.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

Доступно только с помощью [**ипропертисторе**](/windows/win32/api/propsys/nn-propsys-ipropertystore).

## <a name="data-type"></a>Тип данных

VT \_ CLSID

## <a name="applies-to"></a>Применение

-   [DSP для записи речи](voicecapturedmo.md)

## <a name="remarks"></a>Комментарии

Задайте это свойство, если вы используете DSP в режиме фильтрации, а значение свойства [мфпкэй вмааекма " \_ \_ получить \_ \_ статистику TS](mfpkey-wmaaecma-retrieve-ts-statsproperty.md) " является вариантным \_ true.

Если свойство [**мфпкэй \_ вмааекма \_ Получение \_ \_ статистики TS**](mfpkey-wmaaecma-retrieve-ts-statsproperty.md) имеет \_ значение true, то DSP собирает статистику о метках времени на звуковых устройствах и сохраняет эти сведения в реестре. Эти статистические данные могут изменяться для каждого сочетания устройства аудиозаписи и устройства воспроизведения звука. Поэтому приложение должно назначить идентификатор GUID для распознавания конкретного сочетания используемых устройств. Приложение должно отследить этот GUID, чтобы он мог назначить тот же идентификатор GUID при использовании той же пары звуковых устройств.

Если вы используете DSP в исходном режиме, не нужно задавать это свойство. DSP автоматически создает идентификатор GUID на основе значения свойства [мфпкэй \_ вмааекма \_ Device \_ indexes](mfpkey-wmaaecma-device-indexesproperty.md) .
