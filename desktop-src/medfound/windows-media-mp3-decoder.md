# <a name="windows-media-mp3-decoder"></a>Декодер MP3 Windows Media

Декодер MP3 Windows Media декодирует звуковые файлы, закодированные в следующих форматах.

-   ISO/IEC 11172-3 (аудио MPEG-1), уровень 3
-   Стандарт ISO/IEC 13818-3 (MPEG-2 Audio), низкое расширение частоты выборки

## <a name="class-identifier"></a>Идентификатор класса

Идентификатор класса (CLSID) для декодера MP3 Windows Media представлен константой **CLSID \_ CMP3DecMediaObject**. Вы можете создать экземпляр декодера MP3, вызвав **CoCreateInstance**.

## <a name="interfaces"></a>Интерфейсы

Объект декодера MP3 предоставляет интерфейс **имедиаобжект** , чтобы объект можно было использовать в качестве объекта мультимедиа DirectX (DMO), и он предоставляет интерфейс **имфтрансформ** , чтобы объект можно было использовать в качестве Media Foundation преобразования (MFT).

Декодер MP3 Windows Media ведет себя как DMO или MFT в зависимости от того, какие интерфейсы получены и какая версия Windows работает. В следующей таблице показаны условия, при которых декодер MP3 Windows Media ведет себя как DMO или MFT.



| Операционная система | Поведение декодера                                                                                                                                                                               |
|------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Windows XP       | Декодер MP3 Windows Media всегда ведет себя как DMO.                                                                                                                                           |
| Windows Vista    | По умолчанию декодер MP3 Windows Media ведет себя как DMO. Если вы получаете интерфейс **имфтрансформ** или интерфейс **ипропертисторе** в декодере MP3 Windows Media, он ведет себя как MFT. |
| Windows 7        | По умолчанию декодер MP3 Windows Media ведет себя как DMO. Если вы получаете интерфейс **имфтрансформ** для декодера MP3 Windows Media, он ведет себя как MFT.                                    |



 

## <a name="input-formats"></a>Форматы входных данных

В следующей таблице показан тег формата аудио, который представляет тип входных данных, поддерживаемый декодером MP3 Windows Media.



| Формат константы тега      | Формат значения тега | Формат аудио     |
|--------------------------|------------------|------------------|
| \_Формат волны \_ MPEGLAYER3 | 0x55             | ISO MPEG, уровень 3 |



 

## <a name="output-formats"></a>Форматы выходных данных

В следующей таблице показаны Теги звукового формата, представляющие типы выходных данных, поддерживаемые декодером MP3 Windows Media.



| Формат константы тега       | Формат значения тега | Формат аудио                                                                |
|---------------------------|------------------|-----------------------------------------------------------------------------|
| формат ВОЛНового \_ \_ PCM         | 0x0001           | Формат PCM (при использовании в качестве DMO или MFT)                                   |
| \_Формат волны \_ IEEE \_ float | 0x0003           | IEEE с плавающей запятой (при использовании в MFT)                                   |
| формат ВОЛНового \_ \_ расширения  | 0xFFFE           | Формат PCM/IEEE в структуре **вавеформатекстенсибле** (при использовании в файле MFT) |



 

Декодер MP3 Windows Media поддерживает и перечисляет следующие типы выходных данных.

-   Тип выходных данных, который имеет ту же частоту выборки и число каналов, что и тип входных данных.
-   Выходные данные моно для стерео ввода.
-   Типы выходных данных с битовыми глубиной 8 и 16.
-   Вывод с плавающей запятой, если декодер ведет себя как MFT.

Декодер MP3 Windows Media поддерживает, но не перечисляет следующие типы выходных данных.

-   Тип вывода с половиной частоты выборки входного типа.
-   Тип выходных данных, который имеет одну четвертую частоту выборки для типа входных данных.
