# <a name="mf_mt_video_3d_first_is_left-attribute"></a>\_ \_ \_ Первый трехмерный видеоролик \_ \_ — \_ левый атрибут

Для формата 3D-видео указывает, какое представление является левым представлением.

## <a name="data-type"></a>Тип данных

**Bool** , сохраненный как **UINT32**

## <a name="remarks"></a>Remarks

Для трехмерного видео каждый пример видео содержит два представления, которые обозначены как *первое представление* и *второе представление*. Точный макет представлений в памяти определяется атрибутом [ \_ \_ \_ трехмерного видео в \_ формате MF MT](mf-mt-video-3d-format.md) .



| Формат               | Первое представление              | Второе представление               |
|----------------------|-------------------------|---------------------------|
| Упакованная параллельно  | Левая половина буфера | Правая половина буфера  |
| Упакованные сверху вниз | Верхняя половина буфера  | Нижняя половина буфера |
| MultiView            | Индекс буфера 0          | Индекс буфера 1            |
| Базовое представление            | Весь кадр            | Отсутствует               |



 

По умолчанию первое представление представляет собой левое представление, а второе — в правильном представлении. Если левое и правое представления меняются местами, установите в поле \_ \_ \_ \_ \_ \_ тип мультимедиа значение **false** для параметра 3D-видео MF.

> [!Note]  
> В формате *базового представления* (**MFVideo3DSampleFormat \_ басевиев**) сохраняется только базовое представление, поэтому этот атрибут не применяется.
