# <a name="mfpkey_wmadec_ltrtoutput-property"></a>МФПКЭЙ \_ вмадек \_ Лтртаутпут, свойство

Указывает, следует ли выполнять декодер Lt-Rt свертывания.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

Доступно только с помощью [**ипропертисторе**](/windows/win32/api/propsys/nn-propsys-ipropertystore).

## <a name="data-type"></a>Тип данных

**Логическое значение VT \_**

## <a name="default-value"></a>Значение по умолчанию

**ВАРИАНТ \_ false**

## <a name="remarks"></a>Remarks

Если это свойство имеет значение VARIANT \_ false, а выходные данные — стерео, то аудио-декодер использует простой канал. Если это свойство имеет значение \_ true, то аудио-декодер выполняет Lt-Rt (с матрицей) по вертикали до стерео, а затем любой окружающий декодер Dolby можно использовать для декодирования вокруг матрицы. Например, звуковой декодер может выполнять Lt-Rt выгнуть содержимое 5,1 или 7,1.

Это свойство поддерживается только в том случае, если декодер выступает в качестве объекта мультимедиа DirectX (DMO). Отсутствие какого-либо вида поддерживается, если декодер выступает в качестве Media Foundation преобразования (MFT).

в Windows Vista при получении интерфейса **ипропертисторе** для звукового декодера декодер выступает в качестве MFT. следовательно, это свойство нельзя использовать в Windows Vista. сведения о том, когда декодер выступает в качестве DMO или MFT, см. на страницах справочника по отдельным кодекам в разделе [объекты кодека](codecobjects.md).
