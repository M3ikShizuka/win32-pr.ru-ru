# <a name="mfpkey_vtype-property"></a>МФПКЭЙ \_ втипе, свойство

Указывает логику, которую кодек будет использовать для обнаружения чередующихся исходных видео.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

g \_ всзвмвквтипе

## <a name="data-type"></a>Тип данных

VT \_ I4

## <a name="default-value"></a>Значение по умолчанию

0

## <a name="remarks"></a>Remarks

Для этого свойства можно задать одно из следующих значений.



| Значение | Описание                                                                                                                                 |
|-------|---------------------------------------------------------------------------------------------------------------------------------------------|
| 0     | Кодек будет использовать стандартную логику обнаружения типов кадров.                                                                                 |
| 1     | Кодек будет обрабатывать все исходные видеоматериалы как чередующиеся кадры.                                                                          |
| 2     | Кодек будет обрабатывать все исходные видеоматериалы как поля с чередованием видео.                                                                 |
| 3     | Кодек автоматически определит, являются ли входные видеокадры перечередованием кадров или поля с чередованием видео.                      |
| 4     | Кодек автоматически определит, являются ли входные видеокадры последовательными кадрами, чередованием кадров или полями с чередованием видео. |



 

Это свойство определяет метод кодирования изображения, используемый для последовательной или чередования кодирования видео.

Если тип видео не указан, кодек будет использовать прогрессивную кодировку кадров для сеансов прогрессивной кодировки и поле с чередованием кодирования для сеансов кодирования с чередованием. Тип сеанса кодирования видео (прогрессивный или с чередованием) задается с помощью свойства [мфпкэй \_ интерлацедкодинженаблед](mfpkey-interlacedcodingenabledproperty.md) .

> [!Note]  
> Для создания чередующихся выходных данных свойство [мфпкэй \_ интерлацедкодинженаблед](mfpkey-interlacedcodingenabledproperty.md) должно иметь значение Variant \_ true. в противном случае установка свойства мпфкэй втипе не \_ будет действовать.

 

При кодировании видео с чередованием можно указать несколько методов кодирования изображений. Как правило, самый эффективный способ кодирования видео с чередованием — использовать метод с чередованием (2). Если исходное видео содержит очень малое движение, то может оказаться более подходящим метод с чередованием кадров (1) или методом автоматического кадра/поля (2).

При кодировании смешанного содержимого (содержащего как прогрессивные, так и чередующиеся кадры) лучше использовать значение автокадров/поле/прогрессивный метод (4).
