# <a name="mfpkey_wmaaecma_retrieve_ts_stats-property"></a>МФПКЭЙ \_ вмааекма \_ Получение \_ \_ Свойства статистики TS

Указывает, хранится ли в реестре DSP для записи речи статистика по метке времени.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

Доступно только с помощью [**ипропертисторе**](/windows/win32/api/propsys/nn-propsys-ipropertystore).

## <a name="data-type"></a>Тип данных

Логическое значение VT \_

## <a name="default-value"></a>Значение по умолчанию

ВАРИАНТ \_ false

## <a name="applies-to"></a>Применяется к

-   [DSP для записи речи](voicecapturedmo.md)

## <a name="remarks"></a>Remarks

Алгоритмы отмены акустического эха (AEC) зависят от точных меток времени в звуковых потоках. На практике метки времени часто неидеальныы, а различные звуковые устройства могут демонстрировать различные тарифы и отклонения. Если параметр AEC включен, DSP собирает статистические данные о метках времени и использует эти сведения для компенсации неточностей.

Если значение этого свойства равно \_ true, то DSP сохраняет статистические данные, собираемые в реестре. В следующий раз, когда DSP выполняет контекст AEC, используя ту же пару звуковых устройств, он считывает статистические данные из реестра, что позволяет поставщику схемы эффективности работать более эффективно.

Если для этого свойства задать значение \_ true и вы используете DSP в режиме фильтрации, необходимо также задать свойство [мфпкэй \_ вмааекма \_ девицепаир \_ GUID](mfpkey-wmaaecma-devicepair-guidproperty.md) . В режиме исходного кода это необязательно.

Значение этого свойства по умолчанию — VARIANT \_ false. DSP использует это свойство только в том случае, если включена обработка AEC.
