# <a name="codecapi_avencslicecontrolsize-property"></a>КОДЕКАПИ \_ авенкслицеконтролсизе, свойство

Задает размер среза в мегабайтах (МБ), битах или MB.

## <a name="data-type"></a>Тип данных

**Ulong** (VT \_ UI4)

## <a name="property-guid"></a>GUID свойства

**КОДЕКАПИ \_ авенкслицеконтролсизе**

## <a name="remarks"></a>Remarks

**Кодировщики H. 264/AVC:**

Значение параметра КОДЕКАПИ \_ авенкслицеконтролсизе управляется свойством [кодекапи \_ авенкслицеконтролмоде](codecapi-avencslicecontrolmode.md) . В следующей таблице показано, как \_ Свойства кодекапи авенкслицеконтролсизе и кодекапи \_ авенкслицеконтролмоде управляют размером и количеством срезов в кадре.



| \_Параметр АВЕНКСЛИЦЕКОНТРОЛМОДЕ кодекапи | Смысл значения                                                                                                                                                                                                                                                                                                                                                                                           |
|-----------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 0                                       | Это целое число, указывающее размер каждого среза в кадре в единицах макроблоккс. <br/> Кодировщик должен отклонить параметр, если значение больше числа макроблоккс в кадре.<br/>                                                                                                                                                                         |
| 1                                       | Это целое число, которое указывает размер каждого среза в кадре в единицах битов. <br/> Кодировщик должен запустить новый срез на макроблокк, который приводит к превышению этого значения числом битов в срезе (поэтому размер каждого среза всегда будет меньше или равен этому значению). Это означает, что размер последнего среза может быть значительно меньше этого значения. <br/> |
| 2                                       | Это целое число, указывающее размер каждого среза в кадре в единицах макроблокк строк. <br/> Кодировщик должен отклонить параметр, если значение больше числа строк макроблокк в кадре.<br/>                                                                                                                                                                 |



 

Если приложение не устанавливает значение для [кодекапи \_ авенкслицеконтролмоде](codecapi-avencslicecontrolmode.md), кодировщик должен вернуть ошибку.

Рекомендуемое значение по умолчанию — наличие одного среза для всего фрейма.

Некоторые Кодировщики могут параллельно кодировать срезы, и производительность может зависеть от параметров элемента управления "срез". Например, кодирование кадра в виде одного среза может быть медленнее, чем при кодировании кадра как нескольких срезов.

Параметры элемента управления среза являются динамическими и могут быть изменены во время сеанса кодирования.
