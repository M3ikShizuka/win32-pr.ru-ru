# <a name="mfpkey_complexityex-property"></a>МФПКЭЙ \_ комплекситекс, свойство

Указывает сложность алгоритма кодировщика.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

g \_ всзвмвккомплекситекс

## <a name="data-type"></a>Тип данных

VT \_ I4

## <a name="default-value"></a>Значение по умолчанию

Значение по умолчанию зависит от версии кодировщика видео, как показано в следующей таблице.



| Версия кодировщика                 | Значение по умолчанию |
|---------------------------------|---------------|
| Windows Кодировщик Media Video 9   | 3             |
| Windows Кодировщик Media Video 7/8 | 1             |



 

## <a name="possible-values"></a>Возможные значения

Возможные значения зависят от версии кодировщика видео, как показано в следующей таблице.



| Версия кодировщика                 | Возможные значения  |
|---------------------------------|------------------|
| Windows Кодировщик Media Video 9   | 0, 1, 2, 3, 4, 5 |
| Windows Кодировщик Media Video 7/8 | 0, 1             |



 

## <a name="remarks"></a>Remarks

Более низкие значения приводят к тому, что кодек использует менее сложные алгоритмы кодирования. Хотя более простые алгоритмы обеспечивают более низкое качество вывода, процесс кодирования выполняется быстрее и требует меньше вычислительной мощности. Это может быть важно при кодировании содержимого из действующего источника, так как кодировщик должен быстро обрабатывать входные данные, чтобы поддерживать источник.

Любое значение, присвоенное этому свойству, будет проигнорировано, если свойство [мфпкэй \_ компрессионоптимизатионтипе](mfpkey-compressionoptimizationtypeproperty.md) имеет значение 1. В этом случае для МФПКЭЙ \_ комплекситекс автоматически устанавливается значение 3.
