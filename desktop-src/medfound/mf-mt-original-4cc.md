# <a name="mf_mt_original_4cc-attribute"></a>\_ \_ Исходный \_ атрибут 4CC MF

Содержит исходный кодек FOURCC для видеопотока.

## <a name="data-type"></a>Тип данных

**UINT32**

## <a name="getset"></a>Get/Set

Чтобы получить этот атрибут, вызовите метод [**имфаттрибутес:: UInt32**](/windows/desktop/api/mfobjects/nf-mfobjects-imfattributes-getuint32).

Чтобы задать этот атрибут, вызовите [**имфаттрибутес:: SetUINT32**](/windows/desktop/api/mfobjects/nf-mfobjects-imfattributes-setuint32).

## <a name="applies-to"></a>Применяется к

[**имфмедиатипе**](/windows/desktop/api/mfobjects/nn-mfobjects-imfmediatype)

## <a name="remarks"></a>Remarks

В зависимости от исходного файла источник AVI Media может установить этот атрибут для типов мультимедиа, которые он предлагает.

Файл AVI содержит заголовок потока для каждого потока в файле. Источник AVI Media преобразует заголовок потока в тип мультимедиа. Для сжатых видеопотоков в заголовке потока содержится объект FOURCC, идентифицирующий видеокодек. В большинстве случаев источник AVI Media преобразует это значение непосредственно в GUID подтипа, как описано в разделе [GUID подтипа видео](video-subtype-guids.md). В некоторых случаях, однако, он сопоставляет исходное значение FOURCC с другим параметром FOURCC, эквивалентным. Если это так, источник мультимедиа сохраняет исходное значение FOURCC в типе носителя, используя \_ \_ Исходный атрибут 4CC MF MT \_ .

Сопоставления FOURCC хранятся в реестре в следующем разделе:

**HKey \_ \_Корневой** \\ **медиафаундатион** классов \\ **MapVideo4cc**

Каждая запись имеет значение **DWORD** . Имя записи является шестнадцатеричным представлением FOURCC без префикса "0x" и первым символом, показываемым первым в строке. Например, код FOURCC "abcd" будет выглядеть как "61626364". Значение записи является эквивалентным кодом FOURCC.

Константа GUID для этого атрибута экспортируется из мфууид. lib.
