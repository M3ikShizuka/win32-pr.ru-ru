# <a name="mfpkey_constrainenclatency-property"></a>МФПКЭЙ \_ констраиненклатенци, свойство

Указывает, ограничивается ли кодировщик максимальным требованием к задержке.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

Доступно только с помощью [**ипропертисторе**](/windows/win32/api/propsys/nn-propsys-ipropertystore).

## <a name="data-type"></a>Тип данных

**Логическое значение VT \_**

## <a name="default-value"></a>Значение по умолчанию

**ВАРИАНТ \_ false**

## <a name="remarks"></a>Remarks

Если оставить это свойство по умолчанию со значением **Variant \_ false**, кодировщик перечислит набор режимов по умолчанию примерно 2000 миллисекунд для задержки кодировщика. Если для этого свойства задать значение **\_ true**, необходимо также указать максимальную задержку кодировщика, задав свойство [**мфпкэй \_ максенклатенцимс**](mfpkey-maxenclatencymsproperty.md) . В этом случае кодировщик создает режимы, которые отвечают требованиям к задержке, и перечисляет только эти режимы. Кодировщик гарантирует выборку выходных данных, как только кодировщик получил входные данные за период времени, равный **мфпкэй \_ максенклатенцимс**.
