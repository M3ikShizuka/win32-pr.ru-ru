# <a name="mfpkey_numbframes-property"></a>МФПКЭЙ \_ нумбфрамес, свойство

Задает число двунаправленных прогнозирующих кадров (B-кадров).

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

g \_ всзвмвкнумбфрамес

## <a name="data-type"></a>Тип данных

VT-I4

## <a name="default-value"></a>Значение по умолчанию

0

## <a name="remarks"></a>Remarks

по умолчанию Windows Media Video 9 использует только интрафрамес (I-кадры), также известные как ключевые кадры или закрепленные фреймы, которые являются полностью закодированными кадрами, и прогнозные кадры (P-кадры), которые кодируются в отличие от предыдущего I-кадра. B-кадры отличаются от кадров P-Frame, так как они сохраняют как различия предыдущего кадра, так и отличия от следующего кадра.

При настройке кодека для использования B-кадров он будет использовать указанное число кадров B-фреймов между каждой парой кадров любого типа I или P.

Например, если последовательность кадров без кадров "B" имеет значение "ИППППППППИ", то одинаковое кодирование последовательности с двумя кадрами B — "ИББПББПББИ".

Для большинства содержимого подходят один или два кадра в виде «B-кадров». При более высоких скоростях данных один B-кадр обычно является оптимальным выбором. Три или более редко полезны.

Допустимый диапазон значений для этого свойства — от 0 до 7.
