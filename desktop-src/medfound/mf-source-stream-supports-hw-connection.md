# <a name="mf_source_stream_supports_hw_connection-attribute"></a>В \_ исходном \_ потоке MF \_ поддерживается \_ \_ атрибут подключения HW

Указывает, поддерживает ли источник мультимедиа поток данных оборудования.

## <a name="data-type"></a>Тип данных

**Bool** , сохраненный как **UINT32**

## <a name="remarks"></a>Комментарии

Этот атрибут используется, когда источник мультимедиа использует прокси-сервер и может передавать данные на аппаратную шину, не отправляя данные в ЦП. Например, веб-камера может доставлять видео в формате H. 264 непосредственно в Интегрированный аппаратный декодер.

В этом сценарии источник и декодер по-прежнему представлены в Microsoft Media Foundation объектом [источника мультимедиа](media-sources.md) и [преобразованием Media Foundation](media-foundation-transforms.md) (MFT). Однако потоки данных между этими двумя объектами на уровне конвейера не передаются только на уровне оборудования, как показано на следующей схеме.

![Схема, на которой показан источник аппаратного прокси-сервера.](images/proxy-mft3.png)

Соединение между источником мультимедиа и MFT согласовывается следующим образом.

1.  Конвейер запрашивает источник мультимедиа для интерфейса [**имфмедиасаурцеекс**](/windows/desktop/api/mfidl/nn-mfidl-imfmediasourceex) . (Этот интерфейс является необязательным для поддержки источников мультимедиа.)
2.  Конвейер вызывает [**имфмедиасаурцеекс:: жетстреаматтрибутес**](/windows/desktop/api/mfidl/nf-mfidl-imfmediasourceex-getstreamattributes) , чтобы получить указатель [**имфаттрибутес**](/windows/desktop/api/mfobjects/nn-mfobjects-imfattributes) .
3.  Конвейерные запросы к \_ исходному \_ потоку \_ MF \_ поддерживают \_ атрибут подключения HW. Если атрибут представлен и равен **true**, источник мультимедиа поддерживает аппаратные подключения.
4.  Конвейер проверяет, является ли таблица MFT также прокси-сервером, проверяя атрибут [атрибута " \_ \_ \_ \_ атрибут аппаратного URL-адреса перечисления MFT](mft-enum-hardware-url-attribute.md) " в MFT. Дополнительные сведения см. в разделе [Hardware МФТС](hardware-mfts.md).
5.  Конвейер задает атрибут [ \_ \_ \_ атрибута потока MFT, подключенного к](mft-connected-stream-attribute.md) MFT. Значением этого атрибута является указатель [**имфаттрибуте**](/windows/desktop/api/mfobjects/nn-mfobjects-imfattributes) , полученный из источника мультимедиа на шаге 2.
6.  Конвейер задает **значение true** для файла [MFT, \_ подключенного к атрибуту \_ \_ \_ аудиопотока](mft-connected-to-hw-stream.md) , как к источнику мультимедиа, так и к таблице MFT.
