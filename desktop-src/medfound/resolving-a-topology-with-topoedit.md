# <a name="resolving-a-topology-with-topoedit"></a>Разрешение топологии с помощью Топоедит

Существует два типа топологий:

-   Частичная топология. Исходный узел подключен непосредственно к выходному узлу. В некоторых случаях частичная топология может иметь некоторые узлы промежуточного преобразования, такие как эффекты, но не все. Узлы преобразования, которые опущены, обычно являются декодерами или преобразованием форматов МФТС, такими как преобразователи цветов и звуковые фильтры.

-   Полная топология. Исходный узел подключен к выходному узлу через узел преобразования. Этот тип топологии должен содержать каждый узел для обработки данных.

Топоедит может играть только полные топологии. Топоедит использует объект-загрузчик топологии, предоставляемый Media Foundation, для преобразования частичной топологии в полную топологию путем вставки необходимых преобразований. Процесс создания полной топологии называется разрешением топологии.

Сведения о типах топологий см. в разделе "частичные топологии" статьи [о топологиях](about-topologies.md).

Перед разрешением топологии убедитесь в том, что:

-   Топология содержит исходный узел и выходной узел.

-   Исходный и выходной узлы соединены допустимым соединением с узлом. Во время разрешения топологии загрузчик топологии проверяет тип носителя узлов на совместимость. Если соединение с недопустимым узлом существует, то процесс завершается ошибкой и выводится сообщение об ошибке.

Чтобы разрешить топологию, в меню **топология** выберите пункт **Разрешить топологию**.

На панели инструментов отображается состояние топологии: **\[ разрешено \]** или **\[ не \] разрешено**.

Если Топоедит разрешает топологию успешно, **состояние топологии** устанавливается как **\[ разрешенное \]** , а элементы управления воспроизведением — включены.

Каждый раз при внесении изменений в топологию **состояние топологии** устанавливается в значение " **\[ не \] разрешено** ", что указывает на необходимость повторного разрешения.
