# <a name="mft_message_notify_end_of_stream"></a>\_уведомление об \_ \_ окончании \_ \_ потока в сообщении MFT

Уведомляет Media Foundation преобразование (MFT) о завершении входного потока.

## <a name="message-parameter"></a>Параметр сообщения

Параметр *улпарам* содержит идентификатор входного потока, указанного в виде значения **типа DWORD** . В 64-разрядных приложениях поместите это значение в младшие 32-разряды типа **ulong \_**.

## <a name="remarks"></a>Remarks

Чтобы отправить это сообщение, вызовите [**имфтрансформ::P роцессмессаже**](/windows/desktop/api/mftransform/nf-mftransform-imftransform-processmessage).

Клиенту не требуется отсылать это сообщение.

После завершения потока клиент может снова вызвать [**ProcessInput**](/windows/desktop/api/mftransform/nf-mftransform-imftransform-processinput) , чтобы отправить новые данные для этого потока. В этом случае клиент должен задать атрибут небесперебойности (атрибут [**мфсампликстенсионности \_ небесперебойности**](mfsampleextension-discontinuity-attribute.md) ) для первого входного примера после завершения потока. (Клиент всегда должен устанавливать этот атрибут в первом новом примере после завершения потока, независимо от того, отправил ли клиент сообщение MFT об **\_ \_ \_ окончании \_ \_ потока** сообщения. Дополнительные сведения об обработке прекращений см. в разделе [Базовая модель обработки MFT](basic-mft-processing-model.md).

После отправки этого сообщения для каждого входного потока клиент обычно отправляет команду **\_ \_ \_ очистки сообщения MFT** , а затем выполняет сбор оставшихся выходных данных. Однако клиент не обязан выполнять сток MFT. Если клиент не выполняет сток MFT, то MFT, как правило, отменяет любые необработанные данные при следующем вызове метода [**ProcessInput**](/windows/desktop/api/mftransform/nf-mftransform-imftransform-processinput), когда обнаруживает небесперебойность потока. Кроме того, клиент может очистить MFT перед вызовом **ProcessInput**.

Это сообщение не удаляет входной поток или не сбрасывает тип носителя.

### <a name="implementation"></a>Реализация

MFT не требуется для ответа на это сообщение.
