# <a name="mestreamtick-event"></a>Событие Местреамтикк

Сигнализирует, что поток мультимедиа не имеет данных, доступных в указанное время.

## <a name="event-values"></a>Значения событий

Возможные значения, полученные из [**имфмедиаевент:: GetValue**](/windows/desktop/api/mfobjects/nf-mfobjects-imfmediaevent-getvalue) , включают следующее.



| VARTYPE           | Описание                                                                   |
|-------------------|-------------------------------------------------------------------------------|
| VT \_ i8<br/> | Время, когда происходит разрыв, в 100-наносекундных единицах.<br/> <br/> |



## <a name="remarks"></a>Комментарии

Это событие сигнализирует о пропуске в данных. Событие уведомляет подчиненные компоненты, чтобы не ждать каких бы то ни было данных в указанное время.

Событие должно быть отправлено любым объектом, создающим отметки времени для образцов мультимедиа в потоке. В зависимости от формата данных это может быть:

-   Поток мультимедиа в источнике мультимедиа (интерфейс [**имфмедиастреам**](/windows/desktop/api/mfidl/nn-mfidl-imfmediastream) ) или
-   Преобразование декодера (интерфейс [**имфтрансформ**](/windows/desktop/api/mftransform/nn-mftransform-imftransform) ).

Во время пропуска объект должен отправить событие примерно так часто, как это обычно создает образцы. Для видео отправьте одно событие для каждого отсутствующего кадра. Для звука Отправьте событие по крайней мере один раз в секунду во время разрыва. Значение события — это метка времени отсутствующего примера. Отправьте столько событий Местреамтикк, сколько требуется для заполнения зазоров в данных.

Если источник мультимедиа имеет несколько потоков и имеется разрыв в нескольких потоках, каждый поток должен отправить события Местреамтикк. Например, если имеется разрыв в данных аудио и видео, то оба потока отправляют событие.

Событие Местреамтикк не завершает запрос [**имфмедиастреам:: рекуестсампле**](/windows/desktop/api/mfidl/nf-mfidl-imfmediastream-requestsample) . Источник мультимедиа по-прежнему должен отправить событие [мемедиасампле](memediasample.md) для каждого вызова **рекуестсампле**.

Приемники мультимедиа не могут напрямую использовать это событие. Чтобы сообщить о пропуске потока в приемник мультимедиа, вызовите [**имфстреамсинк::P лацемаркер**](/windows/desktop/api/mfidl/nf-mfidl-imfstreamsink-placemarker) с маркером **\_ \_ деления маркера мфстреамсинк** . Конвейер Media Foundation преобразует события Местреамтикк в маркеры **\_ \_ деления мфстреамсинк** , когда это необходимо.

Не устанавливайте атрибут [**\_ прекращения мфсампликстенсион**](mfsampleextension-discontinuity-attribute.md) для следующего примера носителя после события местреамтикк. Атрибут **\_ прекращения мфсампликстенсион** подразумевает, что отметка времени не является непрерывной с предыдущими штампами времени, тогда как местреамтикк предполагает, что отметки времени являются непрерывными, но некоторые данные отсутствуют.

> [!Note]  
> Более ранняя версия документации неправильно объявила, что пример после события Местреамтикк должен иметь атрибут [**мфсампликстенсион \_**](mfsampleextension-discontinuity-attribute.md) .
