# <a name="mft_message_command_marker"></a>\_ \_ маркер команды сообщения \_ MFT

Помечает точку в потоке. Это сообщение относится только к [асинхронному МФТС](asynchronous-mfts.md).

## <a name="message-parameter"></a>Параметр сообщения

Произвольное значение. MFT возвращает значение клиенту в событии [метрансформмаркер](metransformmarker.md) .

## <a name="remarks"></a>Remarks

Чтобы отправить это сообщение, вызовите [**имфтрансформ::P роцессмессаже**](/windows/desktop/api/mftransform/nf-mftransform-imftransform-processmessage).

MFT реагирует на этот мессажеас:

1.  MFT создает столько выходных образцов, сколько можно из существующих входных данных, отправляя событие [метрансформхавеаутпут](metransformhaveoutput.md) для каждого выходного примера.
2.  После создания всех выходных данных MFT отправляет событие [метрансформмаркер](metransformmarker.md) . Это событие должно быть отправлено после всех событий [метрансформхавеаутпут](metransformhaveoutput.md) .

Клиент не обязан отправить это сообщение и должен отправить это сообщение только в асинхронный МФТС. Синхронная Таблица MFT не будет отсылать событие [метрансформмаркер](metransformmarker.md) в ответ на это сообщение.

### <a name="implementation"></a>Реализация

Асинхронный МФТС должен отвечать на это сообщение, как описано. Синхронное МФТС должно игнорировать это сообщение.
