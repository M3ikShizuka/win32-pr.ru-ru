# <a name="codecapi_avencvideouseltrframe-property"></a>КОДЕКАПИ \_ авенквидеауселтрфраме, свойство

Указывает, что текущий кадр кодируется с помощью одного или нескольких фреймов LTR.

## <a name="data-type"></a>Тип данных

**Ulong** (VT \_ UI4)

## <a name="property-guid"></a>GUID свойства

**КОДЕКАПИ \_ авенквидеауселтрфраме**

## <a name="property-value"></a>Значение свойства

Значение этого элемента управления включает два поля, где каждое поле имеет 16 бит.

## <a name="remarks"></a>Комментарии

**Кодировщики H. 264/AVC:**

Это свойство не должно вызываться, если отложенный вызов, использующий кадр LTR, был вызван с помощью \_ Свойства кодекапи авенквидеауселтрфраме, а кодировщик еще не создал кадр, в котором использовался LTR. Иными словами, кодировщик не должен ставить в очередь запросы КОДЕКАПИ \_ авенквидеауселтрфраме.

Если запрос КОДЕКАПИ \_ авенквидеауселтрфраме отправляется, когда другой \_ запрос кодекапи авенквидеауселтрфраме все еще находится в состоянии ожидания, то старый запрос следует удалить.

Вызов КОДЕКАПИ \_ авенквидеауселтрфраме в кадре небазового слоя является допустимым и применяется к кадру, не являющемуся базовым, без задержки в кадре базового слоя.
