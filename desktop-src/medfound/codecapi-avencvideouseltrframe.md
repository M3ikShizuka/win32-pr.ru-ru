# <a name="codecapi_avencvideouseltrframe-property"></a>КОДЕКАПИ \_ авенквидеауселтрфраме, свойство

Указывает, что текущий кадр кодируется с помощью одного или нескольких фреймов LTR.

## <a name="data-type"></a>Тип данных

**Ulong** (VT \_ UI4)

## <a name="property-guid"></a>GUID свойства

**КОДЕКАПИ \_ авенквидеауселтрфраме**

## <a name="property-value"></a>Значение свойства

Значение этого элемента управления включает два поля, где каждое поле имеет 16 бит.




| Значение | Значение | 
|-------|---------|
| <span id="The_first_field"></span><span id="the_first_field"></span><span id="THE_FIRST_FIELD"></span><dl><dt><strong>Первое поле</strong></dt><dt>битов [0.. 15]</dt></dl> | Указывает, какие фреймы LTR разрешено использовать для кодирования текущего кадра. <br /><strong>Кодировщики H. 264/AVC:</strong><br /> Это битовая карта, указывающая, какие кадры LTR можно использовать в качестве ссылки на этот кадр. Младший значащий бит соответствует индексу LTR 0, второй наименее значащий бит соответствует индексу LTR 1 и т. д.<br /> Это значение не должно быть равно 0.<br /> Наибольший индекс, заданный этим значением, не должен превышать максимальное число кадров LTR, указанное в свойстве <a href="codecapi-avencvideoltrbuffercontrol.md">CODECAPI_AVEncVideoLTRBufferControl</a> меньше одного.<br /> | 
| <span id="The_second_field"></span><span id="the_second_field"></span><span id="THE_SECOND_FIELD"></span><dl><dt><strong>Второе поле</strong></dt><dt>битов [16.. 31]</dt></dl> | Флаг, указывающий, требуются ли дополнительные ограничения для кодирования последующих кадров. <br /><strong>Кодировщики H. 264/AVC:</strong><br /> 1 является единственным допустимым значением для этого поля. Все остальные значения являются недопустимыми и зарезервированы для будущего использования.<br /> Если флаг равен 1, кодировщик должен закодировать последующие кадры в порядке кодировки в соответствии со следующими ограничениями:<br /><ul><li>Он не должен использовать краткосрочные ссылочные кадры в порядке кодировки, отличном от текущего кадра, или в будущем кодировке в порядке кодировки.</li><li>Он не должен использовать кадры LTR, не описанные в последнем элементе управления CODECAPI_AVEncVideoUseLTRFrame.</li><li>Он может использовать фреймы LTR, обновленные после текущего кадра.</li></ul> | 




 

## <a name="remarks"></a>Комментарии

**Кодировщики H. 264/AVC:**

Это свойство не должно вызываться, если отложенный вызов, использующий кадр LTR, был вызван с помощью \_ Свойства кодекапи авенквидеауселтрфраме, а кодировщик еще не создал кадр, в котором использовался LTR. Иными словами, кодировщик не должен ставить в очередь запросы КОДЕКАПИ \_ авенквидеауселтрфраме.

Если запрос КОДЕКАПИ \_ авенквидеауселтрфраме отправляется, когда другой \_ запрос кодекапи авенквидеауселтрфраме все еще находится в состоянии ожидания, то старый запрос следует удалить.

Вызов КОДЕКАПИ \_ авенквидеауселтрфраме в кадре небазового слоя является допустимым и применяется к кадру, не являющемуся базовым, без задержки в кадре базового слоя.
