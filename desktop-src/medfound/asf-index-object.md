# <a name="asf-indexer"></a>Индексатор ASF

*ИНДЕКСАТОР* ASF — это компонент уровня вмконтаинер, который используется для чтения или записи объектов индекса в файле ASF. Сведения о структуре файла ASF см. в разделе [Структура файлов ASF](asf-file-structure.md).

Приложение может использовать индексатор для выполнения поиска на основе времени презентации или создания новых записей индекса для файла ASF. Индексатор ASF реализует интерфейс [**имфасфиндексер**](/windows/desktop/api/wmcontainer/nn-wmcontainer-imfasfindexer) .




| Тип индекса | Описание | 
|------------|-------------|
| Индекс на основе времени презентации | Предоставляет индексирование на основе времени представления для потоков аудио и видео в блоках индексов, чтобы сделать индексирование более эффективным. Каждый блок индекса ссылается на элементы индекса, которые содержат смещение в байтах. <br /> Смещение — это позиция искомого пакета данных относительно начала объекта данных ASF.<br /> GUID_NULL должен использоваться в качестве типа GUID для идентификатора индекса. Для получения дополнительных сведений; см. раздел <a href="using-the-indexer-to-write-a-new-index.md">Использование индексатора для записи нового индекса</a>.<br /> | 
| Индекс времени | Облегчает поиск по времени в потоках, содержащих метаданные времени. Коды времени соответствуют формату SMPTE (<em>hours: minutes: seconds: frames</em>). Каждый блок индекса ссылается на элементы индекса, которые содержат смещение в байтах. <br /> Смещение — это позиция искомого пакета данных относительно начала объекта данных ASF.<br /><blockquote>[!Note]<br />Объекты индекса времени в настоящее время не поддерживаются.</blockquote><br /><br /> | 
| Индекс на основе фрейма | Предоставляет индексирование на основе кадров для видеопотоков. Индексы в индексе на основе кадров находятся в виде номеров кадров, а первый кадр для потока в ASF-файле соответствует записи 0 в объекте индекса на основе фрейма. Каждый блок индекса ссылается на элементы индекса, которые содержат смещение в байтах.<br /><blockquote>[!Note]<br />Объекты индексов на основе кадров в настоящее время не поддерживаются.</blockquote><br /><br /> |
