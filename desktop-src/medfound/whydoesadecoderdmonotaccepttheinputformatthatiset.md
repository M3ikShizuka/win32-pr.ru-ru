# <a name="why-does-a-decoder-not-accept-the-input-format-that-i-set"></a>Почему декодер не принимает заданный формат ввода?

Существует множество причин, по которым декодер может отклонить формат. Наиболее распространенные отсутствующие или неправильные данные расширенного формата. Расширенные данные формата представляют собой сведения, относящиеся к кодеку, которые добавляются к структуре, описывающей тип носителя.

при перечислении типа выходных данных с помощью объекта кодировщика DMO элемент **пбформат** структуры [**\_ \_ типа мультимедиа**](/previous-versions/windows/desktop/api/mediaobj/ns-mediaobj-dmo_media_type) будет указывать на структуру **вавеформатекс** . Эта структура содержит добавленные к нему данные расширенного формата, а размер этих данных хранится в элементе **вавеформатекс. кбсизе** . Независимо от контейнера, используемого для хранения сжатых данных, необходимо сохранить структуру **вавеформатекс** и использовать ее в типе ввода для декодера. Без данных расширенного формата декодер не может распаковать содержимое.

Для видеоформатов необходимо вручную извлечь данные расширенного формата и добавить их в структуру **видеоинфохеадер** . Дополнительные сведения см. [в разделе Использование частных данных видеокодека](usingvideocodecprivatedata.md).
