# <a name="mf_transcode_topologymode-attribute"></a>\_Атрибут топологимоде для ПЕРЕКОДИРОВАНИЯ MF \_

Указывает для топологии кодирования, будет ли загрузчик топологии загружать преобразования на основе оборудования.

Режим топологии указывает, можно ли использовать аппаратные преобразования (например, аппаратные кодеки) в топологии перекодирования. Приложение может сохранить этот атрибут в профиле перекодирования путем вызова [**имфтранскодепрофиле:: сетконтаинераттрибутес**](/windows/desktop/api/mfidl/nf-mfidl-imftranscodeprofile-setcontainerattributes).

## <a name="data-type"></a>Тип данных

**[**MF \_ Перекодировать \_ \_ Флаги топологимоде**](/windows/desktop/api/mfidl/ne-mfidl-mf_transcode_topologymode_flags)** , хранящиеся в виде **UINT32**

## <a name="getset"></a>Get/Set

Чтобы получить этот атрибут, вызовите метод [**имфаттрибутес:: UInt32**](/windows/desktop/api/mfobjects/nf-mfobjects-imfattributes-getuint32).

Чтобы задать этот атрибут, вызовите [**имфаттрибутес:: SetUINT32**](/windows/desktop/api/mfobjects/nf-mfobjects-imfattributes-setuint32).

## <a name="remarks"></a>Комментарии

Этот атрибут является необязательным. Он должен иметь одно из следующих значений.



| Значение                                              | Описание:                                                                                                                                                                                                                                                                       |
|----------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **топологимоде (MF), \_ \_ \_ \_ разрешенное оборудование** | Загрузчик топологии будет загружать аппаратные МФТС, такие как аппаратные декодеры, если они доступны.<br/> Загрузчик топологии автоматически возвращается к программному декодированию, если аппаратный декодер не найден или если аппаратный декодер не может подключиться по какой-либо причине.<br/> |
| **MF \_ перекодировать \_ \_ только программное обеспечение топологимоде \_**    | Загрузчик топологии будет загружать только МФТС программного обеспечения, включая программные декодеры.                                                                                                                                                                                                    |



 

Значение по умолчанию — **MF \_ \_ топологимоде \_ Software \_**.

Если загрузчик топологии вставляет в топологию основную таблицу оборудования, в узле Topology задается атрибут [ \_ \_ \_ \_ атрибута «URL-адрес перечисления MFT](mft-enum-hardware-url-attribute.md) ». Чтобы проверить наличие в файле MFT оборудования, перечислите узлы в разрешенной топологии и проверьте наличие этого атрибута.

Константа GUID для этого атрибута экспортируется из мфууид. lib.
