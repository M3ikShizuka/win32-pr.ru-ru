# <a name="mft_decoder_expose_output_types_in_native_order-attribute"></a>\_Декодер MFT \_ предоставляют \_ выходные \_ типы \_ в \_ \_ атрибуте собственного порядка

Указывает, предоставляет ли декодер выходные типы ИЙУВ/I420 (подходящие для перекодирования) перед другими форматами.

## <a name="data-type"></a>Тип данных

**UINT32**

## <a name="remarks"></a>Remarks

Этот атрибут является указанием для декодера, чтобы упорядочить список типов вывода в определенном порядке, в зависимости от предполагаемого использования: воспроизведение или перекодировать.

Для большинства форматов кодирования (H. 264, MPEG-2, WMV) видеодекодеры в Microsoft Media Foundation поддерживают несколько распространенных выходных данных YUV, включая NV12, YV12, YUY2, ИЙУВ и I420. Декодер предлагает упорядоченный список типов выходных данных с помощью метода [**имфтрансформ:: жетаутпутаваилаблетипе**](/windows/desktop/api/mftransform/nf-mftransform-imftransform-getoutputavailabletype) .

Для воспроизведения NV12 является наиболее эффективным и широко поддерживаемым форматом. Таким образом, по умолчанию декодеры обычно предлагают NV12 в качестве первого выходного типа в списке. Это уменьшает время, необходимое для разрешения типа носителя при создании топологии воспроизведения. Однако для перекодировки ИЙУВ или I420 более эффективны для ЦП и обычно предпочтительны кодировщиками.

Если декодер поддерживает этот атрибут, то атрибут имеет следующее поведение:

-   Если атрибут имеет ненулевое значение, ИЙУВ и I420 отображаются первым в списке типов выходных данных мультимедиа. Этот параметр наиболее эффективен для перекодирования.
-   Если атрибут равен нулю, NV12 отображается первым в списке типов выходных данных мультимедиа. Этот параметр наиболее эффективен для воспроизведения и является значением по умолчанию.

Чтобы задать этот атрибут, сделайте следующее:

1.  Вызовите метод [**имфтрансформ:: OutAttribute**](/windows/desktop/api/mftransform/nf-mftransform-imftransform-getattributes) в декодере, чтобы получить указатель [**имфаттрибутес**](/windows/desktop/api/mfobjects/nn-mfobjects-imfattributes) .
2.  Вызовите [**имфаттрибутес:: SetUINT32**](/windows/desktop/api/mfobjects/nf-mfobjects-imfattributes-setuint32) , чтобы добавить атрибут.
