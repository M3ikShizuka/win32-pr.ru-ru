# <a name="mfpkey_enccomplexity-property"></a>МФПКЭЙ \_ енккомплексити, свойство

Указывает сложность алгоритма кодирования. Значение является целым числом от 0 до 100, где 0 указывает наименьший сложный алгоритм, а 100 — наиболее сложный алгоритм.

## <a name="constant-for-ipropertybag"></a>Константа для Ипропертибаг

Доступно только с помощью [**ипропертисторе**](/windows/win32/api/propsys/nn-propsys-ipropertystore).

## <a name="data-type"></a>Тип данных

**VT \_ UI4**

## <a name="default-value"></a>Значение по умолчанию

100 для Windows media audio 10 и Windows media audio 10 Professional

100. выпуск Windows Vista Windows Media Audio 10 без потерь

0 для выпуска Windows 7 Windows Media Audio 10 без потерь

## <a name="remarks"></a>Remarks

Если свойство [**мфпкэй \_ констраинекомплексити**](mfpkey-constrainenccomplexityproperty.md) имеет значение **Variant \_ true**, кодировщик корректирует сложность его алгоритма в соответствии со значением этого свойства.

для кодировщика Windows media audio 10 и кодировщика Windows media audio 10 Professional, если значение этого свойства равно 100, кодировщик помещает высокую нагрузку на цп и выдает наивысшее качество вывода. По мере уменьшения значения этого свойства потребность в ЦП снижается, но качество вывода также уменьшается.

для кодировщика Windows Media Audio 10 без потерь, если значение этого свойства равно 0, кодировщик помещает низкий спрос на цп. По мере увеличения значения этого свойства потребность в ЦП увеличивается, а размер выходных данных кодировщика немного сокращается. Выходные данные без потерь зависят от значения этого свойства.

Если оставить это свойство по умолчанию со значением **Variant \_ false**, кодировщик использует алгоритм по умолчанию. алгоритм по умолчанию зависит от используемого кодировщика и версии Windows. В следующей таблице описывается поведение по умолчанию для различных сочетаний.



| Операционная система | Поведение по умолчанию                                                                                                                                                                                                |
|------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Windows Vista    | Windows media audio 10, Windows media audio 10 Professional и Windows media audio 10 без потерь по умолчанию используют самый сложный алгоритм.                                                    |
| Windows 7        | кодировщики Windows media audio 10 и Windows media audio 10 Professional по умолчанию используют наиболее сложный алгоритм. кодировщик Windows Media Audio 10 без потерь по умолчанию использует наименее сложный алгоритм. |



 

Если свойство [**\_ констраинекомплексити мфпкэй**](mfpkey-constrainenccomplexityproperty.md) имеет значение **Variant \_ false**, кодировщик игнорирует это свойство.
