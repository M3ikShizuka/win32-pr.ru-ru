# <a name="progress-bars"></a>Индикаторы выполнения

> [!NOTE]
> Это руководство по проектированию было создано для Windows 7 и не Обновлено для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

С помощью индикатора выполнения пользователи могут отслеживать ход выполнения длительной операции. Индикатор выполнения может либо Показать Приблизительный процент завершения (в конце), либо указать, что операция выполняется (неопределенное).

Исследования удобства использования показали, что пользователи знают о времени ответа в течение одной секунды. Следовательно, следует рассмотреть операции, которые выполняются в течение двух секунд или дольше, чтобы быть продолжительными, и требуется некоторый тип отзывов о ходе выполнения.

![снимок экрана с типичным индикатором выполнения ](images/progress-bars-image1.png)

Типичный индикатор выполнения.

> [!Note]  
> Рекомендации, связанные с [макетом](vis-layout.md) , представлены в отдельной статье.

 

## <a name="is-this-the-right-control"></a>Выбор правильного элемента управления

Чтобы определиться, ответьте на вопросы:

-   **Операция будет выполнена примерно через пять секунд или меньше?** Если это так, используйте [индикатор действия](inter-mouse.md) , так как отображение индикатора выполнения для такой короткой длительности будет отвлекаться от нежелательного. Если операция обычно занимает пять секунд или меньше, но иногда занимает больше времени, начните с занятого указателя и преобразуйте его в индикатор выполнения через пять секунд.
-   **Используется ли неопределенный индикатор выполнения для ожидания выполнения задачи пользователем?** Если это так, не используйте индикатор выполнения. Индикаторы выполнения предназначены для выполнения компьютера, а не для выполнения пользователем.
-   **Является неопределенным индикатором выполнения в сочетании с анимацией?** В этом случае следует использовать только анимацию. Неопределенный индикатор выполнения фактически является универсальной анимацией и не добавляет значение в анимацию.
-   **Является ли операция фоновой задачей с очень длинными (более двух минут), для которой пользователи больше интересуются при выполнении?** Если это так, используйте вместо них [уведомления](mess-notif.md) . В этом случае пользователи выполняют другие задачи в то же время и не отслеживают ход выполнения. Использование уведомления позволяет пользователям выполнять другие задачи без прерывания работы. Примеры таких длительных операций включают печать, резервное копирование, сканирование системы, а также передачу или преобразование данных.
-   **После завершения операции пользователи смогут воспроизвести результаты?** Если да, используйте вместо него ползунок. Примеры таких операций включают запись видео и звука и воспроизведение.

    ![снимок экрана проигрывателя и ползунка мультимедиа ](images/progress-bars-image2.png)

    В этом примере ползунок используется для индикации хода выполнения при воспроизведении звука. Это позволит пользователям воспроизвести результаты позже.

## <a name="design-concepts"></a>Принципы проектирования

Во время длительной операции пользователям необходимо общее представление о выполняемых операциях. Им также необходимо знать:

-   Что была запущена длительная операция.
-   Этот ход выполнения и операция в конечном итоге завершаются (и, следовательно, не заблокировались).
-   Приблизительный процент выполнения операции (и, следовательно, оставшийся процент).
-   Значение, если необходимо отменить операцию, если не стоит ждать.
-   Значение, если необходимо продолжить ожидание или выполнить какие-либо действия по завершении операции.

**Используйте индикаторы хода выполнения для операций, требующих ограниченного количества времени,** даже если это время не может быть точно предсказуемым. Неопределенные индикаторы выполнения показывают, что выполняется, но не предоставляет никакой другой информации. Не выбирайте неопределенный индикатор выполнения, основанный только на возможном отсутствии точности.

Например, предположим, что для выполнения операции требуется пять шагов, а для каждого из этих действий требуется ограниченное количество времени, но время для каждого шага может сильно различаться. В этом случае следует использовать индикатор хода выполнения и отображать ход выполнения каждого шага пропорционально количеству времени, которое обычно занимает каждый шаг. Использовать неопределенный индикатор выполнения только в том случае, если индикатор выполнения с завершением работы приведет к неправильной блокировке операции.

**Если вы выполняете только одно действие...**

Убедитесь, что вы предоставляете отзыв о ходе выполнения для длительных операций и что приведенные выше сведения четко взаимодействуют. По возможности используйте индикаторы выполнения по завершении.

## <a name="guidelines"></a>Рекомендации

### <a name="general"></a>Общее

-   **При выполнении длительной операции Предоставьте отзыв о ходе выполнения.** Пользователям не следует пригадаться, если выполняется ход выполнения.
-   **Ясно указывает на реальный ход выполнения.** Индикатор выполнения должен быть изменен при выполнении. Если диапазон ожидаемого времени выполнения велик, рассмотрите возможность использования нелинейного масштабирования для обозначения хода выполнения в течение более длительного времени. Вы не хотите, чтобы пользователи заблокировали работу программы, когда она не была заблокирована.
-   **Ясно указывает на отсутствие хода выполнения.** Если ход выполнения не выполняется, индикатор выполнения не должен быть изменен. Вы не хотите, чтобы пользователи ожидают неопределенное время операции, которая не будет завершена.
-   **Укажите полезные сведения о ходе выполнения.** Предоставление дополнительных сведений о ходе выполнения, но только в том случае, если пользователи могут выполнять с ними какие-либо действия. Убедитесь, что текст отображается достаточно долго, чтобы пользователи могли прочитать его.

    ![снимок экрана с индикатором хода выполнения, отображающий скорость передачи ](images/progress-bars-image11.png)

    В этом примере пользователи могут видеть скорость передачи. Низкая скорость передачи подразумевает необходимость использования сетевого подключения с высокой пропускной способностью.

-   **Не выводятся ненужные сведения.** Обычно пользователи не интересуют сведения о выполняемой операции. Например, пользователи программы установки не зависят от того, какой файл копируется или что системные компоненты зарегистрированы, так как они не имеют ожидания об этих деталях. Обычно правильно обозначенный индикатор выполнения содержит достаточно информации, поэтому предоставьте дополнительные сведения о ходе выполнения только в том случае, если пользователи могут выполнять с ними какие-либо действия. Предоставление сведений о том, что пользователи не интересуются, делает работу с пользователем более сложной и технической. Если вам нужны более подробные сведения для отладки, не выводите их в сборках выпуска.

    **Правильно:**

    ![снимок экрана хода установки ](images/progress-bars-image12.png)

    В этом примере индикатор выполнения с меткой имеет все необходимое.

    **Правильно:**

    ![снимок экрана с индикатором хода выполнения, отображающий скорость передачи ](images/progress-bars-image11.png)

    В этом примере проводник Windows копирует файлы, выбранные пользователем, поэтому отображение копируемых файлов имеет смысл.

    **Неправильно**:

    ![снимок экрана с ходом регистрации ](images/progress-bars-image13.png)

    В этом примере программа установки предоставляет сведения, не имеющие смысла для пользователя.

-   **Предоставляет полезные анимации.** Если все готово, анимация повышает удобство работы пользователей, помогая пользователям визуализировать операцию. Хорошая анимация влияет только на текст. Например, индикатор выполнения для команды Outlook Delete отображает корзину для назначения, если файлы можно восстановить, но без корзины, если восстановить файлы невозможно.

    ![снимок экрана с ходом выполнения удаления ](images/progress-bars-image14.png)

    В этом примере отсутствие корзины приводит к необратимому удалению файлов. Эти дополнительные сведения не будут передаваться по эффективному использованию только текста.

-   **Не используйте ненужные анимации.** Анимация может быть неверной, так как обычно выполняется в отдельном потоке от реальной задачи и, следовательно, может предлагать ход выполнения, даже если операция была заблокирована. Кроме того, если операция выполняется медленнее, чем ожидалось, пользователи иногда предполагают, что анимация является частью причины. Следовательно, используйте анимацию только при четком обоснование. не используйте их для того, чтобы попытаться разработать пользователей.
-   **Разместить анимацию по центру индикатора выполнения.** Помещайте анимацию над метками индикатора выполнения, если они есть. Если справа от индикатора выполнения находится кнопка Отмена или остановить, включите кнопку при определении центра.
-   **Воспроизводить звуковой результат при завершении операции, только если он очень длинный (более двух минут), редко и важен.** Если пользователь, скорее всего, не покинет важную операцию во время его обработки, то звуковые эффекты восстанавливают вмешательство пользователя. Использование звукового эффекта при завершении в других обстоятельствах приведет к нежелательному неудобствому.
-   **Не украсть фокус ввода для отображения обновления или завершения хода выполнения.** Пользователи часто переключаются на другие программы в ожидании и не должны прерываться. Фоновые задачи должны оставаться в фоновом режиме.
-   **Не беспокойтесь о технической поддержке.** Так как обратная связь, предоставляемая индикаторами выполнения, не очень точная и является пределом, индикаторы выполнения не являются хорошим механизмом предоставления информации для технической поддержки. Следовательно, если операция может завершиться неудачно (как в случае программы установки), не следует предоставлять дополнительные сведения о ходе выполнения, которые полезны только в технической поддержке. Вместо этого предоставьте альтернативный механизм, например файл журнала для записи сведений о технической поддержке.

    **Неправильно**:

    ![снимок экрана индикатора выполнения, показывающего имя сервера ](images/progress-bars-image15.png)

    В этом примере индикатор выполнения отображает сведения, предназначенные для технической поддержки.

-   **Не помещайте процент завершения или любой другой текст на индикаторе выполнения.** Такой текст недоступен и не совместим с использованием тем.

    **Неправильно**:

    ![снимок экрана индикатора выполнения с текстом на панели ](images/progress-bars-image16.png)

    В этом примере текст в процентах индикатора выполнения недоступен.

-   **Не объединяйте индикатор выполнения со занятым указателем.** Используйте один или другой, но не оба одновременно.
-   **Не используйте вертикальные полосы выполнения.** Горизонтальные линии выполнения имеют более естественное сопоставление и более эффективный поток.

### <a name="determinate-progress-bars"></a>Прервать индикаторы выполнения

-   **Используйте индикаторы хода выполнения для операций, требующих ограниченного количества времени,** даже если это время не может быть точно предсказуемым. Неопределенные индикаторы выполнения показывают, что выполняется, но не предоставляет никакой другой информации. Не выбирайте неопределенный индикатор выполнения, основанный только на возможном отсутствии точности.
-   **Ясно указывает на фазу выполнения.** Индикатор выполнения должен иметь возможность указать, находится ли операция в начале, середине или конце операции. Например, индикаторы выполнения, которые немедленно прокрутки до 99 процента завершения, остаются в течение длительного времени, особенно неинформативные и неприятные. В таких случаях индикатор выполнения должен быть изначально установлен не более 33 процентов, чтобы указать, что операция все еще находится на начальном этапе.
-   **Ясно указывает на завершение.** Не отключайте индикатор выполнения до 100%, пока операция не будет завершена.
-   **Укажите время, оставшееся для оценки, если это можно сделать точно.** Точность оставшихся оценочных оценок удобна, но оценки, которые отменяют отметку или возобновляются, значительно не полезны. Перед предоставлением точных оценок может потребоваться выполнить некоторую обработку. В этом случае не следует отображать потенциально неточные оценки за этот начальный период.
-   **Не перезапускайте ход выполнения.** Индикатор выполнения теряет свое значение, если он перезапускается (возможно, из-за завершения операции), так как пользователи не могут узнать, когда операция будет завершена. Вместо этого все шаги, выполняемые в операции, совместно используют часть хода выполнения, и индикатор выполнения переходит в состояние завершения один раз.

    **Неправильно**:

    ![снимок экрана с перезапуском индикатора выполнения ](images/progress-bars-image17.png)

    В этом примере операция перемещена на шаг копирования файлов и сбрасывает индикатор выполнения для этого шага. Теперь пользователи не имеют представления о том, как выполнялся прогресс или сколько времени осталось.

-   **Не заархивируйте ход выполнения.** Как и при перезапуске, при резервном копировании индикатор выполнения теряет свое значение. Всегда повышайте ход выполнения монотонно. Однако можно иметь оставшееся время, которое увеличивается (а также уменьшается), так как скорость выполнения может быть различной.

### <a name="indeterminate-progress-bars"></a>Неопределенные индикаторы выполнения

-   **Используйте неопределенные индикаторы выполнения только для операций, общий ход выполнения которых невозможно определить.** Используйте неопределенные индикаторы выполнения для операций, требующих неограниченное количество времени или обращающихся к неизвестному количеству объектов. Используйте время ожидания для предоставления границ операциям, основанным на времени.
-   **Преобразуйте в индикатор выполнения завершения, если можно определить общий ход выполнения.** Например, если для определения количества объектов требуется значительно больше двух секунд, можно использовать неопределенный индикатор выполнения во время подсчета объектов, а затем преобразовать их в индикатор выполнения.
-   **Не следует объединять неопределенные индикаторы выполнения с оценками процента завершения или оставшегося времени.** Если эти сведения можно указать, используйте индикатор выполнения для завершения.
-   **Не объединяйте неопределенные индикаторы выполнения с анимацией.** Неопределенный индикатор выполнения фактически является универсальной анимацией, поэтому следует использовать одну или другую, но не то и другое.

    **Правильно:**

    ![снимок экрана с ходом выполнения обнаружения сервера ](images/progress-bars-image18.png)

    В этом примере для отображения текущей операции используется только анимация.

### <a name="modeless-progress-bars"></a>Немодальные индикаторы выполнения

-   **Если пользователи могут выполнять какое-либо повышение производительности во время выполнения операции, предоставьте немодальные отзывы.** Может потребоваться отключить подмножество функций, требующих завершения операции.
-   **Если в окне есть адресная строка, отобразится немодальный ход выполнения в адресной строке.**

    ![снимок экрана с индикатором выполнения в составе адресной строки ](images/progress-bars-image19.png)

    В этом примере в адресной строке отображается немодальный ход выполнения.

-   В противном случае, **Если окно имеет строку состояния, отобразится немодальный ход выполнения в строке состояния.** Вставьте любой соответствующий текст в строку состояния слева.

    ![снимок экрана индикатора выполнения в строке состояния ](images/progress-bars-image7.png)

    В этом примере в строке состояния отображается немодальный ход выполнения.

### <a name="modal-progress-bars"></a>Модальные индикаторы выполнения

-   **Разместить модальные индикаторы выполнения на страницах хода выполнения или в** [диалоговых окнах хода выполнения](win-dialog-box.md).
-   **Укажите кнопку, чтобы остановить операцию, если она занимает больше нескольких секунд или может не завершиться.** Пометка кнопки "Отмена" при отмене возвращает окружение в предыдущее состояние (без побочных эффектов). в противном случае пометка кнопки "остановить", чтобы показать, что она оставляет частично завершенную операцию. Можно изменить метку кнопки с "Отмена" на "остановлено" в середине операции. Если в некоторый момент она не может вернуть среду в предыдущее состояние. Центрирование кнопки команды по вертикали с индикатором выполнения вместо того, чтобы выровнять верхние края.

    **Правильно:**

    ![снимок экрана с ходом ожидания сети ](images/progress-bars-image20.png)

    В этом примере остановка сетевого подключения не имеет побочного действия, поэтому используется Отмена.

    **Правильно:**

    ![снимок экрана со индикатором выполнения "оставшееся время копирования" ](images/progress-bars-image21.png)

    В этом примере остановка копирования оставляет все скопированные файлы, поэтому кнопка Command будет помечена как стоп.

    **Неправильно**:

    ![снимок экрана с индикатором хода выполнения поиска и кнопкой "Закрыть" ](images/progress-bars-image22.png)

    В этом примере остановка поиска оставляет без побочных эффектов, поэтому кнопка должна быть помечена как отмена.

### <a name="time-remaining"></a>Оставшееся время

Для отработки индикаторов хода выполнения:

-   **Используйте следующие форматы времени.** Начните с первого из следующих форматов, где самая крупная единица времени не равна нулю, а затем замените на следующий формат, когда самая крупная единица времени станет нулевой.

    **Для индикаторов выполнения:**

    **Если связанная информация отображается в формате двоеточия:**

    Оставшееся время: ч ч, м мин

    Оставшееся время: m минут, с

    Оставшееся время: s с

    **Если пространство экрана имеет уровень "Премиум":**

    h час, осталось м мин

    m мин, с оставшихся секунд

    осталось секунд:

    **В противном случае:**

    ч часы, осталось м мин

    m мин, с осталось секунд

    осталось секунд:

    **Для строк заголовка:**

    чч: мм осталось

    мм: остаток SS

    0: остаток SS

    В этом компактном формате сначала отображаются наиболее важные сведения, поэтому она не усекается на панели задач.

-   **Выполните точную оценку, но не принимайте в точности значение false.** Если самая крупная единица — часы, укажите минуты (если они понятны), а не секунды.

    **Неправильно**:

    чч ч, mm минуты, СС секунд

-   **Поддерживайте оценку до текущей даты.** Время обновления осталось по крайней мере каждые 5 секунд.
-   **Сосредоточьтесь на оставшееся время** , так как это наиболее важные для пользователей сведения. Укажите общее время, прошедшее только при наличии ситуаций, когда прошло много времени (например, когда задача может повторяться). Если Оценка оставшегося времени связана с индикатором выполнения, не имеет процента завершения текста, так как эти сведения передаются самим индикатором выполнения.
-   **Будьте грамматически правильно.** Если число равно единице, используйте единицы во единственном числе.

    **Неправильно**:

    1 минуты, 1 с

-   **Используйте выделение прописных букв, как в предложении.**

### <a name="progress-bar-colors"></a>Цвета индикатора выполнения

-   **Используйте красную или желтую полосу выполнения только для указания состояния хода выполнения, а не окончательных результатов задачи.** Красный или желтый индикатор выполнения указывает, что пользователям необходимо предпринять определенные действия для выполнения задачи. Если условие не удается восстановить, оставьте индикатор выполнения зеленым и отобразите сообщение об ошибке.
-   **Если имеется восстанавливаемое условие, которое не позволяет выполнить дальнейшие операции, включите красный индикатор выполнения.** Отобразите сообщение, объясняющее проблему и рекомендуемое решение.
-   **Включите желтый индикатор выполнения, чтобы указать, что пользователь приостановил задачу или что существует условие, которое помешать прогрессу** , но выполнение продолжается (например, при низкой сетевой связи). Если пользователь приостановил работу, измените метку кнопки приостановить на возобновление. Если ход выполнения задается, отобразится сообщение, объясняющее проблему и рекомендуемое решение.

### <a name="meters"></a>Приборов

-   **Используйте индикаторы выполнения только для хода выполнения.** Используйте измерительные единицы, чтобы указать проценты, не связанные с ходом выполнения.

## <a name="recommended-sizing-and-spacing"></a>Рекомендуемое изменение размера и расстояния

![Диаграмма, показывающая размеры и пробелы в строке хода выполнения ](images/progress-bars-image23.png)

Рекомендуемое изменение размера и расстояния для индикаторов выполнения.

-   Всегда используйте рекомендуемую высоту индикатора хода выполнения.
    -   **Исключение:** Если родительское окно не поддерживает рекомендованную высоту, можно использовать другую высоту.
-   Используйте минимальную ширину, если хотите сделать индикатор выполнения ненавязчивым.
-   Не используйте ширину, превышающую рекомендованный максимум. Индикатор выполнения не должен заполнять доступное пространство.
-   Центрировать индикатор выполнения по горизонтали, если размер окна намного шире, чем максимальная рекомендуемая ширина.

## <a name="labels"></a>Метки

### <a name="progress-bar-labels"></a>Метки индикатора выполнения

-   **Используйте краткую метку со статическим элементом управления "текст" для указания действия, выполняемого операцией.** Начните метку с глагола (например, копирование) и заканчивайте многоточием. Эта метка может измениться динамически, если операция имеет несколько шагов или обрабатывает несколько объектов.
-   Не присваивайте уникальный [ключ доступа](glossary.md) , так как элемент управления не является интерактивным.
-   Используйте [прописные буквы в стиле предложения](glossary.md).
-   Если операция не была напрямую инициирована пользователем, можно включить дополнительную метку для предоставления контекста и приносимся к прерыванию. Начать эту лишнюю метку с помощью фразы, подождите, пока. Эта метка не должна изменяться во время операции.

    ![снимок экрана индикатора выполнения с меткой ](images/progress-bars-image24.png)

    В этом примере пользователю предлагается подождать, так как пользователь не инициировал операцию напрямую.

-   Поместите метку над индикатором выполнения и выровняйте метку по левому краю индикатора выполнения.

-   Можно изменить метку кнопки с "Отмена" на "остановлено" в середине операции. Если в некоторый момент она не может вернуть среду в предыдущее состояние.
