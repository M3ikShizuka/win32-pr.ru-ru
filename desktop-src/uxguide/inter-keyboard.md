# <a name="keyboard"></a>Клавиатура

> [!NOTE]
> Это руководство по проектированию было создано для Windows 7 и не Обновлено для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

Клавиатура является основным устройством ввода, используемым для ввода текста в Microsoft Windows. Для обеспечения специальных возможностей и эффективности большинство действий можно выполнить с помощью клавиатуры.

Клавиатуры также могут ссылаться на виртуальные, экранные клавиатуры и писать панели, используемые компьютерами без физической клавиатуры, такой как компьютеры на базе планшетов.

![снимок экрана экранной клавиатуры ](images/inter-keyboard-image1.png)

Экранная клавиатура Windows и технология сенсорного ввода.

![снимок экрана планшетной панели рукописного ввода Windows ](images/inter-keyboard-image2.png)

Планшетная панель рукописного ввода Windows и технология сенсорного ввода.

Существует шесть основных типов ключей:

-   Ключ символа отправляет литеральный символ в окно с фокусом ввода.
-   Ключ модификатора, объединенный с другим ключом, изменяет значение связанного с ним ключа, например CTRL, Alt, Shift и клавишу с эмблемой Windows.
-   Клавиши навигации — это направленные стрелки, а также домашние, конечные и страничные.
-   Ключи редактирования: INSERT, Backspace и DELETE.
-   Функциональные клавиши — это F1 через F12.
-   Системные ключи помещают систему в режим или выполняют системную задачу, например печать экрана, Caps Lock и Num Lock.

Ключи доступа — это ключи или сочетания клавиш, используемые для взаимодействия со всеми элементами управления или элементами меню с помощью клавиатуры. Сочетания клавиш — это ключи или сочетания клавиш, используемые опытными пользователями для выполнения часто используемых команд для повышения эффективности. Windows указывает клавиши доступа, подчеркивая назначение клавиш доступа.

![снимок экрана с клавишами доступа и сочетаниями клавиш ](images/inter-keyboard-image3.png)

В этом примере показаны ключи доступа и сочетания клавиш.

Чтобы устранить ненужные визуальные элементы, Windows скрывает подчеркивание клавиш доступа по умолчанию и отображает их только при нажатии клавиши ALT. Чтобы обеспечить согласованность с Windows, изображения в руководство по UX также отображаются с подчеркиванием ключа доступа, если только в рекомендации не включены ключи доступа.

Чтобы улучшить осведомленность о назначении клавиш доступа в программе в течение всего процесса разработки, их можно отображать постоянно. На панели управления перейдите в центр специальных возможностей и нажмите кнопку **сделать эту клавиатуру более удобной для использования**. Установите флажок **подчеркивать сочетания клавиш и ключи доступа** .

**Примечание.** Рекомендации, связанные со [специальными возможностями](inter-accessibility.md) , представлены в отдельной статье.

## <a name="design-concepts"></a>Принципы проектирования

### <a name="elements-of-keyboard-navigation"></a>Элементы навигации с помощью клавиатуры

Пользователи взаимодействуют с окном, используя клавиатуру, переходя к элементам управления, выполняя выделение и выполнение команд. Следующие элементы работают вместе, чтобы сделать это.

![снимок экрана — диалоговое окно "изменение цветов" ](images/inter-keyboard-image4.png)

Чтобы проиллюстрировать элементы навигации с помощью клавиатуры в следующем списке, мы будем называть это диалоговое окно.

-   **Фокус ввода.** Элемент управления с фокусом ввода получает большую часть ввода с клавиатуры. Фокус ввода обозначен пунктирным прямоугольником, называемым прямоугольником фокуса. Некоторые вводы с клавиатуры отправляются в элементы управления, у которых нет фокуса ввода, как описано далее.

    ![снимок экрана первой строки в диалоговом окне "изменение цветов" ](images/inter-keyboard-image5.png)

    Первый элемент управления "основной цвет" имеет фокус ввода, как показано пунктирным прямоугольником.

-   **Клавиши TAB и позиции табуляции.** Клавиша Tab является основным механизмом для перехода в пределах окна. Клавиша Tab посещает только те элементы управления, которые имеют позицию табуляции. У всех интерактивных элементов управления должны быть остановки перехода (если они не объединены в группу), а у неинтерактивных элементов управления, таких как метки, не должны.
-   **Последовательность табуляции.** Все элементы управления с позициями табуляции отображаются в порядке табуляции. Нажатие клавиши TAB перемещает фокус ввода к следующему элементу управления в последовательности табуляции, а нажатие клавиш SHIFT + TAB перемещает фокус ввода на предыдущий элемент управления.
-   **Группы управления.** Набор связанных элементов управления можно внести в группу и назначить одну позицию табуляции. Группы элементов управления используются для создания наборов, которые работают как один элемент управления, например переключатель. Их также можно использовать, если элементов управления слишком много и переход с помощью клавиши TAB недостаточно удобен.

    ![снимок экрана групп Basic и Custom Colors ](images/inter-keyboard-image6.png)

    Основные цвета и пользовательские цвета — это группы управления, что дает этому диалоговому окну пять табуляций. Существует много элементов управления, которые могут оказаться неэффективными без использования групп управления.

-   **Клавиши со стрелками.** Клавиши со стрелками перемещают фокус ввода между элементами управления в группе. При нажатии клавиши со стрелкой вправо фокус перемещается к следующему элементу управления в последовательности табуляции, а нажатие клавиши Стрелка влево перемещает фокус ввода на предыдущий элемент управления. Главная, конечная, работа и вниз также имеют ожидаемое поведение в группе. Пользователи не могут переходить из группы управления с помощью клавиш со стрелками.
-   **Кнопки по умолчанию.** Windows с кнопками команд и ссылками на команды имеют одну кнопку по умолчанию, обозначенную выделенной границей, которая является кнопкой, нажатой при нажатии клавиши ВВОД. По умолчанию назначена одна Командная кнопка или ссылка на команду по умолчанию. Однако кнопка по умолчанию перемещается, когда пользователь переходит на другую командную кнопку или ссылку на команду. Следовательно, любая кнопка или Командная ссылка с фокусом ввода также всегда является кнопкой по умолчанию.

    ![снимок экрана кнопок "ОК" и "Отмена" ](images/inter-keyboard-image7.png)

    Кнопка ОК обычно является кнопкой по умолчанию, как показано выделенной границей. Однако если пользователь должен был перейти на кнопку Отмена, он станет кнопкой по умолчанию и будет активирован с помощью клавиши ВВОД.

-   **Клавиши пробел, ввод и ESC.** Пробел активирует элемент управления с фокусом ввода, тогда как клавиша ВВОД активирует кнопку по умолчанию. Нажатие клавиши ESC отменяет или закрывает окно.
-   **Ключи доступа.** Ключи доступа используются для взаимодействия с элементами управления непосредственно вместо перехода по клавише TAB. Они объединяются с клавишей Alt и обозначаются подчеркнутой буквой в метке.
-   **Метки ключей доступа.** Хотя некоторые элементы управления содержат собственные метки, например кнопки команд, флажки и переключатели, другие элементы управления имеют внешние метки, такие как списки и представления в виде дерева. Для внешних меток ключ доступа назначается метке, а при вызове переходит к следующему элементу управления в последовательности табуляции. Кнопки с надписью ОК, отмена и закрыть не имеют назначенных клавиш, так как они вызываются с вводом и ESC.

    ![снимок экрана с метками "b" и "подчеркнутым" ](images/inter-keyboard-image8.png)

    Нажав клавиши Alt + B, можно перейти к выбранному базовому цвету, нажав клавиши ALT + D, кнопку определить пользовательские цвета, ввести вызов кнопки ОК и ESC вызывает отмену.

-   **Поведение ключа доступа.** Когда вызывается клавиша доступа и он назначается уникальным образом, происходит щелчок связанного элемента управления. Если назначение не является уникальным, связанный элемент управления получает фокус ввода. Если пользователь вводит один и тот же ключ доступа снова, фокус ввода передается следующему элементу управления в последовательности табуляции с тем же самым назначением.

Хотя этот механизм довольно сложен, он также довольно интуитивно понятен. Пользователи сразу же изменяют эти сведения, хотя многие могут объяснить, как они работают.

### <a name="keyboard-support-for-accessibility-and-advanced-users"></a>Поддержка клавиатуры для специальных возможностей и опытных пользователей

**В Windows проектирование для клавиатуры сводится к тому, чтобы обеспечить хорошо спроектированную навигацию по клавишам, клавиши доступа и сочетания клавиш для опытных пользователей.**

Чтобы обеспечить доступность функциональных возможностей программы для широкого спектра пользователей, включая тех, кто имеет инвалидность и нарушения, все интерактивные элементы пользовательского интерфейса должны иметь доступ к клавиатуре. Как правило, это означает, что наиболее часто используемые элементы пользовательского интерфейса доступны с помощью одного ключа доступа или сочетания клавиш, а для менее часто используемых элементов может потребоваться переход на новую вкладку или клавишу со стрелкой. Для этих пользователей полнота более важна, чем согласованность.

Чтобы обеспечить эффективную функциональность программы для опытных пользователей, часто используемые элементы пользовательского интерфейса также должны иметь сочетания клавиш для прямого доступа с клавиатуры. Опытные пользователи чаще применяют клавиатуру, так как с ее помощью можно быстрее вводить команды, при этом не требуется убирать руки с клавиатуры. Для таких пользователей эффективность и согласованность критически важна. Полнота важна только для наиболее часто используемых команд.

При проектировании доступа с клавиатуры для этих двух групп существуют небольшие различия, поэтому Windows предоставляет два независимых механизма прямого доступа к клавиатуре. Благодаря эффективному использованию ключей доступа и сочетаний клавиш можно предоставить программам эффективный и последовательный доступ с клавиатуры, который будет использовать все преимущества.

### <a name="access-keys"></a>Ключи доступа

Клавиши доступа обладают следующими характеристиками

-   Они используют клавишу ALT и буквенно-цифровую клавишу.
-   Они предназначены главным образом для специальных возможностей.
-   Они назначаются всем меню и большинству элементов управления диалоговых окон.
-   Они не предназначены для запоминания, поэтому указываются непосредственно в интерфейсе пользователя подчеркнутым символом метки соответствующего элемента управления.
-   Они действуют только в текущем окне и перемещают фокус на соответствующий пункт меню или элемент управления.
-   Они не назначаются последовательно, поскольку невозможно добиться всесторонней согласованности. Однако клавиши доступа следует назначать согласованно для часто используемых команд, особенно для кнопок фиксации.
-   Они локализуются.

**Так как ключи доступа не предназначены для запоминания, они назначаются символу, который находится на ранней странице метки, чтобы облегчить их поиск,** даже если имеется ключевое слово, которое появляется позже в метке.

**Правильно:**

![снимок экрана первого символа в метке подчеркнутый ](images/inter-keyboard-image9.png)

**Неправильно**:

![снимок экрана с двадцатью символом в конце строки ](images/inter-keyboard-image10.png)

В правильном примере ключ доступа назначается символу, который находится на раннем этапе метки.

### <a name="shortcut-keys"></a>Сочетания клавиш

Сочетания клавиш, напротив, имеют следующие характеристики.

-   Они главным образом используют сочетания клавиш CTRL и функциональных клавиш (в Windows также используются сочетание ALT и не буквенно-цифровой клавиши и клавиша Windows).
-   Они предназначены в основном для опытных пользователей.
-   Они назначаются только наиболее часто используемым командам.
-   Они предназначены для запоминания и описываются только в меню, подсказках и справке.
-   Они действуют во всем приложении, но не дают результата там, где они не применяются.
-   Их необходимо назначать согласованно, поскольку они запоминаются и не описываются напрямую.
-   Они не локализуются.

**Так как сочетания клавиш предназначены для запоминания, наиболее часто используемые сочетания клавиш лучше всего используют буквы из первых или наиболее запоминающих символов в ключевых словах команды,** например CTRL + C для копирования и CTRL + Q для запроса.

Неодинаковое значение для хорошо известных сочетаний клавиш — неплохое и вызывающее ошибки.

**Неправильно**:

![снимок экрана кнопки "вперед" с подчеркиванием "w" ](images/inter-keyboard-image11.png)

В этом примере сочетание клавиш CTRL + F является стандартным сочетанием клавиш Find, поэтому назначение его пересылке — неприятность и вероятность ошибок. Ctrl + W будет лучшим, запоминающимся.

Наконец, так как они предназначены для запоминания, **сочетания клавиш для конкретных приложений имеют смысл только для тех программ и компонентов, которые выполняются достаточно часто, чтобы пользователи могли запоминать о них.** Нечасто используемые программы и компоненты не требуют сочетаний клавиш. Например, программам установки и большинству мастеров не требуются специальные сочетания клавиш, а также нечасто используемые команды в приложении для повышения производительности.

### <a name="assigning-access-keys-in-dialog-boxes"></a>Назначение клавиш доступа в диалоговых окнах

По возможности присваивайте уникальные ключи доступа всем интерактивным элементам управления, за исключением тех, для которых обычно не назначены ключи доступа. Однако на английском языке имеется всего 26 символов. Некоторые символы могут не отображаться ни в одной из меток, и в них может не быть символов различения, что сокращает этот номер. Кроме того, необходимо предусмотреть несколько неназначенных символов для упрощения локализации. Следовательно, в одном диалоговом окне можно назначить только 20 уникальных ключей доступа.

Если имеется диалоговое окно с более чем 20 интерактивными элементами управления, либо не присваивайте клавиши доступа некоторым элементам управления, либо, в редких случаях, назначьте одинаковые ключи доступа.

![снимок экрана — диалоговое окно "Шрифт" ](images/inter-keyboard-image12.png)

При наличии этого множества интерактивных элементов управления не всем из них требуется назначенный ключ доступа.

Используйте следующую общую процедуру для назначения ключей доступа:

-   Сначала назначьте ключи доступа [кнопкам фиксации](glossary.md) и ссылкам на команды. Используйте стандартную таблицу назначений клавиш доступа при ее применении; в противном случае используйте первую букву первого слова.
-   Пропустите элементы управления, которым не назначены ключи доступа.
-   Назначьте уникальные ключи доступа оставшимся элементам управления (начиная с наиболее часто используемых):
    -   Если это возможно, назначьте ключ доступа в соответствии со стандартной таблицей назначений клавиш доступа.
    -   В противном случае:
        -   Предпочитать символы, которые отображаются на ранней метке, в идеале первый символ первого или второго слова.
        -   Предпочтение конкретному согласныму или гласному, например «x», в «Exit».
        -   Предпочитать символы с широкими ширинами, например w, m и прописные буквы.
        -   Старайтесь не использовать символы, которые затрудняют отображение подчеркивания, например, буквы, которые находятся в одном пикселе, буквы с нижними выносными буквами и буквы рядом с буквой с нижним выносным элементом.
-   Если не все элементы управления могут иметь уникальные ключи доступа (начинаются с наименее часто используемых):
    -   Если имеются группы связанных элементов управления, например:
        -   Один набор переключателей
        -   Набор связанных флажков
        -   Набор связанных элементов управления в поле группы

Назначьте ключи доступа для меток группы, а не отдельных элементов управления. Как правило, это делается обратно. (При этом убедитесь, что для этих элементов управления определена группа управления.)

-   Если по-прежнему не все элементы управления могут иметь уникальные ключи доступа:
    -   Вы можете назначить неуникальные ключи доступа, если:
        -   В противном случае элементы управления могли бы оказаться слишком сложными для перехода к.
        -   Неуникальные ключи доступа не конфликтуют с ключами доступа часто используемых элементов управления.
    -   В противном случае доступ к оставшимся элементам управления можно получить с помощью навигации по клавишам Tab и клавиши со стрелкой.

![снимок экрана групп с разными ключами доступа ](images/inter-keyboard-image13.png)

В этом примере есть повторяющиеся элементы управления, поэтому ключи доступа назначаются группам переключателей.

### <a name="preventing-accidental-commands"></a>Предотвращение случайных команд

Если окно, отображаемое из контекста (не инициированное пользователем), не задает фокус ввода, то существует хороший шанс, что это окно будет получать входные данные, предназначенные для другого окна. Кроме того, ключи доступа вступают в силу при нажатии без нажатия клавиши ALT, если в диалоговом окне нет элементов управления, которые принимают текстовые данные (например, текстовые поля и списки). Поэтому в следующем примере нажатие клавиши "r" активирует кнопку "Перезагрузить сейчас".

Очевидно, что такие входные данные могут иметь значительные непредвиденные последствия.

**Неправильно**:

![снимок экрана кнопки "Перезагрузить сейчас", "r" подчеркнутый ](images/inter-keyboard-image14.png)

В этом примере введите текст с пробелом, «r» или «ввод случайно перезапускает Windows».

Конечно, лучшим решением этой проблемы не является кража фокуса ввода. Вместо этого следует либо отбавиться от [кнопки панели задач](winenv-taskbar.md) программы, либо отобразить уведомление, чтобы привлечь внимание пользователя.

Тем не менее, если необходимо отобразить такое окно, лучше не назначать кнопки и ключи доступа по умолчанию и передавать начальные данные фокуса элементу управления, отличному от кнопки фиксации.

**Правильно:**

![снимок экрана: кнопка "перезапустить", "r" не подчеркнута ](images/inter-keyboard-image15.png)

В этом примере очень труднее выполнять случайный перезапуск Windows.

**Если вы выполняете только шесть вещей...**

1.  Проектирование хорошей навигации с помощью клавиатуры с разумным порядком табуляции и соответствующими группами элементов управления, начальным фокусом ввода и кнопками по умолчанию.
2.  Назначение клавиш доступа ко всем меню и большинству элементов управления.
3.  Назначьте клавиши доступа символу, который отображается на ранней странице метки, чтобы облегчить их поиск.
4.  Назначьте сочетания клавиш наиболее часто используемым командам.
5.  Попробуйте назначить сочетания клавиш для первых или наиболее запоминающих символов в ключевых словах.
6.  Присвойте одинаковому значению хорошо известные сочетания клавиш.

## <a name="guidelines"></a>Рекомендации

### <a name="interaction"></a>Взаимодействие

-   **Не используйте клавишу Shift для изменения команд в меню или диалоговых окнах.** Это необнаружение и непредвиденные последствия.

    **Неправильно**:

    ![снимок экрана с диалоговым окном "подтвердить замену папки" ](images/inter-keyboard-image16.png)

    В этом примере, начиная с Windows XP, при удерживании клавиши Shift значение Да заменяется на все.

-   **Не отключайте элемент управления с фокусом ввода. Это может помешать окну получать ввод с клавиатуры.** Вместо этого перед отключением элемента управления с фокусом ввода переместите фокус ввода на другой элемент управления.
-   **Если окно отображается вне контекста, потенциально удивительного пользователя, может потребоваться предотвратить значительные непредвиденные последствия.**
    -   Не присваивайте кнопку по умолчанию.
    -   Не присваивайте ключи доступа.
    -   Присвойте начальным входным данным элемент управления, отличный от кнопки фиксации.

### <a name="keyboard-navigation"></a>Навигация с помощью клавиатуры

-   **Всегда показывать индикатор фокуса ввода. Исключение:** вы можете временно подавить индикатор фокуса ввода, если:
    -   Индикатор фокуса ввода визуально отвлекается от него (как в случае с большим представлением списка, не представленным в подробном представлении).
    -   Скорее всего, перед использованием клавиши Ввод стоит другой ввод с клавиатуры, например ALT или клавиша со стрелкой.
    -   Индикатор фокуса ввода отображается при любом вводе с клавиатуры.
-   **Назначайте начальные входные данные элементу управления, с которым пользователи с наибольшей вероятностью взаимодействуют первыми,** что часто является первым интерактивным элементом управления. Если первое интерактивное управление не является хорошим выбором, рассмотрите возможность изменения макета окна.
-   **Присвойте вкладкам значения табуляции для всех интерактивных элементов управления, включая поля редактирования только для чтения. Отличи**
    -   Групповые наборы связанных элементов управления, которые ведут себя как один элемент управления, например переключатели. Такие группы имеют одну позицию табуляции.
    -   Правильно содержат группы, чтобы клавиши со стрелками циклически переходили в пределах группы и остаться в группе.
-   **Порядок табуляции должен следовать за чтением, который обычно проходит слева направо, сверху вниз.** Рассмотрите возможность создания исключений для часто используемых элементов управления, поместив их раньше в последовательность табуляции. На вкладке должны быть циклически перебираться все позиции табуляции в обоих направлениях без остановки.
-   **В пределах табуляции порядок клавиш со стрелками должен передаваться слева направо, сверху вниз,** без исключений. Клавиши со стрелками должны циклически проходить по всем элементам в обоих направлениях без остановки.
-   **Представьте кнопки фиксации в следующем порядке:**
    -   Ладно, \[ \] /ЕС
    -   \[Не \] /но
    -   Отменить
    -   Применить (при наличии)

где \[ это \] и не \[ делать, это не \] относится к конкретным ответам на основную инструкцию.

-   **Выберите самый безопасный (чтобы предотвратить утрату данных или доступ к системе), а также наиболее безопасную командную кнопку или ссылку на команду по умолчанию.** Если безопасность и безопасность не являются факторами, выберите наиболее вероятный или удобный ответ.
-   **Навигация с помощью клавиатуры не должна изменять значения элементов управления или выдавать сообщение об ошибке.** Никогда не требовать от пользователей изменять начальное значение элемента управления во время навигации. Вместо этого инициализируйте элементы управления, которые проверяются на выходе с допустимыми значениями, и проверяют значение элемента управления только при его изменении.

### <a name="menu-access-keys"></a>Ключи доступа к меню

-   **Назначьте ключи доступа всем пунктам меню.** Исключений нет.
-   **Для элементов динамического меню (например, недавно использовавшихся файлов) назначьте ключи доступа числовыми.**

    ![снимок экрана элементов меню с числовыми ключами доступа ](images/inter-keyboard-image17.png)

    В этом примере программа Paint в Windows назначает числовые ключи доступа недавно использованным файлам.

-   **Назначение уникальных ключей доступа на уровне меню.** Ключи доступа можно использовать в различных уровнях меню.
-   **Упрощение поиска ключей доступа:**
    -   Для наиболее часто используемых пунктов меню выберите символы в начале первого или второго слова Метки, лучше всего первый символ.
    -   Для редко используемых пунктов меню выберите буквы, которые являются отличительными согласными или гласными в метке.

### <a name="dialog-box-access-keys"></a>Ключи доступа диалогового окна

-   **Везде, где это возможно, следует назначать уникальные ключи доступа всем интерактивным элементам управления или их меткам.** [Текстовые поля только для чтения](ctrl-text-boxes.md) — это интерактивные элементы управления (так как пользователи могут прокручивать их и копировать текст), поэтому они пользуются преимуществами клавиш доступа. **Не присваивайте ключи доступа:**
    -   **Кнопки ОК, отмена и закрыть.** Для ключей доступа используются клавиши ВВОД и ESC. Однако всегда назначайте ключ доступа для элемента управления, который означает ОК или Отмена, но имеет другую метку.

        ![снимок экрана диалогового окна с кнопками "Да" и "нет" ](images/inter-keyboard-image18.png)

        В этом примере для кнопки положительной фиксации назначен ключ доступа.

    -   **Метки групп.** Как правило, отдельным элементам управления в группе присваиваются ключи доступа, поэтому метке группы не требуется. Однако при наличии нехватки ключей доступа назначьте ключ доступа к метке группы, а не к отдельному элементу управления.
    -   **Универсальные кнопки справки,** доступ к которым осуществляется с помощью F1.
    -   **Метки ссылок.** Часто бывает слишком много ссылок для назначения уникальных ключей доступа, а подчеркивания ссылок скрывают символы подчеркивания клавиш доступа. Пользователи получают доступ к ссылкам с помощью клавиши TAB.
    -   **Имена вкладок.** Переключение между вкладками осуществляется с помощью клавиш CTRL + TAB и CTRL + SHIFT + TAB.
    -   **Кнопки обзора с меткой "...".** Им нельзя назначить ключи доступа уникальным образом.
    -   **Непомеченные элементы управления,** такие как элементы управления "Счетчик", кнопки графического команд и непомеченные элементы управления раскрытием.
    -   **Статический текст или метки без метки для элементов управления, которые не являются интерактивными,** например индикаторы выполнения.

-   **Сначала назначьте ключи доступа для кнопки фиксации, чтобы убедиться, что они имеют стандартные ключевые назначения.** Если стандартное назначение ключа не используется, используйте первую букву первого слова. Например, клавиша доступа "Да" и "нет кнопки фиксации" всегда должны быть "Y" и "N", независимо от других элементов управления в диалоговом окне.
-   **Для кнопок с отрицательной фиксацией (отличных от Cancel) с фразой как "не" назначьте ключ доступа "n" в "не".** Если не имеет фразы в виде "не", используйте стандартное назначение клавиши доступа или назначьте первую букву первого слова. Таким образом, все запреты и не имеют единообразного ключа доступа.
-   **Чтобы упростить поиск ключей доступа, назначьте клавиши доступа символу, который находится на ранней странице метки,** в идеале первый символ, даже если имеется ключевое слово, которое появляется в метке позже.
-   **Назначьте не более 20 ключей доступа,** поэтому для упрощения локализации требуется несколько неназначенных символов.
-   **Если для назначения уникальных ключей доступа существует слишком много интерактивных элементов управления, то можно назначить неуникальные ключи доступа,** если:
    -   В противном случае элементы управления могли бы оказаться слишком сложными для перехода к.
    -   Неуникальные ключи доступа не конфликтуют с ключами доступа часто используемых элементов управления.
-   **Не используйте строки меню в диалоговых окнах.** В этом случае сложно назначить уникальные ключи доступа, так как элементы управления диалогового окна и пункты меню имеют одни и те же символы.

### <a name="shortcut-keys"></a>Сочетания клавиш

-   **Назначьте сочетания клавиш наиболее часто используемым командам.** Нечасто используемые программы и компоненты не нуждаются в сочетаниях клавиш, так как вместо этого пользователи могут использовать ключи доступа.
-   **Не делайте сочетание клавиш единственным способом выполнения задачи.** Пользователи также должны иметь возможность использовать мышь или клавиатуру со вкладками, стрелками и клавишами доступа.
-   **Не назначайте разные значения для хорошо известных сочетаний клавиш.** Так как они зафиксированы, неоднородное значение для хорошо известных сочетаний клавиш — беспроблемные и подверженные ошибкам.
-   **Не пытайтесь назначить сочетания клавиш для программы на уровне системы.** Сочетания клавиш программы вступят в действие только в том случае, если программа имеет фокус ввода.
-   **Задокументируйте все сочетания клавиш.** Ярлыки документов в элементах строки меню, подсказки на панели инструментов и одна статья справки, в которой содержатся все используемые сочетания клавиш. Это помогает пользователям узнать о назначении сочетаний клавиш, которые не должны быть секретными.

    -   **Исключение:** Не отображать сочетания клавиш в контекстных меню. Контекстные меню не отображают сочетания клавиш, так как эти меню оптимизированы для повышения эффективности.

    ![снимок экрана с подсказкой для сочетания клавиш с полужирным шрифтом ](images/inter-keyboard-image19.png)

    Сочетание клавиш описано в подсказке.

-   **Если программа назначает множество сочетаний клавиш, предоставьте возможность настраивать назначения.** Это позволит пользователям переназначить конфликтующие сочетания клавиш и выполнить миграцию из других продуктов. Большинство программ не назначайте достаточно сочетаний клавиш, чтобы эта функция была необходима.

### <a name="choosing-shortcut-keys"></a>Выбор сочетаний клавиш

-   **Для хорошо известных сочетаний клавиш используйте стандартные назначения.**
-   **Для нестандартных назначений ключей используйте следующие Рекомендуемые сочетания клавиш для более часто используемых команд.** Рекомендуется использовать эти сочетания клавиш, так как они не конфликтуют с хорошо известными сочетаниями клавиш и просты в использовании.
    -   Ctrl + G, J, K, L M, Q, R или T
    -   Ctrl + любое число
    -   F7, F8, F9 или F12
    -   Shift + F2, F3, F4, F5, F7, F8, F9, F11 или F12
    -   Alt + любой ключ функции, кроме F4
-   **Используйте следующие Рекомендуемые сочетания клавиш для редко используемых команд.** Эти сочетания клавиш не имеют конфликтов, но труднее нажать на них, чтобы они часто требовали две руки.
    -   Ctrl + любой ключ функции, кроме F4 и F6
    -   CTRL + SHIFT + любая буква или цифра
-   **Легко запоминать часто используемые сочетания клавиш:**
    -   Используйте буквы вместо чисел или функциональных клавиш.
    -   Попробуйте использовать букву, которая находится в первом слове или в наиболее подсчитанном символе в ключевых словах команды.
-   **Используйте функциональные клавиши для команд, которые имеют небольшой результат,** например команды, применяемые к выбранному объекту. Например, F2 Переименование выбранного элемента.
-   **Сочетания клавиш CTRL используются для команд, которые имеют большой результат,** например команды, применяемые ко всему документу. Например, CTRL + S сохраняет текущий документ.
-   **Используйте сочетания клавиш Shift для команд, расширяющих или дополняющих действия стандартной сочетания клавиш.** Например, сочетание клавиш ALT + TAB циклически переключается на открытые основные окна, а клавиши ALT + SHIFT + TAB — в противоположном порядке. Аналогично, при нажатии клавиши F1 отображается справка, а при нажатии клавиши Shift + F1 отображается контекстная справка.
-   **При использовании клавиш со стрелками для перемещения или изменения размера элемента используйте клавиши CTRL + стрелка для более детального управления.**

### <a name="choosing-shortcut-keys-what-not-to-do"></a>Выбор сочетаний клавиш (что не нужно делать)

-   **Не следует различать местоположения ключей.** Например, Windows может различать левую и правую клавиши Shift, Alt, CTRL, [логотип Windows](glossary.md)и [приложения](glossary.md), а также клавиши на цифровой клавиатуре. Назначение поведения только одному расположению ключа — путаница и непредвиденная ситуация.
-   **Не используйте клавишу модификатор логотипа Windows для сочетаний клавиш программы.** Ключ эмблемы Windows зарезервирован для использования в Windows. Даже если сочетание клавиш Windows не используется в Windows сейчас, оно может быть в будущем.
-   **Не используйте ключ приложения в качестве модификатора сочетания клавиш.** Вместо этого используйте клавиши CTRL, Alt и Shift.
-   **Не используйте сочетания клавиш, используемые в Windows для сочетаний клавиш.** Это приведет к конфликту с сочетаниями клавиш системы Windows, если программа имеет фокус ввода.
-   **Не используйте сочетание клавиш ALT + буквенно-цифровые сочетания клавиш.** Такие сочетания клавиш могут конфликтовать с ключами доступа.
-   **Не используйте следующие символы для сочетаний клавиш:** @ $ {} \[ \] \\  ~  \| ^ '  < >. Для этих символов требуются разные сочетания клавиш для разных языков или они зависят от языкового стандарта.
-   **Избегайте использования сложных сочетаний клавиш, например,** трех или более ключей (например, CTRL + ALT + пробел) или клавиш, которые находятся далеко от клавиатуры (например, CTRL + F5). Используйте простые сочетания клавиш для часто используемых команд.
-   **Не используйте сочетания клавиш CTRL + ALT,** так как Windows интерпретирует это сочетание в некоторых языковых версиях как клавиш AltGr ключ, который создает буквенно-цифровые символы.

### <a name="keyboard-and-mouse-combinations"></a>Сочетания клавиш и кнопок мыши

-   Для ссылок используйте клавиши Shift + щелчок для перемещения по новому окну и Ctrl + щелчок для перехода с помощью новой вкладки. Этот подход согласуется с Windows Internet Explorer.

## <a name="documentation"></a>Документация

При ссылке на клавиатуру:

-   Используйте экранную клавиатуру для обращения к представлению клавиатуры на экране, на котором пользователь касается входных символов.
-   Присвойте сочетания клавиш, начиная с клавиши модификатора. Показывать клавиши модификатора в следующем порядке: эмблема Windows, приложение, CTRL, Alt, Shift. Если используется модификатор Нумпад, вставьте его непосредственно перед ключом, который он изменяет.
-   Не используйте все прописные буквы для клавиш клавиатуры. Вместо этого следует следовать капитализации, используемой стандартными клавиатурами, или строчными буквами, если ключ не помечен на клавиатуре.
    -   Для сочетаний клавиш в алфавитном порядке используйте прописную букву.
    -   Произношение страницы вверх, страницы вниз, на экране печати и с блокировкой прокрутки.
    -   Произношение знака плюс, знака минус, дефиса, точки и запятой.
    -   Для клавиш со стрелками используйте стрелку влево, стрелка вправо, стрелка вверх и стрелка вниз. Не используйте графические метки для клавиш со стрелками.
    -   Используйте ключ эмблемы Windows и ключ приложения для ссылки на ключи, помеченные значками. Не используйте для этих ключей графические метки.

**Правильно:**

пробел, TAB, ввод, страница вверх, Ctrl + Alt + Del, ALT + W, Ctrl + знак "плюс"

**Неправильно**:

ПРОБЕЛ, TAB, ввод, стр. вверх, CTRL + ALT + DEL, Alt + w, CTRL + +

-   Укажите сочетания клавиш со знаком "плюс" без пробелов.

**Правильно:**

Ctrl + A, Shift + F5

**Неправильно**:

Ctrl + A, Shift + F5

-   Чтобы отобразить сочетание клавиш, включающее знаки препинания, для которых требуется клавиша SHIFT, например вопросительный знак, добавьте к сочетанию клавишу Shift и присвойте имя или символ нажатого клавиши. Использование имени несмещенного ключа, такого как 4, а не $, может привести к путанице в работе пользователей или даже неверной. Например,? знаки и/не всегда сдвигаются на каждой клавиатуре.

**Правильно:**

Ctrl + Shift +?, Ctrl + Shift + \* , Ctrl + Shift + запятая

**Неправильно**:

Ctrl + Shift +/, CTRL +?, CTRL + SHIFT + 8, CTRL +\*

-   В первую очередь следует использовать ключ и с именем ключа, если это необходимо для ясности, например клавишу F1. Во всех последующих ссылках используйте ключ только по имени, например, нажав клавишу F1.
-   Дополнительные сведения см. в разделах доступ к ключам и сочетания клавиш в разделе Программирование и другая техническая документация. Не используйте сочетания клавиш, горячие клавиши и сочетания клавиш. В любом другом случае используйте сочетание клавиш, особенно в пользовательской документации.

При ссылке на взаимодействие:

-   Используйте нажатие клавиш, not Депресс, Зачеркнутый, попадание или тип, если нажатие и немедленное освобождение ключа инициирует действие в программе или выполняет навигацию внутри документа или пользовательского интерфейса.
-   Используйте тип, а не ввод, чтобы направить пользователей к вводу текста.
-   Используйте в ситуациях, когда нажатие клавиши может вызвать путаницу, например при ссылке на тип ключа, например клавиши со стрелками или функциональные клавиши. В таких случаях при нажатии клавиши может показаться, что пользователи должны будут одновременно нажимать все ключи.
-   Используйте удержание при нажатии и удержании ключа, например, клавиши-модификатора.
-   Не используйте клавишу как синоним для щелчка.

Примеры:

-   Введите свое имя и нажмите клавишу ВВОД.
-   Нажмите клавиши CTRL + F, а затем введите текст, который требуется найти.
-   Чтобы сохранить файл, нажмите клавишу Y.
-   Чтобы переместить точку вставки, используйте клавиши со стрелками.
