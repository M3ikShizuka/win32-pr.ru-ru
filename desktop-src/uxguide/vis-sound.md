# <a name="sound"></a>Звук

> [!NOTE]
> это руководство по проектированию было создано для Windows 7 и не обновлялось для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

*Звук* — это звуковой элемент пользовательского интерфейса. При правильном использовании звук может быть эффективной формой связи, которая устанавливает связь между текстовом и эмоциональном с пользователями. Звуки можно использовать отдельно или в качестве дополнения к визуальному пользовательскому интерфейсу. Например, Добавление звукового эффекта к уведомлению увеличивает вероятность того, что оно будет замечено, особенно если пользователь не смотрит на экран при возникновении события.

![снимок экрана с диалоговым окном "звук" ](images/vis-sound-image1.png)

На вкладке Звуки элемента панели управления звуком пользователи могут вносить изменения в системные звуки.

В этой статье рассматривается использование звуков в рамках программы в качестве ответа на события и действия пользователя, а также интеграция элемента управления звуком с Windows. В нем не рассматривается использование музыки и речи.

**Примечание.** Рекомендации, связанные с [уведомлениями](mess-notif.md) и [фирменной символикой](exper-branding.md) , представлены в отдельных статьях.

## <a name="is-this-the-right-user-interface"></a>Это правильный пользовательский интерфейс?

Чтобы решить, следует ли использовать звук, учитывайте следующие вопросы.

-   **Существует ли понятное преимущество для использования звука?** Поскольку недостатки использования звука могут легко перевешивают преимущества, используйте звук только при наличии четкого преимущества.
-   **Подходит ли использование звука?** Использует ли звук, чтобы привлечь внимание к вещи, которые достойной внимание? Пропустил ли пользователь звук, если он отсутствовал? Сосредоточьтесь на звуках, которые сообщают пользователям, скорее всего изменяют их поведение или предоставляют полезные отзывы.
-   **Используется ли отменяющий звук?** Есть ли у вас частые и жарринг звуки? Могут ли пользователи уменьшить системный том или громкость программы в результате использования звука?
-   **Используется ли звук в качестве основной формы связи?** Во многих случаях, например для пользователей с определенным уровнем потери слуха, звук не следует использовать в качестве основного средства связи. Звук более эффективен в качестве дополнения к другим средствам связи (например, к тексту или визуальным компонентам).
-   **Являются ли основные целевые пользователи ИТ-специалистами?** Звук обычно неэффективен для задач, нацеленных на ИТ-специалистов, так как многие задачи выполняются автоматически. Кроме того, звук не масштабируется для них, представим выполнение сотен задач за раз и получение звуков после их завершения или сбоя.

## <a name="design-concepts"></a>Принципы проектирования

Как правило, звук достигает любой или всех следующих целей.

-   **Уведомление.** Звук может быть связан с конкретными событиями. Например, звук "Новая почта" сообщает пользователям о поступлении почты, не нарушая их текущую задачу.
-   **Оставлять.** Звук может предоставить отзыв о конкретных действиях пользователя. Например, при отправке ползунка в элементе управления "Громкость" незаметное звуковое значение предоставляет отзыв об уровне текущего параметра.
-   **Фирменной символики.** Звук может быть связан с конкретным содержимым для фирменной символики продукта, приложения или службы. Windows использует для запуска операционной системы звук таким образом.
-   **Сфере.** Звук часто используется для улучшения развлекательных продуктов и для повышения привлекательности продукта. Например, большинство игр, обучающих приложений и потребительских продуктов используют звук для развлечения пользователей и улучшения их работы.

Некоторые звуки могут выполнять несколько этих задач одновременно. например, при запуске Windows звуковое сопровождение означает, что процесс запуска завершен и рабочий стол готов к использованию. Кроме того, он обеспечивает эффективную форму фирменной символики продукта и даже мгновенно применяет пользователей.

Все звуки, которые не удовлетворяют этим целям, скорее всего, будут устранены.

### <a name="inappropriate-use-of-sound"></a>Недопустимое использование звука

**Несмотря на преимущества звука, подходящее использование звука требует значительных ограничения, что в противном случае может сделать программу ненужной.** Пользователи будут полностью отключать свой звук, если они станут раздражены частями, повторяющимися, жарринг, нарушениями, плохо спроектированными звуками. в части это обусловлено тем, что по своей природе звук требует внимания и трудно пропускаться. Советы по поиску разумного баланса см. в разделе [рекомендации по проектированию звука](#sound-design).

Поскольку недостатки использования звука могут легко перевешивают преимущества, используйте звук только при наличии четкого преимущества. **Если вы сомневаетесь, не используйте звук.**

### <a name="make-sound-supplemental"></a>Сделать звук дополнительным

Даже если звук используется соответствующим образом, существует множество ситуаций, в которых звук может не действовать для всех пользователей:

-   Некоторые пользователи могут работать в бесшумной среде, где звуки не могут быть слышны.
-   Некоторые пользователи могут работать в тихом окружении, требующем, чтобы звук был выключен или установлен на низком томе.
-   Некоторые пользователи могут столкнуться с нарушениями слуха или потерей.
-   На компьютере могут отсутствовать динамики.

По этим причинам **звук, используемый для уведомлений и отзывов, никогда не должен быть единственным методом связи,** а должен дополнять визуальные или текстовые подсказки.

### <a name="desirable-characteristics-of-sound"></a>Желательные характеристики звука

Как правило, звук должен быть следующим:

-   от средней до высокой частоты (600 Герц \[ Гц \] до 2 килохертз \[ кГц \] ).
-   Short (меньше одной секунды).
-   Soft или умеренный объем в томе.
-   имеет.
-   приятный, а не Будильник или жарринг.
-   не текстовом.
-   не повторяются.

При использовании звука меньше. **Идеальный звуковой результат — это тот, который пользователи едва заметить, но они пропустили, если они отсутствовали.**

**Распространенное заблуждение заключается в том, что звуки для критических событий должны быть громкыми и жарринг для получения внимания пользователя.** Это не так, потому что звук на самом деле является дополнительным средством взаимодействия. В случае критического сообщения об ошибке, его изображение (возможно, в модальном диалоговом окне), его значок (значок ошибки), а также текст и тон все вместе, чтобы сообщить характер ошибки. эффективный звук ошибки может быть немного громче, чем обычно Windowsный звук, но не должен быть значительно громче.

### <a name="characteristics-of-windows-sounds"></a>характеристики Windows звуков

помимо этого общего вызова для минимизации, Windows sound aesthetic использует световой, чистый и аириный звук, с мягкими эффектами затухания и исчезновения (мягкими краями) для предотвращения внезапной, жарринг, перкуссиве эффектов. Они разработаны так, чтобы быть неочевидными, доброй и согласными. Windowsные звуки используют эхо, переглаголы и обравенство для достижения естественного внешнего поведения.

звуковая схема по умолчанию для Windows обычно не использует instrumental или распознаваемые ежедневные звуки, которые имеют более конкретные или музыкальные данные. Примерами таких звуков являются музыкальные инструменты, такие как пианино или противоречие, животные, шумы в окружающей среде, речь, голоса, звуковые эффекты, такие как фильмы или другие звуки людей. кроме того, Windowsные звуки не предназначены для восприятия как музыка (то есть, например, мелодиес несколько заметок). это делает Windows звучит функционально отличающимся от других типов звуков.

так как Windowsные звуки были профессионально спроектированы для того, чтобы иметь желательные характеристики и полагаться на обширную аудиторию, **рассмотрите возможность использования этих встроенных Windowsных звуков при необходимости.**

### <a name="designing-your-own-sounds"></a>Разработка собственных звуков

Если необходимо создать собственные звуки, создайте их с помощью описанных выше характеристик. Стремитесь сделать их дополненными связанными с ними задачами или событиями.

Понимание того, что создание исходных звуков сложно сделать особенно для звуков, предназначенных для широкой аудитории. Звук может быть поларизинг элементом дизайна. Для каждого пользователя, обожает звук, будет много, кто его не нравится.

**Разработайте звуки программы как группу, чтобы они были взаимосвязаны с вариациями темы.** Средства аудита программы должны быть согласованы с визуальным интерфейсом. Кроме того, «тоновый» звук должен координироваться с помощью [тона текста](text-style-tone.md). Рассмотрите, как текст с приятный, естественный тон может быть достаточно подсветкой, когда Харша, сигнализация о звуках.

**Если вы выполните только четыре вещи...**

1.  Используйте звук с ограничения, чтобы обеспечить четкое общее преимущество пользователя. Если вы сомневаетесь, не используйте звук.
2.  при необходимости используйте встроенные Windows звуки.
3.  Если вы разрабатываете собственные звуки, убедитесь, что они имеют желаемые характеристики звука и, как вариант, в теме.
4.  Не думайте, что звуки должны быть громкыми и жарринг для получения внимания пользователя.

## <a name="guidelines"></a>Рекомендации

### <a name="usage"></a>Использование

-   **Используйте звук с ограничения** , чтобы обеспечить четкое общее преимущество пользователя. Сосредоточьтесь на звуках, которые сообщают пользователям, скорее всего изменяют их поведение или предоставляют полезные отзывы. Если вы сомневаетесь, не используйте звук.
-   **Выберите звук и его характеристики в зависимости от того, как он используется.** Описание каждого шаблона использования см. в таблице, приведенной в предыдущем разделе.
-   **Для уведомлений и отзывов не используйте звук в качестве единственного способа связи.** Вместо этого используйте звук в качестве дополнительного метода, чтобы усилить визуальные или текстовые подсказки. Это гарантирует, что пользователи смогут видеть информацию, если они не прослушивают звук.
-   **Не воспроизводят звуковые или Харша звуки часто.** Это не является обязательным и приводит к ухудшению взаимодействия с пользователем. Чем чаще звук воспроизводится, тем менее «ненавязчиво». Для привлечения внимания звуки не должны быть громкыми или Харша.
-   **Не выдавать звуковой сигнал.** Звуковые сигналы не подходят для современных программ. Звуковые сигналы не могут иметь определенные значения, и пользователи не могут управлять ими.
    -   **Исключение:** Критические системные функции могут подавать звуковой сигнал, чтобы предупредить пользователей о ситуациях, которые они должны посетить немедленно, например, критически низкий уровень зарядки аккумулятора.

### <a name="playback"></a>Воспроизведение

-   **Не повторяйте звук более двух раз подряд.**
-   **Для последовательной последовательности связанных событий звука воспроизводится звук только при первом событии.** Старайтесь не использовать несколько звуков, так как они могут конфликтовать друг с другом или иным образом привести к неприятные пользователя.

### <a name="sound-selection"></a>Выбор звука

-   **Выберите приятный звуки.** Не используйте неприятные, предупреждающие звуки, такие как звуковые сигналы, сбои и нарушения.
-   **Используйте короткие звуковые сигналы** (менее одной секунды).
-   **используйте звуки примерно на том же томе, что и типичный Windowsный звук.** Пользователям не нужно преобразовывать том при запуске компьютера или программы, особенно в общедоступных средах, таких как собрания и презентации. звуковые файлы Microsoft Windows находятся в папке «носитель» в папке «Windows».
-   **Не выбирайте звуки, требующие локализации.** Это можно сделать с помощью звуков, не использующих речь, или имеющих зависящие от культуры значения или различия.

### <a name="windows-system-sounds"></a>Windows системных звуков

-   **при необходимости используйте встроенные Windows системные звуки.**
-   **Выберите использование системных звуков на основе их связанного значения, а не только самого звука.** Системные звуки должны использоваться постоянно.

### <a name="sound-design"></a>Разработка звука

При создании собственных звуков:

-   **Создавайте звуки с желаемыми характеристиками звука.**
-   **Создавайте звуки в обычном диапазоне до высоких частот (от 600 Гц до 2 кГц).** Не используйте низкие частоты, так как они находятся дальше, сложнее размещать и могут быть оповещены.
-   **установите относительную амплитуду обычных звуков на уровень типичного Windows звука.** Windows звучит соответствующим образом для домашних и рабочих сред. Использование различных уровней для звуков приведет к принудительной корректировке тома пользователями.
    -   Установите важные звуки, которые будут немного громче. Такие звуки включают в себя завершение действий, сбои действий и важные системные события.
    -   Установка часто встречающихся звуков в несколько мягких. К ним относятся ФИС, звук фирменной символики и звуковые эффекты.
-   **выберите "звуки" в соответствии со значением Windows звуков.** чтобы создать пользовательскую версию Windowsного звука, сохраните тот же шаг и интервал, но измените оркестрации или тимбре. не назначайте разные значения для звуковых сигналов с похожими и интервалами, как Windows звуки.
-   **Разработайте звуки программы так, чтобы они были взаимосвязаны с вариантами темы.** Средства аудита программы должны быть согласованы с визуальным интерфейсом.
    -   **Разработка сцен переходов и звуковых переходов вместе.** Например, если сцена постепенно плавно исчезает, все звуки также должны постепенно передаваться. Не насмаркуйте визуальные переходы, используя внезапное переключение звука.
-   **Звук должен иметь формат WAV.** Рекомендуется использовать формат PCM для 16-разрядных (44,1 кГц) стерео несжатого импульсного кода. Если размер файла важен, используйте сжатый или монаурал (моно) формат, но имейте в виду, что существует легко определяемая степень потери качества, которая может поразиться на вашем приложении плохо.

### <a name="mixing"></a>Объединение

-   **Не устанавливайте том или отключите в программе элементы управления.** вместо этого разрешите пользователям управлять параметрами относительных томов между приложениями с помощью микшера Windowsого тома. Если программа имеет регулятор громкости, будет несколько мест, где пользователи настраивают свои параметры, что может привести к путанице.

    ![снимок экрана микшера громкости Windows ](images/vis-sound-image8.png)

    микшер громкости Windows позволяет пользователям управлять основным параметром тома, а также томом для каждой программы, которая в данный момент воспроизводит звук.

<!-- -->

-   **Исключение:** Если основная цель программы — воспроизведение аудио-или видеозаписи или создание видео, может оказаться полезным наличие в программе регулятора громкости. Используйте ползунок для этой цели и немедленно предпримете отзыв, когда пользователь изменит том.

### <a name="windows-integration"></a>интеграция Windows

-   **зарегистрируйте звуки программы в реестре Windows звуках.** это позволит Windows микшера тома добавить ползунок для программы.
-   **Зарегистрируйте настраиваемые звуковые события программы.** это позволит отобразить их в элементе панели управления "звук" Windows. Создайте следующий ключ для каждого пользовательского звукового события: HKEY \_ текущий \_ пользователь \| аппевентс \| метки событий \| EventName = имя события.
-   **Не выполнять звуки для звуковых событий программы.** Вместо этого укажите звуки, которые будут воспроизводиться с помощью записей реестра. Это позволяет пользователям настраивать звуковые события с помощью элемента панели управления звуком.
    -   **Исключение:** Вы можете выполнять звуки, используемые для фирменной символики.
-   **Не следует предоставлять пользователям возможность настраивать звуки в параметрах программы.** вместо этого воспользуйтесь элементом панели управления "Windows звуки" для этой цели.
-   **Рекомендуется не назначать звуковым сигналам часто возникающие события по умолчанию.** Не нужно, чтобы пользователи настроили свой способ из неприятного начального опыта.

### <a name="directsound-programming-issues"></a>Проблемы программирования DirectSound

-   Для программ DirectSound, имеющих свой собственный регулятор громкости, **по умолчанию установите для программного тома значение 100 процентов.** Это позволит максимально увеличить динамический диапазон звука.
-   **Не блокируйте другие события звука, запустив программу в монопольном режиме.** Это может препятствовать правильной работе других программ. Например, использование монопольного режима предотвращает использование компьютера в качестве устройства телефонии.

## <a name="text"></a>Текст

-   Не используйте фразу звуковой адаптер. Вместо этого используйте звуковую карту.
-   Используйте устройство для универсального использования с колонками, наушниками и телефонами.
-   Используйте контроллер для обращения к звуковому оборудованию, управляющему устройствами, например звуковыми картами и наборами микросхем.
-   Используйте звуковую схему фразы для описания набора звуков для распространенных событий программы, таких как вход в систему или получение нового сообщения электронной почты. Используйте тему "фраза для настольных ПК", чтобы описать коллекцию визуальных элементов и звуков для рабочего стола компьютера.
-   Используйте термин «звук» для широкого обозначения речи, музыки и звуков. используйте термин «звук» для более точного обозначения программы и Windows звуков, описанных в этой статье.
