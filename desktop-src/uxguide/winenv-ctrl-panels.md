# <a name="control-panels"></a>Панели управления

> [!NOTE]
> Это руководство по проектированию было создано для Windows 7 и не Обновлено для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

Используйте элементы панели управления, чтобы помочь пользователям настроить функции системного уровня и выполнить связанные задачи. Программы, имеющие пользовательский интерфейс, должны быть настроены непосредственно из пользовательского интерфейса.

С помощью панели управления в Microsoft Windows пользователи могут настраивать функции системного уровня и выполнять связанные задачи. Примерами конфигурации компонентов на уровне системы являются установка оборудования и программного обеспечения, Настройка, безопасность, Обслуживание системы и управление учетными записями пользователей.

Термин панель управления относится ко всей функции панели управления Windows. Отдельные панели управления называются элементами панели управления. Элемент панели управления считается верхним, когда он напрямую доступен с домашней страницы панели управления или со страницы категории.

![снимок экрана с категорией "речь" панели управления ](images/winenv-ctrl-panels-image1.png)

Типичный элемент панели управления.

Главная страница панели управления — это Главная точка входа для всех элементов панели управления. В нем перечислены элементы по их категориям, а также наиболее распространенные задачи. Он отображается, когда пользователь наводит кнопку Панель управления в меню Пуск.

На странице Категория панели управления перечислены элементы в одной категории, а также наиболее распространенные задачи. Он отображается, когда пользователи щелкают имя категории на домашней странице.

Элементы панели управления реализуются с помощью [потоков задач](glossary.md) или страниц свойств. Для Windows Vista и более поздних версий потоки задач — это предпочтительный пользовательский интерфейс.

**Разработчики:** Сведения о создании элементов панели управления см. в разделе [элементы панели управления](/previous-versions//bb776838(v=vs.85)).

**Примечание.** Рекомендации, связанные с [таблицами свойств](win-property-win.md) , представлены в отдельной статье.

## <a name="is-this-the-right-user-interface"></a>Это правильный пользовательский интерфейс?

Чтобы определиться, ответьте на вопросы:

-   **Является ли целью Настройка функций системного уровня?** В противном случае используйте другую точку интеграции. Сделайте функции приложения настраиваемыми непосредственно из пользовательского интерфейса с помощью диалоговых окон "Параметры", а не с помощью панели управления. Для служебных программ, которые не используются для установки, настройки или связанных задач (например, устранение неполадок), используйте меню Пуск в качестве точки интеграции.
-   **Имеет ли функция системного уровня собственный интерфейс?** Если да, то в этом пользовательском интерфейсе пользователи должны переключиться на внесение изменений. Например, программа резервного копирования системы должна быть настроена из параметров программы, а не с помощью панели управления.
-   **Будут ли пользователи часто менять конфигурацию?** Если да (скажем, несколько раз в неделю), рассмотрите альтернативные решения, возможно, в дополнение к использованию панели управления. Например, параметр главного тома Windows можно настроить непосредственно из значка в области уведомлений. Некоторые параметры можно настроить автоматически. В проводнике Windows, например, вкладка совместимость свойств приложения позволяет запускать приложение в режиме цвета 256, а не требовать от пользователей изменять видеорежим вручную.
-   **Являются ли целевыми пользователями ИТ-специалисты?** В этом случае следует использовать оснастку [консоли управления (MMC)](/previous-versions/windows/desktop/mmc/microsoft-management-console-start-page) , которая разработана специально для задач управления системой. В некоторых случаях лучшим решением будет наличие элемента панели управления для общих пользователей и оснастки MMC для ИТ-специалистов.

    ![снимок экрана окна "Управление компьютером" ](images/winenv-ctrl-panels-image2.png)

    В этом примере оснастка MMC "Локальные пользователи и группы" обеспечивает управление пользователями, предназначенные для ИТ-специалистов. Другие пользователи скорее всего будут использовать элемент учетные записи пользователей на панели управления.

-   **Является ли функция OEM, используемой только во время начальной настройки системы?** Если это так, используйте центр приветствия Windows в качестве точки интеграции.

Элементы панели управления необходимы, поскольку многие функции уровня системы не имеют более очевидных или прямых точек интеграции. Пока панель управления не должна отображаться как "одно место" для всех параметров конфигурации. **Программы с пользовательским интерфейсом должны быть настроены непосредственно из пользовательского интерфейса, а не с помощью элементов панели управления.**

**Неправильно**:

![снимок экрана элемента "Свойства браузера" панели управления ](images/winenv-ctrl-panels-image3.png)

В этом примере Windows Internet Explorer не должен быть представлен в панели управления, поскольку его собственный пользовательский интерфейс является лучшей точкой интеграции.

### <a name="create-a-new-control-panel-item-or-extend-an-existing-one"></a>Создать новый элемент панели управления или расширить существующий?

Чтобы определиться, ответьте на вопросы:

-   **Может ли функция выражаться как задачи, которые могут подключаться к существующему расширенному элементу панели управления?** Следующие элементы панели управления являются расширяемыми: устройства Bluetooth, дисплей, Интернет, клавиатура, мышь, сеть, питание, система, беспроводная связь (инфракрасная).
-   **Свойства и задачи заменяют функции существующего расширяемого элемента панели управления?** В этом случае следует расширить существующий элемент панели управления, так как это приводит к упрощению взаимодействия с пользователем. В противном случае создайте новый элемент панели управления.

## <a name="design-concepts"></a>Принципы проектирования

**Концепция панели управления основана на реальной метафоре.** Реальная панель управления — это набор элементов управления (датчиков, выключателей, датчиков и экранов), используемых для мониторинга и управления устройством. Пользователям таких панелей управления часто требуется обучение, чтобы понять, как их использовать.

В отличие от реальных аналогев, **структуры панели управления Windows оптимизированы для первого пользователя.** Пользователи не выполняют большую часть задач панели управления очень часто, так что обычно они не запоминают, как их сделать и, в своюмся, придется каждый раз их переучить.

Для проектирования элемента панели управления, который удобен и удобно использовать:

-   Убедитесь, что свойства необходимы.
-   Представление свойств с точки зрения пользователей вместо технологий.
-   Представление свойств на правильном уровне.
-   Разработка страниц для конкретных задач.
-   Разработка страниц для обычных пользователей и защищенных администраторов.

При проектировании и оценке элементов, которые необходимо включить в панель управления, Определите общие задачи, выполняемые пользователями, и убедитесь, что для выполнения этих задач существует четкий путь. Пользователи обычно выполняют следующие типы задач с элементами панели управления:

-   Исходная конфигурация
-   Редкие изменения (для большинства параметров)
-   Частые изменения (для нескольких важных параметров)
-   Откат параметров к исходному или предыдущему состоянию
-   Устранение неполадок

**Если вы выполняете только одно действие...**

Разрабатывать страницы панели управления для конкретных задач и представлять их с точки зрения пользователей вместо технологий.

## <a name="usage-patterns"></a>Варианты использования

Для элементов панели управления можно использовать последовательность задач или страницу свойств. Ниже приведены их шаблоны использования.

### <a name="task-flow-patterns"></a>Шаблоны потока задач

Элементы потока задач используют центральную страницу для представления вариантов высокого уровня и периферийных страниц для выполнения фактической настройки.

**Страницы концентратора**

-   Страницы концентратора на основе задач. Эти страницы концентратора представляют наиболее часто используемые задачи. Их лучше всего использовать для нескольких часто используемых или важных задач, когда пользователям требуется дополнительное руководство и объяснение. На страницах-концентраторах отсутствуют кнопки фиксации. Гибридные страницы на основе задач также имеют непосредственно определенные свойства или команды. Гибридные страницы-концентраторы настоятельно рекомендуются, когда пользователи, скорее всего, используют панель управления для доступа к этим свойствам и командам.
-   Страницы концентратора на основе объектов. Эти страницы концентратора представляют доступные объекты, используя элемент управления представления списка. Они лучше всего использовать при наличии нескольких объектов. На страницах-концентраторах отсутствуют кнопки фиксации.

**Периферийные страницы**

-   Страницы задач. Эти периферийные страницы представляют собой задачу или шаг в задаче с определенной главной инструкцией на основе задач. Они лучше подходят для задач, которые используют дополнительные рекомендации и пояснения.
-   Страницы формы. Эти периферийные страницы представляют собой набор связанных свойств и задач на основе общей основной инструкции. Они лучше подходят для функций, имеющих много свойств и использующих непосредственную одностраничную презентацию, например дополнительные свойства.

### <a name="property-sheet-patterns"></a>Шаблоны листов свойств

-   Страницы свойств лучше использовать в устаревших элементах с множеством параметров, предназначенных для опытных пользователей. Новые элементы могут добиться того же результата с помощью последовательности задач, используя шаблон страницы формы.

## <a name="guidelines"></a>Рекомендации

### <a name="property-sheet-control-panel-items"></a>Элементы панели управления листа свойств

-   **Не используйте вкладки свойств для новых элементов панели управления.** Вместо этого используйте последовательности задач для создания простой работы и полного использования функций классификации и поиска на домашней странице панели управления.

### <a name="task-flow-control-panel-items"></a>Элементы панели управления потоком задач

**Общие сведения**

-   **Поддерживайте наиболее важное содержимое и элементы управления видимыми без прокрутки.** Пользователи не пропрокручиваются для просмотра содержимого страницы, если у них нет причины. Можно сделать кнопки фиксации всегда видимыми, поместив их в [область команд](glossary.md) , а не в область содержимого. Не Разбейте страницы так, чтобы избежать прокрутки.
    -   **Можно выполнить вертикальную прокрутку длинных страниц,** если наиболее важные элементы управления видны без прокрутки.
    -   **Не используйте горизонтальную прокрутку.** Вместо этого следует переконструировать содержимое страницы и использовать вертикальную прокрутку. Страницы могут иметь горизонтальные полосы прокрутки, только если они очень узки.
-   **Для перехода между страницами:**
    -   Используйте [ссылки задач](glossary.md) для запуска задачи.
    -   Используйте ссылки задач или кнопку Далее, чтобы перейти к следующей странице многошаговой задачи.
    -   Используйте кнопки фиксации для выполнения задачи.
    -   Используйте кнопку назад в строке меню, чтобы вернуться к предыдущим просмотренным страницам. Не добавляйте кнопку назад в область команд.
    -   Используйте адресную строку, чтобы напрямую вернуться на домашнюю страницу панели управления.
    -   Используйте раздел показать также ссылки в области задач для перехода к страницам в других элементах панели управления. В противном случае Навигация должна остаться в одном элементе панели управления.
-   **В адресной строке помещается только Домашняя страница панели управления.** При нажатии этой ссылки на домашнюю страницу панели управления возвращается невыполненная работа без [подтверждения](https://msdn.microsoft.com/library/windows/desktop/aa511273.aspx).
-   **Не помещайте кнопку "Закрыть" на страницах панели управления.** Пользователи могут закрыть окно панели управления с помощью кнопки Закрыть в строке заголовка.

**Ссылки и кнопки задач**

-   **Если страница имеет небольшой набор фиксированных параметров, используйте связи задач вместо сочетания переключателей и кнопки Далее.** Это позволяет пользователям выбирать ответ одним щелчком.
-   Вы можете разместить ссылки на задачи и кнопки в следующих местах (в порядке их обнаружения):
    -   [Область команд](glossary.md) (для командных кнопок только на периферийных страницах).
    -   [Область содержимого](glossary.md):
        -   Кнопки команд
        -   Ссылки на задачи
        -   Другие ссылки
    -   Ссылки в [области задач](glossary.md) (только для страниц-концентраторов).
-   **Основано на расположении связей задач и кнопок по важности и необходимости для обнаружения.**
    -   **Помещайте только кнопки фиксации в области команд.**
    -   **Помещайте критические задачи в область содержимого.** Командные кнопки обычно нарисуют наибольшее внимание, поэтому Зарезервируйте их для команд, которые должны видеть пользователи. Ссылки на задачи также нарисуют внимание, но меньше, чем кнопки команд.
    -   **Зарезервируйте область задач и простые ссылки для вторичных (менее важных) задач.** Область задач является наименее обнаруживаемой областью страницы задач, а простые ссылки не отображаются в виде кнопок команд и ссылок на задачи.
-   Для ссылок на задачи, представленные в области содержимого:
    -   **Если число ссылок превышает семь, сгруппируйте ссылки на категории.** Укажите заголовки для каждой из групп.
    -   **Для менее семи ссылок Представьте ссылки в одной группе без заголовка.**
-   **Представление ссылок и кнопок задач в логическом порядке.** Перечислить ссылки задач по вертикали, кнопки по горизонтали.
-   В категории **разделите команды на связанные группы.** Представьте группы задач, поместив наиболее часто используемые первые, и внутри каждой группы сначала поместите наиболее часто используемые задачи. **Полученный порядок должен быть примерно ниже вероятности использования, но также имеет логический поток.**
    -   **Исключение:** Ссылки на задачи, приводящие к выполнению всех задач, должны быть помещены первыми.
-   **Если имеется много ссылок на задачи, присвойте наиболее важным задачам более наглядный вид** , используя значок 24x24 пикселей и две строки текста. Для менее важных задач используйте значок 16x16 пикселей или значок без значка и одну строку текста ссылки.

    ![снимок экрана элементов с большими и мелкими значками ](images/winenv-ctrl-panels-image4.png)

    В этом примере важно, чтобы важные команды были более заметным видом.

-   **Четкое физическое разделение между часто используемыми командами и разрушительными командами.** В противном случае пользователи могут случайно щелкнуть необратимые команды. Вам может потребоваться изменить порядок команд, чтобы объединить разрушающие команды.
-   **Предоставьте механизм для отмены команд непосредственно на странице.** Пользователям не нужно переходить в другое место для отмены ошибки.
-   **Для связей задач используйте все значки ссылок задач по умолчанию или все пользовательские значки.** Не смешивать их. Рассмотрите возможность использования пользовательских значков, только если:
    -   Они помогают пользователям в процессе понимания задач.
    -   Они соответствуют [стандартам значков Aero](vis-icons.md).
    -   Они имеют ненавязчивый внешний вид.

**Диалоговые окна**

При использовании потоков задач обычно требуется, чтобы задача переходила со страницы на страницу в пределах одного окна, но следующие условия являются исключениями. Используйте диалоговые окна в следующих случаях.

-   Запрашивать у пользователей имя пользователя и пароль администратора. Для этой цели всегда используйте диалоговое окно Диспетчер учетных данных. (Должен быть [модальным](glossary.md).)
-   Для подтверждения команды на месте с помощью диалогового окна задачи или сообщения. (Должен быть модальным.)
-   Для получения входных данных для встроенных команд, таких как для новых команд, добавления, сохранения как, переименования и печати.

    ![снимок экрана — диалоговое окно "удаление сетевых расположений" ](images/winenv-ctrl-panels-image5.png)

    В этом примере команда DELETE выполняется в диалоговом окне, а не на отдельной странице.

-   Для выполнения вторичных автономных задач. Использование [немодального](glossary.md)дополнительного окна позволяет выполнять такие задачи независимо от основного потока задач.

### <a name="hub-pages"></a>Страницы концентратора

**Общие сведения**

-   Используйте страницы концентратора на основе задач в следующих случаях:
    -   **Существует небольшое количество часто используемых или важных задач.**
    -   **Конфигурация состоит из одного или двух объектов** (например, мониторов, клавиатуры, мыши, игровых устройств).
    -   **Конфигурация применяется ко всей системе** (примеры: Дата и время, безопасность, параметры электропитания).
-   Используйте страницы концентратора на основе объектов в следующих случаях:
    -   **Конфигурация может содержать несколько объектов** (примеры: учетные записи пользователей, сетевые подключения, принтеры).
    -   **Конфигурация применяется только к выбранному объекту**.
-   **Не используйте центральную страницу, если элемент панели управления содержит одну страницу** , содержащую все связанные задачи и свойства.

**Списки объектов**

-   **Список элементов в логическом порядке.** Сортировать именованные объекты в алфавитном порядке, числа в числовом порядке и даты в хронологическом порядке.
-   Для концентраторов на основе объектов **Предоставьте команды представления объектов в области задач, если возможность изменения представления важна для задач**. Возможность изменять представления важна, если имеется много объектов, и представление по умолчанию не подходит для всех сценариев. Пользователи могут изменять представление списка, даже если в контекстном меню представления списка нет явных команд, но оно является менее обнаруживаемым.

Дополнительные рекомендации по представлению списков объектов см. в разделе [представления списков](ctrl-list-views.md).

**Взаимодействие**

-   **Не помещайте кнопки фиксации на страницах концентратора.** Центральные страницы являются основными точками запуска. Пользователи никогда не «фиксируют» страницы концентратора, они никогда не выполняются. И кнопки фиксации на страницах-концентраторах делают все задачи, инициированные из концентратора, непонятными (пользователи поинтересоваться, нужно ли зафиксировать эти задачи).
    -   **Исключение:** Если изменение параметра требует [повышения прав](glossary.md), используйте кнопку применить с [значком щита безопасности](winenv-uac.md). Отключить кнопку фиксации после применения изменений.
-   **Рассмотрите возможность размещения наиболее полезных свойств непосредственно на страницах-концентраторах.** Такие гибридные страницы-концентраторы настоятельно рекомендуются, если пользователи, скорее всего, используют панель управления для доступа к этим свойствам.

    ![снимок экрана со страницей центра параметров питания ](images/winenv-ctrl-panels-image6.png)

    В этом примере наиболее полезными параметрами для элемента панели управления "Параметры электропитания" являются непосредственно на странице центра.

-   **Используйте модель немедленной фиксации для всех параметров на гибридных страницах-концентраторах, чтобы изменения были сделаны немедленно.** Опять же, пользователи никогда не зафиксируют страницу центра. Если для параметра требуется кнопка фиксации, не следует размещать его на странице центра.
-   **Рассмотрите возможность размещения простых, "одношаговых" команд непосредственно на страницах-концентраторах, а не с помощью навигационных ссылок.**
-   **Подтвердите команды на месте, эффекты которых невозможно легко отменить.** Используйте [диалоговое окно задачи](win-dialog-box.md) или окно [сообщения](glossary.md).

    ![снимок экрана с диалоговым окном "подтвердить удаление" ](images/winenv-ctrl-panels-image7.png)

    В этом примере команда Delete подтверждается с помощью диалогового окна.

-   **Для страниц концентратора, основанных на задачах, необходимо задать для каждой задачи ссылку на задачу и значок.** Можно также указать необязательное описание для каждой ссылки. Однако попробуйте сделать так, чтобы ссылки на задачи были понятны, и предоставить дополнительные описания только тем ссылкам, которые им необходимы.

    ![снимок экрана со страницей центра производительности компьютера ](images/winenv-ctrl-panels-image8.png)

    В этом примере у каждой задачи есть ссылка на задачу и значок.

-   **Для страниц концентраторов, основанных на объектах, одним щелчком выбирается объект, а при двойном щелчке — выбор объекта и переход к странице по умолчанию.** Обычно страницей по умолчанию является страница свойств или страница концентратора на основе задач.
-   **Страница концентратора на основе объектов может переходить к концентратору, основанному на задачах, для выбранных объектов.** Однако такие вторичные концентраторы следует избегать, так как они делают элемент панели управления слишком косвенным.

**Области задач**

Области задач используются для представления ссылок на команды, представления и связанные элементы панели управления.

-   Для областей задач в концентраторах, основанных на задачах, ссылки представлены в следующем порядке:
    -   **Вторичные команды**. Представление основных задач только в области содержимого. Используйте область задач для вторичных, необязательных задач. Рассмотрим задачу, которая является первичной, если пользователи должны обнаружить ее в важных сценариях. дополнительный, если пользователи не могут его обнаружить.
    -   **См. также**. Необязательные ссылки, которые переходить к связанным элементам панели управления.
-   Для областей задач в концентраторах на основе объектов существуют ссылки в следующем порядке:
    -   **Представления объектов**. Необязательные ссылки, используемые для управления представлением объектов.
    -   **Исправлены команды**. Команды, которые не зависят от выбранных в данный момент объектов.
    -   **Контекстные команды**. Команды, зависящие от выбранных в данный момент объектов и, следовательно, не всегда отображаются.
    -   **См. также**. Необязательные ссылки, которые переходить к связанным элементам панели управления.
-   **Не используйте области задач на периферийных страницах.** В отличие от страниц-концентраторов, резервные страницы должны быть направлены на выполнение задачи. Вы не хотите рекомендовать пользователям переходить на другую страницу до их завершения.

**См. также ссылки**

-   **Предоставьте сведения о ссылках в области задач, чтобы помочь пользователям найти связанные элементы панели управления или соответствующий элемент панели управления, если они имеют неправильный значок.** Ссылка на элементы. Пользователи, скорее всего, будут связаны с элементом панели управления.

    ![снимок экрана центра действий "см. также" ссылки ](images/winenv-ctrl-panels-image9.png)

    В этом примере элемент панели управления "центр действий" ссылается на связанные элементы панели управления.

-   **Ссылка на определенную страницу задачи, если это наиболее вероятно для пользователей.** В противном случае свяжите со всем элементом панели управления. Используйте имя панели управления, не добавляя фразу, панель управления.

### <a name="spoke-pages"></a>Периферийные страницы

**Общие сведения**

-   **Страницы задач используются для часто используемых или важных задач, когда пользователям требуются дополнительные рекомендации и пояснения.**
-   **Страницы форм можно использовать для функций, которые имеют много параметров и получают преимущества от прямой презентации с одной страницей.** Идеальные задачи для таких страниц обычно подразумевают очевидные изменения в нескольких простых свойствах.
-   **Не используйте области задач на периферийных страницах.**

**Взаимодействие**

-   **Попробуйте ограничить основные задачи одной страницей.** Если требуется более одной страницы, можно:
    -   **Используйте промежуточные периферийные страницы для дополнительных или необязательных действий.** Промежуточные периферийные страницы фиксируются в виде последней резервной страницы.
    -   **Используйте независимые окна для независимых вспомогательных задач.** Независимые окна фиксируются самостоятельно и независимо от основной задачи.

Это позволяет сохранить значение кнопок фиксации для основной задачи Clear и однозначно. Пользователи всегда должны быть уверены в том, что они зафиксированы.

-   **Не используйте ссылки см. также в последовательности задач.** Эти ссылки на связанные, но отличающиеся элементы панели управления. Хотя переход на другой элемент допустим на страницах-концентраторах, он не находится на периферийных страницах, так как это прерывает задачу.
-   **Не используйте периферийные страницы для простых входов или подтверждений.** Вместо этого используйте модальные диалоговые окна.

**Взаимодействие (промежуточные периферийные страницы)**

-   **Используйте ссылки задач или кнопку Далее для перехода к следующей странице.** Переход к следующему шагу всегда должен быть очевидным.
-   **Можно иметь ссылки навигации на дополнительные шаги задачи.** Чтобы избежать путаницы при фиксации пользователями задачи, эти дополнительные страницы должны быть промежуточными страницами в пределах одного элемента панели управления. Они не должны иметь кнопки фиксации, но должны зафиксироваться при фиксации основной задачи.

**Взаимодействие (последние резервные страницы)**

-   **Используйте кнопки фиксации для выполнения задачи.** Используйте [модель отложенной фиксации](glossary.md) для периферийных страниц, чтобы изменения не вносились в явном виде (если пользователи переходят с помощью обратного, закрытого или адресной строки, изменения будут отменены). Кнопки фиксации представляют собой визуальную подговорить, что пользователь собирается выполнить задачу. Не используйте ссылки для этой цели.
-   **Не подтверждайте кнопки фиксации (включая Cancel).** Это может раздражать. Исключения:
    -   Действие имеет серьезные последствия и, если оно неверно, нелегко могут быть исправлены.
    -   Это действие может привести к значительному снижению времени или усилий пользователя.
    -   Действие явно не согласуется с другими действиями.
-   **Не подтверждайте, если пользователи отменяют изменения** , перемещаясь с помощью команды назад, закрытия или адресной строки. Однако вы можете проверить, может ли потенциально непреднамеренное перемещение привести к значительной потере времени или усилий пользователя.
-   **Не используйте ссылки на команду или навигацию** (в том числе см. также ссылки). На конечных страницах пользователи должны явно завершить или отменить задачу. Пользователям не рекомендуется переходить в другое место, поскольку это может привести к неявной отмене задачи.
-   **Когда пользователи завершают или отменяют задачу, они должны быть отправлены обратно на страницу центра, с которой была запущена задача.** Если такой страницы нет, закройте окно панели управления. Не следует рассчитывать, что периферийные страницы всегда запускаются с другой страницы.
-   **Удаление устаревших "зафиксированных" страниц из стека обозревателя Windows** при возвращении пользователей обратно на страницу, с которой была запущена задача. Пользователи никогда не увидят страниц, на которые они уже зафиксировались при нажатии кнопки "назад". Пользователи всегда должны вносить дополнительные изменения, полностью выполняя задачу, а не щелкая кнопку назад, чтобы изменить устаревшие страницы.
    -   **Разработчики:** Эти устаревшие страницы можно удалить с помощью API-интерфейсов Итравеллог:: Финдтравелентри () и Итравелложекс::D Елетинтри ().

**Кнопки фиксации**

**Примечание.** Кнопки отмены считаются кнопками фиксации.

-   **Подтвердите задачи, используя кнопки фиксации, которые являются конкретными ответами на основную инструкцию вместо универсальных меток, таких как ОК.** Метки на кнопках фиксации должны быть понятны по своему усмотрению. Старайтесь не использовать ОК, поскольку это не конкретный ответ на основную инструкцию и, следовательно, проще мисундерстанд. Кроме того, ОК обычно используется с модальными диалоговыми окнами и неправильно подразумевает закрытие окна элемента панели управления.
    -   **Исключения:**
        -   Используйте кнопку ОК для страниц без параметров.
        -   Используйте ОК, если конкретный ответ все еще является универсальным, например сохранение, выбор или выбор, как при изменении определенного параметра или коллекции параметров.
        -   Нажмите кнопку ОК, если на странице есть переключатели, отвечающие на основную инструкцию. Для поддержки модели отложенной фиксации нельзя использовать связи задач на заключительной странице.

            ![снимок экрана с веб-ограничениями с кнопкой "ОК" ](images/winenv-ctrl-panels-image10.png)

            В этом примере переключатели, а не кнопки фиксации, являются ответами на основную инструкцию.
-   **Укажите кнопку Отмена, чтобы разрешить пользователям явно отменять изменения.** Хотя пользователи могут неявным образом отменять задачу, не подтверждая их изменения, при условии, что кнопка отмены позволяет им сделать это с большей уверенностью.
    -   **Исключение:** Не вынимайте кнопку Отмена для задач, в которых пользователи не могут вносить изменения. Кнопка ОК действует аналогично отмене в данном случае.
-   **Не используйте кнопки Закрыть, готово или завершить фиксацию.** Эти кнопки обычно используются с модальными диалоговыми окнами и неправильно подразумевают закрытие окна элемента панели управления. Пользователи могут закрыть окно с помощью кнопки Закрыть в строке заголовка. Кроме того, «готово» и «готово» являются ошибочными, так как пользователи возвращаются на страницу, с которой была запущена задача, поэтому они не выполняются.
-   **Не отключайте кнопки фиксации.** В противном случае пользователям нужно будет определить, почему кнопки фиксации отключены. Лучше оставить кнопки фиксации включенными и дать полезное сообщение об ошибке при возникновении проблемы.
-   **Убедитесь, что кнопки фиксации отображаются на странице без прокрутки.** Если страница является длинной, можно сделать кнопки фиксации всегда видны, поместив их в [область команд](glossary.md), а не в область содержимого.

    ![снимок экрана диалогового окна "Автозапуск" ](images/winenv-ctrl-panels-image11.png)

    В этом примере размер области содержимого не ограничен, поэтому кнопки фиксации помещаются в область команд.

-   Выровняйте кнопки фиксации по правому краю и используйте этот порядок (слева направо): кнопки положительной фиксации, отмена и применить.

**Кнопки предварительного просмотра**

-   Убедитесь **, что кнопка Предварительный просмотр позволяет применить ожидающие изменения сейчас, но восстановить текущие параметры, если пользователи отходят от страницы, не внося изменений.**
-   **Кнопки предварительного просмотра можно использовать на любой периферийной странице.** Для страниц-концентраторов не требуются кнопки предварительного просмотра, так как они используют [модель немедленной фиксации](glossary.md).
-   **Рассмотрите возможность использования кнопки предварительного просмотра вместо кнопки Применить для страниц панели управления.** Кнопки предварительного просмотра легче понять, и их можно использовать на любой периферийной странице.
-   **Кнопка "предварительный просмотр" доступна только в том случае, если на странице есть параметры (по крайней мере одна) с эффектами, которые могут видеть пользователи.** Пользователи должны иметь возможность просматривать изменения, оценивать изменения и вносить дальнейшие изменения в зависимости от оценки.
-   **Всегда включайте кнопку предварительного просмотра.**

**Динамические предварительные версии**

Элемент панели управления имеет динамический просмотр, когда можно сразу увидеть результат изменений на странице.

-   **Используйте динамический просмотр для параметров экрана в следующих случаях:**
    -   Этот результат очевиден, как правило, потому, что он применяется ко всему монитору.
    -   Этот результат можно применить без значительной задержки.
    -   Этот результат является надежным и может быть отменен легко.

        ![снимок экрана — диалоговое окно "изменение параметров цвета" ](images/winenv-ctrl-panels-image12.png)

        В этом примере результаты настройки цвета и внешнего вида Windows отображаются немедленно. Это позволяет пользователям вносить изменения с минимальными усилиями.

-   **Используйте кнопки сохранить изменения и Отмена для кнопок фиксация.** При сохранении изменений сохраняются текущие параметры, тогда как отмена вернется к исходным параметрам. Вместо ОК используется "сохранить изменения", чтобы ясно убедиться, что все предварительно просмотренные изменения еще не применены.
-   **Не используйте кнопку Применить.** Динамическая предварительная версия делает ненужными.
-   **Восстановите все изменения, если пользователи переходят** с помощью команды назад, закрытия или адресной строки. Чтобы сохранить изменения, пользователи должны зафиксировать их явным образом.

**Применить кнопки**

-   Убедитесь, что **кнопка "Применить" означает применение ожидающих изменений (сделанных с момента запуска задачи или последнего применения), но остается на текущей странице.** Это позволит пользователям оценивать изменения перед переходом к другим задачам.
-   **Используйте кнопки Применить только на конечных страницах.** Кнопки применить нельзя использовать на промежуточных страницах для поддержания немедленной модели фиксации.
    -   **Исключение:** Вы можете использовать кнопки Применить на гибридной странице центра, если изменение параметра требует [повышения прав](glossary.md). Дополнительные сведения см. в разделе [взаимодействие со страницей центра](#hub-pages).
-   **Используйте кнопку Применить, только если на странице есть параметры (по крайней мере одна) с эффектами, которые пользователи могут оценивать осмысленным образом.** Обычно кнопки Применить используются, когда параметры делают видимые изменения. Пользователи должны иметь возможность применять изменения, оценивать изменения и вносить дальнейшие изменения в зависимости от оценки.
-   **Включите кнопку "Применить" только при наличии ожидающих изменений.** в противном случае отключите его.
-   **Назначить "A" в качестве ключа доступа.**

### <a name="control-panel-integration"></a>Интеграция панели управления

Чтобы интегрировать элемент панели управления с Windows, можно:

-   **Зарегистрируйте элемент панели управления (включая его имя, описание и значок)**, чтобы система Windows знала о ней.
-   Если элемент панели управления является верхним уровнем (см. ниже):
    -   Свяжите его с соответствующей **страницей категории**.
    -   **Укажите ссылки на задачи (включая их имя, описание, ключевые слова и командную строку),** чтобы указать основные задачи и разрешить пользователям переходить непосредственно к задачам.
-   **Укажите условия поиска** , которые помогут пользователям находить ссылки на задачи с помощью функции поиска панели управления.

    Обратите внимание, что эти сведения можно указать только для отдельных элементов панели управления. Эти сведения нельзя добавить или изменить для существующих элементов панели управления, которые вы расширяете.

**Страницы категорий**

-   **Добавьте элемент панели управления на страницу категории, только если:**

    -   Большинству пользователей это необходимо. Пример: центр управления сетями и общим доступом
    -   Он используется много раз. Пример: System
    -   Он предоставляет важные функции, которые не доступны в других местах. Пример: принтеры

    Элементы панели управления, соответствующие этим критериям, называются верхним уровнем.

-   **Не добавляйте элемент панели управления на страницу категории, если:**

    -   Он редко используется или используется для одноразовой настройки. Пример: центр начальной настройки
    -   Он предназначен для опытных пользователей или специалистов по ИТ. Пример: Управление цветом
    -   Он не применяется к текущей конфигурации оборудования или программного обеспечения. Пример: Windows SideShow (если не поддерживается текущим оборудованием).

    Удаление таких элементов панели управления из страниц категорий упрощает поиск элементов верхнего уровня. Учитывая их использование, эти элементы панели управления достаточно обнаружить с помощью поиска или контекстных точек входа.

-   **Свяжите элемент панели управления верхнего уровня с категорией, в которой пользователи, скорее всего, будут искать их.** Это решение должно основываться на тестировании пользователей.
-   **Рекомендуется также связать элемент панели управления верхнего уровня с второй из наиболее вероятных категорий.** Необходимо связать элемент панели управления с двумя категориями, если пользователи, скорее всего, будут искать основные задачи в нескольких местах.
-   **Не связывайте элемент панели управления с более чем двумя категориями.** Если элементы панели управления отображаются в нескольких категориях, значение категоризации будет несвязанным.

**Ссылки на задачи**

-   **Свяжите элемент панели управления с его основными задачами.** На странице категории можно отображать до пяти задач, но для поиска на панели управления используются дополнительные задачи. Используйте те же выражения, что и для ссылок на задачи, возможно, удалив некоторые слова, чтобы сделать ссылки задач более сжатыми.
-   **Предпочитать, что ссылки на задачи ведут к разным местам в элементе панели управления.** Наличие нескольких ссылок на одно и то же место может вызвать путаницу.

**Условия поиска**

-   **Зарегистрируйте условия поиска для элемента панели управления, которые пользователи, скорее всего, будут использовать для их описания.** Эти условия поиска должны включать:

    -   Настроенные компоненты или объекты.
    -   Основные задачи.

    Эти условия поиска должны основываться на тестировании пользователей.

-   **Также включите общие синонимы для этих терминов поиска.** Например, мониторы и отображения являются синонимами, поэтому должны быть включены оба слова.
-   **Включить альтернативные слова или разрывы слов.** Например, пользователи могут искать веб-сайт и веб-сайт. Попробуйте также предоставить распространенные опечатки.
-   **Рассмотрим одноименные и множественные существительные формы.** Функция поиска панели управления не выполняет автоматический поиск обеих форм. Укажите формы, для которых пользователи, скорее всего, будут искать.
-   **Используйте простые представленные глаголы.** При регистрации подключения в качестве условия поиска функция поиска не будет автоматически искать подключения, подключения и подключения.
-   **Не беспокойтесь о случае.** Функция поиска не учитывает регистр.

### <a name="standard-users-and-protected-administrators"></a>Обычные пользователи и защищенные администраторы

**Для изменения многих параметров требуются права администратора.** Если процессу требуются права администратора, Windows Vista и более поздние версии требуют, чтобы [обычные пользователи](glossary.md) и [защищенные администраторы](glossary.md) могли явно повышать свои привилегии. Это помогает предотвратить запуск вредоносного кода с правами администратора.

Дополнительные сведения и примеры см. в разделе [Управление учетными записями пользователей](winenv-uac.md).

### <a name="schemes-and-themes"></a>Схемы и темы

Схема — это именованная коллекция визуальных параметров. Тема — это именованная коллекция параметров в системе. К примерам схем и тем относятся экраны, мышь, Телефон и модем, параметры электропитания, звуковые и звуковые параметры.

-   **Разрешить пользователям создавать схемы в следующих случаях:**

    -   **Пользователи, скорее всего, будут изменять параметры.**
    -   **Скорее всего, пользователи будут изменять параметры в виде коллекции.**

    Схемы полезны, когда пользователи находятся в другой среде, например в другом физическом расположении (страна или регион, часовой пояс). Использование компьютера в различных ситуациях (от батарей, закрепленных и незакрепленных); или использование компьютера для другой функции (презентации, воспроизведение видео).

-   **Укажите по крайней мере одну схему по умолчанию.** Схема по умолчанию должна быть хорошо именованной и применяться к большинству пользователей в большинстве случаев. Пользователям не нужно создавать собственную схему.
-   **Предоставьте предварительную версию** или другой механизм, чтобы пользователи могли видеть параметры в схеме.

    ![снимок экрана — диалоговое окно "Персонализация" ](images/winenv-ctrl-panels-image13.png)

    В этом примере элемент панели управления персонализации содержит предварительную версию параметров рабочего стола и внешнего вида.

-   **Укажите команды "Сохранить как" и "Удалить".** Команда переименования не является обязательной. пользователи могут переименовывать схемы, сохраняя их под нужным именем и удаляя исходную схему.
-   Если параметры не могут быть применены без схемы, **не разрешайте пользователям удалять все схемы.** Пользователям не нужно создавать собственную схему.
-   Если схемы не полностью независимы (например, схемы управления питанием зависят от текущего режима работы ноутбука), убедитесь, **что существует простой способ изменения параметров, применяемых во всех схемах.** Например, в схемах управления питанием следует убедиться, что пользователи могут задавать, что происходит при закрытии крышки портативного компьютера в одном месте.

### <a name="miscellaneous"></a>Разное

-   **Используйте расширения панели управления для функций, которые заменяют или расширяют существующие функциональные возможности Windows.** Следующие элементы панели управления являются расширяемыми: устройства Bluetooth, дисплей, Интернет, клавиатура, мышь, сеть, питание, система, беспроводная связь (инфракрасная).

### <a name="default-values"></a>Значения по умолчанию

-   **Параметры в элементе панели управления должны отражать текущее состояние компонента.** В противном случае это приведет к ошибочному получению и, возможно, приведет к нежелательным результатам. Например, если в параметрах отражены рекомендации, но не текущее состояние, пользователи могут нажать кнопку Отмена, а не вносить изменения и подумать, что никаких изменений не требуется.
-   **Выберите самый безопасный (чтобы предотвратить потери данных или доступ к системе) и наиболее безопасное начальное состояние.** Предположим, что большинство пользователей не изменяют параметры.
-   **Если безопасность и безопасность не являются факторами, выберите первоначальное состояние, которое наиболее вероятно или удобно.**

### <a name="main-instructions"></a>Основные инструкции

-   **Для страницы центра используйте основную инструкцию для объяснения цели пользователя с помощью элемента панели управления.** Основная инструкция должна помочь пользователям определить, выбрал ли он нужный элемент панели управления. Помните, что пользователи могли выбрать элемент панели управления с ошибками и на самом деле ищут параметры, которые являются частью другого элемента панели управления.

    **Примеры**

    Обеспечьте безопасность и актуальность компьютера

    Оптимизируйте свой компьютер, чтобы упростить просмотр, прослушивание и управление

-   **Для страниц с периферийными страницами используйте основную инструкцию, чтобы объяснить, что делать на странице.** Инструкция должна быть конкретной инструкцией, императивным направлением или вопросом. Хорошие инструкции сообщают о цели пользователя на странице, а не на том, чтобы сосредоточиться на механике манипуляций с ним.

    **Неправильно**:

    Выбор задачи уведомления

    **Правильно:**

    Укажите способ обработки входящих данных

    Правильная версия повышает связь с целью, полученной страницей.

-   **При возможности используйте конкретные команды.** Конкретные команды более важны для пользователей, чем универсальные.
-   Используйте [прописные буквы в стиле предложения](glossary.md).
-   **Не включайте завершающие периоды, если инструкция является инструкцией.** Если инструкция является вопросом, включите последний вопросительный знак.

### <a name="supplemental-instructions"></a>Дополнительные инструкции

-   **Для страницы центра используйте дополнительную вспомогательную инструкцию, чтобы более подробно объяснить назначение элемента панели управления.**
-   **Для страниц с периферийными страницами используйте дополнительную вспомогательную инструкцию для предоставления дополнительных сведений, полезных для понимания или использования страницы.** Вы можете предоставить более подробную информацию и определить терминологию.
-   Используйте полные предложения и [прописные буквы в стиле предложения](glossary.md).

### <a name="page-text"></a>Текст страницы

-   **Не изменяйте основную инструкцию в области содержимого.**
-   **Используйте слово «страница» для ссылки на саму страницу.**
-   **Используйте второй человек (вы, ваш), чтобы сообщить пользователям, что делать** в основной инструкции и области содержимого. Часто используется второе лицо.

    **Пример**.

    Выберите изображения, которые нужно напечатать.

-   **Используйте первого пользователя (I, Me, My), чтобы позволить пользователям сообщить элементу панели управления, что делать** в области содержимого, которая реагирует на основную инструкцию.

    **Пример**.

    Печать фотографий на камере.

### <a name="task-links"></a>Ссылки на задачи

-   **Выберите сокращенный текст ссылки, который четко взаимодействует и отличает связь задачи.** Она должна быть понятной и соответствовать основной инструкции. Пользователям не нужно было бы понять, что означает ссылка, или как она отличается от других ссылок.
-   **Всегда запускать связи задач с глаголом.**
-   Используйте [прописные буквы в стиле предложения](glossary.md).
-   Не используйте знаки препинания в конце метки.
-   **Если ссылка на задачу требует дальнейшего объяснения, укажите описание в отдельном элементе управления "текст",** используя полные предложения и закрывающие знаки препинания. Тем не менее, добавьте такие пояснения, только если это необходимо, но не добавлять объяснения ко всем связям задач, так как для другой ссылки задачи требуется одна.

Дополнительные сведения и примеры см. в разделе [ссылки](ctrl-command-links.md).

### <a name="commit-buttons"></a>Кнопки фиксации

-   **Используйте определенные метки кнопок фиксации, которые имеют смысл в своей работе и соответствуют основной инструкции.** В идеале пользователям не нужно читать что-нибудь еще, чтобы понять метку. Пользователи гораздо чаще читают метки кнопок команд, чем статический текст.
-   **Всегда запускать метки кнопок фиксации с помощью глагола.**
-   **Не используйте "Закрыть", "Готово" или "Готово".** Эти метки кнопок лучше подходят для других типов окон.
-   Используйте [прописные буквы в стиле предложения](glossary.md).
-   Не используйте знаки препинания в конце метки.
-   Назначьте уникальный [ключ доступа](glossary.md).
    -   **Исключение:** Не присваивайте клавиши доступа для кнопок отмены, так как ESC является ключом доступа. Это упрощает назначение других ключей доступа.

## <a name="documentation"></a>Документация

При ссылке на домашнюю страницу панели управления или страницы категорий:

-   В пользовательской документации обратитесь к панели управления, используя прописные буквы в стиле заголовка и пропустив предыдущую определенную статью.

    **Пример**.

    В панели управления откройте **Центр безопасности**.

-   В разделе Программирование и другая техническая документация перейдите на страницу панель управления домашняя страница и Категория панели управления, не заменяя слова на прописные. Предыдущая статья является необязательной.

Для элементов панели управления:

-   При ссылке на отдельный элемент панели управления используйте « \[ имя элемента панели управления \] в панели управления» или обычно «элемент панели управления» в пользовательской документации. Не используйте приложение, программу или панель управления для ссылки на отдельные элементы панели управления.
-   При ссылке на страницу концентратора элемента панели управления используйте "Главная страница" \[ имя элемента панели управления \] ".
-   По возможности отформатируйте имя панели управления, используя полужирный текст. В противном случае заключите имя в кавычки, только если это необходимо для предотвращения путаницы.

Примеры:

-   На панели управления откройте **родительский** контроль.
-   Вернитесь на главную страницу **родительского контроля** .
