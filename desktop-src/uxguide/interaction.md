# <a name="interaction"></a>Взаимодействие

> [!NOTE]
> это руководство по проектированию было создано для Windows 7 и не обновлялось для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

Взаимодействие — это множество способов взаимодействия пользователей с вашим приложением, включая касание, клавиатуру, мышь и т. д. Используйте эти рекомендации, чтобы создать интуитивно понятные и отличительные возможности, оптимизированные для сенсорного взаимодействия, но постоянно работающие на устройствах ввода.

## <a name="design-for-a-touch-first-experience"></a>Разработка для сенсорного взаимодействия

Главное — проектировать приложение так, чтобы сенсорный ввод являлся главным методом ввода для пользователей. при использовании элементов управления платформой поддержка сенсорной панели, мыши и пера не требует дополнительного программирования, так как Windows предоставляет его бесплатно.

Однако имейте в виду, что пользовательский интерфейс, оптимизированный для сенсорного ввода, не всегда будет лучше традиционного интерфейса. Оба предоставляют преимущества и недостатки, уникальные для технологии и приложения. При переходе к пользовательскому интерфейсу Touch важно понимать основные различия между касанием (включая сенсорную панель), пером, мышью и вводом с клавиатуры. не следует использовать привычные свойства и поведения устройств ввода, так как касание Windows 8 не просто эмулирует эту функциональность.

Как вы вскоре обнаружите, для сенсорного ввода требуется другой подход к проектированию пользовательского интерфейса.

## <a name="visuals-for-feedback"></a>Визуальные элементы для обратной связи

соответствующая визуальная обратная связь во время взаимодействия с приложением помогает пользователям распознать, изучать и адаптироваться к способу интерпретации взаимодействий в приложении и Windows визуальной обратной связи может указывать на успешные взаимодействия, состояние системы ретранслятора, улучшить смысл контроля, сократить количество ошибок, помочь пользователям понять систему и устройство ввода и обеспечить взаимодействие.

Визуальная обратная связь особенно важна, когда сенсорный ввод применяется для операций, которые требуют правильного и точного задания расположения. Отображение обратной связи всякий раз, когда обнаруживается сенсорный ввод, позволит пользователю понять правила наведения, которые действуют в вашем приложении и его элементах управления.

## <a name="optimize-for-accuracy"></a>Оптимизировать для точности

Сенсорный ввод включает всю контактную зону пальца. Эта геометрия контакта используется для определения наиболее вероятного целевого объекта. Измените размер элементов управления, чтобы обеспечить удобный пользовательский интерфейс с помощью объектов и элементов управления, которые просты и надежны для целевого объекта.

Размер, место и расположение элементов управления для помощи в устранении пальца и руки перекрытия, где пользовательский интерфейс скрывается самим пользователем взаимодействием.

Расположите меню, всплывающие окна и подсказки над контактной областью, когда это возможно.

## <a name="constrain-for-confidence"></a>Ограничение на достоверность

Избегайте и сыром взаимодействия с помощью ограничений пользовательского интерфейса.

-   Точки привязки могут упростить работу в нужных местах. Они задают логические остановки в содержимом приложения. В результате точки привязки выступают в качестве механизма разбиения на страницы для пользователя и снижают выносливости от чрезмерного скользящего, прокрутки или поворота. С их помощью можно работать с неточным вводом пользователя и обеспечить отображение определенного подмножества содержимого или сведений о ключе.
-   Направленные "направляющие", которые подчеркивают ось движения (по вертикали или по горизонтали).

## <a name="avoid-timed-interactions"></a>Избегайте взаимодействия по времени

Взаимодействия не должны различаться по времени. Одно и то же взаимодействие должно приводить к определенному результату, сколько бы времени ни потребовалось на его выполнение. Зависимые от времени активации вводят обязательные задержки для пользователей, в результате чего обесценивается иммерсивный характер прямого манипулирования и снижается скорость восприятия ответа системы.

Обычно взаимодействия на основе времени зависят от неочевидных пороговых величин: времени, расстояния или скорости, определяющих, какая команда будет выполнена. Время взаимодействия не имеет визуальной обратной связи до тех пор, пока система не выполнит действие, и пользователи должны получить произвольные и невидимые пороговые значения для достижения результата. Мгновенная визуальная обратная связь при взаимодействии повышает заинтересованность пользователей и укрепляет уверенность в себе.

Взаимодействия, непосредственно влияющие на объекты и имитирующие то, что происходит в реальности, легче обнаружить, понять и запомнить. Они не связаны с неочевидными или абстрактными взаимодействиями.

**Примечание.** Исключением является то, что вы используете определенные временные взаимодействия для обучения и исследования (например, для нажатия и удерживания). Использование соответствующих описаний и визуальных подсказок оказывает значительное воздействие на использование расширенных взаимодействий.
