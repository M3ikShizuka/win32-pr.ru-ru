# <a name="tabs"></a>знаки табуляции;

> [!NOTE]
> Это руководство по проектированию было создано для Windows 7 и не Обновлено для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

Вкладки предоставляют способ представления связанных сведений на отдельных страницах с метками.

![снимок экрана с пятью вкладками ](images/ctrl-tabs-image1.png)

Типичный набор вкладок.

Вкладки обычно связаны с окнами свойств (и наоборот), но вкладки можно использовать в любом окне любого типа.

Элементы управления "Вкладка" представляют Манила папки с вкладками, используемые для организации сведений в шкафов, часто встречающихся в США. (Папки Манила не используются во всем мире.)

> [!Note]  
> Рекомендации, связанные с [макетом](vis-layout.md), [меню вкладок](cmd-menus.md), [диалоговыми](win-dialog-box.md)окнами и [окнами свойств](win-property-win.md) , представлены в отдельных статьях.

 

## <a name="is-this-the-right-control"></a>Выбор правильного элемента управления

Чтобы определиться, ответьте на вопросы:

-   **Можно ли разместить элементы управления на одной странице, достаточном размере?** Если это так, используйте одну страницу.
-   **Есть ли только одна вкладка?** Если это так, используйте одну страницу.
-   **Связаны ли вкладки друг с другом очевидным образом?** В противном случае рассмотрите возможность разделения данных на отдельные окна связанных сведений.
-   **Если используется для параметров, то являются ли параметры на разных страницах полностью независимыми?** Изменяет ли параметр на одной странице Параметры, влияющие на другие страницы? Если они не являются независимыми, используйте страницы задач или [Мастер](win-wizards.md) .
-   **Являются ли вкладки главными одноранговыми элементами друг друга или иерархическая связь?** Если иерархия используется, рассмотрите возможность использования последовательного раскрытия или дочерних [диалоговых окон](win-dialog-box.md) для отображения связанных сведений.
-   **Используются ли вкладки для показа шагов в задаче?** Можно использовать "табуляции" для отображения шагов в задаче, только если они представлены как шаги, и существует очевидный, альтернативный способ перехода к текстовому шагу, например "Далее". В противном случае, если шаги необходимы, используйте страницы в потоке страниц или [мастере](win-wizards.md). Если действия являются необязательными, отобразите необязательные шаги с помощью модальных [диалоговых](win-dialog-box.md) окон.
-   **Являются ли вкладки разными представлениями одних и тех же данных?** Если это так, рассмотрите возможность использования [разворачивающейся кнопки](ctrl-command-buttons.md) или [раскрывающегося списка](/windows/desktop/uxguide/ctrl-drop) для изменения представлений. Хотя вкладки можно эффективно использовать для изменения представлений, альтернативы являются более легковесными.

**Если вы выполняете только одно действие...**

Убедитесь, что информация на вкладках связана, но параметры на разных страницах независимы. Последняя выбранная вкладка не должна иметь специального значения.

## <a name="guidelines"></a>Рекомендации

### <a name="general"></a>Общее

-   **Используйте горизонтальные вкладки, если:**
    -   Окно содержит семь или меньше вкладок.
    -   **Все вкладки умещаются в одной строке даже при локализации пользовательского интерфейса.**
-   **Используйте вертикальные вкладки, если:**
    -   Окно свойств содержит восемь или более вкладок.
    -   **Для использования горизонтальных вкладок требуется более одной строки.**

        ![снимок экрана окна свойств с одиннадцатью параметрами ](images/ctrl-tabs-image6.png)

        В этом примере на вертикальных вкладках размещается восемь или более вкладок.

-   **Не следует вкладывать вкладки или объединять горизонтальные вкладки с вертикальными вкладками.** Вместо этого Сократите число вкладок, используйте только вертикальные вкладки или используйте другой элемент управления, например раскрывающийся список.
-   **Не прокручивать горизонтальные вкладки.** Горизонтальная прокрутка не является обнаруживаемой. Однако вы можете прокручивать вертикальные вкладки.

    **Неправильно**:

    ![снимок экрана с усеченной меткой горизонтальной вкладки ](images/ctrl-tabs-image7.png)

    В этом примере горизонтальные вкладки прокручивается.

-   Для вкладок в окне или области с изменяемым размером при необходимости на странице разместите полосу прокрутки, а не окно или панель. Вкладки всегда должны быть видимыми и не могут прокручиваться при просмотре.

    ![снимок экрана с вертикальной вкладкой страницы с полосой прокрутки ](images/ctrl-tabs-image8.png)

    В этом примере на странице вкладки есть полоса прокрутки, а не панель.

-   **Убедитесь, что вкладки выглядят как вкладки, а не другой тип элемента управления.**

    **Неправильно**:

    ![снимок экрана окна с кнопками для вкладок ](images/ctrl-tabs-image9.png)

    В этом примере эти вкладки выглядят как кнопки команд.

### <a name="interaction"></a>Взаимодействие

-   **Если элементы управления применяются только к странице, поместите их в рамку страницы с вкладками.**
-   **Когда элементы управления применяются ко всему окну, размещайте их за пределами страницы с вкладками.**
-   **Не присваивайте эффекты изменениям вкладок.** Вкладки должны быть доступны в любом порядке. Изменение текущей вкладки никогда не должно иметь побочных эффектов, применять параметры или выдавать сообщение об ошибке.
-   **Не назначайте специальное значение для последней выбранной вкладки.** Выбор вкладки предназначен для перехода на последнюю вкладку пользователя — не параметр.
-   **Не вынимайте параметры на странице, зависящей от параметров на других страницах.** Вместо этого следует разместить зависимые параметры на той же странице.
-   **Если пользователи, скорее всего, начинают с последней отображаемой вкладки, сделайте эту вкладку сохраненной и выберите ее по умолчанию.** Сделайте эти параметры постоянными для каждого окна отдельно для каждого пользователя. В противном случае выберите первую страницу по умолчанию.

### <a name="icons"></a>Значки

-   **Не вымещайте значки на вкладках.** Значки обычно добавляют ненужные визуальные элементы, потребляют пространство на экране и часто не улучшают понимание пользователей. Добавляйте только значки, которые помогают в охвате, например стандартные символы.

    **Неправильно**:

    ![снимок экрана окна со значками на вкладках ](images/ctrl-tabs-image10.png)

    В этом примере значки добавляют визуальные элементы ненужным образом и незначительно улучшают понимание пользователей.

    **Исключение:** Если недостаточно места для вывода значимых меток, можно использовать четко распознаваемые значки:

    **Правильно:**

    ![снимок экрана вкладок со значками и метками краткие ](images/ctrl-tabs-image11.png)

    В этом примере окно настолько узким, что значки лучше взаимодействуют с вкладками, чем метки.

-   **Не используйте логотипы продуктов для графики вкладок.** Вкладки не для [фирменной символики](exper-branding.md).

### <a name="dynamic-window-surface-pattern"></a>Шаблон области динамических окон

-   **Не используйте полосы прокрутки на страницах вкладок.** Функции табуляции аналогичны полосам прокрутки, чтобы увеличить эффективность области окна. Один механизм должен быть достаточным.
-   **Использовать краткие метки вкладок.** Используйте одно или два слова, которые ясно описывают содержимое страницы. Более длинные метки потребляют пространство экрана, особенно при локализации меток.
-   **Используйте определенные значимые метки вкладки.** Не используйте универсальные метки вкладок, которые могут применяться к любой вкладке, например Общие, расширенные или параметры.
-   **Если вкладка не применяется к текущему контексту, и пользователи не ее предполагают, удалите ее.** Это упрощает пользовательский интерфейс, и пользователи не пропускают его.

    **Неправильно**:

    ![снимок экрана окна параметров с недоступным именем вкладки ](images/ctrl-tabs-image12.png)

    В этом примере вкладка расположения файлов неверно отключена, если в качестве редактора электронной почты используется Microsoft Word. Вместо отключения этой вкладки ее следует удалить, так как пользователи не хотят просматривать или изменять расположения файлов в этом контексте.

-   **Если вкладка не применяется к текущему контексту и пользователи могут ожидать:**

    -   Отобразить вкладку.
    -   Отключите элементы управления на странице.
    -   Включите текст, поясняющий, почему элементы управления отключены.

    Не отключайте вкладку, так как это не говорит себя и запрещает исследование. Пользователи, которые ищут конкретное значение, будут вынуждены просматривать все остальные вкладки.

    ![снимок экрана окна с параметрами вкладки «вид» затемнены ](images/ctrl-tabs-image13.png)

    В этом примере ни один из параметров представления не применяется в режиме чтения. Однако пользователи могут ожидать, что они будут применяться на основе метки вкладки, поэтому страница отображается, но параметры отключены.

### <a name="multiple-views-and-documents-patterns"></a>Несколько представлений и шаблонов документов

-   **Используйте метки вкладки представление или документ.**
-   **Избегайте чрезмерно длинных имен вкладок.** При необходимости можно либо задать максимальный размер имени, либо усечь отображаемую метку вкладки с помощью многоточия. Более длинные метки потребляют пространство экрана, особенно при локализации меток.
-   **Если вкладка не применяется к текущему контексту, удалите вкладку.**

### <a name="exclusive-options-pattern"></a>Шаблон монопольных параметров

-   **Не используйте этот шаблон.** Вместо этого используйте переключатели или раскрывающийся список.

    **Неправильно**:

    ![снимок экрана окна с двумя вкладками "создать" ](images/ctrl-tabs-image14.png)

    В этом примере вкладки неправильно используются в качестве замены переключателей.

    **Правильно:**

    ![снимок экрана окна с двумя переключателями ](images/ctrl-tabs-image15.png)

    В этом примере вместо этого правильно используются переключатели.

## <a name="labels"></a>Метки

-   Вкладки меток на основе их шаблона. Используйте существительные вместо глаголов без закрывающих знаков препинания. Дополнительные сведения см. в приведенных выше руководствах по шаблонам.
-   Используйте выделение прописных букв, как в предложении.
-   Не присваивайте ключ доступа. Вкладки доступны через сочетания клавиш (CTRL + TAB, CTRL + SHIFT + TAB, Ctrl + Page Up, Ctrl + Page Down). Недостаточность выбора ключа доступа, поэтому не назначение клавиш доступа для вкладок облегчает назначение их другим элементам управления.

## <a name="documentation"></a>Документация

При ссылке на вкладки:

-   Используйте точный текст метки, включая его регистр букв, и включите вкладку Word.
-   Чтобы описать взаимодействие с пользователем, используйте кнопку.
-   По возможности отформатируйте метку, используя полужирный текст. В противном случае заключите метку в кавычки, только если это необходимо для предотвращения путаницы.
-   Поскольку несколько способов использования могут быть неоднозначными, особенно для Всемирной аудитории, используйте только вкладку существительное, чтобы ссылаться только на элемент управления "Вкладка". Для других целей уточните значение с помощью дескриптора: клавиши TAB, позиции табуляции или знака табуляции на линейке.

Пример: в меню **Сервис** выберите пункт **Параметры**, а затем перейдите на вкладку **вид** .
