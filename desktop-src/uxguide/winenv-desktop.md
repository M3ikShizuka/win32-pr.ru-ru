# <a name="desktop"></a>Рабочий стол

> [!NOTE]
> это руководство по проектированию было создано для Windows 7 и не обновлялось для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

Рабочий стол — это Рабочая область пользователя для своих программ. Это не способ информирования вашей программы или ее торговой марки. Не позлоупотребляйте!

рабочий стол — это рабочая область на экране, предоставляемая Microsoft Windows, аналогичная физическому рабочему столу. Она состоит из [рабочей области](glossary.md) и [панели задач](winenv-taskbar.md). Рабочая область может охватывать несколько мониторов.

Активный монитор — это монитор, в котором запущена активная программа. монитор по умолчанию — это один из меню, панели задач и области уведомлений.

## <a name="design-concepts"></a>Принципы проектирования

Windows desktop имеет следующие точки доступа к программе:

-   **Рабочая область.** Область экрана, где пользователи могут выполнять свою работу, а также сохранять программы, документы и их ярлыки. Хотя технически Рабочий стол включает панель задач, в большинстве контекстов она относится только к рабочей области.
-   **Кнопка запуска.** точка доступа для всех программ и специальных мест Windows (документы, изображения, музыка, игры, компьютер, панель управления) со списками недавно использовавшихся для быстрого доступа к недавно использованным программам и документам.
-   **Быстрый запуск. удалено из Windows 7.** Точка прямого доступа для программ, выбранных пользователем.
-   **Заполнен.** Точка доступа для запуска программ с присутствием рабочего стола. Технически панель задач охватывает всю строку с кнопки Пуск в область уведомлений. в большинстве контекстов панель задач — это область между, содержащей кнопки панели задач. Эта область иногда называется таскбанд.
-   **Дескбандс. Не рекомендуется.** Минимально функциональные и долгосрочные программы, такие как языковая панель. Программы, свернутые в дескбандс, не отображают кнопки панели задач при сворачивании.
-   **Область уведомлений.** Краткосрочный источник уведомлений и состояния, а также точка доступа для компонентов, связанных с системой и программами, которые не имеют присутствия на рабочем столе.

Никогда не просматривайте их как способы повышения осведомленности о программе или ее [торговой марке](exper-branding.md).

**Если вы выполняете только одно действие...**

-   Не придерживайтесь настольной системы — не контролируйте пользователей. Если целевые пользователи, скорее всего, часто используют вашу программу, то во время установки необходимо указать ярлык на рабочем столе, который не выбран по умолчанию.

## <a name="guidelines"></a>Рекомендации

-   **Если пользователи, скорее всего, часто используют вашу программу, во время установки необходимо указать ярлык программы на рабочем столе.** Большинство программ не будут использоваться достаточно часто для предоставления этого параметра.
-   **Представим, что параметр не выбран по умолчанию.** Требовать от пользователей выбора параметра важно, поскольку после того, как нежелательные значки находятся на рабочем столе, многие пользователи неохотно удалить их. Это может привести к ненужному рабочему столу.
-   **Если пользователь выбирает параметр, укажите только один ярлык программы.** Если продукт состоит из нескольких программ, предоставьте ярлык только основной программе.
-   **На рабочем столе следует размещать только ярлыки программ.** Не помещайте саму программу или другие типы файлов.

    **Правильно:**

    В неправильном примере программа, а не ярлык, копируется на Рабочий стол.

-   **Выберите метку, которая может отображаться без усечения.** Пользователи не должны видеть многоточие.

    **Правильно:**

    В неправильном примере метка ярлыка программы настолько длинна, что она усекается.
