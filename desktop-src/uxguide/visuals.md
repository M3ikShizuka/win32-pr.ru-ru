# <a name="visuals"></a>Визуальные элементы

> [!NOTE]
> Это руководство по проектированию было создано для Windows 7 и не Обновлено для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

В число визуальных элементов входят визуальные элементы, отличные от элементов управления. Эти рекомендации помогут принять решения о макете, шрифтах, цвете, значках и т. д. в приложении.

## <a name="in-this-section"></a>В этом разделе



| Раздел                                                       | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|-------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Макет](vis-layout.md)<br/>                         | Макет — это размер, интервал и размещение содержимого в окне или на странице. Эффективный макет очень важен для того, чтобы помочь пользователям найти то, что они ищут быстро, а также сделать внешний вид визуально привлекательным. Эффективный макет может сделать разницу между структурами, которые пользователи немедленно понимают, и теми, которые оставляют пользователей в головоломке и перегруженных.<br/>                                                                      |
| [Шрифты](vis-fonts.md)<br/>                           | Пользователи взаимодействуют с текстом больше, чем с любым другим элементом в Microsoft Windows. Segoe UI (произносится «см.») — это системный шрифт Windows. Стандартный размер шрифта был увеличен до 9 пунктов.<br/>                                                                                                                                                                                                                                                       |
| [Цвет](vis-color.md)<br/>                           | Color — это важный визуальный элемент большинства пользовательских интерфейсов. Помимо чистого эстетичность, цвет имеет связанные значения и найдет ответы эмоциональном. Чтобы избежать путаницы, необходимо постоянно использовать цвет. Чтобы получить нужные эмоциональном ответы, необходимо соответствующим образом использовать цвет.<br/>                                                                                                                                                    |
| [Значки](vis-icons.md)<br/>                           | Значки являются виде рисунков представлениями объектов, важно не только для Aesthetic причин в рамках визуального удостоверения программы, но также и для утилитарные причин, которые являются сокращением для передачи данных, которые пользователи воспринимают почти мгновенно. В Windows Vista появился новый стиль иконографи, который обеспечивает более высокий уровень детализации и сложности для Windows.<br/>                                                                          |
| [Стандартные значки](vis-std-icons.md)<br/>              | Стандартные значки — это значки ошибок, предупреждений, сведений и вопросительных знак, которые являются частью Windows.<br/>                                                                                                                                                                                                                                                                                                                                                  |
| [Анимации и переходы](vis-animations.md)<br/> | Стратегическое использование анимации и переходов позволяет упростить понимание программы, более плавную работу, более естественную и более качественную работу. Но использование анимаций и переходов может сделать программу нежелательным и даже неприятной. <br/>                                                                                                                                                                           |
| [Графические элементы](vis-graphic.md)<br/>              | *Графические элементы* визуально отображают связи, иерархии и выделение. К ним относятся фоны, плакаты, стекла, агрегаты, разделители, тени и маркеры.<br/>                                                                                                                                                                                                                                                                                      |
| [Звук](vis-sound.md)<br/>                           | *Звук* — это звуковой элемент пользовательского интерфейса. При правильном использовании звук может быть эффективной формой связи, которая устанавливает связь между текстовом и эмоциональном с пользователями. Звуки можно использовать отдельно или в качестве дополнения к визуальному пользовательскому интерфейсу. Например, Добавление звукового эффекта к уведомлению увеличивает вероятность того, что оно будет замечено, особенно если пользователь не смотрит на экран при возникновении события.<br/> |
