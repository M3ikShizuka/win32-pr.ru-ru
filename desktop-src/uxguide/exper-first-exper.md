# <a name="first-experience"></a>Первый интерфейс

> [!NOTE]
> Это руководство по проектированию было создано для Windows 7 и не Обновлено для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

При идеальном первом опыте пользователи устанавливают программу и используют ее немедленно, не отвечая на ряд вопросов или изучая множество вещей.

Пользовательский интерфейс первого интерфейса позволяет пользователям переходить с первого на новую программу или функцию на ежедневное использование.

При работе с программами Windows начальное первое взаимодействие происходит при запуске программы установки. Обычно программы установки:

-   Требовать от пользователя принять лицензионное соглашение (EULA).
-   Запросите ключ продукта.
-   Представьте необходимые параметры, связанные с конфигурацией, включая установку дополнительного программного обеспечения.
-   Скопируйте программное обеспечение на жесткий диск пользователя.
-   Представление параметров программы, относящихся ко всем пользователям.

![снимок экрана — диалоговое окно "введите ключ продукта" ](images/exper-first-exper-image1.png)

В рамках типичного процесса установки Windows.

Затем первый интерфейс переходит к первому использованию программы или компонента. Первый способ использования может:

-   Показать параметры программы, которые применяются только к текущему пользователю.
-   Предлагаемые учебники по продуктам и возможностям.

![снимок экрана с диалоговым окном "центр приветствия" ](images/exper-first-exper-image2.png)

Первый интерфейс использования.

**Примечание.** Рекомендации, связанные с [параметрами программы](win-property-win.md) , представлены в отдельной статье.

## <a name="is-this-the-right-user-interface"></a>Это правильный пользовательский интерфейс?

Для принятия решения примите во внимание следующие вопросы.

### <a name="setup-experience"></a>Процесс установки

Применяются ли следующие условия?

-   Для использования программы требуются правильные параметры, и они применяются ко всем пользователям.
-   Параметры изменяют основные принципы работы или имеют решающее значение для личной идентификации пользователя в программе.
-   Нет безопасного по умолчанию, пользователь, скорее всего, будет выбирать параметры, которые не являются параметрами по умолчанию, или для параметров по умолчанию требуется согласие пользователя.
-   Пользователю вряд ли придется изменять параметры после установки.
-   Для изменения параметров требуется повышение прав.

Если это так, рассмотрите возможность представления параметров во время установки.

### <a name="first-use-experience"></a>Опыт первого использования

Применяются ли следующие условия?

-   Для использования программы или компонента требуются правильные параметры или задачи. они применяются к отдельным пользователям.
-   Параметры изменяют основные принципы работы или имеют решающее значение для личной идентификации пользователя в программе.
-   Нет безопасного по умолчанию, пользователь, скорее всего, будет выбирать параметры, которые не являются параметрами по умолчанию, или для параметров по умолчанию требуется согласие пользователя.
-   Пользователи, скорее всего, будут лучше выбирать в контексте программы, чем в программе установки.
-   Пользователю вряд ли придется изменять параметры с помощью параметров.

Если это так, рассмотрите возможность представления задач и параметров во время первого использования программы или компонента.

## <a name="design-concepts"></a>Принципы проектирования

При идеальном первом опыте пользователи устанавливают программу (или даже просто запускаются, если не нуждаются в установке) и используют ее непосредственно без ответа на ряд вопросов или изучения множества вещей.

Это идеальный вариант для большинства программ, поэтому вы должны стремиться к такой идеальной работе, когда это возможно. Однако эта цель часто недоступна для программ, требующих значительной интеграции систем, имеет много дополнительных функций или имеет влияние на конфиденциальность. Например, если программа имеет функции, которые могут раскрывать персональные данные недоверенным сторонам, то перед включением этих [функций необходимо получить](https://www.microsoft.com/mscorp/twc/default.mspx) согласие пользователя.

### <a name="questions-arent-choices"></a>Вопросы не выбраны

Для получения ответов на вопросы необходимо ответить на них, прежде чем пользователи смогут продолжать работу. Вопросы во время первого процесса — это проблемы, с которыми пользователи должны перейти, прежде чем они смогут использовать вашу программу. Напротив, варианты выбора необязательны. Пользователи не должны отвечать на них или могут просматривать их только тогда, когда они хотят.

Поэтому параметры, представленные в главном потоке мастера установки, представляют собой вопросы, тогда как параметры за пределами главного потока установки или в диалоговом окне "параметры программы" являются вариантами. Ненужные вопросы делают первый интерфейс программы слишком громоздким и длинным, тем самым устраняя негативные ожидаемые результаты, и пользователи имеют дело с началом работы с программой.

### <a name="use-the-first-experience-when-you-must"></a>Используйте первый интерфейс при необходимости

Выводятся параметры и задачи для пользователей во время первого взаимодействия, когда необходимо, но, как правило, существуют лучшие альтернативы.

|---------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------|

| Вопросы первого использования<br/>              | Выберите нужные значения по умолчанию и разрешите пользователям изменять параметры программы.<br/>                                                            |
| Задачи первого использования<br/>                  | Вместо этого следует использовать контекстный режим.<br/>                                                                                                           |
| Первое использование объявлений функций<br/> | Сделайте наиболее распространенные и важные задачи обнаруживаемыми и контекстными.<br/>                                                                   |
| Учебники по первому использованию<br/>              | Самостоятельное создание программных функций.<br/>                                                                                                 |

**Если вы выполняете только одно действие...**

Постарайтесь как можно более простую работу. Вы сможете работать над вашей программой прямо сейчас. Выберите безопасные, безопасные, удобные значения по умолчанию и задайте вопросы во время установки и используйте их только при необходимости.

## <a name="guidelines"></a>Рекомендации

### <a name="general"></a>Общее

-   **Ограничьте первый опыт задач и параметров, необходимых для использования программы или компонента, и включайте их только при отсутствии лучшего альтернативы.** Альтернативные варианты см. в предыдущей таблице.
    -   **Исключение:** Добавляйте настройки персонализации или настройки программы к первому интерфейсу, если их настройка является частью основного интерфейса или имеет решающее значение для личной идентификации пользователя в программе.

![снимок экрана-диалоговое окно "введите имя компьютера" ](images/exper-first-exper-image3.png)

Windows запрашивает у пользователей имя компьютера и выбирает фон во время установки, так как эти параметры помогают формировать эмоциональном подключение к продукту.

-   **Используйте процесс установки** для задач и параметров, если они применяются ко всем пользователям или изменение параметров требует повышения прав.
-   **Используйте интерфейс первого использования** для задач и параметров, если они применяются к отдельным пользователям.

### <a name="presentation"></a>Уровень представления

-   **Предпочитать необязательные задачи и параметры обязательными задачами и параметрами.** Избегайте принудительной настройки программы пользователями.

    ![снимок экрана с диалоговым окном "Найдено новое оборудование" ](images/exper-first-exper-image4.png)

    Диалоговое окно Найдено новое оборудование делает необязательным установку драйвера, а не делает его обязательной задачей.

-   **При практическом использовании необязательные задачи и параметры изменяются из основного потока задач.** Например, многие программы установки предоставляют пользовательский путь установки для удаления редко изменяемых параметров из основного потока задач.

    ![снимок экрана с полными и пользовательскими переключателями ](images/exper-first-exper-image5.png)

    Процесс установки, который упрощает основной поток задач, если пользователь не планирует настраивать установку.

-   **Не перегружать пользователей с помощью задач и параметров:**
    -   **Начните с простого приложения.** Начните с простых, настроек персонализации и хода выполнения в сторону более сложных, технических задач и настроек. Например, программа установки Windows запускается с личными данными и заканчивается конфигурацией сети.
    -   **Используйте контекстный первый интерфейс** для задач и параметров, если они применяются только к тем функциям, которые не являются фундаментальными для основной программы.

        ![снимок экрана — диалоговое окно "Настройка звука и видео" ](images/exper-first-exper-image6.png)

        Windows Live Messenger имеет контекстную настройку для аудио и видео, так как они используются дополнительными компонентами.

-   **Не показывать все сразу.** Объедините, чтобы использовать один пользовательский интерфейс, а не несколько поверхностей пользовательского интерфейса, или отображать задачи в разное время, а не все одновременно.

    **Неправильно**:

    ![снимок экрана с пятью перекрывающимися диалоговыми окнами ](images/exper-first-exper-image7.png)

    В этом примере первый интерфейс использования является огромным.

-   **Выражают вопросы и варианты в терминах пользователей и задач, но не в плане технологий.** Укажите параметры, которые пользователи понимают и четко отличают. Обязательно предоставьте достаточно информации, чтобы пользователи могли принимать взвешенные решения.
-   **Если необходимость в личной информации не очевидна, объясните, почему программе требуется информация и как она будет использоваться.**

    ![снимок экрана с текстом, в котором используется адрес электронной почты ](images/exper-first-exper-image8.png)

    В этом примере приложение электронной коммерции объясняет, как будут использоваться персональные данные.

-   **Применяйте первый интерфейс в полноэкранном режиме только в том случае, если пользователи не могут выполнять другие задачи.** Например, программа установки Windows представлена в полноэкранном режиме, чтобы запретить пользователям выполнять другие задачи во время установки Windows. Первый интерфейс не должен быть полноэкранным.

### <a name="settings"></a>Параметры

-   **Для всех параметров выберите самый безопасный (чтобы предотвратить потери данных или доступ к системе), наиболее безопасное и частное значения по умолчанию.** Если безопасность и безопасность не являются факторами, выберите наиболее вероятное или удобное значение. Выбор хорошего значения по умолчанию — эффективный способ упростить первый интерфейс.
-   **Требовать от пользователей согласие на следующее:**

    -   Параметры с юридическими последствиями, например соглашения о лицензировании конечных пользователей (EULA). Такие параметры не могут иметь значения по умолчанию.
    -   Функции, выполняющие автоматические изменения конфигурации системы, такие как автоматическое обновление Windows.
    -   Функции, которые предоставляют персональные данные (PII) или сведения о системе.
    -   Изменение рабочего стола пользователя после добавления записей в меню «Пуск», например добавление значков на Рабочий стол или панель быстрого запуска.
    -   Дополнительное программное обеспечение, например улучшения продукта, подписки и продукты сторонних производителей.

    ![снимок экрана с диалоговым окном "Выбор нужных компонентов" ](images/exper-first-exper-image9.png)

    В этом примере пользователи хотят использовать улучшения продукта, подписки и продукты сторонних производителей.

-   **Если параметр настоятельно рекомендуется, добавьте к метке "(рекомендуется)".** Для переключателей и флажков обязательно добавьте к метке элемента управления, а не дополнительным примечаниям.
-   **Если параметр предназначен только для опытных пользователей, добавьте к метке "(дополнительно)".** Для переключателей и флажков обязательно добавьте к метке элемента управления, а не дополнительным примечаниям.

### <a name="tasks"></a>Задания

-   **Помогите пользователям передавать время ожидания в производство.**
    -   Если время ожидания обычно составляет от одного до двух минут, рассмотрите возможность предоставления полезной информации, когда пользователи ожидают, например, представление новых возможностей во время установки.
    -   Если время ожидания обычно превышает две минуты, упрощение пользователей для выполнения других задач. Отображать предполагаемое время ожидания, рекомендовать пользователям выполнять что-то другое и сделать задачу заметной, изменяя экран.
-   **Продумайте демонстрацию учебников во время первого впечатления.** Скорее всего, пользователи хотят сразу же использовать программу и заинтересовать учебники позже.
-   **Не используйте уведомления об объявлении компонентов в первом интерфейсе.** Вместо [уведомления об объявлении функций](mess-notif.md)Разработайте функцию, чтобы упростить обнаружение в контекстах, где это необходимо, или не делать ничего особенного и позволить пользователям самостоятельно обнаруживать эту функцию.
-   **Не используйте уведомления во время первоначальной работы Windows.** Чтобы улучшить его первый опыт, Windows 7 подавляет все уведомления, отображаемые в течение первых нескольких часов использования. Создайте программу, предполагая, что пользователи не увидят таких уведомлений.
