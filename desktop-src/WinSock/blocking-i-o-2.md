# <a name="blocking-inputoutput"></a>Блокировка ввода-вывода

простейшая форма ввода-вывода в сокетах Windows 2 — это блокировка операций ввода-вывода. Как упоминалось в [флагах и режимах атрибутов сокета](socket-attribute-flags-and-modes-2.md), сокеты создаются в блокирующем режиме по умолчанию. Любая операция ввода-вывода с блокирующим сокетом не будет возвращаться до полного завершения операции. Таким же потоком в каждый момент времени может выполняться только одна операция ввода-вывода. Например, если поток выдает операцию Receive и в настоящее время нет доступных данных, поток блокируется до тех пор, пока данные не станут доступны и помещаются в буфер потока. Хотя это и просто, это необязательно является самым эффективным способом ввода-вывода (см. сведения о [псевдолокализации и истинной блокировке](pseudo-vs--true-blocking-2.md) для получения дополнительных сведений).
