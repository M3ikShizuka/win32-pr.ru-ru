# <a name="network-location-awareness-service-provider-nla"></a>Поставщик службы обнаружения сетевых расположений (NLA)

персональные компьютеры, на которых работает Microsoft Windows, часто имеют множество сетевых подключений, например несколько сетевых карт (NIC), подключенных к разным сетям, или физическое сетевое подключение и коммутируемое подключение. Windows Сокеты способны перечислить доступные сетевые интерфейсы в течение некоторого времени, но некоторые важные сведения о сетевых подключениях ранее были недоступны. сюда входят такие сведения, как логическая сеть, к которой подключен Windows компьютер, или несколько интерфейсов, подключенных к одной сети.

поставщик службы сетевого расположения, обычно называемый NLA, позволяет приложениям Windows sockets 2 выявление логической сети, к которой подключен Windows компьютер. кроме того, NLA позволяет приложениям Windows sockets обнаруживать, какому физическому сетевому интерфейсу данного приложения были сохранены определенные сведения. NLA реализуется как универсальный поставщик службы разрешения имен Windows sockets 2.
