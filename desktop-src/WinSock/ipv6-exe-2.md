# <a name="ipv6exe"></a>Ipv6.exe

Все настройки IPv6 выполняются с помощью средства Ipv6.exe. Это средство в основном используется для запросов и настройки интерфейсов IPv6, адресов, кэшей и маршрутов. Ниже приведены подкоманды IPv6.

<dl> <dt>

<span id="ipv6_if__if__"></span><span id="IPV6_IF__IF__"></span>**IPv6, если** \[ наличии\#\]
</dt> <dd>

Отображает сведения об интерфейсах. Если указан номер интерфейса, отображается только информация об этом интерфейсе. В противном случае отображаются сведения обо всех интерфейсах. Выходные данные включают адрес канального уровня интерфейса и список IPv6-адресов, назначенных интерфейсу, включая текущий MTU интерфейса и максимальный (true) MTU, который может поддерживать интерфейс.

Интерфейс \# 1 — это псевдо-интерфейс, используемый для замыкания на себя. Интерфейс \# 2 — это псевдо-интерфейс, используемый для настроенного туннелирования, автоматического туннелирования и туннелирования 6to4. Другие интерфейсы нумеруются последовательно в порядке их создания. Этот порядок меняется от одного компьютера к другому.

Адреса канального уровня в *AA* - *BB* - *CC* - *дд*/с -  - форматом *FF* — это адреса Ethernet. Адрес уровня связи *в.* *б*. *в*. *d* — это интерфейсы с 6 по 4. Дополнительные сведения о 6-over-4 см. в RFC 2529.

Два псевдо-интерфейса не используют обнаружение соседей IPv6.

</dd> <dt>

<span id="ipv6_ifc_if___forwards___advertises___-forwards___-advertises___mtu__bytes___site_site-identifier_"></span><span id="IPV6_IFC_IF___FORWARDS___ADVERTISES___-FORWARDS___-ADVERTISES___MTU__BYTES___SITE_SITE-IDENTIFIER_"></span>**ИФК IPv6** , если \# \[ Пересылка \] \[ объявлений \] \[ -переслано \] \[ -объявлений \] \[ MTU \# байт сайт \] \[ -идентификатор\]
</dt> <dd>

Управляет атрибутами интерфейса. Интерфейсы могут пересылаться, в этом случае они пересылают пакеты с адресами назначения, не назначенными интерфейсу. Интерфейсы можно объявлять, в этом случае они отправляют объявления маршрутизатора. Эти атрибуты могут управляться независимо друг от друга. Интерфейс либо отправляет запросы маршрутизатора и получает объявления маршрутизатора, либо получает запросы маршрутизатора и отправляет объявления маршрутизатора.

Поскольку два псевдо-интерфейса не используют обнаружение соседей, они не могут быть настроены для отправки объявлений маршрутизатора.

Пересылка может быть сокращена как форв и объявлена как расширенная.

Также можно задать MTU для интерфейса. Новое значение MTU должно быть меньше или равно максимальному значению MTU (true) канала (по протоколу IPv6, если) и больше или равно минимальному значению параметра IPv6 MTU (1280 байт).

Идентификатор сайта для интерфейса также можно изменить. Идентификаторы сайтов используются в \_ поле идентификатора области sin6 \_ с адресами, локальными для сайта.

</dd> <dt>

<span id="ipv6_ifd_if_"></span><span id="IPV6_IFD_IF_"></span>**IPv6 IFD** , если\#
</dt> <dd>

Удаляет интерфейс. Невозможно удалить петлевой и туннельный псевдо-интерфейсы.

</dd> <dt>

<span id="ipv6_nc__if___address__"></span><span id="IPV6_NC__IF___ADDRESS__"></span>**IPv6 NC** \[ Если \# \[ адрес\]\]
</dt> <dd>

Отображает содержимое кэша соседей. Если указан номер интерфейса, то отображаются только содержимое кэша соседей этого интерфейса. В противном случае отображаются содержимое всех кэшей соседей интерфейса. Если указан интерфейс, можно указать IPv6-адрес для вывода только этой записи кэша соседей.

Для каждой записи кэша соседей отображаются интерфейс, адрес IPv6, адрес канального уровня и состояние REACH.

</dd> <dt>

<span id="ipv6_ncf__if___address__"></span><span id="IPV6_NCF__IF___ADDRESS__"></span>**НКФ IPv6** \[ Если \# \[ адрес\]\]
</dt> <dd>

Очищает указанные записи кэша соседей. Очищаются только записи кэша соседей без ссылок. Так как записи кэша маршрутов содержат ссылки на записи кэша соседей, кэш маршрутов должен быть сброшен первыми. Записи таблицы маршрутизации также могут содержать ссылки на записи кэша соседей.

</dd> <dt>

<span id="ipv6_rc__if__address_"></span><span id="IPV6_RC__IF__ADDRESS_"></span>**версия-кандидат IPv6** \[ Если \# адрес\]
</dt> <dd>

Отображает содержимое кэша маршрутов. Кэш маршрутов — это имя реализации IPv6 для целевого кэша. Если указан интерфейс и адрес, отображается запись кэша маршрутов для достижения адреса через интерфейс. В противном случае отображаются все записи кэша маршрутов.

Для каждой записи кэша маршрутов отображаются адрес IPv6 и текущий интерфейс следующего прыжка и адрес соседнего узла. Предпочтительный исходный адрес для использования с этим назначением, текущий MTU для достижения этого места назначения через интерфейс и определение того, является ли это записью кэша для конкретного интерфейса. Для адреса назначения также отображается любой адрес (для мобильных устройств).

Адрес назначения может иметь несколько записей кэша маршрутов — столько же, сколько один для каждого исходящего интерфейса. Однако адрес назначения может иметь не более одной записи кэша маршрутов, которая не зависит от интерфейса. Конкретный интерфейс — запись кэша маршрутов используется только в том случае, если приложение явно указывает на исходящий интерфейс.

</dd> <dt>

<span id="ipv6_rcf__if___address__"></span><span id="IPV6_RCF__IF___ADDRESS__"></span>**RCF IPv6** \[ Если \# \[ адрес\]\]
</dt> <dd>

Очищает указанные записи кэша маршрутов.

</dd> <dt>

<span id="ipv6_bc"></span><span id="IPV6_BC"></span>**IPv6 BC**
</dt> <dd>

Отображает содержимое кэша привязки, в котором содержатся привязки между домашними адресами и адресами для мобильных устройств IPv6.

Для каждой привязки отображаются домашний адрес, адрес для ввода в конце, порядковый номер привязки и время существования.

</dd> <dt>

<span id="ipv6_adu_if__address__lifetime_VL__PL____anycast___unicast_"></span><span id="ipv6_adu_if__address__lifetime_vl__pl____anycast___unicast_"></span><span id="IPV6_ADU_IF__ADDRESS__LIFETIME_VL__PL____ANYCAST___UNICAST_"></span>**аду IPv6** , если \# /Аддресс \[ время существования Корпоративная \[ \] \] \[ \] \[ рассылка ключом/pl рассылки\]
</dt> <dd>

Добавляет или удаляет назначение адресов одноадресной рассылки или адреса многоадресной рассылки в интерфейсе. Адрес одноадресной рассылки действует, если не указано пустое значение.

Если значение не указано, время существования бесконечно. Если указано только допустимое время существования, предпочтительное время жизни равно допустимому времени существования. Можно указать бесконечное время жизни или конечное значение в секундах. Предпочтительное время существования должно быть меньше или равно допустимому времени существования. Задание нулевого времени жизни приводит к удалению адреса.

Вы можете сократить время существования в течение срока жизни.

Для адресов многоадресной рассылки единственными допустимыми значениями времени жизни являются нуль и Infinite.

</dd> <dt>

<span id="ipv6_spt"></span><span id="IPV6_SPT"></span>**СПТ IPv6**
</dt> <dd>

Отображает текущее содержимое таблицы префиксов сайта.

Для каждого префикса сайта команда отображает префикс, интерфейс, к которому применяется префикс сайта, и время существования префикса в секундах.

Префиксы сайта обычно настраиваются в объявлениях маршрутизатора по-своему. Они используются в функции [**функцию getaddrinfo**](/windows/desktop/api/Ws2tcpip/nf-ws2tcpip-getaddrinfo) для фильтрации недопустимых адресов локального сайта.

</dd> <dt>

<span id="ipv6_spu_prefix_if___lifetime_L_"></span><span id="ipv6_spu_prefix_if___lifetime_l_"></span><span id="IPV6_SPU_PREFIX_IF___LIFETIME_L_"></span>префикс **IPv6 СПУ** , если \# \[ время жизни L\]
</dt> <dd>

Добавляет, удаляет или обновляет префикс в таблице префиксов сайта.

Требуется префикс и номер интерфейса. Время существования префикса сайта, указанное в секундах, по умолчанию равно бесконечности, если не указано. При указании нулевого времени жизни префикс сайта удаляется.

Эта команда не требуется для обычной конфигурации узлов или маршрутизаторов.

</dd> <dt>

<span id="ipv6_rt"></span><span id="IPV6_RT"></span>**IPv6 RT**
</dt> <dd>

Отображает текущее содержимое таблицы маршрутизации.

Для каждой записи таблицы маршрутизации команда отображает префикс маршрута, интерфейс связи или соседа следующего прыжка в интерфейсе, значение предпочтения (с меньшим приоритетом) и время существования в секундах.

Записи таблицы маршрутизации также могут иметь атрибуты **публикации** и **устаревания** . По умолчанию они возрастают (время существования меньше, чем оставшаяся константа) и не публикуются (не используются при формировании объявлений маршрутизатора).

Для узлов записи таблицы маршрутизации обычно настраиваются в объявлениях маршрутизатора по-своему.

</dd> <dt>

<span id="ipv6_rtu_prefix_if___nexthop___lifetime_L___preference_P___publish___age___spl_site-prefix-length_"></span><span id="ipv6_rtu_prefix_if___nexthop___lifetime_l___preference_p___publish___age___spl_site-prefix-length_"></span><span id="IPV6_RTU_PREFIX_IF___NEXTHOP___LIFETIME_L___PREFERENCE_P___PUBLISH___AGE___SPL_SITE-PREFIX-LENGTH_"></span>префикс **IPv6 рту** , если \# \[ /некссоп \] \[ время жизни L \] \[ предпочтение P \] \[ Публикация \] \[ возраст \] \[ СПЛ сайт-длина префикса\]
</dt> <dd>

Добавляет или удаляет маршрут в таблице маршрутизации. Требуется префикс маршрута. Префикс может быть ссылкой на указанный интерфейс или следующий прыжок, указанный с адресом соседа в интерфейсе. Маршрут может иметь время существования в секундах, с бесконечным значением по умолчанию и предпочтением со значением по умолчанию 0 или наиболее предпочтительным. При указании нулевого времени жизни маршрут удаляется.

Если маршрут указан как опубликованный, указывая, что он будет использоваться при создании объявлений маршрутизатора, это не возрастет. Время существования маршрута не учитывается, поэтому оно фактически бесконечно, но значение используется в объявлениях маршрутизатора. При необходимости маршрут может быть указан как опубликованный маршрут, который также возрастает. Неопубликованный маршрут по умолчанию всегда возрастает.

Необязательный подпараметр СПЛ можно использовать для указания длины префикса сайта, связанной с маршрутом. Длина префикса сайта используется только при отправке объявлений маршрутизатора.

Время жизни может быть сокращено как жизненный, предпочтения как преф и опубликовано как Pub.

</dd> </dl>
