# <a name="application-behavior"></a>Поведение приложения

Другим аспектом разработки приложений, который следует рассмотреть, является разница в поведении локальных или интракомпутер операций и поведение при выполнении операций между двумя компьютерами, подключенными к сети. Существуют поведения приложений, которые могут хорошо работать на локальном компьютере, но при работе по сети это приводит к значительному снижению производительности и потреблению ресурсов. при разработке приложений Windows sockets следует избегать следующих поведений приложений.

## <a name="behaviors-to-avoid"></a>Варианты поведения, которые следует избегать

-   Приложения для разговора.

    Некоторые приложения выполняют много небольших транзакций. При объединении сетевых издержек, связанных с каждой такой транзакцией, этот результат умножается. В сети небольшие транзакции могут потреблять столько ресурсов и столько же времени, сколько и большие транзакции. Лучшим подходом является объединение большого количества небольших транзакций в одну большую транзакцию.

-   Сериализованные транзакции.

    Ненужная сериализация транзакций может привести к снижению производительности и повлиять на масштабируемость. Например, 1000. для завершения сериализованных транзакций требуется по крайней мере 1000 \* RTT. Лучшим подходом является параллельное выполнение несвязанных транзакций. Когда сериализованные приложения объединяются с приложениями в чате, скорость реагирования может быть серьезно затруднна.

    > [!Note]  
    > Правильная десериализация приложения может быть сложной. Если переход от сериализованного к параллельному оказывается слишком сложным, рассмотрите возможность объединения нескольких транзакций в меньшее количество больших транзакций.

     

-   Транзакции FAT.

    Приложения, отправляющие ненужные байты в сети, считаются приложениями FAT. Приложения, отправляющие ненужные байты в сети, увеличивают нагрузку на сеть и снижают производительность. Эта ситуация часто возникает из неэффективных структур данных или неэффективной потоковой передачи данных. Для устранения этой проблемы Оптимизируйте структуры данных или используйте сжатие.
