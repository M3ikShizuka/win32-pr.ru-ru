# <a name="reliable-multicast-programming-pgm"></a>Безопасное многоадресное программирование (PGM)

В этом разделе описывается реализация протокола многоадресной рассылки PGM в Windows, которая часто называется надежной многоадресной рассылкой. Надежная многоадресная рассылка реализуется через сокеты Windows в Windows Server 2003 и более поздних версиях.

**Windows XP:** Протокол PGM поддерживается только в том случае, если установлена служба очередей сообщений Майкрософт (MSMQ) 3,0.

PGM — это надежный и масштабируемый протокол многоадресной рассылки, позволяющий получателям обнаруживать потери, запрашивать повторную отправку потерянных данных или уведомлять приложение о неустранимой потере. PGM — это надежный протокол, который означает, что получатель отвечает за обеспечение получения всех данных, абсолвинг отправителя за получение.

Протокол PGM подходит для приложений, для которых требуется дублирование данных многоадресной рассылки из нескольких источников в несколько получателей. Протокол PGM не поддерживает подтвержденную доставку и не гарантирует упорядочивания пакетов от нескольких отправителей.

Дополнительные сведения о протоколе PGM см. в документе RFC 3208, доступном по адресу [www.IETF.org](https://www.ietf.org/).
