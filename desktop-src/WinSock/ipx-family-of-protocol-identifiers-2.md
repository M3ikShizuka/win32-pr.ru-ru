# <a name="ipx-family-of-protocol-identifiers"></a>Семейство идентификаторов протоколов IPX

Параметр *протокола* в [**Socket**](/windows/desktop/api/Winsock2/nf-winsock2-socket) и [**всасоккет**](/windows/desktop/api/Winsock2/nf-winsock2-wsasocketa) — это идентификатор, который устанавливает тип сети и метод для определения различных транспортных протоколов, работающих в сети. В отличие от IP, IPX не использует одно значение протокола для выбора транспортного стека. Поскольку нет необходимости использовать определенное значение для каждого транспортного протокола, мы можем назначить их удобным для приложений Winsock. Мы не будем использовать значения 0 – 255, чтобы избежать конфликтов с соответствующими \_ значениями протокола PF inet.



| Имя         | Значение       | Типы сокетов              | Описание                                         |
|--------------|-------------|---------------------------|-----------------------------------------------------|
| Зарезервировано     | 0–255       |                           | Зарезервировано для \_ значений протокола PF inet.              |
| НСПРОТО \_ IPX | 1000-1255 | Сокк \_ ДГРАМ Сокк \_ RAW     | Служба датаграмм для IPX.                           |
| НСПРОТО \_ SPX | 1256        | Сокк \_ Stream Сокк \_ секпкт | Надежный обмен пакетами с использованием пакетов фиксированного размера. |



 

> [!Note]  
> Если \_ указан нспрото SPX, Протокол SPX II автоматически используется, если обе конечные точки способны сделать это.
