# <a name="windows-sockets-compatibility-issues"></a>Windows Проблемы совместимости сокетов

Windows сокеты 2 по-Windows поддерживают все семантики сокетов 1,1 и вызовы функций, за исключением тех, которые обработаны с псевдо-блокировкой. поскольку сокеты Windows 2 выполняются только в 32-битных запланированных средах с вытеснением, нет необходимости реализовывать псевдо-блокировку, обнаруженную в Windows сокетах 1,1. это означает, что код ошибки всаеинпрогресс никогда не будет указан и что следующие функции Windows сокеты 1,1 недоступны для приложений Windows sockets 2:

-   всаканцелблоккингкалл
-   всаисблоккинг
-   всасетблоккингхук
-   всаунхукблоккингхук

Windows Программы сокетов 1,1, написанные для использования псевдо-блокировки, будут работать правильно, так как они связаны с Winsock.dll или Wsock32.dll. обе функции продолжают поддерживать полный набор функций Windows сокетов 1,1. чтобы программы стали Windows сокеты 2 приложения, необходимо внести некоторые изменения в код. В большинстве случаев разумное использование потоков можно заменить на обработку, которая была выполнена с помощью блокирующей функции-ловушки.
