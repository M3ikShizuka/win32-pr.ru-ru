# <a name="event-object-signaling"></a>Сигнализация объекта события

[**вспевентселект**](/previous-versions/windows/hardware/network/ff566287(v=vs.85)) работает точно так же, как и [**вспасинкселект**](/previous-versions/windows/desktop/legacy/ms742267(v=vs.85)) , за исключением того, что вместо того, чтобы Windows сообщение отослано при возникновении любого назначенного события сети демона \_ (например, "демон \ \_ чтение", "запись демо- \_ " и т. д.), задается заданный объект события.

Кроме того, тот факт, что \_ произошло определенное сетевое событие демона "% XXX", запоминается поставщиком услуг. Это необходимо, так как при возникновении всех заданных сетевых событий, а также при появления в нем одного объекта события будет получен сигнал. Вызов [**вспенумнетворкевентс**](/previous-versions/windows/hardware/network/ff566284(v=vs.85)) приводит к тому, что текущее содержимое памяти сетевого события копируется в буфер, предоставляемый клиентом, и память сетевого события для очистки. Клиент также может обозначать определенный объект события, который будет удаляться атомарно вместе с памятью сетевых событий.
