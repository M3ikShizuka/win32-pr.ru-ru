# <a name="error-reporting-and-parameter-validation"></a>Отчеты об ошибках и проверка параметров

Схема для отчетов об ошибках отличается между интерфейсами SPI и API. Windows Поставщики услуг сокетов сообщают об ошибках вместе с возвратом функции, в отличие от подхода, основанного на потоках, который используется в API. 32.dll Ws2 \_ использует код ошибки для каждой функции поставщика услуг, чтобы обновить значение ошибки потока, полученное с помощью функции API [**всажетластеррор**](/windows/desktop/api/winsock/nf-winsock-wsagetlasterror) . Однако поставщики услуг по-прежнему требуются для поддержания ошибок на основе сокетов, которые можно получить с помощью параметра, чтобы использовать \_ сокет с ошибками.

32.dll Ws2 \_ выполняет проверку параметров только для вызовов функций, реализованных полностью внутри самой себя. Поставщики услуг отвечают за выполнение всех собственных проверок параметров.
