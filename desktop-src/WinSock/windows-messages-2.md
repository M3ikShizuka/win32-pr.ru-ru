# <a name="windows-messages"></a>Windows Ошибка

Windows В сокетах 1,1 появился механизм асинхронного выбора для предоставления сетевых событий, не использующих опрос или блокировку. Функция [**вспасинкселект**](/previous-versions/windows/desktop/legacy/ms742267(v=vs.85)) используется для регистрации интереса в одном или нескольких сетевых событиях, как указано в предыдущем примере, и для предоставления маркера окна, используемого для уведомления. когда происходит заданное сетевое событие, в указанное окно отправляется указанное клиентом сообщение Windows. Для выполнения этой задачи поставщик услуг должен использовать функцию [**впупостмессаже**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpupostmessage) .

в Windows это может быть не самым эффективным механизмом уведомлений и неудобным для управляющих программ и служб, которые не хотят открывать окна. Для решения этой проблемы появился механизм сигнализации объекта событий, описанный в следующем примере.
