# <a name="windows-messages"></a>Сообщения Windows

Сокеты Windows 1,1 предоставили механизм асинхронного выбора для предоставления сетевых событий, не использующих опрос или блокировку. Функция [**вспасинкселект**](/previous-versions/windows/desktop/legacy/ms742267(v=vs.85)) используется для регистрации интереса в одном или нескольких сетевых событиях, как указано в предыдущем примере, и для предоставления маркера окна, используемого для уведомления. При возникновении назначенного сетевого события в указанное окно отправляется указанное клиентом сообщение Windows. Для выполнения этой задачи поставщик услуг должен использовать функцию [**впупостмессаже**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpupostmessage) .

В Windows это может быть не самым эффективным механизмом уведомлений и неудобным для управляющих программ и служб, которые не хотят открывать окна. Для решения этой проблемы появился механизм сигнализации объекта событий, описанный в следующем примере.
