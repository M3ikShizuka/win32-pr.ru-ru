# <a name="blocking-operations"></a>Блокирующие операции

Понятие блокировки в среде Windows было очень важным. В средах Windows Sockets 1,1 блокирующие вызовы не были бы рекомендуются, так как они были отключены для постоянного взаимодействия с системой. Кроме того, они используют псеудоблоккинг методику, которая по различным причинам не всегда работает должным образом. Однако в запланированных с приоритетами средах Windows, блокирующие вызовы гораздо более осмысленны, могут быть реализованы собственными службами операционной системы и фактически являются предпочтительным механизмом. API Winsock 2 больше не поддерживает псуедоблоккинг, но поскольку оболочки совместимости Windows Sockets 1,1 должны продолжать эмуляцию этого поведения, поставщики услуг должны поддерживать это, как описано ниже.
