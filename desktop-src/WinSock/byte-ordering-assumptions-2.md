# <a name="byte-ordering-assumptions"></a>Предположения порядка байтов

Поставщик услуг должен обрабатывать все компоненты [**SOCKADDR**](sockaddr-2.md) , за исключением параметра семейства адресов, как если бы они направляются в сетевой последовательности байтов, а параметр семейства Address — в порядке байтов на локальном компьютере. Это обязанность приложения Winsock, обеспечивающая правильную компоновку адресов, содержащихся в структурах **SOCKADDR** . API Winsock предоставляет ряд подпрограмм преобразования для упрощения этой задачи. В настоящее время эти подпрограммы понимают преобразование между естественным порядком байтов на локальном узле и порядком байтов в сети с обратным порядком байтов или с прямым порядком байтов. Архитектура Winsock может поддерживать другие схемы порядка байтов в будущем.
