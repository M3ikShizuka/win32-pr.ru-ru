# <a name="windows-sockets-2"></a>Windows Сокеты 2

## <a name="purpose"></a>Назначение

Windows Сокеты 2 (Winsock) позволяют программистам создавать расширенные Интернет, интрасети и другие приложения, поддерживающие работу в сети, для передачи данных приложений по сети независимо от используемого сетевого протокола. с помощью Winsock программисты получают доступ к расширенным возможностям Microsoft® Windows® сетевые возможности, такие как многоадресная рассылка и качество обслуживания (QoS).

Winsock соответствует модели Windows Open System Architecture (воса). Он определяет стандартный интерфейс поставщика услуг (SPI) между прикладным программным интерфейсом (API), его экспортированными функциями и стеками протоколов. Он использует парадигму сокетов, которая была впервые популярна с помощью Berkeley Software Distribution (BSD) UNIX. позднее оно было адаптировано для Windows в Windows сокетах 1,1, с которыми приложения Windows sockets 2 являются обратно совместимыми. Программное обеспечение Winsock, ранее выровненное по протоколу TCP/IP. Некоторые методы программирования, которые работали с TCP/IP, не работают с каждым протоколом. в результате API Windows сокеты 2 добавляют функции, необходимые для работы с несколькими протоколами.

## <a name="developer-audience"></a>Аудитория разработчиков

Windows Сокеты 2 предназначены для использования программистами C/C++. необходимо ознакомиться с Windows сетью.

## <a name="run-time-requirements"></a>Требования к среде выполнения

Windows сокеты 2 можно использовать на всех Windowsных платформах. если существуют определенные реализации или возможности Windows сокеты 2 ограничения платформы, они четко указаны в документации.
