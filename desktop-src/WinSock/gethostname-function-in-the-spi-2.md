# <a name="gethostname-function-in-the-spi"></a>Функция onhostname в SPI

Запрос [**всалукупсервицебегин**](/windows/desktop/api/Winsock2/nf-winsock2-wsalookupservicebegina) использует \_ имя узла СВЦИД в качестве идентификатора GUID класса службы. Если *лпсзсервицеинстанценаме* имеет значение null или ссылается на пустую строку (то есть. ""), локальный узел должен быть разрешен. В противном случае должно произойти Поиск по указанному имени узла. В целях эмуляции Зовите Ws2 [**,**](/windows/desktop/api/winsock/nf-winsock-gethostname) \_32.dll задаст указатель null для *лпсзсервицеинстанценаме* и укажите имя возвращаемого значения Луп, \_ \_ чтобы имя узла возвращалось в параметре *лпсзсервицеинстанценаме* . Если приложение использует этот запрос и указывает \_ возвращаемый Луп, \_ адрес узла будет указан в структуре **ксаддр \_ info** . \_ \_ Действие возврата BLOB-объекта Луп не определено для этого запроса. Сведения о порте будут по умолчанию равняться нулю, если только *лпсзкуеристринг* не ссылается на службу, например FTP, и в этом случае будет предоставляться полный транспортный адрес указанной службы.
