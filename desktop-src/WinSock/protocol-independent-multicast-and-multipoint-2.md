# <a name="protocol-independent-multicast-and-multipoint"></a>Protocol-Independent многоадресной рассылки и MultiPoint

Windows Сокеты 2 предоставляют универсальный метод для использования возможностей MultiPoint и многоадресной рассылки для транспортов. Этот универсальный метод реализует эти функции точно так же, как позволяет получить доступ к базовым возможностям транспорта данных многочисленных транспортных протоколов. Термин MultiPoint используется в дальнейшем для ссылки на многоадресную и MultiPoint связь.

Текущие реализации MultiPoint (например, многоадресная рассылка IP-адресов, ST-II, T. 120 и ATM UNI) могут сильно различаться. Как узлы присоединяются к сеансу MultiPoint, определяется ли конкретный узел как Центральный или корневой узел, а также происходит ли обмен данными между всеми узлами или только между корневым узлом и различными конечными узлами в разных реализациях. структура [**\_ сведений всапротокол**](/windows/win32/api/winsock2/ns-winsock2-wsaprotocol_infoa) для сокетов Windows 2 используется для объявления различных атрибутов multipoint протокола. изучив эти атрибуты, программист знает, какие соглашения следует соблюдать с помощью подходящих функций Windows сокетов 2 для настройки, использования и уничтожения сеансов multipoint.

Ниже приведены сводные сведения о функциях Winsock, поддерживающих MultiPoint.

-   Биты с двумя атрибутами в [**структуре \_ сведений всапротокол**](/windows/win32/api/winsock2/ns-winsock2-wsaprotocol_infoa) .
-   Четыре флага, определенные для параметра *dwFlags* функции [**всасоккет**](/windows/desktop/api/Winsock2/nf-winsock2-wsasocketa) .
-   Одна функция [**всажоинлеаф**](/windows/desktop/api/Winsock2/nf-winsock2-wsajoinleaf)для добавления конечных узлов в сеанс MultiPoint
-   Два кода команд [**всаиоктл**](/windows/desktop/api/Winsock2/nf-winsock2-wsaioctl) для управления замыканием на себя MultiPoint и создание области для многоадресных передач. (Последнее относится к параметрам срока жизни или TTL многоадресной рассылки.)

> [!Note]  
> включение этих функций multipoint в сокеты Windows 2 не исключает применения существующего интерфейса, зависящего от протокола, такого как параметры сокета диринг для многоадресной IP-рассылки.
