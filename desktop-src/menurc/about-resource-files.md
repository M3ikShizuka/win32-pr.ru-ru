# <a name="about-resource-files"></a>Сведения о файлах ресурсов

чтобы включить ресурсы в приложение на основе Windows с помощью версии-кандидата, выполните следующие действия.

1.  Создание отдельных файлов для курсоров, значков, растровых изображений, диалоговых окон и шрифтов.
2.  Создайте скрипт определения ресурса (RC-файл), описывающий ресурсы, используемые приложением.
3.  Скомпилируйте скрипт с RC. Дополнительные сведения см. [в разделе Использование версии-кандидата (Командная строка RC)](using-rc-the-rc-command-line-.md).
4.  Свяжите файл скомпилированного ресурса (RES) с исполняемым файлом приложения с вашим компоновщиком.

Файл ресурсов — это текстовый файл с расширением RC. Файл может использовать однобайтовые, двухбайтовые или символы Юникода. Синтаксис и семантика предварительной версии RC аналогичны компиляторам Microsoft C/C++. Однако версия-кандидат поддерживает подмножество директив препроцессора, определяет и директивы pragma в скрипте.

В файле скрипта определяются ресурсы. Для ресурса, который существует в отдельном файле, таком как значок или курсор, сценарий указывает ресурс и файл, который его содержит. Для некоторых ресурсов, таких как меню, в сценарии существует полное определение ресурса.

В следующих разделах описываются сведения, которые может содержать файл скрипта:

-   [Комментарии](comments.md), которые должны игнорироваться версией-кандидатом.
-   [Предопределенные макросы](predefined-macros.md), которые не принимают аргументы и не могут быть переопределены.
-   [Директивы препроцессора](preprocessor-directives.md), которые указывают RC на выполнение действий над сценарием перед его компиляцией.
-   [Операторы препроцессора](preprocessor-operators.md), которые используются с директивой **\# define** .
-   [Директивы Pragma](pragma-directives.md)
-   [Операторы определения ресурсов](resource-definition-statements.md), которые содержат имена и описания ресурсов.
