# <a name="searching-for-files"></a>Поиск файлов

По умолчанию версия-кандидат выполняет поиск файлов заголовков и файлов ресурсов (таких как файлы значков и курсоров) в текущем каталоге, а затем в каталогах, заданных переменной среды INCLUDE. (Переменная среды PATH не влияет на каталоги, поиск которых выполняется в RC.)

## <a name="adding-a-directory-to-search"></a>Добавление каталога для поиска

С помощью параметра **/i** можно добавить каталог в список каталогов поиска RC. Затем компилятор выполняет поиск в каталогах в следующем порядке:

1.  Текущий каталог
2.  Каталог или каталоги, указанные с помощью параметра **/i** , в том порядке, в котором они отображаются в командной строке RC.
3.  Список каталогов, заданных переменной среды INCLUDE, в порядке их перечисления в переменной, если не указан параметр **/x**

В следующем примере компилируется файл определения ресурса MyApp. rc:

**RC/i c: \\ Источник: \\ /i d: \\ Resources MyApp. RC**

При компиляции скрипта MyApp. RC версия-кандидат ищет файлы заголовков и файлы ресурсов в текущем каталоге, затем в C: \\ Источник \\ и D: \\ Resources, а затем в каталогах, заданных переменной среды include.

## <a name="ignoring-the-include-environment-variable"></a>Пропуск переменной среды INCLUDE

При определении каталогов для поиска в версии-КАНДИДАТе можно запретить использовать переменную среды INCLUDE. Для этого используйте параметр **/x** . Затем компилятор ищет файлы только в текущем каталоге и в любых каталогах, указанных с помощью параметра **/i** .

Следующая команда компилирует файл скрипта MyApp. rc:

**RC/x/i c: \\ Источник \\ материалов MyApp. RC**

При компиляции скрипта MyApp. RC версия-кандидат сначала ищет файлы заголовков и файлы ресурсов в текущем каталоге, а затем в C: \\ Источник \\ . Он не выполняет поиск в каталогах, заданных переменной среды INCLUDE.
