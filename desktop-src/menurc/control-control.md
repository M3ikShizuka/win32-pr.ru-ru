# <a name="control-control"></a>Элемент управления

Определяет определяемый пользователем элемент управления.

``` syntax
CONTROL text, id, class, style, x, y, width, height [, extended-style]
```

<dl> <dt>

<span id="class"></span><span id="CLASS"></span>*см*
</dt> <dd>

Переопределенное имя, символьная строка или 16-битовое целочисленное значение без знака, определяющее класс. Это может быть любой из классов элементов управления. Список классов элементов управления см. в первом списке, следующем за этим описанием. Если значением является переопределенное имя, предоставленное приложением, оно должно быть строкой, заключенной в двойные кавычки (").

</dd> <dt>

<span id="style"></span><span id="STYLE"></span>*стиль*
</dt> <dd>

Переопределенное имя или целочисленное значение, определяющее стиль данного элемента управления. Точное значение *стиля* зависит от значения *класса* . В разделах, следующих за этим описанием, показаны классы элементов управления и соответствующие стили.

</dd> </dl>

Дополнительные сведения об общем синтаксисе инструкции Control см. в разделе [Общие параметры управления](common-control-parameters.md).

## <a name="remarks"></a>Комментарии

Шесть возможных классов элементов управления описаны в следующих разделах.

### <a name="the-button-control-class"></a>Класс элемента управления Button

Элемент управления "Кнопка" — это небольшое прямоугольное дочернее окно, которое пользователь может включить или отключить, щелкнув его мышью. Элементы управления "Кнопка" могут использоваться отдельно или в группах и могут быть помечены или отображаться без текста. Элементы управления "Кнопка" обычно меняют внешний вид, когда пользователь щелкает их.

Стили кнопок описаны в следующем разделе: [стили кнопок](../controls/button-styles.md).

### <a name="the-combo-box-control-class"></a>Класс элемента управления "поле со списком"

Элементы управления "поле со списком" состоят из поля выбора, похожего на элемент управления редактирования и поле со списком. Список может отображаться в любое время или отбрасываться, когда пользователь выбирает "всплывающее окно" рядом с полем выбора.

В зависимости от стиля поля со списком пользователь может или не может изменять содержимое поля выбора. Если список является видимым, ввод символов в поле выбора приведет к выделению первой записи, соответствующей введенным символам. И наоборот, при выборе элемента в списке выделенный текст отображается в поле выбора.

Стили элемента управления "поле со списком" описаны в следующем разделе: [стили полей со](../controls/combo-box-styles.md)списками.

### <a name="the-edit-control-class"></a>Класс элемента управления Edit

Элемент управления "поле ввода" — это прямоугольное дочернее окно, в котором пользователь может вводить текст с клавиатуры. Пользователь выбирает элемент управления и передает ему фокус ввода, нажимая внутри него указатель мыши или нажимая клавишу TAB. Пользователь может ввести текст, когда элемент управления отображает мигающий курсор вставки. Мышь может использоваться для перемещения курсора и выбора символов для замены или для позиционирования курсора для вставки символов. Для удаления символов можно использовать клавишу BACKSPACE.

Элемент управления "поле ввода" использует шрифт фиксированной высоты и отображает символы Юникода. Они разворачивают символы табуляции в столько пробелов, сколько требуется для перемещения курсора на следующую позицию табуляции. Позиции табуляции считаются на каждые восьмой знак.

Стили элемента управления Edit описаны в следующем разделе: [Edit Control Styles](../controls/edit-control-styles.md).

### <a name="the-list-box-control-class"></a>Класс элемента управления "список"

Элементы управления "список" состоят из списка символьных строк. Элемент управления используется каждый раз, когда приложению требуется предоставить список имен, таких как имена файлов, которые пользователь может просматривать и выбирать. Пользователь может выбрать строку, указав строку с помощью мыши и нажав кнопку мыши. Если строка выбрана, она выделяется и сообщение уведомления передается родительскому окну. Полосу прокрутки можно использовать с элементом управления "список" для прокрутки слишком длинных или слишком широких списков для окна управления.

Стили элемента управления "список" описаны в следующем разделе: [стили списка](../controls/list-box-styles.md).

### <a name="the-scroll-bar-control-class"></a>Класс элемента управления Scroll-Bar

Элемент управления "полоса прокрутки" — это прямоугольник, который содержит бегунок прокрутки и имеет стрелки направления на обоих концах. Полоса прокрутки отправляет сообщение уведомления родительскому элементу всякий раз, когда пользователь щелкает мышью в элементе управления. Родительский элемент отвечает за обновление расположения бегунка (при необходимости). Элементы управления "полоса прокрутки" имеют те же внешний вид и функции, что и полосы прокрутки, используемые в обычных окнах. Но в отличие от полос прокрутки, элементы управления "полоса прокрутки" можно располагать в любом месте окна и использовать при необходимости для обеспечения прокрутки окна.

Стили полосы прокрутки описаны в следующем [разделе.](../controls/scroll-bar-control-styles.md)

### <a name="the-static-control-class"></a>Статический класс элемента управления

Статические элементы управления — это простые текстовые поля, квадраты и прямоугольники, которые можно использовать для обозначения, бокса или разделения других элементов управления. Статические элементы управления не принимают входные данные и не предоставляют выходных данных.

Стили статических элементов управления описаны в следующем разделе: [стили статических элементов управления](../controls/static-control-styles.md).
