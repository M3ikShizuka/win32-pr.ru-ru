# <a name="mrmcreateconfig-function"></a>Функция Мрмкреатеконфиг

\[Некоторые сведения относятся к предварительно выпущенному продукту, который может быть значительно изменен перед коммерческой выпуском. Майкрософт не предоставляет никаких гарантий, явных или подразумеваемых, относительно приведенных здесь сведений.\]

Создает новый инициализированный файл конфигурации PRI, определяющий заданные по умолчанию квалификаторы. Дополнительные сведения и пошаговые руководства по использованию этих API см. в разделе [пакетные API индексирования ресурсов (PRI) и пользовательские системы сборки](/windows/uwp/app-resources/pri-apis-custom-build-systems).

## <a name="syntax"></a>Синтаксис


```C++
HRESULT HRESULT MrmCreateConfig(
  _In_     MrmPlatformVersion platformVersion,
  _In_opt_ PCWSTR             defaultQualifiers,
  _In_     PCWSTR             outputXmlFile
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*платформверсион* \[ окне\]
</dt> <dd>

Тип: **[ **мрмплатформверсион**](mrmplatformversion.md)**

Версия платформы (*targetOsVersion*), используемая для создаваемого файла конфигурации.

</dd> <dt>

*дефаулткуалифиерс* \[ в необязательное\]
</dt> <dd>

Тип: **пквстр**

Список квалификаторов ресурсов по умолчанию. Например, L "язык-EN-US \_ Scale-100 \_ контраст — Standard"

</dd> <dt>

*аутпутксмлфиле* \[ окне\]
</dt> <dd>

Тип: **пквстр**

Путь к создаваемому файлу конфигурации.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **HRESULT**

\_ОК, если функция прошла успешно, в противном случае — другое значение. Чтобы определить успешность или сбой, используйте макросы Success () или FAILed () (определенные в файле Winerror. h).
