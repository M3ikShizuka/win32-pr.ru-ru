# <a name="user-defined-resource"></a>Ресурс User-Defined

Определяемая пользователем инструкция определения ресурса определяет ресурс, содержащий данные, относящиеся к приложению. Данные могут иметь любой формат и могут быть определены как содержимое заданного файла (если задан параметр *filename* ) или как последовательность чисел и строк (если указан *необработанный блок данных* ).

``` syntax
nameID typeID filename
```

Имя *файла указывает файл* , содержащий двоичные данные ресурса. Содержимое файла включается в качестве ресурса. RC не интерпретирует двоичные данные каким-либо образом. Ответственность за то, чтобы данные должным образом совпадали с архитектурой целевого компьютера, лежит на программисте.

Определяемый пользователем ресурс также можно полностью определить в скрипте ресурсов, используя следующий синтаксис:

``` syntax
nameID typeID  {  raw-data  }
```

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="nameID"></span><span id="nameid"></span><span id="NAMEID"></span>*nameID*
</dt> <dd>

Уникальное имя или 16-разрядное целое число без знака, идентифицирующее ресурс.

</dd> <dt>

<span id="typeID"></span><span id="typeid"></span><span id="TYPEID"></span>*Типа*
</dt> <dd>

Уникальное имя или 16-разрядное целое число без знака, определяющее тип ресурса. Если указано число, оно должно быть больше 255. Числа от 1 до 255 зарезервированы для существующих и будущих переопределенных типов ресурсов.

</dd> <dt>

<span id="filename"></span><span id="FILENAME"></span>*файлов*
</dt> <dd>

Имя файла, содержащего данные ресурса. Параметр должен быть допустимым именем файла; Он должен быть полным путем, если файл не находится в текущем рабочем каталоге.

</dd> <dt>

<span id="raw-data"></span><span id="RAW-DATA"></span>*необработанные данные*
</dt> <dd>

Необработанные данные, состоящие из одного или нескольких целых чисел или строк символов. Целые числа могут быть заданы в десятичном, восьмеричном или шестнадцатеричном формате. для совместимости с 16-разрядными Windows целые числа хранятся в виде значений слов. Целое число можно сохранить как значение типа DWORD, указав для целого числа суффикс "L".

Строки заключаются в кавычки. RC не добавляет автоматически завершающий нуль-символ к строке. Каждая строка является последовательностью указанных символов ANSI, если она не указана в виде строки расширенных символов с префиксом "L".

Блок данных начинается с границы **DWORD** , а RC не выполняет заполнение или выравнивание данных в блоке *необработанных данных* . Ответственность за то, чтобы обеспечить правильное выравнивание данных внутри блока, лежит на программисте.

</dd> </dl>

## <a name="example"></a>Пример

В следующем примере показано несколько определяемых пользователем инструкций:

``` syntax
array   MYRES   data.res
14      300     custom.res
18 MYRES2
{
   "Here is an ANSI string\0",    // explicitly null-terminated 
   L"Here is a Unicode string\0", // explicitly null-terminated 
   1024,                          // integer, stored as WORD 
   7L,                            // integer, stored as DWORD 
   0x029a,                        // hex integer 
   0o733,                         // octal integer 
}
```
