# <a name="renaming-the-compiled-resource-file"></a>Переименование скомпилированного файла ресурсов

По умолчанию при компиляции ресурсов версия-кандидат присваивает файлу скомпилированного ресурса (. res) базовое имя RC-файла и помещает его в тот же каталог, где находится RC-файл. Затем необходимо вызвать CVTRES, чтобы преобразовать RES-файл в формат двоичного ресурса (. РБЖ), который может быть понят компоновщиком. В следующем примере компилируется MyApp. RC и создается скомпилированный файл ресурсов с именем MyApp. res в том же каталоге, что и MyApp. rc:

**Версия-кандидат MyApp. RC**

Параметр **/FO** присваивает результирующему RES-файлу имя, которое отличается от имени соответствующего файла. rc. Например, чтобы присвоить результирующему RES file имя NewFile. Res, используйте следующую команду:

**Версия-кандидат: NewFile. RES MYAPP. RC**

Параметр **/FO** также может поместить RES-файл в другой каталог. Например, следующая команда помещает скомпилированный файл ресурсов MyApp. res в каталог C: \\ исходный \\ ресурс:

**RC-FO c: \\ исходный \\ ресурс \\ MyApp. RES MYAPP. RC**
