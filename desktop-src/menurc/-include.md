# <a name="-include"></a>относится

Директива **\# include** заставляет компилятор ресурсов обработать файл, указанный в параметре *filename* . Этот файл должен быть файлом заголовка, который определяет константы, используемые в файле определения ресурса. Файл может использовать однобайтовые, двухбайтовые или символы Юникода.

``` syntax
#include filename
```

<dl> <dt>

<span id="filename"></span><span id="FILENAME"></span>*файлов*
</dt> <dd>

Имя включаемого файла. Если файл находится в текущем каталоге, строка должна быть заключена в двойные кавычки. Если файл находится в каталоге, указанном переменной среды INCLUDE, строка должна быть заключена в символы «меньше» и «больше» (<>). Необходимо указать полный путь, заключенный в двойные кавычки ("), если файл не находится в текущем каталоге или в каталоге, указанном переменной среды INCLUDE.

</dd> </dl>

## <a name="remarks"></a>Remarks

Используйте следующую инструкцию в файле заголовка, чтобы окружить инструкции, которые могут быть скомпилированы компилятором C, но не версией-КАНДИДАТом:

``` syntax
#ifndef RC_INVOKED
```

Таким образом можно использовать те же включаемые файлы в файлах c и RC.

## <a name="example"></a>Пример

в этом примере файлы заголовков обрабатываются Windows. h и мидефс. h при компиляции файла определения ресурса:

``` syntax
#include <windows.h>
#include "headers\mydefs.h"
```
