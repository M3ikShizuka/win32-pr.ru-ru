# <a name="iwiadevmgr2registereventcallbackclsid-method"></a>Метод IWiaDevMgr2:: Регистеревенткаллбаккклсид

Метод **IWiaDevMgr2:: регистеревенткаллбаккклсид** регистрирует приложение для получения событий, даже если приложение не работает.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT RegisterEventCallbackCLSID(
  [in]               LONG lFlags,
  [in]               BSTR bstrDeviceID,
  [in]         const GUID *pEventGUID,
  [in, unique] const GUID *pClsID,
  [in]               BSTR bstrName,
  [in]               BSTR bstrDescription,
  [in]               BSTR bstrIcon
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*лфлагс* \[ окне\]

Тип: **Long**

|----------------------------------|---------------------------------------------------|
| \_ \_ обратный вызов события регистрации WIA \_   | Зарегистрируйтесь для события.                           |
| \_обратный вызов для отмены регистрации \_ события WIA \_ | Удалите регистрацию для события.            |
| \_ \_ обработчик по умолчанию для WIA Set \_       | Задайте приложение в качестве обработчика событий по умолчанию. |



 

</dd> <dt>

## <a name="return-value"></a>Возвращаемое значение

Тип: **HRESULT**

Если этот метод завершается успешно, возвращается значение **S \_ ОК**. В противном случае возвращается код ошибки **HRESULT** .

## <a name="remarks"></a>Remarks

Приложения WIA 2,0 используют этот метод для регистрации для получения событий аппаратного устройства. После вызова **IWiaDevMgr2:: регистеревенткаллбаккклсид** приложение регистрируется для получения событий устройства WIA 2,0, даже если оно не работает.

При возникновении события система WIA 2,0 определяет, какое приложение зарегистрировано для получения события. Он использует функцию [CoCreateInstance](/windows/win32/api/combaseapi/nf-combaseapi-cocreateinstance) и CLSID, указанные в параметре *пклсид* , для создания экземпляра приложения, а затем вызывает метод [**имажеевенткаллбакк**](/windows/desktop/api/wia_xp/nf-wia_xp-iwiaeventcallback-imageeventcallback) для передачи сведений о событии в приложение.

Приложение может вызвать метод [**енумрегистеревентинфо**](-wia-iwiaitem2-enumregistereventinfo.md) для перечисления сведений о регистрации событий.

Если приложение не является зарегистрированным компонентом модели COM и несовместимо с архитектурой WIA 2,0, используйте метод [**IWiaDevMgr2:: регистеревенткаллбаккпрограм**](-wia-iwiadevmgr2-registereventcallbackprogram.md) для регистрации приложения для событий устройства.

> [!Note]  
> В многопоточном приложении нет никакой гарантии, что обратный вызов уведомления о событии возвращается в том же потоке, который зарегистрировал обратный вызов.
