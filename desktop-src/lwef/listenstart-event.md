# <a name="listenstart-event"></a>Событие Листенстарт

\[Microsoft Agent является устаревшим по отношению к Windows 7 и может быть недоступен в последующих версиях Windows.\]

<dl> <dt>

<span id="Description"></span><span id="description"></span><span id="DESCRIPTION"></span>**Nописание**
</dt> <dd>

Происходит при запуске режима прослушивания (распознавание речи).

</dd> <dt>

<span id="Syntax"></span><span id="syntax"></span><span id="SYNTAX"></span>**Syntax**
</dt> <dd>

 *Подагент. * * * листенстарт (ByVal* *  *чарактерид * * *)**



| Часть          | Описание                                            |
|---------------|--------------------------------------------------------|
| *чарактерид* | Возвращает идентификатор прослушивающего символа в виде строки. |



 

</dd> </dl>

### <a name="remarks"></a>Remarks

Это событие отправляется всем клиентам, когда запускается режим прослушивания, так как пользователь нажал на ключ прослушивания или клиент ввода-клиента, который называется методом [**Listen**](listen-method.md) со значением **true**. Это событие можно использовать, чтобы избежать диктовки символа при включенном режиме прослушивания.

Если включить режим прослушивания с помощью метода [**Listen**](listen-method.md) , а затем пользователь нажмет клавишу прослушивания, режим прослушивания сбрасывается и продолжит работу до завершения времени ожидания ключа прослушивания, отпускания клавишу прослушивания, или пользователь завершает говорить, в зависимости от того, что происходит позже. В этом случае, когда режим прослушивания уже включен, вы не получаете дополнительное событие **листенстарт** , когда пользователь нажимает клавишу прослушивания.

Событие возвращает символ для клиентов, которые в настоящий момент загрузили этот символ. Все остальные клиенты получают символ null (пустая строка).
