# <a name="microsoft-agent-speech-output-tags"></a>Теги речевого вывода агента Microsoft Agent

\[Microsoft Agent является устаревшим в Windows 7 и может быть недоступен в последующих версиях Windows.\]

Службы Microsoft Agent поддерживают изменение речевого вывода с помощью специальных тегов, вставленных в текстовую строку речи. Эти теги помогают изменить характеристики выходного выражения символа.

Теги вывода речи используют следующие правила синтаксиса:

-   Все теги начинаются и заканчиваются символом обратной косой черты ( \\ ).
-   Один символ обратной косой черты не включен в теге. Чтобы включить символ обратной косой черты в текстовый параметр тега, используйте двойную обратную косую черту ( \\ \\ ).
-   В тегах не учитывается регистр. Например, \\ значение «смола» совпадает с \\ « \\ смолой» \\ .
-   Теги зависят от пробелов. Например, \\ RST \\ не совпадает с \\ RST \\ .

Если не указано иное или не изменено другим тегом, речевое вывод оставляет набор характеристик, заданный тегом внутри текста, указанного в одном методе [**говорите**](speak-method.md) . После завершения метода **произношения** речевые данные автоматически сбрасываются через определяемые пользователем параметры.

Некоторые теги содержат строки в кавычках. Для некоторых языков программирования, таких как Visual Basic Scripting Edition (VBScript) и Visual Basic, это означает, что может потребоваться использование двух кавычек для обозначения параметра тега или сцепления символа двойной кавычки в составе строки. Последнее показано в этом Visual Basic примере:


```
Agent1.Characters("Genie").Speak "This is \map=" + chr(34) + "Spoken text" _
+ chr(34) + "=" + chr(34) + "Balloon text" + chr(34) + "\."
```



В языках C, C++ и Java™ программировании перед символами обратной косой черты и двойными кавычками. Пример:


```
BSTR bszSpeak = SysAllocString(L"This is \\map=\"Spoken text\"=\"Balloon text\"\\");

pCharacter->Speak(bszSpeak, ......);
```



Для иностранных языков, поддерживающих символы в двухбайтовой кодировке (DBCS), для указания строковых параметров можно использовать двухбайтовые символы. Однако Используйте однобайтовые символы для всех других параметров и символов, которые используются для определения тега, включая сам тег.

Поддерживаются следующие теги:

-   [**Chr**](chr-tag.md)
-   [**CTX**](ctx-tag.md)
-   [**Точек**](emp-tag.md)
-   [**LST**](lst-tag.md)
-   [**Таблица**](map-tag.md)
-   [**мрк**](mrk-tag.md)
-   [**пау**](pau-tag.md)
-   [**Медлен**](pit-tag.md)
-   [**RST**](rst-tag.md)
-   [**Политики**](spd-tag.md)
-   [**Т**](vol-tag.md)

Теги в основном предназначены для настройки выходных данных преобразования текста в речь (TTS). Только теги [**МРК**](mrk-tag.md) и [**Map**](map-tag.md) можно использовать с речевым выходом на основе файлов.

> [!Note]  
> Microsoft Agent не поддерживает все теги, описанные в пакете Microsoft Speech SDK. Параметры также могут различаться в зависимости от выбранного модуля TTS. Вы можете задать конкретный модуль TTS с помощью [**ттсмодеид**](ttsmodeid-property.md).
