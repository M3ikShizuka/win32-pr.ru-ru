# <a name="speech-output"></a>Речевое вывод

\[Microsoft Agent является устаревшим по отношению к Windows 7 и может быть недоступен в последующих версиях Windows.\]

Как и речевые входные данные, речевая передача является знакомой и естественной формой взаимодействия, поэтому она также является соответствующим дополнением в интерфейсе на основе символов. Однако в речевых выводах также есть обязательства. В некоторых средах речевое вывод может не быть предпочтительным или звуковым. Кроме того, речь невидима и не имеет постоянного выхода, что сильно полагается на краткосрочную память. Эти факторы ограничивают емкость и скорость обработки больших объемов информации. Аналогичным образом, речевое вывод может также нарушать ввод данных пользователем, особенно если речь представляет собой входной метод. Модули распознавания речи обычно имеют небольшую поддержку, позволяющую пользователю прерывать работу, когда речь или другая звуковая система имеет выходной канал.

В результате не следует использовать речь в качестве монопольной формы выходных данных. однако, поскольку Microsoft Agent представляет символы как часть интерфейса Windows, он предоставляет несколько преимуществ по сравнению с окружениями, поддерживающими только речь. Символы можно сочетать с другими формами ввода и вывода, чтобы сделать доступными параметры и действия, что обеспечивает более эффективный интерфейс, чем тот, который является исключительно визуальным или основанным на речи.

Кроме того, чтобы сделать вывод речи более видимым, Microsoft Agent включает возможность создания символа с помощью всплывающей подсказки. Другие параметры позволяют определить, как текст отображается в выноске и когда всплывающее сообщение удаляется. Вы также можете определить, какой шрифт использовать. Хотя можно задать текстовые атрибуты для символов, имейте в виду, что пользователь может переопределить эти параметры.
