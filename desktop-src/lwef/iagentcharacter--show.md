# <a name="iagentcharactershow"></a>Иажентчарактер:: показывать

\[Microsoft Agent является устаревшим в Windows 7 и может быть недоступен в последующих версиях Windows.\]

``` syntax
HRESULT Show(
   long bFast,      // play Showing state animation flag
   long * pdwReqID  // address of request ID
);
```

Отображает символ.

-   Возвращает значение S \_ , чтобы указать, что операция выполнена успешно. Когда функция возвращает значение, *пдврекид* содержит идентификатор запроса.

<dl> <dt>

<span id="bFast"></span><span id="bfast"></span><span id="BFAST"></span>*бфаст*
</dt> <dd>

Отображение флага анимации состояния. Если этот параметр имеет **значение true**, анимация **отображения** состояния воспроизводится после того, как символ виден. Если задано **значение false**, анимация не воспроизводится.

</dd> <dt>

<span id="pdwReqID"></span><span id="pdwreqid"></span><span id="PDWREQID"></span>*пдврекид*
</dt> <dd>

Адрес переменной, которая получает идентификатор запроса на [**Отображение**](/windows/desktop/lwef/iagentcharacter--show) .

</dd> </dl>

Избегайте установки значения **true** для параметра *бфаст* без предварительного воспроизведения анимации; в противном случае рамка символов может отображаться, но не иметь изображения для отображения. В частности, обратите внимание, что при вызове функции [**MoveTo**](iagentcharacter--moveto.md) , если символ не отображается, анимация не воспроизводится. Поэтому при вызове метода " **Показать** " с параметром *бфаст* , для которого задано значение **true**, изображение отображаться не будет. Аналогично, если вызвать [**Hide**](/windows/desktop/lwef/iagentcharacter--hide), а затем **Показать** , что для *бфаст* задано значение **true**, изображение не будет отображено.

При использовании протокола HTTP для доступа к данным символов и анимации используйте метод [**Prepare**](/windows/desktop/lwef/iagentcharacter--prepare) , чтобы обеспечить доступность анимации **отображаемого** состояния перед вызовом этого метода.
