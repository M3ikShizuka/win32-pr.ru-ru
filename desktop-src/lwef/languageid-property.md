## <a name="remarks"></a>Комментарии

Если для символа не задано значение **LanguageID** , идентификатор языка будет соответствовать ТЕКУЩЕМу идентификатору языка системы, если установлена соответствующая библиотека DLL языка агента. в противном случае язык символа будет АНГЛИЙСКИМ (US).

Это свойство также определяет язык для текста всплывающей подсказки по слову, команды во всплывающем меню символа и модуль распознавания речи. Он также определяет язык по умолчанию для вывода TTS.

Если вы попытаетесь задать **LanguageID** для символа, а библиотека DLL языка агента для этого языка не установлена или шрифт не отображается для идентификатора языка, то агент выдает ошибку, и **LanguageID** остается в последнем задании.

Установка этого свойства не вызывает ошибку, если для языка нет соответствующих обработчиков речи. Чтобы определить, доступен ли совместимый модуль распознавания речи для **LanguageID**, проверьте [**срмодеид**](srmodeid-property.md) или [**ттсмодеид**](ttsmodeid-property.md). Если параметр **LanguageID** не задан, для него будет задано значение по умолчанию для параметра идентификатор языка пользователя.

Это свойство применяется только к символу, используемому вашим клиентским приложением. Этот параметр не влияет на другие клиенты символов или других символов клиентского приложения.

> [!Note]  
> Если задать для **LanguageID** язык, поддерживающий двунаправленный текст (например, арабский или иврит), но в системе, где выполняется приложение, не установлена поддержка двунаправленного письма, текст в выноске слова будет отображаться как логический, а не как порядок отображения.
