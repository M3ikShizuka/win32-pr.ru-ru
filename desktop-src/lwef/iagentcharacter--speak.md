# <a name="iagentcharacterspeak"></a>Иажентчарактер:: говорите

\[Microsoft Agent является устаревшим в Windows 7 и может быть недоступен в последующих версиях Windows.\]

``` syntax
HRESULT Speak(
   BSTR bszText,    // text to speak
   BSTR bszURL,     // URL of a file to speak
   long * pdwReqID  // address of a request ID
);
```

Говорит текст или звуковой файл.

-   Возвращает значение S \_ , чтобы указать, что операция выполнена успешно.

<dl> <dt>

<span id="bszText"></span><span id="bsztext"></span><span id="BSZTEXT"></span>*бсзтекст*
</dt> <dd>

Текст, который должен говориться с символом.

</dd> <dt>

<span id="bszURL"></span><span id="bszurl"></span><span id="BSZURL"></span>*бсзурл*
</dt> <dd>

URL-адрес (или спецификация файла) звукового файла, используемого для речевого вывода. Это может быть стандартный звуковой файл (. WAV) или лингвистическо Улучшенный звуковой файл (. ЛВВ).

</dd> <dt>

<span id="pdwReqID"></span><span id="pdwreqid"></span><span id="PDWREQID"></span>*пдврекид*
</dt> <dd>

Адрес переменной, которая получает идентификатор запроса [**говорите**](/windows/desktop/lwef/iagentcharacter--speak) .

</dd> </dl>

Для использования этого метода с символом, настроенным для диктовки с помощью механизма преобразования текста в речь (TTS); просто укажите параметр *бсзтекст* . В параметре Бсзтекст можно включить символы вертикальной черты ( \| ), чтобы назначить альтернативные строки, чтобы каждый раз, когда сервер обрабатывает метод, случайно выбрать другую строку.  Поддержка вывода TTS определяется при компиляции символа с помощью редактора символов Microsoft Agent.

Если для символа нужно использовать выходные данные звукового файла, укажите расположение файла в параметре *бсзурл* . При использовании протокола HTTP для загрузки звукового файла используйте метод [**Prepare**](/windows/desktop/lwef/iagentcharacter--prepare) , чтобы обеспечить доступность файла перед использованием этого метода. Можно использовать параметр *бсзтекст* для указания слов, отображаемых в тексте всплывающего слова символа. При указании лингвистическо расширенного звукового файла (. ЛВВ) для параметра *бсзурл* и не указывайте текст, параметр *бсзтекст* использует текст, хранящийся в файле.

Метод [**говорите**](/windows/desktop/lwef/iagentcharacter--speak) использует последнюю воспроизводимую анимацию, чтобы определить, какую анимацию речи следует воспроизвести. Например, если вы предшествует команду **говорите** с параметром [**иажентчарактер::P**](iagentcharacter--play.md) «**жестуреригхт**», то на сервере будет воспроизводиться **жестуреригхт** , а затем анимация **жестуреригхт** . Если последняя воспроизводимая анимация не имеет анимированной анимации, то Microsoft Agent воспроизводит анимацию, назначенную этому состоянию **речи** .

Если вы выберете [**разговор**](/windows/desktop/lwef/iagentcharacter--speak) и канал звука занят, звуковые выходные данные не будут слышны, а текст отобразится в выноске. Для отображаемого текста свойство [Enabled](enabled-property.md) всплывающей программы должно иметь **значение true** .

Автоматическое разбиение слов Microsoft Agent в подсказке, разбивает слова, используя пробелы (например, пробел и знак табуляции). Однако он может нарушить работу слова, чтобы вместить всплывающее сообщение. В таких языках, как японский, китайский и тайский, если пробелы не используются для разбиения слов на слова, вставьте символ нулевой ширины (0x200B) в Юникоде между символами, чтобы определить логические разрывы слов.

> [!Note]  
> Задайте идентификатор языка символа (с помощью [**иажентчарактерекс:: сетлангуажеид**](iagentcharacterex--setlanguageid.md) перед использованием метода [**говорите**](/windows/desktop/lwef/iagentcharacter--speak) , чтобы обеспечить отображение текста внутри всплывающей подсказки.
