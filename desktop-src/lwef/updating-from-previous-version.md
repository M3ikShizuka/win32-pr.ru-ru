# <a name="updating-from-previous-version"></a>Обновление с предыдущей версии

\[Microsoft Agent является устаревшим по отношению к Windows 7 и может быть недоступен в последующих версиях Windows.\]

Приложения, созданные и скомпилированные с помощью предыдущей версии 1,5 агента Microsoft Agent, должны работать без изменений в новой версии 2,0. Свойству [**Connected**](connected-property.md) больше не может быть присвоено значение **false**; Некоторые свойства объекта Спичинпут, которые были заменены, все еще существуют, но больше не включают значения, а сервер больше не запускает события [**перезапуска**](https://www.bing.com/search?q=**Restart**) и [**завершения работы**](https://www.bing.com/search?q=**Shutdown**) .

Однако при обновлении приложений для использования элемента управления агента 2,0 может потребоваться изменить код. если вы установили версию 2,0 Microsoft Agent и загрузили проект Visual Basic использовать более раннюю версию элемента управления агента, Visual Basic включает параметр, который автоматически отображает сообщение о том, что обнаружена новая версия элемента управления. Чтобы обеспечить правильную работу приложения, всегда выбирайте более позднюю версию.

для других языков программирования (например, Microsoft Office 97 VBA) для обновления элемента управления может потребоваться сначала удалить элемент управления агента 1,5 и сохранить код. Затем закройте среду программирования, перезапустите среду программирования, перезагрузите код и вставьте новый элемент управления. Избегайте редактирования приложения с поддержкой агента 2,0 в том же экземпляре среды программирования, в которой вы редактируете приложение с поддержкой агента 1,5 в. Некоторые среды программирования могут не справиться с различиями между двумя версиями элементов управления.

После установки выпуска Agent 2,0 вы сможете удалить выпуск агента 1,5 в системе. Однако если агент 1,5 установлен в выпуске 2,0, может потребоваться переустановить 2,0.
