# <a name="requirements-for-speech-recognition-engines"></a>Требования для модулей распознавания речи

\[Microsoft Agent является устаревшим по отношению к Windows 7 и может быть недоступен в последующих версиях Windows.\]

Подсистема распознавания речи также должна быть полностью совместимой командой и контролем (C&C) в соответствии с SAPI 4,0. Он должен поддерживать несколько грамматик в двоичном формате, описанных в спецификации, и позволяет активировать или деактивировать эти грамматики в режиме реального времени.

Обратите внимание, что SAPI 4,0 требует, чтобы модули распознавания речи поддерживали широкие символы, интерфейсы Юникода. Однако при поддержке этих интерфейсов подсистема не должна зависеть от преобразования данных в Юникоде в ANSI, так как ядро может работать неправильно в некоторых системах. например, японский модуль, преобразующий юникод в ANSI, может не работать в системе Microsoft Windows 95 на английском языке.

Кроме того, чтобы считаться совместимым с Microsoft Agent, обработчик должен вернуть объекты Results после успешного распознавания фразы (через Исрграмнотифисинкв::P Храсефиниш). Эти объекты результатов должны поддерживать Исрресбасик в соответствии с требованиями спецификации. Кроме того, они должны поддерживать Исрресскоре. Несмотря на то, что Microsoft Agent работает с ядром, поддерживающей только Исрресбасик, или даже с ядром, которая не возвращает никаких объектов результатов, производительность обычно значительно снижается с учетом таких ядер. Многие приложения используют значения достоверности, предоставляемые ядром, для управления их реакцией на различные команды.
