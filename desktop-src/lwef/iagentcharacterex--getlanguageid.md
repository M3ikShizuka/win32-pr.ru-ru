# <a name="iagentcharacterexgetlanguageid"></a>Иажентчарактерекс:: Жетлангуажеид

\[Microsoft Agent является устаревшим по отношению к Windows 7 и может быть недоступен в последующих версиях Windows.\]

``` syntax
HRESULT GetLanguageID(
   long * plangID  // address of language ID setting
);
```

Возвращает идентификатор языка, заданный для символа.

-   Возвращает значение S \_ , чтобы указать, что операция выполнена успешно.

<dl> <dt>

<span id="plangID"></span><span id="plangid"></span><span id="PLANGID"></span>*плангид*
</dt> <dd>

Адрес переменной, получающей параметр идентификатора языка для символа.

</dd> </dl>

Длинное целое число, указывающее идентификатор языка для символа. идентификатор языка (LANGID) для символа — это 16-разрядное значение, определенное Windows, состоящее из основного идентификатора языка и дополнительного идентификатора языка. Ниже приведены примеры значений для некоторых языков. Чтобы определить значения других языков, см. документацию по пакету SDK для платформы.

| Баскский                | 0x042d | Японский              | 0x0411 |
| Китайский (упрощенное письмо)  | 0x0804 | Корейский                | 0x0412 |

Если не задать этот идентификатор языка для символа, ИДЕНТИФИКАТОРом языка символа будет текущий идентификатор языка системы.

Этот параметр также определяет язык для вывода данных в формате TTS, текста всплывающей подсказки в Word, команд во всплывающем меню символа и модуля распознавания речи. Чтобы определить, доступен ли совместимый модуль распознавания речи для языка символа, используйте [**иажентчарактерекс:: жетсрмодеид**](iagentcharacterex--getsrmodeid.md) или [**Иажентчарактерекс:: жетттсмодеид**](iagentcharacterex--getttsmodeid.md).

Это свойство применяется только к символу, используемому вашим клиентским приложением. Этот параметр не влияет на другие клиенты символов или других символов клиентского приложения.

> [!Note]  
> Если для идентификатора языка задан язык, поддерживающий двунаправленный текст (например, арабский или иврит), но в системе, в которой выполняется приложение, не установлена поддержка двунаправленного письма, текст будет отображаться в выноске слова логический, а не в порядке отображения.
