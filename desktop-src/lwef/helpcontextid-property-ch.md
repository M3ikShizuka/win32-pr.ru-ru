# <a name="helpcontextid-property-characters-object"></a>Свойство Хелпконтекстид (объект characters)

\[Microsoft Agent является устаревшим по отношению к Windows 7 и может быть недоступен в последующих версиях Windows.\]

<dl> <dt>

<span id="Description"></span><span id="description"></span><span id="DESCRIPTION"></span>**Nописание**
</dt> <dd>

Возвращает или задает связанный контекстный номер для символа. Используется для предоставления контекстной справки для символа.

</dd> <dt>

<span id="Syntax"></span><span id="syntax"></span><span id="SYNTAX"></span>**Syntax**
</dt> <dd>

*Символы Agent. ***("**_чарактерид_*_")._ *  \[  =  *Номер* хелпконтекстид\]



| Часть     | Описание                                   |
|----------|-----------------------------------------------|
| *Число* | Целое число, указывающее допустимый контекстный номер. |



 

</dd> </dl>

## <a name="remarks"></a>Remarks

Для поддержки контекстной справки для символа назначьте номер контекста символу, который используется для связанного раздела справки при компиляции файла справки. Это свойство применяется только к клиенту символа. параметр не влияет на других клиентов символа или других символов клиента.

если вы создали файл справки Windows для приложения и настроили свойство [**HelpFile**](helpfile-property.md) символа, агент автоматически вызывает справку, когда [**хелпмодеон**](helpmodeon-property.md) имеет значение **True** и пользователь щелкает символ. Если в [**хелпконтекстид**](helpcontextid-property.md)имеется контекстный номер, агент вызывает справку и ищет раздел, определяемый текущим контекстным номером. Текущий контекстный номер — это значение **хелпконтекстид** для символа.

> [!Note]  
> для создания файла справки требуется компилятор справки Microsoft Windows.
