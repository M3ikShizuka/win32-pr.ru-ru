# <a name="xtyp_unregister-transaction"></a>КСТИП \_ отменить регистрацию транзакции

Функция обратного вызова платформа динамических данных Exchange (DDE) получает **КСТИП \_ отмену регистрации** [*транзакции, когда*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)платформа динамических данных серверное приложение библиотеки управления Exchange (ддемл) использует функцию [**дденамесервице**](/windows/desktop/api/Ddeml/nf-ddeml-ddenameservice) для отмены регистрации имени службы или в случае, если не ддемл приложение, поддерживающее системный раздел, завершается.


```C++
#define     XCLASS_NOTIFICATION      0x8000
#define     XTYPF_NOBLOCK            0x0002
#define     XTYP_UNREGISTER         (0x00D0 | XCLASS_NOTIFICATION | XTYPF_NOBLOCK)
```

## <a name="remarks"></a>Комментарии

Эта транзакция фильтруется, если приложение указало флаг **КБФ \_ пропустить \_ регистрации** в функции [**ддеинитиализе**](/windows/desktop/api/Ddeml/nf-ddeml-ddeinitializea) .

Приложение не может блокировать этот тип транзакции; \_код возврата блока CBR игнорируется.

Приложение должно использовать параметр *hsz1* для удаления имени службы из списка серверов, доступных пользователю. Приложение должно использовать параметр *hsz2* для указания того, какой экземпляр приложения завершил работу.
