# <a name="xtyp_error-transaction"></a>КСТИП \_ Ошибка транзакции

Функция обратного вызова платформа динамических данных Exchange (DDE), [*ддекаллбакк*](/windows/win32/api/ddeml/nc-ddeml-pfncallback), Получает транзакцию **\_ ошибки кстип** при возникновении критической ошибки.


```C++
#define     XCLASS_NOTIFICATION      0x8000
#define     XTYPF_NOBLOCK            0x0002
#define     XTYP_ERROR              (0x0000 | XCLASS_NOTIFICATION | XTYPF_NOBLOCK )
```

<dl> <dt>

*утипе* 
</dt> <dd>

Тип транзакции.

</dd> <dt>

</dt> <dd>

</dt> <dd>

Обработчик диалога, связанный с ошибкой. Этот параметр имеет **значение NULL** , если ошибка не связана с диалогом.

</dd> <dt>

*hsz1* 
</dt> <dd>

Не используется.

</dd> <dt>

</dd> <dt>

</dt> <dd>

</dd> <dt>

*dwData1*

Код ошибки в слове низкого порядка. В настоящее время поддерживается только следующий код ошибки.



| Значение                                                                                                                                                                      | Значение                                                                                      |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------|
| <span id="DMLERR_LOW_MEMORY"></span><span id="dmlerr_low_memory"></span><dl> <dt>**ДМЛЕРР \_ нехватка \_ памяти**</dt> </dl> | Недостаточно памяти; может быть потеряно предложение, вставка или выполнение данных, иначе может произойти сбой системы.<br/> |

## <a name="remarks"></a>Комментарии

Приложение не может блокировать этот тип транзакции; код **возврата \_ блока CBR** игнорируется. Библиотека управления платформа динамических данных Exchange (ДДЕМЛ) пытается освободить память путем удаления некритических ресурсов. Приложение с заблокированными диалогами должно разблокировать их.
