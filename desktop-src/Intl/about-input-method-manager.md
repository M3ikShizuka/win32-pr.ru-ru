# <a name="about-input-method-manager"></a>О диспетчере методов ввода

Использование функций IMM в приложении, поддерживающем IME, позволяет избавляет пользователей от необходимости помнить все возможные символьные значения. Вместо этого он позволяет редактору IME отслеживать нажатия клавиш пользователя, предвидит символы, которые может потребоваться пользователю, и предоставляет список потенциальных символов, из которых можно выбрать.

> [!Note]  
> IMM выполняет аналогичные операции с [платформой текстовых служб](../tsf/text-services-framework.md), используемыми приложениями, взаимодействующими с текстовыми службами.

 

По умолчанию IMM предоставляет окно IME, в котором пользователь вводит нажатия клавиш и представления и выбирает кандидатов. Приложения могут использовать функции и сообщения IMM для создания собственных окон IME и управления ими, предоставляя пользовательский интерфейс при использовании возможностей преобразования IME.

Модуль IMM включен только в локализованных операционных системах Windows для восточноазиатских языков (китайского, японского, корейского). В этих системах приложение вызывает [жетсистемметрикс](/windows/win32/api/winuser/nf-winuser-getsystemmetrics) с SM дбксенаблед, \_ чтобы определить, включен ли IMM.

**Windows 2000:** Полнофункциональная поддержка IMM предоставляется во всех локализованных версиях языка. Однако IMM включается, только если установлен Азиатский языковой пакет. Приложение, поддерживающее IME, может вызвать [жетсистемметрикс](/windows/win32/api/winuser/nf-winuser-getsystemmetrics) с SM \_ имменаблед, чтобы определить, включен ли IMM.
