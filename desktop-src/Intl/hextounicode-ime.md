# <a name="hextounicode-ime"></a>Хекстауникоде IME

Расширенное редактирование 3,0 поддерживает редактор IME для Хекстауникоде, позволяющий пользователю выполнять преобразование между шестнадцатеричными и символами Юникода с помощью горячих клавиш одним из двух способов.

При использовании первого метода преобразования пользователь вводит код символа в шестнадцатеричном виде, а затем вводит ALT + X. Редактор IME заменяет шестнадцатеричные цифры перед точкой вставки символом Юникода. Если текущий шрифт не поддерживает код символа, выбирается подходящий шрифт, который поддерживает его. Для преобразования из Юникода в шестнадцатеричный формат пользователь вводит SHIFT + ALT + X. Эта запись заменяет символ Юникода, предшествующий точке вставки, шестнадцатеричными цифрами. В частности, этот метод позволяет пользователю определить символ, обозначенный индикатором "отсутствующий глиф". Если шестнадцатеричный код символа следует сразу за определенными шестнадцатеричными символами (несимвольными), пользователь выбирает конкретные цифры для преобразования перед вводом ALT + X. Проблема с этим первым методом заключается в том, что ALT + X иногда используется как сочетание клавиш для команды Exit (то есть выход). например, в Microsoft Office эта команда отображается в меню " **файл** " только как вариант.

Второй способ преобразования между шестнадцатеричными и символами Юникода включает цифровую панель. С помощью этого метода пользователь вводит ALT + Нумпад Numbers (со значениями больше 255), чтобы вводить символы Юникода с помощью десятичных значений. Этот метод не так удобен, как первый, поскольку пользователь не может видеть введенные шестнадцатеричные цифры. Кроме того, пользователь не может исправить шестнадцатеричные цифры, за исключением повторного ввода всех цифр.
