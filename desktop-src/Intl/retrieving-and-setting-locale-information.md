# <a name="retrieving-and-setting-locale-information"></a>Извлечение и установка сведений о языковых стандартах

Приложение должно иметь возможность извлекать и задавать конкретные сведения о доступных языковых [стандартах и языках](locales-and-languages.md). Каждый элемент сведений о языковых стандартах, например название определенного дня недели или символ, используемый в качестве десятичного разделителя, имеет соответствующую константу. Доступные константы определяются в [константах сведений о языковых стандартах](locale-information-constants.md).

Приложение всегда хранит сведения о языковых стандартах и управляет ими в виде строки, завершающейся нулем. Двоичные данные не допускаются, и любые числовые значения должны быть указаны как текст. Каждый тип данных имеет определенный формат. Кроме того, несколько типов связаны друг с другом, так что изменение одного типа также изменяет значение другого типа.

Чтобы получить сведения о языковых стандартах, приложение вызывает [**GetLocaleInfo**](/windows/desktop/api/Winnls/nf-winnls-getlocaleinfoa) или [**GetLocaleInfoEx**](/windows/desktop/api/Winnls/nf-winnls-getlocaleinfoex) с константой, соответствующей требуемой информации. Приложение может вызвать [**сетлокалеинфо**](/windows/desktop/api/Winnls/nf-winnls-setlocaleinfoa) , чтобы задать элемент сведений о языковых стандартах.

> [!Note]  
> Хотя [идентификатор локали](locale-identifiers.md) может поддерживаться, он недоступен для использования приложением, если также не установлен соответствующий языковой стандарт.

 

Так как большинство констант сведений о языковых стандартах являются взаимоисключающими, в каждый момент времени может обрабатываться только один тип информации. Исключениями из этого правила являются [языковой стандарт. \_ используйте параметр \_ CP \_ ACP](locale-use-cp-acp.md), [ \_ \_ номер возврата языкового стандарта](locale-return-constants.md)и [языковой стандарт \_ наусероверриде](locale-nouseroverride.md), который можно сочетать с другими константами с помощью двоичного файла или.

> [!Caution]  
> Использовать ЯЗЫКовой стандарт \_ наусероверриде настоятельно не рекомендуется, так как он отключает предпочтения пользователя.

 

Как и в ряде приложений, например Microsoft Active Directory, приложение может поддерживать свои строки в базе данных, поддерживающей сортировку. Дополнительные сведения см. [в разделе Обработка сортировки в приложениях](handling-sorting-in-your-applications.md).
