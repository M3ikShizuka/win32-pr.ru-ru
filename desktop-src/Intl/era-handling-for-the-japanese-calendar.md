# <a name="era-handling-for-the-japanese-calendar"></a>Работа с эрами японского календаря

У многих календарей есть эры, например AD/BC или CE/–. В японском календаре годы описываются ненгō, сочетанием номера года и названия эры. Например, 2009 — Хэйсэй 21. В прошлом названия эре в японском языке меняются часто, но теперь японские Эр меняются только в случае германской. Windows и Microsoft .NET исторически поддерживали четыре современные эры в этой политике: меижи, таишō, шōва и хэйсэй.

в Windows 7, Windows Server 2008 R2 и платформа .NET Framework 4, корпорация майкрософт распознает, что в будущем могут быть добавлены дополнительные эр. в этих версиях Windows данные эры хранятся в реестре с ключом:<dl> HKEY \_ локальный \_ компьютер \\ система \\ CurrentControlSet \\ контроль \\ NLS \\ календари \\ японская \\ эры  
</dl>

при необходимости к этому ключу можно добавить дополнительные эр с помощью обычного Центр обновления Windows процесса. Этот ключ можно просмотреть с помощью редактора реестра (Regedit.exe). пример ключа и значений, поставляемых в Windows 7:

``` syntax
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\Calendars\Japanese\Eras]
"1868 01 01"="明治_明_Meiji_M"
"1912 07 30"="大正_大_Taisho_T"
"1926 12 25"="昭和_昭_Showa_S"
"1989 01 08"="平成_平_Heisei_H"
```

Название каждого значения эры — это дата начала эры в григорианском календаре. Значение содержит название эры в японском, сокращенное имя на японском языке, имя на английском и сокращенное название на английском языке:<dl> "гггг мм дд" = "JE \_ аже \_ EE \_ аи"  
</dl>where

-   "Гггг мм дд" — это дата начала эры в году, месяц, день, где год состоит из 4 цифр, день — 2 цифры, а месяц — также 2 цифры. Каждая часть даты отделяется пробелом.
-   "JE" — это японское название эры, за которым следует символ подчеркивания.
-   "Аже" — это сокращенное название эры в японском языке, за которым следует символ подчеркивания.
-   "EE" — это английское название японской эры, за которым следует символ подчеркивания.
-   "АИ" — это сокращенное название японской эры в японском языке.

одним из соображений для разработчиков приложений является возможность добавления дополнительных эр с помощью Центр обновления Windows или других средств. В этом случае приложение может столкнуться с более чем ожидаемой четырьмя Эр для японского календаря. Тест-инженеры тестирования могут добавлять в реестр дополнительную эру; Однако он должен быть ограничен только тестовыми компьютерами, так как он влияет на поведение всего компьютера.

Ниже приведен пример такого ключа, который можно использовать для тестирования. Это изменение можно выполнить с помощью редактора реестра. (Это пример только для тестового использования и не предназначен для прогнозирования будущих добавлений.)

``` syntax
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\Calendars\Japanese\Eras]
"2020 09 01"="仮名_仮_Test Era_X"
```

обратите внимание, что это касается только компьютеров с Windows 7 и более поздних версий или платформа .NET Framework 4 и более поздних версий. Разработчикам приложений рекомендуется протестировать свои приложения с помощью таких дополнительных тестовых Эр, чтобы гарантировать, что их приложения будут продолжать работать при добавлении дополнительных эр в будущем.
