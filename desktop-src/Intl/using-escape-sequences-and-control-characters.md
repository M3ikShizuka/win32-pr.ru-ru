# <a name="using-escape-sequences-and-control-characters"></a>Использование escape-последовательностей и управляющих символов

Когда приложение преобразует строки из ASCII или [кодовой страницы Windows (ANSI)](code-pages.md) в Юникод, оно должно преобразовывать escape-последовательности, посимвольно в Юникод. При преобразовании ASCII или другого 8-разрядного текстового файла в Юникод существует вероятность, что впоследствии он будет преобразован обратно. Преобразование escape-последовательностей в Юникод на основе символьных символов вместо объединения их как одного символа Юникода позволяет выполнить обратный пересчет без необходимости распознать и проанализировать escape-последовательности. Например, ESC + A должно стать 0x001B (ESC), 0x0041 (A), а не 0x411B.

Первые 32 16-битные значения кода в Юникоде предназначены для управляющих символов 32. Эта спецификация поддерживает существующее использование управляющих символов в целях форматирования. Приложения Юникода могут обрабатывать эти управляющие символы точно так же, как они обрабатывают их эквиваленты в кодировке ASCII.
