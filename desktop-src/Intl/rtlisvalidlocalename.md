# <a name="rtlisvalidlocalename-function"></a>Функция Ртлисвалидлокаленаме

Определяет, установлен или поддерживается ли языковой стандарт, заданный по имени, в операционной системе.

> [!Note]  
> Эта функция доступна только для использования в Windows Vista. Он может быть изменен или недоступен в последующих версиях. Приложения должны использовать [**исвалидлокаленаме**](/windows/desktop/api/Winnls/nf-winnls-isvalidlocalename).

 

## <a name="syntax"></a>Синтаксис


```C++
BOOL RtlIsValidLocaleName(
  _In_ LPCWSTR LocaleName,
  _In_ ULONG   Flags
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*LocaleName* \[ окне\]
</dt> <dd>

[Имя языкового стандарта](locale-names.md) для проверки. Этот параметр может указывать имя [пользовательского языкового стандарта](custom-locales.md).

</dd> <dt>

*Флаги* \[ окне\]
</dt> <dd>

Флаги, указывающие, считаются ли Нейтральные языковые стандарты допустимыми. В настоящее время единственным определенным флагом является [языковой стандарт, \_ допускающий \_ нейтральные](locale-allow-neutral.md). Значение по умолчанию — нет.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

При успешном выполнении возвращает ненулевое значение или 0 в противном случае.

## <a name="remarks"></a>Комментарии

Эта функция похожа на [**исвалидлокаленаме**](/windows/desktop/api/Winnls/nf-winnls-isvalidlocalename). Единственное отличие заключается в том, что \_ Если \_ параметр locale разрешен нейтральный, **Ртлисвалидлокаленаме** возвращает **значение true** для имени, которое соответствует нейтральной национальной настройке (например, EN), тогда как [**исвалидлокаленаме**](/windows/desktop/api/Winnls/nf-winnls-isvalidlocalename) возвращает **значение true** только для определенного языкового стандарта (например, EN-US). Нейтральные языковые стандарты используются в рамках стратегии загрузки ресурсов в Windows Vista и более поздних версиях. Только небольшой класс строго специализированных приложений используют **ртлисвалидлокаленаме** и Set locale \_ разрешают \_ нейтральные, поскольку Нейтральные языковые стандарты имеют очень ограниченный объем использования. Ни одна из функций, описанных при [вызове функций "locale Name"](calling-the--locale-name--functions.md) , не принимает в качестве входных данных Нейтральные языковые стандарты.
