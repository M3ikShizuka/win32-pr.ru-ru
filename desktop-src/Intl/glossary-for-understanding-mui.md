# <a name="glossary-for-understanding-mui"></a>Глоссарий для понимания многоязыкового интерфейса пользователя

-   [Глобализация](#globalization)
-   [Пригодность к локализации](#localizability)
-   [Локализация](#localization)
-   [Интернационализация](#internationalization)

## <a name="globalization"></a>Глобализация

Термин " *глобализация* " указывает на необходимость использования платформы в любом случае. на практике эта концепция реализована в Windows, поскольку для Windows операционной системы, начиная с выпуска Windows Vista, используется один и тот же двоичный файл, поддерживающий развертывание одного образа в многоязычном контексте и предоставляя api NLS для извлечения данных, зависящих от языкового стандарта.

## <a name="localizability"></a>Пригодность к локализации

Термин " *Локализация* " означает, что упрощается локализация продукта, что, в свою очередь, позволяет снизить затраты на локализацию. Это можно сделать с помощью эффективного использования API-интерфейсов ресурсов, контроля экстента и количества ресурсов, добавляемых в систему, и улучшения средств, используемых для работы с этими ресурсами.

## <a name="localization"></a>Локализация

*Локализация* — это процесс адаптации программы для определенного локального рынка, который включает в себя преобразование пользовательского интерфейса, изменение размеров диалоговых окон, настройку функций (при необходимости) и проверку результатов, чтобы гарантировать, что программа все еще работает. В наиболее часто используемом контексте Локализация нацелена на преобразование пользовательского интерфейса.

## <a name="internationalization"></a>Интернационализация

В качестве синонима для глобализации в некоторых местах или в качестве глобализации и локализации в других *местах локализация* лучше всего подходит для описания всех действий, улучшающих продукт во всем мире.
