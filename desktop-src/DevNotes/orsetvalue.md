# <a name="orsetvalue-function"></a>Функция Орсетвалуе

Задает данные для значения указанного раздела реестра в автономном кусте реестра.

## <a name="syntax"></a>Синтаксис


```C++
DWORD ORSetValue(
  _In_     ORHKEY Handle,
  _In_opt_ PCWSTR lpValueName,
  _In_     DWORD  dwType,
  _In_opt_ const BYTE *lpData,
  _In_     DWORD  cbData
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Обработчик* \[ окне\]
</dt> <dd>

Маркер открытого раздела реестра в автономном кусте реестра.

</dd> <dt>

*лпвалуенаме* \[ в необязательное\]
</dt> <dd>

Имя устанавливаемого значения. Если значение с таким именем еще не существует в ключе, функция добавляет его в ключ.

Если *лпвалуенаме* имеет **значение NULL** или является пустой строкой, функция задает тип и данные для безымянного или значения по умолчанию ключа.

Дополнительные сведения см. в разделе [ограничения размера элементов реестра](../sysinfo/registry-element-size-limits.md).

Разделы реестра не имеют значений по умолчанию, но могут иметь одно безымянное значение, которое может быть любого типа.

</dd> <dt>

*двтипе* \[ окне\]
</dt> <dd>

Тип данных, на который указывает параметр *лпдата* . Список возможных типов см. в разделе [типы значений реестра](../sysinfo/registry-value-types.md).

</dd> <dt>

*лпдата* \[ в необязательное\]
</dt> <dd>

Сохраняемые данные.

Для типов, основанных на строках, таких как REG \_ SZ, строка должна завершаться нулем. Для \_ \_ типа данных reg Multi SZ строка должна завершаться двумя символами NULL.

</dd> <dt>

*cbData* \[ окне\]
</dt> <dd>

Размер информации, на которую указывает параметр *лпдата* в байтах. Если данные имеют тип REG \_ SZ, REG \_ expand \_ SZ или REG \_ Multi \_ SZ, *cbData* должен включать размер завершающего нуль-символа или символов.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System.

## <a name="remarks"></a>Remarks

Размеры значений ограничены доступной памятью. Длинные значения (более 2048 байт) должны храниться в виде файлов с именами файлов, хранящимися в реестре. Это помогает реестру работать эффективно. Элементы приложения, такие как значки, точечные рисунки и исполняемые файлы, должны храниться в виде файлов и не помещаться в реестр.
