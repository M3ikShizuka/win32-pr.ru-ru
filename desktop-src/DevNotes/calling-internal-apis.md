# <a name="calling-internal-apis"></a>Вызов внутренних API

файл заголовка винтернл. h предоставляет прототипы внутренних интерфейсов api Windows. Нет связанной библиотеки импорта, поэтому разработчики должны использовать динамическую компоновку во время выполнения для вызова функций, описанных в этом файле заголовка.

функции и структуры в винтернл. h являются внутренними по отношению к операционной системе и могут изменяться от одного выпуска Windows к следующему и, возможно, даже между пакетами обновления для каждого выпуска. Чтобы обеспечить совместимость приложения, вместо этого следует использовать эквивалентные открытые функции. Дополнительные сведения можно найти в файле заголовка Винтернл. h, а также в документации по каждой функции.

Если вы используете эти функции, вы можете обращаться к ним через динамическую компоновку во время выполнения с помощью [**LoadLibrary**](/windows/win32/api/libloaderapi/nf-libloaderapi-loadlibrarya) и [**GetProcAddress**](/windows/win32/api/libloaderapi/nf-libloaderapi-getprocaddress). Это дает коду возможность корректно реагировать, если функция была изменена или удалена из операционной системы. Однако изменения сигнатуры могут быть недоступны для обнаружения.
