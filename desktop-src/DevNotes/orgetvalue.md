# <a name="orgetvalue-function"></a>Функция Оржетвалуе

Извлекает тип и данные для указанного значения реестра в автономном кусте реестра.

## <a name="syntax"></a>Синтаксис


```C++
DWORD ORGetValue(
  _In_        ORHKEY Handle,
  _In_opt_    PCWSTR lpSubKey,
  _In_opt_    PCWSTR lpValue,
  _Out_opt_   PDWORD pdwType,
  _Out_opt_   PVOID  pvData,
  _Inout_opt_ PDWORD pcbData
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Обработчик* \[ окне\]
</dt> <dd>

Маркер открытого раздела реестра в автономном кусте реестра.

</dd> <dt>

*лпсубкэй* \[ в необязательное\]
</dt> <dd>

Имя раздела реестра. Этот ключ должен быть подразделом ключа, указанного параметром *Handle* . Этот параметр может принимать значение **NULL**.

Имена ключей не чувствительны к регистру.

</dd> <dt>

*лпвалуе* \[ в необязательное\]
</dt> <dd>

Имя значения реестра. Если этот параметр имеет **значение NULL** или является пустой строкой, функция получает тип и данные для безымянного ключа или значения по умолчанию, если таковые имеются. Дополнительные сведения см. в разделе [ограничения размера элементов реестра](../sysinfo/registry-element-size-limits.md).

Ключи не имеют автоматически неименованное или значение по умолчанию. Безымянные значения могут быть любого типа.

В именах значений регистр не учитывается.

</dd> <dt>

*пдвтипе* \[ out, необязательно\]
</dt> <dd>

Указатель на переменную, которая получает код, указывающий тип данных, хранящихся в указанном значении. Список возможных кодов типов см. в разделе [типы значений реестра](../sysinfo/registry-value-types.md). Если тип не является обязательным, этот параметр может иметь **значение NULL** .

</dd> <dt>

*пвдата* \[ out, необязательно\]
</dt> <dd>

Указатель на буфер, который получает данные значения. Если данные не требуются, этот параметр может иметь **значение NULL** .

Если данные являются строкой, функция проверяет наличие завершающего нуль-символа. Если он не найден, строка сохраняется с нулевым символом конца, если буфер достаточно велик для размещения дополнительного символа. В противном случае функция завершается с ошибкой и возвращает \_ Дополнительные \_ данные.

</dd> <dt>

*пкбдата* \[ в, out, необязательно\]
</dt> <dd>

Указатель на переменную, указывающую размер буфера, на который указывает параметр *пвдата* в байтах. При возврате функции эта переменная содержит размер данных, копируемых в *пвдата*.

Параметр *пкбдата* может иметь **значение NULL** , только если *пвдата* имеет **значение NULL**.

Если данные имеют \_ тип SZ, REG \_ Multi \_ SZ или REG \_ expand \_ SZ, этот размер включает в себя любой завершающий нуль символ или символы. Дополнительные сведения см. в подразделе "Примечания".

Если буфер, указанный параметром *пвдата* , недостаточно велик для хранения данных, функция возвращает ошибку \_ больше \_ данных и сохраняет требуемый размер буфера в переменной, на которую указывает *пкбдата*. В этом случае содержимое буфера *пвдата* не определено.

Если *пвдата* имеет **значение NULL**, а *пкбдата* не имеет **значение NULL**, функция возвращает ошибку \_ Success и сохраняет размер данных в байтах в переменной, на которую указывает *пкбдата*. Это позволяет приложению определить лучший способ выделения буфера для данных значения.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System.

## <a name="remarks"></a>Remarks

Приложение обычно вызывает функцию [**оренумвалуе**](orenumvalue.md) для определения имен значений, а затем вызывает функцию **оржетвалуе** для получения данных для имен.
