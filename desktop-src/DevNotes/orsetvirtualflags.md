# <a name="orsetvirtualflags-function"></a>Функция Орсетвиртуалфлагс

Устанавливает флаги виртуализации для указанного открытого раздела реестра в автономном кусте реестра.

## <a name="syntax"></a>Синтаксис


```C++
DWORD ORSetVirtualFlags(
  _In_ ORHKEY Handle,
  _In_ DWORD  dwFlags
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Обработчик* \[ окне\]
</dt> <dd>

Маркер открытого раздела реестра в автономном кусте реестра.

</dd> <dt>

*dwFlags* \[ окне\]
</dt> <dd>

Этот параметр управляет поведением реестра при сбое операции создания или открытия ключа в виртуализированном кусте. Этот параметр может принимать одно или несколько следующих значений.



| Значение                                                                                                                                                                                                                                                    | Значение                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="REG_KEY_DONT_SILENT_FAIL"></span><span id="reg_key_dont_silent_fail"></span><dl> <dt>**Reg \_ КЛЮЧ \_ не \_ Автоматический \_ сбой**</dt> <dt>4</dt> </dl> | Если этот флаг установлен и операция открытия завершается сбоем для ключа, для которого включена виртуализация, реестр не пытается повторно открыть ключ. Если этот флаг снят, реестр пытается повторно открыть раздел с МАКСИМАЛЬным \_ допустимым доступом.<br/>                                                                                    |

Если этот флаг снят, реестр пытается создать ключ в виртуальном хранилище. Вызывающий объект должен иметь ключ \_ Read Right для родительского ключа.<br/> |

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System.

## <a name="remarks"></a>Remarks

Виртуализация реестра — это промежуточная технология обеспечения совместимости приложений, которая позволяет выполнять операции записи в реестре, которые имеют глобальное влияние на расположение пользователей. Это перенаправление прозрачно для приложений, читающих или записывающих в реестр.

виртуализация реестра поддерживается начиная с Windows Vista. однако корпорация майкрософт планирует удалить ее из будущих версий Windowsной операционной системы, так как другие приложения совместимы с Windows Vista. Поэтому приложения не должны зависеть от поведения виртуализации реестра в системе.

Виртуализация реестра включена только в следующих целях:

-   32-разрядные интерактивные процессы
-   Ключи в **\_ \_ \\ программном обеспечении для локального компьютера hKey**
-   Ключи, в которые администратор может выполнять запись

Дополнительные сведения см. в разделе [Виртуализация реестра](../sysinfo/registry-virtualization.md).
