# <a name="orenumkey-function"></a>Функция Оренумкэй

Перечисляет подразделы указанного открытого раздела реестра в автономном кусте реестра. Функция получает сведения о одном подразделе при каждом вызове.

## <a name="syntax"></a>Синтаксис


```C++
DWORD OREnumKey(
  _In_        ORHKEY    Handle,
  _In_        DWORD     dwIndex,
  _Out_       PWSTR     lpName,
  _Inout_     PDWORD    lpcName,
  _Out_opt_   PWSTR     lpClass,
  _Inout_opt_ PDWORD    lpcClass,
  _Out_opt_   PFILETIME lpftLastWriteTime
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Обработчик* \[ окне\]
</dt> <dd>

Маркер открытого раздела реестра в автономном кусте реестра.

</dd> <dt>

*двиндекс* \[ окне\]
</dt> <dd>

Индекс извлекаемого подраздела. Этот параметр должен быть равен нулю для первого вызова функции и затем увеличивается для последующих вызовов.

Поскольку подразделы не упорядочены, каждый новый подраздел будет иметь произвольный индекс. Это означает, что функция может возвращать подразделы в любом порядке.

</dd> <dt>

*лпнаме* \[ заполняет\]
</dt> <dd>

Указатель на буфер, который получает имя подраздела, включая завершающий нуль-символ. Функция копирует в буфер только имя подраздела, а не полную иерархию ключей. В случае сбоя функции данные не копируются в буфер.

Дополнительные сведения см. в разделе [ограничения размера элементов реестра](../sysinfo/registry-element-size-limits.md).

</dd> <dt>

*лпкнаме* \[ в, out\]
</dt> <dd>

Указатель на переменную, указывающую размер буфера, заданного параметром *лпнаме* в **вчарс**. Этот размер должен включать завершающий нуль символ. Если функция завершается с ошибкой, переменная, на которую указывает *лпкнаме* , содержит количество символов, хранящихся в буфере, не включая завершающий символ null.

</dd> <dt>

*лпкласс* \[ out, необязательно\]
</dt> <dd>

Указатель на буфер, который получает строку класса перечисления, завершающуюся нулем. Этот параметр может принимать значение **NULL**.

</dd> <dt>

*лпккласс* \[ в, out, необязательно\]
</dt> <dd>

Указатель на переменную, указывающую размер буфера, заданного параметром *лпкласс* в **вчарс**. Размер должен включать завершающий нуль символ. Если функция выполнена, *лпккласс* содержит количество символов, хранящихся в буфере, не включая завершающий символ null. Этот параметр может иметь **значение NULL** , только если *Лпкласс* имеет **значение NULL**.

</dd> <dt>

*лпфтластвритетиме* \[ out, необязательно\]
</dt> <dd>

Указатель на структуру [fileTime](/windows/win32/api/minwinbase/ns-minwinbase-filetime) , которая получает время последнего написания перечисленного подраздела. Этот параметр может принимать значение **NULL**.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System. Ниже перечислены возможные коды ошибок.

-   Если буфер *лпнаме* слишком мал для получения имени ключа, функция возвращает данные об ошибке \_ \_ .
-   Если нет доступных подразделов, функция возвращает ошибку больше не содержит \_ \_ \_ элементов.

## <a name="remarks"></a>Комментарии

Чтобы перечислить подразделы, приложение должно сначала вызвать функцию **оренумкэй** с параметром *двиндекс* , для которого установлено значение 0. Приложение должно увеличить значение параметра *двиндекс* и вызвать **оренумкэй** , пока не будет больше подразделов (то есть функция возвращает ошибку, что больше \_ нет \_ \_ элементов).

Приложение может также установить *двиндекс* в индекс последнего подраздела при первом вызове функции и уменьшить индекс до перечисления подраздела с индексом 0. Чтобы получить индекс последнего подраздела, используйте функцию [**оркуеринфокэй**](/windows/win32/api/winreg/nf-winreg-regqueryinfokeya) .

Хотя приложение использует функцию **оренумкэй** , оно не должно вызывать никакие функции реестра вне сети, которые могут изменить перечисленный ключ.
