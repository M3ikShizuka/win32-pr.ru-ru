# <a name="orenumvalue-function"></a>Функция Оренумвалуе

Перечисляет значения указанного открытого раздела реестра в автономном кусте реестра. Функция получает сведения для одного значения в указанном ключе каждый раз при вызове функции.

## <a name="syntax"></a>Синтаксис


```C++
DWORD OREnumValue(
  _In_        ORHKEY Handle,
  _In_        DWORD  dwIndex,
  _Out_       PWSTR  lpValueName,
  _Inout_     PDWORD lpcValueName,
  _Out_opt_   PDWORD lpType,
  _Out_opt_   PBYTE  lpData,
  _Inout_opt_ PDWORD lpcbData
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Обработчик* \[ окне\]
</dt> <dd>

Маркер открытого раздела реестра в автономном кусте реестра.

</dd> <dt>

*двиндекс* \[ окне\]
</dt> <dd>

Индекс извлекаемого значения. Этот параметр должен быть равен нулю для первого вызова функции, а затем увеличиваться для последующих вызовов.

Поскольку значения не упорядочены, любое новое значение будет иметь произвольный индекс. Это означает, что функция может возвращать значения в любом порядке.

</dd> <dt>

*лпвалуенаме* \[ заполняет\]
</dt> <dd>

Указатель на буфер, который получает имя значения в виде строки, завершающейся нулем. Размер этого буфера должен быть достаточным для включения завершающего нуль символа.

Дополнительные сведения см. в разделе [ограничения размера элементов реестра](../sysinfo/registry-element-size-limits.md).

</dd> <dt>

*лпквалуенаме* \[ в, out\]
</dt> <dd>

Указатель на переменную, указывающую размер буфера, на который указывает параметр *лпвалуенаме* в символах. Когда функция возвращает значение, переменная получает количество символов, хранящихся в буфере, не включая завершающий символ null.

</dd> <dt>

*лптипе* \[ out, необязательно\]
</dt> <dd>

Указатель на переменную, которая получает код, указывающий тип данных, хранящихся в указанном значении. Список возможных кодов типов см. в разделе [типы значений реестра](../sysinfo/registry-value-types.md). Параметр *лптипе* может иметь **значение NULL** , если код типа не является обязательным.

</dd> <dt>

*лпдата* \[ out, необязательно\]
</dt> <dd>

Указатель на буфер, который получает данные для записи значения. Если данные не требуются, этот параметр может иметь **значение NULL** .

Если *лпдата* имеет **значение NULL** , а *лпкбдата* не равен **null**, функция сохраняет размер данных в байтах в переменной, на которую указывает *лпкбдата*. Это позволяет приложению определить лучший способ выделения буфера для данных.

</dd> <dt>

*лпкбдата* \[ в, out, необязательно\]
</dt> <dd>

Указатель на переменную, указывающую размер буфера, на который указывает параметр *лпдата* в байтах. Когда функция возвращает значение, переменная получает число байтов, хранящихся в буфере.

Этот параметр может иметь **значение NULL** , только если *Лпдата* имеет **значение NULL**.

Если данные имеют \_ тип SZ, REG \_ Multi \_ SZ или REG \_ expand \_ SZ, этот размер включает в себя любой завершающий нуль символ или символы. Дополнительные сведения см. в подразделе "Примечания".

Если буфер, указанный параметром *лпдата* , недостаточно велик для хранения данных, функция возвращает ошибку \_ больше \_ данных и сохраняет требуемый размер буфера в переменной, на которую указывает *лпкбдата*. В этом случае содержимое *лпдата* не определено.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System.

Если буфер *лпдата* слишком мал для получения значения, функция возвращает данные об ошибке \_ \_ .

## <a name="remarks"></a>Remarks

Чтобы перечислить значения, приложение должно сначала вызвать функцию **оренумвалуе** с параметром *двиндекс* , имеющим значение 0. Приложение должно увеличить *двиндекс* и вызвать функцию **оренумвалуе** до тех пор, пока не будет больше значений (до тех пор, пока функция не вернет ошибку больше \_ \_ \_ элементов).

Приложение может также установить *двиндекс* в индекс последнего значения первого вызова функции и уменьшить индекс до перечисления значения с индексом 0. Чтобы получить индекс последнего значения, используйте функцию [**оркуеринфокэй**](orqueryinfokey.md) .

При использовании **оренумвалуе** приложение не должно вызывать никаких функций автономного реестра, которые могут изменить запрашиваемый ключ.

Если данные имеют \_ тип SZ, REG \_ Multi \_ SZ или REG \_ expand \_ SZ, строка может быть не сохранена с правильными символами, завершающимися нулем. Таким образом, даже если функция возвращает ошибку \_ успешного выполнения, приложение должно убедиться, что строка должна завершиться должным образом, прежде чем использовать ее; в противном случае она может перезаписать буфер. (Обратите внимание, что REG \_ В нескольких \_ SZ строках должно быть два завершающих символа null.)

Для определения максимального размера буферов имен и данных используйте функцию [**оркуеринфокэй**](orqueryinfokey.md) .
