# <a name="oropenkey-function"></a>Функция Оропенкэй

Открывает указанный раздел реестра в автономном кусте реестра.

## <a name="syntax"></a>Синтаксис


```C++
DWORD OROpenKey(
  _In_     ORHKEY  Handle,
  _In_opt_ PCWSTR  lpSubKeyName,
  _Out_    PORHKEY phkResult
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Обработчик* \[ окне\]
</dt> <dd>

Маркер открытого раздела реестра в автономном кусте реестра.

</dd> <dt>

*лпсубкэйнаме* \[ в необязательное\]
</dt> <dd>

Указатель на строку в Юникоде, содержащую имя открываемого раздела реестра. Этот ключ должен быть подразделом ключа, определяемого параметром *Handle* .

Имена ключей не чувствительны к регистру.

Если этот параметр имеет **значение NULL** или указатель на пустую строку, функция возвращает тот же переданный обработчик. Если ключ, заданный параметром *Handle* , является корневым ключом Hive, функция возвращает ошибку " \_ Недопустимый \_ параметр".

Дополнительные сведения см. в разделе [ограничения размера элементов реестра](../sysinfo/registry-element-size-limits.md).

</dd> <dt>

*фкресулт* \[ заполняет\]
</dt> <dd>

Указатель на переменную, которая получает маркер открытого ключа. Используйте функцию [**орклосекэй**](orclosekey.md) , чтобы закрыть ключ после завершения использования маркера.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System.

Если возвращаемый маркер будет маркером корневого ключа Hive, функция возвращает ошибку " \_ Недопустимый \_ параметр".

Если указанный ключ был помечен как удаленный, эта функция возвращает \_ ключ ошибки \_ deleted.

## <a name="remarks"></a>Комментарии

Функцию **оропенкэй** нельзя использовать для открытия корневого ключа в кусте реестра вне сети. Чтобы получить маркер корневого ключа Hive, используйте функцию [**оропенхиве**](oropenhive.md) для загрузки Hive в память.
