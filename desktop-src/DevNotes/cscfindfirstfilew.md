# <a name="cscfindfirstfilew-function"></a>Функция Кскфиндфирстфилев

\[Эта функция не поддерживается и не должна использоваться.\]

Выполняет поиск файла в кэше автономные файлы, который соответствует указанным критериям.

## <a name="syntax"></a>Синтаксис


```C++
HANDLE WINAPI CSCFindFirstFileW(
  _In_  LPCWSTR          lpszFileName,
  _Out_ WIN32_FIND_DATAW *lpFind32,
  _Out_ LPDWORD          lpdwStatus,
  _Out_ LPDWORD          lpdwPinCount,
  _Out_ LPDWORD          lpdwHintFlags,
  _Out_ FILETIME         *lpOrgFileTime
);
```

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена, возвращаемое значение является маркером поиска, используемым при последующем вызове метода [**кскфинднекстфилев**](cscfindnextfilew.md) или [**кскфиндклосе**](cscfindclose.md).

Если функция выполняется неудачно, возвращается значение **INVALID\_HANDLE\_VALUE**.

## <a name="remarks"></a>Remarks

Эта функция не имеет связанной библиотеки импорта или файла заголовка. его необходимо вызвать с помощью функций [**LoadLibrary**](/windows/win32/api/libloaderapi/nf-libloaderapi-loadlibrarya) и [**GetProcAddress**](/windows/win32/api/libloaderapi/nf-libloaderapi-getprocaddress) .
