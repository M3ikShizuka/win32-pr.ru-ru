# <a name="detecting-kernel-mode-callbacks"></a>Обнаружение обратных вызовов Kernel-Mode

большая часть кода для Windows операционной системы выполняется в режиме ядра. режим процессора переключается из пользовательского режима в режим ядра каждый раз, когда поток приложения вызывает функцию из API Windows, который, в свою очередь, вызывает внутреннюю системную функцию, которая должна выполняться в режиме ядра. Режим процессора возвращается в пользовательский режим перед возвратом управления функции, чтобы обеспечить защиту системных данных.

Если поток ожидает завершения обратного вызова режима ядра, то сторона пользовательского режима в потоке будет задерживаться при вызове функции **звкаллбаккретурн** .
