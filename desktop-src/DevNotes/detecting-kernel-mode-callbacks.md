# <a name="detecting-kernel-mode-callbacks"></a>Обнаружение обратных вызовов Kernel-Mode

Большая часть кода для операционной системы Windows выполняется в режиме ядра. Режим процессора переключается из пользовательского режима в режим ядра каждый раз, когда поток приложения вызывает функцию из API Windows, которая, в свою очередь, вызывает внутреннюю системную функцию, которая должна выполняться в режиме ядра. Режим процессора возвращается в пользовательский режим перед возвратом управления функции, чтобы обеспечить защиту системных данных.

Если поток ожидает завершения обратного вызова режима ядра, то сторона пользовательского режима в потоке будет задерживаться при вызове функции **звкаллбаккретурн** .
