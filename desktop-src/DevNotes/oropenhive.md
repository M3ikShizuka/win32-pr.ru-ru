# <a name="oropenhive-function"></a>Функция Оропенхиве

Загружает указанный файл куста реестра в память и проверяет куст.

## <a name="syntax"></a>Синтаксис


```C++
DWORD OROpenHive(
  _In_  PCWSTR  lpHivePath,
  _Out_ PORHKEY phkResult
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*лфивепас* \[ окне\]
</dt> <dd>

Указатель на строку в Юникоде, указывающую имя файла куста реестра, который будет загружен в память. Это может быть файл Hive, сохраненный с помощью функции [**орсавехиве**](orsavehive.md) или созданный с помощью функции [регсавекэй](/windows/win32/api/winreg/nf-winreg-regsavekeya) или [регсавекэйекс](/windows/win32/api/winreg/nf-winreg-regsavekeyexa) . Размер файла должен быть менее 4 ГБ, и вызывающая сторона должна иметь \_ \_ доступ к файлу для чтения данных. Дополнительные сведения см. в разделе [безопасность и права доступа к файлам](../fileio/file-security-and-access-rights.md).

</dd> <dt>

*фкресулт* \[ заполняет\]
</dt> <dd>

Указатель на переменную, которая получает маркер корневого ключа загруженного куста реестра. Если не удается открыть файл куста реестра или выполнить проверку не удается, функция присваивает этому параметру **значение NULL**.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является ненулевым кодом ошибки, определенным в файле Winerror. h. [](/windows/win32/api/winbase/nf-winbase-formatmessage) \_ \_ \_ Для получения обобщенного описания ошибки можно использовать функцию FormatMessage с флагом формата Message от System. Ниже перечислены возможные коды ошибок.

-   Если файл пуст или имеет размер более 4 ГБ, функция возвращает ошибку \_ баддб.
-   Если вызывающий объект не имеет необходимых прав доступа для открытия файла, функция возвращает сообщение об ошибке \_ отказа в доступе \_ .
-   Если куст реестра не проходит проверку, функция возвращает ошибку \_ без \_ файла реестра \_ .

## <a name="remarks"></a>Комментарии

Функция **оропенхиве** является единственной автономной функцией реестра, которая проверяет куст реестра. Если проверка не удалась, попытка восстановления куста не выполняется.
