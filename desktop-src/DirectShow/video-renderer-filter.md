## <a name="remarks"></a><span data-ttu-id="50787-140">Комментарии</span><span class="sxs-lookup"><span data-stu-id="50787-140">Remarks</span></span>

<span data-ttu-id="50787-141">В отладочной версии Quartz.dll, если \_ уровень отладки трассировки журнала равен 5 или выше, модуль подготовки отчетов отображает штампы времени каждого кадра в окне видео.</span><span class="sxs-lookup"><span data-stu-id="50787-141">In the debug version of Quartz.dll, if the LOG\_TRACE debug level is set to 5 or higher, the Video Renderer displays each frame's time stamps on the video window.</span></span> <span data-ttu-id="50787-142">Эти номера не отображаются в розничной версии библиотеки DLL.</span><span class="sxs-lookup"><span data-stu-id="50787-142">These numbers do not appear in the retail version of the DLL.</span></span> <span data-ttu-id="50787-143">Дополнительные сведения см. в разделе [отладочные функции вывода](debug-output-functions.md).</span><span class="sxs-lookup"><span data-stu-id="50787-143">For more information, see [Debug Output Functions](debug-output-functions.md).</span></span>

<span data-ttu-id="50787-144">Следующие примечания предназначены для разработчиков фильтров:</span><span class="sxs-lookup"><span data-stu-id="50787-144">The following remarks are intended for filter developers:</span></span>

<span data-ttu-id="50787-145">Модуль подготовки видео принимает форматы YUV, если видеоадаптер поддерживает поверхности наложения YUV.</span><span class="sxs-lookup"><span data-stu-id="50787-145">The Video Renderer accepts YUV formats if the video graphics card supports YUV overlay surfaces.</span></span> <span data-ttu-id="50787-146">Однако при первом подключении к вышестоящему фильтру для модуля подготовки видео требуется формат RGB, соответствующий глубине цвета текущих параметров монитора.</span><span class="sxs-lookup"><span data-stu-id="50787-146">When it first connects to the upstream filter, however, the Video Renderer requires an RGB format that matches the color depth of the current monitor settings.</span></span> <span data-ttu-id="50787-147">Например, если текущим параметром дисплея является 24-разрядный цвет, то вышестоящий фильтр должен иметь возможность предоставлять 24-битное видео RGB.</span><span class="sxs-lookup"><span data-stu-id="50787-147">For example, if the current display setting is 24-bit color, the upstream filter must be able to provide 24-bit RGB video.</span></span> <span data-ttu-id="50787-148">Когда граф фильтра переключается в состояние которых запущены, модуль подготовки видео согласовывает динамическое изменение формата с соответствующим цветовым пространством YUV.</span><span class="sxs-lookup"><span data-stu-id="50787-148">When the filter graph switches to a runing state, the Video Renderer negotiates a dynamic format change to the appropriate YUV color space.</span></span>

<span data-ttu-id="50787-149">При соединении с типом RGB модуль подготовки видео гарантирует, что он сможет использовать GDI в случае, если DirectDraw недоступен.</span><span class="sxs-lookup"><span data-stu-id="50787-149">By connecting with an RGB type, the Video Renderer ensures that it can use GDI in case DirectDraw is not available.</span></span> <span data-ttu-id="50787-150">Он переключается на GDI, если другое приложение использует видеопамять, если прямоугольник видео размещает два монитора в системе с несколькими мониторами или если прямоугольник видео полностью скрыт другим окном.</span><span class="sxs-lookup"><span data-stu-id="50787-150">It will switch to GDI if another application is using the video memory, if the video rectangle straddles two monitors on a multi-monitor system, or if the video rectangle is completely obscured by another window.</span></span>

> [!Note]  
> <span data-ttu-id="50787-151">Модуль подготовки видео не выполняет этот тип изменения динамического формата и не требует типа мультимедиа RGB, так как он никогда не использует GDI для отрисовки.</span><span class="sxs-lookup"><span data-stu-id="50787-151">The Video Mixing Renderer does not perform this type of dynamic format change, and does not require an RGB media type, because it never uses GDI for rendering.</span></span>
