# <a name="cenumpins-class"></a>Класс Ценумпинс

![Иерархия классов ценумпинс](images/filter03.png)

`CEnumPins`Класс реализует перечислитель для ПИН-кодов.

Этот класс реализует интерфейс [**иенумпинс**](/windows/desktop/api/Strmif/nn-strmif-ienumpins) . Он вызывает следующие методы [**кбасефилтер**](cbasefilter.md) :

-   [**Кбасефилтер:: жетпин**](cbasefilter-getpin.md): получает ПИН-код для фильтра, на который ссылается индекс, начинающийся с нуля.
-   [**Кбасефилтер:: жетпинкаунт**](cbasefilter-getpincount.md)— получает общее количество ПИН-кодов в фильтре.
-   [**Кбасефилтер:: жетпинверсион**](cbasefilter-getpinversion.md): определяет, изменились ли ПИН-коды.

Если фильтр динамически создает или уничтожает ПИН-коды, то при каждом изменении ПИН увеличивается ее версия. При изменении номера версии объект перечислителя больше не синхронизируется с фильтром. После того как перечислитель не синхронизирован, методы в `CEnumPins` возвращают \_ \_ \_ \_ Несинхронизированное перечисление в VFW E \_ . Вызовите метод [**ценумпинс:: Reset**](cenumpins-reset.md) для повторной синхронизации перечислителя.
