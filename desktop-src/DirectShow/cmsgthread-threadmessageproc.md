# <a name="cmsgthreadthreadmessageproc-method"></a>Кмсгсреад. Среадмессажепрок, метод

Обрабатывает запросы. Это Чистая виртуальная функция-член.

## <a name="syntax"></a>Синтаксис


```C++
virtual LRESULT ThreadMessageProc(
   UINT     uMsg,
   DWORD    dwFlags,
   LPVOID   lpParam,
   CAMEvent *pEvent
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Uiacp* 
</dt> <dd>

Код запроса.

</dd> <dt>

*dwFlags* 
</dt> <dd>

Необязательный параметр флага для запроса.

</dd> <dt>

*лппарам* 
</dt> <dd>

Необязательный указатель на дополнительные данные или блок возвращаемых данных.

</dd> <dt>

*певент* 
</dt> <dd>

Необязательный указатель на объект события.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Любые ненулевые возвращаемые значения приводят к выходу потока. Возвращает нуль, если только запрос на выход не был обработан в последнее время.

## <a name="remarks"></a>Remarks

Эта Чистая виртуальная функция должна быть переопределена в производном классе. Он будет вызываться один раз для каждого запроса, помещенного в очередь с помощью вызова функции-члена [**кмсгсреад::P утсреадмсг**](cmsgthread-putthreadmsg.md) .

Функция члена определяет четыре параметра. Как правило, для указания запроса используется параметр *uiacp* , а остальные три параметра будут дополнительными. Вызывающее приложение может предоставить указатель на объект [**камевент**](camevent.md) в параметре *певент* , если это требуется приложению. Это событие необходимо задать после обработки события с помощью такого выражения, как:


```C++
pEvent->SetEvent
```



Один код запроса необходимо отложить, чтобы сообщить рабочему потоку о выходе из него. После получения этого запроса возвращается 1 из этой функции члена. Возвращает 0, если не нужно, чтобы рабочий поток завершил работу.
