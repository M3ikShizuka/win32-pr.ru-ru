# <a name="cammsgeventwaitmsg-method"></a>Каммсжевент. Ваитмсг, метод

`WaitMsg`Метод ожидает получения сигнала о событии при диспетчеризации отправленных сообщений.

## <a name="syntax"></a>Синтаксис


```C++
BOOL WaitMsg(
   DWORD dwTimeOut = INFINITE
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*двтимеаут* 
</dt> <dd>

Необязательное значение времени ожидания в миллисекундах.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает **значение true** , если событие является сигнальным, или **значение false** , если истекло время ожидания.

## <a name="remarks"></a>Комментарии

Этот метод вызывает функцию PeekMessage для обработки сообщений. Вызовите этот метод вместо [**камевент:: wait**](camevent-wait.md) , если поток должен обрабатывать сообщения при ожидании события. Если поток не обрабатывает сообщения и другой поток отправляет сообщение, может произойти взаимоблокировка.

Например, предположим, что создается поток, а затем блокируется до инициализации потока. Если поток отправляет сообщение в окно с помощью вызова функции SendMessage, это приведет к взаимоблокировке. Это обусловлено тем, что SendMessage не возвращает значение, пока сообщение не будет обработано. Вызов Ваитмсг позволяет вернуть вызов SendMessage, предотвращая взаимоблокировку.
