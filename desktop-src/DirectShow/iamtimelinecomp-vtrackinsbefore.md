# <a name="iamtimelinecompvtrackinsbefore-method"></a>Метод Иамтимелинекомп:: Втраккинсбефоре

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`VTrackInsBefore`Метод вставляет виртуальную запись в композицию с указанным приоритетом.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT VTrackInsBefore(
   IAMTimelineObj *pVirtualTrack,
   long           Priority
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пвиртуалтракк* 
</dt> <dd>

Указатель на интерфейс [**иамтимелинеобж**](iamtimelineobj.md) виртуальной записи.

</dd> <dt>

*Приоритет* 
</dt> <dd>

Приоритет вставки виртуальной записи или – 1, чтобы вставить виртуальную дорожку в конец списка приоритетов. Список приоритетов определяет, какие клипы видимы. Дополнительные сведения см. в разделе "Примечания".

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает одно из следующих значений **HRESULT** :



| Код возврата                                                                                   | Описание                               |
|-----------------------------------------------------------------------------------------------|-------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>          | Успешно.<br/>                       |
| <dl> <dt>**E \_ INVALIDARG**</dt> </dl>  | Недопустимый аргумент.<br/>              |
| <dl> <dt>**\_интерфейс E**</dt> </dl> | Объект не является виртуальной дорожкой.<br/> |



 

## <a name="remarks"></a>Remarks

Каждая виртуальная дорожка в составе имеет уникальный уровень приоритета. Уровни приоритета находятся в диапазоне от 0 до *n* – 1, где *n* — число виртуальных дорожек в композиции. Для видеогрупп виртуальная дорожка скрывает все виртуальные дорожки с более низким уровнем приоритета, за исключением тех мест, где дорожка пуста или содержит переход. Виртуальные дорожки можно рассматривать как слои в окончательной композиции. Транспортер 1 размещается поверх записи 0, а на дорожке 2 — на уровне Track 1 и т. д.

Если для параметра *Priority* задано значение-1, виртуальная дорожка вставляется в конец списка с более высоким значением приоритета, чем в существующих дорожках. Если указать значение приоритета, которое уже существует в композиции, каждая запись с равным или более высоким приоритетом перемещается на один уровень приоритета.

**Пример**: Track A имеет приоритет 0, а для Track B — приоритет 1. Если элемент Track C вставляется с приоритетом 0, трассировка переходит к приоритету 1, а Track B переходит к приоритету 2.

Если заданный приоритет больше текущего числа дорожек в композиции, метод завершается ошибкой.
