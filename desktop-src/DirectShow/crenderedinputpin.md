# <a name="crenderedinputpin-class"></a>Класс Крендерединпутпин

![Иерархия классов крендерединпутпин](images/rbase04.png)

Класс **крендерединпутпин** является базовым классом для реализации входного ПИН-кода в модуле подготовки отчетов. Этот класс предназначен для фильтров модуля подготовки отчетов, которые не являются производными от класса [**кбасерендерер**](cbaserenderer.md) . (Фильтры, производные от **кбасерендерер** , должны использовать класс [**крендереринпутпин**](crendererinputpin.md) для входного ПИН-кода.)

Чтобы использовать этот класс, необходимо выполнить по крайней мере следующее:

-   Объявите новый класс ПИН-кода, наследующий **крендерединпутпин**.
-   В классе ПИН-кода объявите объект критической секции для хранения блокировки потоковой передачи. Для этой цели можно использовать класс [**ккритсек**](ccritsec.md) . Дополнительные сведения см. в разделе [потоки и критические разделы](threads-and-critical-sections.md).
-   Переопределите [**крендерединпутпин:: EndOfStream**](crenderedinputpin-endofstream.md) для удержания потоковой блокировки.
-   Реализуйте методы [**имеминпутпин:: Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive), [**Кбасепин:: чеккмедиатипе**](cbasepin-checkmediatype.md)и [**кбасепин:: жетмедиатипе**](cbasepin-getmediatype.md) .
-   В фильтре реализуйте [**кбасефилтер:: жетпин**](cbasefilter-getpin.md) , чтобы вернуть экземпляр класса ПИН-кода.

Этот класс можно использовать в модуле подготовки отчетов, который имеет более одного входного ПИН-кода. Этот класс наследует класс [**кбасеинпутпин**](cbaseinputpin.md) .
