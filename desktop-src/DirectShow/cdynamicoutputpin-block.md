# <a name="cdynamicoutputpinblock-method"></a>Метод Кдинамикаутпутпин. Block

`Block`Метод блокирует или разблокирует поток данных из ПИН-кода. Этот метод реализует метод [**ипинфловконтрол:: Block**](/windows/desktop/api/Strmif/nf-strmif-ipinflowcontrol-block) .

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Block(
   DWORD  dwBlockFlags,
   HANDLE hEvent
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*двблоккфлагс* 
</dt> <dd>

Флаг, указывающий, следует ли блокировать или разблокировать ПИН-код. Необходимо установить одно из следующих значений.

Нуль: разблокировать поток данных из ПИН-кода.

\_Закрепление \_ \_ блока управления потоком \_ : Блокировка потока данных из ПИН-кода.

</dd> <dt>

*хевент* 
</dt> <dd>

Указатель на объект события или **значение NULL**.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение **HRESULT** . Возможные значения включают в себя, показанные в следующей таблице.



| Код возврата                                                                                                                    | Описание                                              |
|--------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------|
| <dl> <dt>**S \_ false**</dt> </dl>                                        | ПИН-код уже разблокирован.<br/>                     |
| <dl> <dt>**\_ОК**</dt> </dl>                                           | Успешно.<br/>                                      |
| <dl> <dt>**E \_ INVALIDARG**</dt> </dl>                                   | Недопустимый аргумент.<br/>                             |
| <dl> <dt>**\_ \_ ПИН-код VFW E \_ уже \_ заблокирован**</dt> </dl>                   | ПИН-код уже заблокирован в другом потоке.<br/>     |
| <dl> <dt>**\_ \_ ПИН-код VFW E \_ уже \_ заблокирован \_ в \_ этом \_ потоке**</dt> </dl> | ПИН-код уже заблокирован в вызывающем потоке.<br/> |



 

## <a name="remarks"></a>Комментарии

Дополнительные сведения об этом методе см. в разделе [**ипинфловконтрол:: Block**](/windows/desktop/api/Strmif/nf-strmif-ipinflowcontrol-block). На внутреннем уровне этот метод вызывает один из следующих защищенных методов:

-   Block (асинхронный): [ **кдинамикаутпутпин:: асинчронаусблоккаутпутпин**](cdynamicoutputpin-asynchronousblockoutputpin.md)
-   Block (синхронная): [ **кдинамикаутпутпин:: синчронаусблоккаутпутпин**](cdynamicoutputpin-synchronousblockoutputpin.md)
-   Разблокировать: [ **кдинамикаутпутпин:: унблоккаутпутпин**](cdynamicoutputpin-unblockoutputpin.md)

Разблокировка всегда выполняется синхронно.
