# <a name="ctransinplacefiltercompleteconnect-method"></a>Ктрансинплацефилтер. Комплетеконнект, метод

`CompleteConnect`Метод завершает соединение с закреплением.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT CompleteConnect(
   PIN_DIRECTION direction,
   IPin          *pReceivePin
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*direction* 
</dt> <dd>

Член перечисленного типа с [**\_ направлением ПИН-кода**](/windows/win32/api/strmif/ne-strmif-pin_direction) , указывающий, какой ПИН-код фильтра делает соединение.

</dd> <dt>

*прецеивепин* 
</dt> <dd>

Указатель на интерфейс [**Ипин**](/windows/desktop/api/Strmif/nn-strmif-ipin) другого ПИН-кода в этой попытке подключения.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает **значение HRESULT**. Возможные значения включают в себя, показанные в следующей таблице.



| Код возврата                                                                                           | Описание                                     |
|-------------------------------------------------------------------------------------------------------|-------------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>                  | Успешно.<br/>                             |
| <dl> <dt>**VFW \_ E \_ не \_ в \_ графе**</dt> </dl> | Фильтр не находится на графе фильтра.<br/> |



 

## <a name="remarks"></a>Комментарии

Этот метод переопределяет метод [**ктрансформфилтер:: комплетеконнект**](ctransformfilter-completeconnect.md) .

Поведение фильтра зависит от порядка подключения ПИН-кода:

-   Если входной ПИН-код подключен первым, подключение использует временный распределитель. Когда выходной ПИН-код подключен, фильтр повторно подключает входной ПИН-код. Повторное подключение входного ПИН-кода вызывает повторное согласование распределителя в вышестоящем фильтре. На этом этапе входной ПИН-код предлагает распределитель из подчиненного фильтра. Дополнительные сведения см. в разделе [**ктрансинплацеинпутпин:: распределителе**](ctransinplaceinputpin-getallocator.md).
-   Если выходной ПИН-код подключен первым, то закрепление вывода не выбирает распределитель. Когда входной ПИН-код подключен, он согласовывает распределитель для обоих подключений. Если входные и выходные типы носителей не совпадают, фильтр повторно подключает выходной ПИН-код с помощью входного типа.

Фильтр выполняет все пересоединения ПИН-кода, вызывая метод [**кбасефилтер:: реконнектпин**](cbasefilter-reconnectpin.md) . Метод **реконнектпин** , в свою очередь, вызывает метод [**IFilterGraph2:: реконнектекс**](/windows/desktop/api/Strmif/nf-strmif-ifiltergraph2-reconnectex) в диспетчере графов фильтров.
