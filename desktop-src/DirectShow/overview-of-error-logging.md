# <a name="overview-of-error-logging"></a>Общие сведения о ведении журнала ошибок

\[Этот API не поддерживается и может быть изменен или недоступен в будущем.\]

чтобы предоставить приложениям максимальную гибкость при обработке ошибок, [DirectShow службы редактирования](directshow-editing-services.md) используют механизм обратного вызова. Приложение реализует метод для регистрации ошибки. Во время выполнения, если возникает ошибка, DES вызывает предоставленный метод. Метод принимает параметры, описывающие ошибку. Действия метода с этими сведениями помогут вам. (Он должен возвращать как можно быстрее, или же может мешать выполнению программы.)

Метод обратного вызова регистрации ошибок содержится в COM-интерфейсе [**иамеррорлог**](iamerrorlog.md). Приложение должно реализовать этот интерфейс. Как и все COM-интерфейсы, **иамеррорлог** наследует интерфейс **IUnknown** , поэтому приложение также должно его реализовать.

Существует несколько способов реализации этих COM-интерфейсов. Можно использовать библиотеку активных шаблонов (ATL), которая предоставляет стандартные реализации методов **IUnknown** . DirectShow также предоставляет базовый класс C++ [**кункновн**](cunknown.md), который упрощает реализацию COM-интерфейса. Сведения об использовании **кункновн** см. [в разделе Реализация IUnknown](how-to-implement-iunknown.md).

Пример кода в этой статье определяет автономный класс C++, который реализует **IUnknown** и **иамеррорлог**. Результат не является истинным COM-объектом, так как он не поддерживает **CoCreateInstance**. Однако этот подход достаточно для целей примера.
