# <a name="ctransformfiltertransform-method"></a>Ктрансформфилтер. Transform, метод

`Transform`Метод преобразует входной пример для создания примера вывода.

## <a name="syntax"></a>Синтаксис


```C++
virtual HRESULT Transform(
   IMediaSample *pIn,
   IMediaSample *pOut
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*ПИН* 
</dt> <dd>

Указатель на интерфейс [**имедиасампле**](/windows/desktop/api/Strmif/nn-strmif-imediasample) образца входных данных.

</dd> <dt>

*Тоска* 
</dt> <dd>

Указатель на интерфейс [**имедиасампле**](/windows/desktop/api/Strmif/nn-strmif-imediasample) образца вывода.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Базовый класс возвращает E \_ непредвиденное значение.

Производный класс должен возвращать значение **HRESULT** , указывающее на успешное выполнение или ошибку. Возможные значения включают в себя, показанные в следующей таблице.



| Код возврата                                                                             | Описание                            |
|-----------------------------------------------------------------------------------------|----------------------------------------|
| <dl> <dt>**S \_ false**</dt> </dl> | Не Доставьте этот пример.<br/> |
| <dl> <dt>**\_ОК**</dt> </dl>    | Успешно.<br/>                    |



 

## <a name="remarks"></a>Remarks

Переопределите этот метод для получения выходных данных. Считывает входные данные из образца, указанного параметром *ПИН-кода* , и записывает новые данные в образец, заданный параметром *тоска* .

Перед тем как фильтр вызовет этот метод, он копирует свойства из примера входных данных в образец Output. `Transform`Метод должен устанавливать любые свойства, которые отличаются между двумя образцами, используя методы **имедиасампле** или интерфейс [**IMediaSample2**](/windows/desktop/api/Strmif/nn-strmif-imediasample2) (если он доступен).

Если фильтр не должен доставлять этот пример (например, для поддержки контроля качества), метод должен вернуть \_ значение false.
