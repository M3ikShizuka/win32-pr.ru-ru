# <a name="cbasepincbasepin-constructor"></a>Кбасепин. Кбасепин, конструктор

Метод конструктора.

## <a name="syntax"></a>Синтаксис


```C++
CBasePin(
   TCHAR         *pObjectName,
   CBaseFilter   *pFilter,
   CCritSec      *pLock,
   HRESULT       *phr,
   LPCWSTR       pName,
   PIN_DIRECTION dir
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*побжектнаме* 
</dt> <dd>

Строка, содержащая имя отладки для объекта. Дополнительные сведения см. в разделе [**кбасеобжект**](cbaseobject.md).

</dd> <dt>

*пфилтер* 
</dt> <dd>

Указатель на фильтр, создавший этот ПИН-код.

</dd> <dt>

*плокк* 
</dt> <dd>

Указатель на [**ккритсек**](ccritsec.md) блокировку, используемый для сериализации изменений состояния. Может быть той же критической секцией, что и блокировка фильтра, [**кбасефилтер:: m \_ плокк**](cbasefilter-m-plock.md).

</dd> <dt>

*фр* 
</dt> <dd>

Указатель на переменную, которая получает значение **HRESULT** , указывающее на успешное выполнение или ошибку метода. Перед созданием объекта инициализируйте значение до " \_ ОК". Значение изменяется только в случае возникновения ошибки.

</dd> <dt>

*pName* 
</dt> <dd>

Строка расширенных символов, содержащая имя ПИН-кода. Дополнительные сведения см. в разделе [**кбасепин:: куерипининфо**](cbasepin-querypininfo.md).

</dd> <dt>

*dir* 
</dt> <dd>

Элемент перечисления [**\_ направления ПИН-кода**](/windows/win32/api/strmif/ne-strmif-pin_direction) , указывающий направление ПИН-кода.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Критическая секция, заданная параметром *плокк* , сериализует состояние ПИН-кода, включая его состояние соединения, выбора распределителя, типа носителя и состояния операций записи на диск. Не используйте этот критический раздел для сериализации операций потоковой передачи. Дополнительные сведения см. [в разделе поток данных в графе фильтра](data-flow-in-the-filter-graph.md).

Фильтр может создавать ПИН-коды в своем методе-конструкторе, поэтому на этом этапе указатель *пфилтер* может не ссылаться на допустимый объект. Сохранение указателя, но не разыменование его в конструкторе закрепления.
