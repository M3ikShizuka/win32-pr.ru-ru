# <a name="overview-of-event-notification"></a>Обзор уведомления о событиях

фильтр уведомляет Graph Manager о событии, публикуя уведомление о событии. Событие может быть ожидаемым, например концом потока, или сообщением об ошибке, например при сбое визуализации потока. фильтр Graph Manager обрабатывает некоторые события фильтра по отдельности и оставляет другие для обработки приложению. если фильтр Graph Manager не обрабатывает событие фильтра, уведомление о событии помещается в очередь. Граф фильтра также может ставить в очередь собственные уведомления о событиях для приложения.

Приложение получает события из очереди и реагирует на них в зависимости от типа события. уведомление о событии в DirectShow, таким образом, аналогично схеме очереди сообщений Microsoft Windows. приложение может также отменить поведение фильтра Graph Manager по умолчанию для определенного типа событий. затем фильтр Graph Manager помещает эти события непосредственно в очередь, чтобы приложение было обработано.

Этот механизм позволяет

-   фильтр Graph Manager для взаимодействия с приложением.
-   фильтры для взаимодействия с приложением и фильтром Graph Manager.
-   Приложение для определения степени участия в обработке событий.
