# <a name="ctransinplacefilterdecidebuffersize-method"></a>Ктрансинплацефилтер. ДеЦидебуфферсизе, метод

`DecideBufferSize`Метод задает требования к буферу выходного контакта.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT DecideBufferSize(
   IMemAllocator        *pAlloc,
   ALLOCATOR_PROPERTIES *pProperties
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*паллок* 
</dt> <dd>

Указатель на объект [**имемаллокатор**](/windows/desktop/api/Strmif/nn-strmif-imemallocator) , используемый выходным закреплением.

</dd> <dt>

*pProperties* 
</dt> <dd>

Указатель на запрошенные свойства распределителя для счетчика, размера и выравнивания, как указано в [**структуре \_ свойств распределителя**](/windows/win32/api/strmif/ns-strmif-allocator_properties) .

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение **HRESULT** . Возможные значения включают в себя, показанные в следующей таблице.



| Код возврата                                                                            | Описание        |
|----------------------------------------------------------------------------------------|--------------------|
| <dl> <dt>**\_ОК**</dt> </dl>   | Успешно<br/> |
| <dl> <dt>**\_Ошибка E**</dt> </dl> | Failure<br/> |



 

## <a name="remarks"></a>Комментарии

Этот метод вызывается, когда классу **ктрансинплацефилтер** требуется предоставить размер буфера для подчиненного фильтра. Если фильтр **ктрансинплацефилтер** уже подключился к вышестоящему потоку, он использует свойства распределителя для вышестоящего подключения. В противном случае размер буфера устанавливается в 1 байт в качестве временного значения заполнителя. При соединении вышестоящего фильтра класс **ктрансинплацефилтер** повторно согласовывает нисходящий распределитель. Дополнительные сведения о процессе подключения ПИН-кода в этом классе см. в разделе [**класс ктрансинплацефилтер**](ctransinplacefilter.md).
