# <a name="cbaserendererwaitforrendertime-method"></a>Кбасерендерер. Ваитфоррендертиме, метод

`WaitForRenderTime`Метод ожидает время показа текущего образца.

## <a name="syntax"></a>Синтаксис


```C++
virtual HRESULT WaitForRenderTime();
```



## <a name="parameters"></a>Параметры

Этот метод не имеет параметров.

## <a name="return-value"></a>Возвращаемое значение

Возвращает одно из следующих значений **HRESULT** .



| Код возврата                                                                                           | Описание                                                               |
|-------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>                  | Успешно.<br/>                                                       |
| <dl> <dt>**\_ \_ изменение состояния VFW \_ E**</dt> </dl> | Состояние фильтра изменилось до того, как было получено время презентации.<br/> |



 

## <a name="remarks"></a>Комментарии

Этот метод ожидает, пока не произойдет одно из следующих событий:

-   Получено время презентации в примере, после чего образец можно визуализировать.
-   Фильтр останавливает или начинает запись данных.

При наступлении времени презентации отображается событие [**кбасерендерер:: m \_ рендеревент**](cbaserenderer-m-renderevent.md) . При изменении состояния происходит сигнал о событии [**кбасерендерер:: m \_ среадсигнал**](cbaserenderer-m-threadsignal.md) . Этот метод ожидает оба события. Производный класс может переопределить этот метод для ожидания дополнительных событий, если это необходимо.

Этот метод вызывает метод [**кбасерендерер:: онваитстарт**](cbaserenderer-onwaitstart.md) , когда начинается ожидание, и метод [**Кбасерендерер:: онваитенд**](cbaserenderer-onwaitend.md) по завершении ожидания. Ни один из методов не выполняет никаких действий в базовом классе, но производный класс может их переопределить.
