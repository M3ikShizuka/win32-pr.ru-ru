# <a name="cbaseoutputpin-class"></a>Класс Кбасеаутпутпин

![Иерархия классов кбасеаутпутпин](images/filter06.png)

`CBaseOutputPin`Класс является абстрактным базовым классом, который реализует выходной ПИН-код.

Этот класс является производным от [**кбасепин**](cbasepin.md). Он отличается от **кбасепин** в следующих отношениях:

-   Он подключается только к входным ПИН-данным, поддерживающим интерфейс [**имеминпутпин**](/windows/desktop/api/Strmif/nn-strmif-imeminputpin) .
-   Он поддерживает транспорт локальной памяти через интерфейс [**имемаллокатор**](/windows/desktop/api/Strmif/nn-strmif-imemallocator) .
-   Он отклоняет уведомления конца потока, очистки и создания сегментов. (Они не должны отправляться на выходной ПИН-код.)
-   Он предоставляет методы для доставки образцов в нисходящем направлении.

Когда ПИН-код подключается, он запрашивает распределитель памяти из входного ПИН-кода. При этом создается новый объект распределителя. Закрепление вывода отвечает за настройку свойств распределителя. Это достигается с помощью чистого виртуального метода [**кбасеаутпутпин::D еЦидебуфферсизе**](cbaseoutputpin-decidebuffersize.md). Переопределите этот метод в производном классе. Если входной ПИН-код имеет любые требования к буферу, они передаются в метод **деЦидебуфферсизе** .

Вызовите метод [**кбасеаутпутпин:: жетделиверибуффер**](cbaseoutputpin-getdeliverybuffer.md) , чтобы получить пустой пример носителя. Вызовите метод [**кбасеаутпутпин::D еливер**](cbaseoutputpin-deliver.md) , чтобы доставлять образцы в нисходящем направлении.

Производный класс должен переопределять чистый виртуальный метод [**кбасепин:: чеккмедиатипе**](cbasepin-checkmediatype.md) для проверки типа носителя во время подключения ПИН-кода.
