# <a name="cbasepinqueryinternalconnections-method"></a>Кбасепин. Куеринтерналконнектионс, метод

`QueryInternalConnections`Метод получает контакты, которые подключены внутренне к этому ПИН-коду (в рамках фильтра). Этот метод реализует метод [**Ипин:: куеринтерналконнектионс**](/windows/desktop/api/Strmif/nf-strmif-ipin-queryinternalconnections) .

## <a name="syntax"></a>Синтаксис


```C++
HRESULT QueryInternalConnections(
   IPin  *apPin,
   ULONG *nPin
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Списке* 
</dt> <dd>

Адрес массива указателей [**Ипин**](/windows/desktop/api/Strmif/nn-strmif-ipin) .

</dd> <dt>

*нпин* 
</dt> <dd>

В поле входные данные задает размер массива. Когда метод возвращает значение, ему присваивается число указателей, возвращаемых в массиве.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает одно из значений **HRESULT** , приведенных в следующей таблице.



| Код возврата                                                                               | Описание                         |
|-------------------------------------------------------------------------------------------|-------------------------------------|
| <dl> <dt>**S \_ false**</dt> </dl>   | Недостаточный размер массива.<br/> |
| <dl> <dt>**\_ОК**</dt> </dl>      | Успешно.<br/>                 |
| <dl> <dt>**\_Ошибка E**</dt> </dl>    | Ошибка.<br/>                 |
| <dl> <dt>**E \_ нотимпл**</dt> </dl> | Не реализован.<br/>         |



 

## <a name="remarks"></a>Комментарии

В некоторых фильтрах входные ПИН-коды соответствуют определенным выходным закреплениям. Для каждого ПИН-кода этот метод заполняет массив указателями на соответствующие сигналы. Если каждый входной ПИН-код предоставляет данные для каждого выходного контакта, возвращается E \_ нотимпл.
