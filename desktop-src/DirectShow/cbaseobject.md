# <a name="cbaseobject-class"></a>Класс Кбасеобжект

класс **кбасеобжект** является абстрактным классом для реализации DirectShowных объектов. Для реализации объектов модели COM используйте класс [**кункновн**](cunknown.md) , производный от **кбасеобжект**.



| Методы класса                                      | Описание                            |
|----------------------------------------------------|----------------------------------------|
| [**кбасеобжект**](cbaseobject-cbaseobject.md)     | Метод конструктора.                    |
| [**~ Кбасеобжект**](cbaseobject--cbaseobject.md)   | Метод деструктора.                     |
| [**обжектсактиве**](cbaseobject-objectsactive.md) | Возвращает число активных объектов. |



 

## <a name="remarks"></a>Remarks

большая часть DirectShow базовых классов является производной от **кбасеобжект**. этот класс предоставляет помощь при отладке, сохраняя количество всех DirectShow объектов, активных во время выполнения. Число объектов хранится в переменной статического члена класса:


```
class CBaseObject
{
private:
    static LONG m_cObjects;  // Total number of objects active. 
/* ... */
};
```



В отладочных сборках библиотека DLL будет утверждать, если она выгружается, пока число объектов больше нуля. Это упрощает мониторинг утечек, вызванных проблемами подсчета ссылок.

Конструктор **кбасеобжект** принимает один аргумент — имя отладки для объекта. Это имя хранится в глобальной таблице в библиотеке DLL. Функция [**дбгдумпобжектрегистер**](dbgdumpobjectregister.md) форматирует список объектов, активных в библиотеке DLL, и отправляет их в выходные данные отладки.
