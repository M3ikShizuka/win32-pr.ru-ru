# <a name="cpullpin-class"></a>Класс Кпуллпин

![Иерархия классов кпуллпин](images/pulpin01.png)

`CPullPin`Класс обеспечивает поддержку входных ПИН-кодов, которые извлекают данные через интерфейс [**иасинкреадер**](/windows/desktop/api/Strmif/nn-strmif-iasyncreader) . Используйте этот класс, если реализуется фильтр, использующий модель извлечения для запроса данных из вышестоящего фильтра. дополнительные сведения см. в разделе Flow данных в Graphе фильтров и в модели извлечения.

Этот класс не является производным от **кбасепин** или реализует интерфейс [**Ипин**](/windows/desktop/api/Strmif/nn-strmif-ipin) , а некоторые имена методов конфликтуют с **Ипин**, поэтому его лучше использовать в качестве вспомогательного объекта в ПИН-коде. Чтобы использовать этот класс, выполните следующие действия.

1.  Создайте производный вспомогательный класс от `CPullPin` класса и производные от **кбасепин** классом входного ПИН-кода. Объявите экземпляр `CPullPin` объекта в качестве переменной-члена класса закрепления.
2.  переопределите метод [**кбасепин:: чеккконнект**](cbasepin-checkconnect.md) , чтобы вызвать [**кпуллпин:: Подключение**](cpullpin-connect.md). Этот метод запрашивает другой ПИН-код для **иасинкреадер**.
3.  Переопределите метод [**кбасепин:: бреакконнект**](cbasepin-breakconnect.md) , чтобы вызвать [**кпуллпин::D соединения**](cpullpin-disconnect.md).
4.  Переопределите метод [**кбасепин:: Active**](cbasepin-active.md) , чтобы вызвать [**Кпуллпин:: Active**](cpullpin-active.md). Этот метод запускает рабочий поток, который извлекает образцы из вышестоящего фильтра. При подключении к контактам можно указать, должен ли рабочий поток выполнять асинхронные или синхронные запросы на чтение.
5.  Переопределите метод [**кбасепин:: Inactive**](cbasepin-inactive.md) для вызова [**Кпуллпин:: Inactive**](cpullpin-inactive.md). Этот метод завершает работу рабочего потока.
6.  Реализуйте чистый виртуальный метод [**кпуллпин:: Receive**](cpullpin-receive.md) для обработки входящих образцов и их доставки в нисходящий.
7.  Чтобы задать координаты начала и окончания или найти поток, вызовите метод [**кпуллпин:: Seek**](cpullpin-seek.md) . Этот метод приостанавливает рабочий поток и очищает граф фильтра.
8.  Реализуйте чистые виртуальные [**кпуллпин:: EndOfStream**](cpullpin-endofstream.md), [**Кпуллпин:: бегинфлуш**](cpullpin-beginflush.md)и [**кпуллпин:: ендфлуш**](cpullpin-endflush.md) , как описано в комментариях к этим методам.
9.  Реализуйте чистый виртуальный метод [**кпуллпин:: OnError**](cpullpin-onerror.md) для управления ошибками потоковой передачи.
