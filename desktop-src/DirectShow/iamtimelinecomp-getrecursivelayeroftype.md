# <a name="iamtimelinecompgetrecursivelayeroftype-method"></a>Метод Иамтимелинекомп:: Жетрекурсивелайерофтипе

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`GetRecursiveLayerOfType`Метод выполняет упорядочение по глубине виртуальных дорожек, содержащихся в этой композиции, и извлекает из этого упорядочения *n*-й виртуальный объект Track.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetRecursiveLayerOfType(
  [out] IAMTimelineObj      **ppVirtualTrack,
        long                WhichLayer,
        TIMELINE_MAJOR_TYPE Type
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*ппвиртуалтракк* \[ заполняет\]
</dt> <dd>

Получает указатель на интерфейс [**иамтимелинеобж**](iamtimelineobj.md) виртуальной записи.

</dd> <dt>

*вхичлайер* 
</dt> <dd>

Указывает, какую виртуальную дорожку следует получить, проиндексировать из нуля.

</dd> <dt>

*Тип* 
</dt> <dd>

Член перечислимого [**типа \_ \_ временной шкалы**](timeline-major-type.md) , указывающий, следует ли включать в поиск записи.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает одно из следующих значений **HRESULT** :



| Код возврата                                                                                  | Описание                                 |
|----------------------------------------------------------------------------------------------|---------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>         | Успешно.<br/>                         |
| <dl> <dt>**E \_ INVALIDARG**</dt> </dl> | Нет объекта указанного типа.<br/> |
| <dl> <dt>**\_указатель E**</dt> </dl>    | **Пустой** аргумент указателя.<br/>       |



 

## <a name="remarks"></a>Комментарии

Как правило, приложению не требуется вызывать этот метод.

Если параметр *типа* относится к \_ ОСНОВНОМУ типу временной шкалы \_ \_ , то упорядочение по глубине включает в себя дорожки. В противном случае он включает только композиции и группы. Сам объект включается в упорядочение.

Например, в следующем расположении, начиная с композиции а, упорядочение будет иметь вид B, C, F, D, E, A.

![жетрекурсивелайерофтипе](images/composition.png)

Если метод завершается успешно, интерфейс **иамтимелинеобж** , который он возвращает, имеет необработанный счетчик ссылок. Не забудьте освободить интерфейс по завершении его использования.
