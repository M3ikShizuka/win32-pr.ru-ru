# <a name="new-segments"></a>Новые сегменты

*Сегмент* — это группа образцов мультимедиа, использующих общее время начала, время окончания и скорость воспроизведения. Метод [**Ипин:: невсегмент**](/windows/desktop/api/Strmif/nf-strmif-ipin-newsegment) сигнализирует о начале нового сегмента. Он позволяет исходному фильтру уведомлять нисходящие фильтры о том, что сведения о времени и частоте изменились. Например, если фильтр источника выполняет поиск новой точки в потоке, он вызывает **невсегмент** с новым временем начала.

Некоторые нисходящие фильтры используют сведения о сегменте при обработке образцов. Например, в формате, использующем сжатие по кадрам, если время окончания попадает в разностный кадр, фильтру источника может потребоваться отправить дополнительные выборки после времени окончания. Это позволяет декодеру декодировать окончательный разностный кадр. Чтобы определить правильный окончательный кадр, декодер обозначает время окончания сегмента. В качестве другого примера можно использовать частоту сегментов, а также частоту дискретизации для создания правильных звуковых выходных данных.

В модели отправки фильтр источника инициирует вызов **невсегмент** . В модели извлечения это выполняется фильтром анализатора. В любом случае фильтр вызывает **невсегмент** для входного ПИН-кода, который распространяет вызов к следующему фильтру, пока вызов не достигнет модуля подготовки отчетов. Фильтры должны сериализовать вызовы **невсегмент** с другими вызовами потоковой передачи, например [**Имеминпутпин:: Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive).

Время потока сбрасывается в ноль после каждого нового сегмента. Метки времени для выборок, доставленных после начала сегмента с нуля.
