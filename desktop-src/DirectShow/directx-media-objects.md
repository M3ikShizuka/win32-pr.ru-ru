# <a name="directx-media-objects"></a>Объекты мультимедиа DirectX

> [!Note]  
> Дмос были заменены [Media Foundation преобразованиями](/windows/desktop/medfound/media-foundation-transforms) (МФТС). Интерфейсы DMO по-прежнему поддерживаются. Однако при написании пользовательского кодека или подключаемого модуля обработки звука и видео следует рассмотреть возможность его реализации как MFT.

 

Объекты мультимедиа DirectX (дмос) — это компоненты потоковой передачи данных на основе COM. В некоторых отношениях дмос похожи на фильтры Microsoft DirectShow. Как и фильтры DirectShow, дмос принимает входные данные и использует их для создания выходных данных. Однако прикладные программные интерфейсы (API) для дмос гораздо проще, чем соответствующие API-интерфейсы для DirectShow. В результате дмос проще в создании, тестировании и использовании. Дмос можно использовать во многих сценариях:

-   Приложения, основанные на DirectShow, могут использовать дмос через фильтр DirectShow, называемый фильтром [оболочки DMO](dmo-wrapper-filter.md) . Различие между фильтрами и дмос является прозрачным для приложения. Приложение не вызывает напрямую интерфейсы API DMO.
-   Приложения, основанные на Microsoft DirectSound, могут использовать звуковые эффекты дмос. Опять же, приложение изолируется от низкоуровневых API DMO с интерфейсами API DirectSound более высокого уровня.
-   Приложения могут использовать дмос напрямую.

Таким же путем написания DMO вы создаете компонент, который можно использовать в широком спектре приложений. Эта документация содержит следующие разделы:
