# <a name="cbasecontrolwindowget_owner-method"></a>Кбасеконтролвиндов. Get, \_ метод Owner

`get_Owner`Метод получает текущего владельца окна.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT get_Owner(
   OAHWND *Owner
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Владелец* 
</dt> <dd>

Указатель на владельца окна.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение **HRESULT** .

## <a name="remarks"></a>Комментарии

Окно видео может воспроизводиться в среде документа. Для этого окно должно быть дочерним для другого окна (так что оно обрезается и перемещается соответствующим образом). Это свойство позволяет устанавливать и извлекать владелец окна. Если окно принадлежит другому окну, оно просто вызывает функцию Microsoft Win32 **сетпарент** . Приложение, вызывающее эту функцию, изменит стили окна, чтобы установить \_ дочерний бит WS для.

Когда окно владеет другим окном, оно автоматически пересылает определенные наборы сообщений (в частности, в сообщениях мыши и клавиатуры). Это позволяет приложению выполнять простые изменения в режиме горячей замены и другие взаимодействия.

Эта функция-член предназначена для вызова внешними объектами через интерфейс [**ивидеовиндов**](/windows/desktop/api/Control/nn-control-ivideowindow) и, следовательно, блокирует критический раздел для синхронизации с соответствующим фильтром. Вызовите функцию члена [**кбасеконтролвиндов:: жетовнервиндов**](cbasecontrolwindow-getownerwindow.md) , чтобы получить это свойство, если не вызывается из внешнего объекта.
