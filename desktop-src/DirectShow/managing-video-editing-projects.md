# <a name="managing-video-editing-projects"></a>Управление проектами редактирования видео

\[Этот API не поддерживается и может быть изменен или недоступен в будущем.\]

Следующие советы помогут вам управлять проектами в [службах редактирования DirectShow](directshow-editing-services.md).

Изменения в временной шкале

-   При изменении временной шкалы после построения графа фильтра вызовите [**ирендеренгине:: коннектфронтенд**](irenderengine-connectfrontend.md) еще раз, чтобы перестроить внешний интерфейс. Как правило, это не влияет на оставшуюся часть графа. Однако иногда перед перестроением внешнего интерфейса модулю подготовки отчетов необходимо удалить всю диаграмму. (Например, это происходит при добавлении или удалении группы.) Метод **коннектфронтенд** возвращает параметр S \_ warn \_ аутпутресет, чтобы сообщить, что он удалил граф. В этом случае приложение должно перестроить раздел отрисовки графа.
-   Чтобы полностью удалить все объекты из временной шкалы, вызовите метод [**иамтимелине:: клеараллграупс**](iamtimeline-clearallgroups.md) .

**Очистка**

-   По завершении использования подсистемы визуализации вызовите метод [**ирендеренгине:: скрапит**](irenderengine-scrapit.md) . Как и в случае с любым COM-объектом, не забудьте освободить каждый указатель интерфейса по окончании его использования.
-   Механизм визуализации не сохраняет счетчик ссылок на временной шкале. Не освобождайте временную шкалу до завершения ее использования и всегда вызывайте **скрапит** в подсистеме рендеринга первыми.
-   При освобождении всех ссылок на временную шкалу не используйте объекты в этой временной шкале, даже если на них хранятся счетчики ссылок.

**Несколько экземпляров временной шкалы**

-   Не переносите объекты временной шкалы между временными шкалами. Каждый объект на временной шкале должен быть создан этой временной шкалой. Временная шкала содержит внутренний кэш со сведениями о создаваемых им объектах. Перемещение объектов временной шкалы может привести к нарушению работы кэша.
-   Никогда не используйте один и тот же экземпляр механизма визуализации с несколькими временными шкалами. Модуль подготовки отчетов содержит кэш со сведениями о временной шкале. Несколько временных шкал нарушит кэш и приведут к непредсказуемым результатам. Если требуются две активные временные шкалы, создайте отдельные экземпляры модулей подготовки отчетов для каждой временной шкалы.
-   Временная шкала может использовать более одного механизма рендеринга, но не одновременно. Удалите старый обработчик прорисовки перед использованием другого механизма рендеринга. (Обычно это делается при переключении с использования базового механизма рендеринга для предварительного просмотра в модуль интеллектуального преобразования для записи файлов.)

**Сохраняемость**

-   Граф фильтра не является постоянным при сохранении проекта в XML-файл. Таким образом, теряются все сведения, относящиеся к смарт-сжатию, формату сжатия или параметрам сжатия. Приложение может восстановить эти параметры после загрузки проекта.
