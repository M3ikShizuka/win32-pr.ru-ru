# <a name="iamtimelinecompgetcountoftype-method"></a>Метод Иамтимелинекомп:: Жеткаунтофтипе

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`GetCountOfType`Метод извлекает количество объектов заданного типа, содержащихся в этой композиции, и всех виртуальных дорожках, рекурсивно.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetCountOfType(
   long                *pVal,
   long                *pValWithComps,
   TIMELINE_MAJOR_TYPE MajorType
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*pVal* 
</dt> <dd>

Получает число объектов указанного типа, содержащихся в этой композиции и всех виртуальных дорожках, рекурсивно.

</dd> <dt>

*пвалвискомпс* 
</dt> <dd>

Получает счетчик, возвращенный в *Pval* , и число композиций, в которых выполняется поиск, включая это.

</dd> <dt>

*мажортипе* 
</dt> <dd>

Элемент перечисляемого типа " [**\_ основной \_ тип временной шкалы**](timeline-major-type.md) ", указывающий тип объекта для подсчета.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает \_ значение ОК, если успешно, или \_ указатель E в противном случае.

## <a name="remarks"></a>Комментарии

Как правило, приложение не будет вызывать этот метод. Он вызывается модулем визуализации.

При подсчете композиций значение, возвращаемое в *Pval* , равно нулю, а значение, возвращаемое в *пвалвискомпс* , — число композиций. Значение *\* пвалвискомпс* включает композицию, для которой вызывается метод. Например, при вызове этого метода для пустой композиции *\* пвалвискомпс* равно 1.

Группы не могут находиться внутри композиций, поэтому этот метод нельзя использовать для подсчета групп. (Возвращенное число всегда будет равно нулю.) Для подсчета групп вызовите метод [**иамтимелине:: жетграупкаунт**](iamtimeline-getgroupcount.md) .
