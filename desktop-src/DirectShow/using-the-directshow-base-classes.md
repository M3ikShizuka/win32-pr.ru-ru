# <a name="using-the-directshow-base-classes"></a>использование базовых классов DirectShow

чтобы использовать базовые классы в DirectShow, необходимо создать и связать библиотеку базовых классов.

библиотека базовых классов предоставляется в качестве примера пакета sdk в пакете средств разработки программного обеспечения (sdk) Microsoft Windows <https://go.microsoft.com/fwlink/p/?linkid=62332> . Точное расположение зависит от версии установленного пакета SDK, но относительный путь:

*(Корень примеров пакета SDK)* \\ DirectShow \\ басеклассес

заголовок: Потоки. h

Библиотека. Пример сборки розничной и отладочной версий библиотеки:

-   Розничная версия: Стрмбасе. lib
-   Отладочная версия: Стрмбасд. lib.

Дополнительные сведения о настройке среды сборки см. в разделе [Настройка среды сборки](setting-up-the-build-environment.md).

## <a name="preprocessor-symbols"></a>Символы препроцессора

когда вы включаете заголовочный файл Потоки. h, следующие символы препроцессора имеют особое значение:

-   PERF: reserved. Не используйте этот символ препроцессора.
-   ВФВРОБУСТ: включает проверку указателя в розницу. Дополнительные сведения см. в разделе [макросы проверки указателя](pointer-validation-macros.md). В отладочных сборках необязательно определять ВФВРОБУСТ.

> [!Note]  
> в Windows Vista и более поздних версий макросы проверки указателя пусты.
