# <a name="iamerrorloglogerror-method"></a>Метод Иамеррорлог:: LogError

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

Метод **LogError** записывает ошибку в журнал. Приложениям не требуется вызывать этот метод. Он вызывается внутренним образом в ответ на ошибки отрисовки.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT LogError(
       LONG    Severity,
       BSTR    ErrorString,
       LONG    ErrorCode,
       HRESULT hresult,
  [in] VARIANT *pExtraInfo
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Уровень серьезности* 
</dt> <dd>

Зарезервировано. Не используется.

</dd> <dt>

*ErrorString* 
</dt> <dd>

Строковое значение, содержащее текст ошибки.

</dd> <dt>

*ErrorCode* 
</dt> <dd>

Код ошибки.

</dd> <dt>

*состав* 
</dt> <dd>

Значение HRESULT, возвращенное вызовом метода, вызвавшего ошибку.

</dd> <dt>

*пекстраинфо* \[ окне\]
</dt> <dd>

Указатель на вариант, содержащий любые дополнительные сведения об ошибке.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение параметра *HRESULT* .

## <a name="remarks"></a>Комментарии

В этом методе не следует освобождать **вариант** , на который указывает *пекстраинфо*. Кроме того, **вариант** станет недопустимым после возврата метода, поэтому не пытайтесь ссылаться на него позже.

Реализуйте этот метод, чтобы возвращать как можно быстрее. Не делайте вызовы функций внутри этого метода, которые могут препятствовать выполнению программы. Например, не вызывайте функции, которые отправляют сообщения в окне, блокируют события или иным образом могут блокировать выполнение. Это может привести к тому, что компьютер перестает отвечать на запросы.

Список ошибок, определенных DES, а также значение и тип данных **варианта** , на который указывает *пекстраинфо*, см. в разделе [ошибки отрисовки](rendering-errors.md).
