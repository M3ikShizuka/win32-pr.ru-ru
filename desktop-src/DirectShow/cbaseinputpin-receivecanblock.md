# <a name="cbaseinputpinreceivecanblock-method"></a>Кбасеинпутпин. Рецеивеканблокк, метод

`ReceiveCanBlock`Метод определяет, могут ли вызовы метода [**имеминпутпин:: Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive) блокироваться. Этот метод реализует метод [**имеминпутпин:: рецеивеканблокк**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receivecanblock) .

## <a name="syntax"></a>Синтаксис


```C++
HRESULT ReceiveCanBlock();
```



## <a name="parameters"></a>Параметры

Этот метод не имеет параметров.

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение **HRESULT** . Возможные значения включают перечисленные в следующей таблице.



| Код возврата                                                                             | Описание                                                 |
|-----------------------------------------------------------------------------------------|-------------------------------------------------------------|
| <dl> <dt>**S \_ false**</dt> </dl> | ПИН-код не будет блокироваться при вызове метода **Receive**.<br/> |
| <dl> <dt>**\_ОК**</dt> </dl>    | ПИН-код может блокироваться при вызове метода **Receive**.<br/>    |



 

## <a name="remarks"></a>Комментарии

Возвращает \_ значение false, если вызовы метода **Receive** гарантированно не блокируются. В противном случае возвратите значение S \_ OK или код ошибки. Если метод **Receive** вызывает **Получение** по нисходящему ПИН-коду, то он может блокироваться; `ReceiveCanBlock` необходимо учитывать этот фактор.

Вышестоящий фильтр может использовать этот метод для определения своей стратегии работы с потоками. Если метод **Receive** может блокироваться, вышестоящий фильтр может принять решение использовать рабочий поток, который помещает данные в буфер. Реализация этой стратегии приведена в описании класса [**каутпуткуеуе**](coutputqueue.md) .

В базовом классе этот метод возвращает \_ значение ОК, если выполняется любое из следующих условий.

-   Фильтр не имеет выходных закрепления.
-   Входной ПИН-код, подключенный к этому фильтру, сигнализирует, что он может блокироваться.
-   Входной ПИН-код, подключенный к этому фильтру, не поддерживает интерфейс [**имеминпутпин**](/windows/desktop/api/Strmif/nn-strmif-imeminputpin) .
