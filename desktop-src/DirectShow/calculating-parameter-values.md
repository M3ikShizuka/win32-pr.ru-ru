# <a name="calculating-parameter-values"></a>Вычисление значений параметров

Возможно, входной буфер может быть очень большим. В идеале, когда объект DMO обрабатывает буфер, параметры будут точно соответствовать их кривым по всему пакету данных. Однако у DMO есть некоторые отклонения в том, как он вычисляет эти значения.

-   Наиболее точный подход заключается в вычислении точного значения для каждой атомарной единицы данных. Например, каждый звуковой образец. Этот подход является наиболее ресурсоемким.
-   Другой подход состоит в том, чтобы разделить данные на более мелкие единицы фиксированного размера, например 100. При таком подходе создается «лестничное пошаговое выполнение». Для некоторых параметров это может быть приемлемым. В звуковых эффектах он может создавать звуковые артефакты.
-   Компромисс заключается в использовании предыдущей методики, но в каждом пакете выполняет линейную интерполяцию значения параметра для каждого образца.

Эти проблемы особенно важны для обработки звука. Одна секунда звука может содержать 48 000 звуковых выборок на канал, что является большим количеством вычислений, но с учетом таких артефактов, как присвоение псевдонимов.
