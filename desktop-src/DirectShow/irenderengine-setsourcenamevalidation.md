# <a name="irenderenginesetsourcenamevalidation-method"></a>Метод Ирендеренгине:: Сетсаурценамевалидатион

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`SetSourceNameValidation`Метод указывает, как подсистема подготовки отчетов проверяет имена файлов.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT SetSourceNameValidation(
   BSTR          FilterString,
   IMediaLocator *pOverride,
   LONG          Flags
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*филтерстринг* 
</dt> <dd>

Значение **BSTR** , содержащее пары строк фильтра, в формате, обязательном для элемента **лпстрфилтер** структуры [**OpenFileName**](/windows/win32/api/commdlg/ns-commdlg-openfilenamea) . Указатель мультимедиа использует этот фильтр, если он представляет собой диалоговое окно открытия файла для конечного пользователя.

</dd> <dt>

*поверриде* 
</dt> <dd>

Необязательный указатель на интерфейс [**имедиалокатор**](imedialocator.md) указателя мультимедиа, используемый вместо значения по умолчанию. Чтобы использовать указатель носителя по умолчанию, установите для этого параметра значение **null**. Дополнительные сведения см. в разделе "Примечания".

</dd> <dt>

*Флаги* 
</dt> <dd>

Побитовое сочетание [**флагов проверки имени файла**](file-name-validation-flags.md) , определяющее поведение указателя мультимедиа. \_ \_ Должен присутствовать флаг проверки СФН валидатеф. Флаг СФН \_ валидатеф \_ хлинкмутед не влияет на этот метод.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает одно из следующих значений **HRESULT** :



| Код возврата                                                                                            | Описание                                    |
|--------------------------------------------------------------------------------------------------------|------------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>                   | Успешно.<br/>                            |
| <dl> <dt>**E \_ должен \_ инициализировать модуль \_ подготовки отчетов**</dt> </dl> | Не удалось инициализировать модуль визуализации.<br/> |



 

## <a name="remarks"></a>Remarks

С помощью параметра *поверриде* можно предоставить собственную пользовательскую реализацию интерфейса [**имедиалокатор**](imedialocator.md) . Например, указатель мультимедиа по умолчанию не уведомляет приложение о найденных файлах (или не может его найти). Чтобы обойти это ограничение, можно реализовать пользовательский указатель мультимедиа, сделав его оболочкой для версии по умолчанию. Затем передайте [**имедиалокатор:: финдмедиафиле**](imedialocator-findmediafile.md) напрямую вызовет версию по умолчанию и изучите возвращаемое значение.

В настоящее время этот метод не проверяет динамически загружаемые источники. См. раздел [**ирендеренгине:: сетдинамикреконнектлевел**](irenderengine-setdynamicreconnectlevel.md).
