# <a name="directly-hosting-a-dmo"></a>Непосредственное размещение DMO

В этом разделе описывается, как приложение может работать в качестве непосредственного клиента DMO. Приложение предоставляет входные данные для DMO, объекты DMO создают выходные данные, а приложение использует выходные данные для отрисовки, дальнейшей обработки или что-либо еще. Приложение отвечает за такие проблемы, как выделение памяти, время и синхронизация, а также многопоточность. Эти требования будут зависеть от природы приложения.

Сведения в этом разделе также применяются при написании компонента, который выступает в качестве слоя между приложением и DMO (например, с элементом управления ActiveX, в котором размещен объект DMO). Кроме того, следует прочесть этот раздел, если вы пишете DMO, так как он описывает функциональные возможности, которые должен реализовать ваш объект DMO.
