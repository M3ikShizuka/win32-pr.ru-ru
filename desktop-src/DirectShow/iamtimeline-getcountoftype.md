# <a name="iamtimelinegetcountoftype-method"></a>Метод Иамтимелине:: Жеткаунтофтипе

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`GetCountOfType`Метод извлекает количество объектов указанного типа, содержащихся в указанной группе и всех ее дочерних элементах.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetCountOfType(
   long                Group,
   long                *pVal,
   long                *pValWithComps,
   TIMELINE_MAJOR_TYPE MajorType
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Группа* 
</dt> <dd>

Номер индекса группы, для которой необходимо получить число.

</dd> <dt>

*pVal* 
</dt> <dd>

Получает число объектов указанного типа, содержащихся в группе, и всех его виртуальных дорожек рекурсивно.

</dd> <dt>

*пвалвискомпс* 
</dt> <dd>

Получает счетчик, возвращенный в *Pval* , и число композиций, в которых выполняется поиск, включая это.

</dd> <dt>

*мажортипе* 
</dt> <dd>

Элемент перечисляемого типа " [**\_ основной \_ тип временной шкалы**](timeline-major-type.md) ", указывающий тип объекта для подсчета.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает одно из следующих значений **HRESULT** .



| Код возврата                                                                                  | Описание                           |
|----------------------------------------------------------------------------------------------|---------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>         | Успешно.<br/>                   |
| <dl> <dt>**E \_ INVALIDARG**</dt> </dl> | Недопустимый номер группы.<br/>      |
| <dl> <dt>**\_указатель E**</dt> </dl>    | **Пустой** аргумент указателя.<br/> |



 

## <a name="remarks"></a>Комментарии

Вызов этого метода эквивалентен вызову [**иамтимелинекомп:: жеткаунтофтипе**](iamtimelinecomp-getcountoftype.md) для указанной группы. Дополнительные сведения см. в разделе "Примечания" этого метода.

Как правило, приложение не будет вызывать этот метод. Он вызывается модулем визуализации внутри.
