# <a name="canstep-method"></a>Метод Канстеп

> [!Note]  
> этот компонент доступен для использования в операционных системах Microsoft Windows 2000, Windows XP и Windows Server 2003. В последующих версиях он может быть изменен или недоступен.

 

`CanStep`Метод определяет, может ли декодер MPEG-2 в локальной системе выполнять пошаговое выполнение кадра в указанном направлении.

``` syntax
[ bCanStep = ] MSWebDVD.CanStep(bDirection)
```

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="bDirection"></span><span id="bdirection"></span><span id="BDIRECTION"></span>*бдиректион*
</dt> <dd>

Логическое значение, используемое в качестве флага для указания направления (вперед или назад) возможностей пошагового выполнения декодера.



| Значение | Описание                                          |
|-------|------------------------------------------------------|
| Да  | Можно ли дедекодерировать шаг назад?                           |
| false | Можно ли отдекодеровать шаг вперед? Это значение по умолчанию. |



 

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает логическое значение true, если декодер может выполнить шаг в указанном направлении, и false в противном случае.

## <a name="remarks"></a>Remarks

не все аппаратные декодеры MPEG-2 поддерживают новые возможности пошагового выполнения пакетов в DirectShow® 8,0. Этот метод запрашивает у декодера возможности пошагового выполнения кадра. Если декодер может выполнять пошаговое выполнение, приложение может использовать метод [**Step**](step-method.md) для пошагового перехода к кадрам. Этот метод всегда будет возвращать **значение true** , если используется программный декодер.
