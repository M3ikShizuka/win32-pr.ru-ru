# <a name="irenderengine-interface"></a>Интерфейс Ирендеренгине

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`IRenderEngine`Интерфейс визуализирует проект [служб редактирования DirectShow](directshow-editing-services.md) (DES), создавая граф фильтра из временной шкалы.

DES предоставляет два компонента, которые реализуют этот интерфейс:

-   *Базовый механизм визуализации* создает несжатые выходные данные. Вы можете использовать выходные данные для предварительного просмотра или направить его с помощью фильтров сжатия и записать в файл.
-   *Интеллектуальный модуль визуализации* создает сжатые выходные данные, используя *интеллектуальное* повторное сжатие. При использовании интеллектуального сжатия исходный файл сжимается только в том случае, если его формат отличается от формата выходных данных. Источник с соответствующим форматом записывается непосредственно в выходной файл. В зависимости от сценария интеллектуальное повторное сжатие может значительно повысить время визуализации.

Модуль интеллектуального просмотра также поддерживает интерфейс [**исмартрендеренгине**](ismartrenderengine.md) .

Хотя приложение может создать граф фильтра и передать его в механизм визуализации, типичным сценарием является создание графа фильтра модулем визуализации. Построение графа является процессом, сооснованным на двух стадиях. Сначала создайте интерфейс, вызвав метод [**ирендеренгине:: коннектфронтенд**](irenderengine-connectfrontend.md) . Затем подключите выходные контакты на внешнем интерфейсе к нужным фильтрам отрисовки:

-   Модули подготовки видео и звука для предварительного просмотра или
-   Программы сжатия, мультиплексоры и записи файлов для создания окончательных выходных данных.
