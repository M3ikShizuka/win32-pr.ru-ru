# <a name="dv-splitter-filter"></a>Фильтр разделителя DV

Этот фильтр разделяет поток цифрового видео (DV) с чередованием в его компонентах видео и звуковых потоков.

## <a name="remarks"></a>Remarks

Кадры DV содержат аудио и видео в одном кадре. Фильтр разделителя DV извлекает звуковые данные и доставляет их в виде одного или двух звуковых потоков из ПИН-кодов звука. Исходный DV-кадр доставляется из ПИН-кода вывода видео в виде видеокадра. Тип носителя в кадре видеозаписи изменяется с носителя с \_ чередованием на Mac \_ , но в противном случае данные не изменяются. Тип носителя изменится на сигнал о том, что звуковые данные в кадре следует игнорировать. Разделитель DV не задает время носителя в выходных примерах; Если вы создаете нисходящий фильтр, для которого требуется время работы с мультимедиа, то можете наследовать значения времени от числа кадров.

Только один выходной закрепление за раз предоставляет интерфейсы [**имедиапоситион**](/windows/desktop/api/Control/nn-control-imediaposition) и [**имедиасикинг**](/windows/desktop/api/Strmif/nn-strmif-imediaseeking) .

Фильтр разделителя DV может принимать изменения динамического формата в потоке аудио. Однако если фильтр [мультиплексора AVI](avi-mux-filter.md) является нисходящим, изменение формата будет отклонено. В этом случае разделитель DV останавливает воспроизведение аудиопотока. Это ограничение влияет только на запись файлов типа 2. Для файлов типа 1 поток с чередованием не разбивается в первую очередь. В предварительной версии фильтр мультиплексора AVI не является подчиненным.

Если источником DV является Динамическая камера, обычно нет причин, по которым формат аудио не изменится. Однако формат может измениться при передаче с ленты Втр, которая содержит несколько разнородных источников.

В дополнение к данным аудио и видео в каждом кадре DV содержатся метаданные. Эти метаданные могут изменяться от кадра к кадру. Приложения могут анализировать метаданные путем проверки входных образцов или видеороликов. однако DirectShow не предоставляет прямой поддержки для синтаксического анализа метаданных DV. Дополнительные сведения см. в IEC 61834-4.
