# <a name="cim_sparegroup-class"></a>\_Класс CIM спареграуп

Класс **CIM \_ спареграуп** является производным от класса [**CIM \_ редунданциграуп**](cim-redundancygroup.md) и указывает, что один или несколько агрегированных элементов могут быть запасными. Запасы определяются с помощью ассоциации [**CIM \_ актсасспаре**](cim-actsasspare.md) . Примером запасного диска является использование избыточных сетевых интерфейсов в компьютерной системе, где одна сетевая карта является основной, а другая — запасной. Основная сетевая карта будет членом группы запчастей, связанной с использованием класса [**CIM \_ редунданцикомпонент**](cim-redundancycomponent.md) , а другая сетевая карта будет связана с использованием отношения **CIM \_ актсасспаре** .

> [!IMPORTANT]
> Классы CIM (модель CIM) в DMTF (распределенная задача управления) являются родительскими классами, на которых строятся классы WMI. В настоящее время WMI поддерживает только [схемы версии CIM 2. x](https://dmtf.org/standards/cim/schemas).

 

Следующий синтаксис упрощен из кода MOF-файл (MOF) и включает все его унаследованные свойства. Свойства перечислены в алфавитном порядке, а не в MOF.

## <a name="syntax"></a>Синтаксис

``` syntax
[Abstract, UUID("{64B86CA6-E3D1-11d2-8601-0000F8102E5F}"), AMENDMENT]
class CIM_SpareGroup : CIM_RedundancyGroup
{
  string   Caption;
  string   Description;
  datetime InstallDate;
  string   Name;
  string   Status;
  string   CreationClassName;
  uint16   RedundancyStatus;
};
```
