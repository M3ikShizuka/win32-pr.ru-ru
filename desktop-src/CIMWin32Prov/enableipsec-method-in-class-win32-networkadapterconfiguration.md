# <a name="enableipsec-method-of-the-win32_networkadapterconfiguration-class"></a>Метод Енаблеипсек \_ класса Win32 NetworkAdapterConfiguration

Метод  [класса WMI](/windows/desktop/WmiSdk/retrieving-a-class) енаблеипсек включает протокол IPsec на сетевом адаптере с поддержкой TCP/IP.

В этом разделе используется синтаксис MOF-файл (MOF). Дополнительные сведения об использовании этого метода см. [в разделе вызов метода](/windows/desktop/WmiSdk/calling-a-method).

## <a name="syntax"></a>Синтаксис


```mof
uint32 EnableIPSec(
  [in] string IPSecPermitTCPPorts[],
  [in] string IPSecPermitUDPPorts[],
  [in] string IPSecPermitIPProtocols[]
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Ипсекпермитткппортс* \[ окне\]
</dt> <dd>

Список портов, которым предоставляется разрешение на доступ для TCP. Числовое значение 0 (ноль) означает, что разрешение на доступ предоставляется для всех портов. Пустой массив указывает, что нет ни одного порта, которым предоставляется разрешение на доступ.

</dd> <dt>

*Ипсекпермитудппортс* \[ окне\]
</dt> <dd>

Список портов, которым предоставляется разрешение на доступ к UDP. Числовое значение 0 (ноль) означает, что разрешение на доступ предоставляется для всех портов. Пустой массив указывает, что нет ни одного порта, которым предоставляется разрешение на доступ.

</dd> <dt>

*Ипсекпермитиппротоколс* \[ окне\]
</dt> <dd>

Список протоколов, разрешенных для запуска по IP-адресу. Числовое значение 0 (ноль) означает, что разрешение на доступ предоставляется для всех протоколов. Пустой массив указывает, что нет протоколов, которым предоставлено разрешение на доступ.

</dd> </dl>

Дополнительные сведения о кодах ошибок см. в разделе [**константы WMI Error**](/windows/desktop/WmiSdk/wmi-error-constants) или [**вбемерроренум**](/windows/desktop/api/wbemdisp/ne-wbemdisp-wbemerrorenum).
