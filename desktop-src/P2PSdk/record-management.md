# <a name="record-management"></a>Управление записями

Для управления записями можно работать со сведениями, хранящимися в структурах [**одноранговых \_ записей**](/windows/desktop/api/P2P/ns-p2p-peer_record) . При создании, обновлении или удалении записей изменения, внесенные в граф, реплицируются на все узлы. В следующей таблице приведены правила обновления и автоматического обновления записей.



| Задача управления     | Правило                                                                                                                                                                                                            |
|---------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Обновление записи   | Не изменяйте время истечения срока действия или увеличьте его. Нельзя уменьшить срок действия.                                                                                                                      |
| Обновление записи | Используйте флаг [**\_ \_ \_ автообновления флага одноранговой записи**](/windows/desktop/api/P2P/ns-p2p-peer_record) для автоматического обновления записи. Этот флаг следует использовать только в том случае, если узел, публикующий запись, находится в режиме «в сети». В противном случае не используйте этот флаг. |
