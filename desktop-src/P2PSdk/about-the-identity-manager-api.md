# <a name="about-identity-manager"></a>О программе Identity Manager

API однорангового диспетчера удостоверений позволяет создавать и перечислять одноранговые удостоверения в одноранговых приложениях, а также управлять ими. Вы можете использовать удостоверения одноранговых узлов, созданные с помощью этого API, в качестве входных данных для API поставщика пространства имен однорангового группирования и PNRP (Peer Name Resolution Protocol).

## <a name="peer-identity-manager-best-practices"></a>Рекомендации для диспетчера одноранговых удостоверений

Каждый пользователь должен иметь несколько одноранговых удостоверений, которые могут использоваться для одноранговых действий. Например, у пользователя может быть два одноранговых удостоверения для работы и свободного использования.

Хорошо спроектированное одноранговое приложение позволяет пользователю выбрать одноранговый идентификатор для использования. Пользователь может создать удостоверение однорангового узла, если ни одно из существующих одноранговых идентификаторов не подходит для использования с приложением.

Хорошо спроектированное однорангическое приложение также управляет количеством создаваемых удостоверений. Если приложение создает временный одноранговый узел, приложение должно удалить удостоверение однорангового узла, если удостоверение не требуется. Если приложение не выполняет это обслуживание, диспетчер одноранговых удостоверений может не иметь возможности создавать одноранговые удостоверения, пока не будут удалены некоторые идентификаторы одноранговых узлов.
