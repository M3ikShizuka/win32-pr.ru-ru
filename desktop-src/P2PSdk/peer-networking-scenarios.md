# <a name="peer-networking-scenarios"></a>Сценарии одноранговых сетей

Одноранговая инфраструктура предлагает множество вариантов разработки приложений одноранговых сетей. В этом разделе содержатся сведения о следующих сценариях одноранговой сети.

-   Совместное работа с приложениями
-   Приложения для обмена мгновенными сообщениями или чат
-   Приложения для распространения содержимого

## <a name="collaborative-applications"></a>Совместное работа с приложениями

Одноранговую инфраструктуру можно использовать для создания приложений совместной работы, которые позволяют пользователям интерактивно обмениваться идеями, независимо от того, находятся ли они в одном конференц-зале или через Интернет. Например, можно создать общее приложение, похожее на доску.

> [!Note]  
> Одноранговая инфраструктура не поддерживает потоковую передачу аудио или видео.

 

## <a name="instant-messaging-or-chat-applications"></a>Приложения для обмена мгновенными сообщениями или чат

Приложения для обмена мгновенными сообщениями и чат, использующие центральные серверы, можно обновить для использования одноранговой инфраструктуры. Например, приложение разговора может использовать одноранговую инфраструктуру в качестве альтернативы централизованному серверу. Одноранговая инфраструктура улучшает масштабируемость и позволяет пользователям участвовать в разговоре и обмене сообщениями без сервера.

## <a name="content-distribution-applications"></a>Приложения для распространения содержимого

Одноранговую инфраструктуру можно использовать в качестве механизма распространения содержимого. Например, можно распространить информационный бюллетень на коллеги, которые подписываются на службу. Поскольку данные в одноранговой инфраструктуре синхронизируются, подписчики получают информационный бюллетень, даже если издатель не находится в режиме «в сети».
