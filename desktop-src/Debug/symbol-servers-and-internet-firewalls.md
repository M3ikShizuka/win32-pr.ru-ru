# <a name="symbol-server-and-internet-firewalls"></a>Сервер символов и брандмауэры Интернета

Некоторые системы используют брандмауэры Интернета или прокси-серверы, требующие проверки подлинности для всего трафика Интернета. Ранние версии сервера символов не смогли получить доступ к символам из Интернета, если только система не использовала клиент брандмауэра, который обработал проверку подлинности прозрачно.

Начиная с DBGHELP 6,1 сервер символов поддерживает прокси-серверы, требующие такой проверки подлинности. Сервер символов использует любой сервер, настроенный в качестве значения по умолчанию в параметрах локальной сети компьютера. Чтобы найти его, откройте элемент " **Свойства обозревателя** " на панели управления, перейдите на вкладку " **подключения** " и нажмите кнопку " **Параметры локальной сети**". Это также можно сделать в Internet Explorer, выбрав пункт **Свойства обозревателя** в меню **Сервис** . Сервер символов протестирован на многих торговых марках прокси-серверов с использованием методов проверки подлинности "базовый" и "запрос — ответ".

Чтобы определить конкретный прокси-сервер для использования с сервером символов, задайте \_ \_ \_ для переменной среды прокси-сервера символов NT имя (или IP-адрес) прокси-сервера, за которым следует номер порта. Разделите эти два значения двоеточием. Пример:

**задать значение \_ \_ \_ Прокси-сервер символов NT =**_myproxyserver_*_: 80_*

При использовании отладчика WinDbg настройте путь к символам, чтобы он указывал на хранилище символов, которое вы хотите использовать. Единственное отличие состоит в том, что в системе отображается диалоговое окно, в котором необходимо ввести идентификатор пользователя и пароль для передачи на прокси-сервер. Если ввести неверные данные, диалоговое окно будет отображено повторно. Если нажать кнопку **Отмена** , диалоговое окно отключается и сервер символов будет отключен для использования через Интернет.

При использовании последних версий cdb.exe или ntsd.exe эта функция по умолчанию отключена. Однако вы можете включить или отключить эту функцию с помощью команды! SYM Extension следующим образом:

-   Включение запроса идентификатора пользователя и пароля: **! SYM запросы**.
-   Чтобы отключить запрос идентификатора пользователя и пароля: **! SYM выводит запросы**.

Если включить запрос, потребуется перезагрузить символы с помощью команды. Reload.

API DbgHelp был расширен для поддержки этих изменений. Функция [**симболсерверсетоптионс**](/previous-versions//ms680676(v=vs.85)) поддерживает \_ параметр прокси-сервера ссрвопт. Если параметр data имеет **значение NULL**, используется прокси-сервер по умолчанию, определенный в **свойствах обозревателя** . В противном случае передается строка с нулевым нулем, указывающая имя и номер порта прокси-сервера. Имя и порт разделяются двоеточием следующим образом: *myproxyserver*: 80. Функция [**симсетоптионс**](/windows/desktop/api/Dbghelp/nf-dbghelp-symsetoptions) поддерживает \_ параметр симопт No \_ приглашений. При этом все запросы на проверку с сервера символов отключаются.
