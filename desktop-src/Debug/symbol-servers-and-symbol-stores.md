# <a name="symbol-server-and-symbol-stores"></a>  Сервер символов и хранилища символов

Правильная настройка символов для отладки может быть сложной задачей, особенно для отладки ядра. Часто требуется знание имен и выпусков всех продуктов на компьютере. Отладчик должен иметь возможность размещать файлы символов, соответствующие каждому выпуску продукта и пакету обновления. Это может привести к созданию слишком длинного пути к символу, состоящего из длинного списка каталогов.

Для упрощения этих трудностей при координации файлов символов используйте *сервер символов*. Сервер символов позволяет отладчикам автоматически получать правильные файлы символов без названий продуктов, выпусков или номеров сборки. средства отладки для Windows содержат сервер символов SymSrv.

Сервер символов активируется путем включения в путь к символам определенной текстовой строки. Каждый раз, когда отладчику нужно загрузить символы для вновь загруженного модуля, он вызывает сервер символов для размещения соответствующих файлов символов. Сервер символов находит файлы в *хранилище символов*. Это коллекция файлов символов, индексов и средств, которые администратор может использовать для добавления и удаления файлов. Файлы индексируются в соответствии с уникальными параметрами, такими как метка времени и размер изображения. средства отладки для Windows содержат инструмент для хранения символов с именем SymStore.
