# <a name="schema-extensions"></a>Расширения схемы

Архитектура схемы ADSI предоставляет возможность добавления новых классов схемы в контейнер класса схемы и новых свойств в существующий объект класса схемы во время выполнения. Последняя возможность не требует нового кода. Это важная функция для пространств имен, которые позволяют использовать расширяемые службы каталогов. Компонент поставщика должен разрешать эту расширяемость и иметь возможность получать доступ к экземпляру класса и значения его свойств и сохранять их. В типичной расширяемой службе каталогов эти сведения хранятся в базе данных службы каталогов так же, как и любые другие определения классов и свойств схемы.

> [!Note]  
> В терминологии COM-интерфейса в существующий класс схемы можно добавлять только свойства, а не методы. Добавление нового метода потребует добавления новой реализации этого метода и, таким образом, требует дополнительного кода.

 

Пример схемы конкретного поставщика см. в разделе [Управление схемой](schema-management.md).
