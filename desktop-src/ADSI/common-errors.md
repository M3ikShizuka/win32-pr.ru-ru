# <a name="common-errors-adsi"></a>Распространенные ошибки (ADSI)

Все ошибки, относящиеся к ADSI, имеют шестнадцатеричную форму 80005xxx. Наиболее распространенные коды ошибок описаны в следующей таблице.



| Шестнадцатеричный код ошибки ADSI | Описание                                                                                                                                         |
|---------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| 80005000<br/> | Передан недопустимый путь ADSI. Эта ошибка возникает при передаче неправильно сформированного значения ADsPath в **GetObject при привязке к объекту** .<br/> |
| 8000500D<br/> | Не удается найти свойство ADSI в кэше свойств.<br/>                                                                                 |
| 8000500E<br/> | Объект ADSI существует. При попытке создать объект ADSI с тем же именем, что и у существующего объекта ADSI, возникнет эта ошибка.<br/>    |



 

Полный список кодов ошибок ADSI см. в разделе [Общие коды ошибок ADSI](generic-adsi-error-codes.md).

## <a name="com-errors"></a>Ошибки COM

Так как ADSI состоит из COM-объектов, он возвращает коды стандартных ошибок COM. В следующей таблице перечислены коды ошибок COM, наиболее часто встречающиеся при программировании с помощью ADSI.



| Код шестнадцатеричной ошибки COM  | Описание                                                                                                                   |
|---------------------|-------------------------------------------------------------------------------------------------------------------------------|
| 80004005<br/> | Незаданная ошибка. Причина сбоя COM-объекта определена интерфейсом ADSI. <br/>                                  |
| 800041E4<br/> | Объект не найден. Эта ошибка в основном возникает из-за неправильного написания строки ADsPath при привязке к объекту.<br/> |



 

Дополнительные примеры ошибок COM, которые могут возникнуть в программировании ADSI, см. в разделе [Общие коды ошибок COM](generic-com-error-codes.md) .

## <a name="win32-errors"></a>Ошибки Win32

Любой код ошибки шестнадцатеричной формы 8007xxxx — это стандартный код ошибки Win32. Если вы преобразуете последние четыре цифры из шестнадцатеричного в десятичное, вы можете получить доступ к ошибке из командной строки Windows 2000:

**NET HELPMSG <number>**

В командной строке выше " &lt; Number &gt; " — это десятичное число, полученное путем преобразования последних четырех цифр кода ошибки из шестнадцатеричного числа. Эта командная строка предоставит более полезное описание ошибки Win32, которая может быть полезной для отладки сценария.
