# <a name="directory-services-today"></a>Службы каталогов сегодня

Часто можно найти несколько административных каталогов, развернутых в одной организации. К этим каталогам относятся каталоги сетевых ресурсов, такие как каталоги на основе LDAP, такие как служба Microsoft Active Directory Directory, Служба каталогов операционной системы Windows, а также каталоги приложений, такие как Microsoft Exchange.

![Развертывание нескольких каталогов](images/ds2chal.png)

## <a name="the-directory-challenge"></a>Запрос к каталогу

Несколько каталогов в организации имеют сложные задачи для пользователей, администраторов и разработчиков. Эти проблемы имеют ограниченное развертывание в каталоге. Пользователи сталкиваются с несколькими входами и различными интерфейсами для информации в нескольких каталогах. Администраторы сталкиваются с сложностью управления несколькими каталогами. Конечным пользователям и администраторам необходимо, чтобы разработчики приложений использовали имеющийся административный каталог, но разработчики дилеммой о том, какой из них использовать. Каждый каталог предлагает уникальные интерфейсы приложений. Разработчик должен выбрать конкретную реализацию каталога или поддерживать несколько версий своего приложения. В результате разработчики редко используют существующие службы каталогов.
