# <a name="directory-definition"></a>Определение каталога

В примере компонента поставщика используется относительно простая структура каталога для пояснения связи между компонентами и отображения минимальных требований, необходимых для поставщика ADSI.

Каталог для примера компонента поставщика состоит из двух корневых узлов: "Сиэтл" и "Торонто". Сиэтл содержит еще два подуровне, "Бельвью" и "Redmond". Каждая из этих записей содержит несколько учетных записей пользователей. Запись "Торонто" не имеет дополнительных подуровней, но непосредственно содержит несколько учетных записей пользователей. На следующем рисунке показаны два корневых узла, подключенных к сети.

![Определение каталога](images/dssmdo.png)

В иерархических терминах узел пространства имен содержит "Сиэтл" и "Торонто". "Сиэтл" содержит "Бельвью" и "Redmond". "Бельвью" и "Redmond" содержат набор учетных записей пользователей. "Торонто" напрямую содержит учетные записи пользователей без промежуточных узлов Организации.

Пример компонента поставщика представляет эту структуру только с двумя Active Directoryными типами объектов: объектом-контейнером и конечным объектом. "Сиэтл", "Торонто", "Бельвью" и "Redmond" являются объектами-контейнерами, а каждая учетная запись пользователя является конечным объектом.

В примере компонента поставщика создается класс схемы с именем "подразделение" для типа объекта контейнера и класс схемы с именем "User" для учетной записи пользователя.

Свойства каждого класса схемы, их методы и правила, управляющие отношениями включения для этих объектов, определяются в [управлении схемой](schema-management.md).
