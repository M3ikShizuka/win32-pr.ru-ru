# <a name="enumerating-container-objects"></a>Перечисление объектов контейнера

По соглашению все элементы в перечислении в ADSI должны иметь один и тот же тип данных автоматизации. Например, перечисление не должно возвращать некоторые элементы в качестве **варианта** типа **VT \_ I4** и других в качестве **варианта** типа **VT \_ BSTR**.

Чтобы перечислить список элементов, поддерживаемых объектом, клиент запрашивает создание объекта перечисления для указанного типа информации. В ADSI клиент может перечислить объекты в объектах пространства имен, универсальных объектах контейнера, объектах коллекции, объектах элементов или объектах схемы. ADSI предоставляет фильтр, который можно задать и изменить для ограничения совпадений в любом перечислении, несмотря на то, что свойство [**иадсконтаинер. Filter**](iadscontainer-property-methods.md) . Примеры реализаций объектов перечислителя можно найти в примере кода компонента поставщика для следующих объектов контейнера ADs.



| Тип объекта         | code         |
|---------------------|--------------|
| Универсальный контейнер   | ценумобж. cpp |
| Контейнер пространства имен | ценумнс. cpp  |
| Контейнер схемы    | ценумсч. cpp |



 

Сведения на стороне клиента см. в разделе [коллекции и группы](collections-and-groups.md).
