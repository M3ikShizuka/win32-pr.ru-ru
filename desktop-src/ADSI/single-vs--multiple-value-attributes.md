# <a name="single-vs-multiple-value-attributes"></a>Сравнение одного и нескольких атрибутов значений

Атрибуты, которые могут существовать в каталоге, обычно определяются в схеме для каталога. Определение схемы атрибута задает ряд характеристик атрибута, например тип данных и возможность наличия у экземпляра атрибута нескольких значений.

Экземпляр атрибута с одним значением может содержать одно значение. Экземпляр многозначного атрибута может содержать одно значение или несколько значений. Active Directory не создает атрибуты с пустыми значениями — либо атрибут содержит допустимое значение, либо он не существует в объекте.

> [!Note]  
> В Active Directory и большинстве серверов LDAP порядок значений в атрибуте с несколькими значениями не определен. Кроме того, каждое значение атрибута с несколькими значениями должно быть уникальным.

 

ADSI обычно загружает данные схемы, если каталог поддерживает схему, как Active Directory. Так как ADSI знает синтаксис атрибутов в схеме, указывать тип атрибута при доступе к нему не требуется. ADSI маршалирует значения атрибутов в соответствующий тип данных, как определено в схеме.

Если каталог не имеет схемы, укажите тип данных при доступе к атрибуту.

> [!Note]  
> Active Directory, Exchange, Windows NT 4,0 и сервер сайта имеют схему. Кроме того, Active Directory имеет расширяемую схему.
