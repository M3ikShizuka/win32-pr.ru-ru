# <a name="binding-string"></a>Строка привязки

Из-за количества объектов, доступных из службы каталогов, могут возникнуть конфликты именования. Строка привязки, которая обычно называется ADsPath, позволяет указать конкретный объект, не вызывая конфликтов именования. Это можно применить к одному поставщику службы каталогов или нескольким поставщикам служб каталогов.

ADsPath — это строка, однозначно определяющая объект ADSI в службе каталогов. Поскольку объекты ADSI существуют в контексте пространства имен базовой службы каталогов, часть синтаксиса имени ADsPath зависит от конкретного поставщика.

В следующей таблице перечислены поставщики ADSI, предоставляемые по умолчанию.



| Поставщик         | Описание                                                                                                                                                     |
|------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| WinNT<br/> | Используется для связи с контроллерами домена Windows. Дополнительные сведения о команде WinNT ADsPath см. в разделе [WinNT ADsPath](winnt-adspath.md).<br/>           |
| LDAP<br/>  | Используется для связи с серверами LDAP, например Active Directory. Дополнительные сведения о LDAP ADsPath см. в разделе [LDAP ADsPath](ldap-adspath.md).<br/>  |
| ADs<br/>   | Предоставляет реализацию [**иадснамеспацес**](/windows/desktop/api/Iads/nn-iads-iadsnamespaces) , которую можно использовать для перечисления всех поставщиков ADSI, установленных на клиенте.<br/> |



 

Используйте эти имена поставщиков для доступа к пространству имен поставщика по умолчанию. Например, если выполнить привязку к LDAP, ADSI привязывается к контейнеру, содержащему объект домена, вошедший в систему. Если выполнить привязку к WinNT, ADSI привязывается к контейнеру, содержащему объекты, которые соответствуют всем доменам в сети.

Начальными элементами строки ADsPath являются программный идентификатор (progID) поставщика ADSI, за которым следует "://", за которым следует синтаксис, заданный пространством имен поставщика. Строка progID может быть или не учитываться с учетом регистра в зависимости от поставщика. Строки progID для поставщиков, перечисленных выше, учитывают регистр.

Строка пути может быть или не учитываться с учетом регистра в зависимости от поставщика. Строки пути для поставщиков, перечисленных выше, не учитывают регистр.

Ниже приведены примеры Адспасс.

``` syntax
LDAP://CN=Jeff Smith,CN=users,DC=fabrikam,DC=com
LDAP://server01/CN=Jeff Smith,CN=users,DC=fabrikam,DC=com
 
WinNT://MyDomain/ComputerName,Computer
WinNT://MyDomain/UserAccount
```

Чтобы найти все поставщики, установленные на компьютере, выполните привязку к поставщику ADs, как показано в следующем примере кода.


```VB
Set x = GetObject("ADs:")
For Each provider In x
    provider.Name
Next
```



С помощью поставщика LDAP можно указать значение ADsPath либо в форме различающегося имени X. 500 (DN), начиная с тега CN, либо можно указать его иерархическую инверсию, начиная с тега O. Форма, используемая в начальном параметре ADsPath, определяет порядок тегов.

В следующей таблице перечислены специальные символы ADsPath.



| Имя                      | Знак           | Описание                                                                                                                                                                                           |
|---------------------------|---------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Двойная кавычка<br/>   | "<br/>        | Используется для заключения в кавычки любой части ADsPath, которая может содержать специальный символ, чтобы строка считалась буквальной. Например, "CN = Name/prefix".<br/>                                     |
| Обратная косая черта<br/>      | \\<br/>       | Используется для перед специальными символами для обозначения того, что они должны использоваться в качестве литералов. Дополнительные сведения и список специальных символов см. в разделе [отличительные имена](/previous-versions/windows/desktop/ldap/distinguished-names).<br/> |
| Подряд<br/>          | /<br/>        | Разделитель компонентов.<br/>                                                                                                                                                                       |
| Угловые скобки<br/> | <><br/> | Разделять путь ADsPath в другом соглашении об именовании.<br/>                                                                                                                                       |



 

Чтобы разделить путь ADsPath в спецификации поиска или в составе URL-адреса, используйте левую и правую угловую скобку (< >). Например, " &lt; WinNT://mydomain/UserAccount &gt; ".

Некоторые поставщики ADSI могут добавить ограничения синтаксиса из-за требований к пространству имен.

## <a name="active-directory-binding-options"></a>Параметры привязки Active Directory

Active Directory предоставляет возможность привязки к объекту с помощью нескольких других типов строк привязки, таких как глобальный уникальный идентификатор (GUID) COM или идентификатор безопасности (SID). Дополнительные сведения см. [в разделе Binding to Active Directory](/windows/desktop/AD/binding-to-active-directory-domain-services).
