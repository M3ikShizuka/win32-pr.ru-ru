# <a name="creating-a-new-group"></a>Создание новой группы

Джо Ворден, администратор предприятия, должен создать новую группу. Он хочет защитить некоторые ресурсы, такие как файлы, Active Directory объекты или другие объекты, на основе членства в этой группе. В следующем примере кода показано, как создать новую группу.


```VB
Set ou = GetObject("LDAP://OU=Sales,DC=Fabrikam,DC=COM")
Set grp = ou.Create("group", "CN=Management")
grp.Put "samAccountName", "mgmt"
grp.Put "groupType", ADS_GROUP_TYPE_DOMAIN_LOCAL_GROUP Or ADS_GROUP_TYPE_SECURITY_ENABLED
grp.SetInfo
```



Эта группа, управление, будет создана в подразделении Sales. Во первых, Джо должен создать объект ADSI для подразделения Sales. Во-вторых, он должен установить атрибут [**SamAccountName**](/windows/desktop/AD/group-objects) для этого объекта, так как он является обязательным атрибутом для обратной совместимости. в этом примере, когда задается **samAccountName** , Windows NT средства 4,0, такие как диспетчер пользователей, видят атрибут в качестве **руководства** , а не **управления**.

В третьих, Джо должен указывать тип группы. в домене Windows 2000 существует три типа групп: глобальная, доменная локальная и универсальная. Кроме того, группа несет свою характеристику безопасности. Группа может быть группой с поддержкой безопасности или без защиты. По сути, группы с поддержкой безопасности — это те, которым могут быть предоставлены или запрещены права доступа к ресурсам, аналогичные пользователю. Например, предоставление группе доступа к общей папке предполагает, что все члены группы имеют доступ к общей папке. Списки рассылки нельзя использовать аналогичным образом — вы не можете, например, предоставить список рассылки право доступа к общей папке. во время обновления группы Windows NT 4,0 переносятся как группы с поддержкой безопасности. Незащищенные группы в Active Directory похожи на списки рассылки в Exchange. таким образом, создание групп или списков рассылки аналогично операциям в Windows 2000. в основном режиме Windows 2000 (основной режим означает, что все контроллеры домена находятся на компьютерах с сервером Windows 2000). группы могут быть вложены на любой уровень.
