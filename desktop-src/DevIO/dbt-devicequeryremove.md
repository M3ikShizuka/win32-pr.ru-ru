# <a name="dbt_devicequeryremove-event"></a>\_Событие ДБТ девицекуериремове

Система передает \_ событие ДБТ девицекуериремове Device, чтобы запросить разрешение на удаление устройства или носителя. Это сообщение является последней возможностью для подготовки приложений и драйверов к удалению. Однако любое приложение может отклонить этот запрос и отменить операцию.

Чтобы транслировать это событие устройства, система использует сообщение [**WM \_ девицечанже**](wm-devicechange.md) с параметром *wParam* Set ДБТ \_ девицекуериремове и *lParam* Set, как описано ниже.


```C++
LRESULT CALLBACK WindowProc(
  HWND hwnd,       // handle to window
  UINT uMsg,       // WM_DEVICECHANGE
  WPARAM wParam,   // device-change event
  LPARAM lParam    // event-specific data
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*HWND* 
</dt> <dd>

Дескриптор окна.

</dd> <dt>

*Uiacp* 
</dt> <dd>

Идентификатор сообщения [**WM \_ девицечанже**](wm-devicechange.md) .

</dd> <dt>

*wParam* 
</dt> <dd>

Задайте для параметра значение ДБТ \_ девицекуериремове.

</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на структуру, идентифицирующую удаляемое устройство. Структура состоит из независимого от событий заголовка, за которым следуют элементы, зависящие от события, которые описывают устройство. Чтобы использовать эту структуру, рассматривайте структуру как [**\_ \_ HDR**](/windows/desktop/api/Dbt/ns-dbt-dev_broadcast_hdr) -структуру для разработки вещания, а затем проверьте ее член **дбч \_ типа устройства** , чтобы определить тип устройства.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает **значение true** , чтобы предоставить разрешение на удаление устройства.

Верните ШИРОКОВЕЩАТЕЛЬный \_ запрос \_ Deny, чтобы запретить разрешение на удаление устройства.

## <a name="remarks"></a>Remarks

Необходимо закрыть все дескрипторы устройства, иначе удаление устройства завершится ошибкой.
