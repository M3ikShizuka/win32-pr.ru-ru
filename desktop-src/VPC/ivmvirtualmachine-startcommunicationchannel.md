# <a name="ivmvirtualmachinestartcommunicationchannel-method"></a>Метод Ивмвиртуалмачине:: Старткоммуникатиончаннел

\[Windows Virtual PC больше не доступна для использования в Windows 8. Вместо этого используйте [поставщик WMI Hyper-V (v2)](/windows/desktop/HyperV_v2/windows-virtualization-portal).\]

Настройка коммуникационного канала между узлом и гостевой операционной системой.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT StartCommunicationChannel(
  [in] VMEndpointType inHostEndpointType,
  [in] BSTR           inHostEndPointName,
  [in] VMEndpointType inGuestEndpointType,
  [in] BSTR           inGuestEndpointName
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*инхостендпоинттипе* \[ окне\]
</dt> <dd>

Этот параметр должен быть **вмендпоинт \_ помощью канала NamedPipe** (0).

</dd> <dt>

*инхостендпоинтнаме* \[ окне\]
</dt> <dd>

Уникальное имя канала. Эта строка должна иметь следующую форму: " \\ \\ . \\ pipe \\ *pipeName*". *PipeName* часть имени может содержать любой символ, кроме обратной косой черты, включая цифры и специальные символы. Длина строки имени канала может доставлять до 256 символов. Имена каналов не чувствительны к регистру.

</dd> <dt>

*ингуестендпоинттипе* \[ окне\]
</dt> <dd>

Этот параметр должен быть **вмендпоинт \_ tcpip** (1).

</dd> <dt>

*ингуестендпоинтнаме* \[ окне\]
</dt> <dd>

Номер порта, на котором прослушивается TCP-сервер в гостевой системе.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Текущая реализация поддерживает только интерфейс именованных каналов на узле и интерфейс TCP/IP в гостевой операционной системе.
