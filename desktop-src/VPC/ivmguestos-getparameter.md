# <a name="ivmguestosgetparameter-method"></a>Метод Ивмгуестос:: Parameter

\[Windows Virtual PC больше не доступна для использования в Windows 8. Вместо этого используйте [поставщик WMI Hyper-V (v2)](/windows/desktop/HyperV_v2/windows-virtualization-portal).\]

Извлекает именованный параметр в гостевой операционной системе.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetParameter(
  [in]          BSTR inParameterName,
  [out, retval] BSTR *outParameterValue
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*инпараметернаме* \[ окне\]
</dt> <dd>

Имя параметра. Длина должна составлять от 1 до 255 символов и не может содержать символ обратной косой черты ( \\ ).

</dd> <dt>

*аутпараметервалуе* \[ out, retval\]
</dt> <dd>

Значение параметра.

</dd> </dl>

## <a name="remarks"></a>Remarks

Виртуальная машина должна быть запущена, и при вызове этого метода должны быть установлены компоненты интеграции. Этот метод поддерживается только для гостевых операционных систем под управлением Windows.

После установки компонентов интеграции в реестр гостевой операционной системы автоматически добавляется следующий ключ:

**HKEY \_ локальный \_ компьютер \\ программное обеспечение \\ \\ виртуальные машины Microsoft \\ Гостевые \\ Параметры**

При запуске гостевой операционной системы в разделе **параметров** заполняются следующие строковые значения реестра:

-   **HostName**
-   **фисикалхостнаме**
-   **фисикалхостнамефулликуалифиед**
-   **VirtualMachineName**
