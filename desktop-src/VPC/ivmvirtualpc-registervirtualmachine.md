# <a name="ivmvirtualpcregistervirtualmachine-method"></a>Метод Ивмвиртуалпк:: Регистервиртуалмачине

\[Windows Virtual PC больше не доступен для использования в Windows 8. Вместо этого используйте [поставщик WMI Hyper-V (v2)](/windows/desktop/HyperV_v2/windows-virtualization-portal).\]

Регистрирует существующую конфигурацию виртуальной машины и извлекает объект виртуальной машины.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT RegisterVirtualMachine(
  [in]          BSTR              configurationName,
  [in]          BSTR              configurationPath,
  [out, retval] IVMVirtualMachine **virtualMachine
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*configurationName* \[ окне\]
</dt> <dd>

Имя регистрируемой виртуальной машины. Длина имени не может превышать 80 символов, а общая длина имени и пути не может превышать **максимальный \_ путь** (260) символов. Указанное имя может содержать расширение VMC. Если этот параметр имеет **значение NULL** или является пустой строкой, параметр *configurationPath* должен указывать полный путь к файлу конфигурации.

</dd> <dt>

*configurationPath* \[ окне\]
</dt> <dd>

Путь к папке, содержащей существующий файл конфигурации. Если параметр *configurationName* имеет **значение NULL** или является пустой строкой, он должен указывать полный путь к существующему файлу конфигурации.

</dd> <dt>

*virtualMachine* \[ out, retval\]
</dt> <dd>

Указатель на новый объект [**ивмвиртуалмачине**](ivmvirtualmachine.md) , представляющий эту виртуальную машину.

</dd> </dl>

## <a name="remarks"></a>Remarks

В именах виртуальных машин не учитывается регистр, например "MyVM" и "MyVM" ссылаются на одну и ту же виртуальную машину.
