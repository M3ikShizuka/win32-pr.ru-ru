# <a name="ivmkeyboard-interface"></a>Интерфейс Ивмкэйбоард

\[Windows Virtual PC больше не доступен для использования в Windows 8. Вместо этого используйте [поставщик WMI Hyper-V (v2)](/windows/desktop/HyperV_v2/windows-virtualization-portal).\]

Управляет устройством клавиатуры в виртуальной машине. **Ивмкэйбоард** для виртуальной машины можно получить с помощью свойства [**Ивмвиртуалмачине:: Keyboard**](ivmvirtualmachine-keyboard.md) .

## <a name="members"></a>Элементы

Интерфейс **ивмкэйбоард** наследует от интерфейса [**IDispatch**](/windows/win32/api/oaidl/nn-oaidl-idispatch) . **Ивмкэйбоард** также имеет следующие типы членов:

-   [Методы](#methods)
-   [Свойства](#properties)

### <a name="methods"></a>Методы

Интерфейс **ивмкэйбоард** содержит следующие методы.



| Метод                                                       | Описание                                                                   |
|:-------------------------------------------------------------|:------------------------------------------------------------------------------|
| [**Нажимает**](ivmkeyboard-ispressed.md)                   | Определяет, является ли указанный ключ недоступным.<br/>                      |
| [**прессандрелеасекэй**](ivmkeyboard-pressandreleasekey.md) | Имитирует нажатое клавиша, а затем освобождается.<br/>              |
| [**PressKey**](ivmkeyboard-presskey.md)                     | Имитирует нажатой клавиша.<br/>                                |
| [**релеасекэй**](ivmkeyboard-releasekey.md)                 | Имитирует освобождение ключа.<br/>                                    |
| [**типеасЦиитекст**](ivmkeyboard-typeasciitext.md)           | Имитирует ряд вводимых в гостевой системе ключей ASCII.<br/>       |
| [**типекэйсекуенце**](ivmkeyboard-typekeysequence.md)       | Имитирует разделенный запятыми список ключей, вводимых в гостевой системе.<br/> |



 

### <a name="properties"></a>Свойства

Интерфейс **ивмкэйбоард** имеет следующие свойства.



| Свойство                                                                | Тип доступа           | Описание                                                                                                |
|:------------------------------------------------------------------------|:----------------------|:-----------------------------------------------------------------------------------------------------------|
| [**хасексклусивеакцесс**](ivmkeyboard-hasexclusiveaccess.md)<br/> | Чтение/запись<br/> | Указывает, имеет ли этот объект исключительный контроль над устройством клавиатуры виртуальной машины.<br/> |



 

## <a name="remarks"></a>Remarks

Ключи можно ввести в виртуальную машину несколькими способами. Чтобы ввести обычную последовательность символов ASCII, используйте метод [**типеасЦиитекст**](ivmkeyboard-typeasciitext.md) . Если требуется больше гибкости, **ивмкэйбоард** имеет несколько методов, предназначенных для использования с кодами клавиш, приведенными в следующем списке. Метод [**типекэйсекуенце**](ivmkeyboard-typekeysequence.md) может принимать разделенную запятыми строку кодов клавиш, которая будет нажата и отпущена в пределах виртуальной машины. Помимо этих кодов клавиш, ключевые слова можно использовать для принудительного нажатия клавиши или только для освобождения. Ключевые слова UP и DOWN применяются только к коду ключа, непосредственно следующему за ключевым словом.

Чтобы избежать одновременной попытки доступа нескольких сценариев, приложений или пользователей к одному и тому же устройству клавиатуры, задайте для свойства [**Хасексклусивеакцесс**](ivmkeyboard-hasexclusiveaccess.md) **значение true**. Если один процесс получает монопольный доступ, любые попытки других процессов отправить входные данные на устройство клавиатуры будут проигнорированы до тех пор, пока не будет освобожден монопольный доступ.
