# <a name="ivmvirtualpccreatevirtualmachine-method"></a>Метод Ивмвиртуалпк:: Креатевиртуалмачине

\[Windows Virtual PC больше не доступна для использования в Windows 8. Вместо этого используйте [поставщик WMI Hyper-V (v2)](/windows/desktop/HyperV_v2/windows-virtualization-portal).\]

Создает новую конфигурацию виртуальной машины и извлекает объект виртуальной машины.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT CreateVirtualMachine(
  [in]          BSTR              configurationName,
  [in]          BSTR              configurationPath,
  [out, retval] IVMVirtualMachine **virtualMachine
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*configurationName* \[ окне\]
</dt> <dd>

Имя создаваемой виртуальной машины. Длина имени не может превышать 80 символов, а общая длина имени и пути к файлам VMC и VMCX не может превышать **максимальную длину \_ пути** (260) символов. Файлы Extensions. VMC и. vmcx будут добавлены в конец имени виртуальной машины при создании файлов конфигурации. Если этот параметр имеет **значение NULL** или является пустой строкой, параметр *configurationPath* должен указывать полный путь к файлу VMC.

</dd> <dt>

*configurationPath* \[ окне\]
</dt> <dd>

Путь к папке, в которой будет содержаться файл VMC. Если эта папка не существует, она будет создана. Если *configurationName* имеет **значение NULL** или является пустой строкой, необходимо указать полный путь к новому файлу конфигурации.

</dd> <dt>

*virtualMachine* \[ out, retval\]
</dt> <dd>

Указатель на новый объект [**ивмвиртуалмачине**](ivmvirtualmachine.md) , представляющий эту виртуальную машину.

</dd> </dl>

## <a name="remarks"></a>Комментарии

В именах виртуальных машин не учитывается регистр, например "MyVM" и "MyVM" ссылаются на одну и ту же виртуальную машину.
