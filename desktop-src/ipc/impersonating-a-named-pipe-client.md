# <a name="impersonating-a-named-pipe-client"></a>Олицетворение клиента именованного канала

*Олицетворение* — это возможность выполнения потока в контексте безопасности, отличном от процесса, владеющего потоком. Олицетворение позволяет серверному потоку выполнять действия от имени клиента, но в пределах ограничений контекста безопасности клиента. Клиент обычно имеет более низкий уровень прав доступа. Дополнительные сведения см. в разделе [олицетворение](/windows/desktop/SecAuthZ/client-impersonation).

Серверный поток именованного канала может вызвать функцию [**имперсонатенамедпипеклиент**](/windows/desktop/api/namedpipeapi/nf-namedpipeapi-impersonatenamedpipeclient) , чтобы предположить маркер доступа пользователя, подключенного к клиентской стороне канала. Например, сервер именованных каналов может предоставить доступ к базе данных или файловой системе, к которой сервер канала имеет привилегированный доступ. Когда клиент канала отправляет запрос на сервер, сервер олицетворяет клиента и пытается получить доступ к защищенной базе данных. Затем система предоставляет или запрещает доступ сервера на основе уровня безопасности клиента. После завершения работы сервера он использует функцию [**RevertToSelf**](/windows/desktop/api/securitybaseapi/nf-securitybaseapi-reverttoself) для восстановления исходного маркера безопасности.

[Уровень олицетворения](/windows/desktop/SecAuthZ/impersonation-levels) определяет операции, которые сервер может выполнять при олицетворении клиента. По умолчанию сервер олицетворяется на уровне олицетворения Секуритимперсонатион. Однако когда клиент вызывает функцию [**CreateFile**](/windows/desktop/api/fileapi/nf-fileapi-createfilea) для открытия маркера клиентской стороны канала, клиент может использовать \_ флаг Security SQOS \_ Present для управления уровнем олицетворения сервера.
