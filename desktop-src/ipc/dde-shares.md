# <a name="dde-shares"></a>Общие ресурсы DDE

\[Сетевой DDE больше не поддерживается. Nddeapi.dll имеется в Windows Vista, но все вызовы функций возвращают НДДЕ \_ не \_ реализован.\]

Общие ресурсы DDE — это ресурсы компьютера. Они похожи на общие файловые ресурсы, так как они используются для управления доступом к ресурсу. При использовании файловых ресурсов ресурс является файлом. При использовании общих ресурсов DDE ресурс динамически обменивается данными. Тип обмена данными определяется серверным приложением, предоставляющим данные, и клиентским приложением, которое запрашивает данные.

Сервер вызывает функцию [**нддешареадд**](nddeshareadd.md) для создания общего ресурса DDE, который хранится в диспетчере общей базы данных DDE (DSDM).

Клиент запускает сеанс DDE, подключаясь к общему ресурсу DDE. Клиент должен вызвать функцию [**ддеинитиализе**](/windows/win32/api/ddeml/nf-ddeml-ddeinitializea) для инициализации ддемл и вызвать функцию [**ддеконнект**](/windows/win32/api/ddeml/nf-ddeml-ddeconnect) для подключения к общему ресурсу DDE. В вызове **ддеконнект** клиент указывает имя службы следующим образом:

\\\\*Имя компьютера* \\ НДДЕ $

где *ComputerName* — имя компьютера, на котором работает серверное приложение. НДДЕ $ указывает, что в разделе [**ддеконнект**](/windows/win32/api/ddeml/nf-ddeml-ddeconnect) указано имя общего ресурса DDE на удаленном компьютере с именем *ComputerName*.

Существует три типа общих ресурсов DDE: старый стиль, новый стиль и статический. Обычно поддерживается только статический тип. Имена статических общих ресурсов используют следующее соглашение: *ShareName*$.
