# <a name="about-network-dde"></a>О сетевом DDE

\[Сетевой DDE больше не поддерживается. Nddeapi.dll имеется в Windows Vista, но все вызовы функций возвращают ндде \_ не \_ реализованы.\]

Сетевой DDE используется для инициации и обслуживания сетевых подключений, необходимых для сеансов DDE между приложениями, работающими на разных компьютерах в сети. *Сеанс* DDE — это взаимодействие между клиентскими и серверными приложениями. В приложении вы используете сетевой DDE, а также DDE и библиотеку управления DDE (ДДЕМЛ).

DDE — это форма межпроцессного взаимодействия, которая использует общую память для обмена данными между приложениями. Приложения могут использовать DDE для однократной передачи данных или для текущих операций обмена и обновления данных. Дополнительные сведения о DDE см. в разделе [платформа динамических данных Exchange](../dataxchg/dynamic-data-exchange.md).

ДДЕМЛ упрощает задачу добавления возможностей DDE в приложение. Вместо отправки, публикации и обработки сообщений DDE напрямую приложение использует функции, предоставляемые ДДЕМЛ для управления сеансами DDE. дополнительные сведения о ддемл см. в разделе [платформа динамических данных Exchange управления библиотекой](../dataxchg/dynamic-data-exchange-management-library.md).

## <a name="network-dde-files"></a>Файлы сетевого DDE

Чтобы использовать элементы API сетевого DDE, необходимо включить файл заголовка Нддеапи. h в исходные файлы и включить файл Нддеапи. lib в строку связи. Необходимо также убедиться, что файл NDDEApi.dll загружен.
