# <a name="windows-projected-file-system-projfs"></a>Windows Плановая файловая система (Прожфс)

Windows проектируемая файловая система (прожфс) позволяет приложению пользовательского режима, называемому "поставщиком", выполнять проецирование иерархических данных из резервного хранилища данных в файловую систему, что делает их отображаемыми в файловой системе как файлами и каталогами. например, простой поставщик может проецировать Windows реестр в файловую систему, делая разделы и значения реестра в виде файлов и каталогов соответственно. Примером более сложного поставщика является [VFS для Git](https://github.com/Microsoft/VFSForGit), который используется для виртуализации очень большого репозиториев Git.

> [!NOTE]
> Прожфс предназначен для использования с высокоскоростными резервными хранилищами данных. Одна из целей проектирования состоит в том, чтобы сделать прогнозируемые данные, как будто они были локально представлены, скрывая тот факт, что данные могут быть удаленными. Таким образом, Прожфс не предоставляет следующие механизмы для создания отчетов о ходе отзыва данных. Указывает на состояние файла в сети и вне сети; и другие возможности, которые могут быть желательны при работе с резервными хранилищами данных, которые выполняются слишком долго. Для таких сценариев вместо этого рекомендуется использовать [API облачных файлов](../cfapi/cloud-files-api-portal.md).
