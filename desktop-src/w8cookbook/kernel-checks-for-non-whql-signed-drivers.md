# <a name="kernel-checks-for-non-whql-signed-drivers"></a>Проверки ядра для драйверов, не подписанных WHQL

\[Некоторые сведения относятся к предварительно выпущенному продукту, который может быть значительно изменен перед коммерческой выпуском. Корпорация Майкрософт не предоставляет никаких гарантий, явных или подразумеваемых, относительно предоставленной здесь информации.\]

для устройств, на которых чистая установка Windows 10 и где включена безопасная загрузка (обратите внимание, что это стандартный вариант для всех новых устройств, начиная с выпуска Windows 8 0), все новые драйверы должны быть подписаны лабораториями WHQL/сисдев, а не просто использовать перекрестно подписанные сертификаты. Обновленные устройства и случаи, когда драйверы были подписаны с помощью перекрестных сертификатов, прежде чем эта политика вступит в действие, исключается из этой политики, чтобы обеспечить обратную совместимость. эта политика была объявлена как апрель 2015, однако она будет применена к выпуску Windows годовщина, выпущенной 2016 августа.

В случаях, когда драйвер не подписан должным образом, он будет проявляться как уведомление PCA о том, что драйверы заблокированы, если они не прошли требования к подписи. Это предотвращает непрозрачное блокирование автономного взаимодействия пользователя драйвера в ядре.

Дополнительные сведения см. в [этой записи блога](https://techcommunity.microsoft.com/t5/Windows-Hardware-Certification/bg-p/WindowsHardwareCertification/), которая сообщает об изменении подписи драйвера.
