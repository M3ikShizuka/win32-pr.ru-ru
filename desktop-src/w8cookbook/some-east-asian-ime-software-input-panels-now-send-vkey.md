# <a name="some-east-asian-ime-software-input-panels-now-send-vkey"></a>Некоторые панели ввода программного обеспечения IME для восточноазиатских языков теперь отправляют ключ

## <a name="platforms"></a>Платформы

<dl> Клиенты — Windows 8.1  
серверы — Windows Server 2012 R2  
</dl>

## <a name="description"></a>Описание

если выбран один из восточно-азиатских редакторов ime, то при нажатии клавиши на панели ввода программного обеспечения вкэйс не отправляется. Windows 8

в Windows 8.1 отправляются вкэйс для следующих конфигураций:



| Язык            | IME                                 | Магазин Windows | Рабочий стол |
|---------------------|-------------------------------------|---------------|---------|
| Китайский (упрощенный)  | Пиньинь Майкрософт, Microsoft Wubi    | Да           | Да     |
| Китайский (традиционный) | Microsoft Чанглие, Microsoft Quick | Да           | Да     |
| Китайский (традиционный) | Бопомофо (Майкрософт)                  | нет            | Нет      |
| Корейский              | "Microsoft IME"                       | Да           | Нет      |
| Японский            | "Microsoft IME"                       | нет            | нет      |



 

## <a name="manifestations"></a>Проявлениями

Если пользователь выбирает IME, который не отправляет Вкэйс, функции, активируемые с помощью триггера, не работают.

## <a name="solution"></a>Решение

Приложения, которые не используют одну из перечисленных выше конфигураций IME, должны быть спроектированы таким образом, чтобы пользователи могли выполнять нужную задачу без использования функций, требующих Вкэйс.

если пользователь выбирает IME, который отправляет вкэйс, но приложение было спроектировано без этого ожидания, приложение должно быть изменено, чтобы определить, какая версия Windows работает, и ответить соответствующим образом.
