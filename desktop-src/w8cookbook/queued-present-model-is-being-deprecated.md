# <a name="queued-present-model-is-being-deprecated"></a>Текущая модель в очереди устарела

## <a name="platforms"></a>Платформы

**Клиенты** — после Windows 8  
**Серверы** — после Windows Server 2012  


## <a name="description"></a>Описание

в Windows выпуске после Windows 8 эти api-интерфейсы будут возвращать E \_ нотимпл:

-   двмсетпресентпараметерс
-   двмсетдксфрамедуратион
-   двммодифипревиаусдксфрамедуратион

Кроме того, этот API принимает только значение NULL для параметра HWND:

-   двмжеткомпоситионтимингинфо

Мы будем прекращать поддержку Двмжеткомпоситионтимингинфо с HWND! = NULL и удалим связанные функции. Если указано значение HWND, отличное от NULL, этот API вернет E \_ INVALIDARG.

Мы также рекомендуем использовать API Двмжеткомпоситионтимингинфо и предлагать, что разработчики переключаются на модель перелистывания DXGI и соответствующий API статистики DX Present.

## <a name="manifestation"></a>Проявление

Приложения, использующие поставленную в очередь модель, будут работать неправильно. Точный уровень манифеста зависит от конкретного приложения, но может варьироваться от неправильного времени презентации при неожиданном выходе приложения. На практике мы не будем видеть многие (если они есть) такие приложения. эта модель использовалась проигрывателем windows media player, которая не будет использоваться в Windows 8 (и, возможно, в старом проигрывателе Zune). В настоящее время у нас нет сведений о других приложениях, фактически использующих эту модель.

## <a name="solution"></a>Решение

разработчикам необходимо использовать режим представления DXGI с отражением, а не в очереди (доступен в среде выполнения DX9 с Windows 7, а также в средах выполнения содержимым dx10 и DX11 в Windows 8).

## <a name="tests"></a>Тесты

выполните общие тесты, чтобы убедиться, что входящие Windows компоненты и основные продукты продолжают работать в следующей версии Windows.
