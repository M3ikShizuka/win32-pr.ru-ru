# <a name="ime-mode-model-changed-from-per-user-to-per-thread"></a>Модель режима IME изменена с "на пользователя" на "на поток"

## <a name="platforms"></a>Платформы

<dl> Клиенты — Windows 8.1  
серверы — Windows Server 2012 R2  
</dl>

## <a name="description"></a>Описание

в Windows 8 режим преобразования ime и режим предложений ime основаны на контексте пользователя и изменяют режим приложения, затронутый всеми другими приложениями. Это поведение можно отключить с помощью параметра "Позвольте мне задать другой метод ввода для каждого окна приложения" в разделе "Дополнительные параметры" панели управления "язык".

чтобы улучшить совместимость, в Windows 8.1 режимы хранятся во входном контексте независимо от того, как установлен элемент управления "разрешить мне устанавливать различные методы ввода для каждого окна приложения". в Windows 8.1 параметр «позвольте мне задать другой метод ввода для каждого окна приложения» влияет только на выбор самого метода ввода.

При запуске приложения в режиме IME задаются следующие значения по умолчанию:



| &nbsp;   | Панель программного ввода | Аппаратная клавиатура |
|----------|----------------------|-------------------|
| KOR, JPN | Включен                   | Off               |
| CHS, CHT  | Включено                   | Включено                |



 

## <a name="manifestations"></a>Проявлениями

Когда пользователь изменяет режим IME в приложении, это изменение не влияет на другие приложения.
