# <a name="windows-components-installed-on-demand"></a>Компоненты Windows, установленные по запросу

## <a name="platform"></a>Платформа

**Клиенты —** Windows 8.1  







## <a name="description"></a>Описание

Два компонента Windows будут установлены по запросу в Windows 8.1: DirectPlay и NTVDM. Эти компоненты перечислены в необязательных компонентах в узле "устаревшие компоненты". Эти компоненты устанавливаются локально в составе операционной системы и сжимаются как дополнительные компоненты. Когда приложение обращается или вызывает один из этих компонентов (обычно при первом запуске приложения), установка активируется автоматически. Пользователи будут уведомлены о том, что компонент будет установлен, и они должны подтвердить действие для продолжения. Для этого требуется повышение прав, если пользователь не является администратором. После первоначальной установки пользователю не нужно выполнять никакие другие действия для повторного использования компонента.

## <a name="manifestation"></a>Проявление

Когда приложение ссылается на устаревший компонент или вызывает его в дополнительных компонентах (при первом запуске), приложение будет приостановлено и откроется диалоговое окно "компонент по запросу", запрашивающее разрешение пользователя на установку компонента. Если пользователь нажимаем кнопку "ОК", они также могут увидеть запрос на повышение прав, когда они должны ввести свои учетные данные. Затем компонент будет установлен и приложение будет возобновлено.

## <a name="mitigation"></a>Меры по снижению риска

Чтобы эти компоненты не открывались в интерфейсе пользователя по запросу, можно предварительно установить DirectPlay и NTVDM с помощью DISM или пользовательского интерфейса дополнительных компонентов.

## <a name="solution"></a>Решение

Настоятельно рекомендуется избегать ссылок или вызова компонентов, которые были указаны как устаревшие необязательные компоненты в Windows в будущих версиях приложения. Устаревшие необязательные компоненты содержат только старые, менее используемые компоненты, которые могут вызвать проблемы с производительностью и безопасностью пользователей.

## <a name="tests"></a>Тесты

Для обеспечения совместимости не требуется никаких дополнительных тестовых размещений. Важно помнить, что это диалоговое окно появляется при вызове или ссылке на дополнительный компонент, и эта установка требует повышения прав.
