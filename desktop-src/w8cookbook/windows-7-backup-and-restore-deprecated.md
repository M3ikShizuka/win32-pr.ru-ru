# <a name="windows-7-backup-and-restore-deprecated"></a>нерекомендуемое резервное копирование и восстановление Windows 7

## <a name="platform"></a>Платформа

**клиенты** — Windows 8 


## <a name="description"></a>Описание

При отсутствии изменений в поведении на резервное копирование и восстановление эта функция считается устаревшей и не будет обновлена. Он использовался редко, и его функциональность заменена новой функцией истории файлов. она будет поставляться в Windows 8 и энтузиастов, которые обновились с Windows 7 до Windows 8 или зависят от резервного копирования и восстановления, а также от архивации образа диска, будут по-прежнему иметь возможность его использовать. Однако все точки доступа к резервному копированию и восстановлению, за исключением панели управления, были удалены. приложение панели управления, используемое для резервного копирования и восстановления, было переименовано в Windows 7 восстановление файлов.

Изготовители оборудования, устанавливающие раздел реестра для отключения уведомлений об архивации в своих образах, больше не нуждаются в этом.

Мы не рекомендуем использовать обе функции одновременно. Проверка истории файлов. Если расписание резервного копирования существует и активно, а если оно найдено, оно не позволит пользователям включать его. в этом случае пользователям, желающим использовать журнал файлов, потребуется удалить расписание программа архивации данных.

## <a name="manifestation"></a>Проявление

рабочие процессы могут быть прерваны, и документация, которая ссылается на предыдущий метод доступа к функции программа архивации данных и восстановления, потребуется обновить, чтобы отразить эти изменения.

## <a name="mitigation"></a>Меры по снижению риска

Приложения, которые могут запускать резервное копирование и восстановление, должны быть перезаписаны, чтобы проверить, включен ли журнал файлов, и позволить пользователям выбрать предпочтительный метод.
