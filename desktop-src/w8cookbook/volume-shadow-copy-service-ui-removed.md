# <a name="previous-versions-ui-removed-for-local-volumes"></a>Пользовательский интерфейс предыдущих версий удален для локальных томов

## <a name="platform"></a>Платформа

**Клиенты** — Windows 8 


## <a name="description"></a>Описание

Более ранние версии Windows позволяли пользователям восстанавливать предыдущие версии отдельных файлов из "теневых копий" локальных томов. Эти теневые копии создаются функцией "предыдущие версии" через службу теневого копирования томов (VSS). В Windows 7 пользователи могли настраивать и планировать теневые копии в пользовательском интерфейсе защиты системы, доступном через свойства системы.

Однако предыдущие версии редко использовались и негативно влияли на общую производительность Windows; в результате функция была удалена. В Windows 8 эти функции больше не доступны:

-   Возможность просмотра, поиска или восстановления предыдущих версий файлов с помощью пользовательского интерфейса предыдущих версий
-   Возможность настройки или планирования предыдущих версий файлов с помощью пользовательского интерфейса защиты системы

Однако пользователи по-прежнему могут использовать вкладку предыдущие версии при доступе к общим папкам на сервере Windows.

## <a name="manifestation"></a>Проявление

Параметр предыдущие версии больше не доступен для локальных томов в меню свойства проводника Windows.

## <a name="solution"></a>Решение

Разработчики, которым требуется создавать теневые копии локальных томов, по-прежнему могут сделать это путем вызова API-интерфейсов VSS в пользовательском коде.
