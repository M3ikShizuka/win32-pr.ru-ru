# <a name="secure-boot-feature-signing-requirements-for-kernel-mode-drivers"></a>Требования к подписывания функций безопасной загрузки для драйверов режима ядра

## <a name="platforms"></a>Платформы

**клиенты** — Windows 8  
**серверы** — Windows Server 2012  


## <a name="description"></a>Описание

в Windows 8 и Windows Server 2012, в том числе WinPE, ядро заблокировано, чтобы предотвратить появление вредоносных программ, введенных пакетами загрузки или корневых наборов, от обхода требований безопасности Windows операционной системы для подписанных драйверов.

Это изменение влияет на все драйверы режима ядра для устройств, поддерживающих безопасную загрузку по интерфейсу UEFI. для Windows 8 сертификации для клиентских компьютеров требуется безопасная загрузка UEFI, а для серверов — необязательно. Он не влияет на драйверы пользовательского режима.

Стандартная разработка для драйверов в режиме ядра включает в себя использование режима отладки в режиме ядра или параметра тестсигнинг данных конфигурации загрузки (BCD) &lt; &gt; . Оба эти параметра отключены, если включена безопасная загрузка.

## <a name="manifestation"></a>Проявление

Драйверы режима ядра не будут выполняться, если они не подписаны должным образом доверенным центром сертификации (ЦС). Операционная система не позволит запускать недоверенные драйверы и стандартные механизмы, такие как отладка ядра и тестсигнинг, не будут разрешены.

## <a name="mitigation"></a>Меры по снижению риска

Для тестирования драйверов необходимо отключить безопасную загрузку в BIOS, а также удовлетворить другие требования к подписи тестирования (см. ниже).

При более широком распространении драйверов они должны быть правильно подписаны доверенным центром сертификации.
