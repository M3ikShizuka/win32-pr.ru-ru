# <a name="virtual-disk-service-is-transitioning-to-windows-storage-management-api"></a>служба виртуальных дисков переходит в API управления служба хранилища Windows

## <a name="platforms"></a>Платформы

**клиенты** — Windows 8  
**серверы** — Windows Server 2012  


## <a name="description"></a>Описание

начиная с Windows 8 и Windows Server 2012, интерфейс COM службы виртуальных дисков заменяется API управления служба хранилища, интерфейсом программирования на основе WMI. для управления подсистемами хранения, (Windows) дисками, разделами и томами настоятельно рекомендуется использовать API управления служба хранилища. дополнительные сведения см. в разделе [Windows служба хранилища управления API](/previous-versions/windows/desktop/stormgmt/windows-storage-management-api-portal).

Для всех случаев использования, за исключением зеркальных томов загрузки (с использованием зеркального тома для размещения операционной системы), динамические диски являются устаревшими. для данных, требующих устойчивости к сбою диска, используйте дисковые пространства, отказоустойчивое решение для виртуализации хранилища. дополнительные сведения см. в разделе [дисковые пространства Technical Preview](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh831739(v=ws.11)).

вы можете продолжать использовать DiskPart, diskraid и управление дисками в течение срока действия, но эти средства не будут работать с дисковые пространства или с другими новыми инструментарий управления Windows (WMI) (WMI) Windows api управления служба хранилища или с помощью встроенных служебных программ управления хранением или клиентов.


| API-интерфейсы | служба хранилища Подсистемы | Базовые диски | Динамические диски | Дисковые пространства |
| --- | --- | --- | --- | --- |
| ПОДДЕРЖКИ | Да | Да | Да | Нет |
| WMI | Да | Да | Нет | Да |
| DiskPart | Недоступно | Да | Да | Нет |
| DiskRAID |  Да | Недоступно | Недоступно | Недоступно |
| Графический интерфейс для руководства по дискам | Недоступно | Да | Да | Нет |
| PowerShell | Да | Да | Нет | Да |
| дисковые пространства Панель управления | Недоступно | Нет | Нет | Да |


Результатом этого перехода будет повышение устойчивости, доступности и масштабируемости хранилища. унифицированный язык написания сценариев и программирования, сокращение затрат на управление хранением и упрощенное управление удаленными хранилищами.

## <a name="manifestation"></a>Проявление

программы DiskPart и diskraid, используемые в среде VDS, не поддерживают новые дисковые пространства. аналогичным образом, новая служебная программа служба хранилища PowerShell не поддерживает устаревшие динамические диски.

## <a name="mitigation"></a>Меры по снижению риска

корпорация майкрософт настоятельно рекомендует создавать новые приложения управления хранением на Windows служба хранилища api управления, и вы планируете перенести существующие приложения, основанные на инфраструктуре VDS, в api Windows служба хранилища управления во время стандартных циклов обновления.
