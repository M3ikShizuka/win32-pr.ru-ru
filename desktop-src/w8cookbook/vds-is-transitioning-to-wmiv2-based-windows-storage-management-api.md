# <a name="virtual-disk-service-is-transitioning-to-windows-storage-management-api"></a>Служба виртуальных дисков переходит на API управления хранилищами Windows

## <a name="platforms"></a>Платформы

**Клиенты** — Windows 8  
**Серверы** — Windows Server 2012  


## <a name="description"></a>Описание

Начиная с Windows 8 и Windows Server 2012, интерфейс COM службы виртуальных дисков заменяется API управления хранением, интерфейсом программирования на основе WMI. Для управления подсистемами хранения, дисками, разделами и томами настоятельно рекомендуется использовать API управления хранилищами. Дополнительные сведения см. в разделе [API управления хранилищем Windows](/previous-versions/windows/desktop/stormgmt/windows-storage-management-api-portal).

Для всех случаев использования, за исключением зеркальных томов загрузки (с использованием зеркального тома для размещения операционной системы), динамические диски являются устаревшими. Для данных, требующих устойчивости к сбою диска, используйте дисковые пространства — отказоустойчивое решение для виртуализации хранилища. Дополнительные сведения см. в статье [Предварительная версия дисковых пространств](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh831739(v=ws.11)).

Вы можете продолжать использовать DiskPart, DiskRAID и Управление дисками в течение срока действия, но эти средства не будут работать с дисковыми пространствами или с другими новыми API-интерфейсами управления хранилищами Windows инструментарий управления Windows (WMI) (WMI) или встроенными служебными программами управления хранением данных или клиентами.


| Программные интерфейсы | Подсистемы хранения | Базовые диски | Динамические диски | Дисковые пространства |
| --- | --- | --- | --- | --- |
| ПОДДЕРЖКИ | Да | Да | Да | Нет |
| WMI | Да | Да | Нет | Да |
| DiskPart | Недоступно | Да | Да | Нет |
| DiskRAID |  Да | Недоступно | Недоступно | Недоступно |
| Графический интерфейс для руководства по дискам | Недоступно | Да | Да | Нет |
| PowerShell | Да | Да | Нет | Да |
| Панель управления дисковых пространств | Недоступно | Нет | Нет | Да |


Результатом этого перехода будет повышение устойчивости, доступности и масштабируемости хранилища. унифицированный язык написания сценариев и программирования, сокращение затрат на управление хранением и упрощенное управление удаленными хранилищами.

## <a name="manifestation"></a>Проявление

Служебные программы DiskPart и DiskRAID, используемые в среде VDS, не поддерживают новые дисковые пространства. Аналогичным образом, Новая служебная программа PowerShell для хранения данных не поддерживает устаревшие динамические диски.

## <a name="mitigation"></a>Меры по снижению риска

Корпорация Майкрософт настоятельно рекомендует создавать новые приложения управления хранением в API управления хранилищем Windows и планирует перенести существующие приложения, основанные на инфраструктуре VDS, в API управления хранилищами Windows во время стандартных циклов обновления.
