# <a name="new-file-history-feature"></a>Новая функция истории файлов

## <a name="platform"></a>Платформа

**клиенты** — Windows 8 


## <a name="description"></a>Описание

Новая функция журнала файлов заменяет существующую функцию резервного копирования и восстановления и обеспечивает защиту пользовательских файлов, хранящихся в библиотеках пользователей. Предоставляется набор интерфейсов API, позволяющих разработчикам программно включать журнал файлов и выбирать конкретный целевой объект хранилища. Документация по этим API-интерфейсам доступна на сайте MSDN.

это может повлиять на рабочие процессы, связанные с защитой данных и документацией, которые зависят от функции программа архивации данных и восстановления.

Мы не рекомендуем использовать обе функции одновременно. Журнал файлов проверяет, существует ли расписание резервного копирования и является ли оно активным. Если он найден, он не позволит пользователям включать его. в этом случае пользователям, желающим использовать журнал файлов, потребуется удалить расписание программа архивации данных.

## <a name="manifestation"></a>Проявление

рабочие процессы могут быть прерваны, а документация по предыдущему методу для доступа к функции программа архивации данных и восстановления потребуется обновить, чтобы отразить эти изменения.

## <a name="solution"></a>Решение

Пересмотреть приложения, содержащие рабочие процессы, на которые отрицательно повлияла новая функция журнала файлов, для устранения конфликтов и внедрения нового набора API.

## <a name="tests"></a>Тесты

API позволяет разработчикам определить, включен ли журнал файлов.
