# <a name="resilient-file-system"></a>Устойчивая файловая система

## <a name="platform"></a>Платформа

**Серверы** — Windows Server 2012 

## <a name="description"></a>Описание

Отказоустойчивая файловая система (ReFS) — это новая локальная файловая система. Это позволяет максимально повысить доступность данных, несмотря на ошибки, которые могут привести к потере данных или простою. Целостность данных гарантирует, что критически важные для бизнеса данные защищаются от ошибок и доступны при необходимости. Его архитектура разработана для обеспечения масштабируемости и производительности в эпоху постоянно увеличивающихся размеров наборов данных и динамических рабочих нагрузок.

Основные функции ReFS:

-   **Целостность**: ReFS хранит данные, чтобы защитить их от многих распространенных ошибок, которые могут привести к потере данных. Метаданные файловой системы всегда защищены. При необходимости данные пользователя могут быть защищены отдельно для каждого тома, каталога или каждого файла. Если происходит повреждение, ReFS может обнаруживать и при настройке с помощью дисковых пространств автоматически исправлять повреждение. В случае системной ошибки ссылки на ReFS предназначены для быстрого восстановления после этой ошибки без потери пользовательских данных.
-   **Доступность**. ReFS предназначена для определения приоритета доступности данных. Если в ReFS возникает повреждение и его не удается исправить автоматически, оперативный процесс восстановления локализуется в области повреждения, что не требует какого-либо времени. Вкратце, если происходит повреждение, ReFS будет оставаться в сети.
-   **Масштабируемость**. ReFS разработана для наборов данных на сегодняшний день и размеров набора данных завтра; Она оптимизирована для обеспечения высокой масштабируемости.
-   **Совместимость приложений**. чтобы развернуть AppCompat, ReFS поддерживает набор функций NTFS и интерфейсов API Win32, которые широко приняты.
-   **Упреждающий код ошибки**. возможности целостности ReFS используются средством проверки целостности данных ("средство очистки"), которое периодически проверяет том, пытается определить скрытые повреждения, а затем заблаговременно запускает восстановление поврежденных данных.
