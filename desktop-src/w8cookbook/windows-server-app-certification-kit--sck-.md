# <a name="microsoft-platform-ready-test-tool"></a>Средство тестирования, готовое для платформы Майкрософт

## <a name="platform"></a>Платформа

 **Серверы** — windows Server 2012 \| Windows Server 2012 R2  

## <a name="description"></a>Описание

Средство тестирования готовности платформы Майкрософт (MPR) используется для обеспечения готовности приложений платформы и проверки соответствия требованиям сертификации для приложений Windows Server 2012 и Windows Server 2012 R2.

Корпорация Майкрософт настоятельно рекомендует включать средство тестирования многопротокольного управления в процессы сборки и тестирования программного обеспечения, обеспечивая готовность платформы до выпуска приложений на рынке. Средство тестирования многопротокольного алгоритма также является рекомендуемым средством для тестирования бизнес-приложений и оценки серверных продуктов на совместимость с платформой перед принятием решений о приобретении.

## <a name="requirements"></a>Требования

Средство тестирования многопротокольного управления включает автоматические тесты для:

-   Установщик Windows
-   Установка и основная функциональность
-   Драйверы и безопасность
-   Рекомендации

Самостоятельное тестирование и отправка результатов для большинства серверных приложений займет не более 2 часов; более сложные приложения могут занять около 4 часов.

Все драйверы должны отдельно проходить тестирование сертификации оборудования Windows и быть подписаны корпорацией Майкрософт для Windows Server 2012.

## <a name="usage"></a>Использование

Чтобы протестировать приложения, выполните следующие действия.

1.  Установите последнюю конфигурацию Windows Server 2012 (с полным ГРАФИЧЕСКИм интерфейсом сервера, минимального интерфейса сервера или Server Core) в соответствии с требованиями приложения.
2.  Ознакомьтесь с требованиями к сертификации.
3.  В чистой системе запустите средство тестирования многопротокольного режима в режиме полного пользовательского интерфейса или в режиме командной строки.
4.  Завершите процесс самостоятельного тестирования.
5.  Проверьте результаты и при необходимости исправьте приложение.
6.  Войдите в систему и отправьте результаты на портал готовности платформы Майкрософт для перечисления в каталоге Windows Server и использовании логотипа.
