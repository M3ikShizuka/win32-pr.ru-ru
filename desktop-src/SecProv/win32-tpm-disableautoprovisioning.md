# <a name="win32_tpmdisableautoprovisioning-method"></a>\_TPM Win32::D метод исаблеаутопровисионинг

Отключает автоматическую подготовку доверенного платформенного модуля, если в настоящее время он включен. Операция не действует, если автоматическая подготовка уже отключена. по умолчанию автоматическая подготовка включена для Windows 8 и Windows Server 2012.

Этот метод доступен только локальным администраторам.

## <a name="syntax"></a>Синтаксис


```C++
uint32 DisableAutoProvisioning(
  [in, optional] BOOL OnlyForNextBoot
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Онлифорнекстбут* \[ в необязательное\]
</dt> <dd>

Если задано **значение true**, автоматическая подготовка отключена только для следующей загрузки. Если задано значение **false** или не указано, автоматическая подготовка будет отключена.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Могут возвращаться все ошибки доверенного платформенного модуля, а также ошибки, относящиеся к [базовым службам TPM](../tbs/tbs-return-codes.md) .

Ниже приведены общие коды возврата.



| Возвращаемый код и значение                                                                                                                                 | Описание                           |
|---------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl> | Метод успешно выполнен.<br/> |
