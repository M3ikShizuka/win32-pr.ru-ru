# <a name="iskeyprotectoravailable-method-of-the-win32_encryptablevolume-class"></a>Метод Искэйпротектораваилабле \_ класса Win32 енкриптаблеволуме

Метод **искэйпротектораваилабле** класса [**Win32 \_ енкриптаблеволуме**](win32-encryptablevolume.md) указывает, доступны ли для тома предохранители.

Если указан тип предохранителя, то метод указывает, доступны ли для тома средства защиты указанного типа.

## <a name="syntax"></a>Синтаксис


```mof
uint32 IsKeyProtectorAvailable(
  [in, optional] uint32  KeyProtectorType,
  [out]          boolean IsKeyProtectorAvailable
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Кэйпротектортипе* \[ в необязательное\]
</dt> <dd>

Тип: **UInt32**

Целое число без знака, указывающее тип запрашиваемого предохранителя ключа тома.

Если этот параметр не указан, запрашиваются все доступные предохранители ключа тома.



| Значение                                                                         | Значение                                                          |
|-------------------------------------------------------------------------------|------------------------------------------------------------------|
| <dl> <dt>0</dt> </dl>  | Все типы.<br/> Все предохранители ключа запрашиваются.<br/> |
| <dl> <dt>1</dt> </dl>  | Доверенный платформенный модуль (TPM) (доверенный платформенный модуль).<br/>                        |
| <dl> <dt>2</dt> </dl>  | Внешний ключ.<br/>                                         |
| <dl> <dt>3</dt> </dl>  | Числовой пароль.<br/>                                   |
| <dl> <dt>4</dt> </dl>  | TPM и ПИН-код.<br/>                                          |
| <dl> <dt>5</dt> </dl>  | TPM и ключ запуска.<br/>                                  |
| <dl> <dt>6</dt> </dl>  | TPM, ПИН-код и ключ запуска.<br/>                          |
| <dl> <dt>7</dt> </dl>  | Открытый ключ.<br/>                                           |
| <dl> <dt>8</dt> </dl>  | Кодов.<br/>                                           |
| <dl> <dt>9</dt> </dl>  | Сертификат TPM<br/>                                       |
| <dl> <dt>10</dt> </dl> | Идентификатор безопасности (SID)<br/>                             |



 

</dd> <dt>

*Искэйпротектораваилабле* \[ заполняет\]
</dt> <dd>

Тип: **Boolean**

Логическое значение, указывающее, существует ли на томе предохранитель ключа тома указанного типа.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **UInt32**

При сбое этот метод возвращает один из следующих кодов или другой код ошибки.



| Возвращаемый код и значение                                                                                                                                                         | Описание                                                                                                |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl>                         | Метод успешно выполнен.<br/>                                                                      |
| <dl> <dt>**Д \_ INVALIDARG**</dt> <dt>2147942487 (0x80070057)</dt> </dl> | Параметр *кэйпротектортипе* указан, но не ссылается на допустимый тип предохранителя ключа.<br/> |



 

## <a name="remarks"></a>Комментарии

Файлы MOF-файл (MOF) содержат определения для классов инструментарий управления Windows (WMI) (WMI). MOF-файлы не устанавливаются в составе Windows SDK. Они устанавливаются на сервере при добавлении связанной роли с помощью диспетчер сервера. Дополнительные сведения о файлах MOF см. в разделе [MOF-файл (MOF)](../wmisdk/managed-object-format--mof-.md).
