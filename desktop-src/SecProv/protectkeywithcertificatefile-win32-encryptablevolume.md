# <a name="protectkeywithcertificatefile-method-of-the-win32_encryptablevolume-class"></a>Метод Протекткэйвисцертификатефиле \_ класса Win32 енкриптаблеволуме

Метод **протекткэйвисцертификатефиле** класса [**Win32 \_ енкриптаблеволуме**](win32-encryptablevolume.md) проверяет [*идентификатор объекта*](../secgloss/o-gly.md) (OID) расширенного использования ключа (EKU) предоставленного сертификата.

## <a name="syntax"></a>Синтаксис


```mof
uint32 ProtectKeyWithCertificateFile(
  [in, optional] string FriendlyName,
  [in]           string FileName,
  [out]          string VolumeKeyProtectorID
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*FriendlyName* \[ в необязательное\]
</dt> <dd>

Тип: **строка**

Строка, указывающая назначенный пользователем строковый идентификатор для предохранителя ключа. Если этот параметр не указан, параметр *FriendlyName* создается с использованием имени субъекта в сертификате.

</dd> <dt>

*Имя файла* \[ окне\]
</dt> <dd>

Тип: **строка**

Строка, указывающая расположение и имя CER-файла, используемого для включения BitLocker. сертификат шифрования должен быть экспортирован в формате cer ([*Distinguished Encoding Rules*](../secgloss/d-gly.md) (DER) binary [*x. 509*](../secgloss/x-gly.md) или Base-64 encoded x. 509). Сертификат шифрования может быть создан из PKI Майкрософт, стороннего PKI или самозаверяющего.

</dd> <dt>

*Волумекэйпротекторид* \[ заполняет\]
</dt> <dd>

Тип: **строка**

Строка, однозначно определяющая созданный предохранитель ключа, который может использоваться для управления предохранителем ключа.

Если диск поддерживает аппаратное шифрование, а BitLocker не использует нестандартное владение, для строки идентификатора задается значение "BitLocker", а предохранитель ключа записывается в метаданные диапазона.

</dd> </dl>

Тип: **UInt32**

## <a name="remarks"></a>Remarks

Если OID не совпадает с идентификатором, связанным с контроллером службы в реестре, этот метод завершается ошибкой. Это не дает пользователю вручную настраивать предохранители агента восстановления данных (DRA) на томе. Службы агентов DRA должны быть установлены только службой.
