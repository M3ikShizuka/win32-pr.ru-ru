# <a name="getphysicalpresenceresponse-method-of-the-win32_tpm-class"></a>Метод Жетфисикалпресенцереспонсе \_ класса TPM Win32

Метод **жетфисикалпресенцереспонсе** класса [**\_ TPM Win32**](win32-tpm.md) возвращает результаты выполнения операции физического присутствия доверенного платформенного модуля. Используйте метод [**сетфисикалпресенцерекуест**](setphysicalpresencerequest-win32-tpm.md) для запроса операции.

## <a name="syntax"></a>Синтаксис


```mof
uint32 GetPhysicalPresenceResponse(
  [out] uint32 Request,
  [out] uint32 Response
);
```

*Ответ* \[ заполняет\]
</dt> <dd>

Тип: **UInt32**

Целочисленное значение, указывающее результаты выполнения операции физического присутствия TPM.

Операция физического присутствия завершается успешно, если физически представленный пользователь подтверждает операцию и если операция выполняется без ошибок.

Это значение может содержать любую ошибку доверенного платформенного модуля. В следующей таблице перечислены некоторые распространенные ответы об ошибках.



| Значение                                                                                                                                                                                                                                                                    | Значение                                                             |
|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------|
| <dl> <dt>0</dt> </dl>                                                                                                                                                                                             | Запрошенная операция TPM выполнена успешно.<br/>              |
| <span id="TPM_E_PPI_USER_ABORT"></span><span id="tpm_e_ppi_user_abort"></span><dl> <dt>**Доверенный платформенный модуль \_ \_ \_ Пользовательское \_ прерывание по PPI**</dt> <dt>2150171393 (0x80290301)</dt> </dl>       | Пользователь отклонил запрошенную операцию TPM.<br/>           |
| <span id="TPM_E_PPI_BIOS_FAILURE"></span><span id="tpm_e_ppi_bios_failure"></span><dl> <dt>**Доверенный платформенный модуль \_ E \_ PPI \_ BIOS \_ сбой**</dt> <dt>2150171394 (0x80290302)</dt> </dl> | Ошибка BIOS при выполнении операции TPM.<br/> |



 

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **UInt32**

Могут возвращаться все ошибки доверенного платформенного модуля, а также ошибки, относящиеся к базовым службам TPM.

В следующей таблице перечислены некоторые распространенные коды возврата.



| Возвращаемый код и значение                                                                                                                                                                      | Описание                                                                                      |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl>                                      | Метод успешно выполнен.<br/>                                                            |
| <dl> <dt>**Доверенный платформенный модуль \_ E \_ PPI \_ ACPI \_ сбой**</dt> <dt>2150171392 (0x80290300)</dt> </dl> | Произошла ошибка оборудования. Для получения дополнительных сведений обратитесь к изготовителю компьютера.<br/> |
