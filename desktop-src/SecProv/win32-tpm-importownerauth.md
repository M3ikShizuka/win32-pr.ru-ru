# <a name="win32_tpmimportownerauth-method"></a>\_Метод Win32 TPM:: импортовнераус

Импортирует сведения авторизации владельца для доверенного платформенного модуля, который уже принадлежит реестру операционной системы. Эта операция сначала проверит, правильно ли указаны сведения авторизации владельца. Если это верно, метод импортирует эти сведения в реестр.

Этот метод доступен только локальным администраторам.

## <a name="syntax"></a>Синтаксис


```C++
uint32 ImportOwnerAuth(
  [in] STRING OwnerAuth
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Овнераус* \[ окне\]
</dt> <dd>

Допустимые сведения авторизации владельца для TPM.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Могут возвращаться все ошибки доверенного платформенного модуля, а также ошибки, относящиеся к [базовым службам TPM](../tbs/tbs-return-codes.md) .

Ниже приведены общие коды возврата.



| Возвращаемый код и значение                                                                                                                                 | Описание                           |
|---------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl> | Метод успешно выполнен.<br/> |



 

## <a name="remarks"></a>Remarks

Этот метод особенно полезен в сценариях, где доверенный платформенный модуль находится в состоянии "готов к снижению функциональности", и требует, чтобы импорт авторизации владельца был полностью готов или в сценариях с двойной загрузкой, где одна из операционных систем владеет доверенным платформенным модулем, а сведения авторизации владельца для TPM недоступны в другой операционной системе.
