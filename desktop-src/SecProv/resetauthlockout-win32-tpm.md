# <a name="resetauthlockout-method-of-the-win32_tpm-class"></a>Метод Ресетауслоккаут \_ класса TPM Win32

Метод **ресетауслоккаут** класса [**\_ TPM Win32**](win32-tpm.md) сбрасывает период времени ожидания или другой механизм, реализуемый производителями TPM для защиты от атак из словаря на основе значений авторизации доверенного платформенного модуля. При атаке по словарю злоумышленник пытается угадать правильное значение авторизации TPM, выполнив неполную попытку выполнить все возможные значения.

Используйте этот метод, если доверенный платформенный модуль заблокирован из-за слишком большого числа неудачных попыток входа в систему авторизации владельца или других значений авторизации. Если доверенный платформенный модуль заблокирован, некоторые или все команды, выданные доверенному платформенному модулю, будут возвращать ошибку, а \_ \_ Блокировка защиты от TPM 0x80280803 — \_ \_ запущена.

> [!Note]  
> Этот метод можно использовать только один раз, если доверенный платформенный модуль заблокирован. Если авторизация владельца, предоставленная этому методу, неверна, доверенный платформенный модуль блокируется на весь период ожидания, а дополнительные попытки сброса блокировки завершатся неудачей.

 

## <a name="syntax"></a>Синтаксис


```mof
uint32 ResetAuthLockOut(
  [in, optional] string OwnerAuth
);
```

## <a name="remarks"></a>Комментарии

Этот метод вызывает команду TPM \_ ресетлокквалуе в доверенном платформенном модуле. Точное поведение этого метода зависит от производителей доверенного платформенного модуля. Документация от изготовителя компьютера или доверенного платформенного модуля может предоставить дополнительные сведения о реализации механизма атак с помощью антисловарей.

Как правило, производители могут обнаруживать атаки из словарей, отслеживая неудачные проверки подлинности. Если число или частота сбоев становится достаточно высокой, TPM блокирует дальнейшие команды в течение определенного времени. Как правило, начальное время ожидания будет небольшим, чтобы позволить законному пользователю исправить ситуацию. Если сбои продолжаются, то продолжительность каждого последующего времени ожидания может увеличиваться быстро.

Файлы MOF-файл (MOF) содержат определения для классов инструментарий управления Windows (WMI) (WMI). MOF-файлы не устанавливаются в составе Windows SDK. Они устанавливаются на сервере при добавлении связанной роли с помощью диспетчер сервера. Дополнительные сведения о файлах MOF см. в разделе [MOF-файл (MOF)](../wmisdk/managed-object-format--mof-.md).
