# <a name="disableautounlock-method-of-the-win32_encryptablevolume-class"></a>Метод Дисаблеаутаунлокк \_ класса Win32 енкриптаблеволуме

Метод **дисаблеаутаунлокк** класса [**Win32 \_ енкриптаблеволуме**](win32-encryptablevolume.md) удаляет внешний ключ, сохраненный в текущем томе операционной системы, чтобы при подключении не автоматически разблокировался том данных, например, когда к компьютеру подключены съемные устройства памяти.

Если автоматическая разблокировка отключена или приостановлена, для разблокировки тома необходимо использовать методы [**унлокквисекстерналкэй**](unlockwithexternalkey-win32-encryptablevolume.md) или [**унлокквиснумерикалпассворд**](unlockwithnumericalpassword-win32-encryptablevolume.md) .

## <a name="syntax"></a>Синтаксис


```mof
uint32 DisableAutoUnlock();
```



## <a name="parameters"></a>Параметры

Этот метод не имеет параметров.

## <a name="return-value"></a>Возвращаемое значение

Тип: **UInt32**

При сбое этот метод возвращает один из следующих кодов или другой код ошибки.



| Возвращаемый код и значение                                                                                                                                                                       | Описание                                                                                 |
|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl>                                       | Метод успешно выполнен.<br/>                                                       |
| <dl> <dt> **\_ Том ФВЕ \_ E \_ не \_ привязан**</dt> <dt>2150694935 (0x80310017)</dt> </dl> | Автоматическая разблокировка тома отключена.<br/>                                   |
| <dl> <dt>**ФВЕ \_ E \_ не \_ активировано**</dt> <dt>2150694920 (0x80310008)</dt> </dl>      | Средство BitLocker не включено в томе. Добавьте предохранитель ключа, чтобы включить BitLocker.<br/> |
| <dl> <dt>**ФВЕ \_ E \_ не \_ DATA \_ Volume**</dt> <dt>2150694937 (0x80310019)</dt> </dl>   | Этот метод не может быть выполнен для текущего тома операционной системы.<br/>      |



 

## <a name="remarks"></a>Комментарии

Если ошибки не возвращаются, этот метод удаляет из реестра все идентификаторы предохранителя томов и внешние ключи, используемые для автоматического разблокировки тома.

Файлы MOF-файл (MOF) содержат определения для классов инструментарий управления Windows (WMI) (WMI). MOF-файлы не устанавливаются в составе Windows SDK. Они устанавливаются на сервере при добавлении связанной роли с помощью диспетчер сервера. Дополнительные сведения о файлах MOF см. в разделе [MOF-файл (MOF)](../wmisdk/managed-object-format--mof-.md).
