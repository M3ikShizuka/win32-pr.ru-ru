# <a name="protectkeywithnumericalpassword-method-of-the-win32_encryptablevolume-class"></a>Метод Протекткэйвиснумерикалпассворд \_ класса Win32 енкриптаблеволуме

Метод **протекткэйвиснумерикалпассворд** класса [**Win32 \_ енкриптаблеволуме**](win32-encryptablevolume.md) защищает ключ шифрования тома с помощью специального отформатированного пароля с 48 цифрами. Этот числовой пароль можно использовать для восстановления после сбоев проверки подлинности других предохранителей ключа (например, TPM).

Для тома создается предохранитель ключа типа "числовой пароль".

Используйте метод [**иснумерикалпассвордвалид**](isnumericalpasswordvalid-win32-encryptablevolume.md) для проверки формата числового пароля.

## <a name="syntax"></a>Синтаксис


```mof
uint32 ProtectKeyWithNumericalPassword(
  [in, optional] string FriendlyName,
  [in, optional] string NumericalPassword,
  [out]          string VolumeKeyProtectorID
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*FriendlyName* \[ в необязательное\]
</dt> <dd>

Тип: **строка**

Строка, указывающая назначенный пользователем идентификатор для этого предохранителя ключа. Если этот параметр не указан, используется пустое значение.

</dd> <dt>

*Нумерикалпассворд* \[ в необязательное\]
</dt> <dd>

Тип: **строка**

Строка, указывающая специально отформатированный числовой пароль в 48 цифр.

Числовой пароль должен содержать 48 цифр. Эти цифры можно разделить на 8 групп из 6 цифр, с последней цифрой в каждой группе, указывающей на значение контрольной суммы для группы. Каждая группа из 6 цифр должна быть кратной 11 и должна быть меньше 720896. При условии, что группа из шести цифр помечена как x1, x2, X3, X4, x5 и X6, контрольная сумма X6 вычисляется как – x1 + x2 – X3 + X4 – x5 mod 11.

Группы цифр можно разделять пробелами или дефисами. Таким образом, "XXXXXX-XXXXXX-XXXXXX-XXXXXX-XXXXXX-XXXXXX-XXXXXX-XXXXXX" или "XXXXXX XXXXXX XXXXXX XXXXXX (xxxxxx) XXXXXX XXXXXX" может также содержать допустимые числовые пароли.

Если числовой пароль не указан, он создается случайным образом. Используйте метод [**жеткэйпротекторнумерикалпассворд**](getkeyprotectornumericalpassword-win32-encryptablevolume.md) для получения пароля, созданного случайным образом.

</dd> <dt>

*Волумекэйпротекторид* \[ заполняет\]
</dt> <dd>

Тип: **строка**

Строка, которая является уникальным идентификатором, связанным с созданным предохранителем, который можно использовать для управления предохранителем ключа.

Если диск поддерживает аппаратное шифрование, а BitLocker не использует нестандартное владение, для строки идентификатора задается значение "BitLocker", а предохранитель ключа записывается в метаданные диапазона.

</dd> </dl>
