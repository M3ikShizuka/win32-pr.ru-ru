# <a name="isphysicalpresencehardwareenabled-method-of-the-win32_tpm-class"></a>Метод Исфисикалпресенцехардваринаблед \_ класса TPM Win32

Метод **исфисикалпресенцехардваринаблед** класса [**\_ TPM Win32**](win32-tpm.md) указывает, можно ли установить физическое присутствие на платформе с аппаратным сигналом. Это значение настраивается производителем платформы в зависимости от структуры платформы. Документация производителя платформы может предоставить дополнительные сведения.

## <a name="syntax"></a>Синтаксис


```mof
uint32 IsPhysicalPresenceHardwareEnabled(
  [out] boolean IsPhysicalPresenceHardwareEnabled
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Исфисикалпресенцехардваринаблед* \[ заполняет\]
</dt> <dd>

Тип: **Boolean**

Если **значение равно true**, физическому присутствию на платформе можно назначить аппаратный сигнал.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **UInt32**

Могут возвращаться все ошибки доверенного платформенного модуля, а также ошибки, относящиеся к базовым службам TPM.

Ниже приведены общие коды возврата.



| Возвращаемый код и значение                                                                                                                                 | Описание                           |
|---------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl> | Метод успешно выполнен.<br/> |



 

## <a name="remarks"></a>Комментарии

Файлы MOF-файл (MOF) содержат определения для классов инструментарий управления Windows (WMI) (WMI). MOF-файлы не устанавливаются в составе Windows SDK. Они устанавливаются на сервере при добавлении связанной роли с помощью диспетчер сервера. Дополнительные сведения о файлах MOF см. в разделе [MOF-файл (MOF)](../wmisdk/managed-object-format--mof-.md).
