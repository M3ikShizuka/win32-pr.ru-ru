# <a name="enableautounlock-method-of-the-win32_encryptablevolume-class"></a>Метод Енаблеаутаунлокк \_ класса Win32 енкриптаблеволуме

Метод **енаблеаутаунлокк** класса [**Win32 \_ енкриптаблеволуме**](win32-encryptablevolume.md) позволяет автоматически разблокировать том данных при подключении тома.

Автоматическое снятие блокировки сохраняет внешний ключ в операционной системе, который может автоматически разблокировать том на текущем томе операционной системы.

Чтобы использовать этот метод, том операционной системы уже должен быть защищен шифрование диска BitLocker или должен выполняться шифрование. Кроме того, уже должен существовать внешний ключ для тома данных. Используйте [**протекткэйвисекстерналкэй**](protectkeywithexternalkey-win32-encryptablevolume.md) для создания внешнего ключа, который может автоматически разблокировать том.

## <a name="syntax"></a>Синтаксис


```mof
uint32 EnableAutoUnlock(
  [in] string VolumeKeyProtectorID
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

</dt> <dd>

Строка, определяющая предохранитель ключа типа "внешний ключ", используемый для автоматического разблокировки тома.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **UInt32**

При сбое этот метод возвращает один из следующих кодов или другой код ошибки.



| Возвращаемый код и значение                                                                                                                                                                           | Описание                                                                                                                                                                  |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl>                                           | Метод успешно выполнен.<br/>                                                                                                                                        |
| <dl> <dt>**ФВЕ \_ E с \_ заблокированным \_ томом**</dt> <dt>2150694912 (0x80310000)</dt> </dl>          | Том заблокирован.<br/>                                                                                                                                             |
| <dl> <dt>**ФВЕ \_ E \_ не \_ активировано**</dt> <dt>2150694920 (0x80310008)</dt> </dl>          | Средство BitLocker не включено в томе. Добавьте предохранитель ключа, чтобы включить BitLocker. <br/>                                                                                 |
| <dl> <dt>**Д \_ INVALIDARG**</dt> <dt>2147942487 (0x80070057)</dt> </dl>                   | Параметр *волумекэйпротекторид* не ссылается на допустимый предохранитель ключа типа "внешний ключ".<br/>                                                          |
| <dl> <dt>**ФВЕ \_ E \_ не \_ DATA \_ Volume**</dt> <dt>2150694937 (0x80310019)</dt> </dl>       | Этот метод не может быть выполнен для текущего тома операционной системы.<br/>                                                                                       |

| <dl> <dt> **ФВЕ \_ E \_ \_ с привязкой к тому \_ уже**</dt> <dt>2150694943 (0x8031001F)</dt> </dl> | Автоматическое снятие блокировки тома включено ранее.<br/>                                                                                                    |



 

## <a name="remarks"></a>Комментарии

При наличии допустимого предохранителя ключа тома типа "внешний ключ" связанный 256-разрядный внешний ключ извлекается из предохранителя и сохраняется в реестре выполняющейся операционной системы вместе с ИДЕНТИФИКАТОРом предохранителя ключа тома.

Если внешний ключ, связанный с ИДЕНТИФИКАТОРом предохранителя ключа тома, удаляется, функции автоматического разблокировки тома отключаются или приостанавливаются.

> [!Note]  
> Съемные носители в настоящее время не поддерживаются.

 

Файлы MOF-файл (MOF) содержат определения для классов инструментарий управления Windows (WMI) (WMI). MOF-файлы не устанавливаются в составе Windows SDK. Они устанавливаются на сервере при добавлении связанной роли с помощью диспетчер сервера. Дополнительные сведения о файлах MOF см. в разделе [MOF-файл (MOF)](../wmisdk/managed-object-format--mof-.md).
