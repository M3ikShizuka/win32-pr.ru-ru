# <a name="getexternalkeyfilename-method-of-the-win32_encryptablevolume-class"></a>Метод Жетекстерналкэйфиленаме \_ класса Win32 енкриптаблеволуме

Метод **жетекстерналкэйфиленаме** класса [**Win32 \_ енкриптаблеволуме**](win32-encryptablevolume.md) возвращает имя файла, содержащего внешний ключ, если этот внешний ключ сохранен в расположение файла с помощью метода [**савикстерналкэйтофиле**](saveexternalkeytofile-win32-encryptablevolume.md) .

Этот метод применим только для предохранителей ключей типа "внешний ключ", "TPM, ПИН-код и ключ запуска" или "TPM и ключ запуска".

## <a name="syntax"></a>Синтаксис


```mof
uint32 GetExternalKeyFileName(
  [in]  string VolumeKeyProtectorID,
  [out] string FileName
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Волумекэйпротекторид* \[ окне\]
</dt> <dd>

Тип: **строка**

Уникальный строковый идентификатор, используемый для управления предохранителем ключа зашифрованного тома.

</dd> <dt>

*Имя файла* \[ заполняет\]
</dt> <dd>

Тип: **строка**

Строка, указывающая имя с расширением, но без пути к файлу для файла, содержащего внешний ключ.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **UInt32**

При сбое этот метод возвращает один из следующих кодов или другой код ошибки.



| Возвращаемый код и значение                                                                                                                                                                  | Описание                                                                                                                                                            |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**С \_ ОК**</dt> <dt>0 (0x0)</dt> </dl>                                  | Метод успешно выполнен.<br/>                                                                                                                                  |
| <dl> <dt>**Д \_ INVALIDARG**</dt> <dt>2147942487 (0x80070057)</dt> </dl>          | Параметр *волумекэйпротекторид* не ссылается на предохранитель ключа типа "внешний ключ", "TPM, ПИН-код и ключ запуска", "TPM и ключ запуска".<br/> |
| <dl> <dt>**ФВЕ \_ E с \_ заблокированным \_ томом**</dt> <dt>2150694912 (0x80310000)</dt> </dl> | Том заблокирован.<br/>                                                                                                                                       |
| <dl> <dt>**ФВЕ \_ E \_ не \_ активировано**</dt> <dt>2150694920 (0x80310008)</dt> </dl> | Средство BitLocker не включено в томе. Добавьте предохранитель ключа, чтобы включить BitLocker. <br/>                                                                           |



 

## <a name="remarks"></a>Комментарии

Файлы MOF-файл (MOF) содержат определения для классов инструментарий управления Windows (WMI) (WMI). MOF-файлы не устанавливаются в составе Windows SDK. Они устанавливаются на сервере при добавлении связанной роли с помощью диспетчер сервера. Дополнительные сведения о файлах MOF см. в разделе [MOF-файл (MOF)](../wmisdk/managed-object-format--mof-.md).
