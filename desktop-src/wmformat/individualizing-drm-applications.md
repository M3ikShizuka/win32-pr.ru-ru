# <a name="individualizing-drm-applications"></a>Приложения DRM индивидуализинг

Для повышения безопасности компонент DRM приложений может быть *индивидуальным*. Отдельное приложение — это одно из приложений, которое получило обновление до компонентов DRM, которое отличает его от всех остальных копий того же приложения. Владельцы содержимого могут потребовать, чтобы их защищенные файлы воспроизводились только в приложении, которое было индивидуальным.

Процесс индивидуализации начинается, когда приложение обращается к службе Microsoft индивидуализации, которая затем устанавливает обновление системы безопасности на компьютере пользователя. Поскольку служба индивидуализации обрабатывает сведения от пользователя, необходимо отобразить политику конфиденциальности Майкрософт или указать ссылку на эту страницу на веб-сайте Майкрософт: <https://privacy.microsoft.com/privacystatement> .

Индивидуальная настройка может быть выполнена различными способами. Например, если пользователь воспроизводит защищенный файл, может быть выполнена отдельная Индивидуальная защита. Запрос лицензии отправляется в [*службу лицензирования Windows Media*](wmformat-glossary.md), которая проверяет сертификат приложения, запрашивающего [*лицензию*](wmformat-glossary.md). Если компонент DRM приложения не является индивидуальным, служба лицензирования Windows Media отказывается выдавать лицензию, поскольку она является политикой службы лицензирования Windows Media, выдавать лицензии только отдельным игрокам. Приложение может уведомить пользователя о необходимости обновления приложения. Если пользователь согласен, для индивидуализируйте компонента DRM приложения предоставляется обновление системы безопасности.

Для повышения удобства работы пользователей можно запустить процесс индивидуализации в качестве этапа обновления системы безопасности во время установки. После этого пользователь не прерывается для получения обновления системы безопасности при попытке воспроизведения защищенного файла. Кроме того, можно активно рекомендовать индивидуальную возможность, добавив к приложению команду меню **обновления безопасности** или кнопку.

> [!Note]  
> DRM не поддерживает версию этого пакета SDK на базе x64.
