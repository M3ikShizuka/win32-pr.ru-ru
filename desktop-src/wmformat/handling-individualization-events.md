# <a name="handling-individualization-events"></a>Обработка событий индивидуализации

Когда приложение с поддержкой DRM пытается открыть защищенный файл, компонент DRM проверяет атрибут [**DRM \_ дрмхеадер \_ индивидуализедверсион**](drm-drmheader-individualizedversion.md) в файле, который указывает минимальный уровень версии, необходимый для доступа к содержимому. все уровни компонента DRM работают со всеми 7,0 и более поздними версиями проигрыватель Windows Media и пакетом SDK Windows Media Format. Если уровень отдельных версий компонента DRM меньше требуемой версии, компонент DRM отправит событие **ВМТ \_ требуется \_ индивидуально** в метод [**ивмстатускаллбакк:: OnStatus**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmstatuscallback-onstatus) приложения. После этого приложение должно отобразить сообщение или диалоговое окно, предлагающее пользователям либо запустить, либо отменить обновление системы безопасности. Этот запрос необходим, поскольку в целях обеспечения конфиденциальности пользователи должны предоставить свое разрешение до установки обновления безопасности на компьютере.

> [!Note]  
> В заголовке содержимого указаны только первые две цифры для DRM \_ дрмверсион \_ индивидуализедверсион. Иными словами, для запроса компонента DRM уровня 2.2.0.1 заголовок будет содержать значение "2,2".

 

Чтобы запустить обновление безопасности и/или выполнить индивидуальную настройку, вызовите метод [**ивмдрмреадер:: индивидуализируйте**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmdrmreader-individualize) с параметром *dwFlags* , равным 1.

Необходимо выполнить обработку события **ВМТ \_ индивидуализируйте** в приложении. Это событие будет запускаться несколько раз компонентом DRM с состоянием процесса индивидуализации, указанным в параметре *pValue* , который приводится к указателю на структуру [**\_ \_ состояния WM индивидуализируйте**](wm-individualize-status.md) .

После успешного создания компонента DRM приложение получит событие **ВМТ \_ без \_ прав \_** , указывающее, что приложение теперь может получить лицензию на содержимое.

> [!Note]  
> DRM не поддерживает версию этого пакета SDK на базе x64.
