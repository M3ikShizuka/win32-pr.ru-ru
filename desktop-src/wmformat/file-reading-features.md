# <a name="file-reading-features"></a>Функции чтения файлов

чтение файлов ASF — это одна из основных возможностей пакета SDK для Windows Media Format. Поддерживаются два типа чтения: асинхронные и синхронные. Асинхронное чтение файлов обрабатывается объектом Reader. Синхронный объект чтения используется для синхронного чтения файлов. Дополнительные сведения о различных объектах чтения см. в разделе [объект Reader](reader-object.md) и [синхронный объект Reader](synchronous-reader-object.md).

В самом базовом сценарии асинхронного чтения файлов необходимо реализовать метод обратного вызова, который будет вызываться объектом Reader при готовности образцов. После того как вы начнете чтение файла, приложение ждет доставки образцов в метод обратного вызова. Асинхронное чтение полезно для приложений проигрывателя и поддерживает функции, недоступные при синхронном чтении. если приложению требуется считывать файлы из сетевой папки или взаимодействовать с сервером, на котором работает службы Windows Media, необходимо использовать объект reader. Недостатком объекта Reader является то, что для каждого доставляемого выхода используется отдельный поток. Кроме того, объект Reader не так гибок, как синхронное средство чтения, способное доставлять образцы.

В синхронном средстве чтения не нужно использовать методы обратного вызова. Вместо этого следует выбрать часть файла для чтения и извлечения образцов по одному с помощью вызовов метода. Синхронный читатель хорошо подходит для приложений, поддерживающих редактирование содержимого, где очень важно использовать быстрый доступ к конкретным образцам. Поскольку синхронное средство чтения не использует методы обратного вызова, вы можете создавать приложения для чтения файлов ASF с минимальными затратами на написание кода. однако синхронный модуль чтения не может открыть файл из сетевого расположения или взаимодействовать с сервером, на котором работает службы Windows Media, или считывать файлы, защищенные с помощью [*DRM*](wmformat-glossary.md).
