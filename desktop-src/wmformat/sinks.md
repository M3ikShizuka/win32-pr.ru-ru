# <a name="sinks"></a>Приемники

объект модуля записи пакета SDK Windows Media Format доставляет обработанное содержимое в приемники. Каждый приемник является объектом, который доставляет данные. Точка доставки зависит от типа приемника. Существует три типа приемников: приемники файлов, приемники сети и приемники push-уведомлений.

## <a name="file-sinks"></a>Приемники файлов

Приемники файлов записывают содержимое ASF в файл на локальном или сетевом диске. При использовании объекта модуля записи для записи файла без явного добавления приемника файла модуль записи создаст его, используя имя, передаваемое в [**ивмвритер:: сетаутпутфиленаме**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriter-setoutputfilename). Объекту модуля записи можно назначить несколько приемников файлов для записи содержимого в несколько файлов одновременно.

С помощью приемника файлов можно управлять множеством аспектов файла. Следующие функции доступны в приемнике файлов.

-   Мониторинг статистики файлов. Можно отслеживать размер и длительность создаваемого файла.
-   Создание файла с частичным содержимым. Приемники файлов можно настроить для начала записи содержимого в определенное время и для завершения записи в определенное время. Это позволяет создавать несколько файлов, содержащих разные разделы одного содержимого, на одном и том же этапе записи.

## <a name="network-sinks"></a>Приемники сети

Сетевые приемники пересылают содержимое на сетевой адрес. Чтение клиентов может подключиться к адресу для получения вещания.

## <a name="push-sinks"></a>Приемники push-уведомлений

приемники Push-уведомлений доставляют содержимое из модуля записи на сервер, на котором работает службы Windows Media. Приемники push-уведомлений обычно используются в сценариях, где один компьютер кодирует динамическое содержимое и доставляет его на один или несколько серверов для широкого распределения. Использование приемника push-уведомлений позволяет выделить компьютеры для выполнения конкретных задач, экономя пропускную способность и вычислительную мощность на каждом сервере.
