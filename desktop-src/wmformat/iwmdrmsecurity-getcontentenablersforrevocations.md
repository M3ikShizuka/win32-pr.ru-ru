# <a name="iwmdrmsecuritygetcontentenablersforrevocations-method"></a>Метод Ивмдрмсекурити:: Жетконтентенаблерсфорревокатионс

Метод **жетконтентенаблерсфорревокатионс** получает интерфейсы включения содержимого, которые обеспечивают возобновление компонентов на основе отозванных сертификатов.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetContentEnablersForRevocations(
  [in]      BYTE              **rgpbCerts,
  [in]      DWORD             *rgpdwCertSizes,
  [in]      GUID              **rgpguidCerts,
  [in]      DWORD             cCerts,
  [in]      HRESULT           hResultHint,
  [out]     IMFContentEnabler **prgContentEnablers,
  [in, out] DWORD             *pcContentEnablers
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*ргпбцертс* \[ окне\]
</dt> <dd>

Массив сертификатов для получения содержимого созидателями. Число элементов в массиве должно быть задано параметром *кцертс*.

</dd> <dt>

*ргпдвцертсизес* \[ окне\]
</dt> <dd>

Массив, содержащий размеры сертификатов в массиве *ргпбцертс* . Число элементов в массиве должно быть задано параметром *кцертс*.

</dd> <dt>

*ргпгуидцертс* \[ окне\]
</dt> <dd>

Массив, содержащий типы сертификатов в массиве *ргпбцертс* . Число элементов в массиве должно быть задано параметром *кцертс*. Для каждого элемента массива используйте одно из следующих значений.



| Константа GUID                 | Описание                                                    |
|-------------------------------|----------------------------------------------------------------|
| \_приложение РЕВОКАТИОНТИПЕ \_ WMDRM    | Указывает сертификат приложения.                          |
| \_устройство WMDRM ревокатионтипе \_ | Указывает сертификат устройства.                                |
| \_КАРДЕА WMDRM ревокатионтипе \_ | Указывает сертификат Windows Media DRM для сетевых устройств. |



 

</dd> <dt>

*кцертс* \[ окне\]
</dt> <dd>

Число сертификатов для получения содержимого созидателями. Это число элементов в массиве *ргпбцертс* , массиве *ргпдвцертсизес* и массиве *ргпгуидцертс* .

</dd> <dt>

*хресулсинт* \[ окне\]
</dt> <dd>

Возвращенное значение, полученное из операции, которая завершилась сбоем из-за отозванного сертификата. Если вы не вызываете в ответ на вызов метода со сбоем, задайте для параметра значение S \_ ОК.

</dd> <dt>

*пргконтентенаблерс* \[ заполняет\]
</dt> <dd>

Массив, который получает адреса только что созданных интерфейсов **имфконтентенаблер** . Задайте значение **null** , чтобы получить количество созидателями содержимого в параметре *пкконтентенаблерс* .

</dd> <dt>

*пкконтентенаблерс* \[ в, out\]
</dt> <dd>

Число элементов в массиве *пргконтентенаблерс* . Если *пргконтентенаблерс* имеет **значение NULL**, это значение устанавливается равным числу необходимых созидателями содержимого на выходе.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Метод возвращает **значение HRESULT**. Допустимые значения включают, но не ограничиваются, значения, приведенные в следующей таблице.



| Код возврата                                                                          | Описание                      |
|--------------------------------------------------------------------------------------|----------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl> | Метод выполнен успешно.<br/> |



 

## <a name="remarks"></a>Комментарии

Если вы используете интерфейс **имфконтентенаблер** для обновления отозванных компонентов, необходимо уточнить процесс для пользователя. Это уточнение необходимо, так как процесс обновления отправляет данные с клиентского компьютера на веб-сайт корпорации Майкрософт.

При вызове **имфконтентенаблер:: аутоматиценабле** программа включения содержимого запускает браузер по умолчанию с адресом службы обновления на веб-сайте Майкрософт. Уникальный идентификатор, идентифицирующий отозванный компонент, передается в службу обновления. Затем служба перенаправляет браузер на веб-страницу, с которой пользователь может загрузить и установить новую версию отозванного компонента.
