# <a name="enumerating-licenses-in-the-local-license-store"></a>Перечисление лицензий в локальном хранилище лицензий

Перечисление — это процесс получения сведений о лицензиях в локальном хранилище лицензий с пошаговым выполнением по одному. Вы можете создать перечисление лицензий, вызвав метод [**ивмдрмлиценсеманажемент:: креателиценсинумератион**](iwmdrmlicensemanagement-createlicenseenumeration.md).

Наиболее распространенной причиной перечисления лицензий в магазине является поиск определенной лицензии для расшифровки какого-либо содержимого.

Интерфейс [**ивмдрмлиценсе**](iwmdrmlicense.md) выступает в качестве портала к отдельным результатам лицензии и в качестве перечислителя. При создании перечисления лицензий первая лицензия в списке загружается в интерфейс **ивмдрмлиценсе** . Большинство методов **ивмдрмлиценсе** позволяют получать сведения о лицензии или создавать объекты для шифрования и расшифровки содержимого на основе лицензии. Однако существует два метода управления перечислением: [**GetNext**](iwmdrmlicense-getnext.md) и [**ресетенумератион**](iwmdrmlicense-resetenumeration.md). **OnNext** загружает следующую лицензию из списка в интерфейс. **Ресетенумератион** возвращает перечисление в состояние, в котором он находился при первоначальном создании. При сбросе перечисления первая лицензия в списке загружается обратно в интерфейс **ивмдрмлиценсе** .

Когда вы достигли последней лицензии в списке, следующий вызов метода **GetNext ВОЗВРАЩАЕТ ошибку** "больше \_ нет элементов" \_ \_ .

Если приложение выполняет действие с содержимым, которое охватывает DRM, следует проверить лицензии в локальном хранилище лицензий на наличие прав и других ограничивающих факторов, таких как уровни защиты выходных данных (Оплс).
