# <a name="indexes"></a>Индексы

Распространенным требованием для приложений, считывающих цифровые файлы мультимедиа, является возможность поиска в определенной точке содержимого. Поиск может быть затруднительным, поскольку нет никакой гарантии, что различные потоки в файле имеют выборки с одновременным временем начала. Эта проблема решена с использованием *индексов*. Индекс — это объект в ASF-файле, который выдает образцы видео со временем презентации. Для звуковых потоков не требуется индекс, так как звуковые данные более тесно подключены к времени презентации, чем данные видео.

объект индексатора пакета SDK Windows Media Format может создавать три различных типа индексов: временные индексы, индексы на основе кадров и индексы кода времени SMPTE.

Временные индексы являются наиболее распространенным типом. Они просто приравнивают видеороликов в соответствии со временем презентации.

Индекс на основе кадров соответствует видеороликам с номерами видеокадров и временем презентации. Номера кадров особенно полезны в приложениях, которые редактируют видео.

Индекс кода времени SMTP — это редкий тип индекса. Он использует код времени SMPTE в качестве базиса для индекса и может использоваться только для потоков, которые имеют отметки времени SMPTE, включенные в выборку. Дополнительные сведения о коде времени SMPTE см. в разделе [поддержка кода времени SMPTE](smpte-time-code-support.md).

Файл ASF может содержать индекс каждого типа для каждого потока видео, который он содержит. По умолчанию для каждого видеопотока в файлах, созданных объектом модуля записи, добавляется временный индекс. Вы можете изменить параметры автоматического индексирования для файлов в соответствии с вашими потребностями.
