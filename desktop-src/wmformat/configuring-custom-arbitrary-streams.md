# <a name="configuring-custom-arbitrary-streams"></a>настройка настраиваемых произвольных Потоки

При использовании собственного произвольного типа данных необходимо создать значение идентификатора GUID, которое будет служить основным идентификатором типа носителя. Когда модуль записи встречает поток в профиле с основным типом, который он не распознает, предполагается, что поток является пользовательским произвольным набором данных. Он примет примеры, паккетизе их и объединит их с образцами из других потоков в файле, не проверяя данные каким бы то ни было.

Можно также создать собственные идентификаторы GUID подтипа для определения подкатегорий пользовательских данных. Модуль записи будет полностью игнорировать эти подтипы, но они будут сохранены в разделе заголовка файла ASF, поэтому приложение для чтения может получить их и принимать решения на их основе.

Для произвольного потока требуется Побитовая скорость и окно буфера, а также должна быть структура [**\_ \_ типа мультимедиа WM**](/previous-versions/windows/desktop/api/wmsdkidl/ns-wmsdkidl-wm_media_type) со значениями, которые очищаются, за исключением основного типа и подтипа носителя (если используется один).
