# <a name="encrypting-and-importing-media-samples"></a>Шифрование и импорт примеров мультимедиа

для каждого примера носителя, зашифрованного с помощью Windows media DRM, необходимо создать расширяющее значение, которое должно быть строго больше предыдущего (монотонно возрастает). Используйте новое расширяющее значение, чтобы создать транзитный ключ шифрования, применяя алгоритм шифрования SHA-1 к вектору инициализации, сцепленному с расширяющим значением.

Затем зашифровать пример в соответствии с алгоритмом RC4, используя созданный транзитный ключ. Перед передачей образца в пакет SDK приложение должно связать значение соли с примером, задав атрибут расширения.

Ниже приведены шаги для шифрования примеров носителей.

1.  Вызовите метод **QueryInterface** образца объекта, чтобы получить интерфейс [**INSSBuffer3**](/previous-versions/windows/desktop/api/wmsbuffer/nn-wmsbuffer-inssbuffer3) .
2.  Увеличение расширяющего значения.
3.  Зашифровать пример с помощью алгоритма шифрования RC1. Для шифрования создается ключ путем сцепления вектора инициализации и расширяющего значения.
4.  Укажите значение соли для пакета SDK, вызвав метод [**инссбуффер:: SetProperty**](/previous-versions/windows/desktop/api/Wmsbuffer/nf-wmsbuffer-inssbuffer3-setproperty).
