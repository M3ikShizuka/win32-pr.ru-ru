# <a name="configuration-persistence"></a>Сохраняемость конфигурации

ни одно из свойств, доступных для записи, описанных в этой документации, не сохраняется с помощью пакета SDK для Windows Media. Каждое приложение, использующее пакет SDK, должно предоставлять собственные процедуры сохранения при необходимости и вызывать соответствующий метод Set для каждого экземпляра пакета SDK. Таким образом, изменения конфигурации, вносимые одним приложением, не влияют на другое приложение. Например, изменение времени буферизации в одном проигрывателе не влияет на другой игрок.

Единственным исключением из этого правила является информация об учетных данных. Если приложение указывает, что при возвращении из вызова **аккуирекредентиалс** необходимо сохранить сведения об идентификаторе пользователя и пароле, пакет SDK сохранит эти сведения.
