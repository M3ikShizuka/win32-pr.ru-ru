# <a name="selecting-bit-rates"></a>Выбор скорости

Для файлов, которые будут передаваться по сети, необходимо тщательно продумать, какие скорости следует использовать. В большинстве случаев можно одновременно добавить скорость всех потоков в файле, чтобы получить общее представление о доступной пропускной способности, необходимой для потоковой передачи файла. Однако для каждого потока также требуется определенный объем издержек. Эти дополнительные сведения приведены в следующей таблице.



| Диапазон битовой скорости (кбит/с) | Дополнительная пропускная способность, необходимая для дополнительной нагрузки (кбит/с) |
|-----------------------|---------------------------------------------------|
| 10 – 16               | 3                                                 |
| 17 – 30               | 4                                                 |
| 31 – 45               | 5                                                 |
| 46 – 70               | 6                                                 |
| 71 – 225              | 7                                                 |
| >225               | 9                                                 |



 

Обычные затраты, необходимые для потока, не занимают расширения модулей данных. Каждое расширение единицы обработки данных добавляет к размеру образца, к которому он присоединен. В зависимости от типа расширения единицы данных это может значительно увеличить скорость потока.

Также необходимо учитывать, что теоретическая максимальная пропускная способность, доступная через сетевое подключение, не является практичной целевой скоростью. Средняя доступная пропускная способность для любого конкретного подключения будет значительно меньше пропускной способности подключения из-за сетевого трафика и многих других факторов.
