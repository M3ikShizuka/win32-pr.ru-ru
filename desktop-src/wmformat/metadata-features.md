# <a name="metadata-features"></a>Компоненты метаданных

Метаданные используются в файлах ASF для описания содержимого и свойств файла. Все создаваемые файлы ASF должны содержать соответствующие метаданные. (Общие сведения см. в разделе [метаданные](metadata.md).) пакет SDK для Windows Media Format включает поддержку редактирования метаданных с помощью объекта модуля записи, объекта редактора метаданных, а также объектов чтения и синхронного модуля чтения. Включена собственная поддержка для широкого спектра атрибутов метаданных. Список предопределенных атрибутов см. в разделе [атрибуты](attributes.md) .

поддержка метаданных, обеспечиваемая различными объектами пакета SDK для Windows Media Format, является гибкой и мощной. Основные функции метаданных приведены в следующем списке:

-   Гибкий размер атрибута. Атрибуты метаданных имеют неограниченный размер.
-   Атрибуты уровня потока. Метаданные в ASF-файлах могут быть назначены файлу целиком или определенному потоку.
-   Дублирующиеся атрибуты. Именованный атрибут может использоваться несколько раз в одном файле. Эта функция относится к конкретному использованию при назначении описательных атрибутов содержимого. Например, песня может иметь несколько авторов, каждая из которых требует наличия отдельного атрибута **Author** в файле.
-   Несколько языков. С каждым атрибутом связан язык. Можно задать поддерживаемые языки, а затем назначить их каждому создаваемому атрибуту. Так как вы можете дублировать атрибуты, можно предоставить наиболее важные атрибуты на нескольких языках для достижения более крупной аудитории. Если язык не указан, будет использоваться язык по умолчанию (полученный из операционной системы компьютера, на котором выполняется приложение).
-   Сложные атрибуты. Некоторые предопределенные атрибуты поддерживают структурированные данные. Для этих атрибутов тип данных является двоичным, но значением является структура, определенная в этом пакете SDK.
