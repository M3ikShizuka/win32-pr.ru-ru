# <a name="filtering-layer-identifiers"></a>Фильтрация идентификаторов слоя

Идентификаторы уровней платформы фильтрации Windows (WFP) представлены с помощью идентификатора GUID. Эти идентификаторы определяются следующим образом.

Суффиксы v4 и V6 в конце идентификаторов слоя указывают, находится ли слой в сетевом стеке IPv4 или в стеке сети IPv6.

</dt> <dd> <dl> <dt>



Этот слой фильтрации находится в пути получения сразу после синтаксического анализа IP-заголовка полученного пакета, но до того, как будет выполнена обработка заголовка IP-адреса. Расшифровка или пересборка IPsec не выполнялась.


</dt> </dl> </dd> <dt>

<span id="FWPM_LAYER_INBOUND_IPPACKET_V4_DISCARD___FWPM_LAYER_INBOUND_IPPACKET_V6_DISCARD"></span><span id="fwpm_layer_inbound_ippacket_v4_discard___fwpm_layer_inbound_ippacket_v6_discard"></span>**ФВПМ \_ уровень \_ входящего трафика \_ ИППАККЕТ \_ v4/фвпм на уровне входящего трафика \_ \_ \_ \_ иппаккет \_ V6 \_**
</dt> <dd> <dl> <dt>



Этот слой фильтрации находится в пути получения для проверки полученных пакетов, которые были отклонены на сетевом уровне.


</dt> </dl> </dd> <dt>

</dt> <dd> <dl> <dt>



Этот слой фильтрации находится в пути отправки непосредственно перед тем, как отправленный пакет оценивается для фрагментации. Вся обработка заголовка IP-адреса завершена, и все заголовки расширений установлены. Проверка подлинности и шифрование IPsec уже выполнены.

## <a name="remarks"></a>Комментарии

Эти идентификаторы слоя фильтрации также называются идентификаторами уровней фильтрации управления. API WFP также содержит набор [идентификаторов уровней фильтрации времени выполнения](/windows-hardware/drivers/network/management-filtering-layer-identifiers), документированных в комплекте драйверов Windows (WDK). Идентификаторы уровней фильтрации времени выполнения — это LUID, и поэтому они меньше, чем размер 64 бит, по сравнению с идентификаторами слоя фильтрации управления, которые имеют размер 128 бит.

Идентификаторы уровней фильтрации управления и идентификаторы уровней фильтрации времени выполнения указывают на один и тот же слой.

Идентификаторы уровней фильтрации управления используются функциями, которые взаимодействуют с ядром базовой фильтрации (BFE) из пользовательского режима или режима ядра (например, [**FwpmFilterAdd0**](/windows/desktop/api/Fwpmu/nf-fwpmu-fwpmfilteradd0)).

Идентификаторы уровней фильтрации времени выполнения используются функциями, которые взаимодействуют с обработчиком фильтров только в режиме ядра (например, [FwpsFlowAssociateContext0](/windows-hardware/drivers/ddi/fwpsk/nf-fwpsk-fwpsflowassociatecontext0), [FwpsStreamInjectAsync0](/windows-hardware/drivers/ddi/fwpsk/nf-fwpsk-fwpsstreaminjectasync0)), и в структурах данных, поступающих непосредственно из ядра (например, [**фвпм \_ net \_ event \_ классифицировать \_ DROP0**](/windows/desktop/api/Fwpmtypes/ns-fwpmtypes-fwpm_net_event_classify_drop0)).
