# <a name="ale-multicastbroadcast-traffic"></a>Многоадресный или широковещательный трафик ALE

Весь входящий многоадресный и широковещательный трафик на уровнях применения уровня приложения (ALE) сопоставляется с одним глобальным [потоком ALE](ale-stateful-filtering.md). Трафик ответа для входящих многоадресных и широковещательных пакетов классифицируется на уровне [**\_ \_ \_ проверки подлинности \_ подключения ALE \_ V {4 \| 6} уровня фвпм**](management-filtering-layer-identifiers-.md) , и для каждого ответа создаются отдельные потоки Ale.

Исходящий трафик многоадресной и широковещательной рассылки на уровнях ALE создает 4-секундный поток ALE. По умолчанию проверка подлинности исходящего пакета ALE или многоадресной рассылки будет разрешать входящий трафик, будь то одноадресный, многоадресный или широковещательный, с любого удаленного адреса в течение 4 секунд. Такой поток ALE может обновляться или храниться только при последующем исходящем трафике, соответствующем потоку ALE.

> [!Note]  
> Время существования, равное 4 секундам, определяется встроенным [**\_ вызываемым выноски фвпм \_ параметры SET Authentication \_ \_ \_ Connect \_ Layer \_ V {4 \| 6}**](built-in-callout-identifiers.md). Чтобы изменить время существования по умолчанию — 4 секунды, добавьте фильтр, ссылающийся на **фвпм \_ выноски \_ \_ Параметры Authentication \_ \_ Connect \_ Layer \_ V {4 \| 6}** . Дополнительные сведения см. в разделе [Настройка потока ALE](ale-flow-customization.md) .
