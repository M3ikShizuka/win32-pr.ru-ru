# <a name="basic-types"></a>Базовые типы

API регистрации сертификатов поддерживает следующие основные типы ASN. 1.

## <a name="bit-string"></a>БИТОВАЯ СТРОКА

Тег Encoding: 0x03

Имя Certreq.exe: БИТОВАЯ \_ строка

Разрядная или двоичная строка является произвольно длинным массивом битов. Определенные биты можно идентифицировать с помощью целых чисел и назначенных имен, как показано в следующем примере.

``` syntax
Versions ::= BIT STRING{ version-1(0), version-2(1) } 
```

Ключи сертификатов и подписи часто представляются в виде битовых строк.

``` syntax
---------------------------------------------------------------------
-- ASN.1 type example: BIT STRING
-- Tag number: 0x03
---------------------------------------------------------------------
SubjectPublicKeyInfo ::= SEQUENCE 
{
  algorithm           AlgorithmIdentifier,
  subjectPublicKey    BIT STRING
} 
```

## <a name="boolean"></a>BOOLEAN

Тег Encoding: 0x01

Имя Certreq.exe: логическое значение

Логический тип может содержать одно из двух значений: **true** или **false**. В следующем примере показана структура ASN. 1 для расширения сертификата базовых ограничений. В поле **cA** указывается, является ли субъект сертификата центром сертификации (CA). Значение по умолчанию — **false**.

``` syntax
---------------------------------------------------------------------
-- ASN.1 type example: BOOLEAN
-- Tag number: 0x01
---------------------------------------------------------------------
BasicConstraints ::= SEQUENCE 
{
  cA                  BOOLEAN DEFAULT FALSE,
  pathLenConstraint   INTEGER OPTIONAL
}
```

## <a name="integer"></a>INTEGER

Тег Encoding: 0x02

Имя Certreq.exe: ЦЕЛОе число

Целочисленное значение обычно может быть любым положительным или отрицательным целочисленным значением. В следующем примере показана структура ASN. 1 для открытого ключа RSA. Обратите внимание, что поле **публицекспонент** ограничивается положительным целым числом меньше 4 294 967 296.

``` syntax
---------------------------------------------------------------------
-- ASN.1 type example: INTEGER
-- Tag number: 0x02
---------------------------------------------------------------------
HUGEINTEGER ::= INTEGER

RSAPublicKey ::= SEQUENCE 
{ 
  modulus         HUGEINTEGER,    
  publicExponent  INTEGER (0..4294967295) 
} 
```

## <a name="null"></a>NULL

Тег Encoding: 0x05

Имя Certreq.exe: **null**

Тип **null** содержит один байт 0x00. Его можно использовать в любом месте, где запрос сертификата должен указывать на пустое значение. Например, **алгорисмидентифиер** — это последовательность, которая содержит идентификатор объекта (OID) и необязательные параметры.

``` syntax
---------------------------------------------------------------------
-- ASN.1 type example: NULL
-- Tag number: 0x05
---------------------------------------------------------------------
AlgorithmIdentifier ::= SEQUENCE 
{
  algorithm           OBJECT IDENTIFIER,
  parameters          ANY OPTIONAL    
}
```

Если при кодировании структуры нет параметров, для указания пустого значения используется **значение NULL** .

``` syntax
30 0d            ; SEQUENCE (d Bytes)
|  |  |  06 09          ; OBJECT_ID (9 Bytes)
|  |  |  |  2a 86 48 86 f7 0d 01 01  01
|  |  |  |     ; 1.2.840.113549.1.1.1 RSA (RSA_SIGN)
|  |  |  05 00          ; NULL (0 Bytes)
```

## <a name="object-identifier"></a>ИДЕНТИФИКАТОР ОБЪЕКТА

Тег Encoding: 0x06

Имя Certreq.exe: \_ идентификатор объекта

API регистрации сертификатов использует идентификаторы объектов (OID) в качестве универсального указателя на идентификаторы алгоритма, атрибуты и другие элементы PKI. OID обычно представлены в точечно-десятичной строке, например "2.16.840.1.101.3.4.1.42". Отдельные элементы в строке, разделенные точками, представляют дуги и листья в дереве центра регистрации, которые однозначно идентифицируют объект и зарегистрированную в нем организацию. Например, предыдущий идентификатор объекта может быть расширен до совместного стандарта ISO-ITU-t (2) Country (16) US (840) (1) gov (101) КСОР (3) Нисталгорисмс (4) Аесалгс (1) с 256 добавлением. 42.

``` syntax
---------------------------------------------------------------------
-- ASN.1 type example: OBJECT IDENTIFIER
-- Tag number: 0x06
---------------------------------------------------------------------
AlgorithmIdentifier ::= SEQUENCE 
{
  algorithm           OBJECT IDENTIFIER,
  parameters          ANY OPTIONAL    
}
```

## <a name="octet-string"></a>СТРОКА ОКТЕТА

Тег Encoding: 0x04

Имя Certreq.exe: строка ОКТЕТа \_

Строка октета является произвольно большим массивом байтов. Однако в отличие от типа **битовой строки** определенные биты и байты в строке не могут назначаться имена. Слово «октет» предназначено для независимого от платформы способа ссылки на слово памяти. В контексте API регистрации сертификатов октет и Byte взаимозаменяемы.

``` syntax
---------------------------------------------------------------------
-- ASN.1 type example: OCTET STRING
-- Tag number: 0x04
---------------------------------------------------------------------
AuthorityKeyId ::= SEQUENCE 
{
  keyIdentifier       [0] IMPLICIT OCTET STRING OPTIONAL,
  certIssuer          [1] EXPLICIT NAME
  certSerialNumber    [2] IMPLICIT INTEGER OPTIONAL
}
```
