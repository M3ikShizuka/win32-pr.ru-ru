# <a name="integer-certificate-enrollment-api"></a>ЦЕЛОе число (API регистрации сертификатов)

Целочисленные значения кодируются в TLV-Triplet, который начинается с значения **тега** 0x02. Поле **значения** TLV Triplet содержит закодированное целое число, если оно положительное, или его дополнение, если оно отрицательное. Если целое число положительное, а бит высокого порядка имеет значение 1, в содержимое добавляется ведущий 0x00, чтобы указать, что число не является отрицательным. Например, байт высокого порядка в 0x8F (10001111) равен 1. Таким образом, в содержимое добавляется начальный нуль-байт, как показано на следующем рисунке.

![Кодировка der логического типа данных](images/der-tlv-integer.png)

Если целое число содержит менее 128 байт, поле *length* должно содержать только один байт для указания длины содержимого. Если целое число превышает 127 байт, в поле *length* в разряде 7 устанавливается значение 1, а для битов с 6 по 0 укажите число дополнительных байтов, используемых для определения длины содержимого. Дополнительные сведения см. в разделе [кодированная Длина и байты значений](about-encoded-length-and-value-bytes.md).

В следующем примере из [PKCS \# 10 с кодировкой ASN. 1](pkcs--10-encoded-asn-1.md)показана кодировка для открытого ключа 128 байта. Первый байт содержит значение **тега** для **целочисленного** типа данных, 0x02. Второй и третий байты содержат значение **длины** . Бит 7 второго байта равен 1, так как имеется более 127 байт содержимого. Биты от 0 до 6 второго байта указывают количество требуемых конечных байтов, в данном случае один — для точного указания длины содержимого. Третий байт указывает число байтов содержимого, 0x81. Четвертый байт 0x00 добавляется к содержимому, чтобы указать, что целое число действительно является положительным значением, хотя бит знака ведущего содержимого (0x8F) имеет значение 1.

``` syntax
02 81 81          ; INTEGER (81 Bytes)
|  00
|  8f e2 41 2a 08 e8 51 a8  8c b3 e8 53 e7 d5 49 50
|  b3 27 8a 2b cb ea b5 42  73 ea 02 57 cc 65 33 ee
|  88 20 61 a1 17 56 c1 24  18 e3 a8 08 d3 be d9 31
|  f3 37 0b 94 b8 cc 43 08  0b 70 24 f7 9c b1 8d 5d
|  d6 6d 82 d0 54 09 84 f8  9f 97 01 75 05 9c 89 d4
|  d5 c9 1e c9 13 d7 2a 6b  30 91 19 d6 d4 42 e0 c4
|  9d 7c 92 71 e1 b2 2f 5c  8d ee f0 f1 17 1e d2 5f
|  31 5b b1 9c bc 20 55 bf  3a 37 42 45 75 dc 90 65
```

В следующем примере показано, как кодируется целочисленное значение 0x03. Байт **тега** содержит 0x02, а **Длина** байта указывает на один байт содержимого.

``` syntax
02 01             ; INTEGER (1 Bytes)
|  03
```
