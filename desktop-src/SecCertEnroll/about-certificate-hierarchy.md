# <a name="certificate-hierarchy"></a>Иерархия сертификатов

По мере увеличения числа выданных сертификатов в инфраструктуре открытых ключей (PKI) может оказаться затруднительным, чтобы один центр сертификации (CA) мог эффективно отразить выданные сертификаты. Один из способов решения этой проблемы — создать иерархию сертификатов, в которой ЦС делегирует полномочия выдавать сертификаты подчиненным органам, которые, в свою очередь, могут делегировать полномочия своим подчиненным пользователям. Каждый ЦС делегирует полномочия, выдавая сертификат ЦС подчиненному. Первоначальный ЦС в цепочке называется корнем, и не требуется, чтобы сущность установила отношения доверия с любым центром сертификации, который находится в [цепочке сертификатов](about-certificate-chain.md) , отличной от той, в которой находится сущность.

На следующем рисунке показана иерархия сертификатов, состоящие из одного корневого центра сертификации, двух центров сертификации, подчиненных корневому центру (один для отдела маркетинга, а другой — для отдела производства) и центров сертификации, подчиненных этим.
