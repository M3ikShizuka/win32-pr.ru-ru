# <a name="getjob-function"></a>Функция Жетжоб

Функция **жетжоб** извлекает сведения об указанном задании печати.

## <a name="syntax"></a>Синтаксис


```C++
BOOL GetJob(
  _In_  HANDLE  hPrinter,
  _In_  DWORD   JobId,
  _In_  DWORD   Level,
  _Out_ LPBYTE  pJob,
  _In_  DWORD   cbBuf,
  _Out_ LPDWORD pcbNeeded
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хпринтер* \[ окне\]
</dt> <dd>

Маркер принтера, для которого извлекаются данные задания печати. Используйте функцию [**опенпринтер**](openprinter.md) или [**аддпринтер**](addprinter.md) для получения маркера принтера.

</dd> <dt>

Идентификатор *задания* \[ окне\]
</dt> <dd>

Определяет задание печати, для которого извлекаются данные. Чтобы получить идентификатор задания печати, используйте функцию [**AddJob**](addjob.md) или функцию [**стартдок**](/windows/desktop/api/Wingdi/nf-wingdi-startdoca) .

</dd> <dt>

*Уровень* \[ окне\]
</dt> <dd>

Тип сведений, возвращаемых в буфер *пжоб* . Если *уровень* равен 1, *пжоб* Получает структуру [**\_ сведений о \_ задании 1**](job-info-1.md) . Если *уровень* равен 2, *пжоб* Получает структуру [**\_ сведений о \_ задании 2**](job-info-2.md) .

</dd> <dt>

*пжоб* \[ заполняет\]
</dt> <dd>

Указатель на буфер, получающий сведения о задании [**\_ \_ 1**](job-info-1.md) или сведения о задании [**\_ \_ 2**](job-info-2.md) , содержащие сведения о задании. Буфер должен быть достаточно большим, чтобы хранить строки, на которые указывает члены структуры.

Чтобы определить требуемый размер буфера, вызовите **жетжоб** с параметром *кббуф* , присвойте ему значение 0. **Жетжоб** завершается ошибкой, [**GetLastError**](/windows/desktop/api/errhandlingapi/nf-errhandlingapi-getlasterror) возвращает ошибку, \_ недостаточную \_ для буфера, а параметр *пкбнидед* возвращает размер (в байтах) буфера, необходимого для хранения массива структур и их данных.

</dd> <dt>

*кббуф* \[ окне\]
</dt> <dd>

Размер массива в байтах.

</dd> <dt>

*пкбнидед* \[ заполняет\]
</dt> <dd>

Указатель на значение, указывающее число байтов, скопированных при выполнении функции, или число байтов, необходимых, если *кббуф* слишком мал.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена, возвращаемое значение будет ненулевым.

Если функция выполняется неудачно, возвращается нулевое значение.

## <a name="remarks"></a>Комментарии

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.
