# <a name="representing-attributes-as-featureoption-constructs-or-as-parameters"></a>Представление атрибутов в виде конструкций "функция-параметр" или "Параметры"

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Автор документа PrintCapabilities определяет атрибуты устройства, которые составляют конфигурацию. В документе PrintCapabilities автор может представить атрибут устройства как конструкцию функции или варианта или как конструкцию параметра.

Если атрибут устройства имеет относительно небольшое число возможных состояний, не относящихся ни к одному из очевидных шаблонов, то лучше выбрать конструкцию "функция-параметр". Например, если допустимыми значениями для атрибута устройства Пажемедиасизе являются значения Letter, Legal, A4ISO, tabloid и Envelope10, то для представления лучше выбрать конструкцию "функция-параметр". Из-за сложности и неоднозначности, связанной с выражением допустимых значений без явного перечисления, конструкция параметра не подходит для атрибута Пажемедиасизе.

Если атрибут устройства может быть представлен диапазоном целых чисел, лучше выбрать представление параметра, особенно для диапазонов, включающих множество значений. Например, если атрибут устройства Копикаунт для конкретной модели принтера может находиться в диапазоне от 1 до 99 999, то этот атрибут следует классифицировать как параметр, определив экземпляр Параметердеф. Назначьте значения для экземпляров свойств MinValue и MaxValue элемента Параметердеф, чтобы определить диапазон значений для атрибута Жобкопикаунт. Из-за большого количества значений, которые должны быть явно перечислены как элементы параметров, представление "функция/параметр" не подходит для атрибута устройства Жобкопикаунт.
