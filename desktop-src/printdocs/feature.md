# <a name="feature"></a>Компонент

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Элемент Feature содержит полный список элементов параметров и свойств, которые полностью описывают атрибут устройства, параметр форматирования задания или другие соответствующие характеристики.

## <a name="element-tag"></a>Тег элемента

<Feature>

## <a name="xml-attributes"></a>XML-атрибуты

В следующей таблице перечислены атрибуты XML, которые могут быть связаны с этим элементом.



| Атрибут XML   | Сведения                                                                                              |
|-----------------|------------------------------------------------------------------------------------------------------|
| name<br/> | Содержит имя функции, стандартной или закрытой функции. <br/> |



 

Дополнительные сведения см. в разделе [XML Attributes](xml-attributes.md) .

## <a name="configuration-dependencies"></a>Зависимости конфигурации

Элементы компонента могут не иметь зависимостей конфигурации.

## <a name="element-usage"></a>Использование элементов

### <a name="relationship-to-xml-attributes"></a>Связь с XML-атрибутами

В представлении "функция-параметр" атрибут устройства представлен элементом Feature. Атрибут Device однозначно идентифицируется атрибутом Name в элементе Feature атрибута устройства, как показано в следующем примере. В этом примере атрибут устройства имеет разрешение.

``` syntax
<Feature name="Resolution" />
```

Схема печати определяет набор атрибутов имени для определенных экземпляров компонента. Эти атрибуты имени служат для того, чтобы определить набор предварительно определенных экземпляров компонента, связанных с определенными настраиваемыми атрибутами устройства. Эти имена экземпляров компонентов следует использовать всякий раз, когда они применимы, так как они увеличивают переносимость документа PrintCapabilities и PrintTicket, полученные от них. Закрыто определенные экземпляры компонентов могут быть введены, если определенные атрибуты устройства не соответствуют ни одному из экземпляров компонента, определяемых схемой. Сведения о синтаксисе атрибутов имени и соглашениях, которые применяются к определяемым схемой и частным именам, см. в разделе [атрибуты XML](xml-attributes.md).

### <a name="relationship-to-option-element"></a>Связь с элементом параметра

Каждое из возможных состояний представлено элементом Option. Каждое определение параметра содержит один или несколько элементов Скоредпроперти, которые вместе однозначно описывают или характеризуют представленное состояние. Методика, используемая для создания определений параметров, описана в разделе [определения параметров](option-definitions.md). Все элементы Option, связанные с определенным элементом компонента, находятся в качестве дочерних элементов элемента Feature.

### <a name="subfeatures"></a>Его части

Платформа схемы печати также позволяет объединять элементы компонентов в иерархическую структуру. То есть элемент Feature может сам содержать один или несколько дочерних элементов компонента (подкомпонентов). Это может быть полезно для организации связанных элементов компонента или для элементов компонента, управляющих аспектами функции устройства. Одним из примеров является устройство, которое поддерживает сшивание. Такое устройство может предложить пользователю выбрать место для размещения сшивания, например, в левом верхнем углу или в правом верхнем углу или на левом краю или вдоль левого края. Пользовательский интерфейс для этого устройства должен иметь возможность предоставлять пользователю наиболее высокий уровень выбора, в этом случае следует использовать сшивание. Только после того, как пользователь решил использовать сшивание, он должен быть представлен вторым вариантом выбора, скрепкой. Иерархия функций добавляет дополнительную структуру, которая делает возможным такой пользовательский интерфейс. Платформа схемы печати позволяет подкомпонентам иметь собственные дочерние функции, тем самым разрешая Неограниченный уровень вложенности.

Платформа схемы печати также позволяет элементам параметров отображаться на том же уровне, что и подкомпоненты. то есть в качестве одноуровневых элементов в пределах одного родительского элемента Feature. Это позволяет пользователю принять решение высокого уровня (использовать сшивание) перед выбором подкомпонентов. В этом примере корневой элемент Feature, "степлер", может содержать два элемента параметров: "on" и "OFF", а также подфункции с именем "Стаплелокатион".

## <a name="example"></a>Пример

``` syntax
<psf:Feature name="psk:JobOutputBin">
  <psf:Property name="psf:SelectionType">
    <psf:Value xsi:type="xs:string">psk:PickOne</psf:Value>
  </psf:Property>
  <psf:Option constrained="psk:None">
    <psf:ScoredProperty name="psk:Bin">
      <psf:Value xsi:type="xs:string">SorterBin</psf:Value>
    </psf:ScoredProperty>
    <psf:ScoredProperty name="psk:MediaSheetCapacity">
      <psf:Value xsi:type="xs:integer">100</psf:Value>
    </psf:ScoredProperty>
  </psf:Option>
</psf:Feature>
```
