# <a name="enumprinterkey-function"></a>Функция Енумпринтеркэй

Функция **енумпринтеркэй** перечисляет подразделы указанного ключа для указанного принтера.

Данные принтера хранятся в реестре. При перечислении данных принтера не следует вызывать функции реестра, которые могут изменить данные.

## <a name="syntax"></a>Синтаксис


```C++
DWORD EnumPrinterKey(
  _In_  HANDLE  hPrinter,
  _In_  LPCTSTR pKeyName,
  _Out_ LPTSTR  pSubkey,
  _In_  DWORD   cbSubkey,
  _Out_ LPDWORD pcbSubkey
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хпринтер* \[ окне\]
</dt> <dd>

Описатель принтера, для которого функция перечисляет подразделы. Используйте функцию [**опенпринтер**](openprinter.md) или [**аддпринтер**](addprinter.md) для получения маркера принтера.

</dd> <dt>

*пкэйнаме* \[ окне\]
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая указывает ключ, содержащий подразделы для перечисления. Используйте символ обратной косой черты в \\ качестве разделителя, чтобы указать путь с одним или несколькими подразделами. **Енумпринтеркэй** перечисляет все подразделы ключа, но не перечисляет подразделы этих подразделов.

Если *пкэйнаме* является пустой строкой (""), **енумпринтеркэй** перечисляет ключ верхнего уровня для принтера. Если *пкэйнаме* имеет **значение NULL**, **енумпринтеркэй** возвращает ошибку \_ Недопустимый \_ параметр.

</dd> <dt>

*псубкэй* \[ заполняет\]
</dt> <dd>

Указатель на буфер, который получает массив имен подразделов, заканчивающихся нулем. Массив завершается двумя символами NULL.

</dd> <dt>

*кбсубкэй* \[ окне\]
</dt> <dd>

Размер (в байтах) буфера, на который указывает *псубкэй*. Если для *кбсубкэй* задано значение 0, то параметр *пкбсубкэй* возвращает требуемый размер буфера.

</dd> <dt>

*пкбсубкэй* \[ заполняет\]
</dt> <dd>

Указатель на переменную, которая получает число байтов, полученных в буфере *псубкэй* . Если размер буфера, указанный параметром *кбсубкэй* , слишком мал, функция возвращает ошибку \_ больше \_ данных, а *пкбсубкэй* указывает требуемый размер буфера.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение является кодом системной ошибки. Если *пкэйнаме* не существует, возвращаемое значение — " \_ файл ошибки \_ не \_ найден".

## <a name="remarks"></a>Remarks

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.
