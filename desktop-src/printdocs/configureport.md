# <a name="configureport-function"></a>Функция Конфигурепорт

Функция **конфигурепорт** отображает диалоговое окно настройки порта для порта на указанном сервере.

## <a name="syntax"></a>Синтаксис


```C++
BOOL ConfigurePort(
  _In_ LPTSTR pName,
  _In_ HWND   hWnd,
  _In_ LPTSTR pPortName
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*pName* \[ окне\]
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая указывает имя сервера, на котором существует указанный порт. Если этот параметр имеет **значение NULL**, то порт является локальным.

</dd> <dt>

*HWND* \[ окне\]
</dt> <dd>

Обработано с родительским окном диалогового окна настройки порта.

</dd> <dt>

*ппортнаме* \[ окне\]
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая указывает имя порта для настройки.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена, возвращаемое значение будет ненулевым.

Если функция выполняется неудачно, возвращается нулевое значение.

## <a name="remarks"></a>Remarks

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.

 

Перед вызовом функции **конфигурепорт** приложение должно вызвать функцию [**енумпортс**](enumports.md) , чтобы определить допустимые имена портов.
