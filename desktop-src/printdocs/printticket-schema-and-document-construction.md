# <a name="printticket-schema-and-document-construction"></a>Создание схемы и построения документов PrintTicket

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Текущий метод указания сведений о конфигурации устройства с использованием структуры DEVMODE имеет несколько ограничений. Во-первых, структура DEVMODE представляет собой двоичную структуру, которая может привести к проблемам различных версий. Во-вторых, он делится на нерасширяемую открытую часть и частную часть, доступ к которой можно получить только по драйверам, а затем только по конкретному драйверу, который его создал. Формат PrintTicket Express предоставляет сведения о конфигурации с помощью платформы схемы печати на основе XML, тем самым устраняя эти недостатки структуры DEVMODE.

Схема PrintTicket решает каждую из двух упомянутых проблем. Во-первых, схема PrintTicket является текстовым файлом на основе XML, поэтому проблемы с расширяемостью и управлением версиями устраняются. Во-вторых, сведения о конфигурации доступны для всех клиентов, что означает, что любой клиент или поставщик может хранить и извлекать любые сведения, содержащиеся в PrintTicket. Параметры описаны с помощью того же метода, который используется платформой схемы печати и производным документом PrintCapabilities. По этой причине PrintTicket предоставляет все возможные преимущества переносимости модели определения параметров для реализации. Дополнительные сведения см. в разделе [Print Schema Framework](print-schema-framework.md) . Целевая аудитория для этого раздела включает следующие группы:

-   Реализации интерфейса поставщика PrintTicket/PrintCapabilities

-   Потребители для PrintTicket

-   Клиенты интерфейса поставщика PrintTicket/PrintCapabilities

Члены первой категории в приведенном выше списке называются поставщиками PrintTicket в оставшейся части этого раздела. Члены последних двух категорий называются потребителями PrintTicket.

## <a name="relationship-to-print-schema-and-printcapabilities-schema"></a>Связь с схемой печати и схемой PrintCapabilities

Схемы PrintTicket и PrintCapabilities являются специализированными частями схемы печати. Основные структурные различия между этими подмножествами схемы печати заключается в том, что схема PrintTicket содержит свойства и экземпляры Параметеринит, которые не содержатся в схеме PrintCapabilities, а схема PrintCapabilities включает свойства и экземпляры Параметердеф, которые не содержатся в схеме PrintTicket. За исключением этих различий, схемы PrintCapabilities и PrintTicket обычно отражают друг друга в содержимом, совместном использовании компонентов, параметрах, Скоредпроперти и экземплярах значений. Все такое общее содержимое должно поддерживаться в актуальном состоянии. Например, если в схеме PrintCapabilities вносится изменение в функцию Медиасизе, то в схеме PrintTicket должны быть внесены те же изменения.
