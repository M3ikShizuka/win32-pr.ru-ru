# <a name="setprinterdataex-function"></a>Функция Сетпринтердатаекс

Функция **сетпринтердатаекс** задает данные конфигурации для принтера или сервера печати. Функция сохраняет данные конфигурации в разделе реестра принтера.

## <a name="syntax"></a>Синтаксис


```C++
DWORD SetPrinterDataEx(
  _In_ HANDLE  hPrinter,
  _In_ LPCTSTR pKeyName,
  _In_ LPCTSTR pValueName,
  _In_ DWORD   Type,
  _In_ LPBYTE  pData,
  _In_ DWORD   cbData
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хпринтер* \[ окне\]
</dt> <dd>

Маркер принтера или сервера печати, для которого функция задает данные конфигурации. Используйте функцию [**опенпринтер**](openprinter.md), [**OpenPrinter2**](openprinter2.md)или [**аддпринтер**](addprinter.md) для получения маркера принтера.

</dd> <dt>

*пкэйнаме* \[ окне\]
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая указывает ключ, содержащий заданное значение. Если указанный ключ или подразделы не существуют, функция создает их.

Чтобы сохранить данные конфигурации, которые могут быть опубликованы в службе каталогов (DS), укажите один из следующих предопределенных разделов реестра.



| Значение                                                                                                                                                                      | Значение                                                                                         |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------|
| <span id="SPLDS_DRIVER_KEY"></span><span id="splds_driver_key"></span><dl> <dt>**SPLDS_DRIVER_KEY**</dt> </dl>    | Драйверы принтера этот ключ используется для хранения свойств драйвера.<br/>                             |
| <span id="SPLDS_SPOOLER_KEY"></span><span id="splds_spooler_key"></span><dl> <dt>**SPLDS_SPOOLER_KEY**</dt> </dl> | Зарезервировано. Используется только диспетчером очереди печати для хранения свойств внутреннего диспетчера очереди сообщений.<br/>       |
| <span id="SPLDS_USER_KEY"></span><span id="splds_user_key"></span><dl> <dt>**SPLDS_USER_KEY**</dt> </dl>          | Приложения используют этот ключ для хранения свойств принтера, например номеров ресурсов принтера.<br/> |



 

Значения, хранящиеся в ключе SPLDS_USER_KEY, публикуются в службе каталогов только при наличии соответствующего свойства в схеме. Администратор домена должен создать свойство, если оно еще не существует. Чтобы опубликовать определяемое пользователем свойство после использования **сетпринтердатаекс** для добавления или изменения значения, вызовите [**сетпринтер**](setprinter.md) с *Level* = 7 и установите для элемента **двактион** [**PRINTER_INFO_7**](printer-info-7.md) значение **DSPRINT_UPDATE**.

Можно указать другие ключи для хранения данных конфигурации, не являющихся службами DS. Используйте символ обратной косой черты ( \\ ) в качестве разделителя, чтобы указать путь с одним или несколькими подразделами.

Если *хпринтер* является обработчиком для принтера, *Пкэйнаме* имеет **значение NULL** или является пустой строкой, **сетпринтердатаекс** возвращает **ERROR_INVALID_PARAMETER**.

Если *хпринтер* является обработчиком для сервера печати, *пкэйнаме* игнорируется.

Не используйте **SPLDS_SPOOLER_KEY**. Чтобы изменить свойства принтера диспетчера очереди печати, используйте [**сетпринтер**](setprinter.md) с *Level* = 2.

</dd> <dt>

*пвалуенаме* \[ окне\]
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая определяет данные для задания.

Для принтеров эта строка указывает имя значения под ключом *пкэйнаме* .

Для серверов печати эта строка является одной из предопределенных строк, перечисленных в следующем разделе "Примечания".

</dd> <dt>

*Тип* \[ окне\]
</dt> <dd>

Код, указывающий тип данных, на который указывает параметр *pData* . Список возможных кодов типов см. в разделе [типы значений реестра](/windows/desktop/SysInfo/registry-value-types).

Если *пкэйнаме* указывает один из стандартных ключей службы каталогов, *тип* должен быть **REG_SZ**, **REG_MULTI_SZ**, **REG_DWORD** или **REG_BINARY**. Если используется **REG_BINARY** , *cbData* должен быть равен 1, а служба каталогов обрабатывает их как логическое значение.

</dd> <dt>

*pData* \[ окне\]
</dt> <dd>

Указатель на буфер, содержащий данные конфигурации принтера.

</dd> <dt>

*cbData* \[ окне\]
</dt> <dd>

Размер массива в байтах.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена, возвращаемое значение будет **ERROR_SUCCESS**.

Если функция завершается ошибкой, возвращается значение ошибки.

## <a name="remarks"></a>Remarks

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.
