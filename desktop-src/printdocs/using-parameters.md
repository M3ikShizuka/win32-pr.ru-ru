# <a name="using-parameters"></a>Использование параметров

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Помимо правильной оценки параметра, содержащего экземпляры ParameterRef (см. раздел о [параметрах](referencing-parameters.md)), поставщики PrintCapabilities или PrintTicket и клиенты должны быть готовы к обработке следующих ситуаций, в которых участвуют параметры.

Клиенты ПОЛЬЗОВАТЕЛЬСКОГО интерфейса должны запрашивать у пользователя инициализаторы параметров (значения для элементов Параметеринит) для назначенных параметров в соответствующее время, чтобы соответствующие элементы Параметеринит были вставлены в PrintTicket. Клиенты пользовательского интерфейса должны иметь возможность отличать два типа параметров, безусловно обязательный и условно обязательный, и должны уметь обрабатывать каждый тип соответствующим образом. Для неусловного обязательного параметра пользовательский интерфейс должен убедиться, что для этого типа параметра указан элемент Параметеринит. Для условно обязательного параметра пользовательский интерфейс должен предоставить значение Параметеринит, если на параметр ссылается параметр, выбранный в PrintTicket. Обязательное состояние параметра указывается в экземпляре Параметердеф. Дополнительные сведения см. в разделе [элементы параметердеф и параметеринит](parameterdef-and-parameterinit-elements.md). Клиенты пользовательского интерфейса должны проверить указанные пользователем значения Параметеринит, чтобы убедиться, что они удовлетворяют требованиям, указанным в экземпляре Параметердеф.

Поставщики PrintTicket должны также проверять экземпляры Параметеринит, предоставляемые PrintTicket, чтобы убедиться в наличии всех необходимых параметров и удовлетворять требованиям, указанным в экземпляре Параметердеф. Код проверки должен предоставлять разумные значения по умолчанию в случае, если Параметерините значение недопустимо или отсутствует. Обратите внимание, что Параметердеф позволяет указать значение по умолчанию для этой цели. Кроме того, при наличии ограничений, затрагивающих параметры, например: "Если *копикаунт* больше N, не используйте входной лоток небольшой емкости", код проверки PrintTicket должен обнаружить это ограничение и изменить PrintTicket, чтобы избежать активации ограничения.

Если существует зависимость PrintCapabilities от параметров, указанных в PrintTicket, поставщики PrintCapabilities должны отслеживать значения Параметеринит и создавать соответствующий PrintCapabilities документ.
