# <a name="printcapabilities-document-construction-checklist"></a>Контрольный список для создания документа PrintCapabilities

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

[Сводка по типам элементов](summary-of-element-types.md) обсуждает различные элементы, составляющие документ PrintCapabilities. В этом разделе содержится контрольный список, который авторы документов PrintCapabilities могут использовать для создания документа PrintCapabilities, описывающего устройство.

1.  Определяет все атрибуты устройств, которые вносят вклад в конфигурацию устройства. Для каждого атрибута устройства определите, должен ли он быть представлен как конструкция функции или варианта или как конструкция параметра.

2.  Для каждой функции устройства определите, может ли она быть представлена компонентом, определенным в ключевых словах «Печать схемы». В противном случае необходимо будет ввести новую закрытую функцию (и соответствующий атрибут имени).

    -   Для экземпляров определенных компонентов, определяемых ключевыми словами схемы печати, необходимо определить каждое из доступных состояний, для которых можно установить эту функцию. Каждое состояние соответствует параметру экземпляра компонента. Определите, какие из этих состояний соответствуют экземплярам параметров печати, определяемым схемой, связанным с этой функцией, и укажите, для каких состояний требуется настраиваемый экземпляр параметра. В разделе [определения параметров](option-definitions.md) представлены сведения о создании новых экземпляров параметров и о том, как создавать экземпляры новых параметров на основе существующих экземпляров параметров.

    -   Для нестандартных экземпляров компонентов следует определить характеристики, которые можно использовать для различения одного варианта от другого. Представьте каждую из этих характеристик с помощью элемента Скоредпроперти, а в каждом экземпляре параметра назначьте каждому Скоредпроперти значение, относящееся к этому параметру. Убедитесь в наличии достаточного числа элементов Скоредпроперти, чтобы каждый параметр для данной функции был уникальным. Нестандартные экземпляры функций и параметров по своей природе являются подпортами. То есть другой драйвер не сможет найти эквивалентную функцию или параметр для сопоставления с нестандартным компонентом или параметром, указанным в PrintTicket, создаваемом вашим драйвером.

3.  Определите, должен ли любой из параметров содержать элементы ParameterRef. Дополнительные сведения см. в разделе [конструкции параметров](parameter-constructs.md) и [элементы ссылок на параметры](parameter-reference-elements.md).

4.  Для параметров определите, соответствует ли один из экземпляров Параметердеф, определенных в ключевых словах «Печать схемы», соответствующим образом. Если да, скопируйте экземпляр Параметердеф из ключевых слов схемы Print и измените значение каждого экземпляра изменяемого свойства для лучшего соответствия. Если ни один из экземпляров Параметердеф в ключевых словах схемы Print не является адекватным соответствием, создайте собственный экземпляр Параметердеф. Дополнительные сведения см. [в разделе Параметры в документе PrintCapabilities](parameters-in-the-printcapabilities-document.md).

5.  Убедитесь, что все экземпляры Property и Скоредпроперти, необходимые для документа ключевых слов схемы печати, находятся в документе PrintCapabilities и правильно инициализированы.

6.  При необходимости добавьте дополнительные экземпляры свойств и подсвойств. Вы можете ввести закрытые экземпляры свойств, если есть аспекты устройства, которые необходимо определить, не охваченные экземплярами свойств, определенными в ключевых словах «Печать схемы».

7.  Обратите внимание на соглашение об пространствах имен для атрибутов имени. Это относится к закрытым атрибутам имени, а также к тем, которые определены в ключевых словах «Печать схемы».

8.  Дочерние элементы одного и того же типа элемента не могут быть вложены в глубину более 10 элементов. Это правило применяется независимо к каждому типу элементов, которые могут быть определены.

Обратите внимание, что XML-содержимое документа с возможностями печати должно быть закодировано с помощью UTF-8 или UTF-16.

Обратите внимание, что набор компонентов, параметров и экземпляров Параметердеф не должен изменяться независимо от моментального снимка. Экземпляры Скоредпроперти, которые составляют каждый экземпляр параметра и значение, присваиваемое каждому элементу Скоредпроперти, также не должны изменяться. То же самое относится к экземплярам свойств, которые составляют каждый экземпляр Параметердеф.

Список дополнительных экземпляров свойств, которые необходимо предоставить для полного определения конструкций и параметров функции и варианта, см. в разделе [параметердеф](parameterdef.md) и [параметеринит](parameterinit.md). Например, каждый компонент должен указать его поведение пользовательского интерфейса, в частности, можно ли выбрать только один или несколько экземпляров параметров для каждого компонента одновременно. В документе по ключевым словам схемы печати определяются экземпляры свойств, где они должны находиться в документе PrintCapabilities и какие экземпляры значений, определенные в ключевых словах схемы печати, доступны.

Поставщик PrintCapabilities отвечает за эмиссию соответствующего значения для всех экземпляров свойств, зависящих от конфигурации. Например, если скорость печати зависит как от цветового режима, так и от используемого разрешения, поставщик PrintCapabilities должен отметить параметры цветового режима и разрешения, указанные в предоставляемом клиентом PrintTicket, и должны сообщить правильное значение скорости печати. Обратите внимание, что каждый экземпляр Скоредпроперти должен быть одиночным значением. экземпляр его значения не может измениться при изменении конфигурации устройства.

Обратите внимание, что экземпляры свойств, определенные в ключевых словах схемы Print, должны находиться в расположении, указанном здесь. Они не могут отображаться в произвольном месте документа PrintCapabilities. Экземпляры свойств, определенные частным образом, могут находиться в любом месте даже в виде вложенных свойств в экземплярах свойств, определенных схемой.

Обратите внимание, что функциональный конфликт между параметрами определяется как два неконфликтующих элемента схемы печати, которые имеют похожие функции, но являются различными функциями. Примером может быть Жобдуплексаллдокументсконтигуаусли и Документдуплекс; Оба типа представляют дуплексную функцию устройства, но отличаются в приложении функции, одна применяется ко всему заданию непрерывно, а другая — к документам. В случае указания двух таких элементов приоритет определяется производителем PrintCapabilities и объектом PrintTicket. Производитель PrintCapabilities должен правильно указывать ограничения между конфликтующими элементами с помощью атрибута CONSTRAINED. Элементы в открытой схеме печати, которые демонстрируют этот семантический конфликт, идентифицируются в их определении.
