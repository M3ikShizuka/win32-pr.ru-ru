# <a name="deleteprinter-function"></a>Функция Делетепринтер

Функция **делетепринтер** удаляет указанный объект принтера.

## <a name="syntax"></a>Синтаксис


```C++
BOOL DeletePrinter(
  _Inout_ HANDLE hPrinter
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хпринтер* \[ в, out\]
</dt> <dd>

Обрабатывает объект принтера, который будет удален. Используйте функцию [**опенпринтер**](openprinter.md) или [**аддпринтер**](addprinter.md) для получения маркера принтера.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена, возвращаемое значение будет ненулевым.

Если функция выполняется неудачно, возвращается нулевое значение.

## <a name="remarks"></a>Комментарии

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.

 

Если для указанного принтера имеются оставшиеся задания печати, **делетепринтер** помечает принтер на ожидание удаления, а затем удаляет его после печати всех заданий печати. Не удается добавить задания печати на принтер, помеченный для ожидающего удаления.

Принтер, помеченный для ожидающего удаления, не может удерживаться, но его задания печати можно хранить, возобновлять и перезапускать. Если принтер удерживается и имеются задания для принтера, **делетепринтер** завершается с ошибкой \_ отказа в доступе \_ .

Обратите внимание, что **делетепринтер** не закрывает переданный ему обработчик. Поэтому приложение по-прежнему должно вызывать [**клосепринтер**](closeprinter.md).
