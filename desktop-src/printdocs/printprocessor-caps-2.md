# <a name="printprocessor_caps_2-structure"></a>\_Структура принтпроцессор Cap \_ 2

Представляет сведения о принтера.

## <a name="syntax"></a>Синтаксис


```C++
typedef struct _PRINTPROCESSOR_CAPS_2 {
  DWORD dwLevel;
  DWORD dwNupOptions;
  DWORD dwPageOrderFlags;
  DWORD dwNumberOfCopies;
  DWORD dwNupDirectionCaps;
  DWORD dwNupBorderCaps;
  DWORD dwBookletHandlingCaps;
  DWORD dwDuplexHandlingCaps;
  DWORD dwScalingCaps;
} PRINTPROCESSOR_CAPS_2, *PPRINTPROCESSOR_CAPS_2;
```



## <a name="members"></a>Участники

<dl> <dt>

**двлевел**
</dt> <dd>

Значение, указывающее номер версии структуры.

</dd> <dt>

**двнупоптионс**
</dt> <dd>

Битовая маска, представляющая различные числа страниц документа, которые принтер может печатать на одной стороне физического листа. Младший значащий бит представляет одну страницу документа на каждой стороне, следующий бит представляет две страницы документа на стороне и т. д. Например, 0x0000810B указывает, что принтер поддерживает 1, 2, 4, 9 и 16 страниц документа на физическую сторону.

</dd> <dt>

**двпажеордерфлагс**
</dt> <dd>

Значение флага, указывающее порядок, в котором будут печататься страницы. Это может быть **Обычная \_ Печать**, **обратная \_ Печать** или **буклетная \_ Печать**.

</dd> <dt>

**двнумберофкопиес**
</dt> <dd>

Максимальное число копий, которое может быть обработано принтером.

</dd> <dt>

**двнупдиректионкапс**
</dt> <dd>

Доступные закономерности при печати нескольких страниц документа на одной стороне листа бумаги. Возможны следующие флаги:



| Значение                     | Значение                                                                                             |
|---------------------------|-----------------------------------------------------------------------------------------------------|
| ППКАПС \_ вправо, \_ затем \_ вниз | Страницы отображаются в строках справа налево, каждая последующая строка под ее предшественником.                 |
| ППКАПС \_ вниз, \_ затем \_ вправо | Страницы отображаются в столбцах сверху вниз, каждый последующий столбец справа от его предшественника. |
| ППКАПС \_ влево, \_ затем \_ вниз  | Страницы отображаются в строках слева направо, каждая последующая строка под ее предшественником.                 |
| ППКАПС \_ вниз, \_ затем \_ влево  | Страницы отображаются в столбцах сверху вниз, каждый последующий столбец слева от его предшественника.  |



 

</dd> <dt>

**двнупбордеркапс**
</dt> <dd>

Может быть только ППКАПС \_ границей. \_ это означает, что при печати нескольких страниц документа на одной стороне физического листа принтер может сообщить, следует ли печатать границу вокруг области изображения каждой страницы документа.

</dd> <dt>

**двбуклесандлингкапс**
</dt> <dd>

Может быть только ППКАПСная \_ брошюра \_ , указывающая, что принтер может печатать стиль буклета.

</dd> <dt>**двдуплексхандлингкапс**</dt> <dd> 

| Значение                                         | Значение                                                                                                                                                                                                                                                                                          |
|-----------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ППКАПС \_ обратные \_ страницы \_ на \_ обратный \_ дуплекс  | При печати в обратных и дуплексных режимах процессор может печатать порядок каждой пары страниц, поэтому вместо печати в порядке 4, 3, 2, 1, они будут напечатаны в порядке 3, 4, 1, 2.                                                                                                       |
| ППКАПС \_ не \_ Отправить \_ Дополнительные \_ страницы \_ для \_ дуплексного | При использовании двусторонней обработки обработчик заданий печати может давать не отправку дополнительной страницы при нечетном числе страниц документа. Процессор будет принимать значение так, как это возможно, но в тех случаях, когда ненужная пустая страница может привести к неправильному выходу, дополнительные страницы все еще могут быть отправлены. |



 

</dd> <dt>

**двскалингкапс**
</dt> <dd>

Может быть только ППКАПС \_ квадратным \_ масштабированием, указывая, что принтер может масштабировать изображение страницы.

</dd> </dl>

## <a name="remarks"></a>Remarks

значения всех членов структуры предоставляются функцией **жетпринтпроцессоркапабилитиес** , которая описана в поднаборе драйверов Windows.

Когда приложение вызывает [**жетпринтердата**](getprinterdata.md), диспетчер очереди вызовов вызывает функцию **жетпринтпроцессоркапабилитиес** обработчика заданий печати и задает имя значения, имеющего формат _DataType_ **принтпроккапс \_**, где *DataType* — это имя входного типа данных.
