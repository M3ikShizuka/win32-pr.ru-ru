# <a name="printer_notify_info-structure"></a>\_Структура сведений об уведомлении принтера \_

Структура **\_ \_ сведений об уведомлении принтера** содержит сведения о принтерах, возвращаемых функцией [**финднекстпринтерчанженотификатион**](findnextprinterchangenotification.md) . Эта функция возвращает эти сведения после выполнения операции ожидания на объекте уведомления об изменении принтера.

## <a name="syntax"></a>Синтаксис


```C++
typedef struct _PRINTER_NOTIFY_INFO {
  DWORD                    Version;
  DWORD                    Flags;
  DWORD                    Count;
  PRINTER_NOTIFY_INFO_DATA aData[1];
} PRINTER_NOTIFY_INFO, *PPRINTER_NOTIFY_INFO;
```



## <a name="members"></a>Члены

<dl> <dt>

**Version**
</dt> <dd>

Версия этой структуры. Установите для этого элемента значение 2.

</dd> <dt>

**Флаги**
</dt> <dd>

Битовый флаг, указывающий состояние структуры уведомлений. Если \_ \_ установлен бит уведомление об \_ ОТброшенном принтере, это означает, что некоторые уведомления пришлось отменять.

</dd> <dt>

**Количество**
</dt> <dd>

Число элементов [**\_ \_ \_ данных об уведомлении принтера**](printer-notify-info-data.md) в массиве **адата** .

</dd> <dt>

**адата**
</dt> <dd>

Массив структур [**\_ \_ \_ данных с уведомлением принтера**](printer-notify-info-data.md) . Каждый элемент массива определяет одно поле задания или сведения о принтере и предоставляет текущие данные для этого поля.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Если элемент **flags** имеет \_ установленный бит уведомления о принтерах \_ \_ , это указывает на то, что произошло переполнение или ошибка, а уведомления могли быть потеряны. В этом случае необходимо вызвать [**финднекстпринтерчанженотификатион**](findnextprinterchangenotification.md) и указать \_ \_ \_ флаг обновления параметров принтера для получения всех текущих данных. Пока вы не запрашиваете эту операцию обновления, система не будет отправлять дополнительные уведомления для этого объекта уведомления об изменении.
