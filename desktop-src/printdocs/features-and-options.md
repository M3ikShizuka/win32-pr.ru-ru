# <a name="features-and-options"></a>Функции и параметры

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Функции, параметры и конструкции параметров используются в документе PrintCapabilities для представления атрибутов устройства, которые вносят вклад в определение конфигурации устройства. Для примера конструкции "функция-вариант" рассмотрите печатающее устройство, которое может печататься в нескольких решениях. Атрибут устройства, определяющий разрешение, может быть представлен как экземпляр компонента, при этом каждое из возможных значений разрешения вывода представлено в виде отдельного экземпляра параметра. Один экземпляр параметра может представлять разрешение 300 dpi, другое может представлять разрешение 600 dpi и т. д.

Следует отметить, что схема печати требует, чтобы список компонентов, параметров и экземпляров Параметердеф, сообщаемых в каждом документе PrintCapabilities, оставался постоянным независимо от конфигурации. Это позволяет однозначным образом выразить конфигурации и зависимости в конфигурациях. Следствием этого требования является то, что у каждой функции и подфункции должен быть четко определенный параметр, независимо от настройки любой другой функции или подфункции. Таким образом, каждая подфункция должна иметь параметр, эквивалентный отсутствию op (параметр Off, disabled или None), который автоматически выбирается для всех подкомпонентов, когда пользователь выбирает в родительском компоненте параметр без-Op. И наоборот, если для одной из подкомпонентов задан параметр Enabled (включен), родительский компонент и другие связанные с ним подфункции также будут включены. Тем временем поставщик PrintTicket должен гарантировать (во время проверки PrintTicket), что определяются параметры для всех экземпляров компонента и компонентов, независимо от конфигурации устройства, и что параметры подфункции соответствуют параметрам родительского компонента. Это гарантирует, что устройству не будет присвоено несоответствие PrintTicket, где некоторые подфункции указывают, что, например, сшивание включена, но другие подфункции указывают, что Ассоциация отключена.

Обратите внимание, что авторы PrintCapabilities не обязаны использовать возможности вложенности элементов функций. При желании они могут представлять все экземпляры компонентов в плоской структуре как элементы того же уровня. Обратите внимание, что вложенная коллекция экземпляров компонентов может быть сведена просто путем перемещения всех подкомпонентов на корневой уровень. Единственная мера предосторожности, которую следует предпринять, заключается в том, чтобы убедиться, что атрибуты имени этих экземпляров компонентов уникальны.
