# <a name="value"></a>Значение

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Элемент value связывает литерал с типом.

## <a name="element-tag"></a>Тег элемента

<Value>

## <a name="xml-attributes"></a>XML-атрибуты

В следующей таблице перечислены атрибуты XML, которые могут быть связаны с этим элементом.



| Атрибут XML       | Сведения                                                                                                                                                                          |
|---------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| xsi:type<br/> | Задает тип данных value, который должен быть одним из следующих типов, определенных XSD: строка, целое число, десятичное, QName. Если он отсутствует, типом данных по умолчанию является String.<br/> |



 

Дополнительные сведения см. в разделе [XML Attributes](xml-attributes.md) .

## <a name="element-information"></a>Сведения об элементе

В следующей таблице перечислены элементы, которые могут быть родительскими для этого элемента, элементы, которые могут быть дочерними для этого элемента, и любые ограничения на сам элемент.



| Категория                   | Сведения                                                                                                                                                   |
|----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|
| Родительские элементы<br/> | параметеринит <br/> Свойство<br/> скоредпроперти<br/>                                                                                   |
| Дочерние элементы<br/>  | Допускается только символьное или целочисленное содержимое.<br/>                                                                                                |
| Этот элемент<br/>    | Разрешено содержимое null. Символьное содержимое должно соответствовать синтаксису, определяемому типом данных.<br/> Дублирование дочерних одноуровневых элементов не разрешено.<br/> |



 

## <a name="configuration-dependencies"></a>Зависимости конфигурации

Элементы значения, отображаемые в элементе Скоредпроперти, могут не иметь зависимостей конфигурации. Элементы значений, которые отображаются в элементах свойств, могут иметь произвольную зависимость от конфигурации.

## <a name="element-usage"></a>Использование элементов

Элемент value может присутствовать в свойстве или элементе Скоредпроперти. Целью элемента value является представление значения как стандартного типа данных XML. Тип данных указывается как XML-атрибут элемента value, xsi: Type. Обратите внимание, что поддерживаются не все определяемые XSD или XML-типы. Список поддерживаемых типов см. в разделе [Сводка по типам элементов](summary-of-element-types.md). Элемент value может содержать только символьное содержимое. Другие элементы не могут отображаться как содержимое в элементе value.

> [!Note]  
> Некоторые свойства и элементы Скоредпроперти, определяемые схемой печати, не содержат элемент value, поскольку их назначение просто является родителями вложенных свойств. Не следует добавлять элемент value в такие свойства, как свойства, которые не содержат элемент value.

 

Чтобы обеспечить соответствие платформе схемы печати, которая требует, чтобы элемент value или вложенные элементы присутствовали в элементах, поддерживающих значения, отсутствующее или неопределенное значение должно быть представлено представлением элемента value как пустого элемента. то есть, как <Value></Value>.

## <a name="example"></a>Пример

Определите значение типа Decimal и инициализируйте его значением "128,5".

``` syntax
<Value xsi:type="decimal">128.5</Value>
```
