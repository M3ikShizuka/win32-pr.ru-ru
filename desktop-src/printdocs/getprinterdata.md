# <a name="getprinterdata-function"></a>Функция Жетпринтердата

Функция **жетпринтердата** получает данные конфигурации для указанного принтера или сервера печати.

В Windows 2000 и более поздних версиях Windows вызов **жетпринтердата** эквивалентен вызову [**жетпринтердатаекс**](/windows/desktop/printdocs/getprinterdataex) с параметром *Пкэйнаме* , имеющим значение "принтердривердата".

## <a name="syntax"></a>Синтаксис


```C++
DWORD GetPrinterData(
  _In_  HANDLE  hPrinter,
  _In_  LPTSTR  pValueName,
  _Out_ LPDWORD pType,
  _Out_ LPBYTE  pData,
  _In_  DWORD   nSize,
  _Out_ LPDWORD pcbNeeded
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хпринтер* \[ окне\]
</dt> <dd>

Маркер принтера или сервера печати, для которого функция получает данные конфигурации. Используйте функцию [**опенпринтер**](openprinter.md), [**OpenPrinter2**](openprinter2.md)или [**аддпринтер**](addprinter.md) для получения маркера принтера.

</dd> <dt>

*пвалуенаме* \[ окне\]
</dt> <dd>

Указатель на строку, завершающуюся нулем, которая определяет извлекаемые данные.

Для принтеров эта строка представляет собой имя значения реестра в разделе "Принтердривердата" принтера в реестре.

Для серверов печати эта строка является одной из предопределенных строк, перечисленных в следующем разделе "Примечания".

</dd> <dt>

*птипе* \[ заполняет\]
</dt> <dd>

Указатель на переменную, которая получает значение, указывающее тип данных, полученных в *pData*. Функция возвращает тип, указанный в вызове [**сетпринтердата**](setprinterdata.md) или [**сетпринтердатаекс**](setprinterdataex.md) , который хранит данные. Если тип данных не требуется, присвойте этому параметру **значение NULL** .

</dd> <dt>

*pData* \[ заполняет\]
</dt> <dd>

Указатель на буфер, который получает данные конфигурации.

</dd> <dt>

*нсизе* \[ окне\]
</dt> <dd>

Размер (в байтах) буфера, на который указывает *pData* .

</dd> <dt>

*пкбнидед* \[ заполняет\]
</dt> <dd>

Указатель на переменную, которая получает размер данных конфигурации в байтах. Если размер буфера, указанный параметром *нсизе* , слишком мал, функция возвращает **ошибку \_ больше \_ данных**, а *пкбнидед* указывает требуемый размер буфера.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение **ошибки \_ Success**. Если функция завершается ошибкой, возвращается значение ошибки.

## <a name="remarks"></a>Комментарии

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.

 

**Жетпринтердата** извлекает данные конфигурации принтера, которые были заданы функцией [**сетпринтердатаекс**](setprinterdataex.md) или [**сетпринтердата**](setprinterdata.md) .

Чтобы запросить эти значения, вызовите функцию [**процедура RegQueryValueEx**](/windows/win32/api/winreg/nf-winreg-regqueryvalueexa) .

| **статусупдатеинтервал** | **REG \_ DWORD** | Указывает интервал (в минутах), в течение которого монитор порта обновляет Диспетчер очереди с состоянием порта.<br/> |



 

В Windows 7 и более поздних версиях Windows задания печати, отправленные на сервер печати, по умолчанию подготавливаются к просмотру на клиенте. Следующие значения настраивают отрисовку заданий печати на стороне клиента и могут быть считаны, если в *пвалуенаме* заданы следующие значения.



| Параметр                      | Тип данных      | Описание                                                                                                                                                                                                                                                                                                                                                                                                       |
|------------------------------|----------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **емфдеспулингсеттинг**     | **REG \_ DWORD** | Значение 0 или, если это значение отсутствует в реестре, включает отрисовку заданий печати на стороне клиента по умолчанию.<br/> Значение 1 отключает отрисовку заданий печати на стороне клиента.<br/>                                                                                                                                                                                                          |
| **форцеклиентсидерендеринг** | **REG \_ DWORD** | Значение 0 или, если это значение отсутствует в реестре, приведет к отображению заданий печати на клиенте. Если задание печати не может быть визуализировано на клиенте, оно будет подготовлено к просмотру на сервере. Если задание печати не может быть подготовлено к просмотру на сервере, оно завершится ошибкой.<br/> Значение 1 будет отображать задания печати на клиенте. Если задание печати не может быть визуализировано на клиенте, оно завершится ошибкой.<br/> |
