# <a name="print-schema-language-notes"></a>Заметки языка схемы печати

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

В этой спецификации слова, используемые для определения значимости каждого требования, записываются в верхнем регистре. Эти слова используются в соответствии с их определениями в RFC 2119, и их значения выводятся ниже:



| Термин                                                                | Описание                                                                                                                                                                                                                                                                                         |
|---------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="MUST"></span><span id="must"></span>ПРИХОДИТ<br/>       | Это слово (или прилагательное ОБЯЗАТЕЛЬНОе) означает, что элемент является абсолютным требованием спецификации.<br/>                                                                                                                                                                               |
| <span id="SHOULD"></span><span id="should"></span>СЛЕДУЕТ<br/> | Это слово (или прилагательное) означает, что в определенных обстоятельствах могут существовать действительные причины для пропуска этого элемента, но все последствия должны быть понятны, и следует тщательно взвесить их, прежде чем выбирать другой курс.<br/>                                     |
| <span id="MAY"></span><span id="may"></span>МАЯ<br/>          | Это слово (или прилагательное необязательно) означает, что этот элемент действительно является необязательным. Например, одна из реализаций может включать элемент, так как конкретный рынок или сценарий требует его или потому, что он расширяет продукт. Другая реализация может опускать один и тот же элемент.<br/> |
