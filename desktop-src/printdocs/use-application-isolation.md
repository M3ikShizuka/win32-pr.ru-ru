# <a name="how-to-use-application-isolation"></a>Как использовать изоляцию приложений

Приложения могут объявлять изоляцию драйвера принтера в манифесте приложения, чтобы изолировать приложение от драйвера принтера и повысить надежность приложения. Windows служба печати позволяет драйверам принтера выполняться в процессах, отдельных от процесса, в котором работает диспетчер очереди печати. С помощью этой функции приложение предотвращает его аварийное завершение в случае возникновения ошибки в драйвере принтера.

изоляция драйвера принтера реализована в Windows 7 и Windows Server 2008 R2.

### <a name="prerequisites"></a>Предварительные требования

-   управляемый код или приложение магазина Windows, которое использует Windows печати.

## <a name="instructions"></a>Instructions

### <a name="update-the-app-manifest"></a>Обновление манифеста приложения

Включение изоляции драйвера принтера требует добавления элемента **принтердриверисолатион** в манифест приложения. Это делается так.

1.  Измените манифест приложения, добавив элемент **принтердриверисолатион** со значением **true** в элемент **виндовссеттингс** элемента **Application** , как показано в этом примере.
    ```XML
    <application xmlns="urn:schemas-microsoft-com:asm.v3">
        <windowsSettings>
            <printerDriverIsolation xmlns="http://schemas.microsoft.com/SMI/2011/WindowsSettings">true</printerDriverIsolation>
        </windowsSettings>
    </application>
    ```

    

2.  Перестройте приложение.
