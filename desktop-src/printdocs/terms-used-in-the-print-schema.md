# <a name="terms-used-in-the-print-schema"></a>Термины, используемые в схеме печати

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

В следующем списке содержатся термины, характерные для схемы печати и связанных с ней технологий.



| Термин                                                                                                                                                                 | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>имя атрибута<br/>                                                                          | Имя атрибута — это маркер в левой части знака равенства в стандартном XML-атрибуте. Стандартным представлением XML-атрибута является *атрибут Name*= "*значение атрибута*". Несмотря на то, что термин «имя атрибута» не следует путать с атрибутом Name, определенным в платформе «Печать схемы». Атрибут Name, определенный в платформе Print Schema Framework, является определенным экземпляром имени атрибута. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>кандидатов<br/>                                                                                         | Функцию. Любой параметр, который относится к списку, вычисляемому в процессе оценки. См. условия, параметры оценки или сопоставления и параметр ссылки.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Настройка<br/>                                             | Описывает одно из состояний, которое устройство может предположить или поместить в. Устройство выполняет свою задачу по-разному в каждом состоянии. Существует различие между поведением устройства в каждом из этих состояний. Например, в одном состоянии носитель извлекается из одного лотка и заканчивается в другом лотке. Изменение одного или обоих лотков определяет другое состояние.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Противоречит<br/>                                                         | Сведения, необходимые для определения допустимости конфигурации. Ограничение, как правило, имеет вид: функция a, установленная в параметре параметр1, несовместима с параметром Феатуреб, равным параметру параметр2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>соответствующие элементы<br/>                                                  | Два элемента с одинаковым расположением относительно заданной пары исходных элементов. Элементы источника обычно являются двумя сравниваемыми элементами параметров.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>параметры<br/>                                                                                               | Функцию. Параметр, назначенный либо схемой печати, либо поставщиком PrintCapabilities в качестве параметра, назначенного функции, если PrintTicket не содержит явных параметров для этой функции. Каждый компонент имеет назначенный параметр по умолчанию.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>атрибут конфигурации устройства<br/>                          | Атрибут устройства, который участвует в определении конфигурации устройства. Атрибут конфигурации устройства представлен в платформе схемы печати с помощью конструкций "функция" или "параметр" или "параметр". Обратите внимание, что атрибут, используемый в этом контексте, не имеет связи ни с каким атрибутами XML, ни для печати элементов атрибута, определенных платформой схемы.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>Тип элемента<br/>                                                                                | Тип каждого XML-элемента, определенного в инфраструктуре печати схемы, определяется его тегом элемента XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>Общие элементы<br/>                                                              | См. соответствующие элементы.<br/> Функциями. Определяемая платформой схемы структура, представляющая определенный атрибут конфигурации устройства. Каждый экземпляр компонента представляет один атрибут устройства. Экземпляр компонента определяется путем явного определения каждого из его состояний.<br/> Параметр Identity. Параметр, который не имеет наблюдаемого действия при выборе; по сути, «No-Op». Результат всегда будет одинаковым, независимо от того, сколько раз выбран этот параметр.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>имя экземпляра<br/>                                                                             | XML-атрибут, определяющий конкретный экземпляр типа элемента. См. атрибут Name.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>Клиент интерфейса<br/>                                                                    | Сущность, которая потребляет документы PrintCapabilities и конструирует и потребляет PrintTicket. Распространенными примерами клиентов интерфейсов являются модули ПОЛЬЗОВАТЕЛЬСКОГО интерфейса и модули подготовки отчетов.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>поставщик интерфейса<br/>                                                              | Сущность, которая реализует и предоставляет функциональные возможности с помощью интерфейса. В схеме печати это количество для создания документов PrintCapabilities и проверки PrintTicket. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>расположение элемента<br/>                                                  | Расположение конкретного экземпляра элемента в иерархии элементов. Расположение определяется как определенным элементом, так и всеми предками этого элемента. Иными словами, Location является эквивалентом полного пути в файловой системе. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>атрибут Name<br/>                                                                          | Обозначение, заданное атрибуту XML с именем атрибута Name. Этот атрибут отображается в большинстве типов элементов, определенных в платформе схемы печати. Он идентифицирует и определяет экземпляры каждого типа элемента. Следующие два термина ссылаются на атрибуты имени: правило уникальности одного и того же уровня, а также соглашение о пространстве имен.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Соглашение об пространствах имен<br/>                 | Указывает, что атрибуты имени, определяемые схемой, а также любой объект, определенный схемой, существуют в выделенном пространстве имен, общедоступном пространстве имен, а атрибуты закрытого имени должны существовать в другом пространстве имен, которое может быть идентифицировано как принадлежащее определяющей стороне.<br/> Функцию. Конструкция платформы для печати схемы, используемая для представления состояний, которые могут принимать или помещать каждый компонент. Например, если функция представляет размер носителя, обычные значения параметров могут быть буквами, юридическими, A4, а также другими.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Параметр<br/>                                                             | Схема печати поддерживает возможность описания параметризованных данных в целях ввода данных пользователем. Существует три типа элементов структуры схемы печати, которые играют роли в параметрической среде входа: Параметердеф, Параметеринит и ParameterRef. Элемент Параметердеф характеризует параметр (его диапазон значений, тип данных и другие характеристики). Для инициализации параметра используется элемент Параметеринит. Элемент ParameterRef используется для ссылки на конкретный экземпляр элемента Параметеринит.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>параметризованные<br/>                                                                             | Функцию. Экземпляр параметра, содержащий по крайней мере один элемент Скоредпроперти, который в свою очередь содержит элемент ParameterRef, а не элемент value.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>абсолютное соответствие<br/>                                                              | Параметры. Два экземпляра параметров считаются полностью согласованными, если для каждого элемента Скоредпроперти в одном экземпляре параметра существует соответствующий экземпляр Скоредпроперти в другом экземпляре параметра, и оба элемента value в соответствующих экземплярах Скоредпроперти идентичны.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Схема PrintCapabilities<br/> | Схема, определяющая синтаксис и семантику, используемые документом PrintCapabilities. Схема PrintCapabilities и схема PrintTicket являются подмножествами схемы печати. См. также раздел Печать платформы схемы.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>Схема PrintTicket<br/>                         | Схема, определяющая синтаксис и семантику, используемые PrintTicket. Схема PrintTicket и схема PrintCapabilities являются подмножествами схемы печати. См. также раздел Печать платформы схемы.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Платформа для печати схемы<br/>         | Схема, определяющая синтаксис и семантику, используемые технологиями печатной схемы. Например, платформа схемы печати определяет коллекцию типов элементов и их использование, но не перечисляет все экземпляры элементов.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Вывод ключевых слов схемы<br/>             | Схема, определяющая отдельные экземпляры свойств, компонентов и параметров, которые обычно используются для описания устройств и форматирования заданий. Эта схема определяет общий язык, который делает переносимыми документы PrintCapabilities и PrintTicket. Также называется общедоступной схемой.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>закрыто — определено<br/>                                                                 | Любые экземпляры элементов, которые отображаются в документе PrintCapabilities или PrintTicket, которые не определены явным образом в ключевых словах схемы Print. Эти экземпляры элементов должны быть явно дополнены закрытым пространством имен для каждого поставщика.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>IsReference<br/>                                                                                         | Функцию. Параметр, по которому сравнивается каждый вариант-кандидат. Параметр ссылки обычно исходит от PrintTicket. Ознакомьтесь с условиями, оценкой или сопоставлением, а также вариантом кандидатов.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>корневой уровень<br/>                                                                                      | Корневой уровень — это основа иерархической структуры платформы схемы печати. Элемент корневого уровня определен как дочерний элемент элемента PrintCapabilities или PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>префикс области<br/>                                                                          | Текстовая метка, предварительно добавленная к ключевому слову схемы для предоставления контекстной области. Допустимыми префиксами для схемы печати являются "задание", "документ" и "страница". Например, префиксом области для Жобинпутбин является "Job".<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>скоредпроперти<br/>                                         | Конструкция платформы для печати схемы, представляющая характеристики, которые отличают или отличают один экземпляр параметра от другого. Например, разница между буквой и бумагой с юридическим размером.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>Оценка или сопоставление<br/>                                                              | Функцию. Процесс выбора из списка экземпляра параметра (Candidate), который наилучшим образом соответствует заданному экземпляру параметра (Reference). Тесно связанные термины являются вариантами кандидатов и ссылками. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Правило уникальности одного уровня<br/>     | Указывает, что все родственные элементы одного и того же типа элементов должны иметь уникальные атрибуты имени. Некоторые типы элементов исключены из этого правила. Эти исключения определяются в платформе схемы печати.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>однозначная и многозначная<br/>                    | Свойство. Свойство, значение которого является инвариантным (не зависит от конфигурации), считается однозначным. Такое свойство имеет только одно значение в любой возможной конфигурации. Свойство, зависящее от конфигурации, считается многозначным.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>суффиксы и суффиксы<br/>                                                                       | Ключевое слово схемы, представляющее имя компонента или параметра. Например, суффиксом для Жобинпутбин является "Инпутбин'.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Проверки<br/>                                                         | Процесс проверки того, что PrintTicket соответствует схеме, и представляет конфигурацию, которая может быть выполнена конкретным устройством.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Определено\_<br/>                                                   | Представленные здесь данные представляют собой заполнитель для конкретного драйвера, устройства или приложения, которые нужно создать. Он не передает, что данные должны быть строкой.<br/>                                                                                                                                                                                                                                                                                                                                                            |
