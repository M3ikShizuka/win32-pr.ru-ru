# <a name="constraints-and-printticket-validation"></a>Проверки ограничений и PrintTicket

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Как обсуждалось в разделе схемы PrintCapabilities, некоторые возможные результаты настройки, выраженные в документе PrintCapabilities, недопустимы для устройства. Говорят, что недопустимые конфигурации содержат конфликты ограничений (термин, используемый в мире файлов PPD и ГПД). Чтобы предотвратить конфликты ограничений, поставщики PrintTicket должны поддерживать операцию проверки PrintTicket, которую клиенты используют для выполнения проверки на их PrintTicket. Эта операция должна определять, может ли заданная конфигурация быть выполнена на устройстве. Если конфигурация не может быть выполнена (поскольку указанные компоненты и элементы параметров не существуют на текущем устройстве или из-за ограничения конфигурации), операция должна изменить входной PrintTicket, чтобы он содержал допустимые Неограниченные параметры. Эта операция может также удалить или проверить другие сведения в PrintTicket. Обратите внимание, что при обнаружении конфликта ограничений код проверки должен изменить значение одного из атрибутов устройства, чтобы избежать конфликта ограничений. [Определения параметров](option-definitions.md) предполагает, что для определения того, какой атрибут устройства следует изменить, следует использовать определенный драйвер устройства, чтобы лучше сохранить первоначальную намерение пользователя. Код проверки может жестко присвоить процессу оценки предпочтение одному атрибуту устройства для другого или использовать сведения, предоставляемые определенным экземпляром свойства в PrintTicket, для решения проблемы. Поскольку в ключевых словах схемы Print не определено свойство, позволяющее клиентам указывать относительный приоритет каждого атрибута устройства, все закрытые элементы свойств PrintTicket, используемые для этой цели, скорее всего, будут игнорироваться другими поставщиками PrintTicket.
