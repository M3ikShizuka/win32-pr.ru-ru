# <a name="print-schema-related-technologies"></a>Технологии печати Schema-Related

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

для платформа .NET Framework 3,0, Windows Vista и более поздних выпусков технологии PrintCapabilities и PrintTicket расширяют возможности схемы печати, чтобы обеспечить более широкую работу по печати.

## <a name="printcapabilities"></a>PrintCapabilities

Технология PrintCapabilities — это способ публикации настраиваемых параметров пользователя описание атрибутов и параметров для каждого задания. PrintCapabilities публикуются в документе XML, который называется документом PrintCapabilities, состоящим из терминов, определенных в ключевых словах и закрытых расширениях схемы печати. Документ PrintCapabilities можно рассматривать как "моментальный снимок" пользовательской конфигурации состояния текущего устройства, а также описание возможных конфигураций. Устройства (или драйверы устройств) создают документ PrintCapabilities (моментальный снимок) своего текущего набора настраиваемых параметров при запросе клиентов, которые могут быть либо приложениями, либо подсистемой печати. В этом документе описаны все настраиваемые PrintCapabilities, доступные на устройстве в настоящее время, такие как параметры завершения и параметры макета страницы. Документ PrintCapabilities явным образом описывает все атрибуты устройства и допустимые параметры для каждого атрибута. С помощью платформы печати схемы атрибуты устройства можно точно описать и эффективно сравнивать. Используя ключевые слова, содержащиеся в документе о ключевых словах схемы печати, и структуру, определенную в платформе схемы печати, устройства могут позволить клиентам более эффективно использовать PrintCapabilities. Дополнительные сведения см. в разделе [PrintCapabilities Schema and Document строительство](printcapabilities-schema-and-document-construction.md).

по сравнению с подсистемой печати в Microsoft Windows Server 2003 и более ранних версиях технология PrintCapabilities позволяет клиентским и печатным компонентам подсистемы прозрачно просматривать сведения, содержащиеся в текущем двоичном файле system PrintCapabilities системы Win32. Это позволяет клиенту запрашивать PrintCapabilities, принимать единообразный и хорошо понятный моментальный снимок XML и использовать его для создания PrintTicket для устройства без вызова пользовательского интерфейса драйвера.

## <a name="printticket"></a>PrintTicket

Технология PrintTicket является преемником текущей структуры DEVMODE. Это документ на основе расширяемого языка разметки, который указывает и сохраняет сведения о форматировании задания и конфигурации задания печати. Экземпляр PrintTicket назначает определенные параметры устройства и передает намерение пользователя. Существует два типа PrintTicket: универсальные PrintTicket, которые не формируются для конкретного устройства; и PrintTicket, относящиеся к конкретному устройству, которые создаются для конкретного устройства. Универсальные компании PrintTicket, предназначенные для переноса на другие устройства, получают их содержимое путем выбора параметров для каждого атрибута устройства, описанного только в ключевых словах «Печать схемы». PrintTicket, зависящий от устройства, получают свое содержимое из документа PrintCapabilities, в котором выбираются параметры для каждого атрибута устройства, объявленного этим документом. Эти PrintTicket также могут включать в себя частные расширения, специфичные для одной модели устройства или семейства моделей устройств. Дополнительные сведения см. в разделе [схема и построение документа PrintTicket](printticket-schema-and-document-construction.md).

По сравнению с текущей подсистемой печати, технология PrintTicket позволяет всем компонентам и клиентам подсистемы печати иметь прозрачный доступ к информации, которая в настоящее время хранится в общедоступной и закрытой частях структуры DEVMODE, используя четко определенный формат XML. Эта схема позволяет устранить текущие проблемы, возникшие при обновлении драйверов, а также сценарии понижения и несоответствия драйверов в драйверах, предназначенных для технологии PrintTicket. В настоящее время эти сценарии могут привести к утрате параметров и, следовательно, негативного взаимодействия с клиентами. Кроме того, PrintTicket позволяет использовать новые сценарии, такие как предоставление драйверу принтера возможности предоставлять свои частные параметры DEVMODE приложениям и настраиваемым подключаемым модулям в согласованном и однозначном порядке. Это позволяет сделать компоненты печати более прозрачными и более аккуратно обрабатывать параметры миграции. Интерфейсы PrintTicket будут предоставляться приложениям через методы для объектов управляемого кода, которые также будут доступны для скриптов. в новой платформе приложений, построенной на объектах управляемого кода в платформа .NET Framework 3,0, PrintTicket является стандартным способом описания параметров документа.
