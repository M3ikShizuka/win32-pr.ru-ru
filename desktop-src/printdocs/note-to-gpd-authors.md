# <a name="note-to-gpd-authors"></a>Примечание для авторов ГПД

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Для авторов документов PrintCapabilities, знакомых с файлами ГПД, некоторые ключевые слова ГПД не имеют эквивалентов в документе PrintCapabilities. В следующей таблице содержатся ключевые слова ГПД без эквивалента документа PrintCapabilities и причина, по которой нет эквивалента.



| Ключевое слово ГПД                                                                            | Объяснение                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|----------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| \*Ограничения<br/> \*инвалидкомбинатион<br/> \*конфликтприорити<br/> | Ограничения не определены в документе PrintCapabilities, так как клиенты PrintCapabilities не должны обрабатывать, применять или разрешать их. Эти задачи оставлены для поставщика PrintTicket во время проверки PrintTicket. Подключаемые модули Unidrv могут предоставлять собственный код проверки PrintTicket, или же они могут использовать Unidrv для выполнения этой проверки. В последнем случае Unidrv применяет все ограничения, определенные в файле ГПД.<br/> Монолитные драйверы должны предоставлять собственный код проверки PrintTicket и должны предоставлять собственный метод выражения и применения ограничений.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                       |
| \*дефаултоптион<br/>                                                             | Существует метод PrintTicket, возвращающий PrintTicket по умолчанию, который по определению имеет все параметры по умолчанию для каждого компонента.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| \*Объект featuretype<br/>                                                               | Компонент можно разделить на три различные категории:<br/> Функция, параметры которой определены в PrintTicket. Этот тип функции называется документом-прикрепленным, поскольку эти параметры непосредственно определяют способ обработки документа.<br/> Функция, параметры которой отражают атрибуты физических устройств, которые не управляются пользователями, например объем памяти устройства или наличие дополнительных надстроек, таких как подача бумаги или дуплексные устройства. Этот тип функции называется устройством-наклейкой или принтером. Состояние этого типа компонента важно, так как оно может ограничивать параметр, принадлежащий функции, прикрепленной к документу.<br/> Возможность прикрепления устройства или принтера может быть дополнительно классифицирована как функция наклейки принтера ПОЛЬЗОВАТЕЛЬСКОГО интерфейса или функция автоматической установки принтеров. Функция наклейки принтера пользовательского интерфейса должна отображаться в пользовательском интерфейсе, который может быть установлен администратором. Устройство автоматически обнаруживает функцию автоматической закрепления принтеров.<br/> |
| \*Переключить... \* Ситуации<br/>                                                         | Поставщик PrintCapabilities должен реализовать метод, создающий документ PrintCapabilities, который перечисляет функции прикрепления принтеров или документов, в зависимости от типа, указанного вызывающим объектом. По этой причине нет необходимости представлять одни и те же сведения с помощью самого документа PrintCapabilities.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
