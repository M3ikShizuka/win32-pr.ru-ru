# <a name="lineofferingmode_-constants"></a>\_Константы линеофферингмоде

Константы битового флага **линеофферингмоде \_** (TAPI версии 1,4 и более поздние) описывают различные подсостояния вызова предложения. Режим будет доступен в виде состояния вызова для приложения после трданситионс состояния вызова и в [**строке \_ каллстате**](line-callstate.md) , указывающей на вызов линекаллстате \_ . Эти значения используются, когда вызов осуществляется на адресе, который является общим (передается через мост) с другими станциями (см. [**\_ константы линеаддрессшаринг**](lineaddresssharing--constants.md)), в первую очередь на электронные ключевые системы.

<dl> <dt>

<span id="LINEOFFERINGMODE_ACTIVE"></span><span id="lineofferingmode_active"></span>**ЛИНЕОФФЕРИНГМОДЕ \_ Active**
</dt> <dd> <dl> <dt>



Указывает, что вызов уведомляется на текущей станции (будет сопровождаться ЛИНЕДЕВСТАТЕными \_ сообщениями), и если какое-либо приложение настроено на автоматический ответ, это можно сделать. Если режим вызова равен нулю, приложение должно предположить, что значение активно (это может быть ситуация на немостном адресе). (TAPI версии 1,4 и более поздней)


</dt> </dl> </dd> <dt>

<span id="LINEOFFERINGMODE_INACTIVE"></span><span id="lineofferingmode_inactive"></span>**ЛИНЕОФФЕРИНГМОДЕ \_ неактивен**
</dt> <dd> <dl> <dt>



Указывает, что вызов предлагается на нескольких станциях, но текущая станция не является оповещением (например, это может быть станция помощника, где состояние предложения — это Совет, например мигающий лампочку); программное обеспечение на станции, настроенном для автоматического ответа, предпочтительно не отвечать на вызов, так как это должно быть прерогативе на основной станции (оповещение), но для соединения вызова может использоваться [**линеансвер**](/windows/desktop/api/Tapi/nf-tapi-lineanswer) . (TAPI версии 1,4 и более поздней)


</dt> </dl> </dd> </dl>

## <a name="remarks"></a>Комментарии

Не расширяемый. Все 32 бит зарезервированы.

Для обеспечения обратной совместимости поставщик услуг должен проверить согласованную версию API в строке и не использовать эти значения ЛИНЕОФФЕРИНГМОДЕ, \_ если они не поддерживаются в согласованной версии. Приложения, которые не компания cognizant ЛИНЕОФФЕРИНГМОДЕ, \_ скорее всего, предполагают, что вызов, находящиеся в \_ ПРЕДЛОЖЕНии линекаллстате, находится в линеофферингмоде \_ Active.

Неактивные \_ значения линеофферингмоде Active и линеофферингмоде \_ используются, когда вызов осуществляется на адресе, который используется совместно с другими станциями (с мостом; см. [ \_ константы линеаддрессшаринг](lineaddresssharing--constants.md)), в первую очередь на электронные ключевые системы. Если для параметра Состояние вызова предложения задано значение "активно", это означает, что вызов уведомляется на текущей станции (он будет сопровождаться ЛИНЕДЕВСТАТЕными \_ сообщениями), и если какое-либо приложение настроено на автоматический ответ, это можно сделать. Если режим вызова имеет состояние "Неактивный", вызов предлагается на нескольких станциях, но текущая станция не является оповещением (например, это может быть станция помощника, где состояние предложения — это Совет, например мигающий источник); программное обеспечение на станции, настроенном для автоматического ответа, предпочтительно не отвечать на вызов, так как это должно быть прерогативе на основной станции (оповещение), но для соединения вызова можно использовать [**линеансвер**](/windows/desktop/api/Tapi/nf-tapi-lineanswer) . Если режим вызова равен нулю, приложение должно предположить, что значение активно (это может быть ситуация на немостном адресе).
