# <a name="extended-telephony-services"></a>Расширенные службы телефонии

API содержит механизм, позволяющий поставщикам поставщиков услуг расширить API телефонии с помощью расширений для конкретных устройств. Расширенные службы телефонии (или службы для конкретных устройств) включают все расширения API, определенные определенным поставщиком услуг. Поскольку API определяет только механизм расширения, поставщик услуг должен полностью указать определение поведения службы Extended-Telephony.

## <a name="tapi-2x-extended-telephony"></a>Расширенная телефония TAPI 2. x

Механизм расширения позволяет поставщикам поставщиков служб определять новые значения для некоторых типов перечисления и битовых флагов, а также добавлять элементы в большинство структур данных. Интерпретация расширений осуществляется с помощью идентификатора расширения поставщика услуг, идентификатора спецификации поддерживаемого набора расширений, которые могут быть разными производителями. Специальные функции и сообщения, такие как [**линедевспеЦифик**](/windows/win32/api/tapi/nf-tapi-linedevspecific) и [**фонедевспеЦифик**](/windows/win32/api/tapi/nf-tapi-phonedevspecific) , предоставляются в API, что позволяет приложению напрямую взаимодействовать с поставщиком услуг. Поставщик услуг также определяет параметры для каждой функции.

Для присвоения идентификаторов расширений поставщикам не требуется регистрироваться. Вместо этого служебная программа под названием ЕКСТИДЖЕН (Extidgen.exe) предоставляется в пакете SDK для платформы, который позволяет локальное поколение идентификаторов расширений. Этот уникальный идентификатор состоит из адреса адаптера Ethernet, случайного числа и времени суток. Идентификатор назначается набору расширений (до распределения), а не каждому отдельному экземпляру реализации этих расширений.
