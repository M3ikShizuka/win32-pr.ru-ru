# <a name="lineanswermode_-constants"></a>\_Константы линеансвермоде

Константы побитового флага **линеансвермоде \_** описывают, как действует существующий активный вызов на устройстве линии, отвечая на вызов другого *предложения* в той же строке.

<dl> <dt>

<span id="LINEANSWERMODE_DROP"></span><span id="lineanswermode_drop"></span>**ЛИНЕАНСВЕРМОДЕ \_ DROP**
</dt> <dd> <dl> <dt>



Текущий активный вызов будет автоматически удален.


</dt> </dl> </dd> <dt>

<span id="LINEANSWERMODE_HOLD"></span><span id="lineanswermode_hold"></span>**ЛИНЕАНСВЕРМОДЕ \_ удержание**
</dt> <dd> <dl> <dt>



Текущий активный вызов будет автоматически помещен в удержание.


</dt> </dl> </dd> <dt>

<span id="LINEANSWERMODE_NONE"></span><span id="lineanswermode_none"></span>**ЛИНЕАНСВЕРМОДЕ \_ None**
</dt> <dd> <dl> <dt>



Ответ на другой вызов в той же строке не влияет на существующий активный вызов в строке.


</dt> </dl> </dd> </dl>

## <a name="remarks"></a>Remarks

Без расширяемости. Все 32 бит зарезервированы.

Если вызов поступает в (предлагается) в момент, когда другой вызов уже активен, то новый вызов подключается с помощью вызова [**линеансвер**](/windows/desktop/api/Tapi/nf-tapi-lineanswer). Воздействие на существующий активный вызов зависит от возможностей устройства линии. Первый вызов может быть нетронут, он может быть автоматически удален или автоматически помещен в удержание.
