# <a name="call-state-timer"></a>Таймер состояния вызова

В настоящее время все временные вызовы остаются в приложении. Это может быть утомительным, если приложение наблюдает за большим количеством вызовов, и если имеется несколько приложений, которые, возможно, на нескольких серверах, они могут быть необходимы для поддержки таймеров в одних и тех же вызовах. Поэтому он имеет более осмысленное значение для обработки времени вызова сервером.

Элемент **тстатинтритиме** в [**линекаллстатус**](/windows/desktop/api/Tapi/ns-tapi-linecallstatus) позволяет получать сведения о времени вызовов в состояниях. Элемент (типа **систиме**) указывает время, когда было введено текущее состояние.
