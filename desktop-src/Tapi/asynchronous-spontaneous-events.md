# <a name="asynchronous-spontaneous-events"></a>Асинхронное безсобытие

Существует еще один класс асинхронных событий, помимо описанных выше. Эти события «недоступны» в том смысле, что они не являются прямыми результатами соответствующих запросов. Эти события обнаруживаются в таких случаях, как при поступлении входящего вызова, или когда исходящий вызов поступает из "звонка" в состояние "ответ". Когда TAPI сначала инициализирует взаимодействия с ТСПИ для конкретного устройства, он передает указатель на процедуру обратного вызова, которая вызывается для создания отчетов о таких событиях. Кроме того, этот указатель процедуры является обработчиком устройства, который поставщик услуг включает в качестве одного из фактических параметров обратного вызова.
