# <a name="itmediacollectiondelete-method"></a>Итмедиаколлектион::D удалить метод

\[ В Windows Vista, Windows Server 2008 и последующих версиях операционной системы нельзя использовать встречные средства и элементы управления Конференц-телефонией и интерфейсы. API клиента RTC предоставляет аналогичные функциональные возможности.\]

Метод **Delete** удаляет носитель, соответствующий указанному индексу.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Delete(
  [in] LONG Index
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Индекс* \[ окне\]
</dt> <dd>

Индекс удаляемого носителя.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод может возвращать одно из этих значений.



| Код возврата                                                                                                                              | Описание                                                                                              |
|------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>                                                     | Метод успешно выполнен.<br/>                                                                             |
| <dl> <dt>**E \_ INVALIDARG**</dt> </dl>                                             | Значение параметра *индекса* меньше 1 или больше текущего числа элементов.<br/> |
| <dl> <dt>**E \_ OUTOFMEMORY**</dt> </dl>                                            | Недостаточно памяти для выполнения операции.<br/>                                          |
| <dl> <dt>**\_Ошибка E**</dt> </dl>                                                   | Внутренняя ошибка (возникает, только если предыдущий вызов вернул ошибку).<br/>                      |
| <dl> <dt>**E \_ нотимпл**</dt> </dl>                                                | Этот метод еще не реализован.<br/>                                                           |
| <dl> <dt>**ЗНАЧЕНИЕ \_ HRESULT \_ из \_ кода ошибки ( \_ \_ уничтоженный BLOB-объект сдпблб CONF \_ )**</dt> </dl> | BLOB-объект SDP не существует.<br/>                                                                   |



 

## <a name="remarks"></a>Комментарии

Большинство списков C и C++ основаны на 0, но этот индекс основан на 1 для Visual Basic совместимости, то есть первый элемент имеет номер индекса 1.
