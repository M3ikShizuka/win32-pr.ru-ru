# <a name="lineforwardmode_-constants"></a>\_Константы линефорвардмоде

Константы побитового флага **линефорвардмоде \_** описывают условия, при которых вызовы к адресу можно перенаправить.

<dl> <dt>

<span id="LINEFORWARDMODE_BUSY"></span><span id="lineforwardmode_busy"></span>**ЛИНЕФОРВАРДМОДЕ \_ занят**
</dt> <dd> <dl> <dt>



Пересылка всех вызовов в состоянии Busy, независимо от их происхождения. Используйте это значение при пересылке внутренних и внешних вызовов в состоянии Busy, и отсутствие ответа не может контролироваться отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYEXTERNAL"></span><span id="lineforwardmode_busyexternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусекстернал**
</dt> <dd> <dl> <dt>



Пересылка всех внешних вызовов в состоянии Busy. Используйте это значение при пересылке внутренних и внешних вызовов в состоянии Busy, и отсутствие ответа можно контролировать отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYINTERNAL"></span><span id="lineforwardmode_busyinternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусинтернал**
</dt> <dd> <dl> <dt>



Пересылка всех внутренних вызовов на занятии. Используйте это значение при пересылке внутренних и внешних вызовов в состоянии Busy, и отсутствие ответа можно контролировать отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYNA"></span><span id="lineforwardmode_busyna"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусина**
</dt> <dd> <dl> <dt>



Перенаправлять все вызовы на "занят" или "нет", независимо от их происхождения. Используйте это значение при пересылке внутренних и внешних вызовов в состоянии Busy, и отсутствие ответа не может контролироваться отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYNAEXTERNAL"></span><span id="lineforwardmode_busynaexternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусинаекстернал**
</dt> <dd> <dl> <dt>



Пересылка всех внешних вызовов в состоянии занятости/без ответа. Используйте это значение, если Пересылка вызовов перегружена, и ни один ответ не может управляться отдельно для внутренних вызовов.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYNAINTERNAL"></span><span id="lineforwardmode_busynainternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусинаинтернал**
</dt> <dd> <dl> <dt>



Пересылка всех внутренних вызовов по занятости/без ответа. Используйте это значение, если Пересылка вызовов перегружена, и ни один ответ не может управляться отдельно для внутренних вызовов.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYNASPECIFIC"></span><span id="lineforwardmode_busynaspecific"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусинаспеЦифик**
</dt> <dd> <dl> <dt>



Пересылка занятости/отсутствие ответа на все вызовы, порожденные по указанному адресу (Выборочная переадресация вызовов).


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_BUSYSPECIFIC"></span><span id="lineforwardmode_busyspecific"></span>**ЛИНЕФОРВАРДМОДЕ \_ бусиспеЦифик**
</dt> <dd> <dl> <dt>



Пересылка занятых всех вызовов, поступающих по указанному адресу (Выборочная переадресация вызовов).


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_NOANSW"></span><span id="lineforwardmode_noansw"></span>**ЛИНЕФОРВАРДМОДЕ \_ ноансв**
</dt> <dd> <dl> <dt>



Пересылать все вызовы без ответа независимо от их происхождения. Используйте это значение, если переадресация вызовов для внутренних и внешних вызовов не может управляться отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_NOANSWEXTERNAL"></span><span id="lineforwardmode_noanswexternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ ноансвекстернал**
</dt> <dd> <dl> <dt>



Пересылать все внешние вызовы без ответа. Используйте это значение при пересылке внутренних и внешних вызовов без ответа, которые можно контролировать отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_NOANSWINTERNAL"></span><span id="lineforwardmode_noanswinternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ ноансвинтернал**
</dt> <dd> <dl> <dt>



Пересылать все внутренние вызовы без ответа. Используйте это значение при пересылке внутренних и внешних вызовов без ответа, которые можно контролировать отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_NOANSWSPECIFIC"></span><span id="lineforwardmode_noanswspecific"></span>**ЛИНЕФОРВАРДМОДЕ \_ ноансвспеЦифик**
</dt> <dd> <dl> <dt>



Пересылка без ответа на все вызовы, созданные по указанному адресу (Выборочная переадресация вызовов).


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_UNAVAIL"></span><span id="lineforwardmode_unavail"></span>**ЛИНЕФОРВАРДМОДЕ \_ НЕсвободно**
</dt> <dd> <dl> <dt>



Вызовы перенаправляются, но условия, при которых происходит пересылка, неизвестны и никогда не будут известны поставщику услуг. (TAPI версии 1,4 и более поздней)


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_UNCOND"></span><span id="lineforwardmode_uncond"></span>**ЛИНЕФОРВАРДМОДЕ \_ унконд**
</dt> <dd> <dl> <dt>



Пересылать все вызовы безусловно, независимо от их происхождения. Используйте это значение, если неусловная пересылка для внутренних и внешних вызовов не может управляться отдельно. Безусловное перенаправление переопределяет перенаправление перенаправления "занят" и/или не имеет условий ответа.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_UNCONDEXTERNAL"></span><span id="lineforwardmode_uncondexternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ ункондекстернал**
</dt> <dd> <dl> <dt>



Пересылка всех внешних вызовов безусловно. Используйте это значение, если неусловная пересылка для внутренних и внешних вызовов может управляться отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_UNCONDINTERNAL"></span><span id="lineforwardmode_uncondinternal"></span>**ЛИНЕФОРВАРДМОДЕ \_ ункондинтернал**
</dt> <dd> <dl> <dt>



Пересылать все внутренние вызовы безусловно. Используйте это значение, если неусловная пересылка для внутренних и внешних вызовов может управляться отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_UNCONDSPECIFIC"></span><span id="lineforwardmode_uncondspecific"></span>**ЛИНЕФОРВАРДМОДЕ \_ ункондспеЦифик**
</dt> <dd> <dl> <dt>



Безусловно пересылать все вызовы, порожденные по указанному адресу (Выборочная переадресация вызовов).


</dt> </dl> </dd> <dt>

<span id="LINEFORWARDMODE_UNKNOWN"></span><span id="lineforwardmode_unknown"></span>**ЛИНЕФОРВАРДМОДЕ \_ неизвестный**
</dt> <dd> <dl> <dt>



Вызовы перенаправляются, но условия, при которых произойдет пересылка, в настоящее время не известны. Возможно, условия могут быть известны в будущем. (TAPI версии 1,4 и более поздней)


</dt> </dl> </dd> </dl>

## <a name="remarks"></a>Комментарии

Без расширяемости. Все 32 бит зарезервированы.

Битовые флаги, определенные ЛИНЕФОРВАРДМОДЕ \_ , не являются ортогональными. Безусловное перенаправление игнорирует любое определенное условие, например «занят» или «нет ответа». Если неусловная пересылка не действует, перенаправление на занятый и без ответа можно контролировать отдельно или не отдельно. Если управление осуществляется отдельно, \_ Флаги линефорвардмоде Busy и линефорвардмоде \_ ноансв можно использовать отдельно. Если управление не осуществляется отдельно, \_ необходимо использовать флаг линефорвардмоде бусина. Аналогично, если переадресация внутренних и внешних вызовов может контролироваться отдельно, то ЛИНЕФОРВАРДМОДЕ \_ внутренние и линефорвардмоде \_ Внешние флаги можно использовать отдельно. в противном случае используется сочетание.

Возможности адресации указывают, какие режимы перенаправления доступны для каждого адреса, назначенного линии. Приложение может использовать [**линефорвард**](/windows/desktop/api/Tapi/nf-tapi-lineforward) для установки условий переадресации в параметре.

Для обеспечения обратной совместимости поставщик услуг должен проверить согласованную версию API в строке и не использовать эти значения ЛИНЕФОРВАРДМОДЕ, \_ Если согласованная версия не поддерживает их.
