<span data-ttu-id="47306-108">Чтобы обеспечить оптимальное взаимодействие различных версий приложений, версий самой TAPI и версий поставщиков услуг разных поставщиков, Microsoft телефонии <span data-ttu-id="47306-113">Эти два типа согласования версий обрабатываются двумя различными процедурами: [**тспи \_ расширения не требуются, можно пропустить согласование версии расширения.</span><span class="sxs-lookup"><span data-stu-id="47306-114">Extension version negotiation can be skipped if extensions are not wanted.</span></span> <span data-ttu-id="47306-115">Если эти диапазоны введены при наложении согласования, поставщик услуг должен вернуть значение в перекрывающейся части диапазона в результате согласования.</span><span class="sxs-lookup"><span data-stu-id="47306-115">If these ranges input during negotiation overlap, the service provider must return a value within the overlapping portion of возвращает ошибку.</span><span class="sxs-lookup"><span data-stu-id="47306-117">If the ranges do not overlap, the two parties are incompatible and the function returns an error.</span></span>

<span data-ttu-id="47306-118">Результаты согласования просто указывают, что поставщик службы готов к работе по определенному номеру версии, но не фиксирует поставщик услуг для этого.</span><span class="sxs-lookup"><span data-stu-id="47306-118">Results of a negotiation simply indicate that the service provider is willing to operate at a particular version number, but do not commit the service provider to doing so.</span></span> <span data-ttu-id="47306-119">Например, TAPI может выполнить повторное согласование, чтобы определить идеальную версию после согласования возможной версии.</span><span class="sxs-lookup"><span data-stu-id="47306-119">For example, TAPI can renegotiate to determine an ideal version after having negotiated a possible version.</span></span> <span data-ttu-id="47306-120">Версия интерфейса ТСПИ фиксируется только при открытии строки с помощью [**тспи \_ линеопен**](/windows/win32/api/tspi/nf-tspi-tspi_lineopen) и остается недоступной до закрытия устройства.</span><span class="sxs-lookup"><span data-stu-id="47306-120">The TSPI interface version is only committed when a line is opened using [**TSPI\_lineOpen**](/windows/win32/api/tspi/nf-tspi-tspi_lineopen) and survives until the device is closed.</span></span> <span data-ttu-id="47306-121">Версия расширения зафиксируется при вызове функции [**тспи \_ линеселектекстверсион**](/windows/win32/api/tspi/nf-tspi-tspi_lineselectextversion) и остается в неработоспособном виде до тех пор, пока выбор не будет отменен путем выбора версии расширения 0.</span><span class="sxs-lookup"><span data-stu-id="47306-121">The extension version is committed when the [**TSPI\_lineSelectExtVersion**](/windows/win32/api/tspi/nf-tspi-tspi_lineselectextversion) function is called, and survives until the selection is canceled by selecting extension version zero.</span></span>

<span data-ttu-id="47306-122">Выбор версии расширения может происходить много раз, включая, когда действует версия расширения.</span><span class="sxs-lookup"><span data-stu-id="47306-122">Extension version selection can happen many times, including while an extension version is in effect.</span></span> <span data-ttu-id="47306-123">Так как поставщик услуг фиксируется в версии расширения, его диапазон поддерживаемых версий ограничивается точностью до этой версии расширения.</span><span class="sxs-lookup"><span data-stu-id="47306-123">Because the service provider is committed to the extension version, its range of supported versions narrows to exactly that extension version.</span></span> <span data-ttu-id="47306-124">Например, рассмотрим поставщик услуг, который обычно совместим с версиями расширений 1,0 – 5,5.</span><span class="sxs-lookup"><span data-stu-id="47306-124">For example, consider a service provider that is normally compatible with extension versions 1.0 through 5.5.</span></span> <span data-ttu-id="47306-125">Если версия 3,0 действует, пока вызывающий объект пытается согласовать версию в диапазоне от 1,0 до 5,5, согласование возвращает 3,0.</span><span class="sxs-lookup"><span data-stu-id="47306-125">If version 3.0 is in effect while a caller attempts to negotiate a version within the range 1.0 to 5.5, the negotiation returns 3.0.</span></span>
