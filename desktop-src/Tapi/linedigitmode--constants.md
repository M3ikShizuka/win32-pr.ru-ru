# <a name="linedigitmode_-constants"></a>\_Константы линедигитмоде

Константы **линедигитмоде \_** описывают различные типы создания неизменяемых цифр.

<dl> <dt>

<span id="LINEDIGITMODE_DTMF"></span><span id="linedigitmode_dtmf"></span>**ЛИНЕДИГИТМОДЕ \_ DTMF**
</dt> <dd> <dl> <dt>



Использует сигналы DTMF для сигнализации цифр. Допустимые цифры: от 0 до 9, "" \* , " \# ", "A", "B", "C" и "d".


</dt> </dl> </dd> <dt>

<span id="LINEDIGITMODE_DTMFEND"></span><span id="linedigitmode_dtmfend"></span>**ЛИНЕДИГИТМОДЕ \_ дтмфенд**
</dt> <dd> <dl> <dt>



Использует сигналы DTMF для сигнализации цифр и обнаружения краев вниз. Допустимые цифры: от 0 до 9, "" \* , " \# ", "A", "B", "C" и "d".


</dt> </dl> </dd> <dt>

<span id="LINEDIGITMODE_PULSE"></span><span id="linedigitmode_pulse"></span>**ЛИНЕДИГИТМОДЕ \_ импульс**
</dt> <dd> <dl> <dt>



Использует роторные импульсные последовательности для сигнализации цифр. Допустимые цифры: от 0 до 9.


</dt> </dl> </dd> </dl>

## <a name="remarks"></a>Комментарии

Без расширяемости. Все 32 бит зарезервированы.

При создании или обнаружении цифр можно указать цифровой режим. Обратите внимание, что количество импульсов генерируется путем создания и нарушения цепи локального цикла. Эти импульсы продаются параметром. Удаленный сервер просто просматривает это как серию нечередующихся щелчков звука. Таким образом, обнаружение цифр, отправленных в виде импульсов, должно иметь возможность обнаруживать эти последовательности от 1 до 10 звуковых щелчков.
