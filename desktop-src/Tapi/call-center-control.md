# <a name="call-center-control"></a>Управление центром обработки вызовов

Базовый интерфейс TAPI 2,2 можно использовать для управления центрами обработки вызовов и другими элементами сетевой инфраструктуры телефонии (например, серверами IVR и голосовой почты) через Управление вызовами сторонних производителей. Для создания ACD-прокси, которые будут выполняться на серверах, в TAPI 2,2 были добавлены функции для предоставления дополнительной помощи разработчикам приложений.

В следующих разделах описываются функции TAPI 2,2, которые можно использовать для реализации элементов управления центра обработки вызовов.

-   [Моделирование центра обработки звонков](modeling-of-a-call-center.md)
-   [Станций](stations.md)
-   [Прогнозирующий набор](predictive-dialing.md)
-   [Очереди вызовов и точки маршрутизации](call-queues-and-route-points.md)
-   [Мониторинг и контроль агента ACD](acd-agent-monitoring-and-control.md)
-   [Управление состоянием станции](station-status-control.md)
-   [Управление состоянием станции](station-status-control.md)
-   [Таймер состояния вызова](call-state-timer.md)
-   [Таймеры событий мультимедиа](media-event-timers.md)

Интерфейс TAPI 3. x — предпочтительный API для написания приложений ACD Agent. Сведения об использовании этих интерфейсов и методов см. в разделе [элементы управления центра обработки вызовов](./about-call-center-controls.md) . Однако при необходимости TAPI 2,2 остается пригодным для полноценных наборов приложений.
