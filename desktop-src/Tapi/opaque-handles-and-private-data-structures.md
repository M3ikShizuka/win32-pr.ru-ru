# <a name="opaque-handles-and-private-data-structures"></a>Непрозрачные дескрипторы и закрытые структуры данных

Непрозрачные дескрипторы используются в ТСПИ для ссылки на структуры данных, представляющие линии, телефоны и внешний вид вызовов. Они отображаются как параметры большинства функций и обратных вызовов. Существует несколько функций, которые ссылаются на устройство, используя идентификатор устройства вместо непрозрачного маркера. В этом случае ссылка указывает на само устройство, а не на структуру данных. Функции, которые работают таким образом, обычно вызываются на ранних этапах инициализации до создания структур данных и обмена непрозрачными дескрипторами.

Поставщик услуг должен решить, как интерпретировать эти дескрипторы. Поставщик услуг обычно использует указатель на структуру данных или индекс в массиве структур данных в качестве непрозрачного маркера. Единственное ограничение заключается в том, что поставщик услуг не может использовать значение 0 в качестве [хдрвлине](hdrvline.md), Хдрвкалл или [хдрвфоне](hdrvphone.md). Значение 0 или **null** для маркера имеет специальное значение в некоторых операциях.
