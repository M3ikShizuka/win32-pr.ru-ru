# <a name="ikeyframecontrol-interface"></a>Интерфейс Икэйфрамеконтрол

\[**Икэйфрамеконтрол** недоступен для использования в Windows Vista, windows Server 2008 и последующих версиях операционной системы. API клиента RTC предоставляет аналогичные функциональные возможности.\]

Интерфейс **икэйфрамеконтрол** реализуется с помощью MSP-интерфейса [h. 323](h323-msp.md) и доступен только для объектов потока h. 323. Этот интерфейс предоставляет методы, обеспечивающие создание и обработку терминалов, которые могут взаимодействовать между клиентами H323 и SDP. В нем есть два метода, которые позволяют приложению запрашивать удаленную конечную точку для обновления видеоизображения с помощью опорного кадра.

## <a name="members"></a>Элементы

Интерфейс **икэйфрамеконтрол** наследует от интерфейса [**IUnknown**](/windows/desktop/api/unknwn/nn-unknwn-iunknown) . **Икэйфрамеконтрол** также имеет следующие типы членов:

-   [Методы](#methods)

### <a name="methods"></a>Методы

Интерфейс **икэйфрамеконтрол** содержит следующие методы.



| Метод                                                                  | Описание                                                                                 |
|:------------------------------------------------------------------------|:--------------------------------------------------------------------------------------------|
| [**периодикупдатепиктуре**](ikeyframecontrol-periodicupdatepicture.md) | Настраивает таймер в потоке, который периодически запрашивает обновления изображений.<br/> |
| [**упдатепиктуре**](ikeyframecontrol-updatepicture.md)                 | Запрашивает у отправителя видео обновление изображения.<br/>                                     |
