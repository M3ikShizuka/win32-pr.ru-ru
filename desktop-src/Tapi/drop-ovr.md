# <a name="drop"></a>Drop

Удаление или отключение сеанса прекращает связь. Приложение может отправлять сведения о пользователе во время отключения, если поставщик услуг его поддерживает.

Обычными причинами для удаления сеанса является то, что пользователь запросил отключение или отбросил другой конец сеанса. Операция DROP также может вызываться, когда TAPI предлагает сеанс приложения. Если поставщик услуг поддерживает это, то в результате приложение отклоняет вызов.

При вызове операции удаления связанные сеансы иногда также могут быть затронуты. Например, удаление вызова конференции может удалить всех отдельных участников. Сообщения об изменении состояния отправляются в приложение для всех вызовов, состояние которых затронуто.

В различных конфигурациях с мостом или с поддержкой моста, когда в вызове несколько Сторон, операция Drop может не очистить вызов. Например, в ситуации с мостом вызов не может быть удален, так как состояние других станций в вызове может управляться. Вместо этого вызов можно просто изменить на неактивное состояние, когда оно останется подключенным к другим станциям.

После операции DROP идентификатор сеанса и большинство ресурсов, связанных с сеансом, будут использоваться для большинства операций запроса. Если приложению больше не требуются эти ресурсы, оно должно [завершить сеанс](terminate-a-session-ovr.md) , чтобы избежать утечек памяти.
