# <a name="supplementary-telephony-services"></a>Дополнительные службы телефонии

Дополнительные службы телефонии — это коллекция всех служб, определенных API, не входящих в базовый набор телефонии. Он включает все так называемые дополнительные функции, имеющиеся в современных УАТС, такие как удержание, перенос, Конференция, припарк и т. д. Все дополнительные функции считаются необязательными. Это значит, что поставщик услуг решает, какие из этих служб она делает или не предоставляет.

Приложение может запросить линию или телефонное устройство для набора дополнительных служб, которые он предоставляет, используя такие функции, как [**линежетдевкапс**](/windows/win32/api/tapi/nf-tapi-linegetdevcaps) или [**линежетаддресскапс**](/windows/win32/api/tapi/nf-tapi-linegetaddresscaps). Одна дополнительная служба может состоять из нескольких вызовов функций и сообщений. API телефонии, а не разработчик поставщика услуг, определяет поведение каждой из этих дополнительных функций. Поставщик услуг должен предоставлять дополнительную службу телефонии, только если он может реализовать точное значение, определенное API. В противном случае эта функция должна быть предоставлена в качестве расширенной службы телефонии.

Как упоминалось в основных службах телефонии, службы телефонных устройств считаются необязательными. Таким образом, все службы телефонных устройств являются частью дополнительной телефонии. Список функций дополнительной телефонии см. в [справочнике по быстрой функции TAPI](./tapi-quick-function-reference.md).
