# <a name="lineaddressmode_-constants"></a>\_Константы линеаддрессмоде

Константы побитового флага **линеаддрессмоде \_** описывают различные способы идентификации адреса на устройстве линии.

<dl> <dt>

<span id="LINEADDRESSMODE_ADDRESSID"></span><span id="lineaddressmode_addressid"></span>**ЛИНЕАДДРЕССМОДЕ \_ ADDRESSID**
</dt> <dd> <dl> <dt>



Адрес указывается с небольшим целым числом в диапазоне от 0 до *двнумаддрессес* минус один, где *двнумаддрессес* — это значение в возможностях устройства линии.


</dt> </dl> </dd> <dt>

<span id="LINEADDRESSMODE_DIALABLEADDR"></span><span id="lineaddressmode_dialableaddr"></span>**ЛИНЕАДДРЕССМОДЕ \_ диалаблеаддр**
</dt> <dd> <dl> <dt>



Адрес указывается по номеру телефона.


</dt> </dl> </dd> </dl>

## <a name="remarks"></a>Remarks

Старшие 16 бит можно назначить для расширений для конкретных устройств. 16 младших разрядов зарезервированы.

Эта константа используется для выбора адреса в строке, в которой происходит вызов. В обычной модели адрес выбирается посредством идентификатора адреса. Идентификаторы адресов — это механизм, используемый для идентификации адресов по всему интерфейсу TAPI. Однако в некоторых средах, при совершении вызова, часто бывает более практично указывать исходящий адрес вызова по номеру телефона, а не по идентификатору адреса. Одним из примеров является возможность моделирования большого количества станций (третьих сторон) на коммутаторе с помощью одного линейного устройства с множеством адресов. Линия представляет набор всех станций, каждая из которых сопоставляется с адресом с собственным основным номером телефона и идентификатором адреса.
