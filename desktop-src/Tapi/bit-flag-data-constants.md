# <a name="bit-flag-data-constants"></a>Константы данных Bit-Flag

Для расширяемых констант данных в виде битовых флагов поставщик поставщика услуг может определять новые значения для указанных битов. Так как большинство констант флагов имеют значение **DWORD**, для распространенных расширений обычно зарезервировано определенное число младших битов, а оставшиеся верхние биты доступны для расширений, зависящих от поставщика. Распространенным битовым флагам назначается ноль, а расширения, зависящие от поставщика, должны быть назначены с 31-й стороны. Эта схема обеспечивает максимальную гибкость при назначении битовых позиций общим расширениям, в отличие от расширений, зависящих от поставщика. Предполагается, что поставщик определяет новые значения, которые являются естественными расширениями битовых флагов, определенных API.

Расширяемые структуры данных имеют поле размера вариабли, зарезервированное для использования в конкретных устройствах. Так как поле имеет размер вариабли, поставщик услуг определяет объем информации и интерпретации поля. Поставщик, определяющий поле для конкретного устройства, должен сделать эти естественные расширения исходной структуры данных, определенной API.
