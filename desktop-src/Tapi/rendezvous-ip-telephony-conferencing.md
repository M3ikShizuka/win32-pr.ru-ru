# <a name="rendezvous-ip-telephony-conferencing"></a>Встречные конференции с IP-телефонией

\[ В Windows Vista, Windows Server 2008 и последующих версиях операционной системы нельзя использовать встречные средства и элементы управления Конференц-телефонией и интерфейсы. API клиента RTC предоставляет аналогичные функциональные возможности.\]

Встречные элементы управления TAPI 3 позволяют программисту создавать приложения, которые могут создавать и обнаруживать многоадресные IP-конференции мультимедиа.

Ключевые компоненты встреч:

[Элементы управления каталога](directory-controls.md) представляют собой абстракцию списков конференций на сервере ILS или NTDS.

[Элементы управления "хранилище больших двоичных объектов](conference-blob-controls.md) " представляют сведения, относящиеся к Конференции, такие как время начала и окончания. Для протоколов. BLOB-объекты протокола SDP предоставляются специальными интерфейсами. Подробные сведения см. в RFC 2327 под названием «SDP: Протокол описания сеанса». Текущую копию этого документа RFC можно найти с помощью поисковой системы в Интернете.

[Многоадресные COM-интерфейсы](multicast-com-interfaces.md) — это оболочки COM для функций MADCAP, которые ранее известны как мдхкп. Эти интерфейсы позволяют приложению получать адреса многоадресной рассылки с сервера распределения адресов многоадресной рассылки.

Следующий материал содержит общие сведения и примеры использования встречных элементов управления для IP-телефонии и конференций.
