# <a name="linebearermode_-constants"></a>\_Константы линебеарермоде

Константы побитового флага **линебеарермоде \_** описывают различные режимы носителя вызова. Когда приложение выполняет вызов, оно может запросить конкретный режим носителя. Эти режимы используются для выбора определенного качества обслуживания для запрошенного подключения из базовой телефонной сети. Режимы носителя, доступные в данной строке, являются возможностью устройства в строке.

<dl> <dt>

<span id="LINEBEARERMODE_ALTSPEECHDATA"></span><span id="linebearermode_altspeechdata"></span>**ЛИНЕБЕАРЕРМОДЕ \_ алтспичдата**
</dt> <dd> <dl> <dt>



Альтернативная перенаправление речи или неограниченных данных в одном вызове (ISDN).


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_DATA"></span><span id="linebearermode_data"></span>**\_данные линебеарермоде**
</dt> <dd> <dl> <dt>



Неограниченное перемещение данных при вызове. Скорость передачи данных указывается отдельно.


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_MULTIUSE"></span><span id="linebearermode_multiuse"></span>**ЛИНЕБЕАРЕРМОДЕ \_ многократное**
</dt> <dd> <dl> <dt>



Режим многократное, определенный ISDN.


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_NONCALLSIGNALING"></span><span id="linebearermode_noncallsignaling"></span>**ЛИНЕБЕАРЕРМОДЕ \_ нонкаллсигналинг**
</dt> <dd> <dl> <dt>



Это соответствует сигнальному соединению, не связанному с вызовом, от приложения к поставщику услуг или коммутатору (который рассматривается TAPI как поток мультимедиа).


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_PASSTHROUGH"></span><span id="linebearermode_passthrough"></span>**\_транзитный линебеарермоде**
</dt> <dd> <dl> <dt>



При активации вызова в ЛИНЕБЕАРЕРМОДЕ \_ passthrough поставщик услуг предоставляет прямой доступ к подключенному оборудованию для управления приложением. Этот режим используется главным образом приложениями, которым необходимо временное прямое управление асинхронными модемами, доступ к которым осуществляется через [функции связи](/windows/desktop/DevIO/communications-functions), для настройки или использования специальных функций, которые не поддерживаются поставщиком услуг.


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_RESTRICTEDDATA"></span><span id="linebearermode_restricteddata"></span>**ЛИНЕБЕАРЕРМОДЕ \_ рестриктеддата**
</dt> <dd> <dl> <dt>



Служба носителя для цифровых данных, в которой только семь младших бит каждого октета могут содержать данные пользователя (например, для службы Switchd 56kbit/s).


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_SPEECH"></span><span id="linebearermode_speech"></span>**ЛИНЕБЕАРЕРМОДЕ \_ речь**
</dt> <dd> <dl> <dt>



Это соответствует передаче речи G. 711 в вызове. Сеть может использовать такие методы обработки, как аналоговая передача, отмена эха, сжатие и распаковка. Битовая целостность не гарантируется. Речь не поддерживает типы мультимедиа для факсов и модемов.


</dt> </dl> </dd> <dt>

<span id="LINEBEARERMODE_VOICE"></span><span id="linebearermode_voice"></span>**ЛИНЕБЕАРЕРМОДЕ \_ голоса**
</dt> <dd> <dl> <dt>



Это обычная служба носителя с аналоговым голосовым стандартом 3,1 кГц. Битовая целостность не гарантируется. Голосовое сообщение может поддерживать типы мультимедиа факсов и модемов.


</dt> </dl> </dd> </dl>

## <a name="remarks"></a>Remarks

Старшие 16 бит можно назначить для расширений для конкретных устройств. 16 младших разрядов зарезервированы.

Обратите внимание, что режим носителя и тип мультимедиа являются разными понятиями. Режим носителя вызова указывает на качество телефонного подключения, которое предоставляется в основном в сети. Тип носителя вызова указывает на тип информационного потока, который передается через этот вызов. Факсы группы 3 или модемы данных — это типы мультимедиа, использующие вызов с режимом носителя 3,1 кГц.
