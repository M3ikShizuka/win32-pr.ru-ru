# <a name="event-notification"></a>Уведомление о событии

Уведомление о событии — это основное средство, с помощью которого приложение получает информацию от TAPI и поставщиков услуг. Эти сведения могут быть состоянием асинхронной операции, вызывается приложением, или могут быть связаны с процессом, запущенным за пределами приложения, например уведомлениями о новых входящих вызовах.

**Интерфейс TAPI 2. x:** Приложения обрабатывали уведомления одним из трех способов: скрытого окна, обработчика события или порта завершения. Дополнительные сведения об этих механизмах уведомления см. в разделе "Примечания" для [**линеинитиализикс**](/windows/win32/api/tapi/nf-tapi-lineinitializeexa). Приложение указывает механизм, устанавливая элемент **двоптионс** структуры [**линеинитиализикспарамс**](/windows/win32/api/tapi/ns-tapi-lineinitializeexparams) перед вызовом [**линеинитиализикс**](/windows/win32/api/tapi/nf-tapi-lineinitializeexa).

Функция [**линесетстатусмессажес**](/windows/win32/api/tapi/nf-tapi-linesetstatusmessages) позволяет приложению указать, какие сообщения уведомления должны быть получены для событий, связанных с изменениями состояния для указанной строки или любого из ее адресов.

**Интерфейс TAPI 3. x:** Приложения обработают общее уведомление с помощью [подключаемых объектов](../com/events-in-com-and-connectable-objects.md)com уровня "Стандартный". [**Иттапиевентнотификатион**](/windows/desktop/api/Tapi3if/nn-tapi3if-ittapieventnotification) — это исходящий интерфейс, который должен быть зарегистрирован в ОБЪЕКТЕ контейнера TAPI, а [**Иттапиевентнотификатион:: Event**](/windows/desktop/api/Tapi3if/nf-tapi3if-ittapieventnotification-event) — это метод вызовов TAPI для определения ответа приложения. Метод [**иттапи::p UT \_ EVENTFILTER**](/windows/desktop/api/tapi3if/nf-tapi3if-ittapi-put_eventfilter) указывает TAPI, какие события интересны приложению. Если фильтр событий не указан, приложение не будет принимать уведомления о каких-либо событиях. Метод [**иттапи:: регистеркаллнотификатионс**](/windows/desktop/api/tapi3if/nf-tapi3if-ittapi-registercallnotifications) сообщает TAPI о типах носителей и адресах, для которых приложение будет обрабатывать входящие сеансы. Дополнительные сведения об обработке событий в TAPI 3 см. в разделе Общие сведения о [событиях](events.md) или пример кода [регистрации событий](register-events.md) .

Поставщики услуг телефонии реализуют [**тспи \_ Линесетдефаултмедиадетектион**](/windows/win32/api/tspi/nf-tspi-tspi_linesetdefaultmediadetection) и [**тспи \_ линесетстатусмессажес**](/windows/win32/api/tspi/nf-tspi-tspi_linesetstatusmessages). TAPI вызывает эти функции для указания набора всех событий строк, адресов и типов мультимедиа, запрошенных приложениями.
