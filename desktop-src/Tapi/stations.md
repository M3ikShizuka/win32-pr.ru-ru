# <a name="stations"></a>Станций

Наборы станций, отслеживаемые с помощью сторонней связи, моделируются как линейное устройство и, возможно, связанное телефонное устройство. Линейное устройство может иметь несколько адресов, если модель терминала поддерживает несколько номеров каталогов (DN). Несколько представлений вызовов одного DN можно моделировать как один адрес, поддерживающий несколько вызовов.

Вызовы между двумя станциями в коммутаторе имеют два дескриптора вызова, один из которых предоставляет представление вызовов от первой станции (на устройстве линии), а другой — для представления вызовов со второй станции (на устройстве линии). Например, [**линемакекалл**](/windows/desktop/api/Tapi/nf-tapi-linemakecall) стороннего производителя, размещенного приложением на сервере, перенаправляется на линейное устройство, связанное с станцией, из которой вызывается вызов. в этой строке будет создан маркер вызова по адресу, указанному в [**линекаллпарамс**](/windows/desktop/api/Tapi/ns-tapi-linecallparams) (таким образом, он обеспечивает контроль над тем, какое различающееся имя используется на телефоне, поддерживающем несколько DNs). Когда вызов предлагается конечному адресу, создается новый обработчик вызова, отображающий вызов в состоянии *предложения* . приложения будут иметь представление о том, что он был еще одним вызовом элемента **двкаллид** в [**линекаллинфо**](/windows/desktop/api/Tapi/ns-tapi-linecallinfo) , равным обоим вызовам. Оба вызова приведут к *простою* , когда вызов был удален; вызов можно удалить из стороннего приложения, выполнив [**линедроп**](/windows/desktop/api/Tapi/nf-tapi-linedrop) на любом из дескрипторов вызова.
