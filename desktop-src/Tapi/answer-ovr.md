# <a name="answer"></a>Ответ

Операция ответа представляет собой типичный ответ приложения на предлагаемый сеанс связи. В случае успеха эта операция приведет к переходу сеанса в состояние подключения.

В некоторых средах телефонии (например, ISDN), где оповещения пользователя отделены от предложения о вызовах, приложение имеет возможность [принять](accept-ovr.md) вызов перед ответом или отклонить ([Удалить](drop-ovr.md)) или [перенаправить](redirect-ovr.md) вызов *предложения* .

Если операция ответа выполняется для нового сеанса, когда сеанс уже активен, то воздействие этого на существующий активный сеанс зависит от возможностей поставщика услуг. Первый вызов может быть нетронут, он может быть автоматически удален или автоматически помещен в удержание. TAPI будет отправлять соответствующие уведомления о событиях, касающиеся обоих вызовов.

В ситуации с мостом, если вызов подключен, но находится в активном состоянии, его можно объединить с помощью операции ответа.

Приложение может отправить сведения о пользователе во время ответа, если поставщик услуг его поддерживает.

**Интерфейс TAPI 2. x:** См. [**линеансвер**](/windows/win32/api/tapi/nf-tapi-lineanswer).

**Интерфейс TAPI 3. x:** См. раздел [**итбасиккаллконтрол:: Answer**](/windows/desktop/api/tapi3if/nf-tapi3if-itbasiccallcontrol-answer).
