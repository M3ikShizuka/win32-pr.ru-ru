# <a name="general-design-goals"></a>Общие цели проектирования

Следующий список содержит цели проектирования для TAPI MSP.

-   Базовые классы были сохранены простыми, и члены и методы появились только при крайней необходимости.
-   Простое наследование. Нет множественного наследования между классами, хотя для интерфейсов используется несколько наследований.
-   Блокировка происходит только в одном направлении, чтобы предотвратить взаимоблокировку. Методы для объекта вызова, требующего блокировки вызова, могут вызывать методы в потоке, для которого требуется блокировка потока. Однако методы в потоке, требующем блокировки потока, никогда не будут вызывать метод для вызова, для которого требуется блокировка вызова.
-   Рефкаунтс используются для защиты доступа. Все обратные вызовы, отправленные в пул потоков, содержат рефкаунтс. Значение refcount отменяется при отмене ожидания. Объект Address имеет рефкаунтс на терминалах. объекты Call имеют рефкаунтс по адресу и на Потоки. Объекты потока имеют рефкаунтс на вызовы и терминалы. у терминалов есть рефкаунтс на Потоки. Циклическая рефкаунтс прерывается при вызове метода Shutdown объектов Address и Call.
