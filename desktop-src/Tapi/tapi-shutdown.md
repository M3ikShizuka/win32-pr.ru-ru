# <a name="tapi-shutdown"></a>Завершение работы TAPI

Правильное завершение сеансов связи и всего приложения требуется для предотвращения того, что ресурсы остаются связанными в вызовах или приложениях, которые больше не нужны.

TAPI предоставляет ряд функций и методов, которые помогут в процессе. Очевидно, что приложение должно также принять ответственность за правильное освобождение выделенной памяти, будь то структура данных или ссылки COM.



| Функции TAPI 2. x                                                            | Описание                                                            |
|-------------------------------------------------------------------------------|------------------------------------------------------------------------|
| [**линерегистеррекуестреЦипиент**](/windows/win32/api/tapi/nf-tapi-lineregisterrequestrecipient) | Отменяет регистрацию приложения в качестве обработчика для вызовов телефонной связи. |
| [**линеклосе**](/windows/win32/api/tapi/nf-tapi-lineclose)                                       | Отключает приложение от имени руководителя для вызовов в данной строке.  |
| [**линешутдовн**](/windows/win32/api/tapi/nf-tapi-lineshutdown)                                 | Завершает использование абстракции строки приложением.            |



 



| Интерфейсы или методы интерфейса TAPI 3. x                                            | Описание                                                                |
|---------------------------------------------------------------------------|----------------------------------------------------------------------------|
| [**ИТТАПИ:: Унрегистернотификатионс**](/windows/desktop/api/tapi3if/nf-tapi3if-ittapi-unregisternotifications) | Удаляет все регистрации уведомлений о событиях, выполняемые приложением. |
| [**ИТТАПИ:: Shutdown**](/windows/desktop/api/tapi3if/nf-tapi3if-ittapi-shutdown)                               | Отключает приложение как Диспетчер вызовов.                             |
