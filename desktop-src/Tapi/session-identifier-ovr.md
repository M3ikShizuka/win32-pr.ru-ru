# <a name="session-identifier"></a>Идентификатор сеанса

TAPI назначает идентификаторы сеансов, уникальные для каждого сеанса и приложения. Приложение использует идентификатор сеанса для взаимодействия с TAPI в отношении определенного сеанса. Приложение обычно получает идентификатор, создавая новый сеанс связи или когда TAPI предлагает сеанс.

Идентификатор сеанса нельзя использовать для передачи информации другому приложению. Для этой цели следует использовать [идентификатор вызова](call-id-ovr.md), который может быть назначен поставщиком услуг.

Сведения об операциях, которые создают сеансы и возвращают идентификатор сеанса, см. в разделе [Инициация сеанса](initiate-a-session-ovr.md) . См. раздел [ответ на сеанс, инициированный в других](respond-to-session-initiated-elsewhere-ovr.md) случаях для операций, которые получают идентификаторы сеансов из TAPI. Сведения о освобождении ресурсов сеанса см. в разделе [Завершение сеанса](terminate-a-session-ovr.md) .

Все поставщики служб должны поддерживать некоторую форму идентификации сеанса.

**Интерфейс TAPI 2. x:** Основным идентификатором сеанса связи является обработчик вызова.

**Интерфейс TAPI 3. x:** Сеанс представлен [объектом Call](call-object.md) , а первичный идентификатор — указателем на интерфейс [**иткаллинфо**](/windows/desktop/api/tapi3if/nn-tapi3if-itcallinfo) .
