# <a name="line_monitormedia-message"></a>Строка \_ сообщения монитормедиа

Сообщение TAPI **Line \_ монитормедиа** отправляется при обнаружении изменения типа носителя вызова. Отправка этого сообщения контролируется с помощью функции [**линемонитормедиа**](/windows/desktop/api/Tapi/nf-tapi-linemonitormedia) .


## <a name="return-value"></a>Возвращаемое значение

Нет возвращаемого значения.

## <a name="remarks"></a>Remarks

Сообщение **Line \_ монитормедиа** отправляется в приложение с включенным мониторингом мультимедиа для обнаруженного типа носителя.

Так как эта метка времени могла быть создана на компьютере, отличном от того, на котором выполняется приложение, оно полезно только для сравнения с другими подобными сообщениями с метками времени, созданными на том же устройстве ( [**строка \_ гасердигитс**](line-gatherdigits.md), [**строка \_ Generate**](line-generate.md), [**строка \_ монитордигитс**](line-monitordigits.md), [**\_ монитортоне**](line-monitortone.md)), чтобы определить их относительную временную шкалу (разделение между событиями). После приблизительно 49,7 дней счетчик тактов может быть заключен в оболочку. приложения должны учитывать это при выполнении вычислений.

Если поставщик услуг не создает отметку времени (например, если она была создана с помощью более ранней версии TAPI), TAPI предоставляет метку времени в ближайшем к поставщику услуге, который создает событие, чтобы синтезированная метка времени максимально точна.
