# <a name="unowned-calls"></a>Ненадлежащие вызовы

Чтобы предотвратить ненадлежащие вызовы в системе телефонии, TAPI удаляет вызовы, которые получили сигнал от поставщика услуг, если не удается опознать приложение для первоначального владельца вызова. В TAPI версии 2,0 и более поздних интерфейс TAPI вызывает функцию [**тспи \_ линедроп**](/windows/win32/api/tspi/nf-tspi-tspi_linedrop) для удаления вызова.

Поставщик услуг может заранее определить, доступно ли приложение, чтобы стать владельцем нового вызова. TAPI всегда информирует поставщика служб о типах носителей, для которых доступно приложение с помощью функции [**\_ линесетдефаултмедиадетектион тспи**](/windows/win32/api/tspi/nf-tspi-tspi_linesetdefaultmediadetection) .

Например, если поставщик услуг определяет, что имеется активный вызов в строке, имеющей \_ режим линемедиамоде интерактивевоице, он должен проверить обнаружение носителей по умолчанию перед созданием [**строк \_ невкалл**](line-newcall.md) и [**Line \_ каллстате**](/previous-versions/windows/desktop/legacy/ms725219(v=vs.85)) messages. Если обнаружение носителей по умолчанию показывает, что ни одно приложение не ищет \_ вызов ИНТЕРАКТИВЕВОИЦЕ линемедиамоде, поставщик услуг не должен отправить сообщения, так как TAPI немедленно отправит его.
