# <a name="imulticastcontrol-interface"></a>Интерфейс Имултикастконтрол

\[**Имултикастконтрол** недоступен для использования в Windows Vista, windows Server 2008 и последующих версиях операционной системы. API клиента RTC предоставляет аналогичные функциональные возможности.\]

Интерфейс **имултикастконтрол** реализуется ипконф MSP и доступен только для объектов многоадресного вызова. Этот интерфейс предоставляет методы, обеспечивающие создание и обработку терминалов, которые могут взаимодействовать между клиентами H323 и SDP. Интерфейс **имултикастконтрол** управляет режимом многоадресного замыкания на себя.

В режиме MM \_ без \_ замыкания на себя многоадресная рассылка отключена, что означает, что два приложения на одном компьютере, которые присоединяются к одной группе многоадресной рассылки, получат пакеты друг друга. Этот режим имеет лучшую производительность, если на компьютере всегда только одно приложение, соединяющее группу многоадресной рассылки. \_ \_ В качестве режима по умолчанию используется mm без замыкания на себя.

\_Режим полного замыкания на mm подходит \_ только для тестирования. Все отправленные пакеты также будут циклически возвращаться. Это можно использовать для проверки работоспособности устройств.

\_Режим селективного \_ замыкания на себя mm используется для включения нескольких приложений на одном компьютере для приподключения к одной группе многоадресной рассылки. Пакеты передаются обратно из стека, и каждый сеанс RTP отвечает за фильтрацию ненужных пакетов.
