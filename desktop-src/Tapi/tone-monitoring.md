# <a name="tone-monitoring"></a>Мониторинг тона

Мониторинг тонового режима отслеживает поток мультимедиа при вызове указанных тонов. Гудок описывается частотой и ритмичностью его компонентов. Реализация API может обеспечить одновременное отслеживание нескольких различных тонов. Приложение может отметить каждый тон, чтобы отличать различные звуки, для которых он запрашивает обнаружение.

Приложение может включить или отключить мониторинг тонового объекта при указанном вызове с помощью [**линемонитортонес**](/windows/desktop/api/Tapi/nf-tapi-linemonitortones). С помощью этой функции приложение указывает, какие звуки должны обнаруживаться при указанном вызове. Если включен мониторинг тона, обнаруженные цифры приводят к тому, что приложение будет уведомлено с помощью [**строки \_ монитортоне**](line-monitortone.md) Message. Это сообщение предоставляет маркер вызова, на котором был обнаружен тон, а также тег приложения для тона.

Область отслеживания тона связана с временем существования вызова. Отслеживание тона в вызове завершается, как только вызов *отключается* или переходит в *состояние простоя*.

> [!Note]  
> Мониторинг тонов, цифр или типов мультимедиа часто требует использования ресурсов, для которых поставщик услуг может иметь только ограниченный объем. Запрос на наблюдение может быть отклонен, если ресурсы недоступны. По той же причине приложение должно отключить все ненужные средства мониторинга.
