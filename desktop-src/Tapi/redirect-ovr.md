# <a name="redirect"></a>перенаправление

Перенаправление сеанса позволяет приложению отклонения от предлагаемого сеанса к другому адресу без ответа на вызов. Приложение может выполнять перенаправление на основе вызова по требованию. Например, приложение может разрешить пользователю указывать различные перенаправления на основе сведений об ИДЕНТИФИКАТОРе звонящего. После успешного перенаправления вызова состояние обычно переходит в режим бездействия и связанные ресурсы должны быть освобождены.

Перенаправление отличается от переадресации. После настройки [пересылки операция](forward-ovr.md) выполняется TAPI, поставщиком услуг или коммутатором без дальнейшего участия в приложении. Перенаправление отличается от [перемещения](transfer-ovr.md) в этом перенаправлении не требует первого ответа на вызов.

Не все поставщики служб поддерживают использование этой операции.

**Интерфейс TAPI 2. x:** См. [**линередирект**](/windows/win32/api/tapi/nf-tapi-lineredirect).
