# <a name="wfddisplaysinkstart-function"></a>Функция Вфддисплайсинкстарт

Выполняет необходимую инициализацию, чтобы вызывающее приложение стало приемником дисплея Miracast. Приложение должно вызывать его один раз при запуске.

## <a name="syntax"></a>Синтаксис


```C++
DWORD WINAPI WFDStartDisplaySink(
  _In_     USHORT                                 uDeviceCategory,
  _In_     USHORT                                 uSubCategory,
  _In_opt_ PVOID                                  pCallbackContext,
  _In_     WFD_DISPLAY_SINK_NOTIFICATION_CALLBACK *pfnCallback
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*удевицекатегори* \[ окне\]
</dt> <dd>

Категория устройства.

</dd> <dt>

*усубкатегори* \[ окне\]
</dt> <dd>

Подкатегория устройства.

</dd> <dt>

*пкаллбаккконтекст* \[ в необязательное\]
</dt> <dd>

Необязательный указатель контекста, который передается функции обратного вызова, указанной в параметре *пфнкаллбакк* .

</dd> <dt>

*пфнкаллбакк* \[ окне\]
</dt> <dd>

Указатель на функцию обратного вызова, вызываемую при наличии уведомления для приложения. Уведомления, которые могут быть отправлены, описаны в [**\_ \_ \_ \_ обратном вызове уведомлений о приемнике для монитора WFD**](wfd-display-sink-notification-callback.md).

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция завершается успешно, возвращается значение ошибки \_ Success.

Если функция завершается ошибкой, возвращаемое значение может быть одним из следующих кодов возврата.



| Код возврата                                                                                          | Описание                                                    |
|------------------------------------------------------------------------------------------------------|----------------------------------------------------------------|
| <dl> <dt>**Ошибка \_ не \_ поддерживается**</dt> </dl> | Приемник дисплея не поддерживается на этом оборудовании.<br/> |



 

## <a name="remarks"></a>Комментарии

Эта функция выполняет следующие задачи:

-   Делает компьютер обнаруживаемым
-   Задает сведения о ОДНОРАНГОВОм устройстве, отражающие указанную категорию и подкатегорию
-   Устанавливает наборы устройств Miracast для всех Wi-Fi прямых кадров с типом устройства в качестве приемника.
-   Регистрирует указанный обратный вызов для вызова при наличии входящего подключения.
