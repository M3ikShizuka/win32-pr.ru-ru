# <a name="dot11_auth_algorithm-enumeration"></a>\_Перечисление алгоритма проверки подлинности DOT11 \_

Тип **перечисления \_ \_ алгоритма авторизации DOT11** определяет алгоритм проверки подлинности беспроводной локальной сети.

## <a name="syntax"></a>Синтаксис


```C++
typedef enum _DOT11_AUTH_ALGORITHM { 
  DOT11_AUTH_ALGO_80211_OPEN        = 1,
  DOT11_AUTH_ALGO_80211_SHARED_KEY  = 2,
  DOT11_AUTH_ALGO_WPA               = 3,
  DOT11_AUTH_ALGO_WPA_PSK           = 4,
  DOT11_AUTH_ALGO_WPA_NONE          = 5,
  DOT11_AUTH_ALGO_RSNA              = 6,
  DOT11_AUTH_ALGO_RSNA_PSK          = 7,
  DOT11_AUTH_ALGO_IHV_START         = 0x80000000,
  DOT11_AUTH_ALGO_IHV_END           = 0xffffffff
} DOT11_AUTH_ALGORITHM, *PDOT11_AUTH_ALGORITHM;
```



## <a name="constants"></a>Константы

<dl> <dt>

<span id="DOT11_AUTH_ALGO_80211_OPEN"></span><span id="dot11_auth_algo_80211_open"></span>**\_Открытая проверка подлинности DOT11 \_ Algo \_ 80211 \_**
</dt> <dd>

Указывает алгоритм проверки подлинности с открытым системным стандартом IEEE 802,11.

</dd> <dt>

<span id="DOT11_AUTH_ALGO_80211_SHARED_KEY"></span><span id="dot11_auth_algo_80211_shared_key"></span>**\_Общий ключ проверки подлинности DOT11 \_ Algo \_ 80211 \_ \_**
</dt> <dd>

Указывает алгоритм проверки подлинности с общим ключом 802,11, который требует использования ключа WEP для проверки подлинности 802,11.

</dd> <dt>

<span id="DOT11_AUTH_ALGO_WPA"></span><span id="dot11_auth_algo_wpa"></span>**\_Проверка подлинности DOT11 \_ Algo \_ WPA**
</dt> <dd>

Указывает алгоритм защиты Wi-Fi защищенного доступа (WPA). Проверка подлинности по порту IEEE 802.1 X выполняется с помощью запрашивающего устройства, средства проверки подлинности и сервера аутентификации. Ключи шифра динамически получаются через процесс проверки подлинности.

Этот алгоритм действителен только для типов BSS \_ инфраструктуры типов Dot11 BSS \_ \_ .

При включении алгоритма WPA станция 802,11 связывается только с точкой доступа, для которой в элементе данных WPA (IE) содержится набор проверки подлинности с типом 1 (802.1 X).

</dd> <dt>

<span id="DOT11_AUTH_ALGO_WPA_PSK"></span><span id="dot11_auth_algo_wpa_psk"></span>**\_Проверка подлинности DOT11 \_ Algo \_ WPA \_ PSK**
</dt> <dd>

Указывает алгоритм WPA, использующий общие ключи (PSK). Проверка подлинности по порту IEEE 802.1 X выполняется с помощью запрашивающего устройства и средства проверки подлинности. Ключи шифра динамически извлекаются с помощью предварительно общего ключа, который используется как для запрашивающего, так и для средства проверки подлинности.

Этот алгоритм действителен только для типов BSS **\_ \_ \_ инфраструктуры типов Dot11 BSS**.

Когда алгоритм WPA PSK включен, станция 802,11 связывается только с точкой доступа, в которой ответы на маяки или зонды содержат набор проверки подлинности типа 2 (предварительный ключ) в WPA IE.

</dd> <dt>

<span id="DOT11_AUTH_ALGO_WPA_NONE"></span><span id="dot11_auth_algo_wpa_none"></span>**\_ALGOная проверка подлинности DOT11 \_ \_ WPA \_ None**
</dt> <dd>

Это значение не поддерживается.

</dd> <dt>

<span id="DOT11_AUTH_ALGO_RSNA"></span><span id="dot11_auth_algo_rsna"></span>**\_ALGOская проверка подлинности DOT11 \_ \_ рсна**
</dt> <dd>

Указывает алгоритм беспроводного сопоставления сети (РСНА) 802.11 с ненадежным доступом. WPA2 — это один из таких алгоритмов. Проверка подлинности по порту IEEE 802.1 X выполняется с помощью запрашивающего устройства, средства проверки подлинности и сервера аутентификации. Ключи шифра динамически получаются через процесс проверки подлинности.

Этот алгоритм действителен только для типов BSS **\_ \_ \_ инфраструктуры типов Dot11 BSS**.

Если включен алгоритм РСНА, то Станция 802,11 связывается только с точкой доступа, в которой ответы на сигналы маяка или зонды содержат набор проверки подлинности типа 1 (802.1 X) в среде RSN IE.

</dd> <dt>

<span id="DOT11_AUTH_ALGO_RSNA_PSK"></span><span id="dot11_auth_algo_rsna_psk"></span>**DOT11 \_ AUTH \_ Algo \_ рсна \_ PSK**
</dt> <dd>

Указывает алгоритм 802.11 i РСНА, использующий PSK. Проверка подлинности по порту IEEE 802.1 X выполняется с помощью запрашивающего устройства и средства проверки подлинности. Ключи шифра динамически извлекаются с помощью предварительно общего ключа, который используется как для запрашивающего, так и для средства проверки подлинности.

Этот алгоритм действителен только для типов BSS **\_ \_ \_ инфраструктуры типов Dot11 BSS**.

Если включен алгоритм РСНА PSK, то Станция 802,11 связывается только с точкой доступа, в которой ответы на сигналы маяка или зонды содержат набор проверки подлинности типа 2 (предварительный ключ) в обозревателе RSN IE.

</dd> <dt>

<span id="DOT11_AUTH_ALGO_IHV_START"></span><span id="dot11_auth_algo_ihv_start"></span>**\_Начало проверки подлинности DOT11 \_ Algo \_ \_**
</dt> <dd>

Указывает начало диапазона, который указывает собственные алгоритмы проверки подлинности, разработанные IHV.

Перечислитель **\_ \_ \_ \_ запуска Algo для проверки** на основе DOT11 действителен, только если драйвер минипорта работает в режиме расширяемой станции (екстста).

</dd> <dt>

<span id="DOT11_AUTH_ALGO_IHV_END"></span><span id="dot11_auth_algo_ihv_end"></span>**\_Завершение проверки подлинности DOT11 \_ Algo \_ IHV \_**
</dt> <dd>

Указывает конец диапазона, который указывает собственные алгоритмы проверки подлинности, разработанные IHV.

Перечислитель **\_ \_ \_ \_ завершения проверки подлинности DOT11 Algo** действителен только в том случае, если драйвер минипорта работает в режиме екстста.

</dd> </dl>
