# <a name="ibackgroundcopyjobsetnotifyinterface-method"></a>Метод использованием метода ibackgroundcopyjob:: Сетнотифинтерфаце

Определяет реализацию интерфейса [**ибаккграундкопикаллбакк**](ibackgroundcopycallback.md) для оптимизации доставки. Используйте интерфейс **ибаккграундкопикаллбакк** для получения уведомлений о событиях, связанных с заданием.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT SetNotifyInterface(
   IUnknown *pNotifyInterface
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пнотифинтерфаце* 
</dt> <dd>

Указатель интерфейса [**ибаккграундкопикаллбакк**](ibackgroundcopycallback.md) . Чтобы удалить текущий указатель интерфейса обратного вызова, присвойте этому параметру **значение NULL**.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод возвращает следующие значения **HRESULT** , а также другие.



| Код возврата                                                                              | Описание                                                     |
|------------------------------------------------------------------------------------------|-----------------------------------------------------------------|
| <dl> <dt>S_OK * * * *</dt> </dl> | Указатель интерфейса уведомления успешно установлен.<br/> |



 

## <a name="remarks"></a>Комментарии

Этот метод следует вызывать только при реализации интерфейса [**ибаккграундкопикаллбакк**](ibackgroundcopycallback.md) . Используйте метод **сетнотифинтерфаце** в сочетании с методом [**сетнотифифлагс**](ibackgroundcopyjob-setnotifyflags.md) , чтобы указать тип уведомления, которое требуется получить.

Интерфейс уведомления станет недействительным после завершения работы приложения. Оптимизация доставки не сохраняет интерфейс уведомления. В результате процесс инициализации приложения должен вызвать метод **сетнотифинтерфаце** для тех существующих заданий, для которых вы хотите получить уведомление. Если необходимо собрать сведения о состоянии и ходе выполнения, произошедшие с момента последнего запуска приложения, следует опросить сведения о состоянии и ходе выполнения во время инициализации приложения.

Только владелец, создатель или администратор задания могут зарегистрироваться для получения уведомлений.
