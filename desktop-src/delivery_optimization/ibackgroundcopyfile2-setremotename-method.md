# <a name="ibackgroundcopyfile2setremotename-method"></a>Метод IBackgroundCopyFile2:: Сетремотенаме

Изменяет удаленное имя на новый URL-адрес в задании загрузки.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT SetRemoteName(
  [in] LPCWSTR RemoteName
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Ремотенаме* \[ окне\]
</dt> <dd>

Строка, заканчивающаяся нулем и содержащая имя файла на сервере. Сведения об указании удаленного имени см. в разделе элемент **ремотенаме** .

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод возвращает следующие возвращаемые значения, а также другие.



| Код возврата                                                                                  | Описание                                                                                                           |
|----------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>S_OK * * * *</dt> </dl>     | Успешно<br/>                                                                                                    |
| <dl> <dt>**E_INVALIDARG**</dt> </dl> | Новое удаленное имя является недопустимым URL-адресом, или новый URL-адрес слишком длинный (длина URL-адреса не должна превышать 2 200 символов).<br/> |



 

## <a name="remarks"></a>Комментарии

Как правило, этот метод вызывается, если требуется изменить URL-адрес, используемый для перемещения файла, или если необходимо изменить имя или путь к файлу.

Этот метод не выполняет сериализацию при возврате. Чтобы сериализовать изменение, [**приостановите**](ibackgroundcopyjob-suspend.md) задание, вызовите этот метод (если изменяется несколько файлов в задании, используйте цикл) и [**возобновите**](ibackgroundcopyjob-resume.md) задание. Вызов метода **использованием метода ibackgroundcopyjob:: Resume** сериализует изменение.

Если метка времени или размер файла нового удаленного имени отличаются от предыдущего удаленного имени, либо новый сервер не поддерживает возобновление контрольной точки (для удаленных имен HTTP), выполните загрузку повторно. В противном случае перемещение возобновляется с той же самой позицией на новом сервере. Не перезапускают уже передаваемые файлы.
