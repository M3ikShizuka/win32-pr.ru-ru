# <a name="eap-implementation-details"></a>Сведения о реализации EAP

Точки доступа (ТД), например служба удаленного доступа (RAS), взаимодействуют с реализациями EAP с помощью вызовов функций, которые должны быть экспортированы сторонней библиотекой DLL EAP. Иными словами, протокол EAP является API поставщика, то есть подключаемые модули или библиотеки DLL реализуют код для того, чтобы стать поставщиком EAP, но не должны вызывать сами интерфейсы API EAP. Например, программист библиотеки DLL EAP создает код для подпрограммы инициализации EAP и размещает этот код в предопределенной функции EAP [**расеапинитиализе**](/previous-versions/windows/desktop/legacy/aa363527(v=vs.85)). Затем во время выполнения диспетчер соединений AP вызывает функцию **расеапинитиализе** , которая содержит код, для инициализации реализации EAP.

-   [Инициализация протокола проверки подлинности](authentication-protocol-initialization.md)
