# <a name="eap-installation"></a>Установка EAP

Поставщики реализуют ЕАПС, также известные как протоколы проверки подлинности, в библиотеках динамической компоновки (DLL). Библиотека DLL для протокола проверки подлинности должна размещаться как на клиентской, так и на серверной компьютерах. Для простоты клиентские и серверные библиотеки DLL могут быть идентичными. Однако это не обязательно. Кроме того, имейте в виду, что одна и та же библиотека DLL может поддерживать несколько протоколов проверки подлинности.

Поставщик должен предоставить программное обеспечение для установки и удаления библиотеки DLL. Программа установки также должна создать соответствующие ключи и значения для протокола проверки подлинности в системном реестре и удалить их после удаления.

При установке каждой DLL-библиотеки EAP необходимо создать следующий раздел реестра.

**HKey \_ Локальный \_ компьютер** \\ **System** \\ **CurrentControlSet** \\ **Services** \\ **RASMAN** \\ **PPP** \\ **EAP** \\ **&lt; еаптипеид &gt;**

В предыдущем пути **&lt; еаптипеид &gt;** — это идентификатор протокола проверки подлинности. Поставщик должен получить этот идентификатор из центра "назначенные номера Интернета (IANA)".

Дополнительные сведения и описание поддерживаемых значений для этого ключа см. в разделе [значения реестра протокола проверки подлинности](authentication-protocol-registry-values.md).
