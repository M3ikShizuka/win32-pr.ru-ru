# <a name="client-side-configuration-user-interface"></a>Пользовательский интерфейс настройки Client-Side

Поставщик, реализующий протокол проверки подлинности, может также предоставить пользовательский интерфейс конфигурации для протокола. Пользовательский интерфейс конфигурации может быть реализован в той же библиотеке DLL, что и протокол проверки подлинности, или в отдельной библиотеке DLL. Кроме того, Библиотека DLL, реализующая пользовательский интерфейс конфигурации, может поддерживать несколько протоколов проверки подлинности. Путь к библиотеке DLL для пользовательского интерфейса конфигурации хранится в параметре реестра RAS \_ EAP \_ valueName \_ конфигуи в разделе ключа для протокола проверки подлинности. Дополнительные сведения о создании этого значения реестра см. в разделе [Установка EAP](eap-installation.md).

Библиотека DLL для пользовательского интерфейса конфигурации должна экспортировать точки входа для следующих функций:

[**расеапинвокеконфигуи**](/previous-versions/windows/desktop/api/Raseapif/nf-raseapif-raseapinvokeconfigui)

[**расеапфримемори**](/previous-versions/windows/desktop/api/Raseapif/nf-raseapif-raseapfreememory)

Когда пользователь создает запись конфигурации для конкретного подключения, будь то клиент RAS или беспроводная сеть, пользователь может выбрать протокол проверки подлинности, который служба должна использовать с этой записью. Если протокол проверки подлинности является настраиваемым, служба вызывает [**расеапинвокеконфигуи**](/previous-versions/windows/desktop/api/Raseapif/nf-raseapif-raseapinvokeconfigui) для вызова пользовательского интерфейса настройки. Пользовательский интерфейс конфигурации сохраняет сведения о конфигурации, возвращенные **расеапинвокеконфигуи** , в записи конфигурации.

Сведения о конфигурации должны быть общими для всех пользователей на клиентском компьютере. Сведения, относящиеся к конкретному пользователю или пользователям, не должны храниться в записи. Протокол проверки подлинности должен получать сведения, относящиеся к пользователю, с помощью [функций идентификации](obtaining-identity-information.md) или [интерактивного пользовательского интерфейса](interactive-user-interface.md). Протокол проверки подлинности может хранить эти сведения в реестре, передавая их в службу проверки подлинности в параметре *пеапаутпут* параметра [**расеапмакемессаже**](/previous-versions/windows/desktop/legacy/aa363532(v=vs.85)).

Сведения о конфигурации также не должны быть специфичными для текущего компьютера. Она должна быть переносимой с компьютера на компьютер.

Когда служба проверки подлинности вызывает функцию [**расеапбегин**](/previous-versions/windows/desktop/legacy/aa363520(v=vs.85)) для протокола проверки подлинности, она передает [**\_ \_ входную структуру PPP EAP**](/windows/desktop/api/Raseapif/ns-raseapif-ppp_eap_input) , содержащую указатель на сведения о конфигурации. После завершения вызова **расеапбегин** служба проверки подлинности вызывает [**расеапфримемори**](/previous-versions/windows/desktop/api/Raseapif/nf-raseapif-raseapfreememory) , чтобы освободить память, занятую сведениями о конфигурации. Поэтому протокол проверки подлинности должен скопировать сведения о конфигурации в буфер закрытой памяти во время вызова **расеапбегин**.

Поставщик может добавить значение в раздел реестра для протокола проверки подлинности, который указывает сведения о конфигурации по умолчанию для протокола. Поставщик может также добавить значение, указывающее, требуется ли пользователю вводить сведения о конфигурации при создании записи телефонной книги. Дополнительные сведения см. в разделе [значения реестра протокола проверки подлинности](authentication-protocol-registry-values.md).
