<span data-ttu-id="30265-104">Значения, формируемые системой, создаются этапом IA (на основе пользовательской [семантики](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-semantics)ввода) для обеспечения определенной эффективности операций шейдера.</span><span class="sxs-lookup"><span data-stu-id="30265-104">System-generated values are generated by the IA stage (based on user-supplied input [semantics](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-semantics)) to allow certain efficiencies in shader operations.</span></span>

<span data-ttu-id="30265-105">Присоединив данные, такие как идентификатор экземпляра (видимые для VS), идентификатор вершины (видимый для VS) или идентификатор примитива (видимый для GS/PS), последующий этап шейдера может искать эти системные значения для оптимизации обработки на этом этапе.</span><span class="sxs-lookup"><span data-stu-id="30265-105">By attaching data, such as an instance id (visible to VS), a vertex id (visible to VS), or a primitive id (visible to GS/PS), a subsequent shader stage may look for these system values to optimize processing in that stage.</span></span> <span data-ttu-id="30265-106">Например, этап VS может искать идентификатор экземпляра для получения дополнительных данных по вершинам шейдера или выполнения других операций. этапы GS и PS могут использовать примитивный идентификатор для получения данных одного примитива таким же образом.</span><span class="sxs-lookup"><span data-stu-id="30265-106">For instance, the VS stage may look for the instance id to grab additional per-vertex data for the shader or to perform other operations; the GS and PS stages may use the primitive id to grab per-primitive data in the same way.</span></span>

## <a name="primitiveid"></a><span data-ttu-id="30265-123">PrimitiveID</span><span class="sxs-lookup"><span data-stu-id="30265-123">PrimitiveID</span></span>

<span data-ttu-id="30265-124">Каждый этап шейдера использует примитивный идентификатор для идентификации каждого примитива.</span><span class="sxs-lookup"><span data-stu-id="30265-124">A primitive id is used by each shader stage to identify each primitive.</span></span> <span data-ttu-id="30265-125">Это 32-разрядное целое число без знака, значение по умолчанию — 0.</span><span class="sxs-lookup"><span data-stu-id="30265-125">It is a 32-bit unsigned integer whose default value is 0.</span></span> <span data-ttu-id="30265-126">Он назначается примитиву, когда примитив обрабатывается стадией IA.</span><span class="sxs-lookup"><span data-stu-id="30265-126">It is assigned to a primitive when the primitive is processed by the IA stage.</span></span> <span data-ttu-id="30265-127">Чтобы сообщить этапу IA о создании примитивного идентификатора, присоедините семантику примитивного идентификатора к объявлению ввода шейдера.</span><span class="sxs-lookup"><span data-stu-id="30265-127">To inform the IA stage to generate a primitive id, attach the primitive-id semantic to the shader input declaration.</span></span>

<span data-ttu-id="30265-128">На этапе IA будет добавлен примитивный идентификатор для каждого примитива для использования шейдером geometry или на этапе построителя текстуры (в зависимости от первого этапа, активного после этапа IA).</span><span class="sxs-lookup"><span data-stu-id="30265-128">The IA stage will add a primitive id to each primitive for use by the geometry shader or the pixel shader stage (whichever is the first stage active after the IA stage).</span></span> <span data-ttu-id="30265-129">Для каждого поиндексированного вызова Draw идентификатор примитива увеличивается на 1, однако при каждом запуске нового экземпляра примитивный идентификатор сбрасывается в 0.</span><span class="sxs-lookup"><span data-stu-id="30265-129">For each indexed draw call, the primitive id is incremented by 1, however, the primitive id resets to 0 whenever a new instance begins.</span></span> <span data-ttu-id="30265-130">Все остальные вызовы Draw не изменяют значение идентификатора экземпляра. Если идентификатор экземпляра переполняется (превышает 2 ³ ² – 1), он переносится в 0.</span><span class="sxs-lookup"><span data-stu-id="30265-130">All other draw calls do not change the value of the instance id. <span data-ttu-id="30265-131">На этапе шейдера пикселей не предусмотрены отдельные входные данные для примитивного идентификатора; Однако все входные шейдеры пикселей, в которых указан примитивный идентификатор, используют постоянный режим интерполяции.</span><span class="sxs-lookup"><span data-stu-id="30265-131">The pixel shader stage does not have a separate input for a primitive id; however, any pixel shader input that specifies a primitive id uses a constant interpolation mode.</span></span>
