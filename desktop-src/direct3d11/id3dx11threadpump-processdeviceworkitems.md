# <a name="id3dx11threadpumpprocessdeviceworkitems-method"></a>ID3DX11ThreadPump: метод:P Роцессдевицеворкитемс

> [!Note]  
> Библиотека служебной программы D3DX (D3DX 9, D3DX 10 и D3DX 11) является устаревшей для Windows 8 и не поддерживается для приложений Магазина Windows.

 

Задает рабочие элементы для устройства после завершения загрузки и обработки.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT ProcessDeviceWorkItems(
  [in] UINT iWorkItemCount
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*иворкитемкаунт* \[ окне\]
</dt> <dd>

Тип: **[ **uint**](/windows/desktop/WinProg/windows-data-types)**

Число рабочих элементов, которое будет задано для устройства.

</dd> </dl>

Тип: **[ **HRESULT**](https://msdn.microsoft.com/library/Bb401631(v=MSDN.10).aspx)**

Возвращаемое значение является одним из значений, перечисленных в [кодах возврата Direct3D 11](d3d11-graphics-reference-returnvalues.md).

## <a name="remarks"></a>Примечания

Когда конвейер потока закончит загрузку и обработку ресурса или шейдера, он будет хранить его в очереди до вызова этого API, после чего обработанные элементы будут установлены на устройство. Это полезно для управления объемом обработки, который тратится на привязку ресурсов к устройству для каждого кадра.

В качестве примера того, как один из них может использовать этот API, скажем, вы приближаетесь к концу одного уровня в игре и хотите начать предварительную загрузку текстур, шейдеров и других ресурсов для следующего уровня. Конвейер потока начнет загрузку, распаковку и обработку ресурсов и шейдеров в отдельном потоке, пока они не будут готовы к установке устройству, и в этом случае они покидают их в очереди. Один из них может не захотеть одновременно устанавливать все ресурсы и шейдеры, так как это может привести к заметному временному снижению производительности игры. Таким образом, этот API может вызываться один раз для каждого кадра, чтобы в каждом кадре было установлено только небольшое число рабочих элементов, таким образом распределяя рабочую нагрузку привязок ресурсов на устройство в нескольких кадрах.
