# <a name="threading-differences-between-direct3d-versions"></a>Различия в потоках между версиями Direct3D

Многие многопоточные модели программирования используют примитивы синхронизации (такие как мьютексы) для создания критических разделов и предотвращения доступа к коду более чем одним потоком за раз.

Однако методы создания ресурсов в интерфейсе [**ID3D11Device**](/windows/desktop/api/D3D11/nn-d3d11-id3d11device) были разработаны для повторного входа, не требуя примитивов синхронизации и критических разделов. В результате методы создания ресурсов являются эффективными и простыми в использовании.



|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Различия между Direct3D 11, 10 и 9:<br/> По умолчанию Direct3D 11 использует преимущественно потокобезопасный режим и позволяет приложениям отказаться от использования D3D11 \_ Create \_ Device \_ синглесреадед. Если приложения не являются потокобезопасными, они должны соответствовать правилам потоковой обработки. Среда выполнения синхронизирует потоки от имени приложения, что позволяет выполнять параллельные потоки. Фактически синхронизация в Direct3D 11 эффективнее, чем использование потокобезопасного уровня в Direct3D 10.<br/> Direct3D 10 может поддерживать выполнение только одного потока за раз. Direct3D 10 является полностью потокобезопасным и позволяет приложению отказаться от этого поведения с помощью D3D10 \_ создать \_ устройство с \_ одним \_ потоком. <br/> Direct3D 9 по умолчанию не является потокобезопасным. Однако при вызове [**креатедевице**](/windows/desktop/api/d3d9/nf-d3d9-idirect3d9-createdevice) или [**креатедевицеекс**](/windows/desktop/api/d3d9/nf-d3d9-idirect3d9ex-createdeviceex) для создания устройства можно указать \_ флаг D3DCREATE многопоточность, чтобы обеспечить безопасность потока API Direct3D 9. Это приводит к значительным затратам на синхронизацию. Поэтому не рекомендуется делать потокобезопасность Direct3D 9 потокобезопасным, так как производительность может быть снижена.<br/> |
