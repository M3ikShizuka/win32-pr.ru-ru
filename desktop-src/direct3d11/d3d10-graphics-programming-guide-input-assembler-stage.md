# <a name="input-assembler-stage"></a>Стадия сборщика входных данных

API Direct3D 10 и более поздних версий разделяет функциональные области конвейера на этапы; первым этапом в конвейере является этап ввода-ассемблера (IA).

Целью этапа входного ассемблера является чтение примитивных данных (точек, линий и (или) треугольников) из заполненных пользователем буферов и сборка данных в примитивы, которые будут использоваться другими этапами конвейера. Этап сборщика входных данных может объединять вершины в несколько различных [типов примитивов](d3d10-graphics-programming-guide-primitive-topologies.md), таких как списки линий, полосы треугольников или примитивы со смежностью. Для поддержки шейдера Geometry были добавлены новые типы-примитивы (например, список линий с соседями или треугольниками).

Информация о смежности доступна приложению только в шейдере геометрии. Если шейдер геометрии вызывается, например с помощью треугольника со смежностью, входные данные будут содержать 3 вершины для каждого треугольника и 3 вершины для данных о смежности для каждого треугольника.

Когда на этапе входного ассемблера запрашивается вывод смежных данных, входные данные должны включать в себя смежные данные. Для этого может потребоваться предоставление фиктивной вершины (с образованием неполноценного треугольника) или, возможно, создание в одном из атрибутов вершины отметки о том, существует ли эта вершина. Это также должно распознаваться и обрабатываться шейдером геометрии, несмотря на то что отбраковка неполноценной геометрии будет происходить на этапе растеризации.

При сборке примитивов в качестве дополнительного назначения для архитектуры IA используется присоединение [созданных системой значений](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-semantics) , чтобы сделать шейдеры более эффективными. Создаваемые системой значения являются текстовыми строками, которые также называют семантическими элементами. Все три этапа шейдера формируются из общего ядра шейдера, и в ядре шейдера используются генерируемые системой значения (например, идентификатор примитива, идентификатор экземпляра или идентификатор вершины), чтобы стадия шейдера могла сократить обработку только на те примитивы, экземпляры или вершины, которые еще не были обработаны.

Как показано на [схеме блок-схемы конвейера](/windows/desktop/direct3d10/d3d10-graphics-programming-guide-pipeline-stages), после того, как этап IA считывает данные из памяти (собирает данные в примитивы и присоединяет сформированные системой значения), данные выводятся на [стадию шейдера вершин](/previous-versions//bb205146(v=vs.85)).
