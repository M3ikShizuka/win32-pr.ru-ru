# <a name="tier-1"></a>Уровень 1

В этом разделе описывается поддержка 1-го уровня.

-   Оборудование с уровнем компонентов не менее 11.0.
-   Отсутствует поддержка сшивания.
-   Отсутствует поддержка Texture1D и Texture3D.
-   Отсутствует поддержка сглаживания с множественной дискретизацией (MSAA) с 2, 8 и 16 выборками Требуется только сглаживание с 4 выборками, за исключением форматов 128 бит/пкс.
-   Отсутствует стандартный шаблон выборки (макет для плиток размером 64 КБ и упаковка хвостовых MIP-карт зависит от поставщика оборудования).
-   Ограничения на доступ к плиткам при наличии повторяющихся сопоставлений, описанных в разделе [ограничения доступа к плиткам с повторяющимися сопоставлениями](tile-access-limitations-with-duplicate-mappings-.md).

### <a name="limitations-affecting-tier-1-only"></a>Ограничения, затрагивающие только уровень 1

-   В мозаичных ресурсах могут сопоставлены **значения NULL** , но чтение из них или запись в них приводит к неопределенным результатам, включая удаленное устройство. Приложения могут обойти это путем сопоставления одной фиктивной страницы всем пустым областям. Будьте осторожны при записи и визуализации на странице, сопоставленной с несколькими расположениями целевого объекта прорисовки, так как порядок записи будет неопределенным.
-   Инструкции шейдеров для закрепления уровня детализации и получения состояния сопоставления недоступны. Дополнительные сведения см. в разделе [HLSL мозаичных ресурсов](hlsl-tiled-resources-exposure.md).
-   Ограничения выравнивания для стандартных фигур мозаики: гарантируется, что только MIPS (начиная с Finest), измерения которых являются кратными для стандартного размера плитки, поддерживают стандартные фигуры мозаичных элементов и могут иметь отдельные плитки с произвольным сопоставлением или несопоставленными. Первый mipmap в мозаичном ресурсе с любым измерением, не кратным размеру стандартного мозаичного элемента, а также всем грубым MIP-карты, может иметь нестандартную форму мозаичного заполнения, сопоставленную с N 64 КБ для этого набора MIPS одновременно (N сообщается в приложении). Эти плитки N считаются упакованными одним блоком, который может быть полностью сопоставлен или полностью не сопоставлен приложением в любой момент времени. При этом сопоставления отдельных плиток N могут располагаться в различных произвольных местах пула плиток.
-   Для мозаичного заполнения ресурсов с любым MIP-карты, не кратным размеру стандартного мозаичного элемента во всех измерениях, не допускается, чтобы размер массива был больше 1.
-   Чтобы переключиться между ссылками на плитки в пуле плиток через ресурс- [буфер](overviews-direct3d-11-resources-buffers.md) для ссылки на те же плитки через ресурс [текстуры](overviews-direct3d-11-resources-textures.md) или наоборот, последний вызов [**упдатетилемаппингс**](/windows/desktop/api/D3D11_2/nf-d3d11_2-id3d11devicecontext2-updatetilemappings) или [**копитилемаппингс**](/windows/desktop/api/D3D11_2/nf-d3d11_2-id3d11devicecontext2-copytilemappings) , который определяет сопоставления с плитками пула плиток, должен быть для одного и того же измерения ресурса (буфер и текстура \* ) в качестве измерения ресурса, которое будет использоваться для доступа к плиткам. В противном случае поведение не определено, включая вероятность сброса устройства. Таким образом, например, вызов **упдатетилемаппингс** для определения сопоставлений плиток для буфера, затем **упдатетилемаппингс** на те же плитки в пуле плиток через ресурс [**Texture2D**](/windows/desktop/direct3dhlsl/sm5-object-texture2d) , а затем доступ к плиткам через буфер является недопустимым. Обходные операции должны либо переопределить сопоставления плиток для ресурса при переключении между буфером и текстурой (или наоборот) или просто никогда не использовать совместно в одном пуле плитки буфера ресурсов и буфера текстур.
-   Фильтрация сокращения минимумов/максимумов не поддерживается. Сведения о фильтрации по снижению минимальных и максимальных значений см. в статье [функции выборки текстур для мозаичных ресурсов](tiled-resources-texture-sampling-features.md).
