# <a name="credential-security-support-provider"></a>Поставщик поддержки безопасности учетных данных

Протокол CredSSP — это поставщик поддержки безопасности, реализуемый с помощью интерфейса поставщика поддержки безопасности ([SSPI](sspi.md)). CredSSP позволяет приложению делегировать учетные данные пользователя от клиента целевому серверу для удаленной проверки подлинности. CredSSP предоставляет зашифрованный канал [протокола безопасности транспортного уровня](transport-layer-security-protocol.md) . Проверка подлинности клиента осуществляется по зашифрованному каналу с помощью простого и защищенного протокола Negotiate (SPNEGO) с [Microsoft Kerberos](microsoft-kerberos.md) или [Microsoft NTLM](microsoft-ntlm.md).

> [!Caution]  
> Это не ограниченное делегирование. CredSSP передает полные учетные данные пользователя на сервер без каких бы то ни было ограничений.

 

Сведения о программе SPNEGO см. в разделе [Microsoft Negotiate](microsoft-negotiate.md).

После проверки подлинности клиента и сервера клиент передает учетные данные пользователя на сервер. Учетные данные дважды шифруются в ключах сеанса SPNEGO и TLS. CredSSP поддерживает вход на основе пароля, а также вход с помощью смарт-карты на основе как [*X. 509*](/windows/desktop/SecGloss/x-gly) , так и PKINIT.

> [!IMPORTANT]
> CredSSP не поддерживает клиенты WOW64.
