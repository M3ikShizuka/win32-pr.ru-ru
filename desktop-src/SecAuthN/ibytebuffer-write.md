# <a name="ibytebufferwrite-method"></a>Метод Ибитебуффер:: Write

\[Метод **Write** доступен для использования в операционных системах, указанных в разделе требования. Он недоступен для использования в Windows Server 2003 с пакетом обновления 1 (SP1) и более поздней версии, Windows Vista, Windows Server 2008 и последующих версиях операционной системы. Интерфейс [**IStream**](/windows/desktop/api/objidl/nn-objidl-istream) предоставляет аналогичные функциональные возможности.\]

Метод **Write** записывает указанное число из байтов в объект потока, начиная с текущего указателя поиска.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Write(
  [in]  BYTE *pByte,
  [in]  LONG cb,
  [out] LONG *pcbWritten
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пбите* \[ окне\]
</dt> <dd>

Адрес буфера, содержащего данные, которые должны быть записаны в поток. Для этого параметра должен быть указан допустимый указатель, даже если значение *CB* равно нулю.

</dd> <dt>

*CB* \[ окне\]
</dt> <dd>

Число байтов данных, которые необходимо попытаться записать в поток. Этот параметр может быть равен нулю.

</dd> <dt>

*пкбвриттен* \[ заполняет\]
</dt> <dd>

Адрес **длинной** переменной, в которой этот метод записывает фактическое число байтов, записанных в объект потока. Вызывающий объект может установить для этого указателя **значение NULL**. в этом случае этот метод не предоставляет фактическое число записанных байтов.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращаемое значение является значением **HRESULT**. Значение S \_ ОК указывает на Успешный вызов.

## <a name="remarks"></a>Комментарии

Метод **ибитебуффер:: Write** записывает указанные данные в объект потока. Указатель поиска корректируется на фактическое число записанных байтов. Число фактически записанных байтов возвращается в параметре *пкбвриттен* . Если число байтов равно нулю, операция записи не действует.

Если указатель поиска в текущий момент находится за концом потока, а число байтов не равно нулю, этот метод увеличивает размер потока до указателя поиска и записывает указанные байты, начиная с указателя поиска. Заполнение байтов, записанное в поток, не инициализируется каким бы то ни было определенным значением. Это совпадает с поведением конца файла в файловой системе MS-DOS FAT.

При наличии нулевого числа байтов и указателя поиска за концом потока этот метод не создает байты заполнения для увеличения потока до указателя поиска. В этом случае необходимо вызвать метод [**ибитебуффер:: SetSize**](ibytebuffer-setsize.md) , чтобы увеличить размер потока и записать байты заполнения.

Параметр *пкбвриттен* может иметь значение, даже если возникает ошибка.

В реализации, предоставляемой моделью COM, объекты потока не имеют разреженных объектов. Все байты заполнения в конечном итоге выделяются на диске и назначаются потоку.

## <a name="examples"></a>Примеры

В следующем примере показана запись байтов в объект потока.


```C++
LONG     lWrite;
HRESULT  hr;

// Write to the buffer.
// byData is an array of 64 bytes.
hr = pIByteBuff->Write(byData,
                       64,
                       &lWrite);
if (FAILED(hr))
  printf("Failed IByteBuffer::Write\n");
```
