# <a name="contents-of-a-digest-challenge"></a>Содержимое запроса дайджеста

Размер запроса на дайджест-доступ должен быть менее 2048 байт. В следующем примере показана задача, назначенная строке символов Сзчалленже.


```C++
szChallenge = "realm=\"Microsoft_Example_Forest\",";
algorithm = "MD5-sess\", qop=\"auth\", nonce=\"0123456789abcdef\"";
```



> [!Note]  
> Строка запроса заключается в двойные кавычки и содержит внедренные двойные кавычки. Перед внедренными двойными кавычками следует использовать обратную косую черту ( \\ ).

 

Запрос дайджеста может содержать следующие директивы.



| Директива         | Описание                                                                                                                                                                                                                                                                                            |
|-------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| realm             | Определяемое реализацией указание для клиента о том, какие [*учетные данные*](/windows/desktop/SecGloss/c-gly) требуются. Клиент должен отобразить эти сведения пользователю, если он запрашивает учетные данные.                                                  |
| алгоритм         | Microsoft Digest поддерживает MD5 и MD5-подряд. Для оптимальной производительности используйте MD5-подряд.                                                                                                                                                                                                                     |
| коп               | Для этой директивы можно задать значение AUTH, auth-int или auth-CONF. Дополнительные сведения см. в разделе [качество защиты и шифров](quality-of-protection-and-ciphers.md).                                                                                                                                       |
| nonce             | Уникальное закодированное значение, созданное сервером для каждой задачи. Клиент не должен изменять это значение.                                                                                                                                                                                       |
| непрозрачный            | Содержит ссылку на установленный [*контекст безопасности*](/windows/desktop/SecGloss/s-gly) . Дополнительные сведения см. [в разделе поддержание контекста безопасности между соединениями](maintaining-the-security-context-between-connections.md). |
| шифр (только SASL) | Список шифров, поддерживаемых сервером. Этот элемент может присутствовать в дайджесте SASL, только если директива КОП указывает auth-CONF. Дополнительные сведения см. в разделе [качество защиты и шифров](quality-of-protection-and-ciphers.md).                                              |
| charset           | Для этой директивы можно задать значение UTF-8, если сервер может обрабатывать имена пользователей и сферы в кодировке UTF-8. Если клиент понимает директиву charset, он может ответить, используя значения в кодировке UTF-8.                                                                                                       |



 

Microsoft Digest формирует строку запроса дайджеста для серверных приложений. Дополнительные сведения см. [в разделе Создание дайджест-запроса](generating-the-digest-challenge.md).
