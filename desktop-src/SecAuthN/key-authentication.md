# <a name="key-authentication"></a>Проверка подлинности с использованием ключа

Многие виды проверки подлинности основаны на идее того, что сущность может доказать свою личность, если она может доказать, что она знает ключ, например пароль, который может быть известен только ему.

Методы проверки подлинности, которые используют секрет, например пароль, должны иметь способ защитить секрет от получения общедоступного набора знаний. Владелец пароля не может пройти до двери и ввести пароль. Кто-то, помимо дуркипер, может прослушиваться; или это может быть неверный дверца. Для сохранения секрета должен быть способ подтвердить, что пользователь знает пароль, не раскрывая пароль. Это идея проверки подлинности секретного ключа, метода проверки, используемого по всему [*протоколу Kerberos*](../secgloss/k-gly.md).

Обратите внимание, что «секрет» в проверке подлинности с помощью секретного ключа заключается в том, что процесс проверки подлинности выполняется в секрете, то есть без фактического разглашения содержимого ключа.

Чтобы проверка подлинности с помощью секретного ключа работала, две стороны транзакции должны совместно использовать криптографический [*ключ сеанса*](../secgloss/s-gly.md) , который также известен только для них и не может быть другим. Ключ является [*симметричным*](../secgloss/s-gly.md); Это единственный ключ, используемый для шифрования и расшифровки. Одна сторона в процессе проверки подлинности подтверждает знание ключа, шифруя сообщение. Другая сторона подтверждает свои знания о ключе, расшифровании сообщения.
