# <a name="receiving-connection-notifications"></a>Получение уведомлений о подключении

Некоторые приложения должны принимать уведомления о событиях соединения, перед событием, сразу после его возникновения или в обоих случаях. Вы можете создать библиотеку DLL для получения уведомлений о событиях соединения, а не на самом деле.

Примером приложения, которое должно получать предварительное уведомление о событии подключения, является служба удаленного доступа (RAS). Служба RAS должна быть уведомлена перед подключением, поскольку перед подключением к сети может потребоваться установить модемное подключение.

Аналогичным образом, приложениям может потребоваться очистка ресурсов после подключения, поэтому требуется уведомление после подключения.

Приложения, заинтересованные в получении аванса и уведомления о событиях соединения по факту, должны предоставить библиотеку DLL, которая экспортирует две функции, [**аддконнектнотифи**](/windows/desktop/api/Npapi/nf-npapi-addconnectnotify) и [**канцелконнектнотифи**](/windows/desktop/api/Npapi/nf-npapi-cancelconnectnotify).

После реализации этих функций необходимо зарегистрировать библиотеку DLL, как описано в статье [Регистрация для получения уведомлений о подключениях](registering-to-receive-connection-notifications.md).
