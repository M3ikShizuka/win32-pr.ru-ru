# <a name="accessing-a-smart-card"></a>Доступ к смарт-карте

Подсистема [*смарт-карт*](/windows/desktop/SecGloss/s-gly) предоставляет несколько средств для подключения поставщика приложения или [*службы*](/windows/desktop/SecGloss/c-gly) к смарт-карте:

-   Приложение может вызвать [**скардконнект**](/windows/desktop/api/Winscard/nf-winscard-scardconnecta) для подключения к карточке, которая находится в данном модуле чтения. Это самый простой способ установить связь со смарт-картой.
-   Приложение может искать определенную смарт-карту в заданной группе читателей. Приложение идентифицирует карточку по ее отображаемому имени и указывает список читателей, в которых может появиться карточка. [*Диспетчер ресурсов*](/windows/desktop/SecGloss/r-gly) выполняет поиск в списке модулей чтения для всех карточек со [*строкой ATR*](/windows/desktop/SecGloss/a-gly) , соответствующей именованной карточке, и возвращает сведения о состоянии приложению. [*Подсистема смарт-карт*](/windows/desktop/SecGloss/s-gly) никогда не помещает графический пользовательский интерфейс или взаимодействует с картой, если вы не получите строку ATR. Однако он предоставляет достаточно информации для приложения или общего элемента управления, чтобы проанализировать пользователя с помощью поиска нужной карты или типа карты. Это приводит к сопоставлению запроса с конкретным модулем чтения, на который направляется дальнейший ввод-вывод.
-   Приложение может запросить список карточек, поддерживающих заданный набор интерфейсов смарт-карт. Затем приложение может использовать список в предыдущем случае. Это позволяет приложениям подключаться к картам в зависимости от их возможностей, не обращаясь к их именам.

Когда приложение ищет карточку, оно предоставляет массив имен модулей чтения, в котором будет выглядеть. Для каждого элемента Reader в массиве диспетчер ресурсов предоставляет следующие сведения:

-   Доступно ли средство чтения для использования этим приложением.
-   Указывает, вставлена ли в этот модуль чтения карта, и если да, то какова ее строка ATR.
-   Совпадает ли строка ATR карточки с любыми строками "ATR" запрашиваемых карточек.

Приложение использует возвращенные сведения для применения дополнительных фильтров к картам или для запроса пользователя на выбор нужной карточки. Обратите внимание, что один или несколько возвращенных списков читателей могут быть открыты для монопольного использования другими приложениями, поэтому доступ к этому списку читателей не гарантируется.
