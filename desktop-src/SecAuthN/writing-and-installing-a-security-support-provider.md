# <a name="writing-and-installing-a-security-support-provider"></a>Написание и установка поставщика поддержки безопасности

Конструкция SSPI позволяет записывать и добавлять дополнительные SSP в систему. [*Поставщик общих служб*](../secgloss/s-gly.md) , относящийся к модели безопасности, можно разработать с относительной простотой или отличной сложностью в зависимости от уровня интеграции с операционной системой. Клиентский поставщик общих служб, который разрешает подключения к новому типу сервера, может быть разработан очень быстро, в то время как полный поставщик общих служб, предоставляющий локальное олицетворение, потребует больших усилий.

SSP устанавливаются путем обновления значения **reg \_ SZ** в реестре, как показано ниже.

**HKey \_ Локальный \_ компьютер** \\ **система** \\ **CurrentControlSet** \\ **элемент управления** \\ **секуритипровидерс**  =  *Provider1.dll, Provider2.dll,*...    <dl> <dt>

            Data type
</dt> <dd>            REG \_ SZ</dd> </dl>

Одна библиотека DLL может содержать несколько SSP.
