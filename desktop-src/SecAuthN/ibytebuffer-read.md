# <a name="ibytebufferread-method"></a>Метод Ибитебуффер:: Read

\[Метод **Read** доступен для использования в операционных системах, указанных в разделе требования. Он недоступен для использования в Windows Server 2003 с пакетом обновления 1 (SP1) и более поздней версии, Windows Vista, Windows Server 2008 и последующих версиях операционной системы. Интерфейс [**IStream**](/windows/desktop/api/objidl/nn-objidl-istream) предоставляет аналогичные функциональные возможности.\]

Метод **Read** считывает указанное число байтов из объекта buffer в память, начиная с текущего указателя поиска.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT Read(
  [out] BYTE *pByte,
  [in]  LONG cb,
  [out] LONG *pcbRead
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пбите* \[ заполняет\]
</dt> <dd>

Указывает на буфер, в который считываются потоковые данные. Если возникает ошибка, это значение равно **null**.

</dd> <dt>

*CB* \[ окне\]
</dt> <dd>

Число байтов данных, которые необходимо попытаться считать из объекта потока.

</dd> <dt>

*пкбреад* \[ заполняет\]
</dt> <dd>

Адрес **длинной** переменной, которая получает фактическое число байтов, считанных из объекта потока. Можно задать для этого указателя значение **null** , чтобы указать, что вы не заинтересованы в этом значении. В этом случае этот метод не обеспечивает фактическое число считанных байтов.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращаемое значение является значением **HRESULT**. Значение S \_ ОК указывает на Успешный вызов.

## <a name="remarks"></a>Комментарии

Этот метод считывает байты из этого объекта потока в память. Объект потока должен быть открыт в \_ режиме чтения стгм. Этот метод корректирует указатель поиска на фактическое число считанных байтов.

Число фактически считанных байтов также возвращается в параметре *пкбреад* .

Примечания для тех, кто вызывает этот метод

Фактическое число считанных байтов может быть меньше, чем число запрошенных байтов, если возникла ошибка или достигнут конец потока во время операции чтения.

Некоторые реализации могут возвращать ошибку, если в процессе чтения достигнут конец потока. Необходимо подготовиться к работе с возвращаемыми ошибками или \_ возвратами ОК при завершении операций чтения потока.

## <a name="examples"></a>Примеры

В следующем примере показано чтение байтов из буфера.


```C++
BYTE     byAtr[32];
long     lBytesRead, i;
HRESULT  hr;

// pAtr is a pointer to a previously instantiated IByteBuffer.
// It was used in an earlier call by ISCard::get_Atr.
// Use IByteBuffer::Read to access the retrieved ATR bytes.
hr = pAtr->Read(byAtr, 32, &lBytesRead);
// Use the ATR value. (This example merely displays the bytes.)
for ( i = 0; i < lBytesRead; i++)
    printf("%c", *(byAtr + i));
printf("\n");
```
