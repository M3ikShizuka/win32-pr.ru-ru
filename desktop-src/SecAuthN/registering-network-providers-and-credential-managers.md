# <a name="registering-network-providers-and-credential-managers"></a>Регистрация поставщиков сети и диспетчеров учетных данных

После создания поставщика сети или диспетчера учетных данных необходимо зарегистрировать его в системе. Это необходимо, чтобы многопротокольный маршрутизатор знал, какие сетевые поставщики установлены. После запуска многопротокольного маршрутизатора он проверяет реестр и загружает найденные поставщики сетевых услуг или диспетчеров учетных данных.

Так как поставщики сетевых услуг и диспетчеры учетных данных связаны друг с друга, они регистрируются в тех же подразделах реестра. Фактически, библиотеки DLL поставщика сетевых услуг также могут быть диспетчерами учетных данных.

Подробные сведения о разделах реестра, которые следует создать для поставщика сети или диспетчера учетных данных, см. в разделе [разделы реестра для проверки подлинности](authentication-registry-keys.md).
