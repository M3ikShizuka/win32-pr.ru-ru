# <a name="gina"></a>GINA

[*GINA*](/windows/desktop/SecGloss/g-gly) работает в [*контексте*](/windows/desktop/SecGloss/c-gly) процесса [*Winlogon*](/windows/desktop/SecGloss/w-gly) и, таким образом, DLL-библиотека GINA загружается на ранних этапах процесса загрузки. Библиотека GINA DLL должна следовать правилам, чтобы обеспечить целостность системы, в частности в отношении взаимодействия с пользователем.

> [!Note]  
> библиотеки dll GINA не учитываются в Windows Vista.

 

Чаще всего GINA используется для связи с внешним устройством, таким как [*устройство чтения*](/windows/desktop/SecGloss/r-gly)смарт-карт. Важно установить параметр Start для драйвера устройства в значение System (Winnt. h: \_ Запуск системы службы \_ ), чтобы убедиться, что драйвер загружен на момент вызова GINA.

БИБЛИОТЕКА GINA предназначена для предоставления настраиваемых процедур идентификации и аутентификации пользователей. GINA по умолчанию делает это путем делегирования отслеживания событий SAS в Winlogon, который получает и обрабатывает список [*безопасных сочетаний*](/windows/desktop/SecGloss/s-gly) клавиш (Sasser) CTL + Alt + Del. Пользовательская GINA несет ответственность за настройку, чтобы получать события SAS (отличные от событий по умолчанию CTRL + ALT + DEL) и уведомлять об этом Winlogon при возникновении событий SAS. Winlogon выполнит оценку своего состояния, чтобы определить, что необходимо для обработки SAS пользовательской модели GINA. Эта обработка обычно включает вызовы функций обработки SAS модели GINA.

Сведения о конкретных функциях экспорта GINA см. в разделе [функции экспорта GINA](authentication-functions.md). Сведения об использовании структур GINA для передачи информации см. в разделе [модели GINA](authentication-structures.md).
