# <a name="sspi-model"></a>Модель SSPI

[*Интерфейс поставщика поддержки безопасности*](../secgloss/s-gly.md) (SSPI) позволяет приложению использовать различные модели безопасности, доступные на компьютере или в сети, не изменяя интерфейс системы безопасности. SSPI не устанавливает [*учетные данные*](../secgloss/c-gly.md) для входа в систему, так как обычно это привилегированная операция, обрабатываемая операционной системой.

[*Поставщик поддержки безопасности*](../secgloss/s-gly.md) (SSP) содержится в [*библиотеке динамической компоновки*](../secgloss/d-gly.md) (DLL), которая реализует SSPI, делая один или несколько [*пакетов безопасности*](../secgloss/s-gly.md) доступными для приложений. Каждый пакет безопасности предоставляет сопоставления между вызовами функции SSPI приложения и функциями реальной модели безопасности. Пакеты безопасности поддерживают такие [*протоколы безопасности*](../secgloss/s-gly.md) , как проверка подлинности [*Kerberos*](../secgloss/k-gly.md) и диспетчер LAN.

Интерфейс SSPI доступен в режиме ядра, а также в пользовательском режиме. чтобы использовать функциональность SSPI в режиме ядра, необходимо установить Windows устанавливаемую файловые системы DDK. Вызывающая модель остается неизменной, но рекомендации режима ядра указаны на страницах справки по функциям. Дополнительные сведения см. в разделе [функции SSPI](authentication-functions.md).
