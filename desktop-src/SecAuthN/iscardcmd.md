# <a name="iscardcmd-interface"></a>Интерфейс Искардкмд

\[Интерфейс **искардкмд** доступен для использования в операционных системах, указанных в разделе требования. он недоступен для использования в Windows Server 2003 с пакетом обновления 1 (SP1) и более поздних версий, Windows Vista, Windows Server 2008 и последующих версиях операционной системы. [Модули смарт-карт](/previous-versions/windows/desktop/secsmart/smart-card-modules) предоставляют аналогичные функции.\]

Интерфейс **искардкмд** предоставляет методы, необходимые для создания и управления [*блоком данных протокола приложений*](../secgloss/a-gly.md) [*смарт-карт*](../secgloss/s-gly.md) (категориях APDU). Этот интерфейс инкапсулирует два буфера:

-   Буфер КАТЕГОРИЯХ APDU содержит последовательность команд, которая будет отправлена на карточку.
-   Буфер Апдурепли содержит данные, возвращаемые из карты после выполнения команды КАТЕГОРИЯХ APDU (эти данные также называются возвращаемыми КАТЕГОРИЯХ APDU).

В следующем примере показано типичное использование интерфейса **искардкмд** . Интерфейс **искардкмд** используется для создания категориях APDU.

**Отправка транзакции на определенную карточку**

1.  Создание интерфейса [**карты**](iscard.md) и подключение к смарт-карте.
2.  Создайте интерфейс **искардкмд** .
3.  Создайте команду смарт-карты КАТЕГОРИЯХ APDU с помощью интерфейса [**ISCardISO7816**](iscardiso7816.md) или одного из **искардкмд** методов сборки.
4.  Выполните команду на смарт-карте, вызвав соответствующий метод интерфейса [**кредитной**](iscard.md) карты.
5.  Вычисление возвращаемого ответа.
6.  При необходимости повторите процедуру.
7.  При необходимости освободите интерфейс **искардкмд** и другие.
