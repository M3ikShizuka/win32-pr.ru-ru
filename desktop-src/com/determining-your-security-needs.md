# <a name="determining-your-security-needs"></a>Определение потребностей в безопасности

Настройка безопасности COM для приложения зависит от того, какой тип безопасности требуется приложению. Существует несколько распространенных ситуаций, которые определяют, что следует делать.

Если вы решили использовать параметры безопасности COM по умолчанию, вам не нужно делать анисингâ "COM" обрабатывать все. Сведения о параметрах по умолчанию см. в разделе Параметры [безопасности COM по умолчанию](com-security-defaults.md).

Кроме того, можно предотвратить удаленные вызовы к компьютеру, отключив DCOM полностью (COM между удаленными компьютерами). Дополнительные сведения см. в разделе [Настройка безопасности System-Wide с помощью DCOMCNFG](setting-machine-wide-security-using-dcomcnfg.md).

Для устаревших или новых приложений в реестре можно задать параметры безопасности на уровне процесса. Дополнительные сведения см. [в разделе Настройка безопасности процессвиде в реестре](setting-processwide-security-through-the-registry.md).

Можно также переопределить параметры безопасности по умолчанию для вызовов определенных интерфейсов в процессе, установив безопасность по умолчанию для оставшейся части процесса (чтобы разрешить COM обработку общих вариантов). Дополнительные сведения см. в разделе [Настройка безопасности на уровне прокси интерфейса](setting-security-at-the-interface-proxy-level.md).

В случае сложных требований безопасности вы можете управлять всей безопасностью программным способом, не разрешая COM выполнять их самостоятельно. Для этого вызовите [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity) , чтобы отключить автоматическую проверку подлинности, а затем Контролируйте все параметры безопасности, установив параметры безопасности для каждого интерфейса прокси. Дополнительные сведения см. в разделе [Настройка безопасности процессвиде с помощью CoInitializeSecurity](setting-processwide-security-with-coinitializesecurity.md) и [Настройка безопасности на уровне прокси интерфейса](setting-security-at-the-interface-proxy-level.md).

В некоторых сценариях может потребоваться отключить безопасность полностью. Вы можете решить, что приложение не нуждается в безопасности, или вы можете отключить безопасность во время разработки, чтобы включить функции безопасности по отдельности. Сведения об отключении безопасности COM см. в разделе Отключение [безопасности](turning-off-security.md).

Безопасность в COM зависит от служб проверки подлинности, администрируемых пакетами безопасности. Поставщик NTLMSSP прекрасно работает для многих приложений, но не обеспечивает более надежную защиту, предоставляемую другими пакетами. Поэтому COM поддерживает пакет безопасности SChannel и протокол безопасности Kerberos V5. Дополнительные сведения об использовании этих пакетов безопасности см. в разделе [com и пакеты безопасности](com-and-security-packages.md).
