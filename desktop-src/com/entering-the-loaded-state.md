# <a name="entering-the-loaded-state"></a>Вход в загруженное состояние

Когда объект входит в загруженное состояние, создаются структуры в памяти, представляющие объект, чтобы можно было вызывать операции с ним. Загружается обработчик объекта или внутрипроцессный сервер. Этот процесс, называемый *созданием экземпляра*, возникает, когда объект загружается из постоянного хранилища (переход от пассивного состояния к загруженному) или когда объект создается в первый раз.

На внутреннем уровне создание экземпляра — двухэтапный процесс. Создается объект соответствующего класса, после которого вызывается метод этого объекта для выполнения инициализации и предоставления доступа к данным этого объекта. Метод инициализации определен в одном из поддерживаемых интерфейсов объекта. Определенный метод инициализации зависит от контекста, в котором создается экземпляр объекта, и расположения данных инициализации.
