# <a name="activation-security"></a>Безопасность активации

Безопасность активации (также называемая безопасностью запуска) помогает контролировать, кто может запускать сервер. Безопасность активации автоматически применяется диспетчером управления службами (SCM) определенного компьютера. После получения запроса от клиента на активацию объекта (как описано в разделе [вспомогательные функции создания экземпляра](instance-creation-helper-functions.md)), SCM проверяет запрос на соответствие сведениям о безопасности активации, хранящимся в реестре. (Безопасность активации также проверяется на наличие активаций для одного компьютера.)

При определении удостоверения клиента активация проверяет флаг маскировки, установленный в вызове клиента в [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity). Если установлен флаг маскировки (для динамической или статической маскировки), для определения удостоверения клиента используется токен потока, если он есть. Если маскировка не задана, вместо токена потока используется токен процесса.

Дополнительные сведения о безопасности активации см. в разделе [**коаусинфо**](/windows/desktop/api/wtypesbase/ns-wtypesbase-coauthinfo) и [**косерверинфо**](/windows/win32/api/objidlbase/ns-objidlbase-coserverinfo).
