# <a name="idl-files"></a>IDL-файлы

COM использует язык MIDL (MIDL) для описания COM-объектов. MIDL является расширением IDL для распределенных вычислительных сред, определенных открытым программным обеспечением, разработанным для определения интерфейсов удаленных вызовов процедур в традиционных клиентских и серверных приложениях. MIDL включает большую часть атрибутов и инструкций языка определения объектов (ODL), изначально используемый для создания библиотек типов для OLE-автоматизации.

В C++ и Java разработчик, создающий объект COM, создает IDL-файл, который компилятор MIDL обрабатывает для создания библиотеки типов, а также для файлов заголовков и прокси-серверов. *Библиотека типов* — это двоичный файл, ОПИСЫВАЮЩИЙ COM-объект или COM-интерфейсы или оба. Библиотека типов — это скомпилированная версия IDL-файла. Однако библиотеки типов поддерживают только семантику ODL. В частности, они не могут представлять всю информацию из IDL-файла, относящегося к атрибутам IDL \[ [**\_ , например size**](/windows/desktop/Midl/size-is) \] . Необходимо создать и использовать прокси-файлы для файлов IDL, затронутых потерей информации в библиотеке типов.
