# <a name="microsoft-rpc"></a>Microsoft RPC

Microsoft RPC — это модель для программирования в распределенных вычислительных средах. Целью RPC является обеспечение прозрачного взаимодействия, чтобы клиент взаимодействует непосредственно с сервером. Реализация RPC Майкрософт совместима с RPC с открытым программным обеспечением (использование).

Вы можете настроить RPC для использования одного или нескольких транспортов, одной или нескольких служб имен и одного или нескольких серверов безопасности. Интерфейсы для этих поставщиков обрабатываются RPC. Поскольку Microsoft RPC предназначен для работы с несколькими поставщиками, вы можете выбрать поставщиков, которые лучше подходят для вашей сети. Транспорт отвечает за передачу данных по сети. Служба имен принимает имя объекта, например моникер, и находит его расположение в сети. Сервер безопасности предлагает приложениям возможность запретить доступ определенным пользователям и (или) группам. Более подробные сведения о безопасности приложений см. в разделе [правила разработки интерфейсов](interface-design-rules.md) .

Помимо библиотек времени выполнения RPC, Microsoft RPC включает в себя язык описания интерфейса (IDL) и его компилятор. Хотя IDL-файл является стандартной частью RPC, корпорация Майкрософт улучшила ее функциональность для поддержки пользовательских COM-интерфейсов. Компилятор язык MIDL (MIDL) использует IDL-файл, описывающий пользовательский интерфейс, для создания нескольких файлов, обсуждаемых в разделе [Создание и регистрация библиотеки DLL прокси](building-and-registering-a-proxy-dll.md).
