# <a name="data-transfer"></a>Передача данных

Модель COM предоставляет стандартный механизм передачи данных между приложениями. Этот механизм является *объектом данных*, который представляет собой любой COM-объект, реализующий интерфейс [**IDataObject**](/windows/desktop/api/ObjIdl/nn-objidl-idataobject) . Некоторые объекты данных, например фрагмент текста, скопированного в буфер обмена, имеют интерфейс **IDataObject** в качестве единственного интерфейса. Другие, такие как объекты составного документа, предоставляют несколько интерфейсов, для которых в качестве интерфейса **IDataObject** используется только один. Объекты данных являются фундаментальными для работы с составными документами, хотя они также имеют широкое приложение вне этой технологии OLE.

Благодаря обмену указателями на объект данных поставщики и потребители данных могут управлять передачей данных единообразно, независимо от формата данных, типа носителя, используемого для передачи данных, или целевого устройства, на котором они должны быть подготовлены для просмотра. В приложение можно включить поддержку для базовых передач буфера обмена, перетаскивания передачи и передачи составных документов OLE с помощью одной реализации интерфейса [**IDataObject**](/windows/desktop/api/ObjIdl/nn-objidl-idataobject). После этого объем кода, необходимый для размещения специальной семантики каждого протокола, является минимальным.
