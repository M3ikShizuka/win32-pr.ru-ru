# <a name="object-handlers"></a>Обработчики объектов

Если серверное приложение OLE является локальным сервером, то есть оно выполняется в собственном пространстве процесса, взаимодействие между контейнером и сервером должно осуществляться через границы процесса. Поскольку этот процесс занимает много ресурсов, OLE использует суррогатный объект, загруженный в пространство процесса контейнера, для выполнения от имени локального серверного приложения. Этот суррогатный объект, известный как *обработчик объектов*, требует наличия запросов к контейнеру служб, которые не требуют внимания серверного приложения, например запросов рисования. Когда контейнер запрашивает, что обработчик объектов не может предоставить, обработчик взаимодействует с серверным приложением, используя механизм взаимодействия СОМ вне процесса.

Обработчик объекта уникален для класса объекта. При создании экземпляра обработчика для одного класса его нельзя использовать для другого. При использовании для составного документа обработчик объектов реализует структуры данных на стороне контейнера, когда доступ к объектам определенного класса осуществляется удаленно.

OLE предоставляет обработчик объектов по умолчанию, который могут использовать приложения локального сервера. Для приложений, требующих особых поведений, разработчики могут реализовать пользовательский обработчик, который либо заменяет обработчик по умолчанию, либо использует его для предоставления определенных поведений по умолчанию.

Обработчик объектов — это библиотека DLL, содержащая несколько взаимодействующих компонентов. Эти компоненты включают удаленные части для управления связью между обработчиком и его серверным приложением, кэш для хранения данных объекта, а также сведения о том, как следует форматировать и отображать данные, а также управляющий объект, который координирует действия других компонентов библиотеки DLL. Кроме того, если объект является ссылкой, Библиотека DLL также включает компонент связывания или *связанный объект*, который отслеживает имя и расположение источника ссылки.

*Кэш* содержит данные и сведения о представлении, достаточные для того, чтобы обработчик отображал загруженный, но не выполняющийся объект в своем контейнере. OLE предоставляет реализацию кэша, используемого обработчиком объектов по умолчанию OLE и объектом Link. Кэш хранит данные в форматах, необходимых обработчику объектов для удовлетворения запросов рисования контейнера. При изменении данных объекта объект отправляет уведомление в кэш, чтобы можно было выполнить обновление. Дополнительные сведения о кэше см. в разделе [Просмотр кэширования](view-caching.md).
