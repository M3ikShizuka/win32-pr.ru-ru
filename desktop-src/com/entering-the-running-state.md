# <a name="entering-the-running-state"></a>Вход в состояние выполнения

Когда внедренный объект выполняет переход в состояние выполнения, обработчик объектов должен нахождение и запуск серверного приложения для использования служб, которые предоставляет только сервер. Внедренные объекты помещаются в состояние выполнения явным образом через запрос контейнера, например, необходимо нарисовать формат, который в данный момент не был кэширован, или неявно OLE в ответ на вызов некоторой операции, например, когда пользователь контейнера дважды щелкает объект.

Когда связанный объект выполняет переход в состояние выполнения, процесс называется *привязкой*. В процессе привязки обработчик объектов запрашивает сохраненное моникер для размещения данных ссылки, а затем запускает серверное приложение.

На первый взгляд, привязка связанного объекта не сложнее, чем выполнение внедренного объекта. Однако следующие моменты усложняют процесс:

-   Ссылка может ссылаться на объект или часть, которая внедрена в другой контейнер. Эта возможность подразумевает возможность вложенных внедрений. Разрешение ссылок на такую иерархию требует рекурсивного прохода по *составному моникеру*, начиная с крайнего правого элемента.
-   Когда источник связи выполняется, OLE привязывается к выполняющемуся экземпляру объекта, а не к другому экземпляру. В случае вложенных внедренных объектов, один из которых является источником ссылки, OLE должен иметь возможность привязки к уже выполняющемуся объекту в любой точке.
-   Для выполнения объекта требуется доступ к области хранения для объекта. При запуске внедренного объекта OLE получает указатель на хранилище во время процесса загрузки, который передается в приложение OLE-сервера. Однако для связанных объектов нет стандартного интерфейса для доступа к хранилищу. Серверное приложение OLE может использовать интерфейс файловой системы или другой механизм.
