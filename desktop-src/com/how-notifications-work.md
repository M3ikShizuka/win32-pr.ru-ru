# <a name="how-notifications-work"></a>Как работают уведомления

Уведомления создаются в приложении Object и передаются в контейнер с помощью обработчика объектов. Если объект является связанным объектом, связанный объект перехватывает уведомления от обработчика объектов и уведомляет контейнер напрямую.

Приложение объекта должно управлять запросами на регистрацию, следить за тем, куда отправлять уведомления и отправлять эти уведомления, когда это необходимо. OLE предоставляет два объекта компонентов для упрощения этой задачи: Олеадвисехолдер для уведомлений составного документа и Датаадвисехолдер для уведомлений об изменении данных.

Приложения объектов определяют условия, которые запрашивают отправку каждого конкретного уведомления, и частоту отправки каждого уведомления. Если нужно отправить несколько уведомлений, не имеет значения, какое уведомление отправляется первыми. их можно отправить в любом порядке.

Время уведомлений влияет на производительность и координацию между приложением объекта и его контейнерами. В то время как уведомления отправляются слишком часто, это приводит к слишком редкому получению уведомлений из-за несинхронизированного контейнера. Частоту уведомлений можно сравнивать с частотой, с которой приложение перерисовывается. Поэтому разумно использовать аналогичную логику для синхронизации уведомлений (как используется для перерисовки).
