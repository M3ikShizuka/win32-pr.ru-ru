# <a name="creating-an-object-through-a-class-object"></a>Создание объекта через объект класса

Благодаря повышению важности компьютерных сетей она стала необходима для взаимодействия клиентов и серверов, независимо от того, находятся ли они на одном и том же компьютере или по сети. Критически важным для этого является возможность клиента запустить сервер, создать экземпляр объекта сервера и получить доступ к методам интерфейсов в объекте.

COM предоставляет расширения для этого базового COM-процесса, которые делают его практически беспрепятственно в сети. Если клиент способен опознать сервер через его CLSID, вызов нескольких простых функций позволяет COM выполнять всю работу по поиску и запуску сервера и активации объекта. Подразделы в реестре позволяют удаленным серверам регистрировать свое расположение, поэтому клиент не требует этих сведений. Для приложений, которые хотят воспользоваться преимуществами сетевых функций, функции создания объектов COM обеспечивают большую гибкость и эффективность.
