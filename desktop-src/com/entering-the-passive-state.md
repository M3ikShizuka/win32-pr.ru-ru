# <a name="entering-the-passive-state"></a>Вход в пассивное состояние

Закрытие объекта принудительно принуждает внедренный или связанный объект в пассивное состояние. Обычно она инициируется из пользовательского интерфейса приложения OLE-сервера, например, когда пользователь выбирает команду Закрыть файл. В этом случае серверное приложение OLE уведомляет контейнер, который освобождает его счетчик ссылок на объект. Когда все ссылки на объект были освобождены, объект можно освободить. Когда все объекты были освобождены, приложение OLE Server может безопасно завершить работу.

Приложение-контейнер также может инициировать закрытие объекта. Чтобы закрыть объект, контейнер освобождает его счетчик ссылок после выполнения необязательной операции сохранения. Можно создавать контейнеры, чтобы освободить объекты, когда они деактивируются после сеанса активации на месте, позволяя пользователю щелкнуть за пределами объекта без потери активного сеанса редактирования.
