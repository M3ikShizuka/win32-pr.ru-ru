# <a name="com-language-translations"></a>Переводы языка COM

Компоненты, созданные с помощью модели COM, можно повторно использовать в приложениях, написанных на любом языке программирования, поддерживающем COM. Это обусловлено тем, что COM является двоичным стандартом и, как таковой, не зависит от языка.

Объекты COM описаны в наиболее соответствующем языке программирования или языках. например, объекты, создаваемые для использования на веб-страницах, обычно задокументированы в системе разработки Microsoft Visual Basic, тогда как объекты уровня системы обычно документируются в C++. Однако, поскольку модель COM не зависит от языка, вы не ограничены использованием объекта на том же языке, на котором он написан или документирован. например, можно написать приложение в JScript, которое использует элемент управления, созданный в C++ и описанный в Visual Basic.

В следующих разделах рассматриваются различия между языками программирования и описывается способ преобразования синтаксиса объектов COM с одного языка на другой. В дополнительных разделах описывается использование COM-объектов в различных языках и средах сценариев.

-   [Синтаксические отличия](syntax-differences.md)
-   [Преобразования типов данных](data-type-conversions.md)
-   [IDL-файлы](idl-files.md)
-   [Преобразование синтаксиса COM-объектов для языков программирования](translating-com-object-syntax-for-programming-languages.md)
-   [Создание сценариев с помощью COM-объектов](scripting-with-com-objects.md)

Цель — решить наиболее распространенные проблемы преобразования языка, возникающие при использовании COM-объектов. Описанные методики и принципы применимы к любому языку программирования или сценарию, поддерживающему COM. Поскольку языки сценариев и языки программирования представляют разные парадигмы программирования, преобразование между языками сценариев и языками программирования не устраняется.
