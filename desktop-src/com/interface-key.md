# <a name="interface-key"></a>Ключ интерфейса

Регистрирует новые интерфейсы, сопоставляя имя интерфейса с ИДЕНТИФИКАТОРом интерфейса (IID). Для каждого нового интерфейса должен существовать один подраздел IID.

## <a name="registry-key"></a>Ключ реестра

**HKey \_ \_ \\ Классы программного \\ обеспечения \\ локального компьютера — интерфейс** \\ *{* IID *}*



| Значение реестра                               | Описание                                                                                            |
|----------------------------------------------|--------------------------------------------------------------------------------------------------------|
| [**BaseInterface**](baseinterface.md)       | Определяет интерфейс, производным от которого является текущий интерфейс.                                  |
| [**нуммесодс**](nummethods.md)             | Содержит число методов в связанном интерфейсе, включая методы из производных интерфейсов. |
| [**проксистубклсид**](proxystubclsid.md)     | Карты идентификатор IID для идентификатора CLSID в 16-разрядных прокси-библиотеках.                                                           |
| [**ProxyStubClsid32**](proxystubclsid32.md) | Карты идентификатор IID для идентификатора CLSID в 32-разрядных прокси-библиотеках.                                                           |



 

## <a name="remarks"></a>Remarks

Ключ **\_ \_ \\ \\ классов программного обеспечения файла hKey на локальном компьютере** соответствует **\_ \_ корневому ключу classes** , который был сохранен для совместимости с предыдущими версиями com.
