# <a name="application-identity"></a>Служба удостоверений приложения

Удостоверение приложения — это учетная запись, используемая для запуска приложения. Идентификатором может быть пользователь, который в данный момент вошел в систему (интерактивный пользователь), пользователь, запустивший сервер, указанный пользователь или служба. Сведения о настройке удостоверения приложения с помощью Dcomcnfg.exe см. в разделе [Настройка безопасности Процессвиде с помощью DCOMCNFG](setting-processwide-security-using-dcomcnfg.md).

В следующих разделах описываются все четыре типа удостоверений приложений и объясняются ограничения каждого типа.

-   [Интерактивный пользователь](interactive-user.md)
-   [Запуск пользователя](launching-user.md)
-   [Указанный пользователь](specified-user.md)
-   [Удостоверение службы](service-identity.md)
