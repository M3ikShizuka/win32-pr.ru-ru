# <a name="self-registration-for-controls"></a>Самостоятельная регистрация для элементов управления

ActiveX элементы управления должны поддерживать самостоятельную регистрацию, реализуя функции [**DllRegisterServer**](/windows/win32/api/olectl/nf-olectl-dllregisterserver) и [**DllUnregisterServer**](/windows/win32/api/olectl/nf-olectl-dllunregisterserver) . ActiveX элементы управления должны регистрировать все стандартные записи реестра для внедряемых объектов и серверов автоматизации.

ActiveX элементы управления должны использовать API категорий компонентов для регистрации себя в качестве элемента управления и регистрации категорий компонентов, которые требуются для поддержки узла, а также всех категорий, реализуемых элементом управления, в разделе [категории компонентов](component-categories.md).

ActiveX элементы управления должны также зарегистрировать раздел реестра [ToolBoxBitmap32](toolboxbitmap32.md) , хотя это необязательно.

Категория вставляемого компонента должна быть зарегистрирована, только если элемент управления подходит для использования в качестве объекта составного документа. важно отметить, что объект составного документа должен поддерживать определенные интерфейсы сверх минимально необходимого для ActiveX элемента управления интерфейса [**IUnknown**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown) . несмотря на то, что элемент управления ActiveX может быть назначен как объект составного документа, документация элемента управления должна четко определять, какое поведение должно быть ожидаемым в этих обстоятельствах.
