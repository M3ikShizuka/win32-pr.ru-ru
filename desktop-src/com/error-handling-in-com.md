# <a name="error-handling-in-com-com"></a>Обработка ошибок в COM (COM)

Почти все COM-функции и методы интерфейса возвращают значение типа **HRESULT**. **HRESULT** (для *результирующего маркера*) — это способ возвращения значений успеха, предупреждения и ошибки. **Значения HRESULT** в действительности не обрабатывают ничего; они являются только значениями с несколькими полями, закодированными в значении. Как и в случае со спецификацией COM, результатом нулевого результата является Success и ненулевое значение указывает на сбой.

На уровне исходного кода все значения ошибок состоят из трех частей, разделенных символами подчеркивания. Первая часть — это префикс, определяющий средство, связанное с ошибкой, вторая часть — E для ошибки, а третья часть — строка, описывающая фактическое условие. Например, **STG \_ E \_ медиумфулл** возвращается, если на жестком диске не осталось места. Префикс **STG** указывает на хранилище, а значение **E** указывает на то, что код состояния представляет ошибку, а **медиумфулл** предоставляет конкретные сведения об ошибке. Многие из значений, которые могут быть возвращены из метода или функции интерфейса, определены в файле Winerror. h.
