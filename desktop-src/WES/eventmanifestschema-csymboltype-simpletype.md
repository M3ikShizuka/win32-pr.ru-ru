# <a name="csymboltype-simple-type-windows-event-log"></a>Простой тип Ксимболтипе (журнал событий Windows)

Определяет допустимое имя символа C/C++.

``` syntax
<xs:simpleType name="CSymbolType">
    <xs:restriction
        base="xs:string"
    >
        <xs:pattern
            value="()|([_a-zA-Z][_0-9a-zA-Z]*)"
         />
    </xs:restriction>
</xs:simpleType>
```

## <a name="patterns"></a>Шаблоны

Простой тип **ксимболтипе** — это **Строка типа xs: String** , которая ограничена следующим шаблоном:

-   `()|([_a-zA-Z][_0-9a-zA-Z]*)`

    Имя символа может быть пустым или содержать буквы, цифры и символы подчеркивания. Если имя пустое, компилятор сообщений создаст имя символа. Если указано имя, имя должно начинаться с символа подчеркивания ( \_ ) или алфавитного знака.

## <a name="remarks"></a>Remarks

Если имя символа пустое, компилятор сообщений использует атрибут **Name** элемента, который вы определяете для создания имени символа. Компилятор заменяет любые символы, отличные от алфавитно-цифровых, и начальные цифры символами подчеркивания. Например, если атрибуту **имени** канала является Microsoft-Windows-Самплепровидер/эксплуатация, компилятор будет использовать Microsoft \_ Windows самплепровидер в \_ \_ качестве имени символа.
