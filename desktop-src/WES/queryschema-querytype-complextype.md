# <a name="querytype-complex-type"></a>Сложный тип QueryType

Определяет набор запросов Selector и суппрессор, которые используются для включения или исключения событий из результирующего набора.

``` syntax
<xs:complexType name="QueryType">
    <xs:choice
        maxOccurs="unbounded"
    >
        <xs:element name="Select">
            <xs:complexType
                mixed="true"
            >
                <xs:attribute name="Path"
                    type="anyURI"
                 />
            </xs:complexType>
        </xs:element>
        <xs:element name="Suppress">
            <xs:complexType
                mixed="true"
            >
                <xs:attribute name="Path"
                    type="anyURI"
                 />
            </xs:complexType>
        </xs:element>
    </xs:choice>
    <xs:attribute name="Id"
        type="long"
        use="optional"
     />
    <xs:attribute name="Path"
        type="anyURI"
        use="optional"
     />
</xs:complexType>
```

## <a name="child-elements"></a>Дочерние элементы



| Элемент                                                    | Тип | Описание                                                                                                                                                                            |
|------------------------------------------------------------|------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**Метьте**](queryschema-select-querytype-element.md)     |      | Запрос XPath, определяющий события, включаемые в результирующий набор запроса. Укажите XPath в текстовом тексте этого элемента. Выражение XPath ограничено 32 выражениями.<br/>   |
| [**Подавление**](queryschema-suppress-querytype-element.md) |      | Запрос XPath, определяющий события, исключаемые из результирующего набора запроса. Укажите XPath в текстовом тексте этого элемента. Выражение XPath ограничено 32 выражениями.<br/> |



## <a name="attributes"></a>Атрибуты



| Имя | Тип   | Описание                                                                                                                                                                                         |
|------|--------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Идентификатор   | long   | Идентификатор, однозначно определяющий этот запрос в списке запросов. Идентификатор отсчитывается от нуля. Если список запросов содержит более одного запроса, необходимо указать идентификатор.<br/> |
| Путь | anyURI | Имя канала или путь к файлу журнала, содержащему события.<br/>                                                                                                            |

## <a name="remarks"></a>Комментарии

Запрос должен содержать по крайней мере одну инструкцию SELECT. Для каждой инструкции "подавлять" должна существовать хотя бы одна инструкция SELECT, указывающая тот же путь. Если запрос SELECT и подавлять возвращает одинаковые события, то приоритет имеет оператор подавлять. При выборе события из нескольких источников события возвращаются в порядке отметок времени. Если используется метка системного времени и интенсивность событий высока, возможно, что несколько событий будут иметь одну и ту же отметку времени. В этом случае порядок событий станет неоднозначным и события могут отображаться неупорядоченно.

Если указать путь для одного из запросов в списке запросов, то все запросы должны указывать путь. Если не указать путь для всех запросов, необходимо указать путь при вызове функции [**евткуери**](/windows/desktop/api/WinEvt/nf-winevt-evtquery) или [**евтсубскрибе**](/windows/desktop/api/WinEvt/nf-winevt-evtsubscribe) .
