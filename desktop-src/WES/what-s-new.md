# <a name="whats-new-windows-event-log"></a>Новые возможности (журнал событий Windows)

На этой странице перечислены новые функции, добавленные в API журнала событий Windows для каждого выпуска.

## <a name="windows-7-and-windows-server-2008-r2"></a>Windows 7 и Windows Server 2008 R2

Ниже приведены изменения, внесенные в схему [евентманифест](eventmanifestschema-schema.md) .

-   Удален сложный тип [**таскевентдефинитионтипе**](eventmanifestschema-taskeventdefinitiontype-complextype.md) . Чтобы обеспечить ту же функциональность, используйте коды операций, относящиеся к задачам (см. элемент **коды операций** в сложном типе [**TaskType**](eventmanifestschema-tasktype-complextype.md) .
-   Добавлены простые типы [**ксимболтипе**](eventmanifestschema-csymboltype-simpletype.md), [**FilePath**](eventmanifestschema-filepath-simpletype.md)и [**стртаблереф**](eventmanifestschema-strtableref-simpletype.md) для ограничения значений, присвоенных атрибутам этих типов.
-   Атрибут **Filters** добавлен в сложный тип [**ProviderType**](eventmanifestschema-providertype-complextype.md) . Поставщики могут использовать фильтры так же, как поставщики используют уровни и ключевые слова, чтобы определить, должны ли они записывать событие.
-   Добавлены следующие типы вывода, которые можно указать для элементов данных, определенных в шаблоне данных события.
    -   Win: Датетимекултуреинсенситиве
    -   Win: HResult
    -   Win: NTSTATUS
-   Типы выходных данных теперь учитываются, пока они не были проигнорированы.

В версию [**компилятора сообщений**](message-compiler--mc-exe-.md) , поставляемого с версией Windows SDK для Windows 7, были внесены следующие изменения:

-   Добавлены аргументы, чтобы компилятор создавал код ведения журнала на основе манифеста. Кроме того, можно запросить создание кода для записи событий в операционные системы до Windows Vista. Список аргументов см. в разделе «аргументы, относящиеся к формированию кода, используемого для регистрации событий» раздела [**компилятора сообщений**](message-compiler--mc-exe-.md) .
-   Теперь компилятор обеспечивает более строгую синтаксическую и семантическую проверку манифеста. Это может привести к тому, что некоторые манифесты, успешно скомпилированные в предыдущих версиях компилятора сообщений, потребовали изменений для успешной компиляции с использованием последней версии.
