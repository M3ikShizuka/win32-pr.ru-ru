# <a name="channelloggingtype-complex-type"></a>Сложный тип Чаннеллоггингтипе

Определяет свойства файла журнала, который осуществляет резервное копирование канала, например его емкость и последовательный или циклический режим.

``` syntax
<xs:complexType name="ChannelLoggingType">
    <xs:sequence
        minOccurs="0"
    >
        <xs:element name="autoBackup"
            type="boolean"
            minOccurs="0"
         />
        <xs:element name="retention"
            type="boolean"
            default="0"
            minOccurs="0"
         />
        <xs:element name="maxSize"
            type="UInt64Type"
            default="1048576"
            minOccurs="0"
         />
        <xs:any
            processContents="lax"
            minOccurs="0"
            maxOccurs="unbounded"
            namespace="##other"
         />
    </xs:sequence>
    <xs:anyAttribute
        processContents="lax"
        namespace="##other"
     />
</xs:complexType>
```

| [**Автоматической архивации**](eventmanifestschema-autobackup-channelloggingtype-element.md) | Логическое                                                           | Определяет, следует ли создавать новый файл журнала, когда текущий файл журнала достигает максимального размера. Параметр [**автоархивации**](eventmanifestschema-autobackup-channelloggingtype-element.md) можно установить в **значение true** , только если параметру [**retention**](eventmanifestschema-retention-channelloggingtype-element.md) присвоено значение **true**. Значение по умолчанию — **false**.<br/> Количество файлов резервных копий, которые может создать служба (ограничивается только доступным местом на диске), не ограничено. Служба выделяет блоки размером 1 МБ, поэтому размер физического размера может дорасти до 1 МБ, превышающий заданный максимальный размер. <br/>                                                                                                                                                                                                                                                      |

## <a name="remarks"></a>Remarks

Для любого типа канала можно указать атрибут **MAXSIZE** .

Атрибут **автоархивации** можно указать только для типов администрирования и операционные каналы.

Для атрибута **retention** можно задать значение false (циклическое ведение журнала) для типов администрирования и операционные каналы. для атрибута **retention** можно задать значение false (циклическое ведение журнала) для типовых каналов аналитики и отладки, но для просмотра событий в Просмотр событий Windows необходимо сначала отключить канал. Обратите внимание, что при повторном включении канала события удаляются из канала.
