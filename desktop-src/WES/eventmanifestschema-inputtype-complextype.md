# <a name="inputtype-complex-type"></a>Сложный тип InputType

Определяет тип входных данных.

``` syntax
<xs:complexType name="InputType">
    <xs:sequence>
        <xs:element name="description"
            type="string"
         />
        <xs:element name="outType"
            type="OutputType"
            minOccurs="0"
            maxOccurs="unbounded"
         />
        <xs:any
            processContents="lax"
            namespace="##other"
         />
    </xs:sequence>
    <xs:attribute name="name"
        type="QName"
        use="required"
     />
    <xs:attribute name="value"
        type="string"
        use="required"
     />
    <xs:attribute name="symbol"
        type="CSymbolType"
        use="required"
     />
</xs:complexType>
```

## <a name="child-elements"></a>Дочерние элементы



| Элемент                                                                  | Тип                                                             | Описание                                                                                                                                                                                                                                                                                                                                                                                                   |
|--------------------------------------------------------------------------|------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**nописание**](eventmanifestschema-description-inputtype-element.md) | строка                                                           | Описание типа входных данных.<br/>                                                                                                                                                                                                                                                                                                                                                                   |
| [**Тип**](eventmanifestschema-outtype-inputtype-element.md)         | [**OutputType**](eventmanifestschema-outputtype-complextype.md) | Список типов выходных данных, которые определяют, как данные подготавливаются к просмотру. Необходимо указать по крайней мере один тип выходных данных. Укажите список, если тип входных данных можно визуализировать как различные типы выходных данных. Например, тип входных данных Win: UInt16 можно визуализировать как короткий без знака, номер порта или шестнадцатеричное число. Один из указанных типов выходных данных должен иметь атрибут default, установленный в значение true.<br/> |
