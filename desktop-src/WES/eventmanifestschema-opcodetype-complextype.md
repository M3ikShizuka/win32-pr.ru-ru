# <a name="opcodetype-complex-type"></a>Сложный тип Опкодетипе

Определяет операцию в компоненте приложения. Используется в сочетании с задачей для задания раздела приложения, в котором регистрируется событие.

``` syntax
<xs:complexType name="OpcodeType"
    mixed="true"
>
    <xs:simpleContent>
        <xs:extension
            base="string"
        >
            <xs:attribute name="name"
                type="QName"
                use="required"
             />
            <xs:attribute name="symbol"
                type="CSymbolType"
                use="optional"
             />
            <xs:attribute name="value"
                type="UInt8Type"
                use="required"
             />
            <xs:attribute name="mofValue"
                type="UInt8Type"
                use="optional"
             />
            <xs:attribute name="message"
                type="strTableRef"
                use="optional"
             />
            <xs:anyAttribute
                processContents="lax"
                namespace="##other"
             />
        </xs:extension>
    </xs:simpleContent>
</xs:complexType>
```

## <a name="attributes"></a>Атрибуты



| Имя     | Тип                                                              | Описание                                                                                                                                                                                                                                                                                                          |
|----------|-------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| message  | [**стртаблереф**](eventmanifestschema-strtableref-simpletype.md) | Локализованное отображаемое имя для кода операции. Строка сообщения ссылается на локализованную строку в разделе " [**STRINGTABLE**](eventmanifestschema-stringtable-resources-element.md) " манифеста. <br/>                                                                                                     |
| мофвалуе | [**UInt8Type**](eventmanifestschema-hexint8type-simpletype.md)   | Зарезервировано только для внутреннего использования.<br/>                                                                                                                                                                                                                                                                           |
| name     | **QName**                                                         | Имя кода операции. Это имя должно быть уникальным в пределах области этого поставщика. <br/>                                                                                                                                                                                                                      |
| символ   | [**ксимболтипе**](eventmanifestschema-csymboltype-simpletype.md) | Символ, используемый для ссылки на код операции в приложении. [**Компилятор сообщений (MC.exe)**](message-compiler--mc-exe-.md) использует символ для создания константы для кода операции в файле заголовка, создаваемого компилятором. Если не указать символ, компилятор создаст его.<br/> |
| значение    | [**UInt8Type**](eventmanifestschema-hexint8type-simpletype.md)   | Значение кода операции. Можно указать значения в диапазоне от 10 до 239. Список предопределенных значений кодов операций см. в разделе Примечания.<br/>                                                                                                                                                                                    |

| Win: начало контроллера домена \_ | 3     | WINEVENT \_ код \_ операции \_ запуска контроллера домена | Событие, представляющее Запуск сбора данных. Это типы событий очистки.                      |
| выигрыш: завершение контроллера домена \_  | 4     | WINEVENT \_ код \_ операции \_ завершения контроллера домена  | Событие, представляющее остановку сбора данных. Это типы событий очистки.                      |

| Win: ответ     | 6     | \_Ответ кода операции WINEVENT \_     | Событие ответа.                                                                                         |
| Win: Resume    | 7     | \_возобновление кода операции WINEVENT \_    | Событие, представляющее возобновление действия после приостановки.                                   |
| Win: Suspend   | 8     | Приостановка WINEVENT \_ кода операции \_   | Событие, представляющее действие, которое приостанавливается в ожидании завершения другого действия.           |
| Win: send      | 9     | \_Отправка кода операции WINEVENT \_      | Событие, представляющее передачу действия в другой компонент.                                   |
