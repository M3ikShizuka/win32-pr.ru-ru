# <a name="eventmanifest-schema"></a>Схема Евентманифест

Схема Евентманифест определяет следующие элементы и типы, которые используются для записи манифеста инструментирования:

-   [Элементы Евентманифестсчема](eventmanifestschema-elements.md)
-   [Простые типы Евентманифестсчема](eventmanifestschema-simple-types.md)
-   [Сложные типы Евентманифестсчема](eventmanifestschema-complex-types.md)

Раздел Elements содержит имена элементов, используемых в манифесте. Однако, чтобы получить подробные сведения для каждого элемента, см. сложный тип, содержащий элемент.

Манифест инструментирования — это XML-файл, определяющий поставщика событий, каналы, на которые записываются события, сами события, условия фильтрации событий, такие как уровни, задачи и коды операций, а также локализованные строки, используемые при отрисовке событий. Соглашение заключается в использовании. Man в качестве расширения для файлов манифеста. Дополнительные сведения о создании манифеста см. в разделе [написание манифеста инструментирования](writing-an-instrumentation-manifest.md).

Windows SDK включает схему в \\ \\ файл include. xsd. Для проверки манифеста можно использовать XSD.

В дополнение к схеме Евентманифест журнал событий Windows также определяет следующие схемы:

-   [Схема событий](eventschema-schema.md)— определяет элементы и типы, используемые для отрисовки события.
-   [Схема запроса](queryschema-schema.md)— определяет элементы и типы, используемые для записи запроса на получение событий из одного или нескольких каналов.
