# <a name="query-schema"></a>Схема запроса

Схема запроса определяет следующие элементы и типы, которые можно использовать для написания структурированного XML-запроса для получения событий из канала или файла журнала:

-   [Элементы Куерисчема](queryschema-elements.md)
-   [Сложные типы Куерисчема](queryschema-complex-types.md)

Раздел Elements содержит имена элементов, используемых в запросе. Однако, чтобы получить подробные сведения для каждого элемента, см. сложный тип, содержащий элемент.

Запрос может содержать одно или несколько выражений XPath, которые используются для включения или исключения событий в результирующем наборе запроса. Можно запрашивать события из нескольких каналов или файлов журналов, но нельзя смешивать каналы и файлы журналов. Можно использовать запрос в любой функции, принимающей XPath (например, функции [**евткуери**](/windows/desktop/api/WinEvt/nf-winevt-evtquery) или [**евтсубскрибе**](/windows/desktop/api/WinEvt/nf-winevt-evtsubscribe) ). Каждый заданный XPath ограничен 32 выражениями. Пример см. в разделе [Использование событий](consuming-events.md).

Windows SDK включает схему в \\ \\ файл включения запроса. xsd.

в дополнение к схеме запроса Windows журнал событий также определяет следующие схемы:

-   [Евентманифест Schema](eventmanifestschema-schema.md)— определяет элементы и типы, используемые для записи манифеста инструментирования.
-   [Схема событий](eventschema-schema.md)— определяет элементы и типы, используемые для отрисовки события.
