# <a name="about-windows-networking"></a>сведения о Windows сети

Приложения могут использовать функции Внет для добавления и отмены сетевых подключений, а также для получения сведений о текущей конфигурации сети.

На следующем рисунке показана структура типичной сети.

![Типичная структура сети](images/csnet-01.png)

на предыдущем рисунке иерархия для ресурсов сервера Windows NT server/Windows 2000 указана подробно как поставщик сети \# 1. Сетевые ресурсы других поставщиков имеют разные иерархические системы. Перед началом работы с сетью приложению не требуются сведения об иерархии. Он может продолжать работу из корневого каталога сети (то есть самого верхнего ресурса контейнера) и получать сведения о ресурсах сети, так как эти сведения необходимы.

Сетевые ресурсы, которые содержат другие ресурсы, называются *контейнерами*. Ресурсы контейнера находятся в прямоугольниках на предыдущем рисунке.

Ресурсы, которые не содержат другие ресурсы, называются *объектами*. На предыдущем рисунке SharePoint \# 1 и SharePoint \# 2 являются объектами. *SharePoint* — это объект, доступный по сети. К примерам SharePoint относятся принтеры и общие каталоги.
