# <a name="transaction-flags"></a>Флаги транзакций

Объект может быть открыт в прямом или транзакционном режиме. При открытии объекта в прямом режиме изменения вносятся немедленно и постоянно. Когда объект открывается в режиме транзакций, изменения помещаются в буфер, чтобы их можно было явным образом зафиксировать или отменить после завершения редактирования. Зафиксированные изменения сохраняются в объекте, в то время как отмененные изменения отбрасываются. Режим доступа по умолчанию используется в режиме Direct.

Режим транзакций не требуется для родительского объекта хранилища, чтобы его можно было использовать во вложенном элементе. Однако транзакция для вложенного элемента вложена в транзакцию для родительского объекта хранилища. Таким образом, изменения, внесенные в дочерний объект, не могут быть зафиксированы, пока они не будут зафиксированы, и оба остаются незафиксированными, пока объект корневого хранилища (родитель верхнего уровня) на самом деле не записывается на диск. Иными словами, изменения перемещаются наружу: внутренние объекты публикуют изменения в транзакциях их непосредственных контейнеров.
