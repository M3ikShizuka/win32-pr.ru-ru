# <a name="storage-and-stream-objects-for-a-property-set"></a>Объекты хранения и потока для набора свойств

Программист указывает, хранится ли набор свойств в хранилище или потоке при создании набора свойств. Непростое \_ значение ПЕРЕЧИСЛЕНИЯ пропсетфлаг, переданное в параметре *грффлагс* в метод [**IPropertySetStorage:: Create**](/windows/desktop/api/Propidl/nf-propidl-ipropertysetstorage-create) , указывает на это. Настройка, где хранится набор свойств, предоставляет необходимые элементы управления приложениями для полного взаимодействия через интерфейс [**IPropertySetStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage) с набором свойств COM.

Если \_ установлен непростой флаг пропсетфлаг, набор свойств хранится в объекте хранилища, и в него могут быть записаны непростые значения свойств. Непростые значения включают значения с объектом **VarType** хранилища VT \_ , потоком VT \_ , \_ хранящимся \_ объектом VT или \_ потокового объекта VT \_ . Дополнительные сведения о значениях **VarType** и способах их использования см. в разделе Структура [**пропвариант**](/windows/win32/api/propidlbase/ns-propidlbase-propvariant) .

Если \_ непростой флаг пропсетфлаг не задан, в набор свойств могут быть записаны только простые значения.

В объекте хранилища непростого набора свойств поток создается с именованным содержимым. Это основной поток набора свойств, содержащий все простые значения свойств. Непростые значения свойств (потоки и хранилища) хранятся в основном объекте хранилища, заданного свойством как подпотоки и хранилища. То есть эти непростые значения хранятся как элементы того же уровня в потоке содержимого. Имя параллельных потоков и хранилищ определяется реализацией и сохраняется в потоке содержимого аналогично тому, как хранится простое строковое свойство.
