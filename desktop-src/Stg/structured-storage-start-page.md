# <a name="structured-storage"></a>структурированные служба хранилища

## <a name="purpose"></a>Назначение

структурированные служба хранилища обеспечивают сохраняемость файлов и данных в COM путем обработки одного файла в виде структурированной коллекции объектов, называемых хранилищами и потоками.

цель структурированного служба хранилища заключается в снижении потерь производительности и издержек, связанных с хранением отдельных объектов в одном файле. структурированный служба хранилища предоставляет решение, определяя способ управления единственной объектной сущностью в качестве структурированной коллекции двух типов хранилищ объектов и потоков с помощью стандартной реализации, называемой составными файлами. Это позволяет пользователю взаимодействовать с составным файлом, а также управлять им, как если бы он был одним файлом, а не с вложенной иерархией отдельных объектов.

## <a name="where-applicable"></a>Где применимо

структурированные служба хранилища можно использовать в операционных системах на основе Microsoft COM.

## <a name="developer-audience"></a>Аудитория разработчиков

структурированная документация служба хранилища предназначена для опытных программистов C и C++ и разработчиков систем на основе COM.

структурированные служба хранилища в основном поддерживают языки программирования C и C++, однако любая технология на основе COM также будет поддерживать любой язык программирования, использующий указатели интерфейса.

основательное понимание технологий COM является необходимым условием для разработки структурированных служба хранилища.
