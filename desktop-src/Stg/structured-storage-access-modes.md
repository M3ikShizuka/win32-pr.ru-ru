# <a name="structured-storage-access-modes"></a>Структурированные режимы доступа к хранилищу

Для управления одновременным доступом к объекту по нескольким процессам и пользователям необходимы механизмы. COM предоставляет эти механизмы, определяя режимы доступа для объектов хранилища и потоков. Режим доступа, указанный для родительского объекта хранилища, наследуется его дочерними элементами, хотя можно полагать дополнительные ограничения на дочернее хранилище или поток. Вложенный объект хранилища или поток может быть открыт в одном и том же режиме или в более ограниченном режиме, чем его родительский элемент, но не может быть открыт в режиме с меньшим уровнем ограничения, чем его родительский элемент.

Режимы доступа задаются с помощью значений, перечисленных в [**константах стгм**](stgm-constants.md). Эти значения служат флагами для передачи в качестве аргументов в методы в интерфейсе [**IStorage**](/windows/desktop/api/Objidl/nn-objidl-istorage) и связанных функциях API. Как правило, несколько флагов объединяются в параметре *грфмоде* с использованием логической операции **или** .

Флаги делятся на шесть групп. В каждый момент времени можно указать только один флаг из каждой группы:

-   [Флаги транзакций](transaction-flags.md)
-   [Флаги создания хранилища](storage-creation-flags.md)
-   [Временные флаги создания](temporary-creation-flags.md)
-   [Флаги приоритета](priority-flags.md)
-   [Флаги разрешений доступа](access-permission-flags.md)
-   [Флаги общего доступа](shared-access-flags.md)
