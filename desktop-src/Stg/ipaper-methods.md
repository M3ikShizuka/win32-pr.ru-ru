# <a name="ipaper-methods"></a>Методы Ипапер

**Стосерве** предоставляет объекты, которые управляются главным образом с помощью собственного интерфейса **ипапер** .

В следующей таблице перечислены методы **ипапер** из ипапер. H в каталоге с родителем \\ Inc.



| Метод    | Описание                                                         |
|-----------|---------------------------------------------------------------------|
| инитпапер | Инициализирует объект бумаги и создает массив рукописных данных.                 |
| Блокировка      | Обеспечивает клиентский контроль над бумагой и блокирует другие клиенты.      |
| Unlock    | Освобождает клиентский контроль над документом.                           |
| Загрузить      | Загрузка бумажного содержимого из составного файла клиента и уведомление приемников. |
| Сохранить      | Сохраняет содержимое бумаги в составной файл клиента.                      |
| инкстарт  | Начинает рисовать цветовую краску на бумаге.                      |
| инкдрав   | Помещает точки данных рукописного ввода в электронную область бумаги.               |
| инкстоп   | Останавливает рисование рукописного ввода на бумаге.                             |
| Очистка     | Очистка текущего содержимого бумаги и уведомление приемников.                 |
| Изменить размер    | Изменяет размер прямоугольника рисунка бумаги и уведомляет приемники.        |
| Отрисовать    | Перерисовывает содержимое объекта Paper и уведомляет приемники.                |



 

Методы, представляющие интерес для этого примера кода в составных файлах, — это [**Загрузка**](ipaper--load.md), [**Сохранение**](ipaper--save.md)и [**перерисовка**](ipaper--redraw.md).

[**Инкстарт**](inkstart-method.md), [**инкдрав**](inkdraw-method.md)и [**инкстоп**](cguipaper-methods.md) — это методы, используемые клиентами для создания командных чертежей для записи последовательностей рукописного ввода. Клиент, как правило, реагирует на \_ сообщение WM лбуттондовн как начало последовательности рукописного ввода, вызывая **инкстарт** в содокументе. Когда пользователь перемещает мышь или перо для рисования при удерживании левой кнопки, клиент будет отвечать на повторяющиеся \_ сообщения WM MOUSEMOVE с соответствующими вызовами **инкдрав**. Когда пользователь отпускает левую кнопку мыши, клиент будет отвечать на \_ сообщение WM лбуттонуп с вызовом **инкстоп**, который отмечает конец последовательности рукописного ввода.

[**Инкстарт**](inkstart-method.md) указывает, что Начальная высота последовательности отрисовки в координатах окна клиента. Он также передает цвет и ширину выбранного в данный момент пера. Клиент поддерживает эти варианты выбора. Собумага просто записывает их при вызове **инкстарт** . [**Инкдрав**](inkdraw-method.md) вызывается многократно, чтобы дать сомнение об успешности координат окна, представляющих движение мыши или пера. [**Инкстоп**](cguipaper-methods.md) инструктирует отметку конца документа.
