# <a name="about-structured-storage"></a>Сведения о структурированном хранилище

Традиционные файловые системы сталкиваются с проблемами при попытке сохранить эффективное несколько типов объектов в одном документе. COM предоставляет решение: файловая система в файле. Структурированное хранилище COM определяет способ обработки одной файловой сущности в виде структурированной коллекции двух типов объектов — хранилищ и потоков, которые выполняются как каталоги и файлы. Эта схема называется *структурированным хранилищем*. Структурированное хранилище предназначено для снижения потерь производительности и издержек, связанных с хранением отдельных объектов в неструктурированном файле.

В этом разделе приводятся общие сведения о преимуществах и принципах структурированного хранилища, наборах свойств и асинхронном структурированном хранилище, а также исторического взгляда на файловые системы:

-   Реализация составного файла
-   Реализация файловой системы NTFS
-   Изолированная реализация
