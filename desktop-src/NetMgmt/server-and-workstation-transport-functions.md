# <a name="server-and-workstation-transport-functions"></a>Транспортные функции сервера и рабочей станции

Функции транспорта сервера управления сетью и рабочей станции обрабатывали привязку и отмену привязки транспортных протоколов к серверу и перенаправитель. Серверные функции транспорта работают с транспортными протоколами, которыми управляет сервер. функции транспорта рабочей станции работают с транспортными протоколами, управляемыми перенаправитель.

Общий доступ к файлам между транспортным устройством и сервером включает два компонента:

-   Серверный компьютер, на котором находятся файлы
-   Клиент SMB, обращающийся к файлам;

Клиентский компьютер обменивается данными с сервером по локальной сети с помощью транспортного протокола; Например, TCP или КСНС. Клиент отправляет запросы на сервер для получения данных. Программное обеспечение на клиентском компьютере, создающем запросы файлов, называется *перенаправитель* , так как он перенаправляет запросы локальных файлов на серверный компьютер. Программное обеспечение на компьютере, которое получает и обрабатывает запросы файлов, называется *сервером* , так как обслуживает клиентов. Формат, характерный для этих запросов, называется протоколом SMB.

Ниже перечислены функции транспортного сервера.

|--------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|

| [**нетсерверкомпутернамедел**](/windows/desktop/api/Lmserver/nf-lmserver-netservercomputernamedel) | Отключает каждый сетевой транспортный протокол из имени эмулированного сервера, установленного предыдущим вызовом функции **нетсерверкомпутернамеадд** .                                                                                                                                                                               |
| [**нетсервертранспортадд**](/windows/desktop/api/Lmserver/nf-lmserver-netservertransportadd)       | Привязывает указанный сервер к транспортному протоколу. (Эта функция поддерживает только уровень информации о [**\_ транспортном сервере \_ \_ 0**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_0) .)                                                                                                                                                |
| [**нетсервертранспортаддекс**](/windows/desktop/api/Lmserver/nf-lmserver-netservertransportaddex)   | Привязывает указанный сервер к транспортному протоколу. (Эта расширенная функция поддерживает уровни информации о транспортном [**сервере \_ \_ \_ 1**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_1), [**сервере \_ \_ \_ 2**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_2)и [**\_ транспортном сервере \_ \_ 3**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_3) .) |
| [**нетсервертранспортдел**](/windows/desktop/api/Lmserver/nf-lmserver-netservertransportdel)       | Отключает транспортный протокол от сервера.                                                                                                                                                                                                                                                                         |
| [**нетсервертранспортенум**](/windows/desktop/api/Lmserver/nf-lmserver-netservertransportenum)     | Перечисляет транспортные протоколы, управляемые сервером.                                                                                                                                                                                                                                                                   |



 

Транспортные функции сервера доступны на следующих уровнях информации:

-   [**\_Сведения о транспортном сервере \_ \_ 0**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_0)
-   [**\_Сведения о транспортном сервере \_ \_ 1**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_1)
-   [**\_Сведения о транспортном сервере \_ \_ 2**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_2)
-   [**\_Сведения о транспортном сервере \_ \_ 3**](/windows/desktop/api/Lmserver/ns-lmserver-server_transport_info_3)

Функции транспорта рабочей станции выполняют аналогичные операции с рабочей станцией.

**Windows Server 2003 и Windows XP/2000:** Перенаправитель не поддерживает функцию [**нетвкстатранспортадд**](/windows/desktop/api/lmwksta/nf-lmwksta-netwkstatransportadd) или функцию [**нетвкстатранспортдел**](/windows/desktop/api/lmwksta/nf-lmwksta-netwkstatransportdel) . Параметры по умолчанию для транспортных протоколов можно изменить вручную с помощью диалогового окна **Свойства подключения к локальной** сети в папке **сеть и удаленный доступ** .

Ниже перечислены функции транспорта рабочей станции.



| Функция                                               | Описание                                                       |
|--------------------------------------------------------|-------------------------------------------------------------------|
| [**нетвкстатранспортадд**](/windows/desktop/api/lmwksta/nf-lmwksta-netwkstatransportadd)   | Подключает перенаправитель к транспортному протоколу.                |
| [**нетвкстатранспортдел**](/windows/desktop/api/lmwksta/nf-lmwksta-netwkstatransportdel)   | Отключает транспортный протокол от перенаправителя.           |
| [**нетвкстатранспортенум**](/windows/desktop/api/Lmwksta/nf-lmwksta-netwkstatransportenum) | Список транспортных протоколов, управляемых перенаправитель. |



 

Функции транспорта рабочей станции доступны на одном уровне информации:

-   [**\_ \_ Сведения о транспорте вкста \_ 0**](/windows/desktop/api/Lmwksta/ns-lmwksta-wksta_transport_info_0)
