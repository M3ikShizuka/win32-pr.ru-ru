# <a name="user-modal-functions"></a>Пользовательские модальные функции

Модальные функции пользователя управления сетью получают и устанавливают параметры на уровне системы, связанные с поведением системы безопасности.

Модальные пользовательские функции перечислены ниже.



| Функция                                     | Описание                                                                                                                                                                                             |
|----------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**нетусермодалсжет**](/windows/desktop/api/Lmaccess/nf-lmaccess-netusermodalsget) | Возвращает глобальные сведения для всех пользователей и глобальных групп в базе данных безопасности, которая является базой данных диспетчера учетных записей безопасности (SAM) или, в случае контроллеров домена, Active Directory. |
| [**нетусермодалссет**](/windows/desktop/api/Lmaccess/nf-lmaccess-netusermodalsset) | Задает глобальные сведения для всех пользователей и глобальных групп в базе данных безопасности.                                                                                                                       |



 

Функции **нетусермодалсжет** и **нетусермодалссет** проверяют и изменяют модальные параметры, которые являются глобальными параметрами, влияющими на каждую учетную запись в базе данных безопасности (например, минимально допустимую длину пароля). Все модальные параметры можно изменить путем вызова **нетусермодалссет**. Большинство модальных элементов также можно изменить с помощью команды **net accounts** . Для модальных функций пользователя управления сетью не требуется, чтобы сервер имел безопасность на уровне пользователя.

Модальные данные пользователя доступны на следующих уровнях:

-   [**\_Сведения о модальных пользователях \_ \_ 0**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_0)
-   [**\_Сведения о модальных пользователях \_ \_ 1**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1)
-   [**\_Сведения о модальных пользователях \_ \_ 2**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_2)
-   [**\_Сведения о модальных пользователях \_ \_ 3**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_3)

Следующие уровни информации допустимы только для [**нетусермодалссет**](/windows/desktop/api/Lmaccess/nf-lmaccess-netusermodalsset) и замены старого способа передачи в *пармнум* для задания определенного поля:

-   [**\_Сведения о модальных пользователях \_ \_ 1001**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1001)
-   [**\_Сведения о модальных пользователях \_ \_ 1002**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1002)
-   [**\_Сведения о модальных пользователях \_ \_ 1003**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1003)
-   [**\_Сведения о модальных пользователях \_ \_ 1004**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1004)
-   [**\_Сведения о модальных пользователях \_ \_ 1005**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1005)
-   [**\_Сведения о модальных пользователях \_ \_ 1006**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1006)
-   [**\_Сведения о модальных пользователях \_ \_ 1007**](/windows/desktop/api/Lmaccess/ns-lmaccess-user_modals_info_1007)

При программировании для Active Directory можно вызвать определенные методы интерфейса Active Directory (ADSI) для достижения тех же функций, которые можно достичь, вызвав модальные функции пользователя управления сетью. Дополнительные сведения см. в разделе [**иадсдомаин**](/windows/desktop/api/iads/nn-iads-iadsdomain).
