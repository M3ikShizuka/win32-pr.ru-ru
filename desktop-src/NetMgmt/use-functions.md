# <a name="use-functions"></a>Использование функций

Функции управления сетью используются для проверки подключений между рабочими станциями и серверами и управления ими (использования). Функции use перечислены ниже.



| Функция                               | Описание                                                                               |
|----------------------------------------|-------------------------------------------------------------------------------------------|
| [**нетусеадд**](/windows/desktop/api/Lmuse/nf-lmuse-netuseadd)         | Создает подключение между локальным компьютером и сервером.                               |
| [**нетуседел**](/windows/desktop/api/Lmuse/nf-lmuse-netusedel)         | Завершает подключение к общему ресурсу.                                                   |
| [**нетусинум**](/windows/desktop/api/Lmuse/nf-lmuse-netuseenum)       | Список всех текущих подключений между локальным компьютером и ресурсами на удаленных серверах. |
| [**нетусежетинфо**](/windows/desktop/api/Lmuse/nf-lmuse-netusegetinfo) | Возвращает сведения о подключении к общему ресурсу.                              |



 

Эти функции применяются только к клиенту блока сообщений сервера (Рабочая станция LAN Manager). Функция **нетусежетинфо** не поддерживает общие папки распределенная ФАЙЛОВАЯ система (DFS). Чтобы получить сведения о подключении для общего ресурса с помощью другого поставщика сети (например, WebDAV или общей папки DFS), используйте функцию [**внетжетконнектион**](/windows/desktop/api/winnetwk/nf-winnetwk-wnetgetconnectiona) .

Соединения отличаются от сеансов. *сеанс* устанавливается впервые, когда Рабочая станция устанавливает подключение к общему ресурсу на сервере. Все дополнительные подключения между рабочей станцией и сервером находятся в одном сеансе, пока сеанс не завершится. Можно установить два типа подключений: подключения устройств (которые могут быть только явными) и подключения UNC (которые могут быть явными или неявными).

*Соединения* устанавливаются отдельно для каждого пользователя. Соединение, сделанное пользователем, удаляется при выходе пользователя из системы. По этой причине функции управления сетью используются только локально, так как подключение, настроенное удаленным пользователем, будет недоступно для других пользователей, даже если пользователь был интерактивно вошел в систему этого компьютера.

Функция [**нетусеадд**](/windows/desktop/api/Lmuse/nf-lmuse-netuseadd) устанавливает явное соединение между локальным компьютером и ресурсом, общим на сервере, перенаправляя имя локального устройства в сетевое имя ресурса удаленного сервера ( \\ \\ *ServerName* \\ *ShareName*). После подключения к имени устройства пользователи или приложения могут использовать удаленный ресурс, указав имя локального устройства.

Неявные подключения UNC выполняются функцией, которая отвечает за подключение. Чтобы установить неявное подключение UNC, приложение передает имя общего ресурса в любую функцию, которая принимает пути в формате UNC. Функция принимает имя UNC и устанавливает подключение к указанному имени общего ресурса. Для всех последующих запросов к этому подключению требуется полное имя общего ресурса.

Функции use доступны на следующих уровнях информации:

-   [**ИСПОЛЬЗОВАТЬ \_ сведения \_ 0**](/windows/desktop/api/Lmuse/ns-lmuse-use_info_0)
-   [**ИСПОЛЬЗОВАТЬ \_ сведения \_ 1**](/windows/desktop/api/Lmuse/ns-lmuse-use_info_1)
-   [**ИСПОЛЬЗОВАТЬ \_ сведения \_ 2**](/windows/desktop/api/Lmuse/ns-lmuse-use_info_2)
