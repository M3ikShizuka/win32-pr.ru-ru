# <a name="network-management-function-buffers"></a>Буферы функций управления сетью

Библиотека времени выполнения RPC обрабатывает буферы, необходимые для функций управления сетью получения данных 32-bit, следующим образом.

-   **Отправка данных на сервер** (данные, указанные \[ в \] параметре in).

    Вызывающий объект должен выделить буфер и освободить его для соответствующей структуры информации (или структур) и передать в функцию переменную указателя. Вызывающему объекту не нужно указывать длину буфера.

    Пример: [ **нетграупадд**](/windows/desktop/api/Lmaccess/nf-lmaccess-netgroupadd)

-   **Получение данных с сервера** (данные, указанные в \[ \] параметрах out).

    Система выделяет буфер для возвращаемых сведений. Вызывающая сторона должна передать переменную указателя в функцию на входе. При успешном возвращении указатель получает адрес выделенного системой буфера, который содержит возвращенные сведения. Это упрощает вызывающий код, поскольку вызывающему объекту не нужно оценивать размер буфера или изменять размер буфера и повторно выдавать функцию.

    Когда вызывающий объект завершил обработку возвращенных данных, он должен освободить память, выделенную системой, вызвав функцию [**нетапибуфферфри**](/windows/desktop/api/Lmapibuf/nf-lmapibuf-netapibufferfree) . Дополнительные сведения об указании размеров буфера см. в разделе [Длина буфера функции управления сетью](network-management-function-buffer-lengths.md).

    Пример: [ **нетграупенум**](/windows/desktop/api/Lmaccess/nf-lmaccess-netgroupenum)
