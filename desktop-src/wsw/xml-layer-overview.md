# <a name="xml-layer-overview"></a>Обзор уровня XML

API XML в ВВСАПИ основан на [модуле чтения XML](xml-reader.md) и объектах [модуля записи XML](xml-writer.md) , которые позволяют выполнять чтение или запись XML-документов только в прямом направлении. XML-слой предоставляет приложению полный доступ к содержимому сообщений и управление им.

## <a name="encoding"></a>Кодирование

API XML поддерживает документы, закодированные как:

-   Text (UTF-8, UTF-16LE, UTF-16BE)
-   Двоичные данные
-   MTOM

## <a name="storage"></a>Память

API XML поддерживает обработку документов, хранящихся как:

-   Буфер в памяти закодированных байтов
-   Поток
-   [XML-буфер](xml-buffer.md)

[XML-буфер](xml-buffer.md) — это структурированное представление XML-документа в памяти. Это более эффективное представление, чем документ, закодированный как байты. Документ XML, хранящийся в XML-буфере, может быть перемещен, прочитан или записан.

## <a name="io"></a>Ввод-вывод

API XML никогда не будет выполнять операции ввода-вывода, если только они не запрошены специально. Кроме того, любые операции ввода-вывода могут инициироваться асинхронно. Дополнительные сведения об асинхронной обработке с помощью XML API см. в разделе [**всфиллреадер**](/windows/desktop/api/WebServices/nf-webservices-wsfillreader) и [**всфлушвритер**](/windows/desktop/api/WebServices/nf-webservices-wsflushwriter) .

## <a name="processing"></a>Обработка

XML API имеет три разных уровня, на которых документ может быть обработан.

Документ может обрабатываться [**узлом**](/windows/desktop/api/WebServices/ns-webservices-ws_xml_node) за раз. Это обеспечивает наиболее детализированную обработку XML-содержимого и обеспечивает полную достоверность данных из документа. На этом уровне будут использоваться функции [**всреадноде**](/windows/desktop/api/WebServices/nf-webservices-wsreadnode) и [**всвритеноде**](/windows/desktop/api/WebServices/nf-webservices-wswritenode) и [**вскопиноде**](/windows/desktop/api/WebServices/nf-webservices-wscopynode) .

Следующий уровень управления — это интерфейсы API, такие как [**всреадстартелемент**](/windows/desktop/api/WebServices/nf-webservices-wsreadstartelement), [**всреадвалуе**](/windows/desktop/api/WebServices/nf-webservices-wsreadvalue) и [**всреаденделемент**](/windows/desktop/api/WebServices/nf-webservices-wsreadendelement). Эти API предоставляют множество видов проверки, пропускают пробелы и комментарии, нормализацию текста и CDATA, чтобы предоставить потребителю более простое представление XML.

Самым высоким уровнем управления является использование API сериализации. Эти API-интерфейсы основываются на сопоставлении между типами данных C и XML, а также могут считывать или записывать сложную структуру в памяти в XML и обратно с помощью одной функции, такой как [**всвритилемент**](/windows/desktop/api/WebServices/nf-webservices-wswriteelement) и [**всреаделемент**](/windows/desktop/api/WebServices/nf-webservices-wsreadelement).

API-интерфейсы канонизации XML могут использоваться для создания канонической формы XML, которая, в свою очередь, может использоваться для создания криптографических подписей по содержимому XML.

## <a name="creating-a-writer"></a>Создание модуля записи

Создание и использование модуля записи для записи в буфер в памяти:

``` syntax
WsCreateWriter              // Create an instance of a WS_XML_WRITER
// Initialize a WS_XML_WRITER_BUFFER_OUTPUT
WsSetOutput                 // Set the encoding and output of the writer along with any other writer properties
// Write Elements
WsGetWriterProperty(..., WS_XML_WRITER_PROPERTY_BYTES, ...)  // Get the generated bytes as a single byte array
// Use the generated bytes
WsFreeWriter                // Free the writer
```

Создание и использование модуля записи для записи в поток:

``` syntax
WsCreateWriter              // Create an instance of a WS_XML_WRITER
// Initialize a WS_XML_WRITER_STREAM_OUTPUT
WsSetOutput                 // Set the encoding and output of the writer along with any other writer properties
// Write Elements
WsFlushWriter               // Force any buffered data to be written
WsFreeWriter                // Free the writer
```

Создание и использование модуля записи для записи в [ \_ \_ буфер WS XML](ws-xml-buffer.md):

``` syntax
WsCreateXmlBuffer           // Create the buffer to write to
WsCreateWriter              // Create an instance of a WS_XML_WRITER
WsSetOutputToBuffer         // Set the output buffer along with any other writer properties
// Write Elements
WsFreeWriter                // Free the writer
// The buffer has the generated document
```

Во всех случаях для форматирования XML-кода может быть задано значение свойства « [**\_ \_ модуль записи \_ \_ XML**](/windows/desktop/api/WebServices/ne-webservices-ws_xml_writer_property_id) ».

## <a name="writing-elements"></a>Запись элементов

Запись элемента в модуль записи:

``` syntax
WsWriteStartElement          // Write a start element
for each attribute
{
// Write an attribute using either
WsWriteStartAttribute    // Write a start attribute
// Write Content
WsWriteEndAttribute      // Write an end attribute
// Or one of the following
WsWriteXmlnsAttribute    // Write an explicit xmlns attribute
}
// Write Elements or Content
WsWriteEndElement
```

Также можно использовать следующие действия:

``` syntax
WsWriteArray                 // Write an array of primitive values as a series of repeated elements
```

## <a name="writing-content"></a>Запись содержимого

Чтобы записать содержимое в элемент или атрибут, можно использовать следующее:

``` syntax
WsWriteChars                 // Write unicode characters from memory
WsWriteCharsUtf8             // Write UTF-8 encoded characters from memory
WsWriteBytes                 // Write binary data encoded as base64
WsPushBytes                  // Direct the writer to request that bytes be written
WsPullBytes                  // Direct the writer to read the bytes to be written
WsWriteValue                 // Write primitive values such as ints and BOOLs
WsWriteText                  // Write an WS_XML_TEXT
WsWriteQualifiedName         // Write a qualified name
```

Следующие возможности могут быть использованы для записи в документ, но не могут использоваться в атрибуте.

``` syntax
WsWriteNode                  // Write a single WS_XML_NODE
WsCopyNode                   // Copy a single node, or an entire WS_XML_ELEMENT_NODE and children from an WS_XML_READER
```

Для записи раздела CDATA в текстовый документ можно использовать следующую команду:

``` syntax
WsWriteStartCData            // Start a CDATA section in a text encoding
// Write Content
WsWriteEndCData              // End a CDATA section in text encoding
```

## <a name="creating-a-reader"></a>Создание читателя

Создание и использование считывателя для чтения из буфера в памяти:

``` syntax
WsCreateReader              // Create an instance of a WS_XML_READER
// Initialize a WS_XML_READER_BUFFER_INPUT
WsSetInput                  // Set the encoding and input of the reader along with any other reader properties
// Read Elements
WsFreeReader                // Free the reader
```

Создание и использование читателя для чтения из потока:

``` syntax
WsCreateReader              // Create an instance of a WS_XML_READER
// Initialize a WS_XML_READER_STREAM_INPUT
WsSetInput                  // Set the encoding and input of the reader along with any other reader properties
WsFillReader                // Populate the reader with data from the underlying stream
// Read Elements
WsFreeReader                // Free the reader
```

Создание и использование считывателя для чтения из [ \_ XML- \_ буфера WS](ws-xml-buffer.md):

``` syntax
WsCreateXmlBuffer           // Create the buffer to write to
WsCreateReader              // Create an instance of a WS_XML_READER
WsSetInputToBuffer          // Set the input buffer along with any other reader properties
// Read Elements
WsFreeReader                // Free the reader
```

## <a name="reading-elements"></a>Чтение элементов

Чтение элемента из средства чтения:

``` syntax
WsReadToStartElement         // Skip whitespace and comments to position the reader on a specific element
for each attribute of interest
{
WsFindAttribute          // Try Locate the attribute
if (found)
{
WsReadStartAttribute // Set the reader to read the attribute
// Read Content
WsReadEndAttribute   // Return the reader to the element
}
}
WsReadStartElement           // Advance the reader past the current element
// Read Elements or Content
WsWriteEndElement            // Advance the reader past the corresponding end element
```

Также можно использовать следующие действия:

``` syntax
WsReadArray                  // Read an array of primitive values as a series of repeated elements
```

## <a name="reading-content"></a>Чтение содержимого

Для чтения содержимого из элемента или атрибута можно использовать следующее:

``` syntax
WsReadChars                 // Read characters to memory as unicode
WsReadCharsUtf8             // Read characters to memory encoded as UTF-8
WsReadBytes                 // Read binary data encoded as base64
WsReadValue                 // Read primitive values such as ints and BOOLs
WsReadQualifiedName         // Read a qualified name
```

Для проверки текущего узла, на котором расположен модуль чтения, можно использовать следующую команду:

``` syntax
WsGetReaderNode             // Get the current node
```

## <a name="using-a-buffer"></a>Использование буфера

При записи в [ \_ XML- \_ буфер WS](ws-xml-buffer.md) можно использовать следующее:

``` syntax
WsGetWriterPosition          // Get the current position of the writer in the document
WsSetWriterPosition          // Set the current position of the writer in the document
WsMoveWriter                 // Move relative to the current position in the document
WsRemoveNode                 // Delete an element or text from a document
```

При чтении из [ \_ XML- \_ буфера WS](ws-xml-buffer.md) можно использовать следующее:

``` syntax
WsGetReaderPosition          // Get the current position of the reader in the document
WsSetReaderPosition          // Set the current position of the reader in the document
WsMoveReader                 // Move relative to the current position in the document
```

Для изменения [ \_ \_ буфера WS XML](ws-xml-buffer.md)можно использовать следующую команду:

``` syntax

WsRemoveNode                 // Delete an element or text from a document
```
