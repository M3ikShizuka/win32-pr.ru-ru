# <a name="implementing-a-device-provider"></a>Реализация поставщика устройства

Чтобы реализовать поставщик устройства, создайте объект, который предоставляет интерфейс [**иупнпдевицепровидер**](/windows/desktop/api/Upnphost/nn-upnphost-iupnpdeviceprovider) . Этот объект должен быть зарегистрирован на узле устройства с помощью метода [**иупнпрегистрар:: регистердевицепровидер**](/windows/desktop/api/Upnphost/nf-upnphost-iupnpregistrar-registerdeviceprovider) . Этот метод принимает следующие параметры:

-   Имя поставщика, которое должно быть уникальным на компьютере.
-   Идентификатор ProgID класса, реализующего поставщик устройства.
-   Строка инициализации, которая передается поставщику устройства при запуске.
-   Идентификатор контейнера. Идентификатор контейнера — это строка, идентифицирующая группу, к которой принадлежит устройство. Все устройства с одинаковым идентификатором контейнера размещаются в одном процессе.
