# <a name="device-host-security-considerations"></a>Вопросы безопасности узла устройства

Использование узла устройства создает проблемы безопасности из-за следующих причин:

-   устройства, размещенные на компьютере под управлением Windows XP, передают объявления во всех сетях.
-   устройства, размещенные на компьютере под управлением Windows XP, позволяют управлять устройствами из всех сетей.

это повышает риск для домашних потребителей, так как такие устройства, как проигрыватель мультимедиа или система с мостом или кондиционирование, размещенные на компьютере под управлением Windows XP, видимы и могут управляться из контрольных точек за пределами дома.

При создании размещенного устройства необходимо принять во внимание некоторые проблемы безопасности.

-   Чтобы сократить область обнаружения и атаки устройств на основе UPnP, срок жизни сообщений SSDP составляет 1. Это означает, что зарегистрированное устройство обнаруживается только контрольными точками в той же сети. В реестре можно настроить более высокий срок жизни.
-   Для регистрации неработающего устройства необходимо предварительно зарегистрировать устройство .dll с COM, для которого требуются права администратора.
-   Для регистрации работающего устройства требуется привилегия администратора, локальной службы или локальной системы.
-   При запуске узел устройства запускается как [LocalService](/windows/desktop/Services/localservice-account). Это дает устройству возможность создавать аудиты и читать раздел реестра **hKey \_ Local \_ Machine** . Устройство имеет доступ к **hKey \_ текущего \_ пользователя**. Учетная запись LocalService может использовать ресурсы, к которым локальная служба предоставляет доступ, а также те, которые предоставляют доступ к Аусентикатедусер. Устройство имеет ограниченный доступ к файловой системе.
-   Списки управления доступом файловой системы должны быть обновлены, чтобы разрешить доступ к каталогу ресурсов для [локальной](/windows/desktop/Services/localservice-account) службы.
-   Если устройство должно иметь более безопасный доступ, можно создать собственный процесс для устройства и зарегистрировать его с помощью [**иупнпрегистрар:: регистерруннингдевице**](/windows/desktop/api/Upnphost/nf-upnphost-iupnpregistrar-registerrunningdevice).
