# <a name="what-are-the-basic-actions-of-the-control-point-api"></a>Каковы основные действия API контрольной точки?

Все контрольные точки выполняют следующие задачи:

-   Поиск доступных устройств на основе UPnP в сети.
-   Определите, какие из этих устройств представляют интерес.
-   Выдача запросов на управление к интересующим устройствам и получение событий, создаваемых устройствами.

Приложение должно сначала найти доступные устройства на основе UPnP в сети, выполнив поиск. Поскольку условия поиска, определенные в архитектуре UPnP, довольно обширны, приложение может найти больше устройств, чем требуется. Поэтому приложение должно проверить характеристики обнаруженных устройств, чтобы определить, какие из них наиболее полно соответствуют критериям поиска. После этого приложение может выдавать управляющие запросы к этим устройствам.

В следующих разделах описывается, как выполнять эти операции с помощью API контрольной точки с технологией UPnP.

-   [Поиск устройств](finding-devices.md)
-   [Описание устройств](describing-devices.md)
-   [Управление устройствами](controlling-devices.md)
