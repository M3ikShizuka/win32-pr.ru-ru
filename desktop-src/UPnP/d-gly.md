# <a name="d-upnp-apis"></a>D (API-интерфейсы UPnP)

Содержит термины, связанные с UPnP, которые начинаются с буквы D.

[A](a-gly.md) B [C](c-gly.md) D [E](e-gly.md) F G [р](h-gly.md) . J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.d_1_gly"></span><span id="UPNP.D_1_GLY"></span>**модем**
</dt> <dd>

Устройство — это контейнер других служб или устройств. Устройство может содержать другие логические устройства.

</dd> <dt>

<span id="upnp.d_2_gly"></span><span id="UPNP.D_2_GLY"></span>**объект управления устройством**
</dt> <dd>

Объект управления устройством является центральной точкой управления для объектов службы устройства. При регистрации устройства его объект управления устройством передается на узел устройства.

</dd> <dt>

<span id="upnp.d_3_gly"></span><span id="UPNP.D_3_GLY"></span>**Описание устройства**
</dt> <dd>

Описание устройства — это список универсальных свойств всех устройств на физическом устройстве, включая службы, иерархию устройства и его свойства.

</dd> <dt>

<span id="upnp.d_4_gly"></span><span id="UPNP.D_4_GLY"></span>**узел устройства**
</dt> <dd>

Узел устройства обрабатывает части функций обнаружения, описания, управления и работы с событиями устройства. Устройства содержат код для обработки функций конкретного устройства, таких как Перемотка ВИДЕОМАГНИТОФОНА.

</dd> <dt>

<span id="upnp.d_5_gly"></span><span id="UPNP.D_5_GLY"></span>**Поставщик устройства**
</dt> <dd>

Поставщик устройств — это сущность, зарегистрированная на узле устройства. Этот поставщик устройств не опубликован в сети. Однако поставщик устройства регистрирует другие устройства и публикует их в сети.

</dd> <dt>

<span id="upnp.d_6_gly"></span><span id="UPNP.D_6_GLY"></span>**Тип устройства**
</dt> <dd>

Тип устройства "Стандартный" обозначается универсальным именем URN: schemas-UPnP-org: Device:, за которым следует уникальное имя, назначенное форумом UPnP.

Между шаблонами устройств на основе UPnP и типами устройств существует связь «один к одному». Поставщики UPnP могут указывать дополнительные типы устройств; они обозначаются urn: domain-name: Device:, за которым следует уникальное имя, назначенное поставщиком, где domain-name — это доменное имя, зарегистрированное для поставщика.

</dd> </dl>
