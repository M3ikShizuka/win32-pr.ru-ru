# <a name="virtual-channel-client-registration"></a>Регистрация клиента виртуального канала

Необходимо сохранить имя библиотеки DLL клиента в реестре.

В реестре добавьте подраздел в одно из следующих расположений:

**HKey \_ Текущие \_ пользовательские** \\ **программные** \\  \\  \\  \\ **надстройки** клиента сервера терминалов Microsoft

**HKey \_ Текущая \_ Пользовательская** \\ **Программа**. \\  \\  \\  \\ **надстройки** для подключения клиента сервера терминалов Microsoft

> [!Note]  
> В пути реестра *Connection* представляет имя соединения в диспетчере соединений.

 

Записи в разделе **\\ \\ ADDIN по умолчанию** применяются ко всем подключениям. Записи в разделе " **\\** *_\\ надстройки_* подключений" применяются только к соединению, определяемому *соединением*. Подключения могут создаваться и управляться с помощью диспетчера соединений.

Подразделу можно присвоить любое имя. Он должен содержать значение **reg \_ SZ** или **reg \_ expand \_ SZ** , а также может содержать значение **reg \_ DWORD** . Синтаксис значения **reg \_ SZ** или **reg \_ expand \_ SZ** выглядит следующим образом.

``` syntax
Name = DLLname
```

Если **Name** — это **значение \_ \_ SZ Expand** , оно может содержать неразвернутые переменные среды, которые расширяются во время выполнения.

Значением *DLLname* может быть полный путь. Если параметр *DLLname* не содержит путь, используется стандартная стратегия поиска DLL. Дополнительные сведения см. в разделе "Примечания" для [**LoadLibrary**](/windows/desktop/api/libloaderapi/nf-libloaderapi-loadlibrarya).
