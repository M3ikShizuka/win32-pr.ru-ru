# <a name="imstscaxeventsonconfirmclose-method"></a>Метод Имстскаксевентс:: Онконфирмклосе

Вызывается, когда клиент вызывает метод [**имсрдпклиент:: RequestClose**](imsrdpclient-requestclose.md) . В ответ на это событие пользователю должно быть предложено подтвердить закрытие подключения. Дополнительные сведения см. в разделе "Примечания".

## <a name="syntax"></a>Синтаксис


```C++
void OnConfirmClose(
  [out] VARIANT_BOOL *pfAllowClose
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пфалловклосе* \[ заполняет\]
</dt> <dd>

Если **вариант \_ true**(значение по умолчанию) указывает, что пользователь хочет закрыть соединение. Если значение **варианта \_ false**, указывает, что пользователь не хочет закрывать соединение. Дополнительные сведения см. в разделе "Примечания".

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Комментарии

Когда приложение-контейнер вызывает метод [**имсрдпклиент:: RequestClose**](imsrdpclient-requestclose.md) , этот метод возвращает значение, указывающее, должен ли контейнер ожидать возникновения события **онконфирмклосе** перед закрытием управляющего соединения. Если **RequestClose** возвращает **контролклосеваитфоревентс**, а пользователь подключен и вошел в сеанс службы удаленных рабочих столов, срабатывает событие **онконфирмклосе** . В этот момент контейнерное приложение может запрашивать у пользователя запрос на закрытие подключения. Если пользователь хочет закрыть подключение, приложение должно задать для параметра *пфалловклосе* значение **Variant \_ true** и продолжить закрытие соединения. Если пользователь не хочет закрываться, приложение должно установить *пфалловклосе* в значение **Variant \_ false** и оставить соединение открытым.

Дополнительные сведения о веб-подключение к удаленному рабочему столу см. в разделе [требования для веб-подключение к удаленному рабочему столу](requirements-for-remote-desktop-web-connection.md).
