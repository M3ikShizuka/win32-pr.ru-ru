# <a name="session-monikers"></a>Моникеры сеанса

Распределенная модель COM (DCOM) обеспечивает активацию объектов для каждого сеанса с помощью предоставленного системой моникера сеанса. Дополнительные сведения о создании и использовании моникеров сеансов, включая пример кода, демонстрирующий, как активировать локальный серверный процесс, см. в следующих разделах.

## <a name="in-this-section"></a>Содержание раздела

<dl> <dt>

[Активация сеанса в сеансе с моникером сеанса](session-to-session-activation-with-a-session-moniker.md)
</dt> <dd>

Активация между сеансами (также называемая межсеансией активации) позволяет клиентскому процессу запускать (активировать) процесс локального сервера в указанном сеансе.

</dd> <dt>

[Использование моникера сеанса](using-a-session-moniker.md)
</dt> <dd>

Активация "сеанс — сеанс" позволяет клиентскому процессу активировать локальный серверный процесс в указанном сеансе.

</dd> </dl>
