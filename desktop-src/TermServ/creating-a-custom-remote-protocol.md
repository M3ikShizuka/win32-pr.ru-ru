# <a name="creating-a-remote-desktop-protocol-provider"></a>Создание поставщика протокол удаленного рабочего стола

В следующих разделах содержатся сведения о создании поставщика протокол удаленного рабочего стола. Диспетчер протоколов реализуется как сервер COM и регистрируется в расположении, которое выполняется при запуске службы службы удаленных рабочих столов.

[Регистрация диспетчера протокола](registering-the-custom-protocol.md)
</dt> <dd>

Необходимо создать хотя бы одну запись значения реестра для диспетчера протоколов, чтобы служба службы удаленных рабочих столов могла создать ее экземпляр.

</dd> <dt>

[Последовательность вызовов методов](method-call-sequence.md)
</dt> <dd>

Служба службы удаленных рабочих столов и поставщик протокола вызывают друг друга в четко определенных последовательностях.

</dd> </dl>
