# <a name="testing-and-debugging"></a>Тестирование и отладка

Существует прослушиватель "echo", реализованный клиентом подключение к удаленному рабочему столу (RDC), который всегда присутствует и ожидает входящих подключений. При записи на стороне сервера модуля динамического виртуального канала (DVC) в качестве быстрой проверки можно открыть конечную точку с именем ECHO. Любая запись в канал, созданная из этой конечной точки, приведет к поставке одних и тех же данных.

Этот метод можно использовать для тестирования реализации ввода-вывода на стороне сервера для измерения времени отклика для подключаемого модуля клиента подключение к удаленному рабочему столу (RDC) или для измерения задержки сети для клиента подключение к удаленному рабочему столу (RDC). Нет ограничений на объем данных, которые могут быть отправлены по этому каналу.
