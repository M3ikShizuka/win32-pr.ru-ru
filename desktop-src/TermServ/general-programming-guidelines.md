# <a name="general-programming-guidelines"></a>Общие рекомендации по программированию

В следующих разделах приводятся общие рекомендации по разработке приложений в среде службы удаленных рабочих столов.

## <a name="in-this-section"></a>Содержание раздела

<dl> <dt>

[Уровень совместимости приложений](application-compatibility-layer.md)
</dt> <dd>

Для запуска устаревших приложений в среде службы удаленных рабочих столов можно использовать службы удаленных рабочих столов уровень совместимости приложений.

</dd> <dt>

[Рекомендации для клиентских и серверных приложений](client-server-application-guidelines.md)
</dt> <dd>

Клиентские и серверные приложения не должны рассчитывать на то, что соединение с одним компьютером эквивалентно сеансу одного пользователя.

</dd> <dt>

[Наблюдение за подключениями и отсоединениями сеансов](monitoring-session-connections-and-disconnections.md)
</dt> <dd>

Чтобы зарегистрировать приложение в службы удаленных рабочих столов, сохраните имя серверного приложения виртуального канала в реестре, добавив подраздел.

</dd> <dt>

[Рекомендации по периферийным устройствам](peripheral-hardware-guidelines.md)
</dt> <dd>

Если устройство оборудования не предназначено для работы в удаленном сеансе, поставщики должны убедиться, что программное обеспечение драйвера устройства принудительно отключает перенаправление устройства, используя системную политику или групповую политику.

</dd> <dt>

[Связывание во время выполнения с Wtsapi32.dll](run-time-linking-to-wtsapi32-dll.md)
</dt> <dd>

Приложение может использовать службы удаленных рабочих столов API для динамической привязки к Wtsapi32.dll во время выполнения. Для этого приложение должно вызвать функцию [**LoadLibrary**](/windows/desktop/api/libloaderapi/nf-libloaderapi-loadlibrarya) для загрузки Wtsapi32.dll.

</dd> </dl>
