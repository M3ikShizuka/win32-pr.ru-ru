# <a name="using-remote-desktop-services-virtual-channels"></a>Использование виртуальных каналов службы удаленных рабочих столов

Для реализации виртуального канала необходимо предоставить серверные и клиентские модули приложения виртуального канала. Серверный модуль может быть приложением пользовательского режима или драйвером режима ядра. Клиентский модуль должен быть библиотекой DLL.

Описание этих модулей см. в следующих разделах.

## <a name="in-this-section"></a>Содержание раздела

<dl> <dt>

[Серверное приложение виртуального канала](virtual-channel-server-application.md)
</dt> <dd>

Серверный модуль приложения, использующего виртуальные каналы, должен быть приложением пользовательского режима, выполняющимся в сеансе клиента на сервере узла сеансов удаленный рабочий стол (узел сеансов удаленных рабочих столов). Обратите внимание, что необходимо предоставить метод для запуска серверного приложения.

</dd> <dt>

[Клиентская библиотека виртуальных каналов](virtual-channel-client-dll.md)
</dt> <dd>

Клиент приложения виртуальных каналов — это библиотека DLL, которая загружается во время инициализации службы удаленных рабочих столов на клиентском компьютере. Библиотека DLL должна быть зарегистрирована на клиентском компьютере.

</dd> <dt>

[Регистрация клиента виртуального канала](virtual-channel-client-registration.md)
</dt> <dd>

Необходимо сохранить имя библиотеки DLL клиента в реестре.

</dd> <dt>

[Постоянные виртуальные каналы удаленного управления](remote-control-persistent-virtual-channels.md)
</dt> <dd>

Постоянный виртуальный канал удаленного управления не закрывается при подключении или отключении удаленного управления для сеанса, подключенного к клиенту. Данные по-прежнему будут передаваться и приниматься через этот канал.

</dd> <dt>

[Динамические виртуальные каналы](dynamic-virtual-channels.md)
</dt> <dd>

Динамические API виртуальных каналов (DVC) расширяют существующие API виртуальных каналов для службы удаленных рабочих столов, называемых статическими интерфейсами API виртуального канала (SVC).

</dd> </dl>

если вы включили приложение виртуального канала в развертывание службы удаленных рабочих столов, можно также сделать приложение доступным для клиентских компьютеров, обращающихся к серверу узла сеансов удаленных рабочих столов с помощью элемента управления удаленный рабочий стол ActiveX. дополнительные сведения см. [в разделе использование элемента управления удаленный рабочий стол ActiveX с виртуальными каналами](using-the-remote-desktop-activex-control-with-virtual-channels.md).
