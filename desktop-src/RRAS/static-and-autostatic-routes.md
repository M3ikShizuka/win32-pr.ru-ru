# <a name="static-and-autostatic-routes"></a>Статические и автостатические маршруты

Как правило, маршруты к удаленным сетям получаются динамически через протоколы маршрутизации. Однако администратор может также *заполнить* таблицу маршрутизации, предоставив маршруты вручную. Эти маршруты называются *статическими*. Статический маршрут связан с интерфейсом, представляющим удаленную сеть. В отличие от динамических маршрутов, статические маршруты сохраняются даже при перезапуске маршрутизатора или отключенном интерфейсе.

*Статический* маршрут получается через протокол маршрутизации, но после его получения ведет себя как статический маршрут. Для получения маршрутов с автостатическим интерфейсом используется следующий процесс: Диспетчер маршрутизаторов IP или IPX выдает запрос о том, что протокол маршрутизации обновляет сведения о маршрутизации для определенного интерфейса. Затем результаты обновления преобразуются в статические маршруты. Обратите внимание, что только некоторые протоколы маршрутизации поддерживают запросы на обновления маршрутов для автостатического маршрута.
