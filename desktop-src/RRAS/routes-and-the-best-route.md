# <a name="routes-and-the-best-route"></a>Маршруты и оптимальный маршрут

Маршрут — это "сетевой путь" к назначению, с которым связана определенная стоимость. Стоимость представляется с помощью административных настроек и определенной для протокола метрики. Маршруты с более низкими затратами предпочтительнее всех остальных маршрутов.

Запись маршрута в таблице маршрутизации включает:

-   Маркер назначения
-   Владелец маршрута
-   Сосед (узел), который предоставил сведения о маршруте
-   Флаги, связанные с состоянием маршрута
-   Флаги, связанные с маршрутом
-   Приоритет и метрика маршрута
-   Список представлений, к которым относится маршрут
-   Сведения, являющиеся частными для владельца маршрута
-   Список следующих прыжков, используемых для достижения места назначения

Следующие значения вместе однозначно определяют маршрут в таблице маршрутизации.

-   Целевая сеть
-   Владелец маршрута
-   Сосед, который указал маршрут

## <a name="metrics-and-preference"></a>Метрики и предпочтения

Каждый маршрут имеет административные настройки (заданные политикой маршрутизации) и зависимую от клиента метрику. Диспетчер таблиц маршрутизации использует эти сведения, чтобы определить, какой маршрут является лучшим маршрутом к назначению. Маршруты с более низким приоритетом — это лучшие маршруты (один из них самый низкий и, следовательно, лучший). Если два маршрута имеют одинаковый приоритет, лучше использовать маршрут с нижней метрикой.

Как правило, предпочтение маршрута определяется предпочтением клиента, который добавил маршрут. Однако для всех маршрутов, добавленных с помощью средства управления Netsh.exe, значение предпочтения можно указать для каждого маршрута.

Предпочтение обычно используется для обозначения приоритета между клиентами. Например, администратор может назначить OSPF более низкому (лучшему) приоритету по сравнению с RIP. В этом случае OSPF-маршруты предпочтительнее RIP-маршрутов.
