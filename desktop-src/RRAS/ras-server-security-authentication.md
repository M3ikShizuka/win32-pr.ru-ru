# <a name="ras-server-security-authentication"></a>Проверка подлинности безопасности сервера RAS

когда сервер RAS Windows NT/Windows 2000 получает вызов, он вызывает функцию [**рассекуритидиалогбегин**](/windows/desktop/api/Rasshost/nf-rasshost-rassecuritydialogbegin) зарегистрированной DLL-библиотеки безопасности RAS, если таковая имеется. Этот вызов уведомляет библиотеку DLL безопасности, чтобы начать проверку подлинности удаленного пользователя. Сервер RAS вызывает **рассекуритидиалогбегин** перед выполнением проверки подлинности PPP или RAS.

Вызов [**рассекуритидиалогбегин**](/windows/desktop/api/Rasshost/nf-rasshost-rassecuritydialogbegin) передает следующие сведения в библиотеку DLL безопасности:

-   Маркер порта для обнаружения соединения.
-   Указатели на буферы, используемые при взаимодействии с удаленным пользователем.
-   Указатель на функцию [**рассекуритидиалогкомплете**](/windows/desktop/api/Rasshost/nf-rasshost-rassecuritydialogcomplete) для вызова после завершения проверки подлинности.

Маркер порта и указатель буфера допустимы до тех пор, пока библиотека DLL безопасности не вызовет [**рассекуритидиалогкомплете**](/windows/desktop/api/Rasshost/nf-rasshost-rassecuritydialogcomplete) для завершения транзакции проверки подлинности.

[**Рассекуритидиалогкомплете**](/windows/desktop/api/Rasshost/nf-rasshost-rassecuritydialogcomplete) уведомляет сервер RAS о результатах проверки подлинности удаленного пользователя в библиотеке DLL безопасности. Если библиотека DLL безопасности сообщает об успешном выполнении, сервер удаленного доступа переходит в систему с проверкой подлинности PPP и RAS удаленного пользователя. Если DLL-библиотека безопасности сообщает, что удаленный пользователь не прошел проверку подлинности или произошла ошибка, сервер RAS зависает и записывает в журнал событий ошибку или неудачную проверку подлинности.
