# <a name="working-with-next-hops"></a>Работа со следующими прыжками

API RTMv2 позволяет клиентам работать со списком следующих прыжков, связанных с маршрутами и назначениями. Клиенты могут добавлять или обновлять следующий прыжок, вызывая [**ртмадднекссоп**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmaddnexthop). Клиенты могут удалить следующий прыжок с помощью [**ртмделетенекссоп**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmdeletenexthop). Клиенты могут искать следующие прыжки, вызывая [**ртмфинднекссоп**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmfindnexthop).

Клиенты также могут напрямую обновить следующий прыжок. Для этого клиент должен сначала заблокировать следующий прыжок с помощью функции [**ртмлоккнекссоп**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmlocknexthop) . Затем клиент может использовать прямой указатель на [**\_ \_ информационную структуру RTM**](/windows/desktop/api/Rtmv2/ns-rtmv2-rtm_nexthop_info) диспетчера таблиц маршрутизации для внесения необходимых изменений. Наконец, клиент может разблокировать следующий прыжок.

> [!Note]  
> Поля адреса и индекса следующего прыжка в следующем прыжке однозначно определяют следующий прыжок и не должны изменяться.
