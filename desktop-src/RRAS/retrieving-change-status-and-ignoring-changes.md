# <a name="retrieving-change-status-and-ignoring-changes"></a>Получение состояния изменения и игнорирование изменений

Клиент может запросить диспетчер таблиц маршрутизации, чтобы выяснить, ожидает ли уведомление об изменении назначения, вызвав [**ртмжетчанжестатус**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmgetchangestatus). Эта функция возвращает **значение true** , пока вызывающий объект не извлечет это изменение, вызвав [**ртмжетчанжеддестс**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmgetchangeddests).

Клиент может использовать этот запрос, чтобы избежать выполнения действия, которое должно быть отменено после получения и обработки уведомления об изменении. Использование этой функции позволяет клиенту эффективно работать, откладывая некоторые операции на более позднее время.

Клиент также может игнорировать ожидающее уведомление об изменении для назначения, вызвав [**ртмигноречанжеддестс**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmignorechangeddests). Последующий вызов [**ртмжетчанжеддестс**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmgetchangeddests) не возвращает это назначение, если после вызова [**ртмигноречанжеддестс**](/windows/desktop/api/Rtmv2/nf-rtmv2-rtmignorechangeddests)не происходит другое изменение.
