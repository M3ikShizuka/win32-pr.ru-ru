# <a name="rpf-alert-callbacks"></a>Обратные вызовы оповещений РПФ

После того как диспетчер групп многоадресной рассылки получает уведомление пакета от нового источника или пакета, предназначенного для новой группы, Диспетчер групп многоадресной рассылки находит маршрут к источнику в представлении многоадресной рассылки таблицы маршрутизации. Затем Диспетчер групп многоадресной рассылки вызывает обратный вызов [**\_ \_ обратного вызова пмгм РПФ**](/windows/desktop/api/Mgm/nc-mgm-pmgm_rpf_callback) к протоколу, владеющему интерфейсом, в котором были получены данные.

После вызова этого обратного вызова протокол маршрутизации может изменить входящий интерфейс, если протокол маршрутизации должен получить данные для группы в другом интерфейсе.
