# <a name="routing-and-remote-access-services-architecture"></a>Архитектура служб маршрутизации и удаленного доступа

На следующем рисунке представлено общее представление архитектуры маршрутизатора Windows 2000.

Компоненты, относящиеся к семейству протоколов IP, отображаются светло-серым цветом. Компоненты, относящиеся к семейству протоколов IPX, показаны темно-серым цветом. Компоненты, общие для всех семейств протоколов, не затенены.

Служба маршрутизации и удаленного доступа предоставляет API-интерфейсы для администрирования и настройки службы. Эти интерфейсы API включают агенты SNMP для IP-адресов и IPX.

Служба RRAS включает протоколы маршрутизации для IP и IPX. Для IP-адреса служба RRAS предоставляет протокол RIP и открытые протоколы маршрутизации OSPF. Для IPX служба RRAS предоставляет протокол IPX RIP и SAP.

RRAS использует один диспетчер таблиц маршрутизации (RTM) для IP-адресов и IPX. Однако каждое семейство протоколов имеет собственный диспетчер маршрутизаторов. RRAS также имеет отдельный компонент для управления группами многоадресной рассылки.

Интерфейс диспетчера динамических интерфейсов (DIM) со службами PPP и TAPI используется для управления интерфейсами PPP, используемыми некоторыми маршрутами.
