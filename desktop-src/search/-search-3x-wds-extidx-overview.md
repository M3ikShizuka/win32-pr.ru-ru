# <a name="extending-the-index-windows-search"></a>Расширение индекса (Windows Search)

можно расширить Windows поиска, чтобы индексировать содержимое и свойства новых форматов файлов, а также хранилища данных с помощью [интерфейсов надстроек данных](./-search-data-addins-interfaces-entry-page.md). чтобы создать надстройки поиска Windows, сторонние разработчики должны сначала реализовать хранилище данных оболочки, а затем разработать обработчик протокола, чтобы Windows поиск мог получить доступ к данным для индексирования. При наличии пользовательского формата файла необходимо разработать обработчик фильтра для индексирования содержимого файла и обработчика свойств для каждого типа файлов в свойствах индекса.

Windows В настоящее время Поиск поддерживает индексирование более 200 типов элементов (например, .txt, .html и .xml форматов файлов) и может работать с несколькими типами хранилищ данных (такими как файловая система NTFS и Microsoft Outlook). Windows поиск использует технологию фильтров и обработчиков протоколов, похожую на SharePoint Server. таким образом, если у вас уже есть реализации для формата файлов, можно обновить реализации для инициализации с помощью потока, используя [IPersistStream](/windows/win32/api/objidl/nn-objidl-ipersiststream) , чтобы фильтр работал с Windowsным поиском.

> [!Note]  
> Обработчики фильтров, обработчики свойств и обработчики протоколов должны быть написаны в машинном коде. Это происходит из-за потенциальных проблем с управлением версиями среды CLR при работе нескольких надстроек.

 

В этом разделе по расширению индекса с помощью надстроек содержатся следующие разделы.
