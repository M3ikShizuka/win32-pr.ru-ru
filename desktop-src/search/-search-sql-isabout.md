# <a name="isabout-term"></a>О термине

**Не рекомендуется**

Эта функция была удалена в Windows 8. При написании новых приложений Избегайте использования этой устаревшей функции. При изменении существующих приложений настоятельно рекомендуется удалить все зависимости от этой функции.

Термин «о программе» сопоставляет столбцы с группой из одного или нескольких условий поиска. Он имеет следующий синтаксис:


```
ISABOUT(<components>) [RANKMETHOD <method>]
```



Необязательный термин РАНКМЕСОД указывает метод вычисления, используемый для ранжирования документов, соответствующих одному или нескольким компонентам. Если РАНКМЕСОД не указан, используется метод ранжирования Джаккарда коэффициента по умолчанию.

Термин «о программе» может иметь один или несколько компонентов. Столбцы, указанные в предикате [Contains](-search-sql-contains.md) , проверяются для каждого компонента. Документ включается в результаты, если хотя бы один из компонентов соответствует. Несколько компонентов разделяются запятыми.

Часть компонента имеет следующий синтаксис:


```
<match_term> [<weight_term>]
```



Можно использовать необязательный термин веса для изменения относительной важности каждого термина в термине «о себе». Если критерий веса не применяется, подразумевается вес 1,0 по умолчанию.

## <a name="isabout-column-weighting"></a>О взвешивании столбцов

Термин «о программе» ранжирует совпадающие документы на основе того, насколько близко каждый документ соответствует набору условий соответствия в запросе. Можно использовать весовые коэффициенты столбцов для повышения важности при совпадении с другими условиями соответствия. Для каждого термина соответствия в термине «о себе» может быть применено весовое значение. Весовой коэффициент применяется к одному выражению соответствия и указывается ключевым словом WEIGHT. Выражение веса имеет два альтернативных синтаксиса:


```
<match_term> WEIGHT(<weight_value>)
<match_term>:(<weight_value>)
```



Значение веса должно находиться в диапазоне от 0 до 1,0 и содержать не более трех десятичных разрядов. Указание весового значения вне этого диапазона приводит к появление сообщения об ошибке. Значение невзвешенного ранжирования для термина умножается на значение веса для этого термина.

Если для условия соответствия не указан вес, подразумевается значение по умолчанию 1,0.

### <a name="example"></a>Пример

В следующем примере весовые коэффициенты применяются к двум терминам соответствия, используя как длинный, так и короткий синтаксис для значений веса.


```
WHERE CONTAINS( System.FileName,
      'ISABOUT("computer" WEIGHT (0.75),"software":0.25)')
```
