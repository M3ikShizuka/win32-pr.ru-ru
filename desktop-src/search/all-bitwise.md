# <a name="all-bitwise-and-some-bitwise"></a>ВСЕ побитовое и побитовое

**Все битовые** и **битовые** ключевые слова используются для проверки битов в целочисленном типе. Если все биты в свойстве соответствуют маске, **все ПОбитовое** значение равно true. Если хотя бы один из наборов битов в свойстве соответствует маске, **ПОбитовое** значение равно true.

Операторы можно применять как к скалярным свойствам (с одним значением), так и к свойствам Vector (несколько значений). В следующем примере кода показано, как проверить значения свойств, используя **все ПОбитовое** и **побитовое**.


```sql
ALL array ALL BITWISE [values?]
ALL array SOME BITWISE [values?]
            
```



## <a name="comparison-operators"></a>Операторы сравнения

В следующей таблице перечислены поддерживаемые операторы сравнения для побитовых тестов.



| Оператор сравнения | Описание  |
|---------------------|--------------|
| =                   | Равно     |
| ! = или <>      | Не равно |



 

Логика побитовых тестов приведена в следующей таблице.



| Побитовый оператор теста и сравнения | Логика                   |
|--------------------------------------|-------------------------|
| = ВСЕ ПОБИТОВОЕ                        | Свойство & маска = маска  |
| = НЕКОТОРОЕ ПОБИТОВОЕ                       | Свойство & Mask! = 0    |
| <> ВСЕ ПОБИТОВЫЕ                 | Свойство & маска! = маска |
| <> ПОБИТОВОГО                | Свойство & Mask = 0     |



 

Следующая таблица истинности использует примеры двоичных и шестнадцатеричных значений для демонстате логики побитовых тестов.



| Свойство в двоичном формате (шестнадцатеричное) | Маска в двоичном формате (HEX) | Свойство & Mask = двоичное (шестнадцатеричное) | = НЕКОТОРОЕ ПОБИТОВОЕ | = ВСЕ ПОБИТОВОЕ |
|--------------------------|----------------------|--------------------------------|----------------|---------------|
| 0001 (0x1)               | 0001 (0x1)           | 0001 (0x1)                     | True           | True          |
| 0001 (0x1)               | 0011 (0x3)           | 0001 (0x1)                     | True           | Неверно         |
| 0011 (0x3)               | 0001 (0x1)           | 0001 (0x1)                     | True           | True          |
| 0010 (0x2)               | 0001 (0x1)           | 0000 (0x0)                     | Неверно          | False         |
| 11110000 (0xF0)          | 00000011 (0x03)      | 00000000 (0x00)                | Неверно          | False         |
| 11110010 (0xF2)          | 11110010 (0xF2)      | 11110010 (0xF2)                | True           | True          |
| 11110010 (0xF2)          | 00000011 (0x03)      | 00000010 (0x02)                | True           | Неверно         |



 

## <a name="example"></a>Пример

Ниже приведен пример **ПОбитового** предиката ALL.


```sql
Select system.itemnamedisplay, system.FileAttributes from SystemIndex Where System.FileAttributes <> ALL BITWISE 0x4 AND Scope = 'file:c:\bitwise'
                
```
