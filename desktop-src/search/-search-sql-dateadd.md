# <a name="dateadd-function"></a>Функция DATEADD

Функция DATEADD выполняет вычисления времени и даты для соответствующих свойств с типами дат. Функция DATEADD используется для получения даты и времени в течение указанного промежутка времени до текущего.

## <a name="syntax"></a>Синтаксис


```
DATEADD (DateTimeUnits, OffsetValue, DateTime)
```



## <a name="arguments"></a>Аргументы

*датетимеунитс*

Указывает единицы параметра *даты и времени* : год, квартал, месяц, неделя, день, час, минута или секунда. Это значение чувствительно к регистру, и кавычки не требуются вокруг параметра.

*оффсетвалуе*

Задает смещение времени в единицах, указанных параметром *датетимеунитс* . **Оффсетвалуе** должно быть отрицательным целым числом. Положительные значения не поддерживаются.

*DateTime*

Задает отметку времени, из которой вычисляется смещение. Это не может быть литералом даты. Он должен быть либо ЖЕТГМТДАТЕ, либо результатом другой функции DATEADD.

## <a name="remarks"></a>Комментарии

Функцию DATEADD можно использовать только в сравнениях литеральных значений и только с правой стороны оператора сравнения.

Функция ЖЕТГМТДАТЕ возвращает текущую дату и время по Гринвичу (GMT). Помните, что это значение не может совпадать с местным временем компьютера.

Не используйте оператор сравнения Equals (=), так как внутреннее представление времени может создавать ошибки округления, которые приводят к непредвиденным результатам сопоставления.

Можно использовать несколько функций DATEADD для объединения единиц смещения.

### <a name="examples"></a>Примеры

В следующем примере предложение WHERE соответствует документам, которые были изменены в течение последних пяти дней:


```
...WHERE System.DateModified <=DATEADD (DAY, -5, GETGMTDATE())
```



В следующем примере предложение WHERE соответствует документам, которые были изменены в течение последних двух дней и четырех часов:


```
...WHERE System.DateModified <=DATEADD (DAY, -2, DATEADD (HOUR, -4, GETGMTDATE()))
```
