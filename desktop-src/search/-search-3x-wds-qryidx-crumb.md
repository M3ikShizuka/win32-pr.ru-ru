# <a name="crumb-argument-windows-search"></a>аргумент с попараметром (поиск Windows)

`crumb`Аргумент поддерживает полные инструкции расширенного синтаксиса запросов (АКС) и особенно полезен в качестве средства управления областью поиска. помимо акс ементс, `crumb` аргумент может принимать специальный `location` параметр Windows Vista и `kind` и `store` параметров в XP, как описано далее в этом разделе.

## <a name="crumb-syntax"></a>Синтаксис с повышей

Синтаксис с обозначением выглядит следующим образом:


```
crumb=<column>:<value>[,<label>][,<column>:<value>[,<label>]]& 
```



&lt;Часть столбца &gt; является любым свойством в системе свойств, а &lt; значение value &gt; является допустимым значением для этого свойства. <label>Часть является необязательным псевдонимом для свойства, которое отображается как указание пользовательского интерфейса.

### <a name="general-examples"></a>Общие примеры


```
crumb=System.Author:paolo&
crumb=store:mapi&
crumb=location:c%3a%5cMyVacationPix,Vacation&
```



 

## <a name="using-crumb-with-vista-location"></a>Использование инструкции with Vista (расположение)

в параметре where Windows Vista поддерживает полный набор акс, а также `location` свойство, которое имеет специальную реализацию, доступную только в Windows Vista. Можно использовать либо строку АКС `location` , либо свойство в одном параметре, но не оба. Если параметр ссылки содержит АКС, все остальное в этом параметре не учитывается.

`location`Свойство позволяет указать путь для поиска. Windows Vista может обходить индексатор и просматривать каталог напрямую, если расположение находится за пределами области обхода индексатора. Следовательно, эти операции поиска могут выполняться медленнее, чем поиск, использующий индексатор.

При указании `location` свойства поддерживаются два дополнительных параметра и необязательно:



| Параметр | Значения                  | Описание                                                                                                                                                                       |
|-----------|-------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| включения | включить, исключить        | Указывает, должен ли запрос включать или исключать элементы из этого пути. По умолчанию используется "include". Windows Vista не поддерживает исключения без включения. (См. пример) |
| рекурсию; | рекурсивный, нерекурсивный | Указывает, должен ли поиск выполнять рекурсию для всех вложенных папок, начиная со значения, определенного в поле Расположение: &lt; значение &gt; . "Recursive" — значение по умолчанию.                             |



 

Чтобы ограничить область поиска с помощью протокола Search-MS:, у вас есть различные параметры в зависимости от целевого объекта области.

Папка на локальном компьютере:

-   Используйте АКС (путь = папка: <URL-адрес в кодировке>)
-   Используйте аргумент Location (путь = расположение: <URL-адрес в кодировке>)

Папка на удаленном компьютере или в сети:

-   Используйте аргумент Location (путь = расположение: <URL-адрес в кодировке>)

Доступ к папке через известный обработчик протокола UNC:

-   Используйте АКС (повышено = Store <UNC protocol handler name> ).
-   Используйте аргумент Location (путь = расположение: <URL-адрес в кодировке>)

### <a name="vista-examples"></a>Примеры для Vista


```
search-ms:query=vacation&crumb=location:shell%3aPersonal,include,recursive&

search-ms:crumb=location:c%3a%5cPictures&crumb=location:c%3a%5cPictures%5cDuplicates,,exclude& 

search-ms:crumb=location:c%3a%5cDocuments&crumb=kind:pics&
```



В первом примере выполняется поиск «отпуска», начиная с расположения shell://Personal (специального ярлыка для папки «Мои документы» пользователя), включая эту папку и все вложенные папки. См. таблицу ниже.

Во втором примере выполняется поиск в C: \\ Pictures, но не в c: \\ рисунки \\ дублируются.

В третьем примере выполняется поиск в документах C: \\ , ограниченных файлами, свойство Kind которых имеет значение pics.

### <a name="constants-for-common-folders"></a>Константы для общих папок

Windows Vista позволяет использовать значения [кновнфолдерид](/previous-versions//bb762584(v=vs.85)) , которые обеспечивают уникальный независимый от системы способ определения специальных папок, часто используемых приложениями, но которые могут не иметь одинакового имени или расположения в любой заданной системе. например, системная папка может иметь значение «c: \\ Windows» на одной системе, а «c: \\ Winnt» — на другой. до Windows Vista использовались [ксидлс](/windows/desktop/shell/csidl) .

Используйте эти расположения в следующем синтаксисе:


```
crumb=location:shell%3a<LocationName>&
```



 

## <a name="using-crumb-with-windows-xp-kind-and-store"></a>использование инструкции with Windows XP (тип и магазин)

для Windows поиска в Windows XP (WDS 3. x) термины акс "kind" и "store" имеют специальную реализацию. Значения "Kind" — это те же [значения, которые используются в WDS 2. x](../lwef/-search-2x-wds-perceivedtype.md). Значения "Store" включают следующее:

-   действует
-   файл
-   аутлукекспресс
-   any

### <a name="xp-examples"></a>Примеры для XP


```
search-ms:query=from:john&crumb=store:outlookexpress,OE%20Mail&
search-ms:query=from:john&crumb=kind:communications&
```



первый пример возвращает сообщения электронной почты Microsoft Outlook Express из джон с пользовательской меткой "OE Mail". Во втором примере выполняется поиск любого обмена данными из Джон.
