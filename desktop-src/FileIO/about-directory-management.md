# <a name="about-directory-management"></a>Об управлении каталогами

Каталог, содержащий один или несколько каталогов, является *родительским* для вложенного каталога или каталогов, а каждый содержащийся каталог является *дочерним* по отношению к родительскому каталогу. Иерархическая структура каталогов называется *деревом каталогов*.

Файловая система NTFS реализует логическую связь между каталогом и файлами, которые он содержит, как *таблицу записей каталога*. При перемещении файла в каталог в таблице создается запись для перемещенного файла, а имя файла помещается в запись. При удалении файла, содержащегося в каталоге, имя и запись, соответствующие удаленному файлу, также удаляются из таблицы. В таблице записей каталога может находиться несколько записей для одного файла. Если в таблице для файла создается дополнительная запись, эта запись называется *жесткой ссылкой* на этот файл. Количество жестких связей, которые можно создать для одного файла, не ограничено.

Каталоги также могут содержать соединения и точки повторного анализа.
