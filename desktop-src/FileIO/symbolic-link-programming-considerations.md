# <a name="programming-considerations-local-file-systems"></a>Рекомендации по программированию (локальные файловые системы)

При работе с символическими ссылками учитывайте следующие моменты программирования:

-   Символьные ссылки могут указывать на несуществующий целевой объект.
-   При создании символьной ссылки операционная система не проверяет, существует ли целевой объект.
-   Если приложение пытается открыть несуществующий целевой объект, возвращается **\_ файл ошибки \_ не \_ найден** .
-   Символьные ссылки — это точки повторного анализа. Дополнительные сведения см. [в разделе Определение того, является ли каталог подключенной папкой](determining-whether-a-directory-is-a-volume-mount-point.md).
-   В определенном пути допускается не более 63 точек повторного анализа (и, следовательно, символьных ссылок).

    **Windows Server 2003 и Windows XP:** Существует ограничение в 31 точки повторного анализа по любому указанному пути.
