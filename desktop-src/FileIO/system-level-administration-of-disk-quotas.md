# <a name="system-level-administration-of-disk-quotas"></a>Администрирование дисковых квот на уровне системы

Системный администратор может устанавливать квоты для конкретных пользователей на томе. Администратор также может задать квоты по умолчанию для тома. Новый пользователь на томе получает квоту по умолчанию, если администратор не установил квоту специально для этого пользователя.

Администратор может запрашивать уровень отслеживания и ограничения квоты (или состояния квоты), квоты по умолчанию и сведения о квоте для каждого пользователя. Сведения о квотах для каждого пользователя содержат ограничение жесткой квоты пользователя, порог предупреждения и использование квоты. Администратор может также включить или отключить принудительную квоту.

Существует три состояния квоты, как показано в следующей таблице.



| Состояние          | Описание                                                                                                                                                                              |
|----------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Квота отключена | Изменения использования квоты не записываются, но ограничения квоты не удаляются. В этом состоянии производительность не зависит от дисковых квот. Это состояние по умолчанию.                         |
| Отслеживание квоты  | Изменения использования квоты записываются, но ограничения квоты не применяются. В этом состоянии события нарушения квоты не создаются, и операции с файлами завершаются неудачей из-за нарушений дисковой квоты. |
| Примененная квота | Изменения использования квоты записываются, и применяются ограничения квот.                                                                                                                           |
