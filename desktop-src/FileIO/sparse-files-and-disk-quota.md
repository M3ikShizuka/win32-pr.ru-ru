# <a name="sparse-files-and-disk-quotas"></a>Разреженные файлы и дисковые квоты

Разреженный файл влияет на квоты пользователей по номинальному размеру файла, а не к фактическому выделенному объему дискового пространства. Это значит, что создание файла размером 50 МБ со всеми нулевыми байтами потребляет 50 Мб квоты этого пользователя. Это означает, что, когда пользователь записывает данные в разреженный файл, ему не нужно беспокоиться о превышении предельной квоты, так как он уже заряжен на место. Системные администраторы не должны подсчитывать размер разреженного файла, который остался небольшим. Поэтому они не должны предоставлять пользователям ограничения на жесткие квоты, превышающие доступное физическое пространство, несмотря на использование разреженных файлов.
