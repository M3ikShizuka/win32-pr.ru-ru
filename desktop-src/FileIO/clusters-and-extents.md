# <a name="clusters-and-extents"></a>Кластеры и экстенты

На кластеры можно ссылаться с двух различных точек зрения: в файле и на томе. Любой кластер в файле имеет *номер виртуального кластера* (VCN), который является его относительным смещением от начала файла. Например, поиск в два раза больше размера кластера, за которым следует Read, будет возвращать данные, начиная с третьего VCN. *Логический номер кластера* (LCN) описывает смещение кластера от какой-либо произвольной точки в томе. Лкнс должен обрабатываться только как порядковый номер или относительное число. Нет гарантированного сопоставления логических кластеров с секторами физических жестких дисков.

*Экстент* — это запуск непрерывных кластеров. Например, предположим, что файл, состоящий из 30 кластеров, записан в двух экстентах. Первый экстент может состоять из пяти непрерывных кластеров, а остальные 25 кластеров.

Никакая связь с диском любого экстента не гарантируется в любом другом экстенте. Например, первый экстент может иметь более высокий уровень LCN, чем последующий экстент.
