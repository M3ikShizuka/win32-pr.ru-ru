<span data-ttu-id="87717-104">В Windows API (с некоторыми исключениями, обсуждаемыми в следующих параграфах) максимальная длина пути — это **максимальный \_ путь**, который определен как 260 символов.</span><span class="sxs-lookup"><span data-stu-id="87717-104">In the Windows API (with some exceptions discussed in the following paragraphs), the maximum length for a path is **MAX\_PATH**, which is defined as 260 characters.</span></span> <span data-ttu-id="87717-105">Локальный путь структурирован в следующем порядке: буква диска, двоеточие, обратная косая черта, имена компонентов, разделенные обратной косой чертой, и завершающий нуль-символ.</span><span class="sxs-lookup"><span data-stu-id="87717-105">A local path is structured in the following order: drive letter, colon, backslash, name components separated by backslashes, and a terminating null character.</span></span> <span data-ttu-id="87717-106">Например, максимальный путь на диске D — это "D: \\ *часть 256-символьного пути* &lt; NUL &gt; ", где " &lt; NUL &gt; " представляет невидимый завершающий нуль-символ для текущей системной кодовой страницы.</span><span class="sxs-lookup"><span data-stu-id="87717-106">For example, the maximum path on drive D is "D:\\*some 256-character path string*&lt;NUL&gt;" where "&lt;NUL&gt;" represents the invisible terminating null character for the current system codepage.</span></span> <span data-ttu-id="87717-107">(Символы < > используются для наглядности и не могут быть частью допустимой строки пути.)</span><span class="sxs-lookup"><span data-stu-id="87717-107">(The characters < > are used here for visual clarity and cannot be part of a valid path string.)</span></span>

<span data-ttu-id="87717-115">Префикс " \\ \\ ? \\ " также можно использовать с путями, созданными в соответствии с соглашением об универсальных именах (UNC).</span><span class="sxs-lookup"><span data-stu-id="87717-115">The "\\\\?\\" prefix can also be used with paths constructed according to the universal naming convention (UNC).</span></span> <span data-ttu-id="87717-116">Чтобы указать такой путь с помощью UNC, используйте параметр " \\ \\ ? ## <a name="enable-long-paths-in-windows-10-version-1607-and-later"></a><span data-ttu-id="87717-126">Включение длинных путей в Windows 10, версии 1607 и более поздних</span><span class="sxs-lookup"><span data-stu-id="87717-126">Enable Long Paths in Windows 10, Version 1607, and Later</span></span>
