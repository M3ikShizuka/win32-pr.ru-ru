# <a name="csidl"></a>CSID

> [!Note]  
> Начиная с Windows Vista, эти значения были заменены на значения [**кновнфолдерид**](knownfolderid.md) . Список новых констант и соответствующих значений CSID см. в этом разделе. Для удобства соответствующие значения **кновнфолдерид** также указаны здесь для каждого значения CSid.
>
> Система с CSID поддерживается в Windows Vista в целях совместимости. Однако новая разработка должна использовать значения [**кновнфолдерид**](knownfolderid.md) , а не значения CSid.

 

Значения "CSID" (Специальный список ИДЕНТИФИКАТОРов элементов) обеспечивают уникальный независимый от системы способ идентификации специальных папок, часто используемых приложениями, но которые могут не иметь одинакового имени или расположения в любой заданной системе. Например, системная папка может иметь значение «C: \\ Windows» на одной системе, а «c: \\ WinNT» — на другой. Эти константы определены в Шлобж. h.

## <a name="remarks"></a>Примечания

Эти значения заменяют использование переменных среды для этой цели. Они в свою очередь заменяются в Windows Vista и позже значениями [**кновнфолдерид**](knownfolderid.md) .

CSID используется в сочетании с одной из четырех функций оболочки, [**шжетфолдерлокатион**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderlocation), [**шжетфолдерпас**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderpatha), [**шжетспеЦиалфолдерлокатион**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderlocation)и [**шжетспеЦиалфолдерпас**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderpatha)для получения пути к особой папке или указателя на список идентификаторов элементов (PIDL).

Объедините * * * * \_ флаг CSid \_ Create * * * * с любым другим ксидлс, за исключением * * * * \_ маркера CSid \_ не \_ проверить * * * *, чтобы принудительно создать связанную папку. Остальные Ксидлс соответствуют папкам файловой системы или виртуальным папкам. Когда идентификатор CSID определяет папку файловой системы, в качестве примера используется часто используемый путь. Можно использовать другие пути. Некоторые Ксидлс могут быть сопоставлены с эквивалентной переменной среды *% variablename%* . Однако Ксидлс более надежны и должны использоваться, если это возможно.
