# <a name="about-synchronization-manager"></a>О диспетчере синхронизации

Диспетчер синхронизации предоставляет централизованную, стандартную технологию для синхронизации файлов для автономного использования на локальном компьютере. Пользователь может подготовить компьютер для автономного использования, обновив необходимые файлы из сетевого хранилища. Затем, когда компьютер возвращается в режим «в сети», любые автономные изменения можно скопировать обратно в сеть. Синкмгр предоставляет общий пользовательский интерфейс, который может использоваться всеми приложениями для синхронизации файлов.

Файлы передаются независимо от протокола. Например, программа электронной почты может передавать свои сообщения с помощью протокола SMTP, НМТП или POP3, в то время как браузер может использовать протокол HTTP, а база данных может использовать RPC. Протокол является прозрачным для Синкмгр.

Синкмгр также не зависит от хранилища и может прозрачно передавать файлы, сообщения электронной почты, HTML-страницы или сведения о базе данных.
