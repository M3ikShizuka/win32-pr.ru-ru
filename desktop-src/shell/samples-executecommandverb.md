## <a name="description"></a>Описание

Этот метод является предпочтительным для реализации команд, так как он обеспечивает наибольшую гибкость, прост и поддерживает незавершенную активацию. В этом примере реализуется изолированный объект серверной модели COM, но предполагается, что реализация команды будет интегрирована в существующие приложения. Для этого основной объект приложения должен зарегистрировать фабрику класса для себя. Этот объект реализует [**интерфейс IDropTarget**](/windows/win32/api/oleidl/nn-oleidl-idroptarget) для команд приложения. Обратите внимание, что COM запускает приложение, если оно еще не запущено, но подключается к работающему экземпляру приложения, если оно существует.

## <a name="building-the-sample"></a>Построение образца

Чтобы построить образец из командной строки, выполните следующую команду:

1.  Откройте окно командной строки и перейдите в каталог проекта **ексекутекоммандверб** .
2.  Введите `msbuild ExecuteCommand.sln`.

чтобы создать пример с использованием Microsoft Visual Studio (предпочтительно):

1.  откройте обозреватель Windows и перейдите в каталог проекта **ексекутекоммандверб** .
2.  Дважды щелкните значок файла ExecuteCommand. sln, чтобы открыть проект в Visual Studio.
3.  В меню **Построение** выберите пункт **Построить решение**.

## <a name="running-the-sample"></a>Запуск примера

1.  перейдите в каталог, содержащий новый исполняемый файл, используя командную строку или обозреватель Windows.
2.  В командной строке введите `ExecuteCommand.exe` . кроме того, в обозревателе Windows дважды щелкните значок ExecuteCommand.exe.
3.  Следуйте инструкциям в отображаемом диалоговом окне.
