# <a name="spfilenotify_copyerror-message"></a>\_Сообщение спфиленотифи коперрор

Уведомление **спфиленотифи \_ коперрор** отправляется в подпрограммы обратного вызова, если во время операции копирования файла возникает ошибка.


```C++
SPFILENOTIFY_COPYERROR
  Param1 = (UINT_PTR) FilePathInfo;
  Param2 = (UINT_PTR) ReturnBuffer;
            
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Параметр1* 
</dt> <dd>

Указатель на структуру [**FilePath**](/windows/desktop/api/Setupapi/ns-setupapi-filepaths_a) .

</dd> <dt>

*Param2* 
</dt> <dd>

Указатель на буфер (максимальный размер символов пути), в \_ котором хранятся новые сведения о пути, указанные пользователем.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Обратный вызов должен возвращать одно из следующих значений.



| Код возврата                                                                                    | Описание                                                                                                                                                                                                                                                                   |
|------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**прервать ФИЛЕОП \_**</dt> </dl>   | Обработка очереди должна быть отменена. [**Сетупкоммитфилекуеуе**](/windows/desktop/api/Setupapi/nf-setupapi-setupcommitfilequeuea) возвращает ноль, а [**GetLastError**](/windows/desktop/api/errhandlingapi/nf-errhandlingapi-getlasterror) Возвращает расширенные сведения об ошибке, такие как ошибка \_ отмены (если пользователь был отменен) или ошибка \_ не \_ хватает \_ памяти.<br/> |
| <dl> <dt>**ФИЛЕОП \_ NewPath**</dt> </dl> | Повторите операцию копирования, используя путь функции обратного вызова, помещенной в буфер, на который указывает параметр *Param2* . Подпрограммы обратного вызова должны обеспечить, чтобы путь не переполнить размер буфера в массиве TCHAR из \_ элементов максимального пути.<br/>                |
| <dl> <dt>**ФИЛЕОП \_ Повтор**</dt> </dl>   | Пользователь пытается повторить операцию копирования.<br/>                                                                                                                                                                                                                   |
| <dl> <dt>**ФИЛЕОП \_ пропустить**</dt> </dl>    | Пользователь пропускает операцию копирования файла.<br/>                                                                                                                                                                                                                      |
