# <a name="authoring-an-inf-file"></a>Создание файла INF

При создании INF-файла для приложения установки также следует обращаться к формату INF-файла, который описан в *общих рекомендациях по* РАЗДЕЛам INF-файлов и INF-файлов и *директивам* пакета средств разработки драйверов для Microsoft Windows 2000.

При создании INF-файлов для приложения установки придерживайтесь следующих правил.

-   В каждом INF-файле требуется раздел **версии** .
-   Разделы должны начинаться с имени раздела, заключенного в квадратные скобки ( \[ \] ).
-   Имена разделов **дестинатиондирс**, **саурцедисксфилес**, **саурцедискснамес**, **Strings** и **Version** должны быть идентичны типу раздела. Например, используйте \[ дестинатиондирс \] . Авторы могут указывать имена других типов разделов.
-   К именам разделов **саурцедискснамес** и **саурцедисксфилес** можно добавить суффикс, зависящий от платформы. Например, чтобы отобразить раздел **саурцедискснамес** , относящийся к Intel, используйте \[ саурцедискснамес. x86 \] . Если функции настройки находят разделы **саурцедискснамес** и **саурцедисксфилес** , относящиеся к платформе, соответствующие платформе пользователя, функции используют разделы, относящиеся к конкретной платформе. В противном случае функции установки используют разделы **саурцедискснамес** и **саурцедисксфилес** , не имеющие суффикса. Функции установки могут программно определять систему пользователя. См. [Пример разделов INF саурцедискснамес и саурцедисксфилес](inf-sourcedisksnames-and-sourcedisksfiles-sections-example.md).
-   Значения могут быть выражены в виде заменяемых строк с помощью формы%*стркэй*%. Чтобы использовать символ% в строке, используйте%%. Стркэй должен быть определен в разделе **Strings** файла INF.
