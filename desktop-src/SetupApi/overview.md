# <a name="overview"></a>Обзор

Программный интерфейс (API) программы установки предоставляет набор функций, которые могут вызываться приложением установки для выполнения операций установки. Эти функции установки работают с INF-файлами Windows для предоставления следующих функций установки.

|---------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Файлы очереди                               | [Очереди файлов](file-queues.md)                                                                                                                                              |
| Установка файлов                            | [Очереди файлов](file-queues.md)<br/> [Настройка приложений](setup-applications.md)<br/> [Создание приложений установки](creating-setup-applications.md)<br/> |
| Обработка ошибок и запрос мультимедиа     | [Запросы к диску и обработка ошибок](disk-prompting-and-error-handling.md)                                                                                                  |
| Обновление записей реестра                   | [Настройка приложений](setup-applications.md)                                                                                                                                |
| Ведение журнала установленных файлов                     | [Журнал файлов](file-log.md)                                                                                                                                                    |
| Хранение недавно использовавшихся исходных путей | [Список источников MRU](mru-source-list.md)                                                                                                                                      |



 

Для большинства функций установки доступны версии Юникода и ANSI. Текстовые файлы в Юникоде должны содержать стандартную метку порядка следования байтов 0xFEFF, чтобы функции программы установки могли обозначать файл как текст в Юникоде.

Хотя API установки поддерживает запросы новых носителей и основных диалоговых окон обработки ошибок, функции установки не предоставляют возможности мастера или универсальный пользовательский интерфейс.

Разработчикам следует определить, могут ли они использовать [установщик Windows](/windows/desktop/Msi/windows-installer-portal) для установки приложений, а не API установки. Установщик Windows сокращает совокупную стоимость владения для клиентов, позволяя им эффективно устанавливать и настраивать ваши продукты и приложения. Установщик также может предоставить вашему продукту новые возможности для объявления функций без их установки, установки продуктов по требованию и добавления пользовательских настроек.
