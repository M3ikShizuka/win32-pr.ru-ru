# <a name="setup-api"></a>API установки

## <a name="purpose"></a>Назначение

API установки предоставляет набор функций, которые приложение установки вызывает для выполнения операций установки.

## <a name="where-applicable"></a>Где применимо

Эти функции установки доступны для разработки приложения установки. API установки больше не следует использовать для установки приложений. Вместо этого используйте [установщик Windows](/windows/desktop/Msi/windows-installer-portal)для разработки установщиков приложений. API установки будет использоваться для установки драйверов устройств.

API установки предназначен для разработки приложений в стиле рабочего стола.

## <a name="developer-audience"></a>Аудитория разработчиков

Разработчик может использовать API установки, если для приложения установки требуются следующие функциональные возможности.

-   Очередь файлов.
-   Установка файлов.
-   Обработка ошибок установки и запрос носителя.
-   Обновление записей реестра.
-   Ведение журнала файлов по мере их установки.
-   Хранение последних использованных путей к источникам.

## <a name="run-time-requirements"></a>Требования к среде выполнения

Сведения о том, какие операционные системы требуются для использования определенной функции, см. в разделе "требования" документации по функции.
