# <a name="hardware-provider-objects"></a>Объекты поставщика оборудования

\[Начиная с Windows 8 и Windows Server 2012, интерфейс COM [службы виртуальных дисков](virtual-disk-service-portal.md) заменяется [API управления хранилищами Windows](/previous-versions/windows/desktop/stormgmt/windows-storage-management-api-portal).\]

Объектная модель VDS включает набор объектов для запроса и настройки сущностей поставщика оборудования. (Обратите внимание, что хотя служба VDS включает поставщика программного обеспечения, для использования объектов поставщика оборудования необходимо приобрести поставщик оборудования и соответствующее оборудование отдельно.) Эти объекты поставщика оборудования представляют физические устройства (такие как подсистемы, диски и контроллеры) и виртуальные устройства (например, LUN и Плекс LUN).

Поставщик оборудования должен создать один COM-объект для каждого физического или виртуального устройства.

На следующем рисунке показана связь между объектом поставщика и набором объектов поставщика оборудования, а также отношение между различными объектами поставщика оборудования.

![Схема, показывающая связь между "поставщик" и "подсистема", "контроллер", "LUN", "Плекс", "диск" и "шпиндель". ](images/vdshwobjects.png)

Объект поставщика может содержать любое количество подсистем. Все поставщики оборудования способны управлять несколькими экземплярами одной и той же модели подсистемы. Многие поставщики оборудования также могут управлять несколькими экземплярами различных моделей подсистем. На одном компьютере может размещаться любое количество поставщиков оборудования.

Объект подсистемы может содержать любое количество контроллеров и дисков, а также может отображать любое количество LUN. Объект LUN состоит по крайней мере из одного плекса LUN, и каждый Плекс LUN сопоставляется с одним или несколькими дисками в зависимости от типа плекса. Объекты контроллера могут управлять вводом-выводом данных для любого числа объектов LUN.
