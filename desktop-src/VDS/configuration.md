# <a name="configuration-overview"></a>Обзор конфигурации

\[начиная с Windows 8 и Windows Server 2012, интерфейс COM [службы виртуальных дисков](virtual-disk-service-portal.md) заменяется [API Windows служба хранилища управления](/previous-versions/windows/desktop/stormgmt/windows-storage-management-api-portal).\]

Если вы не знакомы с объектами, определенными службой VDS, см. раздел [объектная модель VDS](vds-object-model.md).

Сложность конфигурации виртуального диска может варьироваться от простого в настройке. Несамостоятельные, бесплатные и корпоративные виртуальные диски определяют три типичные перспективы конфигурации. Каждая Перспектива представляет несколько других требований:

-   Неработающие виртуальные диски не настраиваются, возможно, только Автоматизируйте секционирование и форматирование новых дисков или временно создает зеркальный том для переноса данных с одного диска на другой без простоев. Такие диски хорошо подходят для ноутбуков и других систем с одним или несколькими дисками.
-   Виртуальные диски с бесплатной поддержкой — это хранилище, которое обеспечивает самостоятельную настройку, самостоятельное восстановление и доступность; использует чередующиеся тома и LUN для достижения лучшей производительности; для достижения лучшей доступности использует зеркальные тома и LUN. и используют пакеты, чтобы сделать режимы сбоя чистыми и содержащимися. Этот уровень конфигурации идеально подходит при замене старых, небольших, медленных системных дисков новыми, большими и быстрыми дисками. Он также идеально подходит для зеркального отображения данных с помощью автоматического резервирования — один запасной диск может защищать множество дисков. Дополнительные сведения см. в разделе [горячее резервирование](hot-sparing.md).

-   Enterprise виртуальные диски содержат очень большие или сложные конфигурации предприятия с дополнительными требованиями для конкретного сайта или приложения. Администраторы настраивайте подсистему хранения для одного приложения, которое может выполняться только редко, например ежемесячное задание пакетного отчета в системе транзакций, принимающей заказы. Enterprise виртуальные диски должны масштабироваться, показывать активность подсистемы хранения в реальном времени и удовлетворять требованиям практических администраторов.

Дополнительные сведения о зеркалах, полосковых элементах и других параметрах конфигурации в службе VDS см. в разделе [Привязка томов и LUN](volume-and-lun-binding.md). Расширенный способ настройки, называемый наложением, позволяет объединять конфигурации, связанные с существующими томами и LUN. Дополнительные сведения см. в разделе [Настройка стека](configuration-stacking.md).
