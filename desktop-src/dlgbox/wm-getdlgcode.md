# <a name="wm_getdlgcode-message"></a>\_Сообщение ЖЕТДЛГКОДЕ WM

Отправляется в процедуру окна, связанную с элементом управления. По умолчанию система обрабатывает все вводы с клавиатуры для элемента управления. система интерпретирует определенные типы ввода с клавиатуры как клавиши навигации диалогового окна. Чтобы переопределить это поведение по умолчанию, элемент управления может ответить на сообщение **WM \_ жетдлгкоде** , чтобы указать типы входных данных, которые необходимо обработать.


```C++
#define WM_GETDLGCODE                   0x0087
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Виртуальный ключ, нажатый пользователем и предлагающий Windows выдать это уведомление. Обработчик должен выборочно обрабатывайте эти ключи. Например, обработчик может принять и обработать **VK \_ return** , но делегировать **VK \_ вкладку** в окно владельца. Список значений см. в разделе [**коды виртуальных клавиш**](/windows/desktop/inputdev/virtual-key-codes).

</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на структуру [**MSG**](/windows/win32/api/winuser/ns-winuser-msg) (или **значение NULL** , если система выполняет запрос).

</dd> </dl>

## <a name="remarks"></a>Комментарии

Хотя функция [**дефвиндовпрок**](/windows/desktop/api/winuser/nf-winuser-defwindowproca) всегда возвращает ноль в ответ на сообщение **\_ жетдлгкоде WM** , процедура окна для предопределенных классов элементов управления возвращает код, соответствующий каждому классу.

Сообщение **WM \_ жетдлгкоде** и возвращаемые значения полезны только с пользовательскими элементами управления диалогового окна или стандартными элементами управления, измененными подклассами.
