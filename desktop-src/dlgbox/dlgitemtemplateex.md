# <a name="dlgitemtemplateex-structure"></a>Структура ДЛГИТЕМТЕМПЛАТИКС

Блок текста, используемый расширенным шаблоном диалогового окна для описания расширенного диалогового окна. Описание формата расширенного шаблона диалогового окна см. в разделе [**длгтемплатикс**](dlgtemplateex.md).

## <a name="syntax"></a>Синтаксис


```C++
typedef struct {
  DWORD     helpID;
  DWORD     exStyle;
  DWORD     style;
  short     x;
  short     y;
  short     cx;
  short     cy;
  DWORD     id;
  sz_Or_Ord windowClass;
  sz_Or_Ord title;
  WORD      extraCount;
} DLGITEMTEMPLATEEX;
```

**Идентификатор справки**

Расширенные стили для окна. Этот элемент не используется для создания элементов управления в диалоговых окнах, но приложения, использующие шаблоны диалоговых окон, могут использовать его для создания других типов окон. Список значений см. в разделе [**Расширенные стили окон**](/windows/desktop/winmsg/extended-window-styles).

Идентификатор элемента управления.

Если первый элемент имеет значение 0xFFFF, в массиве имеется один дополнительный элемент, указывающий порядковый номер предопределенного системного класса. Порядковый номер может быть одним из следующих значений Atom.

**title**
</dt> <dd>

Тип: **SZ \_ или \_**

</dd> <dd>

Массив с переменной длиной из 16-разрядных элементов, который содержит исходный текст или идентификатор ресурса элемента управления. Если первый элемент массива имеет значение 0xFFFF, то массив содержит один дополнительный элемент, указывающий порядковый номер ресурса, например значка, в исполняемом файле. Можно использовать идентификатор ресурса для элементов управления, таких как статические элементы управления "значок", которые загружают и отображают значок или другой ресурс, а не текст. Если первый элемент имеет любое значение, отличное от 0xFFFF, система рассматривает массив как строку в Юникоде, заканчивающуюся нулем, которая указывает начальный текст.

</dd> <dt>

**екстракаунт**
</dt> <dd>

Тип: **слово**

</dd> <dd>

Число байтов данных создания, которые следуют за данным элементом. Если это значение больше нуля, то данные для создания начинаются со следующей границы **слова** . Данные создания могут иметь любой размер и формат. Процедура окна элемента управления должна иметь возможность интерпретировать данные. Когда система создает элемент управления, она передает указатель на эти данные в параметре *lParam* сообщения [**WM \_ CREATE**](/windows/desktop/winmsg/wm-create) , которое отправляется в элемент управления.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Расширенный шаблон для диалогового окна состоит из заголовка [**длгтемплатикс**](dlgtemplateex.md) , за которым следует структура **длгитемтемплатикс** для каждого элемента управления в диалоговом окне.

Каждая структура **длгитемтемплатикс** должна быть согласована на границе **DWORD** . Массивы **виндовкласс** и **Title** переменной длины должны быть согласованы по границам **слов** . Массив данных создания, если он имеется, должен быть согласован по границе **слова** .

Если в массивах **виндовкласс** и **Title** указываются символьные строки, необходимо использовать строки в Юникоде. Используйте функцию [**MultiByteToWideChar**](/windows/desktop/api/stringapiset/nf-stringapiset-multibytetowidechar) для создания строк Юникода из строк ANSI.

Члены **x**, **y**, **CX** и **CY** задают значения в диалоговых окнах. Эти значения можно преобразовать в единицы экрана (в пикселях) с помощью функции [**мапдиалогрект**](/windows/desktop/api/Winuser/nf-winuser-mapdialogrect) .
