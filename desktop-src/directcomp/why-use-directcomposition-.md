## <a name="create-a-visually-engaging-user-interface"></a>Создание визуально привлекательного пользовательского интерфейса

DirectComposition позволяет объединять и анимировать [*визуальные элементы*](directcomposition-glossary.md) для создания визуально ПРИВЛЕКАТЕЛЬНОГО пользовательского интерфейса для приложений на основе Windows. Это может помочь придать приложению уникальный внешний вид, установив удостоверение, которое будет устанавливаться отдельно от других приложений.

DirectComposition также помогает упростить работу с приложениями. Например, с его помощью можно создавать визуальные подсказки и анимированные переходы, отображающие связи между элементами на экране.

## <a name="enable-rich-smooth-animations-for-your-application"></a>Включение насыщенных и гладких анимаций для приложения

DirectComposition — это высокопроизводительный механизм композиции точечных рисунков, использующий графику с аппаратным ускорением для достижения высокой частоты кадров, что приводит к гладкому и согласованному панорамированию, прокрутке и анимации сложного содержимого. Так как он работает в выделенном потоке, отдельном от потока, который используется для отрисовки пользовательского интерфейса приложения, DirectComposition никогда не может «распрепятствовать» потоку пользовательского интерфейса или помешать приложению рисовать свои элементы пользовательского интерфейса.

## <a name="combine-bitmaps-from-a-variety-of-sources"></a>Объединение растровых изображений из различных источников

Многие приложения на базе Windows имеют пользовательский интерфейс, состоящий из элементов, созданных различными графическими платформами. Например, приложение может использовать Windows Forms для создания строки состояния, Windows интерфейс графических устройств (GDI) для создания содержимого главного окна, Microsoft DirectX для графического содержимого и т. д. DirectComposition позволяет объединять содержимое из разнообразных графических платформ и отображать их в одном и том же дочернем окне верхнего уровня без необходимости беспокоиться о том, что происходит, если содержимое разных платформ перекрывается.

## <a name="memory-savings-though-integration-with-dwm"></a>Экономия памяти при интеграции с DWM

Композиции и анимации, созданные DirectComposition, передаются во встроенный компонент Windows, именуемый [Диспетчер окон рабочего стола (DWM)](/windows/desktop/dwm/dwm-overview) для отрисовки на экране. Поэтому на компьютере не требуются специальные компоненты отрисовки или платформы пользовательского интерфейса.

## <a name="keep-what-you-already-have"></a>Не отключайте уже имеющиеся

Код пользовательского интерфейса в существующем приложении на основе Windows представляет значительные инвестиции. В большинстве случаев DirectComposition позволяет создавать и анимировать существующее содержимое пользовательского интерфейса. Это означает, что вы можете использовать DirectComposition для внесения значительных изменений в пользовательский интерфейс приложения без необходимости вносить значительные изменения в существующую базу кода пользовательского интерфейса.
