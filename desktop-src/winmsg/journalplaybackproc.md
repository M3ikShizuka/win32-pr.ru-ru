# <a name="journalplaybackproc-function"></a>Функция Жаурналплайбаккпрок

## <a name="description"></a>Описание

Определяемая приложением или библиотекой функция обратного вызова, используемая с функцией [сетвиндовшукекс](/windows/desktop/api/winuser/nf-winuser-setwindowshookexw) .
Как правило, приложение использует эту функцию для воспроизведения ряда сообщений мыши и клавиатуры, записанных ранее с помощью процедуры-обработчика **жаурналрекордпрок** .
При условии, что процедура обработчика **жаурналплайбаккпрок** установлена, обычная мышь и ввод с клавиатуры отключаются.

Тип **хукпрок** определяет указатель на эту функцию обратного вызова.
**Жаурналплайбаккпрок** — это заполнитель для определяемого приложением или библиотечного имени функции.

```cpp
LRESULT CALLBACK JournalPlaybackProc(
  _In_ int    code,
       WPARAM wParam,
  _In_ LPARAM lParam
);
```

## <a name="parameters"></a>Параметры

### <a name="code-in"></a>код [в]

Тип: **int**

Код, используемый процедурой-обработчиком для определения способа обработки сообщения.
Если *код* меньше нуля, процедура обработчика должна передать сообщение в функцию [каллнекссукекс](/windows/desktop/api/winuser/nf-winuser-callnexthookex) без дальнейшей обработки и возвращать значение, возвращенное **каллнекссукекс**.
Этот параметр может принимать одно из указанных ниже значений.

| Значение | Значение |
|-------|---------|
| **HC_GETNEXT** 1 | Процедура-обработчик должна скопировать текущее сообщение мыши или клавиатуры в структуру [евентмсг](/windows/desktop/api/winuser/ns-winuser-eventmsg) , на которую указывает параметр *lParam* . |
| **HC_NOREMOVE** 3 | Приложение вызвало функцию [PeekMessage](/windows/desktop/api/winuser/nf-winuser-peekmessagew) с параметром времовемсг, для которого задано значение **PM_NOREMOVE**, что означает, что сообщение не удаляется из очереди сообщений после обработки **PeekMessage** . |
| **HC_NOREMOVE** 2 | Процедура-обработчик должна подготовиться к копированию следующего сообщения мыши или клавиатуры в структуру **евентмсг** , на которую указывает *lParam*. После получения кода **HC_GETNEXT** процедура обработчика должна скопировать сообщение в структуру. |
| **HC_SYSMODALOFF** 5 | Удалено Системное модальное диалоговое окно. Процедура-обработчик должна возобновить воспроизведение сообщений. |
| **HC_SYSMODALON** 4 | Отображается Системное модальное диалоговое окно. Пока диалоговое окно не будет уничтожено, процедура обработчика должна прекращать воспроизведение сообщений. |

### <a name="wparam"></a>wParam

Тип: **wParam**

Этот параметр не используется.

### <a name="lparam"></a>lParam

Тип: **lParam** .

Указатель на структуру **евентмсг** , представляющую сообщение, обрабатываемое процедурой-обработчиком.
Этот параметр допустим только в том случае, если параметр *Code* имеет значение **HC_GETNEXT**.

## <a name="returns"></a>Возвращаемое значение

Тип: **lResult**

Чтобы система ожидала перед обработкой сообщения, возвращаемое значение должно быть временем ожидания (в тактах), которое система должна ожидать.

(Это значение можно вычислить, вычисляя разницу между элементами времени в текущем и предыдущем входных сообщениях.)

Для немедленной обработки сообщения возвращаемое значение должно быть равно нулю. Возвращаемое значение используется только в том случае, если код обработчика имеет **HC_GETNEXT**; в противном случае он игнорируется.

## <a name="remarks"></a>Комментарии

Процедура-обработчик **жаурналплайбаккпрок** должна скопировать входное сообщение в параметр *lParam* .
Сообщение должно быть ранее записано с помощью процедуры-обработчика **жаурналрекордпрок** , которая не должна изменять сообщение.

Для получения одного и того же сообщения процедура обработки может вызываться несколько раз с параметром *Code* , для которого задано **HC_GETNEXT** без промежуточного вызова с *кодом* , равным **HC_SKIP**.

Если *код* **HC_GETNEXT** и возвращаемое значение больше нуля, система приостанавливается на число миллисекунд, указанное возвращаемым значением. При продолжении система снова вызывает процедуру-обработчик с *кодом* , имеющим значение **HC_GETNEXT** , чтобы получить то же сообщение.
Возвращаемое значение этого нового вызова **жаурналплайбаккпрок** должно равняться нулю; в противном случае система вернется в спящий режим на число миллисекунд, указанное возвращаемым значением, вызовите **жаурналплайбаккпрок** еще раз и так далее.
Система не отвечает.

В отличие от большинства других процедур глобального обработчика процедуры-обработчики **жаурналрекордпрок** и **жаурналплайбаккпрок** всегда вызываются в контексте потока, который задает обработчик.

После того как процедура подключения вернет управление в систему, сообщение будет продолжать обрабатываться. Если *код* **HC_SKIP**, процедура обработчика должна подготовиться к возврату следующего записанного сообщения о событии при следующем вызове.

Установите процедуру-обработчик **жаурналплайбаккпрок** , указав тип [WH_JOURNALPLAYBACK](about-hooks.md) и указатель на процедуру обработчика в вызове функции **сетвиндовшукекс** .

Если пользователь нажимает клавиши CTRL + ESC или CTRL + ALT + DEL во время воспроизведения журнала, система останавливает воспроизведение, отсоединяет процедуру воспроизведения журнала и отправляет сообщение [WM_CANCELJOURNAL](wm-canceljournal.md) в приложение ведения журнала.

Если процедура-обработчик возвращает сообщение в диапазоне **WM_KEYFIRST** для **WM_KEYLAST**, применяются следующие условия.

* Элемент **param** структуры **евентмсг** указывает виртуальный код клавиши, который был нажат.
* Элемент **парамх** структуры **евентмсг** указывает код сканирования.
* Невозможно указать число повторов.
Событие всегда принимается для представления одного ключевого события.
