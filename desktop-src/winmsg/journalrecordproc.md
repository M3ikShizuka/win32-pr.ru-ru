# <a name="journalrecordproc-function"></a>Функция Жаурналрекордпрок

## <a name="description"></a>Описание

Определяемая приложением или библиотекой функция обратного вызова, используемая с функцией [сетвиндовшукекс](/windows/desktop/api/winuser/nf-winuser-setwindowshookexw) .
Функция записывает сообщения, которые система удаляет из очереди системных сообщений.
Позже приложение может использовать процедуру-обработчик [жаурналплайбаккпрок](journalplaybackproc.md) для воспроизведения сообщений.

Тип **хукпрок** определяет указатель на эту функцию обратного вызова.
**Жаурналрекордпрок** — это заполнитель для определяемого приложением или библиотечного имени функции.

```cpp
LRESULT CALLBACK JournalRecordProc(
  _In_ int    code,
       WPARAM wParam,
  _In_ LPARAM lParam
);
```

## <a name="parameters"></a>Параметры

### <a name="code-in"></a>код [в]

Тип: **int**

Указывает способ обработки сообщения.
Если *код* меньше нуля, процедура обработчика должна передать сообщение в функцию [каллнекссукекс](/windows/desktop/api/winuser/nf-winuser-callnexthookex) без дальнейшей обработки и возвращать значение, возвращенное **каллнекссукекс**.
Этот параметр может принимать одно из указанных ниже значений.

| Значение | Значение |
|-------|---------|
| **HC_ACTION** 0 | Параметр *lParam* является указателем на структуру [евентмсг](/windows/desktop/api/winuser/ns-winuser-eventmsg) , содержащую сведения об сообщении, удаленном из системной очереди. Процедура-обработчик должна записать содержимое структуры, скопировав их в буфер или файл. |
| **HC_SYSMODALOFF** 5 | Удалено Системное модальное диалоговое окно. Процедура обработчика должна возобновить запись. |
| **HC_SYSMODALON** 4 | Отображается Системное модальное диалоговое окно. Пока диалоговое окно не будет уничтожено, процедура обработчика должна закончить запись. |

### <a name="wparam"></a>wParam

Тип: **wParam**

Этот параметр не используется.

### <a name="lparam-in"></a>lParam [in]

Тип: **lParam** .

Указатель на структуру **евентмсг** , содержащую записываемое сообщение.

## <a name="returns"></a>Возвращаемое значение

Тип: **lResult**

Возвращаемое значение игнорируется.

## <a name="remarks"></a>Remarks

Процедура-обработчик **жаурналрекордпрок** должна копировать, но не изменять сообщения.
После того как процедура подключения вернет управление в систему, сообщение будет продолжать обрабатываться.

Установите процедуру-обработчик **жаурналрекордпрок** , указав тип [WH_JOURNALRECORD](about-hooks.md) и указатель на процедуру обработчика в вызове функции **сетвиндовшукекс** .

Процедура обработчика **жаурналрекордпрок** не обязательно должна находиться в библиотеке динамической компоновки.
Процедура-обработчик **жаурналрекордпрок** может находиться в самом приложении.

В отличие от большинства других процедур глобального обработчика процедуры-обработчики **жаурналрекордпрок** и [жаурналплайбаккпрок](journalplaybackproc.md) всегда вызываются в контексте потока, который задает обработчик.

Приложение, в котором установлена процедура-обработчик **жаурналрекордпрок** , должно отслеживать [VK_CANCEL](/windows/desktop/inputdev/virtual-key-codes) код виртуального ключа (который реализуется как сочетание клавиш CTRL + BREAK на большинстве клавиатур).
Этот код виртуального ключа должен интерпретироваться приложением как сигнал о том, что пользователь хочет отключить запись журнала.
Приложение должно ответить, завершая последовательность записи и удаляя процедуру обработчика **жаурналрекордпрок** .
Удаление важно.
Это предотвращает блокировку системы приложением ведения журнала путем зависания внутри процедуры-ловушки.

Эта роль как сигнал для остановки записи жаурнл означает, что сочетание клавиш CTRL + BREAK не может быть записано самостоятельно.
Поскольку сочетание клавиш CTRL + C не имеет такой роли, как сигнал ведения журнала, его можно записать.
Существует два других сочетания клавиш, которые не могут быть записаны: CTRL + ESC и CTRL + ALT + DEL.
Эти два сочетания клавиш приводят к тому, что система останавливает все операции ведения журнала (запись или воспроизведение), удаляет все обработчики ведения журнала и записывает [WM_CANCELJOURNAL](wm-canceljournal.md) сообщение в приложение ведения журнала.
