# <a name="using-the-windows-app-certification-kit"></a>Использование комплекта сертификации приложений для Windows

Чтобы позволить классу настольных приложений лучше получать сертификат, проверять и тестировать его на компьютере, прежде чем отправлять его для сертификации и получения списка в магазине Windows. Чтобы сертифицировать приложение, необходимо установить и запустить [Комплект сертификации приложений для Windows](/previous-versions//mt637081(v=vs.85)). Дополнительные сведения о конкретных тестах в пакете см. в разделе [тесты набора сертификации приложений Windows](windows-app-certification-kit-tests.md).

Подробное рассмотрение процесса сертификации и использования этого средства в см. в статье [сертификация приложения для настольных систем](windows-certification-portal.md).

Текущий выпуск Windows ACK доступен на 14 языках (чешский, английский, французский, немецкий, итальянский, японский, корейский, польский, португальский (Бразилия), Русский, упрощенный китайский, испанский, традиционный китайский и турецкий).

## <a name="prerequisites"></a>Предварительные условия

Перед установкой подтверждения Windows необходимо установить и запустить операционную систему.

1. Установите и запустите операционную систему, в которой вы разрабатываете приложения для.

-   При разработке приложений для Windows 7 можно установить и запустить Windows 7, Windows 8 или Windows 8.1.
-   При разработке приложения для классических приложений Windows 8 или Windows 8 для настольных устройств можно установить и запустить Windows 8 или Windows 8.1.
-   Если вы разрабатываете приложение Windows 8.1 классическое приложение или устройство Windows 8 Desktop, установите Windows 8.1.

2. Установите [Комплект сертификации приложений для windows 3,3](/previous-versions//mt637081(v=vs.85)), который входит в состав пакета средств разработки программного обеспечения (SDK) для Windows Windows 8.1.

**Примечание.** При установке комплекта сертификации приложений Windows 3,3 или более поздней версии на компьютер заменяется любая ранее установленная версия пакета.

## <a name="instructions-to-run-windows-app-certification-kit-33"></a>Инструкции по запуску комплекта сертификации приложений для Windows 3,3

**Проверка классического приложения с помощью комплекта сертификации приложений для Windows 3,3 в интерактивном режиме**

1.  В меню Пуск найдите пакет Windows App CERT Kit.
2.  В наборе сертификации приложений для Windows щелкните категорию проверки тестов, которую нужно запустить. Если вы проверяете классическое приложение, выберите **проверить классическое приложение**.
3.  На следующем экране перейдите к файлу установки классического приложения, которое требуется проверить.
    -   **Примечание.** При необходимости можно использовать [шаги командной строки](#commandlinesteps) , чтобы включить параметры или параметр установки.
4.  Укажите тип использования приложения и нажмите кнопку **Далее**. Набор сертификации приложений для Windows запускает установку классического приложения с помощью файлов установки, чтобы он мог проверить установку.
5.  Если вам предложено перезагрузить систему для завершения установки, выберите **нет**. Если приложению необходимо установить несколько компонентов или внешние зависимости, тщательно выберите имя приложения. Выбранное здесь имя представляет собой имя, присваиваемое приложению, если оно указано в магазине Windows. После завершения проверки сохраните отчет с именем, присвоенным приложению на шаге 6. Комплект сертификации приложений для Windows создает XML-файл отчета и сохраняет его.
6.  Перейдите в папку, где был сохранен отчет, и откройте его, чтобы просмотреть результаты теста. Если тест не пройден и вы имеете право на отказ, сведения, которые необходимо отправить, перечислены здесь. Необходимо отправить подробное описание для каждого возможного запроса отказа.

**Проверка классического приложения Windows с помощью набора сертификации приложений Windows 3,3 из командной строки**

1.  Перейдите в папку, где был сохранен отчет, и откройте его, чтобы просмотреть результаты теста. В этом списке перечислены неудачные тесты с возможным запросом отказа. Необходимо отправить подробное описание для каждого возможного запроса отказа.
2.  В папке, содержащей комплект сертификации приложений для Windows, введите следующие команды в следующем порядке:

    -   <span id="commandLineSteps"></span><span id="commandlinesteps"></span><span id="COMMANDLINESTEPS"></span>`appcert.exe reset`
    -   `appcert test -apptype desktop -setuppath d:\cdrom\setup.exe  -appusage peruser -reportoutputpath [report file name]`

    где: `[report file name]` — полное имя файла XML, который будет создан пакетом для включения в тестовый отчет.

3.  После завершения теста откройте файл отчета с именем \[ имя файла отчета \] и просмотрите результаты теста.

    **Примечание.** Чтобы получить дополнительные сведения о командной строке комплекта сертификации приложений Windows, введите команду appcert.exe/?

    Набор сертификации приложений для Windows должен быть запущен в контексте активного сеанса пользователя, но вы не можете запускать приложения в неинтерактивном сеансе. Так как пакет обрабатывает маркеры для запуска тестов с административными привилегиями или без них, он также зависит от контекста сеанса пользователя. Пакет можно запустить из службы, но служба должна иметь возможность порождать этот процесс в активном сеансе пользователя.

**Использование комплекта сертификации приложений для Windows для проверки приложений Windows 7**

-   Комплект сертификации приложений для Windows заменил комплект Windows Software Logo. Если вам нужен логотип Windows 7 для вашего приложения, используйте набор сертификации приложений Windows для проверочного тестирования и отчета. В комплекте можно определить, на какой операционной системе он работает, и автоматически запустить ее для приложений Windows 7. Выполните те же действия для проверки приложений Windows 7.

**Отправка на сертификацию**

-   После проверки приложения вы можете отправить его для сертификации с [помощью процесса отправки на портале](https://www.microsoft.com/?ref=go).

## <a name="windows-app-certification-kit-tests"></a>Тесты комплекта сертификации приложений для Windows

Мы изменили комплект, чтобы [тесты подтверждения Windows](windows-app-certification-kit-tests.md) были проще в использовании. Теперь в комплекте есть:

-   Новый упрощенный пользовательский интерфейс
-   Улучшено тестирование нескольких пользователей, которое больше не требует настройки второй учетной записи пользователя.
