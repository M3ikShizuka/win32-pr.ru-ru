# <a name="using-the-windows-app-certification-kit"></a>Использование комплекта сертификации приложений для Windows

чтобы дать вашему классическому приложению возможность получить сертификат, проверить и протестировать его на компьютере, прежде чем отправлять его для сертификации и получения списка в магазине Windows. чтобы сертифицировать приложение, необходимо установить и запустить [комплект сертификации Windows приложений](/previous-versions//mt637081(v=vs.85)). дополнительные сведения о конкретных тестах в пакете см. в статье [Windows приложений для проверки комплекта сертификации](windows-app-certification-kit-tests.md).

Подробное рассмотрение процесса сертификации и использования этого средства в см. в статье [сертификация приложения для настольных систем](windows-certification-portal.md).

текущий выпуск Windows ACK доступен на 14 языках (чешский, английский, французский, немецкий, итальянский, японский, корейский, польский, португальский (бразилия), русский, упрощенный китайский, испанский, традиционный китайский и турецкий).

## <a name="prerequisites"></a>Предварительные требования

перед установкой Windows ACK необходимо установить и запустить операционную систему.

1. Установите и запустите операционную систему, в которой вы разрабатываете приложения для.

-   если вы разрабатываете приложения для Windows 7, то можете установить и запустить Windows 7, Windows 8 или Windows 8.1.
-   если вы разрабатываете Windows 8 классическое приложение или Windows 8 классическое приложение для устройств, вы можете установить и запустить Windows 8 или Windows 8.1.
-   если вы разрабатываете Windows 8.1 классическое приложение или Windows 8 классическое приложение для устройств, установите Windows 8.1.

2. установите [комплект сертификации для Windows приложений 3,3](/previous-versions//mt637081(v=vs.85)), который входит в состав пакета SDK для Windows для Windows 8.1.

**Примечание.** при установке комплекта сертификации приложений Windows 3,3 или более поздней версии на компьютер заменяется любая ранее установленная версия пакета.

## <a name="instructions-to-run-windows-app-certification-kit-33"></a>инструкции по запуску Windows комплекта сертификации для приложений 3,3

**проверка классического приложения с помощью пакета сертификации Windows приложений 3,3 в интерактивном режиме**

1.  в меню найдите набор Windows приложений Cert Kit.
2.  в Windowsном наборе сертификации приложений щелкните категорию проверки тестов, которую нужно запустить. Если вы проверяете классическое приложение, выберите **проверить классическое приложение**.
3.  На следующем экране перейдите к файлу установки классического приложения, которое требуется проверить.
    -   **Примечание.** При необходимости можно использовать [шаги командной строки](#commandlinesteps) , чтобы включить параметры или параметр установки.
4.  Укажите тип использования приложения и нажмите кнопку **Далее**. набор сертификации Windows приложений запускает установку классического приложения с помощью файлов установки, чтобы он мог проверить установку.
5.  Если вам предложено перезагрузить систему для завершения установки, выберите **нет**. Если приложению необходимо установить несколько компонентов или внешние зависимости, тщательно выберите имя приложения. выбранное здесь имя — это имя, присваиваемое приложению, если оно указано в хранилище Windows. После завершения проверки сохраните отчет с именем, присвоенным приложению на шаге 6. комплект сертификации Windows App Kit создает XML-файл отчета и сохраняет его.
6.  Перейдите в папку, где был сохранен отчет, и откройте его, чтобы просмотреть результаты теста. Если тест не пройден и вы имеете право на отказ, сведения, которые необходимо отправить, перечислены здесь. Необходимо отправить подробное описание для каждого возможного запроса отказа.

**проверка приложения Windows desktop с помощью пакета сертификации Windows приложений 3,3 из командной строки**

1.  Перейдите в папку, где был сохранен отчет, и откройте его, чтобы просмотреть результаты теста. В этом списке перечислены неудачные тесты с возможным запросом отказа. Необходимо отправить подробное описание для каждого возможного запроса отказа.
2.  в папке, содержащей комплект сертификации Windows приложений, введите следующие команды в следующем порядке:

    -   <span id="commandLineSteps"></span><span id="commandlinesteps"></span><span id="COMMANDLINESTEPS"></span>`appcert.exe reset`
    -   `appcert test -apptype desktop -setuppath d:\cdrom\setup.exe  -appusage peruser -reportoutputpath [report file name]`

    где: `[report file name]` — полное имя файла XML, который будет создан пакетом для включения в тестовый отчет.

3.  После завершения теста откройте файл отчета с именем \[ имя файла отчета \] и просмотрите результаты теста.

    **Примечание.** чтобы получить дополнительные сведения о командной строке комплекта сертификации приложений Windows, введите команду appcert.exe/?

    набор сертификации приложений Windows должен быть запущен в контексте активного сеанса пользователя, но вы не можете запускать приложения в неинтерактивном сеансе. Так как пакет обрабатывает маркеры для запуска тестов с административными привилегиями или без них, он также зависит от контекста сеанса пользователя. Пакет можно запустить из службы, но служба должна иметь возможность порождать этот процесс в активном сеансе пользователя.

**использование комплекта сертификации Windows приложений для проверки приложений Windows 7**

-   комплект сертификации для Windows приложений заменил комплект Windows Software Logo. если вам нужен логотип Windows 7 для вашего приложения, используйте набор сертификации Windows приложений для проверочного тестирования и отчета. в комплекте можно определить операционную систему, на которой он запущен, и автоматически запустить приложение Windows 7. выполните ту же процедуру для проверки приложений Windows 7.

**Отправка на сертификацию**

-   После проверки приложения вы можете отправить его для сертификации с [помощью процесса отправки на портале](https://www.microsoft.com/?ref=go).
