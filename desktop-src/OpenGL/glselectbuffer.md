# <a name="glselectbuffer-function"></a>Функция Глселектбуффер

Функция **глселектбуффер** устанавливает буфер для значений режима выбора.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glSelectBuffer(
   GLsizei size,
   GLuint  *buffer
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*size* 
</dt> <dd>

Размер *буфера*.

</dd> <dt>

*двойной* 
</dt> <dd>

Возвращает данные выбора.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

| Имя                                                                                                  | Значение                                                                                                                               |
|-------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_недопустимое \_ значение GL**</dt> </dl>     | *Размер* был отрицательным.<br/>                                                                                                       |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана, пока режим рендеринга был GL \_ SELECT.<br/>                                                              |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана между вызовом [**глбегин**](glbegin.md) и соответствующим вызовом [**гленд**](glend.md).<br/> |



## <a name="remarks"></a>Комментарии

Функция **глселектбуффер** имеет два параметра: *buffer* является указателем на массив целых чисел без знака, а *size* указывает размер массива. Параметр *buffer* возвращает значения из стека имен (см. [**глинитнамес**](glinitnames.md), [**гллоаднаме**](glloadname.md), [**глпушнаме**](glpushname.md)), если режимом отрисовки является GL \_ SELECT (см. [**глрендермоде**](glrendermode.md)). Функция **глселектбуффер** должна быть выдана, прежде чем режим выбора будет включен, и она не должна выдаваться, пока для режима отрисовки \_ выбрано значение GL SELECT.

Выделение используется программистом для определения примитивов, отображаемых в некоторой области окна. Регион определяется текущими моделвиев и матрицами перспективы.

В режиме выделения фрагменты пикселей не создаются из растрирования. Вместо этого, если примитив пересекает громкость клипа, определяемую фрустум просмотра и определяемыми пользователем отсечениями, этот примитив приводит к попаданию в выделение. (При использовании многоугольников не происходит никаких попаданий, если многоугольник исключается.) При внесении изменений в стек имен или при вызове [**глрендермоде**](glrendermode.md) запись об попадании копируется в *буфер* , если с момента последнего такого события возникли какие-либо обращения (изменение стека имен или вызов **глрендермоде** ). Запись об попадании состоит из числа имен в стеке имен во время события. , за которым следуют минимальные и максимальные значения глубины всех вершин, которые были пройдены с момента предыдущего события; , за которым следует имя стека имен, сначала имя снизу.

Возвращенные значения глубины сопоставлены таким, что самое большое целое число без знака соответствует глубине координат окна 1,0, а ноль соответствует глубине координат окна 0,0.

При вводе режима выбора внутренний индекс в *буфере* сбрасывается в нуль. При каждом копировании записи попадания в *буфер* индекс увеличивается, чтобы он указывал на ячейку, следующую за концом блока намессат, до следующей доступной ячейки. Значение, если запись о попадании больше, чем количество оставшихся мест в *буфере*, по мере копирования данных, которые можно разместить, и установлен флаг переполнения. Если при копировании записи попаданий стек имен пуст, то эта запись состоит из нуля, за которой следуют минимальные и максимальные значения глубины.

Режим выбора выходит из-за вызова **глрендермоде** с аргументом, ОТЛИЧНЫМ от GL \_ SELECT. Всякий раз, когда **глрендермоде** вызывается, когда режим рендеринга — GL \_ SELECT, он возвращает число записей попаданий, скопированных в *буфер*, сбрасывает флаг переполнения и указатель на буфер выбора и инициализирует пустой стек имен. Если бит переполнения был установлен при вызове **глрендермоде** , возвращается отрицательное число записей попаданий.

Содержимое *буфера* не определено до тех пор, пока не будет вызван [**глрендермоде**](glrendermode.md) с аргументом, отличным от GL \_ SELECT.

**Глбегин** / **гленд** примитивы и вызовы [**глрастерпос**](glrasterpos-functions.md) могут привести к попаданию.

Следующая функция получает сведения, связанные с **глселектбуффер**:

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ \_ Длина стека имен GL \_
