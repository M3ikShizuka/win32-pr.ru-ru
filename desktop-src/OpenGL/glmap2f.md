# <a name="glmap2f-function"></a>Функция glMap2f

Функции [**glMap2d**](glmap2d.md) и **glMap2f** определяют двухмерный оценщик.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glMap2f(
         GLenum  target,
         GLfloat u1,
         GLfloat u2,
         GLint   ustride,
         GLint   uorder,
         GLfloat v1,
         GLfloat v2,
         GLint   vstride,
         GLint   vorder,
   const GLfloat *points
);
```

## <a name="remarks"></a>Remarks

Оценивающие предоставляют способ использования сопоставлений полинома или рационального полинома для создания вершин, нормалй, координат текстуры и цветов. Значения, создаваемые средством оценки, отправляются на дальнейшие этапы обработки OpenGL, как если бы они были представлены с помощью команд [**глвертекс**](glvertex-functions.md), [**глнормал**](glnormal-functions.md), [**глтекскурд**](gltexcoord-functions.md)и [**глколор**](glcolor-functions.md) , за исключением того, что созданные значения не обновляют текущие нормальные, координаты текстуры и цвет.

Все коэффициенты полинома или рациональной полиномы любой степени (вплоть до максимальной степени, поддерживаемой реализацией OpenGL) можно описать с помощью оценивающих. К ним относятся почти все поверхности, используемые в компьютерных графиках, в том числе области B-сплайна, поверхности НУРБС, поверхности Безье и т. д.

Оценивающие определяют поверхности на основе полиномов бивариате Бернстеин. Определить **p** (*u*^,*v*^) как

![Формула, показывающая определение p ().](images/map05.png)

где **R** *ИЖ* — это контрольная точка, () — *это* бернстеинй уровень степени

*n* (*Уордер*  =  *n* + 1)

![Уравнение, показывающее степень полинома Бернстеина n.](images/map06.png)

и () — это *j*-бернстеин полином степени *m* (*вордер*  =  *m* + 1)

![Уравнение, показывающее степень Бернстеинности градуса m.](images/map07.png)

Вспомним, что

![Уравнения, отображающие эквивалентность, равное 1.](images/map08.png)

Функция **glMap2** используется для определения базиса и для указания типа создаваемых значений. После определения можно включить и отключить карту, вызвав [**гленабле**](glenable.md) и **глдисабле** с именем Map, одним из девяти предопределенных значений для *целевого объекта*, описанным выше. Если [**glEvalCoord2**](glevalcoord-functions.md) представляет значения *u* и *v*, то бивариате бернстеин вычисляется с помощью *u*^ и *v*^, где

![Уравнение, показывающее определение ^.](images/map09.png)

и

![Уравнение, показывающее определение v ^.](images/map10.png)

Параметр *Target* представляет собой символьную константу, которая указывает, какой тип контрольных точек предоставляется в *пунктах*, и какие выходные данные формируются при вычислении схемы.

Параметры *устриде*, *Уордер*, *встриде*, *вордер* и *points* определяют адресацию массива для доступа к контрольным точкам. Параметр *points* — это расположение первой контрольной точки, которое занимает одно, два, три или четыре смежных расположения памяти в зависимости от того, какая схема определена. В массиве есть точки управления *Уордер* x *вордер* . Параметр *устриде* сообщает, сколько расположений с плавающей запятой или Double пропускается, чтобы переместить внутренний указатель памяти из точки управления **r** *ИЖ* в контрольную точку **r** <sub>(\ i + 1 \) j</sub>. Параметр *встриде* сообщает, сколько расположений с плавающей запятой или Double пропускается, чтобы переместить указатель внутренней памяти с точки управления **r** *ИЖ* на контрольную точку **r**<sub>i (j \ + 1 \)</sub>.

Как и в случае со всеми командами OpenGL, которые принимают указатели на данные, это так, как будто содержимое *точек* было скопировано с помощью **glMap2** перед возвратом. Изменения содержимого *точек* не оказывают влияния после вызова **glMap2** .
