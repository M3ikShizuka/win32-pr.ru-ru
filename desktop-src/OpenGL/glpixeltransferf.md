# <a name="glpixeltransferf-function"></a>Функция Глпикселтрансферф

Функции **глпикселтрансферф** и [**глпикселтрансфери**](glpixeltransferi.md) задают режимы перемещения пикселей.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glPixelTransferf(
   GLenum  pname,
   GLfloat param
);
```

Эта функция не возвращает значение.

## <a name="remarks"></a>Remarks

Функция **глпикселтрансфер** устанавливает режимы зонной обработки, влияющие на работу последующих [**команд глкопипикселс**](glcopypixels.md), [**glCopyTexImage1D**](glcopyteximage1d.md), [**glCopyTexImage2D**](glcopyteximage2d.md), [**glCopyTexSubImage1D**](glcopytexsubimage1d.md), [**glCopyTexSubImage2D**](glcopytexsubimage2d.md), [**глдравпикселс**](gldrawpixels.md), глреадпикселс [**, glTexImage1D,**](glteximage1d.md) [**glTexImage2D**](glteximage2d.md), [**glTexSubImage1D**](gltexsubimage1d.md)и [**glTexSubImage2D**](gltexsubimage2d.md) . [](glreadpixels.md) Алгоритмы, заданные режимами зонной пересылки, работают на пикселах после того, как они считываются из буфера кадров (**глреадпикселс** и **глкопипикселс**) или распакованы из памяти клиента (**глдравпикселс**, **glTexImage1D** и **glTexImage2D**). Операции перемещения пикселей выполняются в одном и том же порядке, независимо от команды, которая привела к операции с пикселем. Пиксельные режимы хранения ([**глпикселсторе**](glpixelstore-functions.md)) управляют распаковкой пикселей, считываемых из памяти клиента, а также упаковкой пикселей, записываемой обратно в память клиента.

Операции перемещения пикселей обрабатывали четыре основных типа пикселей: *Цвет*, *индекс цвета*, *глубину* и *набор элементов*. Цветовые Пиксели состоят из четырех значений с плавающей запятой с неопределенными значениями мантисса и экспоненты, масштабированием, таким, что 0,0 представляет нулевую интенсивность, а 1,0 — полную интенсивность. Цветовые индексы состоят из одного значения с фиксированной запятой с неуказанной точностью справа от двоичной точки. Пиксели глубины состоят из одного значения с плавающей запятой с неопределенными значениями мантисса и экспоненты, масштабируемым таким, что 0,0 представляет минимальное значение буфера глубины, а 1,0 представляет максимальное значение буфера глубины. Наконец, пикселы трафарета состоят из одного значения с фиксированной запятой с неуказанной точностью справа от двоичной точки.

Ниже перечислены операции перемещения пикселей, выполняемые с четырьмя базовыми типами пикселей.

Функцию [**глпикселтрансферф**](glpixeltransfer.md) можно использовать для задания любого параметра перемещения пикселей. Если тип параметра — Boolean, 0,0 подразумевает значение false, а любое другое значение подразумевает значение true. Если *pname* является целочисленным параметром *, параметр округляется до* ближайшего целого числа.

Аналогичным образом **глпикселтрансфери** можно использовать для установки любого из параметров перемещения пикселей. Для логических параметров задано значение false, если *параметр param* равен 0, в противном случае — значение true. Параметр *param* преобразуется в плавающую точку перед тем, как он будет назначен параметрам с реальными значениями.

Если команда [**глдравпикселс**](gldrawpixels.md), [**глреадпикселс**](glreadpixels.md), [**глкопипикселс**](glcopypixels.md), [**glTexImage1D**](glteximage1d.md)или [**glTexImage2D**](glteximage2d.md) помещается в список отображаемых списков (см. [**глневлист**](glnewlist.md) и [**глкалллист**](glcalllist.md)), то используются параметры режима зонной пересылки, применяемые при *выполнении* списка вывода. Они могут отличаться от параметров, когда команда была скомпилирована в список отображений.
