# <a name="glvertexpointer-function"></a>Функция Глвертекспоинтер

Функция **глвертекспоинтер** определяет массив данных вершин.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glVertexPointer(
         GLint   size,
         GLenum  type,
         GLsizei stride,
   const GLvoid  *pointer
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*size* 
</dt> <dd>

Число координат на вершину. Значение параметра *size* должно быть равно 2, 3 или 4.

</dd> <dt>

*type* 
</dt> <dd>

Тип данных каждой координаты в массиве, используя следующие символьные константы: GL \_ Short, GL \_ int, GL \_ float и GL \_ Double.

</dd> <dt>

*шага* 
</dt> <dd>

Смещение в байтах между последовательными вершинами. Если *шаг* равен нулю, вершины плотно упаковываются в массив.

</dd> <dt>

*указатель* 
</dt> <dd>

Указатель на первую координату первой вершины в массиве.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

| Имя                                                                                              | Значение                                       |
|---------------------------------------------------------------------------------------------------|-----------------------------------------------|
| <dl> <dt>**\_недопустимое \_ значение GL**</dt> </dl> | *Размер* не был 2, 3 или 4. <br/>        |

| <dl> <dt>**\_недопустимое \_ значение GL**</dt> </dl> | *шаг* или *число* были отрицательными. <br/> |



## <a name="remarks"></a>Комментарии

Функция **глвертекспоинтер** задает расположение и данные массива координат вершин, используемых при подготовке к просмотру. Параметр *size* задает количество координат на вершину. Параметр *типа* задает тип данных для каждой координаты вершины. Параметр *stride* определяет смещение в байтах от одной вершины к следующей, позволяя упаковать вершин и атрибуты в одном массиве или хранилище в отдельных массивах. В некоторых реализациях хранение вершин и атрибутов в одном массиве может оказаться более эффективным, чем использование отдельных массивов (см. [**глинтерлеаведаррайс**](glinterleavedarrays.md)).

Массив вершин включается при указании \_ \_ константы массива вершин GL с [**гленаблеклиентстате**](glenableclientstate.md). Если этот параметр включен, [**глдраваррайс**](gldrawarrays.md), [**глдравелементс**](gldrawelements.md)и [**гларрайелемент**](glarrayelement.md) используют массив вершин. По умолчанию массив вершин отключен.

Невозможно включить **глвертекспоинтер** в списки вывода.

При указании массива вершин с помощью **глвертекспоинтер** значения всех параметров массива вершин сохраняются в состоянии на стороне клиента, а статические элементы массива могут быть кэшированы. Так как параметры массива вершин являются состояниями на стороне клиента, их значения не сохраняются и не восстанавливаются [**глпушаттриб**](glpushattrib.md) и **глпопаттриб**.

Несмотря на то что ошибка не возникает при вызове **глвертекспоинтер** в парах [**глбегин**](glbegin.md) и [**гленд**](glend.md) , результаты не определены.

Следующие функции извлекают сведения, относящиеся к **глвертекспоинтер**:

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ \_ Размер массива \_ вершин GL

**глжет** с аргументом "шаг" для \_ массива вершин GL \_ \_

**глжет** с аргументом \_ \_ число массивов вершин GL \_

**глжет** с аргументом \_ \_ тип массива \_ вершин GL

[**глжетпоинтерв**](glgetpointerv.md) с аргументом в качестве \_ указателя на массив вершин GL \_ \_

[**глисенаблед**](glisenabled.md) с аргументом " \_ массив вершин GL" \_
