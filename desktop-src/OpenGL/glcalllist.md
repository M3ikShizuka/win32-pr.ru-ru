# <a name="glcalllist-function"></a>Функция Глкалллист

Функция **глкалллист** выполняет список отображений.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glCallList(
   GLuint list
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*list* 
</dt> <dd>

Целочисленное имя выполняемого списка отображаемых данных.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Комментарии

Вызов функции **глкалллист** начинает выполнение именованного списка экранов. Функции, сохраненные в списке отображений, выполняются по порядку, как если бы они были вызваны без использования списка вывода. Если *список* не был определен как список вывода, **глкалллист** игнорируется.

Функция **глкалллист** может отображаться в списке отображения. Чтобы избежать возможности бесконечной рекурсии, полученной в результате вызова друг друга из списков вывода, на уровне вложенности списков отображаемых данных во время выполнения списка отображений размещается ограничение. Это ограничение по крайней мере 64, но зависит от реализации.

Состояние OpenGL не сохраняется и не восстанавливается в результате вызова **глкалллист**. Поэтому изменения, внесенные в состояние OpenGL во время выполнения списка отображения, остаются после завершения выполнения списка отображения. Чтобы сохранить состояние OpenGL в вызовах **глкалллист** , используйте [**глпушаттриб**](glpushattrib.md), [**глпопаттриб**](glpopattrib.md), [**глпушматрикс**](glpushmatrix.md)и [**глпопматрикс**](glpopmatrix.md).

Вы можете выполнять списки вывода между вызовом [**глбегин**](glbegin.md) и соответствующим вызовом [**гленд**](glend.md), если в списке отображаются только те функции, которые разрешены в этом интервале.

Следующие функции извлекают сведения, относящиеся к **глкалллист**:

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ , \_ вложенным в параметр GL Max List \_

[**глислист**](glislist.md)
