# <a name="glpointsize-function"></a>Функция Глпоинтсизе

Функция **глпоинтсизе** задает диаметр растровых точек.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glPointSize(
   GLfloat size
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*size* 
</dt> <dd>

Диаметр растровых точек. Значение по умолчанию — 1,0.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="error-codes"></a>Коды ошибок

Функция [**глжетеррор**](glgeterror.md) может получить следующие коды ошибок.



| Имя                                                                                                  | Значение                                                                                                                               |
|-------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_недопустимое \_ значение GL**</dt> </dl>     | *Размер* меньше или равен нулю.<br/>                                                                                     |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана между вызовом [**глбегин**](glbegin.md) и соответствующим вызовом [**гленд**](glend.md).<br/> |



## <a name="remarks"></a>Комментарии

Функция **глпоинтсизе** задает растровый диаметр как псевдонимов, так и сглаженных точек. Использование кегля, отличного от 1,0, имеет различные эффекты в зависимости от того, включено ли сглаживание точек. Сглаживание точек управляется путем вызова [**гленабле**](glenable.md) и **глдисабле** с аргументом GL в виде \_ точки \_ .

Если сглаживание точки отключено, фактический размер определяется путем округления предоставленного размера до ближайшего целого числа. (Если результат округления имеет значение 0, то это будет так, как если бы размер точки был равен 1.) Если округленный размер нечетный, то Центральная точка (*x*,*y*) фрагмента пикселя, представляющая точку, будет вычисляться как

(*x*<sub>w</sub> + 0,5, *y*<sub>w</sub> +. 5)

где *w* подстрочные индексы обозначают координаты окна. Все пиксели, находящиеся в квадратной сетке скругленного размера, расположенного в центре (*x*,*y*), составляют фрагмент. Если размер четное, Центральная точка

(*x*<sub>w</sub> + 0,5, *y*<sub>w</sub> +. 5)

а центрированные центры фрагментов — это координаты окна половинного целого числа в квадрате округленного размера, расположенного в центре (*x*,*y*). Все фрагменты пикселов, созданные при растрировании точки нонантиалиасед, имеют одинаковые связанные данные. вершина, соответствующая точке.

Если включено сглаживание, то точечная текстура создает фрагмент для каждого квадрата пикселей, пересекающего область, которая имеет диаметр, равный текущему размеру точки и центрированную в точках (*x*<sub>w</sub> ,*y*<sub>w</sub> ). Значение покрытия для каждого фрагмента является координатной областью окна пересечения круговой области с соответствующим квадратом пикселей. Это значение сохраняется и используется на заключительном шаге растрирования. Данные, связанные с каждым фрагментом, — это данные, связанные с растровой точкой.

При включении сглаживания точек поддерживаются не все размеры. Если запрошен неподдерживаемый размер, используется ближайший поддерживаемый размер. Гарантируется, что поддерживается только размер 1,0. другие зависят от реализации. Диапазон поддерживаемых размеров и разница между поддерживаемыми размерами в пределах диапазона можно запросить, вызвав [**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументами \_ \_ диапазон размера главной точки \_ и \_ \_ гранулярность размера точки GL \_ .

Размер точки, указанный параметром **глпоинтсизе** , всегда возвращается, \_ когда \_ запрашивается размер точки GL. Фиксация и округление для точек с псевдонимами и сглаживания не влияют на указанное значение.

Размер точки без сглаживания может относиться к максимуму, зависящему от реализации. Хотя это максимальное значение не может быть запрошено, оно должно быть не меньше максимального значения для точек сглаживания, округленное до ближайшего целого значения.

Следующие функции извлекают сведения, относящиеся к **глпоинтсизе**:

**глжет** с аргументом \_ \_ диапазона размера \_ точки GL

**глжет** с аргументом \_ \_ гранулярность размера \_ точки GL
