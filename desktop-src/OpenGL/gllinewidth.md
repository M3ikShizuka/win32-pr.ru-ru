# <a name="gllinewidth-function"></a>Функция Гллиневидс

Функция **гллиневидс** задает ширину растровых линий.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glLineWidth(
   GLfloat width
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*width* 
</dt> <dd>

Ширина растровых линий. Значение по умолчанию — 1,0.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="error-codes"></a>Коды ошибок

| Имя                                                                                                  | Значение                                                                                                                               |
|-------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_недопустимое \_ значение GL**</dt> </dl>     | *Ширина* меньше или равна нулю.<br/>                                                                                    |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана между вызовом [**глбегин**](glbegin.md) и соответствующим вызовом [**гленд**](glend.md).<br/> |



## <a name="remarks"></a>Комментарии

Функция **гллиневидс** задает растровую ширину для линий с псевдонимами и сглаженными. Использование толщины линии, отличной от 1,0, имеет различные эффекты в зависимости от того, включено ли сглаживание строк. Сглаживание линии управляется путем вызова [**гленабле**](glenable.md) и **глдисабле** с аргументом GL \_ Line \_ Smooth.

Если сглаживание линии отключено, фактическая ширина определяется путем округления предоставленной ширины к ближайшему целому числу. (Если округление приводит к значению 0,0, то это так, как если бы толщина линии была 1,0). Если \| ? x \|  =  \| ? y \| , *i* пикселей заполняются в каждом столбце, который является скругленным, где *i* — округленное значение *Width*. В противном случае *i* пикселей заполняют каждую строку, которая является растрирования.

Если сглаживание линии включено, то при растрировании строки создается фрагмент для каждого квадрата пикселей, пересекающего область, которая имеет ширину прямоугольника, равную текущей ширине линии, длина равна фактической длине линии и по центру для математического сегмента линии. Значение покрытия для каждого фрагмента является координатной областью окна пересечения прямоугольной области с соответствующим квадратом пикселей. Это значение сохраняется и используется на заключительном шаге растрирования.

Если включено сглаживание строк, могут поддерживаться не все значения ширины. Если запрошена неподдерживаемая ширина, используется Ближайшая поддерживаемая ширина. Гарантируется, что поддерживается только ширина 1,0. другие зависят от реализации. Диапазон поддерживаемых ширины и разница между поддерживаемыми ширинами в пределах диапазона можно запросить, вызвав **глжет** с аргументами GL \_ Line \_ толщина \_ диапазона и \_ \_ \_ степенью детализации толщины линии GL.

Ширина линии, заданная параметром **гллиневидс** , всегда возвращается \_ при \_ запросе ширины линии GL. Фиксация и округление для строк с псевдонимами и сглаженных линий не влияют на указанное значение.

Ширина линии без сглаживания может быть перенесена в максимальное значение, зависящее от реализации. Хотя это максимальное значение не может быть запрошено, оно должно быть не меньше максимального значения для сглаженных строк, округленное до ближайшего целого значения.
