# <a name="glrendermode-function"></a>Функция Глрендермоде

Функция **глрендермоде** устанавливает режим растрирования.

## <a name="syntax"></a>Синтаксис


```C++
GLint WINAPI glRenderMode(
   GLenum mode
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*mode* 
</dt> <dd>

Режим растрирования. Принимаются следующие три значения. Значение по умолчанию — « \_ рендеринг GL».



| Значение                                                                                                                                                   | Значение                                                                                                                                                                                                                                                                                                                                                                       |
|---------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="GL_RENDER"></span><span id="gl_render"></span><dl> <dt>**\_рендеринг GL**</dt> </dl>       | Режим рендеринга. Примитивы растрируются, создавая фрагменты пикселей, которые записываются в буфера кадров. Это нормальный режим, а также режим по умолчанию.<br/>                                                                                                                                                                                                      |
| <span id="GL_SELECT"></span><span id="gl_select"></span><dl> <dt>**\_Выбор GL**</dt> </dl>       | Режим выбора. Фрагменты пикселей не создаются и не вносятся изменения в содержимое буфера кадров. Вместо этого в буфере SELECT возвращается запись с именами примитивов, которые были бы вырисованы, если был возвращен режим рендеринга GL \_ , который должен быть создан (см. [**глселектбуффер**](glselectbuffer.md)) перед входом в режим выбора.<br/>               |
| <span id="GL_FEEDBACK"></span><span id="gl_feedback"></span><dl> <dt>**\_обратная связь ГК**</dt> </dl> | Режим обратной связи. Фрагменты пикселей не создаются и не вносятся изменения в содержимое буфера кадров. Вместо этого координаты и атрибуты вершин, которые были бы прорисованы, были \_ возвращены в буфер обратной связи, который должен быть создан (см. [**глфидбаккбуффер**](glfeedbackbuffer.md)) перед входом в режим обратной связи.<br/> |



 

</dd> </dl>

| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана с аргументом GL \_ SELECT перед вызовом [**глселектбуффер**](glselectbuffer.md) по крайней мере один раз.<br/>       |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана с аргументом \_ Feedback GL до того, как [**глбидбаккбуффер**](glfeedbackbuffer.md) был вызван хотя бы один раз.<br/> |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Функция была вызвана между вызовом [**глбегин**](glbegin.md) и соответствующим вызовом [**гленд**](glend.md).<br/>       |



## <a name="remarks"></a>Комментарии

Функция **глрендермоде** принимает один аргумент *mode*, который может принимать одно из трех предварительно определенных значений.

Возвращаемое значение функции **глрендермоде** определяется режимом рендеринга во время вызова **глрендермоде** , а не в *режиме*. Ниже приведены значения, возвращаемые для трех режимов рендеринга.



| Значение        | Значение                                                                 |
|--------------|-------------------------------------------------------------------------|
| \_рендеринг GL   | Ноль.                                                                   |
| \_Выбор GL   | Число записей об попадании, передаваемых в буфер SELECT.             |
| \_обратная связь ГК | Количество значений (не вершин), передаваемых в буфер обратной связи. |



 

Дополнительные сведения о выборе и операциях обратной связи см. в разделе [**глселектбуффер**](glselectbuffer.md) и [**глфидбаккбуффер**](glfeedbackbuffer.md) .

Если возникает ошибка, **глрендермоде** возвращает ноль независимо от текущего режима рендеринга.

Следующая функция получает сведения, связанные с **глрендермоде**:

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ режим рендеринга GL \_
