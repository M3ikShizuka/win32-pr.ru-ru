# <a name="execution-model"></a>Модель выполнения

Для интерпретации команд OpenGL используется модель "клиент-сервер". Код приложения (клиент) выдает команды, интерпретируемые и обрабатываемые OpenGL (сервером). Сервер может и не быть работающим на том же компьютере, что и клиент. В этом смысле OpenGL является прозрачным для сети. Сервер может поддерживать несколько контекстов OpenGL, каждый из которых является инкапсулированным состоянием OpenGL. Клиент может подключаться к одному из этих контекстов. Необходимый сетевой протокол можно реализовать, добавив уже существующий протокол (например, для системы X Window) или используя независимый протокол. Для получения вводимых пользователем данных команды OpenGL не предоставляются.

Система Window, которая выделяет ресурсы буфера кадров, в конечном итоге управляет влиянием команд OpenGL на буфера кадров. Система Window:

-   Определяет, какие части буфера кадров OpenGL могут быть доступны в любой конкретный момент времени.
-   Взаимодействует с OpenGL, как структурированы эти части.

Поэтому нет команд OpenGL для настройки буфера кадров или инициализации OpenGL. Настройка буфера кадров выполняется вне OpenGL в сочетании с системным окном; Инициализация OpenGL выполняется, когда система Window выделяет окно для отрисовки OpenGL.
