# <a name="glreadbuffer-function"></a>Функция Глреадбуффер

Функция **глреадбуффер** выбирает источник цветового буфера для пикселей.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glReadBuffer(
   GLenum mode
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*mode* 
</dt> <dd>

Буфер цвета. Допустимые значения: GL \_ спереди \_ слева, GL \_ спереди справа, GL \_ назад, \_ \_ GL \_ назад, GL \_ \_ спереди, GL \_ назад, ГК \_ Left, GL \_ справа и GL \_ AUX *i*,  где находится между 0 и вспомогательными \_ \_ буферами ГК 1.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Комментарии

Функция **глреадбуффер** задает буфер цвета в качестве источника для последующих команд [**глреадпикселс**](glreadpixels.md) и [**глкопипикселс**](glcopypixels.md) . Параметр *mode* принимает одно из двенадцати или более предопределенных значений. (GL \_ AUX0 через GL \_ AUX3 определяются всегда.) В полностью настроенной системе, на главной странице GL, \_ \_ левой и главной части GL \_ все имя передний \_ левый буфер, GL \_ переднего плана \_ и \_ правое название в ГК, а также \_ \_ \_ имя заднего левого буфера.

Нестерео конфигурации с двойной буферизацией имеют только внешний левый и задний левый буфер. Однобуферные конфигурации имеют передний левый и передний правый буфер, если стерео, и только передний левый буфер, если не стерео. Указание несуществующего буфера для **глреадбуффер** является ошибкой.

По умолчанию *режим* — GL \_ спереди в однобуферных конфигурациях, а GL — \_ в двойных буферизованных конфигурациях.
