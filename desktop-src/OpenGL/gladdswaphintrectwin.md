# <a name="gladdswaphintrectwin-function"></a>Функция Гладдсвафинтректвин

Функция обратного вызова **гладдсвафинтректвин** задает набор прямоугольников, которые должны быть скопированы с помощью [**свапбуфферс**](/windows/desktop/api/wingdi/nf-wingdi-swapbuffers).

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glAddSwapHintRectWIN(
   GLint   x,
   GLint   y,
   GLsizei width,
   GLsizei height
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*x* 
</dt> <dd>

Координата *x*(в координатах окна) левого нижнего угла прямоугольника области подсказки.

</dd> <dt>

*y* 
</dt> <dd>

Координата *y*(в координатах окна) левого нижнего угла прямоугольника области подсказки.

</dd> <dt>

*width* 
</dt> <dd>

Ширина прямоугольника области подсказки.

</dd> <dt>

*height* 
</dt> <dd>

Высота прямоугольника области подсказки.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Комментарии

Функция **гладдсвафинтректвин** ускоряет анимацию, уменьшая объем перерисовки кадров. С помощью **гладдсвафинтректвин** вы указываете набор прямоугольных областей, которые необходимо скопировать при вызове [**свапбуфферс**](/windows/desktop/api/wingdi/nf-wingdi-swapbuffers). Если не указать ни одного прямоугольника с **гладдсвафинтректвин** перед вызовом **свапбуфферс**, весь буфера кадров будет заменен. Использование **гладдсвафинтректвин** для копирования только измененных частей буфера может значительно повысить производительность **свапбуфферс**, особенно при реализации **свапбуфферс** в программном обеспечении.

Функция **гладдсвафинтректвин** Добавляет прямоугольник в область подсказки. Если \_ \_ ЗАДАН флаг ПФД swap Copy структуры формата пикселей [**пикселформатдескриптор**](/windows/win32/api/wingdi/ns-wingdi-pixelformatdescriptor) , **свапбуфферс** использует этот регион для обрезки копирования заднего буфера в передний буфер. Вы не укажете \_ копию ПФД Swap \_ ; она устанавливается с поддержкой оборудования. Область подсказки удаляется после каждого вызова **свапбуфферс**. В некоторых конфигурациях оборудования **свапбуфферс** может игнорировать область подсказки и переключить весь буфер. **Свапбуфферс** реализуется системой, а не приложением.

OpenGL поддерживает отдельную область подсказки для каждого окна. При вызове **гладдсвафинтректвин** для любого контекста отрисовки, связанного с окном, прямоугольники подсказки объединяются в один регион.

Вызовите **гладдсвафинтректвин** с ограничивающим прямоугольником для каждого объекта, рисуемого для рамки, и для каждого прямоугольника, очищенного для стирания предыдущих объектов Frame.

> [!Note]  
> Функция **гладдсвафинтректвин** — это функция расширения, которая не является частью стандартной библиотеки OpenGL, но является частью \_ \_ расширения подсказок для функции подкачки GL \_ . Чтобы проверить, поддерживает ли ваша реализация OpenGL **гладдсвафинтректвин**, вызовите **глжетстринг**( \_ расширения GL). Если он возвращает \_ Указание на выигрывает \_ выставляемую \_ подсказку, **гладдсвафинтректвин** поддерживается. Чтобы получить адрес функции расширения, вызовите **вглжетпрокаддресс**.
