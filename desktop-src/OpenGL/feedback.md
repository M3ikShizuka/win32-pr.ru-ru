# <a name="feedback-mode"></a>Режим обратной связи

В режиме обратной связи каждый объект-примитив, который должен быть растровым, создает блок значений, которые копируются в массив обратной связи. Укажите этот массив с помощью [**глфидбаккбуффер**](glfeedbackbuffer.md), который необходимо вызвать перед переводом OpenGL в режим обратной связи. Каждый блок значений начинается с кода, указывающего на тип-примитив, за которым следуют значения, описывающие вершины примитива и связанные данные. Записи также записываются для точечных рисунков и пиксельных прямоугольников. Не гарантируется, что значения будут записываться в массив обратной связи до тех пор, пока не будет вызван [**глрендермоде**](glrendermode.md) для получения из режима обратной связи OpenGL. [**Глпасссраугх**](glpassthrough.md) можно использовать для предоставления маркера, возвращаемого в режиме обратной связи, как если бы он был примитивом.
