# <a name="glstencilop-function"></a>Функция ГлстенЦилоп

Функция **глстенЦилоп** задает действия теста набора элементов.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glStencilOp(
   GLenum fail,
   GLenum zfail,
   GLenum zpass
);
```

## <a name="remarks"></a>Remarks

Набор элементов, например *z*-буферизация, включает и отключает рисование на уровне отдельных пикселов. Вы рисуете на плоскости трафаретов с помощью примитивов рисования OpenGL, затем визуализируете геометрию и изображения, используя плоскости трафаретов для маскирования частей экрана. Набор элементов обычно используется в алгоритмах многопроходной отрисовки для достижения специальных эффектов, таких как декалс, структурирование и конструктивностьная визуализация геометрических объектов.

Условие теста трафарета позволяет исключить пиксель на основе результата сравнения значения в буфере шаблона и ссылочного значения. Тест включается с вызовами [**гленабле**](glenable.md) и [**глдисабле**](gldisable.md) с аргументом \_ Test трафарета GL \_ и управляется с помощью [**глстенЦилфунк**](glstencilfunc.md).

Функция **глстенЦилоп** принимает три аргумента, которые указывают, что происходит с сохраненным значением набора элементов при включенном наборе элементов. Если проверка набора элементов завершается неудачно, в буферы цветов или глубины пикселя не вносятся изменения, а при *сбое* указывается, что происходит с содержимым буфера трафарета.

Значения буфера наборов элементов рассматриваются как целые числа без знака. При увеличении и уменьшении значения записываются в 0 и 2 *n* 1, где *n* — значение, возвращаемое запросом \_ битов трафарета GL \_ .

Два других аргумента для **глстенЦилоп** указывают действия буфера набора элементов, если последующие тесты в буфере глубины выполнены успешно (*зпасс*) или Fail (*зфаил*). (См. [**глдепсфунк**](gldepthfunc.md).) Они указываются с использованием тех же шести символьных констант, что и *Fail*. Обратите внимание, что *зфаил* игнорируется, если буфер глубины отсутствует или если буфер глубины не включен. В таких случаях команда *Fail* и *зпасс* Указание действия трафарета при сбое и прохождении теста трафарета соответственно.

Изначально Проверка набора элементов отключена. Если буфер трафарета отсутствует, изменение трафарета не может быть выполнено и так, как если бы набор элементов всегда прошел проверку, независимо от любого вызова **глстенЦилоп**.

Следующие функции извлекают сведения, относящиеся к **глстенЦилоп**:

[**сбой глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ трафарета GL \_

**глжет** с аргументом \_ \_ \_ этап глубины прохода в трафарете \_

**сбой глжет** с аргументом \_ \_ \_ глубины этапа для шаблона GL \_

**глжет** с аргументом \_ биты шаблона GL \_

[**глисенаблед**](glisenabled.md) с аргументом \_ Test трафарета GL \_
