# <a name="glfrustum-function"></a>Функция Глфрустум

Функция **глфрустум** Умножает текущую матрицу на матрицу перспективы.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glFrustum(
   GLdouble left,
   GLdouble right,
   GLdouble bottom,
   GLdouble top,
   GLdouble zNear,
   GLdouble zFar
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*слева* 
</dt> <dd>

Координата вертикальной плоскости обрезки слева.

</dd> <dt>

*Правильно* 
</dt> <dd>

Координата для правой вертикальной плоскости обрезки.

</dd> <dt>

*Нижний* 
</dt> <dd>

Координата для нижней горизонтальной плоскости обрезки.

</dd> <dt>

*В начало* 
</dt> <dd>

Координата для нижней горизонтальной плоскости обрезки.

</dd> <dt>

*знеар* 
</dt> <dd>

Расстояния на плоскости обрезки близкой глубины. Должно быть положительным.

</dd> <dt>

*зфар* 
</dt> <dd>

Расстояния до дальней обрезанной плоскости. Должно быть положительным.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Remarks

Функция **глфрустум** описывает матрицу перспективы, которая создает перспективную проекцию. Параметры (*левый*, *Нижний*, *знеар*) и (*правый*, *верхний*, *знеар*) указывают точки на близкой плоскости, которые сопоставлены с нижним левым и верхним правым углами окна соответственно, предполагая, что глаз находится в (0, 0, 0). Параметр *зфар* задает расположение дальней плоскости. Оба *знеар* и *зфар* должны быть положительными. Соответствующая матрица показана на рисунке ниже.

![Диаграмма, показывающая матрицу перспективы, которая создает перспективную проекцию.](images/frust01.png)![Уравнения, отображающие функцию Глфрустум, описывающую матрицу перспективы.](images/frust02.png)

Функция **глфрустум** Умножает текущую матрицу на эту матрицу с результатом замены текущей матрицы. То есть, если M — это текущая матрица, а F — матрица фрустум перспективы, **глфрустум** заменяет M на m F.

Используйте [**глпушматрикс**](glpushmatrix.md) и [**глпопматрикс**](glpopmatrix.md) для сохранения и восстановления текущего стека матриц.

На точность буфера глубины влияют значения, указанные для *знеар* и *зфар*. Чем больше соотношение *зфар* к *знеар* , тем менее эффективный буфер глубины будет различен между поверхностями, расположенными рядом друг с другом. If

![Уравнение, показывающее отношение далеко к ближайшему.](images/frust03.png)

примерно все биты *журнала* с точностью 2 (*r*) теряют точность буфера глубины. Так как *r* приближается к бесконечности, так как *знеар* приближается к нулю, никогда не следует устанавливать *знеар* в ноль.
