# <a name="glgeterror-function"></a>Функция Глжетеррор

Функция **глжетеррор** возвращает сведения об ошибке.

## <a name="syntax"></a>Синтаксис


```C++
GLenum WINAPI glGetError(void);
```



## <a name="parameters"></a>Параметры

У этой функции нет параметров.

## <a name="return-value"></a>Возвращаемое значение

Функция **глжетеррор** возвращает один из следующих кодов ошибок.



| Код возврата                                                                                           | Описание                                                                                                                                                                  |
|-------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_недействительное \_ перечисление GL**</dt> </dl>      | Для перечислимого аргумента задано неприемлемое значение. Функция, вызвавшая ошибку, игнорируется, не имеет побочного действия, Кроме установки флага ошибки.<br/>         |
| <dl> <dt>**\_недопустимое \_ значение GL**</dt> </dl>     | Числовой аргумент выходит за пределы допустимого диапазона. Функция, вызвавшая ошибку, игнорируется, не имеет побочного действия, Кроме установки флага ошибки.<br/>                                    |
| <dl> <dt>**\_Недопустимая \_ Операция GL**</dt> </dl> | Указанная операция запрещена в текущем состоянии. Функция, вызвавшая ошибку, игнорируется, не имеет побочного действия, Кроме установки флага ошибки.<br/>           |
| <dl> <dt>**\_нет \_ ошибок в ГК**</dt> </dl>          | Ошибки не записаны. Значение этой символьной константы гарантированно равно нулю.<br/>                                                                         |
| <dl> <dt>**\_переполнение стека GL \_**</dt> </dl>    | Эта функция вызовет переполнение стека. Функция, вызвавшая ошибку, игнорируется, не имеет побочного действия, Кроме установки флага ошибки.<br/>                            |
| <dl> <dt>**\_ \_ потеря значимости СТЕКа GL**</dt> </dl>   | Эта функция вызовет потерю стека. Функция, вызвавшая ошибку, игнорируется, не имеет побочного действия, Кроме установки флага ошибки.<br/>                           |
| <dl> <dt>**\_ \_ \_ неиспользуемая память в GL**</dt> </dl>    | Недостаточно памяти для выполнения функции. Состояние OpenGL не определено, за исключением состояния флагов ошибок, после записи этой ошибки.<br/> |



 

Обратите внимание, что **глжетеррор** возвращает \_ недопустимую операцию GL, \_ если она вызывается между вызовом [**Глбегин**](glbegin.md) и соответствующим вызовом [**гленд**](glend.md).

## <a name="remarks"></a>Комментарии

Каждой обнаруживаемой ошибке присваивается числовой код и символьное имя. При возникновении ошибки флагу ошибки присваивается соответствующее значение кода ошибки. Никакие другие ошибки не записываются до вызова **глжетеррор** , возвращается код ошибки, а флаг сбрасывается в значение \_ No No \_ Error. Если вызов **глжетеррор** возвращает значение No GL, ошибка не обнаружена \_ \_ с момента последнего вызова **Глжетеррор** или с момента инициализации OpenGL.

Для поддержки распределенных реализаций может существовать несколько флагов ошибок. Если какой-либо из отдельных флагов ошибок записал ошибку, возвращается значение этого флага и этот флаг сбрасывается до значения GL \_ без \_ ошибок при вызове **глжетеррор** . Если ошибка была записана более чем одним флагом, **глжетеррор** возвращает и очищает произвольное значение флага ошибки. Если все флаги ошибок должны быть сброшены, следует всегда вызывать **глжетеррор** в цикле до тех пор, пока не будет возвращена \_ \_ Ошибка GL.

Изначально все флаги ошибок имеют значение \_ No No \_ Error.

Если установлен флаг ошибки, результаты операции OpenGL не определяются, только если \_ произошла незаполненная \_ \_ память. Во всех остальных случаях функция, создающая ошибку, игнорируется и не влияет на содержимое состояния OpenGL или буфера кадров.
