# <a name="porting-polygons-and-quadrilaterals"></a>Перенос многоугольников и Куадрилатералс

При переносе многоугольников и куадрилатералс учитывайте следующие моменты:

-   Прямой эквивалент для **вогнутых участков**(**true**) отсутствует. Вместо этого можно использовать подпрограммы тесселяции в GLU, описанные в разделе [тесселяция многоугольников](tessellating-polygons.md).
-   Режимы многоугольников задаются по-разному.
-   Эти функции рисования многоугольников не имеют прямых эквивалентов в OpenGL: семейства подпрограмм **поли** ; Семейство подпрограмм **полф** ; **ПМВ**, **Народно** и **пклос**; **рпмв** и **рпдр**; **сплф**; и **спклос**.

Если в коде в IRI используется эти функции, необходимо переписать код с помощью [**глбегин**](glbegin.md)( \_ многоугольник GL).

|                          | **глбегин**( \_ четыре **неглендных** )<br/>                       | Интерпретирует четверные вершины как куадрилатералс.    |

## <a name="porting-polygon-modes"></a>Перенос режимов многоугольников

Функция OpenGL [**глполигонмоде**](glpolygonmode.md) позволяет указать, к какой стороне многоугольника применяется режим (задний или передний). Синтаксис:

``` syntax
void glPolygonMode( GLenum face, GLenum mode ); 
 
```

где Face является одним из следующих:



|                      |                                                            |
|----------------------|------------------------------------------------------------|
| GL \_ спереди            | режим, применяемый к внешним многоугольникам                |
| на \_ задний план             | режим, который применяется к фоновым многоугольникам                 |
| \_Передняя \_ и \_ задняя части GL | режим, применяемый как к внешнему, так и к фоновому многоугольнику |



 

\_Внешний \_ и \_ задний режимы GL эквивалентны функции в НЕКОТОРОМ **режиме** IRI GL. В следующей таблице перечислены режимы многоугольников ДИАФРАГМы и их эквивалентные режимы OpenGL.



| Режим "IRI GL" | Режим OpenGL | Значение                                       |
|--------------|-------------|-----------------------------------------------|
| \_точка ПИМ   | \_точка GL   | Рисует вершины в виде точек.                     |
| ПИМ \_ строка    | \_строка GL    | Рисует границы границ в виде сегментов линии.        |
| ПИМ \_ Заливка    | \_Заливка GL    | Рисуется внутренняя заливка многоугольника.                |
| ПИМ, \_ Заливка  |             | Заполняет только внутренние Пиксели в границах. |



 

## <a name="porting-polygon-stipples"></a>Перенос многоугольников Стипплес

При переносе стипплесного многоугольника IRI GL учитывайте следующие моменты:

-   OpenGL не использует таблицы для многоугольников стипплес; сохраняется только один шаблон стиппле. Списки просмотра можно использовать для хранения различных шаблонов стиппле.
-   Размер растрового изображения OpenGL Polygon стиппле всегда равен 32-разрядному шаблону размером 32x32.
-   На кодировку стиппле влияет [**глпикселсторе**](glpixelstore-functions.md).

Дополнительные сведения о переносе многоугольников стипплес см. в разделе [перенос операций пикселей](porting-pixel-operations.md).

В следующей таблице перечислены стиппле функции и их эквивалентные функции OpenGL в компании IRI GL.



| Функция IRI GL | Функция OpenGL                                    | Значение                                               |
|------------------|----------------------------------------------------|-------------------------------------------------------|
| **дефпаттерн**   | [**глполигонстиппле**](glpolygonstipple.md)       | Задает шаблон стиппле.                             |
| **сетпаттерн**   |                                                    | OpenGL сохраняет только один шаблон стиппле многоугольника.        |
| **GetPattern**   | [**глжетполигонстиппле**](glgetpolygonstipple.md) | Возвращает битовую карту стиппле (используемую для возврата индекса). |

## <a name="porting-tessellated-polygons"></a>Перенос Тесселяцических многоугольников

В IRI GL используется **вогнутых участков**(**true**), а затем **бгнполигон** для рисования вогнутых участков многоугольников. GLU OpenGL включает функции, которые можно использовать для рисования многоугольников вогнутых участков.

**Рисование многоугольника вогнутых участков с помощью OpenGL**

1.  Создайте объект тесселяции.
2.  Определите обратные вызовы, которые будут использоваться для обработки треугольников, созданных тесселяцией.
3.  Укажите многоугольник вогнутых участков для тесселяции.

В следующей таблице перечислены функции OpenGL для рисования тесселяцических многоугольников.



| Функция OpenGL GLU                        | Значение                                                            |
|--------------------------------------------|--------------------------------------------------------------------|
| [**глуневтесс**](glunewtess.md)           | Создает новый объект тесселяции.                                 |
| [**глуделететесс**](gludeletetess.md)     | Удаляет объект тесселяции.                                     |
| [*глутесскаллбакк*](glutess.md)           |                                                                    |
| [**глубегинполигон**](glubeginpolygon.md) | Начинает спецификацию многоугольника.                                  |
| [**глутессвертекс**](glutessvertex.md)     | Задает вершину многоугольника в контуре.                           |
| [**глунекстконтаур**](glunextcontour.md)   | Указывает, что следующий ряд вершин описывает новый профиль. |
| [**глуендполигон**](gluendpolygon.md)     | Завершает спецификацию многоугольника.                                    |
