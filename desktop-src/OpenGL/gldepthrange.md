# <a name="gldepthrange-function"></a>Функция Глдепсранже

Функция **глдепсранже** указывает сопоставление значений *z* от нормализованных координат устройства до координат окна.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glDepthRange(
   GLclampd zNear,
   GLclampd zFar
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*знеар* 
</dt> <dd>

Сопоставление ближней плоскости отсечения с координатами окна. Значение по умолчанию равно нулю.

</dd> <dt>

*зфар* 
</dt> <dd>

Сопоставление дальней плоскости с координатами окна. Значение по умолчанию — 1.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Комментарии

После отсечения и деления на *w* координаты *z* находятся в диапазоне от 0,0 до 1,0, соответствующих ближайшим и далеко обрезанным плоскостям. Функция **глдепсранже** задает линейное сопоставление нормализованных координат *z* в этом диапазоне с координатами окна по *оси z*. Независимо от фактической реализации буфера глубины, значения глубины координат окна обрабатываются так, как будто они находятся в диапазоне от 0,0 до 1,0 (например, цветовые компоненты). Поэтому значения, принимаемые функцией **глдепсранже** , преобразуются в этот диапазон до их принятия.

Сопоставление по умолчанию (0, 1) сопоставляет ближайшую плоскость с 0 и дальней плоскости с 1. При таком сопоставлении диапазон буферов глубины используется полностью.

Не обязательно, чтобы *знеар* был меньше *зфар*. Обратные сопоставления, такие как (1, 0), приемлемы.

Следующая функция получает сведения, связанные с **глдепсранже**:

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ диапазона глубины GL \_
