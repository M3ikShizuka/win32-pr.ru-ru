# <a name="error-handling-opengl"></a>Обработка ошибок (OpenGL)

Когда OpenGL обнаруживает ошибку, он записывает текущий код ошибки. Функция, которая вызвала ошибку, игнорируется, поэтому она не влияет на состояние OpenGL или на содержимое буфера кадров. (Если ошибка записана в ГК \_ \_Не хватает \_ памяти, однако результаты функции не определены.) После записи текущий код ошибки не удаляется до тех пор, пока не будет вызвана функция запроса [**глжетеррор**](glgeterror.md) , которая возвращает текущий код ошибки.

Реализации OpenGL могут возвращать несколько текущих кодов ошибок, каждая из которых остается установленной до запроса. Функция **глжетеррор** ВОЗВРАЩАЕТ ошибку GL \_ , \_ когда вы запрашиваете все текущие коды ошибок или если нет ошибок. Поэтому при получении кода ошибки вызовите **глжетеррор** до тех пор, пока \_ не будет \_ возвращена ошибка GL, чтобы убедиться, что все ошибки обнаружены. Список кодов ошибок см. в разделе [коды ошибок OpenGL](opengl-error-codes.md).

Функцию [**глуеррорстринг**](gluerrorstring.md) Glu можно использовать для получения описательной строки, соответствующей коду ошибки, переданной в. Дополнительные сведения о **глуеррорстринг** см. в разделе [Обработка ошибок](handling-errors.md).

> [!Note]  
> Функции GLU часто возвращают значения ошибок, если обнаружена ошибка. Кроме того, библиотека служебной программы OpenGL определяет коды ошибок GLU \_ недопустимое \_ перечисление, Glu \_ недопустимое \_ значение и Glu \_ нехватки \_ \_ памяти, которые имеют то же значение, что и соответствующие коды ошибок OpenGL.
