# <a name="abstract-data-models"></a>Абстрактные модели данных

У каждого приложения и каждой операционной системы есть абстрактная модель данных. Многие приложения явно не предоставляют эту модель данных, но в модели рассматривается способ написания кода приложения. В 32-разрядной модели программирования (известной как модель ILP32), типы данных Integer, long и pointer имеют длину 32 бит. Большинство разработчиков использовали эту модель, не зная ее. Для ведения журнала API Win32 это допустимо (хотя и необязательно) предположение.

В 64-Битвиндовс это предположение о четности в размерах типов данных недопустимо. При создании всех типов данных 64 бит будет тратить место на пространство, так как для большинства приложений увеличение размера не требуется. Однако приложениям требуются указатели на 64-разрядные данные, и им необходима возможность иметь в выбранных случаях 64-разрядные типы данных. Эти рекомендации привели к выбору абстрактной модели данных с именем LLP64 (или P64). В модели данных LLP64 только указатели разворачиваются до 64 бит; все остальные базовые типы данных (Integer и long) остаются в 32 бит в длину.

изначально большинство приложений, работающих на 64-разрядных Windows, будут перенесены из 32-разрядной Windows. Это цель того, что тот же источник, тщательно написанный, должен выполняться как в 32, так и в 64-бит Windows. Определение модели данных не делает эту задачу проще. Однако, чтобы убедиться, что модель данных влияет только на типы данных указателей, это первый шаг. Вторым шагом является определение набора новых типов данных, позволяющих разработчикам автоматически изменять размер данных, связанных с указателями. Это позволяет данным, связанным с указателями, изменять размер при изменении размера указателя с 32 бит на 64 бит. Базовые типы данных сохраняют длину 32 бит, поэтому объем данных на диске не изменяется, данные совместно используются по сети или данные, совместно используемые в файлах, отображенных в памяти. Это освобождает разработчиков от значительных усилий, связанных с переносом 32-разрядного кода в 64-разрядную Windows.
