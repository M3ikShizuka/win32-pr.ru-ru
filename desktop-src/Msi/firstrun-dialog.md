# <a name="firstrun-dialog"></a>Диалоговое окно файла firstrun

Последовательность диалоговых окон файла firstrun собирает сведения об имени пользователя, имени компании и ИДЕНТИФИКАТОРе продукта. В этом диалоговом окне установщик проверяет идентификатор продукта.

Последовательность диалоговых окон файла firstrun обычно не является частью последовательности действий и вызывается функцией [**мсиколлектусеринфо**](/windows/desktop/api/Msi/nf-msi-msicollectuserinfoa) при первом запуске продукта.

Автор пакета установщика может использовать последовательность диалогового окна шаблона или создать другую последовательность. Однако для последовательности диалоговых окон необходимо, чтобы пользователь установил следующие свойства:

-   Свойство [**username**](username.md)
-   [**COMPANYNAME**](companyname.md) , свойство
-   [**PIDKEY**](pidkey.md) , свойство

Идентификатор продукта будет проверяться в диалоговом окне с помощью [действия валидатепродуктид](validateproductid-action.md) или [валидатепродуктид таблице ControlEvent событие](validateproductid-controlevent.md).

Если идентификатор продукта установлен в качестве свойства в командной строке или при преобразовании, то необходимость в повторном вводе идентификатора продукта в диалоговом окне первого запуска можно обойти, управляя отображением с помощью свойства [**ProductID**](productid.md) . После успешной проверки идентификатора продукта свойству **ProductID** присваивается полный, допустимый идентификатор продукта.
