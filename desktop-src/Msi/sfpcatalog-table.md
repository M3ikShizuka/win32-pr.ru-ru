# <a name="sfpcatalog-table"></a>Таблица Сфпкаталог

Таблица Сфпкаталог содержит каталоги, используемые Windows Me.

Таблица Сфпкаталог содержит следующие столбцы.



| Столбец     | Type                       | Ключ | Допускает значения NULL |
|------------|----------------------------|-----|----------|
| сфпкаталог | [Имя файла](filename.md)   | Да   | Нет        |
| Каталог    | [Двоичный](binary.md)       | Нет   | Нет        |
| Зависимость | [Формате](formatted.md) | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="SFPCatalog"></span><span id="sfpcatalog"></span><span id="SFPCATALOG"></span>сфпкаталог
</dt> <dd>

Короткое имя файла для каталога. Поскольку каталоги не имеют версии, каталог, указанный в этом столбце, может перезаписать каталог с тем же именем и уже установлен в локальной системе. См. раздел Управление версиями файлов. [ни один из файлов не имеет версии](neither-file-has-a-version.md).

</dd> <dt>

<span id="Catalog"></span><span id="catalog"></span><span id="CATALOG"></span>Каталога
</dt> <dd>

Двоичные данные для каталога.

</dd> <dt>

<span id="Dependency"></span><span id="dependency"></span><span id="DEPENDENCY"></span>Зависимостей
</dt> <dd>

Каталог, указанный в этом поле, является родительским по отношению к зависимому каталогу, указанному в поле Сфпкаталог. Введите короткое имя родительского каталога в поле зависимости. Это поле является ключом обратно в столбец Сфпкаталог. Соответствие зависимости учитывает регистр.

Если поле зависимости ссылается на другую запись в этой таблице Сфпкаталог, родительский каталог устанавливается перед зависимым каталогом.

Если поле зависимости ссылается на родительский каталог, который отсутствует в поле Сфпкаталог этой таблицы, и если родительский каталог еще не установлен, установка завершается сбоем.

</dd> </dl>
