# <a name="installation-mechanism"></a>Механизм установки

Успешный процесс установки состоит из двух этапов: приобретения и выполнения. Если установка завершилась неудачно, может произойти этап отката.

## <a name="acquisition"></a>Получение

В начале фазы приобретения приложение или пользователь предписывает установщику установить компонент или приложение. Затем установщик проходит через действия, указанные в таблицах последовательности базы данных установки. Эти действия запрашивают базу данных установки и создают скрипт, который предоставляет пошаговую процедуру для выполнения установки.

## <a name="execution"></a>Выполнение

На этапе выполнения установщик передает сведения процессу с повышенными привилегиями и запускает скрипт.

## <a name="rollback"></a>Откат

Если установка завершается неудачно, установщик восстанавливает исходное состояние компьютера. Когда установщик обрабатывает сценарий установки, он одновременно создает скрипт отката. Помимо скрипта отката, установщик сохраняет копию каждого файла, который он удаляет, во время установки. Эти файлы хранятся в скрытом системном каталоге. После завершения установки сценарий отката и сохраненные файлы удаляются. Дополнительные сведения см. в разделе [ROLLBACK Installation](rollback-installation.md).
