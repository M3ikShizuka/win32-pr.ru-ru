# <a name="rollback-of-assemblies-in-the-global-assembly-cache"></a>Откат сборок в глобальном кэше сборок

Двухэтапный процесс расширяет модель транзакций установщик Windows на продукты, содержащие сборки среды CLR. Это позволяет установщику откатить неудачные установки и удаления сборок.

На первом шаге установщик Windows использует платформу Microsoft .NET для создания одного интерфейса для каждой сборки. Установщик Windows использует столько интерфейсов, сколько устанавливаются сборки. Фиксация сборки с помощью одного из этих интерфейсов означает, что сборка готова заменить любую существующую сборку с тем же именем, она еще не заменит ее. Если пользователь отменяет установку или возникает неустранимая ошибка установки, установщик Windows может по-прежнему выполнять откат сборки до предыдущего состояния, освобождая эти интерфейсы.

После того как установщик Windows завершит установку всех сборок и установщик Windows компонентов, установщик может запустить второй шаг установки. На втором шаге используется отдельная функция для выполнения окончательной фиксации всех новых сборок среды CLR. Все существующие сборки с тем же именем заменяются.
