# <a name="validating-an-installation-database"></a>Проверка базы данных установки

Авторы пакетов установки должны всегда выполнять проверку своих пакетов, прежде чем пытаться установить пакет в первый раз и повторно запустить проверку при каждом внесении изменений в пакет. Проверка проверяет базу данных на наличие ошибок, которые могут быть действительны по отдельности, но это приводит к неправильному поведению в контексте всей базы данных. Попытка установить пакет, который не прошел проверку, может повредить систему пользователя. См. раздел [Проверка пакетов](package-validation.md) и [внутренние средства оценки согласованности — ICEs](internal-consistency-evaluators-ices.md).

Образец пакета можно проверить с помощью [Orca.exe](orca-exe.md) или [Msival2.exe](msival2-exe.md). Чтобы просмотреть справку по Msival2.exe изменить каталоги и ввести в командной строке.

Msival2 -?

Файл. cub дарице. cub содержит пользовательские действия ICE, необходимые для выполнения проверки Msival2.exe. Проверка MNP2000.msi Enter

msival2 MNP2000.msi Дарице. cub

Описание сообщений об ошибках и предупреждений, возвращенных проверкой, см. в [справочнике по Ice](ice-reference.md). Исправьте все ошибки в пакете и повторно запустите проверку по мере необходимости, пока пакет не пройдет проверку без ошибок.

После того как пакет проходит проверку, можно установить образец пакета, щелкнув значок MNP2000.msi или из командной строки, используя [Параметры командной строки](command-line-options.md).

После этого будет выполнен пример установки.
