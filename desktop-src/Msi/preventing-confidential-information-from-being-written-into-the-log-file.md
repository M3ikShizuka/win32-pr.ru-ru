# <a name="preventing-confidential-information-from-being-written-into-the-log-file"></a>Предотвращение записи конфиденциальной информации в файл журнала

при использовании установщик Windows можно предотвратить ввод конфиденциальных сведений, например паролей, в файл журнала и сделать видимыми.

-   Установщик никогда не записывает сведения в столбец password [таблицы сервицеинсталл](serviceinstall-table.md) в журнал.
-   Можно запретить установщику записывать свойство, связанное с [элементом управления](edit-control.md) "поле ввода", в журнал, установив [атрибут элемента управления Password](password-control-attribute.md). Свойство, связанное с элементом управления "поле ввода", имеющим атрибут элемента управления "пароль", скрывается, даже если для политики [отладки](debug.md) задано значение 7.
-   Можно запретить установщику записывать закрытое свойство в журнал, включив свойство в свойство [**мсихидденпропертиес**](msihiddenproperties.md) .
    > [!Note]  
    > Этот метод может сделать так, чтобы конфиденциальные сведения были указаны в командной строке, видимой в журнале. Если для политики [отладки](debug.md) задано значение 7, установщик будет записывать в журнал сведения, указанные в командной строке. Это делает свойство, введенное в командную строку, видимым даже в том случае, если свойство включено в свойство [**мсихидденпропертиес**](msihiddenproperties.md) .

     

-   Можно предотвратить запись данных из целевого столбца таблицы [CustomAction](customaction-table.md) в журнал, включив флаг хидетаржет bit в поле Type таблицы CustomAction. Значение этого флага равно 8192 (0x2000). Дополнительные сведения см. в статье [параметр скрытого целевого объекта настраиваемого действия](custom-action-hidden-target-option.md).
