# <a name="specifying-properties"></a>Указание свойств

Windows Свойства установщика являются глобальными переменными, которые установщик использует во время установки. См. разделы в разделе [Свойства](properties.md). если в разделе [импортируется пустая база данных](importing-a-blank-database.md) , использованная uisample.msi из пакета SDK для установщик Windows, то [таблица свойств](property-table.md) в копии MNP2000.msi уже содержит множество свойств, используемых пользовательским интерфейсом. в этом разделе вы добавите дополнительные сведения в таблицу свойств, относящуюся к установке Блокнот образца. См. также [группу "таблицы сведений о программе](program-information-tables-group.md)".

в каждом пакете установки требуется пять свойств, которые необходимо обновить для Блокнот образца в [таблице свойств](property-table.md) MNP2000.msi.

-   [**ProductCode**](productcode.md)
-   [**продуктлангуаже**](productlanguage.md)
-   [**Производителя**](manufacturer.md)
-   [**ProductVersion**](productversion.md)
-   [**ProductName**](productname.md)

Как и не требуется для всех пакетов установки, приложения, которые могут в будущем получить обновление, также должны иметь свойство [**UpgradeCode**](upgradecode.md) . См. статью [Подготовка приложения к последующим основным обновлениям](preparing-an-application-for-future-major-upgrades.md).

Используйте редактор базы данных, чтобы открыть MNP2000.msi и ввести в таблицу свойств следующие данные. Список содержит ссылки на справочные разделы, посвященные встроенным свойствам установщика. Имена свойств, которые не являются ссылками, являются свойствами, определяемыми автором. Многие свойства, импортированные из uisample.msi, предназначены для примера пользовательского интерфейса. В следующем разделе интерфейс пользователя для примера установки обсуждается пользовательский интерфейс.
