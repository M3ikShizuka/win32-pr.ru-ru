# <a name="enableadmintsremote"></a>енаблеадминтсремоте

начиная с Windows Server 2008 и Windows Vista, эта политика больше не действует. Администратор может выполнить установку из сеанса консоли сервера терминалов. Администратор также может выполнить установку из сеанса клиента сервера терминалов.

дополнительные сведения см. в разделе [службы терминалов](/windows/desktop/TermServ/terminal-services-portal) в пакете Microsoft Windows Software Development Kit (SDK).

* * операционные системы, выпущенные до Windows Server 2008 и Windows Vista: * *

Настройка [политики системы](system-policy.md) для компьютера позволяет администраторам выполнять установку из сеанса клиента сервера терминалов. Если эта политика не задана, администраторы могут выполнять установку только из сеанса консоли. Пользователи без прав администратора не могут выполнять установку из сеанса клиента. Значение по умолчанию для этой политики позволяет только администраторам выполнять установку из сеанса консоли. Администраторы всегда могут выполнять [административные установки](administrative-installation.md) из сеанса клиента сервера терминалов или сеанса консоли, независимо от того, задана ли эта политика.

## <a name="registry-key"></a>Ключ реестра

**HKey \_ \_** \\ **политики программного обеспечения** локального компьютера \\  \\  \\  \\ **установщик** Microsoft Windows

## <a name="data-type"></a>Тип данных

**REG \_ DWORD**
