# <a name="patch-table"></a>Таблица исправлений

В таблице Patch указан файл, который должен принимать определенное исправление, и физическое расположение файлов исправлений на образах носителей.

Таблица исправлений содержит следующие столбцы.



| Столбец      | Type                               | Ключ | Допускает значения NULL |
|-------------|------------------------------------|-----|----------|
| Файл\_      | [Идентификатор](identifier.md)       | Да   | Нет        |
| Последовательность    | [Integer](integer.md)             | Да   | Нет        |
| патчсизе   | [даублеинтежер](doubleinteger.md) | Нет   | Нет        |
| Атрибуты  | [Integer](integer.md)             | Нет   | Нет        |
| Header      | [Двоичный](binary.md)               | Нет   | Да        |
| стреамреф\_ | [Идентификатор](identifier.md)       | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="File_"></span><span id="file_"></span><span id="FILE_"></span>File\_
</dt> <dd>

Исправление применяется к файлу, заданному идентификатором в этом столбце. Это первичный ключ таблицы, который является внешним ключом для [таблицы файлов](file-table.md).

</dd> <dt>

<span id="Sequence"></span><span id="sequence"></span><span id="SEQUENCE"></span>Виртуализирован
</dt> <dd>

Это расположение файла исправления в порядке последовательности файлов на изображениях мультимедиа. Порядок последовательности должен соответствовать порядку файлов в CAB-файле пакета исправлений. Это первичный ключ для этой таблицы. Максимальный предел — 32767 файлов, чтобы создать пакет установщик Windows с большим числом файлов, см. раздел [Разработка большого пакета](authoring-a-large-package.md).

</dd> <dt>

<span id="PatchSize"></span><span id="patchsize"></span><span id="PATCHSIZE"></span>патчсизе
</dt> <dd>

Этот столбец задает размер исправления в байтах, записанных в виде длинного целого числа.

</dd> <dt>

<span id="Attributes"></span><span id="attributes"></span><span id="ATTRIBUTES"></span>Атрибута
</dt> <dd>

Целочисленное значение, содержащее битовые флаги, представляющие атрибуты исправления. Вставьте значение 1 в этот столбец, чтобы указать, что сбой применения данного исправления не является неустранимой ошибкой.



| Константа                         | Шестнадцатеричный | Decimal | Описание                                                          |
|----------------------------------|-------------|---------|----------------------------------------------------------------------|
| (нет)                           | 0x000       | 0       | Сбой установки этого исправления является неустранимой ошибкой.                        |
| **мсидбпатчаттрибутеснонвитал** | 0x001       | 1       | Указывает, что сбой применения этого исправления не является неустранимой ошибкой. |



 

</dd> <dt>

<span id="Header"></span><span id="header"></span><span id="HEADER"></span>Заголовок
</dt> <dd>

Этот столбец является заголовком двоичного обновления потока, используемого для проверки исправлений. Если столбец Стреамреф не равен null, этот столбец должен иметь значение NULL \_ . В этом случае поток заголовков исправлений хранится в [таблице мсипатчхеадерс](msipatchheaders-table.md) , чтобы преодолеть ограничения имен потоков, описанные в разделе [ограничения по OLE для потоков](ole-limitations-on-streams.md).

</dd> <dt>

<span id="StreamRef_"></span><span id="streamref_"></span><span id="STREAMREF_"></span>стреамреф\_
</dt> <dd>

Внешний ключ в таблицу Мсипатчхеадерс, указывающий строку, содержащую поток заголовков исправлений.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Эта таблица обрабатывается [действием патчфилес](patchfiles-action.md). Обычно он добавляется в пакет установки путем преобразования из пакета исправлений. Обычно он не создается непосредственно в установочном пакете.
