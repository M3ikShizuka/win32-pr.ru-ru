# <a name="one-file-has-a-version"></a>Один файл имеет версию

Если файл ключа устанавливаемого компонента (Copy-A) имеет то же имя, что и файл, уже установленный в целевом расположении (Copy-B), установщик сравнивает номер версии, дату и язык двух файлов.

Если только один из файлов имеет номер версии, установщик использует логику, показанную на следующей схеме потока, чтобы определить, следует ли заменить все установленные файлы, принадлежащие компоненту. Поскольку установщик устанавливает только компоненты целиком, при замене установленного файла ключей все файлы компонента заменяются.

Обратите внимание, что на этой диаграмме показаны [правила управления версиями файлов](file-versioning-rules.md)по умолчанию, которые можно переопределить, задав свойство [**REINSTALLMODE**](reinstallmode.md) . Значение свойства **REINSTALLMODE** по умолчанию — "omus".
