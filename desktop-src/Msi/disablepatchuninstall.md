# <a name="disablepatchuninstall"></a>дисаблепатчунинсталл

Если для этой [системной политики](system-policy.md) на уровне компьютера установлено значение 1, исправления не могут быть удалены с компьютера пользователем или администратором. установщик Windows может по-прежнему удалить исправление, которое больше не применимо к продукту. Если эта политика не задана, пользователь может удалить исправление с компьютера, только если ему предоставлены права на удаление исправления. Это может зависеть от того, является ли пользователь администратором, установлены ли параметры политики [дисаблемси](disablemsi.md) и [AlwaysInstallElevated](alwaysinstallelevated.md) и установлено ли это исправление в управляемом пользователем, неуправляемом или отдельном для пользователя контексте.

## <a name="registry-key"></a>Ключ реестра

**HKey \_ \_** \\ **политики программного обеспечения** локального компьютера \\  \\  \\  \\ **установщик** Microsoft Windows

## <a name="data-type"></a>Тип данных

**REG \_ DWORD**
