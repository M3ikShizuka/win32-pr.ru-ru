# <a name="duplicatefile-table"></a>Таблица Дупликатефиле

Таблица Дупликатефиле содержит список файлов, которые должны дублироваться, либо в каталог, отличный от исходного файла, либо в тот же каталог, но с другим именем. Исходный файл должен быть файлом, установленным [действием инсталлфилес](installfiles-action.md).

Таблица Дупликатефиле содержит следующие столбцы.



| Столбец      | Type                         | Ключ | Допускает значения NULL |
|-------------|------------------------------|-----|----------|
| филекэй     | [Идентификатор](identifier.md) | Да   | Нет        |
| Компонент\_ | [Идентификатор](identifier.md) | Нет   | Нет        |
| Файл\_      | [Идентификатор](identifier.md) | Нет   | Нет        |
| дестнаме    | [Имя файла](filename.md)     | Нет   | Да        |
| DestFolder  | [Идентификатор](identifier.md) | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="FileKey"></span><span id="filekey"></span><span id="FILEKEY"></span>филекэй
</dt> <dd>

Первичный ключ, нелокализованный токен, идентифицирующий уникальную запись Дупликатефиле.

</dd> <dt>

<span id="Component_"></span><span id="component_"></span><span id="COMPONENT_"></span>См\_
</dt> <dd>

Внешний ключ к первому столбцу [таблицы Component](component-table.md). Если компонент, идентифицируемый ключом, не выбран для установки или удаления, для этой записи Дупликатефиле не выполняются никакие действия.

</dd> <dt>

<span id="File_"></span><span id="file_"></span><span id="FILE_"></span>File\_
</dt> <dd>

Внешний ключ в [таблице File](file-table.md) , представляющий исходный файл, который должен дублироваться.

</dd> <dt>

<span id="DestName"></span><span id="destname"></span><span id="DESTNAME"></span>дестнаме
</dt> <dd>

Локализуемое имя, присваиваемое дублированному файлу. Если это поле пустое, то конечному файлу присваивается то же имя, что и у исходного файла.

</dd> <dt>

<span id="DestFolder"></span><span id="destfolder"></span><span id="DESTFOLDER"></span>DestFolder
</dt> <dd>

Имя свойства, которое является полным путем к месту копирования повторяющегося файла. Если этот каталог совпадает с каталогом, содержащим исходный файл, а имя предложенного повторяющегося файла совпадает с исходным, то никаких действий не происходит.

</dd> </dl>

## <a name="remarks"></a>Комментарии
