# <a name="advertisement"></a>Рекламу

Установщик Windows может объявить о доступности приложения пользователям или другим приложениям без фактической установки приложения. Если приложение объявлено, то пользователям или другим приложениям предоставляются только интерфейсы, необходимые для загрузки и запуска приложения. Если пользователь или приложение активирует объявленный интерфейс, установщик продолжит установку необходимых компонентов, как описано в разделе [Установка по запросу](installation-on-demand.md).

Два типа рекламы — это назначение и публикация. Приложение отображается как установленное для пользователя, если оно назначено пользователю. Меню **Пуск** содержит соответствующие сочетания клавиш, значки отображаются, файлы связаны с приложением, а записи реестра соответствуют установке приложения. Когда пользователь пытается открыть назначенное приложение, он устанавливается по запросу.

Установщик поддерживает объявление приложений и функций в соответствии с операционной системой. Установщик регистрирует сведения о COM-классе для назначенных приложений, начиная с Windows XP. Это позволяет установщику установить приложение при создании экземпляра объявленного класса. Дополнительные сведения см. в разделе [поддержка платформ объявления](platform-support-of-advertisement.md).

Приложение можно опубликовать с сервера, начиная с Windows Server 2003. После этого опубликованное приложение устанавливается с помощью сопоставления файлов или типа MIME. Публикация не заполняет пользовательский интерфейс ни одним из значков приложения. Клиентская операционная система может установить опубликованное приложение, начиная с Windows XP.
