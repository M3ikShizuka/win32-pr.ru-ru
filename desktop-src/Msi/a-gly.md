# <a name="a-windows-installer"></a>A (установщик Windows)

A [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) [F](f-gly.md) [G](g-gly.md) р [. J K](i-gly.md) L [M](m-gly.md) N [O](o-gly.md) [P](p-gly.md) [Q](q-gly.md) [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [U](u-gly.md) [V](v-gly.md) W X Y Z

<dl> <dt>

<span id="_msi_accessibility_using_windows_installer_gly"></span><span id="_MSI_ACCESSIBILITY_USING_WINDOWS_INSTALLER_GLY"></span>**режима**
</dt> <dd>

Реализация разработки для предоставления доступа к пользовательскому интерфейсу установщика для всех пользователей. Дополнительные сведения о специальных возможностях см. в разделе Общие сведения: [Специальные возможности](accessibility.md).

</dd> <dt>

<span id="_msi_acquisition_phase_gly"></span><span id="_MSI_ACQUISITION_PHASE_GLY"></span>**Фаза приобретения**
</dt> <dd>

Этап установки, в течение которого установщик определяет процедуру. этап приобретения начинается, когда приложение или пользователь сообщает [*установщик Windows*](m-gly.md) установить приложение или компонент. Затем установщик запрашивает в [*базе данных*](i-gly.md) сведения о том, что он создает [*Скрипт выполнения*](e-gly.md) для установки. Дополнительные сведения о стадиях установки см. в разделе [механизм установки](installation-mechanism.md).

</dd> <dt>

<span id="_msi_action_gly"></span><span id="_MSI_ACTION_GLY"></span>**поддержки**
</dt> <dd>

многие функции, выполняемые установщик Windows, инкапсулируются в действия. Каждое действие задает выполнение определенной функции, а общая последовательность процедур установки определяется последовательностью действий в [*таблицах последовательности*](s-gly.md). [*стандартные действия*](s-gly.md) встроены в установщик Windows. [*Пользовательские действия*](c-gly.md) записываются автором [*пакета*](p-gly.md)установки.

</dd> <dt>

<span id="_msi_admin_approval_mode_gly"></span><span id="_MSI_ADMIN_APPROVAL_MODE_GLY"></span>**Режим одобрения администратором**
</dt> <dd>

Состояние утверждения, включенное функцией защиты учетных записей (UAC), которое запускает всех пользователей с минимальными правами, включая администраторов. Пользователям необходимо предоставить согласие на повышение прав на установку, требующую прав администратора.

</dd> <dt>

<span id="_msi_advertising_gly"></span><span id="_MSI_ADVERTISING_GLY"></span>**размещения**
</dt> <dd>

Возможность сделать интерфейсы необходимыми для загрузки и сделать приложение доступным без установки приложения. Когда пользователь или приложение активирует объявленный интерфейс, установщик продолжает установку необходимых компонентов. Два типа рекламы — это [*назначение*](/windows) и [*Публикация*](p-gly.md). Дополнительные сведения см. в разделе также [*Install-On-Demand*](i-gly.md). Дополнительные сведения о том, как установщик объявляет приложения, см. в статье [объявление](advertisement.md).

</dd> <dt>

<span id="_msi_application_information_service_gly"></span><span id="_MSI_APPLICATION_INFORMATION_SERVICE_GLY"></span>**Служба сведений о приложениях (AIS)**
</dt> <dd>

системная служба Windows Vista, которая упрощает запуск установок, требующих повышенных привилегий для запуска. Предоставляет пользовательский интерфейс согласия, используемый контролем учетных записей, для запроса пользователя на авторизацию администратора.

</dd> <dt>

<span id="_msi_assigning_gly"></span><span id="_MSI_ASSIGNING_GLY"></span>**предоставление**
</dt> <dd>

Делает приложение доступным и делает его недоступным, как если бы оно было установлено для пользователя, без фактической установки. Назначение добавляет ярлыки и значки в меню " **Пуск** ", связывает соответствующие файлы и записывает записи реестра для приложения. Когда пользователь пытается открыть назначенное приложение, установщик устанавливает приложение. Назначение и [*Публикация*](p-gly.md) — это два метода [*рекламы*](/windows). Дополнительные сведения см. в статье [объявление](advertisement.md).

</dd> <dt>

<span id="_msi_asynchronous_execution_using_windows_installer_gly"></span><span id="_MSI_ASYNCHRONOUS_EXECUTION_USING_WINDOWS_INSTALLER_GLY"></span>**асинхронное выполнение**
</dt> <dd>

[*Настраиваемое действие*](c-gly.md) , в течение которого установщик запускает отдельные потоки настраиваемого действия и текущую установку одновременно. Дополнительные сведения см. в разделе [синхронные и асинхронные настраиваемые действия](synchronous-and-asynchronous-custom-actions.md).

</dd> </dl>
