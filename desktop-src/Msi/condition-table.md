# <a name="condition-table"></a>Таблица условий

Таблицу условий можно использовать для изменения состояния выбора любой записи в [таблице компонентов](feature-table.md) на основе условного выражения.

Таблица условий содержит следующие столбцы.



| Столбец    | Type                         | Ключ | Допускает значения NULL |
|-----------|------------------------------|-----|----------|
| Компонент\_ | [Идентификатор](identifier.md) | Да   | Нет        |
| Level     | [Integer](integer.md)       | Да   | Нет        |
| Условие | [Condition](condition.md)   | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="Feature_"></span><span id="feature_"></span><span id="FEATURE_"></span>Функциями\_
</dt> <dd>

Внешний ключ в столбце одна из таблиц Feature.

</dd> <dt>

<span id="Level"></span><span id="level"></span><span id="LEVEL"></span>Уровню
</dt> <dd>

Условный уровень установки для функции в столбце "компонент" \_ этой таблицы. Установщик устанавливает уровень установки этой функции на уровень, указанный в этом столбце, если выражение в столбце Condition имеет значение TRUE.

</dd> <dt>

<span id="Condition"></span><span id="condition"></span><span id="CONDITION"></span>Выполняет
</dt> <dd>

Если условное выражение принимает значение TRUE, то столбец Level в таблице Feature устанавливается на уровень условной установки.

Выражение в столбце Condition не должно содержать ссылки на установленное состояние любого компонента или компонента. Это обусловлено тем, что выражения в столбце Условие оцениваются до того, как установщик оценивает установленные состояния компонентов и компонентов. Любое выражение в таблице условие, которое пытается проверить установленное состояние компонента или компонента, всегда имеет значение false.

Дополнительные сведения о синтаксисе условных операторов см. в разделе [Синтаксис условных](conditional-statement-syntax.md)инструкций.

</dd> </dl>

## <a name="remarks"></a>Remarks

Компонент можно навсегда отключить, установив для столбца Level значение 0.

Уровень может быть установлен на основе любой условной инструкции, такой как тест для платформы, операционной системы или определенного значения свойства.

Следует тщательно выбрать условия, чтобы функция не была включена при установке, а затем была отключена при удалении. Эта функция будет утеряна, и продукт не будет удален.

Эта таблица упоминается при выполнении [действия костфинализе](costfinalize-action.md) .

Если предварительно [**выбранное**](preselected.md) свойство было установлено в 1, установщик не оценивает таблицу условий. Таблица условий влияет только на установку компонентов, если не задано ни одно из следующих свойств:
