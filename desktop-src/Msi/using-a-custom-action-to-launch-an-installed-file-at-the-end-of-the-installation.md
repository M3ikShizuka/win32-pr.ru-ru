# <a name="using-a-custom-action-to-launch-an-installed-file-at-the-end-of-the-installation"></a>Использование настраиваемого действия для запуска установленного файла в конце установки

В следующем примере показано, как запустить HTML-файл в конце установки. Установщик устанавливает компонент, содержащий файл, а затем публикует событие элемента управления в конце установки для запуска настраиваемого действия, открывающего файл. Этот подход можно использовать для запуска учебника по завершении первой установки приложения.

Пример должен соответствовать следующим спецификациям.

-   Установщик выполняет пользовательское действие только в том случае, если для установки приложения используется [*полный уровень пользовательского интерфейса*](f-gly.md) .
-   Установщик выполняет пользовательское действие только в том случае, если компонент, содержащий HTML-файл, установлен для локального запуска на компьютере.
-   Настраиваемое действие выполняется только при первой установке приложения.
-   При сбое настраиваемого действия установка не завершается ошибкой.

Пример включает гипотетический компонент с именем Tutorial, который управляет по крайней мере одним ресурсом, файлом с именем tutorial.htm. Идентификатор этого файла в столбце File таблицы File — это учебник. В следующем обсуждении предполагается, что вы уже создали ресурсы, необходимые для работы с руководством, и сделали все необходимые записи в таблицах [компонентов](feature-table.md), [компонентов](component-table.md), [файлов](file-table.md), [каталогов](directory-table.md)и [феатурекомпонентс](featurecomponents-table.md) для установки этого компонента. Дополнительные сведения см. [в примере установки](an-installation-example.md).
