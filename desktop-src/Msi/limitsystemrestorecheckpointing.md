# <a name="limitsystemrestorecheckpointing"></a>лимитсистемресторечеккпоинтинг

эта [политика системы](system-policy.md) на компьютере отключает создание контрольных точек с установщик Windows.

Если задано значение 0 или отсутствует, установщик выполняет обычную контрольную точку для установки или удаления. Если задано значение 1, установщик не создает контрольных точек.

эта политика влияет только на контрольные точки, установленные установщик Windows. на компьютерах Windows XP администраторам может потребоваться отключить контрольные точки в установщик Windows для повышения производительности. [**Восстановление системы**](../sr/system-restore-portal.md) также создает дополнительные контрольные точки. дополнительные сведения см. в статьях [точки восстановления системы и установщик Windows](system-restore-points-and-the-windows-installer.md) и [настройка точки восстановления из настраиваемого действия](setting-a-restore-point-from-a-custom-action.md).

## <a name="registry-key"></a>Ключ реестра

Задайте значение с именем Лимитсистемресторечеккпоинтинг в следующем разделе реестра.

**в HKEY \_ локальные \_ компьютеры \\ политики программного обеспечения \\ \\ Microsoft \\ Windows \\ Installer**

## <a name="data-type"></a>Тип данных

**REG \_ DWORD**
