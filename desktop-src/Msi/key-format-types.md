# <a name="key-format-types"></a>Типы форматов ключей

Типы форматов ключей настраиваемых данных могут использоваться в текстовых полях для предоставления ключа в таблицу базы данных. Атрибут Мсмконфигитемноннуллабле является неявным в этом формате данных и не требует установки. Значение NULL никогда не является допустимым значением для этого типа. Ответы на все элементы формата ключа должны быть в [специальном формате кмсм](cmsm-special-format.md).

Существуют следующие типы форматов ключей:

[Тип каталога](directory-type.md)

[Тип файла](file-type.md)

[Тип свойства](property-type.md)

[Тип диалогового окна](dialog-type.md)

[Двоичный тип](binary-type.md)

[Тип значка](icon-type.md)

Настраиваемые элементы типа формата ключа используются в текстовых столбцах для предоставления ключа базы данных, а в целом можно заменить любой текстовой строкой. Отдельные типы могут иметь дополнительные синтаксические ограничения, но эти ограничения не применяются Mergemod.dll. Определенные настраиваемые элементы также могут иметь ограничения семантики характеристик. Например, конкретный настраиваемый элемент может быть ключом к [двоичной таблице](binary-table.md) в строке, содержащей растровое изображение. Такие ограничения не применяются Mergemod.dll, поэтому авторы модулей должны быть готовы к обработке любой строки, удовлетворяющей заданному типу формата ключа. Если иное не указано семантическим значением или атрибутами, значение null является допустимым ответом.
