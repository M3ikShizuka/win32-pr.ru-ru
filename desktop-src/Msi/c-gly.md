<dl> <dt>

<span id="_msi_cabinet_file_using_windows_installer_gly"></span><span id="_MSI_CABINET_FILE_USING_WINDOWS_INSTALLER_GLY"></span>**CAB-файл**
</dt> <dd>

Один файл (обычно с расширением CAB), в котором хранятся сжатые файлы в библиотеке файлов. Формат CAB является эффективным способом упаковки нескольких файлов, поскольку сжатие выполняется по границам файлов, значительно повышая степень сжатия. Сведения о создании ящиков см. в разделе [CAB-файлы](cabinet-files.md).

</dd> <dt>

<span id="_msi_checksum_gly"></span><span id="_MSI_CHECKSUM_GLY"></span>**вычислен**
</dt> <dd>

Вычисленное значение, которое зависит от содержимого файла. Он хранится с данными для обнаружения повреждения файла. Система проверяет это значение, чтобы убедиться, что данные не повреждены.

</dd> <dt>

<span id="_msi_component_using_windows_installer_gly"></span><span id="_MSI_COMPONENT_USING_WINDOWS_INSTALLER_GLY"></span>**см**

Примерами компонентов являются группы связанных файлов, COM-объект или библиотека. Дополнительные сведения см. в разделе [компоненты и компоненты](components-and-features.md).

</dd> <dt>

</dt> <dd>

Накопленные изменения, внесенные в базу данных. Изменения не отражаются в реальной базе данных до тех пор, пока база данных не будет зафиксирована. Дополнительные сведения см. в разделе [фиксация баз данных](committing-databases.md).

</dd> <dt>

Аспект функциональных возможностей пользовательского интерфейса установщика. Типичная таблице ControlEvent событие запускает некоторое действие установщиком при активации элемента управления диалогового окна или другого события. Дополнительные сведения см. в разделе [таблице ControlEvent событие Overview](controlevent-overview.md).

</dd> <dt>
