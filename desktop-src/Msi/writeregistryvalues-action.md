# <a name="writeregistryvalues-action"></a>Действие Вритерегистривалуес

Действие Вритерегистривалуес устанавливает сведения о реестре приложения. Сведения о реестре наследуется [таблицей Component](component-table.md). Значение реестра записывается в реестр, если соответствующий компонент настроен для установки локально или в качестве запуска из источника. Дополнительные сведения см. в разделе [Таблица реестра](registry-table.md).

## <a name="sequence-restrictions"></a>Ограничения последовательности

[Действие инсталлвалидате](installvalidate-action.md) должно следовать перед действием вритерегистривалуес. Если имеется [действие ремоверегистривалуес](removeregistryvalues-action.md), оно должно предшествовать действию вритерегистривалуес.

Пользовательское действие можно использовать для добавления строк в [таблицу реестра](registry-table.md) во время установки, удаления или восстановления транзакции. Эти строки не сохраняются в таблице реестра, и эти данные доступны только во время текущей транзакции. Поэтому пользовательское действие должно выполняться при каждой установке, удалении или исправлении транзакций, для которых требуется информация в этих дополнительных строках. Пользовательское действие должно предшествовать действиям [ремоверегистривалуес](removeregistryvalues-action.md) и вритерегистривалуес в последовательности действий.

## <a name="actiondata-messages"></a>Сообщения Актиондата



| Поле | Описание данных действия                               |
|-------|----------------------------------------------------------|
| \[1\] | Путь к разделу реестра значения, записанного в реестр.       |
| \[2\] | Форматированная текстовая строка имя значения, записанное в реестр. |
| \[3\] | Форматированная текстовая строка значения, записанная в реестр.      |
