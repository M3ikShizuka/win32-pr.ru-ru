# <a name="installation-package"></a>Пакет установки

Пакет установки содержит все сведения, необходимые установщик Windows для установки или удаления приложения или продукта, а также для запуска пользовательского интерфейса программы установки. Каждый пакет установки включает MSI-файл, содержащий базу данных установки, информационный поток сводки и потоки данных для различных частей установки. MSI-файл может также содержать один или несколько преобразований, внутренних исходных файлов, а также внешних исходных файлов или CAB-файлов, необходимых для установки.

Разработчики приложений должны создать установку для использования установщика. Поскольку установщик организует установки вокруг концепции [компонентов и компонентов](components-and-features.md)и хранит все сведения об установке в реляционной базе данных, процесс создания пакета установки в общем случае включает следующие шаги.

-   Найдите функции, которые будут представлены пользователям.
-   Упорядочите приложение по компонентам.
-   Заполните базу данных установки данными.
-   Проверьте пакет установки.

В следующем разделе обсуждаются компоненты и функции установщика. Дополнительные сведения см. в разделе [компоненты и компоненты](components-and-features.md). Возможность выбора компонентов обычно определяется функциями приложения с точки зрения пользователя.

Рекомендуется, чтобы разработчики использовали стандартную процедуру выбора компонентов. Дополнительные сведения см. [в разделе Упорядочение приложений по компонентам](organizing-applications-into-components.md).

Авторы пакетов могут использовать средства создания пакетов сторонних разработчиков или редактор таблиц и установщик Windows SDK для заполнения базы данных установки. Все пакеты установки должны быть проверены на предмет внутренней согласованности. Дополнительные сведения см. в разделе [Проверка пакетов](package-validation.md).
