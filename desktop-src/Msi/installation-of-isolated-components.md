# <a name="installation-of-isolated-components"></a>Установка изолированных компонентов

Windows Установщик выполняет следующие действия во время установки приложения, если пакет содержит изолированные компоненты. Обычно общий компонент \_ является библиотекой DLL, которая совместно используется приложениями-компонентами \_ и другими клиентскими исполняемыми файлами.

## <a name="installation"></a>Установка

-   Скопируйте файлы компонента \_ , общие в ту же папку, что и \_ приложение компонента, только если \_ также устанавливается приложение компонента.
-   Создайте нулевой байтовый файл с коротким именем файла ключа \_ приложения компонента. Нахождение этого файла в той же папке, что и \_ приложение компонента. Добавьте расширение. ЛОКАЛЬНЫЙ с этим именем файла.
-   Увеличьте значение refcount Шареддлл, если бит Мсидбкомпонентаттрибутесшареддллрефкаунт установлен в столбце Attributes [таблицы Component](component-table.md).
-   Регистрация \_ приложения-компонента в качестве клиента \_ общего компонента и регистрация пути к ключу, указывающего на общее расположение общего расположения компонента \_ .
-   Установите все ресурсы компонентного \_ приложения обычным образом.

Если \_ общий доступ к компоненту или его файл ключа уже установлен на компьютере, не копируйте файлы в общее расположение общего расположения компонента \_ .

Если \_ общий компонент или его файл ключа еще не установлены на компьютере:

-   Скопируйте файлы компонента, \_ совместно используемые в общем расположении.
-   Обработать все действия по установке для \_ общего компонента.
-   Если компонент \_ Shared является COM-компонентом, зарегистрируйте полный путь com, \[ чтобы синтаксис $Component \] и \[ \# филекэй \] указывал на общее расположение общего расположения компонента \_ .
