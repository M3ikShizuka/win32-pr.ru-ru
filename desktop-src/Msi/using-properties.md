# <a name="using-properties"></a>Использование свойств

В следующих разделах описывается использование встроенных свойств установщика и дополнительные свойства, определяемые автором пакета. Свойства могут быть либо [открытыми свойствами](public-properties.md) , либо [закрытыми свойствами](private-properties.md). Каждое свойство, которое должно быть определено при инициализации установщика, должно иметь имя и начальное значение, перечисленные в [таблице свойств](property-table.md). Свойства, имеющие значение null, не перечислены в таблице свойств. Можно получить или задать свойства из программ и пользовательских действий, а также задать общие свойства из командной строки. Процесс установки можно изменить с помощью свойств в условных инструкциях.

[Ограничения по именам свойств](restrictions-on-property-names.md)

[Инициализация значений свойств](initialization-of-property-values.md)

[Получение и задание свойств](getting-and-setting-properties.md)

[Использование свойств в условных инструкциях](using-properties-in-conditional-statements.md)

[Использование свойства каталога в пути](using-a-directory-property-in-a-path.md)

> [!Note]  
> Не используйте свойства для паролей и других сведений, которые должны оставаться защищенными. Установщик может записать значение свойства, созданного в таблице свойств, или создать во время выполнения в журнале или системном реестре.
