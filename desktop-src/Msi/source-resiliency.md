# <a name="source-resiliency"></a>Отказоустойчивость источника

Приложения, использующие сетевые ресурсы для [установки по запросу](installation-on-demand.md) , подвержены сбоям источника, если исходное расположение должно измениться по какой-либо причине или быть поврежденным. установщик Windows обеспечивает устойчивость источника для компонентов, устанавливаемых по требованию с помощью исходного списка. Исходный список содержит расположения, поиск которых выполняется установщиком пакетов установки. В этом списке могут находиться сетевые расположения, универсальные указатели ресурсов (URL-адреса) или диски Compact. В случае сбоя одного из этих источников установщик может быстро и без проблем попробовать следующий.

Разработчику приложения не нужно включать какие – либо специальные сведения в пакет установщика, чтобы обеспечить устойчивость источника. После установки приложения установщик может добавить последний успешно использовавшийся источник в качестве записи в исходный список. По умолчанию этот источник представляет собой расположение, из которого изначально устанавливается пакет установщика, и совпадает со свойством [**SourceDir**](sourcedir.md) .

Системный администратор может изменить список источников, применив [Преобразование](merges-and-transforms.md) или изменив свойство [**SourceList**](sourcelist.md) из командной строки или в [таблице свойств](property-table.md).

Установщик начинает поиск источника, проверяя Последнее используемое исходное расположение в списке источников. Если этот поиск завершается неудачей, установщик выполняет поиск по списку сетевых источников, источникам носителей и, наконец, к источникам URL-адресов. Системные администраторы могут изменить этот порядок поиска с помощью системной политики [сеарчордер](searchorder.md) . Если эти операции поиска завершаются неудачей, установщик может [открыть диалоговое окно обзора](browse-dialog.md) , чтобы пользователь мог выполнить поиск исходного кода вручную. Диалоговое окно Обзор не может быть отображено, если для уровня пользовательского интерфейса задано значение нет. Дополнительные сведения см. в разделе [уровни пользовательского интерфейса](user-interface-levels.md).

Как правило, установщик должен отображать диалоговое окно обзора, только если текущий пользователь является администратором или если установка не требует повышенных привилегий. Администратор может управлять отображением диалогового окна "Обзор" для пользователей с политиками [дисаблебровсе](disablebrowse.md) и [алловлоккдовнбровсе](allowlockdownbrowse.md) . Администратор также определяет, могут ли пользователи устанавливать приложения из источников, расположенных на носителе, с помощью политик [дисаблемедиа](disablemedia.md) и [алловлоккдовнмедиа](allowlockdownmedia.md) . использование этих политик зависит от версии установщик Windows.
