# <a name="mergemergeex-method"></a>Метод Merge. Мержеекс

Метод **мержеекс** объекта [**Merge**](merge-object.md) эквивалентен функции [**Merge**](/windows/win32/api/mergemod/nf-mergemod-imsmmerge-merge) , за исключением того, что она принимает дополнительный аргумент. Аргумент *пконфигуратион* — это интерфейс, реализуемый клиентом. Аргумент может иметь значение null. Наличие этого аргумента указывает, что клиент способен поддерживать функциональность конфигурации, но не несет клиенту предоставления данных конфигурации для конкретного настраиваемого элемента.

Метод [**Merge**](merge-merge.md) выполняет слияние текущей базы данных и текущего модуля. Слияние присоединяет компоненты модуля к функции, определяемой *функцией*. Корень дерева каталогов модуля перенаправляется в расположение, заданное параметром *редиректдир*.

## <a name="syntax"></a>Синтаксис


```JScript
Merge.MergeEx(
  Feature,
  RedirectDir,
  pConfiguration
)
```

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Remarks

После завершения слияния компоненты модуля присоединяются к функции, определяемой *компонентом*. Эта функция не создана и должна быть существующей функцией. модуль можно подключить к дополнительным функциям с помощью метода [**Подключение**](merge-connect.md) .

Изменения, вносимые в базу данных, сохраняются только в том случае, если метод [**клоседатабасе**](merge-closedatabase.md) вызывается с параметром *бкоммит* , для которого задано **значение true**.

Если возникают конфликты слияния, включая исключения, они помещаются в перечислитель ошибок для последующего извлечения, но не приводят к сбою слияния. Ошибки могут быть получены с помощью свойства [**Errors**](merge-errors.md) . Ошибки и информационные сообщения отправляются в текущий файл журнала.

Если слияние завершается неудачей из-за неверной конфигурации модуля, функция [**мержеекс**](/windows/desktop/api/Mergemod/nf-mergemod-imsmmerge2-mergeex) возвращает \_ ошибку E. Сюда входят следующие ошибки Мсмеррортипе: **мсмеррорбаднуллсубститутион**, **мсмеррорбадсубститутионтипе**, **мсмеррорбаднуллреспонсе**, **msmErrorMissingConfigItem** и **msmErrorDataRequestFailed**. Эти ошибки приводят к тому, что слияние останавливается сразу же после возникновения ошибки. Объект Error по-прежнему добавляется в перечислитель, когда **мержеекс** возвращает E \_ Fail. Дополнительные сведения об ошибках Мсмеррортипе см. в разделе [**\_ функция Get Type (объект Error)**](/windows/win32/api/mergemod/nf-mergemod-imsmerror-get_type). Все остальные ошибки приводят к тому, что **мержеекс** возвращает \_ значение false и вызовет продолжение слияния.
