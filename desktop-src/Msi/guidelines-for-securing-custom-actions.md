# <a name="guidelines-for-securing-custom-actions"></a>Рекомендации по защите настраиваемых действий

соблюдение следующих рекомендаций при создании установщик Windows пакета с настраиваемыми действиями помогает поддерживать безопасную среду во время установки:

-   Защитите дополнительные файлы, записанные настраиваемым действием.
-   Проверьте длину буфера и допустимость всех данных, считанных настраиваемым действием. Сюда входят свойства, которые могут предоставлять данные настраиваемому действию, в частности те, которые используют открытые свойства, предоставленные пользователем.
-   Не полагайтесь на внешние библиотеки DLL, которые не являются доверенными для системы на всех платформах, на которых должен выполняться установочный пакет.
-   Тщательно продумайте, следует ли использовать настраиваемые действия, использующие [*повышенные*](e-gly.md) привилегии или олицетворение. Пользовательские действия, такие как "открыть URL-адрес после установки", "" запустить программное обеспечение после завершения установки "или" запустить файл ReadMe после завершения установки ", обычно не требуют повышенных привилегий для работы. Если настраиваемое действие должно выполняться с *повышенными* привилегиями, убедитесь, что код настраиваемого действия предотвращает переполнение буфера и случайную загрузку ненадежного кода. Обратите внимание, что на этапе выполнения установки установщик передает сведения процессу с *повышенными* привилегиями и запускает скрипт. Любые настраиваемые действия, выполняемые на этапе выполнения, могут выполняться с *повышенными* привилегиями.
-   Соберите все сведения, предоставленные пользователем во время выполнения последовательности пользовательского интерфейса. Не запрашивать у пользователя какие-либо сведения, которые нельзя задать с помощью общедоступного свойства.
-   Если настраиваемое действие сценария развертывает свойства, примите меры предосторожности о том, что пользовательское действие защищено от возможности внедрения сценария. Сценарий может быть записан в виде открытого текста.

См. также [Безопасность настраиваемых действий](custom-action-security.md).
