# <a name="removeexistingproducts-action"></a>Действие RemoveExistingProducts

Действие RemoveExistingProducts проходит по кодам продуктов, перечисленным в столбце Актионпроперти [таблицы Upgrade](upgrade-table.md) , и удаляет продукты последовательно путем вызова параллельных установок. Для каждой параллельной установки установщик задает для свойства [**ProductCode**](productcode.md) код продукта и задает для свойства [**Remove**](remove.md) значение в поле удалить таблицы Upgrade. Если поле удалить пустое, его значение по умолчанию равно ALL, и установщик удалит весь продукт.

Установщик выполняет только действие RemoveExistingProducts при первой установке продукта. Он не выполняет действие во время установки или удаления [обслуживания](maintenance-installation.md) .

## <a name="sequence-restrictions"></a>Ограничения последовательности

Действие RemoveExistingProducts должно быть запланировано в последовательности действий в одном из следующих расположений.

-   Между [действием инсталлвалидате](installvalidate-action.md) и [действием инсталлинитиализе](installinitialize-action.md). В этом случае установщик удаляет старые приложения полностью перед установкой новых приложений. Это неэффективное размещение действия, поскольку необходимо повторно скопировать все повторно используемые файлы.
-   После [действия инсталлинитиализе](installinitialize-action.md) и перед всеми действиями, создающими скрипт выполнения.
-   Между [действием инсталлексекуте](installexecute-action.md), [действием инсталлексекутеагаин](installexecuteagain-action.md)и [действием функции InstallFinalize](installfinalize-action.md). Как правило, последние три действия запланированы прямо друг за другом: Инсталлексекуте, RemoveExistingProducts и функции InstallFinalize. В этом случае сначала устанавливаются обновленные файлы, а затем старые файлы удаляются. Однако если при удалении старого приложения произойдет сбой, то программа установки произведет откат и удаление старого приложения, и установку нового приложения.
-   После [действия функции InstallFinalize](installfinalize-action.md). Это наиболее эффективный способ размещения действия. В этом случае установщик обновляет файлы перед удалением старых приложений. Во время установки устанавливаются только обновляемые файлы. В случае сбоя удаления старого приложения Установщик выполняет откат только удаления старого приложения.
