# <a name="fastoem-property"></a>ФАСТОЕМ, свойство

свойство **фастоем** предназначено для того, чтобы позволить изготовителям оборудования сократить время, затрачиваемое на установку установщик Windows приложений для конкретного сценария. Не Разрабатывайте свойство **фастоем** в [таблице свойств](property-table.md).

Свойство **фастоем** применимо только в том случае, если выполняются все следующие условия.

-   Приложение устанавливается на том же томе, что и папка с исходными файлами.
-   Исходные файлы удаляются после установки.
-   Во время установки пользовательский интерфейс не отображается. [Уровень пользовательского интерфейса](user-interface-levels.md) — None.
-   Установка выполняется в [контексте установки](installation-context.md)на компьютере.
-   На компьютере достаточно места для успешной установки.
-   Это первое время установки. Установка не является рекламным, переустановкой, удалением или выполнением административной установки.
-   Нет установленных компонентов для запуска из источника.
-   Пакет установки не содержит [изолированных компонентов](isolated-components.md). Поскольку изолированным компонентам требуется, чтобы исходные файлы оставались расположенными в источнике, свойство **фастоем** в настоящее время не может использоваться с изолированными компонентами.

если все предыдущие условия имеют значение true, установка свойства **фастоем** позволяет установщик Windows повысить производительность, выполнив следующие действия.

-   Перемещение, а не копирование файлов на том же томе. Это не гарантирует, что все файлы будут перемещены, а не скопированы. Обратите внимание, что если компьютер имеет несколько жестких дисков, необходимо инициализировать свойство [**рутдриве**](rootdrive.md) в командной строке на том же диске, где находится источник установки.
-   Исключите этот источник из списка источников приложения, чтобы при последующих установках или обслуживании по умолчанию были указаны источники компакт-дисков, указанные в [таблице Media](media-table.md).
-   Оптимизируйте [стоимость файлов](file-costing.md).
-   подавление сообщений о ходе выполнения, отправленных с установщик Windows клиенту.

## <a name="remarks"></a>Remarks

Поскольку сообщения о ходе выполнения не отправляются при установке **фастоем** , рекомендуется, чтобы авторы вручную записывали в раздел значение 1800 для времени ожидания.

**HKLM** \\ **Программное обеспечение** \\ **Политики** \\ **Microsoft** \\ **Windows** \\ **Установщик** \\ **Время ожидания**

**\_ Параметр** timeout имеет тип reg.

чтобы отобразить размер приложения в окне "установка и удаление программ" на панели управления Windows 2000, необходимо вручную записать значение EstimatedSize в раздел.

**HKLM** \\ **Программное обеспечение** \\ **Microsoft** \\ **Windows** \\ **CurrentVersion** \\ **Удалить** \\ **<  Код > продукта**

Это тип **\_ DWORD reg** и равен размеру приложения в килобайтах. Установщик не записывает это значение автоматически.

Используйте следующий пример командной строки, если компакт-диск, поставляемый конечным пользователям, хранит пакет установки приложения в корне компакт-диска. Обратите внимание, что метка тома в [таблице Media](media-table.md) файла .msi должна совпадать с МЕТКОЙ тома компакт-диска.

В этом случае в качестве значения свойства [**медиапаккажепас**](mediapackagepath.md) необходимо указать путь к установочному пакету.
