# <a name="removefolders-action"></a>Действие Ремовефолдерс

Действие Ремовефолдерс удаляет все папки, связанные с набором компонентов, которые будут удалены или запускаться из источника. Эти папки удаляются только в том случае, если они пусты. Если папка удалена, она отменяется регистрацией с соответствующим идентификатором компонента.

## <a name="sequence-restrictions"></a>Ограничения последовательности

Действие Ремовефолдерс должно быть получено после действия [ремовефилес](removefiles-action.md) или любого действия, которое может удалить файлы из папок.

## <a name="actiondata-messages"></a>Сообщения Актиондата



| Поле | Описание данных действия    |
|-------|-------------------------------|
| \[1\] | Идентификатор удаленной папки. |



 

## <a name="remarks"></a>Remarks

Установщик не удаляет автоматически папки, созданные [действием креатефолдерс](createfolders-action.md) во время удаления приложения. Установщику необходимо удалить эти папки, разработав действие Ремовефолдерс в последовательности действий.

Чтобы указать имя и расположение папки, используйте \_ столбец Directory [таблицы креатефолдер](createfolder-table.md). Имя каждой папки в таблице Креатефолдер считается свойством, определяющим расположение папки.

Чтобы указать компонент, владеющий папкой, используйте столбец ComponentId [таблицы Component](component-table.md).
