# <a name="installuisequence-table"></a>Таблица Инсталлуисекуенце

В таблице Инсталлуисекуенце перечислены действия, выполняемые при выполнении [действия по установке](install-action.md) верхнего уровня, а для внутреннего уровня пользовательского интерфейса — полный пользовательский интерфейс или СОКРАЩЕНный пользовательский интерфейс. Установщик пропускает действия в этой таблице, если на уровне пользовательского интерфейса задан базовый пользовательский интерфейс или нет пользовательского интерфейса. См. [сведения о пользовательском интерфейсе](about-the-user-interface.md).

Действия в последовательности установки вплоть до [действия инсталлвалидате](installvalidate-action.md)и диалоговых окон выхода находятся в таблице инсталлуисекуенце. Все действия из Инсталлвалидате в конце последовательности установки находятся в [таблице инсталлексекутесекуенце](installexecutesequence-table.md). Так как таблица Инсталлексекутесекуенце должна быть изолированной, она имеет необходимые действия инициализации, такие как [лаунчкондитионс](launchconditions-action.md), [костинитиализе](costinitialize-action.md), [филекост](filecost-action.md)и [костфинализе](costfinalize-action.md), а также [действие ExecuteAction](executeaction-action.md).

Таблица Инсталлуисекуенце содержит следующие столбцы.



| Столбец    | Type                         | Ключ | Допускает значения NULL |
|-----------|------------------------------|-----|----------|
| Действие    | [Идентификатор](identifier.md) | Да   | Нет        |
| Условие | [Condition](condition.md)   | Нет   | Да        |
| Последовательность  | [Integer](integer.md)       | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="Action"></span><span id="action"></span><span id="ACTION"></span>Поддержки
</dt> <dd>

Имя выполняемого действия. Это либо встроенное действие, пользовательское действие, либо мастер пользовательского интерфейса.

Первичный ключ таблицы.

</dd> <dt>

<span id="Condition"></span><span id="condition"></span><span id="CONDITION"></span>Выполняет
</dt> <dd>

Это поле содержит условное выражение. Если выражение принимает значение false, действие пропускается. Если синтаксис выражения является недопустимым, последовательность завершается, возвращая Иесбадактиондата. Дополнительные сведения о синтаксисе условных операторов см. в разделе [Синтаксис условных](conditional-statement-syntax.md)инструкций.

</dd> <dt>

<span id="Sequence"></span><span id="sequence"></span><span id="SEQUENCE"></span>Виртуализирован
</dt> <dd>

Число в этом столбце определяет расположение последовательности, в которой выполняется это действие.

Положительное значение представляет собой расположение последовательности. Значение NULL указывает, что действие никогда не выполняется. Следующие отрицательные значения указывают на то, что это действие выполняется, если установщик возвращает соответствующий флаг завершения. Каждый флаг завершения (отрицательное значение) можно использовать без более чем одного действия. Несколько действий могут иметь флаги завершения, но они должны быть разными флагами. Флаги завершения (отрицательные значения) обычно используются с [диалоговыми окнами](dialog-boxes.md).



| Флаг завершения          | Значение | Описание                                                                          |
|---------------------------|-------|--------------------------------------------------------------------------------------|
| мсидоактионстатуссукцесс  | -1    | Успешное завершение. Используется с диалоговыми окнами [выхода](exit-dialog.md) .               |
| мсидоактионстатусусерексит | -2    | Пользователь завершает установку. Используется с диалоговыми окнами [усерексит](userexit-dialog.md) .     |
| мсидоактионстатусфаилуре  | –3    | Неустранимое завершение работы. Используется с диалоговыми окнами [FatalError](fatalerror-dialog.md) . |
| мсидоактионстатуссуспенд  | –4    | Установка приостановлена.                                                                |



 

Ноль, все остальные отрицательные числа или значение NULL указывают на то, что действие никогда не выполняется.

</dd> </dl>
