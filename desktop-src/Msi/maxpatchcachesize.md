# <a name="maxpatchcachesize"></a>макспатчкачесизе

если для этой [системной политики](system-policy.md) на уровне компьютера задано значение больше 0, установщик Windows сохраняет старые версии файлов в кэше при применении исправления к приложению. Кэширование может повысить производительность будущих установок, которые в противном случае должны получить старые файлы из исходного источника приложения.

Значение политики Макспатчкачесизе — это максимальный процент дискового пространства, который установщик может использовать для кэша старых файлов. Например, значение 20 указывает, что используется не более 20%. Если общий размер кэша достигает указанного процента места на диске, дополнительные файлы не сохраняются в кэше. Политика не влияет на уже сохраненные файлы.

Если значение политики Макспатчкачесизе равно 0, дополнительные файлы не сохраняются.

Если политика Макспатчкачесизе не задана, по умолчанию используется значение 10, а для сохранения старых файлов можно использовать не более 10% места на диске.

## <a name="registry-key"></a>Ключ реестра

**HKey \_ \_** \\ **политики программного обеспечения** локального компьютера \\  \\  \\  \\ **установщик** Microsoft Windows
