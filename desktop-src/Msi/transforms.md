# <a name="transforms-property"></a>Свойство TRANSFORMs

Свойство **TRANSFORMS** представляет собой список преобразований, применяемых установщиком при установке пакета. Установщик применяет преобразования в том же порядке, в котором они указаны в свойстве. Преобразования могут быть заданы по имени файла или полному пути. Чтобы указать несколько преобразований, разделяйте каждое имя файла или полный путь точкой с запятой (;). Например, чтобы применить три преобразования к пакету, задайте для **преобразований** список имен файлов или список полных путей.

``` syntax
TRANSFORMS=transform1.mst;transform2.mst;transform3.mst
TRANSFORMS=\\server\share\path\transform1.mst;\\server2\share2\path2\transform2.mst;\\server3\share3\path3\transform3.mst
```

Можно указать, что файл преобразования внедряется в хранилище файла .msi, а не в автономный файл, представив префикс имени файла с двоеточием (:). Например, в следующем примере показано, что в файл .msi внедрены файлы. MST и transform3. MST, а также этот файл является автономным файлом.

``` syntax
TRANSFORMS=:transform1.mst;:transform2.mst;transform3.mst
```

Установщик требует преобразования, перечисленные в **ПРЕобразованиях** при каждой установке пакета, объявления, установки по требованию или при обслуживании. Политика [политики трансформссекуре](transformssecure-policy.md) , свойство **TRANSFORMS** и первый символ строки **преобразований** информирует установщик о том, как управлять устойчивостью источника автономных файлов преобразования. Windows Установщик рассматривает параметр [трансформсатсаурце Policy](transformsatsource-policy.md) или [**трансформсатсаурце**](transformsatsource.md) так же, как трансформссекуре Policy и [**трансформссекуре**](transformssecure.md). Обратите внимание, что преобразования, внедренные в файл .msi, не кэшируются и всегда получаются из пакета.

В одном списке **преобразований** нельзя одновременно использовать имена файлов и пути. В одном списке нельзя указывать безопасные и защищенные преобразования. Преобразования, внедренные в пакет, можно включить в список с другими преобразованиями.

``` syntax
@transform1.mst;:transform2.mst 
|\\server\share\path\transform1.mst;:transform2.mst
```

Обратите внимание, что, поскольку разделитель списка для преобразований является символом точки с запятой, точка с запятой не должна использоваться в имени файла преобразования или пути.

## <a name="remarks"></a>Комментарии

в случаях, когда для [политики трансформссекуре](transformssecure-policy.md) или свойства [**трансформссекуре**](transformssecure.md) задано значение установщик Windows, \| при настройке **преобразований** с помощью командной строки необязательно передавать символ @ или. Если список состоит только из имен файлов, расположенных в источнике или состоящих только из полных путей, установщик предполагает, что он защищен-at-source или Secure-Full-Path. Вы по-прежнему не можете смешивать два типа источников преобразования.

Обратите внимание, что установщик использует другой порядок поиска для незащищенных преобразований, применяемых во время первого выполнения установки и обслуживания. Дополнительные сведения см. в разделе [незащищенные преобразования](unsecured-transforms.md).
