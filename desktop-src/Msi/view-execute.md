# <a name="viewexecute-method"></a>View.Exeметод милые

Метод **EXECUTE** объекта [**View**](view-object.md) использует маркер вопросительного знака для представления параметров в инструкции SQL. Дополнительные сведения см. в разделе [синтаксис SQL](sql-syntax.md). Значения этих параметров передаются в качестве соответствующих полей записи параметра.

## <a name="syntax"></a>Синтаксис


```JScript
View.Execute(
  record
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*record* 
</dt> <dd>

Необязательные объекты [**Record**](record-object.md) , которые содержат значения, заменяющие маркеры параметров (?) в SQL-запросе.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Комментарии

Этот метод должен вызываться перед вызовом метода [**Fetch**](view-fetch.md) .

Если SQL-запрос указывает значения с маркерами параметров (?), необходимо указать запись, содержащую все замещающие значения, которые должны находиться в том же порядке и в том же типе данных, что и маркеры параметров. Если этот метод используется с запросами INSERT и UPDATE, маркеры вопросительного знака должны предшествовать всем непараметризованным значениям.

Например, эти запросы являются допустимыми:

Обновление {Table-List} SET {Column} =? , {Column} = {константа}

ВСТАВИТЬ в {Table} ({Column-List}) значения (?, {Constant-List})

Однако эти запросы являются недопустимыми:

Обновление {Table-List} SET {Column} = {константа}, {Column} =?

ВСТАВИТЬ в {Table} ({Column-List}) значения ({Constant-List},? )

В случае сбоя метода можно получить расширенные сведения об ошибке с помощью метода [**ластерроррекорд**](installer-lasterrorrecord.md) .
