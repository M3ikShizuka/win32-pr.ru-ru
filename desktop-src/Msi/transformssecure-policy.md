# <a name="transformssecure-policy"></a>Политика Трансформссекуре

Установка для политики Трансформссекуре значения 1 информирует установщик о том, что преобразования должны быть кэшированы локально на компьютере пользователя в том месте, где у пользователя нет доступа на запись. Установка этой политики аналогична настройке [Свойства трансформссекуре](transformssecure.md) , за исключением того, что область отличается. Настройка политики Трансформссекуре применяется ко всем пакетам, установленным на данном компьютере. Установка свойства ТРАНСФОРМССЕКУРЕ применяется к пакету независимо от компьютера.

цель этой политики — предоставить безопасное хранилище преобразований для пользователей в командировках с Windows 2000. начиная с Windows Server 2003, значение этой политики по умолчанию равно 1.

Если эта политика задана, [Установка обслуживания](maintenance-installation.md) может использовать только преобразование из защищенного кэша. Если установщик обнаружит, что преобразование отсутствует на локальном компьютере, установщик попытается восстановить преобразование. Чтобы установщик мог восстановить преобразование, безопасное преобразование должно находиться в списке источников установочного пакета в полномочном источнике. Дополнительные сведения см. в разделе [устойчивость источника](source-resiliency.md). Установка обслуживания завершается сбоем, если преобразование недоступно или не может быть загружено.

в Windows Server 2003, если эта политика не задана, по умолчанию преобразования хранятся в безопасном месте. для всех версий, предшествовавших Windows Server 2003, если политика не задана, по умолчанию преобразуются в профиль пользователя.

Дополнительные сведения см. в разделе также [безопасные преобразования](secured-transforms.md) .

Windows Установщик интерпретирует [политику трансформсатсаурце](transformsatsource-policy.md) как ту же, что и политика трансформссекуре.
