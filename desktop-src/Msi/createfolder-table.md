# <a name="createfolder-table"></a>Таблица Креатефолдер

Таблица Креатефолдер содержит ссылки на папки, которые необходимо явно создать для конкретного компонента.

Таблица Креатефолдер содержит следующие столбцы.



| Столбец      | Type                         | Ключ | Допускает значения NULL |
|-------------|------------------------------|-----|----------|
| Каталог\_ | [Идентификатор](identifier.md) | Да   | Нет        |
| Компонент\_ | [Идентификатор](identifier.md) | Да   | Нет        |

<dl> <dt>

</dt> <dd>

Внешний ключ в первом столбце [таблицы Directory](directory-table.md).

</dd> <dt>

</dt> <dd>

Внешний ключ в первый столбец [таблицы Component](component-table.md).

</dd> </dl>

## <a name="remarks"></a>Комментарии

Папки в этой таблице создаются при установке компонента. Предпринята попытка удалить эти папки только в том случае, если компонент удален или перемещен в исходный код запуска. Автоматическое удаление не запускается, если папки становятся пустыми. В отличие от этого, папки, созданные установщиком, но не перечисленные в этой таблице, удаляются, когда они становятся пустыми.

Поскольку папки, созданные установщиком, удаляются, когда они становятся пустыми, необходимо создать запись в таблице Креатефолдер, чтобы установить компонент, состоящий из пустой папки.

Эта таблица упоминается при вызове действия [креатефолдерс](createfolders-action.md) или [ремовефолдерс](removefolders-action.md) .

Сведения о том, как защитить папку, см. в [таблице мсилоккпермиссионсекс](msilockpermissionsex-table.md) и таблице [локкпермиссионс](lockpermissions-table.md).
