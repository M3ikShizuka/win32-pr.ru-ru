# <a name="typelib-table"></a>Таблица TypeLib

Таблица TypeLib содержит сведения, которые должны быть помещены в реестр для регистрации библиотек типов.

Таблица TypeLib содержит следующие столбцы.



| Столбец      | Type                               | Ключ | Допускает значения NULL |
|-------------|------------------------------------|-----|----------|
| ID       | [GUID](guid.md)                   | Да   | Нет        |
| Язык    | [Integer](integer.md)             | Да   | Нет        |
| Компонент\_ | [Идентификатор](identifier.md)       | Да   | Нет        |
| Версия     | [даублеинтежер](doubleinteger.md) | Нет   | Да        |
| Описание | [Text](text.md)                   | Нет   | Да        |
| Каталог\_ | [Идентификатор](identifier.md)       | Нет   | Да        |
| Функция\_   | [Идентификатор](identifier.md)       | Нет   | Нет        |
| Стоимость        | [даублеинтежер](doubleinteger.md) | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

## <a name="remarks"></a>Комментарии

Эта таблица упоминается при выполнении [действия регистертипелибрариес](registertypelibraries-action.md) или [унрегистертипелибрариес](unregistertypelibraries-action.md) .

Установщик записывает все сведения о регистрации библиотеки типов в \_ \_ расположение реестра для ЛОКАЛЬНОГО компьютера hKey (HKLM). Это происходит даже при установке отдельных пользователей. Библиотеки типов не могут быть зарегистрированы в расположениях для отдельных пользователей (HKCU).

Авторы пакета установки настоятельно рекомендуем использовать таблицу TypeLib. Вместо этого они должны регистрировать библиотеки типов с помощью таблицы [реестра](registry-table.md) . Причины для предотвращения самостоятельной регистрации:

-   Если установка, использующая таблицу TypeLib, завершается ошибкой и должна быть выполнена откат, то откат может не восстановить состояние компьютера, существовавшего до отката. Библиотеки типов, зарегистрированные до отката, могут быть не зарегистрированы после отката.
