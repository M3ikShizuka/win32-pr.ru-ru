# <a name="installerenablelog-method"></a>Метод Installer. EnableLog

Метод **EnableLog** объекта [**Installer**](installer-object.md) позволяет вести журнал выбранного типа сообщений для всех последующих сеансов установки в пространстве текущего процесса.

## <a name="syntax"></a>Синтаксис


```JScript
Installer.EnableLog(
  logMode,
  logFile
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*logMode* 
</dt> <dd>

Обязательная строка, которая содержит буквы, представляющие типы сообщений для записи в журнал. Строка может быть сочетанием следующих значений.



| Значение | Описание                                                                                            |
|-------|--------------------------------------------------------------------------------------------------------|
| I     | Информационные сообщения.                                                                             |
| w     | Некритические предупреждающие сообщения.                                                                            |
| Д.     | Сообщения об ошибках, которые могут быть неустранимыми ошибками.                                                               |
| f     | Список используемых файлов, которые необходимо заменить.                                                         |
| а     | Уведомление о начале действия.                                                                          |
| r     | Запись данных действия, содержащая содержимое, относящееся к действию.                                              |
| u     | Сообщения запроса пользователя.                                                                                 |
| с     | Параметры инициализации пользовательского интерфейса.                                                                          |
| m     | Сообщение о нехватке памяти.                                                                                 |
| v     | Передает большой объем информации в файл журнала, который обычно не используется для пользователей. Может использоваться для поддержки. |
| p     | Дамп таблицы свойств; "свойство = значение" при завершении ядра                                          |
| \+    | Добавление к существующему файлу журнала.                                                                           |
| !     | Запись каждой строки в файл журнала.                                                                       |
| x     | Дополнительные отладочные сведения. этот параметр доступен только для Windows Server 2003.                      |
| o     | Сообщения о нехватке места на диске.                                                                            |



 

</dd> <dt>

*logFile* 
</dt> <dd>

Обязательная строка, содержащая путь к создаваемому файлу журнала. Чтобы отключить ведение журнала, используйте пустую строку ("").

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Комментарии

Путь к расположению файла журнала должен уже существовать при использовании этого метода. Установщик не создает структуру каталогов для файла журнала.

параметры ведения журнала, заданные с помощью **EnableLog** , переопределяют все существующие параметры политики ведения журнала установщик Windows.

По умолчанию журнал перезаписывает существующий файл журнала. Для добавления к существующему файлу журнала необходимо использовать букву "+" в режиме ведения журнала.

Параметр "!" не рекомендуется, так как он может значительно замедлять установку. Этот параметр может быть полезен при отладке установки.

Следующий пример скрипта включает подробное ведение журнала для установки. В конце установки созданный файл журнала будет иметь значение c: \\ temp \\ install. log.


```VB
    Dim Installer
    Set Installer = CreateObject("WindowsInstaller.Installer")
    Installer.EnableLog "voicewarmup", "c:\temp\install.log"
    Installer.InstallProduct "\\server\share\products\sample\sample.msi"
```
