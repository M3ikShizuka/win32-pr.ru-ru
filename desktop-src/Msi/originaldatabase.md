# <a name="originaldatabase-property"></a>Оригиналдатабасе, свойство

установщик Windows задает для свойства **оригиналдатабасе** путь к базе данных установки, используемой для запуска установки. Если установка запускается из командной строки, это значение зависит от того, есть ли в свойстве [**REINSTALLMODE**](reinstallmode.md) параметр пакета повторного кэширования (флаг-v).



| Метод установки                                                                                                                                                                                  | Значение Оригиналдатабасе                        |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------|
| Любая установка, запущенная путем вызова пути к установочному пакету (.msi файл).                                                                                                              | Путь к установочному пакету (файл .msi). |
| Установка, запущенная из командной строки. Установка не запускается из пути пакета. Параметр повторного кэширования (флаг-v) имеется в свойстве [**REINSTALLMODE**](reinstallmode.md) .     | Путь к базе данных в источнике.           |
| Установка, запущенная из командной строки. Установка не запускается из пути пакета. Параметр повторного кэширования (флаг-v) отсутствует в свойстве [**REINSTALLMODE**](reinstallmode.md) . | Путь к кэшированной базе данных.                  |



 

## <a name="remarks"></a>Remarks

Во время первой установки пользовательское действие, выполняемое перед [действием ресолвесаурце](resolvesource-action.md) , может использовать свойство **оригиналдатабасе** для определения расположения источника установки.
