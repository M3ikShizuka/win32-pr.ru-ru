# <a name="customization"></a>Настройка

поскольку установщик Windows сохраняет все сведения об установке в реляционной базе данных, установку приложения или продукта можно настроить для определенных групп пользователей, применяя операции преобразования к пакету. Преобразования можно использовать для инкапсуляции различных настроек основного пакета, которые требуются для различных рабочих групп. Дополнительные сведения см. в разделе [слияния и преобразования](merges-and-transforms.md).

Во время установки несколько преобразований базового пакета могут быть применены в режиме реального времени. Это обеспечивает механизм эффективного назначения настраиваемых установок разным группам пользователей. Например, это может быть полезно в организациях, где отделы поддержки финансов и штатного отдела требуют различных установок конкретного продукта. Продукт можно сделать доступным для всех пользователей в Организации с помощью [административной установки](administrative-installation.md) базового пакета в одну точку административной установки. Каждая Рабочая группа может автоматически получить соответствующую настройку с помощью преобразования "на лету" при установке продукта.
