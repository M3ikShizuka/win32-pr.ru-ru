# <a name="directorycombo-control"></a>Элемент управления Директорикомбо

\_Элемент управления директорикомбо отображает часть пути, которая в данный момент отображается в [элементе управления паседит](pathedit-control.md). Этот элемент управления не показывает последний сегмент пути, этот сегмент отображается [элементом управления директорилист](directorylist-control.md).

\_Элемент управления директорикомбо отображает все доступные тома в алфавитном порядке и иерархические шаги текущего пути. Если выбранный путь содержит какие-либо папки, которые не существуют, эти файлы будут отображаться с другим значком. Типы отображаемых томов задаются с помощью битов, связанных с элементами управления [ремоваблеволуме](removablevolume-control-attribute.md), [фикседволуме](fixedvolume-control-attribute.md), [ремотеволуме](remotevolume-control-attribute.md), [кдромволуме](cdromvolume-control-attribute.md), [рамдискволуме](ramdiskvolume-control-attribute.md)и [флоппиволуме](floppyvolume-control-attribute.md) .

Элементы управления Паседит, Директорикомбо и Директорилист связаны с одним и тем же свойством со строковыми значениями. Это свойство является путем, выбранным пользователем. Введите имя свойства в столбец свойств [таблицы управления](control-table.md). Это свойство должно иметь начальное значение, содержащее по крайней мере один том и один подуровень. Укажите начальное значение для свойства в столбце значение [таблицы свойств](property-table.md).

Этот элемент управления предназначен для использования в [диалоговом окне обзора](browse-dialog.md) вместе с элементами управления [паседит](pathedit-control.md) и [директорилист](directorylist-control.md) .

## <a name="remarks"></a>Remarks

Этот элемент управления можно создать из класса COMBOBOX с помощью функции [**CreateWindowEx**](/windows/win32/api/winuser/nf-winuser-createwindowexa) . В нем имеются **Стили \_ CBS**, **CBS \_ овнердравфиксед**, **CBS \_ хасстрингс**, **WS \_ Child**, **WS \_ Group**, **WS \_ TABSTOP** и **WS \_ VSCROLL** .
