# <a name="movefile-table"></a>Таблица MoveFile

Эта таблица содержит список файлов, которые будут перемещены или скопированы из указанного исходного каталога в указанный каталог назначения.

Таблица MoveFile содержит следующие столбцы.



| Столбец       | Type                         | Ключ | Допускает значения NULL |
|--------------|------------------------------|-----|----------|
| филекэй      | [Идентификатор](identifier.md) | Да   | Нет        |
| Компонент\_  | [Идентификатор](identifier.md) | Нет   | Нет        |
| SourceName   | [Text](text.md)             | Нет   | Да        |
| дестнаме     | [Имя файла](filename.md)     | Нет   | Да        |
| SourceFolder | [Идентификатор](identifier.md) | Нет   | Да        |
| DestFolder   | [Идентификатор](identifier.md) | Нет   | Нет        |
| Параметры      | [Integer](integer.md)       | Нет   | Нет        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="FileKey"></span><span id="filekey"></span><span id="FILEKEY"></span>филекэй
</dt> <dd>

Первичный ключ, однозначно определяющий определенную запись MoveFile.

</dd> <dt>

<span id="Component_"></span><span id="component_"></span><span id="COMPONENT_"></span>См\_
</dt> <dd>

Внешний ключ в [таблице Component](component-table.md). Если компонент, на который ссылается этот ключ, не выбран для установки или удаления, для этой записи MoveFile не выполняются никакие действия.

</dd> <dt>

<span id="SourceName"></span><span id="sourcename"></span><span id="SOURCENAME"></span>SourceName
</dt> <dd>

Этот столбец содержит Локализуемое имя перемещаемых или копируемых исходных файлов. Этот столбец можно оставить пустым. См. описание столбца SourceFolder. Это поле должно содержать длинное имя файла и может содержать символы-шаблоны ( \* и?).

</dd> <dt>

<span id="DestName"></span><span id="destname"></span><span id="DESTNAME"></span>дестнаме
</dt> <dd>

Этот столбец содержит Локализуемое имя, присваиваемое исходному файлу после его перемещения или копирования. Если это поле пустое, то конечному файлу присваивается то же имя, что и у исходного файла.

</dd> <dt>

<span id="SourceFolder"></span><span id="sourcefolder"></span><span id="SOURCEFOLDER"></span>SourceFolder
</dt> <dd>

Этот столбец содержит имя свойства со значением, которое разрешается в полный путь к исходному каталогу. Если столбец SourceName оставлен пустым, то предполагается, что свойство, названное в столбце SourceFolder, содержит полный путь к самому исходному файлу (включая имя файла).

</dd> <dt>

<span id="DestFolder"></span><span id="destfolder"></span><span id="DESTFOLDER"></span>DestFolder
</dt> <dd>

Имя свойства, значение которого разрешается в полный путь к целевому каталогу.

</dd> <dt>

<span id="Options"></span><span id="options"></span><span id="OPTIONS"></span>Параметры
</dt> <dd>

Целочисленное значение, определяющее режим работы.



| Константа                     | Шестнадцатеричный | Decimal | Значение               |
|------------------------------|-------------|---------|-----------------------|
| (нет)                       | 0x000       | 0       | Скопируйте исходный файл. |
| **мсидбмовефилеоптионсмове** | 0x001       | 1       | Переместите исходный файл. |



 

</dd> </dl>

## <a name="remarks"></a>Комментарии

Если в столбце \* SourceName таблицы MoveFile указан подстановочный знак "", а имя целевого файла указано в столбце дестнаме, то все перемещенные или скопированные файлы сохранили имена в источниках.

Эта таблица обрабатывается [действием мовефилес](movefiles-action.md).
