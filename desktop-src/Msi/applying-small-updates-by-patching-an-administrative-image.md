# <a name="applying-small-updates-by-patching-an-administrative-image"></a>Применение небольших обновлений путем исправления административного образа

При [административной установке](administrative-installation.md) создается исходный образ приложения или продукта в сети. Пользователи рабочей группы, которые имеют доступ к этому административному образу, могут установить продукт из этого источника. Обновление приложения для этой Рабочей группы выполняется в два этапа:

-   Во – первых, к административному образу можно применить небольшое исправление обновления.
-   Во вторых, изменения в небольшом обновлении должны распространяться на пользователей.

**Применение небольшого исправления обновления к административному образу**

1.  Получите небольшое обновление в виде [пакета исправлений](patch-packages.md). Например, небольшое обновление с именем patch. MSP.
2.  Получите путь к административному образу.
3.  Из командной строки используйте:

    **msiexec/a** *\[ путь к административному образу. \] MSI файл* **/p** *patch. MSP*

4.  При этом обновляются файлы приложения и MSI административного образа. Список параметров, которые можно использовать с Msiexec.exe, см. в разделе [Параметры командной строки](command-line-options.md). Установщик Windows автоматически определяет, использует ли административный образ короткие имена файлов и задает свойство [**шортфиленамес**](shortfilenames.md) .
5.  Итоговый административный образ аналогичен тому, который был создан административной установкой с помощью полного компакт-диска, включающего обновление. Когда новые пользователи устанавливают приложение из этого источника, они получают обновленное приложение.

**Распространение небольшого обновления в рабочую группу**

-   Члены Рабочей группы получают изменения путем повторной установки приложения из административного образа с помощью процедуры, описанной в разделе [применение небольших обновлений путем переустановки продукта](applying-small-updates-by-reinstalling-the-product.md).
