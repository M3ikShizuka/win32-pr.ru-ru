# <a name="windows-installer-error-messages-for-developers"></a>Установщик Windows сообщения об ошибках (для разработчиков)

> [!NOTE]  
> Эта статья предназначена для разработчиков программного обеспечения, которые используют установщик Windows для создания пакетов установщика для своих приложений. Если у пользователя возникли трудности с компьютером во время или после установки или удаления приложения, обратитесь в службу поддержки клиентов за программное обеспечение, которое вы пытаетесь установить или удалить. Если вы считаете, что вам нужна поддержка продукта Майкрософт, перейдите на наш сайт технической поддержки по адресу [support.Microsoft.com](https://support.microsoft.com).

Коды ошибок, описанные в этом разделе, возвращаются установщик Windows и содержат коды ошибок 1000 и выше. Коды ошибок 1000 – 1999 — поставлять сообщения об ошибках и должны быть созданы в [таблице ошибок](error-table.md). Коды ошибок, превышающие 2000, являются внутренними ошибками и не имеют созданных строк, но могут возникать, если пакет установки был создан неправильно. Коды ошибок, характерные для функций установщик Windows **MsiExec.exe** и **InstMsi.exe**, см. в разделе [MsiExec.exe и InstMsi.exe сообщения об ошибках](error-codes.md). Список зарезервированных кодов ошибок см. в разделе [Таблица ошибок](error-table.md). В Интернете или на [сайте службы поддержки Майкрософт](https://support.microsoft.com/) можно найти решения для многих сообщений, приведенных в следующей таблице.

Разработчики пакетов установки также могут проверить внутреннюю согласованность их пакетов, используя внутренние средства оценки согласованности. Дополнительные сведения см. в разделе [оценщики внутренней согласованности — ICEs](internal-consistency-evaluators-ices.md).
