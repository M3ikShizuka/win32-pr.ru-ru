# <a name="administrative-installation"></a>Административная установка

Установщик Windows может выполнять административную установку приложения или продукта в сеть для использования в Рабочей группе. Административная установка устанавливает в сеть исходный образ приложения, аналогичный исходному образу на компакт-диске. Пользователи в Рабочей группе, которые имеют доступ к этому административному образу, могут затем установить продукт из этого источника. Пользователь должен сначала установить продукт из сети, чтобы запустить приложение. Пользователь может выбрать Запуск из источника при установке, а установщик использует большую часть файла продукта непосредственно из сети.

Администраторы могут запускать административную установку из командной строки с помощью [параметра командной строки](command-line-options.md)/a.

[Действие администратора](admin-action.md) — это действие верхнего уровня, используемое для запуска административной установки. При выполнении этого действия установщик вызывает действия в таблицах [админексекутесекуенце](adminexecutesequence-table.md) и [админуисекуенце](adminuisequence-table.md) для выполнения административной установки.

Свойство [**админпропертиес**](adminproperties.md) представляет собой разделенный точками с запятой список свойств, которые задаются во время установки администрирования. Установщик использует эти свойства во время установки приложения, выполняемого после администрирования, из административного образа.

Пользователи, у которых нет постоянного доступа к сети, могут установить приложение из административного образа, а иногда в качестве источника резервного копирования должны полагаться на носители, например диски компакт-дисков. В таких случаях длина имен файлов в административном образе и на носителе должны совпадать. Оба файла должны использовать длинные имена файлов, или оба они должны использовать короткие имена файлов. Например, компакт-диск, который поддерживает только короткие имена файлов, может предоставить как исходный носитель для установки административного образа, так и источник резервного копирования.

Если свойство [**шортфиленамес**](shortfilenames.md) задано во время административной установки, это свойство может потребоваться установить повторно с помощью пользователя, который впоследствии применяет исправление к административному образу. При использовании установщик Windows для применения исправления установщик автоматически устанавливает свойство **шортфиленамес** , если в административном образе используются короткие имена файлов.

Если администратор использует пакет со свойством " [**Сводка слов**](word-count-summary.md) " со значением 2 или 3 для выполнения административной установки, пользователи административного образа не могут автоматически переустановиться с исходного носителя. Если административный образ становится недоступным, пользователи, установленные из административного образа, не смогут вернуться к исходному носителю.

Применение [*преобразований*](t-gly.md) во время создания административного образа не имеет действующего действия. Чтобы сделать настроенную версию продукта доступной Рабочей группе, примените преобразование во время установки продукта из административного образа.

Во время административной установки установщик создает исходный образ для всех компонентов продукта, за исключением тех компонентов, которые имеют значение 0 в столбце Level [таблицы Feature](feature-table.md).
