# <a name="isolatecomponents-action"></a>Действие Исолатекомпонентс

Действие Исолатекомпонентс устанавливает копию компонента (обычно это общая библиотека DLL) в частное расположение для использования конкретным приложением (обычно это exe-файл). Это позволяет изолировать приложение от других копий компонента, которые могут быть установлены в общую папку на компьютере. Дополнительные сведения см. в разделе [изолированные компоненты](isolated-components.md).

Действие ссылается на каждую запись [таблицы исолатедкомпонент](isolatedcomponent-table.md) и связывает файлы компонента, перечисленные в \_ общем поле Component, с компонентом, указанным в \_ поле приложение компонента. Установщик устанавливает файлы компонента, \_ совместно используемые в том же каталоге, что и \_ приложение компонента. Установщик создает файл в этом каталоге, нулевое значение длины, наличие короткого имени файла ключа для \_ приложения компонента (как правило, это то же имя файла, что и exe-файл), добавленного с помощью. local. Действие Исолатедкомпонент не влияет на установку \_ приложения компонента. При удалении приложения компонента \_ также удаляются \_ Общие файлы компонента и локальный файл из каталога.

## <a name="sequence-restrictions"></a>Ограничения последовательности

Действие Исолатекомпонентс может использоваться только в [таблицах инсталлуисекуенце](installuisequence-table.md) и [инсталлексекутесекуенце](installexecutesequence-table.md). Это действие должно следовать после [действия костинитиализе](costinitialize-action.md) и перед [действием костфинализе](costfinalize-action.md).

## <a name="actiondata-messages"></a>Сообщения Актиондата

Нет сообщений Актиондата.

## <a name="remarks"></a>Комментарии

Если столбец условия для действия Исолатекомпонентс равен true или оставлен пустым, установщик изолирует все компоненты, перечисленные в [таблице исолатедкомпонент](isolatedcomponent-table.md). Если столбец Condition имеет значение false, установщик игнорирует таблицу Исолатедкомпонент и использует обычные компоненты. Для условия этого действия можно использовать свойство [**редиректеддллсуппорт**](redirecteddllsupport.md) . Дополнительные сведения см. в разделе [Использование таблицы последовательностей](using-a-sequence-table.md).
