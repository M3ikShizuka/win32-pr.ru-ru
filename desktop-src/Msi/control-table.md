# <a name="control-table"></a>Таблица управления

Управляющая таблица определяет элементы управления, которые отображаются в каждом диалоговом окне.

Таблица элементов управления содержит следующие столбцы.



| Столбец        | Type                               | Ключ | Допускает значения NULL |
|---------------|------------------------------------|-----|----------|
| Диалог\_      | [Идентификатор](identifier.md)       | Д   | Нет        |
| Элемент       | [Идентификатор](identifier.md)       | Д   | Нет        |
| Тип          | [Идентификатор](identifier.md)       | Нет   | Нет        |
| X             | [Integer](integer.md)             | Нет   | Нет        |
| Д             | [Integer](integer.md)             | Нет   | Нет        |
| Ширина         | [Integer](integer.md)             | Нет   | Нет        |
| Высота        | [Integer](integer.md)             | Нет   | Нет        |
| Атрибуты    | [даублеинтежер](doubleinteger.md) | Нет   | Д        |
| Свойство      | [Идентификатор](identifier.md)       | Нет   | Д        |
| Текст          | [Формате](formatted.md)         | Нет   | Д        |
| Управление \_ следующим | [Идентификатор](identifier.md)       | Нет   | Д        |
| Справка          | [Text](text.md)                   | Нет   | Д        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="Dialog_"></span><span id="dialog_"></span><span id="DIALOG_"></span>Откроется\_
</dt> <dd>

Внешний ключ к первому столбцу [диалоговой таблицы](dialog-table.md), имя диалогового окна.

</dd> <dt>

<span id="Control"></span><span id="control"></span><span id="CONTROL"></span>Элемента
</dt> <dd>

Имя элемента управления. Это имя должно быть уникальным в пределах диалогового окна, но может повторяться в разных диалоговых окнах. Столбец Control, объединенный со столбцом диалогового окна \_ , формирует первичный ключ для этой таблицы.

</dd> <dt>

<span id="Type"></span><span id="type"></span><span id="TYPE"></span>Тип
</dt> <dd>

Тип элемента управления. Список типов элементов управления см. в разделе [элементы управления](controls.md).

</dd> <dt>

<span id="X"></span><span id="x"></span>X
</dt> <dd>

Горизонтальная координата верхнего левого угла прямоугольной границы элемента управления. Это значение должно быть неотрицательным числом. См. раздел [атрибут управления положением](position-control-attribute.md).

</dd> <dt>

<span id="Y"></span><span id="y"></span>&
</dt> <dd>

Вертикальная координата верхнего левого угла прямоугольной границы элемента управления. Это значение должно быть неотрицательным числом. См. раздел [атрибут управления положением](position-control-attribute.md).

</dd> <dt>

<span id="Width"></span><span id="width"></span><span id="WIDTH"></span>Ширина
</dt> <dd>

Ширина прямоугольной границы элемента управления. Это значение должно быть неотрицательным числом. См. раздел [атрибут управления положением](position-control-attribute.md).

</dd> <dt>

<span id="Height"></span><span id="height"></span><span id="HEIGHT"></span>Равно
</dt> <dd>

Высота прямоугольной границы элемента управления. Это значение должно быть неотрицательным числом. См. раздел [атрибут управления положением](position-control-attribute.md).

</dd> <dt>

<span id="Attributes"></span><span id="attributes"></span><span id="ATTRIBUTES"></span>Атрибута
</dt> <dd>

32-разрядное слово, задающее битовые флаги, применяемые к этому элементу управления. Это должно быть неотрицательное число, а допустимые значения зависят от типа элемента управления. Список всех атрибутов элемента управления и значение, которое необходимо ввести в этом поле, см. в разделе [Control Attributes](control-attributes.md).

</dd> <dt>

<span id="Property"></span><span id="property"></span><span id="PROPERTY"></span>Свойства
</dt> <dd>

Имя определенного свойства, связываемого с этим элементом управления. Переключатели, поля со списком и поля со списками привязываются к группе, так как они связаны с одним и тем же свойством. Этот столбец необходим для активных элементов управления.

</dd> <dt>

<span id="Text"></span><span id="text"></span><span id="TEXT"></span>Полнотекстовым
</dt> <dd>

Локализуемое строка, используемая для задания начального текста, содержащегося в элементе управления. Строка может также содержать внедренные свойства. Синтаксис форматированной строки, содержащей свойства, см. в описании функции [**мсиформатрекорд**](/windows/desktop/api/Msiquery/nf-msiquery-msiformatrecorda) . Укажите размер, шрифт и цвет текста, добавив префикс в текстовую строку { \\ Style}, где Style — это стиль текста, который был создан в столбце система создала шрифт [таблицы система создала шрифт](textstyle-table.md). Текстовая строка усекается, если она слишком длинна и не умещается в элементе управления. Текстовая строка может быть пустой.

Специальная разработка [форматированной](formatted.md) текстовой строки в этом поле является обязательной, если текст должен отображаться [элементом управления "текст](text-control.md) ", расположенным в диалоговом окне с атрибутом траккдискпаце. Это случай, определяемый [битом стиля диалогового окна траккдискспаце](trackdiskspace-dialog-style-bit.md) , который появляется в атрибутах [диалоговой таблицы](dialog-table.md). В этом случае, если отформатированная строка в текстовом столбце таблицы управления начинается с "" \[ и заканчивается на "", \] необходимо добавить пробел в конце строки. Например, если Длгтекстфонт является свойством, для которого будет задано значение "{ \\ длгфонтболд}", в форматированной строке " \[ длгтекстфонт \] myText \[ ProductName \] " требуется пробел в конце после закрывающей скобки. Это дополнительное пространство требуется установщику для правильного вывода текста в элементе управления "текст".

Можно ввести краткую описательную текстовую строку для [элементов управления](selectiontree-control.md) [волумекостлист](volumecostlist-control.md), [ListView](listview-control.md), [директорилист](directorylist-control.md)и селектионтри. Этот текст не виден пользователю, но его могут прочитать средства чтения с экрана в качестве описания элемента управления.

См. также [Специальные возможности](accessibility.md).

</dd> <dt>

<span id="Control_Next"></span><span id="control_next"></span><span id="CONTROL_NEXT"></span>Управление \_ следующим
</dt> <dd>

Имя другого элемента управления в том же диалоговом окне и внешнем ключе во втором столбце таблицы управления. Если фокус в диалоговом окне находится на элементе управления в столбце Control, нажатие клавиши TAB перемещает фокус на элемент управления, указанный в \_ следующем столбце элемента управления. Поэтому этот столбец используется для указания последовательности табуляции элементов управления в диалоговом окне. Связи между элементами управления должны формировать замкнутый цикл. Некоторые элементы управления, такие как статические текстовые элементы управления, можно оставить в цикле. В этом случае это поле можно оставить пустым.

См. также [Специальные возможности](accessibility.md).

</dd> <dt>

<span id="Help"></span><span id="help"></span><span id="HELP"></span>Справка
</dt> <dd>

Необязательные, локализуемые текстовые строки, которые используются с кнопкой "Справка". Строка делится на две части символом-разделителем ( \| ). Первая часть строки используется в качестве текста подсказки. Этот текст используется программами чтения с экрана для элементов управления, содержащих изображение. Вторая часть строки зарезервирована для будущего использования. Символ разделителя обязателен, даже если имеется только один из двух видов текста.

</dd> </dl>

## <a name="remarks"></a>Remarks

Целочисленные значения x, y, ширины и высоты находятся в [единицах установщика](installer-units.md), а не в диалоговых единицах. Единица установщика равна одной двенадцатой высоте размера шрифта с размером в 10 пунктов MS Sans Serif. Координаты элементов управления задаются относительно объявления.
