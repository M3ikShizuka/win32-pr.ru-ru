# <a name="ice28"></a>ICE28

ICE28 обычно используется для проверки того, что действие Форцеребут размещается до или после, и никогда в рамках определенной группы действий в таблицах последовательности действий. Чтобы определить действия, составляющие эту группу, см. раздел [действие форцеребут](forcereboot-action.md) .

ICE28 проверяет действия в следующих таблицах последовательностей:

[Таблица Админуисекуенце](adminuisequence-table.md)

[Таблица Админексекутесекуенце](adminexecutesequence-table.md)

[Таблица Инсталлуисекуенце](installuisequence-table.md)

[Таблица Инсталлексекутесекуенце](installexecutesequence-table.md)

## <a name="result"></a>Результат

ICE28 отправляет сообщение об ошибке, если Форцеребут помещается в последовательность в указанной группе действий.

## <a name="example"></a>Пример

В приведенном примере ICE28 отправляет следующее сообщение об ошибке:

``` syntax
Action: 'ForceReboot' of table InstallExecuteSequence is not permitted in the range 5 to 15 because it cannot separate a set of actions contained in this range.
```

[Таблица Инсталлексекутесекуенце](installexecutesequence-table.md)



| Действие             | Последовательность |
|--------------------|----------|
| форцеребут        | 10       |
| регистермимеинфо   |   5      |
| регистерпрогидинфо | 15       |



 

Порядковый номер 10, присвоенный Форцеребут разрывам, приводит к возникновению ошибки, так как он находится между порядковыми номерами Регистермимеинфо и Регистерпрогидинфо.
