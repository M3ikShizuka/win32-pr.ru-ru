# <a name="side-by-side-assemblies"></a>Параллельные сборки

Параллельный доступ к сборкам — это инфраструктура, которая используется для следующих задач:

-   Безопасный обмен сборками между несколькими приложениями
-   Смещение некоторых негативных последствий общего доступа, например конфликтов DLL.

Вместо единственной версии сборки, которая предполагает обратную совместимость со всеми приложениями, совместное использование параллельных сборок позволяет нескольким версиям сборки COM или Win32 выполняться одновременно в системе. Дополнительные сведения см. в разделе [изолированные приложения и параллельные сборки](../sbscs/isolated-applications-and-side-by-side-assemblies-portal.md).

Параллельные сборки могут устанавливаться как [Общие сборки](shared-assemblies.md) или как [закрытые сборки](private-assemblies.md).

Параллельные сборки недоступны в системах, предшествующих Windows XP.
