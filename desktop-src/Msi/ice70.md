# <a name="ice70"></a>ICE70

ICE70 проверяет правильность указанных целочисленных значений для записей реестра. Значения в форме \# \# str, \# % unexpand str не проверяются. Значения формы \# ксхекс, \# ксхекс, \# Integer и \# \[ Property \] проверяются. В следующей таблице представлен краткий обзор.



| Значение                 | Проверка                                                                    |
|-----------------------|-------------------------------------------------------------------------------|
| \#\#str               | допустимым                                                                         |
| \#% нераскрытной str     | допустимым                                                                         |
| \#Ксхекс, \# ксхекс         | Проверьте Допустимые шестнадцатеричные символы (0 – 9, a-f, A-F). Здесь можно использовать свойства. |
| \#+ int, \# -int, \# int | Проверьте допустимые числовые символы (0-9). Здесь можно использовать свойства.     |



 

Синтаксис для целочисленного значения, который необходимо указать в реестре, — \# целое число, где целое число является числовым.

## <a name="result"></a>Результат

ICE70 сообщает об ошибке, если целочисленные значения для записей реестра указаны неправильно.

## <a name="example"></a>Пример

ICE70 сообщает о следующих ошибках в данном примере.

``` syntax
The value #12xz34 is an invalid numeric value for registry entry Reg1. If you meant to use a string, then the string value entry must be preceded by ## not #.
```

Чтобы устранить эту ошибку, измените значение, чтобы оно использовало все числовые символы. Если требуется, чтобы значение было строкой, перед ним должно стоять два " \# " ( \# \# ), а не только один.

``` syntax
The value #xz34 is an invalid hexadecimal value for registry entry Reg2.
```

Чтобы устранить эту ошибку, допустимые шестнадцатеричные символы: 0-9, A-F и A-f. Только эти символы могут следовать за \# x (или \# x).

[Таблица реестра](registry-table.md) (частичная)



| Реестр | Значение    |
|----------|----------|
| Reg1     | \#12xz34 |
| Reg2     | \#xz34   |



 

## <a name="remarks"></a>Remarks

-   \#\[\]недопустимое MyProperty.
-   \#\[Недопустимое MyProperty (отсутствует закрывающая скобка).
-   \#\[myprop1 \] \[ myprop2 является допустимым. (Несмотря на то, что в последнем случае отсутствует закрывающая квадратная скобка, myprop1 может вычислить значение \# str, поэтому будет использоваться \# \# str \[ myprop2, что является допустимым
-   \#\]\[недопустимое MyProperty
-   Любое внедренное свойство в строке значения не может иметь \[ \] форму $compkey, \[ \# филекэй \] или \[ ! филекэй, \] так как они не являются числовыми. Однако существует одно исключение: \# \[ MyProperty \] \[ $compkey \] (или \[ \# филекэй \] или \[ ! филекэй \] ) является допустимым, так как, как и в предыдущем, \[ MyProperty \] может вычислять \# str.
