# <a name="private-properties"></a>Закрытые свойства

Закрытые свойства используются для внутренних целей установщика, и их значения должны вводиться автором пакета установки или устанавливаться установщиком во время установки на значения, определенные операционной средой. Единственный способ взаимодействия пользователя с закрытыми свойствами — [Управление событиями](control-events.md) в созданном пользователем интерфейсе пакета. Имена закрытых свойств должны включать строчные буквы. См. раздел [ограничения имен свойств](restrictions-on-property-names.md).

Закрытые свойства обычно описывают операционную среду. например, если установка выполняется на Windowsной платформе, установщик устанавливает для свойства [**виндовсфолдер**](windowsfolder.md) значение, указанное в таблице свойств.

Значения закрытых свойств не могут быть переопределены в командной строке. Чтобы удалить закрытое свойство из установки, оставьте его из [таблицы свойств](property-table.md). в Windows XP и Windows 2000 нельзя задать свойство private на этапе пользовательского интерфейса установки, а затем передать значение на фазу выполнения.

Список всех стандартных закрытых свойств, используемых установщиком, см. в разделе [Справочник по свойствам](property-reference.md). Можно определить пользовательское закрытое свойство, введя имя свойства и начальное значение в таблице свойств. Имена закрытых свойств должны всегда содержать строчные буквы.
