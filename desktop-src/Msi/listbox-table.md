# <a name="listbox-table"></a>Таблица ListBox

Строки списка не рассматриваются как отдельные элементы управления, но они являются частью списка, который выступает в качестве элемента управления. Таблица ListBox определяет значения для всех списков.

Таблица ListBox содержит следующие столбцы.



| Столбец   | Type                         | Ключ | Допускает значения NULL |
|----------|------------------------------|-----|----------|
| Свойство | [Идентификатор](identifier.md) | Да   | Нет        |
| Заказ    | [Integer](integer.md)       | Да   | Нет        |
| Значение    | [Формате](formatted.md)   | Нет   | Нет        |
| Текст     | [Формате](formatted.md)   | Нет   | Да        |



 

## <a name="columns"></a>Столбцы

<dl> <dt>

<span id="Property"></span><span id="property"></span><span id="PROPERTY"></span>Свойства
</dt> <dd>

Именованное свойство, которое должно быть привязано к этому элементу. Все элементы, привязанные к одному и тому же свойству, становятся частью одного списка.

</dd> <dt>

<span id="Order"></span><span id="order"></span><span id="ORDER"></span>Порядок
</dt> <dd>

Положительное целое число, используемое для определения порядка элементов, отображаемых в одном окне списка. Если список определен как упорядоченный, то все элементы должны иметь значение заказа. Целые числа не должны быть последовательными. Если список определен как неупорядоченный, то этот столбец игнорируется.

</dd> <dt>

<span id="Value"></span><span id="value"></span><span id="VALUE"></span>Значений
</dt> <dd>

Строка значения, связанная с этим элементом. Если выбрать строку, связанному свойству присваивается это значение.

</dd> <dt>

<span id="Text"></span><span id="text"></span><span id="TEXT"></span>Полнотекстовым
</dt> <dd>

Локализуемый, видимый текст, который будет назначен элементу. Если эта запись или весь столбец отсутствуют, по умолчанию используется соответствующая запись в поле значение.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Содержимое полей Value и Text форматируется функцией [**мсиформатрекорд**](/windows/desktop/api/Msiquery/nf-msiquery-msiformatrecorda) при создании элемента управления, поэтому они могут содержать любое выражение, которое может интерпретировать функция **мсиформатрекорд** . Форматирование происходит только при создании элемента управления и не обновляется, если свойство, вовлеченное в выражение, изменяется в течение жизненного цикла элемента управления.
