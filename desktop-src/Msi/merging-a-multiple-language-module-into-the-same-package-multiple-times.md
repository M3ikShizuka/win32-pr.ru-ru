# <a name="merging-a-multiple-language-module-into-the-same-package-multiple-times"></a>Несколько раз объединить многоязыковый модуль в один пакет

если модуль поддерживает несколько языков, его можно объединить в одну базу данных установщик Windows несколько раз, но убедитесь, что в каждом слиянии используется другой язык. Перед каждым слиянием запросите другой язык из модуля. Полученная .msi база данных содержит запись в [таблице ModuleSignature](modulesignature-table.md) для каждого объединения модуля. Компоненты, которые являются общими для разных языков, существуют только один раз в [таблице Component](component-table.md), но связаны с каждым языком в [таблице модулекомпонентс](modulecomponents-table.md).

При объединении нескольких языков модуля в один пакет каждое слияние должно соответствовать тем же ограничениям кодовых страниц, что и модули с одним языком. Модули не могут содержать строки в разных кодовых страницах.

При многократном объединении модуля в один файл .msi может потребоваться изменить порядок файлов в [таблице File](file-table.md) , чтобы использовать существующие .cab из модуля непосредственно в установке. Порядок файлов в таблице File должен соответствовать порядку файлов в .cab. При многократном объединении модуля в базу данных установки последовательность может быть изменена, поскольку файлы, общие для разных языков, уже существуют в модуле из предыдущей операции слияния и имеют другой относительный порядковый номер.
