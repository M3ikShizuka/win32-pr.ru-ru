# <a name="mergemerge-method"></a>Merge. Merge, метод

Метод **Merge** объекта [**Merge**](merge-object.md) выполняет слияние текущей базы данных и текущего модуля. Слияние присоединяет компоненты модуля к функции, определяемой *функцией*. Корень дерева каталогов модуля перенаправляется в расположение, заданное параметром *редиректдир*.

Метод **Merge** может вызываться только один раз для слияния определенного сочетания .msi и MSM-файлов.

## <a name="syntax"></a>Синтаксис


```JScript
Merge.Merge(
  Feature,
  RedirectDir
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Компонент* 
</dt> <dd>

Имя компонента в базе данных.

</dd> <dt>

*редиректдир* 
</dt> <dd>

Ключ записи в [таблице Directory](directory-table.md) базы данных. Этот параметр может иметь значение null или быть пустой строкой.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Remarks

После завершения слияния компоненты модуля присоединяются к функции, определяемой *компонентом*. Эта функция не создана и должна быть существующей функцией. Обратите внимание, что метод **Merge** получает все ссылки на функции в модуле и замещает ссылку на компонент для всех вхождений идентификатора GUID NULL в базе данных модуля. Дополнительные сведения см. [в разделе ссылки на функции в модулях слияния](referencing-features-in-merge-modules.md).

модуль можно подключить к дополнительным функциям с помощью метода [**Подключение**](merge-connect.md) . обратите внимание, что вызов метода **Подключение** создает только связи компонентов компонента. Строки, которые уже были объединены в базу данных, не изменяются.

Изменения, вносимые в базу данных, сохраняются только в том случае, если метод [**клоседатабасе**](/windows/win32/api/mergemod/nf-mergemod-imsmmerge-closedatabase) вызывается с параметром *бкоммит* , для которого задано **значение true**.

Если возникают конфликты слияния, включая исключения, они помещаются в перечислитель ошибок для последующего извлечения, но не приводят к сбою слияния. Ошибки могут быть получены с помощью свойства [**Errors**](error-object.md) . Ошибки и информационные сообщения отправляются в текущий файл журнала.
