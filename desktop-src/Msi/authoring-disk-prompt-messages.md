# <a name="authoring-disk-prompt-messages"></a>Создание сообщений с приглашением на диск

следуйте приведенным ниже рекомендациям, чтобы создать установщик Windowsную установку, в которой отображается окно сообщения, предлагающее пользователю вставить диск.

**Отображение окна сообщения, предлагающего пользователю вставить диск**

1.  Используйте возможности установщика, чтобы задать строку свойства [**DiskPrompt**](diskprompt.md) в таблице [свойств](property-table.md) . Оно должно включать имя устанавливаемого продукта и параметр заполнителя в строке для заголовка, напечатанного на диске. например, для Microsoft Publisher свойство **DiskPrompt** может иметь значение «Microsoft Publisher: диск \[ 1 \] », где \[ 1 \] — это заполнитель для названия диска.
2.  Введите заголовки каждого из дисков, которые будут запрашиваться, в отдельные строки столбца DiskPrompt таблицы [Media](media-table.md) . Например, первая и только запись в таблицу Media может иметь значение «1 – install».
3.  Во время действия [инсталлфилес](installfiles-action.md) значение из столбца DiskPrompt таблицы [Media](media-table.md) заменяется на заполнитель в строке свойства [**DiskPrompt**](diskprompt.md) .
4.  Сообщение, отображаемое окном сообщения, создается из встроенной строки шаблона в [таблице ошибок](error-table.md). Это ошибка 1302, и строка шаблона: "Вставьте диск: \[ 2 \] ", а \[ 2 \] — заполнитель для свойства [**DiskPrompt**](diskprompt.md) .

в примере отображается следующее сообщение для пользователя: "вставьте диск: Microsoft Publisher: диск 1 — установка".

Обратите внимание, что сообщения с приглашением на диск отображаются всеми [уровнями пользовательского интерфейса](user-interface-levels.md) , кроме None.
