# <a name="package-validation"></a>Проверка пакета

Настоятельно рекомендуется выполнять проверку для каждого нового или только что измененного пакета установки, прежде чем пытаться установить пакет в первый раз.

Проверка пакета включает в себя следующие три процесса:

-   [Внутренняя проверка](internal-validation.md)
-   [Проверка пула строк](string-pool-validation.md)
-   [Внутренние средства оценки согласованности — ICEs](internal-consistency-evaluators-ices.md)

Модули слияния следует проверять с помощью метода, описанного в разделе [Проверка модулей слияния](validating-merge-modules.md).

Evalcom2.dll предоставляет COM-объект, который реализует операции проверки для пакетов установки и модулей слияния. Основной объект реализует интерфейсы для программ C/C++. Дополнительные сведения см. в разделе [Автоматизация проверки](validation-automation.md).
