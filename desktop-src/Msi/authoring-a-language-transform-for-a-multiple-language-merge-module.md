# <a name="authoring-a-language-transform-for-a-multiple-language-merge-module"></a>Создание языкового преобразования для модуля многоязычного слияния

При слиянии модуля с базой данных, имеющей другой язык по умолчанию, средству слияния может потребоваться преобразовать язык в модуль, чтобы предоставить окончательный язык. Дополнительные сведения см. в разделе [модули слияния с несколькими языками](multiple-language-merge-modules.md).

Преобразования языка хранятся в файле MSM модуля и должны иметь имя и формат: установочного модуля. lang \# \# \# \# . Объект \# \# \# \# представляет значение [LangID](localizing-the-error-and-actiontext-tables.md) от до 4 цифр для последнего языка. Например, установочного модуля. Lang1033, установочного модуля. Lang9 и установочного модуля. Lang0 для преобразования в Английский, английский и нейтральный языки. Они совпадают с [внедренными преобразованиями](embedded-transforms.md) и их можно добавить во вложенные хранилища в файле MSM.

Преобразование языка должно выполнять следующие действия:

-   Измените язык по умолчанию в столбце язык [таблицы ModuleSignature](modulesignature-table.md) на новый язык модуля.
-   Измените язык по умолчанию в столбце Language [таблицы модулекомпонентс](modulecomponents-table.md) на новый язык модуля. Преобразование может добавлять или удалять строки из этой таблицы.
-   При необходимости измените язык в столбце Рекуиредлангуаже или добавьте или удалите строки в [таблицу модуледепенденци](moduledependency-table.md).
-   При необходимости измените язык в столбце Ексклудедлангуаже или добавьте или удалите строки в [таблицу модуликсклусион](moduleexclusion-table.md).
-   Преобразование может выполнять любые допустимые операции преобразования в модуль, включая добавление или удаление компонентов, файлов, записей реестра или действий.

Обратите внимание, что применение преобразования языка при открытии модуля не приводит к изменению языка по умолчанию или языков, поддерживаемых модулем, он просто изменяет запрашиваемый язык. Поэтому свойство [**сводки шаблона**](template-summary.md) не изменяется, оно должно уже перечислять все языки, поддерживаемые модулем, с языком по умолчанию, указанным в списке первым.

Все файлы, необходимые для всех возможных преобразований языка, обычно хранятся в одном CAB-файле, входящем в состав модуля. Поскольку языковой преобразованию не рекомендуется изменять этот CAB-файл, лучше использовать глобальную последовательность файлов в файле CAB, [таблице файлов](file-table.md)и преобразовании языка. Дополнительные сведения см. в разделе [упорядочение последовательности файлов в CAB-файле модуля слияния с несколькими языками](ordering-the-file-sequence-in-the-cab-of-a-multiple-language-merge-module.md) .
