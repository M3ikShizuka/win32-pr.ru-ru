# <a name="about-event-logging"></a>Сведения о ведении журнала событий

При возникновении ошибки системный администратор или специалист службы поддержки должен определить причину ошибки, попытаться восстановить потерянные данные и предотвратить повторную попытку. Это полезно, если приложения, операционная система и другие системные службы записывают важные события, такие как условия нехватки памяти или чрезмерные попытки доступа к диску. Затем системный администратор может использовать журнал событий, чтобы определить, какие условия привели к возникновению ошибки, и определить контекст, в котором она возникла. Периодически просмотрев журнал событий, системный администратор может обнаруживать проблемы (например, сбой жесткого диска), прежде чем они могут привести к повреждению.

> [!Note]  
> API ведения журнала событий предназначен для приложений, работающих в операционной системе Windows Server 2003, Windows XP или Windows 2000. в Windows Vista инфраструктура ведения журнала событий была изменена. приложения, предназначенные для работы в Windows Vista или более поздних операционных системах, теперь должны использовать [Windows журнал событий](/windows/desktop/WES/windows-event-log) для регистрации событий.
