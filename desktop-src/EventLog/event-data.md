# <a name="event-data"></a>Данные о событиях

Для каждого события могут сопоставлены связанные с событиями данные. Просмотр событий не интерпретирует эти данные; Дополнительные данные отображаются только в комбинированном шестнадцатеричном и текстовом формате. Максимальное ограничение размера события в четном журнале — 0x3FFFF байт. Таким образом, данные событий следует использовать экономно, в том числе только в том случае, если вы уверены, что он будет полезен для отладки проблемы. Например, многие события, связанные с сетью, включают в себя блоки управления сетью (NCB) как данные, относящиеся к конкретному событию.

При использовании данных, относящихся к конкретному событию, в последней части строки описания должно быть указано Примечание о сведениях, предоставляемых в виде данных, относящихся к конкретному событию. Например, сетевое программное обеспечение может предоставить следующее примечание: "(NCB — это данные события)". В качестве соглашения используйте круглые скобки вокруг таких примечаний, как показано в этом примере.

Можно также использовать данные событий для хранения информации, которую приложение может обрабатывать независимо от Просмотр событий. Например, можно написать средство просмотра, предназначенное для событий, или написать программу, которая просматривает журнал и создает отчет, который включает сведения из данных, относящихся к конкретному событию.
