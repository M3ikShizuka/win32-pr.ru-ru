# <a name="sdk-samples-that-use-the-core-audio-apis"></a>Примеры пакетов SDK, использующих основные API аудио

Windows SDK содержит следующие примеры кода, демонстрирующие использование основных api аудио. следующие примеры находятся в каталоге% мссдк% \\ samples \\ мультимедиа \\ audio, где% мссдк% — это корневой каталог установки Windows SDK на компьютере.




| Образец | деаскриптион | 
|--------|--------------|
| <a href="aecmicarray.md">аекмикаррай</a> | В этом примере используются интерфейсы API Ммдевице, ВАСАПИ, Девицетопологи и Ендпоинтволуме для записи высококачественного потока голоса. пример tне поддерживает отмену акустического эха (aec) и обработку массива микрофона с помощью контекста AEC DMO также называемого DSP-модулем записи голоса, предоставляемого корпорацией майкрософт. | 
| <a href="capturesharedeventdriven.md">каптурешаредевентдривен</a> | Этот пример приложения использует основные API аудио для записи звуковых данных с входного устройства, указанного пользователем, и записывает их в уникальное имя. Файл WAV в текущем каталоге. В этом примере демонстрируется буферизация, управляемая событиями. | 
| <a href="capturesharedtimerdriven.md">каптурешаредтимердривен</a> | Этот пример приложения использует основные API аудио для записи звуковых данных с входного устройства, указанного пользователем, и записывает их в уникальное имя. Файл WAV в текущем каталоге. В этом примере демонстрируется буферизация, управляемая таймером. | 
| <a href="duckingcapturesample.md">дуккингкаптуресампле</a> | Этот пример приложения демонстрирует открытие и закрытие потоков связи и вызов событий дуккинг, которые приложение может получить для реализации затухания потока. Это приложение реализует чат-клиент, который использует основные API-интерфейсы для чтения звуковых данных с коммуникационного устройства и воспроизведения на выходном устройстве. | 
| <a href="endpointvolume.md">ендпоинтволуме</a> | Этот пример приложения использует основные API аудио для изменения объема устройства, указанного пользователем. | 
| <a href="osd.md">ОПЕРАЦИОННОЙ</a> | В этом примере используются API Ммдевице и Ендпоинтволуме для реализации экранного экрана, в котором показаны изменения тома в потоке вывода, который воспроизводится через устройство конечной точки рендеринга звука по умолчанию. экран отображается, когда пользователь корректирует уровень громкости в Windows программе управления томами, Sndvol.exe и исчезает после того, как уровень тома не изменится в течение короткого периода. | 
| <a href="renderexclusiveeventdriven.md">рендерексклусививентдривен</a> | Этот пример приложения использует основные API аудио для отображения звуковых данных на устройстве вывода, указанном пользователем. В этом примере демонстрируется буферизация, управляемая событиями, для клиента отрисовки в монопольном режиме. Для потока с монопольным режимом клиент использует буфер конечной точки с звуковым устройством. | 
| <a href="renderexclusivetimerdriven.md">рендерексклусиветимердривен</a> | Этот пример приложения использует основные API аудио для отображения звуковых данных на устройстве вывода, указанном пользователем. В этом примере демонстрируется буферизация, управляемая таймером, для клиента отрисовки в монопольном режиме. Для потока с монопольным режимом клиент использует буфер конечной точки с звуковым устройством. | 
| <a href="rendersharedeventdriven.md">рендершаредевентдривен</a> | Этот пример приложения использует основные API аудио для отображения звуковых данных на устройстве вывода, указанном пользователем. В этом примере демонстрируется буферизация, управляемая событиями, для клиента отрисовки в общем режиме. Для потока общего режима клиент совместно использует буфер конечной точки с обработчиком аудио. | 
| <a href="rendersharedtimerdriven.md">рендершаредтимердривен</a> | Этот пример приложения использует основные API аудио для отображения звуковых данных на устройстве вывода, указанном пользователем. В этом примере демонстрируется буферизация, управляемая таймером, для клиента отрисовки в общем режиме. Для потока общего режима клиент совместно использует буфер конечной точки с обработчиком аудио. | 
| винаудио | В этом примере используется API Ммдевице и ВАСАПИ для воспроизведения и записи звуковых потоков. Пользовательский интерфейс этого примера приложения позволяет пользователям выбирать устройства конечных точек аудио, изменять уровень громкости локального сеанса аудио, а также воспроизводить файлы WAV и микрофон.<blockquote>[!Note]<br />этот пример является устаревшим в Windows 7.</blockquote><br /> |
