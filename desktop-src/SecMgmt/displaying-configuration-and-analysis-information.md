# <a name="displaying-configuration-and-analysis-information"></a>Отображение сведений о конфигурации и анализе

Расширение оснастки должно отображать текущие сведения о конфигурации и анализе для пользователей.

Чтобы отобразить сведения, узел расширения оснастки "вложение" должен расширить оснастку "Настройка безопасности" с помощью узла "службы". Узел службы — это оснастка консоли управления (MMC), которая управляет службами, установленными в системе.

Ниже перечислены типы узлов, которые могут быть расширены.

-   Службы конфигурации NodeType = {24a7f717-1f0c-11D1-affb-00C04fB984F9}
-   Службы проверки NodeType = {678050c7-1ff8-11D1-affb-00C04fB984F9}

Когда узел службы развернут, MMC уведомляет все зарегистрированные расширения оснастки. Каждая оснастка вложений должна быть вставлена в узел "службы" и выполнять следующие задачи:

-   Вызовите **QueryInterface** , чтобы получить указатель на интерфейс [**исцесвкаттачментдата**](/windows/desktop/api/Scesvc/nn-scesvc-iscesvcattachmentdata) , предоставляемый оснасткой настройки безопасности.
-   Вызовите [**исцесвкаттачментдата:: Initialize**](/windows/desktop/api/Scesvc/nf-scesvc-iscesvcattachmentdata-initialize) , чтобы сообщить оснасткам настройки безопасности о загрузке расширения оснастки и установить контекст для взаимодействия.
-   Вызовите метод [**исцесвкаттачментдата:: GetData**](/windows/desktop/api/Scesvc/nf-scesvc-iscesvcattachmentdata-getdata) , чтобы получить сведения о конфигурации из базы данных. Этот шаг можно выполнить либо при инициализации расширения оснастки, либо при открытии пользователем своего узла.

Дополнительные сведения см. [в разделе Добавление узла расширения оснастки «вложение](adding-an-attachment-snap-in-extension-node.md)».
