# <a name="limitations-of-mutual-authentication-with-kerberos"></a>Ограничения взаимной проверки подлинности с помощью Kerberos

учетная запись клиента и учетная запись службы должны находиться в доменах Windows 2000 native или смешанном режиме, так как службы Kerberos недоступны в доменах прежних версий. Кроме того, учетные записи клиента и службы должны находиться в одном лесу, поскольку KDC клиента использует глобальный каталог для поиска имени субъекта-службы.

служба и клиент должны работать на Windows 2000, в противном случае взаимная проверка подлинности с помощью Kerberos завершится ошибкой, так как более ранние версии Windows не поддерживают kerberos.

Имена участников-служб должны включать имя DNS сервера узла, на котором запущена служба. Необходимо использовать DNS-имя.
