# <a name="limitations-of-mutual-authentication-with-kerberos"></a>Ограничения взаимной проверки подлинности с помощью Kerberos

Учетная запись клиента и учетная запись службы должны находиться в доменах машинного кода Windows 2000 или в смешанном режиме, так как службы Kerberos недоступны в доменах прежних версий. Кроме того, учетные записи клиента и службы должны находиться в одном лесу, поскольку KDC клиента использует глобальный каталог для поиска имени субъекта-службы.

Служба и клиент должны работать под управлением Windows 2000, в противном случае взаимная проверка подлинности с помощью Kerberos завершится ошибкой, так как более ранние версии Windows не поддерживают Kerberos.

Имена участников-служб должны включать имя DNS сервера узла, на котором запущена служба. Необходимо использовать DNS-имя.
