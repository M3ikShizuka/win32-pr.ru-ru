# <a name="domain-trees"></a>Деревья доменов

*Доменное дерево* состоит из нескольких доменов, которые совместно используют общую схему и конфигурацию, образуя непрерывное пространство имен. Домены в дереве также связываются вместе отношениями доверия. Active Directory — это набор из одного или нескольких деревьев.

Деревья можно просмотреть двумя способами. Одно из представлений — отношения доверия между доменами. Другое представление — это пространство имен дерева доменов.

## <a name="viewing-trust-relationships"></a>Просмотр отношений доверия

Вы можете нарисовать схему дерева доменов на основе отдельных доменов и существующих отношений доверия.

Windows 2000 устанавливает отношения доверия между доменами на основе протокола безопасности Kerberos. Доверие Kerberos является транзитивным и иерархическим, если домен A доверяет домену B, а домен б доверяет домену C, а домен A доверяет домену C.

![доверительное отношение дерева доменов](images/domain-trust.png)

## <a name="viewing-the-namespace"></a>Просмотр пространства имен

Можно также нарисовать схему дерева доменов на основе пространства имен. Различающееся имя объекта можно определить, следуя пути вверх по иерархии пространства имен дерева доменов. Это представление полезно для группирования объектов в логическую иерархию. Главным преимуществом непрерывного пространства имен является то, что глубокий поиск из корня пространства имен выполняет поиск всей иерархии.

![дерево доменов пространства имен](images/namespace.png)
