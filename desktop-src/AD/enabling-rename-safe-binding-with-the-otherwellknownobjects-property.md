# <a name="enabling-rename-safe-binding-with-the-otherwellknownobjects-property"></a>Включение привязки Rename-Safe с помощью свойства otherWellKnownObjects

Объекты класса Container имеют атрибут **otherWellKnownObjects** , с помощью которого можно связать идентификатор GUID с различающимся именем (DN) дочернего объекта в контейнере. Если дочерний объект перемещается или переименовывается, сервер Active Directory обновляет DN в значении **otherWellKnownObjects** для этого дочернего объекта. Это позволяет использовать функцию привязки ВКГУИД для привязки к дочернему объекту с помощью идентификатора GUID и DN контейнера, а не различающегося имени дочернего объекта.

Атрибут **otherWellKnownObjects** эквивалентен атрибуту **веллкновнобжектс** , за исключением того, что приложения и службы могут записывать значение **otherWellKnownObjects** , но только система может записывать **веллкновнобжектс**.

Использование атрибута **otherWellKnownObjects** и привязки вкгуид полезно в следующих ситуациях, где требуется переименование привязок в отношении определенного объекта контейнера.

Если объект контейнера содержит другие важные объекты или если важные объекты могут быть переименованы или перемещены.

Значение, если важные объекты существуют для каждого экземпляра объекта контейнера. Например, система использует атрибут **веллкновнобжектс** каждого объекта **домаинднс** для хранения значения для контейнера Users, который существует в каждом экземпляре объекта **домаинднс** . Это позволяет приложениям выполнять привязку к контейнеру пользователей с помощью безопасного переименования путем указания хорошо известного идентификатора GUID и DN контейнера **домаинднс** . Приложения могут использовать атрибут **otherWellKnownObjects** контейнера аналогичным образом.

Если для важных объектов требуется возможность переименования и (или) поиска.

**Добавление безопасного переименования и возможностей поиска**

1.  Добавьте значение в свойство **otherWellKnownObjects** объекта Container при создании важного объекта в этом контейнере. Значение содержит идентификатор GUID, представляющий хорошо известный объект. Имейте в виду, что это не **атрибут objectGUID** и **distinguishedName** для этого объекта.
2.  Используйте функцию привязки ВКГУИД для привязки к важному объекту или поиска в нем.

Атрибут **otherWellKnownObjects** может иметь несколько значений и содержать кортежи GUID/DN известных объектов в контейнерах, в которых они заданы. Атрибут **otherWellKnownObjects** имеет синтаксис днвисбинари, в котором значения имеют следующую форму:


```C++
B:<char count>:<well known GUID>:<object DN>
```



В этом примере " &lt; число символов &gt; " — это число шестнадцатеричных цифр в " &lt; известном идентификаторе GUID &gt; ", которое равно 32 (число шестнадцатеричных цифр в GUID) для **otherWellKnownObjects** и **веллкновнобжектс**. " &lt; хорошо известный идентификатор GUID &gt; " — шестнадцатеричное цифровое представление ИЗВЕСТНОГО идентификатора GUID. " &lt; DN объекта &gt; " — это различающееся имя объекта, представленного этим значением ВКО. Сервер поддерживает ObjectDN часть каждого значения **веллкновнобжектс** и **otherWellKnownObjects** , чтобы она содержала текущее различающееся имя объекта, первоначально указанного при создании значения.

Например, если {df447b5e-aa5b-11d2-8d53-00c04f79ab81} является известным идентификатором GUID объекта MyObject в контейнере MyContainer в домене Fabrikam.com, значение **otherWellKnownObjects** будет указывать известный GUID и различающееся имя для MyObject:


```C++
B:32:df447b5eaa5b11d28d5300c04f79ab81:cn=MyObject,cn=MyContainer,dc=Fabrikam,dc=com
```



Чтобы выполнить привязку к этому объекту, используйте следующую строку привязки ВКГУИД, которая указывает известный идентификатор GUID объекта и DN контейнера:


```C++
LDAP://<WKGUID=df447b5eaa5b11d28d5300c04f79ab81,cn=MyContainer,dc=Fabrikam,dc=com>
```



После привязки к этому объекту можно использовать интерфейсы COM ADSI для поиска, чтения, изменения или удаления объекта.

Дополнительные сведения и пример кода, демонстрирующий Добавление объекта в атрибут **otherWellKnownObjects** , см. в разделе [пример кода для создания объекта контейнера](example-code-for-creating-a-container-object.md).
