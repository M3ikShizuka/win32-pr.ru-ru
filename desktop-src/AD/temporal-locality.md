# <a name="temporal-locality"></a>Временная локализация

*Временная локализация* — это стратегия предотвращения, которая сокращает окно для частичных обновлений. Репликация изменений из заданной исходной реплики продолжается в порядке возрастания USN. Операции записи, которые происходят близко друг к другу, будут иметь USN, которые находятся рядом друг с другом и будут распространяться близко друг к другу вовремя. Приложения, которые создают или обновляют несколько связанных объектов, должны заставлять объекты как можно ближе друг к другу во времени. Например, приложение может собрать все необходимые данные от пользователя и применить его к каталогу, когда пользователь нажмет кнопку "Применить".

Временная локализация также сокращает вероятность конфликта, что приводит к несогласованности внутри объекта.
