# <a name="user-interface-extension-for-new-object-classes"></a>Расширение пользовательского интерфейса для новых классов объектов

Домен Active Directory Services и его административный интерфейс оснастки MMC для администрирования можно настроить для адаптации к требованиям администраторов и пользователей. Домен Active Directory Services позволяют изменять схему путем создания новых классов и атрибутов или изменения существующих классов. Описатели отображения для классов можно изменить в соответствии с новыми элементами пользовательского интерфейса, которые требуются для изменения схемы.

В следующей таблице перечислены атрибуты, которые можно использовать для изменения того, как административные оснастки будут отображать объекты определенного класса.



| attribute                  | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|----------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **дефаулсидингвалуе**     | Атрибут **дефаулсидингвалуе** является атрибутом объекта **classSchema** . Этот атрибут содержит логическое значение, которое, Если TRUE, приводит к скрытию экземпляров класса объекта в административных оснастках и оболочке Windows. Это также означает, что пункт меню для нового класса объектов не отображается в контекстном меню **оснастки «Администрирование** », даже если для объекта **дисплайспеЦифиер** нового класса объекта установлены соответствующие свойства мастера создания. Если этот атрибут имеет значение FALSE, экземпляры класса будут видны в административных оснастках и оболочке. Это также приводит к тому, что пункт меню создает новый экземпляр объекта, который добавляется в меню « **создать** » административных оснасток.<br/> Если для этого атрибута не задано значение, по умолчанию используется значение TRUE. Это означает, что экземпляры объекта по умолчанию скрыты.<br/>                                                                |
| **showInAdvancedViewOnly** | Атрибут **шовинадванцедвиевонли** содержит логическое значение, которое, если true, вызывает отображение экземпляров класса объектов в оснастке "пользователи и компьютеры" только в расширенном представлении и не отображается в оболочке Windows. Если это свойство имеет значение FALSE, экземпляры класса будут отображаться в нормальном режиме в оснастке "пользователи и компьютеры" и в оболочке Windows.<br/> Если для этого атрибута не задано значение, по умолчанию используется значение TRUE.<br/> Этот атрибут можно задать для отдельного объекта, чтобы переопределить значение, заданное для класса объекта. Например, класс **контейнера** имеет для этого атрибута значение true, а в контейнере **пользователя** — false. По этой причине контейнер **пользователя** отображается в оболочке и в нормальном режиме в оснастке "пользователи и компьютеры", но другие контейнеры, для которых не задан параметр **шовинадванцедвиевонли** , отображаются только в расширенном представлении оснастки "пользователи и компьютеры".<br/> |



 

## <a name="creating-display-specifiers-for-new-classes"></a>Создание описателей вывода для новых классов

Чтобы настроить пользовательский интерфейс для нового класса, создайте объект описателя экрана для нового класса для каждого поддерживаемого языкового стандарта, а затем задайте нужные атрибуты для описателя экрана.

## <a name="inheriting-display-specifiers-for-derived-classes"></a>Наследование описателей вывода для производных классов

Новый класс, наследующий от существующего класса, не наследует спецификатор экрана родительского класса. Если новый класс должен использовать некоторые или все свойства спецификатора просмотра родительского класса, создайте новый описатель экрана для нового класса и скопируйте свойства из спецификатора экрана родительского класса в Новый описатель экрана нового объекта. Это необходимо сделать для всех языков, для которых применяются свойства спецификатора для просмотра родительского класса.

Некоторые части набора функций пользовательского интерфейса, такие как пункты меню и мастер создания для класса User, реализуются внутренне и недоступны для использования производным объектом. В этих экземплярах лучше расширить существующий класс, чем использовать производный класс.

## <a name="modifying-existing-classes"></a>Изменение существующих классов

Новые атрибуты можно добавить в существующий класс. Новые компоненты пользовательского интерфейса (страницы свойств, элементы меню и отображаемые имена атрибутов) могут быть добавлены или заменены существующим пользовательским интерфейсом. Кроме того, можно разрабатывать новые страницы свойств, предоставляющие меньшее количество атрибутов класса и создавать контекстные меню с меньшим числом действий. Дополнительные сведения см. в разделе [страницы свойств для использования с описателем экрана](property-pages-for-use-with-display-specifiers.md), [Контекстные меню для использования с описателями экрана](context-menus-for-use-with-display-specifiers.md), [отображаемыми именами классов и атрибутов](class-and-attribute-display-names.md).
