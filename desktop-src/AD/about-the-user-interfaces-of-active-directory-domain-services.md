# <a name="about-the-user-interfaces-of-active-directory-domain-services"></a>Сведения о пользовательских интерфейсах служб домен Active Directory Services

Администраторы и пользователи должны иметь возможность просматривать и изменять объекты службы каталогов в службах Microsoft домен Active Directory Services из пользовательского интерфейса. Администраторы управляют сервером Active Directory, используя различные оснастки консоли управления (MMC), в частности, домен Active Directory служб "пользователи и компьютеры", домен Active Directory "сайты и службы" служб домен Active Directory "домены служб и доверие", а также оснастки "домен Active Directory служб диспетчера схем". Пользователь, которому предоставлены соответствующие разрешения, может также использовать оснастки консоли управления Active Directory для просмотра объектов каталога. Оболочка Windows также может использоваться для просмотра объектов каталога. Оболочка Windows позволяет пользователям просматривать объекты, хранящиеся в каталоге, из элемента каталога в папке «Мое сетевое окружение» на рабочем столе или с помощью диалоговых окон поиска, доступных в меню «Пуск».

Службы домен Active Directory Services поддерживают пользовательский интерфейс, который адаптируется в соответствии с потребностями администраторов и конечных пользователей. Домен Active Directory Services позволяют расширять пользовательский интерфейс, представляющий существующие классы объектов, а также новые классы, добавленные в схему. Следующие элементы можно использовать для управления или расширения пользовательского интерфейса для каждого класса, определенного в схеме:

-   [Страницы свойств для использования с описателем экрана](property-pages-for-use-with-display-specifiers.md)
-   [Контекстные меню для использования с описателем экрана](context-menus-for-use-with-display-specifiers.md)
-   [Отображаемые имена классов и атрибутов](class-and-attribute-display-names.md)
-   [Мастера создания объектов](object-creation-wizards.md)
-   [Значки классов](class-icons.md)
-   [Просмотр контейнеров в виде конечных узлов](viewing-containers-as-leaf-nodes.md)

Начиная с Windows 2000, система предоставляет COM-объекты, реализующие общие диалоговые окна для работы с объектами каталога. Эти диалоговые окна предоставляют общий пользовательский интерфейс без необходимости повторной реализации этих диалоговых окон.

-   [Средство выбора объектов каталога](directory-object-picker.md)
-   [Браузер домена](domain-browser.md)
-   [Обозреватель контейнеров](container-browser.md)

Active Directory административных оснасток, контекстных меню и страниц свойств можно расширить с помощью оснасток расширения MMC. Также могут быть реализованы другие расширения MMC, такие как панели задач, элементы пространства имен, панели элементов управления и панель инструментов. Дополнительные сведения см. [в разделе расширение Active Directory административных оснасток](/previous-versions/windows/desktop/mmc/extending-the-active-directory-administrative-snap-ins).
