# <a name="controlling-access-to-objects-and-their-properties"></a>Управление доступом к объектам и их свойствам

Для управления доступом к объектам приложения следует работать с дескриптором безопасности объекта, а именно со списком управления доступом (DACL) и списком записей управления доступом (ACE).

При создании объекта он получает дескриптор безопасности. Дополнительные сведения и описание правил, которые система использует для создания списка DACL для нового объекта, см. [в разделе как устанавливаются дескрипторы безопасности для новых объектов каталога](how-security-descriptors-are-set-on-new-directory-objects.md). Эти правила показывают, что вы можете:

-   Создайте новый дескриптор безопасности и присоедините его к объекту во время создания. Дополнительные сведения см. в разделе [Создание дескриптора безопасности для нового объекта каталога](creating-a-security-descriptor-for-a-new-directory-object.md).
-   Примените наследуемую запись ACE в любой точке иерархии каталогов, чтобы объекты ACE наследовались объектами вниз по дереву. Объект может наследовать ACE от родительского контейнера. Дополнительные сведения см. в разделе [наследование и делегирование администрирования](inheritance-and-delegation-of-administration.md).
-   Если у вас есть необходимые права доступа, укажите элемент ACE в списке DACL по умолчанию в схеме. Каждое определение класса объектов в схеме содержит дескриптор безопасности по умолчанию, который может иметь DACL по умолчанию. Дополнительные сведения см. в разделе [дескриптор безопасности по умолчанию](default-security-descriptor.md).

Кроме того, можно изменить список DACL существующего объекта. Вы можете:

-   Замените список DACL новым.
-   Прочтите существующий список DACL, измените его и примените измененный список DACL. Дополнительные сведения см. [в разделе Установка прав доступа для объекта](setting-access-rights-on-an-object.md).

В следующем списке приводится наиболее важная функция ACE в домен Active Directory Services. С помощью элемента управления доступом можно выполнять следующие действия:

-   Управление доступом к объекту для выполнения конкретных операций.
-   Управление доступом к определенному свойству или набору свойств объекта.
-   Управление тем, кто может создавать дочерние объекты в контейнере, в том числе о том, кто может создавать дочерний объект определенного типа.
-   Определение закрытого элемента управления — права доступа для типа объекта и контроль над тем, кто может выполнять операции, защищенные закрытыми правами.
-   Примените ACE к объекту-контейнеру в корне поддерева каталога, чтобы защита была автоматически унаследована всеми дочерними объектами в дереве.
-   Примените ACE, который наследуется автоматически конкретным типом дочернего объекта в поддереве.
-   Создайте запись ACE, которая предоставляет права группе безопасности, а не одному пользователю.
-   Примените ACE к групповая политика объектам, чтобы управлять учетными записями и компьютерами, затронутыми политикой.
