# <a name="dynamically-linked-auxiliary-classes"></a>Динамически связываемые вспомогательные классы

Динамически связываемый вспомогательный класс — это класс, присоединенный к отдельному объекту, а не к классу объекта. Динамическая компоновка позволяет хранить дополнительные атрибуты в отдельном объекте без влияния на весь лес расширения определения схемы для всего класса. Например, предприятие может использовать динамическую компоновку, чтобы присоединить вспомогательный класс, относящийся к продажам, к объектам-пользователям своих сотрудников по продажам и другим вспомогательным классам отдела для пользовательских объектов сотрудников в других подразделениях.

Динамическая компоновка не сложна: добавьте имя вспомогательного класса к значениям атрибута **objectClass** объекта. Если вспомогательный класс имеет обязательные атрибуты (**муссаве** или **системмуссаве**), их необходимо задать одновременно. Дополнительные сведения и пример кода см. в разделе [Добавление вспомогательного класса к экземпляру объекта](adding-an-auxiliary-class-to-an-object-instance.md).

Чтобы удалить динамически связываемый вспомогательный класс, очистите значения всех атрибутов вспомогательного класса, а затем удалите имя вспомогательного класса из атрибута **objectClass** объекта.

При динамическом добавлении вспомогательного класса, который является подклассом другого вспомогательного класса, в целевой объект добавляются оба вспомогательных класса. Однако при удалении дочернего вспомогательного класса его родительский класс не удаляется. Каждый класс должен быть явно удален.
