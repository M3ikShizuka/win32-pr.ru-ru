# <a name="schema-ad-ds"></a>Схема (AD DS)

Active Directory схема реализуется как набор экземпляров класса объекта, хранящихся в каталоге. Это сильно отличается от многих каталогов, которые имеют схему, но сохраняют ее как текстовый файл, который считывается при запуске. Хранение схемы в каталоге имеет множество преимуществ. Например, пользовательские приложения могут прочитать его, чтобы узнать, какие объекты и свойства доступны.

Active Directory схему можно динамически обновлять. Это значит, что приложение может расширить схему новыми атрибутами и классами и использовать расширения немедленно. Обновление схемы выполняется путем создания или изменения объектов схемы, хранящихся в каталоге. Как и каждый объект в Active Directory, списки управления доступом (ACL) защищают объекты схемы, поэтому полномочные пользователи могут изменять схему только.

Дополнительные сведения см. в разделе [схема Active Directory](active-directory-schema.md).
