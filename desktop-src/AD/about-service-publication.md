# <a name="about-service-publication"></a>О публикации службы

Служба — это приложение, которое делает данные или операции доступными для сетевых клиентов. Часто служба реализуется как официальная служба на основе Microsoft Win32, но это не является обязательным.

Публикация службы — это процесс создания и обслуживания данных об одном или нескольких экземплярах данной службы, чтобы сетевые клиенты могли находить и использовать эту службу. Публикация службы в службах домен Active Directory Services позволяет клиентам и администраторам переходить с ориентированного на компьютерное представление распределенной системы в представление, ориентированное на службы.

**Операционные системы Microsoft Windows NT 3,51 и более поздних версий:** Распределенная система была группой компьютеров, на которых работают различные службы. Чтобы получить доступ к службе, приложению требуются данные о том, какие компьютеры предложили службу.

**Windows 2000 Server, windows 2000 Advanced Server и windows 2000 Datacenter Server:** Службы публикуют свое существование с помощью объектов домен Active Directory Services. Объекты содержат сведения о привязке, используемые клиентскими приложениями для подключения к экземплярам службы. Чтобы получить доступ к службе, клиенту не нужно знать о конкретных компьютерах: объекты на Active Directory сервере включают эти сведения. Клиент запрашивает сервер Active Directory для объекта, представляющего службу (называемый объектом точки соединения), и использует для подключения к службе данные привязки из объекта.

В следующей таблице приведены примеры привязок.



| Служба      | Привязка                                                                                                                                                                                                                     |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Служба файлов | UNC-имя для общей папки. Например, « \\ \\ MyServer \\ мишаренаме».                                                                                                                                                              |
| Веб-служба  | URL-адрес. Например, " https://www.fabrikam.com ".                                                                                                                                                                                 |
| Служба RPC  | Привязка удаленного вызова процедур (RPC): специальные закодированные сведения, используемые для подключения к серверу RPC. Привязки RPC можно преобразовать в строки и из строк с помощью API-интерфейсов RPC. Например: "нкакн \_ IP \_ TCP:сервер. fabrikam. com". |



 

В распределенной системе компьютеры являются механизмами, и интересными сущностями являются доступные службы. С точки зрения пользователя удостоверение компьютера, предоставляющего определенную службу, не имеет значения. Важно то, чтобы получить доступ к самой службе.

Это также относится к управлению службами. Администратор данной зоны DNS не заинтересован на компьютерах, где запущена служба DNS; администратор хочет администрировать DNS. Скорее всего, будет несколько экземпляров службы DNS, одно из которых является полномочным. Компьютеры, поддерживающие службу DNS, не важны для администратора DNS. Важно то, как управлять службой как одним распределенным ресурсом, а не отдельными процессами, работающими на разных компьютерах.
