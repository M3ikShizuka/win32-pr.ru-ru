# <a name="publishing-with-windows-sockets-registration--resolution"></a>Публикация с помощью & разрешения регистрации сокетов Windows

Службы сокетов Windows могут использовать API регистрации и разрешения (РНР) для публикации служб и поиска служб, опубликованных с помощью РНР. Публикация РНР выполняется в два этапа. На первом шаге устанавливается класс службы, связывающий идентификатор GUID с именем службы. Класс службы может содержать зависящие от службы данные конфигурации. Затем службы могут публиковать себя как экземпляры класса службы. При публикации клиенты могут запрашивать службу каталогов для экземпляров заданного класса с помощью интерфейсов API РНР и выбирать экземпляр для привязки. Если класс больше не требуется, его можно удалить.
