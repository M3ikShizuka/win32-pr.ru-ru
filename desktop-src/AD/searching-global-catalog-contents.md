# <a name="searching-the-global-catalog"></a>Поиск в глобальном каталоге

Домен Active Directory Services также имеют глобальный каталог (GC), который содержит частичную реплику всех объектов в каталоге. Он также содержит частичные реплики для контейнеров схемы и конфигурации. Один или несколько контроллеров домена в домене могут содержать копию глобального каталога. Дополнительные сведения о привязке к глобальному каталогу см. в разделе [Привязка к глобальному каталогу](binding-to-the-global-catalog.md).

Глобальный каталог содержит реплику каждого объекта в службах домен Active Directory Services, но с небольшим количеством их атрибутов. Атрибуты в глобальном каталоге чаще всего используются в операциях поиска, таких как имена и фамилии пользователей, имена входа и т. д. Атрибуты глобального каталога также включают в себя те, которые необходимы для нахождение полной реплики объекта. Глобальный каталог позволяет пользователям быстро находить нужные объекты, не зная о том, в каком домене они хранятся, и не требуя наличия непрерывного расширенного пространства имен на предприятии. Это значит, что можно выполнять поиск по всему лесу.

Таким образом, при привязке к объекту в глобальном каталоге будет осуществляться поиск этого объекта и всей иерархии под ним — без необходимости перехода на другой сервер. Однако поиск может использовать только фильтр запросов, содержащий свойства в глобальном каталоге, и может извлекать только свойства из глобального каталога.

Поиск в глобальном каталоге имеет следующие преимущества.

-   Глобальный каталог позволяет выполнять поиск по всему лесу или любой части леса, а также к контейнерам схемы и конфигурации.
-   Глобальный каталог позволяет выполнить полный поиск на одном сервере. Ссылки или отслеживание ссылок не требуются.

Поиск в глобальном каталоге имеет следующие недостатки.

-   Глобальный каталог содержит небольшое подмножество свойств для каждого объекта. Если фильтр запроса содержит свойства, которых нет в глобальном каталоге, запрос вычислит выражения, содержащие эти свойства, как false. При указании свойств неглобального каталога в списке возвращаемых свойств эти свойства не извлекаются.
-   Для поиска в глобальном каталоге должен быть доступен контроллер домена, содержащий глобальный каталог. Если он недоступен, выполнять поиск по глобальному каталогу нельзя.
-   Глобальный каталог доступен только для чтения. Это означает, что вы не можете выполнить привязку к объекту в глобальном каталоге для создания, изменения или удаления объектов.
