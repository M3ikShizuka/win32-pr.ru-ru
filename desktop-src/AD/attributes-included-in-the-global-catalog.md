# <a name="including-attributes-in-the-global-catalog"></a>Включение атрибутов в глобальный каталог

Глобальный каталог леса содержит частичную реплику каждого объекта в лесу. Для каждого объекта глобальный каталог включает только подмножество атрибутов каждого объекта. Атрибут [**исмемберофпартиалаттрибутесет**](/windows/desktop/ADSchema/a-ismemberofpartialattributeset) объекта [**AttributeSchema**](/windows/desktop/ADSchema/c-attributeschema) имеет значение **true** , если атрибут реплицируется в глобальный каталог.

Атрибуты со следующими характеристиками подходят для хранения в глобальном каталоге:

-   Атрибут является глобально интересным, так как атрибут необходим для поиска объектов, которые могут встречаться в любом месте леса, или потому, что доступ для чтения к атрибуту важен, даже если полный объект недоступен. Примером первого типа является атрибут [**Location**](/windows/desktop/ADSchema/a-location) , который можно использовать для поиска объекта [**очереди печати**](/windows/desktop/ADSchema/c-printqueue) . Примером второго типа является [**telephoneNumber**](/windows/desktop/ADSchema/a-telephonenumber), так как вы можете вызвать пользователя, даже если вы не можете получить доступ к полной реплике своего объекта [**User**](/windows/desktop/ADSchema/c-user) .
-   Изменчивости атрибута очень низкий. Это важно, поскольку если класс атрибута включен в глобальный каталог, изменения каждого значения этого класса атрибута в корпоративном лесу реплицируются на все серверы глобального каталога предприятия.
-   Размер значения атрибута невелик. "Малый" — очень субъективная: При размещении атрибута в глобальном каталоге рассмотрите влияние репликации атрибута на все серверы глобального каталога в предприятии. Чем меньше атрибут, тем меньше его влияние. Поскольку репликация происходит только при изменении атрибута, влияние репликации также меньше, чем изменчивости снижается, поэтому большой атрибут с очень низким изменчивости может иметь меньшее влияние, чем небольшой атрибут с высоким изменчивости.

При принятии решения о необходимости размещения атрибута в глобальном каталоге Помните, что вы намерены повысить репликацию и увеличить объем дискового хранилища на серверах глобального каталога для, что может повысить производительность запросов. Как правило, глобальный каталог используется для поиска интересующего объекта, что позволяет считывать выбранные атрибуты объекта. Если нужные атрибуты реплицируются в глобальный каталог, их можно прочитать непосредственно из глобального каталога. Кроме того, для чтения атрибутов, которые не реплицируются в глобальный каталог, необходимо выполнить дополнительные действия для их извлечения. В этом случае после поиска в глобальном каталоге для поиска интересующего объекта необходимо прочитать различающееся имя объекта из глобального каталога, использовать DN для прямого связывания с полной репликой объекта, которая может находиться на другом сервере, и, наконец, считывать атрибуты, не являющиеся глобальным каталогом, из полной реплики объекта.

Часто запрашиваемые и упоминаемые атрибуты, такие как имя сотрудника и номер телефона, хорошо подходят для включения в глобальный каталог. Нечасто упоминаемый атрибут, например "Дриверверсион" для принтеров, лучше всего выходит из глобального каталога.
