# <a name="naming-contexts-and-directory-partitions"></a>Контексты именования и разделы каталога

Каждый контроллер домена в лесу домена, управляемом службами домен Active Directory Services, включает [*разделы каталога*](/previous-versions/windows/desktop/legacy/ms681901(v=vs.85)). Разделы каталога также называются [*контекстами именования*](/previous-versions/windows/desktop/legacy/ms681918(v=vs.85)). Раздел каталога — это непрерывная часть общего каталога, в которой есть независимая область репликации и данные планирования. По умолчанию служба домен Active Directory для предприятия содержит следующие разделы:

-   [*Секция схемы*](/previous-versions/windows/desktop/legacy/ms681936(v=vs.85)): Секция схемы содержит объекты **classSchema** и **attributeSchema** , которые определяют типы объектов, которые могут существовать в лесу. Каждый контроллер домена в лесу имеет реплику того же раздела схемы.
-   [*Раздел конфигурации*](/previous-versions/windows/desktop/legacy/ms681898(v=vs.85)): раздел конфигурации содержит топологию репликации и другие данные конфигурации, которые должны реплицироваться в пределах леса. Каждый контроллер домена в лесу имеет реплику одного и того же раздела конфигурации.
-   [*Раздел домена*](/previous-versions/windows/desktop/legacy/ms681901(v=vs.85)): раздел домена содержит объекты каталога, такие как пользователи и компьютеры, связанные с локальным доменом. Домен может иметь несколько контроллеров домена, а лес может иметь несколько доменов. Каждый контроллер домена хранит полную реплику раздела домена для своего локального домена, но не сохраняет реплики разделов домена для других доменов.

Windows В сервере 2003 появился *раздел каталога приложений*, который предоставляет возможность управлять областью репликации и разрешать размещение реплик в более подходящем для динамических данных образом. Дополнительные сведения о разделах каталога приложений см. в разделе [сведения о разделах каталога приложений](about-application-directory-partitions.md).

Дополнительные сведения о том, как службы домен Active Directory поддерживают согласованность между различными репликами раздела каталога, см. в разделе [репликация и целостность данных](replication-and-data-integrity.md).
