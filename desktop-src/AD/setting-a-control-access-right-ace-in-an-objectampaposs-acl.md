# <a name="setting-a-control-access-right-ace-in-an-objects-acl"></a>Установка элемента управления право доступа ACE в ACL объекта

Используя интерфейсы ADSI, вы задаете элемент управления ACE right так же, как и запись ACE, относящаяся к конкретному свойству, за исключением того, что свойство [**иадсакцессконтролентри. ObjectType**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods) является **ригхтсгуидом** права доступа к элементу управления. Имейте в виду, что для установки списков управления доступом к объектам каталога можно также использовать API-интерфейсы безопасности Win32.

В следующей таблице перечислены свойства [**иадсакцессконтролентри**](/windows/desktop/api/iads/nn-iads-iadsaccesscontrolentry) для прав доступа Control, которые можно использовать для задания свойств элемента управления доступом.



| Свойство                                                       | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|----------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**AccessMask**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods) | Для прав доступа управления, которые контролируют доступ расширенных прав к специальным операциям, значение AccessMask должно содержать флаг доступа к подсистемам **\_ \_ \_ управления \_ доступом DS** . Для прав доступа управления, определяющих набор свойств, AccessMask содержит **рекламу \_ справа \_ DS \_ Read \_ prop** и (или) **ADSS \_ right \_ DS- \_ \_ prop**.<br/> Для управления правами доступа, которые контролируют проверенные записи, [**AccessMask**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods) содержит **рекламу в \_ прямом \_ \_** доступе к DS.<br/> |
| [**Метки**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods)      | Это значение должно включать флаг **\_ \_ \_ \_ присутствия типа объекта флага ADS** .                                                                                                                                                                                                                                                                                                                                                                                                                       |
| [**ObjectType**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods) | Это значение должно быть форматом [**StringFromGUID2**](/windows/win32/api/combaseapi/nf-combaseapi-stringfromguid2) атрибута **ригхтсгуид** права доступа к элементу управления. Имейте в виду, что в записи ACE строка GUID должна содержать начальные и завершающие фигурные скобки, даже если атрибут **ригхтсгуид** объекта **контролакцессригхт** не содержит фигурные скобки.                                                                                                                                     |
| [**AceType**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods)    | Либо **ADS \_ ACETYPE \_ доступ к \_ разрешенному \_ объекту** , чтобы предоставить доверенному лицу доступ к **\_ \_ \_ \_ объекту** Access Control право или ADS ACETYPE, чтобы запретить доверенному лицу право доступа к элементу управления.                                                                                                                                                                                                                                                                                                     |
| [**Доверенное лицо**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods)    | Субъект безопасности, например пользователь, группа, компьютер и т. д., к которым применяется ACE.                                                                                                                                                                                                                                                                                                                                                                                              |



 

Дополнительные сведения о создании записи ACE см. [в разделе Установка прав доступа для объекта](setting-access-rights-on-an-object.md).

Дополнительные сведения и пример кода для настройки элемента управления доступом см. в разделе [пример кода для настройки элемента управления доступом на объекте каталога](example-code-for-setting-an-ace-on-a-directory-object.md).
