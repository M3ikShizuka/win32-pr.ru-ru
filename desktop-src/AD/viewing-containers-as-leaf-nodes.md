# <a name="viewing-containers-as-leaf-nodes"></a>Просмотр контейнеров в виде конечных узлов

Любой объект в домен Active Directory Services может быть контейнером других объектов. Это может нагружать пользовательский интерфейс, поэтому можно объявить, что объект определенного класса должен отображаться только как лист в пользовательском интерфейсе. Атрибут **треатаслеаф** является атрибутом отображения с одним значением, который определяет, должны ли объекты этого класса отображаться только как конечные объекты. Этот атрибут является логическим значением, которое, если **true**, указывает, что объекты класса должны отображаться только как конечные элементы. **Значение false** указывает, что объекты класса могут отображаться в виде контейнера или листа. Как и все атрибуты спецификатора экрана, атрибут **треатаслеаф** задается для каждого языкового стандарта, поэтому этот атрибут может быть локализован по мере необходимости. Например, при **отображении пользователем** описателя "Английский язык (0409)" в качестве значения атрибута **треатаслеаф** по умолчанию установлено **значение true** . В результате пользовательский интерфейс отображает все **пользовательские** объекты как конечные объекты.

**Задание значения атрибута **треатаслеаф****

1.  Выполните привязку к нужному атрибуту дисплея в нужном языковом стандарте. Дополнительные сведения и пример кода см. в разделе [ДисплайспеЦифиерс Container](displayspecifiers-container.md).
2.  Используйте метод [**iAds::P UT**](/windows/desktop/api/iads/nf-iads-iads-put) , чтобы задать для атрибута **треатаслеаф** **значение true** или **false**.
3.  Чтобы зафиксировать изменения в каталоге, вызовите метод [**iAds:: сетинфо**](/windows/desktop/api/iads/nf-iads-iads-setinfo) .
