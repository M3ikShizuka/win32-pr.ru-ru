# <a name="binding-to-the-search-start-point"></a>Привязка к начальной точке поиска

Начальная точка поиска — это контейнер, который напрямую или косвенно содержит искомые объекты. Поиск не будет выполняться в контейнерах выше начальной точки поиска. Например, возьмем следующую гипотетическую структуру каталогов:

``` syntax
Domain A
    Container 1
        Object 1
        Object 2
    Container 2
        Object 3
        Object 4
```

Если поиск выполняется для всех объектов, а начальная точка поиска — "контейнер 2", то будут получены только объекты "объект 3" и "объект 4". Если поиск выполняется с точкой начала поиска в "контейнер 1", то будет получен объект "Object 1" и "Object 2".

Чтобы выполнить привязку к начальной точке поиска, выполните привязку к ADsPath контейнера, который будет начальной точкой поиска.
