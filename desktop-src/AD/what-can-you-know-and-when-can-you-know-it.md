# <a name="what-can-you-know-and-when-can-you-know-it"></a>Что можно узнать, а когда это вы узнаете?

Приложения, которые чувствительны к состояниям с задержкой, должны распознать состояния проблем и предпринять соответствующие меры. Существует два состояния проблем: отклонение версий и частичное обновление. Отклонения версий не обнаруживаются при проверке службы каталогов (Помните, что аксиома распределенных вычислений, в которых [домен Active Directory службы используют эту модель репликации](why-active-directory-domain-services-uses-this-replication-model.md)). Частичное обновление можно обнаружить, добавив метаданные к объектам, составляющим связанный набор. В последующих разделах этого документа приведены предложения для таких метаданных. Для снижения вероятности отклонения версий и частичного обновления могут потребоваться приложения стратегий. Эти стратегии также обсуждаются в последующих разделах этого документа.
