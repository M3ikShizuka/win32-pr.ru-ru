# <a name="controlling-object-visibility"></a>Управление видимостью объекта

Домен Active Directory Services предоставляют возможность скрывать объекты пользователей, которым были отказано в определенных правах. Если объект скрыт, приложение, выполняющееся с учетными данными пользователя, не сможет перечислять или привязывать объекты.

Если пользователю предоставлено право на управление доступом к [**\_ \_ \_ \_ списку актрл DS**](/windows/win32/api/iads/ne-iads-ads_rights_enum) справа в контейнере, пользователь может просматривать любые дочерние объекты контейнера. Аналогичным образом, если пользователю отказано в разрешении на доступ к **\_ \_ \_ \_ списку актрл DS** право управления доступом в контейнере, пользователь не сможет просматривать какие-либо дочерние объекты контейнера. Это позволяет скрывать содержимое всех контейнеров.

Active Directory сервер можно также перевести в режим специального списка объектов, установив третий символ свойства [**дшеуристикс**](/windows/desktop/ADSchema/a-dsheuristics) в значение "1". Режим объекта списка можно отключить, установив третий символ свойства **дшеуристикс** в значение "0". Если Active Directory сервер находится в режиме объекта списка, объект по-прежнему будет виден, если пользователь предоставил право на отображение [**\_ \_ списка AD актрл \_ DS \_**](/windows/win32/api/iads/ne-iads-ads_rights_enum) прямо на родительском объекте. Однако если пользователь запретил доступ к **\_ \_ списку AD актрл \_ DS \_** прямо на родительском объекте, то определенные дочерние объекты все равно можно сделать видимыми, если пользователь предоставил доступ к **\_ \_ \_ \_ объекту списка DS** справа как к родительским, так и к дочерним объектам. Режим объектов списка позволяет системному администратору предоставлять или запрещать доступ к отдельным объектам для пользователей или групп. Режим объектов списка следует использовать осторожно, так как он требует значительно большего числа вызовов проверки доступа, выполняемых службой каталогов, чтобы определить, является ли объект видимым для пользователя. Поэтому он может негативно сказаться на производительности просмотра или чтения объектов служб домен Active Directory.
