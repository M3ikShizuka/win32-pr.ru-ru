# <a name="access-control-and-object-deletion"></a>Управление доступом и удаление объектов

Домен Active Directory службы позволяют удалить объект, если у вас есть одно из следующих прав доступа:

-   Удаление доступа к самому объекту
-   Реклама \_ справа \_ DS \_ Удаление \_ дочернего объекта для этого типа объектов в родительском контейнере

Имейте в виду, что система проверяет дескриптор безопасности как для объекта, так и для его родителя перед запретом на удаление. Это означает, что элемент управления доступом, который явно запрещает доступ DELETE к пользователю, не оказывает никакого влияния, если пользователь УДАЛЯЕТ \_ дочерний доступ к родительскому объекту. Аналогично, элемент управления доступом, который запрещает удаление \_ дочернего объекта в родительском объекте, может быть переопределен, если разрешен доступ на удаление к самому объекту.

Чтобы выполнить операцию удаления дерева, например с помощью метода [**иадсделетеопс::D елетеобжект**](/windows/desktop/api/iads/nf-iads-iadsdeleteops-deleteobject) , необходимо иметь AD \_ \_ DS right Directory \_ DELETE \_ доступ к объекту. При наличии этого права доступа можно удалить объект и все дочерние объекты независимо от защиты дочерних объектов. Чтобы удалить дерево, если у вас нет ADS с \_ \_ доступом к дереву доменных служб Active Directory \_ \_ , необходимо выполнить рекурсивный обход дерева, удалив каждый объект по отдельности. В этом случае необходимо иметь доступ на удаление или удаление \_ дочернего элемента для каждого объекта в дереве.

> [!WARNING]
> Если у пользователей есть \_ права \_ доступа к дереву доменных служб Active Directory \_ \_ для объекта, это дает возможность удалить все поддерево, включая все дочерние объекты. По этой причине вы можете отменить разрешение на доступ "удаление поддерева" для всех пользователей в родительском контейнере.
