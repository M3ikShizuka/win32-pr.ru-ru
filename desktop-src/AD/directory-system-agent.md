# <a name="directory-system-agent"></a>Системный агент каталога

[*Агент системы каталогов*](/previous-versions/windows/desktop/legacy/ms681901(v=vs.85)) (DSA) — это набор служб и процессов, которые выполняются на каждом контроллере домена и предоставляют доступ к хранилищу данных. Хранилище данных — это физическое хранилище данных каталога, расположенное на жестком диске. В домен Active Directory Services DSA является частью подсистемы локального системного администратора (LSA). Клиенты получают доступ к каталогу с помощью одного из следующих механизмов, поддерживаемых DSA:

-   Клиенты LDAP подключаются к DSA с помощью протокола LDAP. Службы домен Active Directory Services поддерживают LDAP 3,0, определенный стандартом RFC 3377 и LDAP 2,0, который определяется RFC 1777. Клиенты Windows используют LDAP 3,0 для подключения к DSA.
-   Клиенты MAPI, такие как Microsoft Exchange Server, подключаются к DSA с помощью интерфейса удаленного вызова процедуры MAPI.
-   DSA в домен Active Directory Services подключаются друг к другу для выполнения репликации с помощью собственного интерфейса удаленного вызова процедур.
