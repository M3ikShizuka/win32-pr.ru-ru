# <a name="linked-attributes-ad-ds"></a>Связанные атрибуты (AD DS)

Связанные атрибуты — это пары атрибутов, в которых система вычисляет значения одного атрибута (ссылки назад) на основе значений, заданных для другого атрибута (прямая ссылка) в пределах леса. Значение обратной связи для любого экземпляра объекта состоит из DNs всех объектов, которые имеют DN объекта, заданное в соответствующей ссылке вперед. Например, "Manager" и "Reports" — это пара связанных атрибутов, где Manager — это прямая ссылка, а отчеты — обратная ссылка. Теперь предположим, что счет — это менеджер Джо. Если вы храните различающееся имя объекта пользователя Bill в атрибуте "Manager" объекта пользователя Джо, то DN объекта пользователя Джо будет отображаться в атрибуте "Reports" объекта "счет".

Пара ссылок вперед и назад определяется значениями [**LinkId**](/windows/desktop/ADSchema/a-linkid) двух определений [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) . **LinkId** прямой ссылки — это четное, положительное, ненулевое значение, а **LinkId** связанной ссылки назад — это прямой **LinkId** плюс один. Например, **LinkId** для "Manager" — 42, а **LinkId** для "reports" — 43.

Ниже приведен список рекомендаций по определению новой пары связанных атрибутов.

-   Значения [**LinkId**](/windows/desktop/ADSchema/a-linkid) должны быть уникальными для всех объектов [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) . Во избежание конфликтов следует автоматически создать **LinkId** , следуя инструкциям в разделе [Получение идентификатора ссылки](obtaining-a-link-id.md).
-   Обратная ссылка должна иметь соответствующую прямую ссылку, то есть должна существовать прямая ссылка, прежде чем можно будет создать соответствующий атрибут ссылки назад.
-   Обратная ссылка всегда является атрибутом с несколькими значениями. Прямая ссылка может иметь одно или несколько значений. При наличии связи "многие ко многим" используйте многозначную ссылку вперед.
-   Значение [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) прямой ссылки должно быть 2.5.5.1, 2.5.5.7 или 2.5.5.14. Эти значения соответствуют синтаксису, содержащему различающееся имя, например, синтаксис [**Object (DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn) .
-   Значение [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) обратной связи должно быть 2.5.5.1, которое является синтаксисом [**Object (DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn) .
-   По соглашению атрибуты ссылки назад добавляются к значению [**mayContain**](/windows/desktop/ADSchema/a-maycontain) [**верхнего**](/windows/desktop/ADSchema/c-top) абстрактного класса. Это позволяет считывать атрибут ссылки назад из объектов любого класса, поскольку они фактически не сохраняются вместе с объектом, но рассчитываются на основе значений прямой ссылки.
