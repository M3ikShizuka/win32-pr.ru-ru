# <a name="defining-a-new-class"></a>Определение нового класса

При определении нового класса укажите юридические родительские классы нового класса, то есть классы, которые могут содержать экземпляры нового класса. Юридические родительские классы указываются в атрибутах **possSuperiors** и **системпосссупериорс** нового класса, а также в возможных старших классах, унаследованных от его классов, но не из вспомогательных классов.

Будьте специфичны при определении юридических родительских классов для нового класса. Определите, где пользователи должны создавать экземпляры класса. Например, указание "Container" в качестве юридического родителя позволит пользователю создавать экземпляры в любом из стандартных контейнеров (**контейнер**, **OrganizationalUnit** и т. д.), при указании "Computer" создание экземпляров будет разрешено только в экземплярах объекта **Computer** .

**Создание класса**

1.  Выберите имя класса. Дополнительные сведения о создании общего имени и отображаемого имени LDAP для нового класса см. в разделе [присвоение имен атрибутам и классам](naming-attributes-and-classes.md).
2.  Получите идентификатор объекта (OID) для класса. Дополнительные сведения см. в разделе [Получение идентификатора объекта](obtaining-an-object-identifier.md).
3.  Выберите категорию объектов по умолчанию для класса. Дополнительные сведения см. в разделе [класс объектов и категория объектов](object-class-and-object-category.md).
4.  Выберите "Категория класса объектов" для класса. Указывает, является ли класс абстрактным, структурным или вспомогательным. Дополнительные сведения см. в разделе [структурные, абстрактные и вспомогательные классы](structural-abstract-and-auxiliary-classes.md).
5.  Создайте новый объект **classSchema** . Для объекта **classSchema** можно задать множество атрибутов. Следующие атрибуты являются критически важными для определения нового класса:

    -   Классы, от которых наследуется новый класс: **списках subClassOf**, **auxiliaryClass** и **системауксилиарикласс**
    -   Имена и идентификаторы для нового класса: **CN**, **lDAPDisplayName**, **админдисплайнаме**, **schemaIDGUID**, **governsID**
    -   Возможные атрибуты нового класса: **mustContain**, **системмустконтаин**, **mayContain**, **системмайконтаин**
    -   Возможные родители нового класса: **possSuperiors**, **системпосссупериорс**
    -   **objectClassCategory**
    -   **defaultObjectCategory**
    -   **дефаулсидингвалуе**
    -   **рднаттид**
    -   **дефаултсекуритидескриптор**
    -   **Описание** (необязательно)

    Дополнительные сведения и описания этих атрибутов см. в разделе [характеристики классов объектов](characteristics-of-object-classes.md).

    Имейте в виду, что классы, указанные в **списках subClassOf**, **possSuperiors**, **системпосссупериорс**, **auxiliaryClass** и **системауксилиарикласс**, должны существовать, когда новый класс записывается в каталог. в противном случае объект **classSchema** не будет добавлен в каталог. Аналогичным образом, атрибуты, указанные в **mustContain**, **системмустконтаин**, **mayContain** и **системмайконтаин**, должны существовать, иначе операция создания класса завершится ошибкой.

6.  Запишите новый объект **classSchema** в каталог.

**Добавление атрибута в свойство mayContain**

1.  Получите объект **classSchema** для изменяемого класса.
2.  Добавьте новый атрибут в свойство **mayContain** с несколькими значениями.
3.  Запишите измененный объект **classSchema** обратно в каталог.

Новые атрибуты можно создавать одновременно с новыми классами. порядок создания новых атрибутов и классов важен. Дополнительные сведения см. [в разделе что должна делать установка](what-the-installation-must-do.md).

**Добавление новых атрибутов и новых классов в одно и то же время**

1.  Сначала определите все новые атрибуты. Дополнительные сведения см. [в разделе Определение нового атрибута](defining-a-new-attribute.md).
2.  Обновите кэш схемы, прежде чем определять новые классы. Дополнительные сведения см. [в разделе Обновление кэша схемы](updating-the-schema-cache.md).
3.  Создайте новые классы.
4.  Добавьте необходимые атрибуты в новые классы.
5.  Снова Обновите кэш схемы.
