# <a name="mutual-authentication-using-kerberos"></a>Взаимная проверка подлинности с помощью Kerberos

Взаимная проверка подлинности — это функция безопасности, в которой клиентский процесс должен доказать свою личность службе, и служба должна доказать свою личность клиенту, прежде чем трафик приложения передается через подключение клиента или службы.

домен Active Directory Services и Windows обеспечивают поддержку имен участников-служб (SPN), которые являются ключевым компонентом механизма Kerberos, с помощью которого клиент проходит проверку подлинности службы. SPN — это уникальное имя, идентифицирующее экземпляр службы и связанное с учетной записью входа, под которой работает экземпляр службы. Компоненты имени субъекта-службы — это то, что клиент может создать имя субъекта-службы без учетной записи входа в службу. Это позволяет клиенту запросить службу для проверки подлинности учетной записи, даже если у клиента нет имени учетной записи.

-   Как клиентское приложение использует имя субъекта-службы в сочетании с интерфейсом поставщика поддержки безопасности (SSPI) для проверки подлинности службы.
