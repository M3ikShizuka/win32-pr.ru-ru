# <a name="documenting-schema-extensions"></a>Документирование расширений схемы

Помимо предоставления отдельной процедуры установки расширений схемы, характер расширений схемы должен быть четко документирован. Документация должна содержать:

-   Список всех новых классов и атрибутов с описанием каждого из них.
-   Список всех стандартных классов, которые расширены с помощью новых атрибутов и списка атрибутов.
-   Список новых страниц свойств с объяснением того, как использовать каждую из них
-   Объяснение того, как служба использует расширения схемы
