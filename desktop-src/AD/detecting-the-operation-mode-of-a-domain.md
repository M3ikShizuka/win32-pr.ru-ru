# <a name="detecting-the-operation-mode-of-a-domain"></a>Обнаружение режима работы домена

В Windows 2000 домен может работать в двух режимах работы: Mixed и Native. Смешанный режим следует использовать для включения контроллеров домена под управлением Windows NT 4,0 в домене Windows 2000. Смешанный режим не поддерживает универсальные группы и вложенные группы. Если все контроллеры домена в домене работают под Windows 2000, можно использовать собственный режим.

Чтобы программно определить режим работы домена Windows 2000, прочтите свойство [**нтмикседдомаин**](/windows/desktop/ADSchema/a-ntmixeddomain) объекта [**домаинднс**](/windows/desktop/ADSchema/c-domaindns) для этого домена. Нулевое значение (0) означает, что домен работает в собственном режиме. Значение одного (1) указывает, что домен работает в смешанном режиме. Можно также использовать функцию [**дсролежетпримаридомаининформатион**](/windows/desktop/api/Dsrole/nf-dsrole-dsrolegetprimarydomaininformation) для получения режима работы, а также других данных о домене и его состоянии.

Чтобы выполнить привязку к объекту [**домаинднс**](/windows/desktop/ADSchema/c-domaindns) домена учетной записи пользователя, под которой выполняется приложение, используйте бессерверную привязку и RootDSE, чтобы получить различающееся имя для домена, а затем используйте это различающееся имя для привязки к объекту **домаинднс** , который представляет этот домен. Дополнительные сведения о бессерверных привязках и rootDSE см. в статье [Привязка к серверу и RootDSE](serverless-binding-and-rootdse.md).

Дополнительные сведения и пример кода, в котором показано, как программным способом определить режим работы домена, см. в разделе [пример кода для определения режима работы](example-code-for-determining-the-operation-mode.md).
