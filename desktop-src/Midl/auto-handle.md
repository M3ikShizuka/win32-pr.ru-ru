# <a name="auto_handle-attribute"></a>\_атрибут Auto Handle

Атрибут **\[ Auto \_ Handle \]** ACF указывает заглушке автоматически устанавливать привязку для функции, которая не имеет явного параметра обработчика привязки.

> [!Note]  
> Этот атрибут устарел и больше не поддерживается. Рекомендуется использовать параметр [**/robust**](-robust.md) .

 

``` syntax
[ 
    auto_handle [, interface-attribute-list] 
] 
interface interface-name
{
    interface-definition
}
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*Interface-список атрибутов* 
</dt> <dd>

Указывает ноль или более атрибутов, которые применяются к интерфейсу в целом, например [**код**](code.md) [**или код**](nocode.md). Разделяйте атрибуты интерфейса запятыми.

</dd> <dt>

*имя интерфейса* 
</dt> <dd>

Указывает имя интерфейса.

</dd> <dt>

*определение интерфейса* 
</dt> <dd>

Задает инструкции IDL, которые формируют определение интерфейса.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Атрибут **\[ Auto \_ Handle \]** отображается в заголовке интерфейса ACF. Он также отображается в заголовке интерфейса IDL-файла при указании конфигурации компилятора MIDL Switch [**/АПП \_ config**](-app-config.md).

Когда клиент вызывает функцию, которая использует автоматическую привязку и не существует привязки к серверу, заглушка автоматически устанавливает привязку. Привязка повторно используется для последующих вызовов других функций в интерфейсе, который использует автоматическую привязку. Приложению клиента не нужно объявлять или выполнять обработку, связанную с маркером привязки.

Если ACF отсутствует или не включает атрибут [**\[ неявного \_ обработчика \]**](implicit-handle.md) , компилятор MIDL использует **\[ автоматическую \_ обработку \]** и выдает информационное сообщение. Компилятор MIDL также использует при необходимости **\[ автоматическую \_ обработку \]**, чтобы установить начальную привязку для [**\[ \_ маркера \] контекста**](context-handle.md).

Атрибут **\[ Auto \_ Handle \]** может возникать, только если не существует [**\[ неявного \_ обработчика \]**](implicit-handle.md) или [**\[ \_ \] атрибута неявного**](explicit-handle.md) обработчика. Атрибут **\[ Auto \_ Handle \]** может появляться в заголовке интерфейса ACF или IDL не более одного раза.

> [!Note]  
> Нельзя использовать автоматическую привязку (с атрибутом **\[ Auto \_ Handle \]** или по умолчанию) при обработке данных по каналам.
