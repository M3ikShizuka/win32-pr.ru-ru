# <a name="oldnames-switch"></a>/олднамес, параметр

Параметр **/олднамес** направляет компилятор MIDL для создания имен интерфейсов, которые не содержат номер версии.

``` syntax
midl /oldnames
```

## <a name="switch-options"></a>Параметры коммутатора

Этот параметр не имеет параметров.

## <a name="remarks"></a>Комментарии

Компилятор MIDL включает номер версии интерфейса в имя интерфейса, созданного в заглушке (например, iface \_ v1 \_ 0 \_ серверифхандле). Этот формат именования согласуется с форматом, используемым компилятором использование DCE IDL. Однако он отличается от формата именования, используемого компилятором MIDL 1,0. Компилятор MIDL 1,0 не включал номера версий в имена интерфейсов (например, iface \_ серверифхандле). Параметр **/олднамес** позволяет указать компилятору MIDL создавать имена интерфейсов, которые не включают номер версии. Таким образом, формат согласован с именами, созданными компилятором MIDL 1,0.

Если у вас есть код серверного приложения, который был написан для использования с заглушкой, созданной компилятором MIDL 1,0, он ссылается на имя интерфейса, созданное с помощью MIDL (например, в вызове [**рпксерверрегистериф**](/windows/desktop/api/rpcdce/nf-rpcdce-rpcserverregisterif)), необходимо изменить его, чтобы он ссылался на стиль имени интерфейса, поддерживаемого в версии 2,0 или более поздней версии компилятора MIDL. Кроме того, можно использовать параметр **/олднамес** при вызове компилятора MIDL.

## <a name="examples"></a>Примеры

**MIDL/олднамес filename. idl**
