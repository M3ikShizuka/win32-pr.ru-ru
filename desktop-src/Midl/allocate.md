# <a name="allocate-attribute"></a>выделить атрибут

Атрибут **\[ allocate \]** ACF позволяет настраивать выделение и освобождение памяти для типа, определенного в IDL-файле.

``` syntax
typedef [allocate (allocate-option-list) [, type-attribute-list] ] type-name;
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*список выделения и параметров* 
</dt> <dd>

Указывает один или несколько параметров выделения памяти. Выберите **один или несколько узлов \_** либо один из них либо **бесплатный** или **не \_ бесплатный**, либо один из этих пар. **\_** При указании более одного параметра разделяйте параметры запятыми.

</dd> <dt>

*Type-Attribute-List* 
</dt> <dd>

Указывает другие необязательные атрибуты типа ACF. При указании более одного атрибута типа разделяйте параметры запятыми.

</dd> <dt>

*имя типа* 
</dt> <dd>

Указывает тип, определенный в IDL-файле.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Атрибут **\[ выделения \]** имеет следующие допустимые параметры.



| Параметр           | Описание                                                           |
|------------------|-----------------------------------------------------------------------|
| **все \_ узлы**   | Выполняет один вызов для выделения и освобождения памяти для всех узлов.             |
| **один \_ узел** | Делает множество отдельных вызовов для выделения и освобождения каждого узла памяти. |
| **free**         | Освобождает память при возврате из заглушки сервера.                          |
| **не \_ бесплатно**   | Не освобождает память при возврате из заглушки сервера.                  |



 

По умолчанию заглушки могут выделять хранилище для данных, на которые ссылается уникальный или полный указатель, путем вызова метода [**\_ пользовательского \_ выделения MIDL**](midl-user-allocate-1.md) и [**\_ \_ бесплатного пользователя MIDL**](midl-user-free-1.md) по отдельности для каждого указателя.

Можно оптимизировать скорость приложения, указав параметр **все \_ узлы**. Этот параметр указывает заглушке вычислить размер всей памяти, на которую ссылается указатель указанного типа, и выполнить один вызов для [**\_ \_ выделения пользователю MIDL**](midl-user-allocate-1.md). Заглушка освобождает память, делая один вызов для [**пользователя MIDL \_ \_ свободным**](midl-user-free-1.md).

Параметр **без \_** поддержки направляет компилятор MIDL для создания заглушки сервера, которая не вызывает для указанного типа [**функцию \_ MIDL \_ без вызова пользователя**](midl-user-free-1.md) . Параметр **не \_ свободен** позволяет структурам указателей оставаться доступными для серверного приложения после завершения удаленного вызова процедуры и возврата клиенту.

Атрибут **\[ выделения \]** приведет к тому, что любой **\[ выходной \]** параметр, который является указателем на тип, квалифицированный с параметром **все \_ узлы** , перераспределяет память при распаковке данных. Это приложение отвечает за освобождение памяти, выделенной ранее для этого параметра. Пример:

``` syntax
typedef struct thistype 
{ 
    [string] char * PTHISTYPE;  
} * PTHISTYPE
HRESULT proc1 ( [in,out] PTHISTYPE * ppthistype);
```

Тип данных ПСИСТИПЕ будет повторно выделен в [**\[ обратном \]**](out-idl.md) направлении заглушкой перед распаковкой. Поэтому приложение должно освободить память, выделенную ранее для данных этого параметра, или произойдет утечка памяти.

## <a name="examples"></a>Примеры

``` syntax
/* ACF file */ 
typedef [allocate(all_nodes, dont_free)] PTYPE1; 
typedef [allocate(all_nodes)] PTYPE2; 
typedef [allocate(dont_free)] PTYPE3;
```
