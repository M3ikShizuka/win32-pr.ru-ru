# <a name="osf-switch"></a>/ОСФ, параметр

Коммутатор **/ОСФ** принудительно обеспечивает беспроблемную совместимость с использование DCE.

``` syntax
midl /osf
```

## <a name="switch-options"></a>Параметры коммутатора

Этот параметр не имеет параметров.

## <a name="remarks"></a>Remarks

Используйте этот параметр, если приложению требуется обеспечение высокой совместимости с использование DCE для обеспечения переносимости.

В режиме **/ОСФ** пакет RPCSS автоматически включается при использовании полных указателей, аргументы нуждаются в выделении памяти или при использовании атрибута [**включить \_ выделение**](enable-allocate.md) . Это означает, что вам не нужно предоставлять пользовательские функции [**пользовательского \_ \_ выделения MIDL**](/windows/desktop/Rpc/the-midl-user-allocate-function) и [**MIDL \_ \_**](/windows/desktop/Rpc/the-midl-user-free-function) в клиенте и серверном приложении.

Следующие расширенные функции Майкрософт недоступны при компиляции с параметром **/ОСФ** :

-   Абстрактные деклараторы (безымянные параметры) в IDL-файле.
-   Определения интерфейса для COM-объектов.
-   Имена интерфейсов, длина которых превышает 17 символов.
-   Атрибуты только для MIDL, такие как [**\_ маршалирование по сети**](wire-marshal.md), [**\_ маршалирование пользователя**](user-marshal.md)и расширения typelib (ODL).
-   Использование ключевых слов ACF в IDL-файле (параметр MIDL **/АПП \_ config** ).
-   Статические функции обратного вызова на клиенте.
-   Атрибут [**Async**](async.md) .
-   [**\_ кавычки cpp**](cpp-quote.md) и **\# pragma MIDL \_ echo**.
-   типы, константы и строки для расширенных символов типа [**WCHAR \_ t**](wchar-t.md) .
-   инициализация [**перечисления**](enum.md) (разреженные перечислители).
-   Спецификация размера только для [**выхода**](out-idl.md).
-   Смешанные указатели размеров и размеры массивов.
-   Выражения, используемые для описателей размера и дискриминатора.
-   Явные параметры обработки в любой из позиций в списке аргументов. В режиме **/ОСФ** компилятор MIDL ищет явный маркер привязки в качестве первого параметра. Если первый параметр не является дескриптором привязки и указаны один или несколько дескрипторов контекста, то в качестве дескриптора привязки используется крайний левый дескриптор контекста. Если первый параметр не является дескриптором и дескрипторы контекста отсутствуют, процедура использует неявную привязку с помощью [**неявного \_ дескриптора**](implicit-handle.md) или [**автоматической \_ обработки**](auto-handle.md)атрибута ACF.
-   Наследование типа атрибута Pointer. Использование DCE не допускает указателей без атрибутов. Поэтому в режиме **/ОСФ** каждый IDL-файл должен определять атрибуты для своих указателей. Если какой-либо указатель не имеет явного атрибута, IDL-файл должен иметь спецификацию [**указателя \_ по умолчанию**](pointer-default.md) , чтобы установить тип указателя.
-   Несколько интерфейсов в IDL-файле.
-   Определения за пределами блока Interface.
-   Введите квалификаторы, такие как **FAR** и **STDCALL**.
-   Пропуск атрибутов направления.

Следующие расширения языка C/C++ недоступны при компиляции с параметром **/ОСФ** :

-   Битовые поля в структурах и объединениях.
-   Однострочные комментарии разделяются двумя символами косой черты (//).
-   Внешние объявления.
-   Процедуры с многоточием в списке параметров.
-   Тип [**int**](int.md).
-   Введите **void \** _ (за исключением атрибута [_ *context \_ Handle* *](context-handle.md) ).
-   Квалификаторы типов, включая форму с префиксом, совместимым с ANSI, содержат два символа подчеркивания: **\_ \_ cdecl**, **cdecl**, [**const**](const.md), **const**, **\_ \_ Export** **, Export,** **\_ \_ дальнее**, **далеко**, **\_ \_ лоаддс**, **лоаддс**, **\_ \_ NEAR**, **NEAR**, **\_ \_ Pascal**, **Pascal**, **\_ \_ STDCALL**, **STDCALL**, **\_ \_ volatile** и **volatile**.
-   Любая \# [**директива pragma**](pragma.md) warning или **\# pragma** Comment.
-   Сериализация типов.
-   Тип данных [**\_ \_ int3264**](--int3264.md) .
-   Параметр [**/протокол**](-protocol.md) и синтаксис инструкции ndr64.

## <a name="examples"></a>Примеры

**MIDL/ОСФ filename. idl**

**MIDL/ОСФ/АПП \_ config имя_файла. idl**
