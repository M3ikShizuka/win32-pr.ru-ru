# <a name="nocode-attribute"></a>атрибут "без кода"

Атрибут InAttribute используется в заголовках ACF или с отдельными функциями для предотвращения создания кода клиентской заглушки. **\[ \]**

``` syntax
[ 
    nocode 
    [ , ACF-interface-attributes ] 
] 
interface interface-name
{
  [ include filename-list ; ]
  [ typedef [type-attribute-list] typename; ] 
  [ [ nocode [ , ACF-function-attributes ] ] function-name (
        [ ACF-parameter-attributes ] parameter-name ;
        ...);
  ]
    ...
}
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*Атрибуты ACF-Interface-* 
</dt> <dd>

Указывает список из одного или нескольких атрибутов, которые применяются к интерфейсу в целом. Допустимые атрибуты включают в себя либо **\[** [**автоматическую \_ обработку**](auto-handle.md) **\]** , либо **\[** [**неявный \_**](implicit-handle.md) **\]** **\[** [](code.md) **\]** **\[ \]** маркер, либо код или код. При наличии двух или более атрибутов интерфейса они должны быть разделены запятыми.

</dd> <dt>

*имя интерфейса* 
</dt> <dd>

Указывает имя интерфейса. В режиме совместимости с DCE имя интерфейса должно совпадать с именем интерфейса, указанного в IDL-файле. При использовании параметра компилятора MIDL [**/АКФ**](-acf.md)имя интерфейса в ACF и имя интерфейса в IDL-файле могут отличаться.

</dd> <dt>

*список имен файлов* 
</dt> <dd>

Указывает список из одного или нескольких имен файлов заголовков языка C, разделенных запятыми. Необходимо указать полное имя файла, включая расширение.

</dd> <dt>

*Type-Attribute-List* 
</dt> <dd>

Задает список из одного или нескольких атрибутов, разделенных запятыми, которые применяются к указанному типу. Допустимые атрибуты типа включают **\[** [**выделение**](allocate.md) **\]** .

</dd> <dt>

*имени* 
</dt> <dd>

Указывает тип, определенный в IDL-файле. Атрибуты типа в ACF могут применяться только к типам, ранее определенным в IDL-файле.

</dd> <dt>

*ACF-Function-Attributes* 
</dt> <dd>

Задает атрибуты, применяемые к функции в целом, например **\[** [**\_ состояние**](comm-status.md)связи **\]** . Атрибуты функции заключаются в квадратные скобки. Несколько атрибутов функции разделяются запятыми.

</dd> <dt>

*имя функции* 
</dt> <dd>

Указывает имя функции, определенное в IDL-файле.

</dd> <dt>

*ACF-Parameter-Attributes* 
</dt> <dd>

Указывает атрибуты ACF, которые применяются к параметру. Обратите внимание, что к параметру можно применить ноль или более атрибутов. Несколько атрибутов параметров разделяются запятыми. Атрибуты параметра ACF заключаются в квадратные скобки.

</dd> <dt>

*имя параметра* 
</dt> <dd>

Задает параметр функции, определенный в IDL-файле. Каждый параметр функции должен быть указан в одной и той же последовательности и использовать те же имена, что и в IDL-файле.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Атрибут InAttribute может присутствовать в заголовке ACF, или его можно применить к отдельной функции. **\[ \]**

Если атрибут InAttribute присутствует в заголовке ACF, клиентский код заглушки не создается для какой-либо удаленной функции, если он не имеет атрибута функции Code **. \[ \]** **\[** [](code.md) **\]** Можно **\[ \] переопределить атрибут** кода в заголовке для отдельной функции, указав атрибут **\[ Code \]** в качестве атрибута функции.

Если атрибут " **\[ некод \]** " присутствует в списке атрибутов функции, код клиентской заглушки для функции не создается.

Код заглушки клиента не создается, если:

-   Заголовок ACF **\[ \] содержит атрибут InAttribute** .
-   К функции применяется атрибут «без **\[ кода \]** ».
-   **\[** [**Локальный**](local.md) **\]** атрибут применяется к функции в файле интерфейса.

**\[** [](code.md) **\]** В списке атрибутов **\[ \]** функции может присутствовать либо код, либо код ".

При создании заглушек сервера атрибут с **\[ кодом \]** не учитывается. Его нельзя применить при создании заглушек сервера в режиме совместимости с DCE.
