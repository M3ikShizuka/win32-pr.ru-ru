# <a name="midl-binding-handles"></a>Дескрипторы привязки MIDL

Дескрипторы привязки — это объекты данных, представляющие привязку между клиентом и сервером.

MIDL поддерживает базовый тип [**маркера \_ t**](handle-t.md). Дескрипторы этого типа называются "примитивными дескрипторами".

Вы можете определить собственные типы обработчиков с помощью атрибута **\[ Handle \]** . Дескрипторы, определенные таким образом, называются "определяемыми пользователем", "настроенными" или "универсальными" дескрипторами.

Можно также определить обработчик, который хранит сведения о состоянии с помощью **\[** [**атрибута \_ Handle контекста**](context-handle.md) **\]** . Дескрипторы, определенные таким образом, называются дескрипторами "контекста".

Если сведения о состоянии не требуются и вы не намерены вызывать библиотеки времени выполнения RPC для управления маркером, вы можете запросить, чтобы библиотеки времени выполнения предработали автоматическую привязку. Это делается с помощью **\[** [**\_ автомаркера**](auto-handle.md)ключевого слова ACF **\]** .

Можно указать глобальную переменную в качестве маркера привязки с помощью **\[** [**неявного \_ обработчика**](implicit-handle.md)ключевого слова ACF **\]** . Ключевое слово **\[ explicit \_ Handle \]** используется, чтобы указать, что каждая Удаленная функция имеет явно заданный обработчик.

Дополнительные сведения см. в разделе [Binding and Handles](/windows/desktop/Rpc/binding-and-handles).
