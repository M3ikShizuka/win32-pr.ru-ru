# <a name="n-switch"></a>параметр/n

Параметр **/n** указывает глубину композиции для создания файлов метаданных.

``` syntax
mdmerge /n namespace_depth
```

## <a name="switch-options"></a>Параметры коммутатора

<dl> <dt>

*Глубина пространства имен \_* 
</dt> <dd>

Указывает глубину пространства имен, которая должна быть составляться в один файл метаданных.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Ниже приведены возможные форматы значений, которые можно указать с помощью параметра **/n** .



| Формат значения                   | Описание                                                                     |
|--------------------------------|---------------------------------------------------------------------------------|
| Int32 > 0                   | Сформируйте все типы на глубине пространства имен, указанной в параметре.               |
| -1                             | Составьте все типы в один IDL-файл на пространство имен.                              |
| <namespace>: Int32 > 0 | Составьте все типы с совпадающим пространством имен на глубине, указанной в параметре. |
| <namespace>:-1           | Составьте все типы с совпадающим пространством имен в один файл для каждого пространства имен.          |

Ниже приведены правила обработки нескольких экземпляров параметра **/n** .

-   Наиболее конкретный экземпляр имеет приоритет. Например, если указать **– н:а.б.к: 4 – н:а.б: 5**, мдмерже разрешает A. b. c. D на уровне 4, поскольку A. B. c является более конкретным, чем а.б. A. B. E. F разрешается глубиной 5, так как она соответствует A. B, но не А.Б.К.
-   Последний экземпляр имеет приоритет. Например, если указать **– n:5 – n:2**, типы будут состоять на уровне 2.
-   Применяются оба правила. Если указать – Н:а.б.к: 4 – Н:а.б.к: 1, пространство имен A. B. C будет состоять на уровне 1.
