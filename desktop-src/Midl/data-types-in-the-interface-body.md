# <a name="data-types-in-the-interface-body"></a>Типы данных в теле интерфейса

Тело интерфейса, заключенное в фигурные скобки ({}), содержит типы данных, которые будут использоваться в удаленных вызовах процедур и прототипах для функций, которые будут выполняться удаленно. Тело интерфейса может содержать операции импорта, директивы pragma, объявления констант, объявления типов и объявления функций. Кроме режима совместимости с использование, компилятор MIDL также допускает неявные объявления в форме определений переменных.

Обратите внимание, что спецификация использование-DCE для интерфейсов RPC не позволяет использовать несколько интерфейсов в одном IDL-файле. Таким образом, если компиляция выполняется в режиме совместимости с использование ( [**/ОСФ**](-osf.md)) MIDL, файл IDL может содержать только один интерфейс.

Дополнительные сведения об использовании компилятора MIDL для создания библиотек типов см. в разделе [Создание библиотеки типов с помощью MIDL](generating-a-type-library-with-midl-2.md).

В Microsoft RPC файл IDL может содержать несколько интерфейсов, и эти интерфейсы могут быть объявлены прямо (в IDL-файле, определяющем их). Пример:

``` syntax
interface ITwo; //forward declaration
interface IOne 
{
...uses ITwo...
}
interface ITwo 
{
...uses IOne...
}
```
