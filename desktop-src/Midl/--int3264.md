# <a name="__int3264-attribute"></a>\_\_атрибут int3264

Ключевое слово **\_ \_ int3264** указывает целочисленный тип, который имеет следующие свойства:

-   Это 32-разрядная версия на 32-разрядных платформах
-   Это 64-разрядная версия на 64-разрядных платформах
-   Для обратной совместимости это 32-разрядное сетевое соединение. Он усекается на отправляющей стороне и расширяется соответствующим образом (подписанный или неподписанный) на принимающей стороне.

``` syntax
[ signed | unsigned ] __int3264 [ int ] declarator-list;
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*декларатор-список* 
</dt> <dd>

Указывает один или несколько стандартных деклараторов C, таких как идентификаторы, деклараторы указателей и деклараторы массивов. (Деклараторы функций и объявления битовых полей не допускаются в структурах, передаваемых в удаленных вызовах процедур. Эти деклараторы разрешены в структурах, которые не передаются.) Несколько деклараторов разделяются запятыми.

</dd> </dl>

## <a name="remarks"></a>Remarks

Этот тип был введен специально для упрощения переноса 32-разрядных интерфейсов в 64-bit. Он обычно полезен только в случаях, когда существующие интерфейсы отправляют указатели как целые числа или используют аналогичные методы обхода. Корпорация Майкрософт не рекомендует использовать этот тип (или другие открытые типы, производные от него, такие как **int \_ ptr**, **DWORD \_ ptr**), где это можно избежать. В частности, эти типы нельзя использовать в интерфейсах OLE Automation и библиотеках типов. Использовать этот тип или производные от него типы в новых интерфейсах не рекомендуется. Дополнительные сведения об этих проблемах, а также комментарии и рекомендации по другим 64-разрядным вопросам см. в язык MIDL (MIDL): 64-разрядное руководство по переносу.

\_ \_ Ключевому слову **int3264** может предшествовать либо ключевое слово с подписью, либо ключевое слово без знака. Ключевое слово int является необязательным и может быть опущено. Для компилятора MIDL \_ \_ **int3264** целое число подписывается по умолчанию и является синонимом подписанного \_ \_ int3264 int.

Целочисленный тип **\_ \_ int3264** является одним из базовых типов языка IDL. Тип целого числа **\_ \_ int3264** может отображаться в виде спецификатора типа в объявлениях const, объявлениях typedef, общих объявлениях и деклараторах функций (в виде спецификатора возвращаемого типа функции и описателя типа параметра). Сведения о контексте, в котором отображаются описатели типов, см. в разделе IDL.
