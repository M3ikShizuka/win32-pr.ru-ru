# <a name="implicit_handle-attribute"></a>неявный \_ атрибут Handle

Атрибут **\[ неявный \_ Handle \]** ACF задает маркер, используемый для функций, которые не включают явный маркер в качестве параметра процедуры.

``` syntax
implicit_handle(handle-type handle-name)
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*Handle-Type* 
</dt> <dd>

Задает тип данных Handle, например, базовый [**тип или определяемый \_ пользователем тип Handle.**](handle-t.md)

</dd> <dt>

*имя обработчика* 
</dt> <dd>

Задает имя маркера.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Маркер, заданный **\[ неявным \_ \]** атрибутом Handle, используется различными способами в зависимости от характера процедуры. Если процедура является удаленной, то маркер будет использоваться в качестве маркера привязки для удаленного вызова. Неявный обработчик также может использоваться для установления начальной привязки для функции, использующей маркер контекста. Если процедура является процедурой сериализации, то этот маркер используется в качестве маркера сериализации, управляющего операцией. В случае сериализации типа этот маркер используется в качестве маркера сериализации для всех сериализованных типов.

Атрибут **\[ неявного \_ дескриптора \]** задает глобальную переменную, которая содержит дескриптор, используемый любой функцией, требующей неявные дескрипторы.

Неявный тип маркера привязки должен быть либо [**обработчиком \_ t**](handle-t.md) (или типом, основанным на **Handle \_ t**), либо определяемым пользователем типом Handle, указанным с помощью атрибута [**Handle**](handle.md) . Неявный обработчик сериализации должен быть типом, основанным на **Handle \_ t**.

Если неявный тип обработчика не определен в IDL-файле или в файлах, включенных и импортированных IDL-файлом для компьютера MIDL, необходимо предоставить файл, содержащий определение типа Handle, при компиляции заглушек. Используйте инструкцию [**include**](include.md) ACF, чтобы включить файл, содержащий определение типа Handle.

**\[ Неявный атрибут \_ Handle \]** может встречаться не чаще одного раза. **\[ Неявный атрибут \_ Handle \]** может происходить только в том случае, если атрибуты [**\[ Auto \_ Handle \]**](auto-handle.md) и [**\[ explicit \_ Handle \]**](explicit-handle.md) не выполняются.

## <a name="examples"></a>Примеры

``` syntax
/* ACF file */ 
[
    implicit_handle(handle_t hMyHandle)
]

{ 
    // Attribute configuration statements
}
```
