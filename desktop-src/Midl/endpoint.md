# <a name="endpoint-attribute"></a>Атрибут конечной точки

Атрибут **\[ Endpoint \]** указывает хорошо известный порт или порты (конечные точки связи), на которых серверы интерфейса ожидают вызовов.

``` syntax
endpoint("protocol-sequence:[endpoint-port]" [ , ...] )
```

## <a name="parameters"></a>Параметры

<dl> <dt>

*протокол — последовательность* 
</dt> <dd>

Указывает строку символов, представляющую допустимые комбинации протокола RPC (например, "нкакн"), транспортного протокола (например, "TCP") и сетевого протокола (например, "IP"). Список допустимых последовательностей протоколов см. в разделе [константы последовательности протоколов](/windows/desktop/Rpc/protocol-sequence-constants).

</dd> <dt>

*Порт конечной точки* 
</dt> <dd>

Задает строку, которая представляет обозначение конечной точки для указанного семейства протоколов. Синтаксис строки порта зависит от каждой последовательности протоколов.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Атрибут **\[ Endpoint \]** указывает семейство транспортов, например протокол TCP/IP, ориентированный на соединение по протоколу NetBIOS или именованный протокол, ориентированный на подключение. Использование атрибута **\[ Endpoint \]** согласуется с другими методами добавления конечной точки и не предоставляет дополнительные или специальные службы для конечной точки; он просто предоставляет ярлык для вызова API.

> [!Note]  
> Указание конечной точки в. Определение интерфейса IDL не ограничивает доступ к интерфейсу для указанной конечной точки. Добавление конечной точки в. Определение интерфейса IDL позволяет вызывать интерфейс через любую конечную точку в этом процессе и позволяет использовать конечную точку для вызова других интерфейсов в этом процессе.

 

Значение *последовательности протоколов* определяет допустимые значения для *порта конечной точки*. Компилятор MIDL проверяет только общий синтаксис для записи *порта конечной точки* . Ошибки спецификации порта выводятся библиотеками времени выполнения. Сведения о допустимых значениях для каждой последовательности протоколов см. в разделе [константы последовательности протоколов](/windows/desktop/Rpc/protocol-sequence-constants).

Следующие последовательности протоколов, заданные DCE, не поддерживаются компилятором MIDL, который предоставляется с Microsoft RPC: **нкакн \_ OSI \_ ДНК** и **нкадг \_ DDS**.

Убедитесь, что в конечных точках правильно заключены символы обратной косой черты. Эта ошибка обычно возникает, когда конечная точка является именованным каналом.

Сведения о конечной точке, указанные в IDL-файле, используются функциями [**рпксерверусепротсекиф**](/windows/desktop/api/rpcdce/nf-rpcdce-rpcserveruseprotseqif) и [**РПКСЕРВЕРУСЕАЛЛПРОТСЕКСИФ**](/windows/desktop/api/rpcdce/nf-rpcdce-rpcserveruseallprotseqsif)среды выполнения RPC.

## <a name="examples"></a>Примеры

``` syntax
endpoint("ncacn_np:[\\pipe\\rainier]") 

endpoint("ncacn_ip_tcp:[1044]", "ncacn_np:[\\pipe\\shasta]")
```
