# <a name="error-switch"></a>/Error, параметр

Параметр **/Error** определяет типы проверки ошибок, которые созданные заглушки будут выполнять во время выполнения.

> [!Note]  
> Эта функция устарела и больше не поддерживается. Рекомендуется использовать параметр [**/robust**](-robust.md) .

 

``` syntax
midl /error { allocation | stub_data | ref | bounds_check | none | all }
```

## <a name="remarks"></a>Комментарии

Параметр **/Error** выбирает число проверок ошибок, которые будут выполнены для созданных файлов заглушек. Начиная с MIDL версии 5,0, значение по умолчанию — **/Error ALL**.

Проверяемые ошибки перечисления (по умолчанию во всех версиях MIDL) являются ошибками усечения, которые возникают при преобразовании между типами **Long enum** (32-разрядные целые числа) и **короткими типами enum** (представлением сетевых [**данных) и**](enum.md)числом идентификаторов в перечислении, превышающим 32 767.

Проверка ошибок доступа к памяти (также по умолчанию во всех версиях MIDL) предназначена для указателей, которые выходят за пределы буфера в коде маршалирования и для согласованных массивов, размер которых меньше нуля. Используйте флаг **\_ проверки границ/Error** , чтобы проверить наличие других недопустимых границ массива.

При указании **распределения/Error** заглушки включают код, вызывающий исключение, когда [**\_ пользователь MIDL \_ выделил**](midl-user-allocate-1.md) возвращает 0.

Параметр **\_ данных заглушки/Error** предотвращает сбой сервера в данных клиента во время распаковки, что обеспечивает более надежный метод обработки операции распаковки.

Начиная с Windows 2000, базовый модуль маршалирования отчета о выходе выполнения выполняет большинство этих проверок. Это означает, что при использовании одного из полностью интерпретируемых режимов ([**/Oi**](-oi.md), **/OIF**) создания заглушки выбор разных параметров проверки на наличие ошибок не влияет на производительность.
