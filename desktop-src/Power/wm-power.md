# <a name="wm_power-message"></a>\_Сообщение о питании WM

Уведомляет приложения о том, что система, как правило, является персональным компьютером с питанием от аккумулятора, перейдет в режим приостановки.

> [!Note]  
> Сообщение **\_ Power WM** устарело. он предоставляется только для обеспечения совместимости с 16-разрядными приложениями на основе Windows. Приложения должны использовать сообщение [**WM \_ поверброадкаст**](wm-powerbroadcast.md) .

 

Окно получает это сообщение через функцию **WindowProc** .


```C++
LRESULT CALLBACK WindowProc
  HWND   hwnd,    // handle to window
  UINT   uMsg,    // WM_POWER
  WPARAM wParam,  // power-event notification
  LPARAM lParam   // not used
); 
```

## <a name="return-value"></a>Возвращаемое значение

Значение, возвращаемое приложением, зависит от значения параметра *wParam* . Если параметр *wParam* равен **PWR \_ суспендрекуест**, возвращаемое значение — **PWR \_ завершается сбоем** , чтобы предотвратить переход системы в состояние SUSPENDED; в противном случае — **PWR \_ ОК**. Если параметр *wParam* равен **PWR \_ суспендресуме** или **PWR \_ критикалресуме**, возвращаемое значение равно нулю.

## <a name="remarks"></a>Remarks

Это сообщение передается только в приложение, которое работает в системе, которая соответствует спецификации расширенного управления питанием (APM) Basic (BIOS). Сообщение транслируется драйвером управления питанием в каждое окно, возвращаемое функцией **EnumWindows** .

Режим приостановки — это состояние, в котором происходит наибольшее количество энергосбережения, но сохраняются все операционные данные и параметры. Содержимое памяти произвольного доступа (ОЗУ) сохраняется, но многие устройства, скорее всего, будут отключены.
