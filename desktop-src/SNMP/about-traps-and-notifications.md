# <a name="about-traps-and-notifications"></a>О ловушках и уведомлениях

Одним из типов сообщений, которые приложение агента SNMP может отправить в приложение WinSNMP, является асинхронное сообщение, которое информирует приложение о значительных событиях. Примером значительного события является ситуация, когда сетевая связь завершается или происходит сбой проверки подлинности.

Эти типы сообщений называются ловушками в версии с и уведомлениями в разделе SNMPv2C. Реализация Microsoft WinSNMP всегда преобразует ловушки в формат SNMPv2C, как определено в RFC 1908.

При вызове функции [**снмпкреатепду**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpcreatepdu) для создания PDU ловушки можно создать только PDU ловушки SNMPv2c. Единственный тип PDU ловушки, который можно обновить с помощью вызова функции [**снмпсетпдудата**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpsetpdudata) , — это PDU ловушки SNMPv2c.
