# <a name="setting-the-entity-and-context-translation-mode"></a>Настройка режима преобразования объектов и контекста

В приложении WinSNMP можно указать интерпретацию и преобразование параметров сущности и контекста, установив режим преобразования сущностей и контекста. Реализация Microsoft WinSNMP сохраняет режим в базе данных.

Параметр режима преобразования сущностей и контекста определяет способ интерпретации входных строк функцией [**снмпстртоентити**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpstrtoentity) и функцией [**снмпстртоконтекст**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpstrtocontext) . Параметр также определяет тип выходной строки, возвращаемой функциями [**снмпентититостр**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpentitytostr) и [**снмпконтексттостр**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpcontexttostr) . Дополнительные сведения см. [в разделе Поддержка строк адресов IPX в WinSNMP](support-for-ipx-address-strings-in-winsnmp.md).

Реализация возвращает текущий режим преобразования сущности и контекста по умолчанию в параметре *нтранслатемоде* функции [**снмпстартуп**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpstartup) . Чтобы получить текущий режим преобразования сущности и контекста для реализации, приложение может вызвать функцию [**снмпжеттранслатемоде**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpgettranslatemode) в любое время.

Далее следуют допустимые режимы преобразования сущностей и контекста.

| Режим                      | Значение                                                                                                                                                                                                                                                                                   |
|---------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| СНМПАПИ \_ переведено       | Реализация использует свою базу данных для преобразования понятных для пользователей имен для сущностей SNMP и управляемых объектов. Реализация преобразует их в свои компоненты SNMPv2C или.                                                                                                  |
| СНМПАПИ \_ НЕпереведенный \_ v1 | Реализация интерпретирует параметры объекта SNMP как литеральные адреса транспорта SNMP, а контекстные параметры — как литеральные строки сообщества SNMP. Для целевых сущностей SNMPv2 реализация создает исходящие SNMP-сообщения, которые содержат нулевое значение в поле Version. |
| СНМПАПИ \_ НЕпереведенный \_ v2 | Реализация интерпретирует параметры сущности SNMP как адреса транспорта SNMP и контекстные параметры как литеральные строки сообщества SNMP. Для целевых сущностей SNMPv2 реализация создает исходящие SNMP-сообщения, которые содержат значение 1 в поле Version.            |



 

Реализация пытается связать ресурсы в своей базе данных с литеральным адресом транспорта сущности управления.

Чтобы изменить режим преобразования сущности и контекста, приложение WinSNMP должно вызывать функцию [**снмпсеттранслатемоде**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpsettranslatemode) . Если запрошенный режим преобразования является недопустимым, функция завершается ошибкой, а [**снмпжетластеррор**](/windows/desktop/api/Winsnmp/nf-winsnmp-snmpgetlasterror) возвращает \_ Недопустимый режим снмпапи кода ошибки \_ .
