# <a name="the-winsnmp-model"></a>Модель WinSNMP

В основе WinSNMP-совместимой модели имеются следующие основные компоненты.

-   Общедоступное приложение для управления сетью SNMP, то есть *приложение* , совместимое с WinSNMP
-   Слой функции WinSNMP
-   Поставщик службы SNMP с поддержкой WinSNMP, то есть *Реализация* , совместимая с WinSNMP

Приложения управления сетью, которые должны передавать SNMP-сообщения, эффективно работают в среде, управляемой событиями. Это связано с тем, что SNMP является протоколом на основе датаграмм или без подключения между удаленными сущностями, которые не устанавливают виртуальные цепи.

так как приложения Microsoft Windows также управляются событиями, в WinSNMP используется модель программирования, в которой получение и обработка асинхронных приложений управления дисками *сообщений с сообщениями* . Асинхронное событие сообщения может быть запросом операции SNMP приложением диспетчера или ответом на запрос, выполняемым приложением агента.

SNMP отправляет запросы и ответы в виде SNMP-сообщений. SNMP-сообщение — это единица данных протокола SNMP (PDU), а также дополнительные элементы заголовка сообщения, определенные соответствующим RFC. Дополнительные сведения см. в статьях [об SNMP-сообщениях](about-snmp-messages.md), [работе со списками привязок переменных](working-with-variable-binding-lists.md) и [работе с единицами данных протокола](working-with-protocol-data-units.md).
