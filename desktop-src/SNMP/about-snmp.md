# <a name="about-snmp"></a>О SNMP

\[Протокол SNMP доступен для использования в операционных системах, указанных в разделе требования. В последующих версиях он может быть изменен или недоступен. вместо этого используйте [служба удаленного управления Windows](/windows/desktop/WinRM/portal), который является реализацией Microsoft WS-Man.\]

Протокол SNMP использует распределенную архитектуру, состоящую из руководителей и агентов. Агент — это приложение SNMP, которое реагирует на запросы от приложений диспетчера SNMP. Агент SNMP отвечает за получение и обновление сведений о локальном управлении на основе запросов диспетчера SNMP. Агент также сообщает зарегистрированным диспетчерам о возникновении значительных событий или ловушек. Диспетчер — это приложение SNMP, которое создает запросы к приложениям агента SNMP и получает ловушки от приложений агента SNMP.

на компьютерах под управлением Microsoft Windows XP/Windows 2000/Windows NT агент snmp реализуется службой snmp (SNMP.EXE). Диспетчер SNMP обычно является консольным приложением управления SNMP стороннего производителя. Консольное приложение управления не требуется запускать на том же узле, что и агент SNMP. Чтобы использовать сведения, предоставляемые службой Microsoft SNMP, необходимо по крайней мере одно консольное приложение управления SNMP. Система включает библиотеки, поддерживающие консольные приложения управления SNMP, но в настоящее время она не включает в себя консольное приложение управления SNMP.
