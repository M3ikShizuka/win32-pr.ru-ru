# <a name="snmp-service"></a>служба SNMP;

\[Протокол SNMP доступен для использования в операционных системах, указанных в разделе требования. В последующих версиях он может быть изменен или недоступен. Вместо этого используйте [Служба удаленного управления Windows](/windows/desktop/WinRM/portal), который является реализацией Microsoft WS-Man.\]

## <a name="purpose"></a>Назначение

Реализация протокола SNMP в Microsoft Windows используется для настройки удаленных устройств, наблюдения за производительностью сети, аудита использования сети и обнаружения сбоев сети или неправильного доступа.

> [!IMPORTANT]
> API Microsoft Windows SNMP поддерживает только версии протокола до SNMPv2C. Она не поддерживает никакие более поздние версии протокола.

 

## <a name="where-applicable"></a>Где применимо

Протокол SNMP использует распределенную архитектуру, состоящую из приложений управления и агентов. Служба SNMP реализует агент SNMP. Чтобы использовать сведения, предоставляемые службой SNMP, необходимо наличие хотя бы одного узла, на котором выполняется приложение управления SNMP. Можно использовать стороннее программное обеспечение управления SNMP или разработать собственное программное приложение для управления SNMP. Для этой цели доступны следующие API:

-   API управления SNMP — набор функций, которые можно использовать для быстрой разработки основных систем управления SNMP.
-   WinSNMP API, версия 2,0, набор функций для кодирования, декодирования, отправки и получения сообщений SNMP

Кроме того, API агента расширения SNMP определяет интерфейс между службой SNMP и библиотеками DLL агента расширения SNMP сторонних производителей. Функции API служебной программы SNMP можно использовать для упрощения обработки сообщений SNMP.

## <a name="developer-audience"></a>Аудитория разработчиков

API-интерфейсы, перечисленные в предыдущем разделе, предназначены для использования программистами C/C++. Знание протоколов SNMP и SNMPv2C, а также опыта работы с сетевыми и сетевыми принципами управления, являются обязательными.
