# <a name="object-identifiers-snmp"></a>Идентификаторы объектов (SNMP)

Идентификатор SNMP-объекта уникальным образом именует объект и определяет его расположение в древовидной структуре управляющей базы данных (MIB). Идентификаторы объектов — это не зависящие от приложения синтаксические нотации одного (ASN. 1) типов данных, состоящие из последовательности неотрицательных целых чисел или субидентификаторов. Идентификаторы объектов должны иметь как минимум два подидентификатора и не должны превышать 128 подидентификаторов.

В среде программирования WinSNMP используется структура [**смиоид**](/windows/desktop/api/Winsnmp/ns-winsnmp-smioid) для управления идентификаторами объектов. Формат массива идентификаторов объектов в структуре **смиоид** является одним подидентификатором для каждого элемента массива.

Строковое представление числового значения идентификатора объекта разделяет его вложенные идентификаторы точками. Например, "1.2.3.4.5.6".

Дополнительные сведения см. [в статье база данных SNMP Management Base (MIB)](the-snmp-management-information-base-mib-.md) и соответствующие RFC.
