# <a name="exportsystemdefinition-method-of-the-msvm_virtualsystemmanagementservice-class"></a>Метод Експортсистемдефинитион \_ класса Виртуалсистемманажементсервице мсвм

Экспортирует виртуальную машину или моментальный снимок виртуальной машины в файл. Виртуальная машина должна находиться в состоянии "отключено" или "сохранено" для экспорта. Виртуальная машина, связанные с ней параметры конфигурации и связанные с ней параметры ресурсов будут сохранены в результирующем файле.

## <a name="syntax"></a>Синтаксис


```mof
uint32 ExportSystemDefinition(
  [in]  CIM_ComputerSystem REF ComputerSystem,
  [in]  string                 ExportDirectory,
  [in]  string                 ExportSettingData,
  [out] CIM_ConcreteJob    REF Job
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*ComputerSystem* \[ окне\]
</dt> <dd>

Ссылка на систему [**CIM, \_**](/windows/desktop/CIMWin32Prov/cim-computersystem) представляющая экспортируемую виртуальную машину.

</dd> <dt>

*Експортдиректори* \[ окне\]
</dt> <dd>

Полный путь к каталогу, в который должна быть экспортирована виртуальная машина. Если свойство **креатевмекспортсубдиректори** класса [**мсвм \_ Виртуалсистемекспортсеттингдата**](msvm-virtualsystemexportsettingdata.md) в параметре *експортсеттингдата* имеет значение **true**, то этот каталог можно повторно использовать для экспорта нескольких виртуальных машин, и этот метод помещает каждое определение виртуальной машины в отдельный подкаталог под этим путем.

</dd> <dt>

*Експортсеттингдата* \[ окне\]
</dt> <dd>

Встроенный экземпляр класса [**мсвм \_ виртуалсистемекспортсеттингдата**](msvm-virtualsystemexportsettingdata.md) , представляющий параметры для операции экспорта.

</dd> <dt>

*Задание* \[ заполняет\]
</dt> <dd>

Если операция выполняется асинхронно, этот метод возвратит 4096, и этот параметр будет содержать ссылку на объект, производный от [**CIM \_ конкретежоб**](/previous-versions//cc136808(v=vs.85)).

</dd> </dl>
