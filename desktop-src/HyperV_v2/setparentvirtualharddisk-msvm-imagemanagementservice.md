# <a name="setparentvirtualharddisk-method-of-the-msvm_imagemanagementservice-class"></a>Метод Сетпарентвиртуалхарддиск \_ класса) мсвм

Обновляет родительский объект для указанных конечных и дочерних файлов виртуального жесткого диска. Ограничения на использование для этого метода см. в разделе Примечания.

## <a name="syntax"></a>Синтаксис


```mof
uint32 SetParentVirtualHardDisk(
  [in]  string              ChildPath,
  [in]  string              ParentPath,
  [in]  string              LeafPath,
  [in]  boolean             IgnoreIDMismatch,
  [out] CIM_ConcreteJob REF Job
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*ChildPath* \[ окне\]
</dt> <dd>

Полный путь, указывающий расположение файла дочернего виртуального жесткого диска.

</dd> <dt>

*Парентпас* \[ окне\]
</dt> <dd>

Полный путь, указывающий расположение файла родительского виртуального жесткого диска.

</dd> <dt>

*Леафпас* \[ окне\]
</dt> <dd>

Полный путь, указывающий расположение конечного файла виртуального жесткого диска. Параметр может иметь **значение NULL** , если виртуальный жесткий диск находится в автономном режиме, но должен быть указан, если используется виртуальный жесткий диск.

</dd> <dt>

*Игнореидмисматч* \[ окне\]
</dt> <dd>

Указывает, следует ли принудительно задавать родительский диск, если идентификаторы виртуальных дисков не совпадают. Этот параметр следует использовать с осторожностью, так как если новый родительский виртуальный жесткий диск не идентичен исходному родителю, может произойти повреждение данных.

</dd> <dt>

*Задание* \[ заполняет\]
</dt> <dd>

Если операция выполняется асинхронно, этот метод возвратит 4096, и этот параметр будет содержать ссылку на объект, производный от [**CIM \_ конкретежоб**](/previous-versions//cc136808(v=vs.85)).

</dd> </dl>

## <a name="remarks"></a>Комментарии

С этим методом можно использовать только следующие типы виртуальных жестких дисков:

-   Разностный виртуальный жесткий диск
-   Разностные VHDX

Доступ к классу [**\_ ) мсвм**](msvm-imagemanagementservice.md) может быть ограничен фильтром контроля учетных записей. Дополнительные сведения см. в разделе [Управление учетными записями пользователей и инструментарий WMI](/windows/desktop/WmiSdk/user-account-control-and-wmi).
