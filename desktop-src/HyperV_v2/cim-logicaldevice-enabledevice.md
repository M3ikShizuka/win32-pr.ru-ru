# <a name="enabledevice-method-of-the-cim_logicaldevice-class"></a>Метод Енабледевице \_ класса CIM

Метод Енабледевице является устаревшим вместо более общего метода RequestStateChange, который непосредственно пересекается с функциями, предоставляемыми этим методом.

Запрашивает включение объектного ключа ("Enabled", "input Parameter = TRUE") или Disabled (= FALSE). В случае успешного выполнения свойства Статусинфо/EnabledState устройства должны отражать требуемое состояние (включено/отключено). Обратите внимание, что функция этого метода пересекается со свойством RequestedState. Объект RequestedState был добавлен в модель для обслуживания записи (т. е. сохраненного значения) последнего запроса состояния. При вызове метода Енабледевице должно быть задано свойство RequestedState соответствующим образом.

Код возврата должен быть равен 0, если запрос был успешно выполнен, 1, если запрос не поддерживается, и другое значение, если произошла ошибка. В подклассе можно указать набор возможных кодов возврата, используя квалификатор ValueMap в методе. Строки, в которые содержимое ValueMap является переведенными, могут также быть указаны в подклассе как квалификатор массива Values.

## <a name="syntax"></a>Синтаксис


```mof
uint32 EnableDevice(
  [in] boolean Enabled
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Включено* \[ окне\]
</dt> <dd>

Если значение TRUE, включите устройство, если значение равно FALSE, отключите устройство.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение 0 в случае успешного выполнения; в противном случае возвращает ошибку.
