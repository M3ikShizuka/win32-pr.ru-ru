# <a name="calculatevideomemoryrequirements-method-of-the-msvm_synth3dvideopool-class"></a>Метод Калкулатевидеомеморирекуирементс \_ класса Synth3dVideoPool мсвм

Вычисляет объем видеопамяти, необходимый для виртуальной машины RemoteFX.

## <a name="syntax"></a>Синтаксис


```mof
uint32 CalculateVideoMemoryRequirements(
  [in]  uint32 monitorResolution,
  [in]  uint32 numberOfMonitors,
  [out] uint64 requiredVideoMemory
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*monitorResolution* \[ окне\]
</dt> <dd>

Максимальное разрешение монитора для виртуальной машины. Это должно быть одно из следующих значений.



| Значение                                                                        | Значение                                           |
|------------------------------------------------------------------------------|---------------------------------------------------|
| <dl> <dt>0</dt> </dl> | Максимальное разрешение составляет 1024 768.<br/>  |
| <dl> <dt>1</dt> </dl> | Максимальное разрешение составляет 1280 1024.<br/> |
| <dl> <dt>2</dt> </dl> | Максимальное разрешение составляет 1600 1200.<br/> |
| <dl> <dt>3</dt> </dl> | Максимальное разрешение составляет 1920 1200.<br/> |



 

</dd> <dt>

*нумберофмониторс* \[ окне\]
</dt> <dd>

Максимальное число мониторов для виртуальной машины. Минимальное число мониторов — один, а максимальное зависит от максимального разрешения экрана. В следующей таблице определяется максимальное число мониторов, разрешенных для различных способов разрешения.



| Решение             | Максимальное число мониторов |
|------------------------|------------------|
| 1024 768<br/>  | 4<br/>     |
| 1280 1024<br/> | 4<br/>     |
| 1600 1200<br/> | 3<br/>     |
| 1920 1200<br/> | 2<br/>     |



 

</dd> <dt>

*рекуиредвидеомемори* \[ заполняет\]
</dt> <dd>

Получает требуемый объем видеопамяти в байтах.

</dd> </dl>

## <a name="remarks"></a>Комментарии

Этот метод обычно вызывается в системе узла, чтобы определить, достаточно ли памяти на узле для размещения виртуальной машины RemoteFX. Для этого сравните объем видеопамяти, вычисленный этим методом, с помощью свойства [**мсвм \_ Фисикалгпуинфо. аваилаблевидеомемори**](msvm-physicalgpuinfo.md) , чтобы определить, достаточно ли памяти на размещающем компьютере. Эти сведения можно использовать, чтобы определить, можно ли переместить виртуальную машину в систему узла.
