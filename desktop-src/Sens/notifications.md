# <a name="notifications-system-event-notification-service"></a>Уведомления (служба уведомлений о системных событиях)

Служба уведомлений о системных событиях позволяет приложениям, поддерживающим мобильные приложения, получать уведомления от системных событий, Сенс мониторы. Когда происходит запрошенное событие, Сенс уведомляет приложение.

Сенс может уведомлять приложения о трех классах системных событий:

-   Сетевые события TCP/IP, такие как состояние сетевого подключения TCP/IP или качество подключения.
-   События входа пользователя в систему.
-   События аккумулятора и питания от сети.

Например, приложение может подписываться на любые из следующих системных событий:

-   Установка сетевого подключения
-   Уведомление, когда можно достигнуть указанного места назначения в указанных параметрах качества подключения (КОК)
-   Компьютер переключился на питание от аккумулятора
-   Процент оставшегося аккумулятора в указанном параметре
-   Выполняются запланированные события с помощью диспетчера синхронизации

**Windows Server 2008 R2 и Windows 7:** У подписчика должно быть не более 3 минут для ответа на уведомление в интерфейсах [**исенслогон**](/windows/desktop/api/Sensevts/nn-sensevts-isenslogon) и [**ISensLogon2**](/windows/desktop/api/Sensevts/nn-sensevts-isenslogon2) . Через 3 минуты Сенс отменяет вызов подписчиков и разблокирует поток уведомлений. Если требуется длительная операция для ответа на уведомление, вернитесь из **исенслогон** или **ISensLogon2** как можно быстрее и откройте другой поток для обработки.
