# <a name="dailytrigger-object"></a>Объект Даилитригжер

Объект скрипта, представляющий триггер, который запускает задачу на основе ежедневного расписания.

## <a name="members"></a>Элементы

Объект **даилитригжер** имеет следующие типы членов:

-   [Свойства](#properties)

### <a name="properties"></a>Свойства

Объект **даилитригжер** имеет следующие свойства.



| Свойство                                                            | Тип доступа           | Описание                                                                                                                                                                                 |
|:--------------------------------------------------------------------|:----------------------|:--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**дайсинтервал**](dailytrigger-daysinterval.md)<br/>        | Чтение/запись<br/> | Возвращает или задает интервал между днями в расписании.<br/>                                                                                                                      |
| [**Активировано**](trigger-enabled.md)<br/>                       | Чтение/запись<br/> | Наследуется от объекта [**триггера**](trigger.md) . Возвращает или задает логическое значение, указывающее, включен ли триггер.<br/>                                                |
| [**ендбаундари**](trigger-endboundary.md)<br/>               | Чтение/запись<br/> | Наследуется от объекта [**триггера**](trigger.md) . Возвращает или задает дату и время деактивации триггера. Триггер не может запустить задачу после ее деактивации.<br/> |

Возвращает тип триггера.<br/>                                                                                              |



 

## <a name="remarks"></a>Комментарии

Время дня запуска задачи задается свойством [**стартбаундари**](trigger-startboundary.md) .

Интервал 1 создает ежедневное расписание. Интервал 2 создает все остальные календарные расписания и т. д.

При чтении или записи собственного XML-кода для задачи ежедневный триггер указывается с помощью элемента [**счедулебидай**](taskschedulerschema-schedulebyday-calendartriggertype-element.md) схемы планировщик задач.
