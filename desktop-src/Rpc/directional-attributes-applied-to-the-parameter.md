# <a name="directional-attributes-applied-to-the-parameter"></a>Атрибуты направления, применяемые к параметру

Атрибуты направления \[ [в](/windows/desktop/Midl/in) \] и \[ [out](/windows/desktop/Midl/out-idl) \] определяют, как клиент и сервер распределяют и освобождают память. В следующей таблице приводится сводка по применению атрибутов направления для выделения памяти.



| Атрибут Direction    | Память на клиенте                                                                                            | Память на сервере                                                                                                                                        |
|--------------------------|-------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| \[[в](/windows/desktop/Midl/in)\]       | Клиентское приложение должно выделить перед вызовом.                                                           | Заглушка сервера выделяет.                                                                                                                                  |
| \[[исходящий трафик](/windows/desktop/Midl/out-idl)\] | Заглушка клиента выделяется при возврате.                                                                            | Заглушка сервера выделяет только указатель верхнего уровня; серверное приложение должно выделить все внедренные указатели. Сервер также выделяет новые данные по мере необходимости. |
| \[**в**, **out**\]      | Клиентское приложение должно распределять начальные данные, передаваемые на сервер; Заглушка клиента выделяет дополнительные данные. | Заглушка сервера выделяет начальные данные, передаваемые от клиента; серверное приложение выделяет новые данные по мере необходимости.                                        |



 

Во всех этих случаях клиентская заглушка не освобождает память. Клиентское приложение должно освободить память до ее завершения. Заглушка сервера освобождает память при возвращении удаленного вызова процедуры (в соответствии с \[ [](/windows/desktop/Midl/allocate) \] атрибутом выделения ACF).
