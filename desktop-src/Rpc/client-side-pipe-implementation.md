# <a name="client-side-pipe-implementation"></a>Реализация Client-Sideного канала

Клиентское приложение должно реализовать следующие процедуры, которые клиентская заглушка будет вызывать во время пересылки данных:

-   Процедура извлечения (для входного канала)
-   Процедура Push (для выходного канала)
-   Процедура выделения, которая выделяет буфер для данных для перемещения

Все эти процедуры должны использовать аргументы, указанные в файле заголовка, созданном с помощью MIDL. Кроме того, клиентское приложение должно иметь переменную состояния для указания места расположения или размещения данных.

Процедура выделения может также быть простой или сложной по мере необходимости. Например, он может вернуть указатель на один и тот же буфер каждый раз, когда заглушка вызывает функцию, или может выделить отдельный объем памяти каждый раз. Если данные уже находятся в соответствующей форме (например, массив элементов канала), можно координировать процедуру выделения с помощью процедуры Pull, чтобы выделить буфер, который уже содержит данные. В этом случае процедура Pull может быть пустой процедурой.

Выделение буфера должно быть в байтах. Процедуры push и Pull, с другой стороны, управляют элементами, размер которых в байтах зависит от того, как они были определены.
