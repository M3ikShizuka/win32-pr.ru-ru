# <a name="handling-asynchronous-calls"></a>Обработка асинхронных вызовов

Подпрограммы диспетчера асинхронной функции всегда получают асинхронный обработчик в качестве первого параметра. Сервер должен отследить этот обработчик и использовать его для отправки ответа при завершении асинхронного вызова удаленной процедуры.

Если серверу необходимо прервать асинхронную RPC, он вызывает [**рпкасинкаборткалл**](/windows/desktop/api/Rpcasync/nf-rpcasync-rpcasyncabortcall). Эта функция выполняет ту же очистку на стороне сервера, что и [**рпкасинккомплетекалл**](/windows/desktop/api/Rpcasync/nf-rpcasync-rpcasynccompletecall) , и распространяет код исключения (предоставленный серверным приложением) обратно клиенту, за исключением того, что он не выполняет упаковку аргументов out.

Пример асинхронной процедуры см. [в разделе Отправка асинхронного ответа](sending-the-asynchronous-reply.md).
