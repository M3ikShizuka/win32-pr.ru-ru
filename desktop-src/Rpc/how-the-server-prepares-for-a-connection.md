# <a name="how-the-server-prepares-for-a-connection"></a>Как сервер готовится к подключению

Когда серверная программа начинает выполнение, она должна сначала зарегистрировать интерфейс или интерфейсы, которые он содержит, в библиотеке времени выполнения RPC. Затем он создает необходимые сведения о привязке. Серверная программа также должна зарегистрировать конечную точку или конечные точки, которые он прослушивает. Затем он может начать прослушивание клиентских вызовов. Этот процесс показан на следующей схеме.

![серверное приложение RPC подготавливается к подключениям клиентов](images/srvrcon.png)

В зависимости от выбранной структуры, серверное приложение может выбрать другой набор действий. в предыдущем примере и иллюстрации представлен один из способов.

В этом разделе представлены сведения о шагах, которые должен выполнить серверный процесс для подготовки к подключению. Обсуждение состоит из следующих разделов:

-   [Регистрация интерфейса](registering-the-interface.md)
-   [Обеспечение доступности сервера в сети](making-the-server-available-on-the-network.md)
-   [Регистрация конечных точек](registering-endpoints.md)
-   [Ожидание вызовов клиента](listening-for-client-calls.md)
