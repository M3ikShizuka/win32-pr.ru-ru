# <a name="remote-procedure-calls-using-rpc-over-http"></a>Удаленные вызовы процедур с помощью RPC через HTTP

Программы Интернет-браузера обычно используют протокол HTTP в качестве основного средства просмотра в Интернете. Поэтому в настоящее время протокол HTTP видит широкое использование на большинстве компьютеров. Корпорация Майкрософт расширила возможности сервера IIS для предоставления служб удаленного вызова процедур по протоколу HTTP.

Реализация Microsoft RPC-over-HTTP (RPC через HTTP) позволяет клиентам RPC безопасно и эффективно подключаться через Интернет к программам сервера RPC и выполнять удаленные вызовы процедур. Это достигается за счет использования промежуточного посредника, известного как прокси RPC-over-HTTP, или просто прокси RPC.

Прокси-сервер RPC работает на компьютере IIS. Он принимает запросы RPC, поступающие из Интернета, выполняет проверку подлинности, проверку и проверку доступа для этих запросов. Если запрос проходит все тесты, прокси-сервер RPC перенаправляет запрос серверу RPC, который выполняет фактическую обработку. При использовании RPC через HTTP клиент RPC и сервер не обмениваются данными напрямую; Вместо этого они используют прокси-сервер RPC в качестве посредника. Эта модель была выбрана по многим причинам. Дополнительные сведения см. в разделе [RPC over HTTP Security](rpc-over-http-security.md).
