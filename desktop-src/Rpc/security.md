# <a name="security-rpc"></a>Безопасность (RPC)

Благодаря увеличению использования распределенных приложений потребность в безопасной связи между клиентскими и серверными частями приложений имеет первостепенное значение. Библиотека времени выполнения удаленного вызова процедур (RPC) предоставляет стандартизированный интерфейс для служб проверки подлинности как для клиентов, так и для серверов. Службы проверки подлинности в системе узла сервера обеспечивают проверку подлинности RPC. Приложения используют удаленные вызовы процедур, прошедшие проверку подлинности, чтобы гарантировать, что все вызовы поступают от авторизованных клиентов. Они также могут гарантировать, что все ответы сервера поступают с серверов, прошедших проверку подлинности.

В этом разделе представлен обзор проверки подлинности RPC в следующих разделах:

-   [Основы безопасности RPC](rpc-security-essentials.md)
-   [Методы безопасности](security-methods.md)
