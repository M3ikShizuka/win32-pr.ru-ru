# <a name="transmit_as-and-represent_as"></a>Передать \_ как и представить \_ как

Передача \_ as и представляется \_ совместно с одним и тем же макетом, за исключением начального маркера. Этот маркер считывает, что FC \_ передает \_ as или FC \_ \_ как, но базовый код является общим.

Описание имеет следующий макет:

``` syntax
FC_TRANSMIT_AS | FC_REPRESENT_AS
flags<1>
quintuple_index<2>
presented_type_memory_size<2>
transmitted_type_buffer_size<2>
transmitted_type_offset<2>
```

Флаги<1> байт состоят из верхнего флага, а также более низкого выравнивания.

Выравнивание по сети обеспечивает выравнивание передачи для передаваемого типа. Это необходимо при определении размера буфера и использовании переданного размера типа из кода формата.

Флаг может иметь следующие флаги:

``` syntax
#define PRESENTED_TYPE_IS_ARRAY     0x10
#define PRESENTED_TYPE_ALIGN_4      0x20
#define PRESENTED_TYPE_ALIGN_8      0x40
```

ПРЕДСТАВЛЕНный \_ тип \_ является \_ флагом массива. аргумент передается как (или представляет AS) верхнего уровня как массив какого-либо значения, переданного по значению. Интерпретатор [**-Oi**](/windows/desktop/Midl/-oi) использует этот флаг для прохода по такому аргументу (который фактически является указателем в стеке, а не массивом). Два других флага также используются только в предыдущих интерпретаторах для корректного шага над представленным типом в стеке.

Индекс пятиэлементные \_<2> является индексом подпрограммы обратного вызова пятиэлементные (на самом деле это четверное) функций. Таблица обычно используется как для передачи как, так и для представления, и существует очевидное сопоставление для позиции подпрограмм, так как одна и та же служба точек входа может передавать и представлять как коды. Сопоставление происходит от \_ Local => к \_ xmit, to \_ Local => из \_ xmit, Free \_ inst => Free \_ xmit, бесплатное \_ локально => Free \_ inst.

Размер отображаемого \_ типа \_ памяти \_<2> всегда предоставляет размер для представленного/локального типа, включая Unknown — как типы.

\_Размер переданного \_ буфера типа \_<2> равен нулю, при изменении размера или фактическим фиксированном размере. Это оптимизация, которая позволяет пропускать некоторые обратные вызовы при изменении размера буфера.

Переданное \_ \_ смещение типа<2> — это обычное смещение относительно типа к передаваемой строке формата типа.
