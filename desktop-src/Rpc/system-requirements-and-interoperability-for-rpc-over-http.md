# <a name="rpc-over-http-system-requirements-interoperability"></a>Требования к системе для RPC по HTTP, взаимодействие

Microsoft RPC поддерживает RPC через HTTP, как показано в следующей таблице.



| Платформа                             | Поддерживает                       | Примечания                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|--------------------------------------|--------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Windows Server 2003                  | Клиенты, серверы и прокси-сервер RPC | Поддерживает RPC через HTTP v1 и RPC через HTTP v2 клиента и сервера. Прокси-сервер RPC поддерживает RPC через HTTP v2, когда службы IIS работают в режиме IIS 6,0. Прокси-сервер RPC поддерживает RPC через HTTP v1 и RPC через HTTP v2, когда службы IIS работают в режиме IIS 5,0. Однако запуск в режиме IIS 5,0 не рекомендуется. дополнительные сведения см. в разделе [Рекомендации развертывания RPC через HTTP](rpc-over-http-deployment-recommendations.md) . RPC через HTTP-сервер и прокси RPC могут находиться на разных компьютерах. |
| Windows XP с пакетом обновления 1 (SP1) | Клиенты и серверы            | Поддерживает RPC через HTTP v1 и RPC через HTTP v2 клиента и сервера. Не поддерживает прокси-сервер RPC.                                                                                                                                                                                                                                                                                                                                                                                         |
| Windows XP                           | Клиенты и серверы            | Поддерживает только клиент RPC через HTTP v1 и только сервер. Не поддерживает прокси-сервер RPC.                                                                                                                                                                                                                                                                                                                                                                                                         |
| Windows 2000                         | Клиенты, серверы и прокси-сервер RPC | Программа RPC через HTTP Server и RPC-прокси могут работать на разных компьютерах. Клиент RPC через HTTP, сервер и прокси-сервер RPC поддерживают RPC только через HTTP v1.                                                                                                                                                                                                                                                                                                                   |



 

Кроме того, применяются следующие требования.

-   Windows 2000 и более поздних версий требует использования IIS 4,0 или более поздней версии.
-   прокси-сервер RPC через HTTP работает только на выпусках Windows server.
-   если службы IIS выполняются на серверной версии Windows, программа rpc через HTTP server может работать на любом компьютере, на котором прокси-сервер rpc настроен для пересылки трафика. Таким образом, он может выполняться на том же компьютере, что и прокси-сервер RPC, или на другом компьютере.

Чтобы установить подключение RPC через HTTP, все клиенты RPC через HTTP, RPC через HTTP и прокси-сервер RPC должны согласовать используемую версию RPC через HTTP. Если не существует общей версии RPC по протоколу HTTP, которая поддерживает все три (клиент, сервер и прокси-сервер RPC), подключение RPC через HTTP не может быть установлено. В следующей таблице приведена сводка по взаимодействию для различных версий RPC через HTTP.



| RPC через HTTP-клиент | Прокси-сервер RPC      | RPC через HTTP-сервер | Работоспособ?                                      | Используемая версия     |
|----------------------|----------------|----------------------|---------------------------------------------|------------------|
| только v1              | только v1        | только v1              | Да, с ограничениями v1                    | RPC через HTTP v1 |
| только v1              | только v1        | Версии v1 и v2       | Да, с ограничениями v1                    | RPC через HTTP v1 |
| только v1              | Версии v1 и v2 | только v1              | Да, с ограничениями v1                    | RPC через HTTP v1 |
| только v1              | Версии v1 и v2 | Версии v1 и v2       | Да, с ограничениями v1                    | RPC через HTTP v1 |
| только v1              | только v2        | только v1              | Нет                                          |                  |
| только v1              | только v2        | Версии v1 и v2       | Нет                                          |                  |
| Версии v1 и v2       | только v1        | только v1              | Да, с ограничениями v1                    | RPC через HTTP v1 |
| Версии v1 и v2       | только v1        | Версии v1 и v2       | Да, с ограничениями v1                    | RPC через HTTP v1 |
| Версии v1 и v2       | Версии v1 и v2 | только v1              | Да, с ограничениями v1                    | RPC через HTTP v1 |
| Версии v1 и v2       | Версии v1 и v2 | Версии v1 и v2       | Да                                         | RPC через HTTP v2 |
| Версии v1 и v2       | только v2        | только v1              | Нет                                          |                  |
| Версии v1 и v2       | только v2        | Версии v1 и v2       | Да. Это рекомендуемая конфигурация. | RPC через HTTP v2 |



 

например, представьте себе клиент Windows 2000, прокси-сервер Windows Server 2003 с IIS, работающий в режиме iis 6,0, и сервер Windows 2003 RPC через HTTP server. в первой таблице на этой справочной странице показано, что Windows 2000 поддерживает только RPC через HTTP v1. эта же таблица показывает, что Windows сервер 2003 с iis, запущенный в режиме iis 6,0, поддерживает только rpc через http v2, а Windows server 2003 rpc через http server поддерживает rpc через http v1 и rpc через http v2. Этот сценарий описан в строке 6 второй таблицы на этой справочной странице, где показано, что невозможно установить подключение RPC через HTTP. Кроме того, вторая таблица показывает, что для этого сценария существуют два варианта:

-   Если безопасность и надежность не учитываются, IIS можно переключить в режим IIS 5,0, где поддерживается как RPC через HTTP v1, так и RPC через HTTP v2. Это позволит установить подключение RPC через HTTP v1.
-   обновите клиент Windows 98 до Windows XP с пакетом обновления 1 (SP1) и получите питание, безопасность и надежность подключения RPC через HTTP v2.
