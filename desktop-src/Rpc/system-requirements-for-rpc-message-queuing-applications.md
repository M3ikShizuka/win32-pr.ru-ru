# <a name="system-requirements-for-rpc-message-queuing-applications"></a>Требования к системе для приложений RPC-Message Queuing

Чтобы использовать транспорт очереди сообщений в клиентском или серверном приложении RPC, на сервере и на клиентских компьютерах должна быть установлена соответствующая Платформа операционной системы и программное обеспечение очереди сообщений.

Требования для серверных компьютеров:

-   Microsoft Windows Server 2003, Windows XP или Windows 2000 или более поздней версии.
-   SQL Server версии 6,5 или более поздней.
-   Основной контроллер предприятия или основной контроллер сайта очереди сообщений.
-   Транспортная библиотека RPC на стороне сервера (RpcMqSvr.dll).

Требования для клиентских компьютеров:

-   Microsoft Windows Server 2003, Windows XP или Windows 2000 или более поздней версии.
-   Клиент очереди сообщений Майкрософт.
-   DLL-библиотека транспорта RPC на стороне клиента (RpcMqCl.dll).

Если компоненты MSMQ установлены на клиентских и серверных компьютерах, реестры системы автоматически настраиваются для включения транспортного протокола очереди сообщений [нкадг \_ MQ](/windows/desktop/Midl/ncadg-mq) для удаленных вызовов процедур.

Чтобы создать приложение RPC-MSMQ, вам потребуется следующее:

-   Microsoft Windows Server 2003, Windows XP или Windows 2000 или более поздней версии
-   MIDL версии 3.1.76 или более поздней.
