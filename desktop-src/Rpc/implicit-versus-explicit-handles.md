# <a name="implicit-vs-explicit-handles"></a>Неявные и явные дескрипторы

Чтобы объявить маркер сериализации, используйте тип маркера-примитива [**\_ t**](/windows/desktop/Midl/handle-t). Дескрипторы сериализации могут быть явно или неявными. Укажите неявные дескрипторы в ACFе приложения с помощью атрибута **\[ неявного \_ дескриптора \]** . Компилятор MIDL создаст переменную глобального обработчика сериализации. Процедуры сериализации с неявным обработчиком используют эту глобальную переменную для доступа к допустимому контексту сериализации.

При использовании кодировки типов созданные подпрограммы, поддерживающие сериализацию определенного типа, используют глобальный неявный обработчик для доступа к контексту сериализации. Обратите внимание, что удаленным подпрограммым может потребоваться использовать неявный обработчик в качестве маркера привязки. Убедитесь, что для неявного маркера задан допустимый обработчик сериализации перед выполнением вызова сериализации.

Явный маркер указывается в качестве параметра прототипа процедуры сериализации в IDL-файле или может быть указан с помощью **\[ явного атрибута \_ Handle \]** в ACF. Параметр явного обработчика используется для установления надлежащего контекста сериализации для процедуры. Чтобы установить правильный контекст в случае сериализации типа, компилятор создает вспомогательные подпрограммы, в которых в качестве маркера сериализации используется явный параметр [**Handle \_ t**](/windows/desktop/Midl/handle-t) . При вызове процедуры сериализации или подпрограммы поддержки типа сериализации необходимо указать допустимый описатель сериализации.
