# <a name="authentication-level-constants"></a>Константы уровня проверки подлинности

Константы уровня проверки подлинности представляют уровни проверки подлинности, передаваемые различным функциям времени выполнения. Эти уровни перечислены в порядке повышения уровня проверки подлинности. Каждый новый уровень добавляет к проверке подлинности, обеспечиваемой предыдущим уровнем. Если библиотека времени выполнения RPC не поддерживает указанный уровень, она автоматически обновляется до следующего более высокого поддерживаемого уровня.



| Константа                                                                                                                                                                                                                | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="RPC_C_AUTHN_LEVEL_DEFAULT"></span><span id="rpc_c_authn_level_default"></span><dl> <dt>**\_ \_ \_ по умолчанию на уровне RPC C AUTHN \_**</dt> </dl>                    | Использует уровень проверки подлинности по умолчанию для заданной службы проверки подлинности.<br/>                                                                                                                                                                                                                                                                                                                                                    |
| <span id="RPC_C_AUTHN_LEVEL_NONE"></span><span id="rpc_c_authn_level_none"></span><dl> <dt>**\_уровень RPC \_ C \_ AUTHN \_ None**</dt> </dl>                             | Не выполняет проверку подлинности.<br/>                                                                                                                                                                                                                                                                                                                                                                                                        |
| <span id="RPC_C_AUTHN_LEVEL_CONNECT"></span><span id="rpc_c_authn_level_connect"></span><dl> <dt>**\_подключение на уровне RPC C \_ AUTHN \_ \_**</dt> </dl>                    | Проверка проходит успешно только в том случае, когда клиент устанавливает соединение с сервером.<br/>                                                                                                                                                                                                                                                                                                                                                       |
| <span id="RPC_C_AUTHN_LEVEL_CALL"></span><span id="rpc_c_authn_level_call"></span><dl> <dt>**\_ \_ \_ вызов уровня RPC C \_ AUTHN**</dt> </dl>                             | Проверка подлинности выполняется только в начале каждого удаленного вызова процедуры, когда сервер получает запрос. Не применяется к удаленным вызовам процедур, выполненным с помощью последовательностей протоколов на основе подключений (которые начинаются с префикса "нкакн"). Если последовательность протокола в маркере привязки является последовательностью протокола на основе соединения и указан этот уровень, то эта подпрограмма использует \_ \_ \_ константу PKT уровня RPC C AUTHN \_ .<br/> |
| <span id="RPC_C_AUTHN_LEVEL_PKT"></span><span id="rpc_c_authn_level_pkt"></span><dl> <dt>**\_PKT на уровне RPC C \_ AUTHN \_ \_**</dt> </dl>                                | Выполняет проверку подлинности только тех данных, которые получены от ожидаемого клиента. Не проверяет сами данные.<br/>                                                                                                                                                                                                                                                                                                                          |
| <span id="RPC_C_AUTHN_LEVEL_PKT_INTEGRITY"></span><span id="rpc_c_authn_level_pkt_integrity"></span><dl> <dt>**\_ \_ \_ \_ целостность PKT RPC C AUTHN Level \_**</dt> </dl> | Проверка подлинности и проверка того, что ни один из данных, передаваемых между клиентом и сервером, не был изменен.<br/>                                                                                                                                                                                                                                                                                                                          |
| <span id="RPC_C_AUTHN_LEVEL_PKT_PRIVACY"></span><span id="rpc_c_authn_level_pkt_privacy"></span><dl> <dt>**\_Конфиденциальность RPC C \_ AUTHN \_ Level \_ PKT \_**</dt> </dl>       | Включает все предыдущие уровни и гарантирует, что данные с открытым текстом могут быть видны только отправителю и получателю. В локальном случае это подразумевает использование безопасного канала. В удаленном случае это подразумевает шифрование значения аргумента для каждого удаленного вызова процедуры.<br/>                                                                                                                                                                 |



## <a name="remarks"></a>Remarks

Независимо от значения, заданного константой, **нкалрпк** всегда использует \_ безопасность RPC C AUTHN на \_ \_ уровне \_ \_ безопасности PKT.
