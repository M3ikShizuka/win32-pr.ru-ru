# <a name="rpc-ndr-engine-rpc"></a>Подсистема доставки оповещений RPC (RPC)

Механизмом представления данных сети (NDR) удаленного вызова процедур (RPC) является механизм упаковки компонентов RPC и DCOM. Модуль NDR обрабатывает все проблемы, связанные с заглушкой удаленного вызова. В качестве процесса маршалирование ОНД управляется кодом C из заглушек, созданных MIDL, генератором JIT-типа MIDL или заглушками, созданными другими инструментами или написанными вручную. В свою очередь, подсистема доставки ОНД управляет базовым временем выполнения (DCOM или RPC), которое взаимодействует с конкретными транспортами.

Хотя заглушки — это код на языке C, созданный с помощью MIDL, приложениям рекомендуется рассматривать заглушки как непрозрачные и настоятельно не рекомендуется изменять что-либо внутри заглушки. Поведение не определено, если эти подпрограммы NDR вызываются из контекста.
