# <a name="base-types"></a>Базовые типы

Чтобы предотвратить проблемы, связанные с типами данных, зависящими от реализации, на разных архитектурах компьютеров, MIDL определяет собственные базовые типы данных.



| Базовый тип                         | Описание                                                                                                                                                         |
|-----------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**Логическое**](/windows/desktop/Midl/boolean)       | Элемент данных, который может иметь значение **true** или **false**.                                                                                                          |
| [**byte**](/windows/desktop/Midl/byte)             | 8-разрядный элемент данных, который гарантированно передается без каких-либо изменений.                                                                                                 |
| [**char**](/windows/desktop/Midl/char-idl)         | 8-разрядный элемент данных без знака.                                                                                                                              |
| [**Дважды**](/windows/desktop/Midl/double)         | 64-разрядное число с плавающей запятой.                                                                                                                                     |
| [**сделать**](/windows/desktop/Midl/float)           | 32-разрядное число с плавающей запятой.                                                                                                                                     |
| [**Handle \_ t**](/windows/desktop/Midl/handle-t)    | Примитивный маркер, который можно использовать для привязки RPC или сериализации данных.                                                                                            |
| [**Hyper**](/windows/desktop/Midl/hyper)           | 64-разрядное целое число, которое может быть объявлено как [**со знаком**](/windows/desktop/Midl/signed) , так и [**без знака**](/windows/desktop/Midl/unsigned) , также может называться **\_ Int64**.                  |
| [**INT**](/windows/desktop/Midl/int)               | 32-разрядное целое число, которое может быть объявлено как **со знаком** , так и **без знака**.                                                                                         |
| [**\_\_int3264**](/windows/desktop/Midl/--int3264) | Ключевое слово, указывающее целочисленный тип, который имеет 32-разрядное или 64-разрядное свойство.                                                                              |
| [**поддерживаем**](/windows/desktop/Midl/long)             | Модификатор для **int** , который указывает на 32-разрядное целое число. Может быть объявлен как **со знаком** , так и **без знака**.                                                       |
| [**short**](/windows/desktop/Midl/short)           | 16-разрядное целое число, которое может быть объявлено как **со знаком** , так и **без знака**.                                                                                         |
| [**значительные**](/windows/desktop/Midl/small)           | Модификатор для **int** , указывающий 8-разрядное целое число. Может быть объявлен как **со знаком** , так и **без знака**.                                                       |
| [**WCHAR \_ t**](/windows/desktop/Midl/wchar-t)      | Тип расширенных символов, поддерживаемый в качестве расширения Майкрософт для IDL. Поэтому этот тип недоступен при компиляции с помощью параметра [ / **Использование**](/windows/desktop/Midl/-osf) . |



 

Файл заголовка Рпкндр. h предоставляет определения для большинства из этих базовых типов данных. Ключевое слово **int** распознается и передается на 32-разрядных платформах. На 16-разрядных платформах тип данных **int** требует модификатора, например **Short** или **Long**, для указания его длины.

Хотя **значение \* \* void** распознается как универсальный тип указателя стандартом ANSI C, MIDL ограничит его использование. Каждый указатель, используемый в операции удаленной или сериализации, должен указывать на базовые типы или типы, созданные из базовых типов. (Существует исключение: дескрипторы контекста определены как типы **void** . Дополнительные сведения см. в разделе [дескрипторы контекста](context-handles.md).)
