# <a name="authentication-levels"></a>Уровни проверки подлинности

Microsoft RPC обеспечивает несколько уровней проверки подлинности. В зависимости от уровня проверки подлинности источник трафика (участника безопасности, отправившего трафик) может быть проверен при установке соединения, когда клиент запускает новый удаленный вызов процедуры или при обмене пакетами между клиентом и сервером.

Даже при проверке отправителя трафика безопасность остается ненадежной, так как такая проверка не гарантирует, что пакет не был изменен или был поврежден. Он только проверяет, получен ли пакет от данного участника. Для повышения безопасности распределенные приложения могут задать библиотеку времени выполнения RPC, чтобы убедиться в том, что ни один из данных, которыми обмениваются клиент и сервер, не изменяется. Кроме того, с помощью библиотеки RPC можно зашифровать содержимое каждого пакета перед его отправкой. Как правило, приложения, которые хотят защитить свой трафик, должны использовать только два последних уровня — целостность и конфиденциальность.

Имейте в виду, что для более высоких уровней проверки подлинности требуются более высокие вычислительные ресурсы. Вы, как разработчик, должны решить, какое значение важнее для вашего приложения — скорость или безопасность. Большинство разработчиков находят, что при тестировании производительности они могут достичь приемлемых уровней производительности, обеспечивая адекватное обеспечение безопасности.

Клиентская и серверная части распределенного приложения должны использовать одинаковый уровень проверки подлинности. Список уровней проверки подлинности RPC см. в разделе [константы уровня проверки подлинности](authentication-level-constants.md).
