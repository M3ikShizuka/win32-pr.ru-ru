# <a name="choosing-the-type-of-binding-handles-to-use"></a>Выбор типа используемых дескрипторов привязки

**Рекомендации:** Если известно, какой сервер будет использоваться приложением, используйте явные дескрипторы. В противном случае всегда используйте каждую конструкцию явных дескрипторов или используйте универсальные дескрипторы с подпрограммыми **\_ BIND** и **\_ Unbind** .

Не используйте неявные дескрипторы или автоматические дескрипторы. Неявные дескрипторы не являются потокобезопасными, и даже несмотря на то, что безопасность потоков может показаться ненужной, она может быть понижена. Автоматические дескрипторы имеют большие издержки и нуждаются в правильной работе программы установки. Их возможности поиска заменены Active Directoryными службами.

Явные дескрипторы очень эффективны, и многие привлекательные возможности доступны только для явных дескрипторов. Например, если несколько вызовов RPC будут выполняться на одном и том же сервере, можно создать один и тот же обработчик привязки и выполнить все вызовы. Этот подход гораздо более эффективен, чем любой другой метод. Если сервер, на который будет проходить вызов, неизвестен, создайте явный дескриптор привязки для каждого вызова или используйте универсальные дескрипторы привязки.

в Microsoft™ Windows XP время выполнения RPC довольно эффективно при повторном использовании и кэшировании вызовов, поэтому, *если на одном* сервере с *n*-ом вызовом заканчивается один и тот же вызов, то RPC повторно использует ресурсы, выделенные для выполнения *n*– го вызова, обойти необходимость кэширования дескрипторов привязки для повышения производительности.
