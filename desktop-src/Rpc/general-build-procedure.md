# <a name="general-build-procedure"></a>Общая процедура сборки

Процесс создания клиентского или серверного приложения с помощью Microsoft RPC:

-   Разработка интерфейса.
-   Разработка сервера, реализующего интерфейс.
-   Разработка клиента, использующего интерфейс.

Эти шаги показаны на следующем рисунке.

![процесс создания приложения "клиент-сервер" с помощью Microsoft RPC](images/appdev.png)

Можно и часто разрабатывать клиентские и серверные приложения параллельно. Так как клиент и серверные программы зависят от интерфейса, интерфейс между ними должен быть разработан до разработки клиента и сервера, как показано на предыдущей схеме.

В этом разделе обсуждаются шаги, необходимые для создания приложения клиента/сервера с помощью RPC. Сведения представлены в следующих разделах:

-   [Разработка интерфейса](developing-the-interface.md)
-   [Разработка сервера](developing-the-server.md)
-   [Разработка клиента](developing-the-client.md)
