# <a name="criteria-for-name-service-entries"></a>Критерии для записей службы имен

При обработке записей службы имен используются следующие критерии.

-   Если указать имя записи, отличное от **null** , для [**рпкнсбиндинглукупбегин**](/windows/desktop/api/Rpcnsi/nf-rpcnsi-rpcnsbindinglookupbegina), то эта запись будет единственной записью для поиска дескрипторов привязки. Если передать **значение NULL**, будет выполнен поиск всех записей в домене входа в систему. Обратите внимание, что сюда не входят доверенные домены.
-   Если вы предоставляете дескриптор интерфейса, дескрипторы привязки возвращаются из записи, только если раздел интерфейса записи содержит совместимую версию этого интерфейса UUID. То есть основной номер версии должен совпадать с идентификатором UUID интерфейса, а номер вспомогательной версии должен быть больше или равен вашему.
-   Если указать UUID объекта, дескрипторы привязки возвращаются из записи только в том случае, если раздел UUID объекта содержит этот конкретный идентификатор объекта.

Если запись службы имен не сохранила описанные выше условия, собираются все дескрипторы привязки из этих записей. Дескрипторы с последовательностью протокола, не поддерживаемой клиентом, отбрасываются, и остальные дескрипторы возвращаются в качестве выходных данных [**рпкнсбиндинглукупнекст**](/windows/desktop/api/Rpcnsi/nf-rpcnsi-rpcnsbindinglookupnext).
