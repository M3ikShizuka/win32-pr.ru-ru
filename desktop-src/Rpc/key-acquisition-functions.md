# <a name="key-acquisition-functions"></a>Функции приобретения ключей

По умолчанию поставщик общих служб предоставляет ключи шифрования для серверных программ, запрашивающих их. Каждый поставщик общих служб реализует собственную систему создания ключей. Формат ключей, создаваемых поставщиком общих служб, зависит от поставщика общих служб.

RPC предоставляет возможность переопределения метода создания ключей шифрования по умолчанию. Приложение может вызвать функцию [**рпксерверрегистераусинфо**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcserverregisterauthinfo) и передать ему указатель на функцию получения ключа. Функцию приобретения ключа можно написать так, чтобы она создавала ключи с помощью любого выбранного метода. Однако ключ, который он передает в серверную программу, должен соответствовать формату, который требуется для поставщика общих служб.

> [!Note]  
> Большинству приложений не требуется использовать функции приобретения ключа, и можно просто указать **значение NULL** для всех параметров, в которых запрашивается функция получения ключа.
