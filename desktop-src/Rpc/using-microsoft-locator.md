# <a name="using-microsoft-locator"></a>Использование локатора Майкрософт

Служба "Локатор Майкрософт" является службой имен по умолчанию. Библиотека времени выполнения RPC использует ее для поиска серверных программ на системах узлов сервера.

**Примечание**    . Указатель Microsoft RPC не поддерживается в Windows Vista и более поздних операционных системах.

До Windows 2000 локатор Майкрософт не предоставил записей службы постоянных имен. Все записи в службе имен хранились в кэше памяти на главном компьютере серверной программы. Указатель использовал механизм вещания для обнаружения расположения серверов по запросу клиентов. При завершении работы системы узла все записи службы имен теряются.

Начиная с выпуска Windows 2000 локатор Майкрософт теперь поддерживает записи службы постоянных имен. Для этого система использует распределенную службу каталогов для хранения записей службы имен. Этот механизм имеет несколько преимуществ.

-   **Сохраняемости.** Серверным программам больше не требуется экспортировать сведения о привязке в службу имен при каждом запуске. Служба имен теперь хранит информацию до тех пор, пока серверная программа на сетевом администраторе не удалит ее явным образом.
-   **Мощности.** Устраняя большую часть широковещательной рассылки для записей службы имен, локатор сокращает сетевой трафик. Он также находит записи службы имен быстрее.
-   **Взаимодействие между доменами.** Теперь клиенты могут выполнять запросы службы имен в нескольких доменах.

Текущие версии локатора Майкрософт имеют обратную совместимость. Например, хост-система сервера, на котором работает локатор, поставляемая с Windows 2000, может правильно работать в сети, содержащей серверные системы, на которых выполняется локатор, входящий в состав Windows NT 4,0.

В выпуске Windows 2000 локатор Майкрософт теперь поддерживает записи службы имен для групп пользователей. Кроме того, он позволяет создавать профили пользователей.

Кроме того, текущая версия локатора Майкрософт поддерживает использование списков управления доступом в записях службы имен. Эта возможность повышает безопасность сети.

Поддержка самонастраивающийся теперь включена в указатель Майкрософт. Таким образом, он может прозрачно выполнять обработку самонастраивающийся событий, таких как изменение домена. Дополнительные сведения см. в разделе [**рпкнсбиндинжекспортпнп**](/windows/desktop/api/Rpcnsi/nf-rpcnsi-rpcnsbindingexportpnpa) и [**рпкнсбиндингунекспортпнп**](/windows/desktop/api/Rpcnsi/nf-rpcnsi-rpcnsbindingunexportpnpa).
