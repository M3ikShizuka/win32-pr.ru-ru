# <a name="deploying-load-balancing"></a>Развертывание балансировки нагрузки

Ниже показана типичная среда развертывания и вариант использования, в которых используется балансировщик нагрузки RPC.![Балансировка нагрузки RPC](images/rpc-load-balancing.png)

1.  RPC-клиент выполняет подключение RPC/HTTP к ферме серверов.
2.  Подключение перенаправляется по сети в балансировщик нагрузки внешнего интерфейса
3.  Подсистема балансировки нагрузки внешнего интерфейса выбирает сервер для обслуживания запроса. В этом примере балансировщик нагрузки внешнего интерфейса перенаправляет соединение на сервер 1.
4.  Служба балансировщика нагрузки RPC единолично соединение. Он определяет, что это первое подключение клиента, и связывает соединение с локальным сервером, сервером 1.
5.  Клиент выполняет второй запрос RPC/HTTP.
6.  Запрос перенаправляется по сети в балансировщик нагрузки переднего плана.
7.  Подсистема балансировки нагрузки внешнего интерфейса выбирает сервер для обслуживания запроса. В этом случае балансировщик нагрузки внешнего интерфейса выбирает сервер 2 для обслуживания запроса.
8.  Служба RPC Load Balancer, единолично соединение. Он определяет, что подключения от этого клиента обслуживаются сервером 1.
9.  Соединение перенаправляется на сервер 1.
