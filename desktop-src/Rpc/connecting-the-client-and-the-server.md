# <a name="connecting-the-client-and-the-server"></a>Подключение клиента и сервера

Для обмена данными клиентские и серверные программы должны установить сеанс связи через сеть или сети, соединяющие их. После установления соединения клиент может вызывать удаленные процедуры в серверной программе так, как если бы они были локальными для клиентской программы.

В этом разделе приводятся общие сведения о том, как установить соединение между клиентами и серверами для удаленных вызовов процедур. Он не содержит подробного обсуждения этого раздела. Все концепции в этом разделе подробно описаны в последующих главах и в разделе Справочник по функциям RPC.

Обсуждение, представленное в этом разделе, состоит из следующих разделов:

-   [Основная терминология привязки RPC](essential-rpc-binding-terminology.md)
-   [Как сервер готовится к подключению](how-the-server-prepares-for-a-connection.md)
-   [Как клиент устанавливает подключение](how-the-client-establishes-a-connection.md)

Обратите внимание, что в обсуждении предполагается наличие явных дескрипторов привязки. Однако если приложение использует другие типы дескрипторов привязки, может потребоваться изменить шаги, описанные в этом разделе. Дополнительные сведения см. в разделе [Binding and Handles](binding-and-handles.md).
