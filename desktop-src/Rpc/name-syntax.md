# <a name="name-syntax"></a>Синтаксис имени

Microsoft RPC принимает имена, соответствующие следующему синтаксису:

``` syntax
/.:/name[/name...]/.../domainname/name[/name...]
```

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="name"></span><span id="NAME"></span>безымян
</dt> <dd>

Указывает идентификатор, который может содержать любой символ, кроме символа косой черты (/).

</dd> <dt>

<span id="domainname"></span><span id="DOMAINNAME"></span>имя_домена
</dt> <dd>

Указывает имя домена.

</dd> </dl>

Параметр, выбирающий тип синтаксиса Name-Syntax, и строка, указывающая имя, предоставляются многим функциям RPC с интерфейсом службы имен (NSI).

В Microsoft RPC поддерживается только один тип синтаксиса имени, который указывается константой RPC \_ C NS с \_ \_ синтаксисом \_ DCE. Эта константа определена в файле заголовка РПКНСИ. H.

Синтаксис имени, заданный с \_ помощью \_ синтаксиса RPC C NS, \_ \_ является расширением \_ синтаксиса имени службы каталога ячеек использование DCE (CD). Возможность указания имени домена представляет собой расширение для этого синтаксиса. Не существует абсолютного ограничения на количество имен, которые могут быть разделены символами косой черты, если общая строка содержит менее 256 символов.

Косые черты позволяют указать в имени логическую структуру, но они не соответствуют логической структуре самих объектов.
