# <a name="the-idl-and-acf-files"></a>Файлы IDL и ACF

Синтаксис язык MIDL (MIDL) основан на синтаксисе языка программирования C. Если понятие языка в этом описании MIDL не определено полностью, подразумевается определение этого термина на языке C.

Конструкция MIDL определяет два различных файла: файл определения интерфейса (IDL) и файл конфигурации приложения (ACF). Эти файлы содержат атрибуты, которые направляют создание файлов заглушки языка C, управляющих вызовом удаленного вызова процедур (RPC). IDL-файл содержит описание интерфейса между клиентом и программами сервера. Приложения RPC используют файл ACF для описания характеристик интерфейса, характерных для оборудования и операционной системы, которые составляют определенную операционную среду. Целью разделения этой информации на два файла является обеспечение того, что программный интерфейс не отличается от характеристик, влияющих только на операционную среду.

IDL-файл указывает сетевой контракт между клиентом и сервером, т. е. файл IDL указывает, что передается между клиентом и сервером. Хранение этих сведений, отличающихся от сведений об операционной среде, делает IDL-файл переносимым в другие среды. IDL-файл состоит из двух частей: [заголовка интерфейса](the-idl-interface-header.md) и [тела интерфейса](the-idl-interface-body.md).

ACF указывает атрибуты, влияющие только на локальную производительность, а не на сетевой контракт. Microsoft RPC позволяет объединить атрибуты ACF и IDL в одном IDL-файле. Можно также объединить несколько интерфейсов в одном IDL-файле (и его ACF).
