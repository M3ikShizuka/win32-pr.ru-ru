<dl> <dt>

<span id="_rpc_cds_glos"></span><span id="_RPC_CDS_GLOS"></span>**Служба каталогов ячеек (компакт-диски)**
</dt> <dd>

Имя — поставщик службы для среды распределенных вычислений в среде Open Software Foundation.

</dd> <dt>

<span id="_rpc_client_stub_glos"></span><span id="_RPC_CLIENT_STUB_GLOS"></span>**Заглушка клиента**
</dt> <dd>

Созданный MIDL исходный код на языке C. Он содержит все функции, необходимые клиентскому приложению для выполнения удаленных вызовов процедур с использованием модели традиционного вызова функции в автономном приложении. Клиентская заглушка отвечает за упаковку входных параметров и распаковать выходные параметры. См. также [*заглушку сервера*](s-glos.md), [*прокси-*](p-glos.md)сервер.

</dd> <dt>

<span id="_rpc_conformant_array_glos"></span><span id="_RPC_CONFORMANT_ARRAY_GLOS"></span>**согласованный массив**
</dt> <dd>

Массив, размер которого определяется во время выполнения другим параметром, полем структуры или выражением.

</dd> <dt>

<span id="_rpc_connection_oriented_glos"></span><span id="_RPC_CONNECTION_ORIENTED_GLOS"></span>**с ориентацией на подключения**
</dt> <dd>

Протокол связи или транспорт, предоставляющий виртуальный канал, через который пакеты данных получаются в том же порядке, в котором они были переданы. Если подключение между компьютерами завершается сбоем, приложение получает уведомления. Протоколы [*TCP*](t-glos.md) и [*SPX*](s-glos.md) являются примерами протоколов, ориентированных на подключения. См. также [*датаграмма*](d-glos.md).

</dd> <dt>

<span id="_rpc_connectionless_glos"></span><span id="_RPC_CONNECTIONLESS_GLOS"></span>**Служба**
</dt> <dd>

См. сведения о [*датаграмме*](d-glos.md).

</dd> <dt>

<span id="_rpc_context_rundown_glos"></span><span id="_RPC_CONTEXT_RUNDOWN_GLOS"></span>**Очистка контекста**
</dt> <dd>

Уведомление сервера, полученное в результате непредвиденного завершения привязки между клиентскими и серверными приложениями.

</dd> </dl>
