# <a name="configuring-the-name-service-provider"></a>Настройка поставщика службы имен

Если распределенное приложение регистрирует свой интерфейс в службе имен, клиент и сервер должны использовать одну и ту же службу имен. Microsoft RPC взаимодействует с локатором Майкрософт и любым поставщиком службы имен (NSP), который соответствует интерфейсу службы имен Microsoft RPC (NSI), например к службе каталогов ячеек DCE, доступ к которой осуществляется с помощью управляющей программы-службы имени корпорации Digital Equipment (НСИД). Служба локатора Майкрософт, предназначенная для использования в средах Windows, является поставщиком службы имен по умолчанию.

-   [Настройка службы имен для Windows 3. x или MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
