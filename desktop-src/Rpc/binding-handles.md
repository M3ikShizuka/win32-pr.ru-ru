# <a name="binding-handles"></a>Дескрипторы привязки

Привязка — это процесс создания логического соединения между клиентской программой и серверной программой. Сведения, составляющие привязку между клиентом и сервером, представлены структурой, которая называется маркером привязки.

Маркер привязки аналогичен файлу, возвращаемому функцией библиотеки времени выполнения fopen языка C, или обработчику окна, возвращаемому функцией [**CreateWindow**](/windows/win32/api/winuser/nf-winuser-createwindowa) .

Как и в случае с этими дескрипторами, приложение не может напрямую обращаться к информации в дескрипторе привязки и управлять ею. Сведения в структуре данных обработчика привязки доступны только в библиотеках времени выполнения RPC. Вы предоставляете обработчик, библиотеки времени выполнения обращаются к соответствующим данным и манипулирует ими.
