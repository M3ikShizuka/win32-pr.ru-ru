# <a name="array-attributes"></a>Атрибуты массивов

Между массивами и указателями в языке C существует замкнутая связь. При передаче в функцию в качестве параметра имя массива рассматривается как указатель на первый элемент массива, как показано в следующем примере:


```C++
/* fragment */
extern void f1(char * p1);

void main(void)
{
    char chArray[MAXSIZE];

    fLocal1(chArray);
}
```



При локальном вызове можно использовать параметр указателя с марта по памяти и исследовать содержимое других адресов:


```C++
/* dump memory (fragment) */
void fLocal1(char * pch1)
{
    int i;

    for (i = 0; i < MAXSIZE; i++)
       printf("%c ", *pch1++);
}
```



Когда клиент передает указатель на удаленную процедуру, заглушка клиента передает как указатель, так и данные, на которые он указывает. Если указатель не ограничен соответствующими данными, вся память клиента должна передаваться при каждом удаленном вызове. Применяя строгую типизацию в определении интерфейса, MIDL ограничивает обработку заглушки клиента данными, которые соответствуют указанному указателю.

Размер массива и диапазон элементов массива, передаваемых на удаленный компьютер, могут быть постоянными или переменными. Если эти значения являются переменными и, таким способом, определяются во время выполнения, необходимо использовать атрибуты в IDL-файле, чтобы указать, сколько элементов массива передавать. Следующие атрибуты MIDL поддерживают границы массивов.



| attribute                             | Описание                                             | Значение по умолчанию |
|---------------------------------------|---------------------------------------------------------|---------|
| \[[ **первый \_ —**](/windows/desktop/Midl/first-is)\]   | Индекс первого переданного элемента массива.           | 0       |
| \[[ **последний \_ —**](/windows/desktop/Midl/last-is)\]     | Индекс последнего переданного элемента массива.            | \-      |
| \[[ **длина \_ равна**](/windows/desktop/Midl/length-is)\] | Общее число переданных элементов массива.             | \-      |
| \[[ **Максимальное \_ число**](/windows/desktop/Midl/max-is)\]       | Наибольшее допустимое значение индекса массива.                        | \-      |
| \[[ **min \_ —**](/windows/desktop/Midl/min-is)\]       | Наименьшее допустимое значение индекса массива.                         | 0       |
| \[[ **размер \_ равен**](/windows/desktop/Midl/size-is)\]     | Общее число элементов массива, выделенных для массива. | \-      |



 

> [!Note]  
> Атрибут **min \_ не** реализован в RPC. Минимальный индекс массива всегда обрабатывается как ноль.
