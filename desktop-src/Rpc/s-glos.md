# <a name="s-rpc"></a>S (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) [F](f-glos.md) G р [. J](i-glos.md) K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) S [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**масштабируемости**
</dt> <dd>

Возможность полностью использовать дополнительную вычислительную мощность в многопроцессорной системе (2, 4, 8, 32 или больше процессоров). Масштабируемость также позволяет обслуживать большое количество клиентов.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Протокол виртуализированных пакетов (SPP)**
</dt> <dd>

Протокол связи Banyan Vines [*— ориентированный на соединение*](c-glos.md) информационные пакеты по локальным сетям.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**виртуализированные Exchange пакетов (SPX)**
</dt> <dd>

Протокол связи Novell NetWare [*с ориентацией на подключения*](c-glos.md) для передачи информационных пакетов по локальным сетям и глобальным сетям.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**сериализовать**
</dt> <dd>

Процесс маршалирования данных в (кодирование) и распаковки данных из (декодирования) буферов, которыми вы управляете. Это отличается от традиционного использования RPC, когда заглушки и среда выполнения RPC управляют буферами упаковки. Также называется выполе выбора. См. также раздел [*сериализация процедур*](p-glos.md), [*сериализация типов*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**Заглушка сервера**
</dt> <dd>

Созданный MIDL исходный код на языке C, содержащий все функции, необходимые для обработки удаленных запросов серверным приложением с помощью вызовов локальных процедур. См. также [*клиентскую заглушку*](c-glos.md).

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**сессии**
</dt> <dd>

Установленная связь между клиентским приложением и серверным приложением. См. также [*Привязка*](b-glos.md), [*маркер привязки*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**статическая функция обратного вызова**
</dt> <dd>

Удаленная процедура, которая является частью клиентской части распределенного приложения, которую сервер может вызывать для получения сведений от клиента. Атрибут \[ [обратного вызова](/windows/desktop/Midl/callback) \] обозначает статическую функцию обратного вызова.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**Отслеживание статических удостоверений**
</dt> <dd>

Статическое отслеживание идентификаторов указывает, что во время выполнения RPC используются учетные данные безопасности в маркере привязки клиента для всех вызовов RPC. См. также [*динамическое отслеживание идентификаторов*](d-glos.md).

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**Строковая привязка**
</dt> <dd>

Символьная строка, состоящая из параметров [*UUID*](u-glos.md)объекта, [*последовательности протокола*](p-glos.md), [*сетевого адреса*](n-glos.md), [*конечной точки*](e-glos.md)и конечной точки, которые можно использовать для создания [*маркера привязки*](b-glos.md) к указанному серверу.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**строгая типизация**
</dt> <dd>

Применение в компиляторе более тщательного контроля над типами данных. В MIDL и RPC строгая типизация используется для обеспечения согласованности данных на разных компьютерах в распределенной среде.

</dd> </dl>
