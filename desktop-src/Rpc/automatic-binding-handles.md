# <a name="automatic-binding-handles"></a>Автоматические маркеры привязки

Автоматические дескрипторы привязок полезны, когда приложению не требуется конкретный сервер, и когда ему не требуется поддерживать какие-либо сведения о состоянии между клиентом и сервером. При использовании автоматического дескриптора привязки нет необходимости писать код клиентского приложения для работы с привязками и дескрипторами — вы просто указываете использование дескриптора автоматической привязки в файле конфигурации приложения (ACF). Затем заглушка определяет обработчик и управляет привязкой.

Например, операция с отметкой времени может быть реализована с помощью автоматического обработчика. Это не отличается от клиентского приложения, на котором сервер предоставляет его с меткой времени, поскольку может принимать время с любого доступного сервера.

> [!Note]  
> Автоматические дескрипторы не поддерживаются для платформы Macintosh.

 

Вы указываете использование автодескрипторов, включив атрибут \[ [**Auto \_ Handles**](/windows/desktop/Midl/auto-handle) \] в ACF. В примере временной метки используется следующий ACF:

``` syntax
/* ACF file */
[
  auto_handle
]
interface autoh
{
}
```

Если ACF не содержит никаких других атрибутов дескриптора и если удаленные процедуры не используют явные дескрипторы, компилятор MIDL по умолчанию использует автоматические дескрипторы. Он также использует автоматические дескрипторы по умолчанию, если отсутствует ACF.

Удаленные процедуры указываются в IDL-файле. Автоматический обработчик не должен использоваться в качестве аргумента для удаленной процедуры. Пример:

``` syntax
/* IDL file */
[ 
  uuid (6B29FC40-CA47-1067-B31D-00DD010662DA),
  version(1.0),
  pointer_default(unique)
]
interface autoh
{
  void GetTime([out] long * time);
  void Shutdown(void);
}
```

Преимущество автоматической обработки заключается в том, что разработчику не нужно писать код для управления маркером; Суррогаты управляют привязкой автоматически. Это значительно отличается от [примера Hello, World](tutorial.md), в котором клиент управляет неявным примитивным маркером, определенным в ACF, и должен вызвать несколько функций времени выполнения для установки маркера привязки.
