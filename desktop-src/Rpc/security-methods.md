# <a name="security-methods"></a>Методы безопасности

Microsoft RPC поддерживает два разных способа добавления безопасности в распределенное приложение. Первый способ — использовать интерфейс поставщика поддержки безопасности (SSPI), доступ к которому можно получить с помощью функций RPC. Как правило, этот метод лучше использовать. SSPI предоставляет наиболее гибкие и независимые от сети функции проверки подлинности.

Второй способ заключается в использовании функций безопасности, встроенных в транспортные протоколы системы. Метод безопасности на транспортном уровне не является предпочтительным методом. Рекомендуется использовать SSPI, так как он работает на всех транспортировках, на разных платформах и обеспечивает высокий уровень безопасности, включая конфиденциальность.

В этом разделе приведены обзоры безопасности на уровне SSPI и транспорта в следующих разделах:

-   [интерфейс поставщика поддержки безопасности (SSPI)](security-support-provider-interface-sspi-.md)
-   [Безопасность транспорта](transport-security.md)
