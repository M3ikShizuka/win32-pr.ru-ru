# <a name="binding-time-out-constants"></a>Константы времени ожидания привязки

В библиотеке RPC используются константы времени ожидания привязки, чтобы указать относительное количество времени, которое должно быть затрачено на установку привязки к серверу. Время ожидания можно включить с помощью вызова функции [**рпкмгмтсеткомтимеаут**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcmgmtsetcomtimeout) . В следующем списке содержатся допустимые значения времени ожидания.



| Константа/значение                                                                                                                                                                                                                                                                | Описание                                                                                                                                                                                                              |
|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="RPC_C_BINDING_INFINITE_TIMEOUT"></span><span id="rpc_c_binding_infinite_timeout"></span><dl> <dt>**RPC \_ \_ \_ Бесконечное \_ время ожидания привязки C**</dt> <dt> 10</dt> </dl> | Постоянно пытается установить связь.<br/>                                                                                                                                                             |
| <span id="RPC_C_BINDING_MIN_TIMEOUT"></span><span id="rpc_c_binding_min_timeout"></span><dl> <dt>**RPC \_ \_ \_ Минимальное \_ время ожидания привязки C**</dt> <dt>0</dt> </dl>                   | Пытается установить минимальное количество времени для используемого сетевого протокола. Это значение отдает предпочтение времени отклика относительно правильности определения того, работает ли сервер.<br/>                                          |
| <span id="RPC_C_BINDING_DEFAULT_TIMEOUT"></span><span id="rpc_c_binding_default_timeout"></span><dl> <dt>**RPC \_ \_ \_ \_ Время ожидания привязки C по умолчанию**</dt> <dt>5</dt> </dl>       | Пытается использовать средний интервал времени для используемого сетевого протокола. Это значение дает возможность определить, работает ли сервер, и возвращает время ответа равным весу. Это значение по умолчанию.<br/> |
| <span id="RPC_C_BINDING_MAX_TIMEOUT"></span><span id="rpc_c_binding_max_timeout"></span><dl> <dt>**RPC \_ \_ \_ Максимальное \_ время ожидания привязки C**</dt> <dt>9</dt> </dl>                   | Пытается использовать для сетевого протокола самое длинное время. Это значение подходит для определения того, работает ли сервер в течение времени ответа.<br/>                                            |



## <a name="remarks"></a>Remarks

Значения в приведенной выше таблице не могут быть в секундах. Эти значения представляют относительное количество времени в масштабе от нуля до 10. Дополнительные сведения о предотвращении задержки связи см. в статье предотвращение зависаний на стороне клиента.
