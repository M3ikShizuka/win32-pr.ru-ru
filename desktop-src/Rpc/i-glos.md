# <a name="i-rpc"></a>I (RPC)

<dl> <dt>

<span id="_rpc_idempotent_glos"></span><span id="_RPC_IDEMPOTENT_GLOS"></span>**идемпотентными**
</dt> <dd>

Удаленный вызов процедур, который не изменяет состояние и возвращает одну и ту же информацию каждый раз при вызове с теми же входными параметрами.

</dd> <dt>

<span id="_rpc_input_parameter_glos"></span><span id="_RPC_INPUT_PARAMETER_GLOS"></span>**входной параметр**
</dt> <dd>

Параметр, содержащий данные, которые клиентское приложение передает в серверное приложение. Атрибут **in** обозначает входной параметр.

</dd> <dt>

<span id="_rpc_idl_glos"></span><span id="_RPC_IDL_GLOS"></span>**Язык определения интерфейса (IDL)**
</dt> <dd>

Использование — стандартный язык DCE для указания интерфейса для удаленных вызовов процедур. См. также [*MIDL*](m-glos.md).

</dd> <dt>

<span id="_rpc_ip_glos"></span><span id="_RPC_IP_GLOS"></span>**Протокол Интернета (IP)**
</dt> <dd>

Протокол связи сетевого уровня без подключения. См. также [*датаграмма*](d-glos.md).

</dd> <dt>

<span id="_rpc_ipx_glos"></span><span id="_RPC_IPX_GLOS"></span>**сетевой Exchange пакетов (IPX)**
</dt> <dd>

Протокол связи Novell NetWare, использующий сокеты [*датаграмм*](d-glos.md) для маршрутизации информационных пакетов по локальным сетям и глобальным сетям.

</dd> <dt>

<span id="_rpc_intranet_glos"></span><span id="_RPC_INTRANET_GLOS"></span>**интрасети**
</dt> <dd>

Частная сеть, использующая продукты и технологии Интернета (например, веб-серверы), но недоступна внешним пользователям Интернета.

</dd> </dl>
