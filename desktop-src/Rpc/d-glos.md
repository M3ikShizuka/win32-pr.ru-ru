# <a name="d-rpc"></a>D (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) D [E](e-glos.md) [F](f-glos.md) G р [. J](i-glos.md) K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**датаграммы**
</dt> <dd>

Протокол связи или транспорт, в котором пакеты данных маршрутизируются независимо друг от друга. Они могут следовать разным маршрутам и поступать в порядке, отличном от того, откуда они были отправлены. Примерами транспортного уровня и протокола датаграмм являются [*UDP*](u-glos.md) и [*IPX*](i-glos.md) . См. также [*объектно-ориентированное соединение*](c-glos.md).

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**discriminant**
</dt> <dd>

Переменная, указывающая типы данных, которые могут храниться в объединении.

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**размеченное объединение**
</dt> <dd>

(или *запись варианта*) Объединение, включающее дискриминатор как часть структуры данных, чтобы текущий допустимый тип данных передавался вместе с объединением. См. также [*инкапсулированное объединение*](e-glos.md), [*неинкапсулированное объединение*](n-glos.md).

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**Распределенная вычислительная среда (DCE)**
</dt> <dd>

Спецификация Open Software Foundation для набора интегрированных служб, включая удаленные вызовы процедур, распределенные файловые системы и службы безопасности. Использование-DCE RPC Standard является основанием для Microsoft RPC.

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**Динамическая конечная точка**
</dt> <dd>

Зависит от сети — адрес сервера, который запрашивается и назначается во время выполнения. См. также [*хорошо известная конечная точка*](w-glos.md).

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**динамическое отслеживание идентификаторов**
</dt> <dd>

Спецификация того, что библиотека времени выполнения RPC будет использовать учетные данные вызывающего потока, а не маркер привязки, для проверки подлинности каждый раз, когда клиент вызывает удаленную процедуру. См. также [*статическое отслеживание идентификаторов*](s-glos.md).

</dd> </dl>
