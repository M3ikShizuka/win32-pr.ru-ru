# <a name="authorization-services"></a>Службы авторизации

Служба авторизации — это метод, который SSP использует для авторизации доступа к удаленной процедуре. SSP могут предоставлять более одной службы авторизации. Однако они обычно выбирают один из них по умолчанию.

Приложение может использовать метод авторизации по умолчанию для текущего поставщика общих служб или указать его. В настоящее время Microsoft RPC поддерживает два метода авторизации. Один — для предоставления серверу авторизации на основе имени клиентской программы. Другой — для сравнения учетных данных проверки подлинности клиента с списком управления доступом (ACL) сервера.

Список служб авторизации см. в разделе [константы службы авторизации](authorization-service-constants.md).

> [!Note]  
> Рекомендуется, чтобы разработчики использовали RPC \_ C \_ AUTHZ \_ None.
