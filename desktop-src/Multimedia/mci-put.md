# <a name="mci_put-command"></a>Команда MCI « \_ разместить»

Команда MCI « \_ разместить» задает прямоугольники источника, назначения и фрейма. Эта команда распознает цифровые видеоролики и устройства наложения видео.

Чтобы отправить эту команду, вызовите функцию [**мЦисендкомманд**](/previous-versions//dd757160(v=vs.85)) со следующими параметрами.


```C++
MCIERROR mciSendCommand(
  MCIDEVICEID wDeviceID, 
  MCI_PUT, 
  DWORD dwFlags, 
  (DWORD) (LPMCI_GENERIC_PARMS) lpDest
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="wDeviceID"></span><span id="wdeviceid"></span><span id="WDEVICEID"></span>*вдевицеид*
</dt> <dd>

Идентификатор устройства MCI, который должен получить командное сообщение.

</dd> <dt>

<span id="dwFlags"></span><span id="dwflags"></span><span id="DWFLAGS"></span>*dwFlags*
</dt> <dd>

\_Уведомление MCI, \_ Ожидание MCI или, для устройств с цифровыми видео, \_ тест MCI. Дополнительные сведения об этих флагах см. [в разделе Флаги ожидания, уведомления и проверки](the-wait-notify-and-test-flags.md).

</dd> <dt>

<span id="lpDest"></span><span id="lpdest"></span><span id="LPDEST"></span>*лпдест*
</dt> <dd>

Указатель на [**общую структуру \_ \_ пармс MCI**](mci-generic-parms.md) . (Устройства с расширенными наборами команд могут заменить эту структуру структурой, зависящей от устройства.)

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает нуль в случае успеха или ошибку в противном случае.

## <a name="remarks"></a>Комментарии

С типом устройства **дигиталвидео** используются следующие дополнительные флаги:

<dl> <dt>

<span id="MCI_DGV_PUT_CLIENT"></span><span id="mci_dgv_put_client"></span>MCI \_ ДГВ \_ Размещение \_ клиента
</dt> <dd>

Прямоугольник, определенный для MCI \_ ДГВ \_ Rect, применяется к положению клиентского окна. Указанный прямоугольник задается относительно родительского окна окна просмотра. \_ \_ \_ Этот флаг должен быть установлен одновременно для окна размещения MCI ДГВ.

</dd> <dt>

<span id="MCI_DGV_PUT_DESTINATION"></span><span id="mci_dgv_put_destination"></span>\_ДГВ \_ размещения MCI \_
</dt> <dd>

Прямоугольник, определенный для MCI \_ ДГВ \_ Rect, задает прямоугольник назначения. Прямоугольник назначения определяет часть окна клиента, связанную с этим экземпляром драйвера устройства, которая показывает изображение или видео.

</dd> <dt>

<span id="MCI_DGV_PUT_FRAME"></span><span id="mci_dgv_put_frame"></span>MCI \_ ДГВ \_ Размещение \_ фрейма
</dt> <dd>

Прямоугольник, определенный для MCI \_ ДГВ \_ Rect, применяется к прямоугольнику фрейма. Прямоугольник фрейма указывает часть буфера фрейма, используемую в качестве места назначения видеороликов, полученных из прямоугольника видео. Видео должно масштабироваться в соответствии с прямоугольником буфера кадров.

Прямоугольник задается в координатах буфера кадров. Прямоугольником по умолчанию является полный буфер кадра. Указание этого прямоугольника позволяет устройству масштабировать изображение, когда оно поддерживает эти данные. Устройства, которые не могут масштабировать образ, отклоняют эту команду с помощью \_ НЕподдерживаемой \_ функции мЦиерр. Вы можете использовать \_ флаг ЖЕТДЕВКАПС MCI \_ , \_ чтобы определить, масштабируется ли устройство с помощью команды [MCI \_ жетдевкапс](mci-getdevcaps.md) . Устройство возвращает **значение false** , если оно не может масштабировать изображение.

</dd> <dt>

<span id="MCI_DGV_PUT_SOURCE"></span><span id="mci_dgv_put_source"></span>\_источник MCI \_ ДГВ \_
</dt> <dd>

Прямоугольник, определенный для MCI \_ ДГВ \_ Rect, указывает исходный прямоугольник. Исходный прямоугольник указывает, какая часть буфера фрейма должна масштабироваться для размещения в прямоугольнике назначения.

</dd> <dt>

<span id="MCI_DGV_PUT_VIDEO"></span><span id="mci_dgv_put_video"></span>\_видео о \_ размещении MCI ДГВ \_
</dt> <dd>

Прямоугольник, определенный для MCI \_ ДГВ \_ Rect, применяется к прямоугольнику видео. В прямоугольнике видео указывается, какая часть текущего источника презентации хранится в буфере кадров. Прямоугольник задается с помощью естественных координат источника представления. Он позволяет определить кадрирование, которое происходит до сохранения изображений и видео в буфере кадров. Прямоугольником по умолчанию является полная активная область сканирования или полные распакованные изображения и видео.

</dd> <dt>

<span id="MCI_DGV_PUT_WINDOW"></span><span id="mci_dgv_put_window"></span>\_ \_ окно размещения MCI \_ ДГВ
</dt> <dd>

Прямоугольник, определенный для MCI \_ ДГВ \_ Rect, применяется к окну просмотра. Этот прямоугольник относится к родительскому окну окна просмотра (обычно это рабочий стол). Если окно не указано, по умолчанию используется исходный размер и расположение окна.

</dd> <dt>

<span id="MCI_DGV_RECT"></span><span id="mci_dgv_rect"></span>MCI \_ ДГВ \_ Rect
</dt> <dd>

Элемент **RC** структуры, идентифицируемой *лпдест* , содержит допустимый прямоугольник.

</dd> </dl>

Для устройств с цифровыми видео *лпдест* указывает на структуру [**MCI \_ ДГВ \_ \_ пармс**](/previous-versions//dd743397(v=vs.85)) .

Для типа устройства **наложения** используются следующие дополнительные флаги:

<dl> <dt>

<span id="MCI_OVLY_PUT_DESTINATION"></span><span id="mci_ovly_put_destination"></span>\_овли \_ размещения MCI \_
</dt> <dd>

Прямоугольник, определенный для MCI \_ овли \_ Rect, указывает область окна клиента, используемую для вывода изображения. Прямоугольник содержит смещение и видимую область изображения относительно начала координат окна. Если кадр растягивается, источник растягивается до прямоугольника назначения.

</dd> <dt>

<span id="MCI_OVLY_PUT_FRAME"></span><span id="mci_ovly_put_frame"></span>MCI \_ овли \_ Размещение \_ фрейма
</dt> <dd>

Прямоугольник, определенный для MCI \_ овли \_ Rect, указывает область видеобуфера, используемую для получения видеоизображения. Прямоугольник содержит смещение и экстент области буфера относительно источника видеобуфера.

</dd> <dt>

<span id="MCI_OVLY_PUT_SOURCE"></span><span id="mci_ovly_put_source"></span>\_источник MCI \_ овли \_
</dt> <dd>

Прямоугольник, определенный для MCI \_ овли \_ Rect, указывает область видеобуфера, используемую в качестве источника цифрового изображения. Прямоугольник содержит смещение и экстент прямоугольника обрезки для видеобуфера относительно его источника.

</dd> <dt>

<span id="MCI_OVLY_PUT_VIDEO"></span><span id="mci_ovly_put_video"></span>\_видео о \_ размещении MCI овли \_
</dt> <dd>

Прямоугольник, определенный для MCI \_ овли \_ Rect, определяет область видеозаписи видеобуфера. Прямоугольник содержит смещение и экстент прямоугольника обрезки для источника видео относительно его происхождения.

</dd> <dt>

<span id="MCI_OVLY_RECT"></span><span id="mci_ovly_rect"></span>MCI \_ овли \_ Rect
</dt> <dd>

Элемент **RC** структуры, идентифицируемой *лпдест* , содержит допустимый отображаемый прямоугольник. Если этот флаг не указан, то прямоугольник по умолчанию соответствует координатам видеобуфера или окна, на которое выполняется усечение.

</dd> </dl>

Для устройств с наложением видео *лпдест* указывает на структуру [**MCI \_ овли \_ Rect \_ пармс**](mci-ovly-rect-parms.md) .
