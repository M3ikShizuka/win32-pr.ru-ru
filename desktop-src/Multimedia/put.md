# <a name="put-command"></a>Команда размещения

Команда размещения определяет область исходного изображения и окно назначения, используемые для вывода. Эта команда распознает цифровые видеоролики и устройства наложения видео.

Чтобы отправить эту команду, вызовите функцию [**mciSendString**](/previous-versions//dd757161(v=vs.85)) с заданным параметром *лпсзкомманд* , как показано ниже.

``` syntax
_stprintf_s(
  lpszCommand, 
  TEXT("put %s %s %s"), 
  lpszDeviceID, 
  lpszRegions, 
  lpszFlags
); 
```

## <a name="return-value"></a>Возвращаемое значение

Возвращает нуль в случае успеха или ошибку в противном случае.

## <a name="remarks"></a>Remarks

Команда размещения определяет один или несколько следующих прямоугольников при работе с устройствами наложения видео:

-   Прямоугольник видео определяет регион входящего видеоизображения для записи.
-   Прямоугольник фрейма определяет область буфера кадров, которая получает изображение для входящего видео.
-   Исходный прямоугольник определяет, какой регион буфера фрейма копируется в прямоугольник назначения.
-   Прямоугольник назначения определяет область клиентской области окна просмотра, которая получает изображение видео.

Прямоугольник видео связан с прямоугольником фрейма так же, как исходный прямоугольник связан с прямоугольником назначения. Растяжение может происходить от прямоугольника видео до прямоугольника фрейма и из исходного прямоугольника в прямоугольник назначения. Не все устройства поддерживают растяжение, и растяжение должно быть включено (с помощью команды [Set](set.md) ).

Следующая команда определяет три региона для видео, кадра и источника.

``` syntax
put vboard video 120 120 200 200 frame 0 0 200 200 source 0 0 200 200
```

Регионы в этом примере определяются следующим образом:

-   В буфер кадров захватывается регион с размером 200 в 200 пикселей для входных данных видео, начиная с точки с координатами 120 пикселов, из верхнего левого угла.
-   Данные видео будут помещены в область размером 200 в 200 пикселей в левом верхнем углу буфера кадров.
-   Передача выполняется из области 200 – 200-пиксель в левом верхнем углу буфера кадров в конечное окно.
