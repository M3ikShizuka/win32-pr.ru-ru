# <a name="timer-event-operations"></a>Операции с событиями таймера

После установки разрешения таймера приложения можно запустить события таймера с помощью функции [**тимесетевент**](/previous-versions//dd757634(v=vs.85)) . Эта функция возвращает идентификатор таймера, который можно использовать для отмены или идентификации событий таймера. Одним из параметров функции является адрес функции обратного вызова [**тимепрок**](/previous-versions//dd757631(v=vs.85)) , которая вызывается при выполнении события Timer.

Существует два типа событий таймера: *один* и *периодически*. Одно событие таймера происходит один раз после указанного числа миллисекунд. Периодическое событие таймера возникает каждый раз, когда истечет указанное число миллисекунд. Интервал между периодическими событиями называется *задержкой события*. Периодические события таймера с задержкой события 10 миллисекунд или менее потребляют значительную часть ресурсов ЦП.

Связь между разрешением события таймера и длительностью задержки события важна в событиях таймера. Например, если задать разрешение 5 и задержку события 100, службы таймера будут уведомлять функцию обратного вызова через интервал от 95 до 105 миллисекунд.

Активное событие таймера можно отменить в любое время с помощью функции [**тимекиллевент**](/previous-versions//dd757630(v=vs.85)) . Не забудьте отменить все необработанные таймеры, прежде чем освободить память, содержащую функцию обратного вызова.

> [!Note]  
> Таймер мультимедиа выполняется в собственном потоке.
