# <a name="paste-command"></a>вставить команду

Команда Вставить копирует содержимое буфера обмена в рабочую область. Устройство Digital-Video распознает эту команду.

Чтобы отправить эту команду, вызовите функцию [**mciSendString**](/previous-versions//dd757161(v=vs.85)) с заданным параметром *лпсзкомманд* , как показано ниже.

``` syntax
_stprintf_s(
  lpszCommand, 
  TEXT("paste %s %s %s"), 
  lpszDeviceID, 
  lpszItem, 
  lpszFlags
); 
```

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="lpszDeviceID"></span><span id="lpszdeviceid"></span><span id="LPSZDEVICEID"></span>*лпсздевицеид*
</dt> <dd>

Идентификатор устройства MCI. Этот идентификатор или псевдоним назначается при открытии устройства.

</dd> <dt>

<span id="lpszItem"></span><span id="lpszitem"></span><span id="LPSZITEM"></span>*лпсзитем*

Один или несколько следующих флагов.



| Значение                 | Значение                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|-----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| в *прямоугольнике*        | Задает расположение в кадре, куда вставляются данные. Верхний левый угол *прямоугольника* соответствует левому верхнему углу добавленных данных. Если прямоугольник имеет ненулевой размер в X или Y, содержимое буфера обмена масштабируется в этих измерениях при их вставлении в кадр. Если этот параметр опущен, *прямоугольником* по умолчанию выделяется весь фрейм. Если этот флаг указан в режиме "вставить" (по умолчанию), то любой регион за пределами прямоугольника закрашивается сплошным цветом.                       |
| *поток* аудиопотока | Указывает аудиопоток в рабочей области, затронутой командой. Если в буфере обмена существует только один звуковой поток, звуковые данные вставляются в указанный *поток*. Если в буфере обмена существует более одного звукового потока, *поток* указывает начальное число для последовательностей потока. Если вы используете этот флаг, а также хотите вставить видео, необходимо также использовать флаг "поток видео". Это значение по умолчанию.                                                                                                                                                                                                                                                                                                                                                    |

Флаги "Insert" и "overwrite" влияют на то, будут ли во время операции вставки кадры уничтожены или перемещены, а не как данные вставляются в пределах каждого кадра.                                                                                                                                                                                                                                              |
| в *Расположение*         | Указывает позиции в рабочей области, в которую вставляются данные. Если в буфере обмена существует более одного видеопотока, *поток* указывает начальное число для последовательностей потока. ## <a name="return-value"></a>Возвращаемое значение

Возвращает нуль в случае успеха или ошибку в противном случае.

## <a name="remarks"></a>Remarks

В данных, скопированных из буфера обмена, отсутствуют сигналы. Изменение становится постоянным только при явном сохранении данных. Однако воспроизведение действует так, как если бы данные были добавлены.
