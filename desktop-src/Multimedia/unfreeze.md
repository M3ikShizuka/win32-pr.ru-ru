# <a name="unfreeze-command"></a>отменить закрепление команды

Команда unfreeze снова включает получение видео в буфер фрейма после того, как он был отключен командой [Freeze](freeze.md) . Эта команда распознает цифровые видеоролики, видеомагнитофоны и устройства наложения видео.

Чтобы отправить эту команду, вызовите функцию [**mciSendString**](/previous-versions//dd757161(v=vs.85)) с заданным параметром *лпсзкомманд* , как показано ниже.

``` syntax
_stprintf_s(
  lpszCommand, 
  TEXT("unfreeze %s %s %s"), 
  lpszDeviceID, 
  lpszUnfreeze, 
  lpszFlags
); 
```

## <a name="parameters"></a>Параметры

<dl> <dt>

<span id="lpszDeviceID"></span><span id="lpszdeviceid"></span><span id="LPSZDEVICEID"></span>*лпсздевицеид*
</dt> <dd>

Идентификатор устройства MCI. Этот идентификатор или псевдоним назначается при открытии устройства.

</dd> <dt>

<span id="lpszUnfreeze"></span><span id="lpszunfreeze"></span><span id="LPSZUNFREEZE"></span>*лпсзунфризе*
</dt> <dd>

Флаг для повторного включения получения видео в буфер кадров. В следующей таблице перечислены типы устройств, которые распознают команду **разморозки** и флаги, используемые каждым типом.



| Значение        | Значение        |
|--------------|----------------|
| дигиталвидео | в *прямоугольнике* |
| overlay      | в *прямоугольнике* |
| видеомагнитофон          | входные выходные данные   |



 

В следующей таблице перечислены флаги, которые могут быть указаны в параметре **лпсзунфризе** , и их значения.



| Значение          | Значение                                                                                                                                                                                                                                                                                    |
|----------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| в *прямоугольнике* | Указывает регион, в котором будет включено повторное получение видео. Прямоугольник задается относительно источника видеобуфера и задается как *x1 Y1 x2 Y2*. Координаты *x1 Y1* определяют левый верхний угол прямоугольника, а координаты *x2 Y2* определяют ширину и высоту. |
| input          | Разморозить входной образ.                                                                                                                                                                                                                                                                  |
| output         | Разморозить выходное изображение. Если не указано ни «вход», ни «Output», то предполагается «Output».                                                                                                                                                                                                  |



 

</dd> <dt>

<span id="lpszFlags"></span><span id="lpszflags"></span><span id="LPSZFLAGS"></span>*лпсзфлагс*
</dt> <dd>

Может иметь значение "Wait", "notify" или и то, и другое. Для устройств с цифровыми видео и ВИДЕОМАГНИТОФОНА можно также указать "Test". Дополнительные сведения об этих флагах см. [в разделе Флаги ожидания, уведомления и проверки](the-wait-notify-and-test-flags.md).

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает нуль в случае успеха или ошибку в противном случае.

## <a name="examples"></a>Примеры

Следующая команда отменяет фиксацию области видеобуфера.

``` syntax
unfreeze vboard at 10 20 90 165
```
