# <a name="sending-midi-messages-with-stream-buffers"></a>Отправка сообщений MIDI с помощью буферов потоков

Когда приложение работает с буферами потоков, оно использует функцию [**мидистреамаут**](/windows/win32/api/mmeapi/nf-mmeapi-midistreamout) для отправки всех (коротких и длинных) сообщений MIDI на устройство. Чтобы указать блоки потоковых данных, используйте структуры [**мидихдр**](/windows/win32/api/mmeapi/ns-mmeapi-midihdr) и [**мидиевент**](/windows/win32/api/mmeapi/ns-mmeapi-midievent) . Структура **мидихдр** содержит адрес заблокированного блока данных, длину блока данных и некоторые флаги ассортимента. Данные хранятся в виде структур **мидиевент** . Система накладывает ограничение размера 64 КБ на буферы потоков.

После использования **мидистреамаут** для отправки буфера потока данных необходимо дождаться завершения работы драйвера устройства с блоком данных перед его освобождением. При отправке нескольких блоков данных необходимо отслеживать завершение каждого блока данных, чтобы вы могли получить сведения о том, когда отправлять дополнительные блоки. Сведения о различных методах мониторинга завершения блоков данных см. в разделе [Управление блоками данных MIDI](managing-midi-data-blocks.md).
