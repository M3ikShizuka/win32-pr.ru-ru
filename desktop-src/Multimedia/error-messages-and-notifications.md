# <a name="error-messages-and-notifications"></a>Сообщения об ошибках и уведомления

МЦивнд использует MCI для управления устройствами, которые воспроизводят и записывают данные мультимедиа. В общем случае МЦивнд отображает ошибки MCI в диалоговом окне ошибки. Ошибка MCI возникает при сбое команды MCI. Например, если приложение попытается возобновить приостановленное воспроизведение с помощью макроса [**мЦивндресуме**](/windows/desktop/api/Vfw/nf-vfw-mciwndresume) , а текущее устройство не поддерживает Resume, пользователю будет сообщено об ошибке.

МЦивнд предоставляет два способа обработки сообщений об ошибках:

-   Можно предотвратить достижение пользователями сообщений об ошибках. Чтобы предотвратить отображение сообщений об ошибках MCI, укажите \_ стиль окна мЦивндф ноеррордлг при создании экземпляра окна мЦивнд с помощью функции [**МЦивндкреате**](/windows/desktop/api/Vfw/nf-vfw-mciwndcreatea) или [CreateWindowEx](/windows/win32/api/winuser/nf-winuser-createwindowexa) .
-   Их можно перенаправить в приложение для показа. Чтобы перенаправить сообщения об ошибках MCI в приложение, укажите \_ стиль окна мЦивндф нотиферрор при создании экземпляра окна мЦивнд с помощью **МЦивндкреате** или **CreateWindowEx**.

Если уведомление об ошибке включено, МЦивнд отправляет каждое сообщение уведомления ([**мЦивндм \_ нотиферрор**](mciwndm-notifyerror.md)) в основной обработчик сообщений родительского окна мЦивнд. Приложение должно иметь обработчик сообщений для обработки получаемых им сообщений уведомления.

Текстовое описание последнего сообщения об ошибке MCI можно получить с помощью макроса [**мЦивнджетеррор**](/windows/desktop/api/Vfw/nf-vfw-mciwndgeterror) . Этот макрос возвращает текст в определяемом приложением буфере. Если строка ошибки длиннее, чем буфер, МЦивнд усекает строку.

Вы можете направить все уведомления в другое окно с помощью макроса [**мЦивндсетовнер**](/windows/desktop/api/Vfw/nf-vfw-mciwndsetowner) .
