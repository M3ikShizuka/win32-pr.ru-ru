## <a name="remarks"></a>Комментарии

Возможно, вы заинтересованы только в том случае, если метод завершился успешно или с ошибкой. Лучший способ проверить, указывает ли значение **HRESULT** об успешном или неудачном завершении, — передать значение в один из следующих макросов, определенных в файле Winerror. h:

-   Макрос [](/windows/win32/api/winerror/nf-winerror-succeeded) Success возвращает **значение true** для кода успешного выполнения и **значение false** для кода сбоя.
-   Макрос [**Failed**](/windows/win32/api/winerror/nf-winerror-failed) возвращает **значение true** для кода ошибки и **значение false** для кода успешного выполнения.

Значение **HRESULT** для каждого значения **\_ ошибки DXGI** определяется на основе этого макроса, определенного в дксгитипе. h:


```
#define _FACDXGI    0x87a
#define MAKE_DXGI_HRESULT(code) MAKE_HRESULT(1, _FACDXGI, code)
```



Например, **Ошибка DXGI \_ , \_ Недопустимый \_ вызов** , определена как **0x887A0001**:


```
#define DXGI_ERROR_INVALID_CALL                 MAKE_DXGI_HRESULT(1)
```
