# <a name="dxgi-13-improvements"></a>Усовершенствования DXGI 1,3

В графическую инфраструктуру Microsoft DirectX (DXGI) 1,3, которая входит в состав Windows 8.1, добавлены следующие функциональные возможности.

## <a name="trim-dxgi-adapter-memory-usage"></a>Обрезать использование памяти адаптера DXGI

Начиная с Windows 8.1, DXGI 1,3 добавляет возможность очистки и освобождения неиспользуемых ресурсов памяти, выделенных адаптером DXGI. Это позволяет приложениям освобождать временную память при приостановке, уменьшая вероятность того, что приложение будет закрыто для освобождения ресурсов для других приложений. После возобновления работы приложения драйверы устройств, поддерживающие Trim, будут воссоздавать ресурсы по мере необходимости. Начиная с Windows 8.1 все устройства Direct3D, созданные приложением, должны вызывать [**IDXGIDevice3:: Trim**](/windows/desktop/api/DXGI1_3/nf-dxgi1_3-idxgidevice3-trim) при приостановке для уменьшения объема памяти и уменьшить вероятность того, что приложение будет завершено для освобождения системных ресурсов.

## <a name="multi-plane-overlays"></a>Многоуровневые наложения

Начиная с Windows 8.1, DXGI 1,3 поддерживает многоуровневые наложения. Вы можете узнать, поддерживает ли устройство многоуровневые оверлея в оборудовании с помощью [**IDXGIOutput2:: суппортсоверлайс**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgioutput2-supportsoverlays).

## <a name="overlapping-swap-chains-and-swap-chain-scaling"></a>Перекрывающиеся цепочки перестановки и масштабирование цепочки буферов

Начиная с Windows 8.1, DXGI 1,3 поддерживает перекрывающиеся цепочки подкачки. Перекрывающиеся цепочки перестановки используются для отрисовки трехмерной графики при несобственных разрешениях (с масштабированием оборудования) при представлении пользовательского интерфейса при собственном разрешении. Это позволяет играм использовать преимущества более высоких ставок для реагирования на игрового процесса без ухудшения визуального качества элементов пользовательского интерфейса, таких как оценка проигрывателя и текст диалогового окна. На устройствах, поддерживающих наложение нескольких плоскостей, Direct3D будет использовать многоуровневые оверлея для перекрывающихся цепочек подкачки. Создайте цепочку перекачки переднего плана, указав флаг [**\_ \_ \_ \_ \_ уровня переднего плана для цепочки подкачки**](/windows/win32/api/dxgi/ne-dxgi-dxgi_swap_chain_flag) при создании цепочки буферов, а также используйте [**IDXGISwapChain2:: сетматрикстрансформ**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-setmatrixtransform) и [**IDXGISwapChain2:: жетматрикстрансформ**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-getmatrixtransform) для масштабирования цепочки буферов, используемой для игрового процесса.

## <a name="select-backbuffer-subregion-for-swap-chain"></a>Выберите подобласть заднего буфера для цепочки буферов

Начиная с Windows 8.1, DXGI 1,3 можно использовать для выбора подобласти буфера обмена для использования с цепочкой буферов, что позволяет подготовиться к просмотру в более коротком буфере без повторного создания цепочки буфера обмена. См. раздел [**IDXGISwapChain2:: сетсаурцесизе**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-setsourcesize) and [**IDXGISwapChain2:: жетсаурцесизе**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-getsourcesize).

## <a name="lower-latency-swap-chain-presentation"></a>Представление цепочки подкачки с меньшими задержками

Начиная с Windows 8.1, DXGI 1,3 позволяет сократить задержку, позволяя цепи подкачки завершать предыдущий кадр перед началом использования устройства для отрисовки следующего кадра. См. раздел [**IDXGISwapChain2:: жетфрамелатенциваитаблеобжект**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-getframelatencywaitableobject), [**IDXGISwapChain2:: жетмаксимумфрамелатенци**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-getmaximumframelatency)и [**IDXGISwapChain2:: сетмаксимумфрамелатенци**](/windows/desktop/api/dxgi1_3/nf-dxgi1_3-idxgiswapchain2-setmaximumframelatency).
