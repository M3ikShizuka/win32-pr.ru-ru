# <a name="user-identification-attributes"></a>Атрибуты идентификации пользователя

> [!Note]  
> служба проверки подлинности в интернете (IAS) переименовала сервер политики сети (NPS), начиная с Windows Server 2008. Содержимое этого раздела относится как к IAS, так и к NPS. По всему тексту NPS используется для ссылки на все версии службы, включая версии, изначально называемые IAS.

 

Удостоверение пользователя, запрашивающего проверку подлинности, предоставляется в DLL расширения NPS в различных атрибутах.

-   **ратусернаме**
-   **ратстриппедусернаме**
-   **ратфкусернаме**

Каждый атрибут предоставляет удостоверение пользователя в другом формате. Как правило, разработчики должны использовать **ратстриппедусернаме**. Использование атрибутов **ратусернаме** и **ратфкусернаме** является более специализированным.

> [!Note]  
> Атрибут User-Password **ратусерпассворд** уже был расшифрован при его отправке в библиотеку DLL расширения и может использоваться в этой форме.

 

## <a name="ratusername"></a>ратусернаме

Атрибут **ратусернаме** содержит имя, которое было фактически Отправлено "по сети". NPS не может каким либо образом обработать или проверить содержимое этого атрибута. Этот атрибут может быть недоступен, так как пользователь мог определиться с помощью таких средств, как идентификатор звонящего.

При использовании [**радиусекстенсионпроцесс/ex**](/windows/desktop/api/authif/nc-authif-pradius_extension_process_ex), если этот атрибут доступен, он доступен только в точке подключаемого модуля DLL расширения проверки подлинности. Атрибут **ратусернаме** недоступен в точке подключаемого модуля DLL расширения авторизации, так как в библиотеках DLL расширения авторизации функции **радиусекстенсионпроцесс/ex** видят только атрибуты "Outbound".

При использовании [**RadiusExtensionProcess2**](/windows/desktop/api/authif/nc-authif-pradius_extension_process_2), если этот атрибут доступен, он доступен как в точке подключаемого модуля DLL модуля проверки подлинности, так и в точке подключаемого модуля DLL расширения авторизации.

## <a name="ratstrippedusername"></a>ратстриппедусернаме

**Ратстриппедусернаме** — это удостоверение пользователя после удаления области. Дополнительные сведения об отбрасывания сфер см. в разделе " [имена областей](/previous-versions/windows/it-pro/windows-server-2003/cc779938(v=ws.10)) " на странице http: \\ \\ technet2.Microsoft.com.

Этот атрибут может присутствовать в точке подключаемого модуля DLL расширения проверки подлинности, точке подключаемого модуля DLL расширения авторизации или в обоих случаях. Этот атрибут гарантированно имеет следующий формат:

* Домен * **\\** _имя_пользователя_

Где *domain* — это NetBIOS-имя домена.

## <a name="ratfqusername"></a>ратфкусернаме

Атрибут **ратфкусернаме** — это полное имя пользователя.

Этот атрибут может присутствовать в точке подключаемого модуля DLL расширения проверки подлинности, точке подключаемого модуля DLL расширения авторизации или в обоих случаях. Однако формат атрибута может различаться между двумя точками подключаемых модулей. В точке подключаемого модуля DLL расширения проверки подлинности этот атрибут всегда будет иметь вид:

* Домен * **\\** _имя_пользователя_

Формат атрибута **ратфкусернаме** в точке подключаемого модуля DLL расширения авторизации зависит от того, является ли пользователь Active Directory пользователем.

-   Если пользователь является локальным пользователем **ратфкусернаме** имеет тот же формат, что и для точки подключаемого модуля DLL расширения проверки подлинности:

    * Домен * **\\** _имя_пользователя_

    .

-   Если пользователь является Active Directory пользователем, **ратфкусернаме** может содержать имя пользователя в "каноническом" формате. Канонический формат — это формат, используемый Active Directory для определения пользователя. Это путь из корня дерева Active Directory и включает подразделение пользователя (OU).
