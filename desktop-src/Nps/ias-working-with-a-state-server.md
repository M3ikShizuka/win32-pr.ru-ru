# <a name="working-with-a-state-server"></a>Работа с сервером состояний

> [!Note]  
> Служба проверки подлинности в Интернете (IAS) переименовала сервер политики сети (NPS), начиная с Windows Server 2008. Содержимое этого раздела относится как к IAS, так и к NPS. По всему тексту NPS используется для ссылки на все версии службы, включая версии, изначально называемые IAS.

 

Сервер политики сети выполняет проверку подлинности с помощью базы данных, настроенной на сайте сервера политики сети. Эта база данных проверки подлинности может быть пользовательской базой данных для домена Windows или нарисована на основе сведений о пользователе, полученных из Active Directory Windows. На следующей схеме показана типичная конфигурация, которая показывает, как сервер политики сети взаимодействует с базами данных проверки подлинности, например с базой данных пользователя домена Windows или Active Directory. На схеме также показано, как сервер политики сети может взаимодействовать с сервером состояний, предоставляемым третьим лицом. Основное назначение сервера состояний заключается в ограничении числа одновременных сеансов входа, которые может выполнять один пользователь.

![взаимодействие сервера политики сети с сервером состояний и базами данных проверки подлинности](images/ias02.png)

Существует две точки взаимодействия между сервером политики сети и сервером состояний. Одно взаимодействие происходит, когда NPS получает запрос на проверку подлинности от NAS. Сервер состояний предоставляет сведения из своей базы данных, чтобы определить, следует ли принимать или отклонять запрос. Другое взаимодействие происходит, когда NPS получает запросы учета от NAS. Сервер состояний использует эти сведения для обновления базы данных запросов на учет.

Дополнительные сведения о серверах состояний см. в следующих статьях:

-   [Рекомендации по проектированию сервера состояний](/windows/desktop/Nps/ias-state-server-design-considerations)
