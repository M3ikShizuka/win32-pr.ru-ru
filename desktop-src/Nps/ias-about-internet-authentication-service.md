# <a name="about-nps-extensions"></a>Сведения о расширениях NPS

> [!Note]  
> служба проверки подлинности в интернете (IAS) переименовала сервер политики сети (NPS), начиная с Windows Server 2008. Содержимое этого раздела относится как к IAS, так и к NPS. По всему тексту NPS используется для ссылки на все версии службы, включая версии, изначально называемые IAS.

 

В этом разделе описывается, как реализовать библиотеки DLL для расширения функциональных возможностей NPS. В нем описывается взаимодействие между сервером политики сети и библиотеками DLL и представлены некоторые рекомендации по проектированию библиотек DLL.

Сервер политики сети предоставляет две точки расширения: одну для проверки подлинности, а другую для авторизации. Проверка подлинности означает проверку удостоверения пользователя. Авторизация означает определение служб, которые сеть должна предоставить пользователю. Две точки расширения соответствуют библиотекам DLL расширения проверки подлинности и библиотекам расширения авторизации. Каждая точка расширения может поддерживать несколько библиотек DLL.

Сервер политики сети предоставляет службы проверки подлинности и авторизации. Библиотеки DLL расширения проверки подлинности вызываются NPS до встроенной проверки подлинности и авторизации NPS. Библиотеки DLL расширения авторизации вызываются после проверки подлинности и авторизации NPS.

На следующей схеме показан поток пакетов через сервер политики сети RADIUS, развернутый с помощью библиотек расширения.

![процесс проверки подлинности и авторизации NPS](images/ias03.png)

Если библиотека DLL расширения проверки подлинности возвращает ACCEPT, пакет пропускает проверку подлинности NPS и переходит непосредственно к авторизации NPS.

> [!Note]  
> При наличии нескольких библиотек DLL расширения проверки подлинности также могут быть пропущены остальные библиотеки DLL расширения. Дополнительные сведения см. [в разделе Настройка библиотек DLL расширения](/windows/desktop/Nps/ias-setting-up-the-extension-and-authorization-dlls) .

 

Если библиотека DLL расширения проверки подлинности возвращает значение CONTINUE, пакет переходит на проверку подлинности NPS, а затем на авторизацию NPS.

> [!Note]  
> При наличии нескольких библиотек DLL расширения проверки подлинности все остальные библиотеки DLL модулей проверки подлинности вызываются до проверки подлинности NPS.
