# <a name="id3dx10fontpreloadtext-method"></a>ID3DX10Font: метод:P Релоадтекст

Загрузка форматированного текста в видеопамять для повышения эффективности отрисовки на устройстве. Этот метод поддерживает строки ANSI и Юникод.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT PreloadText(
  [in] LPCSTR pString,
  [in] INT    Count
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пстринг* \[ окне\]
</dt> <dd>

Тип: **[ **LPCSTR**](../winprog/windows-data-types.md)**

Указатель на строку символов, которая должна быть загружена в видеопамять. Если для параметров компилятора требуется Юникод, тип данных LPCTSTR разрешается в ЛПКВСТР; в противном случае тип данных разрешается в LPCSTR. См. заметки.

</dd> <dt>

*Количество* \[ окне\]
</dt> <dd>

Тип: **[ **int**](../winprog/windows-data-types.md)**

Число символов в текстовой строке.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Тип: **[ **HRESULT**](https://msdn.microsoft.com/library/Bb401631(v=MSDN.10).aspx)**

Если метод выполнен успешно, возвращается значение S \_ . В случае сбоя метода возвращаемое значение может быть одним из следующих: D3DERR \_ инвалидкалл, D3DXERR \_ INVALIDDATA.

## <a name="remarks"></a>Remarks

Параметр компилятора также определяет версию функции. Если определен Юникод, вызов функции разрешается в Прелоадтекств. В противном случае вызов функции разрешается в Прелоадтекста, так как используются строки ANSI.

Этот метод создает текстуры, которые содержат глифы, представляющие входной текст. Глифы рисуются в виде ряда треугольников.

Текст не будет отображаться на устройстве; ID3DX10Font: для отрисовки текста необходимо по-прежнему вызывать метод:D Равтекст. Однако при предварительной загрузке текста в видеопамять ID3DX10Font::D Равтекст будет использовать значительно меньше ресурсов ЦП.

Этот метод внутренне преобразует символы в глифы с помощью функции GDI [жетчарактерплацемент](/previous-versions//ms534004(v=vs.85)).
