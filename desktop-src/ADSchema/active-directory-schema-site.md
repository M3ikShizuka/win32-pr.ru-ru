# <a name="active-directory-schema-terminology"></a>Терминология схемы Active Directory

Следующие термины обычно используются для ссылки на схему Active Directory.


<dl> <dt>

<span id="Attribute"></span><span id="attribute"></span><span id="ATTRIBUTE"></span>Версию
</dt> <dd>

Элементы данных, используемые для описания объектов, представленных классами, определенными в схеме. Атрибуты определяются в схеме отдельно от классов. Это позволяет применять одно определение атрибута ко многим классам. Например, [**Description**](a-description.md) — это атрибут, который можно применить к любому классу в схеме. Атрибут **Description** определен один раз в схеме, гарантируя согласованность, вместо того чтобы иметь другое определение для **описания** пользователя и **описания** принтера.

> [!Note]  
> *Свойство* Term часто используется с *атрибутом* Term.

 

</dd> <dt>

<span id="Attribute_Instance"></span><span id="attribute_instance"></span><span id="ATTRIBUTE_INSTANCE"></span>Экземпляр атрибута
</dt> <dd>

Вхождение атрибута, определенного в схеме. Этот термин используется для различения определения атрибута и отдельного вхождения атрибута. Например, при хранении объекта пользователя для «Джеффа Иванов» с атрибутом [**Common-Name**](a-cn.md) , имеющим значение «Джефф Иванов», создается экземпляр [**Common-Name**](a-cn.md).

</dd> <dt>

<span id="Class"></span><span id="class"></span><span id="CLASS"></span>См
</dt> <dd>

Формальное описание дискретного идентифицируемого типа объекта, хранящегося в службе каталогов. Например, [**пользователь**](c-user.md), [**очередь печати**](c-printqueue.md)и [**Группа**](c-group.md) являются классами. Более того, существует три категории классов: [структурные классы](classes-structural.md), [абстрактные классы](classes-abstract.md)и [вспомогательные классы](classes-auxiliary.md).

</dd> <dt>

<span id="Class_Instance"></span><span id="class_instance"></span><span id="CLASS_INSTANCE"></span>Экземпляр класса
</dt> <dd>

Вхождение класса, определенного в схеме. Этот термин используется для различения определения класса и отдельного вхождения класса. Например, при сохранении объекта [**пользователя**](c-user.md) для «Джеффа Иванов» в службе каталогов создается экземпляр [**пользователя**](c-user.md).

</dd> <dt>

<span id="Content_Rules"></span><span id="content_rules"></span><span id="CONTENT_RULES"></span>Правила содержимого
</dt> <dd>

Определение возможного содержимого экземпляров класса, которые хранятся в службе каталогов. В службах каталогов NT (NTDS), на которых основано Active Directory, правила содержимого полностью выражаются атрибутами, которые [**должны содержать**](a-mustcontain.md) и [**могут содержать**](a-maycontain.md) атрибуты определений схемы для каждого класса.

</dd> <dt>

<span id="Derivation"></span><span id="derivation"></span><span id="DERIVATION"></span>Рожден
</dt> <dd>

См. раздел *наследование*.

</dd> <dt>

<span id="Directory_Information_Tree"></span><span id="directory_information_tree"></span><span id="DIRECTORY_INFORMATION_TREE"></span>Дерево сведений о каталоге
</dt> <dd>

Сам каталог, представленный в виде древовидной структуры, в которой вершины являются записями каталога (экземплярами класса) и соединяющими строками отношения типа «родители-потомки» между записями.

</dd> <dt>

<span id="DIT"></span><span id="dit"></span>DIT
</dt> <dd>

См. раздел *дерево сведений о каталоге*.

</dd> <dt>

<span id="Control_Access_Rights"></span><span id="control_access_rights"></span><span id="CONTROL_ACCESS_RIGHTS"></span>Управление правами доступа
</dt> <dd>

Класс, описывающий право доступа, не привязанное к ресурсу, но действие. Например, пользователю может быть предоставлено право создавать относительные значения ИДЕНТИФИКАТОРов.

</dd> <dt>

<span id="Inheritance"></span><span id="inheritance"></span><span id="INHERITANCE"></span>Цепочк
</dt> <dd>

Возможность создавать новые классы объектов на основе существующих классов объектов. Новый объект определяется как *подкласс* родительского объекта. Родительский объект превращается в *Суперкласс* нового объекта. Подкласс наследует атрибуты родителя, включая правила структуры и правила содержимого.

</dd> <dt>

<span id="LDAP"></span><span id="ldap"></span>LDAP
</dt> <dd>

См. раздел *протокол упрощенного доступа к каталогу*.

</dd> <dt>

<span id="Lightweight_Directory_Access_Protocol"></span><span id="lightweight_directory_access_protocol"></span><span id="LIGHTWEIGHT_DIRECTORY_ACCESS_PROTOCOL"></span>Упрощенный протокол доступа к каталогу
</dt> <dd>

Стандартный протокол связи Интернета, используемый для связи с NTDS. Поддерживаются LDAP версии 2 и 3.

</dd> <dt>

<span id="NT_Security_Descriptor"></span><span id="nt_security_descriptor"></span><span id="NT_SECURITY_DESCRIPTOR"></span>Дескриптор безопасности NT
</dt> <dd>

См. раздел *дескриптор безопасности*.

</dd> <dt>

<span id="Object"></span><span id="object"></span><span id="OBJECT"></span>Объектами
</dt> <dd>

Единица хранения данных в службе каталогов. Объекты службы каталогов не следует путать с COM-объектами или другими объектно-ориентированными системными объектами, которые имеют исполняемый компонент и поведение во время выполнения. Объекты службы каталогов состоят только из данных. Объект службы каталогов определяется объектом [**class-schema**](c-classschema.md) и группой объектов [**Attribute-Schema**](c-attributeschema.md) , на которые ссылается объект [**class-schema**](c-classschema.md) .

Объекты схемы [**классов**](c-classschema.md) и [**атрибутов**](c-attributeschema.md) — это объекты службы каталогов, которые имеют определения в схеме, как и любые другие объекты. См. раздел *класс*.

</dd> <dt>

<span id="Object_Identifier"></span><span id="object_identifier"></span><span id="OBJECT_IDENTIFIER"></span>Идентификатор объекта
</dt> <dd>

Уникальные числовые значения, выданные различными центрами сертификации для уникальной идентификации элементов данных, синтаксисов и других частей распределенных приложений. Идентификаторы объектов (OID) находятся в приложениях OSI, каталогах X. 500, SNMP и других приложениях, где важна уникальность. OID основаны на древовидной структуре, в которой общепроизводительный центр сертификации, например ISO, выделяет ветвь дерева подразделу, который, в свою очередь, может выделять подветви.

Идентификаторы объектов в NTDS включают некоторые, выданные ISO для классов и атрибутов X. 500, а также некоторые, выпущенные корпорацией Майкрософт. Нотация OID — это пунктирная строка чисел, например "1.2.840.113556.1.5.4", которая преобразуется, как указано в следующем списке. 

| Значение  | Описание                                                                                  |
|--------|----------------------------------------------------------------------------------------------|
| 1      | ISO — "корневой центр", выдано "1,2" в ANSI, который, в свою очередь,                           |
| 2      | ANSI... выдано "1.2.840" для США, который, в свою очередь,...                                            |
| 840    | США... выдано "1.2.840.113556" в Майкрософт...                                               |
| 113556 | Microsoft... где Microsoft внутренне управляет несколькими ветвями OID в разделе "1.2.840.113556". |
| 1      | Microsoft DS                                                                                 |
| 5      | Классы NTDS                                                                                 |
| 4      | Builtin-Domain                                                                               |



 

</dd> <dt>

<span id="OID"></span><span id="oid"></span>КОДА
</dt> <dd>

См. *идентификатор объекта*.

</dd> <dt>

<span id="Schema"></span><span id="schema"></span><span id="SCHEMA"></span>Схемы
</dt> <dd>

Формальное определение содержимого и структуры службы каталогов. Схема определяет все атрибуты и классы. Для каждого класса определены [**ПОСС-старший**](a-posssuperiors.md), [**должен-содержать**](a-mustcontain.md)и [**может содержать**](a-maycontain.md) атрибуты. [**ПОСС — старший**](a-posssuperiors.md) элемент определяет возможные структуры дерева для службы каталогов, указывая, какие классы могут быть родительскими для любого заданного класса. [**Должен содержать**](a-mustcontain.md) и [**может содержать**](a-maycontain.md) список атрибутов для класса, который должен присутствовать для хранения класса и дополнительных атрибутов, которые могут быть заданы при необходимости.

</dd> <dt>

<span id="Security_Descriptor"></span><span id="security_descriptor"></span><span id="SECURITY_DESCRIPTOR"></span>Дескриптор безопасности
</dt> <dd>

Сведения о владении объектом и разрешениях, которые другие пользователи применяют к этому объекту. Свойство [**дескриптора NT-Security-**](a-ntsecuritydescriptor.md) Schema записи схемы содержит строку, представляющую дескриптор безопасности объекта. Дополнительные сведения о формате сведений в этом поле см. в разделе [Формат строки дескриптора безопасности](/windows/desktop/SecAuthZ/security-descriptor-string-format).

</dd> <dt>

<span id="Structure_Rules"></span><span id="structure_rules"></span><span id="STRUCTURE_RULES"></span>Правила структуры
</dt> <dd>

Определение возможной древовидной структуры или структур. В NTDS правила структуры полностью выражаются атрибутом ПОСС-Превосходно, присутствующим в каждом объекте [**class-schema**](c-classschema.md) . См. *схему*.

</dd> <dt>

<span id="Subclass"></span><span id="subclass"></span><span id="SUBCLASS"></span>Подкласс
</dt> <dd>

Объект [**class-schema**](c-classschema.md) , наследующий от другого объекта [**class-schema**](c-classschema.md) . См. раздел *наследование*.

</dd> <dt>

<span id="Superclass"></span><span id="superclass"></span><span id="SUPERCLASS"></span>Суперкласс
</dt> <dd>

Объект [**class-schema**](c-classschema.md) , из которого наследуются один или несколько других объектов [**-схем классов**](c-classschema.md) . См. раздел *наследование*.

</dd> <dt>

<span id="Tree"></span><span id="tree"></span><span id="TREE"></span>Приклад
</dt> <dd>

См. раздел *дерево сведений о каталоге*.

</dd> <dt>

<span id="X.500"></span><span id="x.500"></span>X. 500
</dt> <dd>

Семейство стандартов, разработанное совместно с ISO и ITU, ранее известное как CCITT, которое определяет именование, представление данных и протоколы связи для службы каталогов.

</dd> </dl>
