# <a name="entry-ttl-attribute"></a>Атрибут "Entry-TTL"

Этот рабочий атрибут поддерживается сервером и, по-видимому, присутствует в каждой динамической записи. Атрибут отсутствует, если запись не содержит класс объекта dynamicObject. Значение этого атрибута — это время в секундах, в течение которого запись будет продолжать существовать, прежде чем выводить из каталога. При отсутствии промежуточных операций обновления значения, возвращаемые путем считывания атрибута в двух последовательных операциях поиска, гарантированно будут невозрастающими. Наименьшее допустимое значение равно 0, что означает, что запись может исчезнуть без предупреждения. Атрибут помечен как "без изменения пользователя", так как он может быть изменен только с помощью операции обновления.



| Ввод | Значение |
|-------------------|---------------------------------------------|
| CN                | Срок жизни записи                                   |
| LDAP-отображаемое имя | ентриттл                                    |
| Размер              | 4 байта. Максимальное значение = 1 год, по умолчанию — 1 день. |
| Привилегия обновления  | \-                                          |
| Частота обновления  | \-                                          |
| Attribute-Id      | 1.3.6.1.4.1.1466.101.119.3                  |
| System-ID — GUID    | d213decc-d81a-4384-AAC2-dcfcfd631cf8        |
| Синтаксис            | [**Перечисление**](s-enumeration.md)        |
