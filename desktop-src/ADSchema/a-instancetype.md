# <a name="instance-type-attribute"></a>Атрибут Instance-Type

Значение битов, определяющее способ создания объекта на определенном сервере. Значение этого атрибута может отличаться в разных репликах, даже если реплики синхронизированы.

Этот атрибут может быть нулевым или сочетанием одного или нескольких из следующих значений.

| Значение      | Описание                                                                                        |
|------------|----------------------------------------------------------------------------------------------------|
| 0x00000001 | Заголовок контекста именования.                                                                        |
| 0x00000002 | Экземпляр этой реплики не создан.                                                                  |
| 0x00000004 | Объект доступен для записи в этом каталоге.                                                          |
| 0x00000008 | Контекст именования, приведенный выше в этом каталоге, удерживается.                                       |
| 0x00000010 | Контекст именования находится в процессе, который создается в первый раз с помощью репликации. |
| 0x00000020 | Контекст именования находится в процессе удаления из локального DSA.                          |



 



| Ввод | Значение |
|-------------------|------------------------------------------------|
| CN                | Instance-Type                                  |
| LDAP-отображаемое имя | instanceType                                   |
| Размер              | 4 байта.                                       |
| Привилегия обновления  | Это значение задается администратором схемы. |
| Частота обновления  | При создании объекта.                    |
| Attribute-Id      | 1.2.840.113556.1.2.1                           |
| System-ID — GUID    | bf96798c-0de6-11d0-a285-00aa003049e2           |
| Синтаксис            | [**Перечисление**](s-enumeration.md)           |
