# <a name="description-attribute-ad-schema"></a>Атрибут Description (схема AD)

Содержит описание, отображаемое для объекта. Это значение ограничено одним значением для обратной совместимости в некоторых случаях, но может быть многозначным в других. См. заметки.



| Ввод | Значение |
|-------------------|---------------------------------------------|
| CN                | Описание                                 |
| LDAP-отображаемое имя | description                                 |
| Размер              | \-                                          |
| Привилегия обновления  | Администратор домена или владелец учетной записи.      |
| Частота обновления  | Каждый раз, когда необходимо изменить описание.   |
| Attribute-Id      | 2.5.4.13                                    |
| System-ID — GUID    | bf967950-0de6-11d0-a285-00aa003049e2        |
| Синтаксис            | [**String(Юникод)**](s-string-unicode.md) |

## <a name="remarks"></a>Remarks

Атрибут Description реализуется как атрибут с несколькими значениями в схеме для случаев, когда это разрешено. Для объекта, который не является управляемым классом SAM, описание является многозначным. Для атрибута, который является управляемым классом SAM, атрибут Description является однозначным. Управляемые классы SAM предназначены для таких целей, как субъекты безопасности, поэтому, если у вас есть, например, контейнер или собственный класс, схема позволит использовать несколько значений. Это поведение атрибута Description предназначено для обеспечения обратной совместимости с более ранними операционными системами, так как атрибут существовал в API-интерфейсах SAM до существования AD.
