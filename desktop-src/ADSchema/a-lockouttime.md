# <a name="lockout-time-attribute"></a>Атрибут Lockout-Time

Дата и время блокировки этой учетной записи (в формате UTC). Это значение хранится в виде большого целого числа, представляющего число 100-наносекундных интервалов с 1 января 1601 (UTC). Нулевое значение означает, что учетная запись в настоящее время не заблокирована.



| Ввод | Значение |
|-------------------|----------------------------------------------------------------------------------|
| CN                | Lockout-Time                                                                     |
| LDAP-отображаемое имя | lockoutTime                                                                      |
| Размер              | 8 байт                                                                          |
| Привилегия обновления  | Администратор домена                                                             |
| Частота обновления  | При создании записи пользователя и при необходимости изменения времени блокировки. |
| Attribute-Id      | 1.2.840.113556.1.4.662                                                           |
| System-ID — GUID    | 28630ebf-41d5-11d1-a9c1-0000f80367c1                                             |
| Синтаксис            | [**Пределах**](s-interval.md)                                                   |

## <a name="remarks"></a>Комментарии

Большая часть этого большого целого числа соответствует элементу **двхигхдатетиме** структуры [**fileTime**](/windows/desktop/api/minwinbase/ns-minwinbase-filetime) , а нижняя часть соответствует элементу **двловдатетиме** структуры **fileTime** .

Значение этого атрибута сбрасывается только при успешном входе в учетную запись. Это означает, что это значение может быть не нулевым, но учетная запись не заблокирована. Чтобы точно определить, заблокирована ли учетная запись, необходимо добавить [**длительность блокировки**](a-lockoutduration.md) в это время и сравнить результат с текущим временем, учитывая местные Часовые пояса и летнее время.
